# Provable Benefit of Curriculum in Transformer Tree-Reasoning Post-Training (2511.07372)

- **Year**: 2025.11
- **Authors**: Dake Bu, Wei Huang, Andi Han, Atsushi Nitanda, Hau-San Wong, Qingfu Zhang, Taiji Suzuki; City University of Hong Kong, RIKEN, ISM, University of Sydney, A*STAR, NTU, University of Tokyo
- **Venue**: arXiv preprint; TeX uses an ICML 2026 preprint template
- **Keywords**: curriculum post-training, chain-of-thought, reinforcement learning, test-time scaling, transformer theory, reasoning tree, coverage coefficient, parity
- **One-liner**: 证明课程式后训练能把树推理采样从指数降到多项式。

## 精华提炼

### 管线/方法

1. 论文目标是解释为什么 post-training 阶段的 curriculum 能提升 LLM reasoning，而不是提出一个新的通用训练 recipe。
2. 总体管线是：抽象 sample-complexity 框架 -> 2S-ART 推理树模型 -> transformer 实现 -> RL finetuning 理论 -> test-time scaling 理论 -> parity 仿真和三类真实 reasoning 任务实验。
3. 第一层抽象把 policy 写成给定 prompt 后的 CoT 轨迹分布，性能用正确 CoT 的生成概率或 pass@K 表示。
4. 难度用 coverage coefficient 表示：目标策略相对 base policy 的密度比越大，正确 CoT 在 base model 下越稀有。
5. curriculum 被形式化为中间最优策略序列 pi_0=pi_ref, pi_1, ..., pi_L=pi_star。
6. 关键判据是 stage-wise sample complexity 之和是否小于直接从 base 学到最终目标的 sample complexity。
7. 如果每个 stage 的复杂度近似是最终复杂度的 L-th root，则 direct cost 是 (C*)^L，curriculum cost 是 L*C*，得到指数级改善。
8. 放宽后只要求每一步复杂度是 (C*)^p_l 且 p_l 远小于 L，总 curriculum cost 仍是 L*(C*)^p_max，保持指数到多项式的差距。
9. 第二层模型是 2S-ART: 2-States Conditioned Autoregressive Reasoning Tree。
10. 2S-ART 的输入是 x=(x_1,...,x_d) 加 EOS，每一步从 legal set I_l 中选一个索引 i_l。
11. 状态更新是 z_l = Phi_l(z_{l-1}, v_l)，其中 v_l 是当前 CoT 上被索引选中的 token 或状态。
12. 一个任务由路径 S=(i_1,...,i_k,d+1) 定义，prefix paths 形成 curriculum subtask family。
13. 该模型覆盖 parity、Countdown、Markov-chain reasoning、associative recall、causal-graph reasoning等作者认为的树式或组合式推理。
14. base model 被抽象成 PART: 在每个合法子节点上近似均匀采样。
15. 在 legal set 大小为 Theta(d) 时，长度为 ell 的正确 prefix CoT 概率是 Theta(d^{-(ell+1)})。
16. 因此，随着 reasoning depth 增长，base model 的正确轨迹覆盖率指数下降。
17. 课程策略的作用是把一次深路径搜索拆成多次浅路径识别。
18. 论文考虑两类 curriculum: depth-increasing 和 hint-decreasing。
19. depth-increasing 是先学短 CoT 或浅层 prefix，再逐步延伸到完整 reasoning horizon。
20. hint-decreasing 是早期给较长正确 prefix，让模型只补短 suffix，之后逐步减少 hint。
21. transformer 实现部分把 token embedding U 和正交 position embedding P 拼接成 E[token]。
22. attention 的 K^T Q 被限制在 positional block 上，训练主要改变一个 W 矩阵。
23. attention 用于选择下一个索引或父子边，FFN_l 被假设已能实现 Phi_l 这类原子操作。
24. 对 parity，FFN_l 明确给出一组 ReLU 权重来实现 XOR 更新。
25. 这相当于把 post-training 的难点集中在“找对推理路径”，而不是学习底层算术或逻辑原子技能。
26. 奖励设定有两种：terminal oracle 只看最终 pre-EOS token 是否正确；family oracle 可检查 prefix subtask 的 pre-EOS token 是否正确。
27. terminal-only reward 会产生 reward hacking: parity 中选错索引仍有 1/2 概率得到正确最终 bit。
28. family oracle 是 curriculum 得到强监督信号的关键假设，能把信用分配从最终答案拆到每个深度。
29. RL finetuning 使用 REINFORCE 风格 objective，训练目标是提高 outcome reward 的期望。
30. 论文证明了 policy gradient 在正交 positional block 上可分解，称为 Orthogonal Block Isolation。
31. OBI 让每个候选索引对应的 block gradient 可以独立比较，避免理论上不同深度和不同子节点的梯度混在一起。
32. no-curriculum 的 terminal oracle 下，第一步必须从极稀有的完整正确轨迹中获得信号。
33. 对目标深度 k*，直接训练达到高准确率的样本复杂度下界为 \tilde{\Omega}(d^{2k*+2})。
34. depth-increasing 和 hint-decreasing 使用 family oracle 时，每个 stage 的样本复杂度为 \tilde{O}(d^2)。
35. 总样本复杂度因此是 \tilde{O}((k*+1)d^2)。
36. 学习率设定也很极端：no-curriculum 使用 \tilde{\Theta}(d^{k*+1}) 量级，curriculum 使用 \tilde{\Theta}(d^2) 量级。
37. 这个理论不是普通 SGD 多轮收敛分析，而是强调 one-step 或 online per-step 大学习率更新可以放大正确 block。
38. test-time scaling 部分研究的不是训练，而是在 outcome oracle 下识别正确 reasoning path。
39. 只用 terminal oracle 时，错误 suffix 的 spurious acceptance 会掩盖真实路径信号。
40. 主文给出的 terminal-only test-time scaling 下界是 \tilde{\Omega}(d^{2k*})。
41. 附录 formal proposition 对其中一个版本写成 \tilde{\Omega}(d^{2k*+1})，这里存在主文和附录阶数表述不完全一致的问题。
42. curriculum test-time scaling 使用 LTAR: Layer-wise Truncated Accept-Reject。
43. LTAR 先用 FXRS 强制某个 visible x-token，然后用 family oracle 在该深度外部截断验证。
44. 每一层用 best-arm identification 选择接受率最高的 child，再 commit 到下一层。
45. 其 reward-oracle query complexity 为 \tilde{O}((k*+1)d^2)，model-sampling complexity 为 \tilde{O}((k*+1)d^3)。
46. synthetic parity 实验使用 d=8, k*=3, S*=[0,1,2], beta=1, test set size 8192。
47. parity finetuning 中 no-curriculum 学习率为 1e8，两个 curriculum 学习率为 1e5。
48. 图中报告 curriculum finetuning 在 n=76 左右收敛，而 no-curriculum 约 n=1,600,076 才收敛。
49. parity test-time scaling 中 LTAR 在 n=1,410 收敛，对照 no-curriculum TTS 在 n=251,410 收敛。
50. 10 次 trial 的补充实验显示 curriculum finetuning 约 n=200 达到 >99.9%，no-curriculum 约 n=1e6；LTAR 约 n=8e3，no-curriculum TTS 约 n=1e6。
51. d=16,k*=2 时，curriculum finetuning n=120，no-curriculum n=1,100,120；LTAR n=5，no-curriculum TTS n=700,005。
52. d=32,k*=2 时，curriculum finetuning n=260，no-curriculum 到 n=32,000,260 仍未收敛；LTAR n=310,000，no-curriculum TTS n=20,310,000。
53. 小学习率实验使用 d=8,k*=3,S*=[0,1,2], eta=0.5, batch size 256, 1000 steps，比较 SGD、Adam、Muon、AdamW。
54. 小学习率下 curriculum 仍快于 no-curriculum；no-curriculum 中 Muon 和 SGD 有慢速进展，Adam 和 AdamW 基本失败。
55. 真实任务实验统一使用 Qwen2.5-1.5B-Instruct，LoRA rank 32, alpha 64, dropout 0.1，只调 q_proj 和 v_proj。
56. Countdown、Math、Blocksworld 均用 GRPO，beta=1e-3，entropy coefficient alpha=0，A100 80GB GPU。
57. Countdown 表格中 macro average: base 18.5, no-curriculum 36.0, depth-increasing 40.5, hint-decreasing 42.6。
58. Countdown 的 K=256 pass@K: base 50.7, no-curriculum 66.3, depth 70.6, hint 75.7。
59. Math macro average: base 16.5, no-curriculum 36.3, depth-increasing 38.2, hint-decreasing 33.8；depth curriculum 总体最好。
60. Math 的 hint-decreasing 在大 K 和最难 level 上更有竞争力，例如 K=256 达 68.3，hardest/OOD level 达 15.4。
61. Blocksworld macro average: base 5.1, no-curriculum 8.3, depth 8.4, hint 9.2，提升明显小于 Countdown 和 Math。
62. Blocksworld 中 hint 在低到中等 K 较强，no-curriculum 在 K=64 最强，depth 在 K=256 追上并略优。
63. 真实任务实验支持 curriculum 有经验收益，但没有证明真实 LLM 完全满足 2S-ART 或 PART 假设。
64. 论文的最强贡献是把“success rate 低导致 RL 信号稀缺”形式化为 coverage 和树路径概率。
65. 它也清楚指出了 outcome-only final reward 的局限：最终答案正确不代表推理路径正确。
66. 对需要 verifier 或分层 feedback 的项目，这篇提供了一个可引用的理论脚手架。

### 关键技术细节

- **核心抽象**: coverage coefficient \|\pi'/\pi\|_\infty 连接 base model 覆盖率、正确 CoT 稀有度和采样成本。
- **2S-ART**: legal set 大小为 Theta(d)，路径长度 k*，prefix subtask family 为 {f_{S^1},...,f_{S^{k*}}}。
- **PART base model**: 每层近似均匀选择 legal child，正确 prefix 概率 Theta(d^{-(ell+1)})。
- **Transformer realization**: token embedding U 与 orthogonal positional embedding P 拼接；K^TQ 只训练 positional block W；FFN_l 固定实现 Phi_l。
- **Outcome oracle**: terminal oracle 只验证最终 pre-EOS token；family oracle 可按 subtask depth 验证 prefix 输出。
- **RL 算法**: REINFORCE objective；no-curriculum 一次更新；curriculum 每个 stage 一次 online update。
- **理论复杂度**: direct finetuning \tilde{\Omega}(d^{2k*+2})；curriculum finetuning \tilde{O}((k*+1)d^2)。
- **TTS 复杂度**: terminal-only 主文 \tilde{\Omega}(d^{2k*})；curriculum LTAR 数据复杂度 \tilde{O}((k*+1)d^2)，采样复杂度 \tilde{O}((k*+1)d^3)。
- **Parity synthetic**: d=8,k*=3,S*=[0,1,2], beta=1, n_test=8192；学习率 1e8 vs 1e5。
- **Realistic tasks**: Qwen2.5-1.5B-Instruct + LoRA；Countdown 327,680 train/1,024 test；Math 5,194 train/5,000 test；Blocksworld 5,000 train/482 test。

### 核心结果

| Setting | No curriculum | Curriculum | 结论 |
|---------|---------------|------------|------|
| RL finetuning theory | \tilde{\Omega}(d^{2k*+2}) | \tilde{O}((k*+1)d^2) | 从指数深度瓶颈到多项式 |
| Test-time scaling theory | \tilde{\Omega}(d^{2k*}) in main text | T_data \tilde{O}((k*+1)d^2), T_comp \tilde{O}((k*+1)d^3) | 分层 oracle 避免 suffix reward hacking |
| Parity d=8,k*=3 finetuning | n≈1,600,076 | n≈76 | curriculum 更快锁定正确 attention path |
| Parity d=8,k*=3 TTS | n≈251,410 | n≈1,410 | LTAR 显著降低 oracle 查询 |
| Countdown avg pass@K | 36.0 | 40.5 depth / 42.6 hint | hint-decreasing 最好 |
| Math avg pass@K | 36.3 | 38.2 depth / 33.8 hint | depth-increasing 总体最好 |
| Blocksworld avg pass@K | 8.3 | 8.4 depth / 9.2 hint | 提升小且依赖 K |

- surprising 点：论文把 curriculum 的好处归因到“正确 CoT 轨迹在 base model 下太稀有”，不是传统 curriculum learning 的 smooth optimization 叙事。
- surprising 点：在 parity 中，最终答案 reward 有 1/2 spurious success，因此 final-answer RL 不是简单低效，而是会系统性强化错路径。
- surprising 点：真实任务中 Blocksworld 的 curriculum 收益很小，说明中间任务设计质量比“有 curriculum”本身更关键。

### 明确局限

- 理论依赖 2S-ART 和 PART 假设，真实 LLM 是否近似均匀覆盖 legal child 未被直接验证。
- 关键 transformer 分析假设 positional embeddings 正交，K^TQ 只在 positional block 训练，FFN 原子技能固定且正确。
- family oracle 比常见 final-answer reward 强很多，需要 prefix label、hint、subtask verifier 或可截断 evaluator。
- 大学习率 one-step/online-step 证明与常规大规模 RLHF/GRPO 训练流程有差距。
- parity 的 reward hacking 结构很清楚，但 Countdown、Math、Blocksworld 的 spurious CoT 机制没有同等程度地被审计。
- 主文和附录对 terminal-only TTS 下界阶数有一处 d^{2k*} vs d^{2k*+1} 的不一致，需要引用时谨慎。
- 真实实验只用 Qwen2.5-1.5B-Instruct 和 LoRA，不能外推到更大模型或 frontier models。
- 代码链接在摘要中给出，但本次没有在线核验仓库内容或独立复现实验。
- 真实任务没有独立第三方复现，也没有跨更多 benchmark 的消融。
- 对 0616-fex 如果关心真实系统可靠性，这篇更像理论依据和设计启发，不是充分 empirical proof。

## Read by: 0616-fex (2026-06-16)

从 `0616-fex` 角度，这篇最重要的价值是把 curriculum post-training 的收益拆到 claim 级别：不是泛泛地说 easy-to-hard 有用，而是要求中间任务相对上一阶段的 coverage 不要太差。若我们的课题涉及 verifier、诊断、feature/exploration 或 reasoning data allocation，这篇提供了一个强基线：最终答案 reward 会把真实路径信号压到指数小，分层反馈可以把每层识别变成 O(d^2) 级别。撞车风险集中在“depth-increasing/hint-decreasing 能带来指数到多项式 sample complexity 改善”这个 claim；可差异化的地方是降低 oracle 强度、做真实模型 coverage 测量、证明非树式推理或多解任务、以及给出独立复现和更大模型实验。
