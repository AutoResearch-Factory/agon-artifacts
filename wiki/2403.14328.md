# Distilling Reinforcement Learning Policies for Interpretable Robot Locomotion: Gradient Boosting Machines and Symbolic Regression (2403.14328)

- **Year**: 2024.03
- **Authors**: Fernando Acero, Zhibin Li (University College London, Dept. of Computer Science)
- **Venue**: IEEE conference format (ieeeconf.cls), submitted to an IEEE venue (likely ICRA/IROS)
- **Keywords**: Reinforcement Learning, Policy Distillation, Gradient Boosting Machines, Explainable Boosting Machines, Symbolic Regression, Robot Locomotion, Interpretability, DAgger, Quadruped
- **One-liner**: 将神经网络 RL 四足运动策略蒸馏为 GBM/EBM/符号回归等可解释模型，用改进的 DAgger 算法仅需 10 分钟仿真数据即可蒸馏出性能匹配甚至超越专家策略的玻璃箱策略。

## 精华提炼

### 管线/方法

整个框架分两阶段：**RL 训练 → 策略蒸馏**。

**第一阶段 — 训练神经专家策略**：
1. 环境：IsaacGym 仿真，Unitree A1 四足机器人
2. 算法：PPO (Proximal Policy Optimization)，使用 clipped surrogate objective
3. 状态空间：关节位置/速度、基座姿态/速度、足端高度/接触状态、Reward Machine (RM) 状态和 RM 迭代计数、上一步动作（prev action）、command velocity
4. 动作空间：12 个关节位置目标（4 腿 × 3 关节：hip, thigh, calf），由高频 PD 控制器执行（P=20, D=0.5, 50Hz 控制频率）
5. Reward 定义：使用 reward machines（状态机形式定义奖励函数），不同步态(walk/trot/pace/bound)共享基本观测但有不同的 RM 状态
6. 训练量：1.5k PPO updates × 1024 并行环境 ≈ 205 小时仿真经验（显著少于之前的工作）
7. 指令范围：前进速度 [-1,1] m/s，偏航角速度 [-1,1] rad/s，随机化

**第二阶段 — 蒸馏为可解释策略**：
1. 蒸馏本质是 imitation learning（行为克隆），因此有 distribution shift 问题
2. 提出改进的 DAgger 算法（Algorithm 1）：episode-dependent alternation of expert and distilled policy actions
3. 交替比 (alternation ratio) 1/n 决定 rollout 中使用专家动作的频率，n 随 episode 递增形成 curriculum
4. 蒸馏参数：max_episodes=30，velocity commands 在 [0, 0.25, 0.5, 0.75] 循环，n_f=4（每 4 个 episode 才更新 n），最低交替比 1/8
5. 每个步态仅需 **10 分钟**仿真数据（t=1000 步）
6. 用收集的数据集 D 做监督学习，同时留 20% 作测试集
7. 三类可解释策略：GBM (sklearn)、EBM (InterpretML)、Symbolic Regression (PySR)
8. 符号回归使用一元算子 [sin, tanh, ·², ·³] 和二元算子 [+, -, ×]，最大算子复杂度 4，总体复杂度 90，20 轮迭代

### 关键技术细节

- **Reward Machines**：将奖励函数定义为状态机，增强 MDP 状态空间。逻辑规则本身也辅助策略可解释性。
- **DAgger 改进动机**：原始的"混合 expert 和 imitation rollout"方式对反馈控制策略效果差，产生不稳定步态。episode-dependent alternation 让分布漂移更平滑。
- **GBM**：ensemble of shallow decision trees，使用 feature importance 和 permutation importance 分析特征重要性。
- **EBM**：Generalized Additive Models 扩展，支持单变量 f_i 和成对交互项 f_{i,j}（本质是 learned lookup tables），天然全局+局部可解释。
- **Symbolic Regression**：用遗传算法进化数学表达式，PySR 实现。20 轮迭代运行耗时大，但结果精度不足（R² 仅 0.66-0.73）。
- **观测特征列表**：DoF Position (12)、DoF Velocity (12)、Previous Action (12)、RM State Encoding (4-6)、RM Iters (1)、Command X/Yaw (2)、Base Orientation (3)、Base Linear/Angular Velocity (6)、Foot Height/Contact (8)、Estimated Base Velocity (3) → 共约 60+ 维观测。
- **NN 专家网络**：文中未明确给出隐藏层数和宽度，但指出"not particularly deep, usually having 2 to 5 hidden layers"。

### 核心结果

| 指标 | GBM | EBM | Symbolic |
|------|-----|-----|----------|
| Walk R² | 0.9705 | 0.9787 | 0.6811 |
| Trot R² | 0.9863 | 0.9906 | 0.7334 |
| Pace R² | 0.9752 | 0.9819 | 0.7331 |
| Bound R² | 0.9537 | 0.9637 | 0.6564 |

**Surprising findings**:
1. GBM Walk 策略单独运行（alternation=0）**比神经专家高出 10% 以上**的平均 episode reward！这是一个反直觉结果——更简单、更可解释的模型不仅没有性能损失，反而更好。
2. EBM/GBM 在所有步态的 standalone 评估中都匹配或超越神经专家：trot +3%, pace +2%, bound +7%。
3. 符号策略在混用（alternation 1/2）时有时反而更好（pace 高出 12%, bound 高出 10%），但单独运行时无法维持超过几个步态周期。
4. 所有步态中 **Previous Action 是压倒性的最重要特征**，尤其是对 hip 和 calf 关节。这暗示当前策略本质上在做"平滑插值"而非复杂推理。
5. EBM 和 GBM 的 top features 排名不同（Table 1），说明**可解释性结果严重依赖模型架构**——这对声称"我解释了策略"的工作是一个警示。
6. EBM 的成对交互项发现了类似"signed XOR"的模式：如 trot 策略中 Front Left Hip 的 Previous Action 和 Hind Right Thigh DoF Position 的交互——只有特定组合才产生正向贡献。

### 明确局限

**作者自述**：
- 仅测试了 proprioceptive 策略，扩展到 exteroceptive（视觉等）尚未验证
- 符号策略性能不足，需要更多遗传算法迭代或不同算子组合
- 缺乏不确定性估计（uncertainty estimates）
- 未扩展到机器人操作（manipulation）任务

**我们读出**：
- **没有真机实验**——所有结果全在 IsaacGym 仿真中。sim-to-real gap 未处理。
- **没有标准 benchmark 比较**——自定任务（4 种步态），无与其他可解释 RL 方法的横向对比（如 VIPER、TripleTree）。
- **仅测试了 Unitree A1**——未验证跨机器人形态的迁移能力。
- **神经专家网络架构未详细说明**——只说"2-5 层"，没有具体参数。
- **蒸馏需要数据中始终有 expert action**（最低交替比 1/8），不是真正的"专家不在场"独立学习。
- **没有做 ablation on DAgger modification**——不知道是哪个设计真正贡献了性能：交替比 curriculum、n_f 的选择、还是单纯的数据量。
- **可解释性分析停留在特征重要性层面**——没有做因果干预（counterfactual intervention）、没有用户研究（人类是否真的能理解并信任这些玻璃箱策略）。

### 跟我们的关系

**对 0608-llm-evolved-code-vs-rl 课题**：

**不撞车**：这篇工作在机器人运动控制领域，与"LLM 生成代码 vs RL 训练策略"在研究问题和应用场景上完全不同。没有直接竞争关系。

**可用工具/方法**：
1. **改进的 DAgger 算法**（Algorithm 1）——episode-dependent alternation curriculum 是一个通用技术，可用于任何 expert→student 蒸馏场景。如果我们的课题涉及用 RL 训练 expert 再蒸馏到 LLM/程序，这个 curriculum 方案可以直接借鉴。
2. **GBM/EBM 作为"玻璃箱"可解释模型**——如果我们需要分析 LLM 生成的代码或 RL 训练的策略中的决策逻辑，EBM 提供了天然的可解释性（全局+局部解释、成对交互项可视化）。
3. **"仅需 10 分钟数据蒸馏"的极限**——展示了轻量级蒸馏的可行性。如果 LLM 生成代码后需要快速验证或蒸馏，这个低成本蒸馏思路有价值。
4. **Feature Importance × Permutation Importance 双重验证**——当一个重要性分析方法给出结果时，用另一种独立方法交叉检验，这个方法学可用。
5. **可解释性依赖于模型架构的警示**——GBM 和 EBM 给出不同的 top features，提醒我们在 LLM-evolved-code 场景下做解释时，不能把"模型说什么是重要的"当作真相。

**差异化**：
- 我们的场景是 code generation / program synthesis，不是 continuous control
- 我们的 action space 是离散的（代码/文本），不是连续关节角度
- 符号回归在我们的场景中可能更适用——发现数学/逻辑表达式来解释 LLM-evolved code 的行为

## Read by: 0608-llm-evolved-code-vs-rl (2026-06-26)

### 撞车评估
不撞车。本工作在机器人运动控制（continuous control）领域，使用 GBM/EBM/Symbolic Regression 蒸馏神经策略。我们的课题"LLM-evolved-code-vs-RL"关注的是用 LLM 生成代码/程序与用 RL 训练策略的效果对比，属于程序合成/代码生成领域。两者在问题定义、动作空间、评价指标上完全不同。

### 可用工具
1. **改进的 DAgger（Algorithm 1）**：episode-dependent alternation curriculum 是通用的 expert-student 蒸馏技术。如果我们有 RL-trained expert 需要蒸馏为更简洁的代码/程序表示，这个算法可以直接使用。
2. **EBM 的全局+局部解释能力**：如果我们需要分析 LLM-evolved code 的行为模式，EBM 提供了天然的逐特征、逐交互项的可视化解释——这比 attention weights 等方法更直接。
3. **"玻璃箱替代黑箱"的范式**：整篇论文的叙事框架（black-box→glass-box, interpretability for safety certification）可以适配到代码生成的安全性和可解释性场景。

### 差异化
- 我们的输出是离散程序/代码，本工作是连续关节角度→符号回归的表达式语法可以作为代码生成的候选中间表示。
- Kohler et al. (2503.08322) 的反引文将蒸馏目标从数学表达式换成了程序，与我们的场景更接近，应该关注。
- 本工作强调"可解释性不削弱性能"的证据（GBM/EBM 甚至超越专家），为 LLM-evolved-code vs RL-code 的比较提供了一个有趣视角：更简单的模型可能在实践中更好。
