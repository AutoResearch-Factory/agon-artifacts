# Developing Machine Learning-Based Watch-to-Warning Severe Weather Guidance from the Warn-on-Forecast System (2603.20250)

- **Year**: 2026.03
- **Authors**: Montgomery Flora (CIWRO/NSSL → The Weather Company), Samuel Varga (OU/CIWRO → Verisk), Corey Potvin (NSSL/OU), Noah Lang (AI2ES/Valparaiso → UW-Madison)
- **Venue**: arXiv preprint (AMS ametsocV6.1 投稿格式, 目标应为 WAF/AIES 类期刊)
- **Keywords**: Warn-on-Forecast System (WoFS), ML post-processing, watch-to-warning (2–6 h), severe weather probability, HGBT, U-Net, NMEP baseline, updraft helicity, NCEI Storm Data
- **One-liner**: 用 HGBT 与 U-Net 后处理 WoFS 集合预报, 生成 2–6 小时"任意强对流灾害"概率格点指导, 均显著优于精细校准的 2–5 km UH NMEP 基线。

## 精华提炼

### 管线/方法
完整 pipeline (Fig. data-workflow):
1. **输入源**: WoFS (NSSL 对流可分辨集合, 36 个多物理方案分析成员, 前 18 个起报; 3-km 格距, 900×900 km 可移动域, 每 15 min 同化雷达+卫星, 输出频率 5 min)。数据为 2019–2023 HWT-SFE (Hazardous Weather Testbed Spring Forecasting Experiment) 108 个个例日, 仅取整点起报 (预报时长 6 h), 共 1121 个预报。
2. **空间粗化**: 3 km → 9 km。风暴内 (intrastorm) 变量用最大值滤波 (分布高度偏斜), 环境 (environment) 变量用均匀滤波 (近高斯), 之后 k-d tree 最近邻重采样。作者称粗化几乎不损失技巧但大幅提速 (not shown)。
3. **时间聚合**: 2–6 h 预报窗内, intrastorm 取时间最大值, environment 取时间平均。
4. **表格 (HGBT) 分支**: 三个空间邻域滤波 (半径 9/27/45 km; intrastorm 最大滤波, environment 均匀滤波) 隐式编码集合空间不确定性 → 对滤波后场计算集合统计量 (intrastorm 取更多阶矩) → 展平成表格, 共 **175 个特征**。
5. **深度学习 (U-Net) 分支**: 域边缘裁剪 100×100 → 96×96 (2ⁿ 倍数适配池化), 同样的集合统计量 → **63 通道**; 额外加入起报时刻 MRMS 组合反射率 (未做消融验证其贡献)。
6. **标签**: NCEI Storm Data (即 NOAA Storm Events) 中 2–6 h 窗内 (±15 min 容差) 的大风 ≥50 kt、冰雹 ≥1 in、龙卷报告, 格点化后以 **36 km 半径膨胀**成二值场 ("距任意强对流报告 36 km 内"), 对齐 SPC outlook 定义。早期实验: 9 km 和 18 km 半径几乎无技巧 (not shown)。
7. **模型**: scikit-learn HistGradientBoostingClassifier vs PyTorch U-Net; 基线为 2–5 km UH 的 NMEP + isotonic 回归校准。
8. **输出**: 每格点 2–6 h "任意强对流"概率, 类似 SPC 分类展望但在 watch-to-warning 时段。

### 关键技术细节
- **输入变量 (21 个 WoFS 场)**: intrastorm — UH (0–2, 2–5 km)、组合反射率、0–2 km 平均垂直涡度、柱最大上升气流、80-m 风速、HAILCAST 最大雹径、Okubo-Weiss 数; environment — 直减率 (0–3 km, 700–500 mb)、0–3 km SRH、U/V 切变 (0–6, 0–1, 3–6 km)、STP、SCP、75-mb MLCAPE、MLCIN。
- **HGBT 超参** (hyperopt 贝叶斯优化, 5 折 CV, 目标 AUPDC, 早停 10 轮内需改进 1%): lr=0.1, max leaf nodes=5, max depth=6, min samples leaf=10, L2=0.01, max bins=63。
- **U-Net 超参** (随机搜索 N=100): 4 层, 初始滤波器 64 (逐层翻倍), kernel 3, dropout 0.3, lr=0.001 + ReduceLROnPlateau, bfloat16 训练, 早停 patience=7, 最优迭代训 130 epochs, **32M 参数**; DoubleConv = (Conv2D+BN+ReLU+Dropout)×2, 简单跳连 (对比 U-Net++/3+)。
- **基线调优**: 5 折 CV 网格搜索 UH 阈值 × 邻域半径, 最优 **125 m²s⁻² / 45 km** (CV-mean BSS=0.194), 但对选择不敏感; isotonic 校准 (Platt 式: 各折验证集拼接学校准函数)。
- **训练/测试划分**: 按日期随机 75 训练 / 33 测试个例; 表格训练样本 8.04M 格点, DL 783 patches (7.21M 格点, 部分个例缺 MRMS); 测试统一 96×96 域, 3.11M 格点。
- **验证协议**: bootstrap N=100 + **block bootstrapping** (起报时间按 4 h 间隔分 4 组, 处理相邻起报的相关性), 置换检验显著性; 指标为 NAUPDC (base-rate 归一化的 performance-diagram 面积)、NCSI (归一化最大 CSI)、BSS (以测试集 base rate 为气候基准)、可靠性图。

### 核心结果
| 结论 | 细节 |
|------|------|
| 两个 ML 模型均胜基线 | NAUPDC 与 BSS 均统计显著更高 (p<0.0001 / p<0.001) |
| 优势集中在高概率阈值 | 40–60% 阈值处 ML 的 POD/SR 明显更好; 20–25% 阈值处三者 NCSI 相当 (空间上圈的区域类似) |
| HGBT 指标最好但概率上限 60% | U-Net 可输出到 100% (且与观测频率一致), 基线上限 50%; 概率动态范围是 U-Net 独有卖点 |
| U-Net 更平滑、略过报 30–80% 区间 | 三者经 isotonic 校准后总体都贴 1:1 线, HGBT 校准最好 |
| ML 是后处理工具, 继承 WoFS 位移误差 | 空间大改基线概率分布的情形是例外而非常态 |
| 作者明言不下结论 "HGBT 是否有意义地强于 U-Net" | 显著性 ≠ 实用差异, 需多数据集+更充分 DL 调优才能比较 |

反直觉/值得记的点: (1) 邻域半径从 36 km 缩到 9/18 km 就几乎无技巧 —— 2–6 h 时段的可预报性只支持 SPC-outlook 尺度的空间精度; (2) 精细校准后的纯 UH-NMEP 基线其实已能圈出大致相同区域, ML 的净增量主要是"敢给高概率"。

### 明确局限
作者自述:
- U-Net 与 HGBT 输入不对等 (MRMS 只给了 U-Net 且无消融; 多尺度邻域只给了 HGBT), 两模型对比本质上不严格;
- 标签用 local storm reports, 有已知报告偏差 (引 Flora et al. 2025), 未检查偏差是否传入预测;
- 无可解释性分析; 集合统计量的选择缺乏系统研究 (哪些统计量真正从集合中提取信息是开放问题);
- 仅暖季 HWT-SFE 个例训练, 不保证全年泛化; 需要预报员反馈评估 U-Net 高置信度输出的实际价值。
我读出的额外局限:
- **数据不公开** ("freely available upon request"), WoFS 输出无公共档案, 划分/个例列表也非社区标准 —— 结果无法被第三方直接复现或与后续工作公平比较;
- 单一"任意灾害"目标 (分灾种预测撇给了 Varga 的硕士论文), 无逐灾种结论;
- 108 天全部来自春季实验域 (大平原为主 + 中大西洋次高), 空间抽样偏差明确存在 (Fig. 1 heatmap)。

### 跟我们的关系
对 0713-scs-benchmark (SCS 环境–雷达多模态开放基准):
- **不撞车**: 这是方法/应用论文, 不是数据集论文; 其数据不公开、无 DOI、无社区划分, 与我们"发布整编严谨的开放数据产品"目标零重叠。
- **是最好的 motivating evidence 之一**: NSSL 自家的准业务 ML 指导, 数据都只能 "upon request" —— 直接引用它论证 "WoFS/CAM-ML 后处理这条活跃研究线没有公共基准可比较" 的空白。
- **可直接搬进 benchmark 的规范**: (1) 标签配方 — Storm Events 三灾种阈值 (50 kt / 1 in / 龙卷) + ±15 min 时间容差 + 36 km 膨胀半径, 与我们的事件包标签层同源同构; 其 "9/18 km 无技巧" 负结果直接指导我们 canonical task 的邻域半径设定; (2) 指标包 — NAUPDC/NCSI (base-rate 归一化) + BSS + block bootstrap + 置换检验, 就是 NeurIPS D&B track 要求的 canonical metrics; (3) 基线族 — 校准 NMEP (UH 125 m²s⁻²/45 km + isotonic) 是任何强对流概率任务都该带的廉价非 ML 基线, HGBT/U-Net 参考实现代码已开源可改造。
- **差异化再确认**: 他们的"环境场"是 WoFS 集合预报场、窗口仅 6 h; 我们是 HRRR 分析场 (f00/f01) 长时窗 (-15d/+5d) + MRMS 观测雷达, 以风暴为中心封装。我们的基准可支持"用公开数据复刻 2–6 h watch-to-warning 任务"的可复现版本, 这正是本文做不到的。

## Read by: 0713-scs-benchmark (2026-07-14)

以数据基准视角审读。结论: **无撞车风险, 高复用价值**。
- 本文是"CAM/WoFS ML 后处理"研究线在 watch-to-warning 时段的最新代表作, 但其数据仅 upon request、划分非标准、WoFS 输出无公共档案 —— 恰是我们 benchmark 要填的"无公共可比基准"空白的实证。建议在 proposal 的 gap 论证中与 Flora 2021、Clark & Loken 2022 并列引用。
- 直接采纳: 36 km 邻域"SPC-outlook 式"标签定义 (含 9/18 km 无技巧的负结果依据)、NAUPDC/NCSI/BSS + block bootstrap 指标包、校准 UH-NMEP 作为 canonical 非 ML 基线、frdd-wofs-ml-w2w 开源代码作基线实现起点。
- 差异化清晰: 我们用公开源 (HRRR f00/f01 + MRMS + Storm Events) 的长时窗风暴中心事件包, 可承载本文任务的"可复现开放版", 且环境场时间纵深 (-15d) 远超其 6 h 预报窗 —— 两者互补而非竞争。
- 待跟进: Schmidt et al. 2024 (3D U-Net 冰雹) 与 Flora et al. 2025 (标签偏差) 应进下一轮精读队列; 后者直接关系我们标签质量/不确定性层的设计。
