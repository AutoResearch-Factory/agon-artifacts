# InvarGC: Invariant Granger Causality for Heterogeneous Interventional Time Series under Latent Confounding (2510.19138)

- **Year**: 2025.10
- **Authors**: Ziyi Zhang, Shaogang Ren, Xiaoning Qian, Nick Duffield (Texas A&M University; University of Tennessee at Chattanooga; Brookhaven National Laboratory)
- **Venue**: arXiv preprint (ICLR 2026 conference format; `\iclrfinalcopy` set, non-anonymous)
- **Keywords**: Granger causality, latent confounding, unknown interventions, heterogeneous environments, invariant causal prediction, mechanism shift / sparse deviation, group-Lasso, identifiability, time series causal discovery
- **One-liner**: 用跨环境不变性 (ICP 思想) 同时分离潜在混杂、识别 edge-level 未知干预、并恢复不变 Granger 因果图, 并给出可辨识性证明。

## 精华提炼

### 管线/方法
输入: 来自 N 个异质环境的多元时序 **X** ∈ ℝ^{N×d×T} (每个环境 X_k ∈ ℝ^{d×T}, d 个观测变量, T 个时步)。没有任何标签说明哪个环境被干预、哪些变量/时步被干预。输出: (1) 跨环境不变的 Granger 因果图 W_{0,X_{t+1}X_t} ∈ ℝ^{d×d}; (2) 每个环境的 edge-level 干预图 W_k ∈ ℝ^{d×d}; (3) 潜在混杂子空间 Z。

四个模块 (对应架构图 Fig 2):
1. **Latent Confounder Inference Module (LCIM)** —— 每个环境 k、每个时步 t 各自维护一个可学习潜变量向量 Z_{k,t} ∈ ℝ^p, 与观测拼接得 P_{k,t} = LCIM_k(X_{k,t}, Z_{k,t}) ∈ ℝ^{d+p}。LCIM 逐环境独立 (吸收环境特异的混杂)。
2. **Intervention Identification Network** —— 逐环境线性投影 H_{k,t} = W_k X_{k,t} ∈ ℝ^h, 捕捉"环境特异偏差"(即干预)。
3. **Invariant Granger Causal Network** —— 所有环境**共享**一个 W_0: C_{k,t} = W_0 P_{k,t} ∈ ℝ^h, 捕捉跨环境不变的因果表示。
4. **Next-timestep Embedding & Prediction** —— E_{k,t+1} = Embed(H_{k,t}, C_{k,t}) (捕捉非线性交互), 再线性投影 Ŷ_{k,t+1} = W_out E_{k,t+1} ∈ ℝ^d 预测下一时步。

核心分解是 θ_{k,j}^i = W_{0,j}^i + W_{k,j}^i: 边 j→i 在环境 k 的总机制 = 不变部分 + 环境特异偏差。W_{k,j}^i≠0 ⟺ 该边在该环境被干预。这就是一张学出来的"哪条边、在哪个环境发生了机制漂移"的指示图。

### 关键技术细节
- **损失函数** (Eq. 5 线性版 / Eq. 6 非线性版), 对 W 与 Z 联合优化, 四项:
  1. 预测平方误差 Σ‖X_{k,t+1}^i − (W_0^i P_{k,t} + W_k^i X_{k,t})‖²;
  2. 潜混杂正则 λ_z · Σ_k Σ_l √((1/T)Σ_t Z²_{k,l,t}) —— 对每个潜因子沿时间做 group-Lasso, 把多余潜维压到 0;
  3. (1−α) · Σ_{i,j} ‖(W_{0,j}^i, W_{1,j}^i,…,W_{N,j}^i)‖₂ —— 跨环境的**组稀疏**, 强制不变支撑 (across-group);
  4. α · Σ_{k,i,j} ‖W_{k,j}^i‖₂ —— 对环境特异偏差的**组稀疏**, 强制干预稀疏 (within-group)。
  α∈(0,1) 平衡"组间 vs 组内"稀疏; λ_z>0 控制潜混杂维度。
- **非线性推广**: 假设存在 f_i(·) (不变机制)、g_{k,i}(·) (环境 k 对变量 i 的 edge-level 干预)、h_{k,i}(·) (聚合函数), 用 E[X_{k,t+1}^i | PA, Z_t] = h_{k,i}(f_i(P_{k,t}), g_{k,i}(X_{k,t})) 建模; 用 NN 实现。
- **潜混杂生成模型** (Eq. 2): 把状态扩成 (X_t, Z_t), 一阶 VAR, 且假设 W_{Z_{t+1}X_t}=0 (观测不影响未来潜变量, 潜变量外生)、W_{Z_{t+1}Z_t} 对角。
- **干预阈值**: 检测 W_k 时按"模型权重的数量级"设阈值 (作者原话, 未给具体数值)。
- **超参 λ_z、α、p、h 均未在正文给出具体取值**; 阈值规则模糊。

### 核心结果
- **Baselines**: GC, NGC, eSRU, CUTS/CUTS+, DyNOTEARS, GVAR, JRNGC, LPCMCI。指标: AUROC + AUPRC (稀疏结构下 AUPRC 更可靠)。
- **合成数据** (d=5 观测 + p=1 潜混杂, edge prob 0.3, 3 个环境: 2 观测 + 1 edge-干预, Leaky ReLU 负斜率 0.01): InvarGC 在 Linear 与 Non-Linear 上都拿到**满分 AUROC=1.0000 / AUPRC=1.0000** (次优 eSRU 分别 0.9365/0.8333 与 0.9841/0.9617)。⚠ 满分值得警惕 —— 合成数据恰好完全满足其不变性假设, 属"设计即保证"。
- **TEP (Tennessee Eastman Process)** (33 变量, 960 观测, 21 个异常各作一个环境): Conf-TEP(w/o I) InvarGC 0.7855/0.3567 (最佳); Conf-TEP 0.8121/0.4380 (最佳)。真实稀疏设定下所有方法 AUPRC 都很低。
- **Causal-Rivers** (Stein 2025, 德国河流径流): 4 子集 Random / Confounder / Flood / NoRain+Flood, InvarGC 稳定 top-2; 在 **Confounder 子集显著领先** (0.9048/0.9341), 印证其抗混杂设计。
- **消融**: (i) 去掉 LCIM → 出现虚假相关; (ii) L≥|Z| 且 λ_z 合适 → 仍能恢复; (iii) **|Z|>L → 只能部分吸收混杂, 残留虚假相关、因果图有偏** (与可辨识性极限一致); (iv) 干预网络倾向识别**强干预、漏掉弱干预** (作者诚实承认, 依赖阈值)。

### 明确局限
- **理论只在线性 + 单时滞 (t→t+1) 下成立**; 非线性靠 NN, 无理论保证 (作者自认)。
- **无代码 / 数据发布声明, 无误差棒 / 置信区间 / 多 seed 方差**; 合成满分未做噪声敏感性压力测试。
- 真实数据集靠"mask 一个无父但有多子的变量"人工构造混杂, 构造过程只粗略描述。
- 干预检测漏弱干预, 阈值靠经验。
- 强假设: A2 (不变支撑 + 干预只作用于 X→X 边)、A3 (潜混杂外生且 Z→X 机制不可干预、跨环境不变) —— 真实大气/快变系统很可能违背。
- 只 2 个真实 benchmark, 无真机/业务基线对比。

### 跟我们的关系 (0710-causal-scs = 因果方法用于强对流 Severe Convective Storm 预警)
**不撞车, 但是一个高价值的"反向框架"对照 + "strongest objection"的具体化身。**
- **问题不同**: InvarGC 是因果**发现方法** (恢复不变 Granger 图 + 识别干预), 不做强对流预警、不研究估计不稳定性作为信号。输出是图, 不是预警指标。
- **框架相反, 现象相同**: InvarGC 认定"跨环境机制漂移是稀疏、模块化、edge-level、可干净辨识的", 并把它**正则化掉/分离出去**以拿到不变结构。我们的 idea 反过来: "因果估计器在漂移下的**不稳定/脆弱本身**才是前兆信号"。InvarGC 是"regularize the shift away"学派的代表作, 正好可与 review 中已定位的 Causal-Audit (2604.02488, "假设违背→风险分数") 并列, 作为 idea 差异化的锚点 (方向相反、机制邻近, 但 InvarGC 在**方法侧**、Causal-Audit 在**审计侧**)。
- **它是我们"strongest objection"的活体证据**: idea 的最强反驳是"观测到的不稳定性可能只是估计噪声/misspecification, 不是真实状态信号"。InvarGC 恰好证明: 一个 shift-aware 方法能把干预+混杂干净吸收进 W_k 与 Z。skeptic 会说"你在 PCMCI+/LKIF 上看到的不稳定, 用 InvarGC 这类方法就消失了 → 是 misspecification 不是 regime 信号"。idea 必须论证: 要么不稳定信号在用 shift-aware 方法后仍存活, 要么强对流启动违背 InvarGC 的 A2/A3 (支撑本身变、大尺度强迫混杂非平稳且可被干预), 使"干净分离"根本不成立 —— 这正对上 review 点名的"快变中尺度 vs 慢变 CSD 假设错配"。
- **可当方法/工具迁移**: InvarGC 处理的正是 topic 真实数据会遇到的"潜混杂 + 未知干预 + 异质环境"regime。其 θ=W_0+W_k 分解 (不变 + 逐环境偏差) 给出一个便宜的脆弱性度量思路: 跨滑动窗口拟合一个共享不变模型, 用逐窗偏差范数 ‖W_k‖ 作为**突变型/离散型不稳定指标**候选, 与 PCMCI+/LKIF 的跨方法分歧并列比较。其 edge-level 干预图本身就是一张"哪条边在哪个 regime 漂移"的现成离散指标。
- **该不该引**: 应该引, 放 Related Work 的"shift-aware / 分离漂移"一侧, 与 review 已列的 4 篇 (CNM 2412.16235 / NPG / Causal-Audit 2604.02488 / ST-CND 2606.17553) 并列。它不杀新颖性 (不做强对流 EWS), 但必须讨论以证明"利用不稳定 vs 消除不稳定"的差异, 并正面回应"漂移可被干净辨识"这一威胁。
- **差异化清楚**: InvarGC = 有监督不变性方法, 把干预/混杂建模掉以求稳定结构; 我们 = 把估计器的失败/不稳定本身当前兆。方向相反, 原料 (跨 regime 机制漂移) 相同。
