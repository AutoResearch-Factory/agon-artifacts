# Factored Policy Gradients: Leveraging Structure for Efficient Learning in MOMDPs (2102.10362)

- **Year**: 2021.02
- **Authors**: Thomas Spooner, Nelson Vadori, Sumitra Ganesh (J.P. Morgan AI Research)
- **Venue**: NeurIPS 2021
- **Keywords**: policy gradient, variance reduction, MOMDP, influence network, factor baseline, control variate, minimum biclique covering, factored action space, traffic control, bandit
- **One-liner**: 提出 Factored Policy Gradients (FPGs)，通过 influence network 建模 action-target 条件独立结构，用 factor baseline 削减策略梯度方差，在 1000 维 bandit 和交通信号控制中显著优于传统 VPGs。

## 精华提炼

### 管线/方法
完整的 FPG 管线：

1. **输入**：一个 MOMDP，其 reward 可分解为 $m$ 个目标的加权和 $\psi(s, \bm{a}) = \sum_{j=1}^m \lambda_j \psi_j(s, \sigma_j(\bm{a}))$，每个 $\psi_j$ 仅依赖 action 的某个 partition $\sigma_j(\bm{a})$。

2. **构建 Influence Network**：根据领域知识构建二分图 $\mathcal{G} = (I_\mathcal{A}, I_\Psi, E)$，其中 $I_\mathcal{A}$ 为 action 维度节点（$|\mathcal{A}|$ 个），$I_\Psi$ 为目标节点（$m$ 个），边 $(i,j) \in E$ 表示 $a_i$ 因果性地影响 $\psi_j$。等价地可表示为 influence matrix $\bm{K}_\mathcal{G}$（$|\mathcal{A}| \times m$ 布尔矩阵）。

3. **Policy Factorisation**：将 action 维度按 partition maps $\Sigma = \{\sigma_i^\pi\}_{i=1}^n$ 分组，得到 factored policy $\pi_\theta(\bm{a}|s) = \prod_{i=1}^n \pi_{i,\theta}(\sigma_i^\pi(\bm{a})|s)$，其中各因子相互独立。分组后形成 factored influence network $\mathcal{G}_\Sigma$。

4. **计算 Factor Baselines**：对每个 policy factor $i$，baseline 为 $b_i^F(s, \bar\sigma_i^\pi(\bm{a})) = [(\bm{1} - \bm{K}_\Sigma)\bm{\lambda}\circ\bm{\psi}(s, \bm{a})]_i$，即剔除与当前 factor 无关的目标项。

5. **FPG 估计器**：$\bm{g}^F(s, \bm{a}) = \bm{S}(s, \bm{a})\bm{K}_\Sigma\,\bm{\lambda}\circ\bm{\psi}(s, \bm{a})$，其中 $\bm{S}$ 为 score 矩阵（$|\theta| \times |\Sigma|$）。此估计器无偏且方差低于 VPG。

6. **（可选）Minimum Factorisation**：求 $\mathcal{G}$ 的最小 biclique vertex cover（在 $I_\mathcal{A}$ 上不相交），得到最优 policy factorisation $\Sigma_\mathcal{G}^\star$，最大化方差缩减。

### 关键技术细节

- **Influence Network 的四种原型结构**：Fork（一个 action 影响多个 target）、Collider（多个 action 影响一个 target）、Fork/Collider 混合、Complete（全连接）。这些原型通过 min factorisation 归约为更紧凑的形式。
- **VPG 估计器定义**：$\bm{g}^V(s, \bm{a}) = [\psi(s, \bm{a}) - b(s)] \bm{z}$，其中 $\bm{z} = \nabla_\theta \ln \pi_\theta(\bm{a}|s)$ 为 score 向量。
- **FPG 的特殊情形**：
  - VPG = FPG with $\bm{K}_\Sigma = \bm{1}$（全一矩阵）
  - COMA (Foerster 2018) = FPG with square, diagonal $\bm{K}_\Sigma$（每个 agent 对应一个 target）
  - DRPG (Castellini 2020) = 同上结构
- **方差分解公式**（Prop 2）：$\Delta\mathbb{V}_i = \alpha_i \mathbb{E}[(b_i^F)^2] + 2\beta_i \mathbb{E}[b_i^F]$。第一项为二次项（总是非负，是"免费午餐"），第二项为线性耦合项（可为负，但可通过平移 target 使总体非负，见 Corollary 1）。
- **Search Bandit 实验配置**：
  - $n=1000$ 维连续 bandit，cost 函数 $\text{Cost}(\bm{a}) = \|\bm{a} - \bm{c}\|_1 + \lambda \zeta(\bm{a})$
  - 策略：各向同性高斯，固定协方差 $\text{diag}(\bm{1})$
  - Influence network：每个 action 维对应一个 target $\psi_i(\bm{a}) = -\Delta_i(a_i) - \lambda\zeta(a_i)$，构成 $n$ 个 Fork 结构
  - FPGs 的学习率 0.5，VPGs w/o baseline 仅在学习率 0.001 时稳定
  - 墙钟时间：VPGs w/o baseline ~10534 it/s，FPGs ~9950 it/s，action-dependent baseline VPGs 仅 ~80 it/s（慢两个数量级）
- **Traffic 实验配置**：
  - 使用 Flow 框架 + RLLib 的 PPO + GAE 实现
  - 3×3 网格：9 个 traffic light，9 维 action space
  - 2×6 网格：12 个 traffic light
  - Spatial baseline：仅保留与当前红绿灯直接相连的街道/边产生的 reward 项
  - PPO 参数：discount 0.999，GAE λ=0.97，KL target 0.02，learning rate 5×10⁻⁴
  - 策略网络：3 层全连接，每层 32 单元
  - 50 CPU 并行 rollout，horizon 400
- **共享策略（Shared Policy）**：traffic 实验中用单一 univariate policy 处理所有红绿灯，通过 local info + identity variable 区分，体现了参数共享思想（类似 Foerster 2016 / Gupta 2017）。
- **Minimum Factorisation 的计算**：虽与 NP-complete 的 vertex cover 问题相关，但实践中可在多项式时间内找到（参考 Fleischner 2009 的 biclique covering 算法）。

### 核心结果

| 实验 | 关键发现 |
|------|---------|
| Search Bandit (1000-dim) | FPGs 比 VPGs 收敛更快、更稳定；FPGs 对学习率不敏感，VPGs 对学习率极度敏感 |
| Aliasing 效应 | 当惩罚项 $\zeta$ 覆盖 $n-1$ 维时，VPGs 的 $a_n$ 维度被 aliasing 淹没；FPGs 通过直接归因梯度避免了此问题 |
| Traffic 3×3 | Spatial baseline (FPGs) 不仅学习效率更高，**渐进性能也更优**——可能因为低方差使 SGD 能以更高精度逼近最优解 |
| Traffic 2×6 (bias-variance trade-off) | 随着 spatial baseline 覆盖范围 $n$ 增大（bias↓, variance↑），性能**单调递减**——证实引入少量 bias 换取大幅 variance reduction 是值得的 |
| Bandit 方差分解 | 随 policy factor 和 target 数量增长，二次项迅速主导，即使 $\|\Sigma\|$ 很小也如此；$\bm{K}_\Sigma$ 越稀疏效果越好 |
| Action-dependent baseline vs FPGs | Action-dependent baseline (Wu 2018) 墙钟时间慢 100 倍（80 vs 9950 it/s），而 FPGs 几乎无计算开销 |

**Surprising 发现**：
- Traffic 实验中 FPGs 的渐进性能优于 VPGs。传统观点认为 control variate 只影响方差（收敛速度），不影响最终解。作者假设：固定学习率下，SGD 无法区分最优解周围半径为 $\propto \sqrt{\text{variance}}$ 的球内各点；方差减小 → 精度提高 → 更可能达到真正最优解。
- 在 coupling 较强的 regime（如 apples & oranges 中 $\kappa=0.3, \lambda=1$），FPGs 的优势减弱——说明当全局耦合项主导时，结构分解的收益递减。

### 明确局限

1. **Influence network 需预先指定**：真实问题中不一定事先知道精确的 influence 结构。错误地移除边会引入 bias，限制可达到的解空间（文中提到了 curriculum learning 作为缓解方案）。
2. **无真机实验**：所有实验均为仿真环境（bandit + 交通模拟器），未在物理系统上验证。
3. **仅在连续 action space 上测试**：bandit 和 traffic 均为连续控制。离散 action space（如 Atari）中的 FPGs 效果未验证——文中明确指出 dense/complete influence network 不会带来收益。
4. **无标准 benchmark 比较**：未与常见的 RL benchmark（如 MuJoCo、Atari）比较，实验仅限自定义 bandit 和 Flow 交通环境。
5. **Optimal baseline 未解决**：附录详细说明了 optimal baseline 问题涉及多项式方程组（degree=2, 解空间可达 $2^{|\Sigma|}$），在实践中不可解。Factor baseline 是一个 ansatz，而非最优解。
6. **Minimum factorisation 复杂度分析不完整**：文中声称实践上可在多项式时间内找到 MF，但未给出具体算法或伪代码。引用 Fleischner 2009 但未详细说明适用性。
7. **网络规模有限**：traffic 实验最大仅 2×6=12 个路口，未测试真正大规模（如 100+ 路口）场景。
8. **代码未公开**：文中提及代码使用 Python 和 RLLib，但未见开源 repo 链接（NeurIPS checklist 中 compute resources 部分标记为 No）。

## Read by: 0616-fex (2026-06-22)

### 撞车评估
FPGs 关注的是 **策略梯度方差缩减**（通过结构分解），与 0616-fex 的 exploration 方向无直接竞争关系。两者的核心贡献维度不同：FPGs 解决 "how to learn efficiently given known structure"，而 exploration 解决 "how to discover unknown structure"。不撞车。

### 可迁移工具
1. **Influence Network 表示**：如果你的 factored MDP 需要建模 action 维度和 reward 目标之间的依赖关系，本文的二分图 + $\bm{K}_\Sigma$ 矩阵表示是轻量且原则性的选择。
2. **Minimum Factorisation**：当你需要确定 "哪种 action 分组方式最优" 时，MF 给出了基于图论的答案（minimum biclique cover）。这对设计 factored exploration 策略的参数化方式有指导意义。
3. **Factor Baseline 作为零开销插件**：无论你最终用什么 exploration bonus 或 intrinsic reward，都可以叠加 factor baseline 来削减因多目标耦合引入的梯度噪声，计算开销几乎为零。
4. **Spatial Approximation 范本**：Traffic 实验中的 "仅保留局部连通 reward" 策略是可推广的模式——如果你的 problem domain 有图/空间结构，可以用类似方式构造 local factored reward。

### 差异化方向
- 如果你的工作 focus 在 **exploration**（如 intrinsic reward、uncertainty estimation），那 FPGs 是 orthogonal 的 variance reduction 工具，可以组合使用。
- 如果你的工作涉及 **influence network 的自动发现/学习**（本文的 Future Work (a)），那将是 FPGs 的自然延伸——当前 FPGs 需要人工指定 influence network。
- 如果你的 setting 中 **action space 分解不明确**（如 latent action space），FPGs 的 factorisation 假设可能不成立，需要更灵活的分解方法。
