# PINNACLE: PINN Adaptive ColLocation and Experimental points selection (2404.07662)

- **Year**: 2024.04 (ICLR 2024)
- **Authors**: Gregory Kang Ruey Lau, Apivich Hemachandra, See-Kiong Ng, Bryan Kian Hsiang Low (NUS & CNRS@CREATE)
- **Venue**: ICLR 2024
- **Keywords**: PINN, adaptive collocation, NTK, active learning, point selection, augmented space, convergence degree, Nystrom approximation
- **One-liner**: First algorithm that jointly optimizes all PINN training point types (PDE/IC/BC collocation + experimental) using NTK eigenspectrum analysis, with theory linking convergence degree to generalization bounds.

## 精华提炼

### 管线/方法
1. **Augmented Space Construction**：将输入空间 × {实验点标签, PDE配置点标签, IC/BC配置点标签} 构造成增广空间 Z，定义通用预测算子 F 统一处理所有点类型的残差。
2. **eNTK 特征谱分析**：在增广空间上计算 PINN 的 empirical NTK，通过 Mercer 分解得到特征函数和特征值。分析发现：(a) 残差沿主导特征函数方向衰减更快；(b) eNTK 特征函数在训练中逐渐对齐到真解。
3. **Convergence Degree 准则**：定义 α(Z) = ||ΔR||²_RKHS（残差变化在 RKHS 中的范数），作为衡量候选训练集对收敛贡献的度量。
4. **Nystrom 近似**：用参考集 Z_ref (size p << |pool|) 近似特征函数/特征值，将 α 近似为可计算形式：α ≈ Σ λ̃_i⁻¹ (ṽ_i^T K(Z_ref, Z) R(Z))²。
5. **双变体选点**：PINNACLE-S（按 α 比例概率采样）和 PINNACLE-K（在嵌入空间做 K-Means++ 初始化选择）。
6. **交替训练**：选点阶段 → 训练阶段（GD + eNTK 变化过大时触发重新选点）→ 循环。保留部分历史训练点避免灾难性遗忘。

### 关键技术细节
- **NN 架构**：全连接网络 + tanh 激活，部分实验使用 LAAF（局部自适应激活函数）。各问题架构不同：1D Advection (8×128)、Burgers (4×128)、Navier-Stokes (6×64+LAAF) 等
- **优化器**：Adam，学习率 10⁻⁴（Advection/Burgers）或 10⁻³（其他）
- **训练框架**：基于 DeepXDE 改造的 Jax 实现
- **训练点预算**：Advection 1000 CL 点/200k steps，Burgers 300 CL 点/200k steps，Navier-Stokes 1000 CL + 30 exp pts/1k steps，Eikonal 500 CL + 5 exp pts/1k steps
- **参考集大小 p**：p << |pool|（文中未给具体数字，依赖 NTK 特征值快速衰减）
- **Pseudo-output**：选实验点时用继续训练 100 步后的网络输出作为伪标签
- **训练点保留机制**：每轮选点时随机保留部分旧点，防止灾难性遗忘
- **eNTK 变化触发**：基于 Frobenius 范数相对变化 > δ 触发重新选点（实践中常在 T 步内不会触发）

### 核心结果
- **Forward Problems**：
  - 1D Advection：PINNACLE 两个变体均显著优于 Random/Hammersley/RAD。特别在 Advection 中，baselines 只能学到靠近 IC 的前几个条纹，PINNACLE 学到全部条纹。
  - 1D Burgers：同样显著优于所有 baselines
  - Hard-constrained 1D Diffusion：即使只有一种 CL 点类型，PINNACLE 仍优于 baselines
  - 1D KdV、2D Shallow Water 也有优势
- **Inverse Problems**：
  - 2D Navier-Stokes（估计 λ₁ 和 λ₂）：baselines 无法正确恢复参数，PINNACLE 能准确收敛到真实值
  - 3D Eikonal（学习速度场 v(x,y,z)）：PINNACLE 显著优于 baselines
  - 时间效率：PINNACLE-K 用少 10× 的训练点达到同等精度，且总时间更短
- **Transfer Learning**：
  - 微调预训练 PINN 适应扰动 IC：PINNACLE 利用已有解结构，5× 更少点达成更好重建
- **Surprising findings**：
  - PINNACLE 自动选点行为可解释：早期选更多 IC 点→后期逐渐过渡到 PDE 点，类似人工设计的因果学习策略
  - K-Means++ 变体（PINNACLE-K）通常优于采样变体（PINNACLE-S），但差距不大——说明多样性对于 PINN 训练可能不是必需的
  - 即使只有一种 CL 类型（无跨类型信息），算法仍然有效
- **消融实验**：
  - 移除训练点保留机制 → 性能一致下降（灾难性遗忘）
  - 移除 eNTK 触发重新选点 → Advection 中 PINNACLE-K 性能下降
  - 不使用动态 CL 预算分配 → 性能明显下降（其他方法的 0.5/0.8/0.95 固定比例效果无差别）

### 明确局限
- 作者声明：
  - PINNACLE-S 对 pool size 敏感（重复点多的区域采样概率更高）
  - eNTK 变化触发在实践中不常被激活（训练后期 eNTK 变化小）
  - 理论假设线性化 NTK 和无限宽，实践中有偏差
  - 实验仅在已建立 benchmark 上验证，未测试真实工程场景
- 审读者发现的局限：
  - **无真机实验**：所有训练点来自仿真 PDE，实验点也是仿真数据
  - **计算开销**：每轮选点需计算 eNTK（需 Jacobian），对于大 NN 可达 O(|pool| × p) 复杂度
  - **未与最新 baselines 对比**：如因果学习（Wang 2022）、loss reweighting 等方法在文中只用做解释参考，未做直接对比
  - **未见混沌/湍流**：PDE 均为经典 benchmark，未涉及真正高难度场景
  - **NTK 特征函数质量依赖参考集**：Nystrom 近似的误差界依赖参考集大小和选择方式，实践中未系统消融
  - **跟我们 0616-fex 课题的关系**：本文是改进 PINN 训练效率的工具性工作，不是 failure mode 研究。但其 NTK 特征谱分析方法可用于 failure diagnosis（检查哪些特征函数未被激活/对齐不好）

### 跟我们的关系
- **撞车评估**：不撞车。PINNACLE 是 PINN 训练效率优化工具，0616-fex 是 failure mode 研究。两者定位正交。
- **可用工具/方法**：
  1. **增广空间 NTK 分析框架**：可将此框架用于 failure diagnosis——在 failure 区域检查特征函数对齐情况，找出哪些特征函数未被激活。
  2. **Convergence degree 指标**：可用于衡量不同区域对训练的"价值"，结合 failure region 定位低收敛度区域作为 failure 信号。
  3. **eNTK 特征谱**：特征值的衰减速率、特征函数的形态变化可用作训练健康的诊断信号。
- **该引不该引**：如果我们的 work 涉及 PINN training dynamics 的理解或 failure 分析中的 NTK 视角，应引用本文的增广空间框架和特征谱分析方法。如果不涉及 PINN，则不必引用。
- **差异化**：我们的核心是找出 PINN 为什么会失败（failure taxonomy/causes），PINNACLE 关注的是如何更高效地训练（point selection optimization）。完全不同的研究问题。

## Read by: 0616-fex (2026-06-16)

### 撞车评估
不撞车。PINNACLE 的核心问题是如何在训练预算约束下最优地选择所有类型的训练点（collocation + experimental），这是一个训练效率优化问题。0616-fex 的核心问题是理解 PINN 训练为何会失败（failure modes / failure taxonomy）。虽然两者都涉及 NTK 分析 PINN 训练动态，但研究目标完全不同。

### 可借用的技术洞察
1. **增广空间 NTK 特征函数分析**：如果 0616-fex 需要对 PINN 失败做 NTK-level 的诊断，PINNACLE 的增广空间框架可以直接借用。具体地，可以：(a) 在 failure case 中查看 eNTK 特征谱的形态（特征值衰减速度、特征函数与真解的对齐程度）；(b) 检查 cross-information 项（K_sp 块）是否在 failure 时异常。
2. **Convergence degree 作为局部训练难度指标**：α(z) 衡量一个点对训练收敛的贡献，低 α 区域可能对应 failure-prone 区域。可用于检测 failure 的 early signal。
3. **残差衰减方向分析**：Equation (19) 表明残差沿主导特征函数方向衰减最快——如果某些 failure mode 对应的残差分量主要对齐于低特征值方向，则可诊断出具体衰减受阻的原因。
