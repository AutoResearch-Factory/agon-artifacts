# Data-Efficient Symbolic Regression via Foundation Model Distillation (2508.19487)

- **Year**: 2025.08
- **Authors**: Wangyang Ying, Jinghan Zhang, Haoyue Bai, Nanxu Gong, Xinyuan Wang, Kunpeng Liu, Chandan K. Reddy, Yanjie Fu (Arizona State University, Clemson University, Virginia Tech)
- **Venue**: arXiv
- **Keywords**: symbolic regression, equation discovery, foundation model distillation, EQUATE, symbolic-numeric alignment, evaluator-guided optimization, low-data adaptation, E2E transformer
- **One-liner**: 用EQUATE把SR基础模型蒸馏到小数据任务。

## 精华提炼

### 管线/方法

1. 输入是一个小规模目标数据集 $\mathcal{D}=(X,y)$，任务是从观测数据自动恢复可解释数学表达式。
2. 论文把问题定义为 foundation knowledge transfer for symbolic regression：不是从头训练 SR 模型，而是把已有 SR foundation model 适配到单个小数据任务。
3. 采用的预训练骨干是 Kamienny et al. 的 End-to-End symbolic regression with transformers (E2E)，因为其模型、权重、logits 可通过 Facebook symbolic regression 仓库访问。
4. Step 1 是专门构造 fine-tuning 训练集，而不是直接拿单个目标表格做监督。
5. 对每个测试方程，从训练数据中 replacement sampling 生成 100 个 observation sets。
6. 每个 observation set 包含 200 个数据点，这与 E2E 预训练时输入点数不超过 200 的约束对齐。
7. 对每个采样子集，用预训练 backbone 生成一个相关但不完美的候选方程 $\tilde f$。
8. 另有 10% 方程随机生成，90% 方程由 backbone 生成，用来增加训练分布的多样性。
9. 将 $\tilde f$ 回代到子集特征 $X_i$，得到方程预测标签 $\tilde y_i=\tilde f(X_i)$。
10. 每个 fine-tuning instance 是四元组 $(X_i,y_i,\tilde f_i,\tilde y_i)$。
11. Step 2 构造 data-equation pair embedding，而不是只把 numeric data 编码成一个向量。
12. 数值 encoder 直接复用 E2E 的预训练 data encoder，把 $(X_i,\tilde y_i)$ 转成数值 embedding $E_n$。
13. 符号 encoder 是单层 LSTM，把候选方程转成 prefix expression token sequence 后编码成 $E_s$。
14. 该 LSTM symbolic encoder 输出 512 维 embedding，最大 token sequence length 为 200。
15. 数值与符号的融合使用 attention：以 $E_n$ 为 query，以 $E_s$ 为 key 和 value。
16. attention score 使用 scaled dot-product $Q(E_n)K(E_s)^T/\sqrt{d_k}$，再经 softmax 得到权重。
17. context vector 是 value embedding 的加权和，最终融合为 $E_f=E_n+context$。
18. 这一步的核心 claim 是建立 shared symbolic-numeric embedding space，使数据形态与表达式结构在同一空间对齐。
19. Step 3 训练 evaluator，使 embedding space 中每个点带有可微的 data-equation quality 信号。
20. evaluator 是两层 MLP，hidden dimension 为 1024。
21. evaluator 的目标值 $r$ 同时包含拟合度和复杂度惩罚。
22. 拟合度部分是 $R^2=1-\Sigma(\tilde y_i-y_i)^2/\Sigma(\tilde y_i-\bar y)^2$。
23. 简洁度部分是 $\lambda\exp(-l(\tilde f)/L)$，其中 $l(\tilde f)$ 是 prefix expression 长度，$L$ 是最大序列长度。
24. evaluator loss 是 $\mathcal{L}_{est}=MSE(r,MLP(E_f))$。
25. Step 4 用预训练 decoder 从融合 embedding 自回归重构方程 token。
26. decoder loss 是 token-level negative log likelihood，即 $\mathcal{L}_{rec}=-\sum_j\log P(t_j|E_f,t_{<j})$。
27. 总训练目标是 $\mathcal{L}=\alpha\mathcal{L}_{est}+\beta\mathcal{L}_{rec}$。
28. 实验中 $\alpha=0.05$，$\beta=100$，用于把 evaluator MSE 与 reconstruction loss 调到相近尺度。
29. fine-tuning 训练 10 epochs，学习率 1e-4，优化器 AdamW。
30. 参数更新策略是保留 foundation prior：data encoder 和 decoder 大部分权重冻结，只训练 equation encoder 与 decoder 最后一层相关参数。
31. inference 阶段先从 fine-tuning 训练实例中选 top-performance points 作为初始 embedding。
32. 每个测试函数输入 10 个 data matrices，得到 10 个初始 embeddings。
33. 然后用 evaluator gradient 做 embedding-space ascent，而不是在离散表达式空间里盲搜。
34. 搜索最多 20 steps，总候选方程数量不超过 100。
35. 一旦生成方程在测试集上达到 $R^2>0.99$，搜索提前停止。
36. 最终用 fine-tuned decoder 把优化后的 $E_f^+$ 解码成 prefix tokens，再按预定义规则转回符号方程。

### 关键技术细节

1. 任务设置是 symbolic regression / equation discovery，目标是小数据、领域特定数据上的 foundation model distillation。
2. 方法名是 EQUATE: Equation Generation via QUality-Aligned Transfer Embeddings。
3. 论文早期 tex 中也出现过 FKT/Data2Eqn 等标题草稿，但最终 arXiv metadata 与主摘要使用 EQUATE 和 Data-Efficient Symbolic Regression via Foundation Model Distillation。
4. 数据集一是 Feynman，共 119 个方程，每个方程输入维度小于 10。
5. 数据集二是 Strogatz，共 14 个 nonlinear dynamics 方程，每个方程输入维度为 2。
6. 数据集三是 PMLB black-box 子集，因为 backbone 输入维度限制，最终保留 57 个 feature dimension 不超过 10 的任务。
7. 训练/测试 split 为 75%/25%。
8. 评价指标包括平均测试 $R^2$、达到 $R^2>0.99$ 的方程比例，以及生成方程 complexity。
9. 主对比中直接比较 E2E-Beam、E2E-Sampling、EQUATE-Beam、EQUATE-Sampling。
10. 附录还列出 SRBench 类 baselines：Operon, SBP-GP, TPSR, GP-GOMEA, EPLEX, MRGP, AFP, AFP_FE, AIFeynman, FEAT, gplearn, FFX, ITEA, DSR, BSR, SNIP。
11. 环境为 Ubuntu 22.04.3 LTS，Intel Xeon w9-3475X CPU@4800MHz，1 张 RTX A6000，48GB RAM，Python 3.11.4，PyTorch 2.5.1。
12. 论文特别强调 input dimensionality <= 10 的限制，这是由 E2E encoder-decoder tokenization 和内存/效率约束导致的。
13. 梯度搜索用 evaluator 的 $\partial\theta/\partial E_f$ 指导 embedding 更新，tex 中公式写作 $E_f^+=E_f+\eta\cdot\partial\theta/\partial E_f$。
14. 这不是强化学习式 policy search，而是把离散方程搜索转成 continuous latent optimization。
15. 复杂度权衡由 $\lambda$ 控制，附录给出 $\lambda=0$ 到 $\lambda=1$ 的 trade-off 例子。

### 核心结果

1. Feynman 上，E2E-Beam 的 $R^2>0.99$ 为 0.798，平均 $R^2$ 为 0.9727，complexity 为 54.44。
2. Feynman 上，E2E-Sampling 的 $R^2>0.99$ 为 0.815，平均 $R^2$ 为 0.9730，complexity 为 53.61。
3. Feynman 上，EQUATE-Beam 提升到 0.857 / 0.9765 / 54.78。
4. Feynman 上，EQUATE-Sampling 最好，达到 0.874 / 0.9833 / 57.76。
5. Strogatz 上，E2E-Beam 的 $R^2>0.99$ 只有 0.286，平均 $R^2$ 为 0.7591。
6. Strogatz 上，E2E-Sampling 提升到 0.357 / 0.8156。
7. Strogatz 上，EQUATE-Beam 大幅提升到 0.714 / 0.9766。
8. Strogatz 上，EQUATE-Sampling 达到 0.786 / 0.9818，complexity 还最低，为 50.11。
9. Black-box 上，E2E-Beam 平均 $R^2$ 为 0.6968，complexity 为 81.72。
10. Black-box 上，E2E-Sampling 平均 $R^2$ 为 0.7225，complexity 为 83.18。
11. Black-box 上，EQUATE-Beam 平均 $R^2$ 为 0.7789。
12. Black-box 上，EQUATE-Sampling 平均 $R^2$ 为 0.7985，complexity 为 79.81，是表中最好组合。
13. transfer strategy 对比显示 Frozen 最强：Feynman 0.874 / 0.9833 / 57.76，训练每 5 epochs 358.8s。
14. Feynman 上全量微调约为 0.773 / 0.9442 / 65.95，训练 447.5s，表现低于 Frozen。
15. Feynman 上 LoRA 为 0.823 / 0.9678 / 63.44，训练 400.5s，也低于 Frozen。
16. Feynman 上 random initialization 失败，$R^2>0.99=0$，平均 $R^2=-2.5076$。
17. Strogatz 上 Frozen 是 0.786 / 0.9818 / 50.11，训练 31.1s。
18. Strogatz 上 random initialization 同样失败，平均 $R^2=-1.7721$。
19. Black-box 上 Frozen 平均 $R^2=0.7985$，优于 full fine-tuning 0.7768 和 LoRA 0.7798。
20. noise robustness 图的结论是：随着 golden labels 加噪，E2E 下降更陡，EQUATE 保持更高 $R^2$，Strogatz 上尤其明显。
21. ablation 图显示 equation encoder 和 embedding optimization 都有贡献。
22. 去掉 embedding optimization 的 EQUATE* 略弱于完整 EQUATE，但仍强于去掉 equation encoder 的版本。
23. 去掉 equation encoder 的 EQUATE# 在平均 $R^2$ 上有时接近完整模型，但 $R^2>0.99$ 高精度恢复明显不足。
24. appendix 的 gradient-search case 显示 strogatz_vdp1 上，EQUATE step 0 初始 $R^2=0.9665$，E2E baseline 只有 0.718。
25. 同一 case 中，EQUATE 通过 evaluator-guided search 在 step 4 到达 $R^2=0.9999$。
26. $\lambda$ trade-off 实验显示，从 $\lambda=0$ 到 $\lambda=1$，complexity 从 52.29 降到 43.29，但准确恢复比例从 0.786 降到 0.643。
27. 反直觉/重要点：冻结大部分 pretrained model 比全量微调更好，说明小数据任务中保留 foundation prior 比增加可训练参数更关键。
28. 另一个重要点：EQUATE 的提升在 Strogatz 上远大于 Feynman，说明结构化 dynamics 小数据场景更受益于 symbolic-numeric alignment。

### 明确局限

1. 作者明确承认输入维度受限于 E2E encoder-decoder 架构，目前只能处理 feature dimension 不超过 10 的任务。
2. 高维数据会导致 token sequence 过长，进而影响训练和推理效率。
3. 论文没有给出高维 tabular/scientific 数据的实验证据，因此不能直接外推到真实复杂实验数据。
4. 论文使用公开标准 benchmark，但没有独立第三方复现。
5. 主文最终摘要中代码链接被注释掉；tex 中只有草稿摘要提到 anonymous 4open 链接，因此 artifact 状态不够清楚。
6. 结果主要是平均指标与比例，没有 confidence interval，也没有按方程类别展开失败案例。
7. noise robustness 和 ablation 的关键数值主要在图中，tex 没有完整数字表。
8. evaluator 学到的 quality surface 是否会在 out-of-distribution 方程族上误导搜索，没有被系统分析。
9. 训练集构造依赖 backbone 先生成候选方程；如果 backbone 初始候选过差，embedding 搜索是否还能逃出局部区域不清楚。
10. 论文声称避免 negative transfer，但主要证据来自 frozen/full/LoRA/random 对比，还缺少对不同预训练分布偏差的细粒度实验。
11. 与 GP/SRBench 全套 baseline 的对比大多在图中，主表只详细列 E2E backbone，因此 baseline parity 需要进一步确认。
12. 没有真实实验室科学发现案例；目前是 benchmark equation recovery。
13. 对 0616-fex 如果关注真实 feature/equation extraction，EQUATE 是方法 prior，不是完整应用系统。

### 跟我们的关系

1. 对 0616-fex，最需要注意的撞车点是“把 foundation model 的结构化生成能力迁移到小数据任务，并用 evaluator/quality signal 在 embedding space 中做搜索”。
2. 如果我们的 claim 是“用 evaluator-guided latent optimization 改善结构化表达式生成”，EQUATE 是直接相邻 prior，必须引用并明确差异。
3. 如果我们的任务是 feature extraction / formula extraction / equation extraction，EQUATE 不处理文档或图表抽取，但处理“从数据生成可解释表达式”的后端建模问题。
4. 可借用的工具链包括：预训练 backbone 候选生成、候选质量打分器、symbolic-numeric dual encoder、attention fusion、gradient-based latent search。
5. 可借用的实验设计包括：小数据目标任务、75/25 split、R2 与 $R^2>0.99$、复杂度、noise robustness、freeze/full/LoRA/random transfer ablation。
6. 差异化方向可以放在更高维输入、真实领域数据、独立 verifier、可复现 artifact、失败案例审计、或把 evaluator 变成真正独立的检验 gate。
7. 若 0616-fex 要做多智能体结构抽取或程序图抽取，这篇更多是“structured generation + refinement”的相邻方法，不是直接抢同一问题定义。

## Read by: 0616-fex (2026-06-17)

本次从 0616-fex 视角看，这篇论文是“structured generation after foundation transfer”的强相关 prior，但不是直接处理文档/图像/程序中的 feature extraction。它最像会撞车的 claim 是：先用 foundation model 生成一批不完美结构候选，再训练 evaluator，把离散结构搜索转成连续 embedding search，最后用 decoder 产出更好的结构化表达。若 0616-fex 的新意也在“评价器引导的 latent refinement”，必须把 EQUATE、2505.15572 和 TPSR 一起作为 closest prior，并说明我们的 verifier 是否更独立、对象是否更复杂、以及是否有真实任务复现。可直接借用的是四元组训练数据构造、symbolic-numeric alignment、quality-aware evaluator、冻结大部分 backbone 的小数据 fine-tuning，以及 noise/ablation/complexity 三类实验。差异化最好落在高维真实数据、程序/图结构抽取、独立验证 gate、artifact 复现，或把 evaluator 从拟合分数预测器升级为可否决的外部结构一致性检查器。
