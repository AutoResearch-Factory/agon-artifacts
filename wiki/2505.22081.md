# Can Test-time Computation Mitigate Reproduction Bias in Neural Symbolic Regression? (2505.22081)

- **Year**: 2025.05
- **Authors**: Shun Sato, Issei Sato (The University of Tokyo)
- **Venue**: arXiv (formatted for ICML 2026)
- **Keywords**: neural symbolic regression, reproduction bias, test-time computation, Transformer expressiveness, circuit complexity, TC^0 vs NC^1, MCTS decoding, expression generation, self-verification
- **One-liner**: 发现 NSR 模型存在严重的 reproduction bias（仅复制训练数据中的表达式而无法生成新公式），从电路复杂性理论证明 Transformer 无法在生成 token 时兼顾数值一致性，并提出子树验证引导的 test-time 策略 NSR-gvs。

## 精华提炼

### 管线/方法

**整体管线**（NeSymReS 基线）：
1. **训练数据合成**：用随机 unary-binary tree 生成表达式模板 `e_templ`（max depth 6），对 unary operator 加乘性常数，对变量加 `c_mul * x + c_add` 仿射变换，sampling 范围 [-10,10] 内随机区间
2. **数据采样**：从表达式中采样 n 个 (x, y) 点作为模型输入
3. **Encoder**：5 层 Set Transformer，将 numerical data (b,n,d) 编码为 (b,s,h) 嵌入（b=200, h=512, s=32）
4. **Decoder**：5 层标准 Transformer decoder，自回归生成 prefix notation token 序列
5. **常数拟合**：生成后用 BFGS 算法优化常数占位符 C
6. **训练**：cross-entropy loss + teacher forcing，AdamW lr=1e-4，4000 步后 inverse sqrt decay

**三种 Test-time 策略**：
- **Large Beam Search**：beam size 从 5 扩到 150，不提供额外信息
- **TPSR**：MCTS 解码（P-UCB selection + NSR 模型控制 expansion/simulation），reward = 1/(1+NMSE) + λ exp(-|seq|/L)，k_max=3，rollouts=3
- **NSR-gvs（本文提出）**：迭代 self-verification 循环，从候选池采样 subtree 作为 prompt 输入模型，生成后用 R^2 验证，更新子树分数（running average）。MDP 框架：state = {(subtree, verification_score, count)}，action = prompt sequence，policy = 从 top-k + random 集采样，30 轮迭代

### 关键技术细节

- **训练数据规模**：简化场景 100K 表达式（限制 operator 类型：add, sub, sin, cos, tan, exp，避免 mul/pow），实用场景 10M 表达式（全 operator）
- **表达生成器**：基于 Lample 等人的 unary-binary tree generator，算子集包括 pow2/3/4/5, sqrt, log, exp, sin, cos, asin（unary），add, sub, mul, div（binary）
- **Evaluation metric**：R^2 的多阈值评估（0.5, 0.9, 0.95, 0.99, 0.999, 0.9999, 0.99999），100 个 holdout 数据点
- **Test dataset**：
  - `not_included`：模板不在训练集中的表达式（测泛化）
  - `baseline`：直接从生成器随机采样（模板大概率在训练集中）
  - `AI-Feynman`：91 个物理方程，最多 5 变量
  - `only_five_variables_nc`：5 变量无常数表达式
  - `black-box`：SR-Bench 中 35 个真实数据
- **Transformer 理论 model**：bounded-depth log-precision（O(log k) bit 精度），depth O(1)，hidden dim 受限于 poly(input length)
- **NSR-gvs 超参数**：k=39（top-k），k_rand=9（随机采样），z_thres=0.213，l_max=9（子树最大 token 长度），l_t ~ U(0, floor(15.58+0.42t))（动态总长度）

### 核心结果

| 发现 | 具体数字 |
|------|---------|
| **Reproduction bias 强度** | 简化场景：1000 epochs 后 >97% 生成表达式是训练数据直接复制 |
|  | 实用场景（1.5M 训练表达式）：<12% 是 novel expression（含常数位置），<6% 是 novel tree structure |
| **Reproduction bias 与训练数据规模** | 从 100K 到 1.5M 扩大训练数据，novel 生成比例饱和，不随数据规模持续增长 |
| **理论结果** | Log-precision Transformer 无法求解 LastTokenPrediction 问题（NC^1-hard 但 Transformer 表达能力限于 TC^0），前提是 TC^0 ≠ NC^1 |
| **Large Beam Search** | 减少 reproduction bias 有限（beam=1→150: novel 3.34%→10.04%），但 R^2>0.99 比例大幅提升（13.37%→32.97%） |
| **TPSR** | 大幅减少 reproduction bias（novel: 27.62%→36.33%），但 novel 表达式中几乎无高精度（novel+R^2>0.99 仅 0.45%→2.02%） |
| **NSR-gvs** | novel 比例 43.09%（BS=1），且 novel+R^2>0.99 达 4.91%→8.68%，优于 TPSR |
| **NSR-gvs+TPSR** | novel 比例最高 57.05%，novel+R^2>0.99 达 12.75%，但推理极慢（单表达式 2-5 小时） |
| **关键反直觉发现** | 减少 reproduction bias 与提升数值精度**没有强相关**；有时减少 bias 反而降低精度（TPSR），有时没减少 bias 但提升精度（Large Beam） |
| **NSRwH 实验** | 提供 appearing subtrees 时 novel 比例大幅提升（从 ~8% 到 ~52%），但精度下降 |

### 理论分析概述

- 定义 **LastTokenPrediction(m)** 问题：给定不完整 token 序列（只差最后一个 leaf token），选出使数值损失最小的 leaf token
- 证明路径：Boolean Formula Value Problem（NC^1-complete）→ reduction 到 LastTokenPrediction → 若 Transformer ∈ TC^0 能解决，则 TC^0 = NC^1，矛盾
- Reduction 构造：Boolean formula → 转换为不含 ∨ 形式 → prefix notation → 替换为数学 token → prepend `+` 作为前缀 → 用两个数据点 ((1,0)→1, (0,-1)→0) 使得 x1 和 x2 可区分 0/1 结果
- **局限**：定理只证明最后一个 token 的预测问题（最简单的情况），并非证明无法生成任何表达式；使用的是 log-precision 假设
- NSR-gvs 的 PAC 保证：在假设 Transformer 在 prompt 包含所有 e* 子树时以概率 β≥0 命中 e* 的条件下，burn-in B = ceil((U/r) ln(2^D0 / (δ/2))) 轮 + R = ceil(ln(2/δ)/β) 轮后，best-so-far 表达式以 1-δ 概率达到 ε-optimal

### 明确局限

1. **作者自述局限**：
   - 理论只覆盖最后一个 token 预测，非完整表达式生成
   - 简化实验场景（受限 operator、小数据集）与实用场景有 gap
   - NSR-gvs 的超参数仅在 5 个随机表达式上做了 Bayesian optimization（not extensive）
   - NSR-gvs+TPSR 推理极慢（2-5 小时/表达式），不实用

2. **审读者发现的局限**：
   - 没有真机部署，全部为 synthetic/simulation 数据
   - 没有在任何标准 benchmark（如 SRBench 完整版）上做全面比较
   - 未探索 reproduction bias 的根本原因（是训练数据偏差、loss 函数、还是架构限制？只定位了现象）
   - 理论用的是 log-precision 假设——实际 Transformer 使用更高精度（fp16/32），定理的实际意义有限
   - 未对比 RL-based 方法（DSR）或 LLM-based 方法
   - 未分析 reproduction bias 与 overfitting 的因果关系（文中有 epoch 消融，但未排除其他 confounder）
   - NSR-gvs 的 MDP 框架本质上是个简单的 stochastic local search，method novelty 有限
