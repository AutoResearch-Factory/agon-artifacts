# BO-SA-PINNs: Self-adaptive physics-informed neural networks based on Bayesian optimization for automatically designing PDE solvers (2504.09804)

- **Year**: 2025.04
- **Authors**: Rui Zhang, Liang Li, Stéphane Lanteri, Hao Kang, Jiaqi Li; University of Electronic Science and Technology of China; Université Côte d'Azur/Inria/CNRS/LJAD
- **Venue**: arXiv
- **Keywords**: Physics-informed neural networks, Bayesian optimization, self-adaptive PINNs, PDE solvers, adaptive sampling, loss weighting, DeepXDE, TG activation
- **One-liner**: 用BO+EMA/RAR-D自动配置PINN并提升PDE求解精度。

## 精华提炼

### 管线/方法
- 输入是一类给定 PDE 问题：研究者先确定网络输入/输出、问题域、PDE residual loss、boundary loss、initial loss。
- 作者不试图自动发现 PDE 本身，而是自动配置 PINN 求解器。
- Stage 1 用 Bayesian optimization 选择网络结构、学习率、采样点数量分布、loss 权重。
- BO 的目标函数是最小化预训练后的 L2 relative error，文中写成最大化负误差。
- BO surrogate model 选择 Gaussian process regression。
- BO acquisition function 选择 expected improvement。
- 初始 BO 采样点是 10 个，使用 Latin Hypercube Sampling。
- BO 后续优化运行 20 次迭代。
- 每一个 BO candidate 都构造一个 PINN，用 ADAM 训练 500 iterations，然后记录 L2 relative error。
- BO 返回误差最小的 hyperparameter 组，以及对应 500 iter 预训练模型。
- Stage 2 使用 global self-adaptive mechanism。
- Stage 2 的第一个机制是 EMA 动态更新 residual/boundary/initial loss weights。
- EMA 的直觉是：哪个 loss 最近更大，就给哪个项更高权重。
- loss moving average 使用参数 beta，权重平滑更新使用 gamma。
- 数值实验里 beta=0.999，gamma=0.999。
- Stage 2 的第二个机制是 RAR-D，即 residual-based adaptive refinement with distribution。
- RAR-D 先随机生成 1000 candidate points。
- 对每个 candidate point 用 Stage 1 预训练模型计算 PDE residual。
- residual 被 max-normalize 后转为概率分布。
- 每轮从这个 residual distribution 里随机采样 50 个新训练点。
- 这比纯 residual argmax 更分散，避免只追一个局部高误差点。
- Stage 2 在算法描述中每 1000 iterations 更新一次 loss weights，总阶段大约 5000 ADAM iterations。
- Stage 3 用 L-BFGS 做稳定精修。
- Stage 3 的迭代数随实验而变：Helmholtz 是 10000，Burgers/Poisson 是 3000，框架描述给出 5000-10000。
- 三阶段总体是：BO 低成本粗搜配置，ADAM 自适应调整，L-BFGS 局部收敛。
- 新激活函数 TG 定义为 phi(x)=tanh(x) exp(-x^2/2)。
- TG 的设计动机是保留 tanh 非线性，同时用 Gaussian window 增强局部特征表达。
- 作者在附录证明 TG 连续且非多项式，因此满足 universal approximation theorem 的常用条件。
- 代码基础是 DeepXDE，文中给出 GitHub 链接。

### 关键技术细节
- FCNN hidden layers 搜索范围是 [2,8]。
- ResNet residual blocks 搜索范围是 [1,4]。
- hidden neurons 搜索范围是 [20,80]。
- ADAM learning rate 搜索范围是 [0.001,0.01]，实验离散步长是 0.001。
- domain sampling points 搜索范围是 [500,5000]。
- boundary sampling points 搜索范围是 [100,3000]。
- initial sampling points 搜索范围是 [100,1200]。
- residual loss weight 搜索范围是 [0.01,0.10]。
- boundary loss weight 搜索范围是 [0.01,0.25]。
- initial loss weight 搜索范围是 [0.01,0.10]。
- sampling point 数量用 50 为步长变化。
- loss weights 用 0.0001 为步长变化。
- 作者建议如果超参数超过 10 维，考虑 REMBO 或 HeSBO。
- 作者建议如果已有先验最优点，考虑 prior-BO。
- 所有实验在 Nvidia RTX 4090D 上运行。
- 每个 case 做 5 次随机实验，但只记录最优 L2 relative error；这是一个重要评估细节。
- Helmholtz case 的总迭代是 15500：500 BO/pretrain + 5000 Stage 2 + 10000 Stage 3。
- Helmholtz 的 BO-SA-PINN 架构是 [2,74,74,1]，参数量 5847。
- Helmholtz 的 SA-PINN 对照架构是 [2,50,50,50,50,1]，参数量 7851。
- Helmholtz 的 BO 选中超参：omega_R=0.0144, omega_B=0.2406, lr=0.005, n_d=1350, n_b=1650, layers=2, neurons=74。
- Maxwell epsilon_c=1.0 超参：omega_R=0.0848, omega_B=0.1215, lr=0.007, n_d=1550, n_b=1200, layers=6, neurons=48。
- Maxwell epsilon_c=1.5 超参：omega_R=0.0225, omega_B=0.1081, lr=0.008, n_d=3150, n_b=2500, layers=5, neurons=37。
- Maxwell epsilon_c=4.0 超参：omega_R=0.0327, omega_B=0.1810, lr=0.007, n_d=4700, n_b=1500, layers=5, neurons=21。
- Burgers 超参：omega_R=0.0410, omega_B=0.0593, omega_I=0.0815, lr=0.006, n_d=4250, n_b=1900, n_i=1100, layers=7, neurons=35。
- 3D Poisson 超参：omega_R=0.0767, omega_B=0.1142, lr=0.007, n_d=13600, n_b=1050, layers=3, neurons=74。
- 4D Poisson 超参：omega_R=0.0328, omega_B=0.2335, lr=0.009, n_d=12150, n_b=6500, layers=6, neurons=59。
- 5D Poisson 超参：omega_R=0.0377, omega_B=0.1565, lr=0.009, n_d=9550, n_b=2700, layers=3, neurons=58。

### 核心结果
- 2D Helmholtz：baseline PINN L2 error 是 1.40e-1，40000 iterations。
- 2D Helmholtz：SA-PINN L2 error 是 3.20e-3，20000 iterations。
- 2D Helmholtz：BO-SA-PINN L2 error 是 3.21e-4，15500 iterations。
- Helmholtz ablation：No TG 是 1.52e-3，说明 TG 对这个 case 贡献最大。
- Helmholtz ablation：No EMA 是 7.34e-4。
- Helmholtz ablation：No SA mechanism 是 7.75e-4。
- Helmholtz 激活函数对比：TG 3.21e-4，tanh 1.52e-3，sin 6.91e-4，Gaussian 8.18e-4。
- Helmholtz 采样效率：BO-SA-PINN 初始 3000 点 + RAR-D 500 点，SA-PINN 使用 100400 点，采样点少约 96.5%。
- Helmholtz 总训练时间报告为 614.5 seconds。
- Maxwell disk scattering 测试了 300 MHz，disk radius 0.25，square side length 2。
- Maxwell epsilon_c=1.0：BO-SA-PINN real/imag error 是 7.98e-5 / 1.11e-4，对照 PINN 是 4.66e-4 / 4.38e-4。
- Maxwell epsilon_c=1.5：BO-SA-PINN real/imag error 是 1.74e-2 / 1.17e-2，对照 PINN 是 2.06e-2 / 2.30e-2。
- Maxwell epsilon_c=4.0：BO-SA-PINN real/imag error 是 2.84e-2 / 6.28e-2，对照 PINN 是 2.07e-1 / 1.59e-1。
- Maxwell 高介电对比下提升明显，但绝对误差仍在 1e-2 到 1e-1 量级。
- ResNet 验证：epsilon_c=4.0 下 ResNet BO-SA-PINN 3.00e-2 / 6.29e-2，8500 iterations。
- ResNet 验证的结论是框架可迁移到非 FCNN 网络，但 FCNN/ResNet 的误差差距不大。
- Burgers equation：BO-SA-PINN L2 error 是 3.56e-4，tanh 版本是 3.59e-4，SA-PINN 是 4.80e-4，baseline PINN 是 6.70e-4。
- Burgers case 中 TG 几乎没有提升，说明 TG 不是普适显著增益。
- Burgers 采样：BO-SA-PINN 用 7250 + RAR-D 500 点，SA-PINN 用 10400 点，少约 25.5%。
- 3D Poisson：BO-SA-PINN 4.52e-5，baseline PINN 3.13e-1。
- 4D Poisson：BO-SA-PINN 6.57e-5，baseline PINN 2.48e-1。
- 5D Poisson：BO-SA-PINN 9.00e-5，baseline PINN 2.52e-1。
- 高维 Poisson 结果非常强，但 baseline 是作者按经验自选，不是已调优强 baseline。

### 明确局限
- 作者自己承认 TG 在第三阶段某些 extreme cases 可能 overfit。
- 作者自己承认 BO 不能保证找到 global optimum。
- 作者自己提出需要用 uncertainty metrics 提高 PINN generalization ability。
- 论文没有报告均值、方差、置信区间，只说每个 case 做 5 次随机实验并记录最优值。
- 记录最优值会高估稳定性，特别是 BO + PINN 训练这种随机性很强的流程。
- 与 baseline 的公平性有差异：Helmholtz/SA-PINN 数字来自文献，Poisson baseline 是作者经验调参。
- Maxwell baseline 使用“others have chosen”的超参，但没有系统说明是否强 baseline。
- 没有与更现代的 operator learning 方法直接比较，如 DeepONet、FNO、PINN-transformer 变体等。
- 没有把 BO 开销计入总训练成本的完整 wall-clock 对比；每个 candidate 500 iter，10+20 次评估本身可能不便宜。
- 没有说明 BO surrogate/acquisition 的实现细节、随机种子、停止条件、失败 case。
- RAR-D 每轮 1000 candidate 和 50 points 是固定设计，未做系统 sensitivity analysis。
- EMA beta/gamma 都固定为 0.999，也未报告调参敏感性。
- TG 激活函数的 universal approximation 证明只说明可表达性，不等同于训练优势。
- 论文没有外部独立复现报告。
- 开源代码链接存在，但本文 TeX 中没有 commit hash、环境 lockfile 或数据 artifact 版本说明。
- 测试集中多数 PDE 有解析解或标准 DeepXDE 数据，工程复杂度有限。
- 没有真机物理实验，属于数值 benchmark 验证。

### 跟我们的关系
- 对 `0616-fex`，这篇是自动化 PDE/PINN 求解器配置方向的直接相关工作。
- 如果我们的 claim 是“用 BO 自动选 PINN 网络结构、采样分布、loss 权重”，这篇会明显撞车。
- 如果我们的 claim 是“训练中根据 loss/residual 自适应调整权重和采样”，它也部分撞车，因为用了 EMA 和 RAR-D。
- 如果我们的课题重点是更可靠的评测、agent 生成 PDE solver、自动发现 PDE 结构、或者跨问题泛化，这篇只是相关 baseline，不是完整撞车。
- 可复用的方法是三阶段模板：BO coarse search + adaptive training + L-BFGS refinement。
- 可复用的搜索空间很有价值，尤其是 loss weights、sampling counts、layers、neurons、lr 的具体范围。
- 可复用的 baseline 是 DeepXDE + Helmholtz/Maxwell/Burgers/Poisson 四组任务。
- 需要避免的弱点是只报 best-of-5；我们应报告 mean/std、seed、BO 总成本、同等预算 baseline。
- 这篇应该引用，尤其用于说明 PINN 自动调参与自适应采样已有 BO-SA-PINN 路线。
- 差异化建议：强调 verifier 独立性、完整预算核算、跨 PDE leave-one-family-out 泛化、失败模式诊断，而不是只追单点最优误差。

## Read by: 0616-fex (2026-06-16)

从 `0616-fex` 视角看，这篇论文的核心价值是把 PINN solver design 拆成可自动化的若干旋钮：网络深度/宽度、ADAM 学习率、domain/boundary/initial 采样数量、loss weights，再用 BO 先粗搜，随后用 EMA 和 RAR-D 在训练中修正。它不是一个通用科研 agent，也不是自动发现 PDE 的系统，但它会撞到“自动配置 PINN/PDE solver”的方法 claim。

最接近的重叠 claim 有三个：第一，BO 驱动 PINN hyperparameter search；第二，训练中按 loss 反馈动态调 loss weights；第三，按 residual distribution 加点做 adaptive sampling。若我们的方案也把这三者作为主贡献，需要明确差异，比如更强的 verifier、同等总预算评测、跨 PDE 泛化、uncertainty-aware search 或自动 benchmark construction。

可直接拿来用的是搜索空间和评测表：Helmholtz、Maxwell disk scattering、Burgers、3D/4D/5D Poisson 都适合作为 PDE solver 自动化实验基准。更建议把它作为 baseline 重跑，并强制报告 mean/std、全部 seed、BO search 总耗时、失败 case，而不是沿用它的 best-of-5 口径。引用价值高，但它的科学可信度属于中等：实验具体，开源可期，但独立验证和统计报告不足。
