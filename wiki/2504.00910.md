# Provably Accurate Adaptive Sampling for Collocation Points in Physics-informed Neural Networks (2504.00910)

- **Year**: 2025.04
- **Authors**: Antoine Caradot, Remi Emonet, Amaury Habrard, Abdel-Rahim Mezidi, Marc Sebban (Hubert Curien Laboratory, Jean Monnet University, Saint-Etienne, France)
- **Venue**: ECML 2025
- **Keywords**: PINN, collocation points, adaptive sampling, quadrature method, Hessian-based sampling, PDE residuals, second-order derivative, trapezoidal interpolation
- **One-liner**: Proposes a provably tighter quadrature rule using second-order derivative (Hessian) information to adaptively sample collocation points for PINNs, with theoretical error bounds and experiments on 1D/2D PDEs.

## 精华提炼

### 管线/方法

**总体 pipeline：**
1. 输入：PDE 问题（含初始条件和边界条件），PINN 网络架构，超参数
2. 核心创新点一（Quadrature Rule）：
   - 将积分域 [a,b] 分成 k 个等长子区间 I_j
   - 对每个 I_j，计算 M_j = max|f''(x)|（通过采样 S=100 个等距点估计）
   - 按公式 n_j = ceil(N * sqrt(M_j) / sum_p sqrt(M_p)) 分配每个区间的梯形数量
   - 在各区间内做分段梯形插值，聚合结果得到积分近似
3. 核心创新点二（hessian-RAD）：
   - 基于 RAD (Residual-based Adaptive Distribution) 框架泛化
   - 采样分布 p(x) ∝ γ(x)^τ / E[γ(x)^τ] + c
   - γ(x) = f''(x) 时为 hessian-RAD；γ(x)=f(x) 为 res-RAD；γ(x)=f'(x) 为 grad-RAD
   - 每 1000 epochs 重新采样一次 collocation points
4. 输出：训练好的 PINN 作为 PDE 替代求解器

### 关键技术细节

- **激活函数**：RELU (Newton cooling) / tanh (Brinkman, Poisson)
- **网络架构**：
  - Newton: 4 hidden layers × 100 neurons + fully connected output
  - Brinkman: 3 hidden layers × 20 neurons + fully connected output
  - Poisson 2D: 3 hidden layers × 20 neurons + fully connected output
- **优化器**：ADAM
- **学习率**：Newton η=1e-5, Brinkman/Poisson η=1e-3
- **Collocation points 数量**：Newton N=40, Brinkman N=30, Poisson N=400
- **候选点预算**：4000 (Newton/Brinkman) / 40000 (Poisson) 个候选点用于构建采样分布
- **重采样频率**：每 1000 epochs
- **τ（温度参数）**：1/2；c（平滑参数）：0
- **硬件**：Apple M1 Pro, 16GB RAM
- **理论核心**：
  - 基于 Lagrange 余项定理，梯形插值误差为 E = -1/12 * h^3 * f''(ξ)
  - 均匀采样的总误差上界：B_tot,unif = (1/12) * (b-a)^3/N^2 * max|f''(x)|
  - 改进方法的误差上界：B_tot,refined ≤ B_tot,unif，且 f'' 变化越大不等号越严格
  - 关键技巧：按 sqrt(M_j) 比例分配梯形数量（而非直接按 M_j），使得最终不等式链可闭合

### 核心结果

| PDE | 方法对比 | 关键发现 |
|-----|---------|---------|
| Newton's Law of Cooling (1D) | hessian-RAD vs res-RAD vs grad-RAD vs unif-RAD | hessian-RAD 收敛最快，L2 误差最低，计算时间可接受 |
| Brinkman-Forchheimer (1D) | 同上 | res-RAD 初始收敛更快（因为解有平台区，f'' 变化不大）；hessian-RAD 最终达到最佳预测 |
| Poisson 2D | 同上 | **hessian-RAD 显著领先**：~1000 epochs 即可收敛，其他方法需更多迭代；收敛后 gap 仍然明显 |

**Surprising 结果**：
- Brinkman-Forchheimer 中 Hessian 方法反而慢于 residual 方法：原因是解在大段平台上 f'' 几乎不变 → hessian-RAD 在该区域放置过少 collocation points → 测试误差大。这是方法的明确 failure mode。
- Newton cooling 中四种方法都能学到解，但 Hessian 方法在速度上胜出（"faster convergence" 而非 "better solution"）。
- 2D Poisson 中 Hessian 的优势极其明显，远超 1D 场景。

**Quadrature 实验（非 PINN）**：
- 三个测试函数在 N=25 时，refined 方法相对误差明显低于 uniform（例：5.47% vs 15.3%；1.89% vs 16.4%；0.049% vs 0.59%）
- k 越大（子区间越多），refined 方法优势越大

### 明确局限

**作者自己说的局限**：
1. Brinkman-Forchheimer 中 Hessian 方法在平台区放点太少导致测试误差大——需要混合策略（f + f'/f'' 组合采样）
2. 高维情况下计算 Hessian 代价高——提出两种缓解方案：随机采样子矩阵、利用 gPINN 已有的梯度计算
3. 理论证明无法直接从 1D 扩展到 2D+：因为双线性插值 P(x,y) 有二阶项，Lagrange 余项推理不再适用，需要全新方法
4. k 的选择没有自动化方法——目前是手动选择

**审读者发现的局限**：
1. 三个 PDE 都是 toy examples（解析解已知），未在更具挑战性的 PDE（如 Navier-Stokes、Burgers 方程）上测试
2. 未与更近期的强基线比较（如 R3、PACMANN——尽管它们被引用了但未直接对比）
3. 候选点预算（4000-40000）和重采样频率缺乏消融实验
4. 所有实验均在 Apple M1 CPU 上运行——未在 GPU 上测试，大规模 PDE 的可扩展性未知
5. hessian-RAD 的"provably accurate" claim 仅针对 quadrature rule 本身，不针对 PINN 训练过程中的实际表现——理论保证与实际表现之间存在 gap
6. 仅在 tanh 和 RELU 两种激活函数上测试，缺少对更现代激活函数的对比
7. 对于 PINN 的 "failure modes" 只讨论了 uniform sampling 导致的问题，未系统分类和对比各类 failure mode 的覆盖度

## Read by: 0616-fex (2026-06-16)

### 审读视角

本文从 PINN failure modes 出发，提出了一种基于 Hessian 的自适应 collocation point 采样方法 hessian-RAD。其核心贡献是两层的：(1) 理论上证明基于 f'' 的 quadrature rule 比均匀采样有更紧的误差上界；(2) 实验上将 Hessian 信息用于 PINN 的 collocation point 自适应采样。

与 **0616-fex** 的关系是**间接但有用的**：

- **Failure analysis 视角**：本文无意中提供了一个很好的 failure analysis case——hessian-RAD 在 Brinkman-Forchheimer 上表现不如 res-RAD，原因是平台区 f'' 几乎为零导致采样稀疏。这正是 "failure explanation" 的典型案例：通过分析 Hessian 的分布特征，可以预测方法的成功/失败条件。
- **★-RAD 框架可作为诊断工具**：统一公式 p(x) ∝ γ(x)^τ + c 使得我们可以系统地插入不同的 γ(x)（residual, gradient, Hessian, 甚至组合），观察哪种 criterion 在什么 PDE 特征下表现好——这本质上是一个 empirical failure taxonomy 问题。
- **PINN failure modes 文献链**：本文的引言和 related work 提供了 PINN failure modes 的完整文献链 (Wang 2020 → Doumeche 2023 → adaptive sampling methods)，可复用为 0616-fex 的背景。

**结论**：不撞车，但可作为工具性引用和方法论参考。特别是如果 0616-fex 涉及 PDE 求解器的 failure mode 分析或自适应采样策略的诊断，本文的 ★-RAD 框架和 Hessian-based 方法的 failure case 都是很好的素材。
