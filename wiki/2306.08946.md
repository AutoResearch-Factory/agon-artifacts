# Bootstrap aggregation and confidence measures to improve time series causal discovery (2306.08946)

- **Year**: 2023.06 (arXiv v1); published at CLeaR 2024
- **Authors**: Kevin Debeire, Andreas Gerhardus\*, Jakob Runge\*, Veronika Eyring (\*equal corresponding). DLR — Institut für Physik der Atmosphäre / Institut für Datenwissenschaften; TU Berlin; University of Bremen (IUP).
- **Venue**: CLeaR 2024 (Conference on Causal Learning and Reasoning, PMLR). Code in `tigramite`.
- **Keywords**: causal discovery, time series, bootstrap aggregation (bagging), confidence/uncertainty estimation, PCMCI+, LPCMCI, conditional-independence-based, edge-frequency, majority voting
- **One-liner**: 提出保留时序滞后结构的 bootstrap 重采样，套在 PCMCI+ 等 CI 型时序因果发现算法上，既给每条边一个 bootstrap 频率作为置信度，又用逐边多数投票聚合提升 precision/recall（尤其高自相关/大 N/短 T 的困难场景），代价是算力翻倍。

## 精华提炼

### 管线/方法
完整 pipeline（输入 → 输出）：

1. **输入**：一段多变量平稳时间序列 `D = {w_s}`，`w_s = (x^1_s,...,x^N_s)`，长度 T；一个"待 bootstrap"的时序因果发现算法（本文主用 **PCMCI+**，附录另做 PC、LPCMCI）；bootstrap 次数 B。
2. **保时序依赖的重采样**（核心技术贡献，Sec 3.2）：PCMCI+ 内部用 moving-window 把原始序列变成"样本集" `S^{(X,Y,Z)} = {v_s}`，其中每个 `v_s` 已经把一次 CI 测试所需的、来自不同滞后 `X,Y,Z` 的值按正确 lag 打包在一起（如 `v_s = (x^1_{s-1}, x^2_s, x^3_{s-2}, x^2_{s-1})`）。**关键点**：不能像普通 bootstrap 那样对原始时间索引 `I_D` 有放回重采样（那样会把相隔很远的时刻拼进同一个样本，破坏 lag 结构）；而是对**已打包的样本索引集 `I_S` 做有放回重采样**，得到 `I*_S`，令 `S* = {v_s | s ∈ I*_S}`。这样每个样本内部的 lag 关系被完整保留。
3. **一次重采样、全程复用**：对第 b 个 bootstrap 实现，`I*_S^{(b)}` 只抽一次，然后该实现内**所有** (条件) 独立性检验都用这同一套重采样样本 —— 与标准 bootstrap 语义一致。
4. **产出图集**：对 B 个 bootstrap 数据集各跑一次算法，得到 B 张因果图的 ensemble `C = {G_1,...,G_B}`。
5. **两路使用 ensemble**：
   - **(a) 聚合 (bagging)**：逐边多数投票 → 单张 `G_bagged`（Sec 3.3）。
   - **(b) 置信度**：把每条边的 bootstrap 出现频率 `r_e^boot` 当作该边的置信度（Sec 3.4），可视化为边的粗细。

### 关键技术细节
- **PCMCI+ 基础**（Sec 2.2）：两阶段 CI-based。第一阶段 PC₁ 对滞后对做骨架剪枝，得到滞后父集 `B^-_t`；第二阶段用 **MCI 检验** `X^i_{t-τ} ⫫ X^j_t | S, B^-_t(X^j_t)\{...}, B^-_t(X^i_{t-τ})`，对 `B^-` 的条件化保证即使强自相关也**校准良好**。假设：Causal Sufficiency、Faithfulness、Causal Markov、causal stationarity、一致 CI 检验。自由参数：`α_PC`（CI 显著性水平，经验上是 FPR 上界）、`τ_max`（最大滞后）、CI 检验选择。
- **可能的边类型**：滞后对 (τ>0) 只有 {no edge, →}；同期对 (τ=0) 额外有 {←, ∘−∘ (方向不确定), ×−× (冲突)}。
- **聚合 tie-break**（Alg. 1）：对每个有序顶点对统计各边类型频率，取 argmax；平局优先序（保守，偏好信息量更少的边）：no edge > ×−× > ∘−∘ > {→,←}（同级）；→ 与 ← 平局 → 判为冲突 ×−×。
- **两种聚合策略**：(i) 简单多数投票（默认）；(ii) **alternative**：先只投"有边/无边"定 adjacency（no edge 只有出现 >50% 才能赢），再对确定的 adjacency 投方向。
- **置信度的统计含义**（Sec 3.4）：假设平稳，则 `S^{(X,Y,Z)}` 是某 t-无关分布 `F` 的（非 iid）样本，bootstrap 样本来自其经验分布 `F̂_T`；当 T→∞、B→∞，`r_e^boot` 收敛到"反复从模型重新生成数据"得到的参考频率 `r_e`。**caveat**：逐边聚合忽略边与边之间的相关性，丢失联合结构信息。
- **合成实验设置**（Sec 4）：加性模型 `X^j_t = a_j X^j_{t-1} + Σ_i c_i f_i(X^i_{t-τ_i}) + η^j_t`；自相关 `a_j ~ U[max(0,a-0.3), a]`；噪声 Gaussian(σ~[0.5,2]) 或 Weibull；cross-links `L=⌊1.5N⌋`，平均入度 d=1.5（N 越大越稀疏）；30% 同期 + 70% 滞后 (τ∈{1..5})；每个 setting 跑 **500 个模型**。CI 检验：线性用 ParCorr，非线性用 GPDC（tigramite）。评测：recall/precision/PR-AUC（区分滞后 cross / 同期 / auto）、FPR、冲突率、runtime（AMD EPYC 7763）。
- **确认度实验**：第一实验 250 模型×D=100 独立样本估参考频率，B=1000；第二实验 500 模型、D=5000 估参考、B 从 25 扫到 2000。

### 核心结果
- **Bagged-PCMCI+ 系统性优于 PCMCI+**：adjacency 与同期定向的 PR-AUC 一致更高；同期定向提升尤为明显。
- **困难场景增益最大**：高自相关 a、大变量数 N、短样本 T 时提升最显著。
- **FPR 更低**：在 α_PC ∈ [0.001, 0.1] 全程，Bagged-PCMCI+ FPR 一致低于 PCMCI+，两者都把 FPR 控制在 α_PC 以下 → 可对 Bagged 用更高 α_PC 换 recall 仍守住 FPR。
- **B 的影响**：B=50 与 B=200 差别不大；推荐 B≥100。alternative 聚合对 adjacency PR-AUC 更好，简单多数对同期定向更好。
- **置信度质量**：link-frequency MAE ≈ 3%（滞后/全部），≈ 7%（同期，更难）。参考频率 40–60% 区间有**持续正偏**（高估）。误差随 α_PC→0 下降（1e-2→1e-4），是否 →0 收敛不明。取 99% 的估计其误差棒穿过 x=y 对角线。
- **泛化**：Bagged-PC、Bagged-LPCMCI 同样改善（Bagged-LPCMCI 只在同期定向改善，adjacency 无改善——诚实报告的负结果）。
- **代价**：不并行时 B 从 25→500 runtime 涨约 20 倍；可 embarrassingly parallel。

### 明确局限
- **只有合成数据**：尽管以地球科学/生物/工程为动机，全文无真实数据实验，无标准 benchmark（合成 setup 沿用 Runge 2020）。
- **算力**：主要弱点，B 倍运行时。
- **逐边聚合不保无环性**：可能产生环（作者辩称"环=不确定性/假设违背的提示"，算 feature）；也**忽略边间依赖**，丢联合结构。
- **置信度有持续正偏**（40–60% 区间高估），是否随 B→∞ 收敛到 0 不清楚。
- **渐近一致性只是猜想**（SM Conjecture 1）：有明确 loophole —— bootstrap 数据集采自经验分布而非真实数据生成分布，无法立即断定 bootstrap 上的 CI 检验渐近正确判定真分布的 (in)dependence；建议研究 α_PC→0 极限。**未证明**。
- **平稳性是硬前提**：置信度的理论解释建立在 causal stationarity 上。

### 跟我们的关系
针对 topic `0710-causal-scs`（因果效应**不稳定性**作为强对流 SCS 预警指标；候选方法含 PCMCI+/J-PCMCI+/LKIF）：

- **强相关：这是"PCMCI+ 上的不确定性量化"的基础工具论文，几乎必引、大概率要用**。我们 idea 的不稳定性指标里明确列了"置信区间宽度"和"DAG 结构相邻窗口 edge 变化"；本文的 **bootstrap 逐边频率 `r_e^boot`** 正好是一个现成的、原理清晰的 per-link 置信/不确定度量，且已进 `tigramite`（`Bagged-PCMCI+`/`Bagged-LPCMCI`），可直接在滑窗内调用。
- **保时序结构的 bootstrap 采样是关键可复用件**：我们要在滑动窗口里做因果发现并量化窗内不稳定性，若要 bootstrap 时间序列而不破坏 lag，本文 Sec 3.2 的"重采样样本索引集而非原始时间索引"就是正确做法，避免自造轮子。
- **框架相反、机制邻近——既是工具也是对照/counter-framing**：本文把 bootstrap 不稳定性当作要**消除的噪声**（bagging 平均掉有限样本误差以提精度），置信度里"高频=高信心"；我们 idea 反过来要把**低置信/跨窗口高方差当作信号**。这与 idea review 里已识别的 Causal-Audit（2604.02488，正是引用本文的后续工作）同属"利用/评估违背"这条线。差异化必须讲清：我们不是"用 Bagged-PCMCI+ 的置信度"，而是"在**故意违背平稳性**的非平稳滑窗下，该估计器的脆弱性携带事件特异信息"。
- **重要 caveat（撞车风险点 + 方法陷阱）**：本文置信度的理论正当性**建立在 causal stationarity 上**，而我们 idea 恰恰运行在非平稳 regime —— 不能天真地把 `r_e^boot` 当作干净信号直接用；且本文自曝置信度在 40–60% 区间有**持续正偏**、同期链 MAE~7%，这类系统性偏差可能被误当成"不稳定性信号"，负对照/surrogate 必须能把它排除。
- **不直接撞车**：本文目标是提精度 + 给置信度，**不涉及**早期预警/临界点/instability-as-signal，因此不冲击 idea 的核心新颖性 claim；但它把"bootstrap 边频率作为因果链置信度"这一手段确立为 prior art（Debeire et al. 2024），我们的 novelty 表述要与"只是用 Bagged-PCMCI+ 置信度"划清界限。

## Read by: 0710-causal-scs (2026-07-10)

**审读视角**：SCS 强对流预警课题下，把因果估计不稳定性当预警信号，候选方法含 PCMCI+/J-PCMCI+/LKIF。

- **撞车评估**：不撞核心新颖性。本文只做"提精度 + 给 per-link 置信度"，不碰早期预警/临界点/instability-as-signal。但它把"bootstrap 边频率 = 因果链置信度"确立为 prior art（Debeire et al. 2024），我们的 novelty 表述必须与"直接用 Bagged-PCMCI+ 置信度"划清界限——差异点在于**故意违背平稳性的非平稳滑窗**下，脆弱性是否携带**事件特异**信息。
- **可用工具**：(1) `tigramite` 里的 `Bagged-PCMCI+`/`Bagged-LPCMCI` 可直接产出 per-link bootstrap 频率，作为我们不稳定性指标里"CI 宽度/edge 变化"的现成量化。(2) Sec 3.2 保 lag 结构的重采样是滑窗内 bootstrap 时序数据的正确做法，直接复用避免自造。
- **方法陷阱（务必写进 negative-control 设计）**：置信度理论建立在 causal stationarity 上，而我们运行在非平稳区；本文自曝的 40–60% 区间**持续正偏**和同期链 ~7% MAE 是系统性偏差，极易被误当"不稳定信号"。surrogate/匹配负对照必须能排除这类估计器固有偏差与有限样本效应。
- **差异化落点**：与 Causal-Audit（2604.02488，本文的引用者）一样属"failure-aware causal diagnostics"线，但我们把违背当**正向事件信号**而非风险规避；需在严格预事件窗口、匹配非平稳负对照、经样本量校正的 fragility score 上，证明相对经典 EWS/CAPE-shear 的增量预警价值。
