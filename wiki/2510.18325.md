# GoodRegressor: A Hierarchical Inductive Bias for Navigating High-Dimensional Compositional Space (2510.18325)

- **Year**: 2025.10
- **Authors**: Seong-Hoon Jang (Tohoku University, Institute for Materials Research)
- **Venue**: arXiv preprint (sole author)
- **Keywords**: Hierarchical inductive bias, Interpretable scientific AI, Symbolic regression, Interaction-depth evolution, Compositional complexity, High-dimensional materials space, Lexicographic search
- **One-liner**: 提出 GoodRegressor，一种层次化深度控制的符号回归框架，通过词典序扩展系统地组装非线性描述符交互，在氧离子导体/NASICON/超导氧化物上达到或超越黑箱模型性能，并发现交互深度演化可作为科学数据集层次复杂性的诊断轴。

## 精华提炼

### 管线/方法

完整 pipeline（五模块串联）：
1. **Parser**：输入化学式 + 目标指标 → 解析化学式 → 输出 parsed dataset
2. **Designer**：输入原子信息、结构信息、用户定义特征 → 计算统计描述符（均值〈·〉、标准差 σ(·)、偏度 r(·)）→ 输出 feature-analyzed dataset（记为 X°₁ = {x₁, ..., xₙ}）
3. **Curator**：构造 X₁ = X°₁ ∪ Xᵐ₁（乘法交互，如 xᵢxⱼ）∪ Xᵈ₁（除法交互，如 xᵢ/xⱼ，带符号稳定性约束）→ 输出 curated dataset。对于氧离子导体数据，n(X₁) = 358
4. **Regressor**（核心）：5 步符号回归算法（见下文），MPI 并行化实现，输出 N 个独立模型 M_{f,i}
5. **Designer (post-process)**：对 {M_{f,i}} 做 stacking ensemble → 共识模型 M_{f,Nf}，提取重要交互链 𝕀*，做材料预测

### 关键技术细节

**符号回归核心算法（五步法）**：
- **a. Run through (the jungle)**：给定 n 个活性变量，词典序枚举所有 C(n(X₁), n) 种组合，按 "jumping-jack-flash interval" nⱼ 分配到 n_core ≥ 2048 个 CPU 核上（每核算第 1, 1+nⱼ, 1+2nⱼ... 个模型）。时间限制 1000 秒。每个核保留满足 Full Fisher 条件（F 检验 p < 0.05 + 所有系数的 t 检验 p < 0.05）下验证集 R² 最高的模型 M_core,i
- **b. Swap**：从 M_core,i 开始，移除 p 值最大的变量，逐一替换为未激活变量，保留 R²_val 最高的配置。从最不显著到最显著重复
- **c. Transit**：对活性变量施加 109 种标量变换（多项式系列 x⁻³~x³、对数系列、幂系列 10ˣ 等、指数系列、误差函数系列 erf(x) 及偏移/缩放/平方变体 ~40 种、正弦系列 sin(·) 21 种、余弦系列 cos(·) 21 种），逐个测试保留最高 R²_val 的形式
- **d. Pick**：cross-core 选取全局最优（最高 R²_val）。以 n→n-1 个变量，但从扩展候选池 X₂ = X₁ ∪ X°_opt ∪ Xᵐ_opt ∪ Xᵈ_opt 重新搜索。循环直至 ⟨R²⟩_cv 不再提升。最终模型记为 M_f
- **e. Bagging**：不同 train-validation split 重复 N=10 次，产生 M_{f,1}...M_{f,10}，stacking ensemble 产生 M_{f,10}

**搜索空间规模**：
- 简单线性组合：N∨ = C(358, 20) ≅ 2.86×10²³（远小于 long double 上限 10⁴⁹³²）
- 引入 109 种变换后：N∨' = 1.60×10²⁶
- 引入交互项后：N∨'' = 1.44×10²⁸
- 模型索引理论上限：10⁴⁹³²（GCC）或 10⁴⁹³（MSVC）

**超参数与设置**：
- 初始活性变量数 n = 20（逐步递减至最优窗口）
- Bagging N = 10 iterations
- 每轮 train:validation = 8:2
- 5-fold cross-validation，20% hold-out testing
- 交互深度演化：n = 20→19→...→1，n 越小 = 交互深度越大（更少的加性分量迫使模型依赖高阶复合交互项）
- Stacking 最小二乘聚合：M_{f,i} = m₀ + Σⱼ mⱼ M_{f,j}

**数据集**：
- 氧离子导体：n_data = 483, n(X₁) = 358, 目标 Ea 和 log₁₀A
- NASICON：n_data = 180, n(X₁) = 211, 目标 log₁₀ σ_Na,RT
- 超导氧化物：n_data 从 11964 过滤到 1358（去除成分相似的低 Tc 样本），n(X₁) 含 Hubbard U/J 参数

**特征池（20 种基础属性）**：O_M（氧金属比）、M（原子质量）、Z（价态）、r（Shannon 离子半径）、B/G（体/剪模量）、ρ/ρ_molar（密度/摩尔密度）、η（离子填充率）、n/l（主/角量子数）、α（热膨胀）、κ（热导率）、χ-χ_O（电负性差）、ν（泊松比）、θ_D（德拜温度）、n_c（氧配位数），加上均值/标准差/偏度统计共约 60 个基础特征，curator 扩展到 358

### 核心结果

**Benchmark 性能**（vs 9 个基线）：
| 数据集 | 指标 | GoodRegressor | 最佳黑箱 | 最佳白箱 |
|--------|------|---------------|----------|----------|
| 氧离子导体 Ea | ⟨R²⟩ | 0.726 | 0.667 (XGBoost) | SISSO OOM |
| 氧离子导体 Ea | ⟨RMSE⟩ | 205 meV | 225 meV | - |
| 氧离子导体 logA | ⟨R²⟩ | 0.654 | 0.574 (XGBoost) | - |
| NASICON | ⟨R²⟩ | 0.862 | 0.828 (XGBoost) | 0.817 (PySR) |
| 超导 Tc | ⟨R²⟩ | 0.536 | 0.657 (LightGBM) | - |

**交互深度演化的三个系统特征**：
1. **氧离子导体**：尖锐最优窗口（18≥n≥13 for Ea，20≥n≥10 for logA），深度<n=20 时性能显著退化。交互深度敏感型
2. **NASICON**：最优在 n≈20，维度依赖性弱，禁用交互无显著退化。弱耦合型
3. **超导氧化物**：明显最优窗口（17≥n≥13），但峰较宽。强但部分冗余的层次纠缠型

**三个可观测量定义层次复杂性分类学**：
- 最优深度位置（optimal depth location）
- 最优窗口的锐度（sharpness of the optimal window）
- 禁用交互时的性能退化幅度（degradation magnitude）

**交互分析发现**（氧离子导体 Ea）：
- l∃=1 最重要特征：⟨n_c⟩（主量子数均值）
- l∃=2 最重要交互：{⟨n_c⟩, O_M}（主量子数 × 氧金属比）
- l∃=3 最重要交互：{⟨n_c⟩, O_M, ⟨ν⟩}（加入泊松比），三维偏依赖图显示：高 ⟨n_c⟩ + 高 O_M 区 Ea 最低，且受 ⟨ν⟩ 调制——更大的泊松比使晶格更容易"呼吸"，促进离子迁移
- logA 的最重要单特征：⟨r⟩（离子半径），二特征交互：{⟨r⟩, ⟨Z⟩}

**材料预测能力**：
- 磷灰石型 La₁₀.₆Si₅.₄Al₀.₆O₂₇ 预测 Ea=494meV（vs 实验 Nd 类似物 658meV）
- NASICON: Na₃.₄Y₀.₄Hf₁.₆Si₂PO₁₂ 预测 σ=3.94×10⁻² S/cm
- 超导：Cu₂.₈W₀.₁₈Mo₀.₀₂Ca₁.₈Ba₀.₂Ce₀.₈₁Lu₀.₀₉Ca₀.₁O_z 预测 Tc=275K（可能含数学伪影）

**Stacking 收敛性**：Rall 在 i=5→10 改善可以忽略，10 个 ensemble 模型足够

**可复现性**：两独立 stacking ensemble 预测值 Pearson r=0.980，且识别相同最优材料

### 明确局限

1. **单作者、未发表**：arXiv 预印本，未经同行评审，sole author
2. **仅有材料科学验证**：虽然声称 domain-agnostic，但全在三类材料数据集上验证（氧离子导体、NASICON、超导氧化物），缺乏其他科学领域（化学、生物、物理）的验证
3. **搜索空间巨大但依赖启发式**：109 种变换 + 词典序跳跃采样 + swap/transit 局部优化均为启发式设计，没有理论保证能找到全局最优
4. **计算资源要求高**：需要 ≥2048 CPU 核心和 MPI 并行，小实验室难以复现
5. **超导 Tc 的 R² 仅 0.536**：远低于黑箱 SOTA（LightGBM 0.657），且最高预测 Tc=275K/287K 作者也承认"可能是数学伪影"
6. **特征工程仍依赖领域知识**：初始特征池（Supplementary Table 1 的 20 种属性）仍由材料科学家手动选择，不是完全自动化的端到端系统
7. **没有与 FEX/深度符号回归比较**：虽与 PySR、Φ-SO 比较，但没有跟基于 RL 或基于 Transformer 的符号回归方法（如 FEX）比较
8. **没有消融各组件**：未做 run-through vs swap vs transit vs pick 的消融实验，不确定每步贡献
9. **交互深度定义绑定到算法机制**：交互深度的增加依赖于 n 的递减（更少的加性分量），此定义与算法耦合，缺乏独立的复杂度度量

### 跟我们的关系

对 **0616-fex (FEX: RL for Symbolic Regression)** 课题：

**撞车评估**：不直接撞车。GoodRegressor 使用词典序枚举 + 局部优化的确定性搜索策略，FEX 使用 RL（强化学习）搜索。两者共享相同的目标问题（符号回归的高维搜索），但方法论正交。GoodRegressor 的出现说明符号回归领域正在从"算法竞赛"转向"归纳偏置设计"——这一点与 FEX 使用 RL 作为归纳偏置的思路高度一致。

**可用的工具/方法**：
1. **交互深度演化 (interaction-depth evolution)** 概念可以直接移植到 FEX：在 FEX 的 RL 框架中引入深度控制轴，让 RL 自动选择深度而非手动调 n
2. **三个复杂度分类学量**（最优深度位置、窗口锐度、禁用退化幅度）≥ 可以在 FEX 实验中使用来诊断不同数据集的层次复杂性
3. **109 种标量变换列表** 可以作为 FEX 的算子库参考
4. **交互链提取 (𝕀* with n∃ and z∃)** 是一个很好的可解释性输出，FEX 可以借鉴来展示发现的重要特征交互
5. **Stacking ensemble 策略** 简单有效，FEX 的 RL 生成的多模型也可以用类似方式聚合

**差异化方向**：
- FEX 使用 RL → 更 adaptive 的搜索，可能搜索效率更高
- GoodRegressor 使用确定性词典序 → 可复现性更好，但受限于枚举策略
- FEX 可以突出 RL 在更高维搜索空间中（如 n(X₁) > 500）的优势
- GoodRegressor 缺少与 RL-based 符号回归的比较 → 这是 FEX 论文可以强调的 gap

## Read by: 0616-fex (2026-06-25)

GoodRegressor 是一篇与 FEX 高度相关但不撞车的论文。它从归纳偏置设计的角度重新审视符号回归——这正是 FEX 项目的核心叙事方向。

**方法论层面的差异**：
- GoodRegressor：词典序枚举 + 直觉启发 (swap/transit/pick) + MPI 大规模并行
- FEX：强化学习驱动的序列决策搜索

这两种方法共享一个核心洞察：在高维符号空间中，不加约束的随机搜索无法有效探索，需要结构化的归纳偏置。GoodRegressor 选择"层次深度控制"作为偏置，FEX 选择"RL 的策略梯度"作为偏置。

**FEX 可以从这篇论文中获得的关键启发**：
1. **交互深度作为可诊断轴**：可以在 FEX 实验中对不同数据集做深度扫描，展示 FEX 自动发现的深度窗与 GoodRegressor 枚举窗的一致性/差异
2. **交互链分析 (n∃, z∃)**：可以作为 FEX 的标准输出，增强可解释性
3. **109 种标量变换作为 FEX 算子库的参考 baseline**
4. **三个复杂度分类量** 可用于 FEX 论文的实验部分，展示 FEX 能处理的复杂度谱系

**FEX 的竞争优势（需要突出）**：
- RL 不需要 MPI 并行 + 2048 核 —— 更 practical
- RL 可以自适应当前发现质量决定搜索深度 —— 比固定"n→n-1"策略更灵活
- FEX 可以声称 "不需要手动定义交互深度"，让 RL 自主发现最优复杂度
