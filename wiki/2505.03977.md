# Call for Action: Towards the Next Generation of Symbolic Regression Benchmark (2505.03977)

- **Year**: 2025.05
- **Authors**: Guilherme Seidyo Imai Aldeia (UFABC), Hengzhe Zhang (VUW), Geoffrey Bomarito (NASA Langley), Miles Cranmer (Cambridge), Alcides Fonseca (ULisboa), Bogdan Burlacu (FH Upper Austria), William G. La Cava (Harvard Medical School), Fabricio Olivetti de Franca (UFABC)
- **Venue**: arXiv / GECCO 2025 (nonacm acmart format)
- **Keywords**: symbolic regression, benchmark, SRBench, performance profile, model complexity, energy consumption, hyperparameter tuning, reproducibility
- **One-liner**: SRBench 2.0 将 symbolic regression 基准方法从 14 种扩至 25 种，引入双轨道 (black-box + 第一原理/现象学) 数据集、performance profile 可视化和能耗分析，发现无单一算法在所有数据集上占优，并提出社区驱动的 living benchmark 及弃用标准。

## 精华提炼

### 管线/方法

**整体 benchmark pipeline**：
1. **数据集选择** (双轨道):
   - **Black-box track**: 从 PMLB 中筛选 12 个回归数据集。筛选方法: 先排除所有算法 R^2 > 0.99 或线性回归即可 R^2 > 0.99 的数据集; 然后用 t-SNE 对数据集元特征 (样本量、特征数、各算法在 SRBench 1.0 的表现) 降维; k-means 聚 12 类，每类取离质心最近者。额外约束 Friedman 类数据集不超过 25%。
   - **Phenomenological & first-principles track**: 12 个数据集，部分来自真实测量 (multiviewsr, cranmer2023)，部分人工生成加噪声。包含吸收光谱、Bode、Hubble、Kepler、Leavitt、Planck、Rydberg 等经典物理/天文学关系。
2. **超参数调优**: 对每个方法做 grid search，训练数据上 3-fold CV 选最佳配置 (按平均 R^2)。FFX 无超参数除外，Bingo/Brush/GP-GOMEA 不兼容 GridSearchCV 接口，用默认值。off-the-shelf 配置被强制加入候选集 (因为初步实验表明某些问题上默认配置优于调优)。
3. **训练与评估**: 75%/25% train/test split，每个 (算法, 数据集, seed) 组合重复 30 次独立运行 (共 25x24x30 = 18,000 次实验)。时间预算: 超参搜索 6 小时、最终训练 1 小时，通过 SIGALRM 终止。
4. **模型大小度量**: 将模型转换为 SymPy 表达式，统计节点数 (不做 simplify，因为 SymPy simplify 不可靠且可能增大表达式)。
5. **能耗监控**: eco2AI 库通过 Linux 命令监控 CPU/内存估算能耗 (在最终训练阶段测量)。
6. **结果可视化**:
   - **Performance profile plot**: Dolan-More 风格，x 轴 R^2 阈值，y 轴达到该阈值的运行次数比例 P[R^2>=x]。AUC 作为聚合度量 (1.0=全部运行 R^2=1)。
   - **Cluster map**: 数据集 x 算法的 AUC 热力图，单元格大小反比模型大小。
   - **Pareto front**: 第一原理轨道用 R^2 vs 模型大小的 Pareto 前沿，标注 ground-truth 位置。

### 关键技术细节

- **25 种算法**: 涵盖 GP-based (AFP, AFP_fe, AFP_ehc, Brush, EPLEX, FEAT, GeneticEngine, GPGomea, GPlearn, GPZGD, Operon, PySR, TIR, ITEA), 神经网络 (EQL, NeSymRes, uDSR), Transformer (E2E, TPSR), 贝叶斯 (BSR), 确定性 (FFX), 迭代 (Rils-Rols), 分段树 (Ps-Tree), 概率采样 (Qlattice)
- **硬件环境**: 单计算集群, 每个 job 10GB RAM, GPU 仅供支持的算法, 不允许子进程或多核
- **总计算量**: 单核连续运行需 1 年 43 周 5 天
- **参数优化 (const. opt.)** 是区分性能的关键特征: 最优算法几乎都具备。方法包括线性缩放 (linear scaling)、基于 OLS 的参数优化、梯度/非线性优化。
- **具体发现**: GPZGD (Z-score 标准化 + SGD 参数优化) 位列 top performer。Brush (multi-armed bandits) 和 GP-GOMEA (linkage learning) 同样表现突出。
- **Ground-truth 恢复**: 仅 ITEA 成功恢复了 Leavitt 定律 (-0.94 * log(P))。其他数据集上 SR 算法要么产生更复杂但更精确的表达式，要么更简单但更不精确。
- **噪声影响**: Hubble 和 Tully-Fisher 的 ground-truth 因原始数据噪声本身不能达到完美 R^2，这种情况下更复杂模型得到更高 R^2 实际上是过拟合噪声而非捕获潜在关系。

### 核心结果

| 发现 | 细节 |
|------|------|
| No free lunch | 无单一算法在所有数据集上占优; 无算法在所有问题上失败 |
| 参数优化是关键 | 几乎所有 top performer 都包含常数优化步骤 (线性/梯度/非线性) |
| 超参调优不一定优于默认 | 某些问题 off-the-shelf 配置胜于 grid search 调优 |
| 能耗差异显著 | Python 算法能耗最高，C++ 算法 Operon 因内嵌调优也偏高 |
| GPU vs CPU 差距不大 | TPSR (GPU) vs CPU 算法能耗差异有限，但 TPSR 执行时间更长 |
| 数据集聚类 | 优质算法在一个数据集集群上表现一致好; 有些弱算法在特定数据集上反而最优 |
| 第一原理恢复困难 | 12 个第一原理数据集中仅 1 个被完美恢复 (ITEA/Leavitt) |
| FFX 在 557 数据集上 AUC 最佳 | 但 Rils-Rols 以略低 AUC 换取了更小模型 |
| uDSR 与 eco2AI 不兼容 | 能耗测量失败 |

**Surprising/反直觉**: 
- Hyperparameter tuning 不一定优于 off-the-shelf 默认配置，作者强制将默认配置加入 grid search 候选集。
- GPU 方法在能耗上并未表现显著优势。
- 增加运行次数从 10 到 30 次揭示了更多变异性和可靠性差异。

### 明确局限

**作者自己说的局限**:
- 能耗测量非精确 (通过 Linux OS 命令估算): "not exact, this provides a rough yet informative profile"
- 时间管理不精确: 有 max_time 的算法确实在时限内停止，但有些 (如 NeSymRes) 远早于时限终止，导致不公平比较
- uDSR 与 eco2AI 不兼容，能耗数据缺失
- SymPy 简化不可靠，不做表达式化简导致模型大小测量可能偏保守
- benchmark 结果强烈依赖于所选数据集和超参设置
- 未包含不确定性信息或误差条用于模型评估
- 多种算法仓库缺乏维护 (bugs 频出)，需要社区共同决定弃用标准

**审读发现的局限**:
- **没有真机验证**: 纯 benchmark，不涉及任何实际物理实验或真机部署
- **没有标准 benchmark 对比**: 虽自称 SRBench 2.0，但与 SRBench 1.0 的结果不可直接比较 (数据集不同、算法版本不同、条件不同)
- **没有做 ablation study**: 未系统分析哪个组件对性能贡献最大
- **数据集规模小**: 24 个数据集，每类 12 个，代表性存疑
- **第一原理数据集的噪声比例不一致**: (10%, 10%, 10%, 1%, 20%) 缺乏统一性考量
- **只报告 max aggregation**: 可能掩盖最差情况
- **统计检验缺失**: 无 paired statistical test (如 Wilcoxon) 来判断显著性差异
- **与 FEX/dim-lift-skeleton 课题**: 这是 benchmark 论文，不提出新方法。与 FEX 可能有关联——FEX 若涉及符号回归作为子模块，可用此 benchmark 评估。不撞车。

### R-Score（Reliability score / 科学可信度）

| R1 Evidence | R2 Independ | R3 Honesty | R4 Reprod | R5 Subst | Tier |
|-------------|-------------|------------|-----------|----------|------|
| 4 | 3 | 4 | 5 | 5 | A (21) |

- **R1 Evidence Grounding (4)**: 所有 claim 可追溯到具体实验 (30 runs, AUC, Pareto fronts)，代码和数据已公开。但能耗数据精度差 (-1)。
- **R2 Verification Independence (3)**: benchmark 自身是评估他人算法的第三方，但 hyperparameter tuning 用的是同一套 pipeline (scikit-learn GridSearchCV)，未引入独立验证者评估 benchmark 设计合理性 (-1)。作者团队与多个被评估算法有关联 (FEAT, ITEA, Ps-Tree 等) (-1)。
- **R3 Honesty (4)**: 诚实披露了能耗测量不精确、时间管理不公、bug 多等局限，且承认 ground-truth 不一定完美。但未报告 negative results 的细节 (如哪些算法在哪些数据集上完全失败) (-1)。
- **R4 Reproducibility (5)**: 全部脚本和数据公开 (GitHub, PMLB)，容器化环境，明确时间/资源约束，符合开放科学最高标准。
- **R5 Substantiation (5)**: 大量真实人力投入 (维护多种算法的 wrapper、修 bug、实际跑 ~1.5 年 CPU 时间的实验)，每个 claim 有独立验证痕迹。benchmark 作为第三方检验工具本身就是 substantiation 的体现。
- **Tier A (21)**: 这是扎实的 benchmark 工作，reproducibility 满分，主要扣分在能耗精度和 fairness of time comparison。

3. **Performance profile 可视化**: 若 FEX 有多种变体（如不同 feature extractor 配置），此可视化方式可展示不同设置下的性能鲁棒性
4. **Pareto front (复杂度 vs 精度)**: FEX 旨在发现可解释的 skeleton，此 trade-off 图天然契合 FEX 的评估需求
5. **能耗监测方法 (eco2AI)**: 若需报告 FEX 的计算开销，可复用

**该引不该引**: 若 FEX 论文在 related work 中讨论到符号回归或方程发现 benchmark，应引用本文及 SRBench 1.0。若 FEX 整体是 feature learning 方法不涉及符号回归，则不必引用。

**差异化思路**: FEX 定位为 skeleton discovery / feature extraction，与纯符号回归的差异在于：(1) FEX 可能先做 representation learning 再输出结构化表示，而非在 raw feature space 上搜索; (2) FEX 的 skeleton 可能是图结构/树结构而非常规数学表达式。这两点都超出了 SRBench 当前覆盖范围，构成差异化叙事。
