# Consistency evaluation of benchmarks used for causal discovery (2606.01789)

- **Year**: 2026.06 (arXiv submission; ACL-style preprint template with `[preprint]` option active — no confirmed peer-reviewed venue found in the source)
- **Authors**: Yuzhe Zhang (Independent researcher, Australia), Chihui Chen (UNSW, Australia), Lina Yao (UNSW, Australia), Chen Wang (CSIRO, Australia)
- **Venue**: arXiv preprint (ACL template)
- **Keywords**: causal discovery, benchmark quality auditing, benchmark consistency, d-separation/conditional independence, LLM-as-judge, domain-knowledge alignment, literature-grounded verification, bnlearn Bayesian-network benchmarks
- **One-liner**: First systematic audit of whether 11 popular causal-discovery benchmark ground-truth graphs (Sachs, Asia, Child, Alarm, …) still align with current domain literature, via an LLM pipeline checking d-separation-implied (in)dependencies against 38,081 retrieved papers.

## 精华提炼

### 管线/方法

This is a pure measurement/auditing paper — no new causal-discovery algorithm is proposed. It has two parts: (A) a bibliometric survey of benchmark *usage*, and (B) a 3-component pipeline that scores benchmark *quality* against domain literature.

**(A) Benchmark usage survey** (Section 2, feeds into which 11 benchmarks get evaluated in part B):
1. Paper selection: search NeurIPS/ICML/ICLR/AAAI/IJCAI/ACL/EMNLP (last 5 years) for keyword "causal"; manually filter to papers proposing *time-independent* causal-discovery methods (excludes causality-in-language / multimodal causality papers); then manually check each paper's experiments and drop pure-synthetic-only papers.
2. Output: 96 papers using ≥1 empirical benchmark, 69 unique benchmarks total, 21 of which appear in ≥3 papers ("popular").
3. Separately, an arXiv-metadata bibliometric pass (not restricted to top venues) searches title/abstract for causal-discovery+LLM keyword pairs, tracking the raw yearly paper count and how many also mention "benchmark" — used only for trend-plotting (Section 2.2), not for benchmark selection.

**(B) Domain-consistency evaluation pipeline** (Section 3 — the paper's actual contribution), run once per (benchmark, variable pair):
1. **Domain research paper retrieval.** Query = synonym-expanded boolean OR/AND over the two variable names (`(Vi syn1 OR Vi syn2 …) AND (Vj syn1 OR Vj syn2 …)`), sent to PubMed (medical/biology graphs) or OpenAlex (other domains). Abstract-level filter keeps only papers whose abstract contains a term from both synonym sets. Open-access PDFs are downloaded and OCR'd to JSON via the Mistral OCR API.
2. **Conditional-association extraction** (Algorithm 1, deterministic, no LLM): for a target pair (X, Y) in the DAG, if adjacent → "always correlated"; else if no collider-free path exists → "mutually independent" (unconditionally); else enumerate all *minimal* separating sets Z (no proper subset of Z also separates X, Y) and emit one d-separation query per minimal Z. A single pair can therefore generate multiple queries.
3. **Consistency verification** (the one LLM step): for each (paper, query) combination, prompt an LLM with the paper's extracted text plus the natural-language query ("Is X and Y mutually independent conditioned on Z?"); the LLM must return strict JSON with `has_data_analysis` (bool), the query answer (`Independent`/`Correlated`/`Unknown`), a short reasoning trace, and a verbatim supporting quote. "Unknown" fires whenever the paper lacks a concrete data-analysis section that answers that specific query.
4. **Aggregation.** Per benchmark: `#Con` = papers whose answer matches the graph-implied relationship, `#InCon` = papers that contradict it, `#Unknown` excluded from the ratio. **Inconsistency rate** `R_InCon = #InCon / (#InCon + #Con)`.

### 关键技术细节

- **LLM verifier**: Qwen3-30B-A3B-Thinking-2507, self-hosted on a single H100 80GB GPU — an open-weight, mid-size reasoning model, chosen explicitly under compute-budget constraints (see 明确局限).
- **Synonym expansion**: medical/biology variables get 15 synonyms merged from the UMLS ontology + ChatGPT; all other domains get 15 synonyms from ChatGPT only.
- **Search budget**: up to 500 most-relevant papers per variable pair; benchmarks with >200 variable pairs are pair-sampled down to 200 (random sampling, no seed reported). Theoretical cap ≈100,000 papers/benchmark; actual total = 38,081 full papers across all 11 evaluated benchmarks.

### 核心结果

- **Headline inconsistency rates** (stated explicitly in text, not read off a figure): Sachs 26.70% (lowest/best), Insurance 31.6%, Ecoli 32.9% — the three most literature-aligned benchmarks. Diabetes 45.8%, Child 54.0%, Asia 57.9% — the three most misaligned. (Cancer, Alzheimer, Arctic Sea Ice, and Alarm's raw pre-cutoff number appear only in a figure, not as text numbers, so are not reproduced here to avoid guessing pixel values.)
- **Motivating anecdote** (Section 1, worked through in detail): the Asia benchmark encodes `Xray ⊥ Bronchitis | Smoking`. Modern deep-learning-based chest-X-ray diagnosis literature (cited: chen2020diagnosis, ntiamoah2021recycling, nishino2014practical) can detect bronchitis from X-ray images, i.e. domain research suggests X-ray and Bronchitis are *not* actually independent given Smoking — directly contradicting the 1988-vintage benchmark graph. Asia is simultaneously the single most misaligned of the 11 evaluated benchmarks (57.9%) *and* one of the three most-used benchmarks in LLM-based papers — the paper's central cautionary example.
- **Construction-year cutoff re-analysis**: recomputing R_InCon using only papers published *after* each benchmark's construction year changes rankings non-trivially. Fmri improves sharply (42.6% → 28.6%, moving into 2nd place), while Alarm gets worse (39.7% → 47.2%). After the cutoff, all pre-2000 benchmarks except Insurance rank last (Diabetes, Alarm, Child, Asia) — age predicts misalignment even after removing papers that predate the benchmark, i.e. it's not just "the benchmark predates the evidence," there is genuine post-construction drift too.
- **Domain field is not predictive**: 9 of 11 evaluated benchmarks are medical/biology; the 2 non-bio benchmarks (Insurance, Arctic Sea Ice) don't cluster at either extreme, so the paper concludes inconsistency correlates with *age*, not *field*.
- **Sachs case study / hypothesis for why it's best**: attributed to its *hybrid* construction — an automated Bayesian-network reconstruction step *followed by* manual domain-expert revision (vs. purely statistical or purely hand-built graphs for the other benchmarks). The paper explicitly recommends this hybrid process as a construction standard going forward.
- **"Growing inconsistency" hypothesis**: raw inconsistent-paper *counts* (not rates) rise for most benchmarks well after their construction year, attributed to later research studying higher-dimensional, more complex scenarios (e.g. previously-latent confounders becoming measured) rather than to the original benchmark being simply "refuted."

### 明确局限

Two limitations stated explicitly by the authors:
1. **LLM verifier accuracy / compute tradeoff** — a small open-weight model (Qwen3-30B) was used purely due to the retrieved-corpus size (up to ~100k papers/benchmark in theory); the authors state a stronger LLM would likely improve accuracy but needs a bigger budget. The reported 90% spot-check accuracy is against only 100 samples out of 38,081 papers processed.
2. **Retrieval efficiency / database coverage** — a single query-expansion pass per database is not guaranteed comprehensive; PubMed works well for biomedical topics but other domains fall back to OpenAlex + institutional subscriptions, and open-access coverage is a hard bottleneck (per the appendix count table, "Unknown" papers vastly outnumber "Correlated"+"Independent"+"Inconsistent" combined for every benchmark, e.g. Alarm: 23,667 Unknown vs. 3,220 decided).

What we additionally read out from the source (not stated as limitations by the authors):
- **No arbitration between "paper is wrong" and "graph is wrong."** A sentence acknowledging exactly this ambiguity ("it does not necessarily conclude whether the paper is wrong or the causal graph is wrong...") exists in the LaTeX source but is commented out (`%`) and does not appear in the compiled paper — the authors flagged this conceptual gap while drafting but never resolved or published it. The inconsistency rate is silently treated as a graph-quality signal throughout, without ever ruling out that some fraction of "inconsistent" papers are themselves the noisy/biased ones.
- **No downstream/consequential validation.** The paper never re-runs a causal-discovery method (LLM-based or statistical) against an "updated" version of a misaligned benchmark to show the misalignment actually changes evaluation outcomes in practice — the misalignment is measured but its practical effect on method rankings is asserted, not demonstrated.
- **Static graphs only, no temporal/regime structure.** The method operates on one fixed DAG per benchmark and a global inconsistency rate; there is no notion of a graph or effect that itself evolves within a single analysis — all "change over time" here is about the literature being newer than the graph, not about anything dynamic inside the causal-discovery process itself.
- **Single verifier, single team, no cross-model check** — one LLM, one lab, no ensemble/majority vote and no adversarial second verifier (see R2 below).

- **R1 Evidence Grounding (4)**: Every headline number traces to a specific, reproducible-in-principle count (papers retrieved, Correlated/Independent/Inconsistent/Unknown tallies per benchmark, in an appendix table that hand-sums exactly to the abstract's 38,081), plus a published external artifact (verifier-accuracy website with raw samples + labels). - **R2 Verification Independence (3)**: Structurally decent — the object being audited (the benchmark DAG) is an external, pre-existing artifact the pipeline does not itself generate, and the literature used to check it is independently retrieved. But the *verifier* is a single self-hosted open-weight LLM with no cross-model or cross-lab check, and its own accuracy was validated by the same authors on a small (100-paper) manual sample rather than an independent auditor.
- **R3 Honesty (4)**: Reports both extremes candidly (Sachs as best, Asia/Child/Diabetes as worst), reports a ranking *reversal* under the construction-year-cutoff re-analysis (Alarm gets worse, not better) rather than only the flattering re-analysis direction, and states both limitations plainly. Docked from 5 because the "is the paper wrong or the graph wrong" ambiguity was drafted (as a LaTeX comment) and then silently dropped rather than being addressed or left as an explicit open question in the published limitations section.
- **R4 Reproducibility (3)**: Full LLM prompt text, the deterministic extraction algorithm (Algorithm 1), exact model/hardware, and exact synonym-expansion recipe are all given in the appendix — strong component-level transparency. But no code repository was found in the source, the 38,081-paper corpus itself can't be redistributed (mixed open-access/subscription), and the 200-pair sampling isn't seeded — so exact end-to-end reproduction is not turnkey even though every component is individually well-specified.
- **R5 Substantiation (4)**: Large, concrete, cross-checkable empirical labor — 96 hand-screened papers across 7 venues over 5 years, 69 benchmarks catalogued, 11 fully evaluated against 38,081 real retrieved papers, an H100 LLM deployment, a manual accuracy audit with a public artifact. Docked from 5 because the practical-impact claim (misalignment matters for evaluation) is argued rather than demonstrated, and several numeric claims live only in figures that could not be cross-verified against text during this read.

### 跟我们的关系

No direct claim overlap with `causal-scs-indicator`. This paper audits whether a *fixed, published* causal-discovery benchmark's ground-truth DAG matches current domain literature; `causal-scs-indicator` (per the current topic file `topics/0710-causal-scs.md` and `workspace/causal-scs-indicator/README.md`) is about whether the *instability of causal-effect estimates across sliding time windows / methods / assumption violations* — not benchmark ground-truth accuracy — can serve as a pre-event diagnostic for severe convective storm onset. The two projects share no claim, no method, and no domain (medical/biology Bayesian-network benchmarks vs. 3km-resolution atmospheric fields). Nothing here should be read as prior art for the dispersion/jump-channel instability indicator itself.

That said, two indirect, worth-citing connections:

1. **Supports our explicit non-goal.** The topic file states the project deliberately does *not* try to "identify the true atmospheric DAG." This paper is a concrete, large-scale empirical demonstration of exactly why that's reasonable: even the most-used, decades-old, hand-curated real-world benchmark graphs (Asia, Child, Alarm) are measurably out of step with their own domain literature, and the paper's own best-case benchmark (Sachs) still shows 26.7% inconsistency. If ground-truth DAGs this well-studied remain contested, treating "ground truth" as the evaluation target for a 3km-resolution atmospheric system (with far less standing per-edge literature than a decades-studied medical benchmark) is materially less defensible — citable motivation for our indicator-not-accuracy framing, not because it's about weather, but because it is the most direct evidence in the current literature that benchmark ground truth itself decays.
2. **Hailfinder is a near-miss, not a hit.** Reference [98] in this paper's own bibliography is *Hailfinder: A Bayesian system for forecasting severe weather* (Abramson et al., 1996) — a Bayes-net benchmark literally built for severe-weather forecasting, one of the 21 "popular" benchmarks this paper's survey identified, but excluded from the actual consistency evaluation purely for size (compute budget, not quality). It is not used by, and shares no method with, `causal-scs-indicator` (which targets real 3km environmental fields and event records, not a hand-built 1996 Bayes net), but it's worth knowing this benchmark exists — and per this paper's own age-correlates-with-inconsistency finding, a 30-year-old severe-weather causal graph would likely score poorly if ever audited the same way.

No tool/method reuse is recommended: the LLM-literature-consistency pipeline depends on a per-variable-pair scientific-literature search returning enough papers to decide consistency (thousands per benchmark here), which has no analogue for physical atmospheric variables (there is no comparable "does X-ray correlate with Bronchitis conditioned on Smoking"-style literature to query for, e.g., CAPE vs. low-level shear). Cite only as motivating evidence for the "ground truth is unstable, so don't chase it" framing, not as a method precedent.

## Read by: 0710-causal-scs (2026-07-11)

This paper does not collide with `causal-scs-indicator` at the claim level — it is a benchmark-ground-truth-quality audit for static Bayesian-network benchmarks (medical/biology domain), while our project is about the temporal *instability* of causal-effect estimates as a pre-event diagnostic signal for severe convective storms. No shared method, no shared domain, no shared evaluation object. The one genuinely useful takeaway is indirect but citable: this paper is the best available empirical evidence that even the most decades-studied, hand-curated real-world causal-discovery benchmarks (e.g. Sachs at 26.7% inconsistency, its *best* case) remain measurably misaligned with their own domain literature, and that misalignment gets worse with benchmark age. That is a clean, literature-grounded justification for the topic's explicit non-goal of "identifying the true atmospheric DAG" — if this well-funded, single-domain audit still finds pervasive ground-truth drift after decades of medical/biology research, treating recovery of a "true" atmospheric DAG as the target for a comparatively younger, higher-dimensional 3km weather dataset would be on much weaker footing. Secondarily, the paper's own reference list surfaces Hailfinder (Abramson et al. 1996), a severe-weather-forecasting Bayes net — not usable by this project directly, but useful background if a related-work section ever needs to acknowledge prior probabilistic-graphical-model work on severe weather. No LLM-literature-consistency-pipeline reuse is recommended: that pipeline needs a per-variable-pair scientific literature corpus to query, which has no analogue for physical atmospheric variables.
