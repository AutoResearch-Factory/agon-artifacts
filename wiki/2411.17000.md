## 精华提炼

### 管线/方法
1. 数据构建：TERRA-MODIS Level-1B (MOD02) 5分钟 swath → 逐日全球合成（约270天/年 × 10年，2000.03至今）→ 1km分辨率128×128像素图像块。14个MODIS波段（波段1,2,3,6,7,21,26,27,28,29,30,31,32,33）覆盖可见光-近红外-热红外，且特意选择为与GOES-ABI 16通道中14个光谱最接近的对应波段，为下游跨传感器迁移预先铺路。
2. 预处理：250m/500m原生分辨率波段用Elliptical Weighted Averaging (EWA)聚合到1km；重叠观测取最低view zenith angle以降低几何畸变；只保留solar zenith angle<72°的像元（排除低照度）。反射率波段：DN→反射率（线性offset+scale×100）；热红外波段：DN→radiance→Planck方程反演亮温→温度校正（tcs斜率/tci截距）。反射率与亮温分别独立归一化到[0,1]（反射率×0.01；亮温做全局min-max）。
3. 采样策略：非纯地理随机采样，而是基于云/地表覆盖类型聚类采样，四象限（NW/NE/SW/SE）分层，覆盖海洋、全谱系陆地覆盖类型、云与非云样本，避免稀有覆盖类型欠采样、常见类型过采样。
4. 预训练：SimMIM框架 + SwinV2 Giant架构（30亿参数）。输入128×128×14图像块，8×8 patch随机遮蔽（图2重建示例标注遮蔽60%，正文未单独给出统一mask ratio数值），模型预测被遮蔽区域原始像素值，window size=12，训练50 epoch，共100M图像块。
5. 下游任务（3D云retrieval）：把预训练好的SatVision-TOA backbone迁移到GOES-ABI观测（不同传感器、不同平台），接FCN decoder，预测CloudSat/CALIPSO在卫星轨迹curtain上的垂直云掩膜。Baseline是同一FCN decoder但encoder从零训练；两者共享decoder架构与训练数据（7000训练/1300验证ABI图像块），唯一变量是encoder是否经过MODIS预训练。

### 关键技术细节
- 模型规模实验：SwinV2 Huge（6.58亿参数）与SwinV2 Giant（30亿参数）两种backbone；预训练数据规模2M/26M/100M三档，构成部分组合的scaling研究（细节在Appendix Model Configuration Comparison及三张表）。
- 硬件：NASA NCCS Discover超算（SCU17分区，40核AMD Rome CPU + 4×NVIDIA A100-40GB）用于2M/26M规模实验；Oak Ridge Frontier超算（64核AMD EPYC + 4×AMD Instinct MI250X）用于100M规模的最终Giant模型。总计68,329 GPU小时（7.8 GPU年）。
- 预训练优化：fused AdamW（β1=0.9, β2=0.99）+ OneCycle余弦学习率调度；100M配置峰值学习率0.00159727，有效batch size 16,384（跨256 GPU，DeepSpeed ZeRO分布式）；bfloat16混合精度。
- 下游微调：4×NVIDIA A100，输入128×128×14，有效batch size=8，AdamW，线性warmup（5e-7→3e-4峰值）。
- 评测指标：图像重建用SSIM（而非MSE，因为SSIM衡量结构相似性、对边缘/纹理保真更敏感）；3D云retrieval用mIOU（主指标，因正负样本类别不平衡）+ pixel accuracy + AUC（ROC曲线下面积）。
- 数据体量：100M图像块，覆盖10年以上MODIS记录，总计超过100TB原始数据。

### 核心结果
- 图像重建（128个留出图像块测试集，随机遮蔽）：100M配置SSIM=0.9289，为四种（Huge/Giant × 2M/26M/100M）配置中最高；scaling单调递增：Huge-2M(0.9032) < Giant-2M(0.9084) < Giant-26M(0.9115) < Giant-100M(0.9289)。
- 3D云retrieval（ABI跨传感器迁移，CloudSat/CALIPSO curtain标签）：SVTOA-Giant-100M达mIOU=0.4638、accuracy=95.74%、AUC=0.98，对比FCN-baseline（从零训练）mIOU=0.2185、accuracy=93.20%、AUC=0.93——mIOU绝对提升0.245（相对提升约112%）。摘要另提及false negative rate相对baseline降低超过50%（未在正文表格中单列具体数值）。
- Scaling配置对比（Appendix Table 3）：mIOU随规模单调提升 FCN-baseline(0.2185) < SVTOA-Giant-2M(0.3695) < SVTOA-Huge-2M(0.3838) < SVTOA-Giant-26M(0.4144) < SVTOA-Giant-100M(0.4638)；一个具体的、反直觉的细节是2M规模下Huge（6.58亿参数）反而略优于Giant（30亿参数）（0.3838 vs 0.3695），说明小数据量下更大模型未必占优，Giant的规模优势要到100M数据量级才充分体现。
- 逐通道SSIM（Appendix Table 2）显示100M配置在几乎全部14个通道上都优于或持平26M/2M配置，但个别通道在26M配置出现局部非单调下降（如Ch28: 2M=0.94 → 26M=0.87 → 100M=0.95），论文原样列出未作平滑或回避处理。

### 明确局限
- 只用白天Terra-MODIS（约9AM当地时间过境）训练，模型未见过夜间或其他当地时间的数据，作者明确承认这会限制模型在涉及**昼夜变化**（diurnal variation，如云生命周期）的下游任务中的能力。
- 跨传感器迁移（MODIS→ABI）未做任何显式的辐射定标或图像重投影以匹配MODIS footprint，只是挑选14个光谱最接近的ABI通道直接输入；下游ABI训练/测试样本被迫限定在1:30PM当地时间（受CloudSat/CALIPSO太阳同步轨道约束），这一限定本身正是为了规避作者在Objectives中承认的"MODIS只在9AM训练但ABI覆盖15分钟刷新的完整昼夜循环"这一失配，属于用样本选择绕开问题而非解决问题。
- 下游任务只验证了一个（3D云retrieval），且只对比一个baseline（同结构FCN从零训练），没有与其他同类地球观测基础模型（SatMAE、Prithvi-HLS、Scale-MAE，均在Intro中被提及作为"高分辨率cloud-free局限"的反例）做同任务横向对比。
- 图像重建评测集仅128个留出图像块，下游训练/验证集仅7000/1300个ABI图像块，规模较小；正文和附录均未报告任何置信区间、方差或多次重复实验结果，全部数字都是单次运行的点估计。
- 论文以类似workshop/技术报告的短篇格式呈现（NeurIPS风格`arxiv.sty`模板，无独立Related Work小节，仅在Intro中穿插相关工作），截至本次核查未确认是否已在同行评审venue正式发表（OpenAlex/S2元数据均未返回journal-ref）。
- 与我们课题的本质差异：SatVision-TOA处理的是MODIS**传感器辐射/反射率原始图像**（14光谱通道，单一2D空间快照，无气压层结构，无时间维度），而非causal-scs-indicator项目所需的HRRR**再分析衍生大气状态变量**（U/V/T/RH/omega/Z，4个气压层，96小时历史，`[24,96,64,64]`张量）。SatVision-TOA的MIM预训练目标仅做**单帧空间掩码重建**，完全不建模时序动态。

### 跟我们的关系
- **撞车判断：不撞车**。SatVision-TOA是纯粹的计算机视觉/遥感图像基础模型论文（MODIS TOA辐射图像的自监督MIM预训练 + 跨传感器3D云retrieval微调），不涉及因果推断、时序不稳定性、PCMCI+/LKIF或任何形式的"预警指标"构造，与causal-scs-indicator"用因果效应估计的不稳定性作为强对流预警指标"这一核心claim在方法论、数据类型和研究问题上完全无重叠。
- **数据类型的本质差异**：项目proposal v2明确将输入定义为HRRR**再分析衍生的大气状态变量**（`[24,96,64,64]`张量：U,V,T,RH,omega,Z共6变量×4气压层×96小时历史×64×64空间网格）；而SatVision-TOA的输入是MODIS**原始传感器辐射/反射率图像**（14光谱通道×128×128空间×单一时刻快照，无气压层结构，无时间维度）。这是"大气状态场"（NWP/reanalysis state）与"卫星传感器观测"（satellite sensor imagery）两条不同谱系的基础模型——proposal review已明确排除的HRRRCast/StormCast/Stormscope属于前一谱系但因分辨率/气压层/时序结构不匹配被拒；SatVision-TOA连"状态场"这个前提都不满足，属于相关性更弱的第二谱系（遥感影像基础模型，与SatMAE/Prithvi-HLS/Scale-MAE同类）。
- **对modernization讨论的间接佐证、但非正面证据**：proposal v2的review指出当前设计的自监督预训练目标是"纯reconstruction"（SimMIM式掩码重建），被reviewer批评"容易靠局部空间插值解出，不强迫encoder学习与危险演化相关的时序动力学"，建议改为Prithvi WxC式的masked+forecasting混合目标。SatVision-TOA恰好是"reconstruction-only"路线的一个公开案例——其MIM预训练本身只做单帧空间掩码重建，不含时间维度，达到了不错的下游精度提升，但这不能反驳reviewer的modernization建议：SatVision-TOA的下游任务（单帧3D云结构反演）本身也不要求时序推理能力，二者是"目标与任务自洽"而非"reconstruction-only目标足以支撑需要时序理解的任务"的反例。
- **不该引用为方法学基础或backbone候选**：SatVision-TOA的SwinV2骨架和MIM预训练框架均为单帧2D多光谱图像设计，不支持项目所需的24通道×96小时4D张量输入；采用它需要与HRRRCast/StormCast同等量级的架构改造（新增时序聚合模块），而proposal已明确把这类改造排除在"intentionally excluded"之列（不新增可学习融合权重、不新增transformer/DiT骨干）。
- **可能值得留意的旁支线索**：反引文中的MAESTRO（多时相MAE）和HighFM（高频地球观测基础模型）是SatVision-TOA之后明确往"时序"方向扩展遥感基础模型的后续工作。若未来课题真的转向"大规模气象基础模型预训练"路线（目前v10/v2均未采纳），这两篇比SatVision-TOA本身更值得优先查看，因为它们已经在处理时序输入。

## Read by: 0710-causal-scs (2026-07-13)

本文（SatVision-TOA）是NASA团队的MODIS TOA辐射图像基础模型：SwinV2 Giant（30亿参数）+ SimMIM掩码重建预训练于1亿张单帧图像块，跨传感器迁移到GOES-ABI做3D云垂直结构retrieval，mIOU相对FCN baseline几乎翻倍（0.4638 vs 0.2185）。

- **撞车评估**：无撞车风险。本文不涉及因果推断、时序不稳定性或预警指标构造，与causal-scs-indicator的核心claim（因果效应估计不稳定性`(D,J)`能否为强对流预警提供增量信息）在方法、数据、任务上完全不重叠。
- **数据谱系差异**：本文处理MODIS原始传感器辐射图像（单帧、14光谱通道、无气压层、无时间维度），项目需要的是HRRR再分析大气状态变量（96小时历史、4气压层、6变量）——两者属于地球观测基础模型的不同分支，前者与SatMAE/Prithvi-HLS/Scale-MAE同类，后者与Prithvi WxC/Aurora/HRRRCast同类，项目proposal已确认自己站在后一分支。
- **对v2 review的modernization讨论有间接参考价值，但不是可直接采纳的正面案例**：v2 review批评当前self-supervised目标"纯reconstruction、不强迫学时序动力学"，SatVision-TOA是reconstruction-only路线的真实公开案例，其成功建立在下游任务本身不需要时序推理的前提上——不能用来反驳review建议改用masked+forecasting混合目标（Prithvi WxC路线）。
- **不建议采纳为backbone**：SwinV2+MIM框架仅支持单帧2D多光谱输入，接入项目的24通道×96小时4D张量需要与HRRRCast/StormCast同等量级的新架构改造，而v2 proposal已明确排除"新增transformer/DiT骨干"和"新增可学习融合权重"。
- **结论**：本文对causal-scs-indicator当前v2阶段（Gate 0/Pre-P2/P2三个门控）没有直接方法或数据可借鉴之处，仅作为"reconstruction-only MIM在遥感基础模型中的真实工程案例"背景知识记录；其反引文中的MAESTRO、HighFM等时序扩展工作比本文更值得在未来"气象基础模型预训练"路线被激活时优先查看。
