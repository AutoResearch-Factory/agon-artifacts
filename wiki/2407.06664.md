# PDEformer-1: A Foundation Model for One-Dimensional Partial Differential Equations (2407.06664)

- **Year**: 2024.07 (v2: 2025.01)
- **Authors**: Zhanhong Ye (PKU), Xiang Huang (Huawei), Leheng Chen (PKU), Zining Liu (PKU), Bingyang Wu (PKU), Hongsheng Liu (Huawei), Zidong Wang (Huawei), Bin Dong (PKU, New Cornerstone)
- **Venue**: arXiv preprint (Under review)
- **Keywords**: PDE foundation model, graph Transformer, implicit neural representation, computational graph, neural operator, zero-shot inference, inverse problem, diffusion-convection-reaction
- **One-liner**: 用计算图表示 PDE 符号形式 + Graphormer 编码 + Poly-INR 解码，在 300 万个 1D PDE 样本上预训练出一个通用神经求解器，零样本推理精度超过专门训练的 FNO/DeepONet。

## 精华提炼

### 管线/方法
**输入 → 处理 → 输出 pipeline：**

1. **PDE 形式 → 计算图（Graph Construction）**：将 PDE 的符号形式转化为有向无环图（DAG），节点类型包括 UF（未知场变量）、SC（标量系数）、CF（系数场）、IC（初始条件）、|xL/|xR（边界值）、dt/dx（微分算子）、+、×、−（一元取反）、sin/cos、=0 等。边为同质有向边，表示操作数关系。

2. **数值信息嵌入 → 节点输入特征**：SC 和边界值节点 → Scalar Encoder（3层 MLP，256 hidden）；函数（IC/CF）→ Function Encoder（类 DeepSet + softmax 加权的 permutation-invariant 网络，4 branch nodes，每次采 256 个散点）；其余节点 → Scalar-Encoder(0)。

3. **图编码（Encoding）**：Graphormer（基于 Ying et al. 2021 修改版）处理计算图。关键修改：(a) 去掉了原版 Graphormer 的 edge encoding（因为边同质）；(b) 增加 disconnected node pair masking（d_ij = -∞ 当节点间无路径相连接，实验验证此 masking 提升精度）；(c) 最短路径距离编码 attention bias（截断到 14）。对每个 UF 节点，额外连接 L 个 m_ℓ 节点，用于收集各层的 latent code µ_ℓ。

4. **解码预测（Decoding）**：采用改良版 Poly-INR（Singh et al. 2023b），L=8 层 hidden layers，每层 256 神经元。每层由对应的 µ_ℓ ∈ R^de 通过两个独立的 3 层 MLP hypernet 生成 scale 和 shift modulation。激活函数：leaky-ReLU (slope 0.2) + 裁剪到 [-256, 256]。最终线性层输出解 u(t,x)。整个预测是 mesh-free 的——可在任意 (t,x) 坐标查询。

### 关键技术细节

- **模型规模**：4 种配置 (S/M/L/XL)，参数量分别为 3.60M/6.92M/22.4M/58.2M。核心差异在 Graphormer 层数/宽度和 INR 层数/宽度。
  - XL：Graphormer 12层、embed_dim 768、32 heads；Poly-INR 12层、dim_hidden 512
- **预训练数据**：300 万样本，扩散-对流-反应方程家族 (方程1)。每个系数 c_ik 以 0.5 概率置零，否则从 U([-3,3]) 采样。κ(x) 限制在 [10^-3, 1]。周期/非周期各半。边界条件随机选择 Dirichlet/Neumann/Robin。
- **数值求解器**：Dedalus V3（谱方法），256 空间网格点，δt_solver = 4×10^-4，δt_data = 0.01（101 个时间步）。丢弃 ∥u∥_L∞ > 10 的异常样本。
- **训练设置**：XL 模型在 48 NPU 上训练 125 小时，batch_size=640，Adam optimizer，lr=0.0001，cosine schedule + 10 epoch warmup，500 epochs。Loss 函数为 nRMSE = ∥u-û∥_L2 / ∥u∥_L2。
- **Finetuning**：用较少数据（100~9k 样本）在目标 PDE 上微调，使用相对 L2 error 作为损失。
- **推理速度**：单样本 23.70ms（1 NPU），对比 FNO 2.74ms、DeepONet 7.41ms、U-Net 5.17ms、传统 FD 2072ms、Dedalus 411ms。

### 核心结果

| 实验 | PDEformer-1 Zero-Shot | PDEformer-1 Finetune | 最佳 Baseline |
|------|----------------------|---------------------|--------------|
| Burgers ν=0.1 (InD) | 0.0061 | 0.0034 | 0.0171 (FNO) |
| Burgers ν=0.01 (InD) | 0.0180 | 0.0119 | 0.0492 (FNO) |
| Advection β=0.1 (InD) | 0.0120 | 0.0062 | 0.0125 (FNO) |
| Burgers ν=0.001 (OoD) | 0.0388 | 0.0265 | 0.0727 (FNO) |
| Advection β=1.0 (OoD) | 0.9036 | 0.0099 | 0.0152 (FNO) |
| Reaction-Diffusion (OoD) | 0.0377 | 0.0009 | 0.0019 (DeepONet) |

- **Scaling law**：Test-Error ∝ Data-Amount^(-α)，Test-Error ∝ Training-Time^(-β)，呈现清晰的幂律关系。
- **Few-shot learning**：仅用 100 个 OoD 样本 finetune 即可超越所有从零训练的专门模型。
- **Finetuning 速度**：~100 次迭代即接近收敛，而 FNO 从零训练 1000+ 次迭代仍误差更高。预训练过的 FNO (FNO-FT) 迁移效果远不如 PDFormer-1。
- **结构泛化 (Trigonometric)**：在 J_max=4 的数据上训练后，能零样本泛化到 J=5 的未见 PDE（nRMSE 5.05%），表明模型学到了求和操作的语义含义。
- **新算子泛化 (Wave Equation)**：在预训练数据不含二阶时间导数的情况下，finetune 后达到 nRMSE 5.1%（周期边界）/ 4.6%（非周期边界）。
- **逆问题**：使用 PSO 优化 + PDEformer-1 作为 forward surrogate，成功恢复标量系数（含 10% 观测噪声）、完成系统辨识（7 个系数）、恢复源场 s(x)（梯度下降）、恢复波速场 c(x)（3 种 Lu 形式）。

**反直觉发现**：Advection β=1.0 的零样本推理相对 L2 error 高达 0.9036（几乎无用），但仅需 100 个样本 finetune 后降到 0.0099——说明预训练模型学到了有用的 representation，只是数值范围偏移太大导致零样本失效，而非架构根本局限。

### 明确局限

1. **仅限 1D**：当前仅在 1D 时间依赖方程上验证，实际科学/工程问题多为 2D/3D，作者明确承认"practical value is still relatively limited"。
2. **仅限单变量**：虽然架构理论上支持多变量 PDE 系统，但所有实验仅涉及单未知场变量 u。
3. **方程形式受限**：预训练只覆盖扩散-对流-反应类型的多项式非线性（至多三次方）。三角/波方程需要 finetune 才能处理。
4. **无真机对比**：对比仅限神经网络方法（FNO/DeepONet/U-Net）+ 传统数值方法（FD/Dedalus），没有与工业级 PDE 求解器或真实物理实验对比。
5. **逆问题依赖 PSO**：系数恢复使用 PSO（非梯度优化），每次需要大量前向推理调用，缺乏端到端的逆问题方法。
6. **数据依赖 Dedalus**：预训练数据完全依赖谱方法求解器生成，对于 Dedalus 无法求解的 PDE（会被丢弃），模型也无法学习，存在系统性偏差。
7. **无不确定性量化**：虽然提到了未来方向，当前版本无 UQ 能力。
8. **计算图表达能力边界**：缺少除法、一般幂运算、指数/对数等节点（虽声称可扩展，但未实验）。
9. **跟我们的关系（0616-fex）**：PDEformer-1 是 PDE 基础模型赛道的重要 baseline。其核心思路（计算图 + Graphormer）与我们可能探索的方向（如 FEX 在 PDE 求解中的符号回归/表达式发现）具有互补性——PDEFeformer 做 forward solving，FEX 可能做符号发现/方程还原。

### 跟我们的关系

对 **0616-fex** 课题：
- **不撞车**：PDEformer-1 的核心是 forward PDE solving（给定方程形式+数值信息 → 预测解），而 FEX 的核心通常是符号回归/方程发现（给定观测数据 → 推断方程形式）。两者方向正交互补。
- **可用的工具/方法**：
  - **计算图表示**：PDEFeformer 的 PDE→DAG 表示方法可以直接用于将 FEX 发现的候选方程形式化为结构化输入。
  - **Graphormer 架构**：其图编码方式对有向无环的数学表达式编码效果很好，可用于 FEX 的表达式编码/评估。
  - **预训练模型作为 surrogate**：如果将 FEX 用于 PDE 发现，PDEFeformer 可作为快速 forward evaluator 来评估候选方程的拟合质量，替代传统数值求解器的巨大计算开销。
  - **Poly-INR decoder**：其 mesh-free 输出特性使得可以在任意观测点评估解，适合与稀疏/非均匀观测数据对接。
- **该引**：如果 0616-fex 工作涉及 PDE 符号发现并用神经网络做 forward model，PDEFeformer-1 是必引的前置工作。
- **差异化**：PDEFeformer 是"给定方程求解"，FEX 是"从数据发现方程"。可以结合——用 FEX 发现方程结构，用 PDEFeformer 架构做方程表示和快速验证。注意 PROSE（Liu et al. 2023）已经做了部分符号表达式预测的工作，需要仔细区分。

## Read by: 0616-fex (2026-06-24)

### 撞车评估
**不撞车。** PDEformer-1 的方向是 "给定 PDE 形式 → 预测解"（forward PDE solving），而 0616-fex 的方向更接近符号回归/方程发现（"给定观测数据 → 推断 PDE 形式"）。两者是同一个大问题（AI for PDE）的正反两个方向，互补而非竞争。

### 可用工具
1. **PDE 计算图表示**：PDEFeformer 的 DAG 表示方法 (Section 3.2 + Appendix A) 是一种结构化的 PDE 编码方式，可以直接用于表示 FEX 发现的候选方程，实现与 forward solver 的无缝对接。
2. **Pre-trained PDEFeformer-1 as forward surrogate**：如果 FEX 在搜索方程空间时需要反复评估候选方程与观测数据的拟合度，用 PDEFeformer-1 替代传统数值求解器作为 forward model 可以大幅加速（23ms vs 410ms per sample）。
3. **Graphormer for expression encoding**：其图 Transformer 对有向无环数学表达式的编码能力已经验证有效，可以直接复用到 FEX 的表达式特征提取中。

### 差异化策略
- 本文 PDEFeformer-1 解决的是 **"求解"**（solve），0616-fex 应聚焦 **"发现"**（discover）。
- PROSE (2309.16816) 是 PDEFeformer 引用的最接近 FEX 的工作（预测算子和符号表达式），需要重点与 PROSE 区分——FEX 的优势可能在更显式的符号搜索（如遗传编程/强化学习搜索），而非端到端神经网络预测。
- 一个自然的结合点：**FEX 发现方程结构 → PDEFeformer 的 DAG 表示做验证 → PDEFeformer 预训练模型做快速 forward evaluation**。
