# Alleviating Overfitting in Transformation-Interaction-Rational Symbolic Regression with Multi-Objective Optimization (2501.01905)

- **Year**: 2025.01
- **Authors**: Fabricio Olivetti de Franca, Federal University of ABC
- **Venue**: arXiv preprint
- **Keywords**: symbolic regression, genetic programming, Transformation-Interaction-Rational, multi-objective optimization, NSGA-II, overfitting, SRBench, Pareto model selection
- **One-liner**: 用 NSGA-II 把 TIR 符号回归改成误差和表达式大小双目标，主要缓解小样本过拟合。

## 精华提炼

### 管线/方法

1. 论文研究对象是 Transformation-Interaction-Rational, 即 TIR 符号回归表示。
2. 输入是标准表格回归数据集，每个样本有 d 个变量和一个连续目标值。
3. TIR 把模型限制为一个有理形式：先构造两个 Interaction-Transformation 表达式 p 和 q。
4. 每个 IT 表达式是多个 transformed interaction terms 的线性组合。
5. 单个 interaction term 是变量幂次的乘积，形如 prod x_i^{k_ij}。
6. 每个乘积项再经过一元 transformation function, 如 id, tanh, sin, cos, log, exp, sqrt。
7. TIR 的整体形式是 g(p(x) / (1 + q(x)))，其中 g 是可逆函数。
8. 这个表示的核心偏置是减少任意树状 GP 的搜索空间，避免深层非线性函数链。
9. 数值系数不是靠随机常数在树里漂移，而是通过变换目标后的 OLS 拟合。
10. 对 noiseless 训练目标，作者把 y = g(p/(1+q)) 重排为 g^{-1}(y) = p - g^{-1}(y) q。
11. 这样 p 和 q 中的线性系数可以合并到一个线性最小二乘问题里求解。
12. 论文的问题起点是旧版 TIR 在 SRBench 小数据集上会出现训练 R2 很高、测试 R2 很差的过拟合。
13. 旧版缓解方法是在小数据集上给 fitness 加表达式大小惩罚：f'(x)=f(x)-c*l(x)。
14. 旧版经验参数是 c=0.01，且只在某些小数据启发式触发时启用。
15. 小数据启发式包括 samples <= 100, features <= 6, 以及 samples * features <= 1000。
16. 作者认为这些启发式和惩罚系数本质上是额外超参数，会增加调参负担。
17. 本文的核心改动是把 TIR 的搜索目标改成 multi-objective optimization。
18. 第一个目标是最大化验证或测试相关的 R2 accuracy。
19. 第二个目标是最小化表达式树节点数，作为 simplicity 或 parsimony proxy。
20. 多目标选择使用 NSGA-II 的 fast non-dominated sorting 和 crowding distance 思路。
21. 父代选择从普通 tournament 改为 crowding tournament。
22. reproduction 从单目标 generational/tournament selection 改为对父代和 offspring 合并后做 non-dominated sorting。
23. 若 Pareto front 超过种群容量，则使用 crowding distance 保持多样性。
24. 论文直觉是：在同等 accuracy 下，较小表达式会支配较大表达式，因此 Pareto 压力会自然实现 Occam bias。
25. 作者测试了四个变体。
26. TIRMOO 是纯多目标版本，最后返回 Pareto front 上最准确的模型。
27. TIRMOO-points 是多目标版本，但在 samples * features <= 1000 的数据集上使用 penalized accuracy objective。
28. TIRMOO-Select 返回 Pareto front 上 accuracy 达到最佳 accuracy 95% 以内的最小模型。
29. TIRMOO-Sel-points 只在小数据集上采用 TIRMOO-Select 的选择策略，其他数据集返回最准确模型。
30. 关键点是论文把“搜索时产生 Pareto front”和“最后选哪个模型”区分开。

### 关键技术细节

1. 初始种群中的每个个体表示为三元组 (g, p, q)。
2. g 从可逆函数集合 G 中随机选取。
3. p 是非空 IT expression，q 可以为空。
4. 每个 IT expression 通过反复随机生成 term 直到空 term 或 token budget 用尽。
5. 生成一个 term 时，从尚未选择的变量集合中无放回抽样变量，或以 1/(d+1) 概率停止。
6. budget 设置为 max(5, min(15, floor(n_samples/10)))。
7. 种群大小固定为 1000。
8. 进化代数固定为 500。
9. crossover probability 是 30%。
10. mutation probability 是 70%。
11. transformation functions 是 [id, tanh, sin, cos, log, exp, sqrt]。
12. invertible functions 是 [id, atan, tan, tanh, log, exp, sqrt]。
13. error measure 使用 R2。
14. k_ij 幂次范围只搜索两组超参：(-5, 5) 和 (0, 3)。
15. crossover 会先在第一个 parent 中随机选交叉点。
16. 若交叉点在 g/root, child 继承第一个 parent 的 g,p 和第二个 parent 的 q。
17. 若交叉点在 p, child 继承第一个 parent 的 g,q，并混合两个 parent 的 p。
18. 若交叉点在 q, child 继承第一个 parent 的 g,p，并混合两个 parent 的 q。
19. mutation 在 insert-node, remove-node, change-var, change-exponent, change-function 中均匀选择。
20. 接近 budget limit 时不使用 insert-node。
21. 单 term 特殊情形下不使用 remove-node。
22. 实验基准是 SRBench 的 122 个真实回归问题。
23. 其中 62 个是 Friedman benchmark 的变体。
24. 每个算法每个数据集运行 10 个固定随机种子。
25. 训练测试划分比例是 0.75/0.25。
26. 如果训练集超过 10000 个样本，则 subsample 训练数据。
27. 超参选择使用 halving grid search。
28. grid search 使用 5-fold cross-validation。
29. SRBench 规则限制最多 6 组用户定义超参，本文 TIRMOO 只用 2 组 k_ij range。
30. 结果统计主要看 test set R2，用每个数据集的 median 汇总，并用 1000 次 bootstrap 估计 0.95 confidence interval。
31. 统计比较还用了 Nemenyi test 的 critical difference diagram，alpha=0.05。
32. 作者还报告 top-5 rank frequency histogram。
33. 运行环境是 Intel DevCloud, Intel Xeon Gold 6128 CPU @ 3.40GHz, single thread。
34. 原 SRBench 其他算法结果来自公开 repository，原始实验 CPU 是 Xeon E5-2690 v4 @ 2.60GHz，因此 runtime 对比有硬件偏差。

### 核心结果

1. 全 benchmark 上，TIRMOO 和 TIRMOO-Sel-points 的 median R2 约为 0.95。
2. 这把它们放在 Operon 和 SBP-GP 附近。
3. Operon 仍然整体最强，主要优势来自标准树 GP 加 nonlinear parameter optimization。
4. Friedman 子集上，Operon R2 高于 0.95，TIRMOO, TIRMOO-Sel-points, TIRMOO-points 接近该区间。
5. Friedman 子集决定了整体 rank 的很大部分，因为 122 个数据集中 62 个是 Friedman 变体。
6. non-Friedman 子集上，top-15 算法差距很小，error bar 大量重叠。
7. points 小数据集子集上，top-15 差距同样不明显。
8. 小数据集上，带简化机制的 TIR/TIRMOO 变体 error bar 更窄，说明结果变异更小。
9. plain TIRMOO 没有进入小数据集 plot 的重点队列，说明“Pareto front 上取最准确模型”不足以缓解小样本过拟合。
10. TIRMOO-Sel-points 是作者认为最好的折中方案。
11. 它说明模型选择策略本身很关键，不只是搜索过程中是否用了 Pareto front。
12. 在 critical difference diagram 中，overall 结果里 Operon 和 SBP-GP 没有显著差异。
13. TIRMOO, TIRMOO-Sel-points, TIRMOO-points 与 SBP-GP 也没有显著差异。
14. Friedman 子集 top-5 的 average rank 约在 2 到 6 之间，明显优于后部算法。
15. non-Friedman 和 points 子集 top-15 average rank 约在 9 到 14 之间，算法间没有显著差异。
16. Kruskal-Wallis 和 Mood's median test 也没有发现这些选择中的 top-15 有显著差异。
17. top-5 frequency 上，Operon overall 接近 70% 数据集进入 top-5。
18. TIRMOO-Sel-points 和 TIRMOO overall 约 55% 数据集进入 top-5。
19. Friedman 数据上，Operon, TIRMOO-Sel-points, TIRMOO 分别约 88%, 68%, 65% 进入 top-5。
20. non-Friedman 和小数据上，TIRMOO 最好变体 top-5 频率降到约 35% 和 25%。
21. 单目标 TIR-points 在 non-Friedman 上约 42% top-5，与 Operon 持平。
22. TIR-points 在小数据上约 48% top-5，优于 Operon 的约 35%。
23. size comparison 显示 TIRMOO-Select 的 median model size 约为其他 TIR/TIRMOO 变体的一半。
24. points 小数据集上，penalization 能把 TIR/TIRMOO 的 median size 降到约一半。
25. TIRMOO-Select 和 TIRMOO-Sel-points 的选择策略能比 TIR-points 维持更小的 median model size。
26. runtime 上，MOO 版本的 median runtime 接近单目标版本，但作者提醒 MOO 只用了 2 组超参，单目标用了 6 组。
27. 如果 MOO 用同样大小的 grid search，runtime 预计约 3 倍。
28. 示例 687_sleuth_ex1605 中，TIRMOO 返回很复杂的有理表达式，train R2=0.87, test R2=-0.43。
29. 同一数据上 TIRMOO-Select 返回较简单表达式，train R2=0.74, test R2=0.20。
30. TIRMOO-points 返回更简单的 tanh 表达式，train R2=0.64, test R2=0.53。
31. 这个例子支持“最准确 front 成员可能仍过拟合，稍低训练精度的小模型泛化更好”。
32. hypervolume 表显示 TIRMOO front 的 hypervolume 通常较高。
33. 对多个示例数据集，TIRMOO-points 的 hypervolume 低于 TIRMOO。
34. 作者解释为 penalization 会降低 accuracy objective 的 front quality。
35. surprising 结果：多目标搜索本身不是银弹，最后选择 front 上哪个模型比是否生成 front 更影响小数据泛化。
36. surprising 结果：更强的 penalization 可能在小数据集更有效，但会牺牲 Pareto front 的 hypervolume。

### 明确局限

1. 论文没有提出新的符号回归表示，核心增量是在 TIR 搜索上接入 MOO 和 front selection。
2. 没有证明 NSGA-II 一定比 penalized single objective 更好；小数据集提升很小且统计上不强。
3. 作者明确说小数据集上差异 barely noticeable，只是 TIRMOO-Sel-points 略有优势。
4. 论文依赖 SRBench 的固定 protocol，真实科学建模场景中的 extrapolation 需求未单独评测。
5. 复杂度只用 expression tree node count 作为 proxy，没有衡量线性性、平滑性、单调性、物理合理性或 MDL。
6. TIR 的表达空间受限，优点是可解释和低复杂度，缺点是可能漏掉需要函数嵌套或非 TIR rational form 的真实规律。
7. TIRMOO-Select 的 95% threshold 仍然是一个人为启发式，只是从 fitness 惩罚转移到了 model selection。
8. points heuristic 即 samples * features <= 1000 仍是经验规则。
9. 论文没有把 AIC, BIC, MDL 等模型选择准则纳入实验，只作为 future work。
10. 与所有 SRBench 比较一样，多个 Friedman 变体可能放大某类算法优势。
11. runtime 比较受硬件和 grid size 不一致影响，不宜过度解读。
12. 没有报告完整独立复现实验，只说明结果和处理脚本公开在 GitHub。
13. 没有使用外部真实科学方程恢复 benchmark 来证明发现的表达式有物理解释价值。
14. 论文对失败案例有说明，但没有系统分析哪些数据属性导致 TIRMOO 失败。
15. 如果 0616-fex 关注的是自动发现 feature/expression，而不是传统 tabular symbolic regression，则这篇只能提供搜索和模型选择经验，不能直接覆盖任务定义。

### 跟我们的关系

1. 对 `0616-fex` 的撞车风险取决于 FEX 的实际定义。
2. 如果 FEX 是 feature/expression extraction 或 formula/expression discovery，这篇与“用符号回归发现可解释表达式”高度相关。
3. 直接撞车的 claim 是：用多目标优化同时追求预测误差和表达式简洁度，以缓解小样本符号回归过拟合。
4. 另一个撞车点是：最后不应无条件选 Pareto front 上 accuracy 最高的表达式，而应按数据规模和不确定性选择更小模型。
5. 不撞车的地方是：本文限定在 TIR rational symbolic regression 表示，且 benchmark 是 SRBench tabular regression。
6. 如果我们的 FEX 要做 LLM-assisted equation discovery、feature construction、科学假设生成或神经符号 pipeline，本文只是相关 baseline，不覆盖完整系统。
7. 可直接借用的工具思想包括：TIR 的 rational IT 表示、OLS 拟合数值系数、NSGA-II front、crowding tournament、node-count complexity objective。
8. 可直接借用的实验设计包括：按 samples * features <= 1000 切小数据集、报告 train/test R2 的最坏和最好 seed、比较 front selection 策略。
9. 可借用的 negative lesson 是：多目标优化不自动等于泛化，front 后处理和 selection rule 要单独设计。
10. 这篇应该引用，尤其当我们讨论符号表达式搜索、小样本过拟合、Pareto selection 或 SRBench baseline。
11. 差异化建议：不要只说“multi-objective for expression simplicity”；要强调新的 FEX setting、不同 verifier、不同 complexity metric、或跨任务泛化证据。
12. 若做 novelty check，需要继续查 2025-2026 年关于 multi-objective symbolic regression、Pareto/MDL/AIC model selection、digital twin symbolic regression 的后续工作。

## Read by: 0616-fex (2026-06-16)

本次按 `0616-fex` 视角看，这篇是一个需要纳入相关工作的符号回归方法论文，而不是完整撞车。它的核心贡献不是新的 feature/expression extraction 框架，而是把 TIR 符号回归的搜索从单目标 accuracy 改成 accuracy 与 expression size 的双目标 NSGA-II，并系统比较小数据集上不同 Pareto front 选择策略。

最值得借用的是它把“生成候选表达式”和“选择最终表达式”拆开：plain TIRMOO 会产生 Pareto front，但如果最终仍选 front 上最准确的模型，小数据过拟合仍然明显；TIRMOO-Select 或 TIRMOO-Sel-points 这类更偏小模型的后处理才真正改善泛化。这对 FEX 很关键，因为表达式发现系统往往会生成多个候选式，最终 verifier/selector 的独立性和偏好函数可能比搜索器本身更决定结果。

如果我们的 claim 是“用多目标优化或复杂度惩罚提升符号表达式泛化”，这篇会构成近邻 prior art，需要明确差异：新的表达式空间、不同数据模态、独立 verifier、不同复杂度准则如 MDL/AIC/BIC、或在 2025-2026 新 benchmark 上的发现。若我们的 FEX 更偏 LLM/agent 辅助假设生成，这篇不撞系统层面，但必须引用其 SRBench 小样本过拟合结论，并把 TIR/TIRMOO 作为强 baseline 或 ablation。下一轮检索应围绕 multi-objective symbolic regression, Pareto model selection, MDL symbolic regression, structural risk minimization GP, digital twin symbolic regression 展开。
