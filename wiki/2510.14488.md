## 精华提炼

### 管线/方法

**Core idea (Guess2Graph / G2G framework).** Many causal-discovery algorithms contain subroutines that sample a *sequence* of actions (which edge to test next, which conditioning set to try next) uniformly at random, where — critically — under oracle/consistent CI tests *any* sequence yields the same asymptotically-correct answer. G2G's insight: replace uniform sampling with expert-guided sampling in exactly these subroutines. Because any sequence remains asymptotically valid, statistical consistency is untouched; but because finite-sample CI tests are error-prone, a *good* sequence can substantially reduce error and runtime. This is explicitly framed as an application of the "algorithms with predictions" / learning-augmented-algorithms paradigm (Mitzenmacher & Vassilvitskii 2020) to purely observational causal discovery — first such application (prior work applied the paradigm to interventional causal learning, `active-learning-advice-arnab-2023`).

**Pipeline for one discovery run:**
1. Query an expert ψ (human or LLM) for a single one-shot causal structure guess Ĝ (skeleton + a topological ordering; no confidence/uncertainty required).
2. Extract two orderings from Ĝ via a shared subroutine EOE (Extract Orderings from Expert): an edge ordering 𝖮 (predicted non-edges first, predicted edges last) and a subset ordering 𝖫 (subsets predicted to be d-separating first).
3. Run a constraint-based skeleton-discovery algorithm, but every place it would sample a test sequence uniformly at random, sample according to 𝖮/𝖫 instead.
4. Return the learned skeleton (edge orientation via standard v-structure + Meek rules is *not* itself guided — justified because it only consumes the CI-test outcomes already produced by the guided skeleton phase).

**Decomposition into two universal subroutines** (Section "G2G in Constraint-Based Discovery"):
- **Edge Loop (EL)**: for conditioning-set size range [k_min, k_max], iterate over edges in order 𝖮, calling Edge Prune for each edge that satisfies a validity rule R.
- **Edge Prune (EP)**: for an edge and conditioning-set size k, test all size-k subsets of the adjacency set in order 𝖫, removing the edge on the first independence result.
- Any constraint-based algorithm (PC, SGS, rPC-approx, etc.) is just a particular choice of how EL/EP are invoked and what R is (Appendix gives an explicit PC vs. rPC-approx decomposition table).

**Guided versions:**
- **EL-Guess**: partitions 𝖢 into 𝖢∖Ŝ (predicted false edges, tested first) and 𝖢∩Ŝ (predicted true edges, tested last), random order within each bucket. This ordering changes *accuracy*, not just runtime, because correctly removing a false edge shrinks the adjacency sets of its endpoints (helping later true-edge tests survive), while correctly retaining a true edge changes nothing.
- **EP-Guess**: within a single edge's tests, places conditioning sets the expert predicts are d-separating before others. This ordering changes only *runtime* (Lemma: accuracy is provably invariant to EP's ordering, since edge removal depends on whether *any* test in the set returns independent, not which one runs first).

**Two instantiations built from these guided subroutines:**
- **PC-Guess**: standard PC's per-conditioning-set-size loop (ℓ = 0 … d−1), just fed guided 𝖮/𝖫 instead of uniform. Inherits PC's "statistical-conditioning bias" — must exhaust all conditioning-set sizes below k before it can even attempt the size-k test that would remove a false edge whose minimal d-separating set has size k. This blocks full exploitation of a perfect expert (concrete 4-node chain counterexample given in appendix: a false edge with minimal d-sep set {x₂} still forces one guaranteed-fail test at ℓ=0, and inflates every neighboring edge's adjacency set for that whole round).
- **gPC-Guess** ("Guided PC-Guess", the paper's proposed *redesign*, earlier draft names LAPC-G2G/SGS-GUESS): drops the level-by-level structure entirely — single pass, tests each edge across *all* conditioning-set sizes 0…|V|−1 at once, validity rule reduced to "edge still present." This makes the algorithm immediately responsive to expert guidance (false edges removed the instant they're reached in 𝖮, regardless of their minimal d-sep set size) at the cost of running some large-conditioning-set tests earlier than is "statistically optimal" when the expert is wrong.

### 关键技术细节

- **Analysis metric**: the paper deliberately optimizes *perfect recovery probability* Φ = P(every edge in the returned skeleton matches ground truth), NOT an error-tolerant metric like expected edge-count-correct. Reason: false-positive and false-negative errors have *opposing* effects on downstream adjacency sets (FPs inflate them, FNs shrink them), so which effect dominates is graph-dependent and analytically intractable (formalized in Appendix "Complexities of Error Propagation in Edge Loop"). Φ factors cleanly into a product of conditionals via the chain rule, sidestepping this.
- **Two structural lemmas drive everything**: (1) *edge-asymmetry* — adding vertices to an adjacency set strictly decreases P(correctly keep a true edge) but strictly increases P(correctly remove a false edge); (2) *edge-ordering-asymmetry* — correctly removing a false edge before testing a true edge can only help (or not hurt) the true edge, while the reverse never helps. Together these give *false-before-true* (Lemma): swapping an adjacent true-then-false pair to false-then-true is never worse and often strictly better for Φ — the foundation for why EL-Guess's bucket ordering is provably good.
- **Expert model**: binary symmetric channel — for each edge (or each candidate d-separating set), the expert independently reports the correct label with probability p_ψ (resp. p_d-sep), no calibrated confidence required. p_ψ = 0.5 ⇔ uninformative/random guess.
- **Novel proof technique**: all monotonicity results (accuracy increases with p_ψ; runtime decreases with p_d-sep) are proved by (a) constructing a *monotone coupling* between two experts of different accuracy using shared randomness (same random permutations π_T/π_F, same uniform draws r_i, only the accuracy threshold differs), (b) showing the resulting orderings differ only by which "inversions" (true edge before false edge) exist, (c) invoking the **weak Bruhat order on permutations** (Yessenov 2005, Prop 2.2: v ≤ w in weak Bruhat order iff Inv(v) ⊆ Inv(w)) to show the higher-accuracy ordering is reachable from the lower-accuracy one via a sequence of provably-beneficial adjacent swaps, (d) applying **Strassen's Coupling Theorem** to lift pointwise dominance to expectation dominance. This coupling + weak-Bruhat-order combination is a clean, reusable technique for "does guidance help" proofs beyond this paper.
- **Correctness proof** (asymptotic, Theorem "guess acc correct"): under oracle CI tests both PC-Guess and gPC-Guess eventually test every edge/conditioning-set pair regardless of order (inherits PC's classic any-ordering correctness result), so recovery is order-independent in the infinite-data limit; under merely *consistent* CITs, a union bound over the finite (≤ |V|²·2^|V|-2) set of possible tests drives the failure probability to 0 as n→∞ — independent of expert quality.
- **Design criteria (C1–C3)** the whole framework is built to satisfy: C1 Statistical Consistency (recovers true graph as n→∞ regardless of expert), C2 Monotonic Improvement (perf increases monotonically with p_ψ), C3 Finite-Sample Robustness (perf with guidance ≥ without, once p_ψ ≥ 0.5). PC-Guess only gets C2/C3 *per-iteration* (cascading effects across ℓ block an end-to-end proof — an explicitly acknowledged open gap); gPC-Guess gets all three *end-to-end* (Theorem "sgs performance").
- **CI tests used**: Fisher's Z-test (α=0.05) for linear-Gaussian synthetic data; chi-square test for the discretized Sachs data.
- **Synthetic DGP**: linear SEMs, Erdős–Rényi graphs, edge weights uniform in [−2.5,−1.5]∪[1.5,2.5] (avoids faithfulness violations near zero), standardized data, variables randomly permuted before discovery to avoid ordering leakage. Sparse (ER1, ≈d edges) vs. dense (ER3, ≈3d edges); main text: d=20, n=100, 30 trials/setting.
- **Real data**: Sachs protein-signaling network (11 nodes: Raf, Mek, Plcg, PIP2, PIP3, Erk, Akt, PKA, PKC, P38, Jnk), n=100 subsample, known ground-truth DAG.
- **LLM expert**: Claude Opus 4.1 via AWS Bedrock, single fixed chain-of-thought prompt per trial (4-step reasoning: biological function → direct interaction → pathway/cascade → regulatory relationship; explicitly told "missing edges is worse than including uncertain ones" to bias toward high recall), variable names shuffled per trial, 30 independent trials on Sachs.
- **Baselines**: PC-Stable (order-independent variant, Colombo & Maathuis 2014); PC/gPC = PC-Guess/gPC-Guess fed a p_ψ=0.5 "expert" (isolates the value of guidance from the value of the algorithmic redesign itself).
- **Compute**: single EC2 instance, AMD EPYC 7R13, 192 vCPUs, 740 GiB RAM; `causal-learn` for PC/PC-Stable implementations.

### 核心结果

| Setting | PC-Guess | gPC-Guess |
|---|---|---|
| Synthetic (ER3, d=20, n=100), perfect expert vs. baseline | up to ~5 pts F1 | up to ~30 pts F1 |
| Sachs + simulated expert | flat/modest gain | +30 pts F1 |
| Sachs + Claude Opus 4.1 expert | — | +15 pts F1 boost, ~10 pts above other baselines |
| Worst-case expert (p_ψ=0, fully adversarial) | small drop | ~8 pts F1 drop from baseline (bounded, not catastrophic) |

- Both algorithms show monotonic F1 improvement with expert accuracy p_ψ in both synthetic and real data — confirms the theory empirically, including for the metric (F1) the theory doesn't directly target.
- **Central empirical claim**: simple *augmentation* of an existing algorithm (PC-Guess) leaves most of the framework's potential on the table; *redesigning* the algorithm's internal structure to be more receptive to guidance (gPC-Guess) is what fully realizes the gains — same underlying G2G framework, dramatically different payoff.
- Runtime (number of CI tests) decreases with p_ψ for both, more dramatically in dense graphs.
- Value of guidance *increases* with dimensionality d (the regime — high-dim, low-sample — where causal discovery is hardest and expert help matters most): at d=5 the gPC-Guess vs. PC-Stable gap is ~7 pts, at d=30 it's ~18 pts.
- Gains shrink as sample size n grows (expected — consistency means all methods converge).
- d-separation-accuracy (p_d-sep, which only affects EP/runtime) barely matters above a low threshold (~25%) in dense graphs.

### 明确局限

- **Theory–practice metric gap**: all formal guarantees are stated for the strict "perfect recovery" probability Φ, but experiments report F1/skeleton-accuracy — the paper argues error-tolerant metrics are analytically intractable (graph-structure-dependent) rather than proving F1-specific bounds. This is honestly flagged, not swept under the rug.
- **PC-Guess has no end-to-end guarantee** — only per-iteration monotonicity/robustness; whether these compose across the ℓ=0…d−1 loop is stated as an open question because early-iteration improvements have "complex cascading effects" on later iterations.
- **Extensions to score-based (SE-GES-Guess) and ANM/FCM-based (RESIT-Guess) methods are conjectures only** — sketched in the appendix with matching intuition but explicitly "we have not completed formal proofs."
- **LLM validation is thin**: one dataset (Sachs, 11 nodes), one model (Claude Opus 4.1), one fixed prompt, 30 trials. The paper explicitly warns *elsewhere* (related work) that other papers' LLM-causal-discovery results on well-known benchmarks may reflect training-data memorization/test-set leakage rather than genuine reasoning — yet doesn't run a memorization control (e.g., a held-out or synthetic-renamed variant of Sachs) for its own Claude result.
- **Expert model is a simple i.i.d. binary symmetric channel** per edge/d-sep-set — doesn't model realistic correlated LLM error patterns (e.g., systematic errors on hub nodes, or errors correlated with variable-name semantics).
- **Guess-quality validation and iterative reprompting are appendix sketches, not implemented/evaluated**: the "Guess Quality" (permutation-based null-distribution p-value test) and "Expert Self-Consistency" (Faller et al. 2024-style overlapping-subset agreement score) mechanisms for deciding *whether to trust* a given guess at all are proposed only qualitatively, with no experiments — this is exactly the piece the main framework punts on ("we focus on the simplest case — a single expert whose guesses are always accepted").
- Only skeleton recovery is guided; orientation (v-structures + Meek rules) is unguided by construction, with the propagation argument untested empirically in isolation.
- No code/artifact released at submission time ("will release upon acceptance").
