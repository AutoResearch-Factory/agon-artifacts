# Univariate Skeleton Prediction in Multivariate Systems Using Transformers (2406.17834)

- **Year**: 2024.06
- **Authors**: Giorgio Morales and John Sheppard, Gianforte School of Computing, Montana State University
- **Venue**: ECML PKDD 2024 / arXiv
- **Keywords**: symbolic regression, neural symbolic regression, Multi-Set Transformer, symbolic skeleton, post-hoc explanation, multivariate regression, counterfactual sampling, precision agriculture
- **One-liner**: 用多集合 Transformer 生成每变量符号骨架解释黑盒回归。

## 精华提炼

### 管线/方法
- 论文要解决的问题不是直接拟合一个最低 MSE 的多变量公式，而是提取每个输入变量与响应之间的单变量函数骨架。
- 输入是多变量回归数据 `X, y`，假设真实系统可由含一元算子和二元算子的数学表达式表示。
- 第一步训练一个黑盒回归模型 `f_hat`，论文实验中使用 feed-forward NN，以 MSE 作为训练目标。
- 该 NN 的作用是作为可查询的系统替身，给人工构造的 counterfactual 输入点估计响应。
- 对每个变量 `x_v` 单独处理，其他变量固定，只有 `x_v` 在观测范围 `[x_v_min, x_v_max]` 内变化。
- 对同一个 `x_v` 构造 `N_S` 个输入集合，每个集合里其他变量使用一组固定随机值，因此每个集合对应同一个单变量函数族的不同常数。
- 论文默认使用 `N_S=10` 个集合，每个集合有 `n=3000` 个 input-response pairs。
- 每个集合被表示为 `S^(s) in R^{n x 2}`，每行是 `[x_v, y_hat]`。
- 这些 `N_S` 个集合一起输入 Multi-Set Symbolic Skeleton Prediction, 简称 MSSP。
- MSSP 的目标是输出一个共同 skeleton `e_hat(x_v)`，描述所有集合共享的函数形式，只把常数留作占位符 `c_i`。
- Multi-Set Transformer 的 encoder 先用共享的 set encoder `phi` 独立编码每个集合。
- 每个 `phi` 是若干 Induced Set Attention Blocks, ISAB, 加一个 Pooling by Multi-head Attention, PMA。
- 所有集合的 latent vectors 再通过额外 PMA 聚合为唯一 latent representation `Z`。
- Decoder 是标准 Transformer decoder 风格的条件自回归生成器。
- 输出序列是 prefix notation 的 symbolic skeleton token 序列。
- 推理时按 token 自回归生成，每一步取 vocabulary 上 softmax 概率最高的 token。
- 这个过程对每个变量重复一次，最终得到 `e_hat(x_1), ..., e_hat(x_t)`。
- 作者把这些单变量 skeleton 解释为黑盒 NN `f_hat` 的 post-hoc explanation。
- 论文当前不把这些单变量 skeleton 合成为完整多变量表达式。
- 未来工作明确提出要研究如何保证各变量 skeleton 兼容并合并为完整表达式。

### 关键技术细节
- 黑盒回归 NN 训练目标是 `argmin_theta 1/N_R sum_j (f_hat(x_j)-y_j)^2`。
- Synthetic pretraining 数据包含 `10^6` 个预生成表达式。
- Validation set 包含 `10^5` 个表达式。
- 表达式以 prefix notation 存储，并转为 skeleton token 序列监督 decoder。
- 训练 batch 中每个样本都是一组 `N_S` 个集合，而不是单个点集。
- `generateSets` 会从表达式中抽取常数占位符，并采样多个不同常数版本。
- 常数从 `U(-10, 10)` 独立采样。
- 输入支持点从 `U(-x_limit, x_limit)` 采样，其中 `x_limit ~ U(1, 10)`。
- `avoidNaNs` 会处理 `log`, `sqrt`, `exp`, `sinh`, `cosh`, `tanh`, `asin`, `acos`, `tan`, division 等可能产生 undefined 或极大值的问题。
- `log` 和 `sqrt` 被当作左边界算子，`exp/sinh/cosh/tanh` 被限制右侧避免过大值。
- `asin/acos` 被当作双边界算子，输入需落在 `[-1, 1]`。
- 对 division/tan 这类 singularity，算法重采样或避开奇异点附近的输入区间。
- 表达式生成不是 Biggio/Lample 式随机插入 operator，而是 preorder recursive tree generation。
- 节点类型按 unary:binary:leaf 的权重 2:1:1 采样。
- 作者禁止一些容易直接化简或数值爆炸的组合，例如 `log(exp(x))`, `exp(exp(x))`, `sinh(sinh(x))`。
- Vocabulary 包含 `SOS`, `EOS`, `c`, `x`, `abs`, `acos`, `asin`, `atan`, `cos`, `cosh`, `div`, `exp`, `log`, `mul`, `pow`, `sin`, `sinh`, `sqrt`, `tan`, `tanh`, 整数 token 和 `E`。
- 主实验里的 operator 集包括 `+`, `*`, `/`, `abs`, `acos`, `asin`, `atan`, `cos`, `cosh`, `exp`, `log`, `pow2` 到 `pow5`, `sin`, `sinh`, `sqrt`, `tan`, `tanh`。
- 选定 architecture: encoder ISAB blocks `ell=3`。
- Decoder blocks `M=5`。
- Embedding size `d=512`。
- Attention heads `h=8`。
- 论文正文称 MST 需要约 24.2M parameters，附录表写 23,094,304 parameters，存在一个小的数字不一致。
- E2E baseline 约 93.5M parameters，NeSymReS 约 26.4M parameters。
- Synthetic evaluation 使用 13 个多变量方程 E1 到 E13。
- 每个 synthetic problem 使用 10,000 个均匀采样点。
- 训练/验证划分为 90%/10%。
- E2 使用 3 个 hidden layers 的 FFNN。
- E1, E4, E5, E7 使用 5 个 hidden layers 的 FFNN。
- 其他 synthetic problems 使用 4 个 hidden layers。
- 每层 500 个 ReLU nodes。
- Baselines 包括 PySR, TaylorGP, NeSymReS, E2E。
- GP 方法最大 10,000 iterations，测试 population size 100, 200, 500, 1000，作者称超过 500 没有明显收益。
- NeSymReS 和 E2E 使用在线预训练模型。
- NeSymReS 因训练限制不能处理 E4 和 E5，因为它只支持最多 3 个变量。

### 核心结果
- 作者的主要 claim: MST 生成的单变量 skeleton 在所有测试问题和变量上都匹配或等价于 target skeleton。
- 他们强调传统 MSE 评测可能奖励错误函数形式，例如二次近似可以在窄区间内逼近 `cos(-x/10)^2`。
- 因此论文提出 skeleton-level evaluation，而不是直接用预测误差评价公式。
- 对每个 target skeleton，随机采样常数得到 `f_target(x_v)`。
- 对估计 skeleton 用 GA 拟合常数，目标是最小化 test support 上的绝对误差和。
- Test support 使用 `N_test=3000` 个点，并从比训练更宽的区间 `U(2*x_min, 2*x_max)` 采样。
- 每个 skeleton 评测重复 30 次，报告 mean 和 std。
- GA population size 为 500。
- GA 停止条件是 20 generations 后 objective 变化小于 `10^-5`。
- GA 使用 tournament selection, binomial crossover, generational replacement。
- 统计显著性用 Tukey HSD, `alpha=0.05`。
- E2 示例里，MST 对 `x_1` 输出二次骨架，对 `x_2` 输出平方根骨架，对 `x_3` 输出正弦骨架，均与 target 同型。
- E2 的 skeleton error: MST 在 `x_1` 和 `x_3` 为 `0 +/- 0`，`x_2` 为 `0.02 +/- 0.03`。
- E2 的 `x_2` 上，PySR 是 `5.4 +/- 5.0`，TaylorGP 是 `4.2 +/- 5.3`，NeSymReS 是 `4.6 +/- 5.0`，E2E 是 `4.2 +/- 5.3`。
- E6 的 `x_1` 上，target 是 `tanh` 型，MST 输出 `tanh`，E2E 输出 `atan` 型。
- 作者指出 E2E 的 `atan` 可以通过常数拉伸在数值上像 `tanh`，但函数形式并不真正一致。
- 这个例子支撑了论文的评价动机: 低误差不等于解释正确。
- E4 中 E2E 能输出等价但更大的多项式绝对值表达式，MST 输出更简洁的四次多项式骨架。
- 作者认为 MST 相比 E2E 的优势之一是参数更少，同时更专注于每变量函数形式。
- 在 E5, E6, E8, E9, E13 等问题里，一些 baseline 也能低误差，但函数形式不如 MST 对齐。
- Real-world 部分分析冬小麦 Field A 的 nitrogen fertilizer response curves。
- 该田块被分为 4 个 management zones, MZs。
- 每个 MZ 随机选 `N_S=10` 个 field sites，把 N-rate `Nr` 与 relative yield `ry` 曲线作为 MSSP 输入。
- 传统 quadratic-plateau 和 exponential 模型假设一个预定义函数形式。
- MST 则为每个 MZ 发现不同 skeleton。
- MZ1: MST `r_bar=0.0489`，quadratic-plateau `0.0680`，exponential `0.2303`。
- MZ2: MST `0.0495`，quadratic-plateau `0.0725`，exponential `0.1825`。
- MZ3: MST `0.0479`，quadratic-plateau `0.0879`，exponential `0.1965`。
- MZ4: MST `0.0237`，quadratic-plateau `0.0615`，exponential `0.2249`。
- Real-world 结果说明不同 MZ 可能需要不同函数形式，而不是整块田统一使用一个经验曲线。
- 这部分没有真实 closed-form ground truth，所以证据是 fitting error 和 domain plausibility，不是 skeleton recovery。

### 明确局限
- 该方法解释的是训练出的 NN `f_hat`，不是直接证明解释了真实系统 `f`。
- 如果 `f_hat` 学错、过拟合或在人工采样区域外 extrapolate 失败，MST 的 skeleton 会继承这些错误。
- 人工构造集合时固定其他变量，这是一种 counterfactual slicing，依赖输入空间中这些组合是有意义的。
- 对强交互系统，单变量切片可能给出局部或条件性的解释，而非全局因果机制。
- 当前论文不输出完整多变量表达式，这对 symbolic regression 的最终建模目标是明显短板。
- 作者自己承认还需要解决 univariate skeleton 的兼容性和合并顺序问题。
- Skeleton 复杂度受 pretraining 表达式生成器限制。
- 作者举例说，`c_1 + c_2*x_2^2/sin(c_3*e^{c_4*x_2})` 这类需要 8 个 operators 的 skeleton 超出训练集能力。
- Hyperparameter selection 是 one-factor-at-a-time，不是系统 ablation。
- 作者明确说未来还需要研究 `N_S` 和 `n` 对性能的影响。
- Synthetic problems 是作者选定或改造的 13 个方程，覆盖面仍有限。
- Baseline 比较集中在 SR 模型，没有和 broader explanation 方法直接比较。
- Real-world N-response 任务中 MZ 内共享函数形式是额外假设。
- Field A 实验借助前作生成的 response curves 和 management zones，外部独立数据集不足。
- 论文没有提供独立复现实验报告。
- 代码和数据链接在 tex 中声明为 GitHub `NISL-MSU/MultiSetSR`，但本次未在线验证 artifact 完整性。
- Evaluation metric 仍然依赖 GA 找常数，若 GA 未找到好常数，可能误判 skeleton 不等价。
- 反过来，某些不同函数形式在有限测试区间内仍可能被评为相近。
- 对 0616-fex 若目标是 feature-wise explanation，这篇很近；若目标是 full equation extraction，则它只是前半段。
- 对 0616-fex 若目标要求生成与检验分离，这篇的 verification independence 不够强。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 视角看，这篇是强相关文献，尤其适合作为 feature-wise/function-wise symbolic explanation 的 prior art。它的核心贡献不是一般意义上的 transformer SR，而是把 multivariate SR 改写成多个 MSSP 子问题：每个变量生成 10 个 counterfactual input-response sets，其他变量固定，变量本身变化，然后用 Multi-Set Transformer 输出共享 skeleton。若我们的方案也做“固定其他变量、解释单变量影响、输出符号骨架”，需要把这篇列为直接竞品。

撞车最严重的点是方法 claim：post-hoc distillation of an opaque regression model into univariate symbolic skeletons。它已经有 synthetic equations、skeleton-level metric、PySR/TaylorGP/NeSymReS/E2E 比较，以及 N-response real-world case。差异化最好不要停留在“我们也用 Transformer 解释变量”，而应该转到完整多变量表达式合成、独立 verifier、uncertainty-aware sampling、交互项识别、或更严格的 evidence grounding。

可用工具上，MSSP 任务定义、ISAB/PMA multi-set encoder、prefix skeleton decoder、`avoidNaNs` 合成表达式生成、GA-based skeleton equivalence metric 都值得复用。下一轮搜索应重点查 `2511.04124` 和 `2605.31276`，因为前者已经补上本文 future work 里“合并单变量 skeleton”的缺口，后者把农业 N-response 应用继续推进。若 0616-fex 要做新贡献，需要避开这两篇的覆盖范围。
