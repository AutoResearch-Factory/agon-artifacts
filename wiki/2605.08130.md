# Additive Atomic Forests for Symbolic Function and Antiderivative Discovery: Generating Primitives, Product-Rule Closure, and Derivative-Matching Optimisation (2605.08130)

- **Year**: 2026.05
- **Authors**: Reda Belaiche, Universite Paris-Est Creteil
- **Venue**: arXiv
- **Keywords**: symbolic regression, antiderivative discovery, derivative matching, additive atomic forests, EML primitive, SOL primitive, product-rule closure, self-expanding library
- **One-liner**: 用导数匹配和自扩展原子库同时发现函数及其原函数。

## 精华提炼

### 管线/方法
- 输入是单变量或多变量数据点，核心训练目标不是直接拟合候选函数 `F`，而是拟合其解析导数 `F'` 到观测目标 `y`。
- 输出同时包含两个符号对象：匹配数据的函数 `F'`，以及由原子结构直接给出的原函数 `F`。
- 第一层思想是 derivative algebra：每个原子都保存一个函数值和一个解析导数，形式为 `(phi, phi')`。
- product rule 是主要闭包算子：若已知 `(f, f')` 和 `(g, g')`，则自动生成 `(fg, f'g + fg')`。
- chain rule 是第二个闭包算子：若已知内部函数及外部 elementary function 的导数，就能生成复合函数对。
- 线性组合提供 additive forest 顶层结构：`F = sum_k c_k T_k`，因此 `F' = sum_k c_k T'_k`。
- 作者强调不调用 Risch 算法，也不对数据做数值积分；原函数来自森林每个原子的构造。
- 初始种子包含有理幂 `x^(p/q)`，其中 `p in {-4,...,15}`，`q in {1,2,3,4}`，不有限的候选在数据网格上丢弃。
- 第一类生成 primitive 是 EML：`eml(u,v)=exp(u)-ln(v)`，导数为 `exp(u)u' - v'/v`。
- 第二类 supplementary primitive 是 SOL：`sol(u,v)=sin(u)-cos(v)`，导数为 `cos(u)u' + sin(v)v'`。
- EML 的作用是提供指数和对数族，并依赖 2603.21852 的结论声称 elementary functions 的理论完备性。
- SOL 的作用是把 `sin x`、`cos x` 放到 depth 1，避免纯 EML 通过复数 Euler 公式构造三角函数时需要约 depth 8。
- 混合 grammar 为 `S -> 1 | x | eml(S,S) | sol(S,S)`，作者认为保留 EML 完备性的同时降低三角结构深度。
- canonical tree 的叶子是 `sigma(alpha)*1 + sigma(beta)*x`，训练后 snap 到常数 1 或变量 x。
- enhanced forest 允许四类节点：LeafNode、EMLMasterFormula、SOLMasterFormula、MultiplicativeNode。
- MultiplicativeNode 显式表示乘积结构，例如 `x exp(x)`，避免把乘法硬编码进一个深 primitive tree。
- 物化库模式先构造候选原子矩阵，再对导数矩阵做稀疏线性组合搜索。
- 导数矩阵 `D in R^{N x M}` 存储每个原子导数在样本点的值，预计算 `G = D^T D` 和 `d = D^T y`。
- `K=1` 搜索是向量化标量回归，系数为 `d_k / G_kk`，复杂度在预计算后为 `O(M)`。
- `K=2` 搜索用 Cramer rule 批量解所有二元子问题，作者设置 determinant 阈值 `|Delta| > 1e-30`，系数上限 `|c_k| <= 20`。
- `K>=3` 用 beam search：从 `K-1` 的 top 200 结果扩展，每轮保留 top 500 候选。
- 精确候选要求训练 MSE 极小后再用独立网格验证；文中把 `MSE < 1e-15` 作为需要独立验证的强候选阈值。
- 梯度模式使用 Adam，在树叶 softmax 参数上训练 derivative-matching loss。
- softmax temperature 在训练前 80% 从 1.0 线性退火到 0.1，推动叶子向 one-hot 选择收敛。
- 深树训练时使用 `log(1 + |r|^2)` 作为稳定化 loss 变体。
- 梯度裁剪阈值为 norm `<= 5`。
- 多 restart 并行训练，snap 后若 `MSE < 1e-10` 则停止并验证。
- 作者提出知识库增长机制：每次发现新函数 `f` 后，把 `(f, f')` 重新加入库，后续任务可继续组合、复合和相乘。
- 这个增长机制是方法叙事的核心，但论文没有给出跨任务持续学习的严格实验，只给了设计论证和若干任务内示例。

### 关键技术细节
- 理论对象是 elementary functions field `E`，从 `C(x)` 在 `exp`、`log` 和 composition 下闭包得到。
- Product-rule closure 命题证明较直接：product、composition、linear combination 都保持已知导数对。
- conditional completeness theorem 依赖外部 EML 完备性：若 EML 对 `{1,x}` 终端完备，则 additive forest width 1 即可表示任意 elementary function。
- additive depth theorem 说明若多个 summand 各自 depth 较浅，forest 顶层求和可避免在单树里额外编码加法。
- 复杂度命题给出结构化导数图大小 `O(K 2^d)`，这里是 computation graph 复杂度，不是展开后最简符号项数。
- Layer 1 对线性内函数 `a x + b` 生成 EML 与 SOL depth-1 原子，斜率和截距为整数超参数。
- Layer 2 生成 quadratic argument 的 `exp(g)`、`-ln(g)`、`sin(g)`、`cos(g)`、`1/g`、部分 `arctan` 和 `arcsin`。
- Layer 3 对已有原子两两乘积，形成 EML x SOL cross-products，例如 `e^x sin x`、`e^{-x} cos 2x`、`x e^{-x} sin x`。
- Layer 4 做 depth-2 和 depth-3 nestings：`exp(phi)`、`sin(phi)`、`cos(phi)`、`-ln(phi)`、`1/phi`、`arctan(phi)`，并乘以 `x`。
- Deduplication 用数据网格相关性阈值：若 `|corr(psi, phi)| > 0.999`，则拒绝新候选。
- 分类实验把输入变量展开到 atom library，再用 L1/L2 regularised logistic regression 拟合。
- 分类实验是 17 个 benchmark 的 5-fold CV，指标只报 raw accuracy。
- Feynman 实验用自建 multi-variable library，greedy expansion 到 depth 3，搜索 `K=1,2,3,4`。
- Feynman 结果计数为 exact pass 14/45、close 18/45、fail 13/45。
- Feynman exact pass 全部发生在 `K <= 2`，其中 11 个在 `K=1`，3 个在 `K=2`。
- Feynman 失败解释为 8 个需要尚未在库中出现的 composed arguments `f(x_i x_j / x_k)`，3 个非可分，2 个 rational-of-sums。
- Feynman 运行总时间报告为 76 秒，硬件为 NVIDIA T4。
- SPARC/RAR 示例使用 175 个星系和 3389 个数据点。
- RAR 搜索目标为 `g_obs/a0 = sum_k c_k phi_k(g_bar/a0)`，标准质量光度比为 disk 0.5、bulge 0.7，`a0 = 1.2e-10 m/s^2`。
- RAR 最好 `K=1` 原子为 `y = x^{2/3}`，wMSE 0.074；最好 `K=2` 为 0.070。
- 作者明确指出 `x^{2/3}` 对应 `nu(x)=x^{-1/3}`，不满足标准 MOND 的大 x 和小 x 渐近约束。

### 核心结果
- 分类 benchmark 上，作者报告 atoms 在 17 个数据集中的 13 个 match or exceed XGBoost。
- Haberman：atomic 75.8%，XGBoost 66.0%，提升 +9.8%，但作者承认类别不平衡时 raw accuracy 可能误导。
- sklearn diabetes 二分类化：atomic 75.8%，XGBoost 70.1%，提升 +5.7%。
- Heart disease：atomic 83.5%，XGBoost 79.8%，提升 +3.7%。
- Pima：atomic 77.0%，XGBoost 74.1%，提升 +2.9%。
- Breast cancer：atomic 97.4%，XGBoost 96.3%，提升 +1.1%。
- 失败项包括 Ionosphere、Glass、Parkinsons、Adult，其中 Adult 差距仅 -0.6%。
- Synthetic regression ground truth 为 `3e^{-x0} + 2 sin(x1) + 0.5 x2^2`，atomic `R^2=0.999`，XGBoost 0.995，ridge 0.939。
- sklearn diabetes regression：atomic `R^2=0.481`，XGBoost 0.336，ridge 0.479。
- Haberman 解释性公式示例只选一个 atom：`F(x2)=4.404 arctan(x2)-0.859`，导数 `4.404/(1+x2^2)`。
- Heart disease 示例中主变量是 `x11`，`tanh` 饱和显示从 0 到 1 条 diseased vessel 风险跳变最大。
- Heart disease 导数对比为 `partial F / partial x11 | x11=0 = 21.93`，而 `x11=2` 时为 1.07。
- Feynman symbolic regression benchmark exact recovery 31%，另有 40% relMSE < 0.01。
- RAR 中 `x^{2/3}` 的 wMSE 0.074 略低于 MOND simple 0.077、standard 0.078、RAR curve 0.079。
- 一个反直觉点是，作者把 antiderivative discovery 转化为 derivative matching，避免直接做 integration。
- 第二个反直觉点是，SOL 并非理论上必要，却在三角结构上大幅降低搜索深度，实际可能比完备性更重要。
- 第三个反直觉点是，classification 结果来自一维原子展开加线性模型，而不是常规符号回归直接拟合公式树。
- 最值得保留的技术 insight 是：如果每个 atom 都携带导数，那么搜索导数组合时原函数表达式几乎免费得到。

### 明确局限
- 这是单作者 arXiv preprint，论文内没有给出代码仓库、artifact、随机种子和完整实验日志。
- EML 完备性是条件性引用，本文没有独立证明完整 elementary function generation。
- SOL 的新意是实用 depth reduction，不是单独完备 primitive；论文也承认 SOL 不能产生 `exp` 或 `ln`。
- 分类实验只报 raw accuracy，作者自己指出 Haberman 等不平衡数据需要 balanced accuracy、AUC 或 F1。
- 17 个分类 benchmark 的 preprocessing、split seed、超参选择细节不足，容易有 benchmark leakage 或 tuning bias 风险。
- Feynman 实验是 representability test，不是完整 symbolic regression protocol；输入数据由已知公式生成。
- Feynman exact recovery 只有 14/45，说明 depth-3、`K<=4` 下的库仍有明显结构盲区。
- RAR 示例只是 unconstrained empirical atom search，不满足物理先验，作者明确禁止过度物理解读。
- Self-expanding KB 是论文最有吸引力的叙事，但缺少跨任务累计后性能提升的定量实验。
- 没有和 PySR、AI Feynman 在同一 symbolic regression protocol 上做充分 head-to-head。
- 梯度模式和物化库模式之间的关系没有完全实验化：分类主要是特征展开，Feynman 是库搜索，gradient forest 示例较少。
- Dedup 用数据网格相关性，可能删除在训练域相关但外推行为不同的表达式。
- 对噪声数据、离群值、稀疏采样、domain singularity 的鲁棒性实验不足。
- 多变量 interaction 需要 cross-product budget，论文失败分析已经显示这是瓶颈。
- Risch/Liouville 关系主要是概念对齐，没有把算法复杂性或可判定性纳入比较。

## Read by: 0616-fex (2026-06-17)

从 `0616-fex` 视角看，这篇论文是强相关文献。它不是普通“再做一个符号回归器”，而是把问题重写成 derivative matching：先让候选森林 `F` 的解析导数 `F'` 拟合数据，再把 `F` 当作原函数直接输出。这一点对任何 function/expression extraction 课题都有直接撞车风险，尤其是如果我们的贡献也声称“同时发现函数和导数/积分”。

可用工具层面，最值得借鉴的是物化 atom library 的线性代数搜索。`K=1` 扫描、`K=2` Cramer rule、`K>=3` beam search 都适合做可复现 baseline，而且比纯遗传式 symbolic regression 更容易记录完整候选和失败原因。EML/SOL 组合也值得纳入搜索空间设计，特别是 SOL 用一个 depth-1 primitive 解决三角函数浅层可得性的问题。

差异化需要避开本文已经占位的叙事：自扩展原子库、product-rule closure、derivative pair、antiderivative without integration 都不能再作为单独新意。`0616-fex` 更应该强调本文没有做扎实的部分：公开代码和 artifact、严格 benchmark、噪声鲁棒、多变量 interaction、跨任务持续增长实验、以及独立 verifier 对候选表达式的否决权。若我们能把“生成表达式”和“验证表达式”分离成两个独立系统，这会明显强于本文的自产自检设置。
