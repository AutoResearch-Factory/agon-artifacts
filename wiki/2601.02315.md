- **Year**: 2026.01
- **Authors**: Saurabh Kaushik$^{1,*}$, Lalit Maurya$^{2,*}$ (co-first), Beth Tellman$^{1}$ — $^1$Center for Sustainability and the Global Environment (SAGE), Univ. of Wisconsin–Madison; $^2$Portsmouth AI and Data Science Centre (PAIDS), Univ. of Portsmouth
- **Venue**: arXiv preprint, formatted with `wacv.sty` (likely a WACV 2026 submission/camera-ready; arXiv metadata only lists category `cs.CV`, no confirmed acceptance)
- **Keywords**: Geo-Foundation Models, Prithvi, CNN–Transformer fusion, adapter/PEFT fine-tuning, attention-based multi-scale fusion, flood inundation mapping, arbitrary-channel input, remote sensing
- **One-liner**: Frozen adapter-tuned Prithvi encoder + parallel CNN/CAM branch, fused via learned per-scale attention, beats most geo-foundation models on two flood-segmentation benchmarks and lifts Prithvi past its native 6-channel limit — but does not clearly beat the strongest U-Net baseline in its own table.

## 精华提炼

### 管线/方法

Input tensor $X \in \mathbb{R}^{B\times C\times H\times W}$ is split by a **fixed** channel-index partition into two disjoint, complementary streams — this is the core trick that lets the model take an arbitrary number of channels $C$ even though Prithvi natively only accepts 6:

- $\mathcal{I}_{AP} = \{1,2,3,7,11,12\}$ (the 6 HLS-like bands Prithvi was pretrained on) → routed to the **Adapted Prithvi** transformer branch.
- $\mathcal{I}_{CNN} = \{1,\dots,C\}\setminus\mathcal{I}_{AP}$ (everything else) → routed to a parallel **CNN** branch.

**Branch 1 — Adapted Prithvi (transformer):**
- Backbone: pretrained Prithvi ViT (600M params), kept **frozen**.
- Fine-tuning: a *Dynamic Prompt Adapter* inserted at the start of every ViT block: $y=\mathrm{blk}(x+f_{\text{adapter}}(x))$, $f_{\text{adapter}}(x)=\sigma(W_2\sigma(W_1x+b_1)+b_2)$, with $W_1\in\mathbb{R}^{d\times32}$, $W_2\in\mathbb{R}^{32\times d}$ — a 32-dim low-rank residual bottleneck. Only adapters + heads train → trainable params drop from 650M to **45.5M**, GPU memory −50–80%.
- Multi-level extraction: token features pulled from ViT layers $\{7,15,23,31\}$ (embedding dim 1280), reshaped token→2D via a token-to-image (T2I) transform.
- 4 separate FPN heads upsample and shrink channels per level: FPN1 (8× up, 1280→640→320→160), FPN2 (4× up, 1280→640→320), FPN3 (2× up, 1280→640), FPN4 (1×, keeps 1280) → a 4-level transformer feature pyramid $\mathbf{F}^{AP}=\{\mathbf{F}_1,\dots,\mathbf{F}_4\}$.

**Branch 2 — CNN:**
- 4 sequential Residual Blocks (He et al. 2016 style: $y=\sigma(x)+F(x;W)$, two 3×3 conv + BN + ReLU, 1×1-conv shortcut when dims differ), each paired with a **Convolutional Attention Module (CAM)**: channel attention via global-avg+max pooling, then spatial attention via pooled spatial maps + conv → a 4-level CNN pyramid $\mathbf{F}^{CNN}$.
- Best CNN channel width found by ablation: `[128,256,512,1024]` (beats `[32,64,128,256]` and `[64,128,256,512]`).

**Fusion — M²FAF (Multi-Scale Multi-Level Feature Attention Fusion), per level $i$:**
1. Resize+project CNN map to transformer resolution: $\hat{\mathbf{F}}_i^{CNN}=\mathrm{Conv}_{1\times1}(\mathrm{Interp}(\mathbf{F}_i^{CNN}, \mathrm{size}(\mathbf{F}_i^{AP})))$.
2. Attention mask: $\mathbf{A}_i=\sigma(\mathrm{Conv}_{1\times1}([\mathbf{F}_i^{AP};\hat{\mathbf{F}}_i^{CNN}]))$.
3. Stabilizing bias $\beta$: $attn_i' = attn_i(1-\beta)+\beta$, swept $\beta\in[0.2,0.8]$; **best at $\beta=0.8$** (favors transformer/semantic stream).
4. Convex-combine: $\mathbf{F}_i^{fuse}=attn_i'\odot\mathbf{F}_i^{AP}+(1-attn_i')\odot\hat{\mathbf{F}}_i^{CNN}$.

**Decoder:** UperNet — Pyramid Pooling Module on the coarsest map for global context, then top-down lateral fusion (L4→L1) across the 4 fused levels, final conv → per-pixel flood/no-flood mask, scored by IoU / mIoU / m-F1(Dice).

### 关键技术细节

- Backbone pretraining: Prithvi 2.0 (600M, and a separate 300M variant used as comparator), MAE-style pretrain on 4.2M HLS (6-band Landsat+Sentinel-2) samples — Prithvi's *own* paper describes it as multitemporal, but this work only ever uses single-timestamp input.
- Trainable parameter budget: 45.5M — about 7% of the 650M full 600M-encoder configuration.
- Datasets: **Sen1Floods11** (446 hand-labeled Sentinel-1/2 pairs, official split reused, plus a geographically held-out Bolivia site for OOD testing) and **FloodPlanet** (298 Sentinel-2 images with 3m PlanetScope-derived labels, resized to 320×320, no official split → self-defined 4-fold CV, 70/10/20 per fold).
- Optimization: cross-entropy loss, AdamW, StepLR; 20-trial HPO (60 epochs, patience 10) then final fit (patience 20); seed fixed at 42, batch size 8, single NVIDIA RTX A6000. The HPO search algorithm/library itself is never named.
- Two ablations: (1) module ablation (Prithvi-adapter × Residual-block × CAM × M²FAF, ticked on/off) — full combo best at 68.74 mIoU / 81.45 mDice; (2) CNN width ablation, monotonic with channel count.
- Per-fold std-dev on FloodPlanet: Prithvi-CAFE 0.02 mIoU — reasonably stable but not the single most stable entry in the table (U-Net's row shows 0.01).
- Minor internal inconsistencies spotted while reading: prose states "TerraMind-Base" std-dev is 0.14, but the results table's "TerraMind" row lists 0.01 (unclear if two different TerraMind variants are being conflated); prose quotes TransNorm mIoU as 60.19 while the table shows 60.09 for the same cell.

### 核心结果

- **Sen1Floods11 (in-distribution test)**: Prithvi-CAFE = 83.41 IoU-water / 90.50 mIoU / 97.80 m-F1 @ 45.5M params. Beats Prithvi-2.0-600M (82.50), Prithvi-2.0-300M (82.20), TerraMindv1-B (82.90), DOFA (81.54), SpectralGPT (81.02), U-Prithvi (82.22), the "U-Net" reimplementation from Kostejn et al. (80.69), DeepSARFlood (72.22), MM-UNet (73.84), RemoteCLIP/GFM-Swin/S12-DINO (all lower). **Does not beat "U-Net Base" (Li et al. 2023): 84.03 IoU / 90.80 mIoU / 95.40 m-F1 @ only 31M params** — the strongest baseline in the paper's own table. The text calls this "only marginal improvement (0.63) [by U-Net]," i.e., implicitly concedes the loss without stating it plainly in the abstract, whose "outperforms baseline U-Net" claim is true only against the weaker Kostejn U-Net (80.69), not against U-Net Base.
- Also close: Prithvi-2.0-300M-LoRA (Marti-Escofet et al. 2025) reaches 90.04 mIoU with only **5.5M** trainable params — an 8× parameter-efficiency edge over Prithvi-CAFE's 45.5M for a ~0.5-point mIoU gap the paper never discusses.
- **Sen1Floods11 Bolivia (geographic OOD holdout)** — the paper's strongest, least-ambiguous result: Prithvi-CAFE = 81.37 IoU / 88.87 mIoU / 96.87 m-F1, beating U-Net Base (70.57/82.54), Prithvi 1.0 (72.42/82.89), U-Prithvi (79.68/87.70), and all PEFT variants (LoRA 87.57, ViT-Adapter 84.94, Full-FT 82.07 mIoU). +10.8 IoU over U-Net Base under true geographic generalization.
- **FloodPlanet (4-fold CV)**: Prithvi-CAFE = 64.70(0.02) IoU-water / 68.74(0.03) mIoU / 81.45(0.02) m-F1 @ 45.5M — clean, consistent win over every comparator here (Prithvi-2.0-600M 61.91, -300M 62.03, TerraMind 62.33, U-Net 60.14, DOFA 59.15, TransNorm 60.09/60.19, UViT 59.16), margin +2.4 to +7.2 mIoU.
- Ablation surprise: the config with **no adapter fine-tuning at all** (frozen Prithvi as-is + CNN + M²FAF) still scores 67.06/79.65 mIoU/mDice — second-best, ahead of "adapter-only, no CNN/fusion" (65.43/76.81) — suggesting the CNN+fusion path contributes more than the adapter fine-tuning itself.
- t-SNE embedding visualization: Adaptive-Prithvi (45.5M trainable) shows visibly separable flood/non-flood clusters while tuning only ~7% of encoder parameters, qualitatively comparable to fully fine-tuned Prithvi-600M/TerraMind/DOFA.
- Failure mode: dense cloud cover degrades every model, Prithvi-CAFE included (Supplementary Fig. S1); authors propose SAR fusion (their architecture already supports arbitrary channel counts) as the fix but do not test it in this paper.

### 明确局限

- The strongest same-table baseline (U-Net Base, Li et al. 2023, 31M params) still edges out Prithvi-CAFE on the primary in-distribution benchmark's headline metric (IoU 84.03 vs 83.41, mIoU 90.80 vs 90.50) — a real, checkable gap between the abstract's framing and the paper's own table.
- Headline motivation ("extend Prithvi beyond 6 channels, handle any number of channels") is never actually exercised with a genuinely different modality (e.g., SAR + optical); all reported experiments stay within optical Sentinel-1/2/PlanetScope imagery.
- FloodPlanet has no official split; the paper's 4-fold CV is self-defined, so future comparisons are only valid if the exact fold assignment is released with the code.
- HPO protocol underspecified (algorithm/library for the "20 trials" not named).
- Single, small team (3 authors, 2 institutions), one GPU class (RTX A6000); most other-GFM numbers in Table 1 are copied from each method's own paper rather than re-run in-house under identical preprocessing — comparisons are standard-practice but not strictly apples-to-apples.
- No temporal / multi-timestep modeling anywhere, despite Prithvi-EO-2.0 itself being explicitly built as a "multitemporal" foundation model — this paper only ever does single-timestamp, per-pixel static segmentation.
- No causal, counterfactual, or distributional-robustness machinery of any kind (expected for this genre — a supervised-segmentation benchmark paper — not a deficiency for its own goals, just a marker of how different its validity bar is from causal-effect-estimation work).

### 跟我们的关系

Different research genre from `0710-causal-scs`: this is a supervised-segmentation architecture paper (beat SOTA IoU on public flood benchmarks) with **zero** causal-inference content — no PCMCI+/LKIF, no rolling causal-effect estimation, no dispersion/jump (D,J) instability channels, no sham/confound controls, no paired-trajectory falsification design, no null-calibration audit. It does not compete with, threaten, or overlap the novelty of the causal-scs-indicator project's central claim (whether rolling causal-instability features add SCS warning value beyond raw-state features). Zero scoop risk from this specific paper.

Where it **is** technically adjacent: our own v10/v10b pilot (`run_cnn_fusion_gpu_pilot.py`, `cnn_fusion_helpers.py`) already runs a small from-scratch 3D CNN over the 4D atmospheric grid (pressure-level × time × lon × lat), with matched-architecture raw-vs-raw+causal branches — i.e. we independently arrived at "two feature streams over a gridded field, matched-capacity ablation between information source A and A+B" as an evaluation harness. Prithvi-CAFE's M²FAF (per-scale, sigmoid-gated, $\beta$-stabilized convex combination of a pretrained-encoder stream and a from-scratch CNN stream) is a more sophisticated version of exactly that "fuse two streams" pattern our v10 pilot currently does more crudely (global pooling then concatenation before a shared head). If a future causal-scs-indicator iteration wants a less naive way to combine the raw-field CNN branch with the (D,J) causal-feature branch than a flat concat, CAFE's per-scale attention gate ($attn_i'=attn_i(1-\beta)+\beta$, then $attn_i'\odot A + (1-attn_i')\odot B$) is a directly transplantable few-line mechanism — no need to adopt Prithvi itself, just the fusion-gate idea, and it would slot into the existing `cnn_fusion_helpers.py` position without new estimators.

Prithvi itself is **not** directly reusable for our HRRR/synthetic 4D pressure-level fields: it is pretrained on 6-band HLS optical/spectral reflectance imagery, a different physical modality from NWP pressure-level state variables (geopotential height, wind, temperature at 1000/500 hPa) that our project uses. Even this paper's "arbitrary channel count" extension stays within optical/SAR remote-sensing bands, never generalizing to arbitrary physical variables with different units/statistics. Not worth citing as related work unless we specifically adopt an attention-gated fusion mechanism, in which case cite as the source of that specific technique rather than as a competing method.

## Read by: 0710-causal-scs (2026-07-13)

No novelty conflict with `causal-scs-indicator`: this is a pure supervised-segmentation architecture paper (Prithvi transformer + CNN fusion for flood-map IoU), with no causal-inference machinery of any kind. Do not cite as competing work.

Only transferable piece: the M²FAF attention-gated fusion mechanism (per-scale sigmoid gate + $\beta$-stabilized convex combination of two feature streams) is a strictly better version of the naive concat/mean fusion our v10/v10b CNN pilot (`cnn_fusion_helpers.py`) currently uses to combine the raw-field branch with the (D,J) causal-feature branch. Worth a look if a future iteration needs a learned, per-scale way to weight causal vs. raw features instead of a flat combiner — implementable in a few lines without adopting Prithvi or any pretrained GFM (which is domain-mismatched for HRRR pressure-level grids anyway, since Prithvi is pretrained on 6-band optical HLS reflectance data, not NWP state variables).

Also worth noting for calibration of our own honesty practices: this paper's abstract claims to "outperform baseline U-Net" while its own table shows a stronger U-Net variant (U-Net Base, Li et al. 2023) still winning on the primary in-distribution metric — a useful negative example of the kind of selective-baseline framing `causal-scs-indicator`'s pre-registered-gate discipline (EXPERIMENT_LOG.md, fixed thresholds before each run) is explicitly designed to avoid.
