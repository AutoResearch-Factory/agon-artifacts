# Improving exploration in policy gradient search: Application to symbolic optimization (2107.09158)

- **Year**: 2021.07
- **Authors**: Mikel Landajuela, Brenden K. Petersen, Soo K. Kim, Claudio P. Santiago, Ruben Glatt, T. Nathan Mundhenk, Jacob F. Pettit, Daniel M. Faissol; Lawrence Livermore National Laboratory
- **Venue**: arXiv; source uses ICLR 2021 conference final-copy template
- **Keywords**: symbolic regression, policy gradient search, deep symbolic optimization, hierarchical entropy, soft length prior, risk-seeking policy gradient, neural-guided search, exploration
- **One-liner**: 用层级熵和软长度先验改善神经符号搜索的探索效率。

## 精华提炼

### 管线/方法

- 论文研究对象是 symbolic optimization，重点实验场景是 symbolic regression。
- 输入是一组小规模合成数据点，以及一个由符号 token 构成的表达式库。
- token 库包含二元算子、 unary 算子、终端变量和常数。
- 候选解是变长 token 序列，解释为一棵表达式树或数学表达式。
- 搜索 policy 是 autoregressive RNN，按位置依次生成 token。
- 第 i 步 RNN 输出 logits `psi_i`，经 softmax 得到 token 分类分布。
- 采样出的完整序列 `tau` 用黑盒 reward `R(tau)` 评价。
- policy 的基本更新来自 REINFORCE / policy gradient。
- 每个 batch 采样 M 条候选序列，再根据 reward 更新 RNN 参数。
- 训练目标不是监督拟合标签，而是在组合符号空间中寻找高 reward 表达式。
- 论文继承 Petersen et al. 2019 的 deep symbolic regression 框架。
- baseline 使用 risk-seeking policy gradient，而不是普通期望 reward 最大化。
- risk-seeking 更新用经验 `(1-eps)` 分位数 `R_eps` 作为 baseline。
- 同时 batch 只保留 top `eps` fraction 的高 reward 样本参与更新。
- 这样优化更偏向“找到最好表达式”的 best-case performance。
- 论文指出该类 RNN 搜索有两个探索失败模式。
- 第一个失败模式是 early commitment。
- early commitment 指早期 token 的分布很快塌缩，导致整棵搜索树的早期分支不再被探索。
- 第二个失败模式是 initialization bias。
- initialization bias 指零初始化或均衡 token-type prior 会让初始长度分布强烈偏向最长表达式。
- 方法一是 hierarchical entropy regularizer。
- 标准 entropy regularizer 是对所有时间步 entropy 求和。
- 作者认为标准求和会把有效探索压力集中到较晚 token，因为早期 token entropy 很快接近 0。
- hierarchical entropy 把第 i 个 token 的 entropy 乘以 `gamma^(i-1)`。
- 其中 `gamma < 1`，越靠前的位置权重越高。
- 直觉是让模型持续探索表达式树的早期分支。
- 方法二是 soft length prior。
- 作者先构造一个按 token 类型均衡的 prior `psi_o^eq`。
- 该 prior 让二元、 unary、终端三类 token 的总概率各占三分之一。
- 在这个 prior 下，表达式长度期望为无穷大。
- 实际加最大长度约束后，初始采样会大量堆在最大长度处。
- soft length prior 给第 i 步 logits 额外加一个位置相关项。
- 当 `i < lambda` 时，它惩罚 terminal token，避免表达式过早结束。
- 当 `i > lambda` 时，它惩罚 binary token，避免表达式继续扩张过长。
- 惩罚形状是 Gaussian logit penalty，强度由 `sigma` 控制。
- 因此 SLP 不是硬性长度截断，而是给 RNN 一个可学习的长度偏好。
- 输出是更高的 recovery rate、更少的求解步数、更短的最终表达式。

### 关键技术细节

- policy 记为 `pi(tau; theta)`，候选序列为 `tau = [tau_1, ..., tau_T]`。
- 每个 token 条件分布为 `pi(. | tau_1:(i-1); theta) = softmax(psi_i)`。
- reward `R(tau)` 是黑盒函数，论文没有把方法限定在特定 reward 形式上。
- 参数更新式包含 reward advantage 项和 entropy regularization 项。
- 更新学习率记为 `alpha`。
- baseline 记为 `b`，risk-seeking 情况下设为 `R_eps`。
- entropy weight 记为 `eta`。
- 标准 entropy regularizer 是 `sum_i H[pi(. | prefix_i)]`。
- hierarchical entropy regularizer 是 `sum_i gamma^(i-1) H[pi(. | prefix_i)]`。
- SLP 里的目标长度中心是 `lambda`。
- SLP 里的宽度参数是 `sigma` 或 `sigma^2`。
- benchmark 主实验写明使用 `lambda=10, sigma^2=5`。
- 图示长度分布 caption 里写的是 `lambda=10, sigma^2=20`，看起来是展示图与主实验设置不同。
- hyperparameter grid 包含 `eta in {0.001, 0.005, 0.01, 0.02, 0.03}`。
- hierarchical entropy 的 `gamma` grid 是 `{0.7, 0.75, 0.8, 0.85, 0.9}`。
- SE 最佳 `eta=0.005`。
- HE 最佳 `eta=0.02, gamma=0.85`。
- SLP 最佳 `eta=0.005`。
- SLP+HE 最佳 `eta=0.03, gamma=0.7`。
- 其他超参数沿用 Petersen et al. 2019。
- 实验数据集是 Nguyen symbolic regression suite，共 12 个 benchmark。
- 每个 variant 对 12 个 Nguyen 任务做 100 次 independent runs 的平均。
- 最大求解步数上限为 2,000。
- 评测指标包括 exact recovery rate、steps to solve、best expression length。
- 实验代码基础是 open-source Deep Symbolic Optimization。
- 论文没有在 tex 正文中给出完整 per-task 表格、置信区间或显式随机种子列表。

### 核心结果

| Variant | Recovery | Steps | Best length |
|---------|----------|-------|-------------|
| SE | 83.8% | 592.6 | 19.86 |
| HE | 86.0% | 545.6 | 19.16 |
| SLP | 84.0% | 516.2 | 16.17 |
| SLP+HE | 88.4% | 429.1 | 14.14 |

- HE 单独使用时，recovery 从 83.8% 提升到 86.0%。
- HE 单独使用时，平均求解步数从 592.6 降到 545.6。
- SLP 单独使用时，recovery 几乎不变，只从 83.8% 到 84.0%。
- SLP 单独使用时，求解步数下降更明显，从 592.6 到 516.2。
- SLP 单独使用时，表达式长度从 19.86 降到 16.17。
- SLP+HE 组合效果最好，recovery 达到 88.4%。
- SLP+HE 平均求解步数 429.1，比 SE 少约 27.6%。
- SLP+HE 最终表达式长度 14.14，比 SE 短约 28.8%。
- 图 2 左显示标准 entropy 下早期 token entropy 会较早塌缩。
- 图 2 左的 HE 曲线显示前几个 token 的 entropy 保持更久。
- 图 2 右显示没有 SLP 时初始长度分布堆在最大长度附近。
- 图 2 右显示有 SLP 时长度分布更平滑，更接近可学习的先验。
- 一个反直觉点是：普通 entropy regularization 并不保证搜索树早期分支保持探索。
- 另一个关键点是：看似“均衡”的 token-type prior 会导致长度分布严重偏长。
- 作者声称这些结果超过 Petersen et al. 2019 的 SR state of the art。
- 论文也声称 baseline 已优于若干商业 SR 软件，因此这里是在 neural-guided SR 上继续推进。

### 明确局限

- 主要实验只覆盖 Nguyen 12 个标准 synthetic SR benchmark。
- 没有展示 Feynman、ODE、真实科学数据、噪声数据或高维真实任务上的结果。
- exact recovery 是清晰指标，但可能高估方法在真实科学发现中的可靠性。
- 作者没有报告 wall-clock time、GPU/CPU 成本或 memory footprint。
- 作者没有给每个 benchmark 的完整分项结果，正文只有平均表。
- 作者没有给置信区间、standard error 或显著性检验。
- 100 次 independent runs 是优点，但负面案例和失败表达式没有展开。
- SLP 的效果可能依赖表达式文法、最大长度和 token library 的 arity 比例。
- HE 的最佳 `gamma` 和 `eta` 需要 grid search，迁移到新任务时可能要重调。
- 论文没有证明 HE 一定优于更一般的位置加权 entropy 或 count-based tree exploration。
- 论文没有和 Monte Carlo Tree Search、genetic programming 最新变体做大规模公平比较。
- 论文使用 risk-seeking DSR 作为基础，HE/SLP 与 risk-seeking filter 的贡献可能互相耦合。
- SLP 是手工先验，不是从数据自动学习出来的结构先验。
- 方法针对离散符号序列很自然，但对连续参数、单位约束、维度约束没有直接处理。
- 作者自己承认 symbolic spaces 的组合性质使 exploration 是核心瓶颈，但没有解决 reward misspecification。
- 对科学方程发现来说，找到高 reward 表达式不等于物理可解释或可外推。

## Read by: 0616-fex (2026-06-16)

从 `0616-fex` 视角看，这篇论文的核心价值不是新 benchmark，而是把 neural symbolic search 的探索失败拆成两个很具体的机制：早期 token 分支塌缩，以及初始长度分布偏向最大长度。若我们的工作也生成表达式、特征或方程，尤其使用 policy gradient、LLM policy optimization 或 token-by-token 采样，那么 HE 和 SLP 都是必须比较的 prior art。

撞车点主要在 claim 层：如果我们声称“用 token 位置相关正则改善表达式搜索探索”，本文已经做了最小而清晰的版本；如果我们声称“给表达式长度加入 soft prior 能提升 SR”，本文也已经覆盖。差异化应放在更具体的任务设定上，例如 feature extraction、physics-informed constraints、noisy real data、dimensional consistency、LLM proposal + independent verifier，或把长度/结构 prior 从手工 Gaussian 改为数据驱动。

可直接复用的实验设计是 SE/HE/SLP/SLP+HE 四组 ablation、token-position entropy 曲线、初始长度分布诊断、100-run exact recovery 评估。建议引用它作为 RL symbolic regression exploration 的基础文献，同时在下一轮重点查 2025-2026 的 FePySR、DrSR、LLM-Based Scientific Equation Discovery 和 DSO survey/monograph，判断 `0616-fex` 是否已被 LLM 方程发现方向部分覆盖。
