# Oscillatory State-Space Models as Inductive Biases for Physics-Informed Neural PDE Solvers (2606.02623)

- **Year**: 2026.06
- **Authors**: Abhishek Chandra (KTH Royal Institute of Technology), Taniya Kapoor (Wageningen University & Research)
- **Venue**: arXiv preprint; NeurIPS 2026 style file in source
- **Keywords**: PINN, state-space model, LinOSS, oscillatory dynamics, spectral basis, neural PDE solver, inverse PDE, high-dimensional PDE
- **One-liner**: 用振荡 SSM 和谱基函数改造 PINN 时序求解。

## 精华提炼

### 管线/方法
1. 输入是一类时间依赖 PDE：空间域 \(\Omega\)、时间区间 \([0,T]\)、初始条件 \(u_0(x)\)、边界条件和 PDE residual \(\mathcal{F}\)。
2. 论文不再用标准 PINN 的坐标 MLP 直接学习 \((x,t)\mapsto u(x,t)\)。
3. 核心 ansatz 是 \( \hat u(x,t)=D(x)\sum_{k=1}^K c_k(h(t))\phi_k(x)+g(x) \)。
4. 其中 \(\phi_k(x)\) 是固定解析空间基，\(D(x)\) 是硬边界条件因子，\(g(x)\) 处理非齐次边界。
5. 初始条件先在固定 IC 采样点上取样，拼成 \(U_0=[u_0(x_1),...,u_0(x_{N_{\rm IC}})]\)。
6. IC encoder \(g_e\) 把 \(U_0\) 映射到 LinOSS 初始状态 \((y_0,z_0)\in R^{2H}\)。
7. 时间演化由 LinOSS cell 负责：\(\dot y=z,\dot z=-Ay+Bs(t)\)。
8. \(A\) 是非负对角矩阵，参数化为 \(A=\mathrm{ReLU}(\hat A)\)，每一维对应一个 forced harmonic oscillator。
9. 隐状态的角频率是 \(\omega_i=\sqrt{A_i}\)，连续时间特征值位于虚轴 \(\pm i\sqrt{A_i}\)。
10. 论文使用两种离散化：OSSM-PINN-IMEX 和 OSSM-PINN-IM。
11. IMEX 版本保持自主演化部分辛结构，离散谱在单位圆上，适合守恒/振荡问题。
12. IM 版本引入可控耗散，离散谱落在单位圆内，适合耗散动力学。
13. 递推写成 \(\xi_{n+1}=M\xi_n+F_n\)，并用 associative parallel scan 计算整段时间轨迹。
14. 这个 scan 使时间 rollout 在并行硬件上具有 \(O(\log N_t)\) depth，而不是顺序 ODE rollout 的 \(O(N_t)\) depth。
15. 得到的时间隐状态 \(h(t_n)=y(t_n)\) 输入 modewise decoder \(g_d:R^H\to R^K\)。
16. decoder 输出每个时间步的 \(K\) 个空间模态系数 \(c_k(h(t_n))\)。
17. 空间端用问题相关解析基：周期区间用 Fourier features，Dirichlet 区间用 sine modes。
18. 多维矩形域用 tensor-product basis，非矩形三角域构造能在三条边上消失的解析基。
19. 若基函数本身不能满足边界条件，则用 \(D(x)\) 乘 raw modal sum 实现 hard BC。
20. 空间导数不通过空间 MLP autograd，而是预计算基函数及其导数，再按 Leibniz rule 组合。
21. 时间导数由预测序列上的有限差分给出，默认使用 6 阶 finite difference，边界处用 one-sided stencil。
22. loss 是 PDE residual loss 加 IC loss，forward 默认 \(\lambda_{\rm PDE}=1,\lambda_{\rm IC}=100\)。
23. inverse problem 额外加入 data loss，默认 \(\lambda_{\rm data}=100\)，未知 PDE 系数作为 trainable scalars 一起优化。
24. 优化流程是 Adam warm-up 接 L-BFGS，L-BFGS 使用 strong-Wolfe line search。
25. 实现默认 double precision；消融显示 FP32 会明显降低精度。

### 关键技术细节
26. LinOSS hidden width 在主超参数表中为 128。
27. coefficient MLP width 也是 128，并启用 locally adaptive activation。
28. L-BFGS history size 为 50，learning rate 为 1.0，max inner iterations 为 20。
29. L-BFGS gradient tolerance 为 \(10^{-12}\)，change tolerance 为 \(10^{-14}\)。
30. 实验硬件是单张 NVIDIA RTX 5090 Laptop GPU，24 GiB VRAM。
31. 软件环境为 Ubuntu 24.04、PyTorch 2.10.0+cu128、Python 3.11.6。
32. Baselines 包括 PINNsFormer、PINNMamba、ML-PINN 和 NeuSA。
33. PINNsFormer/Mamba 类方法把时间作为 pseudo-sequence，空间网格也被卷入 sequence token，因此显存随 \(N_t\times N_x\) 增长。
34. NeuSA 使用固定谱基和显式 RK4 rollout，但其时间演化不是从 PDE residual 学出来的，并且 rollout 顺序化。
35. OSSM-PINN 的关键区别是：时间演化可学习、具有振荡谱约束、可并行 scan，并与空间基解耦。
36. 论文给出 residual consistency proposition：basis truncation error、LinOSS coefficient approximation error 和 \(\Delta t^{2p}\) 三项共同控制残差。
37. finite-mode corollary 表示若真实解恰好是有限模态且 cell/decoder 可精确表示，残差只剩有限差分误差项。
38. Cell-swap 消融固定 encoder、decoder、basis、optimizer 和参数预算，只替换 temporal cell。
39. 对流 \(\beta=50\) 上 LinOSS-IMEX rMAE 为 \(3.039e{-5}\)，S5 为 0.008，LRU/Mamba/NCDE 等接近 0.86-0.99。
40. Reaction 上 LinOSS-IMEX rMAE 为 0.003，最近的 Mamba 为 0.072，仍差一个数量级以上。
41. 关键组件消融显示去掉 LinOSS 后 convection、Euler-Bernoulli、wave 的 rMAE 都约为 1。
42. 去掉 Fourier/spectral basis 后 convection IMEX rMAE 从 \(3.04e{-5}\) 退化到 0.710，wave 到 1.00。
43. 把 6 阶时间差分换成低阶后 convection rMAE 从 \(3.04e{-5}\) 退化到 \(2.75e{-2}\)。
44. FP32 消融同样恶化：convection IMEX rMAE 到 \(7.97e{-4}\)，wave 到 \(8.30e{-3}\)。

### 核心结果
45. Forward benchmark 覆盖 convection、reaction、wave、Euler-Bernoulli、Taylor-Green vortex、Schrodinger 等 12 个设置。
46. Convection \(\beta=50\)：OSSM-IMEX rMAE \(3.04e{-5}\)，最佳 baseline NeuSA 为 \(3.24e{-3}\)，提升约 107x。
47. Convection \(\beta=100\)：OSSM-IMEX rMAE \(1.48e{-5}\)，NeuSA 为 \(3.24e{-3}\)，提升约 219x。
48. Reaction：OSSM-IM rMAE \(2.92e{-3}\)，PINNsFormer 为 \(1.12e{-2}\)，提升约 4x。
49. Wave：OSSM-IMEX rMAE \(4.15e{-5}\)，NeuSA 为 \(5.48e{-3}\)，提升约 132x。
50. Euler-Bernoulli classical：OSSM-IM rMAE \(6.58e{-5}\)，ML-PINN 为 0.328，NeuSA 为 3.35，Transformer/Mamba OOM。
51. Euler-Bernoulli extended \([0,8\pi]\)：OSSM-IM rMAE \(1.08e{-5}\)，ML-PINN 为 1.06，提升约 \(9.84e4\) 倍。
52. Taylor-Green 2D：四个 baseline 全部 OOM，OSSM-IM rMAE \(2.64e{-3}\)。
53. Triangle heat：NeuSA 被标为 geometry-infeasible，sequence baselines OOM，OSSM rMAE 约 \(7.66e{-3}\)。
54. Schrodinger 5D：OSSM-IM rMAE \(1.11e{-3}\)，其他 baseline OOM。
55. Schrodinger 100D：OSSM-IMEX rMAE \(8.74e{-4}\)，其他 baseline OOM；图中 100D slice 误差低于 \(5e{-4}\)。
56. KdV inverse：OSSM-IM 恢复 \(\lambda_1=0.99932\)、\(\lambda_2=0.0024986\)，相对误差 0.07% 和 0.06%。
57. SST 2D inverse：OSSM-IMEX 恢复 \(c_x=0.49971\)、\(c_y=0.048107\)、\(\kappa=0.010215\)，误差 0.06%、3.79%、2.15%。
58. Problem-adapted basis 上，QHO 从 Fourier basis rMAE \(9.51e{-3}\) 降到 Hermite basis \(1.87e{-4}\)，约 50x。
59. Poschl-Teller problem 从 Fourier basis rMAE \(6.63e{-2}\) 降到 PT basis \(1.68e{-3}\)，约 40x。
60. 频谱诊断显示 OSSM-PINN 能跟踪 reference spectrum 到 FFT noise floor，NeuSA 在高频处有 aliasing plateau。
61. Latent diagnostics 显示 sequence baselines 常退化为低秩 smooth drift，而 OSSM 隐状态保留多维振荡模态。
62. Loss landscape 上，convection \(\beta=100\) 的 \(L_{\rm Lip}^{\log}\)：OSSM-IMEX 为 13.2，PINNMamba 为 16.7，PINNsFormer 为 37.9。
63. 计算成本上，wave benchmark OSSM peak GPU 144 MiB，而 PINNsFormer 1454 MiB、PINNMamba 6301 MiB。
64. 推理延迟上，OSSM 通常比 sequence baselines 快 8-23x；convection \(\beta=50\) OSSM 约 3.8 ms，PINNsFormer 88.65 ms。

### 明确局限
65. 作者明确承认：方法假设目标解能由有限个空间模态和结构化时间演化良好表示。
66. 对 shocks、moving discontinuities、intermittent localized structures、strongly non-modal dynamics，固定全局基可能需要很多 modes。
67. 当前实现使用固定 collocation points、uniform temporal grids 和手选 problem-aware bases。
68. 这对多尺度或高度局部化现象可能低效，作者建议未来做 adaptive residual sampling 和 adaptive temporal refinement。
69. 自动 basis selection 还没有做；目前 Fourier、sine、Hermite、PT、triangle basis 都是人工选择。
70. 论文没有看到公开代码或 artifact 链接，复现实验需要重建不少工程细节。
71. 反引文截至 2026-06-16 在 S2 为 0，说明这篇还太新，没有外部独立验证。
72. 所有 benchmark 似乎都在作者单机环境内完成，没有第三方复现或 cross-lab validation。
73. 很多 baseline 在高维/2D设置下被标为 OOM，这能证明资源优势，但也让 accuracy comparison 变成可运行性比较。
74. SST inverse 的“真实值”来自用 advection-diffusion 方程和给定参数生成的传统解，而不是真实海洋动力学真值。
75. 论文把 NeuSA 标为 stiff/geometry infeasible 的部分需要代码级核查，否则容易受实现选择影响。
76. 对 100D Schrodinger 的设置是高度结构化解析模态和低秩 IC，不代表一般 100D PDE。

## Read by: 0616-fex (2026-06-16)

本次按 `0616-fex` 视角看，这篇是“频谱/振荡归纳偏置 + PDE solver”方向的强相关 prior art。核心创新不是发明新的 SSM，而是把 LinOSS 的虚轴振荡谱放进 PINN temporal coefficient dynamics，并用固定解析空间基把 PDE 求解转成模态系数的时间演化。若我们的课题包含 frequency extrapolation、Fourier basis、PDE time extrapolation、physics-informed residual learning 或 structured sequence prior，这篇会撞到“振荡时序 bias 提升 PDE 泛化/稳定性”的主 claim。

可直接复用的技术路线是：IC encoder 初始化 LinOSS 状态，IM/IMEX rollout 产生 \(h(t)\)，modewise decoder 输出 \(K\) 个谱系数，空间基提供闭式导数和 hard BC，时间导数用 6 阶 finite difference，loss 仍然是 residual + IC/data。可直接复用的实验论证是 cell-swap：固定所有模块只换 temporal cell，证明收益来自 oscillatory eigenstructure 而不是参数量。对我们最有价值的是它的负面边界：固定全局 basis 对 shocks、移动不连续、局部间歇结构可能失效，自动/局部/多分辨率 basis 仍是缺口。

差异化建议：不要只声称“用 SSM/频谱结构解 PDE”，这已经被本文覆盖。更稳的方向是做自动 basis selection、localized wavelet/spectral basis、non-modal transient dynamics、含 shocks 的 conservation laws、真实观测数据上的 inverse PDE，或把 FEX 的贡献定义为“发现/选择/验证频率外推结构”的系统，而不是单个 LinOSS-PINN 架构。引用上应把本文列为直接相关工作，并同时引用 LinOSS、PINNsFormer、PINNMamba/SSM-PINN、NeuSA。
