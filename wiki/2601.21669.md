# Expected Return Causes Outcome-Level Mode Collapse in Reinforcement Learning and How to Fix It with Inverse Probability Scaling (2601.21669)

- **Year**: 2026.01
- **Authors**: Abhijeet Sinha (NUS), Sundari Elango (IIT Madras), Dianbo Liu (NUS)
- **Venue**: ICML 2026 (submission, icml2026 style)
- **Keywords**: outcome-level mode collapse, inverse probability scaling, GRPO, reward-proportional distribution, multimodal RL, KL regularization, expected return, policy gradient
- **One-liner**: 证明 expected-return maximization 内在导致 outcome 层面的 mode collapse（并非探索不足），并提出 Inverse Probability Scaling (IPS) 通过除以 outcome 概率来消除频率偏置，使策略收敛到 reward-proportional 分布。

## 精华提炼

### 管线/方法

整体 pipeline 分为三步：
1. **理论分析阶段**：将 outcome-multimodal RL 归约为 outcome-selection bandit（去除状态动力学和探索噪声），在 softmax 参数化下分析 gradient flow 的 ODE。证明 log-ratio 动力学为 d/dt log(p_i/p_j) = p_i*a_i - p_j*a_j，即概率比例由自身概率 p_i 乘以优势函数 a_i 驱动——这构成了"富者愈富"的正反馈回路。
2. **IPS 原理提出**：将 terminal reward r(o) 替换为 r(o)/p_θ(o)（stop-gradient 分母），得到 IPS 目标 J_IPS = E[ r(o)/p(o) ]。在该目标下，gradient flow 变为 dz_i/dt = r_i - p_i*Σr_k，稳态解为 p_i* = r_i / Σr_k（reward-proportional）。
3. **IPS-GRPO 算法**：在 GRPO 框架内实例化 IPS——对每个 group 内的 G 条轨迹，计算 empirical outcome frequency p̂(o)，将 reward 缩放为 r̃(o) = r(o) / max(p̂(o), ε)，然后执行标准 GRPO 更新。

输入：policy π_θ, terminal reward function r(o), group size G, clipping ε
输出：更新后的 policy，其 outcome 分布趋向 r(o)-proportional

### 关键技术细节

- **理论设定**：outcome-selection bandit（finite outcome set O={1,...,K}），softmax parameterization p=softmax(z)，gradient flow（连续时间 ODE）。核心假设：reward 仅依赖 terminal outcome o=φ(τ)。
- **IPS 的核心 trick**：stop-gradient 分母。p_θ(o) 用作标量除数但不参与梯度回传。这使得 J_IPS(z) = Σ_k p_k * (r_k / p̄_k) = Σ_k r_k = R 是常数（不依赖 z），但 gradient 不为零：∂J_IPS/∂z_i = r_i - R*p_i。
- **稳态解**：p_i* = r_i / Σr_k。势函数 Ψ(z) = log(Σe^{z_k}) - Σ p_k* z_k 沿 gradient flow 单调递减，保证全局收敛到 p*。
- **KL regularization 下的行为**：IPS + KL 退化为 reward-proportional 分布与 reference policy 之间的插值，β 控制 trade-off；KL 不再需要"对抗 collapse"。
- **IPS-GRPO 超参数**：group size G ∈ {4,8,16,32,64}，clipping ε ∈ {0.01,0.1,0.2}。Ablation 显示 bias-variance trade-off：G 大则 p̂(o) 准确但小概率不稳定，ε 大则稳定但偏置增大。最优组合（HypoSpace causal）：G=16, ε=0.2 → RR=43.91%。
- **实验模型**：HypoSpace 使用 Gemma-3 4B Instruct + 4-bit NF4 + LoRA (r=16, α=32, target q/k/v/o)；Drug discovery 使用 Mamba 12-layer (hidden=256, ~5.27M params) pretrained on ChEMBL 33，REINVENT 框架 fine-tune（batch=64, lr=1e-4, KL coeff=128, oracle budget=10K）。
- **Drug discovery reward**：4-component geometric mean（docking score via QuickVina2-GPU, QED, HBD count, synthesizability via Syntheseus/MEGAN）。SYNTH 和 ALL-AMIDE 两个任务。

### 核心结果

| 实验 | 指标 | GRPO | FlowRL | IPS-GRPO |
|------|------|------|--------|----------|
| Hyper-grid n=2 | ℓ1 distance ↓ | 0.028 | 0.028 | **0.003** |
| Hyper-grid n=4 | ℓ1 distance ↓ | 5.2e-4 | 5.2e-4 | **3.7e-4** |
| HypoSpace Causal | Recovery Rate ↑ | 16.02% | 12.82% | **43.91%** |
| HypoSpace 3D Voxel | Recovery Rate ↑ | 31.61% | 26.85% | **90.00%** |
| HypoSpace Boolean | Recovery Rate ↑ | 9.50% | 10.74% | **60.74%** |
| Drug SYNTH (0.80) | Yield ↑ | REINVENT: 5695 | — | **7371** |
| Drug ALL-AMIDE (0.80) | Yield ↑ | REINVENT: 1 | — | **1478** |

**Surprising findings**:
- 即使 reward 完全相同，expected-return 也会因 path multiplicity 差异（35 paths vs 7 paths）导致 collapse。GRPO 会崩塌到路径数多的 outcome A。IPS-GRPO 保持两者频率稳定。
- FlowRL 在 HypoSpace 上并不比 GRPO 好（甚至更差），说明 flow-based 方法在此类任务上无优势。
- ALL-AMIDE 任务 REINVENT 几乎找不到分子（Yield=1），IPS-GRPO 找到 1478 个，差异极大。

### 明确局限

1. **IPS 依赖 finite outcome empirical frequency estimation**：在大或连续 outcome space 中，group-based 频率估计噪声大，probability clipping 引入 bias-variance trade-off。
2. **IPS 不适用于 Actor-Critic 方法**（如 PPO with learned critic）：因为这类方法依赖 value estimates 而非显式的 outcome frequencies。IPS 目前仅适用于 group-based on-policy 方法（如 GRPO）。
3. **IPS 针对 terminal rewards**，扩展到 dense-reward 或 continuous-outcome 场景仍需工作。
4. **没有真机实验**：所有实验在模拟环境（grid、LLM LoRA fine-tune、CLM）中完成。
5. **没有与 entropy regularization 的系统消融对比**：虽然理论证明 IPS 独立于 entropy/KL，但实验未展示不同 entropy 强度下 IPS 与 baseline 的对比。
6. **收敛到 reward-proportional 分布可能是缺点**：如果 reward 本身 biased，IPS 会忠实地反映这种 bias，而不会像 standard RL 那样可能通过 collapse 意外纠正 reward misspecification。

### 跟我们的关系

对 **0616-fex** 课题（FEX 的 RL 搜索效率）：

1. **撞车评估**：不撞车。本篇完全不涉及 FEX 或符号回归。但核心洞察——"RL 的 expected-return 目标内在导致 outcome 多样性崩塌"——对 FEX 的 RL controller 设计有直接启发：
   - FEX 的 controller 用 policy gradient 搜索表达式树，搜索空间中可能存在多个等价的符号表达式（不同树结构表示同一数学公式），这正是 outcome-multimodal 场景。
   - 如果 FEX 的 controller 也受 mode collapse 影响（偏向某些"容易发现"的表达式形式而漏掉等价但结构不同的表达），IPS 可能是一种轻量级修复。

2. **可用工具/方法**：
   - **IPS-GRPO** 可作为 FEX controller 的替代优化器（当前 FEX 用 vanilla policy gradient / REINFORCE）。IPS-GRPO 仅需 group-based sampling + reward rescaling，不引入 auxiliary model，可 drop-in 替换。
   - **IPS 原理本身**更通用：任何 outcome-multimodal RL 都可将 terminal reward 除以 outcome frequency。如果 FEX 的 binary tree 表达式可聚类为 outcome（如根据数学等价性 hash），即可直接应用。
   - **Section 6 的"Why IPS works"分析**（path multiplicity bias）对诊断 FEX controller 的搜索偏置有方法论价值——可以类比分析"表达式 A 比表达式 B 有更多生成路径"导致的 bias。

3. **差异化空间**：
   - 本篇针对 LLM post-training 和分子生成，未涉及符号搜索/数学表达式空间。
   - FEX 场景的独特 challenge：outcome space 是组合爆炸的（expression trees），empirical frequency estimation 在小 group 中极不准确。如何定义 FEX 的"outcome"（expression equivalence class?）是未解决的问题。
   - 可以将 IPS 与 FEX 的 tree search 结合，形成"IPS-guided tree search"，在搜索过程中保持 expression diversity。

### 撞车评估
**不撞车。** 本文研究 RL 中 outcome-level mode collapse 的理论根源和 IPS 修正，完全不涉及 FEX（Finite Expression Method）或 PDE 求解。但理论上高度相关：FEX 的 RL controller 在表达式树空间中搜索，等价表达式对应相同"outcome"，同样可能受 mode collapse 影响。

### 可用工具
- **IPS 原理**：可被 FEX controller 直接采用，将 reward 按表达式频率缩放，避免搜索崩塌到"容易找到"的表达式形式。
- **IPS-GRPO**：group-based 的轻量级算法，可 drop-in 替换 FEX 当前使用的 vanilla policy gradient。
- **Section 6 的诊断方法**（path multiplicity bias + quiver/force field 可视化）：可用于分析 FEX controller 的搜索偏置。

### 差异化
- 本文针对 LLM post-training 和分子生成，未涉及数学表达式/符号搜索空间。
- FEX 场景的核心挑战是：如何在组合爆炸的表达式树空间中定义"outcome"的等价类（语义等价 vs 句法等价的 tree hashing）。
- IPS 的 empirical frequency estimation 在 FEX 的大型搜索空间中可能极不可靠——这是需要解决的独特问题。
