# Architecture-Induced Recoverability Bias in Differentiable Symbolic Regression (2604.23256)

- **Year**: 2026.04
- **Authors**: Chakshu Gupta, Theodore J. LaGrow; Georgia Institute of Technology
- **Venue**: arXiv
- **Keywords**: differentiable symbolic regression, recoverability, representability, variable routing, EML operator, gradient asymmetry, hardening, validation selection
- **One-liner**: 固定树路由会强烈改变符号回归可恢复性。

## 精华提炼

### 管线/方法
- 论文研究的是 fixed-tree differentiable symbolic regression，而不是通用 SR benchmark 排名。
- 输入是无噪声数值数据，目标是从数据恢复闭式表达式，并在 hardening 后得到可检查的符号树。
- 核心问题被拆成两个概念：表达式是否在架构可表示集合内，以及训练加 hardening 是否能把它恢复出来。
- 作者定义架构 `A` 的 representable set `E(A) = {H(A, theta)}`，其中 `H` 是 selector snap 后的硬化表达式。
- 作者定义 recoverability `R(A, f*)` 为随机初始化和训练过程下，硬化表达式结构和数值都匹配目标 `f*` 的概率。
- 实验固定深度为 3，因此每棵完整二叉模板有 7 个 internal nodes 和 8 个 leaves。
- 论文比较三种 depth-3 variable-routing architecture：Eq. 6、V16、Hybrid。
- Eq. 6 在每个 internal node 用三路 softmax 选择 `{1, x, f_child}`，因此 `x` 可在各层进入，`y` 只在叶子进入。
- V16 在 internal node 用 sigmoid 选择 `{1, f_child}`，并只在叶子用三路 softmax 选择 `{x, y, 1}`，对 `x/y` 更对称。
- Hybrid 大体继承 V16，但在 root 加四路 softmax，让 `x/y` 可以直接进 root。
- 目标表达式来自五种 active-subtree shape：LR、RL、RR、LL、Balanced。
- LL 会产生三重指数，在训练域内溢出，因此从主实验排除。
- Balanced 是两边 equal-depth subtree 的拓扑，作为链式结构的对照。
- 每个 shape 有 `(x,y)` 和 `(y,x)` 两种 leaf assignment。
- 主 operator 是 EML：`eml(a,b) = exp(a) - ln(b)`。
- EML 的左输入梯度 `exp(a)` 会放大，右输入梯度 `-1/b` 会衰减，这是论文解释路由偏置的关键。
- 为了测试是否只是 EML 特例，作者还构造 SML 和 RML 两个 operator。
- SML 是 `sinh(a) - arctan(b)`，保持左侧梯度放大、右侧梯度有界的方向。
- RML 是 `arctan(a) - sinh(b)`，反转了梯度不对称方向。
- 数据域是 `[1,3]^2` 上 `21 x 21` 网格，共 441 个训练点。
- 训练域远离 0，因为 EML 的 `ln b` 需要 `b > 0`，且 `-1/b` 在 0 附近会导致梯度不稳定。
- 训练用 Adam，学习率 `0.01`。
- search phase 运行 6000 iterations，temperature `tau = 2.5`。
- hardening phase 运行 2000 iterations，`tau` 从 2.5 anneal 到 0.01，并加入 entropy 和 binarity penalties。
- hardening 结束后每个 selector 做 one-hot snap，剪掉未选分支，得到 concrete expression tree。
- V16 和 Hybrid 每个 cell 运行 256 trials：64 seeds 乘以 4 种 initialization strategy。
- Eq. 6 每个 cell 运行 64 trials，使用原始论文的 `randn x 0.1` 初始化。
- V16/Hybrid 的四种初始化是 biased、uniform、xy-biased、random-hot。
- exact recovery 同时要求结构匹配和数值匹配。
- 结构匹配用 canonical signature，处理对称节点的 commutativity，并要求 leaf weight 在 `1e-6` 内。
- 数值匹配要求训练 MSE 小于 `1e-20`，held-out RMSE 小于 `1e-6`。
- held-out 集是 `[0.5,5.0]^2` 上均匀采样的 500 点，比训练网格范围更宽。
- 每次 trial 记录 12 个字段，包括 seed、initialization、architecture、target、operator、train/held-out RMSE、hardened expression、routing、structural match、failure type、wall time。
- 作者额外做 sensitivity screen，覆盖 optimizer、hardening schedule、clamp magnitude、held-out seed、training noise、numerical epsilon。
- sensitivity screen 包括 5248 个 OFAT trials 和 chain cells 的 factorial。
- 主实验固定 Adam `1e-2`、power-law hardening、clamp `1e6`、grid seed `12345`、noise `0`、epsilon `1e-12`。

### 关键技术细节
- 这篇论文的创新点不是提出一个更强 SR 算法，而是隔离并测量 architecture-induced recoverability bias。
- 主要 controlled variable 是 variable-routing architecture；operator、grammar、depth、训练协议尽量固定。
- Eq. 6 的 representable set 是 V16 的严格超集，因为 internal node 可选项包含 V16 的 `{1, f_child}`。
- 但 representable superset 并不保证 recoverability 更高，这是论文最重要的反例。
- Eq. 6 在某些 V16 近乎确定恢复的目标上直接失败，说明失败不是表达式不存在，而是训练加 hardening 到不了。
- one-hot snap 是主要失败门槛：汇总 5376 runs 后，77% soft phase loss 能到 `<0.01`，但只有 32% snap 后还能保持 hardened training MSE `<0.01`。
- 一旦 snap 后仍满足 `<0.01`，这些 run 都满足更严格 exact-match tolerance。
- 这说明 differentiable relaxation 的连续最优和离散 hard expression 之间存在明显落差。
- 梯度诊断使用 `rho_xy = ||grad theta_x L|| / ||grad theta_y L||` 衡量两个变量叶子的梯度信号。
- 另一个诊断是 `rho_LR`，比较 internal node 左右子分支的梯度 norm。
- 梯度在 iteration 1000 测量，报告 10 seeds 的 mean 和 standard error。
- Eq. 6/EML 的 recovered LR target 在 1000 步时 `rho_xy = 0.67 +/- 0.03`，恢复率 100%。
- Eq. 6/EML 的 failed RL target 在 1000 步时 `rho_xy = 1.58 +/- 0.04`，恢复率 0%。
- trajectory 显示 recovered LR target 的 `rho_xy` 会从 iteration 100 的 1.45 短暂跌到 iteration 500 的 0.14，再到 1000 的 0.67。
- failed target 的 `rho_xy` 全程高于 1.14，作者把这解释为必要变量路径的梯度 starvation。
- V16 能在 `rho_xy = 2.31 +/- 0.24` 的 EML/RL target 上达到 99.6%，说明对称 leaf routing 能补偿某些变量梯度不平衡。
- Balanced 的 branch ratio 只有约 2 到 3，弱于成功链式结构上的 10 到 17 倍不对称。
- 作者没有把 gradient ratio 说成因果机制，只说它是 diagnostic correlate。
- Signal-model audit 检查了 Shockley diode、log compression、Friis/path loss、Shannon capacity、Sabine/Hill 等常见公式类。
- Shockley diode 的 normalized 形式 `eml(x,e)` 在 EML 中 depth-1 可表示。
- log-only response `eml(0, .)` 在当前 leaf alphabet `{1,x,y}` 下不可达，因为 zero leaf 不能合成。
- Friis/path loss 和 Shannon capacity 受阻于 addition，Sabine/Hill 受阻于 reciprocal。
- 论文代码链接写在脚注中：`https://github.com/ChakshuGupta13/lab/`。

### 核心结果
- 同一个 target 在不同 architecture-plus-native-training-protocol 下可以从 `0/64` 变到 `64/64`。
- 没有单一 architecture dominate：最佳架构在一个目标上可能是另一个目标的最差架构。
- Eq. 6 像 specialist：两个目标 100%，三个目标 0%，剩下一个低于 2%。
- V16 在四个 chain targets 上都有恢复，范围是 16% 到 99.6%。
- Hybrid 是 RL specialist，在 RL targets 上 95% 到 96%，其他目标接近 0%。
- Eq. 6 在某处达到 100% 而 V16 约 17%；V16 在另一处 99.6% 而 Eq. 6 为 0%。
- Balanced topologies 在所有测试配置中全部失败：`0/3776`，95% Clopper-Pearson upper bound 为 0.10%。
- Balanced 失败不是单一训练设置导致，因为额外六种训练配置也得到 `0/192`。
- EML balanced cells 多在 search 阶段发散，loss 到 `1e16`。
- SML balanced cells 往往 soft fit 可到达，但 hardening 时发生 chain collapse 或 snap mismatch。
- 切换 EML 到 SML 会改变 Eq. 6 的 preferred shapes：LR 从 100% 掉到 0%，RL(xy) 和两个 RR variants 到 100%。
- RML 反转梯度不对称后整体 collapse：Eq. 6 所有 targets 0%，V16 best 仅 2.0% 即 `5/256`。
- Validation selector 的 proof-of-concept 在 jointly-run subset 上把 V16-only baseline 从 `264/768 = 34.4%` 提到 `385/768 = 50.1%`。
- 这个 selector 是训练小 architecture set，hardening 后用 held-out RMSE 选最低者。
- selector 在 subset 上能 match per-target oracle，是因为 exact recovery 和 non-recovery 的 held-out RMSE 相差约五个数量级。
- Shockley diode 上，Eq. 6 恢复 `18/32` seeds，Hybrid 恢复 `22/32`，V16 恢复 `0/32`。
- Shockley diode 上 selector 恢复 `28/32`，因为 Eq. 6 和 Hybrid 的成功 seed 互补。
- V16 在 Shockley diode 上失败是 representational failure，而不是 optimization failure。
- 原因是 V16 的 root 左输入只能是常量或 depth <= 2 的 V16 subtree 输出，枚举后没有表达式等于 `x`。
- 最反直觉的结果是 representable superset Eq. 6 会输给 representable subset V16。
- 另一个反直觉结果是 balanced tree 看起来结构更对称，却在全部 tested configurations 中不可恢复。

### 明确局限
- 结论只在 depth-3 fixed-tree differentiable SR 上严格成立。
- 论文没有证明所有 differentiable SR architecture 都存在相同 bias。
- operator family 主要是 `f(a)-g(b)` 形式，addition、multiplication、composition 作为 primitives 没有系统测试。
- SML/RML 只是梯度不对称方向的 controlled probes，不是实际 SR grammar 的完整替代。
- depth-4 只做了 limited V16 sweep，并且 RL 从 99.6% 掉到 0.8%，提示 depth-3 的 generalist profile 不能外推。
- 训练目标无噪声，selector 依赖 held-out RMSE 的清晰分离；有噪声数据下 exact/non-exact 的 RMSE gap 可能缩小。
- selector 只在三个 jointly-run synthetic configurations 上评估，不能视为完整 benchmark。
- Eq. 6 和 V16 的 native initialization protocol 不同，所以二者部分比较应读作 architecture-plus-native-training-protocol，而不是纯 architecture。
- 作者虽做 sensitivity screen，但 initialization 仍然是 cell-dependent 的主要因素。
- 梯度 ratio 是 correlate，不是经干预验证的 causal mechanism。
- 论文没有和 PySR、DSR、SINDy 等非 fixed-tree 方法做公平 end-to-end 性能比较，因为研究问题不同。
- 论文没有标准真实噪声 benchmark，Shockley diode 只是代表性检查。
- 当前 grammar 不能合成 zero、addition、reciprocal，因此许多 signal-processing 公式在 depth-3 audit 中直接 blocked。
- 代码虽给出链接，但论文正文没有 artifact hash、完整日志位置或独立复现报告。
- 对 `<topic_slug>=0616-fex` 而言，若我们研究的是更宽的 formula extraction 或 symbolic discovery，本篇提供的是 failure mode 和 evaluation warning，不是一个可直接拿来做 SOTA baseline 的完整系统。

### 跟我们的关系
- 对 `0616-fex` 课题，如果我们做 formula extraction / symbolic discovery，本文直接提醒不能只报告 grammar 或 operator set，必须报告 variable routing architecture。
- 本文不完全撞车，除非我们的 claim 是“某个固定 differentiable SR tree 在代表性公式上可稳定恢复闭式表达式”。
- 若我们的方法也使用 soft selector 加 hard snap，one-hot snap failure 是必须测的风险点。
- 可借鉴的评测指标包括 exact structural recovery、held-out RMSE、snap 前后 loss drop、failure type taxonomy。
- 可借鉴的实验设计是把 representability 和 recoverability 分开，并对每个 target 报 seed-level recovery probability。
- 可借鉴的 mitigation 是小 architecture ensemble 加 validation selector，而不是固定单一 routing。
- 差异化方向之一：把该 bias study 扩到 addition/multiplication/composition grammar，而不局限于 `f(a)-g(b)`。
- 差异化方向之二：在有噪声数据、真实系统识别 benchmark、或 LLM-assisted formula extraction 里测试 selector 是否仍有效。
- 差异化方向之三：用干预实验验证 gradient ratio 是否因果，例如重加权路径梯度、冻结路由、或人为均衡左右 branch signal。
- 该文应引用，尤其在讨论 differentiable symbolic regression 的 architecture choice、hardening instability、validation-based model selection 时。
- 如果我们主张新方法更可靠，应对比本文的 depth-3 recoverability matrix 或复现其中几个 high-contrast cells。
- 下一轮检索应围绕 differentiable symbolic regression architecture、hard concrete / softmax hardening、neural symbolic regression recoverability、DARTS instability、symbolic regression benchmark leakage 这些词展开。
