# EditSR: Enhancing Neural Symbolic Regression via Edit-based Rectification (2606.07915)

- **Year**: 2026.06
- **Authors**: Da Li, Xinxin Li, Xin Cui, Jin Xu, Juan Zhang, Junping Yin; Northeast Normal University, East China Normal University, Institute of Applied Physics and Computational Mathematics, Beihang University, National Key Laboratory of Computational Physics, Shanghai Zhangjiang Institute of Mathematics
- **Venue**: arXiv
- **Keywords**: symbolic regression, neural symbolic regression, edit-based rectification, syntax-constrained decoding, NeSymReS, Tagger-Editor, state-transition chain, SRBench
- **One-liner**: 用语法受限编辑器修正神经符号回归输出。

## 精华提炼

### 管线/方法

1. 任务输入是一个观测数据集 $\mathcal{D}=\{(x_n,y_n)\}_{n=1}^N$，目标是恢复可解析的符号表达式结构。
2. EditSR 采用两层框架：第一层是快速神经符号回归模型，第二层是学习到的 edit-based Rectifier。
3. 第一层具体实例化为 NeSymReS，并扩展到最多 10 个变量。
4. NeSymReS 使用 permutation-invariant Set encoder 把数据集编码成 $h$。
5. Decoder 是 Transformer decoder，以 prefix expression sequence 形式自回归生成表达式 skeleton。
6. 第一层训练目标是标准 teacher-forcing cross entropy：$\mathcal{L}_{base}=-\sum_i \log p_\theta(f^*_i|f^*_{<i},h)$。
7. 一次性 autoregressive decoding 容易早期出错，后续 tokens 会在错误的树 arity context 下继续生成。
8. 作者的核心观察是：NeSymReS 失败时常常仍保留部分正确变量、常数或子树。
9. 因此 EditSR 不重启全局搜索，而是把错误表达式当作可修补 draft。
10. 第二层 Rectifier 以当前表达式状态 $f^{(t)}$ 和 dataset encoding $h$ 为输入。
11. Rectifier 每一步输出一个 edit position、edit action 和 edit content。
12. 编辑动作集合为 Keep、Replace、Delete、Rewrite、Insert。
13. Replace 只替换选中 token，且保持 arity 不变。
14. Delete 把内部子树折叠为一个 leaf token。
15. Rewrite 重新生成选中内部子树。
16. Insert 把选中 leaf 扩展为一个新生成的内部子树。
17. 所有非平凡编辑都在 subtree 级别执行，而不是 token 级别执行。
18. 每一步状态都是 parseable expression，避免半截公式在后续步骤污染推理。
19. Tagger 负责对每个位置预测 admissible actions，并选出最高置信的非 Keep position-action pair。
20. Tagger 的 admissible set 由 token arity 决定：leaf 允许 Keep/Replace/Insert，internal node 允许 Keep/Replace/Delete/Rewrite。
21. Editor 负责生成 edit content。
22. Editor 对 Replace/Delete 使用 single-token classification head。
23. Editor 对 Insert/Rewrite 使用 autoregressive subtree generation head。
24. Editor context state $\tilde f^{(t)}$ 用 action-specific placeholders 表示待填补区域。
25. Insert 和 Rewrite 使用长度预算 $S$ 个 placeholder，在当前实现中 $S=5$。
26. Editor 采用 bidirectional and infilling attention：可见 tokens 双向互看，hole 内部自回归生成。
27. 生成子树时同样使用 syntax-constrained feasible token set。
28. Syntax constraint 由 deficit count 管理：每个 token 消耗一个 pending hole，并按照 arity 创建新 holes。
29. token 可行条件是更新后的 deficit 能在剩余长度预算内闭合。
30. 推理时第一层使用 beam search，beam size $B=30$。
31. 如果某个 beam candidate 已满足停止条件，直接接受。
32. 否则每个失败 candidate 都可作为独立初始状态进入 Rectifier。
33. Rectifier 推理采用 greedy edit，不使用 beam search。
34. 每次 edit 后对新表达式做 BFGS constant optimization 并计算 MSE。
35. 如果 $\mathrm{MSE}\le 10^{-5}$，提前停止并返回当前表达式。
36. 否则继续直到 Tagger 选择 Keep 或达到 $T_{\max}=10$。
37. 若没有提前成功，返回 edit trajectory 中 MSE 最小的表达式。
38. 训练 Rectifier 的关键不是在线搜索，而是构造 supervised rectification chains。
39. 作者先从 target expression $f^*$ 出发，随机施加 $T'$ 次 corruption 得到初始错误状态 $f^{(0)}$。
40. corruption depth $T'$ 从 $\{1,\ldots,20\}$ 均匀采样。
41. random edit actions 从预定义 action space 等概率采样，且必须满足 action admissibility。
42. 从 $f^{(0)}$ 到 $f^*$ 的监督链由确定性 state-transition algorithm 构造。
43. State-transition algorithm 在当前树与目标树之间递归对齐。
44. 它在每个 position 比较两类计划：立即对当前位置做 admissible edit，或延迟到 aligned descendants。
45. cost function 对 Replace 设 0/1 成本，对 Delete/Insert/Rewrite 加入 $1+\lambda |subtree|$。
46. 当前实现的 subtree penalty $\lambda=0.1$。
47. 如果目标 subtree 超过 Editor budget $S$，算法构造 budget-limited intermediate subtree。
48. 这让 Rectifier 学会多步局部修补，而不是总想一步 rewrite 大片结构。
49. Rectifier loss 是 Tagger loss 加 Editor loss。
50. Tagger loss 是 position-wise multi-class cross entropy，目标位置为真实 action，其他位置为 Keep。
51. Editor loss 对 edit content 做监督；Replace/Delete 退化为分类损失，Insert/Rewrite 是序列生成损失。
52. 训练流程分两阶段：先训练第一层 NeSymReS，再冻结第一层训练 Rectifier。
53. 第二阶段主要用人工 corruption chains 训练 Rectifier。
54. 之后短期 fine-tuning，用第一层真实输出作为 initial states，使 Rectifier 贴合推理时的错误分布。
55. 这个设计把昂贵修补成本前移到 pretraining，推理时只做少量本地 edit 和 BFGS。

### 关键技术细节

- 训练集包含 100,000,000 个随机生成的 expression skeleton，validation set 为 1,000。
- Skeleton generator 最大变量集合为 $\{x_1,\ldots,x_{10}\}$。
- 最大 unary operators 为 5，最大 binary operators 为 $d+5$，最大 constants 为 3。
- skeleton 最大长度为 30，Rectifier 最大 prefix expression length $L_{\max}=50$。
- constants 从 $\mathcal{U}(-10,10)$ 采样。
- binary operators 采样权重：add 1, sub 0.5, mul 1, div 0.5。
- unary operators 包含 abs, pow2, pow3, pow5, sqrt, sin, cos, tan, arcsin, log, exp。
- unary 权重中 pow2/pow3/sqrt 为 1，sin/cos/log/exp 为 0.5，abs/pow5/tan/arcsin 为 0.1。
- 变量 sampling 同时使用 linear-uniform 和 log-scale sampling。
- evaluation 出现 NaN/Inf 时将对应值替换为 0 以保持数值稳定。
- 训练时加入 0-3 个 distractor variables。
- 训练 noise level 从 $\mathcal{U}(0,0.1)$ 采样。
- model dimension 为 512，attention heads 为 8。
- Set encoder 4 层，first-layer decoder 8 层。
- Tagger 和 Editor 各 4 层。
- learning rate 为 0.0001，使用 cosine decay。
- batch size 为 200。
- NeSymReS 训练 30 epochs，Rectifier 训练 30 epochs，fine-tuning 5 epochs。
- 推理 beam size 为 30。
- 推理最大 rectification iterations 为 10。
- BFGS constant initialization 从 $\mathcal{U}(0,10)$ 采样。
- BFGS restarts 为 10 次。
- Accuracy solution 的判定阈值为 $R^2 \ge 0.999$。
- Symbolic solution 使用 SRBench 标准：预测式与真式可差一个 additive constant 或 multiplicative factor。
- Complexity 用 expression tree node count 衡量。
- benchmark 覆盖 standard benchmarks、SRBench 1.0 和 SRBench 2.0。
- Standard benchmarks 共 91 个问题，最多 3 维，每题 200 samples，平均 complexity 11.49。
- Feynman benchmark 共 120 个问题，最多 9 维，10,000 samples，平均 complexity 16.57。
- ODE-Strogatz benchmark 共 14 个问题，最多 2 维，400 samples，平均 complexity 13.36。
- SRBench 2.0 phenomenological and first-principles 共 12 个问题，最多 4 维，平均 48 samples，含 natural noise。
- SRBench 2.0 black-box 共 12 个问题，最高 51 维，平均 4387 samples；高维问题按 SRBench 2.0 建议做 feature selection 到 3 维。

### 核心结果

- Standard benchmarks 上，EditSR 在多数 benchmark 的 mean $R^2$ 排第一或第二，并且表达式通常更简单。
- Constant: EditSR $R^2=0.9975\pm0.0014$，Complexity $11.92\pm0.26$，比 uDSR/TPSR/ParFam 简洁很多。
- Koza 和 Nguyen: EditSR 达到 $R^2=1.0000$。
- Livermore: EditSR $R^2=0.9978\pm0.0014$，Complexity $10.33\pm0.78$。
- Korns 是相对弱项：EditSR $R^2=0.6278\pm0.0573$，低于 SR4MDL 的 $0.8716\pm0.1572$。
- Mean test time 上 EditSR 为 17.03s，是该比较中最快。
- 对比 test time：uDSR 77.99s，SR4MDL 765.03s，ParFam 367.18s，RILS-ROLS 26.46s，TPSR 174.24s。
- 这说明 EditSR 的 learned rectification 比 TPSR 这类在线 MCTS rectification 快一个数量级左右。
- Noise robustness 上，standard benchmarks 的 ECDF 显示 EditSR 在 0、0.001、0.01、0.1 noise 下都保持较右偏的 $R^2$ 分布。
- Distractor robustness 是本文很有说服力的结果。
- 1 个 distractor 时，EditSR ASR 77.94%, DUR 2.45%, SSR 61.76%。
- 2 个 distractors 时，EditSR ASR 72.06%, DUR 4.41%, SSR 55.88%。
- 3 个 distractors 时，EditSR ASR 65.69%, DUR 13.73%, SSR 48.53%。
- TPSR 在 2/3 distractors 下 ASR 很高，但 DUR 分别达 68.65% 和 78.20%，SSR 只有 13.06%。
- 这个结果支持作者主张：EditSR 不只是数值拟合好，更少把 spurious variables 吸进公式。
- Feynman benchmark 上，EditSR 的 Accuracy solution rate 在三个 noise levels 下都接近 80%。
- Feynman 的 Symbolic solution rate 持续超过 50%，RILS-ROLS 是最接近的 baseline 但仍低于 EditSR。
- ODE-Strogatz 整体更难，但 EditSR 在 Accuracy solution rate 和 Symbolic solution rate 上仍位于 strongest group。
- SRBench 2.0 black-box 上没有模型达到接近完美的 $R^2$ 分布，但 EditSR 避免了低 $R^2$ 加高 Complexity 的坏组合。
- SRBench 2.0 phenomenological and first-principles 上，EditSR 的多数问题 mean $R^2$ 高于 0.95，同时表达式较简单。
- Ablation 显示加 Rectifier 后，Accuracy solution rate 和 Symbolic solution rate 提升最大。
- NeSymReS 常产生过短或结构不完整的表达式，EditSR 的 complexity 有时略升，更多表示补全了缺失结构。
- 按 target complexity 分桶后，medium complexity 起 EditSR 和 NeSymReS 的差距明显扩大。
- very high complexity bucket 中 NeSymReS 的 Symbolic solution rate 下降到 0%，EditSR 仍保持非零表现。
- 成功修补通常只需要 4-6 个 edit steps，长 edit chains 少见。
- Normalized edit distance 的 median 从 step 1 的 0.64 降到 step 7 附近接近 0。
- Insert 是最常用 edit action，其次是 Rewrite。
- Insert 的 normalized edit distance median reduction 最大，说明 NeSymReS 的主要错误常是结构缺失，而不是多余结构。
- Replace 和 Delete 使用较少，Delete 虽稀少但仍有正向修补效果。
- dropout robustness 实验中，dropout 0.1/0.2/0.3 下 EditSR ASR 分别为 79.38%、76.11%、78.06%。
- 同一实验的 SSR 分别为 60.11%、56.78%、59.17%，Complexity 在 11.68-11.96 之间。
- fine-tuning validation curves 中，Tagger 后期超过 90% accuracy，Editor 各分支也接近或达到约 90%。
- 案例分析显示，EditSR 擅长修补已有合理 scaffold 的表达式，例如补 missing powers、factors、denominator terms 和 function arguments。
- 失败案例通常是第一层预测离目标太远，误差不再是局部结构差异；这时 bounded rectification 不足以恢复完整公式。

### 明确局限

- Rectifier 的有效性依赖第一层输出至少有可复用 scaffold。
- 当 NeSymReS 初始预测与目标表达式距离过大时，10 步 bounded local edits 可能无法恢复。
- 作者承认 Rectifier 的 robust training 主要来自人工 random corruption chains，严重 out-of-distribution error pattern 仍需验证。
- Rectifier 当前没有展示跨不同第一层架构的大规模验证，只在 NeSymReS dropout variants 上验证了 error pattern shift。
- 虽然作者说框架模块化，但适配前提较强：表达式表示、vocabulary、constant convention 和 syntax constraints 都要可对齐。
- 没看到独立第三方复现或公开 artifact 证据；论文给了超参数，但 100M skeleton training 成本较高。
- 对 SRBench 2.0 black-box 的高维问题做 feature selection 到 3 维，这会弱化对真实高维符号回归能力的检验。
- 主要结果是作者自报 benchmark 评测，尚缺少外部 lab 或 blind benchmark 验证。
- 该方法恢复的是 expression skeleton，constants 仍依赖 BFGS 后处理；这与许多 neural SR 方法一致，但不是端到端常数发现。
- 对等价表达式的判断仍依赖 SRBench symbolic solution 标准，可能漏掉更复杂代数等价。
- 在 noisy sparse 数据上，多个候选公式数值上都 plausible，局部修补信号会变弱。
- 跟 TPSR 的比较支持效率优势，但 TPSR/MCTS 的强弱取决于搜索预算，未必覆盖所有预算-性能 tradeoff。
- 论文没有把 learned rectifier 和更强 global search first layer 结合，作者把这列为 future work。
- 对真实科学发现场景的人工专家校验较少，主要是 benchmark recovery。
- 对 failure modes 的定量分类还不够细，只说明了 initial prediction 太远会失败。

### 跟我们的关系

- 对 `0616-fex` 来说，若 FEX 指 formula/equation extraction 或 scientific equation discovery，这篇是直接相关 prior。
- 可能撞车的 claim 是：先由神经模型一次性生成公式 draft，再通过 learned edit rectifier 做局部、语法受限、多步修补。
- 如果我们的 claim 是“编辑式公式修复/公式后处理模块”，需要明确区别于 EditSR 的 Tagger-Editor、state-transition chain 和 syntax-constrained subtree edits。
- 如果我们的重点是从文本/图表/实验日志抽取公式，而不是从数值数据恢复表达式，那么不完全撞车，但 EditSR 可作为公式结构修补模块引用。
- 最可借鉴工具是 deficit-based syntax mask，可保证 prefix expression 任意中间状态 parseable。
- 第二个可借鉴点是把修补监督转成 deterministic state-transition chains，而不是依赖在线 reward search。
- 第三个可借鉴点是把修补成本前移到训练，用短 greedy edit chains 替代 inference-time MCTS。
- 对实验设计有启发的是同时报告 Accuracy solution rate、Symbolic solution rate、Complexity、distractor usage rate 和 runtime。
- 对差异化最关键：不要只声称“post-hoc rectification for symbolic regression”，这个已经被 EditSR 明确覆盖。
- 可差异化方向包括：跨 first-layer 架构泛化、真实文献/图表公式抽取、外部 verifier veto、LLM 生成公式的结构修复、或与 global symbolic search 的 tighter collaboration。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 角度看，EditSR 是“公式/方程结构修补”方向的强相关论文。它的核心 claim 不是提出新的 symbolic regression 搜索器，而是把 neural SR 的错误输出视为 draft，再用学习到的 Tagger-Editor Rectifier 做语法受限的 subtree-level edits。若我们的课题要做 formula/equation extraction 后处理，尤其是把 LLM 或 neural parser 输出的错误公式逐步修正，这篇必须引用，并且需要避开“learned edit rectification of symbolic expressions”这一已覆盖表述。

可直接复用的方法包括三块。第一是 deficit count syntax mask，用 pending holes 保证 prefix expression 的任意生成/编辑结果都能闭合。第二是 state-transition chain construction，把错误表达式到目标表达式的修补路径转化成确定性 one-step supervision，而不是在线 RL 或 MCTS。第三是把修补模块拆成 Tagger 和 Editor：先选位置和动作，再局部生成 edit content。这个拆分对 FEX 很实用，因为公式抽取错误经常是局部缺项、错变量、错 denominator 或函数参数错误。

撞车边界也比较清楚。如果 `0616-fex` 的创新点是“对已有公式表达式做多步局部编辑修复，并保持语法有效”，那与 EditSR 高度重合，需要强调我们的输入来源、verifier、目标任务或跨模态场景不同。如果我们的创新点是从论文文本/图像/实验日志抽取公式，再用独立 verifier 做证据约束，EditSR 不是直接替代品，而是一个可借鉴的修补 baseline。最好的差异化是加入独立验证者、claim-level evidence grounding，或验证 edit 后公式是否被原始数据/文本证据支持，而不是只追求 benchmark recovery。
