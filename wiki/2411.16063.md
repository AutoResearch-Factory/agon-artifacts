# VICON: Vision In-Context Operator Networks for Multi-Physics Fluid Dynamics Prediction (2411.16063)

- **Year**: 2026.02
- **Authors**: Yadi Cao, Yuxuan Liu, Liu Yang, Rose Yu, Hayden Schaeffer, Stanley Osher (UCLA, UCSD, NUS)
- **Venue**: TMLR (accepted)
- **Keywords**: in-context operator learning, vision transformer, multi-physics PDEs, fluid dynamics, few-shot generalization, patch-wise operations, temporal flexibility, autoregressive rollout
- **One-liner**: 用 ViT patch 操作替代逐点 token 化，将 ICON 缩放至 2D 稠密流体数据，3 个 benchmark 上比 DPOT/MPP 误差降 38-45%、推理快 28-65%，且在不完美时间采样下仅退化 24%（baseline 71-74%）。

## 精华提炼

### 管线/方法

**问题设定**：多物理场时间依赖 PDE 的前向问题，domain $\Omega \subseteq \mathbb{R}^2$，$c$ 通道数。给定初始 $I_0=10$ 帧，预测后续帧。利用时间齐次 Markov 性：对固定的 $\Delta t$，存在算子 $\mathcal{L}^{(i_p)}_{\Delta t}: \mathbf{u}_t \mapsto \mathbf{u}_{t+\Delta t}$。

**训练 pipeline**：
1. 从轨迹中随机采样 stride $s \sim \mathcal{U}\{1,\dots,s_{\text{max}}\}$（$s_{\text{max}}=5$）
2. 形成 COND-QOI pair：$\langle \mathbf{u}_t, \mathbf{u}_{t+s\Delta\tau} \rangle$
3. 每帧 128×128×7（7 通道 union：密度、$u_x$、$u_y$、压力、涡量、被动标量、节点类型）→ 划分为 8×8=64 个 patch，每 patch 16×16×7
4. 共享线性投影 $f_\phi: \mathbb{R}^{16\times16\times7} \rightarrow \mathbb{R}^{1024}$
5. 注入两类 learned positional encoding：(a) patch 位置编码 $\mathbf{E}_p \in \mathbb{R}^{64\times 1024}$，(b) 函数位置编码 $\mathbf{E}_c, \mathbf{E}_q \in \mathbb{R}^{10\times 1024}$ 区分 COND/QOI 及 pair index
6. 拼接所有 patch token 送入 10 层 transformer（hidden dim=1024, heads=8, FFN dim=2048, dropout=0）
7. 使用交替大小的 block causal attention mask（支持 $N_c \neq N_q$），保证 autoregressive 预测时每 pair 内 COND 可看 COND，QOI 可看同 pair 的 COND+QOI
8. 输出 token 经共享线性解码器 $g_\psi: \mathbb{R}^{1024} \rightarrow \mathbb{R}^{16\times16\times7}$ 回到物理空间
9. Loss：MSE in normalized space，仅计算 pair index $i > I_{\text{min}}=5$ 的 loss（前 5 对仅做 context）

**Prompt Normalization**：对每 sequence 内所有 COND frames 计算 channel-wise mean $\mu$ 和 std $\sigma$，用于归一化所有 COND 和 QOI。$\mathcal{N}_c = \mathcal{N}_q$（因最大 stride=5 较小，scale 分布变化不大）。threshold $\sigma_{\text{min}} = 10^{-4}$。

**推理策略**：
- Single-step rollout：$s=1$ 固定步长自回归
- Flexible-step rollout：先用 $s=1,\dots,s_{\text{max}}$ 扩展初始帧集，之后用 $s_{\text{max}}$ 大步长预测（减少 rollout 步数和误差累积）
- 不完美测量：仅用可用帧形成 pair，不做插值（DPOT/MPP 需插值）

### 关键技术细节

- **模型配置**：10 layers, hidden dim 1024, 8 heads, FFN dim 2048, dropout 0.0, 10 COND-QOI pairs, $I_{\text{min}}=5$，总参数 88M
- **训练**：AdamW, peak LR $10^{-4}$, final LR $10^{-7}$, cosine annealing + linear warmup (20K steps), total 200K steps, weight decay $10^{-4}$, grad clip 1.0, 2× RTX 4090, 58 GPU hrs
- **数据规模**：879K frames total，NS 2,496 轨迹×56 步，Compress 40K 轨迹×21 步，Euler 4K 轨迹×21 步
- **对比 baseline**：DPOT (122M params)、MPP (AViT-B, 116M params)、Specialist U-Net (U-Net-Mod64, per dataset)
- **原版 ICON 不可行**：每帧 128×128×7 ≈ 114K tokens，超出 GPU 内存
- **CNN-ICON 消融**：CNN encoder 压缩整帧为 1 token → 86.5M 参数。VICON 全面优于 CNN-ICON（1.9-3.9×），说明保留 patch 空间结构关键
- **非 in-context ViT baseline**：162M params，stride=1 时接近 VICON，stride=2 时严重退化（95.03 vs 68.03 last step error），验证 in-context 的必要性
- **Cross-trajectory context**：用同 operator 不同轨迹的 pairs 做 context 可行（仅 +0.012 avg error），但实际部署更倾向 intra-trajectory

| NS Last Step (1e-2) | 76.77 | 68.03 | **65.27** | 93.52 | 48.8 |
| Euler Last Step | **39.11** | **39.11** | 48.92 | 65.32 | 55.9 |
| Compress Last Step | 61.41 | 61.41 | 2866 | 185.3 | **39.9** |

- **平均最后步误差降幅**：vs DPOT 37.9%，vs MPP 44.7%
- **推理效率**：8.7 ms/step（DPOT 12.0, MPP 25.7），参数 88M（DPOT 122M, MPP 116M）
- **Flexible-step**：NS 上 stride=3 最优（11.4% 误差降低 vs single-step），Euler/Compress 无收益（因总步数少）
- **不完美测量鲁棒性**：VICON 仅退化 24.41%（相对），DPOT 71.37%，MPP 74.49%
- **TKE 分析**（Euler 湍流子集）：VICON MAE=0.016，MPP=0.049，DPOT=0.012
- **Joint vs Separate training**：Joint 在多数据集上显著优于 separate（Compress last step: 71.17 vs 163.5）
- **Ablation - Patch size**：16×16 最优；8×8 需降 hidden dim（512→256），4×4 需更激进降维（128/256 dim, 5 layers）且训练 188 hrs；<4×4 不可行
- **Ablation - Positional encoding**：function encoding 比 patch encoding 更重要；patch encoding 对 NS（无限刚度）收益大，对 Compress/Euler（空间相关随距离衰减）收益小
- **Ablation - Context length**：10 pairs ≥ 8 pairs > 6 pairs

### 明确局限

1. **仅限 2D**：3D 会导致 token 数立方增长，超出计算预算。作者建议 VAE 潜空间压缩或 crop/window 方法
2. **仅限被动自治系统**：未扩展到有外力驱动的非自治 PDE
3. **自回归误差累积**：最终 pair 的 condition frame 误差在 rollout 中传播
4. **Channel-union 瓶颈**：跨领域扩展到流体以外（不同状态变量）时受限，需 inter-channel attention
5. **不规则域未覆盖**：仅处理规则网格，不支持 graph/mesh（固体力学、分子动力学）
6. **Compress 数据集 artifacts**：checkerboard 伪影，源于 field 值剧烈变化 + 训练/推理 normalization mismatch（训练时 context 从全轨迹均匀采样，推理时仅初始帧）
7. **无真机验证**：全部实验为数值仿真数据
8. **NS 上未全面超越 DPOT**：NS 数据集 DPOT 仍略好（last step 65.27 vs 76.77），但 VICON 在长 rollout 时追平
9. **Specialist U-Net 在单数据集上更强**：NS 和 Compress 上 specialist 显著优于 VICON（但 VICON 是统一的 multi-physics 模型）
10. **Cross-trajectory context 需要 operator 标识**：实际部署中 operator 参数未知，限制了跨轨迹利用

## Read by: 0616-fex (2026-06-24)

### 撞车评估
VICON 不直接撞击 0616-fex 的核心方向。VICON 走的是 **in-context operator learning** 路线——通过 few-shot 示例对隐式编码 PDE 动力学，无需显式 PDE 知识，无需 fine-tuning。这与 PROSE 系列的显式符号编码路线互补但不同。0616-fex 如果是 foundation model experiments（pre-training/fine-tuning 范式），与 VICON 的技术路线有本质差异——VICON 依赖 in-context examples 而非 fine-tuning data。

但是，VICON 的 **multi-physics joint training** 实验（3 数据集联合训练优于分别训练）直接支撑 foundation model 的核心理念：多物理系统联合训练有正向迁移。这个 finding 对 0616-fex 有参考价值。

### 可用技术
1. **Patch-wise 物理场处理**：patch 投影 + 双 positional encoding（spatial + functional）的设计模式可直接用于 2D PDE 输入编码
2. **Prompt/instance normalization**：per-sequence channel-wise normalization 是简单的训练稳定技巧，可移植
3. **Channel union + mask**：多数据集异构通道的统一处理方案
4. **随机 stride 数据增强**：提升模型对不同 $\Delta t$ 的泛化能力
5. **Block causal attention mask**：处理 paired (COND, QOI) 序列的专用 mask 设计
6. **Flexible-step rollout 策略**：减少长序列自回归步数的通用方法

### 差异化要点
- VICON 需要部署时提供 in-context pairs（初始 10 帧），pre-training 模型需要 fine-tuning data
- VICON 的优势场景是**在线部署、参数未知、快速适应**；pre-training 模型的优势是**已知任务、大量 offline 数据**
- 0616-fex 如果追求少样本适应，VICON 的 in-context 范式比 fine-tuning 更有吸引力
- VICON 的不完美测量鲁棒性是 pre-training 模型难以复现的独特能力
