# CogMCTS: A Novel Cognitive-Guided Monte Carlo Tree Search Framework for Iterative Heuristic Evolution with Large Language Models (2512.08609)

- **Year**: 2025.12
- **Authors**: Hui Wang, Yang Liu, Xiaoyu Zhang, Chaoxu Mu (Anhui University, Pengcheng Laboratory)
- **Venue**: IJCAI 2026 (submitted)
- **Keywords**: MCTS, LLM, automatic heuristic design, combinatorial optimization, cognitive guidance, evolutionary search, UCT, self-reflection
- **One-liner**: 将多轮认知引导机制（快速/复杂认知 + 正负知识分类）集成到 MCTS 中，用于 LLM 驱动的自动启发式算法设计，在 TSP/KP/OP/CVRP/MKP 上超越 MCTS-AHD。

## 精华提炼

### 管线/方法

CogMCTS 的完整 pipeline 如下：

**输入**：种子启发式函数 + 任务实例数据集 D

1. **初始化**：用 LLM 生成 N_I=10 个初始节点，每个节点对应一个 Python 实现的启发式函数。使用多角色提示（不同学术背景的虚拟角色）增加初始多样性。所有初始节点挂在虚拟根节点 n_r 下。

2. **MCTS 主循环**（重复直到评估预算 T=1000 用完）：
   - **Selection**：从根节点开始，用 UCT 公式选择最有前途的节点。UCT 使用探索衰减机制 λ = λ₀·(T-t)/T，λ₀=0.1，随迭代推进逐渐从探索转向利用。Q 值做 min-max 归一化。采用 progressive widening 策略，高访问次数的节点才扩展。
   - **Cognitive Guidance（核心创新）**：
     - 构建认知候选集 CCS = UCT 选中节点 ∪ 从精英集 E 中按排名权重采样的 m 个节点
     - **快速认知**：按性能排序后两两配对输入 LLM，做对比分析，提取设计模式和经验知识 E_k
     - **复杂认知**：整合快速认知结果 E_k + 正知识 K⁺ + 负知识 K⁻，由 LLM 生成结构化指导（关键词、建议、规避提示、解释）
   - **Expansion（双轨扩展）**：在选中节点上执行 5 种操作：
     - **i**：用种子函数 + 多角色提示生成初始启发式
     - **em1**：局部对比驱动 — 从 CCS 中配对启发式 + 认知指导 → 生成 m 个新启发式
     - **em2**：全局最优引导 — 用当前全局最优 + 认知指导 → 生成新启发式
     - **m1**：结构变异 — 改变算法形式（新机制/公式/模块）
     - **m2**：参数变异 — 调整关键参数和策略
   - **CKV（一致性知识验证）**：比较第 k 轮 em1 前的全局最优 S^k_{em1} 和第 m 轮 em2 后的全局最优 S^m_{em2}。若不同 → E_k 存入正知识 K⁺；若相同 → 存入负知识 K⁻。思考周期 C_t=2 最优。
   - **Simulation**：在数据集 D 上评估新启发式，计算奖励 R(h)，直接作为 UCT 的 Q 值
   - **Backpropagation**：沿路径回传，Q(s,a) ← max_{children} Q，N(s) ← sum_{children} N

3. **输出**：全局最优启发式代码 C*

### 关键技术细节

- **LLM 后端**：GPT-3.5-turbo 和 GPT-4o-mini（两种都测试了，未用开源模型）
- **UCT 公式**：标准 UCT + min-max 归一化 + 探索衰减，直接沿用 MCTS-AHD 的设计
- **Progressive Widening**：参数设置沿用 MCTS-AHD
- **精英集采样**：按排名 r_i 计算权重 w_i = 1/(r_i+1+N)，归一化后无放回采样 m 个（实验中 real_m=7）
- **评估预算**：T=1000 次启发式评估，每个启发式限 60 秒运行时间
- **每轮扩展**：一次扩展产生 8 个新节点（em1, em2 各生成若干 + m1, m2 各 1 个）
- **搜索树深度限制**：H=10
- **初始节点数**：N_I=10
- **认知候选集大小**：UCT 选中节点 + 最多 7 个精英集采样节点

### 明确局限

**作者自述**：
- 未来工作计划用 Gumbel-MCTS 等轻量模拟策略加速大规模优化
- 未来计划扩展到更复杂的组合优化和真实世界优化问题

**读出来的局限**：
1. **改进幅度有限**：相比 MCTS-AHD 的改进通常在小数点后第二三位，部分任务甚至不如基线（TSP N=20）
2. **只用了闭源 LLM**：GPT-3.5-turbo 和 GPT-4o-mini，未验证开源模型的表现
3. **无 LLM API 调用开销分析**：多轮认知引导（快速+复杂认知）显著增加 LLM 调用次数，但论文完全没报告 token 消耗或 API 成本
4. **CKV 机制缺乏理论支撑**：用全局最优是否变化来判断经验是正是负，这个二元分类过于粗糙
5. **所有基准问题都是标准 CO benchmark**：没有真实世界应用验证
6. **UCT/progressive widening 设计完全沿用 MCTS-AHD**：核心 MCTS 搜索结构没有创新
7. **作者组自引** (PoH 是同组工作)

### 跟我们的关系

对 llm-mcts-sr 课题：

**撞车评估**：
- 本文的核心场景是 LLM+MCTS 搜索启发式算法代码，与 symbolic regression 的"搜索数学表达式"有方法论上的相似性
- 如果 llm-mcts-sr 课题用 MCTS 指导 LLM 搜索符号回归表达式，则搜索框架层面有中等程度的重叠
- 但具体 domain 不同：CogMCTS 面向 CO 问题的启发式设计，而非函数发现/物理方程推导

**可用工具/方法**：
- 认知引导机制（快速+复杂认知）的思路可迁移：在 SR 场景中，快速认知可对比不同表达式的结构差异，复杂认知可整合历史搜索经验
- CKV 正负知识分类思路简单但有启发：根据全局最优是否变化来判断经验价值
- 双轨扩展中 em1（局部对比）和 em2（全局最优引导）的分工值得借鉴
- 精英集的排名权重采样策略（w_i = 1/(r_i+1+N)）可复用

**差异化方向**：
- CogMCTS 的认知引导完全依赖自然语言反馈，而 SR 可以利用数学结构（梯度、残差分析）做更精确的引导
- CogMCTS 没有对搜索空间做任何结构化建模，而 SR 的表达式树天然有结构可利用
- 可以在此基础上引入数学先验（对称性、量纲分析）来替代/增强纯 LLM 认知引导

## Read by: llm-mcts-sr (2026-06-06)

**与 llm-mcts-sr 课题的关系**：CogMCTS 展示了 LLM+MCTS 在"搜索程序/算法"场景中的一种增强模式——通过多轮认知引导（快速认知做对比分析 + 复杂认知整合正负经验）来改善 MCTS 的节点扩展质量。这与用 LLM+MCTS 做 symbolic regression 在方法框架层面高度相似（都是 MCTS 管理搜索树 + LLM 生成候选 + 反馈指导下一轮生成），但具体 domain 不同。

**可借鉴点**：(1) 双轨扩展的 em1/em2 分工（局部对比 vs 全局最优引导）可迁移到 SR 的表达式搜索；(2) CKV 正负知识分类虽粗糙但思路有价值——在 SR 中可用残差变化替代"全局最优是否变化"做更精确的经验评估；(3) 精英集的排名权重采样可直接复用。

**差异化空间**：CogMCTS 完全依赖 LLM 的自然语言反馈做认知引导，而 SR 场景可以利用数学结构信息（梯度、残差模式、量纲分析）做更精确的指导，这是一个明确的差异化方向。此外 CogMCTS 的改进幅度有限且未报告 API 开销，在 SR 场景中需要更注重效率。
