# D2NO: Efficient Handling of Heterogeneous Input Function Spaces with Distributed Deep Neural Operators (2310.18888)

- **Year**: 2023.10
- **Authors**: Zecheng Zhang (FSU), Christian Moya (Purdue), Lu Lu (Yale), Guang Lin (Purdue), Hayden Schaeffer (UCLA)
- **Venue**: arXiv
- **Keywords**: DeepONet, federated learning, operator learning, discretization invariance, heterogeneous input functions, distributed training, neural operators
- **One-liner**: 提出 D2NO，一种分布式 DeepONet 训练策略，允许不同输入函数使用不同数量传感器，通过专用 branch net + 共享 trunk net 架构在保持精度同时降低计算成本。

## 精华提炼

### 管线/方法

**问题设定**：学习非线性算子 G: V → U，输入函数空间 V 中的函数具有异质性（不同正则性/频率/几何特征）。传统 DeepONet 要求所有输入函数在相同传感器上离散化，这迫使对整个数据集采用适应最低正则性函数的传感器密度。

**D2NO 完整管线**：
1. **数据分区**：将异质性数据集 D 按输入函数正则性/特征划分为 C 个子集 D¹, ..., D^C，每个子集内函数具有相似属性。
2. **专用 branch net**：每个 client c 拥有独立的 branch net p^k_c(·)，参数记为 α^c，专门处理其子集的输入函数。不同 client 可使用不同传感器数量/位置。
3. **共享 trunk net**：一个全局 trunk net b_k(x) 作为所有输出函数共享的基函数，参数记为 β，由中心 server 维护。
4. **局部损失**：L^c(α^c | β) = Σ_i ||G(u^c_i) - G_{α^c, β}(û^c_i)||²，每个 client 固定 β 优化自己的 α^c。
5. **全局同步**：L(β | α) = Σ_c L^c(α^c | β)，用全部数据更新共享的 β。
6. **训练循环**（Algorithm 1）：每轮先并行更新所有 α^c（各 client 独立），然后聚合更新 β（全局步骤）。无 FedAvg 式的多轮局部 SGD——每步都是全局同步。

**与传统 DeepONet 对比**：
- 传统：所有 u ∈ D → 统一 branch net p_k(u) → 统一 trunk b_k(x) → 全局 loss
- D2NO：D¹ → branch¹ p¹_k → shared trunk b_k → local loss¹; D² → branch² p²_k → shared trunk b_k → local loss²; 全局 loss 聚合所有 local loss

### 关键技术细节

- **网络架构**：基于 stacked DeepONet。Branch net 将传感器值映射到系数向量 [p_1(u), ..., p_K(u)]；Trunk net 将空间坐标映射到基函数值 [b_1(x), ..., b_K(x)]；输出为内积 Σ_k p_k(u)·b_k(x)。
- **Burgers 实验 1（双峰）**：两个 client，相同 branch 架构（75×100→100×1），但传感器布局不同（各自聚焦峰值区域）。对比 DeepONet 用 75 均匀传感器。
- **Burgers 实验 2（不同属性）**：Client 1 = 峰值函数，branch 75×100→100×1（75 传感器）；Client 2 = 光滑函数，branch 6×50→50×1（6 传感器）。对比 DeepONet 统一 75 传感器。
- **Pendulum 实验（不同频率）**：GRF 长度尺度 ℓ=0.1（高频）→ 100 传感器；ℓ=1.0（低频）→ 10 传感器。各 1000 训练轨迹。10 轮同步，每轮 100 local SGD steps。Adam 优化器。
- **Pendulum 实验（不平衡数据）**：V1 仅 200 样本，V2 有 1800 样本——模拟真实分布式场景中的数据不均衡。
- **效率分析**：D2NO 梯度计算总量 = N·p'·n_epochs ≤ N·p·n_epochs = 传统 DeepONet。关键在于 p'（每个 client 的 branch 参数量）≤ p（统一 branch 参数量）。在实验 2 中，非共享参数从 77K 降到 41.5K。

### 核心结果

| 实验 | DeepONet | D2NO | 提升 |
|------|----------|------|------|
| Burgers 双峰 (L2 rel err) | 18.64% | 8.10% | 56.5% ↓ |
| Burgers 不同属性 (L2 rel err) | 7.53% | 5.34% | 29.1% ↓ |
| Burgers 不同属性 (# unshared params) | 77K | 41.5K | 46.1% ↓ |
| Pendulum 不同频率 (V1/V2 err) | 3.27%/1.18% (m=100) | 5.21%/3.45% | V1略差, V2略差 |
| Pendulum 不同频率 (m=10 baseline) | 16.64%/3.21% | 5.21%/3.45% | 远超轻量baseline |
| Pendulum 不平衡数据 (V1/V2) | 6.34%/0.78% (m=100) | 7.08%/3.01% | V1略差, V2略差 |

**反直觉发现**：
- D2NO 在 Pendulum 上不如 DeepONet(m=100)，但论文将其定位为"competitive while more efficient"。实际上在 V2 上差距更明显（3.45% vs 1.18%）。这说明共享 trunk 可能导致对低频函数的 overfit 不足。
- 论文声称 D2NO "reduces back-propagation steps"，但 Algorithm 1 显示每轮仍然对所有样本做梯度更新——减少的是每个样本的输入维度，而非 update 次数。

### 明确局限

1. **只展示了 2-client 场景**——没有验证扩展到 C>2 时的 trunk 收敛质量。
2. **所有实验都是 1D 问题**（Burgers、单摆）——没有 2D/3D PDE 验证。
3. **理论仅证明 universal approximation**（存在性），没有 convergence rate 或 generalization bound。
4. **没有真正的 federated 部署**——所有数据仍在本地内存中，没有测量通信开销、隐私保护或 non-IID 鲁棒性。
5. **没有与 FNO 系列对比**——只和 vanilla DeepONet 比，但 FNO 本身在规则网格上效率更高。
6. **Pendulum 上 D2NO 精度不如 DeepONet(m=100)**——论文未充分解释为什么共享 trunk 引入精度损失。
7. **50 次独立训练的误差 bar 未报告**——只给了均值，无法判断显著性。
8. **传感器布局策略是手动的**——按峰值区域手动放置，没有自动化的传感器分配方法。

## Read by: 0616-fex (2026-06-24)

D2NO 与 0616-fex 课题的交集在于**异质性输入函数空间的处理**和**分布式算子学习**。核心评估：

**撞车风险**：低到中等。D2NO 的具体机制（手动分区 + 专用 branch + 共享 trunk）是一个明确的工程方案。如果 0616-fex 提出的是全新的分布式算子学习范式（如去中心化、异步、动态 client 加入/退出），则不撞车。但如果 0616-fex 的核心 claim 是"允许不同 sensor 数量处理异质 PDE 数据"，则与 D2NO 高度重叠。

**可复用组件**：
1. Algorithm 1 的 alternating optimization 框架（local branch update → global trunk sync）可作为 baseline。
2. Theorem 1 的证明结构（Lemma 3 函数近似 → Lemma 4 泛函近似 → Theorem 算子近似）可作为任何基于 DeepONet 的分布式架构的理论模板。
3. 异质性数据集构造方法（GRF 不同长度尺度、不同峰值区域）可在实验中复用。

**差异化空间**：
1. D2NO 的 client 划分依赖先验知识（已知函数正则性）→ 自动化/学习驱动的分区策略。
2. D2NO 只在 2-client 场景验证 → 扩展到 C>>2 的 scaling law 分析。
3. D2NO 使用同步全局更新 → 异步/去中心化的 trunk 聚合。
4. D2NO 没有 privacy/communication cost 的实证分析 → 真正的 federated deployment benchmark。
5. D2NO 限于 DeepONet → 推广到 FNO/Transformer-based 神经算子的分布式训练。
