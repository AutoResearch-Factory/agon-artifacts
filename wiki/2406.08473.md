# Strategies for Pretraining Neural Operators (2406.08473)

- **Year**: 2024.06
- **Authors**: Anthony Zhou, Cooper Lorsung, AmirPouya Hemmasian, Amir Barati Farimani (Carnegie Mellon University, Dept. of Mechanical Engineering & Machine Learning)
- **Venue**: TMLR (Transactions on Machine Learning Research), 2024
- **Keywords**: neural operator pretraining, PDE transfer learning, data augmentation, self-supervised learning, physics-informed learning, scaling behavior, generalization
- **One-liner**: 系统性比较 9 种神经算子预训练策略，发现 transfer learning + shift 数据增强效果最好，transformer/CNN 架构比 FNO/DeepONet 更受益于预训练。

### 管线/方法

输入 → 预训练（9216 样本，Heat+Advection+Burgers 各 3072 个 2D 时间序列，分辨率 (nt,nx,ny)=(32,64,64) 或 (32,32,32)）→ Fine-tune（500 样本，预测目标 PDE）→ 评测（Relative L2 error，5 seeds 平均）。

预训练策略分为三大类共 9 种：

**CV 迁移：**
1. *Binary* — 打乱时间轴后二分类「正确顺序 vs 打乱」，二进制交叉熵 loss；
2. *TimeSort* — 沿时间轴 patchify 成 4 块打乱，24 类（4! 排列）分类任务，cross-entropy；
3. *SpaceSort* — 沿空间轴 patchify 打乱，但表现差（周期性边界条件导致不同排列在视觉上无法区分），主实验剔除；
4. *Jigsaw* — 时空 patchify 成 8 块，从 40320 种排列中选 Hamming 距离最大的 1000 类，cross-entropy；

**PDE 预测型：**
5. *Coefficient* — 从 PDE 快照回归方程系数（ν, c），MSE loss；
6. *Derivative* — 回归空间/时间导数 {u_x, u_y, u_xx, u_yy, u_t}，用 FinDiff 生成标签，CNN 上采样后 MSE；
7. *Masked* — 75% 时空 patches 随机 mask + 可学习 mask token，回归重建完整解；

**对比型：**
8. *Transfer* — 用相同物理预测目标在预训练集上训练（监督/自监督，相当于 CV 的 transfer learning）；
9. *PICL* — Physics Informed Contrastive Learning，用 Generalised Contrastive Loss 根据 PDE 系数向量在隐空间聚类样本；

数据增强：Noise（高斯噪声 σ²=1e-7）、Shift（Fourier 域空间平移，Lie 对称性保证物理不变），Scale（乘常数，但 Burgers 的非线性使 scale 不保物理）。

所有预训练策略用 CNN projector head 来适配不同输出维度（分类或回归），预训练 200 epochs、Adam lr=1e-3、OneCycle scheduler。

### 关键技术细节

**模型架构：**
- FNO：4 modes, 48 width, 4 layers, 300k params
- DeepONet：branch/trunk 各 256 维, 各 3 层 MLP, SiLU 激活, 250k params
- OFormer（transformer-based）：hidden dim 32, 2 heads, encoder depth 2, decoder depth 1, latent channels 32, 70k params
- Unet：hidden channels 16, 8 ConvNext blocks, dim scaling (1,2,4), attention 在 bottleneck, 1M params

**数据规模：**
- 预训练：9216 样本（Heat/Adv/Burgers 各 3072），分辨率 (32,32,32) 或 (32,64,64)
- Fine-tune：256-1024 样本（视实验），256 验证样本
- Navier-Stokes：额外 (32,64,64) 分辨率，ν 采样自 {1..9}×10^{-6..-9}，A 振幅 {1..10}×10^{-3}

**评测指标：** Relative L2 error（FNO 原论文定义的标准化方法），5 seeds 平均。

**计算资源：** NVIDIA GeForce RTX 2080Ti (11GB)。

### 核心结果

**1. 预训练策略排名（自回归预测，In-distribution）：**
- 最好的策略在 16 个 (model, PDE) 组合中分布极分散
- Transfer 赢了 7/16 个 case，PICL 赢 4/16，其余分散
- 但 CV 策略（Binary/TimeSort/Jigsaw）几乎从不赢 —— 多数 case 比 None 还差

**2. 不同架构预训练提升幅度（best vs None，自回归）：**
- OFormer 提升最大（Heat: 38.91%, Burgers: 17.12%）
- Unet 其次（Heat: 29.16%, Burgers: 9.71%）
- FNO 提升有限（Heat: 14.43%, NS: 0%）
- DeepONet 中等（Burgers: 15.74%）

**3. 数据增强效果：**
- Shift 增广在 8/12 个 (model, PDE) 组合中表现最优
- Transfer + Shift 在 6/12 组合中同时最优
- 数据增强 consistently 提升性能，即使 baseline 已经 pretrained
- 最佳 aug+pretrain 组合 vs None 提升：OFormer + Noise/Transfer 在 NS 上达 13.23%

**4. Scaling 行为（In-distribution，样本数扫描 100→1000）：**
- 100 样本：pretrain 提升最大（OFormer: 47.60%, Unet: 23.05%）
- 1000 样本：pretrain 增益骤降（OFormer: 13.27%, Unet: 2.995%）
- FNO 在 100 样本时 pretrain 反而更差（-5.953%），猜测是 Fourier modes 迁移困难

**5. 泛化行为：**
- OOD（同方程不同系数）：OFormer 提升 34.06%, Unet 27.70%
- NS（全新方程）：所有模型提升微弱或负面，最大 Unet 仅 1.87%
- DeepONet 在 OOD 上反而更差（-3.38%）

**反直觉发现：**
- 直接迁移 CV 预训练方法到 PDE 几乎全失败 —— 物理预测是高维回归任务，与 CV 的分类任务根本不同
- "应用 X 到 Y"式的无监督 PDE 预训练不如直接用物理预测（Transfer learning），因为 PDE 数据没有「标注成本」—— 数值模拟生成标签不费人工
- FNO 预训练效果差不是方法问题，是 Fourier modes 跨任务迁移困难

### 明确局限

**作者自述：**
1. 预训练策略高度依赖模型和数据集选择，不存在 universal best
2. CV 方法直接迁移失败，需要 PDE-specific 的预训练设计
3. 实验限于 2D 和相对简单的 PDE（Heat, Adv, Burgers, NS）
4. 预训练数据限于 9216 样本，远小于 CV/NLP 的预训练规模
5. 仅评估了 3 种数据增强，multiple augmentations 组合未探讨
6. 解空间仅考虑了周期边界条件

**我读出来的额外局限：**
1. 没有真机/物理实验验证 —— 全靠数值模拟，无法评估 sim2real 差距
2. 对比型的 9 种策略远不是穷举，缺少 denoising diffusion 预训练、next-step prediction 变体
3. PDE 预训练 vs 直接训练的收敛速度缺少系统对比（只比了 error 没比训练时间）
4. 预训练数据集的多样性不足 —— 缺少多尺度、多物理、多几何的对比
5. 所有实验在单一 GPU（RTX 2080Ti）上，计算受限影响实验规模
6. 没有与 DPOT/Poseidon 等同期工作的直接数值对比
7. Auto-regressive rollout 只 roll 到 t=32，长时稳定性未充分评估

## Read by: 0616-fex (2026-06-24)

本次审读从 **0616-fex**（foundation model for PDEs / 科学机器学习基础模型）视角出发：

**撞车评估：** 不撞车。本文是 empirical comparison survey，不是提出新 foundation model 或新预训练方法。如果 0616-fex 在开发新的 PDE pretraining 方法，本文应作为必引的 landscape paper。

**关键收获：**
1. **Transfer learning = 强 baseline**：任何新的 PDE 预训练方法必须 beat transfer learning（即用相同目标在更大数据集上预训练）。这暗示"应用 CV 自监督方法到 PDE"本身不够形成 contribution。
2. **架构选择至关重要**：transformer-based 架构（OFormer）从预训练中受益最大。如果要做 foundation model，应该优先考虑 attention 机制。
3. **数据增强是 free lunch**：Shift augmentation（Fourier 域平移）几乎 always helps，且物理保持性使其理论上合理。
4. **Scarcity 场景下预训练最有价值**：100-250 样本 fine-tune 时 pretrain 增益显著，1000 样本时增益收窄。这对小数据场景的 PDE 建模有实际指导意义。

**可复用的基础设施：**
- Zenodo 上的 Heat/Advection/Burgers 9216 样本数据集
- GitHub 开源的 pretrain + fine-tune pipeline
- 文中详尽的超参数表可直接作为 baseline 配置
