# Improving Monte Carlo Tree Search for Symbolic Regression (2509.15929)

- **Year**: 2025.09
- **Authors**: Zhengyao Huang, Daniel Zhengyu Huang (北京大学 BICMR/CMLR), Tiannan Xiao, Dina Ma, Zhenyu Ming (华为), Hao Shi (清华), Yuanhui Wen (华为)
- **Venue**: NeurIPS 2025 (submitted)
- **Keywords**: symbolic regression, Monte Carlo Tree Search, extreme bandit, UCB, state-jumping actions, genetic programming, combinatorial optimization, reward distribution
- **One-liner**: 提出 UCB-extreme 极值 bandit 策略 + GP 风格 state-jumping 动作改进 MCTS 用于符号回归，理论证明最优性并在 Nguyen/SRBench 上取得 SOTA。

## 精华提炼

### 管线/方法

完整 pipeline：
1. **输入**：数据点 $(x_i, y_i)_{i=1}^n$，预定义符号集 $\{+, -, *, /, \sin, \cos, \exp, \log, \text{const}, \text{var}\}$
2. **表达式表示**：符号表达式 → 二叉树 → 先序遍历序列，max depth = 6
3. **MDP 建模**：状态 = 部分符号序列，动作 = 追加下一个符号，reward = $\frac{1}{1 + \text{NRMSE}(f)}$ 仅在完整表达式时非零
4. **MCTS 核心循环**（Algorithm 1）：
   - **Selection**：用 UCB-extreme 公式选择节点（替代标准 UCB1）
   - **State-jumping 触发**：在每个节点以深度衰减概率 $p_s$ 触发 mutation 或 crossover
   - **ε-random exploration**：以概率 $\epsilon$ 随机选子节点（增强鲁棒性）
   - **Expansion + Simulation**：标准 MCTS 展开和随机 rollout
   - **Bidirectional Propagation**：向上（Backprop）+ 向下（ForwardProp）更新所有节点的 Top-N priority queue
5. **常数优化**：每次评估表达式时用 BFGS 优化内部常数
6. **输出**：最高 reward 对应的符号表达式

### 关键技术细节

**UCB-extreme 策略**（公式 5）：
$$I_{T+1} := \arg\max_k \left\{ \hat{Q}_{k,T_k} + 2c \left(\frac{\ln T}{T_{k,T}}\right)^\gamma \right\}$$
- $\hat{Q}_{k,T_k} = \max_{t:I_t=k} X_{I_t,t}$（用最大值替代均值，这是与标准 UCB1 的核心区别）
- 参数 $\gamma$ 和 $c$ 需满足 $1/\gamma \geq a_1$ 且 $2^{a_1} c^{1/\gamma} \geq 2 + 1/a_1$
- 实际使用 $\gamma = 0.5, c = 1$

**三个理论定理**：
- **Theorem 1**（多项式尾衰减上界）：假设 reward 服从 $P(x;a,b) = 1-(1-x/b)^a$，performance gap $G(T) = O(1/T^{1/a_1})$，regret $R(T) = O(\ln T / T^{1+1/a_1})$
- **Theorem 2**（下界）：基于 KL 散度的信息论下界，证明 UCB-extreme 是最优的
- **Theorem 3**（指数尾衰减负面结果）：若 reward 分布指数衰减，$G(T) \geq \Omega(1/\ln T)$，即需要指数级样本才能找到最优解

**State-jumping 动作**：
- **Mutation**：uniform mutation、node replacement、subtree insertion、subtree shrinkage（各 25% 概率）
- **Crossover**：single-point crossover，随机选节点交换子树
- 触发概率 $p_s = 0.2$（深度衰减），mutation rate $p_m = 0.1$
- 每个 MCTS 节点维护 Top-N=500 priority queue 存储高 reward 轨迹

**Bidirectional Propagation**：
- Backward Propagation（Algorithm 2）：新发现的高 reward 轨迹向上传播到所有祖先节点的 queue
- Forward Propagation（Algorithm 3）：节点展开时，父节点向新子节点传播其 queue 中经过该子节点的轨迹
- 保证 $\hat{V}(s) = \max\{\hat{V}(v') : v' \text{ is child of } v\}$

**实验配置**：
- 10.6 TFLOPS FP32 + 256GB RAM，单核 CPU 运行
- Basic benchmarks: 100 runs × 2M expression evaluations
- SRBench: 10 runs × 500K evaluations 或 48h
- 代码: github.com/PKU-CMEGroup/MCTS-4-SR

### 核心结果

| Benchmark | Ours | DSR | GEGL | NGGP | PySR |
|-----------|------|-----|------|------|------|
| Nguyen (12 datasets) | **93.25%** | 83.58% | 86.00% | 92.33% | 74.41% |
| Nguyen^C (5) | **100%** | 100% | 100% | 100% | 65.4% |
| Jin (6) | **100%** | 70.33% | 95.67% | 100% | 72.17% |
| Livermore (22) | **71.41%** | 30.41% | 56.36% | 71.09% | 46.14% |

- SRBench Black-box (122 tasks): test $R^2$ 排名第二（仅次于 Operon），但模型复杂度显著更低
- Pareto 前沿：与 Operon、GP-COMEA、DSR 并列最优
- 运行时间：Nguyen 平均 157.9s vs NGGP 265.5s（快 1.7x）

**Ablation 关键发现**：
- 去掉 state-jumping（Model B）: 93.25% → 53.08%（最大影响）
- 去掉 UCB-extreme（Model A）: 93.25% → 90.83%
- 标准 UCB1 无增强（Model C）: 仅 10.50%
- State-jumping 将 reward 尾衰减率 $a$ 从 [4,6] 降至 [2,4]（更容易发现高 reward 表达式）

**反直觉发现**：
- State-jumping 的贡献远大于 UCB-extreme 理论改进（从 ablation 看差距巨大）
- 降低尾衰减参数 $a$ 反而提高了搜索效率（看似矛盾但理论一致：小 $a$ 意味着高 reward 更容易被采到）

### 明确局限

**作者自述**：
- Nguyen-12 仍然未解决（recovery rate 仅 22%）
- 极值 bandit 理论保证依赖 reward 分布假设（多项式衰减）
- 未扩展到更广泛的 RL 或组合优化问题

**我读出的局限**：
- 不使用 LLM 或任何学习型先验（纯搜索方法），在复杂表达式上可能不如带预训练的方法
- 仅在 1-2 变量的标准 benchmark 上验证，未测试高维真实科学数据
- 常数优化用 BFGS，对初值敏感，可能遗漏好的表达式
- Priority queue size N=500 是人工选择，没有分析敏感度
- 与 Transformer-based SR 方法（如 TPSR）的比较不够直接——TPSR 也用 MCTS 但有 Transformer prior
- SRBench 结果中 test $R^2$ 低于 Operon，说明在纯精度上并非最优

### 对 FEX 的撞车评估

结论：**方法层面中高重叠，应用层面不直接撞车**。

本文与 FEX 都把符号表达式当作树/序列搜索问题，也都使用有限算子集、表达式深度约束、常数内层优化和 reward-driven search。因此如果 `0616-fex` 的 idea 是“改进 RL/FEX 的表达式树搜索效率”，本文是直接相关 prior，必须引用。

但本文目标是一般数据拟合式 symbolic regression，评价在 Nguyen、Jin、Livermore、SRBench Black-box；FEX 的核心场景是 PDE/PIDE/动力学问题，用 PDE residual、边界条件、积分项或物理约束定义目标，并依赖 FEX 的 `S_k` approximation theory。本文没有 PDE residual、没有高维 PIDE、没有 FEX 的 Adam+BFGS 求解管线，也没有处理 PDE collocation/积分误差。因此它不是 FEX 方法族的直接竞品。

真正的风险在叙事：今后不能再把“RL/MCTS 搜索表达式树 + BFGS 常数拟合 + GP mutation/crossover”包装成 FEX 的新颖点；这些在一般 SR 社区已经很成熟。FEX 论文应把差异化压在 PDE/PIDE 目标、`S_k` 理论、高维性、物理 residual、以及 FEX-specific controller/inner optimizer 诊断上。

### 对 0616-fex 各机会点的影响

- **FEX 搜索复杂度理论**：本文给出一个有用但很窄的极值 bandit 理论模板。若 reward tail 近似 polynomial，UCB-extreme 有 regret bound；若 tail exponential，找到最优表达式需要指数级样本。这可作为 FEX 搜索复杂度论文的“局部 toy model”，但不能直接声称解决 FEX controller 收敛率，因为 FEX reward 受 PDE residual、内层参数优化、非独立采样和策略更新影响。
- **FEX failure taxonomy**：本文的 reward-tail diagnostic 很值得借。它用 fully random node selection、200k evaluations、100 runs、KDE bandwidth 0.25 来估计 tail decay，并展示 state-jumping 如何把 tail 参数从 `[4,6]` 改到 `[2,4]` 或更低。FEX 可照搬这个诊断框架，比较不同 PDE、算子集、树深、频率分离、参数简并下的 reward-tail 形状。
- **FEX curriculum/depth growing**：本文最强结果来自非局部 state-jumping，不是 UCB-extreme。Ablation 显示去掉 state-jumping 后 Nguyen 平均 recovery 从 93.25% 掉到 53.08%，去掉 UCB-extreme 只掉到 90.83%。这提示 FEX 的搜索效率瓶颈可能更需要“模块复用/非局部编辑/高分候选记忆”，而不是只调 RL controller 的 exploration constant。
- **LLM+FEX skeleton**：本文没有 LLM，但它给了明确插口：LLM 可以替代 random rollout、指导 mutation/crossover、生成 candidate skeleton、或估计 action prior。对 FEX 来说，LLM 应该服务 PDE 解结构和算子集缩窄，而不是泛化地做 SR token prediction。
- **EGG/equivalence pruning + FEX**：本文的 Top-N queue 和 bidirectional propagation 会保存许多等价或近等价表达式；FEX 若加入 e-graph/equality pruning，可能同时减少重复评估和改善 queue 质量。这里可与 EGG-SR 对接。
- **Inner optimizer 风险**：本文用 BFGS 优化常数，反引文 SAGE-Fit 明确指出“好结构可能拿到坏分数”来自参数优化失败。FEX 也使用 Adam+BFGS，所以下一步不能只看树搜索；必须把结构搜索失败与内层参数优化失败分开诊断。

### 可直接复用的方法/设置

- 把 FEX controller 的一部分改写成 MCTS-FEX baseline：state 是 partial expression tree，terminal reward 是 PDE residual/BC loss 的变换，selection 用 UCB-extreme。
- 节点维护 Top-N priority queue，本文取 `N=500`；FEX 可先用较小 N 做 ablation，再看 memory/compute tradeoff。
- State-jumping：single-point crossover；mutation 包括 uniform mutation、node replacement、subtree insertion、subtree shrinkage。本文设置 `g_s=0.2`、`g_m=0.1`、`\epsilon=0.2`。
- UCB-extreme 默认 `c=1`、`\gamma=0.5`；但作者自己的 appendix 表明无 state-jumping 时参数非常敏感，FEX 不应把这个当稳健超参。
- 评测模板：Full / no-UCB-extreme / no-state-jumping / UCB1-only 四组消融，再加 reward-tail KDE 诊断。这正好适合作为 FEX failure-taxonomy 的核心实验表。
- 对 FEX high-dimensional PDE，不能只报 final loss；应同时报表达式复杂度、常数优化成功率、queue diversity、tail decay、controller entropy 和 compute budget。
