# Amortized Reasoning Tree Search: Decoupling Proposal and Decision in Large Language Models (2602.12846)

- **Year**: 2026.02
- **Authors**: Zesheng Hong, Jiadong Yu, Hui Pan; The Hong Kong University of Science and Technology (Guangzhou)
- **Venue**: arXiv
- **Keywords**: RLVR, reasoning tree search, flow matching, GFlowNet, process reward model, MATH-500, long-tail reasoning, verifier-guided search
- **One-liner**: 用 flow verifier 解耦生成和决策，恢复 RLVR 压掉的长尾推理路径。

## 精华提炼

### 管线/方法
- 论文的问题设定是：RLVR 在数学推理中提升平均正确率，但会把低概率的正确推理轨迹压到采样不可见区域。
- 作者把这一现象称为 **Normalization Squeeze**，核心机制是有限 rollouts 只更新采样到的轨迹，未采样轨迹的 logit 不变，但 softmax 归一化分母增大，导致概率被动衰减。
- 理论部分把推理轨迹空间记为 `T`，标准 RLVR 近似为在有限样本集 `S_k` 上做稀疏更新。
- 对任意未采样轨迹 `tau notin S_k`，论文推导出 `pi_{k+1}(tau)=pi_k(tau)/(1+alpha_k)`。
- 其中 `alpha_k` 是被奖励样本带来的 partition function shift，只要奖励为正，未采样轨迹就被乘以小于 1 的衰减因子。
- 作者进一步强调，正确性不等于生存；一个正确 trace 只有进入采样集合才会被强化。
- 低温采样设为 `T in [0.6, 0.8]` 时，高概率路径被进一步放大，低概率正确路径更难进入 batch。
- 有限 rollouts 也会形成隐式高通 likelihood filter：若 `pi(tau) << 1/N`，该 trace 在训练中近似不可见。
- 方法上，作者提出 **Amortized Reasoning Tree Search (ARTS)**，目标不是更新 generator，而是保留 base proposer 的高熵覆盖，再训练 verifier 做搜索排序。
- ARTS 的 Proposer 是冻结的 `Qwen-2.5-7B` base model。
- ARTS 的 Verifier 是 `Qwen-2.5-1.5B-Instruct`，用作 causal feature extractor。
- Verifier 输出的是 flow value `F_theta(s)`，而不是普通 PRM 的局部 step quality。
- 训练数据构造叫 **Sparse Deep Fork (SDF)**。
- SDF 对每道数学题先采样 `N=16` 条主轨迹，形成 reasoning backbone。
- 在主轨迹中遍历中间节点，以 `p=0.5` 的概率选 pivot，再从该 pivot 多 rollout 一条 fork。
- 这样训练图从单线轨迹变成稀疏 DAG，成本约为 `O(1.5N)`，远低于完整树展开。
- SDF 的作用是让 verifier 看到同一前缀下的反事实分叉，例如一个错误主路径旁边存在正确 fork。
- 终止节点只用最终答案可验证信号标注，没有人工 dense step label。
- 终止 reward 是手工 soft reward：正确为 `10`，错误为 `0.1`。
- 非零错误 reward 用于避免 `log 0`，作者称为 Log-Flow Smoothing。
- 训练目标是 **Reasoning Flow Matching (RFM)**。
- RFM 对每个非终止状态要求 `log F(s_t)` 接近 `log sum_{s' in C_obs(s_t)} F(s')`。
- 换言之，父节点的 flow 近似等于已观察子节点 flow 之和。
- 终止节点由 `L_Leaf` 回归到 soft reward。
- 由于只观察稀疏子节点，RFM 学到的是 true flow 的 conservative lower bound。
- 作者认为这个 lower bound 对排序足够，因为 reward 设计让任何含正确后续的子树至少贡献 `10`，全错子树只积累低背景流。
- 推理时，ARTS 用 verifier 对候选轨迹或候选子状态排序，实验主体是 Best-of-N re-ranking。
- PRM/RM baseline 聚合 step score 时用 minimum value，ARTS 则用 terminal flow value `F(s_T)`。
- 作者还给出 visibility amplification 推导：若每步 proposal width 为 `K`、深度为 `H`，稀有轨迹发现概率可近似放大为 `K^H * pi_0(tau)`。
- 这个论证支持其核心立场：长尾推理更适合把 compute 放到 inference-time search，而不是用 RL 把路径压进参数。

### 关键技术细节
- Base proposer：`Qwen-2.5-7B`，冻结，不做 RL 或 SFT。
- Verifier backbone：`Qwen-2.5-1.5B-Instruct`。
- 精度与加速：`bfloat16`，Flash Attention 2。
- LoRA：rank `r=16`，`alpha=32`，dropout `0.1`。
- LoRA target：所有 linear projections，包括 `q/k/v/o/gate/up/down`。
- 更新参数量：约 `<0.3%`。
- Value head：2-layer MLP，结构为 Hidden -> GELU -> Dropout -> Scalar。
- 状态聚合：Last-Token Aggregation，从当前 step 最后一个 token hidden state 预测 `F(s_t)`。
- 训练数据来源：MATH training split。
- 训练采样：每题 `N=16` trajectories。
- 采样温度：`T=0.6`。
- nucleus sampling：`top-p=0.95`。
- SDF 分叉概率：`p=0.5`。
- 终止 reward：correct `10`，incorrect `0.1`。
- 评测集：MATH-500。
- 主评测指标：Pass@1、Best-of-N / BoN@2/4/8/16、Gap Closed。
- RL baseline：SimpleRL-Zoo GRPO，初始化于 `Qwen-2.5-7B`，全参数 RLVR fine-tune。
- Verifier baselines：Pointwise PRM，MSE step-level regression；Pairwise RM，Bradley-Terry ranking loss。
- 论文没有给出完整代码仓库或独立复现实验报告，复现性主要依赖 tex 中的实现细节。

### 核心结果
- Base `Qwen-2.5-7B`：Pass@1 为 `57.7%`。
- Base random sampling BoN@16：`73.9%`。
- SimpleRL-Zoo GRPO：Pass@1 为 `74.7%`。
- Pairwise RM BoN@16：`67.4%`，低于 base random BoN@16。
- Pointwise PRM BoN@16：`73.8%`，接近 base random BoN@16。
- ARTS BoN@16：`74.6%`，几乎追平 GRPO 的 `74.7%`，但不改 generator。
- Oracle@16 上界报告为 `88.6%`。
- Gap Closed：Pairwise RM `31.3%`，Pointwise PRM `52.1%`，ARTS `54.7%`。
- BoN scaling 有 crossover：低预算下 PRM 更强，`N=16` 时 ARTS 反超。
- 难度分层中，Level 4：Base `55.3%`，PRM `72.7%`，ARTS `77.3%`。
- Level 5：Base `32.2%`，Pairwise RM `46.3%`，PRM `47.8%`，ARTS `50.0%`。
- 题型分层中，Algebra：PRM `89.5%`，ARTS `88.7%`，PRM 略强。
- Number Theory：PRM `87.1%`，ARTS `80.7%`，PRM 明显更强。
- Geometry：PRM `58.5%`，ARTS `61.0%`。
- Precalculus：PRM `58.9%`，ARTS `62.5%`。
- Counting & Probability：PRM `73.7%`，ARTS `79.0%`，ARTS 高 `+5.3%`。
- Rare Solutions subset：base proposer 16 条中最多 2 条正确，random `8.5%`，ARTS `27.0%`，约 `3.2x`。
- Counting & Probability rare subset：random `7.5%`，ARTS `60.0%`，约 `8.0x`。
- Extinction Zone：GRPO Pass@16 为 `0%` 的问题上，random `2.7%`。
- 同一 Extinction Zone 上 PRM/RM 只有 `1.7%`，比 random 更差。
- 同一 Extinction Zone 上 ARTS 为 `6.9%`，作者称关闭 `25.8%` gap。
- 最反直觉结果：普通 verifier 在极长尾问题上可能压制稀有正确路径，表现低于随机抽样。
- 另一个关键结果：不更新 generator 的 inference search 可以达到全参数 GRPO 的平均准确率水平，但成本从训练转移到推理。

### 明确局限
- 作者承认 ARTS 是 selector，不是 creator；base proposer support 中没有正确路径时，verifier 无法恢复。
- 推理成本高于 greedy RL policy；Best-of-N 或 tree search 增加 latency 和 token cost。
- reward shaping 使用 `10` vs `0.1` 的手工设定，缺乏原则性概率推导。
- 理论分析依赖 function-space orthogonality 假设，即采样 trace 的更新不会通过泛化任意提升未采样 trace；真实 LLM 参数共享下该假设只是近似。
- 实验集中在 MATH-500，外推到代码、agent planning、科学推理仍只是 future direction。
- 反引文查询为 0，说明目前还没有后续独立工作验证或挑战其结论。
- 没有看到标准公开 artifact 或第三方复现，训练数据构造和 verifier 训练细节虽较多，但仍不足以直接复刻全部结果。
- 论文使用 ICLR 2025 样式和 `iclrfinalcopy`，但 arXiv 日期是 2026.02；venue 应按 arXiv 处理。
- SDF 只构造稀疏 DAG，RFM 学到的是 observed support 的 lower bound，若采样没覆盖关键 fork，排序仍可能失败。
- 与 PRM 相比，ARTS 在 Algebra 和 Number Theory 上不占优，说明 flow objective 不是全面替代，而是更偏高分叉、高熵任务。
- Extinction Zone 的定义依赖该文选择的 GRPO baseline，如果换 RL recipe 或 sampling budget，子集组成可能变化。
- 论文没有报告统计显著性区间或多 seed 方差，`74.6%` vs `74.7%` 的接近程度应谨慎解释。

### 跟我们的关系
- 对 `0616-fex` 来说，这篇最核心的可借用点是 **generator/verifier 解耦**：保留高熵 proposer，把选择权交给单独训练的 verifier。
- 如果 `0616-fex` 的 claim 是“用 flow 或 conservation-style objective 做 inference-time exploration / selection”，这篇会高度撞车。
- 撞车最具体的 claim：RFM verifier 学 `F(s) ~= sum F(children)`，用 flow value 而不是局部 PRM 分数做推理路径排序。
- 另一个撞车点：作者已经把长尾推理失败归因于 RLVR 的 mode-seeking 和 finite-sampling visibility barrier。
- 如果我们的贡献只是“RL 会压缩推理多样性”，这篇和 Yue et al. 2025 已经构成直接 prior。
- 如果我们的贡献是 FEX 的新数据构造或 verifier 训练协议，需要和 Sparse Deep Fork 明确区分。
- 可借用的方法：SDF 的低成本 fork 采样、soft terminal reward、last-token flow head、BoN@N re-ranking 评测。
- 可借用的诊断：按 rare-solution subset、extinction subset、题型高熵程度拆分，而不只看平均准确率。
- 可引用的位置：作为“flow-guided inference-time search for long-tail reasoning”的最近 arXiv prior。
- 差异化建议：不要只做 MATH-500；应扩到代码、规划或工具调用，并加入多 seed、artifact、trace-level audit。
- 另一个差异化方向：把 flow verifier 与 RL policy 结合成 expert iteration，而不是停留在 frozen proposer selector。
- 对 novelty 审查而言，接下来搜索关键词应覆盖 ARTS、Reasoning Flow Matching、Sparse Deep Fork、GFlowNet verifier、RLVR coverage collapse。
