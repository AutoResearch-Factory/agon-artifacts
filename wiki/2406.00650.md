# Cluster-Robust Jackknife and Bootstrap Inference for Logistic Regression Models (2406.00650)

- **Year**: 2024.06
- **Authors**: James G. MacKinnon (Queen's University), Morten Ørregaard Nielsen (Aarhus University), Matthew D. Webb (Carleton University)
- **Venue**: arXiv preprint (accompanies public Stata package `logitjack`; group is the same MacKinnon-Nielsen-Webb trio behind the canonical linear-model cluster-bootstrap papers cited throughout)
- **Keywords**: cluster-robust inference, logistic regression, cluster jackknife, wild cluster bootstrap, CRVE, linearization, score vector, clustered standard errors, small-number-of-clusters inference
- **One-liner**: 在完整样本 MLE 处对 logit 的 score/信息矩阵做线性化，把 cluster-jackknife 和 wild bootstrap 从 G+1 次非线性重估算降到 G 次 k×k 矩阵求逆，实测提速约 41 倍。

## 精华提炼

### 管线/方法

- 研究对象：logit 模型 $\Pr(y_{gi}=1\mid X_{gi})=\Lambda(X_{gi}\beta)$，$g=1,\ldots,G$ 个 cluster，cluster $g$ 内 $N_g$ 个观测，共 $N=\sum_g N_g$ 个观测，$k$ 个解释变量（$\beta$ 是 $k$ 维向量）。只假设 cluster 间独立，cluster 内允许任意形式的相关。
- 起点：现有唯一广泛使用的 logit CRVE（记作 CV1，式 (7)）是普通 sandwich 估计量的 cluster 版，Stata/R 默认配 $N(0,1)$ 临界值——论文证明这个组合在有限样本下过度拒绝（over-reject）非常严重。
- 第一类解法（未线性化）：cluster jackknife。依次剔除每个 cluster 重新做完整非线性 MLE，得到 $G$ 个 delete-one-cluster 估计 $\hat\beta^{(g)}$，围绕均值（CV3J）或围绕全样本估计 $\hat\beta$（CV3，更常用）取方差。代价是 $G+1$ 次完整非线性优化。
- 第二类解法（核心创新）：线性化。不真的重新拟合，而是在**全样本 $\hat\beta$ 处**用一步 Newton 近似 $\hat\beta^{(g)}-\hat\beta \approx (\hat J-\hat J_g)^{-1}(\hat s-\hat s_g)$，其中 $\hat s_g,\hat J_g$ 是 cluster $g$ 对 score 向量和信息矩阵的贡献（式 scores/infolrm，均是 $k$ 维/$k\times k$ 的解析量，对 logit 有闭式表达）。由此得到 CV3L（线性化 cluster jackknife）。
- 第三类解法：把同一线性化用在 wild cluster bootstrap 上，得到 WCLR/WCLU 系列（Restricted/Unrestricted）。做法：给每个 cluster 的 score 乘一个均值 0 方差 1 的随机符号变量 $v_g^{*b}$（通常 Rademacher，$G\le12$ 时改用 Webb 6-point 权重），求解一次 $k\times k$ 线性方程得到 bootstrap 系数 $\ddot b^{*b}$，再算 bootstrap $t$ 统计量；"-C"（classic，直接用原始 score）和 "-S"（score-transformed，用 CV3L 式的变换去掉估计过程带来的失真）两个变体，"-S" 系统性更好。
- 全部方法封装进公开 Stata 包 `logitjack`（GitHub + Stata SSC），并给出两个真实数据实证例子做校准检验（含 placebo regression：随机指派一个假处理变量，看各方法在真 null 下的经验拒绝率是否接近名义水平）。

### 关键技术细节

- **变量数 $k$**：仿真中 $k$ 从 2 扫到 20（主设定 $k=7$：1 常数项 + 5 无关回归量 + 1 处理哑变量）；实证例 1（现金奖励对通过率的影响）$k\approx10$（9 个协变量+处理哑变量，无 school FE）；实证例 2（学费对入学率影响）含 year FE + province FE（province 恰是 cluster 变量本身），触发论文专门讨论的 "cluster fixed effects" 复杂情形（$G+k-1$ 个参数需要 generalized inverse 处理，且只有 $K=k-1$ 个 slope 系数能做 cluster-jackknife 推断，$G$ 个 FE 系数本身不能）。
- **聚类粒度 $G$ 与 $N_g$**：仿真主实验 $G\in\{12,18,24,30,36,42,48,60,72\}$，"almost ideal case" 用 $G=50$（与本 proposal 的 `>=50` storm 目标恰好同量级）；cluster 大小不均匀度由 $\gamma\in[0,4]$ 控制，$\gamma=4$ 时最大/最小 cluster 比可达约 47 倍，$\gamma=2$（默认）约 7 倍。两个真实例子：34 个 school（12–146 学生）、10 个 province（3402–37109 人，比例约 11 倍）。
- **计算复杂度（论文原话，Tuition 例子实测）**：计算 CV3（完整 cluster jackknife）占该实验全部计算时间的约 70%，因为需要 $G$ 次额外 logit 估计；CV3L（线性化版）与 CV3 结果几乎相同，但成本只有 CV3 的约 1/41——即约 41 倍加速。Wild bootstrap（WCL 系列）每个 replicate 只是一次 $k\times k$ 线性求解，因此哪怕 $B=999{,}999$（论文两个实证例子里为了"minimize dependence on random numbers"用到近似穷举的 $B=9{,}999{,}999$）依然远比 $G$ 次非线性重估算便宜。
- **关键失效模式**：deleting 某个 cluster 后若该子样本出现完美分类器（perfect classifier），$\hat\beta^{(g)}$ 会发散，导致 CV3/CV3J 剧烈失真——论文要求任何实现都要检测并剔除/标注这类子样本。
- **jackknife 可推广到 $\beta$ 的光滑函数**：论文明确指出若关心的是 $\delta=\beta_2/\beta_3$ 这类 $\beta$ 的光滑函数，只需对全样本和每个 delete-one 样本各算一次 $\hat\delta$，再套用同样的 jackknife 方差公式即可；但这个推广的前提仍是"$\delta$ 是同一个被线性化参数向量的光滑函数"——AUPRC 不满足这个前提，因为它不是 CNN 权重（更不是某个 $k$ 维 $\beta$）的光滑函数，而是预测排序的阶梯函数，这正是本文机制无法跨过的根本边界，而非可以绕过的实现细节。
- **对称 vs 等尾 bootstrap $P$ 值**：论文提供两种 bootstrap $P$ 值（式 bootps 对称、bootpet 等尾），提到 logit 斜率系数常"偏离零方向有偏"，等尾版本可能更合适，但实测两者差异不大；本任务若最终报告的是单侧假设（"(D,J) 是否提升 AUPRC"），这组讨论提示应明确选定单侧/等尾构造并说明理由，而不是默认套用对称区间。
- **推荐的 4 种置信区间构造**：(1) $t(G-1)$ 分位数 + CV3L 标准误（最便宜）；(2) 同上但用 CV3 标准误（更贵，偶尔更准）；(3) WCLU-S 的 studentized bootstrap 区间（非对称，不假设枢轴量正态）；(4) $t(G-1)$ 分位数 + WCLU-S bootstrap 标准误（对称）。反直觉发现：(4) 有时比理论上更"精细"的 (3) 覆盖率更好——不能想当然认为 studentized 区间总是更优，要靠仿真/校准实测比较。
- **诊断报告清单**：论文明确建议任何实证应用都报告：因变量均值、cluster 数、"treated" cluster 数、以及至少一个 cluster 大小差异度量（并引用 CSS_2017 的"有效 cluster 数" $G^*$ 概念）。
- **两个实证例子的具体数字显示方法选择能翻转结论**：现金奖励例（$N=1861,G=34,G_1=16$）中 logit 的 CV1 标准误给出 $P=0.0296$（拒绝），但 CV3/CV3L 给出 $P\approx0.055$（临界，CI 跨零）；学费例（$N=127518,G=10$）里 CV1 给出 $P=0.0216$（拒绝），而更可靠的 WCLR-S bootstrap 给出 $P=0.1551$（远不显著）——placebo regression 校准显示恰恰是给出小 $P$ 值的方法（CV1、WCLU-C）在真 null 下过度拒绝最严重，即"看起来显著"的方法反而最不可信。这对本 proposal 的启示是：POSITIVE/NULL/NEGATIVE 判据不能只看某一种标准误下的点估计+CI 是否跨零，必须像本文一样用 placebo/graph-null 校准去交叉验证，避免选中一个恰好过度拒绝的方法而误判 POSITIVE。
- **Linear Probability Model 等价性附带讨论**：论文证明当 logit 用线性近似（LPM）替代、且用普通 WCR/WCU wild bootstrap 时，结果通常与 WCLR/WCLU 非常接近；这提示如果未来 storm-cluster bootstrap 需要一个更简单、更快的 sanity-check baseline，可以先在一个线性概率/线性判别的替代评分上跑通同一套重采样流程，再迁移到 AUPRC，用于交叉验证重采样管线本身没有 bug（而不是作为最终判据）。

### 核心结果

- CV1 + $N(0,1)$（Stata/R 当前默认）过度拒绝最严重；CV1 + $t(G-1)$ 好一些但仍不可靠。
- CV3/CV3L 用 $t(G-1)$ 临界值后表现始终优于 CV1，是"总是更可靠"的基线选择；CV3L 与 CV3 几乎总是给出几乎相同的推断，仅在线性化明显失准时（$E(y_{gi})$ 接近 0 或 1）出现可观差异。
- WCLR-S（restricted, score-transformed）多数情形下表现最好；WCLU-S 在置信区间构造上通常优于 WCLU-C，WCLU-C 区间"不可推荐"。
- **对本任务最关键的一组结果**：当"treated"（正类）cluster 数 $G_1$ 相对 $G$ 很小时（如 $G=24, G_1=2\text{–}3$），**所有方法**（包括 bootstrap）严重欠拒绝或过拒绝；当 $\pi$（因变量均值）远离 0.5 时同样所有方法退化。相比之下，单纯的 cluster 数量大（如 $G=50$）、cluster 大小不均（$\gamma$ 大）在其他条件理想时，退化程度小得多。换言之：**决定可靠性的第一主因是"少数类 cluster 的绝对数量"，不是 cluster 总数**。
- $G=50$、平衡、无组内相关（"almost ideal case"）时，bootstrap 方法"几乎完美"，CV1 只轻微过拒绝——这是本文给出的、与本 proposal 目标 cluster 数最接近的一个正面基准点。

### 明确局限

- 全文只处理**参数化 GLM（logit/probit 等对称链接的二元响应模型）系数向量**的推断，线性化的前提是模型由一个光滑、闭式可导的一阶条件（score=0）定义，且待估参数是低维（$k$ 从 2 到 20）向量。完全没有涉及非参数/秩统计量（如 AUC、AUPRC）、没有涉及深度网络预测的评估指标推断、也没有涉及"比较两个已训练模型的预测性能"这类 paired 比较问题。
- 未讨论 cluster fixed effects 情形下的边际效应/预测概率的标准误——作者明确说这需要完整的 $G+K$ 维方差矩阵，而现有方法都做不到，将其列为未解决问题。
- 所有仿真都基于一个特定、便于生成 intra-cluster correlation 的 DGP（式 ugi/ygi 混合均匀分布方案），结论"应谨慎解读，因为 DGP 设定方式对结果有实质影响"——作者自己反复强调这一点。
- 论文没有討論 cluster 数远大于 100（如成百上千 cluster）时的行为，也没有讨论 cluster 本身的定义/构造是否恰当（如空间/时间相关单位如何划成 cluster）这一更上游的问题。

### 跟我们的关系

- **直接结论：本文的具体算法（CV3L 线性化、WCL wild bootstrap）不能直接套用为 storm-cluster bootstrap 的算法基础**，原因是对象不匹配：MNW 重采样的是"cluster 对一个低维（$k\sim7$–20）GLM 参数向量的 score 贡献"，核心技巧是在全样本 MLE 处做一步 Newton 线性化以避免 $G$ 次重新优化；而本 proposal 要做推断的对象是 **Delta AUPRC**——两个已训练冻结的 CNN 在 held-out storm 上的排序类（非参数、不可解析求导）性能指标之差，根本不存在一个"$k$ 维参数 + 闭式 score/信息矩阵"可供线性化。
- **更关键的一点：MNW 要解决的计算问题在本任务里根本不存在**。MNW 之所以要发明线性化，是因为 cluster jackknife 的"删掉一个 cluster 重新估计"在 GLM 里意味着重新做一次完整非线性优化（$G+1$ 次），代价高昂（实测占比 ~70% 总计算时间）。而本任务的 raw-only/raw+(D,J) 两个 CNN 都已训练完毕并冻结，"删掉一个 storm cluster 再算一次 AUPRC"根本不需要任何重新训练——只是过滤已算好的 held-out 预测数组后重算一次排序统计量，本身就是廉价操作（$O(n\log n)$ 级别）。也就是说，无论是留一 storm jackknife 还是重复上千次的 resample-with-replacement bootstrap，对我们而言从一开始就是便宜的；MNW 论文最大的卖点（避免 $G$ 次重优化）对我们不构成任何阻碍，因此也不需要被"借用"。**计算复杂度不是 storm-cluster bootstrap 的瓶颈**，这一点可以直接写进 proposal 回应 Validation Focus 缺口，排除"是否需要线性化近似"这一虚假选项。
- **真正能迁移的是方法论纪律，而非算法本身**：(a) $G_1$（少数类/"treated" cluster 数）过小时全部方法失灵这一发现，精确对应到本任务的真实风险——`>=50` storm 里到底有多少个"阳性"（含强对流事件标签）storm cluster，而不是 storm 总数，才是决定 bootstrap CI 能否压到 0.01 实用阈值内的第一主因；这直接支撑并具体化了 proposal 已被要求补的功效分析（应按"阳性 storm cluster 数"而非"storm 总数"做功效核算）。(b) 用 $t(G-1)$ 而非渐近正态做小样本修正的纪律，可直接套用到 storm-cluster bootstrap 的 CI 构造（或其百分位数/BCa 类比）。(c) 论文推荐的诊断报告清单（因变量均值、cluster 数、"treated" cluster 数、cluster 大小变异度）可直接作为 storm-cluster bootstrap 结果表的报告模板。(d) 不要预设某一种 CI 构造（percentile/studentized/对称+t 修正）天然最优，应像本文一样做多种构造的经验横向比较。(e) 本文两个实证例子都用 placebo regression 做校准检验，这与 proposal 已规划的 graph-null 校准 bootstrap（Gate 0）思路一致，构成外部方法论支持证据。(f) MNW-influence（summclust，leverage 诊断）提供了"单个 cluster 对结果贡献过大"的诊断思路，可类比用于检查是否有个别 storm cluster 因贡献了不成比例多的正例而成为高杠杆点。
- **(g) 两个可直接复用的数值细节**：论文默认用 $B=999$ 个 bootstrap replicate（正式实证报告用 $B=999{,}999$ 或 $9{,}999{,}999$ 以压低随机数噪声），这个数量级对我们同样适用且廉价，storm-cluster bootstrap 可以直接采用甚至不必设太低；权重方案上，论文规定 $G\le12$ 时才需要换用 Webb 6-point 权重代替 Rademacher（因为 $G\le12$ 时 Rademacher 只能生成有限种符号组合，不够丰富），而本 proposal 目标 storm cluster 数 `>=50` 远超这个阈值，若未来确有必要采用符号翻转类重采样，直接用标准 Rademacher/纯 resample-with-replacement 即可，不必引入 Webb 权重这类小样本专用机制——这是一个可以直接排除的复杂度分支。
- **下一轮搜索方向**：本文所在的是"GLM 系数 cluster-robust 推断"econometrics 文献线，不是我们真正需要的。真正贴题的是**分组/聚类数据上排序类预测性能指标（AUC/AUPRC）的非参数 cluster bootstrap 推断**——如 DeLong 检验的聚类推广、Obuchowski 风格聚类 ROC 推断、或 ML 文献中关于"分组测试集上正确做 CV/bootstrap 而不泄漏"的工作，这类文献目前完全没有覆盖到。此外 S2 反引文中出现的 MacKinnon 2026 新作《When Can We Trust Cluster-Robust Inference?》（arXiv:2604.02000）标题直接对准"何时可信"这一问题，值得下一轮精读，可能直接提供小 $G_1$/小样本场景下的可信边界判据。

## Read by: 0710-causal-scs (2026-07-13)

本轮是 causal-scs-indicator proposal v1 review（REVISE）针对性文献调研第二轮，目标是判断本文能否作为 storm-cluster bootstrap（storm system 当 cluster 单位，对 raw-only vs raw+(D,J) 两个已训练冻结 CNN 的 Delta AUPRC 做 cluster-robust 推断）的具体算法基础。

**结论：不能直接套用/改造，但方法论纪律高度相关。** 核心原因见上文"跟我们的关系"：本文全部方法建立在"cluster 对一个低维 GLM 参数的 score/信息矩阵贡献可以在 MLE 处闭式线性化"这一结构上，目的是避免 $G$ 次代价高昂的非线性重估算；而 Delta AUPRC 是两个已冻结 CNN 的非参数排序性能指标之差，既没有可线性化的低维参数结构，也没有"重新估计代价高"这个问题需要解决——删一个 storm cluster 重算 AUPRC 本身就是廉价的数组操作。这意味着 proposal 不需要、也不应该尝试把 CV3L/WCL 那套线性化机器搬过来；storm-cluster bootstrap 应该走一条更直接的路径：对 storm cluster 做（留一或重复）resample，重算两个模型的 AUPRC 及其差值，用重采样分布直接构造 CI（percentile/BCa/或 SE+t(G-1) 校正），这在计算上完全负担得起，可以把 $B$ 开到远超 999 而不担心成本。

对 proposal 当前 Validation Focus 缺口（Gate 0 判据翻转不能证明新协议已校准；`>=50` storm 样本量未做功效分析；n=8 seed-CI 曾宽达 ±0.08-0.09）最直接的可操作贡献是：本文用大量仿真证实，**决定 cluster-robust 推断可靠性的第一主因是"少数类（治疗/阳性）cluster 的绝对数量 $G_1$，不是 cluster 总数 $G$**"——这把 proposal 该做的功效分析从"storm 数够不够 50 个"精确重定向为"这 50 个 storm 里有多少个带阳性标签，够不够支撑 0.01 阈值下的可判定性"，并建议直接挪用本文推荐的诊断报告清单（因变量均值、cluster 数、"treated" cluster 数、cluster 大小变异度）作为 storm-cluster bootstrap 结果表的标准报告项。同时，本文两个实证例子采用的 placebo-regression 校准检验，与 proposal 已规划的 graph-null 校准 bootstrap（Gate 0）思路相互印证，构成外部方法论支持。下一轮建议精读反引文中的 MacKinnon 2026 新作 arXiv:2604.02000《When Can We Trust Cluster-Robust Inference?》，以及转向搜索"clustered bootstrap AUC/AUPRC"“grouped bootstrap evaluation metric”一类非参数排序指标聚类推断文献，这才是与 storm-cluster Delta AUPRC 推断真正同构的问题。
