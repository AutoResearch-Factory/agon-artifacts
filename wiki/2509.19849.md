# Analyzing Generalization in Pre-Trained Symbolic Regression (2509.19849)

- **Year**: 2025.09
- **Authors**: Henrik Voigt, Paul Kahlmeyer, Kai Lawonn, Michael Habeck, Joachim Giesen (University of Jena)
- **Venue**: ICLR 2026 (submission)
- **Keywords**: symbolic regression, transformer, generalization gap, out-of-distribution, pre-training domain, SRBench, recovery rate, pattern memorization
- **One-liner**: First systematic empirical study revealing that transformer-based SR models suffer a severe OOPD generalization gap—they memorize structural snapshots rather than learning compositional building blocks.

## 精华提炼

### 管线/方法

**整体分析框架**：该工作不是提出新方法，而是建立了一套系统评估预训练 SR 模型泛化能力的方法论。

1. **三层域划分（核心方法论贡献）**：
   - **IPD (In-Pre-training-Domain)**：训练数据从与预训练数据相同的数值域采样（如预训练用 [-5,5]^d，训练也用此域）
   - **WIPD (Within-Pre-training-Domain)**：训练数据从预训练域内的子超立方体采样（随机选择一个完全被预训练域包含的小超立方体）
   - **OOPD (Out-of-Pre-training-Domain)**：训练数据从与预训练域显式不相交的超立方体采样（从 [-100,100]^d 中随机选取，拒绝与预训练域重叠的）
   - 同时区分 ID（同训练域测试）与 OOD（不同域测试），形成 3×2 的评估矩阵

2. **分布漂移扰动测试**：用 power-law skew 控制训练数据分布偏离预训练分布的程度：
   ```
   x = a + (b-a) * ((x_init - a)/(b-a))^(exp(-ε))
   ```
   ε=0 恢复原分布，ε>0 向右侧集中，ε<0 向左侧集中

3. **鲁棒性 case study**：在 y=x^2 这个简单问题上施加四种扰动：
   - Input Scaling：缩放输入域范围
   - Input Shift：平移输入域
   - Output Noise：加高斯噪声到 y 值
   - Distribution Shift：改变采样分布

4. **评估指标**：
   - Recovery Rate：通过 SymPy 验证表达式是否与 ground truth 符号等价（差或商化简为常数）
   - Accuracy (R²)：分别在 ID 和 OOD 测试集上计算

### 关键技术细节

**评测的 5 个 transformer 方法**：
- **NeSymReS** (Biggio et al., 2021, 2106.06427)：预训练域 [-5,5]^d, d≤3, 两阶段采样（先选小超立方体再均匀采样）
- **E2E** (Kamienny et al., 2022, 2204.10532)：预训练域通过标准化实现无界，d≤10, diverse mixture 采样策略 + 标准化预处理
- **TF4SR** (Lalande et al., 2023, 2312.04070)：预训练域 [-5,5]^d, d≤6
- **SymFormer** (Vastl et al., 2022, 2205.15764)：预训练域 [-5,5]^d, 均匀采样
- **TPSR** (Shojaee et al., 2023, 2303.06833)：混合方法，transformer + MCTS

**Baseline**：PySR (Cranmer et al., 2023)，基于搜索的 SOTA 方法，不依赖预训练

**数据集**：SRBench (La Cava et al., 2021, 2107.14351) 的 144 个 ground truth 公式，主要为物理领域中小复杂度公式

**训练设置**：每个公式每种 regime 采样 255 个训练点

**补充实验的 ablation**：作者自训练了一个基于 E2E 架构的模型，控制预训练数据生成过程：
- 表达式来源：随机树 vs 枚举 DAG（后者更全面、更少偏差）
- 采样策略：uniform vs diverse mixture
- 预处理：是否标准化输入

### 核心结果

**Recovery 性能（图 1a）**：
- PySR：~45% recovery，三个 regime 几乎一致
- NeSymReS：IPD 31% → WIPD 明显下降 → OOPD 进一步退化
- SymFormer：IPD ~12% → 逐 regime 下降
- E2E、TF4SR：recovery 接近零（所有 regime）
- TPSR：IPD 有 recovery → WIPD/OOPD 急剧退化

**ID Accuracy（图 1b）**：
- PySR：一致高精度（~0.95 R²）
- E2E：在所有 regime 保持 ~0.9+ R²（但 recovery 为 0！）→ **高精度是欺骗性的**
- TPSR：IPD/WIPD 最强（~0.99）→ OOPD 恶化
- NeSymReS：~0.6-0.7 R²，随 regime 下降
- TF4SR、SymFormer：低精度

**OOD Accuracy（图 1c）**：
- PySR：一致高精度
- 所有 transformer 方法在 OOD 下大幅下降
- E2E：R² 下降到 ~0.4

**鲁棒性 Case Study（y=x^2）**：
- Input Scaling/Shifting：NeSymReS 和 SymFormer 在 ε=0 附近有尖锐峰值，稍微偏移即 collapse；TF4SR 在 scaling 的正 ε 方向保持高 recovery（例外）
- Output Noise：极小噪声即导致 catastrophic failure，仅 SymFormer 有微弱韧性
- Distribution Shift：NeSymReS 和 SymFormer 在负 ε 方向保持较好 recovery，呈不对称性

**Ablation 发现**：
- 枚举 DAG 预训练 >> 随机树：IPD recovery 从 35% 跌至近 0
- Diverse 采样优于 uniform：更稳定的 IPD/WIPD 表现
- 标准化：牺牲 IPD/WIPD 换取更稳定的 OOPD（trade-off），但不解决根本问题
- 所有 ablation 配置的性能都显著低于 PySR

**最反直觉的发现**：E2E 在 ID 上 R² 高达 ~0.9，但 recovery 率接近 0%。这意味着它只是一个高度灵活的插值器（类似多项式回归），而非真正的符号发现器。

### 明确局限

**作者自己说的**：
1. SRBench 主要为中小复杂度物理公式（138/144 是物理），不覆盖所有科学领域
2. 只评估了 5 个代表性模型，未来架构可能不同
3. 没有明确测试经济学、化学等其他科学领域

**我读出来的**：
1. 没有真机实验，纯 benchmark 分析
2. 所有评测基于 synthetic ground truth，不涉及真实噪声数据
3. Ablation 中自训练的模型是基于 E2E 架构，结论可能受限于特定架构选择
4. 没有探讨为什么 NeSymReS recovery 明显高于 E2E/TF4SR（这可能与两阶段采样策略或架构细节有关）
5. 没有考虑多模态数据（表格+文本+图像），只处理纯数值输入
6. 没有讨论常数优化（constant fitting）对 recovery 率的影响差异

### 跟我们的关系

对 **FEX (RL for Symbolic Regression)** 课题：

1. **不撞车**：该文批评 transformer-based SR 的泛化能力，主张 search-based 方法更鲁棒。FEX 使用 RL（属于 search-based 范式的一种变体），与 transformer 预训练方法不同。该文的结论**间接支持 FEX 的方法论方向**——即需要超越纯 pattern-matching 的搜索/推理机制。

2. **可直接用的工具/方法论**：
   - **IPD/WIPD/OOPD 三层评估框架**：可直接用于评估 FEX 的泛化能力，是一个很好的实验设计模板
   - **四种鲁棒性扰动测试**（scaling/shifting/noise/distribution shift）：可直接复用
   - **SRBench 基准**：已被广泛使用，FEX 应在此上报告结果以便公平比较
   - **Power-law skew 公式**：用于测试分布鲁棒性

3. **该引**：是的。如果 FEX 论文中讨论了 transformer-based SR 的局限性作为动机，这篇是最直接的引用。它提供了 transformer SR 泛化差的经验证据。

4. **差异化定位**：
   - FEX 的 RL 搜索机制 vs transformer 的 pattern-matching：该文为这个对比提供了经验基础
   - 可以声称 FEX 避免了该文指出的 "memorizing structural snapshots" 问题
   - 该文指出 hybrid 方向是未来，FEX 如能与 transformer 组件结合（如用 transformer 生成搜索先验），可能是好的 hybrid 方案
