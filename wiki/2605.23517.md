# Probabilistically checkable proofs for the Existential Theory of the Reals (2605.23517)

- **Year**: 2026.05
- **Authors**: Jack Stade; 机构未在 tex 中列出

- **Keywords**: existential theory of the reals, PCP theorem, MAX-ETR-INV, hardness of approximation, assignment tester, midpoint code, locally decodable codes, gap amplification
- **One-liner**: 把 Dinur 式 PCP 扩展到 ∃R 的 MAX-ETR-INV。

## 精华提炼

### 管线/方法
- 输入对象是 ETR-INV 及其优化版 MAX-ETR-INV：变量限制在 `[1/2,2]`，约束只有 `x=1`、`xy=1`、`x+y=z`。
- 目标是证明存在常数 `c<1`，使 Gap-MAX-ETR-INV_c^1 为 `∃R`-hard。
- 证明先绕到中间问题 ETR-C(q)，约束包括 `x=y`、`x=(1+q)y`、`x+y=z`、`x+y=z+w`、`xy=z`、`xy=zw` 和空约束。
- 对 ETR-C(q) 证明常数 gap hardness，再用 gap-preserving reduction 回到 MAX-ETR-INV。
- 主体沿用 Dinur PCP theorem 的三步 gap amplification。
- Step 1 是 expanderization：把约束图变成常度数、谱扩展良好的图，保持 satisfiable，并把 UNSAT 至少乘以常数 `η1`。
- Step 2 是 graph power：使用 Radhakrishnan 的 lazy random walk 版本，使 UNSAT 至少按 `η2 t min(UNSAT,1/t)` 放大，且常数不依赖 alphabet 大小。
- Step 1 和 Step 2 可直接推广到无限 alphabet，因为它们只依赖约束图结构，不依赖有限字母枚举。
- 真正新内容是 Step 3：为连续实数约束构造 assignment tester，把大 alphabet 降回 `[1/2,2]^4`。
- assignment tester 的输出 alphabet 是 `[1/2,2]`，查询长度 `ell=4`，错误阈值 `zeta=1/1000`，输出约束仍在 `C(q)` 里。
- tester 的核心是一个非标准的 midpoint code，用于在实数或任意 Abelian group 上做常数查询测试和解码。
- midpoint code 把 `G^n x G` 编码成网格 `{0,...,k1}^n -> G` 上的仿射函数 `A(x)=g+sum beta_i x_i`。
- midpoint test 随机选 `x1,y1` 和小偏移 `z`，设 `x2=k1+z-x1`、`y2=k1+z-y1`，用 4 次查询检查 `A(x1)+A(x2)=A(y1)+A(y2)`。
- 若点出界，测试自动通过；这使边界处理不会破坏常数查询。
- decoder 用 2 次查询输出 `A(x+d)-A(x)`，其中 `||d||_1 <= r`。
- 参数取 `k2=4 ceil(delta^-1) r`，`k1=32 ceil(delta^-1)^2 n r`。
- 重要性质不是 Hamming local testability，而是：若 midpoint test 高概率通过，则存在同一个隐含 `alpha`，所有小 `d` 的 decoder 都高概率输出 `sum d_i alpha_i`。
- 作者明确指出该 code 不是通常意义上的 locally testable 或 locally correctable code，因为远离 codeword 的字符串也可能高概率通过。
- 这不是缺陷，而是正好够 assignment tester 使用：它只需要可恢复相关线性信息。
- 线性测试部分把一个整数矩阵 `M` 给出的 Z-linear map `f:G^d -> G^n` 和 midpoint code 结合。
- 如果 linear test 以 `1-epsilon` 通过，且 `epsilon <= 1/32`，则输入向量在至少 `1-4epsilon` 的坐标上等于某个 `img(f)` 中的向量。
- 对一般 C(q) 约束，作者先用 `threeconstraintconversion` 把任意 `V in P(C(q)^n)` 缩小到 `V' subset R^{n+m}`。
- 这里 `m=1+22n`，`V'` 位于 `[-6q,6q]^{n+m}`，由三类约束切出：`x+y=z`、`(1+x)(1+y)=1+z`、单个 `x=q`。
- 对 `V'` 的点 `alpha`，再引入布尔超立方编码 `u_sigma=prod_i (1+alpha_i)^{sigma_i}`。
- 乘法型约束 `(1+x)(1+y)=1+z` 取对数后变线性，因此用乘法群上的 linear test 检查。
- 加法型约束 `x+y=z` 被转成 `u_{sigma+e_i}+u_{sigma+e_j}=u_{sigma+e_k}+u_sigma`，再对 `u_sigma-u_0` 做加法群 linear test。
- 常数约束 `x=q` 通过随机 `sigma` 检查 `u_{sigma+e_i}=(1+q)u_sigma`。
- constraint test 随机选择这三类子测试之一；若通过概率超过 `1-1/192`，得到某个 `alpha in V'`，且 `u_sigma` 至少在 `1-12epsilon` 的 `sigma` 上匹配其编码。
- 为保证所有辅助变量落在 `[1/2,2]`，需要 `q` 足够小；文中给出极粗但只依赖 `n` 的界，例如 `q <= 2^{-2^{n+m+O(1)}}`。
- 最终 assignment tester 的 error-correcting function 是 `f(alpha)_sigma=prod_i (1+q alpha_i)^{sigma_i}`，blow-up size 为 `2^n`。
- 这个 `f` 的距离至少 `1/2`，证明用 DeMillo-Lipton-Schwartz-Zippel lemma 的仿射函数版本。
- tester 对一条二元约束 `V` 做五类测试：左端编码 linear test、右端编码 linear test、`u` 的 constraint test、左端一致性测试、右端一致性测试。
- 一致性测试随机选 `sigma1,sigma2`，让前 `n` 个坐标反相关，使 `mu1-mu2` 均匀分布在 `{-1,0,1}^n`，再检查 `a_mu1 u_sigma2 = a_mu2 u_sigma1`。
- 如果 assignment test 通过概率至少 `999/1000`，则存在 `(alpha,beta) in V`，且左右编码各至少有 `1/50` 的坐标匹配。
- 组合引理把 assignment tester 接到 Dinur Step 2 的大 alphabet 图上，得到 Step 3 常数 `eta3=1/5000`。
- 令 `t=ceil(2/(eta1 eta2 eta3))`，三步合成后得到 gap amplification：UNSAT 要么翻倍，要么达到某个常数 `alpha`。
- 反复应用 `O(log |E|)` 次，把任意非 satisfiable 的 ETR-C(q) 实例放大到常数 UNSAT。
- ETR-C(q) 的 `∃R`-hardness 来自 Abrahamsen-Adamaszek-Miltzow 的 `ETR_{[1/2,2]}^{1,+,*}`，其中 `x=1` 可用 `x*x=x` 替代。
- 因此得到 Gap-MAX-ETR-C(q)_c^1 为 `∃R`-hard。
- 最后把 MAX-ETR-C(q) gap-preservingly 归约到 MAX-ETR-INV。
- 关键缩小 lemma 对单个 C(q) 约束使用最多 70 个辅助变量、1 个 `x=q`、52 个加法约束、18 个 shifted multiplication 约束。
- 取 `q=2^{-k}`、`k>=6`，每个 C(q) 约束被模拟成恰好 `1266+2k` 个 ETR-INV 约束。
- 归约满足 `(1266+2k)^{-1} UNSAT(phi) <= UNSAT(psi) <= UNSAT(phi)`。
- 这保留常数 gap，完成 MAX-ETR-INV 的 `∃R`-hardness of approximation。
- 论文还给出解复杂度结果：存在 satisfiable ETR-INV 实例，任意满足常数比例约束的赋值都需要某变量生成超过 `n^k` 次的有理域扩张。
- 构造思路是先强制 `x_0^p=2`，选择大素数 `p`，再通过 gap amplification 和 ETR-INV 归约保留代数次数障碍。
- 论文最后给两个正向近似算法：多项式时间 8-factor，和 NP 能力下的 2-factor。
- 8-factor 算法把每个变量独立随机赋为 `1` 或 `2`，每个可满足约束至少以 `1/8` 概率满足。
- NP 2-factor 算法把 `x=1, xy=1` 与 `x+y=z` 两类约束分开，后者变成最大可行线性子系统，需要 NP 证书。
- 全文是理论证明，没有实验、实现、benchmark 或实证数据。
- 证明中有少数疑似排版笔误，例如 `Blinearity` lemma 的结论写成 `B(d1)+B(d2)=B(d2)`，按证明应为 `B(d3)`。
- 这些笔误不改变主线，但正式引用或复现证明时需要逐条校对。

### 关键技术细节
- 主约束集：`C(q)={x=y, x=(1+q)y, x+y=z, x+y=z+w, xy=z, xy=zw, null}`，变量域 `[1/2,2]`。
- assignment tester 参数：`ell=4`，`zeta=1/1000`，输出 alphabet `[1/2,2]`，输出 constraint set `C(q)_[1/2,2]^4`。
- Step 3 常数：`eta3=1/5000`。
- midpoint code 查询数：tester 4 queries，decoder 2 queries。
- midpoint code 参数：`k2=4 ceil(delta^-1) r`，`k1=32 ceil(delta^-1)^2 n r`。
- linear test soundness 条件：`epsilon <= 1/32` 时，至少 `1-4epsilon` 坐标与某个 image vector 一致。
- constraint test soundness 条件：`epsilon < 1/192` 时，至少 `1-12epsilon` 的 `u_sigma` 匹配某个 `alpha in V'`。
- constraint shrinking：单个 C(q) 约束转为 `R^74` 中约束，70 个辅助变量，至多 1 个 `x=q`、52 个 `x+y=z`、18 个 `(1+x)(1+y)=1+z`。
- general conversion：对 `P(C(q)^n)` 取 `m=1+22n` 个辅助变量。
- 最终 ETR-INV block size：每个 MAX-ETR-C(q) 约束对应 `1266+2k` 个 ETR-INV 约束。
- 误差传递：`UNSAT(psi)` 在 `UNSAT(phi)` 和 `UNSAT(phi)/(1266+2k)` 之间。
- solution-complexity 构造使用素数 `p` 和二进制 repeated squaring/product，变量数约 `2 ceil(log2 p)+3`。
