# EGG-SR: Embedding Symbolic Equivalence into Symbolic Regression via Equality Graph (2511.05849)

- **Year**: 2025.05
- **Authors**: Nan Jiang (UT El Paso), Ziyi Wang, Yexiang Xue (Purdue)
- **Venue**: arXiv preprint (提交时格式为 Google-style report，含 ICLR Ethics/Reproducibility Statement)
- **Keywords**: symbolic regression, equality graph, e-graph, symbolic equivalence, MCTS, deep reinforcement learning, LLM, rewrite rules
- **One-liner**: 用 e-graph 统一编码数学恒等变换，将符号等价性嵌入 MCTS/DRL/LLM 三类 SR 算法，理论上收紧 MCTS regret bound 并降低 DRL 梯度方差，实验上在三角函数和 Feynman benchmark 上一致提升表达式发现精度。

## 精华提炼

### 管线/方法

**核心思想**：符号回归中存在大量语法不同但语义等价的表达式（例如 $\log(x_1^2 x_2^3)$ 与 $2\log x_1 + 3\log x_2$），传统 SR 算法将它们视为独立候选，导致搜索空间冗余。EGG-SR 提出一个统一框架，通过 equality graph (e-graph) 数据结构紧凑表示等价表达式集合，并将等价性感知嵌入三类主流 SR 方法。

**具体管线**：

1. **EGG 模块（e-graph for grammar-based symbolic expressions）**
   - 输入：一条由 context-free grammar (CFG) 生成规则构成的序列，对应一个符号表达式
   - 用预定义的数学恒等式（交换律、对数恒等式、三角恒等式等 20+ 条双向 rewrite rules，见 Table 3）作为 rewrite rules
   - 对输入表达式构建 e-graph → equality saturation（最多 20 轮迭代）
   - 提取：cost-based extraction（最简化版本）+ random-walk sampling（批量采样等价变体）
   - 实现：纯 Python，扩展自 Willsey et al. (POPL 2021) 的 egg 框架，专门支持 grammar-based 表达式

2. **EGG-MCTS**
   - 基于 Sun et al. (ICLR 2023) 的 Symbolic Physics Learner
   - 核心改动在 backpropagation 步骤：对当前路径构建 e-graph → 采样等价序列 → 检查搜索树中是否存在对应路径 → 若存在则同步更新所有等价路径的 visits 和 reward
   - 类似于 transposition table，但用 e-graph 取代哈希表来检测符号等价性
   - UCT 公式中 visits(s) 变为等价类的累计访问次数

3. **EGG-DRL**
   - 基于 Petersen et al. (ICLR 2021) 的 Deep Symbolic Regression (DSR)
   - 核心改动在 policy gradient estimator：对每个采样序列 $\tau_i$ 构建 e-graph → 采样 K-1 个等价序列 → 将 $\nabla\log p_\theta(\tau)$ 替换为 $\nabla\log[\sum_k p_\theta(\tau_i^{(k)})]$（聚合等价序列概率）
   - sequential decoder 可用 LSTM (3 层, hidden 128) 或 decoder-only Transformer (6 头, hidden 128)

4. **EGG-LLM**
   - 基于 Shojaee et al. (ICLR 2025) 的 LLM-SR
   - 核心改动在 feedback prompt：将 LLM 生成的 Python 函数转为 lambda 表达式 → 解析为表达式树 → 构建 e-graph → 采样 K 个等价变体 → 转回 Python 函数 → 放入下一轮 prompt 的 feedback 中
   - 使用 GPT-3.5 和 Mistral 两种后端

### 关键技术细节

- **Grammar 表示**：CFG $\langle V, \Sigma, R, S\rangle$，非终结符 $V=\{A\}$，终结符含变量和系数，生产规则编码数学运算（如 $A\to A\times A$ 表示乘法），leftmost derivation
- **Rewrite rules**：包含交换律、分配律、对数恒等式（$\log(ab)\leadsto\log a+\log b$, $\log(a^b)\leadsto b\log a$）、指数恒等式、三角恒等式（和差化积、积化和差、二倍角、半角、Pythagorean 等 20+ 条），所有规则双向编码
- **e-graph 数据结构**：e-class（等价类，虚线框）+ e-node（操作节点，实线框），e-node 到 child e-class 有边。Union-Find 维护合并。hashcons 字典存储 enode→eclass 映射
- **Equality saturation**：最多 20 轮迭代。未引入可能导致 e-graph 指数增长的规则（如 $a+b-b\leadsto a-b+b$）
- **系数优化**：BFGS，batch size 1024，8 核并行，最多 20 个自由系数
- **训练数据规模**：训练集/验证集/测试集各 2048 样本
- **评估指标**：NMSE（归一化均方误差），$R^2 = 1-\text{NMSE}$
- **DRL 训练**：Adam, lr=0.009, entropy weight 0.03, gamma 0.7, batch 1024, 200 iterations, 最大序列长 20
- **MCTS**：UCT 探索常数 $\alpha$，rollout 评估

### 核心结果

1. **EGG-MCTS vs MCTS**（三角函数 benchmark，Table 1）：
   - Noiseless: (2,1,1) NMSE <1e-6 vs 0.006; (3,2,2) <1e-6 vs 0.033; (4,4,6) 0.006 vs 0.144; (5,5,5) 0.009 vs 0.147
   - Noisy: 类似趋势，除 (3,2,2) noisy 中 MCTS 略好（0.007 vs 0.012）
   - EGG-MCTS 维护更大更深的搜索树（Figure 3 left）

2. **EGG-DRL vs DRL**（Table 1）：
   - Noiseless 全面优于 DRL：(2,1,1) 0.020 vs 0.030; (3,2,2) 0.161 vs 0.277
   - Noisy: 多数优于 DRL，但 (4,4,6) noisy DRL 反而更好（2.46 vs 5.09）
   - 经验梯度方差显著降低（Figure 3 right）

3. **EGG-LLM vs LLM-SR**（4 个科学 benchmark，Table 2）：
   - GPT-3.5: OOD 上 Oscillation I 0.0004 vs 0.0005, Bacterial growth 0.0198 vs 0.0264, Stress-Strain 0.0419 vs 0.0516
   - Mistral: 同样多数数据集上改善

4. **空间效率**：e-graph 内存远小于 array-based 显式存储（Figure 4），对数和三角恒等式场景指数级节省

5. **时间效率**：EGG 模块计算开销极小，远低于系数拟合和神经网络更新（Figure 5）

### 明确局限

**作者自述局限**：
- 仅测试了 grammar-based SR 方法；SymNet 类（layer-wise symbolic networks）不直接兼容
- 未纳入 E2E-Transformer（Kamienny et al. 2022）的实验对比（仅讨论了如何集成）
- 某些 rewrite rules 存在域限制（如 $\log(ab)$ 要求 $a,b>0$），未系统处理域约束
- 省略了可能导致 e-graph 指数膨胀的规则（如 $a+b-b\leadsto a-b+b$）
- Diffusion-based SR（Bastiani et al. 2025）因实现未公开未对比

**读出来的局限**：
- **实验规模偏小**：主要在三角函数合成 benchmark 和 4 个 LLM-SR 科学问题上测试，缺少 SRBench 或 Nguyen benchmark 等标准大规模对比
- **noisy setting 表现不稳定**：DRL noisy (4,4,6) 中 EGG 反而变差，说明等价变换在噪声下可能引入不良变体
- **理论保证的实际约束**：Theorem 1 建立在 OPD（而非 UCT）框架上；Theorem 2 的方差减少依赖等价类大小，若 rewrite rules 覆盖率低则收益有限
- **LLM 实验缺乏消融**：未量化 feedback prompt 中等价表达式数量 K 的影响
- **与 GP-based e-graph 方法（de Franca & Kronberger 2023/2025）缺乏直接实验对比**，虽然在 related work 中讨论了差异
- **未测试真实物理发现场景**：Feynman 方程只用于 e-graph 可视化，未做 SR 实验

### 跟我们的关系

对 `llm-mcts-sr` 课题：

- **直接相关但不撞车**：本文将 e-graph 嵌入 MCTS/DRL/LLM 用于符号回归，我们的课题若涉及 LLM 或 MCTS 驱动的 SR，则 EGG-SR 是最近的 related work。但本文的核心贡献是利用已知数学恒等式构建 e-graph 来减少搜索冗余，而非改进 LLM/MCTS 本身的搜索策略或架构
- **可用的工具/方法**：
  - EGG 模块（纯 Python，代码开源在 github.com/jiangnanhugo/egg-sr）可直接复用，用于任何 grammar-based SR 的等价性检测

  - DRL 的 Rao-Blackwellization 梯度估计器是通用技巧
- **差异化方向**：
  - 本文的 rewrite rules 是手工编码的数学恒等式；若课题探索自动发现 rewrite rules 或学习等价关系则有差异化
  - 本文未真正将 LLM 和 MCTS 结合使用（EGG-MCTS 和 EGG-LLM 是独立的），若课题研究 LLM-guided MCTS for SR 则可以在 EGG 框架上叠加
  - 本文未涉及 PDE 发现、ODE 发现等更复杂的科学发现任务
- **应引用**：若做 MCTS/DRL/LLM for SR，本文是 2025 年的最新工作，必须引用和对比

## Read by: 0616-fex (2026-06-16)

从 FEX（RL for Symbolic Regression）视角下的分析：

**相关性**：本文高度相关。EGG-DRL 与 FEX 共享同一技术范式——用强化学习（policy gradient）从 grammar-based 搜索空间中发现符号表达式。但两者有不同的技术路线和核心贡献。

**撞车评估**：
- 不直接撞车。EGG-DRL 的核心贡献是在**已有 DRL 框架（DSR, Petersen et al. 2021）上叠加 e-graph 等价性感知的 Rao-Blackwellized 梯度估计器**，而非提出新的 SR 网络架构或搜索策略。
- FEX（若使用独特的网络架构/搜索目标）落在不同设计空间：EGG-DRL 使用 LSTM/Transformer 作为 sequential decoder，FEX 可能有自己的策略网络设计。
- 若 FEX 使用 policy gradient 式的 DRL + grammar-based 搜索，则 EGG 的 Rao-Blackwellization 技巧可直接嫁接，无冲突。
- 若 FEX 采用非 sequential decoder 架构（如递归网络、graph-based 编码等），则需要评估 EGG 模块的兼容性。

**可用工具/方法**：
1. **Rao-Blackwellized 梯度估计器**（Eq. 7）：这是本文最通用的贡献——将等价序列的概率聚合到 $\log[\sum_k p_\theta(\tau^{(k)})]$，通过条件方差分解降低梯度方差。这一技巧不依赖具体架构，可移植到任意 RL-based SR 方法（包括 FEX）。
2. **EGG 模块**：纯 Python 实现的 e-graph 数据结构，支持 grammar-based 表达式的等价类构建和随机采样。可直接用于 FEX 的等价性检测和变体生成。
3. **手工编码的 rewrite rules**（20+ 条数学恒等式，包括三角函数、对数、指数等）：可复用做等价性覆盖。
4. **EGG-MCTS 的等价类 backpropagation**：若 FEX 涉及 MCTS 搜索，该策略可直接使用。

**差异化方向**：
1. FEX 若使用自己的网络架构而非 LSTM/Transformer decoder，则 EGG-DRL 的实验设定未覆盖此情况——可以论文中声称其技巧"与架构无关"，但实验只验证了简单的 sequential decoder，这是差异化实验设计的空间。
2. EGG 使用 **手工预定义** 的 rewrite rules；若 FEX 探索自动学习/发现等价关系，则完全不同。
3. EGG 实验**未使用 SRBench、Nguyen 等标准 benchmark**，只用了三角函数合成数据和 4 个 SciBench 问题——FEX 若在这些 benchmark 上做更系统的对比，可提供更可信的证据。
4. EGG 未覆盖 PDE/ODE 发现、高维物理系统等复杂场景。
5. EGG-DRL 在 noisy (4,4,6) setting 中反而不如 baseline，说明等价变换在噪声下可能有害——这是值得探究的反例。

**应引用**：若 FEX 使用 grammar-based 表达式 + policy gradient，本文必须引用作为 related work。它代表了 2025 年在 SR 中引入等价性感知的最新进展。
