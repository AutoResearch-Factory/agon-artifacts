# H-FEX: A Symbolic Learning Method for Hamiltonian Systems (2506.20607)

- **Year**: 2025.06
- **Authors**: Jasen Lai (U Florida Math), Senwei Liang (LBNL), Chunmei Wang (U Florida Math)
- **Venue**: arXiv preprint (not yet peer-reviewed)
- **Keywords**: Finite Expression Method, Hamiltonian systems, symbolic regression, reinforcement learning, interaction nodes, energy conservation, symplectic integration, mixed optimization, risk-seeking policy gradient
- **One-liner**: Adapts FEX to Hamiltonian systems by enforcing Hamiltonian dynamics via integrators during RL-based symbolic search, and introduces interaction nodes to capture pairwise interaction terms.

## 精华提炼

### 管线/方法

完整 pipeline：观测轨迹数据（从真系统用 RK45 生成）→ 固定二叉树结构 → RL Controller（小 FC 网络，输入 zero vector，输出每节点的 PMF）→ 采样 M 条 operator sequence（ε-greedy, ε=0.2）→ 构建 H-FEX 树表达式 → 用 integrator（Leapfrog/RK2, K=20 substeps）从 surrogate Hamiltonian 生成预测轨迹 → 与观测轨迹算 MSE loss → Adam 优化权重（score phase: lr=0.1, 150步）→ Score = 1/(1+min_loss) → top-K 进 candidate pool → risk-seeking policy gradient（ν=0.25）更新 controller → 循环 100 次 → search 结束后 fine-tune candidate pool（lr=0.001, 300步）→ 选最高分 tree 输出为 closed-form Hamiltonian。

关键区别：(1) 训练时 integrator 每步从观测状态出发（不做自回归），避免误差累积；(2) 评估时从预测状态递归推进；两者均使用多步 integration（K substeps）降低数值误差。

### 关键技术细节

**H-FEX 树结构**：
- 树的节点类型：unary node（如 Id, (·)², (·)³, (·)⁴, exp, sin, (·)⁻¹）、binary node（+, ×, -, ÷）、interaction node（新增）
- 每个 unary/binary node 后有可训练 weight（标量乘子）
- p 和 q 坐标分开输入树的不同分支
- 非可分系统树：3 unary + 1 binary nodes，共 4 weights
- 三体问题树：3 unary + 1 binary + 1 interaction nodes，p 走 unary 分支（只做平方），q 走 interaction 分支

**Interaction Node（核心创新）**：
- 作用在位置坐标 q 的矩阵输入上（r 个 d 维向量 → r×d 矩阵）
- 对每对 (i,j) i≠j 施加 operator 映射到标量
- 本例 operator 集 I = {||qi - qj||², ||qi - qj||, ||qi ⊙ qj||², ||qi ⊙ qj||}
- 产生 pairwise 标量值后馈入后续 tree 结构
- 这是 FEX 家族首次引入 domain-specific node type

**Search Loop 参数**：
- 迭代数 E=100
- 每轮采样 B=10（三体）或 15（非可分）条 operator sequences
- Score 阶段 Adam lr=0.1, 150 steps
- Fine-tune 阶段 Adam lr=0.001, 300 steps
- Candidate pool size K=15
- Risk-seeking ν=0.25（关注 top 75% scores）
- ε-greedy ε=0.2
- Controller：小型 FC NN，输入 zero vector，输出每节点 PMF

**积分器**：
- 训练/评估内部用 K=20 uniform substeps per timestep
- 可分系统：Leapfrog（辛积分器，保能量好）
- 非可分系统：RK2（不依赖可分性假设）
- 数据生成：RK45（自适应步长，rel error < 1e-9）

**训练数据**：
- 非可分系统：120 条轨迹，t∈[0,3]，Δt=0.1（30 点），IC uniform [-1,1]²
- 三体问题：
  - Data1: t∈[0,3]，Δt=0.1（30 点）
  - Data2: t∈[0,7]，Δt=0.1（70 点），collision 多，更 stiff
  - IC：三体放在随机圆上（半径 uniform [0.9,1.2]），给垂直动量产生近似圆轨道，再乘 random factor [0.8,1.2]
- 测试数据：更长时间区间（非可分 t∈[0,60]，三体 t∈[0,30]），30 条

**Loss**：
- L(Θ) = E_{IC}[Σ_n ||p̂_n - p_n||² + ||q̂_n - q_n||²]
- 训练时用 empirical loss，p̂ 从观测状态出发而非预测状态

### 核心结果

**非可分系统（1D p, 1D q）**：
- H-FEX 精确恢复：exp(-1.0002588 p² - 1.1002197 q⁴) vs 真值 exp(-p² - 1.1q⁴)
- 权重学习：α₁≈1.000, α₂≈1.100（真值 1.0 和 1.1），误差 <0.1%
- SINDy 产出 6 次多项式近似，形式完全错误，无法捕捉 exp 结构
- H-FEX 在 [0,60] 上 MSE 保持 ≈10⁻¹⁰，能量相对变化 ≈10⁻¹⁰
- SINDy MSE 随 t 线性增长至 >10⁻¹，能量漂移迅速增大

**三体问题（12D system）**：
- H-FEX 从 30pts 训练数据恢复的 Hamilton 形式：Σ_i (0.5 p_i²) - Σ_{i<j} (1.0/||q_i - q_j||)
  - 动能项系数：0.5005, 0.4981, 0.4996, 0.4985, 0.4976, 0.5024（真值 0.5）
  - 势能项系数：-0.9985, -0.9970, -0.9949（真值 -1.0）
  - 总误差 <1%
- 30pts 训练：H-FEX MSE 比 HNN/SRNN 低约一个数量级，能量漂移明显更小
- 70pts 训练（含碰撞 stiff 区）：H-FEX 仍然最优，比 SANN 也能保持更低 MSE 和能量漂移
- 关键：H-FEX 在 70pts（更多碰撞）下的表现与 30pts 相近，说明 interaction node 稳健

**Surprising 发现**：
- H-FEX 在 3-body 中即使训练数据有限（30 时间点，t∈[0,3]），也能在 t=30 处保持精度，远超训练区间 10 倍
- SINDy 完全无法处理非多项式非线性（exp 函数），而 FEX tree 轻松处理
- H-FEX 对 stiff 数据（碰撞多发）天然鲁棒，不需要 SANN 的 stiffness-aware 机制

### 明确局限

**作者自述**：
1. 依赖 operator dictionary 选择；缺少关键 operator 则精度受损
2. 需要预定义树结构（节点数、类型、weight 位置）
3. 无先验知识时需试错多种 operator 和结构组合
4. 未来方向：自适应调整节点和 weight 位置的动态方法

**我读出的额外局限**：
5. 仅在两个 toy problems 上验证（非可分 2D + 三体 12D），物理系统覆盖面极窄
6. 未与最直接竞争者 DiPietro & Zhu (2022) SISR（symplectic integrated symbolic regression）对比——SISR 也是 Hamiltonian 专用的 symbolic regression，也用了 symplectic integration
7. 未测试噪声鲁棒性——所有数据无噪声生成，实际观测数据必有噪声
8. 树结构是人工设计的（非可分 3U+1B，三体 3U+1B+1I），这本身包含了对系统的先验知识
9. RL controller 输入是 zero vector——无状态，等价于学习一个固定的 multinomial 分布，而非 adaptive policy
10. 无实机数据或真实物理系统验证（纯仿真）
11. dimensionality scalability 未探讨（最大 12D），高维 Hamiltonian 系统（如分子动力学 10³-10⁶ D）不可行
12. search 过程的计算成本未讨论——每次 score 计算需要跑 K=20 substeps integration 多次

### 跟我们的关系

对 **0616-fex** 课题：

- **不撞车**：H-FEX 是 FEX 的一个应用 specialization（Hamiltonian 系统），不是对 FEX 核心方法论的改造。我们的课题如果是改进 FEX 的搜索机制、表达能力、scalability 等核心问题，H-FEX 不构成竞争。
- **可用的工具/方法**：
  1. **Interaction Node 设计模式**：FEX 树中引入 domain-specific node type 的思路可以推广——H-FEX 证明了这种扩展有效。我们可以在其他 domain（如 PDE、网络动力学）引入类似的专用 node。
  2. **integrator-in-the-training-loop**：将物理 simulator（integrator）嵌入 loss 计算的范式，可用于其他物理系统 learning。
  3. **训练/评估分离的 integration 策略**：训练时从观测状态出发（teacher forcing），评估时自回归——这个技巧可推广到其他 sequential prediction 场景。
  4. **Risk-seeking policy gradient + candidate pool + fine-tune** 的三阶段 pipeline 是 FEX 家族的标准做法，可复用。
- **该引**：是 FEX 在物理系统应用的最新工作（2025.06），在 FEX 综述/related work 中应引。
- **差异化**：H-FEX 特定于 Hamiltonian 系统（需要知道系统是 Hamiltonian 并可使用 symplectic integrator）。我们的 FEX 工作如果是 general-purpose 或面向不同物理 paradigm，则定位不同。

## Read by: 0616-fex (2026-06-17)

H-FEX 是 FEX 家族在 Hamiltonian 系统上的直接应用 specialization。它对 0616-fex 课题的核心价值在于：

1. **Domain-specific node 设计范式**：Interaction node 证明了在 FEX tree 中引入物理学先验（pairwise interaction）的有效性。这为 FEX 向其他领域的扩展提供了设计模板。

2. **不构成竞争**：H-FEX 修改的是 FEX 的"应用层"（在搜索 loop 中嵌入 integrator、加入 interaction node），不涉及 FEX 核心搜索机制（RL controller、risk-seeking gradient、tree structure optimization）的改进。我们的课题如果聚焦于 FEX 方法论本身（如更高效的搜索策略、自动 tree 结构学习、scalability 改进），H-FEX 是 parallel work 而非 competing work。

3. **可借鉴的设计元素**：训练/评估分离的 integration 策略（teacher forcing during training, autoregressive during eval）、multi-step integrator 嵌在 loss 计算中的方案，都可以抽象为 FEX 与物理 simulator 交互的通用范式。

4. **局限性对我们是机会**：H-FEX 的树结构仍需人工设计、operator dictionaries 需要先验知识——这正是 FEX 方法论层面需要解决的核心问题。如果我们能做出自动 tree 结构搜索/自适应 operator 选择，直接解决 H-FEX 作者自己指出的 open problem。

5. **引用策略**：在 FEX 应用 survey 或 related work 中引用，展示 FEX 在物理系统 learning 中的 versatility。在 method 部分如果涉及 domain-specific adaptation，可引用 interaction node 设计作为先例。
