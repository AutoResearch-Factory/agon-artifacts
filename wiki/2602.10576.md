## 精华提炼

### 管线/方法

PiT-PO 的整体管线是一个封闭的进化循环，在搜索过程中用强化学习逐步微调 LLM。具体步骻：

1. **多岛进化探索（Island-Based Exploration）**：N=4 个独立"岛"各自维护一个经验缓冲 B_j。每个岛基于当前数据 D 和历史最佳方程构建 prompt，LLM 采样 G=4 个候选方程（sketch），由外部数值优化器（BFGS）拟合系数。每个方程会通过双约束评估得到全局奖励 R_global 和 token 级惩罚 {P_{i,k}}。

2. **双约束评估（Dual-Constraint Evaluation）**：
   - **层级物理约束**：通用层（量纲一致性 P_dim、可微性 P_diff）+ 领域特定层（如湍流建模的可实现性/壁面边界条件/渐近标度/能量一致性）。物理惩罚被门控（gated）：只有当候选方程的 MSE 低于阈值 delta_gate 时才激活，避免早期过度约束。
   - **理论约束（Support Exclusion Theorem）**：根据系数幅度比 tau_i 用定理判定冗余项，对冗余 token 施加 P_tok = p * max(0, -log(|b_i|+eps)) 惩罚。

3. **Token-Aware 策略更新（In-Search LLM Evolution）**：全局奖励 R_global = R_fit - P_cplx - P_phy 标准化后经 GRPO 做组内 advantage 估计，再减去 token 级惩罚 P_{i,k} 得到 token-aware advantage。用 LoRA (r=16) 进行参数高效微调，lr=1e-6，KL beta=0.01。

4. **层级选择（Population Management）**：保留每个岛缓冲中的 top 候选，周期性地用高适应度种子重置表现差的岛。

### 关键技术细节

- **LLM 骨干**：Llama-3.2-1B-Instruct, Llama-3.2-3B-Instruct, Llama-3.1-8B-Instruct（全部 4-bit 量化）
- **RL 算法**：GRPO (Group Relative Policy Optimization)，组大小 G=4，clip epsilon 按标准 GRPO 配置
- **LoRA**：rank=16，仅训练 adapter 权重
- **超参数**：学习率 1e-6，KL 正则化系数 0.01，搜索迭代 2500 次
- **总 batch size**：G x N = 4 x 4 = 16（每个设备）
- **硬件**：单张 NVIDIA RTX 3090
- **物理约束门控阈值**：delta_gate = 1e-3 * MSE_initial
- **冗余检测阈值**：rho = 0.01，即标准化系数比 tau_i <= 0.01 的项被视为冗余
- **token 惩罚尺度**：p = 0.5
- **复杂度惩罚系数**：lambda_len = 5e-3（AST 节点数）
- **拟合奖励**：R_fit = -alpha * log(MSE + epsilon)，alpha=1.0
- **湍流建模**：用 Pope 张量基展开，LLM 预测 3 个标量系数函数 G^1, G^2, G^3（仅含两个变量 I_1, I_2），然后通过固定张量基重建 Reynolds 应力。评估时用 k 缩放后计算 MSE（仅用 4 个张量分量 (0,0),(0,1),(1,1),(2,2)）。
- **多保真度约束调度**：Stage A 仅用 realizability 约束（廉价），Stage B 在出现新 best-MSE 时触发完整湍流约束套件（需要 CFD 后处理）。
- **湍流 CFD 验证**：在 OpenFOAM 中用 simpleFoam 求解 RANS，嵌入学到的符号闭合项。
- **数据源**：NASA Turbulence Modeling Resource 的 parameterized periodic hills DNS 数据。

**关键发现：**
- PiT-PO 是唯一能完全正确恢复 Oscillator 1 真实方程的方法（NMSE 低至 6.41e-31，堪比浮点精度极限）
- PiT-PO 的 1B 模型（99.95% Acc_avg）已超过 LLM-SR 的 Mixtral 8x7B（100.00% Acc，但 NMSE 高了 20 个数量级）
- 在 25,000 秒固定壁钟预算下，PiT-PO 一致优于 LLM-SR
- 消融实验：去掉物理约束或 token 正则化都显著降低 NMSE 并扩大 ID-OOD 泛化差距

**湍流建模验证：**
- 在 periodic hills (Re_b=5600) 上，PiT-PO 学到的非线性本构关系使 Reynolds 应力各向异性分量更接近 DNS
- 改善了分离泡大小预测，壁面摩擦系数 Cf 分布更贴合 DNS
- 优于标准 k-omega SST RANS 模型和 DSRRANS

### 明确局限

1. **计算开销**：虽然性能更强，但 in-search 微调确实引入了额外计算成本。作者承认了这一点但认为加速收敛可弥补开销。
2. **仅在合成/仿真数据上验证**：OOD 测试虽然是合成外推（改变动力学参数），但没有真正的 real-world 实验数据验证。
3. **湍流算例有限**：仅在一个 periodic hills 几何（单一 Reynolds 数）上验证，缺少泛化到不同几何、不同 Re 数的评估。
4. **物理约束的工程依赖**：部分湍流约束需要 CFD 后处理（wall distance 等），使用两阶段调度部分缓解但增加了系统复杂度。
5. **Support Exclusion Theorem 的假设限制**：定理假设真实函数支持集有限 (|S'|<=M) 且系数有界 (A<=|a_j|<=B)。实际中 M、A、B 是输入超参，设定不合理可能使判定失效。
6. **没有在真正的 out-of-distribution 物理场景验证**（如新几何、新流态）。
7. **依赖 LLM 的预训练科学先验知识**：如果方程完全超出 LLM 训练数据分布，初始采样质量可能很低。

从 FEX 课题角度看 PiT-PO：

PiT-PO 的核心贡献是将 LLM 从 frozen 的方程提议者进化为 adaptive 生成器，通过 in-search RL 微调持续内化搜索信号。这个思路与 FEX 中"用 LLM 搜索/构造特征"的场景有潜在的交集——如果 FEX 涉及迭代搜索，可以考虑类似的 RL-based 参数内化机制。

具体可迁移的技术点：
1. **Support Exclusion Theorem** 是一个独立的理论工具，可用于任何需要判定表达式中冗余项的场景。
2. **Gated physical constraints** 的 curriculum scheduling 设计模式可迁移到 FEX 的复杂度控制。
3. **Token-aware advantage estimation** 的思路对 FEX 中处理"部分正确的特征结构"有启发意义。

不撞车：FEX 课题如果聚焦于通用特征构造/工程，而非物理方程发现，则与 PiT-PO 的目标域不同。PiT-PO 在主代码需求（symbolic regression equation discovery）上完成了较完备的工作。
