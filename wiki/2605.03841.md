# Complex Equation Learner: Rational Symbolic Regression with Gradient Descent in Complex Domain (2605.03841)

- **Year**: 2026.05
- **Authors**: Sergei Garmaev, Maurice Gauche, Olga Fink; EPFL Intelligent Maintenance and Operations Systems Laboratory / Mathematics Section
- **Venue**: arXiv
- **Keywords**: symbolic regression, rational symbolic regression, Equation Learner, complex-valued optimization, poles, division operator, frequency response function, structural damage
- **One-liner**: 用复权重EQL稳定学习含极点的有理符号模型。

## 精华提炼

### 管线/方法

1. 论文要解决的是 EQL 类梯度符号回归在 division、log、sqrt 等算子下训练不稳定的问题。
2. 核心诊断不是“网络表达不了有理式”，而是实数域优化会在分母零点附近出现梯度符号相消。
3. 作者用简单例子说明：拟合 `f(x)=1/x`，模型 `1/(x+a)`，当 `a` 是实数时，每个样本的梯度项随 `x` 和 `x+a` 的符号变化。
4. 如果样本覆盖零点两侧，不同区间的样本给出相反梯度，总梯度可能接近 0 或震荡。
5. 论文把这个问题推广到含分母的更复杂表达式，认为这是实数参数空间的结构性病态。
6. 第二类问题是 log/sqrt 的实数域定义域限制：中间激活在训练中可能变成非正数，导致 forward 无定义。
7. 现有“安全”替代如 `log(abs(x))` 或 `sqrt(abs(x))` 能避免崩溃，但改变了可表达的目标函数族。
8. CEQL 的主线是把 EQL 的内部权重扩展到复数域，同时把最终输出投影回实数域来计算损失。
9. 输入和真实标签仍然是实数；复数只用于内部权重、激活和优化路径。
10. 直觉是：复数权重给分母一个虚部自由度，优化轨迹可以绕开实轴上的退化点。
11. 对 division，作者定义了专门的 surrogate：`b_div(x,y)=Re(Re(x)/y)`。
12. 这个设计让 numerator 的有效输出保持实数，同时让 denominator 保留复数自由度。
13. 对其他 unary operator，CEQL 用 `u_i*(x)=u_i(Re(x))+0i`，即先取实部再套原实值算子。
14. 对其他 binary operator，CEQL 用 `b_j*(x,y)=b_j(Re(x),Re(y))+0i`。
15. 对 log 和 sqrt，作者使用复数 log/sqrt 的主分支，并通过 imaginary-part penalty 把中间值拉近主分支。
16. CEQL 网络仍是 EQL 风格的有向无环图：每层先做复权重线性组合，再送入固定 operator library。
17. 每层有 `m` 个 unary summation nodes 和 `2n` 个 binary summation nodes；binary nodes 成对输入给二元算子。
18. 模型输出是最后一层 activations 的加权和，训练时取实部和目标值比较。
19. 相比原始 EQL，CEQL 加了 skip connections，把原始输入拼接到更深层。
20. skip connections 的用途是让浅层目标表达式不必被迫走深计算路径，也改善梯度传播。
21. 训练分三阶段：先弱正则训练，再剪枝和继续稀疏化，最后在固定结构上 fine-tune。
22. Phase 1：优化误差 + 极弱 L1 + 极弱 imaginary penalty；结束后按阈值剪掉小权重，并 cascade cleanup。
23. Phase 2：在剪枝后模型上继续训练，加强 sparsity，并周期性删掉 batch loss impact 最小的连接。
24. Phase 3：关闭 sparsity regularization，只保留误差项和 imaginary-weight penalty 做精调。
25. 最终表达式由剩余 active edges 诱导的 expression tree 抽取。
26. 评价不依赖 exact symbolic match，因为代数等价判定很脆；主指标是 interpolation MSE、extrapolation MSE 和 node count。
27. 作者强调 extrapolation MSE 更接近“是否找到了真实函数形式”，因为测试点在训练域外。

### 关键技术细节

1. synthetic benchmark 共 10 个表达式，编号 E-1 到 E-10。
2. E-1 到 E-4 是线性/多项式；E-5 是 log；E-6 是 sqrt；E-7 到 E-10 是含 1 个或 2 个 poles 的 rational functions。
3. 系数从 signed uniform 采样：符号 `s in {-1,+1}`，幅值 `u ~ U(0.5,3.0)`，保留两位小数。
4. 每个表达式生成训练集、interpolation test、extrapolation test 三套数据。
5. 训练和插值域是 `[-2,2]^d`。
6. 外推域是 `([-4,-2] union [2,4])^d`。
7. 每个表达式训练样本 128 个，插值和外推测试各 8192 个。
8. 产生 non-finite、undefined 或绝对值超过 `10^2` 的样本被丢弃。
9. CEQL benchmark 采用 2 个 symbolic layers。
10. CEQL benchmark operator library 每层包含 `const x2`、`(.)^2 x2`、`const x2`、`mul x2`、`id`、`log(.) x2`、`sqrt(.) x2`、`div x2`。
11. benchmark 优化器是 Adam，初始学习率 `1e-2`。
12. Phase 3 使用 ReduceLROnPlateau，patience 为 2000，factor 为 0.1，最小学习率 `1e-5`。
13. 收敛阈值是 `1e-7`。
14. benchmark 表中 CEQL 的 loss 写为 PR-MSE；方法正文用 MSE 描述主误差项。
15. Phase 1/2/3 epoch 数分别是 `100000 / 200000 / 50000`。
16. imaginary-weight penalty 系数分别是 `1e-10 / 1e-3 / 1e3`。
17. L1 weight penalty 系数分别是 `1e-10 / 1e-7 / 1e-7`。
18. log/sqrt argument penalty 系数分别是 `1e-10 / 1e3 / 1e3`。
19. 剪枝只在 Phase 2 开启，间隔 10000 epochs，每次剪枝比例 0.1。
20. synthetic benchmark 的最少剩余 edges 是 15。
21. PySR baseline 使用 1000 iterations，population size 27，并行 populations 200，max sequence size 20。
22. PySR unary operators 是 `log`、`sqrt(.)`、`.^2`；binary operators 是 `+`、`-`、`*`、`div`。
23. EQL-div baseline 使用 SGD，lr `1e-2`，10000 epochs，batch size 64，2 symbolic layers。
24. EQL-div 只有 `id` unary、`mul` binary 和 output `div`，L1 正则 `1e-3`，division regularization parameter `k=50`。
25. SINDy baseline 使用 polynomial degree 2，STLSQ，threshold `1e-2`，ridge alpha `1e-6`，library 包含 log、sqrt、div 等稳定化版本。
26. FRF 实验使用一根带六个可拆质量块的悬臂钢梁振动数据。
27. 原始频率范围是 0 到 2000 Hz；输入频率在模型中用 `omega`，单位 kHz。
28. 结构条件包括 healthy 和三种逐步 damaged 状态；damage 用移除质量百分比 `d` 表示。
29. FRF 模型输入是 `(omega,d)`，输出是线性尺度的 inertance magnitude。
30. FRF 手工结构是 broadband linear trend 加 20 个 resonant terms：`a omega + b + sum_i A_i / ((omega - h_i(d))^2 + gamma_i)`。
31. 每个 `h_i(d)` 由 CEQL 小网络学习，输入只有 damage variable `d`。
32. FRF 的 `h_i` 网络也是 2 个 symbolic layers，operator library 为 `id`、`const`、`square`、`mul`。
33. FRF 实验初始 resonant components 数 `n=20`，最少剩余 edges 是 50。

### 核心结果

1. 在简单线性/多项式 E-1 到 E-4 上，SINDy 和 PySR 都很强，CEQL 也能恢复低误差表达式。
2. CEQL 在 E-1 的 interpolation/extrapolation MSE 分别约 `1.0e-12 / 3.6e-12`，node count 5。
3. CEQL 在 E-2 的 interpolation/extrapolation MSE 分别约 `2.7e-12 / 7.5e-12`，node count 8。
4. CEQL 在 E-3 的 interpolation/extrapolation MSE 分别约 `6.8e-11 / 2.0e-9`，node count 10。
5. CEQL 在 E-4 的 interpolation/extrapolation MSE 分别约 `9.0e-11 / 2.4e-9`，node count 22。
6. 对 log 表达式 E-5，CEQL 达到 `5.9e-9 / 3.4e-11`，明显优于 SINDy 和 EQL-div。
7. 对 sqrt 表达式 E-6，CEQL 达到 `3.8e-7 / 2.1e-9`；PySR 在该项更低，但 CEQL 避免了实域 undefined 训练问题。
8. 对 rational E-7，CEQL 达到 `8.6e-8 / 1.9e-9`；EQL-div 是 `1.3e3 / 1.8e3`，直接失败。
9. 对 two-variable rational E-8，CEQL 达到 `9.3e-8 / 1.5e-9`；SINDy、EQL-div 都在几十到上万量级误差。
10. 对 quadratic-denominator rational E-9，CEQL 达到 `4.8e-8 / 9.7e-7`。
11. 对 two-pole rational E-10，CEQL 达到 `2.7e-6 / 4.2e-11`。
12. PySR 的外推误差在若干表达式上比 CEQL 更低，但 CEQL 的贡献重点不是绝对击败 PySR，而是让 EQL 型梯度架构能学含 poles/domain constraints 的表达式。
13. CEQL 在 E-1 到 E-4 和 E-6 的 node count 方差为 0，说明这些较简单结构下收敛结构稳定。
14. 对复杂 rational 表达式，CEQL node count 方差变大，作者解释为低影响残余项没有完全被剪掉。
15. appendix 给出的 CEQL 恢复表达式通常与目标式在数值上接近，但并不总是最简代数形式。
16. FRF 实验中，模型捕捉到 broadband trend 和两个主共振峰，位置约在 0.75 kHz 和 1.18 kHz 附近。
17. FRF 预测的 peak shape 更平滑、峰值略低，因为模型只让 resonance location `h_i(d)` 随 damage 变，`A_i` 和 `gamma_i` 固定。
18. peak frequency 对比图显示两处主峰的预测频率跟 measured mean frequency 大体贴近 identity line。
19. FRF appendix 的最终表达式保留多个 rational terms，分母里有关于 `d` 的二次、三次、四次 shift。
20. 论文最有用的实证发现是：含 poles 的 rational symbolic regression 对 EQL 来说主要是优化几何问题，不只是 operator library 是否包含 division。

### 明确局限

1. 没有看到公开代码或 artifact 链接；复现需要重建 CEQL、剪枝和表达式抽取细节。
2. 虽然 appendix 给了大量超参，但 exact pruning implementation、cascade cleanup 和 expression extraction 仍可能影响结果。
3. 论文不主张 trigonometric functions 已解决；作者明确说 sin/cos 在宽域上仍会因为周期性符号变化导致梯度相消。
4. sparsity 机制仍脆弱；过早或过强剪枝会删掉正确分支，过弱剪枝会留下残余项。
5. 对复杂 rational expressions，CEQL 的 recovered expressions 有时不够简洁，需要后处理或人工代数检查。
6. 评价主要是 synthetic expressions 和一个结构振动 FRF case，覆盖面还不等于标准 SR benchmark 全景。
7. synthetic 表达式的 operator set 与模型 library 高度相关，不能证明对未知 operator family 的发现能力。
8. 论文用 extrapolation MSE 作为 symbolic correctness proxy，这比 interpolation 更好，但仍不是严格代数等价证明。
9. FRF 实验是半结构化建模：resonant rational form 是人工指定的，不是完全自由发现。
10. FRF 模型固定 `A_i` 和 `gamma_i` 不随 damage 变化，因此 peak amplitude 和 width 的解释力有限。
11. 复数 log/sqrt 仍有 branch cut 问题；作者用 penalty 缓解，但没有给出全局收敛保证。
12. 复权重带来的收益和成本没有系统 runtime/compute 对比，尤其相对 PySR 的 wall-clock 代价不清楚。
13. 反引文目前只有一篇同组 2026 后续工作，独立第三方采用度还无法判断。
14. 论文中有一些表述和拼写小问题，例如 abstract 里的 `Validate` 大写，不影响主结论但提示稿件仍是 preprint 状态。
15. 如果我们的课题需要严格可审计的“发现新物理规律”，这篇只能提供优化机制和工程证据，不能单独支撑强因果发现。

## Read by: 0616-fex (2026-06-19)

从 `0616-fex` 视角，这篇最重要的是 claim 边界：它已经明确占据了“复数域权重 + EQL 架构 + rational/pole symbolic regression”的组合。若我们的课题计划用类似复权重绕开分母零点梯度病态，必须把它作为 closest prior，而不能把复数优化本身当主要 novelty。

可复用的部分很具体：division surrogate `Re(Re(x)/y)`、log/sqrt 的复数主分支处理、三阶段训练剪枝、以外推误差替代 exact symbolic match、以及 FRF 中 `linear trend + rational resonators` 的结构化表达式。尤其是 FRF/结构损伤方向，本文已经证明这种 rational denominator 可以把 resonance shift 写成显式的 `h_i(d)`，这对做可解释频响、FEA surrogate 或损伤参数识别都可直接借鉴。

差异化建议是避开“又一个 CEQL”叙事，转向作者没有解决好的部分：周期算子学习、自动简化/等价验证、公开可复现实验、condition-dependent damping/amplitude、以及独立 verifier 对发现表达式做 veto。如果 `0616-fex` 要强调真实实验或 FE 数据闭环，最好把 CEQL 作为 pole/expression extraction 模块之一，再用物理约束和跨工况验证形成增量。
