# PeakWeather: MeteoSwiss Weather Station Measurements for Spatiotemporal Deep Learning (2506.13652)

- **Year**: 2025.06
- **Authors**: Daniele Zambon*, Michele Cattaneo* (equal contrib.), Ivan Marisca, Jonas Bhend, Daniele Nerini, Cesare Alippi — USI IDSIA (Lugano) + MeteoSwiss + ETH/EPFL SDSC + PoliMi
- **Venue**: arXiv preprint（结构完全按 NeurIPS Datasets & Benchmarks 模版：dataset + library + baselines）
- **Keywords**: weather station observations, spatiotemporal benchmark, graph neural networks, graph structure learning, wind forecasting, NWP ensemble baseline (ICON-CH1-EPS), topographic descriptors, direct observation prediction (DOP)
- **One-liner**: 发布瑞士 302 站 10 分钟分辨率 8+ 年地面观测 + 地形描述子 + 业务 NWP 集合预报的时空 DL 基准，并用风预报实验证明 DL 在 ~12h 内可胜过 NWP。

## 精华提炼

### 管线/方法
论文是数据集/基准论文，管线分数据产品和示范实验两部分。

**数据产品管线**：
1. 观测源：SwissMetNet（MeteoSwiss 业务观测网，WMO 标准，自动+人工 QC）→ 302 站（160 气象站 + 142 雨量站），8 变量，10 分钟分辨率，2017-01-01 至 2025-10-13。
2. 静态特征：swisstopo DHM25 数字高程模型（50m 分辨率，裸地面）→ 用 MeteoSwiss 开源库 topo-descriptors（github.com/MeteoSwiss/topo-descriptors）在 2km 与 10km 两个邻域尺度计算 TPI、高程 STD、aspect、slope、西东/南北梯度。
3. NWP baseline：ICON-CH1-EPS（MeteoSwiss 2024-05 起业务运行，~1km 网格，每 3h 起报，11 成员集合 = 1 控制 + 10 扰动，33h lead time）→ 取站点最近格点（<1km），逐小时；温度用 0.06 K/m 恒定 lapse rate 做高度订正，相对湿度用 Bolton (1980) Eq.10 从温度/露点导出。起自 2024-05-14。
4. 封装：数据托管 Hugging Face（CC-BY-4.0），配 framework-agnostic Python 库（BSD-3-Clause）做加载、对齐、重采样（含变量特定聚合策略）、u/v 风分量提取、缺失 mask 生成，输出 tensor-ready 格式。

**示范实验管线**（风预报）：10min → 1h 重采样 → 160 气象站 → 目标为风速/风向合成的水平速度向量 (u,v)∈R²，24h horizon，输入窗 24 步；训练 2017–2024.09，验证 2023.10–2024.09，测试最后一年；优化 Energy Score（CRPS 的多变量推广），Monte Carlo 估计（训练 M=16、测试 M=100 样本）。

### 关键技术细节
- 8 变量（SwissMetNet 短名）：温度 tre200s0、湿度 ure200s0、降水 rre150z0（10min 累计）、日照 sre000z0、气压 prestas0、风速 fkl010z0（10min 均值）、阵风 fkl010z1（1s 峰值 10min 最大）、风向 dkl010z0（10min 圆均值）。
- 缺失：总体 ~3%；日照缺失最高 13.02%，降水 3.51%；NaN 表示 + 二值可用性 mask；含传感器搬迁 metadata（95% 位移 <110m，平均 97m）。
- 站网：160 核心气象站 5 近邻平均距离 18km（全部站 12km）。
- 模型族：RNN（2 层 GRU，hidden 128）、TCN（4 层，kernel 3，dilation 2，128 hidden）；MP-RNN/MP-TCN 在时序处理后叠 6 层 diffusion-convolution 消息传递（time-then-space）；MP-TCN-gsl/MP-TCN+ 按 Cini et al. sparse GSL（score-function 梯度估计学 top-k=10 图分布，逐 batch 采样、推理取众数）。文献基线：DCRNN、AGCRN、GraphWaveNet、Chronos-2（120M 参数基础模型，零微调，60 天上下文，独立逐站预测，输出分位数）。
- 站点特异组件：可学习站点嵌入 e^i∈R³²（条件化 encoder/decoder，按 Cini et al. "Taming local effects"）+ 地形描述子 v^i。
- 输入图：站点水平欧氏距离 Gaussian kernel（带宽=距离 std），top-k=8 且相似度 >0.7。
- 概率输出：逐 lead time/站/时刻的（二元）高斯，重参数化训练；持久性基线 PM-st（重复最后观测）与 PM-day（重复 24h 前）。
- 训练：Adam，lr 0.001（epoch 40/60/120 ×0.3），200 epochs × 300 batch × bs 32，早停 40；单模型训练+评测 <3h（NVIDIA L4）。
- 评测：ES + MAE Velocity/Speed/Direction（方向为角距离，风速 <1 m/s 样本剔除）；双测试集（与 ICON 3h 起报对齐的 NWP test set / 全窗口 full test set），结果一致。

### 核心结果
- 站点特异信息作用（TCN 族，MAE Vel）：纯时序 1.112 → 加 topo 或 embedding 1.076；时空模型 1.024 → 全组件 1.010；GSL 替换/增强输入图 → 0.999–1.001。即 **topo 特征与 embedding 对纯时序模型提升大（~3%），对图模型提升小**（消息传递本身已提供局地化）；GSL 一致有效。
- 排行（NWP test set，MAE Vel / ES）：GraphWaveNet 0.994/1.116 ≈ MP-TCN+ 0.999/1.125 > AGCRN 1.038 > DCRNN 1.049 > MP-RNN 1.056 > TCN 1.075 > RNN 1.087 > Chronos-2 1.108/– > **ICON 1.132/1.407** > PM-st 1.519 > PM-day 1.589。
- **DL vs NWP 分水岭**：MP-TCN 系在 nowcasting 至 ~12h 内胜过 ICON；>12h 尤其风向 ICON 反超——物理模型在较长时程/大尺度动力上仍占优。ICON 短时不提升（格点-站点空间失配 + 局地信息同化困难）。ES 指标下 DL 相对更强（ICON 集合仅 11 成员、局地代表性差）。
- 反直觉/值得注意：基础模型 Chronos-2 零微调下不敌图模型甚至不敌 ICON（MAE Speed 1.202 全场最差之一），说明站点级复杂地形风预报对通用时序基础模型仍难。
- 归纳温度实验（20 气象站 + 20 雨量站 held-out）：训练模型迁移到未见站性能降 ~10%，DCRNN unseen MAE 1.633 vs ICON 1.453——**归纳设定下 DL 不敌 NWP（MAE），但 ES 下 DL 仍占优**；且对 held-out 站选择敏感。

### 明确局限
- 作者自述：站点/传感器搬迁造成不一致（已给 metadata）；QC 后仍可能有伪迹（如冻结风速计）；雨量站温度传感器质量较低（只为区分固/液态降水）；日照缺失 13%；ICON 聚合量（降水/日照/阵风）无 lead time 0；归纳实验对 held-out 站敏感。
- 读出来的局限：**无雷达、无卫星、无事件/灾害标签**——纯站点时序 + 静态地形 + NWP，不覆盖对流尺度现象；仅瑞士（气候单一、面积小、302 站）；NWP baseline 只有 2024-05 之后 ~1.5 年，与 8 年观测严重不对齐（限制混合建模）；实验只做了风/温度的常规预报，没碰极端事件评测（如大风警戒命中率）；无标准 canonical splits 之外的任务协议（imputation/virtual sensing 只是提及 + 一个附录实验）；未做统计显著性检验（只报 5 seed 均值±std）。
- 与 benchmark 定位相关：论文自称"首个开放组合长期高频站点数据 + 地形上下文 + 复杂地形 NWP baseline"——这个 claim 较稳，但"支持 DOP"并非独有（WeatherReal、Weather2k、StationBench 同赛道）。

### 跟我们的关系
对 0713-scs-benchmark（强对流环境–雷达多模态事件包基准）：
- **不撞车**：PeakWeather 是通用地面站时序基准（瑞士、常规变量），无雷达、无事件标签、无"环境场时空演变"概念，更无 storm-centric 封装。我们的核心贡献（NOAA Storm Events 标签 + HRRR 长时窗环境场 + MRMS 雷达的事件包）与其空白正交。它 related work 必引，且正好用来论证"现有站点/观测 benchmark 不把环境场当一等公民"。
- **可直接借鉴的做法**：(1) HuggingFace 托管 + framework-agnostic Python 库 + tensor-ready 加载/对齐/重采样 utilities 的交付形态，正对应我们的三层交付物；(2) 二值可用性 mask + NaN 表示 + 缺失统计表（我们的质量标记层）；(3) 传感器搬迁 metadata 的 provenance 披露（ESSD 式整编严谨性范例）；(4) 静态地形描述子多尺度计算（topo-descriptors 库可直接用于我们的静态特征层）；(5) 业务 NWP 集合作为 physics baseline 的对齐方式（最近格点 + 高度订正 + lead time 对齐——我们对 HRRR f00/f01 的处理可参照其写法）；(6) 任务谱定义（forecasting/imputation/virtual sensing/GSL + inductive setting + 双测试集设计）可移植为我们的 canonical tasks；(7) Energy Score/CRPS 概率评测协议。
- **差异化表述**：地域（瑞士 vs CONUS）、粒度（station 时序 vs 事件聚类区域格点多模态）、目标现象（常规风/温 vs 强对流全生命周期）、结构（station-centric vs storm/event-centric）、环境场（无长时窗环境演变 vs ±15/5 天环境窗）。
- **引用建议**：必引，放"现有观测类基准"段，与 WeatherReal/Weather2k/StationBench 并列，指出全部缺失事件标签与对流尺度模态。
