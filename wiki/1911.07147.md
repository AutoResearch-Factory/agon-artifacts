# Causality-based Feature Selection: Methods and Evaluations (1911.07147)

- **Year**: 2019.11
- **Authors**: Kui Yu, Xianjie Guo (Hefei University of Technology); Lin Liu, Jiuyong Li (University of South Australia); Hao Wang, Zhaolong Ling (Hefei University of Technology); Xindong Wu (Mininglamp Academy of Sciences)
- **Venue**: arXiv preprint (JMLR-style survey formatting; exact peer-reviewed publication venue not confirmed by available tool metadata — treat as preprint until verified)
- **Keywords**: causality-based feature selection, Markov blanket / Markov boundary, constraint-based CI testing, PC (parents-children) learning, spouse / collider discovery, score-based BN learning, CausalFS toolbox, distribution-shift invariance
- **One-liner**: Survey + open-source CausalFS toolbox benchmarking 28 Markov-blanket/PC learning algorithms on synthetic BNs and real data, with formal MB-optimality theorems and an explicit CI-test-power-vs-conditioning-set-size tradeoff.

## 精华提炼

### 管线/方法

This is a survey + toolbox + benchmark paper, not a single new algorithm. Its structure:

1. **Formal setup.** Defines Bayesian network (BN), Markov blanket (Mb), and Markov boundary (MB, the *minimal* Mb) for a class variable $C$. States the central theorem: under the faithfulness assumption, $MB(C)$ is unique and equals parents ∪ children ∪ spouses (other parents of $C$'s children) of $C$ (Theorem 2-1, Definition 2-62). A corollary (Proposition 2-0) states $MB(C)$ renders $C$ independent of every other variable — i.e. MB is the minimal, maximum-predictivity feature subset for classification under faithfulness.
2. **Two propositions that every constraint-based algorithm in the survey instantiates:**
   - Proposition 2-1: if $V_i$ is a parent or child of $V_j$, then $V_i \nindep V_j \mid S$ for **every** subset $S$ — a true parent/child is never screened off by conditioning.
   - Proposition 2-2: if $V_i$ is a spouse of $V_k$ (they share a common child $V_j$), then $V_i \indep V_k$ **marginally**, but $V_i \nindep V_k \mid \{S, V_j\}$ once you condition on the common child — the classic v-structure/collider mechanism. The paper's own example: *Allergy* (spouse) and *Lung cancer* are marginally independent but become dependent once conditioned on *Coughing* (their common child).
3. **Catalogs constraint-based MB/PC algorithms in five families** (Table 1): simultaneous MB learning (GSMB, IAMB and 7 variants incl. FBED$^K$, PFBP), divide-and-conquer (MMMB, HITON-MB, semi-HITON-MB, PCMB, IPCMB, MBOR, STMB, CCMB), interleaved PC+spouse learning (BAMB, EEMB), relaxed-assumption learning for non-faithful/non-causally-sufficient data (KIAMB, TIE*, SGAI, LCMB/WLCMB, M3B), and special-purpose learning (multi-dataset MIMB, distribution-shift-invariant MCFS, multi-target MIAMB/MKIAMB, weak-supervision BASSUM/Semi-IAMB).
4. **Score-based analogues** (SLL, S²TMB, S²TMB$^+$, DMB/RPDMB, BSS-MB, LMB-CSEM) — same MB/PC targets, but read the local DAG from a BN structure-learning score instead of raw CI tests; consistently the slowest family and not shown to beat constraint-based methods.
5. **Methods for orienting PC into parents vs. children**: global BN structure learning (NP-hard), local-to-global approaches (MMHC, GGSL, PSL, TC), and approaches that use interventional/experimental data to break ties that observational data alone cannot orient (ODLP, ODLP*).
6. **CausalFS toolbox**: open-source C/C++ implementation of 28 representative algorithms (16 single-MB, 2 multi-MB, 6 PC-only, 4 score-based), 3-layer architecture (application / algorithm / data), https://github.com/kuiy/CausalFS.
7. **Benchmark evaluation** (Section 7): 4 synthetic BNs with fully known ground-truth graphs (Child, Alarm, Pigs, Gene: 20–801 variables, true PC-set sizes 1–41) at two sample sizes (500 and 5,000), scoring Precision/Recall/F1/time against the *true* MB/PC; plus 8 real datasets (UCI + NIPS2003 challenge, including 3 class-imbalanced ones: infant 0.94/0.06, bankruptcy 0.89/0.11, dorothea 0.90/0.10) scoring NB/KNN/SVM accuracy, AUC, compactness, and time.

### 关键技术细节

- **CI test families used**: $\chi^2$/$G^2$ test and mutual information (discrete features), Fisher's Z test (continuous, linear, additive Gaussian noise), kernel-based tests (nonlinear/non-Gaussian continuous).
- **Two generic search frameworks** that every algorithm instantiates: SFBS (standard forward-then-backward) and IFBS (interleaved — triggers the backward/pruning phase immediately after every forward addition, keeping the candidate set smaller throughout the search).
- **AND rule vs. OR rule for symmetry checks**: MMMB/HITON-MB/PCMB require $X \in PC(C) \iff C \in PC(X)$ (AND) to remove false positives — precise, but at small sample sizes can incorrectly delete true positives (drives down recall). MBOR/CCMB use the OR rule ($X\in PC(C)$ if *either* direction tests positive) — more permissive, favors recall over precision.
- **PCMasking** (formalized in Wu et al.'s CCMB, cited in this survey): the named failure mode where an unreliable/underpowered CI test makes a real parent-child dependency test as independent — i.e., a formal name for a conditioning-induced false negative.
- **Explicit statement of the sample/conditioning-set-size tradeoff** (repeated at multiple points, e.g. §3.3.1 on GSMB/IAMB, and the Fast-IAMB "reliable test" rule): the number of samples required for a CI test to be reliable grows **exponentially with the size of the conditioning set** (concretely, Fast-IAMB requires $N / (r_X \cdot r_C \cdot r_S) \geq k$, $k{=}5$, where $r_S$ grows combinatorially with $|S|$). This is a direct, quantitative account of why algorithms that condition on a large candidate set (simultaneous MB learners) are "time efficient but not data efficient," while divide-and-conquer/PC-only methods that condition on small subsets are more data-efficient but slower.
- **Ground-truth synthetic BN suite** ready to reuse: Child (20 vars/25 edges), Alarm (37/46), Pigs (441/592), Gene (801/972) — off-the-shelf, known-graph benchmarks with published max in/out-degree and PC-set-size ranges.
- **Toolbox**: CausalFS, C/C++, 28 algorithms, https://github.com/kuiy/CausalFS.

### 核心结果

- **Theory**: $MB(C)$ (parents + children + spouses) is the minimal, maximum-predictivity set under faithfulness — but this optimality theorem is about the *full* MB, not the parent set alone.
- **Empirically surprising finding on real data** (§7.2): "the classification performance using the PC set of a class variable is not inferior to that of using the MB" — spouses contribute little extra *raw accuracy* once parents+children are already selected, even though only the full MB carries the formal optimality guarantee. Theory and finite-sample practice diverge.
- **Class imbalance is a documented, severe weak spot** — directly analogous to the fragility-regime finding. On the three imbalanced real datasets, several MB/PC algorithms collapse to near-chance AUC even while raw accuracy looked uniformly high across algorithms (accuracy was masking near-chance discrimination): e.g. on *dorothea*, PCMB NB-AUC 0.28/0.68, IPCMB 0.23/0.59; several 0.50 AUC cells appear on *bankruptcy* (GSMB, Fast-IAMB, SVM columns) and *infant* (SLL/S²TMB KNN column). The paper's own conclusion (open problem #4) states plainly: "the majority of existing causality-based methods cannot deal with data sets with imbalanced classes."
- **Open problem #5 (Distribution shift)** explicitly separates two different notions of "optimal": the parent set is framed (via the cited MCFS work) as the minimal *invariant* set across shifted distributions, while MB/PC is the best *in-distribution* predictor — i.e., the survey itself flags that conditioning down to parents-only is a deliberate robustness/invariance trade against raw predictive power, not a strictly-dominant choice.
- Backward-strategy/symmetry-check algorithms (PCMB, IPCMB, MBOR) win on large synthetic BNs (better precision) but symmetry checks are a "double-edged sword" — computationally expensive and, at small sample sizes, prone to deleting true positives.

### 明确局限

- **No dynamical-systems/time-series/bifurcation setting anywhere in the benchmark.** All 4 synthetic BNs and 8 real datasets are i.i.d. tabular data with a single fixed causal DAG — nothing here is tested under regime change or approaching a bifurcation. PCMCI+ (lagged, time-series constraint-based CI testing) is architecturally a descendant of the IAMB/PC family cataloged here, so the CI-test machinery, AND/OR-rule tradeoffs, and the PCMasking failure mode transfer *conceptually*, but the paper provides zero direct evidence about near-bifurcation or nonstationary regimes.
- **No closed-form or information-theoretic bound on the predictive/AUC gap between conditioning levels.** The exponential-sample-requirement statement is qualitative/algorithmic (drawn from the CI-test literature), not a derived formula for how much AUC a conditional test loses relative to a marginal test at a given effect size and conditioning-set size.
- **No factorial isolation of "conditioning" as its own experimental axis.** The paper's own benchmark varies algorithm identity × sample size × dataset — never causal-conditioning-vs-marginal at matched nonlinearity / effect size / hidden-confounding. It is a **template** (known-graph synthetic BN + ground-truth MB/PC + Precision/Recall/F1/AUC against ground truth) for the kind of controlled experiment idea v3 needs, not itself evidence that already answers the design question.
- **The paper's own deepest theoretical source for "why causal vs. non-causal feature selection differ in the low-sample/weak-effect regime" is a different paper it merely cites in passing**: "A Unified View of Causal and Non-causal Feature Selection" (Yu, Liu, Li 2018, arXiv:1802.05844 — reference #23 in this paper's own bibliography; also referenced in a *commented-out* early draft of this paper's conclusion still visible in the tex source, describing causal FS as a "high-order approximation" and non-causal/classical FS as a "low-order approximation" to the same MB objective). This survey only paraphrases that paper's existence (main text, §1: "developing a unified view to link causality-based feature selection with classical feature selection") without reproducing its argument. **arXiv:1802.05844 is very likely the more load-bearing theoretical source and should be the next read**, not this survey.
- **2019 vintage**: no PCMCI/PCMCI+ (Runge et al., contemporaneous with/after this survey), no deep-learning-based causal discovery, no discussion of time series at all.

### 跟我们的关系

**Overall**: no claim collision. This is a 2019 survey of i.i.d.-tabular Markov-blanket/PC feature selection; it never discusses time series, PCMCI/PCMCI+, Granger causality, regime change, or bifurcations. It is pure background/theoretical grounding for 0710-causal-scs, not a competing method — safe and appropriate to cite as foundational context.

**(a) Does it help design a cleaner controlled/factorial comparison to isolate Δ = AUC_conditional − AUC_marginal?**
Partially, as a *template* rather than a ready result. Its own evaluation methodology (§7.1) is exactly the right shape: pick a synthetic generator with a **fully known ground-truth graph**, hold the graph fixed, vary **one axis at a time** (here: sample size, 500 vs. 5,000), and score every algorithm against the *true* MB/PC using Precision/Recall/F1 (and, on real data, AUC). The direct actionable recommendation for the v3 factorial redesign: build (or reuse a toy analogue of) a known-DAG generator and vary **effect size / proximity-to-threshold as its own explicit factor**, independent of nonlinearity, hidden-driver count, and window length — because (see (b) below) this paper's own CI-test machinery predicts that conditioning-set size and effect size *interact multiplicatively* in determining detection power. If effect size isn't pinned down as a controlled factor, any AUC gap between a conditional (parents-only) indicator and a marginal indicator is confounded with exactly the axis most likely to drive it. This paper does **not** run that isolated experiment itself — nothing here directly measures Δ under matched confounds — so it cannot be cited as evidence for the Δ finding, only as a methodological template and as the source of the mechanism the design should target.

**(b) Does it explain why causal conditioning might discard marginal-correlation-carried predictive signal?**
Yes, with one important caveat about which mechanism actually applies. Two candidate mechanisms appear in this paper, and they point in different directions for the v3 pilot's fragility-regime result:

1. *Collider/spouse mechanism (Proposition 2-2)*: conditioning on a common child creates dependence between two variables that were marginally independent (the textbook v-structure/"explaining away"). This is the standard "collider bias" story, but it runs the **wrong direction** for the pilot's finding — it explains how conditioning can *reveal* spouse-carried signal that marginal correlation *misses* entirely (a true spouse is exactly marginally independent of the target), not how conditioning could *lose* signal that a marginal test *does* catch. If anything, this mechanism argues causal conditioning should sometimes catch more, not less, than a marginal baseline — so it does not fit the pilot's chance-level-conditional / above-chance-marginal pattern and should not be the headline mechanism cited.
2. **CI-test statistical-power mechanism (the more likely fit)**: repeated throughout the paper (§3.3.1 on GSMB/IAMB, the Fast-IAMB reliability rule, the general "simultaneous MB learning...time efficient but not data efficient" discussion) is the explicit, quantitative claim that **the number of samples needed for a reliable CI test grows with (in the simplest cases, exponentially in) the size of the conditioning set**, for a *fixed* true effect size. Consequently, for a **fixed sample budget**, a full/parent-conditioned test has systematically lower power (higher Type-II error) than a marginal (zero-order, $S=\emptyset$) test — and that power gap widens exactly as the true effect size shrinks toward a detection threshold. This maps directly onto the fragility regime: if fragility corresponds to weak/borderline dependency strength (by construction, since no true graph change has happened yet), a full-conditioning-set causal test is structurally the worst-powered test available for exactly that regime, while a marginal/"existence" test at $S=\emptyset$ retains maximal power precisely because it pays no conditioning penalty. Under this framing, the "existence threshold" non-causal baseline is not a different kind of method so much as **the zero-order/maximally-powered special case of the same CI-testing family** that PCMCI+'s conditioning-based test sits at the low-power, high-order end of. PCMasking (Wu et al., CCMB, cited in this survey) is the formal name in this literature for exactly this failure mode (a real dependency testing as independent due to an unreliable/underpowered conditional test).
3. A third, complementary angle from the survey's own open-problem framing (§8, and the parent-set-invariance result from MCFS): the parent set is cast as the minimal set that is *invariant across distribution shift*, explicitly *not* claimed to be the most discriminative set within one fixed distribution — the MB (not the parent set) carries the raw-predictivity optimality guarantee. If PCMCI+'s "causal conditioning" indicator is effectively targeting a parents-only (or near-parents) estimate, discarding some raw predictive power relative to a full-MB or marginal estimate is the *expected*, not anomalous, behavior under this paper's own theory — the interesting empirical question is only how large that gap is and whether it's exaggerated by weak-sample/near-threshold conditions (mechanism 2) on top of the structural trade-off.

**Net recommendation for the wiki reader**: cite this survey for (i) the ground-truth-BN benchmark template, (ii) the formal conditioning-set-size vs. test-power tradeoff as the primary mechanism candidate, and (iii) the parent-set-vs-MB predictive/invariance distinction — but do **not** lean on the collider/spouse mechanism as the explanation, since it points the wrong way for this specific finding. The single highest-value next read is **arXiv:1802.05844** ("A Unified View of Causal and Non-causal Feature Selection," Yu/Liu/Li 2018), which this survey itself cites as the paper that formalizes causal vs. non-causal feature selection as high-order vs. low-order approximations to the same Markov-blanket objective — almost exactly the theoretical framing the idea needs. A close second is the 2026 citing paper "Causal feature selection using a conditional independence test with a balanced trade-off between Type I and Type II errors" (Li, Chen, Lin et al.), which by title appears to directly target the Type-I/Type-II tradeoff underlying mechanism 2 above.

**审读视角**：v3 pilot 发现 PCMCI+ causal conditioning 在 fragility regime（趋近分岔但无真实图变化）AUROC 落 chance（0.482），non-causal 边际相关基线通过存在性门槛（0.705），但两者在真实图变化正对照 regime 都表现良好（causal 0.786, non-causal 0.849）。Reviewer 指出这个双 regime 比较混淆了 conditioning 之外的多个轴（非线性/隐藏驱动/效应量/窗口参数）。

**撞车评估**：不撞车。这是一篇 2019 年的 i.i.d. 表格数据 Markov blanket / PC 特征选择综述 + 工具箱 + benchmark 论文，全文未提及时间序列、PCMCI/PCMCI+、Granger 因果、regime change 或分岔。可安全作为背景理论引用，不构成竞争方法。

**可用于设计干净对照实验（问题 a）**：本文 §7.1 的 benchmark 方法论（已知真实图的合成 BN + 固定图结构 + 单轴变化 sample size + 对照真实 MB/PC 算 Precision/Recall/F1/AUC）是理想的 factorial 对照模板。具体建议：构造/复用已知 DAG 的合成生成器，把**效应量/趋近阈值程度**设为独立显式因子，与非线性、隐藏驱动数量、窗口长度分开控制 —— 因为下面机制（b）说明效应量和 conditioning-set 大小在决定检验功效上是**乘性交互**的，不分离这两者，任何 Δ=AUC_conditional − AUC_marginal 的差异都无法排除是效应量混淆而非 conditioning 本身导致。本文自己没有做这个隔离实验（其 benchmark 变化的是算法身份×样本量，不是 conditioning 阶数本身），只能作为方法论模板，不能引用为证据。

**理论机制（问题 b）**：本文提供两个候选机制，方向不同，需要谨慎甄别：
1. Collider/spouse 机制（Proposition 2-2）：条件化在共同子节点上会*产生*新的依赖（v-structure/explaining-away）。这个机制方向反了 —— 它解释因果条件化如何能**捕捉**边际相关捕捉不到的 spouse 信号（真 spouse 边际独立），而不是因果条件化如何**丢失**边际相关能捕捉到的信号。不应作为本 pilot 发现的主要引用机制。
2. **CI 检验统计功效机制（更贴合的候选）**：全文反复强调（§3.3.1 GSMB/IAMB 讨论、Fast-IAMB 可靠性检验规则）CI 检验所需样本量随 conditioning set 大小指数增长；固定样本预算下，全条件化检验的功效系统性低于零阶（边际，S=∅）检验，且这个功效差距在真实效应量趋近检测阈值时会被放大。这与 fragility regime（按定义是弱/临界依赖强度，因为还没发生真实图变化）高度吻合：因果条件化检验恰好是该 regime 下功效最差的检验，而边际"存在性门槛"检验恰好是同一 CI 检验族群的零阶、最大功效特例。CCMB 论文（Wu et al. 2019，本文引用）中的 "PCMasking" 概念是这个失效模式在该文献中的正式名称。
3. 补充视角：MCFS 的 parent-set 不变性结果（本文 §8 open problem #5 引用）明确区分"parent set = 跨分布不变的最小集合"vs"MB = 单一分布内最优预测集合" —— 如果 PCMCI+ 的因果条件化本质上在逼近 parents-only 估计，牺牲一部分原始判别力换取不变性/结构正确性是本文理论下的**预期行为**，不是异常。

**下一步优先阅读**：arXiv:1802.05844（"A Unified View of Causal and Non-causal Feature Selection", Yu/Liu/Li 2018）—— 本文引用但只带过，很可能是机制（b）真正的理论来源（把 causal/non-causal FS 形式化为对同一 MB 目标的高阶/低阶近似）。次优先：2026 反引文 "Causal feature selection using a conditional independence test with a balanced trade-off between Type I and Type II errors"，标题直指机制 2 的 Type-I/Type-II 权衡。
