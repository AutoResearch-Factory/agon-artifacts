# Deep Generative Symbolic Regression (2401.00282)

- **Year**: 2024.01
- **Authors**: Samuel Holt, Zhaozhi Qian, Mihaela van der Schaar (University of Cambridge)
- **Venue**: ICLR 2024
- **Keywords**: Symbolic Regression, Deep Generative Models, Pre-training, Policy Gradients, Equation Invariance, Set Transformer, NMSE Loss, Gradient Refinement
- **One-liner**: 用预训练深度条件生成模型学习方程表示的不变性，在推理时通过梯度精炼实现高维符号回归，在12变量问题上首次显著超越RL方法。

## 精华提炼

### 管线/方法

完整的 DGSR pipeline 分两步：

**第一步：预训练 (Pre-training)**
1. 从方程先验 `p(f)` 中采样生成 100K 个方程，使用 Lample et al. 的方程生成框架
2. 对每个方程 `f`，在定义域 X 上采样 n 个点，构成数据集 `D = {(X_i, f(X_i))}`
3. 用 mini-batch（t=5 个数据集，每个数据集采样 k=500 个方程）训练条件生成模型 `p_θ(f|D)`
4. 优化目标：端到端 NMSE loss（Eq.1），重写为奖励函数 `R(θ)=1/(1+L(θ))`，用 vanilla policy gradient (VPG/REINFORCE) 优化
5. 同时训练 encoder 和 decoder 参数 θ，直到 early stopping（~10K datasets）

**第二步：推理 (Inference)**
1. 给定观测数据集 D，encoder 编码得到 latent vector V
2. Decoder 自回归生成方程 tokens（使用 hierarchical tree state representation）
3. 用 NGPQT（neural guided priority queue training）优化方法精炼 decoder 权重 φ（encoder 冻结）
4. NGPQT = policy gradient (PQT) + genetic programming (DEAP)，GP 组件帮助逃离局部最优
5. 采样 k 个方程 seed GP 初始种群，GP 迭代多代后合并到训练集，用 PQT loss 更新 decoder
6. Monte Carlo 采样 k 个方程，选 NMSE 最低的作为 MAP 估计返回
7. 推理预算：2M 方程评估（或找到真方程提前停止）

### 关键技术细节

**模型架构：**
- Encoder: Set Transformer（Lee et al. 2019）——3 个 ISAB (Induced Set Attention Block) + PMA (Pooling by Multihead Attention)，hidden dim=32, 1 head, 1 output feature, 64 inducing points
- Decoder: 标准 Transformer Decoder（Vaswani et al. 2017）——2 layers, hidden dim=32, 1 attention head, 0 dropout
- 额外路径：已生成的 tokens 经过 hierarchical tree state representation（DSR 的 parent/sibling encoding），通过一个 3-layer transformer encoder 编码为额外 latent vector，与 encoder latent vector 拼接成总 latent vector `U ∈ R^(w+ds)`，其中 w=32, ds=32
- 总参数量：**122,446**（比 NESYMRES 的 26M 和 E2E 的 86M 小约两个数量级）
- Token 约束：使用 DSR/NGGP 的 token selection constraints（限制方程长度、子节点约束、三角函数嵌套约束等）
- 数值常数优化：BFGS 算法（Fletcher 2013），初始猜测为 1.0，token 库中不显式包含常数 token

**预训练超参数：**
- Pre-training datasets: m=100K 方程（pre-compiled）
- Mini-batch: t=5 datasets, k=500 equations per dataset → 2500 equations per mini-batch
- EWMA baseline coefficient α=0.5
- Entropy weight λ_H=0.003
- Minimum/maximum equation length: 4/30
- Adam optimizer, lr=0.001
- Early stopping patience: 100 mini-batch iterations
- Equation generator prior `p(f)`: max_leaves=5, min_leaves=3, 操作符权重分布见表（+和×各10，-和÷各5等），叶子节点 80% 概率为变量、20% 为整数
- 预训练硬件：RTX3090 24GB GPU，~5 小时

**推理超参数：**
- Batch size k=500, entropy weight λ_H=0.003
- 方程长度约束: 4-30 tokens
- PQT queue size q=10, sample selection size=1
- GP generations per iteration=25, crossover prob=0.5, mutation prob=0.5, tournament size=5, mutate tree max=3
- Risk-seeking quantile ε=0.02
- Adam lr=0.001（仅更新 decoder 参数）
- 最大预算：2M equation evaluations（除非提前找到真方程）

**方程生成与预训练数据：**
- 使用 Lample et al. (2019) 的方程生成方法：随机生成方程树 → pre-order 遍历得到 prefix notation → Sympy 解析为函数式方程
- 强制变量连通性约束：如 x5 存在，则 x1...x4 也必须在树中出现
- 可注入数值常数占位符（从 U(-1,1) 采样）
- 预训练数据集不包含任何测试集方程（通过交叉检查排除）
- 每个 benchmark problem set 独立预训练一个模型（因 token 库不同）

**方程表示：**
- Prefix notation（前缀表示法），例如 x1 + x2 → [+, x1, x2]
- 与传统方法的区别：不直接存储为 expression tree 或 raw token sequence，而是通过 encoder 被编码为 set-invariant latent vector
- NMSE loss 的关键优势：x1×x2 和 x2×x1 具有完全相同的 loss 值（不管 token 序列差异）

**Surprising 发现：**
1. DGSR 在噪声+低数据量场景下显著优于 NGGP（n=10 样本时 91.66% vs 66.66%），说明预训练 prior 在真实世界数据稀缺场景中有巨大优势
2. DGSR 可以在 d=2 上预训练，在 d=5 上推理仍有 61.29% 恢复率（vs d=5 预训练的 67.50%），证明跨维度泛化能力
3. 即使去掉 GP 组件，纯 policy gradient 版 DGSR 也比同配置 NGGP 少 ~5x 方程评估次数（29,356 vs 151,231）
4. NESYMRES 用 CE loss 预训练生成几乎 100% 有效方程，但找不准真方程——说明"生成有效方程"和"找到真方程"是两个不同的问题
5. Feynman-8（含指数嵌套分数）所有方法恢复率为 0%，但 DGSR 能找到 NMSE=0.142 复杂度=64 的近似解

**Inference 效率：** DGSR 平均方程评估数低于 NGGP（尤其在 d=12 上：271K vs 829K），但 NESYMRES 最快（256 次采样固定不变，但恢复率低）

### 明确局限

**作者自述：**
1. 高度复杂的方程可能无法发现（变量多、操作符多、高度嵌套）
2. 假设所有变量均被观测，未观测变量场景失败
3. 数值常数优化可能陷入局部最优（初始猜测 1.0 + BFGS）
4. 每个新 token 库/定义域需重新预训练模型

**我读出的额外局限：**
1. 没有真机/真实物理实验验证——全部在合成数据上评测（虽然是标准 benchmark）
2. 预训练需要为每个问题集单独做，缺乏"一次预训练到处推理"的通用性
3. 没有与 SINDy 类方法做对比（只对比了 GP、RL、encoder-decoder 系列）
4. 预训练方程生成器用固定分布，可能与真实科学方程分布不匹配
5. 122K 参数的小模型限制了表示的丰富度——作者自己也说更复杂的方程可能需要更大模型
6. 没有在偏微分方程 (PDE) 发现任务上评估

## Read by: 0616-fex (2026-06-24)

**撞车评估：**
不撞车。DGSR 是做符号回归的具体方法（输入端：观测数据 → 输出端：closed-form equation），而 fex 是实验基础模型的宏大框架（输入端：实验目标/历史数据 → 输出端：实验设计/新知识/假设）。两者的输入输出空间和问题设定完全不同。

**对 fex 可用的工具/方法：**
1. **Permutation-invariant dataset encoding (Set Transformer)**：fex 处理多源实验数据时，不同实验的样本数和变量数不同，Set Transformer 的排列不变性天然适合
2. **Pre-training + gradient refinement 两阶段范式**：fex 可以先在大量合成实验数据上预训练，再在真实实验数据上精炼——这是 DGSR 的核心方法论
3. **端到端 NMSE loss 替代 token-level CE loss**：fex 如果涉及"从数据恢复公式"子任务，NMSE loss 比 CE loss 更适合
4. **Policy gradient + genetic programming 混合优化**：fex 的实验设计模块（离散组合优化）可以用类似的混合优化策略
5. **方程生成框架**（Lample et al. 2019）：可用于 fex 的预训练数据生成——自动合成大量"实验数据-方程"对

**差异化：**
- fex 的目标不是"找方程"而是"做实验"——生成实验方案、预测实验结果、迭代优化实验设计。DGSR 解决了"给定数据找公式"的问题，fex 需要解决的更广泛：在未给定最佳数据采集策略时，主动决定采集什么数据、何时停止、如何更新信念。
- DGSR 的输入是"一个确定的数据集"，fex 的输入是"一个实验目标 + 已有实验历史 + 领域知识"——信息维度更高
- DGSR 的输出是"一个方程"，fex 的输出是"一个实验方案/知识发现"——决策空间完全不同

**该引：** YES。DGSR 是生成式科学发现的重要 baselines，它的 pre-training + refinement 范式、set-invariant encoding、end-to-end NMSE loss 都是 fex 系统设计时的参考锚点。同时，DGSR 的局限（只处理 closed-form equations、假设所有变量已观测、不能主动采样）恰好是 fex 试图突破的方向。
