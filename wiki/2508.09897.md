# Finetuning Large Language Model as an Effective Symbolic Regressor (2508.09897)

- **Year**: 2025.08
- **Authors**: Yingfan Hua, Ruikun Li, Jun Yao, Guohang Zhuang, Shixiang Tang, Bin Liu, Wanli Ouyang, Yan Lu; USTC, Shanghai AI Laboratory, CUHK
- **Venue**: arXiv
- **Keywords**: symbolic regression, LLM fine-tuning, SymbArena, Form-GRPO, LoRA, structure-aware reward, HER inference, form-level consistency
- **One-liner**: 用 SymbArena 微调 Qwen 做强符号回归。

## 精华提炼

### 管线/方法
- 论文目标是把通用 LLM 从“提示式公式猜测器”改造成面向符号回归的专门模型。
- 输入任务是若干组观测数据点，即独立变量 \(x\) 与因变量 \(y=f(x)\) 的配对。
- 输出任务是一个可解释的符号公式，论文中统一要求模型只返回公式字符串。
- 数据侧先构造 SymbArena，一个同时服务训练和评测的符号回归数据集。
- SymbArena 的每个样本包含一个方程形式 \(f_i\) 和对应的数据矩阵 \(d_i\)。
- 方程生成以树结构表示表达式，节点包括变量节点、常数占位符节点和操作符节点。
- 操作符库覆盖一元函数如 `sin`、`log`、`exp`、`sqrt`、`arcsin`，以及二元运算如 `+`、`-`、`*`、`/`。
- 树生成采用递归增量扩展：从局部树出发，逐步采样操作符或终端节点插入扩展位置。
- 生成后做语法有效性检查，过滤括号、符号组合和数学操作不合法的表达式。
- 生成后做唯一性检查，核心依据是 equation skeleton 的字符串相似度。
- 生成后做复杂度检查，树层数低于 4 或高于 12 的公式被过滤。
- 对每个公式随机采样 \(K\) 组变量值，变量来自均匀分布 \(\mathcal{U}(-dom, dom)\) 或高斯分布 \(\mathcal{N}(0, dom)\)。
- 论文实现中 \(dom=10\)，作者认为真实应用可通过去归一化吸收输入尺度差异。
- 训练/测试划分按 equation skeleton 的 unique form 做，避免同构公式跨 train/test 泄漏。
- 测试集还做 reality enhancement：每个公式用 LLM deep research 检索 3 个相似真实科学方程。
- LLM 返回相似性解释后，作者人工过滤低相似或不可信的候选，使测试式子更接近真实科学场景。
- 该现实性增强没有直接使用已有真实方程，目的是减少 LLM 预训练记忆带来的评测污染。
- 方法侧提出 Symbolic-R1，主干是 Qwen2.5-7B-Instruct。
- Symbolic-R1 的训练分两阶段：Instruction Tuning，然后 Reinforcement Fine-Tuning。
- Instruction Tuning 将数据矩阵转为 prompt，prompt 包含共享任务说明和数据特定的 key-value 样本。
- 任务说明告诉模型从 input-output pairs 推导公式，并列出允许的基础数学操作。
- 数据部分把 \(d_i\) 重写成 `x_0=..., x_1=..., f(x)=...` 这样的文本输入。
- SFT 目标仍是标准 cross-entropy，让模型输出 token 与 ground-truth 公式 token 对齐。
- 参数高效微调用 LoRA，论文给出 rank=8、alpha=32。
- Reinforcement Tuning 用作者设计的 Form-GRPO。
- Form-GRPO 的奖励不是单一 R2，而是四类规则奖励的加权和。
- Form-Correct Reward 用 `is_valid` 检查表达式能否转为 NumPy 或 math 可执行表达式。
- 语法有效公式给 +1，非法公式给 -1。
- Form-Similarity Reward 直接使用论文的 form-level consistency metric。
- Numerical Reward 用 \(R^2\) 衡量拟合数据的好坏。
- 由于 \(R^2\) 下界无穷，奖励用 `max(0, R^2)` 截断，非法公式数值奖励为 0。
- Equivalent Reward 奖励 skeleton 与 ground-truth skeleton 完全一致的表达式。
- GRPO 每个输入采样 \(G=8\) 个候选公式，计算组内 reward，再用组内均值和标准差归一化 advantage。
- 作者采用的主 reward 权重是 `R_format=1.0, R_similarity=2.0, R_numerical=2.0, R_equiv=4.0`。
- 推理侧提出 Hypothesis-Experiment-Revision (HER)。
- HER 每轮从 Symbolic-R1 生成 \(K=6\) 个候选公式。
- 每个候选公式先通过数值优化工具优化连续系数。
- Experiment 步用 \(R^2\) 作为候选公式与观测数据一致性的验证指标。
- Revision 步把公式、分数和模型生成的反思评论组成 memory bank。
- memory bank 维护 top-5 记录，按 \(R^2\) 排名保留强候选。
- 下一轮把 memory bank 改写成 reflection prompt，追加到原始数据 prompt 后继续生成。
- 最终 HER 配置为 5 轮，每轮 6 个候选，因此每个样本约产生 30 个候选公式。
- 该框架的实际定位是“微调后的 LLM 负责结构假设生成，数值优化和 R2 负责参数/候选筛选”。

### 关键技术细节
- SymbArena 总计 148,102 个方程。
- 训练集为 147,590 个方程，测试集为 512 个方程。
- 论文摘要称这些方程被组织成 1.83B tokens 的 LLM 可用语料。
- 每次构造 prompt 和评测 baseline 时，统一采样 200 个 input-output 数据点。
- SFT 阶段使用 146,590 个 unique expressions。
- SFT 为每个表达式构造 5 种 prompt variation。
- RFT 阶段使用 1,000 个表达式。
- RFT 每个输入从模型采样 8 个 candidate responses。
- LoRA rank 是 8，alpha 是 32。
- LLM backbone 是 Qwen2.5-7B-Instruct。
- LLM baseline 包括 LLM-SR 和 SGA。
- LLM baseline 测试了 Qwen2.5-7B-Instruct、GPT-3.5-turbo 和 GPT-4o-mini。
- LLM-SR 最大采样公式数 capped at 50。
- SGA 配置为 5 iterations，每轮 exploit 2 个公式、explore 4 个公式。
- 所有 LLM backbone 的 temperature 设置为 0.7。
- 传统 SR baseline 包括 gplearn、AFP、AFP-FE、GP-GOMEA、uDSR 和 PySR。
- PySR 配置为 `maxsize=20, niterations=40`。
- gplearn 配置为 `population_size=1000, generations=20, p_crossover=0.9, max_samples=1.0, parsimony_coefficient=0.001`。
- 实验环境给出 Ubuntu 20.04.6 LTS、Python 3.11.8、PyTorch 2.7.0、CUDA 12.4.1。
- 评测指标分两类：数值精度和形式一致性。
- 数值精度包括 \(R^2\) 和 tolerance-based accuracy \(\text{Acc}_\tau\)。
- \(\text{Acc}_\tau\) 检查 worst-case relative error 是否在容忍阈值内。
- 形式一致性包括 rule-based metric 和 GPT-4o semantic adjudicator。
- rule metric 将公式 skeleton 拆成六个结构特征。
- 六个特征是 operators、functions、variables、constants、structural pattern、complexity。
- 集合型特征用 Jaccard similarity。
- 数值型特征如 constants count 和 complexity 用 `min/max` ratio。
- structural pattern 用字符位置匹配数除以更长字符串长度。
- 六项相似度做无权平均并裁剪到 `[0, 1]`。
- GPT-4o metric 给出 0 到 1 的整体结构相似评分，但这也引入了模型裁判偏差。

### 核心结果
- 在 SymbArena 主表中，Symbolic-R1 达到 `S_struct(gpt-4o)=0.436`。
- 在 SymbArena 主表中，Symbolic-R1 达到 `S_struct(rule)=0.607`。
- 在 SymbArena 主表中，Symbolic-R1 达到 `R^2=0.808`。
- 在 SymbArena 主表中，Symbolic-R1 达到 `Acc_tau=0.404`。
- PySR 是最强传统 baseline，主表为 `S_struct(gpt-4o)=0.368, S_struct(rule)=0.382, R^2=0.663, Acc_tau=0.398`。
- Symbolic-R1 相比 PySR 在 \(R^2\) 上高 0.145，在 rule form consistency 上高 0.225。
- Symbolic-R1 的 `Acc_tau=0.404` 只比 PySR 的 0.398 略高，说明完全数值成功率优势不大。
- 最强 LLM baseline 的 rule form score 是 SGA(gpt-3.5-turbo) 的 0.504。
- Symbolic-R1 的 rule form score 0.607 相比它提升 0.103 绝对值。
- 最强 LLM baseline 的 \(R^2\) 是 LLM-SR(gpt-4o-mini) 的 0.355 或 SGA(gpt-3.5-turbo) 的 0.327。
- Symbolic-R1 的 \(R^2=0.808\) 相对这些 LLM baseline 大约是两倍以上。
- 消融显示，未调 Qwen2.5-7B 无迭代时 \(R^2=0.201\)，`S_struct(rule)=0.466`。
- 未调 Qwen2.5-7B 加 HER 后 \(R^2=0.344\)，说明纯迭代有帮助但不够。
- 仅 IFT 且无迭代时 \(R^2=0.313\)，`S_struct(rule)=0.590`。
- IFT+HER 时 \(R^2=0.624\)，`S_struct(rule)=0.613`。
- IFT+RFT 且无迭代时 \(R^2=0.540\)，`S_struct(rule)=0.574`。
- 完整 IFT+RFT+HER 时 \(R^2=0.808\)，是决定性组合。
- 反直觉点：RFT 后无迭代的 rule form score 0.574 低于 IFT-only 无迭代的 0.590，但 \(R^2\) 从 0.313 升到 0.540。
- 这说明 Form-GRPO 的数值奖励强化了拟合能力，但结构相似性不一定单调提升。
- GRPO 权重消融中，配置 b 的 `R_numerical=2.0` 达到最高 \(R^2=0.814\) 和 `Acc_tau=0.412`。
- 作者最终选择配置 f，因为它的 `S_struct(gpt-4o)=0.436` 和 `S_struct(rule)=0.607` 最好，尽管 \(R^2=0.808\) 略低于配置 b。
- 噪声实验加入 \(\sigma=0.001\) 的 additive Gaussian noise。
- 噪声下 Symbolic-R1 的 \(R^2\) 从 0.808 降到 0.793。
- 噪声下 rule form consistency 从 0.607 降到 0.596。
- 噪声下 PySR 的 \(R^2\) 从 0.663 降到 0.625。
- 跨 benchmark 实验覆盖 Nguyen、Constant、R、Keijzer 和 SRBench。
- Symbolic-R1 的 overall average 为 0.969，略高于 PySR 的 0.968。
- Symbolic-R1 在 Nguyen、R、SRBench 排第一，在 Constant、Keijzer 排第二。
- 可视化 case 中，ground-truth 是 `C*x_1 + C*arctan(C*x_0 + C) + C`。
- Symbolic-R1 唯一恢复同一 symbolic form；多个 baseline 输出过长、多项式化或变量错配形式。

### 明确局限
- 论文核心评测基于作者自建合成 benchmark，真实实验数据或真实科学发现案例不足。
- 测试集 reality enhancement 依赖 LLM deep research 和人工筛选，但没有给出可审计的筛选协议、标注一致性或专家名单。
- GPT-4o 被用作结构相似性裁判之一，生成者与裁判都属于 LLM 系统，独立性有限。
- rule-based form metric 是作者设计的启发式平均，是否符合科学家对“等价结构”的判断还需要外部验证。
- 主文没有明确代码、数据、训练日志和模型 checkpoint 的开放情况；仅凭 tex 不能确认可复现 artifact。
- Acc_tau 只有 0.404，说明即使完整方法在 512 个测试公式上也远未稳定解决符号回归。
- 等价 skeleton 的完全命中率没有在正式表中作为主指标展示，注释表里曾出现接近 0 的 skeleton accuracy，值得追问。
- 论文称迭代 baseline 边际收益低，但 HER 本身也依赖多候选和多轮筛选；公平成本比较需要更清楚。
- 正式主表删除了 time cost 列，而注释表中 Symbolic-R1 的 92.05s 并不比 SGA 的 57.47/69.12/73.81s 更快。
- 测试 prompt 仅输入数值点，不给科学语义，这对公平比较有价值，但不能说明模型能处理真实实验噪声、缺失变量或单位约束。
- 方程生成限制在预设操作符库内，超出库的物理结构、微分方程、积分项、分段函数等没有系统覆盖。
- 系数优化工具的细节没有展开，后续反引文也指出 SR 评分会受到参数优化质量显著影响。
- 作者将训练数据和评测数据都来自同一生成机制，虽然 skeleton split 减少泄漏，但 domain shift 仍弱。

## Read by: 0616-fex (2026-06-16)

这篇对 `0616-fex` 的风险点很明确：如果我们的核心叙事是“为公式/方程任务合成大规模语料，微调 LLM，再用结构奖励和数值反馈做迭代发现”，它已经覆盖了主要 claim。尤其是 SymbArena、Form-GRPO 和 HER 三件套足以构成一个完整系统，不能只在名称上换皮。

可用部分也很清楚：可以借用 skeleton-based train/test split、六项结构相似度、validity/form/numerical/equivalence reward 分解，以及 top-k memory bank 推理框架。但我们应把这些作为 baseline 或 inspiration，而不是创新点本身。

差异化建议放在三处：第一，把生成者和检验者真正分离，例如用外部 CAS、独立优化器、held-out physical constraints 或人工审计作 veto；第二，使用真实实验/文档抽取场景，而不是同一生成器分布下的合成方程；第三，公开数据、代码、checkpoint 和逐样本评测日志，解决本文 R4 偏弱的问题。

这篇应该引用。推荐在 related work 中把它放在“LLM fine-tuning for symbolic regression”核心位置，同时引用 LLM-SRBench、LLM-SR、SGA 和 LaSR 作为它的直接前史。
