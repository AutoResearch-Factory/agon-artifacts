# Noise-Resilient Symbolic Regression with Dynamic Gating Reinforcement Learning (2501.01085)

- **Year**: 2025.01
- **Authors**: Chenglu Sun, Shuo Shen, Wenzhi Tao, Deyi Xue (Tencent IEG), Zixia Zhou (Stanford University)
- **Venue**: arXiv; AAAI 2025 formatted manuscript
- **Keywords**: symbolic regression, reinforcement learning, L0 regularization, noise-resilient gating, action masking, path entropy, PPO, Nguyen benchmark
- **One-liner**: 用 L0 门控和 PPO 熵正则提升噪声变量下符号回归。

## 精华提炼

### 管线/方法

1. 输入是符号回归数据集 `(X, y)`，目标是恢复未知表达式 `f`，使 `y_i = f(X_i)`。
2. 论文中的 "noise" 不是标签噪声，而是输入中加入的无关变量或 irrelevant variables。
3. 整体方法叫 NRSR，核心由两个模块组成：Noise-Resilient Gating Module (NGM) 和 RL 符号表达式搜索。
4. 第一步先训练 NGM，把输入变量维度上的无关变量过滤掉。
5. NGM 使用 L0 gating layer，门控 `G` 是二值变量，`0` 表示抑制输入变量，`1` 表示保留输入变量。
6. 门控后的输入为 `X' = X \odot G`，再送入一个小型回归网络拟合目标 `y`。
7. NGM 优化目标是 MSE 加 L0 正则：既要拟合 `y`，又要让 gate 尽量稀疏。
8. 因为 L0 不可微，作者采用 Louizos et al. 的近似思路，并在附录中写了 Binary Concrete / Gumbel-Softmax 风格的可微门控实现。
9. NGM 训练完成后，不直接把连续 gate 当作动作约束，而是跨 epoch 平均 gate 值。
10. 平均 gate 再用 Otsu threshold 转成离散 gate，论文设定 Otsu threshold scale 为 `1.05`。
11. 如果 gate 全为 0，作者认为模型无法判断变量是否全无关或全必要，此时不做输入过滤。
12. 第二步把 NGM 得到的 gate 和原本 SR 搜索中的 action mask 相乘，得到新的 action mask。
13. 原始 action mask 用来阻止无意义 token 序列，例如 `log(exp(x))` 或 `sin(cos(x))`。
14. 新 action mask 同时保留原本的语法/反函数约束，并禁止选择被 NGM 判断为无关的输入变量。
15. 第三步由 RNN policy 逐 token 生成表达式的前序遍历序列。
16. token library 包括 `{+, -, *, /, sin, cos, log, exp, x_i}`。
17. 生成的 token sequence 被解释为 expression tree，再计算候选表达式在数据集上的拟合质量。
18. reward 使用 `INRMSE = 1 / (1 + NRMSE)`，NRMSE 越小 reward 越高。
19. 每一轮生成一批 trajectories，再按 reward 只保留 top epsilon 样本训练 policy。
20. policy 优化使用 PPO，而不是 DSR 原始 risk-seeking policy gradient。
21. PPO 的 advantage 用 `R(tau) - R_eta`，其中 `R_eta` 是 batch reward 的经验分位数 baseline。
22. 除常规分步 entropy 外，作者提出 Mixed Path Entropy (MPE)。
23. MPE 的新项是 path entropy：对整条 token sequence 的联合概率做熵正则。
24. 直觉是单步 entropy 鼓励每个位置别太早确定，而 path entropy 鼓励整条表达式路径多样。
25. 最终目标写成 `L(theta) = L_p(theta) + alpha H_tau(pi_theta) + beta H(pi_theta)`。
26. 算法终止条件是找到 reward 为 1 的完美表达式，或者达到最大 explored expression number。

### 关键技术细节

1. NGM 网络结构：gating layer、两个 dense layers、output layer。
2. Dense layers 使用 batch normalization 和 ReLU。
3. NGM training samples 为 `20,000`。
4. NGM batch size 为 `256`。
5. NGM hidden size 为 `128`。
6. train dataset ratio 为 `0.8`。
7. NGM optimizer 为 Adam。
8. Adam 参数为 `beta_1 = 0.99`, `beta_2 = 0.999`。
9. NGM learning rate 为 `0.001`。
10. NGM epoch number 为 `20`。
11. NGM L2 regulation weight 为 `1e-5`。
12. L0 gating loss `lambda = 0.25`。
13. Otsu threshold scale 为 `1.05`。
14. SR/RL policy 是 single-layer LSTM 加 dense layer。
15. NRSR 的 LSTM hidden size 为 `32`。
16. NRSR 的 max used samples 为 `2e6`。
17. NRSR 的 SR batch size 为 `1000` trajectories。
18. NRSR policy algorithm 是 PPO。
19. epsilon-risk factor 为 `0.05`。
20. SR optimizer 是 Adam。
21. SR learning rate 为 `5e-5`。
22. MPE/path entropy coefficient `alpha = 0.05`。
23. step entropy coefficient `beta = 0.02`。
24. hierarchical entropy decay `gamma = 0.7`。
25. 每个 Nguyen benchmark 训练独立的 RL policy，不是一个跨任务通用模型。
26. 评测基准是 12 个 Nguyen symbolic regression expressions。
27. 数据分三份：NGM 训练集、表达式 reward 计算集、每轮 best expression 测试集。
28. reward/test 集在论文配置中各只有 `20` 个样本，NGM 使用 `20,000` 样本。
29. 高噪声实验通过额外加入 5 或 10 个无关输入变量构造。
30. 对比方法包括 DSR、HESL、GP-Meld、DGSR、Eureqa。
31. 除 Eureqa 通过 DataRobot API 跑以外，其它方法都放在 unified SR framework 中实现。
32. 硬件为 Nvidia A10 GPU、32-core CPU，Python 3.8。
33. 指标为 RR, EEN, NMSE：recovery rate 越高越好，explored expression number 和 NMSE 越低越好。
34. 主要统计均为每个 benchmark 100 个 random seed replicate 的平均。
35. 附录里 NGM 还在 93 个表达式上测试不同数量无关变量的 gate filtering accuracy。

### 核心结果

| Setting | Method | RR | EEN | NMSE |
|---------|--------|----|-----|------|
| 5 noisy variables | NRSR | 89.1% | 425K | 7.73e-3 |
| 10 noisy variables | NRSR | 89.1% | 423K | 8.52e-3 |
| clean data | NRSR | 89.7% | 408K | 7.66e-3 |

1. 5 个噪声变量下，NRSR 平均 RR 为 `89.1%`，明显高于 DSR/DSP `61.3%`、HESL `63.9%`、GP-Meld `51.0%`、DGSR `72.5%`、Eureqa `35.0%`。
2. 5 个噪声变量下，NRSR 平均 EEN 为 `425K`，比多数 baseline 少很多；GP-Meld 达到 `1.63M`。
3. 10 个噪声变量下，NRSR 仍为 `89.1%` RR，而 DSR/DSP 掉到 `23.2%`，HESL 为 `26.4%`，GP-Meld 为 `35.1%`。
4. 10 个噪声变量下，DGSR 保持 `68.5%` RR，但作者认为代价是需要针对不同输入维度训练多个 pretrained models。
5. clean data 下，NRSR 的 RR/EEN 为 `89.7%/408K`，优于 HESL 的 `88.3%/441K`。
6. clean data 下 Eureqa 的平均 NMSE 最低，为 `1.97e-3`，但 RR 只有 `67.4%`，说明低误差不等同于精确恢复表达式。
7. 消融显示 NGM 是高噪声 setting 的关键：10-noise 下去掉 NGM 后 RR 从 `89.1%` 跌到 `32.9%`。
8. 同时去掉 NGM/MPE/PPO 后，10-noise RR 进一步为 `28.3%`，EEN 上升到 `1.62M`。
9. clean data 下去掉 MPE，RR 从 `89.7%` 降到 `85.3%`，但 NMSE 反而从 `7.66e-3` 到 `5.98e-3`，说明 MPE 更偏向精确恢复率和探索，而不一定最小化平均误差。
10. clean data 下去掉 PPO，RR 降到 `86.7%`，EEN 升到 `462K`。
11. NGM gate accuracy 在 3/5/10 noisy variables 下均为 `98.92%`，20 noisy variables 下为 `96.77%`。
12. 只取 final epoch gate 的 accuracy 为 `60.21%`，远低于 averaged gate + Otsu，说明跨 epoch 平均是重要工程细节。
13. Otsu threshold scale 的最优设置是 `1.05`；`1.0` 为 `95.70%`，`1.1` 为 `94.62%`。
14. L0 gating loss `lambda = 0.25` 最好，`0.1` 和 `0.5` 都是 `96.77%`。
15. MPE 的 EER 在 Nguyen-5 上为 `0.713`，高于 DSR `0.634`、HESL `0.660`、GP-Meld `0.056`、DGSR `0.550`。
16. Nguyen-12 是硬例子，所有方法在 clean/high-noise setting 中 RR 都为 `0`。
17. 作者把 Nguyen-12 失败归因于表达式 token 序列长和 RL sparse reward。
18. 反直觉点：增加 5 到 10 个无关变量时，NRSR 平均 RR 几乎不变，EEN 也几乎不变；这正是 NGM 的主要 claim。

### 明确局限

1. 论文中的噪声是无关输入变量，不是观测噪声、标签噪声、异方差噪声或真实实验测量误差。
2. 主实验集中在 Nguyen benchmark，表达式规模和真实科学发现任务之间仍有明显距离。
3. reward/test dataset 每个 benchmark 只有 `20` 个样本，可能放大精确恢复指标，但也可能低估泛化风险。
4. 没有真实物理、化学、生物、工程数据集，也没有 Feynman/PMLB 这类更广泛 benchmark 的结果。
5. 每个 benchmark 单独训练一个 RL policy，方法还不是跨表达式的 foundation model。
6. 作者承认复杂表达式 Nguyen-12 全部失败，说明该方法没有解决长表达式 sparse reward 的根问题。
7. 论文建议未来做 distributed RL 和 LoRA/foundation model，但这些没有实验验证。
8. baseline 是否完全公平仍需复核；Eureqa 不能调参，DGSR 使用预训练策略，统一框架和商业 API 的算力/时间口径不同。
9. 论文没有给出完整置信区间；只在噪声数量影响上提到统计不显著 `p > 0.05`。
10. 表格和文字存在若干编辑问题，例如 DSR/DSP 命名不一致、clean table 中 `0100` 和缺失数值，降低一点可审计性。
11. arXiv source 写了代码 URL `https://github.com/clivesun01/nrsr`，但本次未做 GitHub 下载或外部复现检查。
12. 如果任务中的 noise 指 label noise 或真实传感器 noise，这篇只能作为 feature selection / irrelevant variable filtering 的相关工作，不能直接声称解决同一问题。

## Read by: 0616-fex (2026-06-16)

这篇对 `0616-fex` 的价值取决于课题里 `fex` 的具体含义。仅从 slug 看，不能硬判断撞车；但如果该 topic 处理的是 noisy features 下的 formula extraction、feature extraction 或 symbolic regression，那么 NRSR 是必须认真处理的近邻工作。

最核心的可借鉴点是把 feature selection 做成训练前置的可微门控模块，再把门控结果接入表达式生成器的 action mask。这比单纯在 reward 里惩罚复杂度更直接，因为它在搜索空间入口处剪掉无关变量。第二个可借鉴点是 path entropy：它把探索压力从单个 token 扩展到整条表达式路径，对 sparse-reward 的 SR 搜索有一定帮助。

撞车风险主要在两个 claim：`L0/Binary Concrete gate filters irrelevant variables before SR` 和 `path-level entropy improves RL expression exploration`。如果我们的方法也围绕这两个点，需要在实验 setting 上明显区分，例如处理 label noise/measurement noise、真实数据、跨任务 policy、长表达式恢复、或可审计 feature relevance。

这篇不完全覆盖真实噪声鲁棒性。作者定义的 high-noise 是加入无关输入变量，且主实验是 Nguyen synthetic benchmark。它没有证明在真实科学观测、分布漂移、标签误差、单位尺度混乱、缺失值或 outliers 下依然有效。对 `0616-fex` 来说，最稳妥的用法是把 NRSR 当作 irrelevant-variable symbolic regression baseline，而不是泛化为所有 noisy data SR baseline。
