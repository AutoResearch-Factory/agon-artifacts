# TRACE: Trajectory Recovery for Continuous Mechanism Evolution in Causal Representation Learning (2601.21135)

- **Year**: 2026.01 (ICML 2026 accepted)
- **Authors**: Shicheng Fan (UIC), Kun Zhang (CMU, MBZUAI), Lu Cheng (UIC)
- **Venue**: ICML 2026
- **Keywords**: causal representation learning, nonlinear ICA, mixture-of-experts, continuous mechanism transition, identifiability theory, mixing trajectory recovery, temporal CRL, regime interpolation
- **One-liner**: 把因果机制的"离散切换"推广为"simplex 上的连续轨迹"，用 MoE 学 atomic mechanism 并给出 latent 变量与混合系数的联合可辨识性理论。

## 精华提炼

### 管线/方法

TRACE 是一个两阶段框架，目标是从观测序列中恢复"因果机制随时间连续演化"的轨迹，而不是像 TDRL/NCTRL 那样只处理离散切换。

**输入 → 输出总览**：
- 输入：训练阶段是 K 个"纯域" (pure-domain) 标注轨迹集合 `{D_k}`，每个域对应一个固定的 atomic mechanism（转移矩阵 `W^(k)`）；推理阶段是一条未标注、机制连续演化的测试轨迹 `{x_t}`。
- 输出：(1) 可辨识的隐变量 `z_t`（up to 置换和逐分量单调变换）；(2) 每个时间步的混合系数 `α(t) ∈ Δ^{K-1}`（K-1 维概率单纯形上的点），刻画当前有效机制 `W(t) = Σ_k α_k(t) W^(k)` 是哪些 atomic mechanism 的凸组合。

**Stage 1（训练，MoE 表征学习）**：
1. 共享编码器 `g_φ^{-1}` 把观测 `x_t` 和滞后上下文 `x_{t-L:t-1}` 映射为高斯后验 `μ_t, σ_t`，reparameterization 采样得到 `z_t`。
2. 每个域 k 对应一个专家网络，用 normalizing-flow 形式的可逆转移函数 `f_k^{-1}` 把 `z_t` 映射到独立噪声 `ε̂_t`（即每个专家学一条转移动力学的密度模型）。
3. 训练期间用确定性 one-hot gating（域标签已知）把样本路由到对应专家，目标函数是标准序列 VAE ELBO：重构项 + 转移先验对数密度 − 后验熵。
4. 训练结束后，用每个域的验证数据计算条件期望 `μ̂^(k) = E[z_t | z_{t-L:t-1}, domain k]`，以域 0 为基线算出差分 `δμ̂^(k) = μ̂^(k) − μ̂^(0)`，拼成 basis matrix `B̂ = [δμ̂^(1), ..., δμ̂^(K-1)]`。

**Stage 2（推理，最小二乘轨迹恢复）**：
1. 对测试观测 `x_t` 用编码器后验均值得到 `z_t`。
2. 用最小二乘投影恢复混合系数：`α̂(t) = Proj_{Δ^{K-1}}( B̂^† (z_t − μ̂^(0)) )`，其中 `B̂^† = (B̂ᵀB̂)^{-1}B̂ᵀ` 是 Moore-Penrose 伪逆，`Proj` 是投影到概率单纯形。
3. 用窗口大小 `w` 做时间平滑：`ᾱ(t) = (1/(2w+1)) Σ_{s=t-w}^{t+w} α̂(s)`，理论上把误差率从逐点的 `O(1)` 改进到 `O(T^{-2/3})`（利用轨迹的有界全变差假设）。
4. 可选：用两点边界校准（affine `α^cal = a·α̂ + b`）纠正系统性 scale 失真；可选仿射适配器处理分布偏移（CORAL loss），主实验未用到。
5. 可选诊断：用双样本 KS 检验比较"纯域残差"和"过渡段残差"的分布，检验凸组合假设是否成立（含一个刻意构造违反假设的合成反例，验证协议确实能检测出违反）。

整个方法的核心几何直觉：把连续机制变化看成 simplex 上的一条轨迹，atomic mechanisms 是顶点，MoE 的每个专家对应一个顶点，Stage 2 的最小二乘投影就是把观测点往这个由顶点张成的仿射子空间上投影求重心坐标。

### 关键技术细节

- **生成模型**：`z_t = f(z_{t-L:t-1}; θ_t) + ε_t`，`x_t = g(z_t)`，`g` 可逆，`θ_t = Σ_k α_k(t)·θ^(k)`（凸组合形式，`α(t)` 在概率单纯形上）。
- **机制参数化**：`θ^(k)` 实例化为权重矩阵 `W^(k) ∈ R^{d×d}`，`f(z; W) = σ(Wz)`，`σ` 常用 LeakyReLU（negative slope 0.2）。
- **Distinguishable Mechanisms 假设**：`{W^(k)}` 两两差分线性无关，蕴含容量约束 `K ≤ d+1`（因为要求每个隐变量维度 k 上的 `(K-1)` 个行差向量在 `R^d` 中线性无关）。论文证明这个约束是充分非必要的：当同时激活的机制数 `K_active` 远小于 `K_total` 时（结构稀疏扰动），可以突破 `K_total ≤ d+1`（K_total=50, d=8 时 Weight Corr 仍达 0.925~0.989）。
- **两个核心定理**：
  - Theorem (Pointwise Recovery)：`‖α̂(t) − α(t)‖ ≤ (1/σ_min)(‖ε̂_t‖ + δ_approx)`，`σ_min` 是 basis matrix 最小奇异值，`δ_approx = O(ε²)` 是一阶泰勒近似残差。
  - Theorem (Smooth Trajectory Recovery)：在有界全变差 `TV(α*) ≤ V` 和亚高斯噪声假设下，用正则化估计器（数据项+相邻时刻平滑惩罚），最优正则化系数 `λ* ≍ T^{1/3}` 时 MSE 达到 `O((V/T)^{2/3}·σ^{2/3}(K-1)^{1/3}/σ_min^{2/3} + δ_approx²/σ_min²)`，是有界全变差信号的 minimax 最优率。
- **实验设置一览**（Table `hyperparams`/`compute`）：
  - Synthetic：`d=8`，`K_total=5~50`，40,000 条轨迹/域，3 层 MLP 编码器（隐层 128），专家为 2 层 masked autoregressive flow，Adam lr=1e-4，100 epoch，1×A100 训练 6-8 小时。
  - Modified CartPole：128×128 灰度图像，`z=(x,v,θ,ω)`，5 个域各改一条因果边，CNN 编码器，2×A100 训练 15-20 小时。
  - 真实数据：UAVDT 无人机航拍车辆转弯（`d=4, L=2`，2 个 atomic mechanism：水平/竖直移动，代理指标是速度方向比 `|v_y|/(|v_x|+|v_y|)`）；CMU MoCap 步态过渡（31 关节骨架，`p=188` 观测维，`d=8, L=3`，2 个 atomic mechanism：走/跑，代理指标是归一化髋部速度）。
- **评测指标**：MCC（隐变量可辨识性，up to 置换+逐分量变换）；Weight Correlation（真实与恢复的 `α(t)` 的 Pearson 相关，对单调变换不变）。

### 核心结果

| 场景 | TRACE | 最强 baseline (NCTRL) | 备注 |
|------|-------|----------------------|------|
| Synthetic (K_active=3) | Weight Corr 0.94±0.05, MCC 0.96±0.03 | NCTRL-soft 0.72 / MCC 0.90 | TDRL/LEAP/iVAE/PCL 无法输出过渡期权重估计 (NA) |
| CartPole (图像) | MCC 0.970, Corr 0.95 | — | 常数方差噪声下 NCTRL 的可辨识性条件失效，TRACE 仍成立 |
| UAVDT 车辆转弯 (真实数据, 26 帧) | Corr 0.960 | NCTRL 0.239 | 朴素启发式 (质心跟踪 0.911, 光流 0.902) 也明显更差 |
| CMU MoCap 步态过渡 (真实数据) | Corr 0.856±0.043 (展示样例 0.917) | NCTRL 产生 5 次伪离散跳变 | — |
| OOD 泛化 (未见过的三域过渡 0→2→4) | 全模型 (两阶段) 0.945 | 仅 Stage1 门控网络 0.313 | 证明 Stage 2 最小二乘投影是 OOD 泛化的关键，不能只靠学到的门控 |
| 数据效率 (10% 训练数据) | MCC 0.752 但 Weight Corr(K=3) 仍 0.960 | — | 轨迹级平滑对噪声的均化效果远强于逐点可辨识性 |
| 稀疏度加成 (d=8, K_total=50) | dense 扰动 0.925 → sparse 扰动 0.979，方差从 ±0.052 降到 ±0.006 | — | 稀疏扰动显著放宽有效容量约束 |

**反直觉发现**：`K_active` 从 5 增到 7 时出现急剧相变（Weight Corr 从 0.968 掉到 0.459），但同期 `W(t)` 恢复始终 >0.995——论文用这个对照实验把"机制学习失败"和"分解阶段的几何病态 (basis 列向量趋于共线)"两种失效原因彻底分离开，证明退化是推理时的投影病态，不是训练学错了。

### 明确局限

- **需要纯域标注数据和已知 K**：训练阶段假设每条轨迹有域标签、且知道 atomic mechanism 数目 K。作者做了鲁棒性实验：域污染 (mechanism-level impurity) 到 50% 时 Weight Corr 仍 ≥0.93；K 误设时轻微 over-specification（0.966 vs 0.971）代价很小，under-specification 对未见过的域完全恢复不了（0.244，"inherent information limit"）。**但完全没有"域标签"这件事本身在真实无监督场景（比如强对流因果结构演化）是不成立的**——这是这篇论文和无监督场景之间最大的 gap。
- **真实数据评测靠物理代理指标，不是真值**：车辆/步态实验都没有 ground-truth 机制标签，只能用速度方向比、髋部速度这类物理代理，且论文自己承认这些代理"conflates kinematic state with dynamic mechanism"，只能在皮尔逊相关（对单调变换不变）意义下评估，绝对尺度不可信。
- **容量瓶颈 `K ≤ d+1` 是充分非必要条件**：稀疏扰动可以放宽，但完全无约束的 `K_active ≫ d` 场景仍是 open problem（附录提出 `W(t)=h(c_t)` 连续参数化作为未来方向，尚未实现）。
- **代码状态存疑**：摘要脚注称代码已在 `github.com/shichengf/trace`，但附录 Software 段又写"代码和预训练模型将在发表后开放"——两处表述不完全一致，实际可复现性需要以发表后的仓库状态为准。
- **理论构建在高斯加性噪声 + LeakyReLU/smooth 激活假设上**：非高斯或强非线性转移函数下的可辨识性论证需要额外条件（附录有讨论但非全面覆盖）。

### 跟我们的关系

与 `0710-causal-scs`（用滑动窗口因果估计的不稳定性/失稳作为强对流预警指标）**不是正面撞车**，但有两层值得注意的方法论关联：

1. **问题设定本质不同，不能直接套用**：TRACE 要求训练阶段有"纯域"标签（哪段轨迹属于哪个 atomic mechanism 是已知的），这在强对流场景里不成立——没有人能标注"当前大气因果结构属于哪个 atomic regime"。TRACE 的 MoE + 已知域标签监督框架，与 causal-scs 完全无监督地在观测变量上跑 PCMCI+/LKIF 并且不知道也不假设离散 regime 存在，是两种根本不同的问题设定。**不构成 novelty 意义上的竞争对手**，也不会出现在 causal-scs v2 review 已经列出的最近同类 (CNM/ST-CND/NPG/Causal-Audit/Laitinen-Lahti/Ashwin-VAR-eigenvalue) 名单里，属于"估计脆弱性当信号"这个元技巧之外的另一条技术路线（identifiability-theory-driven MoE，而不是 estimator-instability-as-EWS）。
2. **方法论上可借鉴的地方，是 causal-scs 当前 Quality review 里被点名的一个缺口**：v2 review 的 Logical gaps 明确写到"尚未证明因果估计器的不稳定性优于同复杂度非因果模型的不稳定性"，本质是在问"观测到的不稳定性，是真实结构变化，还是估计器本身在特定几何条件下的病态伪影？" TRACE 的 `K_active=7` 相变实验给出了一个非常干净的正面示范：**同一套数据下，`W(t)`（机制本身）恢复始终 >0.995，但 `α(t)`（分解系数）恢复因为 basis 列向量趋于共线而崩溃**——这是一个可直接复用的诊断范式（"结构学得对，但分解病态"vs"结构真的错了"），以及 TRACE 提出的 `σ_min` "pre-deployment diagnostic"思路，都可以作为 causal-scs 下一步设计"同复杂度非因果模型脆弱性基线"或"ill-conditioning 排除检验"时的直接参考模板，而不需要重新发明这套区分逻辑。
3. TRACE 的一作 Shicheng Fan 和合作者 Kun Zhang（CMU/MBZUAI，时序因果表征学习方向的主要作者之一）在 2026 年还有两篇紧邻的工作（反引文里发现）：`2605.05524` MOSAIC（面向科学时间序列的稀疏可加可辨识因果模块发现）和 `2605.16030` Mind Dreamer（隐流形上的主动因果干预）。MOSAIC 标题看起来比 TRACE 更贴近"科学时间序列 + 模块化因果发现"，值得后续单独精读，判断是否比 TRACE 本身更接近 causal-scs 的技术路线。

## Read by: 0710-causal-scs (2026-07-11)

**撞车评估**：不撞车。TRACE 的核心贡献（MoE 学习已知 atomic mechanism + 最小二乘恢复混合轨迹的联合可辨识性理论）建立在"训练时有纯域标签、已知机制数 K"的监督设定上，而 causal-scs 是完全无监督地在滑动窗口上跑因果发现（PCMCI+/LKIF）、把估计量本身的离散度 (D) 和跳变 (J) 当作预警信号，不假设、不需要"atomic mechanism"标签。两者都关心"因果机制如何随时间演化"，但 TRACE 关心的是**如何在有监督信息下精确重建演化轨迹**，causal-scs 关心的是**能否在无监督、假设违背的真实环境下，把估计过程本身的不稳定性变成有信息量的信号**——问题目标（重建 vs. 预警）和监督设定（有标签 vs. 无标签）都不同，TRACE 不会出现在 causal-scs 的 novelty closest-prior-work 名单里，也不需要因此调整 claims 边界。

**可借鉴的工具/方法**：
1. **`W(t)` vs `α(t)` 恢复对照消融**（Table exp3_full 展示的 `K_active=7` 相变）是目前读到的最干净的"结构学习正确 vs. 分解阶段几何病态"分离实验设计，直接回应了 causal-scs 当前 v2 review 里 Logical gaps 一栏点名的缺口——"因果估计器的不稳定性是否优于同复杂度非因果模型的不稳定性"本质上就是在问同一件事：观测到的不稳定，是真信号还是病态几何伪影？TRACE 给出的对照实验模板（固定一个量做正确性金标准、另一个量做分解难度探针）可以直接借鉴到 causal-scs 的 Stage-1.5/positive-control regime 设计里。
2. **`σ_min` pre-deployment diagnostic** 和 **KS 检验 + 刻意构造违反假设的合成反例**这套"验证协议本身是否真的能检测违反"的自我证伪测试思路，可以作为 causal-scs 在做 method-internal null-percentile 校准时的参考——目前 causal-scs review 里被指出"未证明不同估计器校准后的百分位在语义上可交换/可聚合"，TRACE 展示的"用已知会违反假设的合成数据验证检验协议本身"是一种可直接复用的自证套路。
3. **不是候选方法**：TRACE 本身由于监督要求，不能作为 causal-scs 真实阶段的候选因果发现/估计方法；但其同作者后续工作 MOSAIC（`2605.05524`，面向科学时间序列的模块化因果发现）标题上更贴近，建议列入下一轮搜索关键词精读。
