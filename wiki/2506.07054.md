# Policy Gradient with Tree Search: Avoiding Local Optimas through Lookahead (2506.07054)

- **Year**: 2025.06
- **Authors**: Uri Koren, Navdeep Kumar, Uri Gadot (Technion); Giorgia Ramponi (University of Zurich); Kfir Levy, Shie Mannor (Technion)
- **Venue**: arXiv preprint
- **Keywords**: policy gradient, tree search, lookahead, local optima, stationary points, MDP diameter, sparse rewards, reinforcement learning
- **One-liner**: 用有限深度 lookahead 减少 PG 局部最优。

## 精华提炼

### 管线/方法

1. 论文处理的对象是有限 MDP：状态集 `S`、动作集 `A`、转移核 `P`、奖励 `R`、折扣因子 `gamma`、初始分布 `mu`。
2. 标准 PG 的基线更新是把策略投影到 simplex：`pi_{k+1}=proj(pi_k + eta_k d^pi(s) Q^pi(s,a))`。
3. 作者强调标准 PG 的问题不是梯度估计方差，而是 one-step greedy 方向只看当前 `Q^pi`，在低覆盖、稀疏奖励、高直径 MDP 中容易停在局部最优。
4. PGTS 的核心替换是把 `Q^pi` 换成 `T^m Q^pi`，其中 `T` 是 optimal Bellman operator。
5. 更新式为 `pi_{k+1}(.|s)=proj[pi_k(.|s)+eta_k d^{pi_k}(s)(T^m Q^{pi_k})(s,.)]`。
6. 当 `m=0` 时，`T^0` 是 identity，因此 PGTS 退化为标准 policy gradient。
7. 当 `m>0` 时，每个被访问状态的动作打分不是单步价值，而是从该动作出发做 `m` 步最优 lookahead，再接回当前策略的 `Q^pi`。
8. 展开式等价于从 `(s,a)` 出发，最大化 `m` 步内的临时策略序列 `pi_1,...,pi_m` 的累计奖励，加上深度末端的 `Q^pi` bootstrap。
9. 计算上并不需要显式枚举完整树；作者说可反复应用 Bellman operator，因此复杂度随 lookahead 深度 `m` 线性增长。
10. 算法输入是 depth `m`、步长 `eta_k`、初始策略 `pi_0`；循环中估计 `Q^{pi_k}`，再只在 `d^{pi_k}(s)>0` 的状态更新策略。
11. 这点是论文的关键设定：它不要求每轮更新全状态空间，只要求更新当前策略访问到的状态，贴近大状态空间实践。
12. 理论分析围绕 stationary policy set `Pi^m` 展开，而不是给出完整有限样本收敛率。

### 关键技术细节

13. Stationary set 定义为 `Pi^m_eta={pi | pi = proj[pi + eta D^pi T^m Q^pi]}`，`D^pi` 是以 occupancy `d^pi(s)` 为对角元素的矩阵。
14. `sigma_pi={s | d^pi(s)>0}` 表示当前策略真实访问到的状态支撑集。
15. Lemma 1 证明 stationary point 与步长无关：对任意 `eta>0`，`pi in Pi^m_eta` 当且仅当在所有 `s in sigma_pi` 上，`pi_s` 最大化 `<pi'_s,(T^m Q^pi)(s,.)>`。
16. 这个 lemma 的直觉是：投影固定点等价于当前策略已经把概率放在 update direction 的 maximizer 上。
17. Proposition 说明全局最优策略 `pi*` 属于所有深度的 stationary set，即 `pi* in Pi^m` 对所有 `m>=0` 成立。
18. Lemma 2 给出 PGTS stationary optimality condition：`pi in Pi^m` 当且仅当对所有访问状态 `s`，`v^pi(s)=max_a (T^m Q^pi)(s,a)`。
19. 对 `m=0` 的标准 PG，这个条件退化为只在访问状态上满足 `v^pi(s)=max_a Q^pi(s,a)`。
20. 这解释了局部最优来源：未访问状态完全不进入 stationarity 检验，PG 可以对外部高奖励区域“看不见”。
21. 主定理证明 `Pi^m subseteq Pi^{m-1}`，即 lookahead 深度增加时，stationary policies 集合按包含关系单调缩小。
22. 进一步，`Pi^infty = intersection_m Pi^m = argmax_pi J^pi`，无限深度下只剩全局最优策略。
23. Corollary 定义 worst local minimum `b^m=min_{pi in Pi^m} J^pi`，得到 `b^m >= b^{m-1}`。
24. 这不是说每次 PGTS 迭代都单调改进；事实上作者强调 `m>=1` 时 PGTS 可能短期降低 return。
25. PGTS 的价值在于 stationary set 变好，而不是保证每一步 objective monotonic。
26. 证明依赖 Bellman operator 的单调性、occupancy 支撑集闭包，以及 simplex projection 的固定点性质。
27. 论文没有给深度阈值 `M_epsilon` 的闭式表达；作者猜测它依赖 MDP diameter、connectivity、reward sparsity。
28. 论文也没有建立有限深度 PGTS 相比标准 PG 的收敛速度定理，只给了实验支持和开放问题。

### 核心结果

29. Ladder MDP：5 个状态、2 个动作 `left/right`，除 `s4` 执行 `right` 得到 `+1` 外奖励为 0。
30. Ladder 的初始策略是所有状态都走 `left`，初始分布集中在 `s0`。
31. 标准 PG 在这个初始策略处梯度为 0，因此停在 return 为 0 的局部最优。
32. PGTS 在 Ladder 上需要至少 `m=4` 才能从 `s0` 看到 `s4` 的奖励并收敛到全局最优；深度小于 4 时仍会被局部解困住。
33. Ladder 中高于 4 的深度没有进一步收益，作者把它解释为该 MDP 直径约为 4。
34. 如果初始状态分布是 uniform，标准 PG 也能找到最优，但信息传播慢；高深度 PGTS 更快把远端奖励传播回来。
35. Random MDP：`S=10`、`A=2`、`gamma=0.9`，转移核和奖励随机生成且连通性好、奖励密集。
36. Random MDP 中 PG 和不同深度 PGTS 表现接近，说明 lookahead 的优势主要出现在稀疏奖励或高直径结构里。
37. Tightrope MDP：4 个状态、2 个动作，终止状态 `s2` 给 `+1`，终止状态 `s3` 给 `-10`。
38. Tightrope 从 `s0` 出发时，PG 发现不了远端正奖励，保持 `left`；PGTS 能发现最优 `right/right` 路径。
39. Tightrope 的 uniform 初始分布展示一个反直觉现象：PG 保持 return 单调改进但会慢，PGTS 初期 return 下降但随后更快达到最优。
40. 这个现象支撑作者的“farsightedness”叙事：PGTS 愿意接受短期损失以进入后续高回报路径。
41. Grid MDP：`10 x 10` 共 100 状态、4 动作，动作以 `0.99` 概率按方向移动，以 `0.01` 概率随机一步转移。
42. Grid 的目标状态是 `s99`，奖励 `+10`；陷阱状态是 `s12` 和 `s32`，奖励 `-10`。
43. Grid 初始策略为各状态均匀动作；在从 `s00` 开始且学习率 `eta=0.1` 时，PG 先学会避开近处陷阱，最终变得保守并停在 `s00`。
44. Grid 中 `m>=1` 的 PGTS 先避开陷阱，再较快识别通向 `s99` 的路径。
45. 作者声称所有代码和结果在匿名仓库 `https://anonymous.4open.science/r/pgts-C77F/`，但 tex 本身没有给 artifact hash 或复现实验脚本细节。
46. 论文表格显式列出 Ladder 和 Tightrope 的 stationary policy set 随深度变化；Ladder 到 `Pi^4` 时只剩 `pi_11111`。
47. 对 Tightrope，表中 `Pi^1` 到更深深度没有继续缩小，这说明有限深度并不必然每加 1 都严格改进。
48. 最核心的可引用结果不是实验曲线，而是 `Pi^m subseteq Pi^{m-1}` 与 worst local minima 单调变好。

### 明确局限

49. 论文只在小型 tabular MDP 上做实验：Ladder、Tightrope、Random MDP、Gridworld。
50. 没有 Atari、MuJoCo、Robotics、D4RL、Procgen 等标准深度 RL benchmark。
51. 没有真机或真实环境实验；所有实验都是受控 MDP。
52. PGTS 需要模型式 Bellman lookahead 或至少可估计的 `T^m Q`，论文没有实现 model-free deep PGTS。
53. 作者明确把 sample-based PGTS compatible with deep neural networks 留作 future work。
54. 作者也明确把 convergence rate 分析留作 future work。
55. 实验主要展示曲线，没有给置信区间、随机种子统计、显著性检验或 wall-clock/sample cost 对比。
56. 对 `m` 的计算成本只说线性随深度增长，没有系统报告深度增加后的实际 runtime 或 sample complexity。
57. 理论证明假设精确 MDP 与精确 Bellman operator；在函数逼近、估计误差、off-policy sampling 下是否仍保留 stationary set 单调性质未证明。
58. 论文把 MCTS、PPO、TRPO 称为不能逃离这些局部 optima，但没有在相同 setting 下系统跑这些算法。
59. 文中英文有若干 typo 和表述不严谨处，例如 “local optimas”，不影响主线但降低成稿成熟度。
60. 对 0616-fex 而言，若课题目标是可部署的大模型/agent 系统，这篇目前更像理论机制和 toy evidence，不是可直接替换现有系统的工程模块。

### 跟我们的关系

61. 对 `0616-fex`，这篇的撞车风险取决于我们是否也声称“用 lookahead/tree search 修复 policy gradient 的局部最优”。
62. 如果我们的 claim 是对 PG/PPO/TRPO 加前瞻搜索或 planning 来改善稀疏奖励探索，这篇属于直接相关，需要引用并区分。
63. 这篇最强 claim 是 stationary set 随 lookahead 深度单调收缩；不要重复包装成新理论贡献。
64. 可借用的方法工具是 `T^m Q^pi` lookahead、只更新 visited states 的 occupancy-gated policy update、以及按 MDP 结构分析 depth 的思路。
65. 可借用的实验模板是 Ladder、Tightrope、10x10 Gridworld 三类：分别测远端稀疏奖励、短期损失换长期收益、近处陷阱导致保守策略。
66. 可做差异化的方向一：从 tabular PGTS 推到 sample-based / model-free / neural policy，并处理估计误差。
67. 可做差异化的方向二：把 depth 选择和 MDP diameter、connectivity、reward sparsity 自动关联，而不是手动调 `m`。
68. 可做差异化的方向三：在真实 benchmark 上比较 PGTS-style lookahead 与 MCTS、entropy exploration、intrinsic reward、model-based RL。
69. 可做差异化的方向四：把 PGTS 的“短期 return 可下降”变成带安全约束或 risk budget 的算法，而不是无约束接受下降。
70. 该引：如果 0616-fex 涉及 RL exploration、policy optimization local optima、tree-search-enhanced gradient，都应该引用。
71. 不该过度引用：如果 0616-fex 主要是 LLM agent workflow 或非 RL 的 feature extraction，这篇只可作为“lookahead 改善局部贪心”的类比，不是核心 prior。
72. 总体判断：中等撞车。它占住了“PG + finite-depth tree search reduces local optima”的理论叙事，但没有覆盖 deep RL、sample efficiency、真实环境和自动深度选择。
