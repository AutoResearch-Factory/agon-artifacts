# Deep Symbolic Optimization: Reinforcement Learning for Symbolic Mathematics (2505.10762)

- **Year**: 2025.05
- **Authors**: Conor F. Hayes 等, Lawrence Livermore National Laboratory
- **Venue**: arXiv
- **Keywords**: symbolic regression, equation discovery, deep symbolic optimization, reinforcement learning, autoregressive models, risk-seeking policy gradient, genetic programming, large-scale pre-training
- **One-liner**: 把符号回归建模为自回归序列搜索，并用 RL/GP/预训练提升发现率。

## 精华提炼

### 管线/方法
- 论文把 Symbolic Optimization 定义为在离散 token 序列空间里寻找最大化黑盒 reward 的表达式。
- 在符号回归场景中，输入是数据集 `D=(X,y)`，输出是一个可解释的数学表达式树。
- 表达式树用预序遍历转成可变长 token 序列 `tau=<tau_1,...,tau_n>`。
- token library 包括变量、常数占位符、二元/一元算子，如加减乘除、sin、cos、exp、log、sqrt、pow 等。
- 生成器是自回归模型，文中实验使用 RNN，方法描述也允许 GPT-like autoregressive model。
- 在第 i 步，自回归模型基于前缀 token 输出 logits，再经 Softmax 采样下一个 token。
- 对树结构的状态建模不仅使用前缀，还把 parent 和 sibling token 作为 RNN 输入，缓解树遍历中相邻 token 与树邻接关系不一致的问题。
- 采样完成后，表达式被实例化并通过 reward function 评价；符号回归里常用 reward 是 `1/(1+NMSE)`。
- 常数处理采用特殊 `const(beta)` token，先采样表达式骨架，再在 reward 计算前用 BFGS 等非线性优化器拟合连续常数。
- 论文也介绍 DisCo-DSO：把离散 token 分布和条件连续浮点值分布联合建模，避免把常数优化完全交给后处理。
- 约束和先验以 logit adjustment vector 形式加入采样：`psi + psi_constraint + psi_prior`。
- prior 只改变概率偏置，不剪枝；constraint 直接把非法 token 的概率压到零或负无穷。
- 这种 in-situ 设计让领域知识在采样过程中生效，而不是采完之后再过滤非法表达式。
- 训练层面，论文把硬离散优化转成对生成分布参数 theta 的连续优化。
- 其核心 lemma 说明：若模型能把概率质量集中到任意候选表达式，最大化期望 reward 的最优分布 mode 对应离散最优解。
- 标准训练目标是 REINFORCE/VPG：最大化 `E_{tau~p_theta}[R(tau)]`。
- 作者指出符号回归的目标通常不是平均 sample 质量，而是训练预算内找到的最好表达式。
- 因此 DSO 使用 risk-seeking policy gradient，聚焦 top epsilon reward 分位数以上的样本。
- RSPG 的 baseline 是 batch 内经验 `(1-epsilon)` reward quantile `R_epsilon(theta)`。
- 梯度只对 `R(tau) >= R_epsilon` 的样本计入，形式近似 `(R(tau)-R_epsilon) * grad log p(tau|theta)`。
- 核心算法每轮执行：采样 N 个表达式、优化常数、计算 reward、取 top epsilon、加 entropy gradient、更新 theta、维护当前 best expression。
- 论文还把 Priority Queue Training 接入 DSO：维护 top-K 高 reward 表达式队列，用最大似然训练生成器模仿队列里的表达式。
- GP seeding 方案中，RNN 先采样 N 个表达式作为 GP 初始种群，GP 运行 S 代，再把 GP elite 与 RNN 样本一起用于训练 RNN。
- 这种混合让 GP 提供局部结构变异和搜索改良，RNN 负责学习更高层的采样分布。
- 大规模预训练分两路：一种是 pre-trained prior，另一种是 encoder-decoder 式合成任务预训练。
- pre-trained prior 使用数学表达式语料学习 next-token 预测，再把 prior logits 以强度 lambda 加入 DSO logits。
- lambda 可解释为 prior 的 inverse temperature，lambda 越大，prior 对采样分布影响越强。
- encoder-decoder 预训练使用 Set Transformer 编码 `(X,y)`，再由自回归 decoder 生成表达式 token。
- 预训练目标既包括 supervised learning，也包括 RL/risk-seeking 风格目标，以减小预训练和 fine-tuning 的目标错配。
- uDSO 把 AIF 递归问题简化、DSO、LSPT、GP 和 LM token 五个模块组合起来。
- LM token 表示由 basis functions 张成的函数空间；实验中 polynomial terms 最高到 3 次。
- uDSO 在线阶段先由 AIF 分解问题，再在每个子问题上用预训练 controller、RNN 学习、GP seeding 和 LM token 搜索表达式。
- 最后 AIF 把子问题解组合回根问题解。

### 关键技术细节
- Nguyen 实验中表达式长度被约束在最小 4、最大 30。
- 约束还包括：operator 的 children 不应全为常数。
- 约束还包括：一元算子的 child 不能是该算子的逆，例如 `log(exp(x))`。
- 约束还包括：三角函数的 descendant 不能再是三角函数，例如禁止 `sin(x + cos(x))`。
- DSO/PQT/VPG/RSPG/GP 在 Nguyen benchmark 上每个实验使用 2,000,000 次 expression evaluations。
- Nguyen-7 和 Nguyen-10 用作调参 benchmark。
- GP 调参网格为 800 个 hyperparameter combinations。
- RSPG、PQT、VPG 各自调参网格为 81 个 combinations。
- Nguyen benchmark 的每个表达式用 100 个 random seeds 复现实验。
- DSO RNN 实验只报告 Wolfram 的一维 benchmark，因为 Wolfram 方法不适用于高维 benchmark。
- GP hybrid ablation 在 Nguyen、R、Livermore 三组 benchmark 上运行，每项 25 次 independent runs。
- Pre-trained Prior 的语料来自约 70 GB Wikipedia dump。
- 从 Wikipedia 中抽取 798,998 个数学表达式，覆盖 41,763 个页面。
- 以 Physics category depth 3 为例，抽到 67,404 个表达式、2,265 个页面、879 个 categories。
- PP 使用 recurrent neural network，hidden layer size 为 256。
- PP 训练 200 epochs，cross-entropy loss 降到 0.367。
- PP 的 inverse temperature lambda 扫描 `0.1,0.2,...,1.0`。
- SRBench 评测包含 130 个有隐藏 analytic solution 的 ground-truth problems。
- SRBench 还包含 122 个来自 PMLB 的 real-world black-box datasets。
- uDSO 的 token library 为加、减、乘、除、sin、cos、exp、log、sqrt、1.0、const、linear/polynomial token。
- uDSO combinatorial ablation 总数为 48：AIF on/off、GP on/off、LM on/off、pre-training SL/RL/off、DSO on/off。
- uDSO ablation 运行 `48 * 130 * 10 = 62,400` 次，最多 500,000 expression evaluations per sub-problem。
- DSO-off ablation 并不是不用 controller，而是把 neural network learning rate 设为 0。

### 核心结果
- 在 Nguyen benchmark 上，RSPG 平均 recovery rate 为 83.6%。
- 同表中 PQT 为 75.2%，VPG 为 46.7%，标准 GP 为 60.1%，Eureqa 为 73.9%。
- RSPG 在 Nguyen-1/2/3/4/6/9/10/11 上达到 100% recovery。
- Nguyen-12 对所有比较方法都是 0%，说明 benchmark 中仍有 DSO 类方法难以恢复的结构。
- Nguyen-7 上 RSPG 只有 35%，PQT 41%，GP 0%，Eureqa 85%；这是一个反直觉结果，说明商业系统在某些低维对数结构上仍可能更强。
- Nguyen-8 上 RSPG 96%，但 PQT 21%、VPG 5%、GP 5%、Eureqa 0%；risk-seeking 目标对某些最佳样本驱动任务很关键。
- GP+PQT hybrid 在 Nguyen benchmark 平均 recovery rate 达到 91.4%，比 RSPG 的 83.6% 高 9.3 个百分点。
- Hybrid ablation 总体 recovery rate 约 74-75%，其中 Trainer=PQT 为 74.92%，Trainer=VPG 为 74.27%，Trainer=RSPG 为 73.95%。
- 在 hybrid 方法里 entropy weight 设为 0 仍有 73.95%，作者解释为 GP 生成样本已提供足够 exploration。
- Pre-trained Prior 把 Nguyen 平均 recovery 从 83.8% 提升到 85.0%。
- PP 把平均求解 steps 从 592.6 降到 536.7。
- PP 把平均 invalid expression rate 从 29.7% 降到 28.9%。
- PP 对 `sqrt(x)`、`sin(x)+sin(y^2)`、`x^y` 的 steps 降幅尤其明显。
- uDSO 在 SRBench ground-truth 任务上同时提升 symbolic solution rate 和 accuracy solution rate。
- uDSO 在 122 个 black-box SR problems 的 test R2 vs complexity 图上位于 Pareto frontier。
- 论文强调此前 SRBench 没有单一方法同时主导 symbolic solution、accuracy solution、Pareto efficiency，而 uDSO 在三项上都处于领先。
- uDSO ablation 的总体趋势是启用组件越多性能越好，其中 GP 和 LM 的边际收益最大。

### 明确局限
- 这篇 2505.10762 更像 DSO 系列工作的综述/章节整合，而不是单一全新实验论文。
- 很多算法细节、超参数和实现细节被指向既有工作，当前 tex 本身没有完整列出全部可复现实验配置。
- 主要实验证据集中在 symbolic regression；其他应用如 power converter、policy search、antibody optimization 更多是引用 DSO 已应用过，而非本章逐项复现实验。
- 标准 benchmark 强，但 Nguyen 系列偏合成；SRBench 更全面，不过仍不能等价于真实科学发现闭环。
- 常数优化仍是核心瓶颈：论文承认 BFGS 等非线性优化是 trial-and-error SR 系统中最昂贵的部分之一。
- DisCo-DSO 被介绍为绕开独立常数优化的一种扩展，但本章没有给它与 uDSO 主结果的统一消融。
- RSPG 的目标与最终 metric 对齐，但代价是可能牺牲平均样本质量；这对需要稳定生成多个候选而非只要 best-of-budget 的任务可能不理想。
- 反例 Nguyen-12 全部方法 0% recovery，说明简单 benchmark 中仍存在结构恢复盲区。
- 商业软件 Eureqa/Wolfram 的超参数不可控或不公开，比较公平性受限。
- 独立验证不充分：SRBench 是公开 pipeline，但实验执行和叙述主要仍由相关作者完成。
- 论文没有给出对 LLM agent 式 equation discovery 的系统比较；2026 近作已经在探索 LLM qualitative evaluator、self-reflective repair 和 parameter evaluator。
- 若用于我们的课题，不能直接把 DSO 的高 recovery rate 外推到带噪、稀疏采样、部分可观测或带物理约束的真实系统。

### 跟我们的关系
- 对 `0616-fex`，若主题是 formula/expression discovery、feature-expression search 或可解释科学建模，这篇是直接 prior。
- 撞车点 1：把表达式搜索表述为 autoregressive token policy + RL reward 的序列决策问题，这已经是 DSO/DSR 的核心 claim。
- 撞车点 2：用 top-quantile/risk-seeking objective 优化 best-of-budget 表现，不能再作为新颖主贡献单独提出。
- 撞车点 3：在采样过程中加入领域约束和先验，而不是事后过滤非法表达式，已经由 DSO 的 logit adjustment/prior/constraint 机制覆盖。
- 撞车点 4：RNN 采样 + GP inner loop + elite feedback 的 hybrid search 已由 GP seeding 线覆盖。
- 撞车点 5：用大规模数学表达式预训练提升符号搜索，已由 Wikipedia PP 和 Set Transformer encoder-decoder 预训练覆盖。
- 可以借用的工具：DSO GitHub、SRBench pipeline、Nguyen/R/Livermore benchmarks、PQT/RSPG/GP seeding、AIF decomposition、LM token。
- 可以借用的方法：把 reward gate 改成独立 verifier，而不是只用 fitting error；把 DSO 的 constraint hook 用作物理约束和语法约束注入点。
- 差异化方向 1：聚焦“生成”和“检验”分离，用独立 evaluator/critic/veto gate 解决 R2，而非只优化同一个 reward。
- 差异化方向 2：处理 2026 SAGE-Fit 指出的 Good Structure, Bad Score 问题，把常数拟合可靠性作为搜索环节的一等对象。
- 差异化方向 3：把近作 STRIDE/DoLQ 风格的 LLM qualitative evaluator 与 DSO token policy 结合，但要防止 LLM 自评替代可审计 evidence。
- 差异化方向 4：从 single best expression 扩展到带置信、证据、反事实测试和 OOD 验证的 expression portfolio。

## Read by: 0616-fex (2026-06-16)

本次按 `0616-fex` 视角审读后，结论是：若我们的课题声称“用 RL 或神经生成器搜索数学表达式/公式/可解释特征”，这篇及其 DSO 系列前作会构成直接 prior，不能忽略。尤其是 autoregressive token policy、top-quantile risk-seeking objective、in-situ constraints、GP seeding、Wikipedia/LSPT 预训练、SRBench 评测，都已经覆盖了常见的“神经符号搜索”主张。

能直接复用的是框架骨架：表达式用 tree preorder token 序列表示；采样时用语法/物理约束调 logits；reward 可以替换为我们的任务指标；RSPG 适合 best-of-budget 发现任务；PQT/GP seeding 适合把局部搜索得到的高质量候选蒸馏回生成器。若 `0616-fex` 要做可解释公式发现，最稳妥的 baseline 应包含 DSO/uDSO、PySR/GP、PhySO 或单位约束 SR、SRBench 风格 protocol。

差异化应避开“又一个 RL symbolic regression controller”。更有价值的切入点是 DSO 没有充分解决的可靠性问题：生成器和检验器不独立、常数拟合会错杀好结构、只看 best expression 而缺少表达式组合证据、真实物理/工程场景下 OOD 验证不足。2026 年 arxiv_tool 检查里，`2605.23272` 的 SAGE-Fit 已经明确攻击 Good Structure, Bad Score；`2605.17790` STRIDE 和 `2605.07323` DoLQ 代表 LLM evaluator/repair 方向；`2601.14693` EGRL-SR 代表 goal-conditioned RL 与历史轨迹利用。这些是下一轮搜索和定位 novelty 时必须纳入的近作。
