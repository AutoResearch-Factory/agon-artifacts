# Transformer Semantic Genetic Programming for d-dimensional Symbolic Regression Problems (2511.09416)

- **Year**: 2025.11
- **Authors**: Philipp Anthes, Dominik Sobania, Franz Rothlauf (Johannes Gutenberg University, Mainz)
- **Venue**: arXiv preprint (nonacm format, targeted at ACM venue)
- **Keywords**: Genetic Programming, Transformer Models, Semantic Operators, Symbolic Regression, Semantic Distance, Program Synthesis, Evolutionary Computation
- **One-liner**: 用预训练的 3.8M-parameter transformer 作为 GP 语义变异算子，在 24 个数据集上 rank 1.58 碾压 stdGP / SLIM_GSGP / DSR / DAE-GP 四个 baseline。

## 精华提炼

### 管线/方法 (Pipeline)

TSGP 的核心是两阶段架构：

**Phase 1: Model Building (离线预训练)**
1. **数据生成**: 对每个维度 d∈{2,3,4,5} 各跑 50 个合成 SR 问题上的 stdGP（pop=2000, 150K unique functions per run），收集所有函数入 archive F^(d)。用 SymPy 做语法归一化去冗余（消掉等价的语法变体，提升训练信号质量）。合成问题由带高斯噪声的线性回归生成，输入从标准正态分布采样。
2. **语义相似搜索**: 对每个 archive 内的函数，在标准化输入集 X（m=500 个随机点，zero mean unit variance）上计算语义向量 s(f) = f(X)。用 FAISS 库做 k-NN 搜索（k=3），找语义最相近的函数对。只保留 0 < SD < 100 的对（排除语义相同和极端 outlier）。总共生成 20M 个训练对（5M per dimension），是 anthes2025transformer 的 4 倍。
3. **模型训练**: Encoder-decoder transformer (Vaswani 架构)，embedding dim=128，2 个 encoder-decoder stacks = 3.8M params。输入 parent function f_i，输出 offspring f_o，额外 condition 在 semantic distance SD(f_i,f_o) 和维度 d 上。训练目标：cross-entropy loss over output tokens。AdamW lr=1e-3 + cosine scheduler，8 epochs，NVIDIA TITAN RTX ~26h。初步实验显示更大模型容量无益。

**Phase 2: Model Inference (在线搜索)**
- 用预训练 transformer 替代标准 GP 的变异算子。每个 generation：对 population 中每个 parent f_i，给定 target SD_t 和问题维度 d，auto-regressive 生成 offspring f_o。
- 语法制约 (Syntax Control, 来自 wittenberg2022using)：每个 decoding step 把非法的 token 概率设零。
- 全 population 在单 batch 内生成，100 programs 耗时 5-10s。
- 搜索中不再用 SymPy。100 generations，pop=100，tournament selection size=5，Ramped Half-and-Half 初始化，max tree depth=17。

### 关键技术细节

- **词汇表 (Vocabulary)**: Special tokens ([start], [end], [UNK], "") + 运算 (add/sub/mul/div/pow) + 变量 x1-x5 + ERC 常数 [-0.5,0.5] step 0.1 + 整数 [-5,5] step 1（SymPy 归一化带来的，如 2*3 → 6）。
- **语义定义**: 函数在标准化输入矩阵 X 上的输出向量 s(f)∈R^m。与搜索中的 evaluate 方式一致，保证 Model Building 和搜索之间的语义一致性。
- **标准化输入空间**: 所有 features standardized to zero mean unit variance。这个标准化也应用于搜索中的数据集预处理，使得预训练模型能 zero-shot transfer 到不同值域的数据集。
- **Primitive set**: {V, ERC, +, -, ×, %, pow}。pow 是因为 SymPy 把 division 表示为 exponentiation，训练数据中有 pow。虽然 SLIM_GSGP 和 DSR 不支持 pow，但在 stdGP 中加入 pow 对性能无正面影响。
- **ERC**: [-0.5, 0.5]，step 0.1。范围较小，适合标准化后的数据。
- **Baseline 配置**: stdGP（90% subtree crossover + 10% subtree mutation），SLIM_GSGP（SLIM+2 mutation, inflate rate 0.2, no crossover），DSR（negative RMSE reward, batch 100, 100 iterations），DAE-GP（LSTM autoencoder, 95% Levenshtein edit corruption, Adam lr=0.001）。
- **Double tournament selection**: 用于 TSGP 的 training data generation，平衡 accuracy 和 program size，减少 bloat。
- **SymPy 归一化的 impact**: Ablation 显示不使用 SymPy → validation accuracy 从 94.92% 降到 92.96%，validation loss 从 0.1585 升到 0.2361。搜索性能在所有 5 个测试数据集上均下降。
- **模型鲁棒性**: 只用 10% 训练数据（2M pairs）→ 性能几乎不变。只训练 1 epoch → 轻微下降。10% 数据 + 1 epoch → 仍可用。数据质量（k 选择和 SymPy）比数据量/训练时长重要得多。
- **LLM baseline 失败**: Qwen3-4B (~1B params, 1000x larger) 作为语义变异算子，即使高温度 t=2 也过早收敛，在所有 benchmark 上远差于 TSGP。说明 specialized small transformer 远胜通用 LLM。

### 核心结果

**Prediction Quality (Table 2 - Test RMSE):**
- TSGP1 (SD_t=1): 平均 rank **1.58** (±1.04)，在所有 baseline 中显著最优。
- stdGP: rank 2.71, SLIM_GSGP: rank 2.67, DSR: rank 3.54, DAE-GP: rank 4.50。
- TSGP 在 24 个数据集中的 16 个上取得 best median test RMSE。尤其在 Feynman 合成数据集上优势巨大（例如 Feynman II 34 2: TSGP 0.0881 vs SLIM 0.2014 vs stdGP 0.2253）。

**Solution Size (Table 4):**
- TSGP1: avg rank 1.58, stdGP: 1.75, SLIM_GSGP: 2.67。
- 这说明 TSGP 同时取得最好性能和最小 solution size——打破 "accuracy vs compactness" trade-off。
- TSGP 在 11/24 数据集上 size 最小，SLIM_GSGP 只在 3 个上最小。
- stdGP 在某些 Feynman 数据集上 size 极小（ESL: 7 nodes, Feynman I 18 4: 25 nodes），这些是已知的简单 ground-truth 公式，stdGP 碰巧找到了。

**SD_t 对搜索行为的影响 (关键发现):**
- SD_t 确实能有效控制语义步长：小 SD_t → 低 actual Euclidean distance → 高语义相似性 → 强 exploitation。
- SD_t=0.1: 几乎每代都有新 best（exploitation 极强），但收敛慢，最终 solution size 大（如 Pollen: 93 nodes）。
- SD_t=5: 探索性强，收敛快，solution size 小（Pollen: 42 nodes），但容易过早 plateau，最终精度可能不如小 SD_t。
- SD_t=1: 是二者之间的 robust balance，多数数据集上综合最优。
- TSGP 比 SLIM_GSGP 和 stdGP 产生语义上更稳定、更可控的变异。stdGP 的语义距离分布非常宽（特别是 75th percentile 很大），且不可控。

**收敛速度 (Figure 3):**
- TSGP1 几乎在所有数据集上收敛速度远超 stdGP 和 SLIM_GSGP。Feynman I 24 6: TSGP1 在 gen 9 就超越 stdGP 的最终性能，gen 21 超越 SLIM_GSGP 的最终性能。
- TSGP 的 IQR 比 stdGP 更小，说明 run-to-run 稳定性更高。

**反直觉/有趣的发现:**
- Analcatdata Apnea1 & 2: stdGP 反而最优，TSGP5（最大步长）比 TSGP0.1 好。这表明有些问题需要大幅度探索，细粒度语义控制反而有害。
- DSR 和 DAE-GP 两个 model-based online 方法性能居然排倒数第一第二（rank 4.50, 3.54），被 stdGP（纯语法算子）碾压。这强烈说明 online model training 在搜索中效率极低。
- Qwen3-4B 作为变异算子完全失败，说明把通用 LLM 当 GP 变异算子远不如专用小型 transformer。
- SymPy 归一化对训练至关重要——这暗示训练数据的 "clarity of signal" 比 "quantity" 更重要。

### 明确局限

**作者自己说的:**
1. 词汇表固定，只支持 d≤5 和 {add,sub,mul,div,pow}。扩展到更高维或不同 operator set 需要重新训练。
2. 训练数据偏向平滑、近线性行为（来自线性回归+噪声的合成问题），虽然对非线性和真实数据集泛化良好，但没有系统研究分布外泛化边界。
3. SD_t 在整个搜索中是固定的。Adaptive scheduling of SD_t 是未来工作方向。
4. 对需要大幅度探索的问题（如 Analcatdata Apnea），低步长语义搜索表现差。

**我读出来的局限:**
5. 没有真机验证（纯软件 benchmark，但这对 SR 领域正常）。
6. 只比较了单 parent 变异算子，没有探索 geometric crossover（双 parent）的可能。SLIM_GSGP 也没有 crossover，但这不代表 TSGP 不需要。
7. 没有跟其他 SOTA SR 方法比较：比如 Operon (Burlacu 2020)、AI-Feynman (Udrescu 2020)、PySR (Cranmer 2023)、E2E transformers (Kamienny 2022) 只在 related work 提了一下，说 one-shot transformer 在搜索中不如搜索方法，但没有做直接 benchmark。这可能是选择性避开了能打的对手。
8. 预训练代价高（26h GPU），虽然作者说 robustness ablation 显示代价可降，但没有给出具体降低后的搜索性能证据。
9. 没有开源训练数据和预训练模型的实际链接（只有 figshare URL，需要验证是否真的公开可用）。
10. 只做了 d=2-5，real-world SR 经常涉及 d>10 甚至上百维——在这个 regime 下 TSGP 不一定适用。
11. 没有和更强的 GP variant 比较，如 Geometric Semantic Crossover (Moraglio 2012) 就不在 baseline 中。

### 跟我们的关系

对 **0616-fex** 课题：

**撞车评估：**
- 不直接撞车。0616-fex 关注 function expression / 符号表达式的生成和搜索，TSGP 做的是用 transformer 改进 GP 的变异算子，属于强化 GP 搜索的工具，不直接竞争 FEX 本身的核心贡献。
- 但如果 FEX 方向涉及用深度学习辅助符号表达式搜索，TSGP 是一个重要的 prior work。

**能用什么工具/方法：**
1. **语义距离控制 (SD_t)**: 这是本论文最巧妙的设计——用一个标量参数控制 offspring 与 parent 的语义偏离程度。如果 FEX 搜索也涉及逐步改进表达式，可以借鉴这种 "step size in semantic space" 的概念。
2. **离线预训练 + 零样本 inference**: TSGP 的 offline model building 思路非常优雅——在合成数据上训练，然后 zero-shot transfer 到真实数据。FEX 的表达式生成/改进也可以考虑类似的 offline pretraining。
3. **SymPy 归一化**: 用 SymPy 消除语法冗余以提升训练信号质量——这是 engineering-wise 很实用的 trick。
4. **语法制约 (Syntax Control)**: 在 auto-regressive decoding 中强制语法正确性——如果 FEX 也用 auto-regressive 表达式生成，这个技术可以直接用。
5. **语义相似度定义的标准化**: 在标准正态分布输入上定义语义一致性——这个设计保证了预训练和搜索之间的 transfer。

**该引不该引：**
- 如果 0616-fex 的方法包含任何形式的 "用神经网络引导符号表达式搜索" 或 "语义空间导航"，TSGP 和 anthes2025transformer 都必须引用。
- 如果 0616-fex 是做纯符号推演/规则基的表达式构造，引用的必要性降低，但仍可作为 related work 中的 "learning-based symbolic regression" 分支。

**差异化在哪：**
- TSGP 的本质是改进 GP 的搜索效率，FEX 可能关注的是表达式结构的发现或表达。TSGP 仍需要 GP 框架（population, selection, generations），而 FEX 可能不依赖 evolutionary framework。
- TSGP 的 transformer 是 "how to search better"，FEX 可能更偏向 "what to find" 或 "how to represent"。

## Read by: 0616-fex (2026-06-16)

这是 TSGP 从 "fixed d=4" 扩展到 "multi-dimensional d∈{2,3,4,5}" 的 extended version，核心创新是证明单个 3.8M-param transformer 可以通过 condition on dimension d 来泛化到不同维度的问题——这对于 FEX 的多维表达式搜索有直接参考价值。

最值得借鉴的三个点：
1. **语义空间的步长控制 (SD_t)**：如果 FEX 中表达式是逐步构建/改进的，可以用类似机制控制每步修改幅度。
2. **SymPy 归一化做训练数据处理**：消除语法冗余后 transformer 学得更好——任何用神经网络合成表达式的方法都能用。
3. **小型专用模型 >> 大型通用 LLM**：Qwen3-4B 完全失败，TSGP 3.8M 参数就能做好——FEX 方向上如果考虑用神经网络，应该走小型专用路线而非调 LLM API。

不撞车：TSGP 依赖 GP 框架，而 FEX 不一定走 evolutionary 路线。TSGP 的 transformer 解决 "怎么更快找到好表达式"，FEX 解决 "怎么表达/构建/发现函数结构"——任务正交，方法可互补。
