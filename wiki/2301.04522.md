# Testing for the appropriate level of clustering in linear regression models (2301.04522)

- **Year**: 2023-01 (v2)
- **Authors**: James G. MacKinnon (Queen's University), Morten Ørregaard Nielsen (Aarhus University), Matthew D. Webb (Carleton University)
- **Venue**: arXiv (econ.EM); acknowledgments name an editor (Serena Ng) + an anonymous associate editor + three anonymous referees, and the LaTeX source contains embedded multi-round author↔referee revision comments — journal name itself not stated in the source text
- **Keywords**: cluster-robust inference, score-variance (SV) test, wild cluster bootstrap, sequential hypothesis testing, nested clustering levels, CRVE, intersection-union principle, Hausman-type specification test
- **One-liner**: 提出比较细/粗两级聚类下得分方差的 Hausman 式检验（SV test）与序贯测试程序，用于判定回归模型正确的聚类层级。

## 精华提炼

### 管线/方法

1. 设定：线性回归 `y = Xβ + u`，`N` 个观测按候选的粗聚类 `g=1..G` 划分，每个粗聚类内部再按候选的细聚类 `h=1..M_g` 划分（细聚类严格嵌套在粗聚类内，这是方法生效的结构性前提，不是简化假设）。
2. OLS 全样本估计 `β̂`，得到残差 `û`。
3. 计算每个观测的经验得分 `ŝ_ghi = X_ghi' û_ghi`，逐级求和得到细聚类得分 `ŝ_gh` 与粗聚类得分 `ŝ_g`。
4. 构造两个"三明治滤芯"方差估计：`Σ̂_f`（假设相关性只存在于细聚类内部，用细聚类得分外积求和）与 `Σ̂_c`（假设相关性延伸到整个粗聚类，用粗聚类得分外积求和）。
5. 检验 `H0: Σ_f = Σ_c`（细聚类已足够）对 `H1: Σ_f ≠ Σ_c`（需要更粗的聚类）。单系数用标量 Hausman 式统计量 `τ_σ`（渐近 N(0,1)，可单尾/双尾）；多系数联合用二次型 Wald 统计量 `τ_Σ`（渐近 χ²(k(k+1)/2)）。
6. 有限样本修正：用 wild (cluster) bootstrap 得到 bootstrap P 值，而非依赖渐近临界值——尤其当备择假设下的粗聚类数较少时，渐近检验会明显过度拒绝。
7. 多层级扩展：若存在 `p` 个从细到粗嵌套的候选聚类层级，用自下而上的序贯检验（intersection-union 原则）：先测层级 0 vs 1，若不拒绝则停在层级 0；若拒绝则测 1 vs 2，以此类推，直到第一个不被拒绝的层级。该程序被证明不会因多重测试而膨胀 family-wise 错误率。
8. 降维：当回归含大量非目标系数（如固定效应）时，先把它们从目标回归量中偏出（partial out），只对真正关心的 `k1` 个系数比较 CRVE，避免直接比较大维矩阵。

### 关键技术细节

- 方差估计 `Var̂(θ̂)` 用"平方和减去和平方"的技巧（Eq. varfast）避免三重求和：`2·Σ_g(Σ_h ŝ²_gh)² − 2·Σ_g Σ_h ŝ⁴_gh`。
- 核心假设（Assumption as:cluster）：细聚类得分在 `g` 和 `h` 两个维度上都独立——这就是 H0（细聚类已足够）的形式化定义。备择假设（as:clusteralt）只要求粗聚类得分独立，粗聚类内部允许任意相关。
- 特征值条件（as:eigen）排除退化情形：细聚类内不能完全负相关；单个细聚类的方差不能主导整个粗聚类方差之和（基本等价于要求每个粗聚类内 `M_g > 1`，即每个粗聚类至少含 2 个细聚类，否则该粗聚类对检验统计量无贡献）。
- 聚类大小条件（as:size）在"聚类内相关程度"与"聚类大小异质性"之间做权衡：相关性越强，允许的聚类越大；聚类大小越同质，允许的聚类数越少。
- 渐近理论要求粗聚类数 `G→∞`（大 G 渐近）。论文也勾勒了固定-G 渐近框架（§4.5, subsec:fixedG），但明确说明这只是"猜想"（conjecture）未经证明，且需要"极强"的逐聚类中心极限定理条件，排除常见的随机效应/因子模型。
- Wild cluster bootstrap：Rademacher 辅助变量按细聚类（`v*_gh`，H0 涉及聚类时用）或按观测（普通 wild bootstrap，检验无聚类时用）抽取；bootstrap DGP 中 `β` 设为 0，因为检验对象是方差结构本身而非 `β` 的约束。推荐 `B=999` 或更多次重抽样。
- 非嵌套结构（如 state × year 双向聚类，或本课题可能出现的"跨日期的同一天气系统"）被显式排除在已证明的理论之外——Remark rem:two-way 明确指出这需要"完全不同、技术上非平凡"的渐近技术，留作未来工作。
- 序贯检验的渐近性质（Theorem thm:seq）：`P(选出过细的层级) → 0`；若真实层级不是候选中最粗的一档，`P(选中正确层级) → 1−α`；若真实层级就是最粗一档，`P(选中正确层级) → 1`——即程序渐近上"宁可选粗不选细"。
- 与其他两种聚类层级检验的对比（§3.3）：Ibragimov & Müller 的 IM 检验（要求按每个粗聚类分别估计模型，遇到粗聚类层面不变的处理变量时不可行；固定 `G`）与 Cai (2022) 的随机化推断检验（同样单尾、固定 `G`，且要求细聚类足够大，因此不能检验"观测独立"这一最细的原假设）。三种方法都以"粗聚类数量足够多"为前提，没有一种能在 `G` 极小时给出可靠推断。

### 核心结果

- 大规模蒙特卡洛（40万次重复/组）：渐近 `τ_Σ` 检验随被联合检验的系数数 `k1` 增加而严重过度拒绝（自由度是 `O(k1²)`，与 White 信息矩阵检验的病理现象类似）；bootstrap 版本在几乎所有配置下（`G_c` 从 6 到 36，每粗聚类的细聚类数 3–12，聚类大小 25–400）都远好于渐近版本。
- 功效：细聚类数从 20 增至 40 时功效大幅提升，从 40 增至 100 时提升幅度显著变小（边际收益递减）；单尾 bootstrap 检验功效最高，equal-tail 检验功效明显偏低、不建议使用。
- 序贯检验实验（8 粗聚类、48 细聚类、2400 观测）：当真实结构是"无聚类"时，程序约 95% 选中"无聚类"、约 4.75% 误选"细聚类"、约 0.25% 误选"粗聚类"——精确复现了名义 `α` 水平，证明序贯程序不膨胀第一类错误。
- STAR 教育实验真实数据例子（3,989 个观测，330 个"班级"聚类，75 个"学校"聚类，平均每校 4.4 个班级）：SV 检验强烈拒绝"无聚类"，支持至少班级层级聚类。但班级 vs 学校（更粗）该选哪个，**取决于具体系数**——"小班效应" `β_s` 的检验结果支持用更粗的学校层级聚类，而"助教班效应" `β_a` 的检验结果支持用更细的班级层级聚类。即：正确的聚类层级不是数据集的单一固定属性，可能随被估计的目标量而变化。
- 渐近与 bootstrap P 值在"每个粗聚类内细聚类数很少"（STAR 例子中平均 4.4 个班级/校）且存在粗层级固定效应时会剧烈分歧（例：`β_a` 的班级 vs 学校检验，渐近 P=0.031 对 bootstrap P=0.344，相差十倍以上）——原因是固定效应吸收后，OLS 残差在粗聚类内部机械性负相关，扭曲渐近分布，此时应信 bootstrap 不信渐近。

### 明确局限

- 方法严格建立在线性回归 OLS 三明治/CRVE 框架之上（得分 `ŝ = X'û`），论文没有为一般 M-estimator、GMM 或黑箱模型输出（如训练好的神经网络在留出集上的排序类指标 AUPRC）提供现成扩展。
- 检验要求研究者**给定并可观测**一个具体的候选粗聚类分组变量——方法不会自动发现或搜索未知的分组结构；STAR 例子里，作者们自己也承认"班级"这个嵌套变量并非数据原生字段，是用学校 ID、教师种族、教龄、学位、职级阶梯、处理状态等多个代理变量拼出来的，且明确指出这可能偶尔把两个班误并为一个班（虽然论证了这不影响嵌套关系的有效性）。
- 非嵌套/双向聚类结构（如 state × year，或本课题里可能出现的"跨日期、跨多个风暴系统的同一天气系统"）被显式排除在已证明的渐近理论之外，反复强调需要"完全不同、技术上非平凡"的技术，留作未来工作。
- 所有理论和仿真都假设粗聚类数 `G` 不会太小：仿真实验从未低于 `G_c=3`，多数配置在 `G_c=6~8` 时渐近检验仍有明显偏差（bootstrap 好得多，但也从未在 `G_c=2` 这类极端情形下验证过）；论文自引的姊妹文献（MacKinnon & Webb 2018，"few (treated) clusters"）专门记录了粗聚类数极少（尤其伴随固定效应）时聚类稳健推断可能严重失真的问题。
- 固定-G 渐近理论（当粗聚类数量本身无法增长时最相关的框架）在本文中只是"猜想"而未证明，且依赖作者自称"极强"的假设，对真正的"极小 G"场景没有已证明的可靠后备方案。
- 没有处理"聚类归属本身不确定/需要估计"的情形——即粗聚类变量不是像"学校""州"这样干净的行政标签，而是需要额外算法或主观判断去构造（例如按气象系统聚类风暴）。

### 跟我们的关系

**核心问题的直接回答：不能直接套用来"验证" storm-cluster bootstrap 的核心假设，但能提供明确、可操作的替代建议和诚实边界。**

具体理由：

1. **统计对象不匹配**：本文的 SV 检验建立在线性回归"三明治方差"框架上，需要显式的回归得分 `ŝ_gh = X_gh'û_gh`。而 causal-scs-indicator proposal 的决定性指标是两个 CNN 在留出 storm 上的 `Delta AUPRC`（一个基于排序的、非参数、来自黑箱神经网络预测的配对差异统计量），没有闭式的回归得分表达。本文没有为这类统计量提供现成的得分/估计方程推导——要生成一个可用的 score-variance 版本，需要独立推导 AUPRC 差异统计量的影响函数/U-统计量表示（类似 DeLong 检验的思路），这是proposal 目前"最小充分干预"范围之外的新理论贡献，不应该被当作现成工具引用。
2. **嵌套结构不天然成立**：本文方法要求候选粗聚类是细聚类的严格嵌套上级（如班级⊂学校，这本身在 STAR 例子里都需要用多个代理变量拼出来）。"同一天"是干净的嵌套变量（天然划分、无歧义），但"同一天气系统"未必是——一个大尺度天气系统可能跨越多天、覆盖大范围区域并催生多个离散风暴系统，把 storm 归属到"系统"本身需要额外的追踪/聚类算法，会引入自己的不确定性，比 STAR 例子里的班级构造更棘手。
3. **粗聚类数量远不够**：本文仿真从未在粗聚类数 `G_c` 低于 3 的情形下测试，`G_c=6~8` 时渐近检验仍有可见偏差；姊妹文献专门警示"少数（处理）聚类"问题。而 proposal 当前只有 `>=2` 个季节作为潜在的"粗聚类"候选——远低于本文方法（以及 IM 检验、Cai 检验，三者都要求足够多粗聚类）已验证过的任何范围，即便理论上能推广到 AUPRC 差异统计量，`G=2` 也无法支撑任何可靠的形式化检验。

**可以拿来用的部分（务实建议）**：
- 采纳 §subsec:infreg 的做法——不追求"证明"某一层级绝对正确，而是在多个"看起来合理"的聚类层级下都报告置信区间（storm-level bootstrap CI 作为主结果 + 一个"day-block"或"season-block" bootstrap CI 作为敷衍成本很低的稳健性检查），CI 若相近则为 storm-level 选择提供非形式化的支持证据，CI 若显著变宽则是诚实的警示信号——这不需要本文的渐近理论，只需要现有 storm-cluster bootstrap 代码换一下重抽样单元。
- 采纳 STAR 例子最重要的启示：**正确的聚类层级可能因目标量而异**（β_s 要粗、β_a 要细）。这直接提示 proposal 不应笼统声称"storm system 是正确的独立重抽样单元"，而应把这一假设显式列为 Failure Modes / non-goal 里"本轮不形式化检验、留待未来工作"的一项，与 proposal 现有的诚实边界风格一致，而不是试图用本文的检验去"证明"它。
- 序贯测试的 intersection-union 逻辑本身是可迁移的**思维框架**（先测细层级是否够用，不够用再逐级变粗，且不膨胀错误率），如果未来数据量增长到有 `>=6-8` 个可信的粗聚类候选（例如按 6-8 个不同天气型/regime 分组，而非只按 2 个季节），可以考虑重新评估是否值得投入去推导 AUPRC 版本的 score-variance 检验；但这在当前 proposal 的资源和 non-goals 约束下不该现在做。

## Read by: 0710-causal-scs (2026-07-13)

本轮是 causal-scs-indicator proposal v1 review（REVISE）触发的针对性文献调研第三轮，聚焦缺口(2)：storm-cluster bootstrap 的核心假设（storm system 是否是正确的独立重抽样单元）能否被本文的聚类层级检验方法直接验证。

**结论：不能直接套用作为形式化验证工具**，原因有三层，且层层递进：(a) 本文的 score-variance 检验绑定在线性回归的三明治方差框架上，要求显式回归得分，而 proposal 的决定性指标 `Delta AUPRC` 来自两个 CNN 的留出集预测差异，没有对应的得分表达，直接套用需要一次独立的、proposal 当前 non-goals 明确排除在外的新理论推导；(b) 本文要求候选粗聚类是细聚类的严格嵌套上级且可观测，"同一天"满足这一点，但"同一天气系统"不必然满足，需要额外的、会引入自身不确定性的归属算法；(c) 即使前两点都能解决，本文仿真从未在粗聚类数低于 3（多数在 6-8 才稳定）的情形下验证过任何版本的检验，而 proposal 只有 `>=2` 个季节作为候选粗聚类，远低于这一门槛，IM 检验和 Cai 检验同样受此限制，不存在"换一个检验就能绕过"的捷径。

**对 proposal 下一版的具体建议**：不要引用 MacKinnon/Nielsen/Webb (2023) 作为 storm-cluster bootstrap 有效性的形式化证据。改为两处修补：(1) 在 Failure Modes 或 non-goals 里显式承认"storm system 是否为正确重抽样单元"本轮不做形式化检验，是留待未来工作（数据量增长到有足够多可信粗聚类候选之后）的已知限制，这与 proposal 现有的诚实边界风格一致；(2) 可选、成本很低地加一个"day-block"或"season-block" bootstrap CI 作为非形式化稳健性检查（借用本文 §subsec:infreg 的建议：在多个看似合理的聚类层级下都报告 CI），不需要本文的渐近理论，只需复用现有 bootstrap 代码换一下重抽样单元；同时应引用本文 STAR 例子的发现——正确聚类层级可能因目标量而异——作为"不能笼统声称 storm-level 普遍正确"的直接文献支持。下一轮建议精读 arXiv:2604.02000（同作者 2026 年新作，题目直指"何时能信任聚类稳健推断"）与 arXiv:2106.05503（Cai，"未知聚类"设定，概念上更贴题）。
