# Multi-Scale Finite Expression Method for PDEs with Oscillatory Solutions on Complex Domains (2510.22497)

- **Year**: 2025.10
- **Authors**: Gareth Hardwick (Purdue University), Haizhao Yang (University of Maryland College Park)
- **Venue**: arXiv
- **Keywords**: Finite Expression Method, FEX-PG, oscillatory PDE, complex domain, symbolic spectral composition, eigenvalue PDE, reinforcement learning, PDE solver
- **One-liner**: 用多尺度 FEX 解振荡 PDE、复杂孔洞域和特征值问题。

## 精华提炼

### 管线/方法

1. 论文目标是把 Finite Expression Method (FEX) 从高维 PDE/PIDE 扩展到三类更难场景：高频振荡解、带多个孔洞的复杂几何域、以及 Laplace 特征值问题。
2. 输入是一个待解 PDE、边界条件、采样点集合、固定表达式树结构、可选的真解构造函数用于 manufactured-solution benchmark。
3. 输出不是黑盒神经网络权重，而是一个由有限数学算子组成的闭式表达式，例如若干 sine/cosine/exp/polynomial 项的组合。
4. 表达式由二叉树 `T` 承载，每个节点从二元算子或一元算子集合中选择，节点序列记为 `e`。
5. 连续参数 `theta` 包括一元算子的权重、bias、输入层的频率缩放系数，以及特征值问题里的 `lambda`。
6. 搜索问题被写成组合优化：在算子序列 `e` 和参数 `theta` 上最小化 PDE residual/boundary residual 组成的 loss。
7. FEX 的外层是强化学习式 controller `chi_Phi`，它为树上每个节点产生一个 probability mass function。
8. 每轮 search 从 controller 采样一批 operator sequences，并用 epsilon-greedy 机制保留探索。
9. 每个候选表达式先做 coarse tuning：先用一阶优化器跑 `T1` 轮，再用二阶方法如 BFGS/LBFGS 跑 `T2` 轮。
10. 候选的 reward/score 采用 `S(e) = (1 + L(e))^{-1}`，其中 `L(e)` 是 coarse tuning 后的 PDE loss。
11. controller 更新采用 Deep Symbolic Regression 中的 risk-seeking policy gradient，只对 top quantile 高分样本优化。
12. 论文强调这个 objective 不惩罚低分探索样本，适合 FEX 中“只需要找到一个极好表达式”的组合搜索。
13. candidate pool 存储每个 batch 里高分的 operator sequence；pool 满后用更好候选替换最差候选。
14. search 结束后对 candidate pool 里的表达式做 fine tuning，通常用 Adam 跑更多迭代 `T4`。
15. 新的 multi-scale FEX 保留 FEX-PG 框架，但修改输入层和周期算子集合，使它能发现高频谱成分。
16. 第一项方法创新是 redesigned linear input layer：每个输入维度 `x_i` 先乘一个可学习系数 `alpha_i`，再进入 unary operator。
17. 这些 `alpha_i` 在 periodic unary operator 中直接变成频率微调参数，可把基础频率调到真实频率。
18. 原始 FEX 输入层总是把多个维度的 unary 输出相加；新输入层允许从二元算子集合中采样加法或乘法。
19. 这个变化非常关键，因为很多 PDE 真解是跨维度乘积，例如 `sin(mu x1) sin(mu x2)` 或十维 sine product。
20. 第二项方法创新是 symbolic spectral composition：把多尺度 sine/cosine 基频显式放入 unary operator set。
21. 实际 operator set 包括 `0, 1, x, x^2, x^3, x^4, exp(x), sin(x), cos(x)`。
22. 周期算子额外扩充为 `sin(3x), sin(6x), ..., sin(24x)` 和 `cos(3x), cos(6x), ..., cos(24x)`。
23. 论文报告使用接近 `pi` 的频率间隔，在 Adam + PDE loss 下是搜索规模和可调精度之间的折中。
24. 基频由 controller 作为离散 operator 选择，精确频率由 `alpha_i` 连续优化。
25. 因此方法不是直接训练一个大网络学高频，而是先离散搜索谱基，再连续微调频率。
26. 第三项方法创新是 eigenvalue formulation：把 `lambda` 作为 FEX 的可学习参数，与 eigenfunction 同时优化。
27. 为避免 `u=0, lambda=0` 的平凡解，loss 加了归一化项。
28. 作者没有直接采用 DeepMartNet 里均值约束 `((1/N) sum |u(x_i)|^p - c)^2`。
29. 他们改成 `min_i {(|u(x_i)|^p - c)^2}`，意图是只惩罚“所有点都近零”的候选，而不是强制均值接近 `c`。
30. eigenvalue loss 包括 PDE residual、边界 residual 和 normalization residual，后两项权重为 `alpha_b`、`alpha_n`。
31. eigenvalue 初始化使用 Rayleigh quotient 思路：`lambda_0 = mean |grad u(x_i)|^2 / mean u(x_i)^2`。
32. 这个初始化复用 loss 里已经需要的导数，因此作者声称不额外增加复杂度。
33. 数值实验全部是 manufactured benchmarks：给定真解，构造 RHS 和边界条件，再检验 FEX 是否恢复真解。
34. 论文比较对象主要是 DeepMartNet 和 MscaleDNN，而不是传统 FEM/FDM 的系统 benchmark。
35. 复杂几何通过域采样和边界采样进入 loss，不需要 mesh。
36. 但论文没有给出面向复杂 CAD/工程几何的通用采样器或几何处理工具链。
37. 论文没有开源代码、运行时间表、硬件配置或完整超参数表；Algorithm 里列出 `T, T1, T2, T3, T4, K, N, eta`，但正文实验只给出部分数值。
38. 100-D Poisson-Boltzmann 例子使用 50 次 search-loop iteration，fine tuning 约 2000 次后绝对相对误差到 `1e-6` 量级。
39. 10-D nonlinear Poisson-Boltzmann 例子相对 `L2` 误差是 `3.3e-6`，对比 DeepMartNet 的 `2.5e-1`。
40. 2-D 复杂孔洞 Poisson 的真解为 `sin(7pi x1) sin(7pi x2)`，测试小孔和大孔两种域。
41. 第一种 2-D 域有 3 个圆孔，中心和半径分别为 `-(0.5,-0.5)`、`(0.5,0.5)`、`(0.5,-0.5)` 与 `0.1, 0.2, 0.2`。
42. 第二种 2-D 域有 3 个圆孔和 1 个椭圆孔；圆孔半径为 `0.3, 0.6, 0.3`，椭圆为 `16(x1+0.5)^2 + 64(x2-0.5)^2 = 1`。
43. 2-D 第一域 10 次 trial 的平均绝对相对误差为 `8.3e-7`，平均相对 `L2` 误差为 `4.9e-7`。
44. 2-D 第二域绝对相对误差为 `1.6e-6`，相对 `L2` 误差为 `8.6e-7`。
45. 论文给出一个恢复表达式：`0.9999 sin(21.9911 x1) sin(21.9911 x2)`，对应真实 `7pi ≈ 21.9911`。
46. 3-D 复杂域是边长 `L=2`、中心在原点的 cube，内部有 125 个随机半径球孔，按网格排列。
47. 3-D multiplicative sine 真解 `sin(mu x1) sin(mu x2) sin(mu x3)`，其中 `mu=7pi`。
48. 该 3-D 例子每次迭代采样 5000 个 interior points 和 5000 个 boundary points，边界点约一半来自 cube 外壁，一半来自球孔表面。
49. 作者在低维 3-D 复杂域中省略 parameter grouping，用来隔离验证新输入层和频率模块本身。
50. 3-D multiplicative sine 的平均相对 `L2` 误差为 `4.1e-14`，接近 double precision machine epsilon。
51. 3-D exponential-sine 真解 `exp(sin(mu x1)+sin(mu x2)+sin(mu x3))`，仍取 `mu=7pi`。
52. 该例使用 2500 个 interior points 和 2500 个 boundary points，每轮迭代点数比前一 3-D 例子少。
53. 3-D exponential-sine 的平均相对 `L2` 误差为 `3.2e-15`，作者认为 fine tuning 几乎是 trivial，真正难点在 search。
54. 10-D Laplace eigenvalue 问题使用零边界条件，关注最小 eigenpair。
55. 真实 eigenfunction 写为 `prod_i sin(pi x_i)`，真实 eigenvalue 为 `10 pi^2`。
56. eigenvalue 实验中 search loop 跑 100 次，`alpha_b = alpha_n = 100`。
57. FEX 输出示例为 `prod_{i=1}^{10} sin(3.14168 x_i)`，`lambda = 98.69143`。
58. 真实 `10 pi^2 ≈ 98.696`，所以 eigenvalue 结果接近但没有达到前面 manufactured Poisson 的机器精度。
59. eigenvalue 例子的平均相对 `L2` 误差为 `3e-3`，对比 DeepMartNet 的 `2.5e-1`。
60. 作者承认 eigenvalue 问题的优化更难，因为 `lambda` 和 `u(x)` 参数互相耦合。
61. 作者建议未来可能为 `lambda` 和函数参数使用不同优化器或学习率。
62. 总表给出七个 benchmark：100-D PB `~10e-7`、10-D PB `3.3e-6`、2-D 小孔 `4.9e-7`、2-D 大孔 `8.6e-7`、3-D sine `4.1e-14`、3-D exp-sine `3.2e-15`、10-D eigen `3e-3`。
63. paper 的核心 claim 是：带显式频率基的符号表达式搜索能绕过普通 NN 的 spectral bias/F-principle 困难。
64. 可信部分在于所有核心实验都有明确 PDE、真解、loss、误差数字和对照论文。
65. 不足部分在于这些都是精心构造的可分离/可表达真解；FEX 的 operator set 已经包含与真解同型的 `sin/cos/exp/product` 结构。
66. 因此它证明的是“当真解在 operator grammar 内，FEX 可以高精度恢复”，不等于证明一般复杂 PDE 的鲁棒求解能力。
67. search 成本没有充分量化；作者也承认表达式搜索的 RL loop 对高维或 stiff problem 仍计算昂贵。
68. 对 widely separated frequencies，例如 `sin(10x1) sin(20x2) sin(30x3)`，作者明确说 RL 动态会不稳定。
69. 论文没有给出失败率、随机种子分布、confidence interval 或完整 ablation。
70. 与传统数值方法比较不足；文中声称 FEM/FDM 对高频和复杂几何代价高，但实验主要对比 NN solver。
71. 与我们的关系上，它是 FEX 方向的强相关近期 prior，尤其覆盖“多尺度频率发现 + 表达式树 + PDE residual”这条路线。
72. 如果 `0616-fex` 计划做振荡 PDE、孔洞域、特征值 PDE 的 FEX 方法，这篇论文高度撞车。
73. 如果 `0616-fex` 计划做更通用的 FEX 工程化、可复现 benchmark、搜索稳定性或自动 operator-set design，这篇可以作为基线而非终点。

### 关键技术细节

- 表达式结构：固定 depth-two tree，节点从二元和一元 operator set 中采样。
- 二元算子：`+`, `-`, `*`。
- 一元算子：`0`, `1`, `x`, `x^2`, `x^3`, `x^4`, `exp(x)`, `sin(x)`, `cos(x)`。
- 多尺度周期算子：`sin(3x)` 到 `sin(24x)`，步长 3；`cos(3x)` 到 `cos(24x)`，步长 3。
- 频率微调：输入层每个维度乘可学习 `alpha_i`，周期函数的实际频率为 base frequency 与 `alpha_i` 的乘积。
- controller：用 policy gradient 更新，每个树节点输出一个离散概率分布。
- 探索策略：epsilon-greedy，epsilon 越大越偏探索。
- reward：`S(e) = (1 + L(e))^{-1}`。
- coarse tuning：一阶优化器 `T1` 步 + BFGS/LBFGS 类二阶优化器 `T2` 步。
- candidate pool：固定容量 `K`，保存高分表达式进入 fine tuning。
- fine tuning：用 Adam 对 pool 里的表达式做长迭代优化。
- eigenvalue loss：PDE residual + `alpha_b` boundary residual + `alpha_n` min-normalization residual。
- eigenvalue 实验超参：`alpha_b = 100`, `alpha_n = 100`, search loop 100 iterations。
- 3-D 孔洞域实验采样：5000 interior + 5000 boundary 或 2500 interior + 2500 boundary。
- 3-D 孔洞数：125 个球孔，cube side length `L=2`。

### 核心结果

| Problem | FEX result | Baseline in cited paper | 备注 |
|---------|------------|-------------------------|------|
| 100-D Poisson-Boltzmann | `~1e-7` | `~5e-3` DeepMartNet | 50 search iterations, fine tuning 后到 `1e-6` 量级 |
| 10-D nonlinear Poisson-Boltzmann | `3.3e-6` | `2.5e-1` DeepMartNet | manufactured quadratic true solution |
| 2-D Poisson, small holes | `4.9e-7` relative `L2` | `1e-2` MscaleDNN | 真解 `sin(7pi x1)sin(7pi x2)` |
| 2-D Poisson, large holes | `8.6e-7` relative `L2` | `8e-3` MscaleDNN | 3 circles + 1 ellipse |
| 3-D Poisson, sine product | `4.1e-14` relative `L2` | `1e-2` MscaleDNN | 125 球孔，double precision |
| 3-D Poisson, exp-sine | `3.2e-15` relative `L2` | `1e-1`/table 写 `1e-0` | 接近浮点误差 |
| 10-D Laplace eigenvalue | `3e-3` relative `L2` | `2.5e-1` DeepMartNet | `lambda=98.69143` vs `10pi^2` |

- 最强结果是 3-D 复杂孔洞域中的 `1e-14`/`1e-15` 级误差。
- 最反直觉处是低维复杂几何对 FEX 几乎没有造成困难，作者认为难点主要在表达式搜索而不是边界几何。
- eigenvalue 结果明显弱于 Poisson manufactured examples，说明同时学 `u` 和 `lambda` 的耦合优化仍是瓶颈。
- paper 的比较优势来自“表达式 grammar 命中真解结构”，不是靠更大模型容量。

### 明确局限

- 没有开源代码、数据、seed、硬件和完整运行时间，复现实验需要重新实现 FEX-PG。
- 没有系统 ablation：无法分离输入层乘法、频率 operator 扩展、Rayleigh 初始化、risk-seeking controller 各自贡献。
- benchmark 多为 manufactured solutions，真解高度匹配 operator set。
- 复杂 domain 仍靠采样，不展示 CAD、mesh、signed distance function 或真实工程几何输入。
- 传统 FEM/FDM 对比缺失，主要对比 NN solver 文献中的误差数字。
- 没有报告 search failure rate；只给平均误差或示例表达式。
- 没有置信区间，2-D 第一域只说明 10 trials，其他多数实验没有完整统计。
- 对 widely separated frequencies，作者自己指出会引入 RL 动态不稳定。
- 对 Navier-Stokes、elastodynamics 等大规模工程 PDE 仍停留在 future work。
- 表达式树固定 depth-two，是否能表达更复杂多尺度局部结构没有验证。
