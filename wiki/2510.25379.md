# A Deep Learning Framework for Multi-Operator Learning: Architectures and Approximation Theory (2510.25379)

- **Year**: 2025.10
- **Authors**: Adrien Weihs (UCLA), Jingmin Sun (JHU), Zecheng Zhang (Notre Dame), Hayden Schaeffer (UCLA)
- **Venue**: arXiv
- **Keywords**: Deep Neural Networks, Approximation Theory, Neural Scaling Laws, Operator Learning, Multi-Operator Learning
- **One-liner**: 提出两种多算子学习架构(MONet/MNO)并建立了连续、可测、Lipschitz 算子的通用逼近定理和量化 scaling laws，在 5 个参数化 PDE 基准上验证了优越性能。

## 精华提炼

### 管线/方法

**问题设定**：区分两种多算子场景：
1. **Multiple Operator Learning**：$G: W \to \{G[\alpha]: U^{(\alpha)} \mapsto V^{(\alpha)}\}_{\alpha \in W}$，$\alpha$ 作为显式网络输入
2. **Learning Several Single Operators**：$\{G^{(j)}: U^{(j)} \to V^{(j)}\}_{j \in J}$，每个独立学习

**架构输入管线**：
- $\alpha$ 函数/向量 → parameter-approximation network ($l_p$ or $L_{ki}$) → scalar outputs
- $u$ 函数（如初值条件）→ 采样到 sensor points $\{u(x_l)\}$ → function-approximation network ($b_{pk}$ or $b_{ki}$) → scalar outputs
- $x$ 空间/时间坐标 → space-approximation network ($\tau_{pk}$ or $\tau_k$) → scalar outputs
- 三者通过 triple product 组合：$\sum l_p(\alpha) \cdot b_{pk}(u) \cdot \tau_{pk}(x)$

**两种架构**：
1. **MONet ($\UAPNetwork$, Def. 4)**: $\sum_{k=1}^N \sum_{i=1}^M \tau_k(x) b_{ki}(u) L_{ki}(\alpha)$，所有子网络为浅层 TW 激活网络（一层隐藏层 + 线性输出）
2. **MNO ($\ScalingNetwork$, Def. 5)**: $\sum_{p=1}^P \sum_{k=1}^{H^{(p)}} l_p(\alpha) b_{pk}(u) \tau_{pk}(x)$，子网络为 $\cF_{\rm NN}$ 类深层 ReLU 网络

**有限维 $\alpha$ 变体**：$\UAPNetworkFinite$ 适用于 $\alpha \in \mathbb{R}^p$ 情形，将 $L_{ki}(\alpha)$ 替换为标准 MLP

**理论证明管线（四层递进）**：
1. 函数逼近 (Theorem 1/Thm 6) → 泛函逼近 (Theorem 6) → 单算子逼近 (Theorem 7) → 多算子逼近 (Theorem 8)
2. 每步证明策略：将输入变量顺序分离，通过 $\eta_k$-net + partition of unity 构造有限维子空间，再在子空间上应用函数逼近

### 关键技术细节

**网络类别定义**（Def. 3）：$\cF_{\rm NN}(d_1, d_2, L, p, K, \kappa, R)$
- ReLU 前馈网络，$L$ 层，宽度 $\leq p$，非零参数 $\leq K$，参数上界 $\kappa$，输出上界 $R$
- 这为 scaling laws 的参数量化提供严格框架

**激活函数**：Tauber-Wiener (TW) 类，包括 sigmoid, tanh, Gaussian, ReLU, sine。实验使用 ReLU。

**Scaling Laws 核心结果**（Theorem 8）：
- 误差 $\varepsilon$ 与总参数量 $N_\#$ 的关系：$\varepsilon \asymp \left(\frac{\log \log N_\#}{\log \log \log N_\#}\right)^{-1/d_W}$
- $d_W$ = 参数函数 $\alpha$ 的定义域维度
- 推导链：$n_{c_W} = \mathcal{O}(\varepsilon^{-d_W})$, $n_{c_U} = \mathcal{O}(\delta^{-d_U})$, 逐层 $N_\# = P^{n_{c_W}}K_3 + N^{d_V}K_1 + H^{n_{c_U}}K_2$，取 log 两次后 Lambert W 反演

**逼近阶对效率的影响**（Remarks 12/13）：
- 原序（函数→泛函→算子）：$\varepsilon \asymp (\frac{\log N_\#}{\log\log N_\#})^{-1/((1+d_V)d_U)}$
- 逆序（泛函→函数→算子）：$\varepsilon \asymp (\frac{\log N_\#}{\log\log N_\#})^{-1/d_U}$
- 逆序更有利，因为 $1/d_U > 1/(d_U(1+d_V))$

**实验设定**：
- PDE 求解：$u_{\text{pred}}(t, x_{\text{spatial}})$ for $(t,x) \in (0,2] \times [0,2]$
- 5 个参数化 PDE：Conservation Laws, Diffusion-Reaction-Advection, Nonlinear Klein-Gordon, Parametric Diffusion-Reaction, Parametric Wave Equation
- 初值条件：PDEBench-style sinusoidal ($u_0(x) = \sum A_i \sin(k_i x + \phi_i)$, $n_{c_U}=64$ sensor points)
- 训练：AdamW, lr=$10^{-4}$, cosine scheduler, weight decay $10^{-4}$, 50 epochs × 2K steps, batch size 150 data × 5 tasks
- 硬件：2× RTX 4090 24GB
- 评测指标：平均相对 $\ell_2$ 误差，$N_{\text{test}} = 80 \times 50$

**模型配置**（所有模型参数 1.15M–1.50M，除 MNO-L 16.7M）：
- MONet: P=1, M=100, N=20
- MNO-S: P=10, H=20, N=20 (1.19M)
- MNO-L: P=40, H=100, N=20 (16.7M)
- 所有 branch/trunk 深度 4-6 层

**$\Lp{2}$ 逼近定理的关键技巧**（Theorem 2）：
- 截断算子 $G_M$ → Lusin 定理保证在紧子集 $W_K$ 上连续 → $\Lp{\infty}$ UAP → 残差控制通过 $\gamma$-网络裁剪
- 正交化引理（Lemma 1）：通过 Gram-Schmidt 将 MONet 重写为 orthonormal trunk/branch 形式，使得 $\Vert \UAPNetwork \Vert_{\Lp{2}} = \Vert L(\alpha) \Vert_{\ell^2}$

| Conservation Laws (OOD) | 9.00% | 6.59% | 8.48% | 7.20% | 6.64% | **5.92%** |
| Diffusion-Reaction-Advection (IN) | 13.63% | 4.91% | 3.95% | 3.80% | 3.39% | **2.51%** |
| Diffusion-Reaction-Advection (OOD) | 15.10% | 7.07% | 7.06% | 6.21% | 5.47% | **4.27%** |
| Nonlinear Klein-Gordon (IN) | 24.03% | 5.67% | 7.73% | 4.53% | 3.56% | **2.50%** |

| Parametric Wave | 56.37% | 9.31% | 13.66% | 6.95% | 5.72% | **4.41%** |

**Surprising/反直觉发现**：
1. DeepONet-C（简单 concatenation）在多个任务上**显著优于标准 DeepONet 和 MIONet**，挑战了"需要专门参数编码"的直觉
2. MNO-S（1.19M 参数）在所有任务上超越 MONet（1.15M），且训练时间相同(30min)，说明深层结构 > 浅层结构
3. Parametric Wave 方程上 DeepONet 误差 56.37% vs MNO-L 4.41%，差距超过一个数量级
4. 误差分布图显示 MNO/MONet 误差更均匀，而其他方法误差集中在激波/高梯度区域
5. 多算子 scaling law 严重退化（double-iterated log），但实际性能远超理论预测——作者推测因为 PDE 解空间存在低维结构

### 明确局限

**作者自己说的**：
1. Multiple operator scaling laws 仅对 Lipschitz 算子成立（Theorem 8），连续/可测算子只有 UAP 无量化律
2. Scaling laws 的退化极严重（log log / log log log），说明 general multiple operator setting 理论上极难——这是 "curse of parametric complexity" 的体现
3. MONet 没有 scaling laws——仅 UAP
4. 实验的 OOD 泛化仅在 PDE 参数空间的均匀扩展上测试，未涉及真正的分布偏移
5. 未建立 lower bounds（仅引用 lanthalerStuart 作为未来方向）
6. 未提供 generalization error / sample complexity 理论

**我读出的**：
1. **无真机/非标准 benchmark**：实验仅用 PDEBench-style 合成数据，未在真实物理数据上测试
2. **仅 1D 空间**：所有实验都是 1D+time，高维空间性能未知
3. **参数空间有限**：$\alpha$ 维度最多 5（向量）或 GP 随机场（函数型），未测试高维参数空间
4. **与 0616-fex 关系**：该论文集中在算子学习的逼近理论上，不是在特征提取方法上的贡献。其 "parameter-approximation network" 可视为一种条件特征提取器（从 PDE 参数中提取条件特征来调制 operator），但这个概念在论文中未被深化为通用特征提取框架
5. **缺乏与其他架构（FNO, Transformer）的对比**：仅对比了 DeepONet/MIONet，未涉及 FNO、GNOT、PDEformer 等

### 跟我们的关系

1. **撞不撞车**：不撞。本文做的是 operator learning 的逼近理论和架构设计，0616-fex 如果是关于特征提取方法，与其不直接竞争。但需要警惕：本文的 "parameter-approximation network" 本质上是一种 **operator-conditioned feature extraction**，若 fex 方向涉及从 input function 或 parametric input 中提取特征来指导下游预测，可能产生方法上的 overlap。

2. **能用什么工具/方法**：

3. **该引不该引**：应该引用，作为多算子学习的理论基线。但需要明确区分——本文是 operator learning theory，不是 feature extraction。

4. **差异化**：本文缺乏对"特征"本身的定义和分析。其网络组件虽然实现了隐式的特征提取，但没有显式的 representation learning 视角、没有 disentanglement 分析、没有 transfer learning 实验。这些都是 fex 课题可以发力的方向。
