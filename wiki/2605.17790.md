# STRIDE: A Self-Reflective Agent Framework for Reliable Automatic Equation Discovery (2605.17790)

- **Year**: 2026.05
- **Authors**: Jiarui Su, Songjun Tu, Bei Sun, Xiaojun Liang; Central South University, Institute of Automation CAS, Pengcheng Laboratory
- **Venue**: arXiv
- **Keywords**: symbolic regression, automatic equation discovery, LLM agent, self-reflection, mixed parameter fitting, semantic memory, OOD robustness, LSR-Synth
- **One-liner**: 用反思式多角色 agent 提升自动方程发现可靠性。

## 精华提炼

### 管线/方法

1. 输入是观测数据集 $\mathcal{D}=\{(\mathbf{x}_i,y_i)\}$、任务说明 $\mathcal{T}$、变量描述、可用算子和表达式约束。
2. STRIDE 把 LLM 方程发现拆成生成、拟合评估、批评修复、记忆更新四个角色，而不是只做一次生成加打分。
3. 第一步是 data-aware sampling: 从训练数据抽取轻量数据提示 $h$。
4. 数据提示包括均值统计、偏置趋势、奇偶模式、主导多项式项、交互项和简单非线性展开项。
5. 这些提示是软约束，不直接硬编码进搜索空间。
6. 第二个上下文来源是 semantic memory $\mathcal{B}$ 中的 elite cases。
7. 第三个上下文来源是任务说明、变量说明和可用操作符。
8. Generator agent $\mathcal{M}_{gen}$ 根据 prompt $p$ 采样一批可执行的参数化 skeleton。
9. 论文默认每个 prompt 接收 2 个 elite exemplars，并生成 4 个候选 skeleton。
10. 每个候选都包含自由数值参数，结构生成和数值验证在系统中明确分离。
11. 第二步是 mixed-fitting evaluator: 先把候选表达式解析成 AST。
12. AST 分析把参数分为线性系数 $\mathbf{w}$ 和非线性或耦合参数 $\mathbf{q}$。
13. 对固定 $\mathbf{q}$，线性参数通过 ridge least squares 求解。
14. ridge 系数在 appendix 中给为 $10^{-10}$。
15. 非线性参数使用 reduced objective 的 4-start Powell search。
16. mixed fit 结果会和 multi-start BFGS fallback 比较。
17. 最终保留 NMSE 更低的一组参数 $\theta^\star$。
18. 评分函数是 accuracy-complexity objective: $W_{fit}[-\log(NMSE+\epsilon)]-W_{comp}C$。
19. 论文给定 $W_{fit}=0.7$、$W_{comp}=0.3$、$\epsilon=10^{-12}$。
20. 复杂度项包括有效参数数、敏感性和曲率。
21. 复杂度权重是 $K_{param}=1.0$、$K_{sens}=0.05$、$K_{curv}=0.01$。
22. 敏感性和曲率最多在 300 个训练点上用一阶、二阶有限差分估计。
23. 第三步是 critic-executor reflective repair。
24. 触发条件是 score 大于 0，并且 Bernoulli 试验以 $\pi_c=0.4$ 成功。
25. 触发后 critic 读取 base equation、fitted parameters、score feedback、变量和任务描述。
26. critic 的动作空间被限制为 Remove、Simplify、Add。
27. executor 把 critic 动作转成最多 4 个修订版 equation skeleton。
28. 修订候选先经过 fast screening。
29. fast screening 使用 one split、one restart、40-second timeout，并且不再调用 critic。
30. top candidate 会进入 full evaluator；如果 runner-up 相对分数差小于 0.01，也会进入 full evaluator。
31. base candidate 和 refined candidate 重新比较，只有更好的候选进入记忆。
32. 第四步是 semantic memory。
33. 记忆由 10-island buffer 组织。
34. 系统先随机选择一个 island。
35. 每个 island 内的方程按 semantic cluster 管理。
36. cluster signature 是该 cluster 中最高分 elite equation 的 score。
37. cluster 采样概率用 softmax，温度 $\tau_c=0.1$。
38. 选中 cluster 后直接取最高分方程作为 exemplar，不再偏好更短程序。
39. 插入记忆前对方程做 canonicalization。
40. 方程被编码为 TF-IDF token 向量，token 包括 operator、variable、function。
41. 两个方程的相似度用 cosine similarity。
42. 相似度超过 0.9 时归入同一 semantic cluster，否则新建 cluster。
43. 每个 cluster 只保留最高分 elite，减少重复 prompt 经验。
44. 这套机制的核心不是更大的 LLM，而是让拟合反馈、局部修复和多样性记忆在闭环中共享状态。
45. 论文默认每个数据集最多生成 2000 个候选方程。
46. data hints 每 25 epochs 注入一次。
47. 评测同时覆盖 ID 和 OOD split，强调结构能否迁移到训练区域之外。
48. 主要对比对象包括 GPlearn、PySR、DSR、uDSR、LLM-SR、LaSR、SR-LLM、SR-Scientist、DrSR。
49. LLM backbone 使用 GPT-5.1 和 Claude-3-Haiku 两套设置。
50. 论文的近似成本估计是 STRIDE 每题约 3.5M tokens、6.7h。
51. 相比纯 generation-centered loop，STRIDE 的关键 claim 是 evaluator feedback 要能影响修复和记忆，而不只是筛选。
52. 相比传统 SR，STRIDE 的关键 claim 是 LLM 负责结构先验，数值优化负责可审计拟合证据。

### 关键技术细节

- 混合拟合不是泛泛的“让 LLM 自查”，而是具体的 AST role detection 加线性/非线性参数分解。
- 对线性部分，给定非线性参数后构造 design matrix $\Phi(\mathbf{q})$，用 ridge least squares 求 $\mathbf{w}^\star(\mathbf{q})$。
- 对非线性部分，用 4-start Powell search 搜索 reduced objective $J(\mathbf{q})$。
- fallback 是 multi-start BFGS，最后取 mixed fit 与 BFGS 中 NMSE 更低者。
- score 中的复杂度不只看表达式长度，还看 effective parameter count、sensitivity、curvature。
- critic 不是自由文本无限重写，而是被 Remove/Simplify/Add 三类动作约束。
- executor 每次最多产出 4 个 refined skeleton，避免 reflection 变成另一轮无约束生成。
- semantic memory 用 TF-IDF 和 cosine similarity，而不是只按 score 或表达式长度做 library 更新。
- similarity threshold 是 0.9；每个 cluster 保留 elite representative。
- 搜索预算对齐到 LLM-based baselines，STRIDE 每个 dataset 最多 2000 candidate expressions。
- early stopping 条件是 training NMSE 低于 $10^{-13}$。
- 评测指标包括 NMSE、pointwise Acc@0.1、Acc@0.001，以及 LSR-Synth 上的 split-level Accmax@0.1。
- OOD 结果被单独报告，说明作者在意 extrapolation 而不是只在训练分布附近拟合。
- benchmark 包括四个 LLM-SR 任务：Oscillator 1、Oscillator 2、E. coli Growth/Bactgrow、Stress-Strain。
- LSR-Synth 部分选择 CRK、PO、MatSci、BPG 四个 suite 中按本地 difficulty ranking 最高的五个任务。

### 核心结果

- 在主表 ID 结果中，GPT-5.1 backbone 下 STRIDE 在 Oscillator 1 达到 Acc0.1=100.00、Acc0.001=100.00、NMSE=3.36e-18。
- 在 Oscillator 2 上，GPT-5.1 下 STRIDE 达到 Acc0.1=100.00、Acc0.001=100.00、NMSE=1.12e-21。
- 在 E. coli Growth 上，GPT-5.1 下 STRIDE 的 Acc0.1=10.44、Acc0.001=0.16、NMSE=5.84e-3，是相对改善但绝对成功率仍不高。
- 在 Stress-Strain 上，GPT-5.1 下 STRIDE 的 Acc0.1=88.28、NMSE=1.82e-2，Acc0.001 不是最佳。
- Claude-3-Haiku 下 STRIDE 在 Oscillator 1/2 仍接近满分，说明收益不完全依赖单一 backbone。
- LSR-Synth ID 中，GPT-5.1 下 STRIDE 在 CRK、MatSci 达到 100.00 Acc0.1 和 100.00 Accmax0.1。
- LSR-Synth ID 中，BPG 上 GPT-5.1 STRIDE Acc0.1=91.88、Accmax0.1=80.00、NMSE=4.16e-7。
- LSR-Synth ID 中，PO 是弱项，GPT-5.1 STRIDE Acc0.1=75.68、Accmax0.1=0.00，但 NMSE=4.17e-4。
- OOD oscillator 结果更能体现 claim: GPT-5.1 下 Oscillator 1 OOD NMSE=5.97e-12，Oscillator 2 OOD NMSE=4.48e-21。
- OOD E. coli Growth 仍然难，GPT-5.1 STRIDE Acc0.1=4.58、Acc0.001=0.05、NMSE=1.41e-1。
- OOD Stress-Strain 上 GPT-5.1 STRIDE Acc0.1=93.77、NMSE=4.79e-2。
- OOD LSR-Synth 中，CRK 和 BPG 改善最明显；GPT-5.1 下 CRK OOD Acc0.1=100.00、Accmax0.1=100.00、NMSE=1.46e-3。
- BPG OOD 中 GPT-5.1 STRIDE Acc0.1=80.00、Accmax0.1=80.00、NMSE=5.09e-2；Claude-3-Haiku 下 BPG OOD Acc0.1=96.60、NMSE=2.46e-4。
- 消融显示 mixed fitting 是最关键部件之一：Oscillator 1 OOD 去掉 mixed fit 后 NMSE 从 5.97e-12 变成 4.66e-6，Acc0.001 从 100.00 降到 69.61。
- Oscillator 2 OOD 去掉 mixed fit 后 NMSE 从 4.48e-21 变成 1.85e-4，Acc0.001 从 100.00 降到 58.67。
- 去掉 critic-executor 也降低严格精度：Oscillator 1 OOD Acc0.001 从 100.00 降到 89.75。
- 去掉 semantic memory 后 Oscillator 1 OOD Acc0.001 降到 84.87，说明只按分数保留记忆会损失结构多样性。
- 成本表给 STRIDE 的相对 OOD Acc0.1 为 3.26x，relative NMSE 为 1x，即相对基线最优。
- 一个反直觉点：更复杂的 agent 工作流并没有只带来 token 增长，论文声称其收敛更稳定，搜索效率更好。
- 另一个反直觉点：semantic memory 不追求最短表达式，而保留 cluster elite，避免短表达式偏置吞掉复杂但正确的结构。

### 明确局限

- 作者承认 AST-based parameter role identification 是 heuristic；复杂耦合表达式可能分错线性/非线性角色。
- BFGS fallback 缓解但不能消除这个问题，因为 fallback 本身仍受初始化和病态 landscape 影响。
- semantic memory 依赖 surface-form canonicalization 和 TF-IDF；代数等价但写法不同的方程可能被分到不同 cluster。
- 实验主要是 controlled benchmark，不是真实科学发现闭环中的新实验或真机/湿实验验证。
- E. coli Growth 的绝对 accuracy 仍低，说明复杂生物增长式子不是已经解决。
- PO suite 的 Accmax0.1 多为 0.00，严格全 split 准确恢复仍然困难。
- prompt templates 以图呈现，完整可复制文本和运行细节仍需要代码仓库核查。
- 代码 URL 在摘要中给出，但我本次没有读取 GitHub 仓库，不能确认 artifact 完整性。
- 使用 GPT-5.1 作为 backbone 的结果依赖外部闭源模型，第三方复现难度高。
- 论文没有独立实验室复现，也没有独立 evaluator 对发现方程做物理/生物/材料领域验真。
- LSR-Synth 选择“每个 suite 最难五题”的 ranking 是作者管线内设定，仍可能影响结论外推。
- 反引文目前只有 1 篇，说明社区复核还很早。
- 近半年 arXiv 检查发现 IGSR 2605.29184 也在做 LLM-driven equation search with granular feedback，是后续/并行近邻，需要在 novelty 叙述中区分。

### 跟我们的关系

- 对 `0616-fex`，如果课题目标包含 LLM/agent 自动公式或方程发现，STRIDE 是直接撞车级 prior work。
- 撞车 claim 1：把方程发现组织成多角色闭环，包括 generator、evaluator、critic/executor、memory。
- 撞车 claim 2：用拟合反馈驱动局部修复，而不是只让 LLM 自我反思。
- 撞车 claim 3：用语义多样性记忆替代只按分数或长度保留经验。
- 撞车 claim 4：在 LSR-Synth/LLM-SRBench 和 OOD split 上证明结构恢复更可靠。
- 可复用工具 1：AST-based mixed parameter fitting，尤其是线性系数闭式/岭回归加非线性 Powell 外循环。
- 可复用工具 2：score = fit reward - complexity penalty，其中 complexity 同时看有效参数、敏感性和曲率。
- 可复用工具 3：critic 动作空间约束为 Remove/Simplify/Add，避免 reflection 失控。
- 可复用工具 4：TF-IDF/cosine semantic clustering memory，保留结构多样的 elite exemplars。
- 差异化方向 1：如果我们做的是 formula extraction 或 equation discovery 的可验证闭环，应强调独立 verifier 或外部实验 gate，而不是自产自评。
- 差异化方向 2：可以把 STRIDE 的 heuristic AST role split 替换为更严格的 symbolic algebra/automatic differentiation/interval verification。
- 差异化方向 3：可以专攻 noisy real-world data、uncertainty estimation、confidence intervals 和失败披露，这正是 STRIDE 明确承认欠缺的地方。
- 差异化方向 4：近邻 IGSR 2605.29184 用 influence score 做 granular feedback 和 pruning；我们的新意要避免只说“granular feedback improves equation search”。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 角度看，STRIDE 是高重叠论文。它不只是把 LLM 接到 PySR 或 BFGS 上，而是把“生成结构、拟合参数、批评修复、记忆复用”分成可交互的闭环角色。若 `0616-fex` 想做自动方程/公式发现、formula extraction 后的可执行建模、或 LLM 生成表达式后的验证修复，这篇必须作为强基线引用。

最可借的模块是 mixed fitting evaluator。它把 free parameters 按 AST 分成线性和非线性，线性部分用 ridge least squares，非线性部分用 4-start Powell，再和 BFGS fallback 比较。这比单纯把表达式交给黑盒 optimizer 更稳定，也能解释为什么一个近似正确 skeleton 不应被差的 coefficient fit 提前淘汰。

第二个可借模块是 constrained critic-executor。critic 只允许 Remove、Simplify、Add 三类动作，executor 只产出少量候选，并且 refined candidates 要重新经过筛选和 full evaluator。这个设计适合我们做“LLM 提出假设，但必须被工具验证和修复”的版本，避免把反思写成不可审计的自然语言。

第三个可借模块是 semantic memory。STRIDE 用 TF-IDF/cosine clustering 保留结构多样的 elite 方程，而不是只保留最高分或最短式子。对 `0616-fex`，这提醒我们 memory 不应只存最终答案，还应存失败但有用的结构族、近似 skeleton、以及带验证记录的局部修复轨迹。

撞车边界也很清楚：如果我们的核心 claim 只是“LLM agent + self-reflection + symbolic regression benchmark improves OOD”，STRIDE 已经覆盖。差异化要落在更强验证：独立 verifier、有否决权的 evaluator、噪声和不确定性建模、真实领域数据、代码/日志级可审计证据，或者比 STRIDE/IGSR 更细粒度的 term-level causal attribution。

近半年 arXiv 检查还发现 IGSR 2605.29184，它用 influence-guided granular feedback 和 pruning 做 LLM-driven equation search，并在 LLM-SRBench 等设置上评测。这和 STRIDE 一起说明 2026 年的拥挤点是“粗粒度 scalar score 不够，必须把反馈变成可操作的结构修改”。`0616-fex` 后续检索关键词应围绕 `granular feedback symbolic regression`、`LLM equation discovery verifier`、`mixed coefficient estimation`、`semantic memory symbolic regression` 展开。
