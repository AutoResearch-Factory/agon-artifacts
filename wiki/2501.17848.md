# Improving Genetic Programming for Symbolic Regression with Equality Graphs (2501.17848)

- **Year**: 2025.01
- **Authors**: Fabrício Olivetti de França (Federal University of ABC), Gabriel Kronberger (University of Applied Sciences Upper Austria)
- **Venue**: arXiv; GECCO '25 proceedings metadata in source
- **Keywords**: symbolic regression, genetic programming, equality graph, equality saturation, subtree crossover, subtree mutation, duplicate detection, SRBench
- **One-liner**: 用 e-graph 避免 GP 重复搜索等价表达式。

## 精华提炼

### 管线/方法
1. 输入是一组符号回归数据点，目标是在表达式空间里搜索一个同时准确且短的数学模型。
2. 基础搜索器是 minimalist GP：ramped half-and-half 初始化种群，tournament selection 选父代。
3. 每代产生与种群规模相同数量的父代对，然后以概率 `pc` 做 subtree crossover。
4. crossover 后以概率 `pm` 做 subtree mutation。
5. 新表达式会进入 e-graph，e-graph 同时存储原表达式和通过一次 equality saturation 得到的等价形式。
6. 每次插入表达式后只执行 single step equality saturation，避免 e-graph 在多表达式历史上指数膨胀。
7. e-graph 的作用不是只做后处理简化，而是作为整个搜索历史的索引。
8. “未访问表达式”定义为：该表达式本身或其已存等价形式在历史 e-graph 中没有出现过。
9. e-graph crossover 的流程是先从 parent 1 随机选替换点。
10. 然后遍历 parent 2 的候选子树，只保留替换后会生成未访问表达式的子树集合。
11. 如果候选集合非空，从集合中随机采样一个子树完成 crossover。
12. 如果候选集合为空，或者这轮不执行 crossover，则返回未修改的 parent 1。
13. e-graph mutation 的流程是先像传统 subtree mutation 一样随机选替换点。
14. 然后用 grow 或 full 方法随机生成新子树。
15. 如果生成的新表达式已经在 e-graph 中出现，则改变新子树根节点。
16. 根节点只从同 arity 且会产生未访问表达式的符号集合中采样。
17. 如果这个集合为空，则保留当前 mutation 结果。
18. 该设计保留了 GP 的随机搜索和冗余 building block 可能性，但显式降低“重评等价表达式”的概率。
19. 单目标版本 `eggp_so` 直接用生成 offspring 替换当前 population。
20. 多目标版本 `eggp_mo` 用 Pareto front、第二层 front 加新 offspring 填充种群，目标是兼顾 exploitation 和 exploration。
21. 运行结束后，从整个历史中提取 Pareto front，而不只看最后一代。
22. 论文还维护按 fitness 和 size 排序的 evaluated-expression 数据库。
23. 该数据库可用 `O(n)` 取 Pareto front，用 `O(log m)` 插入新表达式。
24. 由于 equality saturation 会把如 `x+x+x` 合并到带参数的等价形式，搜索历史也能减少冗余参数化。

### 关键技术细节
1. 实现语言是 Haskell，基于作者的 `srtree` symbolic regression/equality saturation 库。
2. 算法名写作 `eggp`，即 e-graph genetic programming。
3. 核心数据结构是 e-graph，由 e-node、e-class、父节点表和 pattern matching 数据库组成。
4. e-class 中同组 e-node 表示等价表达式，遍历任一 e-node 都得到等价结果。
5. 插入表达式采用 bottom-up 方式：先查 terminal e-class，再把内部节点转换成 children e-class id 构成的 e-node。
6. equality saturation 每次只做一步规则扩展，而不是跑到完全饱和。
7. 论文的非终结符集合是 `+`, `-`, `*`, `divide`, `log(abs(.))`, `exp`, `sqrt(abs(.))`, `abs(x)^y`。
8. 通用 GP 参数：population size 500，generations 200，tournament size 5。
9. mutation probability 是 0.3，crossover probability 是 0.9。
10. 最大深度是 10。
11. 优化目标是 `[MSE, size]`。
12. 参数优化是 `2 x 50` steps；Operon 因不支持多重 restart，用 100 steps。
13. PySR 使用 10 个 island，所以每个 island population 是表中 population size 的十分之一。
14. `eggp` 把训练数据的三分之一作为 validation set 来算 fitness。
15. `eggp` 用剩下三分之二训练数据拟合表达式中的数值参数。
16. SRBench 组做 3-fold cross-validation，并重复 10 次，总计 30 runs。
17. real-world 组已有固定 train/test split，每个实验也跑 30 repetitions。
18. 所有算法限制为 single core，以控制多线程差异。
19. SRBench reduced set 包含 12 个数据集，训练集最多 cap 到 1000 points。
20. real-world set 包含 9 个数据集，覆盖 chemical、flow stress、friction、NASA battery、Nikuradse 等。
21. SRBench 最大表达式 size 为 50；real-world 中多数为 30，flow/NASA/Nikuradse 为 20。
22. 因 Operon 不把 terminal scale coefficient 计入 size，论文把 Operon max size 设为其他算法的约 0.67 倍。
23. baseline 包括同 backend 的 tinyGP、Operon C++ 和 PySR。
24. tinyGP 是测量“只改 subtree operators”收益的 minimalist GP 对照。

### 核心结果
1. 在 SRBench median test `R^2` rank 上，`eggp_mo` 平均 rank 2.50，优于 `eggp_so` 3.08、Operon 2.83、PySR 3.17、tinyGP 3.42。
2. 在 SRBench AUC 上，`eggp_mo` 平均 0.56，`eggp_so` 0.52，Operon 0.50，PySR 0.53，tinyGP 0.46。
3. AUC 是按达到不低于某个 `R^2` 的概率加权，论文把它当作 reliability 指标。
4. 在 SRBench 平均模型大小上，`eggp_mo` 是 28.08，`eggp_so` 30.91，Operon 48.17，PySR 38.40，tinyGP 48.81。
5. 这个 size 结果说明 e-graph 版本显著倾向于更短表达式。
6. 作者解释可能原因之一是插入 e-graph 后的 equality saturation 会产生更简化形式。
7. 另一个原因是插入过程消除了部分 redundant parameters，缓解先前工作指出的过参数化问题。
8. SRBench 中数据集 522、1028、1193 上，所有算法基本都可靠达到相似 `R^2`。
9. SRBench 中数据集 579、606、1089 上，两个 `eggp` 版本更稳定，其他算法有更低分或失败 run。
10. 数据集 557 上所有算法都有失败，失败率约 15% 到 35%。
11. real-world median `R^2` rank 上，`eggp_mo` 平均 2.22，Operon 1.89，`eggp_so` 2.89，PySR 4.78，tinyGP 3.22。
12. real-world AUC 上，`eggp_mo` 平均 0.86，Operon 0.85，`eggp_so` 0.83，PySR 0.67，tinyGP 0.79。
13. real-world 中 Operon 在 median rank 上更强，但 `eggp_mo` 的 AUC 与 Operon 基本相当。
14. flow 和 niku-2 上，即使 Operon 也有明显 misbehavior；`eggp` 在这些数据上有竞争力。
15. runtime 方面，`eggp` 和 tinyGP 大约是 Operon 的 5 到 15 倍。
16. PySR 大约是 Operon 的 5 到 25 倍，视高维或大数据集而定。
17. `eggp` 通常快于 PySR，但明显慢于 Operon。
18. 与 tinyGP 比，加入 e-graph 和 equality saturation 没有显著增加 runtime。
19. 论文最强的经验结论不是绝对 SOTA，而是简单 GP 加 e-graph operators 后能接近或超过强 baseline 的可靠性。
20. surprising 点在于：只改 subtree operators，不加 PySR/Operon 的一堆高级 mutation、island model 或专门参数包装，也能获得接近 SOTA 的表现。

### 明确局限
1. 论文使用默认或 reasonable hyperparameters，没有做全面 per-dataset tuning。
2. 作者明确说 `eggp` 有约 8 个可调超参数，实际应用中仍可能需要调参。
3. PySR 约有 30 个相关超参数，Operon 约有 20 个；若仔细调参，它们可能达到与 `eggp` 相近或更好的结果。
4. equality saturation 只跑 single step，不能保证所有等价表达式都被插入 e-graph。
5. 多步 saturation 的收益和成本是 future work。
6. e-graph 存多表达式历史时有膨胀风险，论文用 single step 是工程折中。
7. 实验强调 small expressions；大表达式、大搜索空间下是否还能保持收益没有充分证明。
8. baseline 是 single-core 限制；Operon 和 PySR 在实际多线程配置下 runtime 可能更好。
9. 论文没有把 e-graph operators 接入 PySR/Operon 的完整 operator set，只在 minimalist GP 中验证。
10. 反过来说，结果不能证明 e-graph 机制在所有 SOTA 系统里都会带来同等收益。
11. `eggp` 的 validation split 设计与其他算法设置不同，虽然合理，但也是一个实验差异源。
12. 评估以 regression benchmark 为主，没有真实科学发现闭环验证。
13. 代码和数据可用性较好，但我没有看到独立第三方复现实验。
14. 论文没有系统报告 e-graph size、memory growth、pattern matching overhead 随 evaluation count 的曲线。
15. 对 0616-fex 如果关心的是通用 agentic discovery，本文只覆盖符号回归搜索器，不覆盖 LLM 规划或自动实验设计。

### 跟我们的关系
1. 对 `0616-fex`，如果课题涉及 formula exploration、symbolic regression、科学发现中的表达式搜索，本文是直接相关工作。
2. 撞车点很具体：凡是“用搜索历史避免重复生成等价公式”的 claim，都需要引用本文并说明差异。
3. 如果我们的方法也用 e-graph/equality saturation 过滤 symbolic expression duplicates，本文是 closest prior。
4. 如果我们的方法只是做 feature extraction 或 LLM 生成候选公式，而没有 GP operator 级别的等价去重，则不完全撞车。
5. 可复用工具是 e-graph 作为“全历史表达式记忆”，而不是只在输出后简化公式。
6. 可复用实验指标是 AUC reliability plot：衡量随机算法多次运行中达到某个 `R^2` 的概率。
7. 可复用对照是 tinyGP vs e-graph GP：隔离“搜索算子去重”本身的收益。
8. 差异化方向一：把 e-graph 历史接入 LLM/agent 生成候选公式的 veto gate，而不是只改 GP subtree operators。
9. 差异化方向二：研究 e-graph memory sharing across tasks/searches，论文只提出可能性，没有完整验证。
10. 差异化方向三：补上 memory-growth、operator hit-rate、duplicate-pruning-rate 的审计曲线，论文缺这一层机制证据。
11. 差异化方向四：做 recent 2025-2026 反引文中 EGG-SR、Zobrist duplicate detection、partial parameter sharing 的横向对比。
12. 该引。即使我们的核心不是 GP，本文也提供了“等价表达式历史索引可以提高 symbolic search reliability”的强证据。
