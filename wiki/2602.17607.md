# AutoNumerics: An Autonomous, PDE-Agnostic Multi-Agent Pipeline for Scientific Computing (2602.17607)

- **Year**: 2026.02
- **Authors**: Jianda Du, Youran Sun, Haizhao Yang (Department of Mathematics & Computer Science, University of Maryland, College Park)
- **Venue**: arXiv (formatted for ICLR 2026)
- **Keywords**: multi-agent LLM, autonomous PDE solver, numerical analysis, coarse-to-fine execution, residual-based verification, classical numerical schemes, GPT-4.1 agent
- **One-liner**: 用多智能体 LLM 框架，从自然语言描述自动设计、实现、调试和验证经典数值 PDE 求解器，在 24 个 PDE 上达到比神经求解器低 6 个数量级的误差。

## 精华提炼

### 管线/方法

完整的 7 步 pipeline（图 1）：

1. **Formulator Agent**：将自然语言 PDE 描述转换为结构化规格说明，包含控制方程、边界条件、初始条件、物理参数。
2. **Planner Agent**：生成 10 个候选数值方案，覆盖不同离散化方法（有限差分 FD、谱方法 Spectral、有限体积 FV、有限元 FEM）和时间推进策略（显式 RK4、隐式 Crank-Nicolson、IMEX、ETDRK4），同时过滤违反数值稳定性和一致性原则的配置。
3. **Feature Agent**：从 PDE 问题和候选方案中提取数值特征。
4. **Selector Agent**：对候选方案打分和排序，进一步过滤不合理/非物理配置，选 top-k（k=5）送入执行。
5. **Coder Agent + Critic Agent**（粗到细执行）：
   - 粗网格阶段：低分辨率运行，Critic 修复逻辑错误（语法、shape mismatch）
   - 逻辑验证通过后提升到高分辨率网格，失败视为稳定性问题，调整时间步长
   - 超过重试上限 M 时触发 Fresh Restart：丢弃当前代码，Coder 从零生成新实现
   - 大尺度时间模拟时使用 history decimation：仅在稀疏间隔存储解快照
6. **Verification**：三指标评估 —
   - 有解析解：相对 L2 误差 e_L2 = ||u - u*|| / (||u*|| + ε)
   - 无解析解：相对 PDE 残差 e_res = ||L(u) - f|| / (||f|| + ε)
   - 隐式解析关系（如守恒律 F(u)=0）：相对隐式残差 e_impl = ||F(u)|| / (||F_ref|| + ε)
   - ε = 1e-12 防止除零
7. **Reasoning Agent**：为最佳方案生成理论分析。

关键参数：粗网格重试上限 M_coarse=4，高分辨率重试上限 M_fine=6，代码生成重试上限 2 次，每次粗/高运行最大 wall-clock 时间 120s。

### 关键技术细节

- **底层 LLM**：仅使用 GPT-4.1（单一模型，未做多模型集成）
- **Planner 产 10 个候选方案 → Selector 打分 → 取 top-5 → Coder 实现**
- **粗到细执行**将调试与稳定性验证解耦：粗网格上快速暴露逻辑 bug，高网格上的失败归结为 CFL/稳定性问题
- **Fresh Restart** 机制是一种"放弃当前路径、重新生成"的逃逸策略，防止在坏代码路径上无限循环
- **History decimation** 通过稀疏存储中间状态避免大尺度时间模拟的内存溢出
- **评分标准**：Selector 综合考虑预期精度、稳定性、计算成本
- **方案选择模式**：周期性 BC → Fourier 谱方法；Dirichlet 抛物型 → FD/FEM 隐式；Dirichlet 椭圆型 → Chebyshev 谱方法

### 核心结果

**CodePDE Benchmark（5 PDEs）nRMSE 对比（表 1）**：

| PDE | Ours | CodePDE | FNO | Best Neural |
|-----|------|---------|-----|-------------|
| Advection | 4.18e-14 | 1.01e-3 | 7.70e-3 | 2.20e-3 (UPS) |
| Burgers | 1.79e-5 | 3.15e-4 | 7.80e-3 | 7.80e-3 (FNO) |
| React-Diff | 8.98e-7 | 1.44e-1 | 1.40e-3 | 1.40e-3 (FNO) |
| CNS | 1.82e-4 | 1.53e-2 | 9.50e-2 | 4.50e-3 (UPS) |
| Darcy | 4.84e-13 | 4.88e-3 | 9.80e-3 | 9.80e-3 (FNO) |
| **Geo Mean** | **9.00e-9** | 5.08e-3 | 9.52e-3 | — |

- Ours 几何平均比 CodePDE 低 ~6 个数量级，比 FNO 低 ~6 个数量级
- 对比 ill-designed central difference baseline（advection 上 nRMSE 高达 7.05e12），展示稳定性感知方案选择的重要性

**24-PDE 全量结果亮点**：
- 19 个有解析解的问题中 11 个达到 ≤1e-6 的 L2 误差
- Poisson 2D：5.41e-16（近机器精度）
- Helmholtz 2D：3.50e-16（近机器精度）
- Euler 1D：5.21e-14
- Schrödinger 1D：5.40e-14
- 明确失败案例：Biharmonic 2D (L2=6.14e-1)、Helmholtz 5D (L2=9.8e-1)
- 端到端运行时间：18.5s 到 228.1s，大多数在 20-130s 范围

**2D Advection 完整演练**（附录）：
- Planner 生成 10 方案（Spectral×3, FD×4, FV×2, FEM×1），分数 45-90
- 执行后 Spectral ETDRK4 (med-res) 残差 8.02e-15，35.3s，被选中
- 高分方案 Spectral RK4 (high-res) 残差反而更大 (1.75e-3)，说明执行后重评估的必要性
- WENO3+RK3 方案完全发散（残差 3.18e4）

**方案选择一致性模式**（表 4）：
- 周期性 BC PDEs → Fourier 谱方法
- Dirichlet 抛物型 PDEs → FD (Crank-Nicolson/implicit) 或 FEM (IMEX)
- Dirichlet 椭圆型 PDEs → Chebyshev 谱方法
- 这种模式显示 Selector/Planner 的数值推理能力确实捕捉了 PDE 结构特性

### 明确局限

**作者自述的局限**：
1. 高维（≥5D）和高阶 PDE 上精度有限（Biharmonic 4 阶、Helmholtz 5D 失败）
2. 评测仅覆盖规则域（regular domains），未涉及复杂几何
3. 绑定单一 LLM（GPT-4.1），未做跨模型实验
4. 生成的代码缺乏形式化收敛性或稳定性保证
5. 机制理论分析较薄，主要是实验驱动的系统论文

**我们读出的额外局限**：
6. 无真机/物理实验验证——所有评测是纯数值的，无与真实物理测量对比
7. 无标准 Benchmark 对齐——自建 200-PDE benchmark 未与任何公开 benchmark 标准对齐，结果可比性有限
8. 未开源代码——R4 Reprod 严重受限
9. 只与 CodePDE（同为 LLM 方案）和神经 baseline 比较，未与成熟数值库（FEniCS, deal.II, PETSc）比较精度/效率
10. 所有 agent 基于同质模型 GPT-4.1，没有 model-diverse verification
11. 未做 ablation：去除粗到细/残差验证/Fresh Restart 的消融实验
12. 200-PDE benchmark 未公布，不知道 PDE 具体是如何抽取/构造的（潜在的 cherry-pick 空间）
13. 仅评测精度和 runtime，未评测内存使用、扩展性、鲁棒性

### 跟我们的关系

对 `0616-fex` 课题的评估：

1. **撞车评估**：低度相关，不直接撞车。AutoNumerics 是 LLM 驱动的 PDE 求解自动化系统，核心在"用 LLM 替代数值分析专家"。而 fex 课题（从仓库名 `fex-dim-lift-skeleton` 推断）关注的是特征提取/维度提升，更偏方法论的底层数值机制。两者的核心 problem 不同。

2. **可用工具/方法**：
   - **粗到细执行策略**可以借鉴：先在小规模/低分辨率上 debug 逻辑，再在高分辨率上验证精度，节省计算资源。这在任何需要在高维/大规模上运行的实验中都能用。
   - **残差自验证机制**：对无真值（ground truth）的实验场景，用残差范数作为代理质量指标，可以移植到 fex 的无监督/自监督场景。
   - **Fresh Restart 想法**：超过重试上限就丢弃重来，避免在坏路径上浪费计算。
   - **PDE-specific**：多方案生成+评分的架构（Planner→Selector）可以迁移到任何"自动选择最优配置"的场景。

3. **引用建议**：如果 fex 课题涉及任何形式的自动化科学计算、自动方案选择、或多 agent LLM 编排，应引用此工作作为相关背景。如果不涉及 LLM，则相关性较弱。

4. **差异化**：AutoNumerics 的目标是"替代数值分析专家"，fex 的目标应是"推进数值方法本身"——前者是自动化，后者是方法创新。两者目标不同，差异化空间大。

## Read by: 0616-fex (2026-06-26)

### 撞车评估
不直接撞车。AutoNumerics 的核心贡献是"用 LLM multi-agent 系统替代数值分析专家的手工调参"，属于自动化工具链。而 0616-fex 课题关注的是维度提升/特征提取的数值方法本身——目标不同，方法不同。AutoNumerics 的 pipeline 使用 "经典数值方法"（FD/FEM/Spectral）+ LLM 编排，fex 更可能关注新数值机制的发明。

### 可借鉴组件
1. **粗到细执行**（coarse-to-fine）：在小规模/lower-dim 上快速验证逻辑正确性，再提升到目标维度/分辨率做精度验证。这是通用的计算实验效率优化策略。
2. **残差自验证**（residual-based self-verification）：在无 ground truth 的场景下用 PDE 残差范数度量质量，可借鉴到 fex 的无监督/自监督评估。
3. **多候选方案生成+执行后重评估**（Planner→Selector→execute→re-rank）：generate-then-filter 的架构，对任何需要选择最优配置/参数的场景都有参考价值。
4. **Fresh Restart 逃逸机制**：避免在 suboptimal 路径上浪费重试配额。

### 注意事项
- 本文未开源代码和 benchmark，无法直接复现或对比。
- 所有实验绑定 GPT-4.1，依赖商业 API，成本不可忽略（140s/wall-clock 背后可能有大量 token 消耗）。
- fex 课题如需引用，应限定在"自动化科学计算工具链"或"multi-agent LLM 编排"的背景。
