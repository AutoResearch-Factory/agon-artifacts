# Neuro-Symbolic AI for Analytical Solutions of Differential Equations (2502.01476)

- **Year**: 2025.02
- **Authors**: Orestis Oikonomou (ETH Zurich/ETH AI Center/IBM Research), Levi Lingsch, Dana Grund, Siddhartha Mishra, Georgios Kissas
- **Venue**: ICML 2026 submission (preprint)
- **Keywords**: Neural-Symbolic AI, Compositional Learning, Context-Free Grammar, Differential Equations, Analytical Solutions, Grammar Variational Autoencoder, Topological Regularisation, Symbolic Regression
- **One-liner**: 提出 SIGS 框架，用上下文无关文法生成数学原子、GVAE 嵌入连续潜在流形、两阶段搜索（结构选择+系数精炼），无数据、无逐问题重训练地发现 PDE 解析解。

## 精华提炼

### 管线/方法

整体管线：**固定 CFG 文法 → 一次性原子语料库生成（23,682 表达式）→ GVAE/TGVAE 训练（固定流形）→ 对每个新 PDE：Ansatz 指定 → Stage I 结构搜索（潜在空间聚类+离散选择）→ Stage II 系数精炼（Adam 梯度下降）→ 输出解析表达式**。

1. **文法构建**：CFG G = {Φ, N, R, S}，51 条产生式规则，包含 sin/cos/exp/tanh/log/sqrt 等一元运算符、+-×/ 等二元运算符、变量 x/y/t、整数/小数常量。文法保证生成表达式语法正确。

2. **原子语料库**：一次性生成 23,682 个原子表达式，来源包括：(a) 物理动机模板（Laplacian 特征模、时间因子、特征相位、过渡层 tanh、局部径向基 Gauss、径向波）；(b) 随机 CFG 组合采样（~15,000 个）。经过有效性检查、规范化、去重。

3. **TGVAE 训练**：GVAE 编码器 3 层 1D Conv（52→64→128→256）→ 32 维 latent；解码器 GRU + grammar mask；~6.1M 参数；AdamW lr=3e-4；KL warmup 7000 步。TGVAE 额外加了三个几何正则项：凸包损失 L_Hull（约束在训练数据支持的区域内）、持续同调损失 L_ph（抑制小洞/碎片）、解码光滑性损失 L_smooth（惩罚二阶大变动）。

4. **Stage I 结构搜索**：对每个 Ansatz 槽位，用 k-means 聚类 tag-admissible 的 latent 编码，枚举跨槽位的 cluster 组合（~1000 种），按 PDE residual 评分，选出最优 cluster，再迭代子聚类细化直到结构 residual 降到阈值（~1e-2 到 1e-3）。

5. **Stage II 系数精炼**：固定符号结构后，用 JAX float64 + Adam + Optax + vmap 并行多起点（100-500 starts）优化数值常数，噪声尺度 η ∈ [0.1, 0.5]，直到 √R(u) < 1e-8。

### 关键技术细节

- **文法规模**：51 条产生式规则，GVAE one-hot 52 通道（含 1 padding），最大序列长度 72
- **潜在空间维度**：32
- **语料库**：23,682 个表达式 → 训练 16,578 / 验证 4,736 / 测试 2,368
- **硬件**：单张 NVIDIA RTX 5080 Laptop 16GB VRAM
- **优化器**：AdamW, lr=3e-4, weight decay=1e-5, batch size=64
- **KL schedule**：线性 warmup 到 1.0，7000 步，初始权重 0.01
- **拓扑权重**：w_Hull=0.8, w_ph=0.8, w_smooth=1e-4
- **持续同调设置**：Rips complex, max 24 points, max dim 1, scales {0.10, 0.50}
- **Hull 设置**：256 个固定方向在 32 维 latent space
- **Stage II**：JAX float64, Adam with Optax, exponential LR decay, JIT + vmap
- **Residual 公式**：R(u) = ||S(u)||² + ||u(0,x)-u₀||² + ||B[u]-g||²（三项等权）
- **评测**：相对 L² error vs 精确解（已知解情况）或 vs FEniCS P2/P4 参考解（无闭式解情况）
- **PDE 网格**：Burgers 128², Diffusion 128², Damped wave 32³, PG 100², KdV 128², SWE 64²×32, CE 64²

### 核心结果

**已知解 PDE（vs baselines 的相对 L² error %）**：

| PDE | SIGS | HD-TLGP(vanilla) | SSDE | PINNs |
|-----|------|------------------|------|-------|
| Burgers | 6.64e-14 | 35.68 | 45.62 | 6.09 |
| Diffusion | 7.16e-13 | 79.73 | 5870 | 2.56 |
| Damped Wave | 1.22e-13 | 178.77 | 1190 | 5.56 |

**未知解 PDE 近似**：Poisson-Gauss (PG-2/3/4) 达到 1–3% 相对 L² error。FEniCS reference 对比。

**耦合非线性系统**（首个能做的方法）：
- Shallow Water (2D): ρ/u/v 误差 <0.05%，3 分 23 秒
- Compressible Euler (steady 2D): ρ/u/v/p 误差 ~10%，5 分 12 秒（首次 symbolic approximation of CE）

**速度对比**（wall-clock time，秒）：
- Burgers: SIGS 11.62s vs HD-TLGP 12057s vs SSDE 393s
- Diffusion: SIGS 14.67s vs HD-TLGP 10909s vs SSDE 485s
- Damped wave: SIGS 8.95s vs HD-TLGP 2227s vs SSDE 379s

**Ablation 关键发现**：
1. 无原子（仅 primitive-level CFG 采样 50,000 次）→ 仅 133 个 admissible，最优误差 ~366%
2. 相同原子 + HD-TLGP tree search（不用 latent manifold）→ 2–423% error
3. TGVAE vs GVAE：off-training decode 效率提升 3.56%（1486→1433 次尝试/1000 valid）
4. KdV 文法缺失 sech/cosh：SIGS 自动发现等价形式 2-2tanh²(x-4t)，误差 6.6e-6
5. Burgers Ansatz over-specification：K=5 仍能抑制冗余项，误差 4.52e-5

**Surprising 发现**：KdV 实验中 SIGS 在缺失 cosh/sech 的情况下，利用 tanh 和恒等式 sech²(x)=1-tanh²(x) 自动找到了等价解。这说明框架对适度文法 misspecification 具有鲁棒性。

### 明确局限

1. **复杂工程问题可扩展性不足**：含间断、多尺度结构、湍流的 PDE 需要富含特殊函数的文法，这些函数难以分解为小原子，或产生长表达式增加搜索复杂度。作者建议与 POD 特征函数或 Neural Operators 混合。
2. **依赖 Ansatz 人工设计**：虽然文法+流形是固定的，但 Ansatz 仍反映人类专家选择。在不那么理论成熟的领域（缺少如 Burgers/Poisson 的经典理论指导），Ansatz 设计成瓶颈。作者建议用 LLM 自动构建 Ansatz。
3. **仅测试了规则域上的经典 PDE**：所有 benchmark 都是矩形/周期性域上的标准 PDE，没有不规则几何、复杂边界条件。
4. **GVAE 训练数据量小**：仅 23,682 个原子表达式，潜在空间覆盖可能不足。
5. **没有真机实验**：纯计算方法，无物理实验验证。
6. **没有开源代码**：声称代码在 github.com/oroikono/SIGS，但论文提交时未必可用。

### 跟我们的关系

**对 0616-fex (FEX: RL for Symbolic Regression) 课题**：

**不撞车，但高度相关**。FEX 做的是 symbolic regression（从数据中恢复表达式），SIGS 做的是 PDE 解析解发现（从方程中恢复表达式）。核心差异：
- 输入：FEX 输入 data；SIGS 输入 PDE equation + BC/IC
- 搜索方法：FEX 用 RL + tree search over primitives；SIGS 用 GVAE latent search over atoms
- SIGS 不依赖数据，纯方程驱动

**可用工具/方法**：
1. **Atom-level 方法**：SIGS 的核心洞见——用预生成的"原子"（component-level expressions）代替 primitive-level 搜索——可直接应用于 FEX。FEX 当前在 primitive 层面搜索，combinatorial explosion 严重。引入 atom corpus 可大幅缩小搜索空间。
2. **GVAE/TGVAE 潜在流形**：SIGS 证明了将离散符号表达式嵌入连续流形并用几何正则约束搜索区域是可行的。这对 FEX 的 RL policy 搜索空间设计有直接启发——可以用 VAE latent space 替换 FEX 的 tree-based action space。
3. **两阶段搜索（结构+系数）**：SIGS 将 symbol structure 搜索和 numerical coefficient 优化解耦——FEX 目前混在一起做。解耦后，RL agent 只需负责结构选择，系数精炼交给 gradient-based optimizer，可能大幅提升效率。
4. **CFG 文法约束**：SIGS 用 CFG 保证生成表达式的语法正确性——FEX 可以考虑加文法 mask 来减少无效 proposal。
5. **PDE residual 作为无数据 loss**：如果 FEX 扩展到 physics-informed 方向，可以用 PDE residual 替代 data-fitting loss，实现无数据 symbolic discovery。

**差异化空间**：
- FEX 可以把 SIGS 的 atom corpus + GVAE 作为 pretrained prior 集成到 RL agent 的 action space 中
- FEX 的 RL 策略可以用于改进 SIGS 的 Stage I 离散选择（当前只是简单的 cluster-based selection）
- FEX + SIGS 结合可能产生一个同时支持 data-driven 和 equation-driven 的统一 symbolic discovery 框架

SIGS 对 FEX 课题的核心启示：

**撞车评估**：不撞车。SIGS 是 equation-driven PDE 解析解发现，FEX 是 data-driven symbolic regression。任务不同但技术高度互补。

**可迁移的工具**：
1. **Atom corpus + CFG grammar**：FEX 可借鉴 SIGS 的预生成原子语料库来结构化 RL agent 的 action space，将搜索从 primitive-level 提升到 atom-level

4. **文法 mask**：CFG 保证语法正确性，减少 FEX agent 的无效 proposal
5. **PDE residual scoring**：FEX 可扩展到 physics-informed 场景，不依赖数据

**差异化**：FEX 的 RL 方法可以反过来改进 SIGS 的 Stage I cluster-based selection，用 RL policy 做更智能的 latent space 探索。两者的结合可能产生统一的 data+equation 双驱动 symbolic discovery 框架。
