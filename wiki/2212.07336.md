# BelNet: Basis Enhanced Learning, a Mesh-Free Neural Operator (2212.07336)

- **Year**: 2022.12
- **Authors**: Zecheng Zhang (CMU), Wing Tat Leung (CityU Hong Kong), Hayden Schaeffer (UCLA)
- **Venue**: arXiv (not published at a top venue in this version)
- **Keywords**: neural operator, mesh-free, discretization-invariant, operator learning, multiscale PDE, projection net, construction net, high-contrast coefficient
- **One-liner**: 提出 BelNet，一种无网格算子学习框架，输入传感器位置和输出评估点可完全不同，在多尺度高对比度 PDE 上全面优于 DeepONet 和 FNO。

## 精华提炼

### 管线/方法
BelNet 的完整 pipeline：输入函数 u 被采样到任意传感器位置 {y_j}（每样本可以不同）→ Projection Nets（K 个独立全连接子网络，每个学习一组 basis function p^k(y)，输出 K×N 矩阵 P）→ 取 Pu 内积经过激活 a_u → Construction Net（全连接网络，输入评估点 x，输出 K 维向量，即 basis q^k(x)）→ 最终输出为 [a_u(Pu)]^T · a_x(Qx + b_x) 的内积。

核心公式：G(u)(x) ≈ G_θ(u)(x) = Σ_{k=1}^K a_x((q^k)^T x + b_x^k) · a_u(û^T W_y^{2,k} (a_y(W_y^{1,k} y + b_y^k)))

输入分两条路径：(1) 传感器位置 y → Projection Nets → 得到 basis P；(2) 函数值 u(y_j) → 与 P 做内积 → 经 a_u 激活。同时评估点 x → Construction Net → 得到 basis Q。两条路径在最终内积处汇合。

### 关键技术细节

- **Architecture**: 两个核心组件。Projection Net: K 个独立子网络，每个结构 [N, N₁, N]（N 为传感器数量，N₁ 为隐藏层宽度）。Construction Net: 单个全连接网络，结构 [d, 100, ..., K]（d 为输出域维度，K 为 basis 数量，与 Projection Net 的 K 匹配）。

- **各实验超参数**:
  - Burgers: N=25, K=10, Projection Net [25, 100, 25] + Tanh, a_u 替换为 ReLU 激活的 10×10 可训练矩阵, Construction Net [2, 100, 100, 100, 10] + Tanh
  - Elliptic: N=100, K=10, Projection Net [200, 100, 100] + ReLU, 去除 a_u（因为问题对 f 线性）, Construction Net [2, 100, 10] + ReLU
  - RTE: N=25, K=30, Projection Net [50, 100, 25] + ReLU, Construction Net [4, 100, 100, 30] + ReLU
  - Richard: N=25, K=10, Projection Net [50, 100, 25] + Tanh, a_u 替换为 Tanh 激活的 10×10 可训练矩阵, Construction Net [3, 100, 100, 10] + Tanh

- **训练数据规模**: Burgers (200 initial conditions × 25 sensors × 5 timestamps), Elliptic (80 source functions × 19×19 grid), RTE (60 scatter coeffs × 25×25×8 velocities), Richard (60 initial conditions × 40×40×7 timestamps)

- **Loss 函数**: 未在论文中明确说明，推断为标准相对 L² loss（因所有结果报告 Euclidean/relative error）

- **两个变体**: fix-BelNet（所有样本共享同一组传感器位置，类似 DON 设定）和 free-BelNet（每样本随机采样不同传感器位置）

- **理论基础**: 从线性积分算子的低秩分解出发：κ(x,y) = Σ_k p_k(y) q_k(x)，非线性推广引入激活函数 g 和非线性依赖。与 FNO 的联系：FNO 的傅里叶域训练等价于 BelNet 中使用傅里叶基作为 p 和 q 的特例。

### 核心结果

| 实验 | DON (stack) | fix-BelNet | free-BelNet | FNO | 传统方法 |
|------|------------|------------|-------------|-----|---------|
| Burgers (相对误差%) | 1.17 | 0.59 | 1.07 | 1.61 | - |
| Elliptic (相对误差%) | 16.44 | 14.43 | 9.66 | - | MsFEM 2.28 |
| RTE CR=2 (相对误差%) | 4.39 | 3.68 | 3.56 | - | - |
| RTE CR=100 (相对误差%) | 19.18 | 16.40 | 13.31 | - | - |
| Richard (相对误差%) | 11.98(unstack)/21.18(stack) | 7.87 | 7.64 | - | GMsFEM 32.43 |

**Surprising findings**:
- free-BelNet 在 elliptic 问题 (9.66%) 上远优于 fix-BelNet (14.43%)，随机采样传感器反而提高了泛化能力（可能是因为增加了数据多样性，类似 data augmentation 效果）
- GMsFEM 在 Richard 问题上表现极差 (32.43%)，而 BelNet 仅用 40×40 mesh 训练便大幅优于 GMsFEM 的 1728 DOF，这在直觉上是反常规的——通常认为专门设计的数值方法应优于通用学习器
- BelNet 参数少于 DON 但性能更好（Burgers: 102K vs 103.7K; Elliptic: 312.4K vs 319.4K）
- RTE 实验中 free-BelNet 对 contrast ratio 的敏感度明显低于其他方法：CR 从 2→100，free-BelNet 误差只增加 9.75 个百分点 vs DON 增加 14.79 个百分点

### 明确局限

- **没有 UAP 证明**: 作者自己指出，BelNet 的 universal approximation theorem 尚未证明，尤其是 mesh-free 设置的推广仍然开放。这是理论上的核心缺失。
- **参数效率问题**: 作者承认 Projection Net 的参数数量可以优化（K 个独立子网络导致参数与 K 线性增长）。仅提到用 random feature projection 做了初步探索，未给出结果。
- **没有真机实验**: 所有实验均为合成 PDE 数据（Burgers, elliptic, RTE, Richard），无真实物理测量数据验证。
- **没有与更 modern 的 baseline 比较**: 只比较了 vanilla DON 和 FNO，没有比较后续的变体如 POD-DON、NOMAD 等。
- **仅关注稳态和短时演化**: 所有测试问题的时间演化相对简单，没有测试混沌动力学或长时间预测的稳定性。
- **计算开销未定量**: 提到 storage cost 问题但没有给出与 DON/FNO 的具体 wall-clock time 或 GPU memory 对比。
- **K 的选择是 heuristic**: 每个实验中 K 的取值（10 或 30）看起来是经验性的，没有给出选择准则或 sensitivity analysis。
- **输入函数的维数限制**: 所有实验中输入函数定义在 1D 或 2D 域，没有测试 3D 问题（计算代价可能爆炸）。

### 跟我们的关系

对 0616-fex 课题的评估：
- **撞车风险低**: BelNet 核心贡献是算子学习中的 mesh-free 特性（离散化不变 + 预测自由 + 域独立），与 fex（有限元相关方向）的关注点不同。BelNet 使用 learning-based projection，而非有限元基函数。
- **可用工具/方法**:
  - BelNet 的 Projection Net + Construction Net 架构设计可借鉴，尤其是将传感器位置显式编码到网络中的思路（即通过 y 坐标影响 projection weights）
  - "mesh-free" 的概念框架（discretization-invariant / prediction-free / domain-independent）可作为评估新方法是否 mesh-free 的 checklist
  - 对于处理不规则网格或随机采样点的问题，BelNet 的架构天然适配
  - 对比实验中 MsFEM 和 GMsFEM 的结果可作为 fex 相关方法的性能参考
- **该引**: 如果 fex 涉及 mesh-free 算子学习或多尺度问题，必须引用。但若 fex 纯粹关注有限元方法的改进（如 basis function 设计），则引用关系较远。
- **差异化**: BelNet 是纯 learning-based，不依赖 PDE 的变分形式或有限元离散。fex 方向若涉及 FEM 与神经网络的混合方法，则在方法论基础上与 BelNet 有本质区别。

## Read by: 0616-fex (2026-06-24)

本次审读从 fex 视角出发，评估 BelNet 与有限元相关研究的关系。

**撞车评估**: BelNet 与 fex 的核心方向不直接撞车。BelNet 是纯神经网络算子学习框架，目标是 mesh-free（即完全摆脱网格依赖）。fex 如果涉及有限元方法与神经网络的结合（如学习 FEM basis、用 NN 增强 FEM），则与 BelNet 存在概念上的张力：BelNet 追求无网格，而 fex 可能拥抱有限元网格。两个方向的哲学基础不同。

**可用工具**:
1. BelNet 的"传感器位置显式编码"机制（通过 y → Projection Nets）可被借鉴到 fex 相关的架构设计中，尤其是当需要处理不规则节点分布时
2. 论文中对多尺度/高对比度问题的实验 setup（elliptic permeability, RTE scattering, Richard's equation）可作为 fex 相关方法的测试 benchmark
3. MsFEM 和 GMsFEM 在本论文中的 baseline 结果可作为 fex 方法的性能参考点
4. FERN (2510.26962) 是目前最接近"有限元 + BelNet"的工作，值得单独深读

**差异化策略**: 如果 fex 方向强调利用 FEM 的数学结构（如变分原理、局部基函数性质），则与 BelNet 的 "学习一切基函数"有本质区别。fex 应强调物理可解释性和计算效率（BelNet 的 Projection Net 参数随 K 线性增长），而非纯粹的 mesh-free 灵活性。
