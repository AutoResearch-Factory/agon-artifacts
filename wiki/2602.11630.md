# Neuro-Symbolic Multitasking: A Unified Framework for Discovering Generalizable Solutions to PDE Families (2602.11630)

- **Year**: 2026.02
- **Authors**: Yipeng Huang, Dejun Xu, Zexin Lin, Zhenzhong Wang, Min Jiang (Xiamen University)
- **Venue**: arXiv
- **Keywords**: symbolic regression, PDE families, multifactorial optimization, affine transfer, C-ADF, physics-informed residual, neural operator baseline, FEX prior
- **One-liner**: 用多任务符号回归同时求同维 PDE 族解析解。

## 精华提炼

### 管线/方法

1. 论文目标不是数值 operator learning, 而是给一组同结构不同参数的 PDE family 同时搜索可解释解析表达式。
2. 输入是一族 PDE tasks: 每个 task 有相同的 PDE skeleton, 但参数不同, 例如 advection velocity、viscosity、diffusion coefficient。
3. 每个 task 的训练/评测数据来自解析解或高精度数值解, 论文统一随机采样 1100 个 spatiotemporal points。
4. 输出是每个 PDE task 对应的解析表达式 `u(x,t)` 或高维变量下的 symbolic solution。
5. 核心方法名是 NMIPS, 即 neuro-assisted multitasking symbolic PDE solver。
6. 主体搜索框架是 multifactorial optimization (MFO), 用一个统一 population 同时优化多个 PDE task。
7. 表达式表示采用 C-ADF chromosome: 一个 main function 加多个 automatically defined functions。
8. 每个 main/ADF 都用 Karva expression 编码, 可按 breadth-first 规则确定性解码成 expression tree。
9. 为适配多任务, 不直接存具体 token, 而是用整数区间表示 function、ADF、terminal、ADF argument 四类元素。
10. 对不同 task, 同一个整数会映射到该 task 的具体 function/terminal set, 形成统一编码空间。
11. Karva head/tail 长度满足 `l = h * (v - 1) + 1`, 多任务版用所有 task function arity 的最大值。
12. 种群初始化在统一整数空间内随机采样, 每个个体带 skill factor, 表示其当前最擅长的 task。
13. 繁殖包含两条路: 一点 crossover + uniform mutation, 以及 differential-evolution-based mutation/crossover。
14. 两条繁殖路由 `rmp` 随机选择, 论文设置 `rmp = 0.2`。
15. offspring 的 skill factor 可以从 parent 或 mating individual 继承, 或沿用 parent 的任务归属。
16. 每个候选表达式先做常数优化, 再计算 task-specific factorial cost。
17. 适应度不是纯数据拟合, 而是数据误差和 PDE 物理约束共同决定。
18. 数据误差使用 RMSE: 比较候选表达式 `u_hat` 与采样点上的 reference solution。
19. PDE 误差包括 PDE residual、initial condition error、boundary condition error。
20. PDE residual 通过自动微分计算候选表达式的高阶偏导, 再代入原 PDE。
21. constant tuning 使用 gradient-based optimizer, 目标是最小化数据误差和 PDE/IC/BC 约束组合 loss。
22. selection 使用 factorial rank、skill factor 和 scalar fitness。
23. factorial rank 是个体在某个 task 上按 factorial cost 排序的位置。
24. skill factor 是个体 rank 最好的 task。
25. scalar fitness 是 `1 / min_rank`, 用于跨任务比较。
26. 每轮 offspring 和 parent 做 one-to-one selection, 若 offspring scalar fitness 更高或 parent 属于 redundant set 则替换。
27. 关键新增模块是 affine transformation-based knowledge transfer。
28. 该模块先按 skill factor 把当前 population 分成两个 task-specific groups。
29. 每个 group 按 scalar fitness 排序, 取 top `N/2` 个体作为 transfer 源。
30. 对每组 population 计算均值向量和方差向量。
31. 两个 MLP 分别从均值/方差输出 affine scaling `gamma` 和 shifting `beta`。
32. transformed chromosome 形如 `(1 + gamma) * normalized(z) + beta`。
33. 变换后用 modulo arithmetic repair 到合法整数范围。
34. 原 population 和 transformed population 合并, 重新评估 factorial cost 和 scalar fitness, 取 top `N`。
35. 直觉上, 这个 affine transfer 不是共享神经网络参数, 而是在 symbolic chromosome distribution 层做结构迁移。
36. 论文声称这样能保留 PDE family 内共享的 mathematical structure, 同时允许各 task 有自己的 coefficient/fitness。

### 关键技术细节

37. benchmark 覆盖 6 类 PDE: 1D Advection、1D Burgers、1D Advection-Diffusion、2D Advection、2D Navier-Stokes、3D Advection。
38. 每类 PDE 都构造 4 个 parameter tasks, 共 24 个 task-level settings。
39. 1D Advection: `u_t + beta u_x = 0`, `beta = {0.1, 0.4, 0.7, 1.0}`。
40. 1D Advection 初值是 2 个 sinusoidal waves 叠加, `n_max = 8`, domain `x in [0,1]`, `t in [0,2]`。
41. 1D Burgers: `u_t + d(u^2/2)/dx = (nu/pi) u_xx`, `nu = {0.001, 0.004, 0.007, 0.01}`。
42. Burgers 数据用 finite difference solver 生成, `M = 100` spatial points, `T = 1000` time steps。
43. 1D Advection-Diffusion: `u_t + beta u_x = alpha u_xx`, `beta = {0.1,0.4,0.7,1.0}`, `alpha ~ U(0.001,0.005)`。
44. Advection-Diffusion 数据用 Crank-Nicolson finite difference, `N_x = 100`, `N_t = 100`。
45. 2D Advection: `u_t + beta_x u_x + beta_y u_y = 0`, `beta_x` 线性取 0.1 到 1.0, `beta_y ~ U(0.1,1.0)`。
46. 2D Advection 初值是两个 sinusoidal waves 的乘积, domain `x,y in [0,1]`, `t in [0,1]`。
47. 2D Navier-Stokes 用 vorticity formulation: `omega_t + u dot grad omega = nu Delta omega`。
48. Navier-Stokes 采用 Taylor-Green vortex, 解析解为 `sin(2*pi*x) sin(2*pi*y) exp(-8*pi^2*nu*t)`。
49. Navier-Stokes viscosity `nu = {0.005, 0.02, 0.035, 0.05}`。
50. 3D Advection: `u_t + beta_x u_x + beta_y u_y + beta_z u_z = 0`, `beta_x` 线性取 0.1 到 1.0, `beta_y,beta_z ~ U(0.1,1.0)`。
51. 3D Advection 初值是三个 sinusoidal waves 的乘积, domain `x,y,z in [0,1]`, `t in [0,1]`。
52. NMIPS 超参数: population size `N = 50`, mutation probability `0.002`, generations `100`, max evaluations `5000`。
53. Advection 类 symbol library 是 `{+, -, *, sin}`, 不包含 PDE parameters。
54. Burgers、Advection-Diffusion、Navier-Stokes 的 symbol library 扩展为 `{+, -, *, sin, exp, log}`。
55. 比较方法有 5 个: PhySO、DSR、SP-GPSR、GNOT、Geo-FNO。
56. PhySO、DSR、SP-GPSR 是 symbolic regression / GP 侧基线。
57. GNOT 和 Geo-FNO 是 neural operator 侧基线, 用于比较可泛化黑箱数值求解器。

### 核心结果

58. 1D Advection 平均 MSE: NMIPS `5.47E-01`, PhySO `5.97E-01`, DSR `5.58E-01`, SP-GPSR `5.77E-01`, GNOT `6.11E-01`, Geo-FNO `7.19E-01`。
59. 1D Burgers 平均 MSE: NMIPS `1.53E-02`, 优于 SP-GPSR `1.63E-02`, DSR `1.69E-02`, PhySO `2.38E-02`, GNOT `1.04E-01`, Geo-FNO `2.29E-02`。
60. 1D Advection-Diffusion 平均 MSE: NMIPS `1.19E-01`, 优于 SP-GPSR `1.25E-01`, DSR `1.26E-01`, PhySO `1.32E-01`, GNOT `3.85E-01`, Geo-FNO `1.39E-01`。
61. 2D Advection 平均 MSE: NMIPS `2.58E-01`, 优于 PhySO/SP-GPSR `2.64E-01`, DSR `2.71E-01`, GNOT `4.19E-01`, Geo-FNO `4.13E-01`。
62. 2D Navier-Stokes 平均 MSE: NMIPS `5.25E-02`, 与 SP-GPSR `5.26E-02`、DSR `5.29E-02` 很接近, 优于 PhySO `6.36E-02`, GNOT `1.01E-01`, Geo-FNO `7.13E-02`。
63. 3D Advection 平均 MSE: NMIPS `1.30E-01`, 优于 SP-GPSR `1.35E-01`, PhySO `1.39E-01`, DSR `1.40E-01`, Geo-FNO `1.97E-01`, GNOT `3.72E-01`。
64. 论文摘要声称最高约 `35.7%` accuracy improvement。
65. 计算成本图显示 NMIPS 在 1D Burgers、1D Advection-Diffusion、2D Navier-Stokes、3D Advection 上通常最快。
66. 1D Burgers 和 1D Advection-Diffusion 中 NMIPS 约 `250-300s`, 其他 symbolic baselines 常在 `2000-4000s`。
67. affine transfer 消融显示 w/ AT 在 6 个 PDE 平均都优于 w/o AT。
68. AT 对 2D Advection 最明显: 平均 MSE 从 `3.13E-01` 降到 `2.58E-01`, 约 `17.6%` reduction。
69. AT 对 2D Navier-Stokes: `6.06E-02` 降到 `5.25E-02`, 约 `13.4%` reduction。
70. AT 对 3D Advection: `1.47E-01` 降到 `1.30E-01`, 约 `11.6%` reduction。
71. noise robustness 在 2D Advection 上测试 `0%/5%/10%/15%` Gaussian noise, NMIPS curve 基本最平且最低。
72. supplement 的 scientific insight 声称 NMIPS 恢复了 transport characteristic line `x - beta t`、advection-diffusion 的 exponential decay envelope、Navier-Stokes 的 separable spatial/temporal skeleton。
73. 反直觉点: 某些表格中 NMIPS 的单 task 单 transfer setting 不总是最佳, 但 task-average 最好；这说明 transfer 有平均收益但不是逐 case 单调增益。
74. 另一个反直觉点: 2D Navier-Stokes 这里用的是 Taylor-Green vortex, 非线性项简化, 所以“复杂流体”证据没有标题看起来那么强。
75. supplement 的 2D Navier-Stokes 表达式表中 NMIPS 给出 `(x - 0.493) sin(y) + 0.037 t`, 与同节稍后宣称的 `exp(c1 t) sin(x) sin(c2 y)` skeleton 并不完全一致, 需要谨慎引用。

### 明确局限

76. 数据集全部是合成 PDE, 很多来自解析解或低维有限差分, 没有真实实验数据。
77. PDE 最高到 3D spatial dimensions, 没有真正高维 PDE 的大规模验证。
78. 每个 PDE family 只有 4 个 parameter tasks, family 内跨度有限。
79. 训练/评测采样点每 task 1100 个, 没有系统 sweep 数据规模。
80. 没有看到开源代码、固定随机种子、完整 artifact 或第三方复现报告。
81. 论文把“同一 PDE family 不同参数”的 generalization 与“跨维/跨结构 generalization”区分得不够清楚。
82. affine transfer 的 MLP 训练细节不充分: 网络结构、学习率、训练步数、loss weighting 没有足够可复现细节。
83. PDE residual、IC、BC、data loss 的权重没有在正文中明确量化。
84. symbolic library 很小, 结果可能依赖人工设计的 operator set。
85. baseline 是否经过同等调参不够透明。
86. Neural operator baselines 与 symbolic regression baselines 的目标不同: 前者通常输出数值 field, 后者输出表达式, 横向比较需要谨慎。
87. 论文声称 “first framework” 需要更多近期工作对照；本文只通过 references 获取了引用列表, 没有做额外 6 个月 arXiv search。
88. 对 negative transfer 的讨论主要是论证式, 没有看到任务相似度失败案例或 adversarial family。
89. pairwise/cross-dimensional exchangeability 这类 FEX 关注点完全不在本文覆盖范围。
90. 如果用于 `0616-fex`, 最大风险是把 NMIPS 误当作“高维 FEX 竞品”；实际上它更像同维参数族 symbolic transfer 的强 prior。

### 跟我们的关系

91. 对 `0616-fex` 来说, 本文是强相关 prior, 因为它同样做 symbolic regression for PDE solutions, 且强调解析表达式而不是黑箱数值场。
92. 但它不直接撞 `fex-dim-lift-skeleton` 的核心 claim: NMIPS 做同维 PDE family 参数迁移, 我们做低维 FEX skeleton 的 cross-dimension exchangeability-gated lifting。
93. 撞车点一: 如果我们只声称“利用 PDE family 的共享结构减少 symbolic search 成本”, NMIPS 已经占位。
94. 撞车点二: 如果我们只说“多任务/迁移可以加速 symbolic PDE solver”, NMIPS 也已经占位。
95. 差异化点一: 我们应强调 FEX RL controller 的 searchability audit, NMIPS 没有讨论 controller 多 seed 是否稳定搜到可 lift skeleton。
96. 差异化点二: 我们应强调 cross-dimension lift/reject decision, NMIPS 的任务维度固定在 family 内, 没有从 2D/3D 到 d=100 的宏提升。
97. 差异化点三: 我们的 gate 包括 coefficient exchangeability 和 high-d top-k PDE probe, NMIPS 的 affine transfer 是 chromosome distribution alignment, 不是 exchangeability certificate。
98. 可借鉴工具一: C-ADF/Karva 表达式编码可作为 FEX 表达式合法性和复用子结构的候选表示。
99. 可借鉴工具二: task-aware scalar fitness / skill factor 可以借鉴到 FEX multi-family search stability scan。
100. 可借鉴工具三: PDE residual + IC/BC + data fit 的 combined fitness 是 FEX 高维 probe 的合理 scoring 模板。
101. 可借鉴工具四: affine distribution transfer 可以作为 FEX seed warm-start 或 skeleton population augmentation 的 baseline。
102. 该引。Related work 中可放在 “same-dimension PDE family transfer / multitask symbolic PDE solving” 小节。
103. 不应把它列为最强 direct competitor；对 `fex-dim-lift-skeleton`, PSR/HD-TLGP/FEX-PG 更直接。
104. 写作时建议一句话界定: “NMIPS transfers symbolic structures across parameterized PDE instances of fixed dimensionality; our problem is certifying when a low-dimensional FEX-discovered skeleton can be lifted across dimensions.”

从 `0616-fex` 视角, 这篇是强相关但非直接撞车。它会威胁任何宽泛 claim, 例如“共享 PDE family 结构可以加速符号 PDE 求解”或“多任务迁移能提升 symbolic PDE solver”。但它不覆盖我们当前 `fex-dim-lift-skeleton` 的关键 claim: 低维 FEX 搜索到的 skeleton 什么时候能跨维提升到 d=100, 什么时候必须拒绝, 以及 searchability 与 liftability 的分离诊断。

可用部分包括: C-ADF/Karva 作为合法表达式编码, task-aware scalar fitness/skill factor, PDE residual + IC/BC + data fit 的 combined fitness, 以及 affine transfer 作为 population warm-start baseline。差异化应明确写成: NMIPS 是 fixed-dimensional parameter-family transfer; 我们是 cross-dimensional exchangeability audit, 包含 search stability、coefficient exchangeability 和 high-dimensional PDE probe。引用时建议放在 “multitask symbolic PDE solving” 或 “same-dimension transfer” 小节, 不要把它当成 FEX cross-dimension lifting 的直接竞品。
