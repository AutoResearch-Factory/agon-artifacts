# RadarQA: Multi-modal Quality Analysis of Weather Radar Forecasts (2508.12291)

- **Year**: 2025.08
- **Authors**: Xuming He (Shanghai AI Lab / 浙大), Zhiyuan You (CUHK), Junchao Gong, Couhua Liu (CMA 地球系统数值预报中心), Xiaoyu Yue, Peiqin Zhuang, Wenlong Zhang†, Lei Bai† (Shanghai AI Lab)
- **Venue**: arXiv preprint (NeurIPS 2025 模板, preprint 选项)
- **Keywords**: radar nowcasting quality assessment, MLLM, SEVIR, expert-aligned evaluation, GRPO, hybrid annotation, VIL, forecast verification
- **One-liner**: 把雷达临近预报的质量评估从分数变成专家式文本报告——用 SEVIR + 7 个 nowcasting 模型构建 RQA-70K 数据集，三阶段训练 Qwen2.5-VL-7B，四任务全面超过 GPT-4o/o1。

## 精华提炼

### 管线/方法

整体是"数据构建 → 多阶段训练 → 四任务评测"三段：

1. **RawRQA-20K 生成**：SEVIR (VIL 模态) 中只取 storm events（7 类：thunderstorm wind 1030 / hail 556 / flash flood 218 / flood 121 / tornado 121 / funnel cloud 58 / heavy rain 55，共 2159 个事件），每个事件切成 3 个 input-target 对（输入 10 帧 → 目标 12 帧）。用 7 个 nowcasting 模型生成预报序列：EarthFormer、PredRNN、CasCast、DGMR、DiffCast、SimVP、NowcastNet（覆盖 RNN/GAN/diffusion/transformer 架构谱系）。VIL 按 SEVIR 惯例离散成 6 个降水强度级并用 SEVIR colormap 渲染成 RGB。帧级任务从每条序列随机抽 1 帧配对 GT。
2. **属性标注（Attribute-Informed Generation 的输入）**：37 个关键属性 = 35 个科学属性库属性 + 帧/序列两级 Overall。其中 **17 个感知型属性人工标注**（外包标注公司 + 专家 golden standard + 试标对齐 + 批次交叉验证 + 专家按批抽检 35%，不合格整批重标），**20 个指标型属性脚本自动计算**（阈值由专家定）。
3. **响应生成**：rating 任务直接由 general attributes 拼 JSON；assessment 任务把全部属性 + 视觉信息喂给 GPT-4o 组稿成流畅报告，并显式指示其纠正不一致。问题池由 GPT-4o 生成 50 条、人工筛 10 条模板。
4. **三阶段训练**（基座 Qwen2.5-VL-7B）：Stage-1 LoRA SFT → Stage-2 GRPO（仅 rating 任务，format reward + accuracy reward）→ Stage-3 小样本 post-training 规整输出格式。
5. **推理/评测**：四任务（帧评级/帧评估/序列评级/序列评估），rating 用 per-attribute accuracy，assessment 用 BLEU/BERTScore/ROUGE_L/METEOR + GPT-4 Score (0-10)。

### 任务范式（四任务七维）

- **Task-1 帧评级**：4 静态属性 Miss / False Alarm / Sharpness / High Value Match + Overall，离散等级（Great/Good/Fair/Poor 四级）。
- **Task-2 帧评估**：文本报告，先描述观测再逐属性评价并解释对 overall 的影响。
- **Task-3 序列评级**：3 动态属性 Dynamic Consistency / Cumulative Precipitation / High Value Retain + Overall。
- **Task-4 序列评估**：基于 22 个序列属性的综合报告（含对流生消、组织度、移向移速等过程描述）。

### 关键技术细节

- **科学属性库**：5 super-categories（Intensity / Precipitation Conservation / Precipitation Dynamic Distribution / Convective Cycle / Morphology）→ 10 sub-categories → 35 属性（15 帧级 + 22 序列级，含 Overall 后 37）。序列属性含 move direction、rotation center、shape/scale/intensity change、convective cell change、genesis/dissipation difference、organization degree、artifacts direction 等——是一份显式的"对流过程评估维度清单"。
- **自动属性计算**（专家定阈值，全部可复现）：降水像素定义 VIL>16（SEVIR 惯例）；miss rate 阈值 [0.1,0.2,0.4]、FAR [0.1,0.2,0.3] 分四级；Sharpness 用 Sobel 均值比的折叠差 d，阈值 [0.5,0.7,0.9]；高值区定义 VIL>219，相对误差转分后阈值 [0.3,0.6,0.8]；累计降水阈值 [0.93,0.97,0.99]；方向描述用 3×3 网格九宫格方位；6 级降水按重要性加权 [1,1.5,2.5,5,10,20] 选主导缺陷等级。
- **RQA-70K 规模**：rating 训练 40,000（帧/序列各 20k）+ assessment 训练 29,000（各 14.5k）；验证 860/410/801/179。
- **训练超参**：Stage-1 LoRA rank 8、lr 1e-4、AdamW、batch 128、5 epochs；Stage-2 GRPO generation number 4、10k rating 样本、batch 32、1 epoch；Stage-3 LoRA rank 4、每子任务 2500 样本、1 epoch。总计约 50 小时 × 8 × A800。
- **奖励设计**：r_fmat（JSON 可解析且键齐全 =1）；r_acc = N_hit/N_all（仅 r_fmat=1 时非零）——纯规则奖励，无学习型 reward model。

### 核心结果

| 任务 | RadarQA | 最强 baseline | 开源 7B 基线 |
|------|---------|---------------|--------------|
| 帧评级 Overall acc | **61.51** | o1 52.67 / GPT-4o 48.84 | Qwen2.5-VL-7B 20.10 |
| 帧评估 GPT-4 Score | **6.87** | Claude3.7 5.89 | 3.81 |
| 序列评级 Overall acc | **66.17** | GPT-4o 45.00 | 7.99 |
| 序列评估 GPT-4 Score | **6.58** | Gemini2.5-pro 5.63 / o1 5.49 | 3.92 |

- **High Value Retain 达 80.52% acc**（GPT-4o 只有 4.99%）——通用 MLLM 几乎完全不能判断高值区随时间的保持，这是 over-smoothing 检测的核心维度。
- **传统指标与专家评级一致性弱**（appendix 关键表）：帧评级 accuracy/SRCC/PLCC 上 CSI 41.74/0.26/0.27、POD 42.79/0.28、FAR 39.07/0.15，通用 IQA 指标 DISTS 53.60/0.55 反而更好，RadarQA 61.51/0.62/0.64 最高。**反直觉点：DISTS/LPIPS 这类感知指标比 CSI/POD/ETS 更贴近气象专家判断**。
- **OOD 泛化**：在雷达反射率重建任务（DiffSR/SRViT/U-Net 生成，训练未见）上 rating 59.94/48.72、assessment 6.22/5.64，仍大幅领先 o1（32.28/17.31、4.34/4.36）；序列侧下降归因于重建任务逐帧独立、无时间连贯性。
- **Ablation**：Stage-1 SFT 带来最大跳变（rating 27.79→64.05 平均）；GRPO 主要提升推理型属性（false alarm 63.37→68.14、dynamic consistency 42.44→64.17）；四任务联合训练全面优于单任务（68.46 vs 66.63 等）；3B 模型只降 ~3-5 点（63.44/59.71 vs 68.46/62.24）。
- **专家研究**：气象学家按内容准确性/信息密度/专家关注点覆盖三准则盲评，GT 与 RadarQA 均显著高于 GPT-4o；序列任务得分高于帧任务（专家知识注入越多越有价值）。
- 开源 MLLM 在 High Value Match 上 acc < 20% ——连 VIL colormap 颜色↔强度映射都学不会。

### 明确局限

作者自陈：(1) 任务范式未统一，不支持两个预报结果间的对比评估（A/B 比较）；(2) 细粒度描述仍不满意；(3) 评估输出能否作为 nowcasting 模型的 feedback/reward 未探索。附录额外披露 GPT-4o 组稿的四类偏差：style bias（报告结构过度统一）、accuracy bias（文本与视觉信息不一致）、redundancy bias、attribute omission bias。

我读出的额外局限：(a) **GPT-4 Score 有循环性**——GT 报告由 GPT-4o 生成、评分由 GPT-4 打，文风逼近 GPT 生成分布的模型天然占优，专家研究只覆盖 assessment 任务的小样本盲评（样本量与专家人数未披露）；(b) 数据只有 SEVIR VIL 单模态、CONUS 区域，**无环境场（CAPE/shear 等）、无事件结局标签**，"质量"只相对雷达 GT 定义，不回答"这个预报误差对预警意味着什么"；(c) 四级离散等级的阈值是专家拍的单一切分，无不确定性表达；(d) 12 帧 × 5min = 1 小时预报窗，未触及更长时效；(e) 人工标注 17 属性的标注者一致性（IAA）未报告。

### 跟我们的关系

对 0713-scs-benchmark（SCS 环境-雷达多模态基准）：

- **不撞车**。RadarQA 做的是"预报质量的 MLLM 文本评估"，数据底座仅 SEVIR VIL 单模态；我们做的是"环境场（HRRR 长时窗）+ 雷达（MRMS）+ 事件标签（Storm Events）以风暴为中心的事件包基准"。它没有环境场、没有事件结局、没有长时窗，与我们的核心 claim（环境数据作为一等公民）零重叠。

- **强参考价值 2：整编严谨性范本**。hybrid annotation（专家 golden standard → 试标对齐 → 批次交叉验证 → 35% 抽检 → 不合格整批重标）+ 全部自动属性公式/阈值显式披露，正是 ESSD 式 provenance 叙述所需的标注质量控制模板。
- **强参考价值 3：论证弹药**。其 appendix 表格实证了 CSI/POD/FAR 与专家判断的一致性（SRCC 0.15-0.29）远低于感知类指标——这直接支持我们 benchmark 设计中"传统逐像素指标不足以刻画对流预报价值、需要对象级/过程级评测"的 motivation。
- **潜在下游用户**：RadarQA 类评估模型目前只能"看雷达评雷达"；我们的事件包若建成，其 conclusion 里"评估输出作为预报模型 reward"的 future work 可在带环境场与事件结局的语境下扩展（如评估报告能否预测预警命中），构成引用我们数据集的天然场景。

## Read by: 0713-scs-benchmark (2026-07-14)

从 SCS 基准数据集视角审读。核心结论：**互补不撞车，且是三重资源**。

1. **撞车评估**：RadarQA 的贡献是"SEVIR 之上的质量评估任务层"（RQA-70K），不是新的多模态观测基准。它不含环境场、不含事件结局标签、时窗仅 1 小时预报段——与我们"长时窗 HRRR 环境场 + MRMS + Storm Events 事件包"的填空 claim 无任何一条重叠。反而佐证了该生态位空缺：连雷达质量评估 SOTA 工作都只能在纯 VIL 上定义"质量"，无法回答误差的预警后果。
2. **可用方法/工具**：(a) 35 属性科学属性库 → 事件包 quality flags 与 canonical task 评测维度的现成 taxonomy；(b) 自动属性的公式+专家阈值（Sobel sharpness、高值保持、累计降水守恒、3×3 方位网格）→ 可直接实现为我们数据管线的自动质量标记；(c) 人工标注 QC 流程（golden standard/试标/35% 抽检/整批重标）→ ESSD 论文 provenance 章节的标注质量模板；(d) CSI/POD 与专家一致性 SRCC 仅 0.15-0.29 的实证表 → 我们 motivation 的直接弹药。
3. **差异化表述建议**：我们的 benchmark 应强调"评估的是预警价值而非像素/结构保真"——RadarQA 已把"结构保真的专家式评估"做到 7B 模型可用，我们的空间在"环境-事件耦合的可预报性与预警评估"。
4. **后续搜索线索**：反引文中 Omni-Weather (2512.21643)、S2SServiceBench (2602.14017) 值得下轮精读，确认气象 MLLM benchmark 谱系是否有人开始把环境场纳入。
