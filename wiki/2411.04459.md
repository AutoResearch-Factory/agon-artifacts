# GPT-Guided Monte Carlo Tree Search for Symbolic Regression in Financial Fraud Detection (2411.04459)

- **Year**: 2024.11
- **Authors**: Prashank Kadam (Vesta Corporation)
- **Venue**: CIKM '24
- **Keywords**: symbolic regression, MCTS, Symbolic GPT, financial fraud detection, rule-based learning, PUCT, interpretable ML, LLM-guided search
- **One-liner**: 用 Symbolic GPT 提供的策略概率引导 MCTS 的 PUCT 选择做符号回归，在金融欺诈检测上比传统 ML 方法更准且可解释。

## 精华提炼

### 管线/方法

完整 pipeline 如下：

1. **输入**：金融交易数据集（含 base features 如 shipping/billing email、phone、address、card number、BIN、device ID、IP；velocity features 为各 base feature 在 15min~90day 窗口的 count/sum；relational velocity features 为一个 base feature 关于另一个的聚合）。所有分类变量做 one-hot encoding。

2. **MDP 建模**：将符号回归问题建模为 MDP：
   - State space S：所有合法的 operand-operator 组合（partial expression）
   - Action space A_s：根据当前 state 条件性选择下一个 operator 或 operand（operand 后必须跟 operator，反之亦然）
   - Transition function T(s,a)：保证表达式数学合法
   - Reward function R(s,a)：与 loss 成反比

3. **MCTS 搜索 + Symbolic GPT 引导**：使用 PUCT (Predictor + Upper Confidence bounds for Trees) 策略选择动作：
   ```
   PUCT(s,a) = Q(s,a) + c * sqrt(log N(s) / N(s,a)) * P_GPT(s,a)
   ```
   其中 P_GPT(s,a) 是 Symbolic GPT 对下一步动作的预测概率。Symbolic GPT 接收当前 partial expression 作为输入，输出下一个可能动作（operand 或 operator）的概率分布，充当 MCTS 的 policy π(s)。

4. **表达式评估**：生成的 trajectory（即完整表达式）用 Binary Cross-Entropy loss 评估：
   ```
   L_BCE(y, ŷ) = -[y log(ŷ) + (1-y) log(1-ŷ)]
   ```
   最大表达式长度为 40，超过后在下一个 operand 处终止。

5. **Top-k 选择与 GPT 重训练**：选择 loss 最低的 top 20% 表达式，赋予 reward R(T) = 1/(L + ε)，用这些表达式重训练 Symbolic GPT。重训练 loss 为 cross-entropy + L2 正则化：
   ```
   L_CE = -Σ y_i log(ŷ_i) + λ Σ θ_j²
   ```

6. **规则提取**：从 top-k 表达式中，通过求解线性方程组提取可解释规则。例如设 Fraud Score = 1 创建不等式触发规则。

7. **迭代收敛**：对数据集中每笔交易重复以上过程直到收敛。

### 关键技术细节

- **Symbolic GPT 模型**：基于 Valipour et al. (2021) 的 SymbolicGPT（arXiv 2106.14131），在大规模符号回归数据集上预训练的 Transformer。本文在金融数据上做 fine-tune。
- **PUCT 策略**：源自 AlphaGo (Silver et al. 2016)，c 为 exploration-exploitation 平衡常数（具体值未给出）。
- **表达式空间**：operand 包括所有 base features + velocity features + relational velocity features；operator 包括一元运算 {sin, cos, log, exp} 和二元运算 {+, -, ×, ÷} 以及常数。
- **数据集**：Proprietary Financial Fraud Dataset (PFFD)，来自 Vesta 的电商客户数据，约 125 万笔交易，13,454 笔欺诈（1.07% 正样本率）。数据已 hash 脱敏。
- **超参数**：k = 0.2（top 20%），最大表达式长度 = 40，ε 为小正数避免除零。
- **评估指标**：Recall 和 AUC。

### 核心结果

| Algorithm | Recall | AUC |
|-----------|--------|-----|
| SVM | 0.536 | 0.507 |
| Random Forest | 0.629 | 0.574 |
| XGBoost | 0.678 | 0.634 |
| LSTM | 0.621 | 0.587 |
| GCN | 0.725 | 0.711 |
| GAT | 0.784 | 0.765 |
| **SR-MCTS** | **0.812** | **0.797** |

SR-MCTS 在 Recall 和 AUC 上均超过所有 baseline，同时提供完全可解释的规则。

**Surprising/反直觉发现**：
- 即使 Symbolic GPT 有幻觉问题，由于它只提供"有限引导"给 MCTS（通过 PUCT 中的 P_GPT 项），幻觉影响被大幅削弱。MCTS 自身的搜索机制（exploration 项）可以纠正 GPT 的错误引导。
- 生成的表达式相当复杂（示例中包含 log、sin、exp 嵌套的 12+ 项），但仍然比黑盒模型更可解释。

### 明确局限

**作者承认的局限**：
- 仅在专有数据集上测试，未来将探索其他领域
- 未来将实验更长表达式、更复杂算子、以及动态检测 k 和终止状态

**读出来的局限**：
1. **无标准 SR benchmark**：没有在 SRBench、Nguyen、Feynman 等标准符号回归数据集上测试。无法评估其作为通用 SR 方法的性能。
2. **无 SR 方法对比**：仅与传统 ML/DL 方法（SVM、RF、XGBoost、LSTM、GCN、GAT）比较，没有与其他 SR 方法（如 PySR、DSR、uDSR、GP-based 方法、其他 MCTS-SR 方法如 SPL/TPSR）比较。
3. **Symbolic GPT 过时**：使用的是 2021 年的 SymbolicGPT，较为古老；后续有更强的 SR 基础模型（如 E2E Transformer, NeSymReS 等）。
4. **数据集不可复现**：PFFD 是专有数据，其他研究者无法复现。
5. **评估指标单一**：只报告了 Recall 和 AUC，缺乏 Precision、F1、表达式复杂度/简洁性分析。
6. **规则提取细节缺失**：第2.5节"Extracting Rules"非常简略，没有具体说明如何从表达式构建规则集。
7. **论文较短**：整体内容偏向工业应用报告（workshop paper 级别），理论深度有限。
8. **无消融实验**：没有对比"无 GPT 引导的纯 MCTS"、"不同 k 值"、"不同表达式长度"等消融。

### 跟我们的关系

对 llm-mcts-sr 课题：

**撞车评估**：这篇论文与我们课题有直接重叠——它用 LLM (Symbolic GPT) 引导 MCTS 做符号回归。但重叠程度有限：
- 它用的是 Symbolic GPT（2021 年的小型专用模型），不是现代大型 LLM
- 它只在金融欺诈检测这个单一应用上做了实验
- 它没有在标准 SR benchmark 上评估
- 它的 LLM 引导方式比较简单（直接用 GPT 输出概率作为 PUCT 的 prior）

**能用的工具/方法**：
- PUCT + LLM prior 的基本框架可以作为 baseline
- MDP formulation（operand-operator 交替）是标准做法，可以借鉴
- 迭代 retraining loop（MCTS 生成 → 选 top-k → 重训 LLM → 重复）的思路有价值
- Binary cross-entropy loss 用于分类型 SR 任务的评估方式

**差异化方向**：
- 使用更强大的现代 LLM（而非 2021 年的 SymbolicGPT）
- 在标准 SR benchmark 上做全面评估
- 更精细的 LLM-MCTS 交互方式（不仅是 prior probability，还可以做 value estimation、expression simplification 等）
- 消融实验证明各组件贡献
- 与其他 MCTS-SR 方法（SPL、TPSR）做系统对比

**该引不该引**：该引。它是 LLM+MCTS+SR 直接交叉的工作，必须在 related work 中讨论。但可以指出其局限性来突出我们的贡献。
