# SymTorch: Symbolic Distillation of Neural Networks (2602.21307)

- **Year**: 2026.02
- **Authors**: Elizabeth S.Z. Tan, Adil Soubki, Miles Cranmer (University of Cambridge, DAMTP)
- **Venue**: arXiv preprint (NeurIPS 2026 format)
- **Keywords**: symbolic regression, neural network interpretability, symbolic distillation, PySR, physics-informed machine learning, GNN interpretability, LLM acceleration, SLIME
- **One-liner**: SymTorch is an open-source PyTorch-native framework for extracting closed-form symbolic expressions from arbitrary neural network components, enabling both scientific discovery and model acceleration.

## 精华提炼

### 管线/方法

**核心流程**: SymTorch 的 symbolic distillation 管线分三步：
1. **Wrapping**: `SymbolicModel` 继承 `nn.Module`，通过 forward hooks 记录目标模块的输入和输出激活值。
2. **Distillation**: 调用 PySR（基于遗传算法的符号回归引擎）在记录到的 I/O 对上搜索符号表达式。对多输出维度分别拟合独立的符号模型。PySR 维护一个复杂度-准确率的 Pareto 前沿。
3. **Switch**: `switch_to_symbolic` 将原始模块完全替换为符号表达式，梯度可流过符号表达式继续训练。`switch_to_block` 恢复原始模块。

**SLIME 实现**:
- 给定黑箱模型 f 和关注点 x*，在局部邻域拟合符号代理模型 s
- 训练集包含：(1) 从数据分布中选取的 J 个最近邻 (2) 从以 x* 为中心的高斯分布中采样的 N_synthetic 个点
- 损失函数：合成点用核权重 MSE（核为高斯核 pi_x），真实邻居点用权重 M
- 相比 LIME 的线性代理模型，SLIME 可捕获非线性关系

**LLM 推理加速管线**:
1. 对每层 MLP 的输入和输出激活分别做 PCA 降维
2. 对降维后的输入输出映射做符号回归
3. 用符号表达式替换原始 MLP（完全移除 MLP 权重矩阵，释放显存）
4. 贪心选择每层对 perplexity 影响最小的层依次替换

### 关键技术细节

**GNN 力恢复实验**:
- 架构：edge model + node model 均为 3 层 MLP，每层 300 隐藏单元，ReLU 激活
- 数据集：4 粒子系统，每个力律 10,000 次模拟 x 1,000 时间步（每 5 步采样），70/15/15 分割
- 训练：Adam + Cosine Annealing，100 epochs，batch size 64，约 110 万优化步
- 数据增强：位置坐标加 std=3 的高斯噪声
- 模型变体：Standard (msg dim=100)、Bottleneck (msg dim=2)、L1 reg (alpha=1e-2)、KL variational (Gaussian prior + KL loss)、Pruning (余弦退火调度逐步缩减维度至 2)
- SR 参数：运算符 {+,-,x,inv,exp,log}，complexity 最高 25，7,000 次迭代，parsimony=0.05，5,000 样本
- 计算时间：~10 分钟 (Apple M4 Max)

**混沌 Lorenz 实验**:
- Lorenz 参数 sigma=10, rho=28, beta=2.7，时间步 0.01
- 3D 状态通过随机线性映射投影至 10D 观测空间
- 架构：线性编码器 R^10->R^3，动力学 MLP [3,64,64,3]（ReLU），解码器为编码器伪逆
- 训练：Adam lr=1e-3, batch 8000, 3000 epochs, 早停
- SR 参数：运算符 {+,-,x}，complexity 最高 35，400 次迭代
- 符号表达式替换后 finetune 编码器/解码器 200 epochs (lr=1e-4)
- 计算时间：~5-10 分钟

**PINN PDE/ODE 发现实验**:
- 物理系统：1D 热方程 (alpha=0.2)、1D 行波方程 (c=0.67)、阻尼谐振子 (delta=2, omega0=20)
- 数据：仅 10 个训练点
- 架构：3 层 x 32 单元 FC，Tanh 激活（PINN 与标准 NN 共享相同架构）
- PINN 损失：L_data + lambda_phys * L_phys + lambda_ic * L_ic + lambda_bc * L_bc
- 热/波方程：lambda_phys=1, lambda_ic=5, lambda_bc=5；振荡器：lambda_phys=1e-4
- SR 参数：{inv, sin, exp, cos}，complexity=3 for trig/exp，parsimony=0.01, 1000 iterations
- 结果：复杂度 13 时恢复精确闭式解，物理常数 3 位有效数字精度

**SLIME vs LIME 基准**:
- 6 分类 + 2 回归数据集，MLP 黑箱模型（2 层 x 64/128 单元）
- 每个数据集 5 个随机局部点，500 合成样本，J=min(100, max(10, 0.05 x n_train))
- 指标：分类用 KL 散度、加权交叉熵、加权概率 MSE、决策边界一致率；回归用加权 R^2、加权 MAE、加权 MSE

**LLM 推理加速实验**:
- 模型：Qwen2.5-1.5B-Instruct，28 层 MLP
- 数据：Wikitext-2-v1，178K tokens/分割
- PCA 降维：输入 1536->32，输出 8960->8
- SR 参数：默认 + 5000 次迭代，6,000 训练样本
- 推理基准：Nvidia A100-SXM4-80GB，5 次 warm-up + 100 次 forward pass 取平均

### 核心结果

**GNN 力恢复**:
- Pruning 变体在所有 4 个力律上成功恢复正确的函数形式，与 bottleneck 性能相当但无需预知系统维度
- Standard 和 L1 模型成功率较低，KL 变体几乎全部失败
- 在 charge 模拟中，pruning MAE=19.40（variance=56417.66）

**混沌 Lorenz**:
- 符号模型在训练轨迹 MSE=15.52，测试 MSE=15.65（NN 分别为 8.22, 8.18）
- 但在 200 条随机轨迹上符号模型和 finetune 模型平均预测误差显著低于 NN（surprising：NN 过拟合单轨迹）
- 发现方程虽非精确 Lorenz 方程，但经过线性变换后恢复正确 attractor geometry

**PINN PDE/ODE 发现**:
- 热方程：sin(3.144 x) * exp(-1.981 t)，alpha 估计值 0.201（真值 0.200）
- 行波方程：sin(3.147 x) * cos(2.105 t)，c 估计值 0.670（真值 0.670）
- 阻尼谐振子：exp(-2.000 t) * cos(-19.625 t)，delta=2.000（真值 2.000），omega 约 19.625（真值 19.900）

**SLIME vs LIME**:
- SLIME 在全部 8 个数据集上全面优于 LIME，KL 散度低 1-4 个数量级，回归加权 MSE 显著更低
- 例如 Iris 数据集：LIME KL=0.00~1.40，SLIME KL=-0.00~0.00

**LLM 算术偏差**:
- 加法：正确公式 x0+x1 在 Pareto 前沿中但非最优选择，表明 LLM 添加了系统误差
- 温度转换：LLM 学习到的公式复杂且不精确，正确公式 (9/5)x+32 在 Pareto 前沿中
- 计数：SR 未能找到正确公式，与有限精度 transformer 的表达能力限制一致

**LLM 推理加速**:
- 替换 1 层：+0.444 PPL，+2.4% throughput，158MB (2.7%) VRAM reduction
- 替换 4 层：+2.547 PPL，+9.8% throughput，630MB (10.7%) VRAM reduction
- 替换 7 层：+6.375 PPL，+18.7% throughput，1103MB (18.7%) VRAM reduction
- SR 的 PPL 代价与 PCA 几乎相同（SR 0.444 vs PCA 0.451 在 1 层时），印证维数降低是瓶颈
- 符号混合模型在 perplexity vs throughput Pareto 前沿上与同规模开源 LLM 竞争

### 明确局限

**作者自述的局限**:
1. SR 随输入维度组合增长，高维组件难以精确全局近似
2. 无合适归纳偏置时，NN 将单一功能分散到多个纠缠组件中，蒸馏困难
3. PCA 降维是 PPL 退化的主因，而非符号近似本身
4. 各层独立优化，收益非累加，贪心选择无法保证全局最优
5. 固定运算符集合限制表达形式
6. 当前方法限于线性坐标变换（与 SINDy autoencoder 的非线性编码器相比）

**读出来的局限**:
1. **无真机验证**：所有实验均为仿真/标准数据集，无真实机器人/物理实验验证
2. **无标准 benchmark 对比 SR 性能**：未在 SRBench 等标准 SR benchmark 上评测
3. **LLM 加速的 PPL 代价不可忽略**：10 层替换后 PPL 从 10.79 升至 30.52，部分应用不可接受
4. **SLIME 的符号结果可读性存疑**：ADR 表中输出表达式极为复杂（多层嵌套函数），其"可解释性"可能不如 LIME 的线性系数直观
5. **依赖 PySR 的随机搜索**：遗传算法的随机性，每次运行可能给出不同结果
6. **未与 other symbolic regression engines 对比**（如 AI Feynman, EQL, Deep Symbolic Optimization）
7. **实验规模有限**：PINN 只有 3 个简单 PDE，GNN 只有 4 个力律，LLM 只有 1 个模型
8. **Our topic 0616-fex 角度**：本文聚焦"蒸馏已训练好的 NN"，而非"指导 NN 学习更好的物理表示"。若我们的课题想用符号表达来增强物理仿真/接触模型，SymTorch 的工具链有用但需要改造——它不能直接产生接触力学中的约束/不等式形式

### 跟我们的关系

对 **0616-fex 课题**：

1. **不直接撞车**：SymTorch 的目标是理解和解释已训练 NN（post-hoc interpretability），而非设计更好的物理仿真学习架构。我们的 FEX 课题关注的是 flexible/soft object 的接触动力学仿真与表示学习，两者目标不同。

2. **可用工具/方法借鉴**：
   - SymTorch 的 `SymbolicModel` + `switch_to_symbolic` 链可用于分析仿真器学习到的物理规律——如果我们训练了一个 GNN 做 soft object 接触动力学预测，可以用 SymTorch 蒸馏出接触力表达式
   - Pruning 变体（自动发现系统维度）对不知道复杂物体真实自由度的情况很有用
   - SLIME 的局部符号代理可用于分析 soft object 变形在特定接触条件下的局部行为

3. **差异化方向**：
   - SymTorch 只处理 post-hoc 蒸馏，不设计 inductive bias 来引导 NN 学到可解释的物理结构
   - 我们的 FEX 可以提出新的 inductive bias（如接触不等约束、材料本构关系的符号先验）
   - SymTorch 未涉及接触力学/碰撞检测等机器人物理仿真的特色问题

4. **该引不该引**：作为 symbolic regression 应用于 NN 理解的系统性开源框架论文，引入 Related Work 的"symbolic interpretability of neural networks"小节。作为工具链在 GNN 力恢复实验上可能有对比价值，但不适合作为核心 baseline。

## Read by: 0616-fex (2026-06-16)

本次审读从 0616-fex（flexible/soft object 接触动力学仿真与表示学习）视角出发：

**撞车评估**：不撞车。SymTorch 是 NN 的 post-hoc 可解释性工具，并非物理仿真方法。其 GNN 力恢复实验证明了"用 SR 从 GNN 中提取物理定律"的有效性，但这是已知结果的复现，非新物理规律的发现。

**可用工具**：
1. SymTorch 的 pruning 变体可用于自动推断物理系统的有效维度
2. SymbolicModel + switch_to_symbolic 的完整管线可用于分析我们自己的仿真模型
3. SLIME 的局部符号解释可用于分析接触边界附近的力场局部行为

**差异化**：
- 我们的 FEX 若产出新的 inductive bias（如接触约束的符号先验），则与 SymTorch 形成互补：SymTorch 做 post-hoc 解释，我们做 training-time 引导
- SymTorch 专注于"恢复已知规律"，FEX 应专注于"发现新规律"或"用仿真验证理论假设"
