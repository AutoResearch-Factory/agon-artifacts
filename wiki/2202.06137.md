# MIONet: Learning multiple-input operators via tensor product (2202.06137)

- **Year**: 2022.02
- **Authors**: Pengzhan Jin (Peking Univ), Shuai Meng, Lu Lu (UPenn, corresponding)
- **Venue**: arXiv (later published in SIAM Journal on Scientific Computing, 2022)
- **Keywords**: neural operator, multiple-input operator, DeepONet, tensor product, universal approximation theorem, Schauder basis, PDE solver
- **One-liner**: 首次提出多输入神经算子 MIONet，基于 Banach 空间张量积证明通用逼近定理，用多个 branch net 编码不同输入函数，用于学习 PDE 的多输入解算子。

## 精华提炼

### 管线/方法

1. **输入**：多输入函数 v1, v2, ..., vn 分别定义在不同的 Banach 空间 X1, X2, ..., Xn 上（如初始条件定义在初始域、边界条件定义在边界域、forcing term 定义在全域），以及输出域坐标 y。
2. **投影（preprocessing）**：每个输入函数 vi 通过 Schauder basis 的 canonical projection φ_qi^i 投影到有限维坐标（如等距采样 100 点），作为对应 branch net 的输入。
3. **Branch nets**：n 个独立 FNN（默认 2 层 200 宽），每个编码一个输入函数，输出 p 维向量。
4. **Trunk net**：一个 FNN 编码输出域坐标 y，输出 p 维向量（与 branch 同维）。
5. **Fusion**（low-rank 版本，默认）：
   - 所有 branch 输出和 trunk 输出做逐元素 Hadamard 积：g1 ⊙ g2 ⊙ ... ⊙ gn ⊙ f
   - 然后做 summation（所有分量求和）加 bias
   - 公式：G̃(v1,...,vn)(y) = S(g̃1 ⊙ g̃2 ⊙ ... ⊙ g̃n ⊙ f̃(y)) + b
6. **输出**：output function u(y) 在任意 y 处的值（无需插值）。

**High-rank vs Low-rank**：
- High-rank：trunk 输出是 p1×p2×...×pn 维张量，参数随 n 指数增长
- Low-rank（默认）：所有 branch/trunk 输出 p 维向量，参数规模可控

### 关键技术细节

- **理论核心**：利用 injective tensor product C(K1×...×Kn, Y) ≅ C(K1) ⊗ε C(K2) ⊗ε ... ⊗ε C(Kn) ⊗ε Y 将多输入算子分解为独立空间的乘积
- **逼近定理（Theorem 1）**：对任意连续多输入算子 G 和 ε>0，存在 p_i, q_i 和连续向量函数 g_i，使 sup ‖G - u⟨g1,...,gn⟩‖ < ε，其中 g_i 编码投影坐标，u 是张量
- **Theorem 2（分离定理）**：总误差 ≤ 算子逼近误差 ε + 投影误差 M·Σ L_i^ε(q_i)，投影误差随 q_i→∞ 趋于 0
- **线性情况（Corollary 5）**：如算子对某输入分量线性，则对应 branch 可用线性网络
- **Bias 作用（Corollary 3）**：加 bias 能降低 M = (max-min)/2，减少误差常数
- **Schauder basis**：Faber-Schauder（C[0,1] 的分片线性基）或 Fourier 基（L²[0,1] 的正交基）
- **网络配置**：branch 和 trunk 均使用 FNN，激活函数 ReLU，优化器 Adam
- **训练**：MSE loss，100K epochs，learning rate 0.001（advection 用 0.0002）
- **数据采样**：每个输入函数在 [0,1] 上等距 100 点采样；输出函数可用等距网格或随机点
- **代码实现**：DeepXDE 库，GitHub: https://github.com/lu-group/mionet

### 核心结果

**ODE system**（非线性，f1, f2 → u1）：
| 模型 | 宽度 | 参数 | L² 相对误差 |
|------|------|------|-------------|
| MIONet | 200 | 161K | **1.69 ± 0.13%** |
| DeepONet (same size) | 312 | 161K | 2.41 ± 0.27% |
| DeepONet (best) | 300 | 151K | 2.26 ± 0.14% |

**Diffusion-reaction system**（D, g → u）：
| 模型 | 宽度 | 参数 | L² 相对误差 |
|------|------|------|-------------|
| MIONet | 200 | 161K | **1.97 ± 0.11%** |
| DeepONet (same size) | 312 | 161K | 5.25 ± 0.38% |
| DeepONet (best) | 400 | 242K | 5.18 ± 0.11% |

**Advection-diffusion system**（D, u0 → u，带周期性）：
| 模型 | 宽度 | 参数 | L² 相对误差 |
|------|------|------|-------------|
| MIONet | 300 | 422K | 1.98 ± 0.07% |
| MIONet (periodic) | 248 | 422K | **1.29 ± 0.09%** |
| DeepONet (same size) | 343 | 424K | 7.83 ± 0.49% |
| DeepONet (best) | 300 | 332K | 7.70 ± 0.69% |

**关键发现**：
- MIONet 在所有多输入问题上显著优于 DeepONet（将多输入拼接传入单 branch 的 baseline）
- Diffusion-reaction 问题上 MIONet 误差仅为 DeepONet 的~1/3（1.97% vs 5.18%）
- 嵌入先验知识（周期性层、线性 branch）可进一步提升精度（1.29% vs 1.98%）
- 多 trunk net 拆分（x 和 t 分别用不同 trunk）是可行的扩展

### 明确局限

- **实验规模有限**：仅 3 个 toy 问题（ODE 系统、diffusion-reaction、advection-diffusion），输入维度均 ≤2，无真实应用场景
- **无高维/大规模测试**：没测试 n>2 输入、高维空间、复杂几何的场景
- **仅使用 FNN**：branch/trunk 只用 FNN，未探索 CNN/ResNet/Transformer 等 backbone
- **无系统 benchmark**：没有与其他可能的 baseline（如简单堆叠多个 DeepONet）做系统对比
- **无训练效率分析**：未报告训练时间、收敛曲线、对超参数的敏感性
- **低秩假设**：low-rank 版本假设算子可被低秩张量逼近，但未对实际问题的秩做定量分析
- **输入端依赖预定义的 sensor 位置**：函数投影需要固定采样点，无法处理非均匀或不规则网格
- **论文自己指出**："more experiments should be done to test the performance of MIONet on diverse problems"
- **跟我们的关系**：本文是 neural operator 的早期工作（2022），方法学成熟但尚未涉及 FEX（Finite Expression）领域

### 跟我们的关系

对 0616-fex 课题：
- **不撞车**：MIONet 是 neural operator 架构（学习算子映射），FEX 是符号回归（发现数学表达式），两者的根本目标和输出形式完全不同。MIONet 输出的是一个可求值的神经网络函数，FEX 输出的是一个可解释的数学公式。
- **可用工具**：
  - MIONet 的 Schauder basis 投影技术（将无限维函数映射到有限坐标）可用于 FEX 的输入预处理
  - 周期性 encoding（cos(2πx), sin(2πx)...）是可用的特征工程技巧
  - Low-rank tensor product 思想可作为 FEX 中组合多个子表达式的理论基础
  - 可利用 MIONet 作为 FEX 的对比 baseline：对一个 PDE 同时跑 MIONet 和 FEX，展示可解释性 vs 精度的 trade-off
- **不引也没问题**：如果 FEX 论文的焦点是符号回归而非 neural operator，MIONet 可以只作为 related work 中 neural operator 方向的代表引用。但如果 FEX 声称可以替代 neural operator 解决某些问题，则必须引用并做对比实验。

## Read by: 0616-fex (2026-06-24)

MIONet 是一篇高质量的方法学论文，在 neural operator 领域有重要贡献（被引 296 次）。对 FEX 课题而言，最相关的 insight 是：

1. **不构成撞车风险**：MIONet 学习的是隐式的神经网络算子映射，输出无法闭合为数学表达式，与 FEX 的符号回归完全正交。FEX 输出可读公式，MIONet 输出可求值的黑盒网络。

2. **可借鉴的技术**：
   - Schauder basis 投影框架可用于 FEX 的输入函数离散化
   - 周期性 encoding（cos/sin features）可直接复用
   - 多 trunk 拆分思路（空间+时间分别编码）适配时空 PDE 的 FEX 特征构造

3. **差异化定位**：如果 FEX 声称解决 PDE 的 operator learning 问题，必须引用 MIONet 并强调"可解释性 vs 黑盒预测"的核心差异。FEX 的优势在于输出显式公式（可分析、可外推、可嵌入理论），这是所有 neural operator 都做不到的。
