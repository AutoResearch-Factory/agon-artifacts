# Improving Coverage Accuracy of Block Bootstrap Confidence Intervals (0804.4361)

- **Year**: 2008.04
- **Authors**: Stephen M. S. Lee, P. Y. Lai（Department of Statistics and Actuarial Science, University of Hong Kong）
- **Venue**: arXiv preprint；正式发表为 Lee, S.M.S. and Lai, P.Y. (2009). "Double block bootstrap confidence intervals for dependent data." *Biometrika*, DOI: 10.1093/biomet/asp018（S2 记录该发表版被引 15 次；arXiv 条目本身在 S2 引文图里显示 0 次，是 preprint/DOI 记录分裂的常见伪影，非真实零影响力）
- **Keywords**: block bootstrap, dependent/time-series data, coverage calibration, Studentization, double/nested (iterated) bootstrap, Edgeworth expansion, block length selection, weakly dependent processes
- **One-liner**: 双层嵌套 block bootstrap，通过覆盖校准或直接蒙特卡洛估计 Studentizing 因子，把依赖数据 block bootstrap 置信区间的覆盖误差从 O(n^{-1/3}) 提升到 O(n^{-2/3})。

## 精华提炼

### 管线/方法

完整管线（以构造 level-α 单侧置信上界为例，双侧类似）：

1. **输入**：严格平稳、离散时间的 d 维依赖序列 X_1,...,X_n；目标参数 θ=H(μ)（"光滑函数模型"，涵盖样本均值/方差/自相关系数/Yule-Walker 估计等一大类估计量）。
2. **第一层 block bootstrap**：选定 block 长度 ℓ，构造重叠 block Y_{j,ℓ}=(X_j,...,X_{j+ℓ-1})（j=1..n-ℓ+1）；从这 n-ℓ+1 个 block 中有放回抽 b=⌊n/ℓ⌋ 个，首尾拼接得第一层 bootstrap 序列 X*（长度 bℓ）。由 X* 算出 bootstrap 分布 G*，得到未修正区间 I(α)=θ̂-n^{-1/2}G*^{-1}(1-α)。此区间覆盖误差量级 O(ℓ^{-1}+ℓn^{-1})，在 ℓ∝n^{1/3} 时最优，误差为 O(n^{-1/3})。
3. **第二层 block bootstrap（关键技巧）**：在**每个第一层 block 内部**（不跨越两个被拼接 block 之间的接缝）取长度 k≤ℓ 的子 block Y*_{i,j,k}；从全部 b·(ℓ-k+1) 个可用子 block 中有放回抽 c=⌊n/k⌋ 个拼接为第二层序列 X**（长度 ck），算出 G**。"子 block 不跨接缝"这一构造是本文对早期怀疑论（Davison-Hall 1993、Choi-Hall 2000、Bühlmann 2002 都认为迭代 block bootstrap 因接缝不连续伪影而无效）的直接技术回应。
4. **G** 的两种用法**：
   - **覆盖校准（coverage calibration）**：解方程 P*( 第一层统计量 ≤ G**^{-1}(1-α̂) ) = 1-α，得到校准后名义水平 α̂，报告 I_C(α)=I(α̂)。
   - **Studentization**：从 G*、G** 直接蒙特卡洛估计第一层统计量标准差 τ̂ 与第二层统计量标准差 τ*，构造 pivotal 分布 J*(x)=P*(第一层统计量/τ* ≤ x)，报告 I_S(α)=θ̂-n^{-1/2}τ̂J*^{-1}(1-α)。
5. **调优规则**：设 k=ℓ/2、ℓ∝n^{1/3}，则 I_C(α) 与 I_S(α) 渐近等价（差异 O_p(n^{-7/6})），覆盖误差降到 O(n^{-2/3})——比未修正的 I(α) 提升整整一个阶。k=ℓ/2 是唯一需要设的超参数，且是固定比例、无需数据驱动搜索。

### 关键技术细节

- **正则条件 (A1)-(A5)**（采自 Götze-Hipp 1983 经 Lahiri 2003 §6.5 修改的标准依赖数据高阶渐近框架）：(A1) 35+δ 阶矩存在（很强的矩条件）；(A2) 长程协方差非退化；(A3) 协方差增长下界；(A4) 通过辅助 sub-σ-field 定义的混合/弱依赖条件（覆盖线性过程、m-依赖、平稳齐次 Markov 链、平稳高斯过程）；(A5) H 四阶连续可微、四阶导数多项式有界（Bhattacharya-Ghosh 1978 光滑函数模型）。
- **主定理（Thm 1）**：(i) I(α) 覆盖概率 = α + ℓ^{-1}·(block bootstrap 方差估计量的 O(1/ℓ) 偏差项，系数含 χ_{2,2}) + n^{-1/2}·(偏度型项) + O(ℓ^{-2}+ℓn^{-1})；(ii) I_C(α)、I_S(α) 覆盖概率 = α + (2ℓ^{-1}-k^{-1})·(同一偏差项) + O(k^{-2}+ℓn^{-1})——**这个系数 (2/ℓ-1/k) 在 k=ℓ/2 时恰好为零**，这正是 k=ℓ/2 能抵消一阶偏差的代数原因。
- **偏差抵消的直观机制**：第一层 block bootstrap 方差估计量偏差为 O(1/ℓ)；把第二层 bootstrap 方差估计量视为对"第一层 bootstrap 方差估计量"本身的估计时，其偏差为 O(1/k-1/ℓ)——两个偏差项共享同一常数 χ_{2,2}，因此第二层重采样"免费"提供了对第一层偏差的估计，完全不需要解析推导 χ_{2,2}。
- I_C 与 I_S 被证明渐近等价（差异 O_p(k^{-2}n^{-1/2}+ℓn^{-3/2})）——意味着实践中可以只做更简单的校准（a），不必再为复杂统计量手推 Studentizing 因子（b）。
- **证明技术**：多层嵌套累积量/特征函数展开，把求和按索引距离切成"近"（L_n=K log n 以内）与"远"（混合条件下指数可忽略）两部分，逐层通过条件期望 E*、E** 传递——附录证明约 600 行，含 6 个引理（block 和的矩、特征函数展开等）。
- **模拟设置**：n=500（ℓ=8, k=4）、n=1000（ℓ=10, k=5）；ARCH(1)/MA(1)/AR(1) 三种过程；θ=均值/方差/滞后1自相关；α∈{0.05,0.10,0.90,0.95}；1000 条独立序列 × 1000 第一层 × 1000 第二层 bootstrap 重采样；对比对象为未修正 I(α)、本文 I_C/I_S，以及两个闭式 Studentization 竞品 I_DH（Davison-Hall 1993）、I_GK（Götze-Künsch 1996，核权重常数 c=0.5）。

### 核心结果

- **方差例子（表2）是未修正 block bootstrap 最差的场景**：ARCH(1)、n=500、α=0.90 时，I(α) 实际覆盖率仅 0.840（名义 0.90，低估 6 个百分点）；I_C 修正到 0.887、I_S 到 0.888，接近腰斩误差，与 I_DH(0.881)、I_GK(0.883) 相当。
- **均值例子（表1）**：五种方法表现接近——均值的 Studentizing 因子本身接近常数，纠偏空间小，是"简单场景"。
- **自相关例子（表3）暴露了闭式方法的脆弱性**：AR(1)/MA(1) 序列在小 α（如 α=0.05）时，I_GK 严重失准（实际覆盖率仅 0.022-0.026，不到名义值一半！），而 I_C/I_S 保持接近 I_DH 与名义水平（如 n=500 AR(1) 下 I_C=0.057）——说明解析推导的 Studentizing 因子（GK）可能对某些统计量系统性偏差，而纯重采样的 I_C/I_S 因为经验估计校正量而更稳健。
- **总体结论**：I_C/I_S "优于或至少不劣于" DH/GK，且不需要为每个新统计量单独解析推导 Studentizing 因子——这种"通用性"（对任意光滑函数模型统计量都适用）是本文相对 Lahiri(1992)/Davison-Hall(1993)/Götze-Künsch(1996) 等此前方法的核心卖点。

### 明确局限

- **作者自陈局限**：(1) 理论限定在光滑函数模型（θ=H(μ̄)）；声称（但未证明）可推广到 von Mises 型泛函/估计方程；(2) 明确排除长程依赖（long-range dependence），列为未来工作——(A3)(A4) 的混合条件要求指数衰减依赖，长记忆过程不满足；(3) 双 bootstrap 的额外计算成本是"必须付出的代价"，论文明确承认。
- **读出的局限（与本任务相关）**：(a) 全部理论和模拟都建立在"单条长平稳序列被切成 block"上——ℓ 与 n 必须联合 →∞（ℓ∝n^{1/3}，b=n/ℓ→∞），完全没有考虑"多个独立物理事件构成 cluster"这种设定；这里的"block"始终是"一条序列的时间切片"，不是"多个独立事件之一"。(b) 35 阶矩条件（A1）对重尾变量是很强的约束（虽然 AUPRC 有界在 [0,1]，未必触发这个问题，但需要留意其构造统计量是否重尾）。(c) 2008 年 preprint，无代码/数据 artifact，需要从公式重新实现。(d) 模拟仅验证了 3 个低阶经典时间序列模型上的标量统计量（均值/方差/滞后1自相关），完全没有验证过类似"两模型配对比较统计量"（如 ΔAUPRC）或多元/高维衍生特征的场景——把 k=ℓ/2 规则和具体误差阶数外推到这类统计量是未经验证的。

### 跟我们的关系

**不撞车判断**：这是一篇纯统计理论论文，不涉及大气科学、因果发现或 CNN，与 `causal-scs-indicator` proposal 的因果估计器/融合机制/骨干网络 non-goals 毫无重叠，不存在 novelty 冲突。它应被当作**统计工具引用**，而非竞争性研究。

**核心问题：本文 k=ℓ/2 的具体调优规则不能直接照搬到 storm-cluster bootstrap。** 原因：本文渐近理论建立在"单条长依赖序列被切成可调长度 block"这一设定——ℓ 和 b=n/ℓ 必须联合 →∞，ℓ∝n^{1/3} 是分析者可自由选择、需要偏差-方差权衡的"分辨率"参数，两层嵌套校准修正的正是这个可调 ℓ 带来的 O(1/ℓ) 偏差。而 proposal 的 storm-cluster bootstrap 里，"storm" 本身就是完整的、物理定义的原子重采样单元，不是从一条长序列里切出来的可调窗口——不存在一个类似 ℓ 的自由参数可供调优。(A1)-(A5) 里 35 阶矩存在、单条序列指数混合衰减这些条件，也是针对"一条序列内部依赖结构"而非"多个 storm 之间是否可交换"设定的，不能直接套用其定理 1 的具体误差阶数（O(n^{-1/3})→O(n^{-2/3})）。

**但可迁移的是更通用的"嵌套/迭代 bootstrap 覆盖校准"算法骨架，与 block 长度机制无关**：本文 Section 2 把第二层分布 G** 的用法明确拆成 (a) 覆盖校准——解一个反解方程得到校准后名义水平 α̂；(b) Studentization——蒙特卡洛估计标准差再做 pivotal 区间；且证明两者渐近等价。这意味着 **proposal 可以只实现更简单的 (a) 路线，不需要为非光滑的 ΔAUPRC（涉及排序/PR 曲线积分）统计量手推 Studentizing 因子**。具体可操作的迁移方案是把"block"换成"storm"（不再切分 storm 内部）：外层对 n 个 storm 有放回重采样 n 个、算 B1 次 ΔAUPRC* 得到外层 bootstrap 分布；内层对每个外层重采样再做一次有放回子重采样（B2 次），仿照本文 Section 2 的校准方程反解出让外层区间实际覆盖率匹配名义 95% 的 α̂。这是标准的（Efron-Tibshirani / Beran / Hall 意义上的）**iid 双层 bootstrap 校准**，不需要"block 内不跨接缝"这类专门构造（那是拼接单条依赖序列才有的伪影，storm 之间没有这个问题）。计算成本 O(B1×B2)，但因为 ΔAUPRC 只是对已缓存的逐 storm 预测重新聚合、不需要重新训练 CNN，即使 B1=B2=999 也很廉价。

**更贴切的理论参照物**：如果要继续深挖"小 n（~50-100）iid/可交换设定下双层 bootstrap 校准"的理论保证，本文同一 senior author（Lee）更早与 Ho 合作的 *Iterated smoothed bootstrap confidence intervals for population quantiles*（math/0504516, 2005）是 iid 版本的迭代校准（针对分位数统计量），比本文更贴近 storm-cluster 的设定，值得作为下一轮检索目标；本文的价值在于提供"更难的依赖数据设定下嵌套校准依然有效"这一上界参照。

**量级旁证**：本文模拟 n=500、ℓ=8（约 62 个 first-level block，与 proposal storm 数量级 50-100 相近）时，方差例子未修正覆盖率 0.840 对比名义 0.90（低估 6 个百分点），校准后修正到 0.887——这是一个数量级相近的具体证据，说明"重采样单元在 50-100 量级时，一层重采样确实会有实质覆盖率偏差，加一层校准能显著但不完全修正它"，可直接引用作为"为什么值得加这层校准"的动机证据，呼应 proposal review 已标记的 v10 阶段 n=8 seed-bootstrap CI 宽度 ±0.08-0.09 检验力不足的历史问题。

**一个重要但非显然的条件性说明**：如果未来发现 storm 之间并非近似独立（强对流事件常呈多日连续爆发/同一槽线连续影响多日，即季节内存在短程序列依赖），那么把"storm-cluster bootstrap"从当前隐含的 iid-cluster 假设升级为"按日期分组的 block-of-storms bootstrap"就会重新变得必要——那种情形下，本文关于"块内子块不跨接缝"的具体构造技巧和 k=ℓ/2 调优规则会重新变得**直接**适用（只是把"block=一段时间序列"换成"block=连续几天的 storm 序列"）。目前 proposal 尚未讨论 storm 间是否存在这种序列依赖，这是一个可以在 Failure Modes 里补充的诊断项。

**该引不该引**：建议在 proposal 下一版 Validation Focus 修补里，把"加一条独立的特权匹配对照校准检验"之外，明确加一句方法引用：对主 Claim 1 的 storm-cluster percentile bootstrap，追加一层 iid 双层校准（nested/iterated bootstrap coverage calibration，参照 Lee & Lai 2008/2009 的校准公式改写为无 block 的 storm-level 版本），而非仅报告未校准的一层 percentile CI；引用位置应放在 Validation Focus/统计方法论部分，不应放进因果推断或强对流预警相关工作段落（领域完全不相关）。

**这篇论文不能解决的问题**：proposal review 里标记的另外三个 Validation Focus 缺口——IID vs regime-迁移 holdout 之间流行率不可比、BH 校正与 CI-阈值判据如何结合、特权匹配对照本身的校准检验——本文完全没有涉及，需要独立的修补方案。本文只回答"给定一个重采样步骤，如何让它的覆盖率更准"这一个具体问题。

**触发背景**：`causal-scs-indicator` proposal v1 review（verdict REVISE）的 Validation Focus 弱项明确指出，`>=50` storm 的样本量沿用 idea 阶段既有估计，未针对 storm-cluster bootstrap CI 能否压到 0.01 实用阈值内做功效/精度分析，且 idea v10 阶段已记录 n=8 seed-bootstrap CI 宽度约 ±0.08-0.09、检验力有限的历史问题未获回应。本次精读的具体问题是：本文的双层嵌套 block bootstrap 能否直接改进 storm-cluster bootstrap 的覆盖率精度。

**结论**：**不能直接照搬，但可迁移其算法骨架**。本文 k=ℓ/2 的具体调优规则和误差阶数证明，是针对"单条依赖时间序列被切成可调长度 block"这一设定；storm-cluster bootstrap 里 storm 是物理定义的原子单元，没有类似 ℓ 的自由参数，因此这套具体机制不能直接套用。但本文 Section 2 提出的**更通用算法**——用第二层嵌套重采样直接反解出使名义覆盖率匹配实际覆盖率的校准 α̂（"coverage calibration"）——不依赖 block 长度机制，可以原样改写为 storm-level 无 block 版本的 iid 双层校准，且计算成本低（复用已缓存的逐 storm 预测，不需重新训练）。建议 proposal 下一版在 Validation Focus 里明确加入这一具体修补：对主 Claim 1 的 storm-cluster percentile bootstrap 追加一层嵌套校准，而非仅报告未校准的一层 CI；理论参照物应转向本文同作者更早的 iid 版本工作 Ho & Lee (2005, math/0504516)。完整推理见上方"跟我们的关系"小节，含一条条件性说明（若 storm 间存在季节内序列依赖，则本文的 block 版本机制会重新变得直接适用）。
