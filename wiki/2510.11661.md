# SR-Scientist: Scientific Equation Discovery With Agentic AI (2510.11661)

- **Year**: 2025.10
- **Authors**: Shijie Xia, Yuhan Sun, Pengfei Liu; Shanghai Jiao Tong University, Shanghai Innovation Institute, GAIR
- **Venue**: arXiv preprint; source uses ICLR 2026 style, acceptance unverified
- **Keywords**: scientific equation discovery, symbolic regression, agentic AI, LLM tools, code interpreter, experience buffer, GRPO, LSR-Synth
- **One-liner**: 让 LLM 用工具长程探索方程，显著提升符号回归精度。

## 精华提炼

### 管线/方法

1. 论文把 scientific equation discovery 明确落在 symbolic regression: 给定数据集 $\mathcal{D}=\{(\mathbf{x}_i,y_i)\}_{i=1}^n$，寻找可解释符号函数 $f(\mathbf{x})$。
2. 核心问题不是提出新的 SR 搜索算子，而是把 LLM 从固定 pipeline 里的 equation proposer 提升为能主动分析数据、评估方程、根据反馈迭代的 agent。
3. 推理阶段由多个 iteration 组成，每个 iteration 有一个 MAPE 精度目标 $G_i$，agent 需要寻找满足目标的方程。
4. agent 轨迹按 ReAct 风格组织: 自然语言 reasoning $r_i$，工具调用集合 $\mathcal{T}_i$，以及工具返回 observation $o_i$。
5. 工具 1 是 `data_analyzer`: 一个 code interpreter，绑定 observed data，允许 agent 写 Python 做数据检查、统计分析、残差分析等。
6. 工具 2 是 `equation_evaluator`: 另一个 code interpreter，agent 提供 Python `equation(...)` 函数骨架，工具用 SciPy BFGS 优化常数参数并返回 MSE、NMSE、MAPE。
7. `equation_evaluator` 最多允许 10 个参数；推理配置里为了公平，BFGS 优化目标使用 MSE，最终实践中按 NMSE 选最优方程。
8. 论文用 MAPE 作为 agent 的目标提示，因为作者认为 MAPE 在不同数据上更适合做统一 precision target。
9. 推理时每个 LLM 的 sampling temperature 是 0.7，单次最大 completion length 是 8192。
10. 每个问题最多 40 个 iterations，每个 iteration 最多 25 个 turns；这个设置把总 LLM calls 控制在约 1000。
11. 为绕过上下文长度限制，系统维护一个 experience buffer $E=\{(e_i,s_i)\}$，记录历史探索过的 equation 和对应 MAPE。
12. 每轮开始时从 buffer 取 top-K 最好方程作为 in-context examples；K 的敏感性实验显示 K=1/3/5 都稳，K=10 下降。
13. stopping condition 是达到最大 iteration 或方程误差足够小；最终提交 observed data 上表现最好的方程。
14. 作者尝试过用 GP 做 experience management，但没有观察到明显提升。
15. 训练框架从数据合成、reward 设计到 GRPO 更新形成完整闭环，目标是让 agent 从 rollout experience 中改进。
16. 训练数据沿用 LLM-SRBench 思路，混合 rule-based 和 model-based synthetic data。
17. 每个 scientific scenario 先由 Claude 4 Sonnet 合成 equation skeleton，skeleton 至少包含一个 known term 和一个 novel term。
18. skeleton 的 term 数限制在 2-4 个，以控制复杂度。
19. 常数实例化不是随机采样，而是让 LLM 按物理意义赋值，再由人类验证。
20. 两名作者独立检查训练 skeleton 与 benchmark equation 是否重复或过近，过近样本会丢弃。
21. 材料科学数据是静态系统: 温度 $T\in[273,573]$ K，应变 $\epsilon\in[0,0.6]$，生成 5000 个 grid points。
22. 化学、生物、物理数据是动态系统: 用 SciPy `solve_ivp` 的 RK45 求解微分方程，时间 $t\in[0,60]$，生成 5000 个点。
23. OOD test set 取最高温度或最高时间的 500 个点；剩余 4500 个点中 500 个做 ID test，其余做 training set。
24. RL rollout 中最大 turns 是 20，训练 60 steps；prompt batch size 是 32，每个 prompt 采样 8 rollouts。
25. RL 基建使用 32 张 NVIDIA H200，`verl` 框架，SGLang 做 rollout engine，FSDP 做 training engine。
26. RL temperature 是 1.0，max response length 是 10240，KL loss coefficient 设为 0 以鼓励探索。
27. reward 使用 log-linear 函数，把 best equation 的 MAPE $s$ 映射到 $[0,1]$，$s_\text{max}=100\%$，$s_\text{goal}=0.1\%$。
28. 作者还比较了 stepwise reward；log-linear 在 Qwen3-Coder-30B 上 overall Acc0.01/Acc0.001 为 40.92/20.69，高于 stepwise 的 37.21/17.83。

### 关键技术细节

1. benchmark 是 LSR-Synth，来自 LLM-SRBench 的 synthetic 部分；作者认为 LSR-Transform 有记忆污染风险，因为近 50% 问题第一轮即可达到很小误差。
2. LSR-Synth 共 129 个问题，覆盖 4 个科学领域: chemistry 36、biology 24、physics 44、material science 25。
3. 每个问题有 training set、ID test set、OOD test set。
4. 主指标是 accuracy-to-tolerance: $\text{Acc}_\tau=\mathds{1}(\max_i |\hat y_i-y_i|/y_i \le \tau)$。
5. 为降低 max operator 对 outlier 的敏感性，评测丢弃最差 5% predictions。
6. 论文认为 $R^2$ 区分度不足；附录表里 LaSR、LLM-SR、SR-Scientist 的 overall $R^2$ 分别是 0.991、0.998、0.998。
7. NMSE 也被作者认为受最大值影响过大；Qwen3-Coder-480B physics 上 LaSR/LLM-SR/SR-Scientist 平均 NMSE 分别是 1.53e-02、3.19e-03、1.29e-03。
8. 非 LLM baseline 包括 GPLearn、E2E、NeSymReS、DSR、uDSR、PySR。
9. LLM baseline 包括 LLM-SR 和 LaSR。
10. 非 LLM 方法每个问题限制 100000 个 equation candidates；LLM 方法限制 1000 次 LLM calls，至多 1000 个候选。
11. GPLearn 设置: population 500、generations 200、tournament size 20，函数集含 add/sub/mul/div/sin/cos/sqrt/log/abs/neg/inv。
12. E2E 设置: max input points 200，refinement trees 10。
13. NeSymReS 设置: 传给 Transformer 的 points 为 500，beam size 32。
14. DSR 设置: samples 100000，batch size 512，learning rate 0.0005，expression length 1-9。
15. uDSR 的 GP 设置: population size 100、generations 20、crossover 0.5、mutation 0.5、tournament size 5、max mutation tree depth 3。
16. PySR 设置: iterations 125、cycles per iteration 550、populations 15、population size 33、maximum size 30、randomization weight 0.1。
17. LaSR 设置: iterations 25、cycles per iteration 550、populations 10、population size 33、maximum size 30、max concepts 20，LLM crossover/mutation/randomization weights 都是 0.02。
18. LLM-SR 使用 `llm-srbench` 默认配置，max LLM calls 1000。
19. backbone 包括 Qwen3-Coder-480B-A35B、GLM-4.5-Air、GPT-OSS-120B、GPT-OSS-20B、Qwen3-Coder-30B-A3B。
20. 主实验只对 Qwen3-Coder-30B 做 RL 训练；附录还在 Qwen3-8B/14B/32B 上验证 RL 有效性。
21. symbolic accuracy 用 GPT-OSS-120B 做初筛，每个 case 评估 10 次；若 10 票不一致，再交给人类 evaluator。
22. 在 121 个 pilot cases 上，10 票一致样本中 LLM 与人类一致率为 98.3%。
23. tool-call analysis 也用 GPT-OSS-120B 分类，54 个 cases 上与人类一致率为 94.4%。
24. tool-call 类型显示 Qwen 和 GLM 约 80% 调用用于 equation evaluation，约 20% 用于 data analysis。
25. GPT-OSS-20B/120B 更倾向自己写 residual error analysis，GPT-OSS-120B 还会直接通过 data analysis 定义 equation constants。
26. RL 后 Qwen3-Coder-30B 增加了 data statistics 调用。
27. 成本估计: GPT-OSS-120B API 单题约 0.25 美元，GPT-OSS-20B 单题约 0.1 美元，未计 cached token 降本。
28. 本地部署 GPT-OSS-120B 在 2 张 H100 上 batch-level 跑 129 个问题，最大 wall-clock time 不超过 5 小时。

### 核心结果

1. GPT-OSS-120B + SR-Scientist 是主表里最强组合: overall Acc0.01 = 63.57，Acc0.001 = 49.35。
2. GPT-OSS-120B 的 LLM-SR baseline 只有 28.16/11.37，LaSR 是 16.02/10.08；因此该 backbone 上提升非常大。
3. Qwen3-Coder-480B 上 SR-Scientist 为 49.09/24.55，LLM-SR 为 41.08/18.09，LaSR 为 11.89/7.49。
4. GLM-4.5-Air 上 SR-Scientist 为 48.32/25.06，LLM-SR 为 35.92/14.47，LaSR 为 14.21/8.53。
5. GPT-OSS-20B 上 SR-Scientist 为 42.64/23.00，LLM-SR 为 33.33/12.92，LaSR 为 12.66/8.53。
6. Qwen3-Coder-30B 原版 SR-Scientist 为 32.30/16.02，RL 后升到 40.92/20.69。
7. 非 LLM baseline 里最强的 uDSR/PySR overall Acc0.01 都是 29.46，远低于 GPT-OSS-120B SR-Scientist。
8. 材料科学上 Qwen3-Coder-480B SR-Scientist 达到 86.67/69.33，GLM 达到 81.33/70.67，Qwen3-Coder-30B + RL 达到 85.33/65.33。
9. 化学上 GPT-OSS-120B SR-Scientist 达到 81.48/64.81，是表中最突出的单领域结果。
10. 生物上 GPT-OSS-120B SR-Scientist 达到 66.67/43.05。
11. 物理上 GPT-OSS-120B SR-Scientist 达到 40.91/34.09，仍是最强，但整体难度更高。
12. symbolic accuracy 绝对值不高，但 SR-Scientist 领先: GLM/GPT/Qwen 分别 7.75/7.00/7.00；PySR 是 4.65，uDSR 是 0.77。
13. OOD 图显示 SR-Scientist 在 OOD data 上总体仍优于其他方法；材料科学从 ID 到 OOD 反而所有方法都有提升。
14. noise robustness 实验加入 $\sigma=\{0.01,0.05,0.1\}$ Gaussian noise；所有方法随噪声下降，但 SR-Scientist 持续领先，尤其 Qwen3-Coder-480B 和 GLM-4.5-Air。
15. GPT-OSS-120B ablation 中移除 `data_analyzer` 后 Acc0.01 从 63.57 跌到 35.66，说明工具化数据分析是关键。
16. GPT-OSS-120B 移除 experience buffer 后 63.57/49.35 跌到 57.36/41.86；随机采样 buffer 而非 top-k 后为 58.14/41.86。
17. Qwen3-Coder-480B 移除 experience buffer 后从 49.09/24.55 跌到 35.66/16.28，随机 top-k 替换后跌到 26.36/13.70。
18. GLM-4.5-Air 移除 experience buffer 后从 48.32/25.06 跌到 37.21/18.61。
19. turn number 实验显示最大 turns 从 10 增加到 25 明显提升；超过 25 后停滞或微降。
20. K 敏感性: K=1 得 63.57/49.61，K=3 得 63.57/49.35，K=5 得 63.31/50.90，K=10 降到 61.50/47.29。
21. 初始 MAPE goal 敏感性: 0.1 得 64.34/48.84，0.01 得 63.31/49.61，0.001 得 63.57/49.35，0.0001 降到 51.74/37.70。
22. Qwen3-32B 附录实验: SR-Scientist 29.46/15.50，RL 后 39.53/20.16。
23. Qwen3-14B: SR-Scientist 18.60/6.98，RL 后 23.26/10.08。
24. Qwen3-8B: SR-Scientist 15.50/4.65，RL 后 20.93/9.30。
25. case study BPG18 显示 agent 通过 high/low population、early/late time 和 $P^2$ correlation 分析，推到 logistic growth 结构。
26. case study PO29 显示 agent 通过 output 与 position/time/velocity 的 correlation 和 linear regression，推到近似 $a=p_0x+p_1$ 的 oscillator 结构。
27. surprising 点: 作者把 autonomous scientist 限定在非常窄的 equation discovery sandbox 内，但仍然靠长程工具交互打败了把 LLM 当 proposer 的 LLM-SR。
28. 另一个反直觉点: 更难的初始目标 0.0001 反而显著伤害性能，说明 agent 的探索需要可达成的 curriculum。

### 明确局限

1. 作者明确说主实验只用 text-only models；多模态输入和 plot analysis 没有验证。
2. 作者明确说 noisy setting 下虽然 SR-Scientist 最好，但仍有明显性能下降。
3. 作者明确说 memory 只给 top-performing equations，agent 可能反复探索历史上差的方程；更抽象的 memory distillation 是未来工作。
4. 论文没有真实湿实验、机器人实验或 lab-in-the-loop；全部结果都在 synthetic symbolic regression benchmark 上。
5. LSR-Synth 虽然有 human/scientific plausibility check，但仍是 LLM 参与合成的 benchmark，不等价于真实科学发现。
6. 训练数据也用 Claude 4 Sonnet 合成 known/novel terms；即使两名作者查重，仍不能完全排除隐性分布重叠。
7. baseline fairness 依赖固定候选数和 LLM call budget；不同方法对 1000 calls 或 100000 candidates 的敏感性可能不同。
8. symbolic accuracy 的第一层 verifier 是 GPT-OSS-120B，虽然不一致样本交给人类，但一致样本未全量人审。
9. 反引文已经出现 2026 年直接竞争方法，例如 STRIDE、Think like a Scientist、LLM-ODE，说明这条线正在快速拥挤。
10. 论文声称 autonomous AI scientist，但实际 autonomy 是在预定义工具、预定义数据格式、预定义目标函数和预定义 benchmark 内发生。
11. `equation_evaluator` 把常数优化交给 BFGS，这降低了 LLM 需要解决的连续优化难度；贡献更偏 agentic search orchestration。
12. 对真实科学任务，数据异常、量纲约束、实验误差模型、变量不可观测性都比 LSR-Synth 更复杂，论文没有覆盖。
13. RL 训练只主跑 Qwen3-Coder-30B；最强 GPT-OSS-120B 没有 RL 后结果。
14. 成本估计基于 common API pricing 和 token 消耗，缺少更细的方差、失败率、重试成本。
15. 论文使用 LLM 做写作 assistant；作者声明未用于核心 idea/method/science content，但外部读者仍只能信赖声明。

### 跟我们的关系

1. 对 `0616-fex`，如果 FEX 指 formula/expression/equation exploration 或 feature-expression discovery，这篇是直接撞车级别的相关工作。
2. 撞车 claim 1: "LLM agent 通过工具主动分析数据并迭代发现方程" 已被 SR-Scientist 明确主张。
3. 撞车 claim 2: "长程工具交互优于单次 proposer 或固定 SR pipeline" 已被 turn-number 和 ablation 支撑。
4. 撞车 claim 3: "experience buffer/top-k memory 帮助科学方程搜索" 已被 ablation 支撑。
5. 撞车 claim 4: "RL 可让 equation-discovery agent 自我进化" 已被 Qwen3-Coder-30B 和 Qwen3-8B/14B/32B 附录实验覆盖。
6. 可用工具: `data_analyzer` / `equation_evaluator` 双工具分离设计很值得复用，尤其是把分析代码和方程评价代码分开。
7. 可用方法: MAPE goal curriculum、BFGS 常数拟合、experience buffer、top-K historical equations、log-linear reward、GRPO with KL=0。
8. 可用评测: LSR-Synth 的 ID/OOD/noise/symbolic accuracy 组合可作为 baseline，但我们需要补真实数据或更强外部分布。
9. 差异化方向 1: 不只发现 equation skeleton，而是加入单位/量纲约束、物理先验约束或可证伪实验设计。
10. 差异化方向 2: 把 memory 从 top equations 升级为失败模式、局部结构、变量变换、残差规律的可检索经验库。
11. 差异化方向 3: 让 verifier 独立于 generator，例如不同模型、符号代数系统、holdout generator 或 human veto。
12. 差异化方向 4: 在真实科学数据、噪声模型、稀疏观测或不可观测变量上验证，而不是只在 synthetic LSR-Synth。
13. 差异化方向 5: 研究 agent 的 data-analysis 行为本身，提出可解释、可复用的 FEX 操作库，而非只看最终 accuracy。
14. 该引必须引；如果我们做 scientific equation discovery、AI scientist、agentic SR、LLM for symbolic regression，这篇属于 2025 年直接 prior work。

## Read by: 0616-fex (2026-06-16)

从 `0616-fex` 视角看，这篇需要作为强 prior work 处理。它已经覆盖 "LLM agent + data-analysis tool + equation-evaluation tool + long-horizon refinement + memory buffer + RL" 这一整套组合，且在 LSR-Synth 上给出比 LLM-SR、LaSR、PySR、uDSR 更强的结果。如果我们的课题只是把 LLM 包成方程 proposer，或者只是让 agent 多轮试 equation，这会明显撞车。
