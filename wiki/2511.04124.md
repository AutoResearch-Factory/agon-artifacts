# Decomposable Neuro Symbolic Regression (2511.04124)

- **Year**: 2026.03
- **Authors**: Giorgio Morales, John W. Sheppard (Montana State University)
- **Venue**: TMLR (Transactions on Machine Learning Research)
- **Keywords**: symbolic regression, neuro-symbolic, multi-set transformer, genetic programming, genetic algorithm, skeleton merging, post-hoc interpretability, decomposable regression
- **One-liner**: SeTGAP: a decomposable SR method that uses Multi-Set Transformer to generate per-variable univariate symbolic skeletons, then merges them via GA+GP cascade to recover multivariate expressions with 100% functional form recovery rate on 13 synthetic benchmarks.

## 精华提炼

### 管线/方法

SeTGAP (Symbolic Regression using Transformers, Genetic Algorithms, and Genetic Programming) 是一个四阶段管线：

**阶段1 — 单变量骨架生成（Univariate Skeleton Generation）**：
- 输入：已训好的回归模型 f_hat（opaque model，实际实验用3-5层、每层500 ReLU的MLP）和原始数据 (X, y)
- 对每个变量 x_v，生成 N_s 组人工输入-响应对 (tilde_X, tilde_y)，其中 x_v 变化、其他变量固定为随机值
- 用预训练的 Multi-Set Transformer g（3层ISAB encoder + 5层decoder，embedding size 512，8 attention heads，在100万合成骨架上预训练）对每组数据生成 univariate skeleton
- 使用 Diverse Beam Search（n_B=3）生成多变体，重复 n_cand=3 次（每次不同的固定值组合），产生最多 9 个骨架候选
- 去除相同和数学等价骨架（用三角函数恒等式简化），用 GA（population=500，tournament selection + binomial crossover）对剩余骨架拟合系数并评估 MSE，保留 top n_cand=3
- 输出：每个变量最多 3 个符号骨架表达式

**阶段2 — 骨架合并（Skeleton Merging）**：
- 核心是 Proposition 1：任意由有限 unary/binary 算子组合而成的函数 f(x) 可表达为规范形式 f(x) = c0 + sum_i c_i prod_j nu_{i,j}(T_{i,j}(x))
- Algorithm merge(ex1, ex2) 递归合并两个骨架：
  - 若都是 Sum：对 exShort 的每个 summand，在 exLong 中找兼容（共享相同算子）的子树；若选到1个则递归merge，若多个则乘积合并
  - 若共享相同 outer operator：递归合并 inner arguments
  - 若都是乘积且全是符号：做 wrapped-product merge
  - 若 outer operator 不同：直接相乘
- Algorithm evolve：对每个合并骨架生成 rep=150 个随机系数实例，独立进化 maxG=300 代（coefficient-only crossover/mutation，骨架结构不变），选最优者

**阶段3 — 级联合并（Cascade Merging）**：
- 按每个变量最优骨架的 MSE 升序排列
- 从 MSE 最低的变量开始，逐步加入新变量，每步最多产生 n_cand^2 = 9 个组合
- 采用 greedy selection：每步保留 top n_cand=3 个合并骨架

**阶段4 — 系数优化（Coefficient Fitting）**：
- 对最终的多变量骨架，用 GA（population=500）在原始数据上最小化 MSE 来拟合系数

### 关键技术细节

- **Multi-Set Transformer 架构**：Encoder（3层ISAB+PMA聚合）+ Decoder（5层标准Transformer decoder blocks），embedding size d=512，8 attention heads
- **训练数据**：100万合成数学表达式（前缀表示法），最大长度20 tokens，额外10万验证集；使用 Adadelta optimizer，lr=0.0001，batch size=16
- **词汇表**：22个tokens（sin, cos, exp, log, sqrt, add, mul, div, pow, abs, tan, tanh, sinh, cosh, SOS, EOS, c, x, integers -3到5, E）
- **训练限制**：表达式最多7个算子、最多2个unary算子、最多1层unary嵌套；不含 arcsin, Pow(.,5), Pow(.,2/3)
- **N_s = 10** 组输入集，每组 **n = 3000** 个点
- **GA 配置**：population=500，tournament selection + binomial crossover + generational replacement，termination when objective change < 10^{-6} for 30 consecutive generations
- **rep=150, maxG=300** for Algorithm evolve
- **P_max=5000** for skeleton combination population（实际从未达到上限）
- **运行环境**：AMD EPYC 7H12（32 physical cores），30GB RAM，无GPU加速
- **典型运行时间**：E1（2变量）约43小时，E4（4变量）约101小时，E2（3变量）约13小时

### 核心结果

**13个合成Benchmark（E1-E13）上的功能形式恢复率（10次独立运行）**：
| 方法 | 成功率 |
|------|--------|
| SeTGAP | **130/130 (100%)** |
| PySR | 47/130 (36%) |
| TaylorGP | 22/130 (17%) |
| NeSymReS | 24/120 (20%, E4/E5不适用) |
| E2E | 7/130 (5.4%) |
| uDSR | 44/130 (34%) |
| TPSR | 17/130 (13%) |

**关键发现**：
- SeTGAP 是唯一在所有13个问题、所有10次迭代中一致恢复正确函数形式的方法
- 其他方法在部分简单问题上也能成功（如 E10 sin(x0*e^x1) 和 E11 x0*log(x1^4)），但在复杂交互问题上完全失败
- SeTGAP 的 extrapolation MSE 在大多数问题上显著优于所有对比方法（包括 opaque NN 本身），表明学到的表达式泛化更好
- 对噪声鲁棒：在 sigma_a = 0.01/0.03/0.05 噪声下仍能恢复正确的函数形式，但含指数项的问题（E3, E5）extrapolation误差较大（系数微小误差被指数放大）

**Feynman 数据集（78个问题）**：
- 符号解恢复率：约61.2%（排名第二）
- R²：在73/78个问题上最低或统计可比（Tukey HSD, p<0.05）
- MSE：在46/78个问题上最低或统计可比（Kruskal-Wallis + Dunn's test with Holm correction）
- 即使骨架正确，MSE可能非零（如 II.34.29a: SeTGAP MSE≈1e-5，GP-GOMEA MSE≈1e-32），因为系数拟合引入微小误差

**SRBench++ 测试**：
- F1-F3：10/10全部成功恢复
- F4：失败（需要8个算子、3个unary算子，超出Multi-Set Transformer能力范围）

### 明确局限

1. **表达能力受限**：Multi-Set Transformer 训练数据仅含最多7个算子、2个unary算子、1层嵌套——这导致F4 failure。作者建议通过 transfer learning 扩展
2. **计算成本极高**：单次运行需数小时到数天（E4: ~101小时），远高于对比方法。但作者认为在科学发现场景中，可解释性优先于速度
3. **依赖 opaque model 质量**：若 f_hat 对目标函数近似差，Multi-Set Transformer 可能无法识别正确骨架
4. **合并过程随机性强**：Algorithm merge 包含多个随机操作（shuffle, sample, random choice），每次运行可能产生不同结果
5. **变量排序启发式**：MSE-based ordering 在某些问题上表现良好，但在 II.6.11 和 II.13.17 上不同排序产生几乎相同的结果——ordering 的影响还需理论分析
6. **仅支持 unary/binary 算子**：不支持微分算子、积分变换等
7. **无真机实验**：全部在合成数据和 Feynman benchmark 上评估
8. **低维限制**：测试最多4个变量，级联合并在高维下计算不可行
9. **无开源代码**（至少文中未提及公开仓库）
10. **三角函数等价检测不完整**：只处理了10种情况，更多等价性留给未来工作

### 跟我们的关系

对 **0616-fex** 课题（关注 feature extraction / representation learning）：

**不撞车**：SeTGAP 的核心贡献是 decomposable SR —— 先将多变量问题分解为单变量骨架预测，再通过结构保持的合并恢复多变量表达式。这与 fex（特征提取/表示学习）在目标和方法上有本质区别。

**可用的工具/方法**：
1. **Multi-Set Transformer**：他们的 Multi-Set Transformer（处理多组输入-响应对、提取共享骨架）是一个有意思的 permutation-invariant 多集合编码器架构，如果我们需要从多个相关数据集中提取共享特征表示，可以参考其 ISAB+PMA 编码设计
2. **Proposition 1 的分解形式**：任意 unary/binary 组合函数可写成 c0 + sum_i c_i prod_j nu_{i,j}(T_{i,j}(x)) 的规范形式——这对设计可解释的特征提取器有启发
3. **GA-based coefficient fitting**：他们的 GA 用于骨架选择和系数优化，如果我们需要从黑盒模型中提取可解释特征，这种 hybrid 方法可以参考
4. **级联变量合并策略**：逐步加入变量、保持已学结构的 cascading 方法，与 curriculum learning 有共通之处
5. **Diverse Beam Search**：在多集合场景下生成多样化骨架候选——如果我们需要从黑盒模型中采样多种特征解释

**差异化**：
- SeTGAP 依赖预训练的 transformer 进行骨架推断——fex 课题可能不需要这种依赖
- SeTGAP 假设已知数学算子集合——fex 可能需要更灵活的表示
- 计算成本是 SeTGAP 的主要缺陷——fex 课题可以关注更高效的分解/合并方法

## Read by: 0616-fex (2026-06-16)

SeTGAP 对 0616-fex 课题的核心启示在于其"分解-合并"范式：将复杂的多变量函数发现分解为单变量关系提取和结构保持的组合。这一思路对 feature extraction 有启发——如果我们能将高维特征交互分解为低维可解释组件，再用结构保持的方法组合，可能得到既准确又可解释的特征表示。

具体而言：
1. Multi-Set Transformer 的 permutation-invariant 多集合编码能力值得关注——如果我们面对多个相关但非同分布的数据源，这种架构可能有用
2. Proposition 1 的规范分解形式为设计可解释特征提取器提供了数学基础
3. Cascade merging 的逐步变量加入策略与 curriculum/渐进式特征学习有概念重叠
4. SeTGAP 的主要弱点（计算成本、算子集限制、低维适用）提醒我们：在 fex 课题中需要更高效的分解/组合方法，且不能预设算子集合

**撞车评估**：不撞车。SeTGAP 针对 symbolic regression 场景设计，依赖已知数学算子集合和预训练 transformer，与我们关注的通用 feature extraction 有本质区别。但其"分解→合并"的思想可以作为 fex 课题的方法论参考。
