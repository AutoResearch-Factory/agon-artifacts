# Rethinking Entropy Interventions in RLVR: An Entropy Change Perspective (2510.10150)

- **Year**: 2025.10
- **Authors**: Zhezheng Hao (Zhejiang Univ), Hong Wang (Tencent), Haoyang Liu, Jian Luo, Jiarui Yu, Hande Dong (Tencent), Qiang Lin, Can Wang, Jiawei Chen (Zhejiang Univ & Hangzhou High-Tech Zone Institute)
- **Venue**: arXiv (ACL format)
- **Keywords**: RLVR, GRPO, entropy collapse, token-level entropy change, entropy dynamics, policy gradient, LLM reasoning, exploration-exploitation, advantage shaping, clip ratio
- **One-liner**: 推导了 token 级熵变的精确一阶近似，揭示四个控制因子，统一了解释现有启发式熵干预方法的框架，并提出 STEER——基于熵变估计的自适应 token 级重加权方法。

## 精华提炼

### 管线/方法

**输入**: query $q$ 从数据集 $\mathcal{D}$ 采样 → GRPO 从 $\pi_{old}$ 采样 $G$ 个 responses $\{o_i\}_{i=1}^G$

**STEER 核心流程**:
1. 对每个 token position $(i,t)$，用 Theorem 1 的 $\Omega(s)$ 公式计算 token 级熵变估计（使用 clipping indicator $\mathbb{I}_{\text{clip}}$、advantage $A$、old policy $\pi_{old}$、当前 policy $\pi_\theta$、token 熵 $\mathcal{H}(s)$）
2. 将 $|\Omega(s)|$ 在 batch 内归一化
3. 通过指数衰减函数 $\lambda(s) = \exp(-\alpha \cdot |\Omega(s)| / \max |\Omega(s)|)$ 映射为 token 权重
4. 将 $\lambda(s)$ 乘入 GRPO loss 中每个 token 的梯度贡献，自适应降低大熵变 token 的更新幅度
5. 等价于调 $\lambda_{\min} = \exp(-\alpha)$ 控制最小权重（实践中设 $\lambda_{\min}=0.7$）

### 关键技术细节

**Theorem 1（Token 级熵变）**:
- 在 logit-independent softmax 假设下，单步 GRPO 更新后的 token 级熵变一阶近似为：
  $$\Omega(s) = -\frac{\eta}{L} \mathbb{E}_{a\sim\pi_\theta}[\frac{\mathbb{I}_{\text{clip}} A}{\pi_{old}} \cdot \pi_\theta(1-\pi_\theta)(\log\pi_\theta + \mathcal{H}(s))]$$
- 逼近误差 $O(\eta^2)$，实践中 $\eta < 10^{-4}$，误差极小
- 该定理可推广到 PPO、RLOO、OPO 等算法（附录有证明）

**四个控制因子**:
1. $\mathbb{I}_{\text{clip}}$：clip 操作，过滤极端 importance ratio 的 token
2. $A$ 和 $\pi_{old}$：advantage 和旧策略概率作为熵变权重
3. $\pi_\theta$：token 生成概率
4. $\mathcal{H}(s)$：当前状态的 token 条件熵

**四象限框架**（$(A, \pi_\theta)$ 空间）:
- Quadrant I ($A>0$, $\pi_\theta$ high)：Exploitation → 熵减
- Quadrant II ($A>0$, $\pi_\theta$ low)：Exploration → 熵增
- Quadrant III ($A<0$, $\pi_\theta$ low)：Suppression → 熵减
- Quadrant IV ($A<0$, $\pi_\theta$ high)：Error-Correction → 熵增

**实验配置**:
- 训练数据：DAPO-Math-17k（17K prompts）
- 模型：Qwen2.5-Math-1.5B/7B/14B, Qwen2.5-7B/14B, Llama-3.2-3B, Mistral-7B
- RL 算法：GRPO（主）、RLOO、OPO（泛化验证）
- 训练超参：LR=1e-6, batch=512 gen / 32 update, rollouts=8, top-p=1.0, temp=1.0, max 200 steps
- STEER 唯一超参 $\lambda_{\min}=0.7$（所有实验统一，无需 per-task 调参）
- 评测：AIME24, AIME25, AMC23, MATH500, Minerva, OlympiadBench（数学）；LCB-v5, Internal, Zeta（代码）
- 硬件：NVIDIA H20 GPU 集群
- 所有主实验结果取两次独立训练平均

### 核心结果

**主表（Qwen2.5-Math-7B, 6 benchmarks）**:
- STEER 平均 48.6%，vs OPO 46.4%（+2.2%），vs Clip-Cov 45.7%（+2.9%），vs GRPO 44.2%（+4.4%）
- 所有 6 个 benchmark 上 STEER 全部最优

**14B 模型**: STEER 45.2% vs GRPO 42.9%（+2.3%）

**代码任务**: STEER 在 3 种模型规模 × 3 个测试集上全部超越 GRPO（至少 +1%）

**Pass@1024**: STEER 在 AIME24/25 的 Pass@256/512/1024 上全部最优

**极端场景**（$\varepsilon_{\text{high}}=5, \varepsilon_{\text{low}}=0.99$，几乎无 clip）:
- GRPO 和 Entro. Adv. 趋向熵坍缩
- Entropy Loss 导致熵爆炸
- Clip-Cov 无法稳定控制
- STEER 初始下降后稳定，最终 entropy 维持在适中水平；同时测试精度最高（48.5% vs Clip-Cov 45.8%）

**泛化**:
- RLOO + STEER: 45.8 → 46.8
- OPO + STEER: 46.4 → 47.5
- Llama-3.2-3B、Mistral-7B 上均有 consistent gain

**熵变估计精度**（vs 前人 Cov 方法）:
- MSE: Ours ~4-8e-4 vs Cov 0.29-5.37（低 3-4 个数量级）
- SRCC: Ours +0.61~+0.72 vs Cov +0.04~+0.08

**关键反直觉发现**: Entropy-aware Advantage 方法（如 Entro. Adv., GTPO）在高熵 token 上分配更大 advantage——作者发现这反而可能**加速熵坍缩**，因为高熵 token 天然具有更大的熵变幅度，放大其 advantage 会放大熵变波动，当 entropy 处于下降阶段时反而加速下降。

**Ablation**: 指数映射 > 线性映射 > 二值映射（证明连续重加权优于硬截断）；$\lambda_{\min} \in [0.5, 0.8]$ 范围内 STEER 表现一致稳健。

### 明确局限

1. **仅限 verifiable reward 场景**：作者明确指出只研究了有可验证 reward 的 RL 训练，未探索无 verifiable reward 的通用 RL 场景
2. **未涉及 agentic / long-horizon RL**：分析框架是否能推广到多步 agent 交互、长期规划等场景未验证
3. **logit-independent 假设**：Theorem 1 推导依赖 "参数独立 softmax" 假设——梯度更新只改变被采样 token 的 logit，不影响其他 token。这是一个弱假设但并非严格成立
4. **仅基于 GRPO 做理论推导**（虽然声称可扩展到其他算法，但主分析以 GRPO 为锚）
5. **KL 散度项被移除**：实验中去掉了原始 GRPO 的 KL 项（引用 DAPO/GPG/Open 等工作经验），但未深入分析这一选择对 entropy 的交互影响
6. **无真机实验**：纯 language benchmark 评测，无 robotics 或 embodied AI 场景
7. **$\lambda_{\min}$ 虽 insensitive 但仍需手工设定**：没有自适应机制
