# Transformers Don't In-Context Learn Least Squares Regression (2507.09440)

- **Year**: 2025.07
- **Authors**: Joshua Hill (UWaterloo), Benjamin Eyre (Columbia), Elliot Creager (UWaterloo / Vector Institute)
- **Venue**: ICML 2025 Workshop on Reliable and Responsible Foundation Models (Oral)
- **Keywords**: in-context learning, linear regression, transformers, out-of-distribution generalization, spectral signature, singular value decomposition, residual stream, representation geometry, OLS, pretraining data bias
- **One-liner**: 证明 transformers 的 ICL 不是"学会了一个回归算法"，而是对预训练分布的过度记忆；OOD 输入会破坏 residual stream 的 spectral signature，导致性能崩溃。

## 精华提炼

### 管线/方法
1. **任务生成**：合成线性回归 prompt，维度 d=20，context 长度 k=40
   - 权重向量 w 从 N(0, Id) 采样，可选投影 P_w 到低维子空间
   - 输入 x 从 N(0, Id) 采样，可选投影 P_x 到低维子空间
   - label y = w^T x（可选加噪声 ε ~ N(0, σ²)）
2. **Transformer 训练**：GPT-2-style (12 layers, 8 heads, hidden dim 256, no dropout)，在大量 ICL prompt 上自回归预训练
   - 最后加一个 linear layer 将 token embedding 投影到标量输出
   - AdamW, lr=1e-4, 500k steps（延长到 1M-1.5M 或加 weight decay 无显著变化）
   - Curriculum：初始只暴露部分维度和少量 context pairs，每 2000 steps 逐步放开
3. **OOD 评估设置**：
   - 将 R^20 分解为两个 10 维正交子空间 A 和 B
   - Input Subspace Restriction：训练时 P_x=P_A（只看到 A 中的输入），测试时分别用 P_A（in-dist）、P_B（正交）、I_d（全空间）
   - Weight Subspace Restriction：训练时 P_w=P_A（只看到 A 中的任务向量），测试时用 P_B、I_d
4. **Spectral Signature 分析**：
   - 收集 final readout head 之前的 residual stream 表示 Z_p ∈ R^{(k+1)×d}
   - 对每个 prompt 的 Z_p 做 SVD：Z_p = U_p Σ_p V_p^T
   - 比较不同分布的奇异值谱、singular vector 对齐度（cosine similarity vs canonical set）
   - 用 bivariate Gaussian 拟合 C_p 前两个分量做 OOD 检测

### 关键技术细节
- **模型架构**：GPT-2-style, 12 layers, 8 attention heads per layer, hidden dimension = 256, no dropout, final linear projection to scalar
- **数据维度**：d=20 (输入维度), k=40 (context 长度), 每个 token 维度 = 20（y_i 首维是 scalar label，其余 19 维补零）
- **训练超参数**：AdamW, lr=1e-4, 500,000 steps（≈9h on RTX A4500）；weight decay 0 vs 1e-6 无显著差异
- **Curriculum 策略**：初始 d_start=15（隐藏最后 15 维）, k_start=11；每 2000 steps d_start--, k_start+=2；直到 d_start=0, k_start=k=40
- **Spectral 分析参数**：batch size b=128 prompts, canonical 集 20 prompts
- **Baselines 对比**：OLS (pseudoinverse), Ridge Regression, Bayesian Linear Regression (Gaussian prior τ=1, posterior 采样 m 个 weight vectors 平均), Kernel Ridge Regression (RBF kernel), Gradient Descent (内循环迭代)
- **Scaling 实验**：train on s∈{1,2,3} 混合分布可以提高 OOD 鲁棒性
- **LLM 扩展实验**：Qwen3-4B, Caesar cipher decode, shift=13 (in-dist, 43% acc) vs shift=4 (OOD, 10% acc)

### 核心结果
| 实验 | 关键发现 |
|------|----------|
| Input Subspace Restriction | Transformer 在正交子空间上性能崩溃，OLS 无此问题；convex combination 实验显示误差随 t 平滑增加 |
| Weight Subspace Restriction | 相同结论——即使权重向量不直接暴露给模型，分布漂移也致命 |
| 同分布内对比 | 在 overdetermined 区域（i > d），OLS 误差比 Transformer 低数个数量级——**即使在 ID 下，transformer 也达不到 OLS** |
| 隐式权重向量提取 | 用 pseudoinverse 从 query predictions 反推的 β，在训练子空间投影的 norm 均值 0.35，正交子空间仅 0.04 |
| Spectral Signature | ID 输入的前 2 个 singular values 主导（sharp drop），OOD 的谱平坦且方差大；前 2 个 singular vectors 高度对齐（cosine near 1） |
| Prediction 依赖 | ID 输入的前 2 个 singular vectors 占据 readout head 幅度的 ~90%（S_∥: 0.897），OOD 仅 0.352 |
| OOD Detection | Gaussian fit 方法：ID 检测率 94.45% ± 4.29%，OOD 仅 19.39% ± 12.46%（T_∥） |
| Scale Shift | OLS/Ridge 对 input scale 完全不敏感，Transformer 在 scale 偏离训练分布时误差急剧上升；多 scale 训练可缓解 |
| Label Noise | 在 noisy labels 上训练的 Transformer 在 clean test set 上表现类似 Ridge，远差于 OLS 和 clean-trained Transformer |
| LLM 扩展 | Qwen3-4B Caesar cipher：OOD shift=4 的谱更平坦、vector 对齐更差，与合成实验一致 |
| 子空间维度变化 | 5/10/15 维训练子空间均表现相同 OOD 失败模式，误差在训练子空间维度数处饱和 |

**Surprising results**：
- 即使 ID，transformer 的渐近误差比 OLS 差数个数量级——这与之前声称"transformer 可以匹配 OLS"的结论直接矛盾
- 分布漂移不仅在 input space 致命，在 weight space（模型从未直接看到的）同样致命
- 仅需 2 个 singular vectors 就足以区分 ID/OOD 并预测 loss

### 明确局限
1. **仅限合成线性回归**：所有主要实验在 d=20 合成数据上完成，结果能否推广到真实 NLP/多模态 ICL 任务未知（附录有 Caesar cipher 初步实验但规模小）
2. **仅 GPT-2 架构**：未测试其他架构（Llama-style, Mamba 等）
3. **Spectral signature 的成因未解**：只知道它在训练中形成，但形成机制、与具体架构组件（layer norm, attention, MLP）的关系未分析
4. **无真机部署验证**：OOD detection 方法是 offline 分析，未集成到实际监控 pipeline 中
5. **训练分布单一**：只在 Gaussian 分布上实验，未测试其他分布族（uniform, mixture 等）
6. **无理论解释**：解释了"不是什么"（不是 OLS/GD），但未给出正面解释"transformer 到底学了什么机制"
7. **OOD 检测阈值需要单独校准**：未给出跨任务泛化的检测阈值
8. **语言模型实验规模小**：仅 Qwen3-4B 一个模型、一个 Caesar cipher 任务、batch 150

## Read by: 0616-fex (2026-06-25)

### 撞车评估：不撞车
本文研究 ICL 中的线性回归算法等价性问题，核心问题是"Transformer 的 ICL 是否等价于已知的学习算法（OLS/GD）"，与 fex（特征提取/表示学习）是不同方向。

### 可借鉴方法：
1. **Spectral Signature OOD 检测 pipeline**：收集 residual stream 表示 → SVD → canonical singular vectors → Gaussian fit → 置信度检测。这个 pipeline 可以直接移植到 fex 场景，用于检测"特征提取器是否遇到了分布外的输入"。
2. **子空间限制 + 正交子空间评估**：通过投影矩阵系统性地制造可控分布漂移（训练子空间 vs 正交子空间 vs 全空间），可评估 fex 模型的特征提取是否对特定方向敏感。
3. **C_p 向量与 loss 的相关性分析**：将表示与 canonical basis 的对齐度作为特征质量的代理指标，可能帮助 fex 在不依赖下游任务标注的情况下评估特征质量。

### 关键洞察：
本文的核心教训是"看起来学会了算法 ≠ 真的学会了算法；transformer 可能只是在匹配训练分布的统计特征"。这个教训对 fex 同样适用——如果 fex 模型的特征提取在某些 benchmark 上表现好，可能需要检查它是否只是记住了训练数据中的特征模式，而非真正学会了通用的特征提取策略。Spectral signature 方法提供了一种不依赖 downstream task 的检测方式。
