# Symbolic Regression via Neural-Guided Genetic Programming Population Seeding (2111.00053)

- **Year**: 2021.11
- **Authors**: T. Nathan Mundhenk, Mikel Landajuela, Ruben Glatt, Claudio P. Santiago, Daniel M. Faissol, Brenden K. Petersen (Lawrence Livermore National Laboratory)
- **Venue**: NeurIPS 2021 (neurips_2021.sty)
- **Keywords**: symbolic regression, genetic programming, neural-guided search, policy gradient, priority queue training, population seeding, hybrid optimization, combinatorial optimization
- **One-liner**: 用 RNN 生成表达式作为 GP 初始种群（而非随机初始化），在 RNN 训练与 GP 进化之间交替迭代，相比纯 DSR 恢复率提升 9.3%，并引入更难的新 benchmark Livermore。

## 精华提炼

### 管线/方法

整体算法流程（Algorithm 1，Figure 1）：

1. **RNN 采样**（序列生成器）：用一个单层 LSTM（32 hidden nodes）自回归地生成一批 $N=500$ 个数学表达式，表示为 pre-order traversal token 序列
2. **GP 种群播种**：将 RNN 生成的这批表达式直接作为 GP 的初始种群 $\mathcal{T}_{\text{GP}}^{(0)} = \mathcal{T}_{\text{RNN}}$（替代传统 GP 的随机初始化）
3. **GP 进化内循环**：运行 $S=25$ 代 GP 进化（mutation + crossover + tournament selection），每代产出新种群
4. **精英选择**：从所有 GP 代的并集 $\bigcup_{s=0}^S \mathcal{T}_{\text{GP}}^{(s)}$ 中选 Top-$M$（$M=1$）最优个体，与原始 RNN 样本合并为训练集
5. **RNN 训练**（外循环）：用合并后的样本集训练 RNN（默认 PQT），更新参数 $\theta$
6. **GP 重启**：下一次迭代 GP 从头开始，以更新后的 RNN 新样本为起点——形成 "RNN 学会更好起点 → GP 精炼 → 反馈 RNN" 的正向循环
7. 重复至总表达式评估数达 2M 上限

### 关键技术细节

- **RNN 架构**：1 层 LSTM，32 hidden units，自回归生成 4-30 token 的 pre-order traversal
- **训练方法**（三种，默认 PQT）：
  - **VPG**（Vanilla Policy Gradient）：REINFORCE + EWMA baseline
  - **RSPG**（Risk-Seeking Policy Gradient）：只对 top-ε 精英样本做梯度更新（$\varepsilon$ 为风险因子）
  - **PQT**（Priority Queue Training）：维护大小为 $k=10$ 的最大奖励优先队列，做监督学习 $\mathcal{L} = \frac{1}{k}\sum_{\tau \in \text{MRPQ}} \nabla_\theta \log p(\tau|\theta)$
- **GP 参数**：
  - 种群大小 $N=500$，代数 $S=25$
  - 四种变异等概率选择：uniform / node replacement / insertion / shrink
  - Crossover 概率 $P_c=0.5$，Mutation 概率 $P_m=0.5$
  - Tournament selection，tournament size $k=5$
  - 表达式约束（禁止嵌套三角函数如 $\sin(1+\cos(x))$，禁止 inversion 如 $\log(e^x)$）
- **奖励函数**：$R(\tau) = 1/(1 + \text{NRMSE}(\tau))$，其中 NRMSE 归一化到 $\sigma_y$
- **优化器**：Adam，学习率 $\alpha = 0.0025$（比 DSR 的 0.0005 大 5 倍，因为 GP 提供更稳定探索）
- **Entropy regularizer weight**：0.005（但 ablation 表明设为 0 影响不大，因 GP 本身提供足够探索）
- **常数优化**：对于 Jin benchmarks，常数通过 reward 计算时内置优化（不限制常数个数）
- **Hardware**：36 核 Intel Xeon E5-2695，单核运行，Nguyen 平均 4.4 min/run

### 核心结果

| 方法 | Nguyen 恢复率 | R 恢复率 | Livermore 恢复率 | 总体 |
|------|--------------|---------|-----------------|------|
| **Ours (PQT)** | **92.33%** | **33.33%** | **71.09%** | **74.92%** |
| GEGL | 86.00% | 33.33% | 56.36% | 64.11% |
| Random Restart GP | 88.67% | 2.67% | 58.18% | 63.57% |
| DSR (RNN only) | 83.58% | 0.00% | 30.41% | 45.19% |

**Surprising 发现**：
- Random Restart GP 本身就非常强（63.57%），超过 DSR 近 18 个百分点，说明简单的多起点 GP 比精心设计的 RL 训练更有效
- Entropy regularizer 在这套混合方法中不太重要（设为 0 后只降 ~1%），与纯 DSR 中高度依赖 entropy 形成鲜明对比——GP 本身提供充足探索
- "全 off-policy" ablation（RNN 只用 GP 样本训练，不含自身样本）几乎不降性能（74.81% vs 74.92%），说明 GP 精炼后的样本质量远超 RNN 原始样本
- Nguyen-12（$x^4 - x^3 + 0.5y^2 - y$）所有方法均为 0% 恢复率——这是 classic benchmark 中的硬骨头，可能因表达式不对称且含分数常数

**最优 GP 代数**：$S=10$ 到 $S=25$ 表现最佳，与最大表达式长度（30 tokens）大致匹配——代数应正比于表达式长度

**SOTA 水平**：
- Jin benchmark 全部完美恢复（Mean RMSE = 0）vs DSR (0.45) vs BSR (2.42)
- Neat benchmark 平均 Median RMSE = 0.1139 vs DSR (0.1756) vs Neat-GP (0.1977)
- +SLP/HER 后进一步提升到 83.89%（尤其是 R benchmark 从 33.33% 跃升至 98.67%）

### 明确局限

1. **仍有未解决问题**：Nguyen-12 (0%), R-1 (4%), R-2 (4%), Livermore-7/sinh (0%), Livermore-8/cosh (0%)——即使 2M 表达式预算也无法恢复
2. **无真机实验**：纯 benchmark 评估，无真实物理/工程数据验证
3. **无噪声数据测试**：作者在 Future Work 中明确提到"plan on doing in-depth analysis with noisy data"，当前所有 benchmark 使用无噪声精确采样
4. **Off-policy 问题未解决**：VPG/RSPG 训练时 GP 样本是 off-policy 的，标准 importance sampling 不适用（GP 分布 intractable），只是实证上 PQT 不受此影响
5. **无标准 benchmark 之外的验证**：仅在 Nguyen/R/Livermore/Jin/Neat 上测试，未在真实科学发现任务（如物理定律发现）中验证
6. **与我们的关系**：该方法是通用符号回归框架，可直接应用于 FEX 的场景（从数据中发现函数表达式）；其 GP+NN 混合思路是 FEX 方法学的重要参考，但它的搜索库局限在 $\{+,-,\times,\div,\sin,\cos,\exp,\log\}$，而 FEX 可能需要更丰富的算子集合

### 撞车评估
不撞车。本文是 2021 年的通用符号回归方法，FEX（0616）的核心命题是 function expression extraction/recovery 在更具体场景中的应用。方法学上，本文的 GP+NN 混合范式可以作为 FEX 的 backbone 之一，但 FEX 仍有差异化空间：
- 本文 token 库固定（基础数学算子），FEX 可扩展物理先验算子
- 本文仅做纯符号回归 benchmark，无 ODE/PDE 发现能力
- 本文无噪声数据处理（作者自己列为 future work）

### 可用的工具
1. **Deep Symbolic Optimization 代码库**（github.com/brendenpetersen/deep-symbolic-optimization）：方法实现的直接基础
2. **Livermore benchmark suite**：22 个更难的符号回归问题，可作为 FEX 的补充评测
3. **约束机制 + 多变异类型 GP**：可复用的工程组件

### 差异化方向
- FEX 若涉及物理定律（ODE/PDE）发现，本文不覆盖
- 噪声鲁棒性是本文的明确弱点，可成为 FEX 的突破点
- 可考虑用 Transformer 替代 LSTM 作为序列生成器（2021 年后的趋势）
