## 精华提炼

### 管线/方法

这是一篇 2 页的理论笔记（note），没有 ML pipeline，只有一条自洽的代数推导链，外加一个用于验证的合成实验图。推导链如下：

1. **符号约定**：标准 2x2 混淆矩阵（TP/FN/FP/TN），定义 `TPR = TP/P`（即 Recall）、`FPR = FP/N`；`P`、`N` 为测试集正负样本数。定义流行率 `π = P/(P+N)`，定义比例 `r = P/N = π/(1-π)`。归一化后的混淆矩阵（除以样本量 n）在 `n→∞` 的总体极限下完全由 `(TPR, FPR, r)` 三个量决定。
2. **ROC 对 r 不变性论证**：ROC（TPR vs FPR）在总体极限下不随 r 变化——这是"immediate"的，因为 TPR、FPR 分别是正类内部、负类内部的比率，不涉及跨类别的样本量比例。经验 ROC 会因采样有限而有波动，但总体 ROC 严格不变。
3. **Precision 关于 r 的闭式解**（论文的核心公式，eq. 1）：`Prec = TP/(TP+FP) = P·TPR / (P·TPR + N·FPR) = TPR / (TPR + FPR/r)`。这一步只是把 TP、FP 用 `P·TPR`、`N·FPR` 代入再除以 `N`——但推论很重要：**在 ROC 上固定同一个工作点 (TPR, FPR)（即固定分类器"能力"不变）时，Precision 是 r 的显式、确定性函数**，`r→∞`（`π→1`）时 `Prec→1`，`r→0`（`π→0`，且 `FPR>0`）时 `Prec→0`。
4. **合成实验验证**（Fig 1，论文唯一的经验图）：2D 高斯生成的二分类问题，训练一个逻辑回归分类器。测试集 A：`r=1`（`P=N=5000`，平衡），画出蓝色 PR 曲线。测试集 B：`r=0.1`（`P=500, N=5000`，不平衡），画出绿色经验 PR 曲线。用 eq. 1 把蓝色曲线（`r=1` 下的经验 `TPR,FPR`）按 `r=0.1` 重新加权，得到红色"预测"曲线。红色与绿色高度吻合，残差归因于绿色曲线样本量更小（因为红色复用了蓝色平衡样本的全部数据做reweighting，绿色是真实抽取的小样本）。
5. **训练集不平衡不影响测试 ROC/PR 的论证**（eq. 2，log-odds 分解）：对概率型/连续打分分类器，`log[p(C+|x)/p(C-|x)] = log[p(x|C+)/p(x|C-)] + log(r_train)`——`r_train`（训练集类别比）只是给判别函数加一个常数平移，不改变按 log-odds 排序阈值扫描得到的混淆矩阵序列本身，因此不影响测试 ROC/PR 曲线的"形状"，只影响某个固定阈值对应哪个工作点。
6. **推广到 F_β**：把 eq. 1 代入 F_β 的加权调和平均定义，代数化简得到 `F_β = (1+β²)·TPR / (TPR + FPR/r + β²)`——同样显式依赖 r。
7. **推广到 Flach & Kull (2015) 的 Precision Gain / Recall Gain**：定义 `PrecG = (Prec-π)/((1-π)·Prec)`、`RecG = (Rec-π)/((1-π)·Rec)`（相对"永远预测正类"这一平凡基线的"增益"，构造上类似气象学中相对气候态基线的技巧分数 skill score，但论文本身未做此类比，是本 wiki 的外部联想，不代表论文正文观点）。用 `1/(1-π)=1+r` 代入 eq. 1 化简，得到本文最重要的结果：**`PrecG = 1 - FPR/TPR`，完全不含 r**；而 `RecG = 1 + r·(1 - 1/TPR)`，对 r 是仿射（线性）依赖。论文强调 PrecG、RecG 各自只依赖 `{TPR, FPR, r}` 三者中的两个。
8. **与前人工作的关系**：指出这一"PR 曲线随 r 变化"的可预测性此前不被充分认知——Fawcett (2006) 画过 `r=1` 和 `r=0.1` 两条 PR 曲线但未指出二者的解析关系；Hoiem et al. (2012) 在比较不同视觉类别的检测 PR 曲线时提出用固定参考 r（"normalized precision"）做跨类别公平比较，是本文结果的一个应用先例；Siblini et al. (2020) 用固定 `r₀` 定义不随 r 变化的 AUPR/F-score/AUPR-Gain。

### 关键技术细节

- 核心公式全部是初等代数变换，**本次精读逐步手动重新推导并核对**：
  - `Prec = TPR/(TPR + FPR/r)` ✓（直接代入验证）。
  - `F_β = (1+β²)TPR / (TPR + FPR/r + β²)` ✓（把 `1/Prec` 代入调和平均定义，通分后与论文式完全一致）。
  - `PrecG = 1/(1-π) - r/Prec = (1+r) - (r·TPR + FPR)/TPR = 1 - FPR/TPR` ✓（`r/Prec` 展开为 `r + FPR/TPR`，与 `1+r` 相减恰好消去 r，仅剩 `1-FPR/TPR`——这一步代数消去是全文最关键、最值得信赖的单一结果，本次精读独立复核无误）。
  - `RecG = 1/(1-π) - r/Rec = (1+r) - r/TPR = 1 + r(1-1/TPR)` ✓。
- Fig 1 实验设置：2D 高斯分类问题 + 逻辑回归，`r=1`（`P=N=5000`）vs `r=0.1`（`P=500, N=5000`）；具体高斯分布参数（均值、协方差）未在正文给出，无代码/数据仓库链接——图本身是"示意验证"而非可精确复现的基准实验。
- 全文所有等式严格意义上是**总体极限（population limit, n→∞）** 下的精确关系；经验（有限样本）ROC/PR 会有额外的采样方差，论文承认但未定量刻画（无方差公式、无置信区间构造）。
- 论文只给出**单一工作点**（固定 `TPR, FPR`）上的 Precision、F_β、PrecG、RecG 随 r 的变化关系，**没有显式讨论 AUPRC（PR 曲线下面积，对 Recall 积分）这一整合量随 r 的变化**——这是一个直接但非平凡的外推，见下方"跟我们的关系"。
- 论文只处理"同一条 ROC 曲线（同一个分类器的判别能力），仅改变 r"的场景，**没有讨论"两个不同总体/regime，可能连 ROC 曲线本身也不同（不只是 r 不同）"这一更一般的场景**——这正是本课题 IID vs regime-迁移 holdout 比较所处的场景，需要额外假设，见下方。

### 核心结果

| 量 | 是否依赖 r | 闭式表达式 |
|----|-----------|-----------|
| Precision（固定 Recall） | 是，显式 | `TPR/(TPR+FPR/r)` |
| F_β（固定 Recall） | 是，显式 | `(1+β²)TPR/(TPR+FPR/r+β²)` |
| Precision Gain (PrecG) | **否，恰好不变** | `1 - FPR/TPR` |
| Recall Gain (RecG) | 是，但只是 r 的仿射函数 | `1 + r(1-1/TPR)` |
| ROC (TPR vs FPR 整条曲线) | 否（总体极限） | 不随 r 变化 |

- 最 surprising 的结果：`PrecG` 表面上看是"相对流行率基线 π 归一化"的量，直觉上应该和 π（进而和 r）有关，但代数上精确抵消为只依赖 `(TPR, FPR)`——这不是近似不变，是恒等式意义上的精确不变。
- Fig 1 的合成实验定量验证了 eq. 1 的预测力：跨 r（`1→0.1`，10 倍流行率变化）预测曲线与经验曲线高度吻合，差异可完全归因于有限样本噪声而非公式误差。
- 训练集不平衡（`r_train`）通过 log-odds 的加性平移论证，被证明不影响**测试** ROC/PR 曲线的形状（只影响阈值到工作点的映射），这是一个独立于主结果但同样有用的推论。

### 跟我们的关系

本节直接回答本轮调研的核心问题：**这篇笔记能否严格论证"IID holdout vs regime-迁移 holdout 因流行率不同，直接比较两者 Delta AUPRC 会混淆技能变化与流行率变化"，并给出流行率调整后的正确比较方法？**

**结论先行：论文提供了完成这一论证所需的关键代数原语（eq. 1 与 PrecG 闭式解），但论文本身没有做到 AUPRC 积分层级、也没有讨论跨总体比较的场景——proposal 如果要引用这篇论文来"严格论证"，必须显式补上以下两步外推，并诚实标注这是外推而非论文原文结论。**

**1）可以直接借用的部分——为什么 Delta AUPRC 跨流行率比较是混淆的（本 wiki 推导，建立在论文 eq. 1 之上）**

标准 AUPRC 定义为 `AUPRC(r) = ∫₀¹ Prec(Rec; r) dRec`。由论文 eq. 1，在总体极限下将 ROC 曲线记为 `FPR(Rec)`（分类器"能力"，与 r 无关），则
`AUPRC(r) = ∫₀¹ [ Rec / (Rec + FPR(Rec)/r) ] dRec`——这是 ROC 曲线 `FPR(·)`（技能）与标量 r（流行率）的联合泛函。**关键推论**：即使两个 holdout 上 causal 分支相对 raw 分支的真实 ROC 级别技能差（`FPR_raw(Rec) - FPR_causal(Rec)`，即"causal 在每个 recall 处比 raw 少多少假阳性"）在两个 regime 下完全相同（即真实"因果增量"零漂移的原假设情形），只要 `r_IID ≠ r_regime`，`Delta(r) = AUPRC_causal(r) - AUPRC_raw(r)` 在两个 holdout 上一般仍不相等——因为 `r ↦ Delta(r)` 是非线性函数。也就是说，**"regime-迁移 Delta 大于 IID Delta"这一观测，在零真实技能漂移的原假设下也可能纯粹由 `r_IID ≠ r_regime` 产生，无法从原始 Delta AUPRC 数字本身区分"真技能变化"与"流行率伪影"**。这正是 proposal review 中 Validation Focus 弱项 (3) 的精确数学根源，本次精读把它从"审稿人的定性判断"补成了"可从论文 eq. 1 严格推导的代数事实"。

**2）流行率调整后的正确比较方法（本 wiki 基于论文 PrecG 结果的外推，两种可行方案）**

- **方案 A（推荐，理论最干净）——用 PrecG 替代 Precision 定义积分指标**：论文证明 `PrecG(Rec) = 1 - FPR(Rec)/TPR(Rec) = 1 - FPR(Rec)/Rec` 完全不含 r。定义 `AUPrecG = ∫₀¹ PrecG(Rec) dRec = ∫₀¹ [1 - FPR(Rec)/Rec] dRec`——这是一个只依赖 ROC 曲线（技能）、不含任何流行率信息的积分量，构造上对 r 完全不变。据此 `Delta_PrecG = AUPrecG_causal - AUPrecG_raw = ∫₀¹ [FPR_raw(Rec)-FPR_causal(Rec)]/Rec dRec`——两个 holdout 各自算出的 `Delta_PrecG` 现在可以直接比较，任何差异只能来自两个 regime 下 causal 相对 raw 的真实 ROC 级别技能差变化，不可能来自流行率不同。**实现成本很低**：proposal 已有的 storm-cluster bootstrap 机制（`factorial_helpers.py`/`cnn_fusion_helpers.py`）可以原样复用，只需把每个 bootstrap 重采样内计算的"每个阈值处的 y 轴量"从 Precision 换成 `1-FPR/Recall`，配对/分箱/CI 构造逻辑不变。需注意数值细节：`Rec→0` 处 `FPR/Rec` 可能因分母趋零而对小样本噪声敏感，需要设定最低 recall 截断或复用现有 AUPRC 在 `Rec=0` 端点的既有约定（如 sklearn 式插值），这是实现层面的通用问题，不是本方案独有的新缺陷。
- **方案 B（更贴合现有协议，保持 AUPRC 量纲）——重投影到共同参考流行率 r₀**：预注册一个共同参考 `r₀`（如两个 holdout 的池化流行率，或直接取 IID holdout 自身的 π 作参考，须在看到主结果前定），对每个 holdout 各自估计的经验 `(TPR, FPR)` ROC 点，用 eq. 1 重新加权到 `r₀` 下的 Precision，得到 `AUPRC(r₀)`，再计算 `Delta(r₀)` 并跨 holdout 比较。优点是**不改变 proposal 已经预注册的 `Delta AUPRC ≥ 0.02`、`CI 下界 > 0.01` 这套阈值的量纲和既有解释**，缺点是需要额外引入一个"参考 r₀"的选择（虽然只要固定且预注册，选择本身不影响比较的合法性）。

两个方案数学上不等价（PrecG 是相对"永远猜正类"基线的归一化增益，量纲与 Precision 本身不同；重投影后的 `AUPRC(r₀)` 仍是 Precision 量纲），但都能达成"消除流行率混淆"的目标。**建议 proposal 把方案 A 列为主指标（因为它不需要任意选定 r₀，是论文结果最直接、最少额外假设的推论），方案 B 作为保持向后兼容、便于向读者解释的次要报告指标**，两者若结论一致可互相印证协议的稳健性。

**3）必须诚实标注的适用边界（本次精读认为这是最容易被忽略、但对 proposal 至关重要的一点）**

论文 eq. 1 的 r-不变性证明，前提是"**同一条 ROC 曲线，只是评估时改变了 r**"——即隐含假设两个待比较总体之间，分类器的类条件打分分布 `p(score|actual 正类)`、`p(score|actual 负类)` 完全相同，唯一变化的是正负类各自的样本比例。**这个假设在 IID holdout vs regime-迁移 holdout 的场景下并不是自动成立的**：regime 迁移（跨季节/跨 regime）很可能不只改变正例流行率，还改变分类器在正类/负类内部的打分分布本身（即 ROC 曲线自己也会漂移）——这恰恰是 proposal 自身"regime 迁移可能揭示 `(D,J)` 更大增量"这一核心假说所依赖的机制。换言之：**方案 A/B 消除的是"纯流行率漂移"这一混淆分量；如果 regime 迁移还伴随真实的 ROC 级别漂移（分布外泛化能力变化），流行率调整后残余的 `Delta_PrecG` 或 `Delta(r₀)` 差异，仍然完全可能包含"raw 与 causal 在新 regime 下泛化能力本身不同"这一 proposal 真正想测量的信号**——这不是缺陷，而正是流行率调整"提纯"出来的、proposal 真正关心的量。因此本次精读的结论是：流行率调整是**必要但可能不充分**的修补——它排除了一个已知的、纯统计的混淆源（流行率），但不能替代、也不涉及 proposal review 中另一条独立弱项——即"regime 迁移 Delta 应大于 IID Delta"这一预测本身借用的 anchor regression / CausalKinetiX 文献假设（良定义的 shift/anchor 变量结构、跨环境局部动力学不变）在本应用中是否成立——这是两个独立问题，**不应在 proposal 修订中被混为一谈**：流行率调整解决"能否公平比较两个数字"，anchor regression 假设解决"两个数字的差异该如何被理论预期解释"，本论文只能且只应该被引用来支撑前者。

**4）具体写入建议**

- Method 节定义 estimand 处，应明确 `Delta AUPRC` 的正式定义要么改为 `Delta_PrecG`（方案 A），要么显式加上"重投影至预注册 r₀"的步骤（方案 B），并引用 Williams (2020) eq. 1 与 Flach & Kull (2015) 的 PrecG 定义作为依据。
- Related Work 节引用本文时，措辞应准确限定其贡献边界：只引用来支撑"Precision/AUPRC 是流行率敏感的度量，PrecG 是其可证明流行率不变的替代量"这一具体技术事实，不要用来支撑"regime 迁移应有更大 Delta"这一预测本身（那部分应继续引用 anchor regression/CausalKinetiX 文献，且需按 review 建议单独说明其假设未被验证）。
- Failure Modes 节可以新增一条诊断：若 `Delta_PrecG`（流行率调整后）与原始 `Delta AUPRC`（未调整）在同一 holdout 内符号/量级差异很大，说明该 holdout 的流行率效应本身就很强，是需要额外报告的稳健性信号。

## Read by: 0710-causal-scs (2026-07-13)

**审读视角结论**：这篇 2020 年短笔记不撞车（不是同一子领域的竞争方法论文），而是一件可以直接拿来加固 proposal Method 节的**数学工具**。它不是"因果特征 vs 强对流预警"这条主线上的相关工作，而是"如何公平比较两个不同流行率数据集上的 AUPRC"这条纯统计方法论支线上的关键引用。

对 proposal v1 REVISE 的具体回应：

1. **Validation Focus 弱项 (3) 的根源已从"审稿人直觉"升级为"可推导的代数事实"**：本文 eq. 1（`Prec=TPR/(TPR+FPR/r)`）本身已经足以证明——固定分类器技能（ROC 曲线）不变、只改变流行率 r，Precision（进而 AUPRC）就会系统性变化。这不是"可能混淆"，而是"结构性必然混淆，除非做流行率调整"。
2. **可执行的修补方案已经具体化到公式级别**：方案 A（用论文证明的 `PrecG=1-FPR/TPR` 构造 `AUPrecG` 作为主指标，对流行率精确不变）与方案 B（用 eq. 1 把两个 holdout 的经验 ROC 重投影到共同参考 r₀，保持 AUPRC 量纲、兼容现有 0.01/0.02 阈值）都可以直接写入下一版 proposal 的 Method 节 estimand 定义，且实现成本低——复用现有 storm-cluster bootstrap 机制，只替换每个阈值处计算的标量。
3. **一个 proposal 尚未意识到、需要显式承认的边界**：本文的 r-不变性证明前提是"两个总体间只有流行率不同，ROC 曲线本身不变"。但 regime 迁移 holdout 存在的意义恰恰是 proposal 怀疑 ROC 曲线本身也会漂移（这才是"`(D,J)` 在 regime 迁移下有更大增量"这一核心假说的机制）。因此流行率调整后残余的 Delta 差异，仍需谨慎解读为"真实 ROC 级别技能差异"，而不能反过来说"流行率调整证明了两个 holdout 完全可比"——调整消除的是已知的统计混淆源，不是让两个总体变成同一个总体。这一点建议作为 Method 节流行率调整步骤后的一句限定语写入，避免读者误读调整后的差异为"纯净"的因果信号。
4. **与另一条独立弱项划清边界**：anchor regression/CausalKinetiX 假设是否成立（是否存在良定义的 shift/anchor 变量、局部动力学是否跨环境不变）与本文的流行率调整是完全独立的两个问题，本文不能也不应该被用来回应前者。下一版 proposal 修订两条弱项时，引用应分别对应，不要合并成一次性的"统计严谨性补丁"叙事。
5. **可迁移性**：Hoiem et al. (2012) 的 normalized precision 与 Siblini et al. (2020) 的固定 r₀ 度量本质上是方案 B 的既有实现，proposal 若采用方案 B，应直接引用这两篇作为方法先例而非自称新颖；PrecG（方案 A）目前看没有在气象/强对流预警文献中被使用的先例（本次调研未检索到），若采用，可以在 Paper Story 里明确标注这是把 Flach & Kull (2015) 的度量学工具首次引入强对流预警评估协议，是一个小的、诚实的、可加分的方法论迁移点（但仍属"协议补丁"性质，不构成独立 contribution，需按 review 已有建议避免过度包装）。
