# Data-driven discovery of digital twins in biomedical research (2508.21484)

- **Year**: 2025.08
- **Authors**: Clémence Métayer (Inserm U1331, Institut Curie), Annabelle Ballesta (Inserm U1331, Institut Curie), Julien Martinelli (Aalto University)
- **Venue**: arXiv
- **Keywords**: biomedical digital twin, ODE discovery, sparse regression, symbolic regression, SINDy, chemical reaction networks, Bayesian uncertainty, LLM-assisted equation discovery
- **One-liner**: 综述 177 个生物医学 ODE 数字孪生自动发现方法，结论是 Bayesian sparse regression 最可靠但仍缺统一 benchmark。

## 精华提炼

### 管线/方法
- 这篇不是提出单一新算法，而是面向“从生物时间序列自动学习 ODE-based digital twins”的方法综述和评估框架。
- 论文先把目标形式化为：给定离散时间点的观测数据 `D={(t_j, y_j)}`，学习状态变量 `x(t)` 的向量场 `f(x, theta)` 和参数 `theta`。
- 作者强调目标不是任意预测器，而是 mechanistic estimator：项最好来自 mass action、Michaelis-Menten、Hill kinetics 等生物可解释动力学。
- 被审视的方法被分成两大类：symbolic regression 和 sparse regression。
- Symbolic regression 从运算符、变量、常数构成的表达式树里搜索公式，优点是灵活，缺点是搜索空间指数增长、易过拟合、常按变量逐方程学习而忽略耦合约束。
- Sparse regression 先构造候选函数库 `Theta(Y)`，再用稀疏正则或 Bayesian prior 选择少量活动项，典型代表是 SINDy / PySINDy 系列。
- 论文把 biological digital twin discovery 的难点拆成 8 个挑战：4 个生物挑战和 4 个方法挑战。
- 4 个生物挑战是：噪声/缺失/不规则采样，多条件或多患者异质性，生物先验知识整合，潜变量/未观测变量。
- 4 个方法挑战是：高维系统可扩展性，未观测导数，候选函数库选择/生成，模型鲁棒性和不确定性量化。
- 评估逻辑是逐方法标注其是否处理这些挑战，而不是只看某个 toy benchmark 的恢复率。
- 文章还提供 Shiny app，用来交互式浏览 reviewed methods 及其挑战标签、实现语言、benchmark 和技术特征。
- 全文最后从评估表出发给出方向：未来更可能来自模块化集成，而不是某个全新单点算法。
- 作者主张的理想管线是 CRN mechanistic grounding + Bayesian uncertainty quantification + deep learning / LLM 的先验整合能力。
- 他们还提出 benchmark 设计建议：要用 curated biological ODE systems，而不只用低维物理 toy systems。
- 这类 benchmark 应系统改变噪声、采样频率、初始条件、条件/患者差异、先验量和先验可信度、未观测物种比例。

### 关键技术细节
- 数学问题定义为 `dot{x}(t)=f(x(t), theta)`, `x(0)=x_0`, `y_j=x(t_j)+epsilon_j`。
- 生物数字孪生的模型对象是 ODE-based mechanistic systems，变量可代表基因、蛋白、药物种类或其他生物状态。
- Symbolic regression 的典型结构是 expression tree，节点为变量/常数/基本运算符，迭代流程包括初始化候选表达式、拟合数值参数、按 loss+complexity 选择、mutation/crossover/grammar/neural decoding 生成下一代。
- 文章明确指出 symbolic regression NP-hard，变量数和运算符数上升时搜索空间爆炸。
- Symbolic methods 中重点列出 Genetic Programming、Neural Symbolic Regression、Grammar-based SR、Bayesian SR、mathematical constraints、application-specific SR 六类。
- ODEFormer 被描述为 Transformer-based symbolic translation：编码时间序列轨迹，直接生成 symbolic dynamics，在 SEIR 和 Lotka-Volterra 等低维生物相关系统上表现强。
- Grigorian et al. 的 hybrid neural ODE + symbolic regression 被作者认为能处理 latent trajectory inference 和 symbolic recovery。
- Reactmine 是 application-specific symbolic/reaction discovery：顺序搜索 synthesis、degradation、enzyme catalysis 等反应，在 3/5 变量真实数据和 7 变量 MAPK synthetic cascade 上展示效果。
- Sparse regression 的核心矩阵为 `Theta(Y) in R^{n x p}`，列可以是常数、线性、多项式、三角函数、非线性或生物特定项。
- SINDy 用 Sequential Thresholded Least Squares 等稀疏优化识别少量活动项，但原始 SINDy 依赖导数估计，面对生物噪声脆弱。
- PySINDy 被作为可用实现提到，覆盖 SINDy 及相关稳健变体。
- Sparse methods 被分成 classical sparse regression、derivative-free sparse regression、Bayesian sparse regression、neural sparse regression、sparse network/reaction discovery 五类。
- Reactive SINDy 构造 mass-action candidate reaction library，并强制 chemical mass conservation，在 E. coli 和 MAPK network 上恢复真反应。
- Bayesian sparse regression 使用 spike-and-slab、horseshoe 等 prior，在 term inclusion 和参数不确定性上比简单 LASSO 更有信息量。
- GP-NODE 结合 sparse dynamics identification、differentiable neural ODE solver 和 Gaussian Process observation model，适合 scarce/noisy/partially observed data。
- 多条件建模中，group sparsity、INSITE、SpReME 这类方法共享结构但允许 condition-specific coefficients。
- NeuralCODE、Gui et al. invariant function learning、MetaPhysiCa、OASIS 则用 neural representation 或 meta-learning 捕捉跨条件不变动力学。
- 先验知识分为 interaction prior、kinetic-form prior、parameter prior。
- Interaction prior 可来自 STRING、SIGNOR、HuRI、humanDB、Reactome、KEGG、BioCyc、BioGRID 等数据库。
- Kinetic prior 包括 conservation laws、dimensional consistency、symmetry、known functional forms、reference model residual learning。
- 作者认为 CRN formalism 是生物里比一般 dynamical-system formulation 更合适的结构先验，因为它编码 stoichiometry、reaction directionality 和 kinetic rate function。
- LLM / foundation model 的角色不是直接替代 ODE solver，而是辅助 candidate generation、candidate plausibility scoring、literature/database prior extraction、expert feedback translation。
- LLM-Lasso 是重要例子：用 RAG 检索文献/数据库，让 LLM 给候选项 `x_i x_j` 的生物可信度打分，再转成 adaptive penalty。
- 作者也明确提醒 LLM 在 biology 的 underspecified setting 中易 hallucinate，prompt engineering 手工且脆弱，simple baselines 在 perturbation prediction 中仍会超过深度模型。
- 高维挑战被定量说明：10 变量系统如果 candidate library 包括一阶和二阶项有 65 个候选项，每个方程选最多 4 个活动项会有超过 722,000 种组合。
- Mangan yeast glycolysis 案例需要约 900 条不同初始条件的 temporal trajectories 才能支撑 7 维模型推断，这说明实验负担很重。
- 处理未观测导数有三类：先数值估计导数，直接在优化中数值求解候选 ODE，或通过 integral / weak formulation 避免求导。
- Weak formulation 在 sparse system identification benchmark 中表现稳健，但现有评估仍偏低维混沌系统。
- 候选函数库优化包括 Mutual Information、entropy filtering、Taylor expansion constraints、dimensional analysis、symmetry filtering、sensitivity analysis、stepwise F-tests、Dropout-SINDy、LLM/Transformer 生成候选项。
- 鲁棒性和不确定性量化包括 bootstrap/bagging、trajectory subsampling、multiple restarts、Bayesian posterior over parameters/terms/equations。

### 核心结果
- 论文综述了 177 个自动学习 biological digital twins 的方法，并在表中筛选/比较最佳 symbolic 与 sparse methods。
- Symbolic regression 的整体结论偏负面：49 个 best-performing symbolic methods 中只有 3 个能处理超过 3/8 个挑战。
- Symbolic methods 中，只有 11/49 处理 missing data 或 unobserved variables，且只有 1 个可能同时处理两者。
- 没有 reviewed symbolic regression method 解决 multiple experimental conditions 这个挑战。
- Symbolic methods 中只有 8/49 避免导数估计，9/49 提供 uncertainty quantification。
- Symbolic methods 中只有 7 个在 4 个或更多 state variables 的系统上测试过，说明高维能力证据弱。
- Sparse regression 更强：55 个 reviewed sparse methods 中有 18 个处理超过 3/8 个挑战。
- Sparse methods 中约一半处理 missing data：25/55。
- Sparse methods 中 22/55 整合 prior knowledge。
- Sparse methods 中 23/55 提供 uncertainty quantification，26/55 避免导数估计。
- Sparse methods 仍有硬伤：只有 8/55 处理 multiple experimental conditions，只有 4/55 处理 unobserved variables。
- Sparse methods 中只有 6 个包含 candidate function selection/refinement，作者认为这是可改进空间。
- 最强 sparse methods 都是 Bayesian frameworks：Bhouri et al. 2022、Jiang et al. 2022、Long et al. 2024、Course et al. 2023、Foo et al. 2025、Fung et al. 2025、North et al. 2022、Sun et al. 2022。
- Bhouri、Jiang、Long 被点名为唯一处理超过 5/8 挑战的 pipelines。
- 这三者共同点是：有 interpolation/trajectory reconstruction，有 sparsity-enforcing parameter priors，在至少 4 维 biological systems 上验证，直接数值求解 ODE 避免 finite-difference 导数估计，并通过 posterior quantification 表达不确定性。
- Bhouri / GP-NODE 进一步处理 unobserved variables。
- 论文的 strongest claim 是：Bayesian sparse regression 目前比 symbolic regression 更适合真实生物 ODE 数字孪生发现。
- 另一个核心 claim 是：没有一个方法覆盖全部 8 个挑战，所以未来应是 modular hybrid framework。
- Surprising 点：LLM/Transformer 的势头很强，但作者没有把它们评为可靠主力；它们更适合作为 prior/candidate generator，而不是未加检验的科学结论生成器。
- 另一个反直觉点：candidate library selection 在 sparse regression 中很关键，但只有少数方法系统做了这件事。

### 明确局限
- 这是一篇 non-systematic review，而不是系统综述；Methods 中说明检索从 PubMed/arXiv 的关键词组合开始，再 backward snowballing 和 citation expansion。
- 方法筛选标准是：能同时学习 ODE structure 和 parameters、与 biological modeling challenges 相关、方法有原创性；这带有作者判断。
- 综述没有做统一复现实验，也没有重新跑 177 个方法，所以表格标签依赖论文报告和作者归纳。
- 对 figure/table 中方法的优劣判断很有用，但不是严格 benchmark 结果。
- Shiny app 增强了可浏览性，但是否长期维护、条目是否完整、标签是否可审计，需要额外检查。
- 文中多处指出现有 benchmark 主要是 synthetic、low-dimensional、fully observed systems，这限制了结论外推到真实生物系统。
- 对 LLM / foundation model 的讨论是 forward-looking，多数可靠性问题还没有实验性解决。
- 论文没有提出可直接运行的新 benchmark dataset，只提出 benchmark 设计原则。
- 论文也没有给出一个完整实现的 hybrid modular framework；CRN + Bayesian + LLM 是路线图，不是 artifact。
- 对高维 single-cell / multi-omics 场景的可操作方案仍偏概念层面，尤其是从 latent gene programs 回到 mechanistic biological variables 的解释问题。
- 文章对临床 digital twin 的落地限制说得较清楚：高维、低样本、噪声、异质性、先验不确定、未观测变量、模型不确定性都没被同时解决。
- 作者之一与 Reactmine 等相关工作有关，Reactmine 被积极讨论；这不等于有问题，但在评价 application-specific CRN methods 时要注意潜在偏好。

### 跟我们的关系
- 对 `0616-fex`，这篇更像“选题地图”和“novelty 风险清单”，不是直接方法撞车。
- 若 `0616-fex` 目标是提出一个新的 biological digital twin / equation discovery / mechanistic model learning pipeline，那么它高度相关，必须引用。
- 直接撞车点：如果我们声称“首次把 LLM 用于 biological ODE candidate term generation / prior scoring”，会撞 LLM-Lasso、LLM-SR、Al-Khwarizmi、ODEFormer 相关路线；需要把 novelty 收窄到具体数据源、验证 gate 或生物 setting。
- 直接撞车点：如果我们声称“首次处理 noisy sparse biological time series 的 ODE discovery”，会撞 GP-NODE、Bayesian spline learning、weak/integral sparse methods、Bayesian Reactive SINDy。
- 直接撞车点：如果我们声称“首次将 CRN prior 引入 equation discovery”，会撞 Reactive SINDy、Bayesian Reactive SINDy、Reactmine 和 CRN inference work。
- 可用工具：PySINDy 是 baseline；Bayesian sparse regression / spike-and-slab / horseshoe prior 是强基线；GP / spline / neural ODE 可做 denoising；weak formulation 可作为避免导数估计的稳健 baseline。
- 可用评估框架：8 个挑战可以直接变成 `0616-fex` 的 claim checklist，每个 claim 都要说明处理哪些挑战、不处理哪些挑战。
- 可用 benchmark 设计：用 BioModels 或 curated ODE systems，系统 sweep 噪声、采样间隔、多条件参数扰动、prior corruption、observability ratio。
- 差异化方向：把 LLM 从“生成方程”降级为“候选项/先验建议器”，再用独立 Bayesian/CRN/solver gate veto，这正好符合 reliability 需求。
- 差异化方向：做多条件 + missing data + partial observability 的组合 benchmark，因为文中指出没有方法真正覆盖全部挑战。
- 差异化方向：提供 artifact：固定数据生成器、评价脚本、方法 wrapper、uncertainty calibration 指标。本文缺的是可执行 benchmark，不是又一篇综述。
- 该引：应在 related work 或 motivation 中引用，尤其用于说明 biological digital twin discovery 的 8 个挑战和 Bayesian sparse regression 的当前优势。

## Read by: 0616-fex (2026-06-16)

从 `0616-fex` 的角度，这篇文章的价值在于给出一个非常实用的 novelty checklist：不要把“用 AI/LLM 发现生物动力学方程”说成空泛新方向，因为 symbolic regression、SINDy、Bayesian sparse regression、NeuralODE、ODEFormer、LLM-SR、LLM-Lasso、Reactmine/Reactive SINDy 都已经覆盖了关键局部 claim。更稳妥的差异化应落在组合挑战和验证机制上：例如同时处理 noisy sparse sampling、多条件/多患者、partial observability、CRN prior、Bayesian uncertainty，并用独立 solver/biological prior gate 否决 LLM 生成项。

如果 `0616-fex` 是方法论文，建议把本文作为 landscape citation：它支持“真实 biomedical digital twin discovery 还没有单一方法覆盖 8 个挑战”的动机，也支持“Bayesian sparse regression 是必须比较的强 baseline”。如果 `0616-fex` 是 benchmark/agent 课题，这篇更有用：可以直接把 8 个挑战改写成 benchmark axes，并把本文点名的方法作为 baseline pool。真正可发表的 delta 不是再写一个 LLM 生成 equation demo，而是给出可执行 benchmark、可审计 prior provenance、独立 verification/veto、uncertainty calibration，以及在 curated biological ODE systems 上的系统实验。
