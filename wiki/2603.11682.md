# Entropy-preserving reinforcement learning (2603.11682)

- **Year**: 2026.03
- **Authors**: Aleksei Petrenko (Apple), Ben Lipkin (MIT), Kevin Chen (Apple), Erik Wijmans (Apple), Marco Cusumano-Towner (Apple), Raja Giryes (Apple), Philipp Krähenbühl (Apple)
- **Venue**: ICLR 2026 (camera-ready)
- **Keywords**: entropy collapse, policy gradient, RL for LLMs, GRPO, DAPO, GSPO, PPO clipping, numerical precision, BF16, FP16, entropy regularization, adaptive clipping, REPO, ADAPO
- **One-liner**: 分析 policy gradient 算法在 LLM 训练中的熵崩塌现象，提出 REPO（修改 advantage 函数）和 ADAPO（自适应非对称 clipping）来主动控制熵，同时在 AppWorld 上用 RLOO + FP16 取得 SOTA。

## 精华提炼

### 管线/方法

**问题设定**：
- 输入：预训练 base model（Qwen-3-8B / 32B），terminal reward（数学答案匹配 / AppWorld 单元测试）
- 在线 policy gradient RL 训练，每次迭代包含：(1) vLLM rollout 采样 K 条轨迹，(2) 优势估计，(3) PPO 风格的 off-policy 梯度更新

**实际训练栈**：
- 3 台 8-GPU H100 节点（1 rollout / 1 training / 1 eval）
- vLLM 推理（每节点 2 实例 × 4 GPU tensor parallelism）
- FSDP2（SimpleFSDP）分布式训练，LoRA（rank=16, α=32）微调 attention key/value/query/output + MLP
- Cut Cross-Entropy（CCE）减少显存，AdamW，恒定 lr=5×10⁻⁵，weight decay=0.01，grad clip max-norm=0.1
- Early stopping rollout：4/6 条轨迹 + 90% 总数到齐即进入训练（防止长尾任务拖慢）

**对比算法**：
| 算法 | 优势估计 | Clipping | 是否 off-policy |
|------|----------|----------|----------------|
| RLOO | 原始 leave-one-out | 无 (strictly on-policy) | 否 |
| GRPO | LNO + std 标准化 | 对称 ε=0.2 | 是 (2 epochs) |
| LOOP | RLOO 非标准化 | 对称 ε=0.2 | 是 |
| DAPO | RLOO | 非对称 ε_low=0.2, ε_high=0.28 | 是 |
| GSPO | RLOO | 轨迹级 clipping ε_low=3e-4, ε_high=4e-4 | 是 |
| REPO-R | GRPO + 优势自适应缩放 | GRPO 对称 | 是 |
| ADAPO | DAPO + 动态 ε_high | 自适应非对称 | 是 |

**REPO 的具体机制**：
- $\adv_{\mathrm{REPO}}(s,a) = \adv(s,a) - \beta_s \cdot \clogp(s,a)$，其中 $\clogp$ 是均值中心的 log-prob
- REPO-D（Decorrelation）：$\beta_s \propto -\Delta \mathcal{H}$，抵消自然熵变化
- REPO-R（Rescale）：$\beta_{s,a} = \zeta \cdot |\adv(s,a)|$，按优势幅度缩放
- 自适应控制器：首轮记录初始熵，之后每轮倍增/减半 ζ，超过 [ζ_min, ζ_max] 翻转符号

**ADAPO 的具体机制**：
- 固定 ε_low=0.2，动态调整 ε_high ∈ [0.2, 0.32]
- 熵 < 初始熵 → ε_high ×1.05（允许更多熵增）
- 熵 > 初始熵 → ε_high ×0.95（限制熵增）

### 关键技术细节

**理论框架**：
- Theorem 1（MDP 熵动力学）：$\Delta \mathcal{H} \approx -\alpha \cdot \mathbb{E}_{a,a'}[\adv(s,a) \cdot \clogp(s,a') \cdot \nabla \log\pi(a) \cdot \nabla \log\pi(a')]$
- Corollary 1（近似形式，假设 score 正交）：$\Delta \mathcal{H} \propto -\mathbb{E}_a[\adv(s,a) \cdot \clogp(s,a) \cdot \pi(a|s)]$
  - 熵变化由 advantage 和 log-prob 的协方差驱动
  - 当 base policy 已校准到 reward：高 prob 动作 → 正 advantage → 熵降
- Theorem 2（PPO 熵界）：$(1-\epsilon_{low}) \cdot \mathcal{H}_{old} \le \mathcal{H}_{new} \le (1+\epsilon_{high}) \cdot \mathcal{H}_{old}$
  - 对称 clipping（ε=0.2）直接限制了每 token 的最大熵变化
- Theorem 3（BF16 Bias）：$E[r_{observed} | r_{true}] > r_{true}$，向上偏置与 $r_{true}$ 成比例
  - 等价于 $\epsilon_{low} > \epsilon_{high}$（与 DAPO 相反！），加速熵崩塌

**数值精度发现**：
- BF16 的 16-bit 输出 cast（FSDP2 MixedPrecisionPolicy 默认行为）产生乘法向上偏置
- 这导致 importance ratio 的 upper clip 更早触发，lower clip 更松——系统性地阻碍低概率动作的提升
- FP16 训练多出 3 个 mantissa 位，显著缩小 vLLM 推理和训练前向的 log-prob 差异
- softmax 反向传播中，当 token 概率 p > 1-2⁻²³ ≈ 0.9999999 时梯度项 (1-p) 舍入到 0（影响 30-40% token）
  - 修复：用 64-bit 重算受影响 token 的梯度项（O(1) 开销）
- 综合效果：FP16 + MPP fix 使 DAPO 在 Qwen-3-8B AppWorld 上从熵崩溃转为熵爆发（定性变化）

**实验设置关键参数**：
- AppWorld 训练：90 题 train split，最多 30 turns（eval 50），32K context
- AIME 训练：563 题来自 NuminaMath-1.5（与 eval 不重叠），4K context 限制（提高难度）
- AIME eval：AIME 2024（30 题）+ AIME 2025（30 题）
- K=6 rollout samples per task
- 每次实验 3 个独立 seed

### 核心结果

**AppWorld Qwen-3-8B（Table 1）**：
| 算法 | Test Normal (mean) | Best TN | Test Challenge (mean) | ΔH |
|------|-------|---------|-------|------|
| RLOO | 0.53 ± 0.05 | 0.64 | 0.32 ± 0.05 | +4% |
| GRPO | 0.46 ± 0.01 | 0.46 | 0.21 ± 0.02 | **-64%** |
| GSPO | **0.56 ± 0.04** | 0.60 | 0.32 ± 0.03 | +2% |
| DAPO | 0.51 ± 0.03 | 0.55 | 0.28 ± 0.01 | +40% |
| ADAPO | 0.53 ± 0.02 | 0.57 | 0.30 ± 0.02 | -2% |
| REPO-R | 0.49 ± 0.01 | 0.49 | 0.25 ± 0.02 | -9% |

**AppWorld Qwen-3-32B（Table 2, SOTA）**：
| 算法 | Test Normal (mean) | Best TN | Test Challenge (mean) | Best TC |
|------|-------|---------|-------|------|
| RLOO | 0.77 ± 0.02 | **0.79** | 0.61 ± 0.06 | **0.71** |
| GRPO | 0.67 ± 0.03 | 0.71 | 0.46 ± 0.03 | 0.49 |
| ADAPO | **0.78 ± 0.02** | **0.82** | 0.58 ± 0.03 | 0.62 |
| REPO-R | 0.73 ± 0.04 | 0.78 | 0.54 ± 0.06 | 0.63 |

Surprising：RLOO（纯 on-policy）在 FP16 下胜过所有 off-policy 方法，尽管 early iterations 更慢。

**AIME Qwen-3-8B（4K context，Table 3）**：
- 所有方法非常接近，RLOO 略胜（0.47 AIME'24 mean）
- 最高分很早就出现 → base model 可能已 overfit 到 AIME

**Sequential Training（顺序训练，Fig. 5）**：
- GRPO 训练的模型在第二阶段训练中显著劣化（熵崩塌后无法探索）
- DAPO 尤其是 REPO-R 训练的模型保留了第二阶段的探索能力
- 跨任务 migration（AIME→AppWorld、AppWorld→AIME）一致：保留熵 = 保留训练能力

**关键图表信息**：
- Fig 1a-1c：每个 algorithm 的平均 per-token entropy 随时间变化 + test accuracy 曲线
- Fig 1d-1f：累计熵 vs test accuracy 正相关，每个点是一次 run 的 checkpoint
- 低熵轨迹 → 低性能；熵轨迹相似但后期不同 → 性能几乎不受影响
- "it's not the destination, it's the journey"（旅程而非终点）

**数值发现的关键证据**：
- BF16 rounding 影响 <0.1% 的 token 但改变了 clipping 行为
- FP16 + MPP fix：DAPO 在 Qwen-3-8B 上从崩溃转为暴涨
- Qwen-2.5 32B 上 LOOP/GRPO 不崩溃但 RLOO 崩溃 → base model 特性决定熵动力学

### 明确局限

1. **没有公开代码/data（写明了"Reproducibility Statement"有 appendix 但没有承诺开源）**
2. **仅在 2 个环境评测**：AppWorld（tool-use agent）和 AIME（数学），缺乏更广泛的 benchmark（GSM8K、MATH、HumanEval、SWE-bench 等）
3. **LoRA 微调**：所有实验用 LoRA rank=16，全参数微调下的熵动力学未经验证
4. **模型规模受限**：仅测试 8B 和 32B，超大模型（70B+）行为未知
5. **AIME "saturation"**：作者承认 base model 可能已 overfit 到 AIME，熵保持对 AIME 的增益有限
6. **REPO-R 在 8B 上弱于 GSPO**：尤其是 Test Normal（0.49 vs 0.56），说明 REPO 在小模型上不一定是最优
7. **REPO-D 仅理论分析，无实验对比**
8. **没有分析 why 仅 <0.1% token 导致如此大差异**——这个 "pivotal token" 现象缺乏因果解释
9. **Sequential training 仅在 2 任务 1 个方向测试**
10. **没有 human evaluation / 生成质量指标**：仅 reward-based 评测
11. **没有分析 REPO 的计算开销**（虽然声称 zero memory cost，但没有 latency 对比）
12. **No exploration of negative β（降熵）有什么用**

5. **Adaptive controller（倍增/减半策略）**：简单的自适应超参数调节机制，可以用于任何需要动态 reguliarization 的场景

### 可迁移的方法论
1. **Entropy trajectory 作为诊断工具**：本文的 cumulative entropy scatter plot（Fig 1 bottom row）是一个优雅的 diagnostics——可以在任何 RL 训练中部署此可视化来判断训练健康状态
2. **BF16 bias 的分析方法**：Theorem 3 的推导范式（quantization error → Jensen inequality → multiplicative bias）可以迁移到分析其他精度敏感的数值计算
3. **Adaptive bidirectional controller**：倍增/减半 + 翻转符号的简单控制器适用于任何需要动态调整 regularization 的场景，代码量极小
4. **"It's not the destination, it's the journey" 的实证方法论**：熵轨迹比最终熵更重要——类似的 temporal analysis 可以应用到评估任何 training dynamic 变量

### 差异化空间
- 本文仅针对 LLM token-level entropy，没有讨论 feature-level / representation-level 的信息保持
- 本文的 REPO 修改的是 advantage 函数而非 model architecture 或 training objective
- 对 representation diversity preservation（与 token distribution diversity 不同）是一个未探索方向
