# RSRM: Reinforcement Symbolic Regression Machine (2305.14656)

- **Year**: 2023.05
- **Authors**: Yilong Xu (Gaoling School of AI, Renmin Univ), Yang Liu (UCAS), Hao Sun (Renmin Univ, corresponding)
- **Venue**: arXiv (uses NeurIPS 2023 style file, status unclear)
- **Keywords**: symbolic regression, Monte Carlo tree search, double Q-learning, expression tree, sub-tree discovery, divide-and-conquer, parity symmetry
- **One-liner**: RSRM 组合 MCTS 搜索 + Double Q-learning 值估计 + 调制子树发现模块，在有限数据上分步发现复杂数学方程，在 Nguyen/LiverMore/R 等 benchmark 上达到 SOTA。

## 精华提炼

### 管线/方法

完整的 SR pipeline：输入数据 → 3 步循环：

1. **数据预处理（奇偶性检测）**：
   - 用 cubic spline 拟合数据得到函数近似
   - 比较 y(-x) 和 y(x)：若误差 < E_sym (10^-5) 判定奇/偶性
   - 偶函数：转换 x→|x|, 用 ŷ = (g(x)+g(-x))/2 的约束形式搜索
   - 奇函数：转换 x→|x| 且反转 y, 用 ŷ = (g(x)-g(-x))/2 约束

2. **RL 引导的表达式搜索（Algorithm 1）**：
   - 用 MCTS 在表达式树上逐层搜索（hierarchical traversal strategy）
   - Greedy selection：不是 UCT 选最高分，而是选当前 reward 最高的 token
   - Simulation：每选一个 token，随机补全整棵树，计算 reward 喂给 Double Q-learning 训练
   - Expansion：Double Q-learning 输出每个子节点的初始概率
   - Back-propagation：基于当前 expression form 回传 reward
   - 参数优化：BFGS（scipy）优化表达式中的参数占位符，初始值用 N(0,1)

3. **GP 微调**（Algorithm 1 之后）：
   - 30 轮 GP, population 500，保留前 20 最佳表达式
   - Mate rate = Mutate rate = 0.5

4. **调制子树发现 MSDB（Algorithm 2）**：
   - 从 RL+GP 产生的最佳表达式中统计高频子表达式 (Split-by-addition)
   - 检测三种 form：A±f(x) / A×f(x) / A^f(x)
   - 固定 A 部分，让后续搜索只优化 f(x)，实现分治
   - 循环迭代直到发现完整表达式

### 关键技术细节

- **表达式树**：内部节点 = 操作符 (+, -, ×, ÷, log, exp, sin, cos, sqrt)，叶节点 = 常量或变量
- **层次遍历策略**：高层节点获得更多搜索迭代（不同于按深度遍历的传统方法）
- **Reward 函数**：R = η^l / (1 + sqrt(Σ(y_i-ŷ_i)²))，η=0.99 为长度折扣，l 为表达式树节点数
- **Double Q-learning**：两个 Q 网络交叉训练（50% 概率），训练时用 simulated reward
- **UCT 公式**：UCT(n) = Q̄(v') + c·sqrt(ln(N(v))/N(v'))，c = sqrt(2)
- **最小选取次数 n₀=3**：访问 < 3 次的子节点优先选择，确保探索
- **RL 训练轮数 t_r = 50**
- **表达式长度范围**：[4, 35]，最大参数数 10
- **Double Q-learning learning rate = 10^-3**
- **操作符集**：+, -, ×, ÷, cos, sin, exp, log, sqrt
- **表达式约束**（来自 DSR）：单目操作符后继不能是逆操作 / 三角函数不能嵌套三角函数 / 最大参数数限制

### 核心结果

**Nguyen Benchmark（12 方程 + 7 含参版本，100 次独立运行）**：
- RSRM 平均 recovery rate：100.00%（所有方程全部 100% 恢复！）
- SPL: 84.68%, NGGP: 94.47%, DSR: 78.74%, GP: 29.00%
- 最难的 Nguyen-12 (x₁⁴-x₁³-0.5x₂²+x₂)：RSRM 100% vs SPL 28% vs NGGP 21%

**LiverMore Benchmark（22 方程，100 次独立运行）**：
- RSRM 平均 recovery rate: 97.95%
- SPL: 51.0%, NGGP: 80.68%, DSR: 50.41%, GP: 50.77%
- Livermore-3（sin(x₁³)cos(x₁²)-1）：仅 55%，是所有方法中最低的
- 高难度方程如 sinh(x₁)：RSRM 100% vs SPL 18% vs NGGP 24%

**R Rational Benchmark（6 方程）**：
- RSRM 平均 68.83%，大幅领先所有 baseline
- SPL、DSR、GP 均为 0%，NGGP 仅 26.67%

**自由落体球物理数据集（11 球，MSE 指标）**：
- RSRM 平均 MSE 0.173，优于 RSRM* (0.242) 和 SPL (0.356)
- RSRM 发现了 h(t) = -4.63t² + sin(0.83t)sin(t) + 46.13 等形式的方程
- 在大多数球类上表现最好，但有个别失败案例（如 yellow whiffle ball 搜索到了 cos 形式，MSE 1.277）

**泛化实验（CDF 外推）**：
- RSRM 总误差比 NGGP 低 2-3 个数量级（曲线1: 1.05e-5 vs 0.00215）
- RSRM 生成的表达式更简洁（如 0.503 + 117.088x/(x²+14702) vs NGGP 的嵌套 cos(exp(...))）
- 但 NGGP 在训练集内插误差更小

**Ablation（LiverMore 上 5 模型对比）**：
- 完整 RSRM: 97.95%
- Model A（去除 Double Q-learning）：94.68% —— 搜索多样性增加但效率下降
- Model B（去除 MCTS）：95.0% —— 更容易过拟合
- Model C（去除 form search/MSDB）：80.45% —— 影响最大
- Model D（去除 preprocessing/奇偶检测）：89.45% —— 对 sinh/cosh 影响最大（sinh: 100%→10%）

### 明确局限

1. **作者自述**：表达式 form 设置缺乏灵活性，限制了不同问题领域的适应性。作者期望未来用神经网络生成更灵活的 slot
2. **代码未公开**："codes will be posted upon final publication"，截至本文阅读时未见公开仓库
3. **无真机/物理实验验证**：只做了数值 benchmark 和一个物理数据集（自由落体球），无真实机器人或物理系统验证
4. **搜索空间有限**：操作符集仅含基本运算和三角函数，无法处理更复杂的数学结构（如积分、微分方程、分段函数）
5. **对超长方程效果未知**：表达树最长 35 节点（实际上只测试到 9 次多项式 Σx^k）
6. **高复杂度方程恢复率仍低**：Livermore-3 仅 55%，R-1 仅 48-49%，说明对嵌套 sin/cos 和有理函数仍吃力
7. **无与其他 DL-based 方法对比**：未对比 End-to-end Transformer (Kamienny 2022)、DGSR (ICLR 2023) 等最新方法
8. **计算成本未报告**：未给搜索时间或 GPU/CPU 使用量
9. **benchmark 规模小**：Nguyen benchmark 仅到 2 变量，实际物理系统远超此复杂度

## Read by: 0616-fex (2026-06-24)

本次审读结论：RSRM 是纯 search-based SR 方法，与 foundation model for expression 方向**正交不撞车**。

- **撞车评估**：无直接竞争。RSRM 依赖 MCTS + RL + GP 三级搜索，需要预定义操作符集，与 FM-based 方法在范式上完全不同
- **可用工具/思路**：
  1. MSDB 的 Split-by-addition + form discovery 可作为 FM 生成表达式后的 mode analysis / clustering 工具
  2. cubic spline + 奇偶性预检测 可做 FM pipeline 的预处理模块
  3. Reward 函数（η^l / (1+RMSE)）可作为 FM 生成质量的评价参考
  4. 分治策略（先固定 A 再搜 f(x)）是可泛化的设计原则
- **差异化定位**：FM 方法的核心优势在于：(a) 无需预定义操作符集，(b) 一次推理覆盖更大搜索空间，(c) 可利用大规模预训练知识。这些都是 RSRM 做不到的
- **引用建议**：作为 SR 领域的 MCTS-based baseline 可引用，但非核心相关文献
