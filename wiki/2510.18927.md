# BAPO: Stabilizing Off-Policy Reinforcement Learning for LLMs via Balanced Policy Optimization with Adaptive Clipping (2510.18927)

- **Year**: 2025.10
- **Authors**: Zhiheng Xi (Fudan), Xin Guo, Yang Nan, Enyu Zhou, Jun Shen, Wenxiang Chen, Jiaqi Liu, Jixuan Huang, Zhihao Zhang, Honglin Guo, Xu Deng, Zhikai Lei, Miao Zheng, Guoteng Wang, Shuo Zhang, Peng Sun, Rui Zheng, Hang Yan, Tao Gui, Qi Zhang, Xuanjing Huang
- **Venue**: arXiv (Fudan University + Shanghai Qiji Zhifeng Co., Ltd.)
- **Keywords**: Off-Policy RL, LLM Reasoning, PPO/GRPO, Adaptive Clipping, Entropy Collapse, Policy Gradient, Importance Sampling, Positive-Negative Balance
- **One-liner**: 通过理论分析揭示 off-policy RL 中正负样本不平衡和 Entropy-Clip Rule 两个关键问题，提出自适应裁剪边界 BAPO 方法，在 AIME 2024/2025 上 32B 模型达 SOTA (87.1/80.0)，超过 o3-mini 和 Gemini-2.5-Flash-Thinking。

## 精华提炼

### 管线/方法

BAPO 整体管线：

1. **基础算法**：以 GRPO (Group Relative Policy Optimization) 为基础框架进行改造。GRPO 本身是对 PPO 在 LLM 场景的简化版——用 group-based rewards 替代 value network。

2. **采样阶段**：从数据集 $\mathcal{D}$ 中采样 batch，用旧策略 $\pi_{\theta_\text{rollout}}$ 生成 $G$ 条响应（stale data 通过 ppo_epoch 复用或 partial rollout 引入）。

3. **优势估计**：基于 reward function $R$ 为每条响应计算 reward 和 advantage $A_t$。

4. **BAPO 核心——自适应裁剪边界调整**：
   - 初始化：$c_\text{low} = a^-$（默认 0.6），$c_\text{high} = a^+$（默认 1.2）
   - 循环检查：当前 positive token 对 loss 的贡献比例 $\rho$ 是否 ≥ 目标阈值 $\rho_0 = 0.4$
   - 不满足时：优先增大 $c_\text{high}$（步长 $\delta_1 = 0.05$，上限 $b^+ = 3.0$）；若 $c_\text{high}$ 已达上限，则增大 $c_\text{low}$（步长 $\delta_2 = 0.02$，上限 $b^- = 0.9$）
   - 对每个 update step 独立动态调整，不需要手动调参

5. **目标函数**：
   $$J^{\text{BAPO}}(\theta) = \mathbb{E}_{\bm{y}\sim \pi_{\theta_\text{rollout}}} \sum_{t=1}^{T} \left[ \min(r_t\cdot A_t, \text{clip}(r_t, c_\text{low}, c_\text{high})\cdot A_t)\right]$$
   关键区别：标准 PPO 使用对称裁剪 [1-ε, 1+ε]；BAPO 针对每个 update step 自适应选择非对称边界。

6. **训练循环**：对 staleness = 0...E 的每轮迭代，先动态确定裁剪边界，再用 BAPO objective 更新策略。

### 关键技术细节

- **基础模型**：DeepSeek-R1-Distill-Qwen-7B、DeepSeek-R1-Distill-Qwen-32B、OctoThinker-Llama3.2-3B-Long-Zero
- **自训练 SFT 模型**：BP-Math-7B 和 BP-Math-32B，基座为 Qwen2.5-Math，经过 SFT 微调
- **RL 数据集**：SkyWork-OR1-RL-Data（广泛使用的高质量数学推理数据集）
- **评测基准**：AIME 2024、AIME 2025、MATH（Llama 实验）
- **训练超参数**：
  - 学习率：$2 \times 10^{-6}$
  - 采样温度：0.6
  - 最大响应长度：8k（初步/验证实验），64k（主实验 BP-Math 模型）
  - BAPO 超参数：$\rho_0=0.4$, $a^-=0.6$, $b^-=0.9$, $a^+=1.2$, $b^+=3.0$, $\delta_1=0.05$, $\delta_2=0.02$（强调未精细调参）
  - 推理时 16 次 rollout 取平均
- **Entropy-Clip Rule 推导**（理论核心）：
  - 基于 tabular softmax 假设，将 PPO 目标函数的梯度分解为 $\mathcal{X}(y_t)$ 指示器
  - 推导出 $\Delta\mathcal{H}(\pi_\theta)\approx-\eta \cdot \text{Cov}[\log \pi_\theta(y_t), A_t \cdot \mathcal{X}(y_t) + C]$
  - 核心发现：熵变化由**未被裁剪的 token** 的控制，由其 log probability 与 advantage 的协方差决定
- **四类 token 对熵的影响**（Proposition 3-4 + Table）：
  - High prob + High adv → 降熵（sharpening distribution）
  - High prob + Low adv → 升熵（smoothing distribution）
  - Low prob + High adv → 升熵（exploration-promoting）
  - Low prob + Low adv → 降熵（over-penalization）
- **Staleness 引入方式**：ppo_epoch 经验复用 + partial rollout（长轨迹分段存储/恢复）
- **对比基准**：SFT、GRPO、clip-higher（DAPO）、非对称裁剪（手动固定版本）

### 核心结果

| 模型 | AIME 2024 | AIME 2025 | 平均 | 对比 |
|------|-----------|-----------|------|------|
| BP-Math-32B (SFT) | 84.4 | 78.1 | 81.3 | baseline |
| BP-Math-32B (GRPO) | 84.6 | 78.8 | 81.7 | +0.4 |
| **BP-Math-32B (BAPO)** | **87.1** | **80.0** | **83.5** | **+2.7/+1.9 vs SFT** |
| BP-Math-7B (SFT) | 66.9 | 59.0 | 62.9 | baseline |
| BP-Math-7B (GRPO) | 69.2 | 59.2 | 64.2 | +1.3 |
| **BP-Math-7B (BAPO)** | **70.8** | **62.5** | **66.7** | **+3.9/+3.5 vs SFT** |

**Surprising/反直觉发现**：
1. GRPO 在强 SFT 模型上收益微弱（32B 仅 +0.2/+0.7 点），而 BAPO 能解锁显著增益——说明 off-policy 的不稳定性是 GRPO 性能瓶颈的关键因素。
2. 32B BAPO 超过 o3-mini-medium（79.6→87.1 AIME24，76.7→80.0 AIME25），甚至超过 DeepSeek-R1 671B（79.8→87.1 AIME24）——小模型通过更好的 RL 优化可以打败大得多的模型。
3. 熵并非越高越好——BAPO 通过 $\rho_0=0.4$ 限制正样本贡献上限，也防止熵无控增长导致的 tail degradation（模型过拟合简单题，放弃难题）。
4. 随着 staleness 增加，传统 GRPO 的 token 裁剪比例急剧上升，但 BAPO 动态适应后裁剪比例保持稳定。

**Llama 基座实验**（OctoThinker-Llama3.2-3B-Long-Zero）：
- GRPO: AIME24 2.5%, AIME25 2.9%, MATH 58.4%
- BAPO: AIME24 5.4%, AIME25 5.8%, MATH 66.0%（相对提升 > 2x）

### 明确局限

1. **无真机实验**：所有实验均为数学推理（AIME/MATH），未在 code、agent、多模态等场景验证。
2. **理论假设简化**：Entropy-Clip Rule 推导基于 tabular softmax policy 假设，实际 LLM 使用神经网络参数化，连续状态空间下该分析严格性有限。
3. **仅与 GRPO 对比**：baseline 只有 GRPO，未与 PPO、REINFORCE、GSPO (2507.18071)、DCPO (2509.02333) 等做直接实验对比，虽然 related work 中讨论了它们。
4. **超参数声称 "未精细调参" 但未见 sensitivity analysis**：$\rho_0=0.4$、$\delta_1=0.05$、$\delta_2=0.02$、$b^+=3.0$ 等设置缺乏消融实验，不清楚是否对性能敏感。
5. **没有开源 SFT 数据细节**：BP-Math-7B/32B 的 SFT 训练细节未披露，其他研究者难以复现完整 pipeline。
6. **没有计算开销讨论**：BAPO 在每个 update step 都反复评估 $\rho$ 并调整边界——相比固定裁剪的额外计算开销未报告。
7. **贡献比例阈值 $\rho_0=0.4$ 的来源不清**：为何选 0.4 而非 0.3 或 0.5 未作说明。
8. **Partial rollout 实验规模偏小**：仅展示 training dynamics 曲线，未给最终 eval 数字。

### 撞车评估
**不撞车。** BAPO 的核心贡献在 RL 优化算法层面（自适应裁剪、正负平衡、熵保持），解决的是 off-policy RL for LLM 的训练稳定性问题。0616-fex 课题如果聚焦在特征提取/表示学习层面，而非 RL 优化器设计，则不存在重叠。但需确认：0616-fex 是否在 pipeline 中使用了 RL for LLM 训练——如果使用，BAPO 是训练稳定性的重要参考；如果不使用 RL，则无直接关系。

### 可用工具
1. **Entropy-Clip Rule** 可以在任何涉及 token 级别熵追踪的场景中用作诊断框架。
2. **正负样本贡献比监控**（$\rho$ 指标）是简单但有效的训练健康度指标，可移植到任何 RL training loop 中。
3. **Staleness 系统化评估方法**（通过 ppo_epoch 和 partial rollout 控制）可复用为鲁棒性测试协议。

### 差异化策略
如果 fex 涉及表示学习用于 RL 训练，可以清晰定位为：fex 提供更好的表示/特征 → BAPO（或类似方法）保证 RL 训练稳定。两者是互补关系而非竞争。在论文中可将 BAPO 列为 "RL training stability" 类别的方法论参考。
