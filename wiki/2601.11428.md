# Diagnosing Failure Modes of Neural Operators Across Diverse PDE Families (2601.11428)

- **Year**: 2026.01
- **Authors**: Lennon Shikhman, Georgia Institute of Technology
- **Venue**: TMLR / arXiv
- **Keywords**: neural operators, PDE surrogate modeling, robustness, distribution shift, FNO, DeepONet, CNO, rollout stability
- **One-liner**: 用统一压力测试揭示神经算子鲁棒失效模式。

## 精华提炼

### 管线/方法
1. 论文把神经 PDE 求解器的可靠性问题定义为结构化泛化问题，而不是普通的 in-distribution 测试误差问题。
2. 输入对象是 PDE 家族中的函数、系数、初值、边界/终端条件、payoff 或显式参数通道。
3. 输出对象是静态解场，或时间依赖系统中的短时预测目标。
4. 被比较的三类架构是 Fourier Neural Operator (FNO)、DeepONet-style model、Convolutional Neural Operator (CNO)。
5. FNO 被视为全局谱混合机制，适合低频或相干模态主导的算子结构。
6. DeepONet 被视为 branch-trunk 的条件算子近似，分离输入函数编码和坐标/查询点编码。
7. CNO 被视为局部多尺度卷积机制，强调 continuous-discrete consistency 和跨分辨率响应。
8. PDE 覆盖五类：nonlinear Schrodinger、Poisson、Navier--Stokes、Black--Scholes、Kuramoto--Sivashinsky。
9. 五类压力测试包括参数/系数 shift、边界或终端条件 shift、分辨率外推、长时 rollout、输入扰动敏感性。
10. 并非每个 PDE 都适用全部 stressor；但同一个 PDE 内的 stress grid 对所有架构固定一致。
11. 所有 stress evaluation 都不在 shifted regime 上 fine-tune，评估的是零适应鲁棒性。
12. 主指标是 degradation factor：每个 seed 的 stress error 除以 baseline relative L2 error。
13. 作者没有只用 degradation factor，还报告 absolute baseline L2 error、rollout growth、rollout amplification、frequency-binned spectral error 和 95% CI。
14. 核心 pipeline 是：生成每个 PDE 的 in-distribution 数据 -> 训练三类神经算子 -> 对每个训练模型跑统一 stress suite -> 按 seed 聚合 -> 比较 baseline 与 stress 下的排序变化。
15. 论文还给出两个解释性机制：rollout composition bound 和 resolution error decomposition。
16. rollout bound 说明一阶预测误差小不代表长时稳定，因为误差会按 learned map 的 Lipschitz 常数迭代放大。
17. resolution decomposition 把有限分辨率误差拆成 projected risk 和 Fourier spectral-tail term，用来解释分辨率压力下的失效。

### 关键技术细节
18. 总训练量是 3 个架构 x 5 个 PDE x 每组 50 seeds，共 750 个训练模型。
19. 优化器统一使用 Adam，learning rate 为 1e-3。
20. Poisson 设置：512 samples，resolution n=128，batch size 8，3000 steps。
21. Black--Scholes 设置：512 samples，n=256，batch size 8，3000 steps。
22. Nonlinear Schrodinger 设置：256 samples，n=256，n_t=20，batch size 4，4000 steps。
23. Navier--Stokes 设置：128 samples，n=64，n_t=20，batch size 2，5000 steps。
24. Kuramoto--Sivashinsky 设置：256 samples，n=128，n_t=20，batch size 8，3000 steps。
25. FNO 使用 width 64、depth 4；1D 用 16 Fourier modes，2D 用 12 x 12 modes。
26. CNO 使用 width 64、depth 5。
27. DeepONet-style model 使用 width 128、depth 2。
28. 所有模型都使用 coordinate channels。
29. baseline evaluation 每个 seed 使用 64 个 in-distribution test samples。
30. 对时间依赖 PDE，baseline rollout summary 使用 5 rollout steps。
31. 参数 shift 示例包括 NLS 中更大的 kappa、Navier--Stokes 中更低的 viscosity nu、Black--Scholes 中更高的 volatility sigma、Poisson 中更 rough 的 coefficients。
32. 边界/终端条件 shift 示例包括 unseen boundary families 和 unseen payoff families。
33. 分辨率外推用更细或更粗的 discretization，并用谱诊断定位频率段误差。
34. 长时 rollout 将 one-step predictor 迭代到训练 horizon 之外，测误差累积和 dynamical instability。
35. 输入扰动测试是在 state channels 上加入小扰动，测 local stability。

### 核心结果
36. 总结性发现：FNO 在五个 PDE family 的 baseline in-distribution error 都最低，但这不稳定地转化为 robustness。
37. Poisson 是最明显的排序反转：FNO baseline 最强，但 perturbation 和 resolution shift 下最不稳；CNO baseline 最差但 stress 下最稳。
38. Poisson degradation：FNO resolution 18.417、input perturbation 11.199；DeepONet resolution 1.882、perturbation 1.545；CNO resolution 1.051、perturbation 1.055。
39. NLS 的主要难点是 nonlinearity shift kappa：FNO 3.872、DeepONet 2.227、CNO 2.618。
40. NLS resolution transfer 下 FNO 1.022、DeepONet 0.985 基本不变，CNO 2.203 明显退化。
41. Navier--Stokes 的参数 shift、resolution shift、input perturbation 都较温和，真正的问题是 long-horizon rollout。
42. Navier--Stokes rollout degradation：FNO 3.296、DeepONet 3.271、CNO 3.138，说明三类架构共享长时迭代失稳。
43. Black--Scholes 的主 stressor 是 payoff structure shift，而非普通输入扰动。
44. Black--Scholes payoff shift：FNO 6.308、DeepONet 2.338、CNO 12.239；DeepONet 最稳，CNO 最差。
45. Black--Scholes volatility shift：FNO 2.068、DeepONet 1.272、CNO 1.357，仍是 DeepONet 更稳。
46. Kuramoto--Sivashinsky 隔离出 chaotic rollout 行为：FNO baseline 最好但 rollout degradation 最差。
47. KS rollout degradation：FNO 1.607、DeepONet 0.886、CNO 1.288；one-step accuracy 和 rollout stability 被明显解耦。
48. 反直觉点一：baseline 低误差可能掩盖 shift 下的 fragility，尤其是 Poisson 和 KS。
49. 反直觉点二：DeepONet baseline 不一定最好，但在 Black--Scholes payoff shift 和若干 rollout 设置上更稳定。
50. 反直觉点三：CNO 的 local multiscale bias 在 Poisson 中很有利，但在 Black--Scholes payoff family extrapolation 中很差。

### 明确局限
51. 作者明确说目标不是最终 architecture leaderboard，也不是穷尽所有 neural PDE solver。
52. 架构只覆盖 FNO、DeepONet-style、CNO，没有系统比较 PINO、WNO、GNO、transformer operator、foundation PDE model 等。
53. PDE family 有五类，但每类的具体数据生成和 stress grid 仍是作者设定的合成协议。
54. 每个 stressor 的强度和组合方式有限，不能代表真实部署中的全部 coupled shift。
55. 所有 shifted evaluation 都不 fine-tune，因此结果说明 zero-adaptation robustness，不说明 adaptation 后性能。
56. 论文有 50 seeds 和 confidence intervals，但检验者不是独立实验室，也没有第三方复现实验。
57. 代码链接在 tex 中给出为 GitHub 仓库，但论文正文没有列出完整 raw logs、随机种子产物、硬件时间成本明细。
58. 解释性理论是 auxiliary mechanism，不是对架构排序的因果证明。
59. 对 0616-fex 来说，最重要局限是它评价的是 neural operator robustness，而不是直接提出新的 PDE foundation model 或新的 failure-explanation architecture。
60. 如果我们的课题需要证明一个新方法更鲁棒，不能只引用本文结论；需要复用 stress protocol 并加入自己的 shifted benchmark。

### 跟我们的关系
1. 对 `0616-fex`，这篇更像是 failure-evaluation / failure-atlas 型工作，而不是一个新求解器。
2. 如果 `0616-fex` 目标是做科学 ML 或 PDE surrogate 的失败解释、鲁棒评估、stress benchmark，它有较强撞车风险。
3. 具体撞车 claim 是：“baseline accuracy 不能代表 structured shift robustness”，以及“不同 PDE family 的 failure mode 和 architecture bias 交互决定鲁棒性”。
4. 不太撞车的部分是：它没有提出新的 fex 方法、没有做 foundation PDE model 训练、没有做自动 failure explanation agent，也没有把 failure diagnosis 闭环到修复策略。
5. 可复用工具是 degradation factor、跨 seed stress suite、resolution spectral diagnostics、rollout diagnostics、Poisson/NLS/NS/BS/KS 五类 pressure map。
6. 可复用论证是 rollout composition bound：one-step error 会被 learned map 的 Lipschitz 行为放大。
7. 可复用论证是 resolution decomposition：有限分辨率误差可拆成 projected risk 与 spectral tail。
8. 该引：只要我们的工作涉及 neural operator、PDE surrogate、robustness under shift、failure mode diagnosis，就应该引用。
9. 差异化方向一：从 evaluation atlas 走向自动 failure explanation 和 repair recommendation。
10. 差异化方向二：加入独立 verifier 或 cross-model/lab gate，提高 R2。
11. 差异化方向三：把 stress test 迁移到真实工程 PDE、非规则几何、多物理耦合或 foundation simulator。
12. 差异化方向四：不仅测 degradation，还定位 causal failure source，并给出可执行的 architecture/data/training fix。
