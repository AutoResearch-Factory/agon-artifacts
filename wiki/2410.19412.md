# Robust Time Series Causal Discovery for Agent-Based Model Validation (2410.19412)

- **Year**: 2024.10 (v3)
- **Authors**: Gene Yu, Ce Guo, Wayne Luk — Dept. of Computing, Imperial College London
- **Venue**: arXiv preprint only, no venue listed. Report document class (`\documentclass{report}`, full `\tableofcontents`), reads as an Imperial College MEng/MSc individual project report rather than a submitted conference/journal paper — no peer review evident.
- **Keywords**: causal discovery, time series, VAR-LiNGAM, PCMCI, cross-validation robustness, agent-based model validation, structural Hamming distance, non-stationarity
- **One-liner**: 对 VAR-LiNGAM/PCMCI 做交叉验证鲁棒化, 用一致性/变异性指标过滤不稳定因果边以提升 ABM 校验可靠性.

## 精华提炼

### 管线/方法
1. **输入**：多变量时间序列数据集 D（三类来源：可控合成 VAR 数据；模拟 fMRI BOLD 数据；用于 ABM 校验场景的"真实世界 vs ABM 输出"配对数据）。
2. **基础因果发现**：对全量 D 跑一次选定的 base 方法（VAR-LiNGAM 或 PCMCI），得到初始因果图 G0，每条候选边 r_ij^0。
3. **K 折切分**：将 D 切成 k 个子集 D_1..D_k（文中未明确是否为保序的连续时间块切分——这是一个未交代清楚的实现细节），对每个子集独立重跑同一 base 方法，得到 G_1..G_k。
4. **一致性/变异性度量**（核心公式，Eq 1-2）：
   - Consistency(r_ij) = (1/k) Σ 1{sign(r_ij^m) = sign(r_ij^0)} —— m 折中符号方向与全量估计一致的比例
   - Variability(r_ij) = std({r_ij^m}) / (|r_ij^0| + ε) —— 折间效应量标准差相对全量估计的归一化幅度
5. **Validate_And_Adjust**：丢弃 Consistency < τ_c 或 Variability > τ_v 的边；保留边可选做加权调整 r_adjusted = (1-w)·r0 + w·mean(r^m)，本研究中 w=0（只用全量估计值，CV 只用来决定去留，不重新估计强度）。
6. **输出**：清洗后的鲁棒因果图 G（RCV-VAR-LiNGAM / RCV-PCMCI）。
7. **下游整合**：接入本文第二个贡献——"Context-Aware ABM Validation Framework"，四阶段流水线：数据一致性预处理 → 属性分析（线性性/平稳性/统计均衡/遍历性检验）→ 因果结构识别（可在 VAR-LiNGAM/PCMCI/RCV-VAR-LiNGAM/RCV-PCMCI/VL-Bootstrap 间切换）→ 校验评估（SHD、F1、F1-directed、Frobenius 范数 + 3 个相似度检验：符号相似度/幅度相似度/联合相似度，改自 Guerini & Moneta 2017 的方法）。

### 关键技术细节
- Base 方法实现：VAR-LiNGAM 用 `lingam` 包 v1.5.2；PCMCI 用 `tigramite` 包 v4.2.0（与我们 topic 候选方法 PCMCI+/J-PCMCI+ 同一代码库家族）。
- k=7 是针对 T=1000 长度序列在合成数据上网格搜索得到的经验值，无理论推导；不同 T/base method 是否需要重新调 k 未讨论。
- 阈值：VAR-LiNGAM 用 τ_c=0.4, τ_v=0.4；PCMCI 用 τ_c=0.7, τ_v=0.4——PCMCI 阈值更高，理由引用 Assaad et al. (2022)："PCMCI 是 recall-oriented，容易产生更多相关但也更多假边，因此需要更严格的一致性门槛才能可靠"。
- 合成数据生成器：VAR(p) 骨架，B_0 下三角矩阵控制同期效应，B_1..B_p 控制滞后效应；可选 tanh 非线性、可选 t 分布非高斯噪声、可选趋势项制造非平稳、随机游走扰动模拟真实数据的不可预测性；含稳定性修正保证过程平稳。控制变量：变量数 {5,20,50}，序列长度 {250,1000,2000}，每种配置生成 10 个数据集。
- fMRI 基准：Smith et al. (2011) 模拟数据，Nauta et al. (2019) 预处理，27 个脑网络，每网络 5-15 变量，T ∈ [50,5000]（均值 774），10-33 条真实因果边，含非线性/自因果/混杂因子，因果充分（无隐变量），因计算成本排除了 >15 变量的网络。
- 评估指标：SHD（边增删反转计数）、F1（无向边存在性）、F1-directed（主指标，符号感知的方向准确率——文中给出一个具体数值例子：真实 A→B→C→D 三边图，若推断出 D→C 而非 C→D，标准 F1=1.0 但 F1-directed=0.667，直观展示两指标差异）、Frobenius 范数（计算但未列表）、Runtime。
- 硬件/软件：MacBook Pro 14" 2023（M2 Pro, 32GB RAM），macOS Ventura，Python 3.8，numpy 1.24.4/pandas 2.0.3/scikit-learn 1.3.2，VS Code venv。
- 全程对比 5 个方法：VAR-LiNGAM、VL-Bootstrap（bootstrap 鲁棒化基线）、RCV-VAR-LiNGAM（本文方法）、PCMCI、RCV-PCMCI（本文方法）。

### 核心结果
- **线性 vs 非线性**：RCV-VAR-LiNGAM 两种场景均最优（F1-directed 0.848 线性 / 0.819 非线性，对比 base VAR-LiNGAM 的 0.758/0.712），且非线性下性能衰减最小。
- **高斯 vs 非高斯噪声**：所有方法在非高斯噪声下更好（符合 LiNGAM 族对非高斯性的依赖）；但 PCMCI 反常——非高斯下反而更差（0.599→0.573），文中专门标注这一反直觉现象。
- **平稳 vs 非平稳**：PCMCI/RCV-PCMCI 在平稳数据上的表现同样很差（F1-directed 0.416/0.433），并非只在非平稳时差——说明 PCMCI 族在此实验设置下整体上限低，而非单纯对非平稳敏感；VAR-LiNGAM 族对非平稳只有轻微衰减（RCV-VAR-LiNGAM 0.864→0.838，降 3%）。
- **稀疏 vs 稠密**：**RCV 在稠密图上优势反转**——稀疏图 RCV-VAR-LiNGAM 领先（0.828 vs base 0.754），但稠密图反而输给原始 VAR-LiNGAM（0.766 vs 0.859）。这是文中明确承认的负面结果，作者猜测 CV 过滤机制在稠密结构下会误删"看似不稳定但实际真实"的边。
- **变量规模 (n=5/20/50)**：RCV-VAR-LiNGAM 在所有规模上 F1-directed 最优（0.792/0.678/0.669），衰减曲线最平；PCMCI 崩溃最快（0.401→0.164）；SHD 随 n² 量级恶化（VAR-LiNGAM 在 n=50 时 SHD=560，PCMCI 达 1740）；VL-Bootstrap 在 n=50 时超时（1 小时上限），计入零分。
- **序列长度 (T=250/1000/2000)**：RCV-VAR-LiNGAM 随数据增多而改善，在 T=1000/2000 最优（0.831/0.833）；但 **T=250 时反而略输给原始 VAR-LiNGAM**（0.765 vs 0.771）——说明 CV 折数据太短时不足以提供有效信息，RCV 并非在任何数据量下都占优。
- **fMRI 基准（更接近真实、非线性、有混杂）**：所有方法性能远低于合成数据（F1-directed ≤ 0.51），证实"合成数据表现"到"类真实数据表现"存在明显 gap；RCV-VAR-LiNGAM 仍最优（0.506 vs VAR-LiNGAM 0.462），运行时 0.78s（比 base 慢 4.5 倍，但绝对值仍很小）。
- **ABM 校验框架试点**（10 个数据集共享同一真实因果结构，1 个记为"真实世界"、9 个记为"ABM 输出"，属于理论上限应为 100% 相似度的自一致性测试）：RCV-VAR-LiNGAM 显著优于原始 VAR-LiNGAM——SHD 10.1 vs 26.7，符号相似度 0.890 vs 0.720，F1-directed 0.659 vs 0.488，代价是运行时慢 5 倍（5.16s vs 1.10s）。这是全文最直接的应用价值展示，但样本量极小（仅 10 个数据集、单次 pilot，无独立重复实验）。

### 明确局限
- **完全没有真实世界（非模拟）实证**：只有合成 VAR 数据 + 模拟 fMRI + 自一致性 pilot（真实因果结构人为设为相同）。作者在 §6.3 明确承认"further validation on diverse real-world datasets is necessary"。
- **未见代码/artifact 发布**：正文中只引用了两个第三方包（`lingam`、`tigramite`）的 GitHub 链接，RCV 封装层和数据生成器本身没有仓库链接。
- **k 折切分方式未交代清楚**：是否为保持时序依赖的连续时间块切分、还是随机切分，文中没有明说——这对于"时间序列交叉验证"这一核心声称是关键缺失，也未对照 blocked/rolling-origin CV 文献做讨论。
- **RCV 并非严格帕累托改进**：自己的表格显示稠密图和短序列（T=250）两种场景下 RCV 反而输给 base 方法，论文诚实报告但未深入解释原因，只有一句猜测。
- **阈值/超参靠手工网格搜索**：τ_c、τ_v、k 均在合成数据上经验调出，无自适应/数据驱动的阈值选择机制（作者列为未来工作）。
- **w=0 意味着"Adjustment"这半个机制在本研究中未被真正使用**——RCV 目前只是全量估计图上的一个过滤/剪枝步骤，并未把交叉验证信息真正融入效应量估计。
- **单一硬件配置**（M2 Pro 笔记本）、较旧的库版本（Python 3.8），未做超出 n=50/T=2000 的扩展性研究——这也是作者自己测出的实际天花板（VL-Bootstrap 在 n=50 已经超时）。
- **Consistency/Variability 是相对单一 anchor（G0）的逐边度量**，不是折间两两对称的度量，因此无法单独刻画"整张图的 regime 变化"，只能刻画局部单边相对全量估计的偏离。
- **与我们课题的关键差异**：本文从未把不稳定性本身当作预测信号——Consistency/Variability 纯粹是 QC 过滤器，目标是产出更干净的单点估计图，从未把这两个指标暴露成下游预警特征，也没有做滑动时间窗口版本，更没有连接到任何"事件发生前兆"预测任务。

### 跟我们的关系

- **直接命中同一个技术原语**：本文 Eq 1-2 的 Consistency/Variability 定义（k 折下因果效应符号一致率 + 效应量标准差归一化）与我们 topic 想构造的"因果效应不稳定性指标"在数学形式上几乎是同一件事的一种现成实现——应该把这两个公式当 baseline 特征直接借用，而不是从零发明。
- **但目的完全相反，这正是我们潜在的 novelty 落脚点**：本文把 Consistency/Variability 当 QC 过滤器——低一致性/高变异性的边直接被扔掉，目标是产出更干净的单点估计因果图（服务于 ABM 校验）。我们的 topic 恰恰要做相反的事：把这种不稳定性保留下来、当作强对流事件的预警信号本身——一个"缺陷"到"特征"的框架反转。但这也说明"因果效应不稳定性可以被结构化度量"这件事本身不是我们的原创，这篇提供了现成的度量公式和调参先例（τc/τv 因 base method 而异，k=7 for T=1000），novelty 审查时需要明确区分"度量方式"（沿用）与"使用目的"（反转）两层。
- **折叠方式的差异是关键 gap**：本文的 k 折是"数据子集"意义上的折叠（是否连续时间块未说清楚），不是"滑动时间窗口"意义上的折叠。我们 topic 明确想用"时间窗口滑动下产生的不稳定表现"作为信号来源之一——本文完全没做这个，只在 §6.3 Future Work 里提了一句"sliding window approach or time-varying parameter models"作为未探索方向，相当于间接承认这是空白点。
- **方法列表高度重合**：我们候选方法 PCMCI+/J-PCMCI+ 都是本文用的 PCMCI（Runge et al. 2019，同一作者后续工作）的直接扩展，同样基于 `tigramite` 包——可以直接复用其调用范式和超参数选择经验（如 PCMCI 需要比 VAR-LiNGAM 更高的一致性阈值，因为 PCMCI 是 recall-oriented、更容易产生假边）。VAR-LiNGAM 不在我们候选列表里，可忽略。
- **应用域完全不同，无场景撞车**：本文场景是金融 ABM 校验 + 模拟 fMRI 基准，我们是强对流气象——没有直接的应用层面重合，不构成 novelty 威胁。
- **一个值得警惕的同期信号**：反引文中 Tusoni et al. 2025《Robust Causal Discovery in Real-World Time Series with Power-Laws》(arXiv:2507.12257) 标题上明显是同一脉络的后续工作（鲁棒时序因果发现，2025-07 发布），值得后续 novelty-check 单独细读，判断它是否比本文更接近我们的"不稳定性即信号"框架。
- **可直接复用的工具**：`tigramite`（PCMCI/PCMCI+/J-PCMCI+）调用方式、F1-directed 指标定义（含符号方向）、SHD 都是我们评估阶段可以直接复用的标准化工具，不需要重新发明。

## Read by: 0710-causal-scs (2026-07-11)

本文是"因果效应不稳定性作为强对流预警指标"这一课题目前读到的、与核心技术原语（因果效应的一致性/变异性度量）撞得最正面的一篇。它把同样的 Consistency/Variability 公式用在完全相反的目的上——过滤噪声而非提取信号——这既是一种风险（说明"因果效应不稳定性可被结构化度量"不是我们的原创），也是一种机会（"把不稳定性当预警信号而非噪声"的框架反转、加上滑动时间窗口设计、加上强对流气象场景，三者叠加构成潜在差异化点，且都在本文的 Future Work 空白区之内）。

后续建议：
1. 优先精读 Tusoni et al. 2025 (2507.12257)，确认它是否已经把"鲁棒性/不稳定性"往信号方向而非过滤方向做了推进。
2. 精读 CD-NOD (1903.01672)，理解非平稳因果发现的标准做法，对照我们"保留假设违背影响而非修正"的立场差异。
3. 若后续要用 fMRI 或类似 benchmark 做方法验证前的低成本 sanity check，可参考本文 §4.1.2 的数据来源链路（Smith 2011 → Nauta 2019 预处理）。
4. 本文的 F1-directed、SHD、Consistency/Variability 公式可直接作为我们评估阶段的标准化基线指标，无需重新定义。
