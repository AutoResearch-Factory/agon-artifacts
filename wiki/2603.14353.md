# LawMind: A Law-Driven Paradigm for Discovering Analytical Solutions to Partial Differential Equations (2603.14353)

- **Year**: 2026.03
- **Authors**: Min-Yi Zheng (SCUT), Shengqi Zhang (EIT), Liancheng Wu (SCUT), Jinghui Zhong (SCUT), Shiyi Chen (EIT/PKU), Yew-Soon Ong (NTU)
- **Venue**: arXiv preprint (Springer Nature sn-jnl template)
- **Keywords**: symbolic regression, PDE analytical solution, evolutionary computation, law-driven discovery, hierarchical search, closed-form solution, physics-constrained optimization, expression tree
- **One-liner**: 提出 LawMind 框架，通过分层进化搜索从 PDE 控制方程直接进行符号推导，在 100 个 benchmark 上全部恢复解析解，并发现了此前未报道的三阶 PDE 闭式解。

## 精华提炼

### 管线/方法

**输入**：PDE 控制方程 N[u(x,t); λ] = 0 + 关联条件 C（初始条件 u(x,0)=g(x)、边界条件或源项）。

**Pipeline（图 1f 的闭环流程）**：

1. **分层搜索空间构建 (Hierarchical Search Space Construction)**：按阶段逐步将变量激活进入 terminal set T^(k)。第一阶段只有 x 和常量，后续阶段逐步加入 t 等变量。每次只让一个新变量及其合法变体进入 T^(k)，严格控制搜索空间膨胀。

2. **结构候选组装 (Structural Assembly)**：在当前 stage k，从 terminal set T^(k)、function set F（数学运算符）和 position set P^(k)（当前表达式树的所有节点）组合候选子树。每个候选 = 选择一个插入位置 p + 一个算子 f + 一个新变量表达式。

3. **进化扩展 (Evolutionary Expansion)**：将候选子树插入当前表达式的指定位置。插入机制：在位置 p 引入新内部节点（算子 f），原节点和新叶子节点分别作为左右子树，左右分配是额外二值选择。Stage 1 从初始条件表达式初始化，后续 stage 从前一 stage 的已验证解出发。

4. **律动符号评估 (Law-Driven Symbolic Evaluation)**：两步验证：
   - Step 1：将候选表达式代入 PDE，进行**符号恒等计算**（非数值采样），判定 residual 是否恒为零。
   - Step 2：检查初始条件兼容性（令 t=0，检查是否化简为 u(x,0)）。
   - Fitness 为严格二值：两步都通过才接受，否则拒绝并继续搜索。

**输出**：满足 PDE + 条件的闭式参数化解 ũ(x,t)（保留自由参数的符号表达式）。

### 关键技术细节

- **表达树表示**：叶子节点 = 自变量和常量 (T)，内部节点 = 数学算子 (F)
- **搜索空间控制**：不分层时复杂度 O((2|F||T|)^(|P|×M))，分层后降为 O(Σ(2|F||T^(k)|)^|P^(k)|)，其中 |T^(k)| ≤ |T| 且 |P^(k)| ≪ |P|×M。核心是将指数增长的 flat search 分解为逐 stage 的求和。
- **符号验证而非数值逼近**：PDE residual 的计算是严格的符号恒等式检查，对任意变量和参数取值成立，而非在 collocation points 上的数值逼近。这是区分于 PINNs 和其他数值方法的关键。
- **参数化自由**：发现的是含自由参数（A, B, C...）的参数化解族，而非单一数值特解。
- **等价性判定**：同一解的不同参数化形式（如参数重标定）通过数学等价性验证而非要求形式完全一致。

**新发现案例**：
- 三阶线性 PDE：描述具有空间变化粘性系数的特殊粘弹性流体（含记忆效应的材料振动行为），形式为 u_t - ∂_x(μ exp(-kx) u_x) + ∂_x(α exp(-kx) u_xt) = 0
- 三阶非线性 PDE：描述广义粘性 Burgers 模型（Helmholtz 算子正则化物性导数），形式为 u_t + u u_x - u_xx - ∂_xx(u_t + u u_x) = 0

**Surprising 发现**：LawMind 在没有任何 transformation template 或 human prior 的情况下，从初始条件出发自动构造出了含 Bessel 函数 I_1、K_1 的复杂解析解，这表明纯符号搜索可以"发现"需要使用特殊函数的解形式。

### 明确局限

1. **搜索可扩展性**（作者自述）：符号变量数、初始条件复杂度、terminal set 大小三者共同决定搜索空间规模，呈组合增长。复杂解可能超出当前搜索过程的实际可达范围。
2. **依赖时间变量 + 初始条件**（作者自述）：当前框架需要时间演化结构，以初始条件作为解构建的结构锚点。无法处理仅有边界条件而无时间变量的 PDE。
3. **无耦合 PDE 系统**（作者自述）：仅处理单方程场景。
4. **无代码/artifact 发布**（审读者发现）：方法描述抽象，缺少具体实现细节（function set 具体包含哪些算子、进化参数设定、计算预算等），第三方复现困难。
5. **100% 成功率可能暗示选择偏差**（审读者发现）：benchmark 的 100 个 PDE 全部有已知参考解，本质上是一个 recovery task 而非 discovery task。对新 PDE 类的泛化能力未在 benchmark 上检验。
6. **无消融实验**：没有分析 hierarchical activation vs flat search 的实证对比、不同 terminal set 策略的影响、或计算成本随问题复杂度的 scaling。

### 跟我们的关系

**对 0616-fex 课题（FEX: RL for Symbolic Regression）**：

- **撞车评估**：**中度相关但不直接撞车**。LawMind 使用进化算法（非 RL）做符号回归用于 PDE 解发现，与 FEX 的 RL 方法共享"用符号表达树表示数学表达式"这一核心表示。但目标不同：LawMind 专攻 PDE 解析解（law-driven evaluation 是核心差异），FEX 是通用符号回归方法。
  - 如果 FEX 被用于 PDE 解发现场景，且也用 PDE residual 做 fitness，则与 LawMind 高度重叠（此时 FEX 的 RL 策略 vs LawMind 的进化策略 + 分层激活是实质差异）。
  - 如果 FEX 保持通用 SR 定位，则在方法层面不直接撞车。但在文献综述和 related work 中必须引用 LawMind。

- **可用工具/方法**：
  1. **分层变量激活策略**（Hierarchical Search Space Construction）：可借鉴到 FEX 中，用于处理多变量符号回归时的搜索空间控制问题。FEX 的 RL agent 可以在不同阶段被限制只能使用部分变量。
  2. **律动符号评估**（Law-Driven Evaluation）：如果 FEX 未来进入 PDE 解发现领域，直接使用 PDE residual 的符号恒等计算作为 reward signal 是显然的思路。LawMind 的两步验证 + 二值 fitness 设计值得参考。
  3. **Benchmark 数据集**：100 个 PDE 的 curated benchmark 可以直接用作 FEX 的评测集，特别是如果 FEX 想展示在物理/数学领域的应用能力。
  4. **新发现 PDE 作为 challenge problem**：LawMind 新发现的 4 个三阶 PDE 解可以作为 FEX 的 test case。

- **差异化**：
  - FEX 使用 RL（policy gradient / actor-critic），LawMind 使用进化搜索（evolutionary insertion），方法论本质不同
  - FEX 的 RL 框架天然支持 learn from experience（跨问题的迁移学习），而 LawMind 是 per-problem 的独立搜索
  - LawMind 的 law-driven evaluation 不需要数据，FEX 通常需要 data-fitting reward；如果 FEX 加上 law-driven reward，就是一个新的 hybrid 方向
