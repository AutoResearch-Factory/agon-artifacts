# CompNO: A Novel Foundation Model approach for solving Partial Differential Equations (2601.07384)

- **Year**: 2026.01
- **Authors**: Hamda Hmida, Hsiu-Wen Chang Joly, Youssef Mesri (Mines Paris - PSL, CEMEF & CAOR)
- **Venue**: arXiv
- **Keywords**: Scientific foundation models, Neural operators, PDE simulation, Computational fluid dynamics, Fourier neural operator, Compositional learning, Parametric PDEs, Boundary condition enforcement
- **One-liner**: 提出 CompNO——将 PDE 分解为基本算子（对流/扩散/非线性对流），各自独立预训练为 Foundation Block，再用轻量 Adaptation Block 组装成目标方程求解器，而非单一大模型预训练。

## 精华提炼

### 管线/方法

CompNO 的 pipeline 三阶段：

1. **Foundation Blocks 预训练**：对每个基本微分算子（对流 ∂_t + β∇, 扩散 ∂_t − νΔ, 非线性对流 ∂_t + u·∇），独立训练一个 Parametric Fourier Neural Operator (PFNO)。每个 PFNO 学习映射 H: (f(x,t), γ) → f(x, t+dt)。输入函数 f(x) 与参数 γ ∈ R^p 拼接后经 lifting operator P: R^{1+p} → R^{d_h} 投影到高维，过 4 层 Fourier integral operator layers（GELU 激活），再经 projection operator L: R^{d_h} → R^1 回到物理空间。预训练后权重冻结存入库。

2. **Adaptation Blocks 组装**：冻结所有 Foundation Block 权重。对目标 PDE（如对流-扩散、Burgers），将输入同时送入相关 Foundation Block，取各 block 的嵌入表示（projector 之前的高维特征），由 Aggregator 组合后投影回物理域。对流-扩散用单层线性层（width 256）；Burgers 用两层 MLP（width 256→512, GELU）。

3. **BC Constraint Operator**：基于 BOON (Saad et al., 2023) 的改进算法。通过评估 FNO kernel 在边界 impulse e_0/e_L 上的响应，修正输入端边界值，再前向传播后将输出边界硬设为 Dirichlet 条件值，保证边界 loss = 0。

输入: 初始状态 F(x,t0) + 参数向量 γ + 边界条件 BCs → 输出: 下一时刻的解 u(x, t+dt)。可链式推演到任意时间步。

### 关键技术细节

- **算子架构**：PFNO 为核心模块，4 层 Fourier integral operator layers，特征维度 d_h=128，GELU 激活。Fourier 层公式：ε_{l+1} = σ(F^{-1}{R_l(F{ε_l})} + W_l(ε_l))，R_l 为频域可训练权重张量。
- **数据集**：PDEBench 1D 子集。5 个方程——对流 (Eq.18)、扩散 (Eq.19)、非线性对流/inviscid Burgers (Eq.20)、Burgers (Eq.21)、对流-扩散 (Eq.22)。域 x∈(0,1), t∈(0,2], dt=0.01。初始条件为叠加正弦波 u_0(x)=Σ A_i sin(k_i x + φ_i)，k_max=8, N=2, A_i∈[0,1], φ_i∈(0,2π)。8000 训练 + 2000 测试。数据生成用有限差分：纯对流用一阶迎风（稳定性），含扩散项用二阶迎风+中心差分。
- **超参数**：
  - 预训练：lr=1e-3, StepLR(gamma=0.5, step=100 epochs), 1000 epochs, batch=50, MSE loss, Adam(weight_decay=1e-4)
  - 组装训练：lr=1e-4, StepLR(gamma=0.5, step=100 epochs), 100 epochs, batch=4, MAE loss
  - 硬件：单 NVIDIA A100 40GB
- **参数范围**：Convection Block: β∈{0.1,0.4,0.7,1,2}; Diffusion Block: ν∈{0.01,0.1,0.2,0.5,1,2}; NL Convection Block: 无参数（用标准 FNO 而非 PFNO，参数量 75k vs 217k）
- **预训练 MSE**：Convection 9.5e-5(train)/9.8e-5(test); Diffusion 5.7e-4/5.8e-4; NL Convection 7.6e-4/8e-4
- **预训练 block 展现时间外推能力**：训练在 t∈[0,2] 上，能外推到更长时间，预测趋势正确

### 核心结果

| PDE | CompNO (RL²) | PDEFormer-FS | ICL-based |
|-----|-------------|-------------|-----------|
| Convection | **0.0071** | 0.0124 | 0.0184 |
| Diffusion | **0.0075** | - | 0.0120 |
| Convection-Diffusion | **0.012** | - | 0.0231 |
| Burgers' (ν=0.1) | 0.028 | **0.0135** | - |
| Burgers' (ν=0.01) | **0.0371** | 0.0399 | - |

- Convection-Diffusion MAE：CompNO-v1 (train 0.008 / test@Nx=2048 0.019 / boundary 0) vs PFNO (train 0.039 / boundary 1.53)。模型在 2× 分辨率泛化良好。
- Burgers' MAE：CompNO-v2 (train 0.01 / test@Nx=2048 0.03 / boundary 0) vs PFNO (train 0.002 / boundary 0.01)。PFNO 在这个简单参数化场景（ν 仅 5 个离散值）略优。
- **Surprising**：CompNO 在低粘性 Burgers (ν=0.01) 反超 PDEFormer，且误差在两个粘度设置间更稳定（0.028→0.0371，而 PDEFormer 从 0.0135→0.0399 近乎翻三倍）。说明组合式架构在复杂物理场景更有优势。
- Ablation study：加 BC correction 使 MAE 从 0.025 降至 0.004。
- Pe/Re 敏感性：Pe≤2（扩散主导）最优；Pe 增大逐渐退化。Re≤1（层流）稳定；Re=10-100 过渡区误差波动增大；Re≥100 部分恢复（捕获大尺度结构但小尺度波动差）。

### 明确局限

1. **仅限 1D**：所有实验都是一维。FNO 本身维度无关，但 2D/3D 扩展未验证。作者承认这是主要局限。
2. **聚合机制简单**：对流-扩散用单层线性，Burgers 用浅层 MLP。在高 Pe/Re 下性能下降，说明当前聚合不足以捕获强非线性耦合。
3. **Foundation Block 库很小**：仅 3 个 block（对流/扩散/非线性对流），只覆盖了 CFD 中的基本算子。要成为真正的 SFM 需要扩展到更多物理过程。
4. **无真机实验**：纯数值 benchmark（PDEBench）。无真实物理系统验证。
5. **未与标准数值方法对比计算成本**：只比了精度，没说比传统有限差分/有限元快多少。作者自己提到了这个缺憾。
6. **固定 dt**：训练在 dt=0.01 上进行，虽然可以通过链式推理或插值处理不同时间分辨率，但未系统性验证。
7. **Burgers 方程 PFNO baseline 更强**：在简单参数化场景下 PFNO 精度更高（0.002 vs 0.01），说明 CompNO 的组装过程引入了一定精度损失。
8. **预训练需要模拟数据**：Foundation Block 仍需传统求解器生成数据预训练，不是无监督方法。

### 跟我们的关系

针对 **0616-fex (FEX: RL for Symbolic Regression)** 课题：

- **不撞车**：CompNO 解决 PDE 正问题（给定方程→求解），FEX 解决逆问题（给定数据→发现符号方程）。方向互补而非竞争。
- **可用的方法/工具**：
  1. **Compositional 思想可借鉴**：FEX 的 operator set 目前是固定的数学原语集。CompNO 的 "预训练基本算子→组装复杂系统" 范式启发：是否可以在 FEX 中引入分层搜索策略——先搜索子表达式（对应 Foundation Blocks），再搜索它们的组合方式（对应 Adaptation）？
  2. **PFNO 的参数化方法**：将 PDE 参数拼接入输入函数的技巧（f*(x)∈R^{1+p} → 高维嵌入）可启发 FEX 如何在 RL 搜索中处理 parametric PDE 发现（方程形式不变但参数变化）。
  3. **BC 硬约束**：如果 FEX 未来用于发现带边界条件的 PDE，BOON 式的 kernel 修正可确保发现方程的解满足边界条件，作为搜索的可行性质检。
- **该引不该引**：如果 FEX 论文讨论 PDE 发现的前向求解器选择或 SFM 相关工作，CompNO 是相关的近期工作（2026.01）。但不属于核心竞品。
- **差异化**：FEX 的核心是符号搜索 + RL，CompNO 的核心是神经算子 + 组合分解。两条技术路线正交，但都强调"从简单组件构建复杂系统"的哲学——这是可在 related work 中建立联系的点。

## Read by: 0616-fex (2026-06-24)

CompNO 与 FEX 不做同一件事——CompNO 是前向 PDE 求解器，FEX 是逆问题符号发现器——因而不存在直接竞争。但 CompNO 的核心理念「分解→预训练→组装」（decompose-pretrain-assemble）对 FEX 有潜在启发：

1. **分层符号搜索**：FEX 当前在扁平 operator set 上搜索。CompNO 提示可以先识别子结构（如扩散项、对流项），再搜索它们的组合方式。类似地，FEX 可以引入两阶段 RL：先学高频子表达式模板（"Foundation Block"），再学如何组合它们。

2. **参数化算子**：CompNO 将 PDE 参数作为模型输入，使同一 block 可处理不同 β/ν。FEX 在发现 parametric PDE 时，可借鉴这种将参数空间纳入搜索的显式编码方式，而非将参数作为待定常数。

3. **前向求解器选择**：如果 FEX 在 RL reward 计算中需要快速前向求解 PDE（而非查表），CompNO 这种轻量、可组合的神经算子可能比传统数值求解器更快，适合嵌入 RL loop 作为 rollout 评估器。

4. **撞车风险评估：LOW**。CompNO 是 neural operator + composition，FEX 是 RL + symbolic regression，技术路线正交。唯一的交叉点是如果 FEX 将来扩展到「自动发现 PDE 的算子分解结构」，那就需要考虑 CompNO 的先验工作。

建议：在 FEX related work 中将 CompNO 列为 SFM 方向的代表性工作之一，强调正交性和互补性，不构成 novelty 威胁。
