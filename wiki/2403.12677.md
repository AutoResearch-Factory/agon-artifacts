# Causal Change Point Detection and Localization (2403.12677)

- **Year**: 2024.03
- **Authors**: Shimeng Huang (1st), Jonas Peters, Niklas Pfister — Univ. of Copenhagen (Dept. of Mathematical Sciences) + ETH Zürich
- **Venue**: arXiv preprint (stats.ME / stat.ML), 2024-03; single-group methodology paper
- **Keywords**: causal change point, invariant causal prediction, structural change, regression change point, change point localization, causal stability loss, seeded binary segmentation, Chow test, offline change point detection
- **One-liner**: 定义"因果变点(CCP)"——只在 Y|X 的因果机制(而非 X 边缘分布)改变时才触发的变点,用 invariance 刻画,并给出检测+定位方法(纯仿真验证)。

## 精华提炼

### 管线/方法
本文是**统计方法论**论文,不是学习/系统论文。核心贡献是一套区分"因果变点"与"非因果结构变点"的定义 + 检测 + 定位框架。

**输入**:一段独立观测序列 $(X_1,Y_1),\dots,(X_n,Y_n)$,$X_i\in\mathbb{R}^{d+1}$(含截距 $X^{d+1}=1$),$Y_i\in\mathbb{R}$。offline 设置(整段数据已知)。

**三层变点定义(从宽到窄)**:
1. **一般变点**:$\mathbb{P}_k^{X,Y}\neq\mathbb{P}_{k-1}^{X,Y}$(联合分布变)。
2. **回归变点 RCP**(Def 2.1):总体 OLS 系数变 $\beta_k^{OLS}\neq\beta_{k-1}^{OLS}$ **或** 残差分布变 $\epsilon_k\overset{d}{\neq}\epsilon_{k-1}$。这就是经典 econometrics 的"structural change"。
3. **因果变点 CCP**(Def 2.4):对**所有**子集 $S\in\mathcal{S}$(含截距的协变量子集),都有 $\beta_k^{OLS}(S)\neq\beta_{k-1}^{OLS}(S)$ **或** $\epsilon_k(S)\overset{d}{\neq}\epsilon_{k-1}(S)$。即**不存在任何一个在 $\{k-1,k\}$ 上保持不变(invariant)的子集**。
- **关系**:CCP $\subseteq$ RCP。RCP 里不是 CCP 的叫 **NCCP(非因果变点)**——那些只是 $X$ 边缘分布漂移导致 OLS 变化、但 $Y$ 的因果机制没变的点。
- **等价刻画**(Prop 2.6):$k$ 是 CCP $\iff$ 不存在 $\{k-1,k\}$-invariant 集 $S$。这把"因果变点"完全建立在 **invariance(可观测量)** 上,**不需要指定因果模型**。

**与真实因果的挂钩**(Sec 2.1,Setting 2.7 = 带隐变量的 sequential 线性 SCM):在"$Y$ 的噪声与其父节点不相关"(式 no_confound: $\mathbb{E}[X^{PA(Y)}g(H,\epsilon^Y)]=0$)的额外假设下,Prop 2.8 证明 CCP $\Rightarrow$ 因果系数 $\beta_i$ 变 或 $Y$ 的噪声项 $g_i$ 分布变。**但若存在隐混淆**(Example 2.9),该蕴含关系失效——CCP 不再等于因果机制改变(诚实地给了反例)。

**检测**(Sec 3):区间 $I$ 无 CCP $\equiv$ 零假设 $\mathcal{H}_0^I$: 存在某 $S$ 是 $I$-invariant。做法:对每个 $S$ 分别检验 $\mathcal{H}_{0,S}^I$(该子集是否 invariant),再用 **min 组合**——只有当**所有** $S$ 的检验都拒绝时才判定有 CCP(Prop 3.2 保证 level $\alpha$,因为 null 下至少存在一个 invariant 的 $S$ 其检验控制水平)。单个 $\mathcal{H}_{0,S}^I$ 的检验:把区间对半分,用 **Chow 检验**(chow1960),或用 pfister2019 的 seq-ICP 重采样法。

**定位**(Sec 4)——两条路线:
1. **Prune-\***(剪候选):给定候选集 $\mathcal{K}$(通常用现成 RCP 定位法如 strucchange/breakpoints 估出),对每个候选 $k_j$ 在区间 $I_j=\{k_{j-1},\dots,k_{j+1}-1\}$ 上做 $\mathcal{H}_0^{I_j}$ 检验来剪枝(Algo 1)。Prop 4.1 给出有限样本界:$\mathbb{P}(\hat{\mathcal{T}}\subseteq\mathcal{T})\geq 1-(|\mathcal{K}|-|\mathcal{T}|)\alpha$ 和 $\mathbb{P}(\mathcal{T}\subseteq\hat{\mathcal{T}})\geq 1-|\mathcal{T}|(1-\beta)$;可用 Bonferroni 修正。缺点:候选集若漏了真 CCP,估计可任意偏。
2. **LossCS**(损失函数):定义 **causal stability loss** $\mathcal{L}_{I,s}(i)$,基于 **minimal OLS instability** $\mathcal{C}_s(I)=\min_{S}\sum_r(V_{P_r^c,P_r}(S)-V_{P_r,P_r}(S))^2$(在长度 $s$ 的分段上度量"非不变程度",$V$ 是残差平方的期望)。性质:无 CCP 时 $\mathcal{C}_s(I)=0$(Prop 4.3);单 CCP 区间里 loss 在真 CCP 处取 0(Corollary)。多 CCP 定位:把 LossCS 插到 **标准二分分割**(vostrikova1981)或 **seeded binary segmentation**(kovacs2020)+ narrowest-over-threshold(baranowski2019)里,再加剪枝。

**输出**:CCP 的存在性判定 + 位置估计 $\hat{\mathcal{T}}$。

### 关键技术细节
- **协变量子集遍历**:CCP 定义与检测都要遍历 $\mathcal{S}$(所有含截距的子集),规模 $2^d$——**对维度指数级**,论文未深入讨论可扩展性(实验只有 4 个协变量)。
- **Chow 检验**(Appendix D):对半分区间,$F(d, l-2d)$ 分布(大样本)或 $F(l_2, l_1-d)$(小样本)。
- **seeded intervals**(Algo 3):decay 参数 $a\in[1/2,1)$,层数 $L=\lfloor 1+\log_{1/a}(n/s)\rfloor$;bottom-up(窄到宽)保证每个区间至多含一个 CCP。
- **loss 的最小分段长** $s$ 是关键超参;实验中设为 $0.1n$ 或 $0.2n$。
- **无神经网络、无训练**:纯经典统计估计(OLS + 假设检验 + 分段搜索)。
- **仿真数据生成**:5 节点 SCM(式 5.1),$X^1\to X^2$, $X^1\to Y$, $X^2\to Y$, $Y\to X^3$, $X^4\to X^3$;15 个参数分成 4 个"因果参数"($\beta^{15},\beta^{25},\mu^Y,\sigma^Y$)和 11 个"非因果参数";改因果参数=CCP,改非因果参数=NCCP。所有实验 **200 次重复**。
- **baseline**:`breakpoints`(bai2003 的多结构变点,R 包 strucchange / zeileis2003)。
- **代码**:https://github.com/shimenghuang/CausalCP(公开)。

### 核心结果
- **检测**:Chow 检验方案在 CCP 位于区间**中点**时 power 最高、靠边界时最低(因为对半分后一侧混了两个分布);"无 CCP"时正确把 type-I error 控在 5%。
- **定位 Exp1**(1 CCP + 2 NCCP,增大 $n$):除 `BP-1` 外所有方法随 $n$ 增大都收敛到真 CCP;`BP-1`(纯结构变点定位)错误地收敛到第二个 **NCCP**——直接证明经典结构变点法会把非因果漂移误判为变点。`LossCS` 优于两种剪枝法(后者常常一个 CCP 都检不出)。
- **定位 Exp2**(单 CCP 不同位置):`LossCS` 在 CCP 靠近中心时准;靠左/右边界时系统性高估/低估位置。
- **定位 Exp3**(2 CCP + 1 NCCP):大样本下 `Prune-BP` 最好;**小样本下 `LossCS-SeededBS-Prune` 最好**(假阳性最少、真阳性最多)。`BP-2` 无效(会把 NCCP 当变点)。`LossCS-StdBS` 必须配剪枝,否则会先切在 NCCP 上产生大量假阳。
- **总结**:两族方法互补——`Prune-*` 有检验保证(小样本可保守),`LossCS-*` 可直接对接各种分割算法且不需先估 RCP。
- **⚠️ 反直觉/关键**:经典 structural-change 定位(strucchange/breakpoints)在存在非因果漂移(NCCP)时会**系统性误报**;本文方法是专门修这个的。

### 明确局限
- **只有仿真,零真实数据**——对一个应用导向的变点方法这是最大短板;所有"ground truth CCP/NCCP"都是作者按参数构造出来的(自证)。
- **仅线性回归**:作者明说可推广到半参/非参(用"invariant functions"代替"invariant sets"),但本文没做。
- **假设观测独立(iid),无时间依赖**:作者列为 future work;真实时间序列几乎都有序列相关。
- **子集遍历 $2^d$ 指数复杂度**:高维不可扩展,论文回避。
- **隐混淆下"因果"解释失效**(Example 2.9,作者诚实承认):CCP 的因果含义依赖"$Y$ 噪声与父节点不相关"的强假设。
- **多 CCP 区间里 loss 未必在真 CCP 取最小**,需要 seeded/剪枝补救。
- **单一响应变量 $Y$**:只处理"$Y$ given $X$"的机制变化,不处理整张 DAG 的结构变化(对比 huang2020 的 CD-NOD 处理全图非平稳)。

### 跟我们的关系（0710-causal-scs）
**这篇是本课题的核心近邻,撞车风险高,必须正面处理。**

- **撞车点(具体到 claim)**:如果我们的"causal-scs-indicator"要做的是"用 invariance 区分因果机制变化 vs. 单纯分布漂移,并给出一个逐时刻的指标",**本文在线性/offline/iid 设定下已经做了**——CCP 定义 = 因果结构变化指示器,causal stability loss $\mathcal{L}_{I,s}(i)$ = 逐时刻连续指标。核心概念 claim(invariance-based 因果变点)不再新。
- **差异化必须走这几条(本文明确留的口子)**:
  1. **非线性/非参机制**——本文只做线性,Discussion 明说未来方向是"invariant functions"。用核方法/神经网络做非参 invariance indicator 是干净的 delta。(注意:反引文 2605.05809 已经用 kernels+copulas 往这个方向走了,先查它撞不撞。)
  2. **在线/流式检测**——本文是 offline;online causal-shift indicator 有空间。
  3. **时间依赖**——本文假设 iid,序列相关下的因果变点是空白。
  4. **真实数据验证**——本文零真实数据,任何带真实/半真实 benchmark 的 causal-scs indicator 都能补这个洞。
  5. **对隐混淆鲁棒**——本文因果解释在混淆下失效(Ex 2.9),鲁棒于混淆的指标是实打实的贡献。
  6. **可扩展性**——本文 $2^d$ 子集遍历,高维需要更聪明的 indicator。
- **可直接复用的工具/方法**:invariance 检验(ICP peters2016 / seq-ICP pfister2019)、Chow 检验、causal stability loss 作为分割算法的插件 loss、seeded binary segmentation(kovacs2020)+ narrowest-over-threshold(baranowski2019)、strucchange/breakpoints(bai2003)作 RCP baseline、CausalCP 开源代码作参考实现与对比 baseline。
- **该不该引**:**必引**,且应作为最直接的 prior work / baseline 来对比与区分。定位差异化时,把它的"RCP vs CCP"框架直接拿来说明我们指标的因果性从何而来。
