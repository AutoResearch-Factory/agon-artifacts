# Prithvi-EO-2.0: A Versatile Multi-Temporal Foundation Model for Earth Observation Applications (2412.02732)

- **Year**: 2024.12 (v3 as of read)
- **Authors**: Daniela Szwarcman, Sujit Roy, Paolo Fraccaro, et al. (IBM Research; NASA Marshall Space Flight Center; Univ. of Alabama in Huntsville; Univ. of Iceland; Jülich Supercomputing Centre; Arizona State Univ.; Oregon State Univ.; Clark Univ.; Boston Univ.; UC Berkeley; Universities Space Research Association — 35 authors, 12 institutions)
- **Venue**: arXiv preprint (IEEE journal-track template; targets IEEE TGRS-style venue)
- **Keywords**: geospatial foundation model, masked autoencoder (MAE), Vision Transformer, multi-temporal remote sensing, HLS (Harmonized Landsat Sentinel-2), self-supervised pretraining, GEO-Bench, temporal/location embeddings, TerraTorch, SME-led downstream evaluation
- **One-liner**: 6 亿参数、训练于 4.2M 组 HLS 时序影像的多时相 ViT-MAE 地球观测基础模型，在 GEO-Bench 与 9 个 SME 真实下游任务上全面超越前代 Prithvi-EO-1.0 及 6 个同类基础模型。

## 精华提炼

### 管线/方法

**预训练数据构建**（第一大贡献）：
1. 数据源：NASA HLS（Harmonized Landsat Sentinel-2）产品，30m 分辨率，2-3 天重访周期，2014-2023 十年跨度。
2. 波段：取 Landsat/Sentinel-2 共有的 6 个波段（Blue, Green, Red, NIR, SWIR1, SWIR2）。
3. Tile 级采样：按 Copernicus Land Cover 100m + RESOLVE Ecoregions 标签计算每个 HLS tile 的 LULC 类别占比与生态区占比；每类取占比最高的 500 tiles 中的 100 个；城市区域过采样至覆盖全球~60% 城市面积（1000 tiles）；额外采样 1000 个高 LULC 熵（异质景观）tiles；846 个生态区中 712 个在 ≥3 个 tile 中有代表（68 个因面积不足被舍弃）；95%-5% train-val 切分，最终 3028 训练 + 163 验证 tiles。
4. Patch/序列级采样：每个 tile 内构建 4 帧时间序列，帧间隔 1-6 个月（保证覆盖季节变化又不过度增加 token 数）；序列切成不重叠的 256×256 patch；丢弃缺失值>1% 或云污染>20% 的样本（用 Fmask 波段判定）；缺失值用最近邻插值填补；每 tile 每 patch 位置最多保留 10 个样本（验证集 2 个）以防云清晰区域过表征；训练/验证空间不重叠。
5. 后处理修正：预训练实验后追加过滤——剔除格陵兰中部持续伪影 tiles，对海洋/沙漠纯净区域降采样（避免同质区域过表征）。
6. 最终数据集：**4.2M 训练 + 46k 验证样本**，每样本 4×256×256×6（T×H×W×C）。

**模型架构**（第二大贡献，两处关键创新）：
1. 基于标准 MAE（ViT encoder-decoder，非对称，随机 mask 一部分 patch，仅可见 patch 过 encoder，decoder 用 mask token 重建，pixel-space MSE loss）。
2. **创新一——3D patch embedding**：把 2D patch embedding 换成 3D 卷积，把 T×H×W 输入切成不重叠的 (t,h,w) cube；因卫星非规则低频重访，设 t=1（不像视频用 t>1）；3D 位置编码 = 分别生成 time/height/width 三个 1D sin/cos 编码后拼接。
3. **创新二——metadata 条件化（时间+地理位置）**：对每张图独立编码日期（year + day-of-year）和中心经纬度（各自 1D sin/cos 编码后拼接），通过**带学习权重的加权和**（而非拼接进 patch embedding）注入到 embedded token；训练时以概率 0.1 随机丢弃 metadata（temporal 和/或 location），让模型学会在缺失 metadata 时仍工作——这是因为下游任务常常没有精确的拍摄日期/坐标可用。
4. 两种规模：300M（ViT-L）和 600M（ViT-H）；每种都训练了带 TL（temporal+location embedding）和不带 TL 的版本。
5. 训练规模：400 epoch；300M 用 80×A100-40GB（~21,000 GPU-hours）；600M 用 240×A100（~58,000 GPU-hours）；global batch size 3840；weight decay 0.05；cosine LR schedule，peak LR 5e-4，40-epoch linear warmup；随机裁剪 256→224，随机水平翻转；训练于 JUWELS 超算（Jülich）。

**评测方法**（第三大贡献）：
1. **Benchmark 轴**：GEO-Bench（6 分类 + 6 分割数据集，0.1m-30m 分辨率跨度），用 TerraTorch-iterate（Optuna 贝叶斯优化）做超参搜索（10 trials），最优超参下 10 个随机种子重复实验，报告 mean/std/max/min。对比 6 个已发表 GFM（MOCO/DINO-ResNet50 对比学习/蒸馏、DeCUR、ScaleMAE、DOFA、Satlas）+ Prithvi-EO-1.0。
2. **应用轴（SME-led）**：三大类 9 个真实下游任务——灾害响应（洪水制图 Sen1Floods11、野火疤痕 HLS-burn-scars、烧毁强度 BAER、滑坡检测 Landstide4Sense）、土地覆盖与作物制图（美国多时相作物分割、欧洲 Sen4Map 土地覆盖+作物分类、法国 PASTIS 作物分割）、生态系统动态（BioMassters 地上生物量、37 个 FLUXNET/AmeriFlux/ICOS 站点 GPP 估算）。全部集成进 TerraTorch 工具链，数据 loader 开源，全部微调实验单 GPU 可跑。

### 关键技术细节

- **架构消融证据**：Prithvi-EO-2.0-100M vs Prithvi-EO-1.0-100M（同架构同参数量，仅预训练数据集不同）在 GEO-Bench 上高 3%——纯粹数据规模/质量收益的干净对照。
- **TL（temporal+location embedding）消融**：带 TL 的版本在 GEO-Bench 整体上持续优于不带 TL 版本，即使不是所有 GEO-Bench 数据集都含时间/位置元数据——说明 metadata 条件化本身起到正则化/组织 embedding 空间的作用，而非单纯"多喂信息"。
- **PASTIS 长序列任务**（20 帧输入）：Prithvi 预训练时只见过 T=4，但在 20 帧微调任务上仍稳健泛化，mIoU 超过 U-TAE baseline 与 DOFA/Satlas/Presto 等其他 GFM——预训练时的短序列不构成对长序列下游任务的硬性上限。
- **小样本鲁棒性**（Landslide4Sense 全量 vs. ~1%/50 张）：U-Net/U-Net++ 在极小样本下性能骤降（mIoU 70.4→59.7，F1 59.7→35.0），而 Prithvi-EO-2.0 降幅显著更小，600M 版本在小样本场景下反超所有模型——大规模预训练模型的样本效率优势在数据稀缺场景下更明显。
- **GPP 估算任务的下游架构**：Prithvi encoder（**冻结**）生成 HLS embedding，与一个独立的 MERRA-2（10 维大气/地表变量）卷积分支拼接后回归 GPP；对比 baseline 是 RF/XGBoost（用 HLS 空间均值 + 植被指数 EVI/NDVI/NDWI/GCI/NIRv/kernel-NDVI 手工特征）与 ResNet18（同输入无手工特征）；Prithvi-EO-2.0-600M-TL 在全部 4 个留一年测试上 R² 最高（0.74-0.88），比不含植被指数的 RF/XGBoost 提升可达 20%，且**不使用任何手工植被指数**——即端到端预训练表征替代了手工特征工程。
- **BioMassters 多模态消融**（唯一出现负结果的任务）：额外加入 Sentinel-1 SAR 波段**未能提升**性能（RMSE 35.47→38.67，4 帧配置），作者归因于 SAR 与光学模态物理机制差异过大，Prithvi 难以直接迁移利用；specialized multimodal baseline（U-Net + temporal attention，同时用 12 帧 S1+S2 全部 15 波段）以 RMSE 27.49 明显优于 Prithvi 最佳配置的 33.40（仅用 S2 12 帧 + LoRA）。

### 核心结果

| 任务 | 最佳 Prithvi-EO-2.0 配置 | vs. 前代/baseline |
|---|---|---|
| GEO-Bench（12 数据集综合） | 600M-TL | 全面超越 Prithvi-EO-1.0 及 6 个外部 GFM，平均提升 8% |
| Sen1Floods11 洪水制图 | 600M-TL | water IoU 83.1 vs. 前代 79.6（+3.5） |
| 野火疤痕制图 | 600M / 600M-TL | burn IoU 83.2 vs. 前代 76.8（+5.6） |
| 滑坡检测（全量） | 300M+Lovasz loss | mIoU 71.3，优于 U-Net/U-Net++ |
| 滑坡检测（~1%小样本） | 600M | mIoU 67.0 vs. U-Net 59.7，小样本鲁棒性明显更优 |
| 美国作物分割 | 600M | mIoU 50.7 vs. U-Net 42.6、前代 42.7 |
| 欧洲 Sen4Map 土地覆盖 | 600M | 全数据比例下 F1 全面超越 ViViT baseline |
| PASTIS 法国作物分割（长序列） | 600M | mIoU 53.4（100%数据），超过 U-TAE baseline 52.9 |
| BioMassters 生物量 | 300M+12帧S2+LoRA | RMSE 33.40，仍逊于专用多模态 baseline 27.49 |
| GPP 通量估算（37 站点） | 600M-TL | R² 0.74-0.88，超过 RF/XGBoost/ResNet18 |

### 明确局限

- **烧毁强度分级任务**：所有模型（含 Prithvi）mIoU 整体偏低（~31%），区分中间严重等级（1-3 级）能力差，作者归因于云烟噪声与标签空间不连续（小块高严重区嵌在大片低严重区中），大 patch size 模型（Prithvi/ViT 系）在此类细粒度边界任务上天然弱于像素级 U-Net。
- **BioMassters 生物量任务**：光学-only 的 Prithvi 天花板明显低于真正多模态（SAR+光学）baseline——因为光学在高生物量区饱和（冠层遮蔽），而 SAR 微波能穿透冠层感知垂直结构，这是光学基础模型的**结构性**局限，非训练规模问题。
- **无原生多模态支持**：直接把 SAR 波段拼进 Prithvi encoder 无效，需要专门的跨模态融合设计（如独立 SAR encoder + 后期融合），作者在 Conclusion 中明确列为 future work。
- **无真正长时间序列建模验证**：预训练固定 T=4，虽然下游 PASTIS/Sen4Map 上验证了对 12-20 帧的泛化，但没有验证更极端长序列（如逐日/逐时尺度）行为。
- **纯 2D 空间 + 单一高度层**：数据结构是 (T, H, W, C)，**完全没有垂直/气压层维度**——不是大气再分析场（无 z 轴），也不处理大气动力学变量；这与本课题需要的 (level × lon × lat × time) 4D 场结构存在根本的数据结构差异（见"跟我们的关系"）。
- **无因果方法论**：全文没有出现任何因果推断/时序因果分析方法，纯粹是表征学习 + 下游微调范式。

**该引吗**：可以作为"黑箱基础模型范式"的背景引用（例如在 Introduction/Related Work 中论证"为什么不能默认端到端模型学到了全部信息"时），但不构成直接的方法竞争对手，不需要作为核心 baseline 复现。

**结论：不撞车，边缘相关，可作背景引用。**

本文是 IBM/NASA 团队发布的地球观测多时相基础模型工程报告，处理的是纯光学地表反射率（HLS：Landsat+Sentinel-2 融合，30m，2D 空间+稀疏月尺度时间，无气压层/垂直维度），完全不涉及大气再分析场、严重对流风暴、或任何因果推断方法。与本课题在方法论、数据结构、应用领域三个维度均无重叠，因此不威胁"因果方法分析 SCS 4D 时空场为该领域未填补的具体空白"这一新颖性论断。

真正有价值的部分是它给了 memory 中"黑箱端到端模型未必学到原始场全部有用信息"这一前提论证提供了一个具体、可引用的相邻领域正例：GPP 通量估算任务中，端到端预训练表征模型（无需任何手工植被指数）显著超越基于手工统计特征（EVI/NDVI/NDWI 等）的 RF/XGBoost baseline，R² 提升可达 20%。这反过来支持本课题下一阶段（P1 ML fusion test）必须选用"真正训练过的 ML/DL 模型"作为 baseline、而非简单统计特征这一已有设计决策——本文可以作为该设计决策的背景引用之一。
