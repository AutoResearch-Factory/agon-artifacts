# ASyMOB: Algebraic Symbolic Mathematical Operations Benchmark (2505.23851)

- **Year**: 2025.05
- **Authors**: Michael Shalyt, Rotem Elimelech, Ido Kaminer; Technion - Israel Institute of Technology
- **Venue**: arXiv
- **Keywords**: symbolic mathematics, LLM reasoning, perturbation benchmark, CAS validation, tool use, math robustness, dataset generation, AI for science
- **One-liner**: 用系统扰动检验 LLM 是否真正会符号数学。

## 精华提炼

### 管线/方法

- 论文提出 ASyMOB, 一个专门评测大学级符号数学操作能力的 benchmark。
- 目标不是评测 word problem, formal proof, multiple choice, 而是隔离纯符号操作能力。
- 覆盖任务包括积分、极限、微分方程、级数、超几何函数。
- 数据集从 100 个 seed problems 出发生成 35,368 个 validated symbolic math challenges。
- 100 个 seed 中, 55 个来自 university-level benchmarks、math olympiads 等来源, 45 个由作者补充创建。
- 题目类别分布为 Integrals 30, Differential Equations 23, Series 22, Limits 15, Hypergeometric Functions 10。
- 每个题目只包含数学表达式、指令和必要 assumptions, 避免语言理解混入评测。
- 核心生成思想是保留原题的数学求解技巧, 只改变符号结构或表面复杂度。
- 第一类扰动是 Symbolic-N: 给原题加入 2 到 5 个符号参数, 再枚举移除部分符号得到不同 N。
- 第二类扰动是 Numeric-All-N: 把所有符号参数替换成固定 digit length 的随机正整数, N 从 0 到 10。
- Numeric-All-0 是一个关键 sanity check, 所有符号变成 1, 理论上应回到原题结构。
- 第三类扰动是 Numeric-One-N: 只把一个符号参数替换成大数字, 其余符号移除, 用来隔离单个大系数的影响。
- 第四类扰动是 Numeric-All-N-S: 对 N=2,3 生成多组随机数字, 用来测同一 seed 下模型输出方差。
- 第五类扰动是 Equivalence-One/All-Easy/Hard: 用恒等于 1 的表达式替换一个或全部符号。
- equivalence identities 覆盖 trigonometric、hyperbolic、logarithmic、series、complex exponential 五类。
- Hard equivalence 故意制造 expression swell, 用来测试模型是否能先化简再求解。
- 论文的评测 pipeline 给模型一个题目, 要求输出 final answer 的 LaTeX 表达式。
- 输出答案先用 regex 提取最终 LaTeX, 再转成 SymPy 表达式。
- LaTeX 到 SymPy 的主路径是 `sympy.parsing.latex.parse_latex`。
- 如果 parse 失败, 只用 Gemini 2.0 Flash 做答案表达式到 SymPy 的翻译, 不用它判数学对错。
- 正误验证分两步: symbolic validation 和 numeric validation。
- symbolic validation 用 `SymPy.simplify` 检查模型答案与参考答案之差是否为 0, 不定积分允许常数差。
- numeric validation 随机采样变量和符号参数, 重复 5 次, 要求相对误差小于 0.002%。
- 对不定积分, numeric validation 检查 5 次差值是否为同一个常数。
- 论文强调不用 LLM-as-judge, 这是 benchmark 可信度的重要设计。
- 实验按 pass@1 评估, 不做 majority voting, 以贴近真实使用中一次回答的可靠性需求。
- 评测模型包括 o4-mini、GPT-4.1、GPT-4o、GPT-4o-mini、Gemini 2.5 Flash、Gemini 2.0 Flash。
- 开源/开放模型包括 DeepSeek-R1、DeepSeek-V3、DeepSeek-Prover-V2-671B、Llama-4-Scout-17B、Qwen2.5-72B、Gemma-3n、Nemotron-Super。
- 对有代码执行能力的模型, 同时比较 code-enabled 和 no-code setups。
- 作者还记录了大规模评测成本: 例如 o4-mini code 约 1849 美元, Gemini-2.5 Flash no-code 约 687 美元, GPT-4o code 约 1595 美元。

### 关键技术细节

- ASyMOB 的总题量为 35,368, 明显大于只靠人工专家题构造的小规模 symbolic benchmark。
- 扰动生成的原则是“不显著改变核心数学挑战”, 例如把 `int x^2 e^x dx` 改成 `int x^2 e^{Fx} dx` 是允许的。
- 但把 `x^2` 改成 `x^{2B}` 会引入 Gamma 函数等新技巧, 论文明确认为这不再是同一求解技能的扰动。
- Symbolic-N 的数量随 seed 可插入符号数变化, 因此不同 N 的平均 seed 难度并不完全一致。
- Numeric-All-N 允许测试 OOD 数字长度, 因为 10 位大系数在训练分布中较少。
- Numeric-All-N-S 每个 seed 取 100 个随机变体子集, 用于估计模型稳定性和方差。
- Equivalence perturbation 的 Easy/Hard 分类根据嵌套运算数和代数化简步骤定义。
- 论文列出 10 个恒等式, 其中 Hard trigonometric 和 Hard hyperbolic 等式对 SymPy 也有挑战。
- validation pipeline 的答案抽取 regex 第一优先级匹配 "The final answer is:" 后的 display math。
- 其他 fallback 包括最后一个 boxed expression、最后一个 display expression、`output='...'` 和 `output="..."`。
- 作者报告 final answer 抽取约 98% 成功。
- LaTeX 到 SymPy 表达式整体约 96.1% 成功。
- 在有效 SymPy 表达式中, 97.6% 能被 symbolic 或 numeric validation 成功验证。
- 无法解析或无法验证的输出从 aggregate statistics 中排除。
- 论文对 seed-cluster dependence 做了额外分析, 承认 35,368 个变体并非完全 IID。
- 每个 seed cluster 变体数量从 195 到 483, 平均 353.7, 中位数 299。
- 他们用 leave-one-out question difficulty 和 model/setup intercept 控制题目难度和模型强弱。
- 控制后 residual seed-clustered interval 平均约 2.4%, 对应约 1440 个 IID-equivalent homogeneous Bernoulli questions。
- 这说明 benchmark 不是 35k 个完全独立样本, 但也远多于 100 个 seed 的信息量。

### 核心结果

- 平均模型成功率从 unperturbed seed subset 的 74.6% 降到 full ASyMOB 的 46.8%。
- 这个下降是全论文最核心的结果: 小扰动足以大幅削弱 LLM 的符号数学表现。
- Table 1 中全数据集最高的是 Gemini-2.5 Flash no-code 78.5%, DeepSeek-R1 78.3%, o4-mini no-code 78.1%。
- o4-mini code 为 76.1%, 比 no-code 略低, 说明 frontier reasoning model 不一定从代码工具受益。
- GPT-4o code 为 35.3%, no-code 只有 16.8%, 说明较弱或较旧模型会明显受益于 code execution。
- Gemini-2.0 Flash code 58.1%, no-code 54.9%, 工具增益存在但不大。
- SymPy baseline 总体为 39.2%, seed 为 100%, symbolic 56.7%, numeric 65.2%, equivalence 21.9%, variance 57.8%。
- SymPy seed 100% 不是自然优势, 因为 seed selection 已用 CAS 验证, 对 SymPy 有选择偏置。
- Equivalence perturbations 对多数模型杀伤最大, GPT-4o code 的 equivalence 只有 15.1%, GPT-4o-mini 只有 3.8%。
- Gemini-2.5 Flash no-code 在 equivalence 上达到 84.7%, 是非常强的 robustness 信号。
- DeepSeek-R1 symbolic 78.8%, total 78.3%, 是 open-weight/开放模型中的最强之一。
- DeepSeek-Prover-V2-671B 在 ASyMOB 上不如 DeepSeek-R1, 暗示 formal proof 能力不直接等价于符号操作能力。
- Limits 是平均最容易的主题, across setups 平均 59.5%。
- Integrals 是平均最难的主题, across setups 平均 42.6%。
- Hypergeometrics 对模型家族差异明显, GPT-4o code 在 Hypergeometrics 为 49.6%, 高于其 total 35.3%。
- Llama-4-Scout 在 Hypergeometrics 为 23.1%, 低于其 total 34.3%。
- Numeric-All-0 只增加少量字符, 有些表达式长度只增加 3-4%, 但某些模型仍出现 50-100% relative degradation。
- 同字符长度下, Symbolic 与 Numeric-1 的表现差异说明失败不只是 token length 问题。
- 强模型在 seed performance 和 perturbation resilience 之间呈现非线性提升, 作者称为 robustness regime shift。
- seed contamination stress test 中, 把原 seed 和答案作为 in-context exemplar 给 Gemini 2.0 Flash、GPT-4o、Nemotron-Super。
- 该设置对 Numeric-All-0 有明显帮助, 但对 Numeric-One-3、Numeric-All-3、Symbolic-3 的平均增益迅速下降。
- 作者因此认为即使 seed 被训练见过, 复杂扰动仍能暴露超越记忆的能力缺陷。
- 论文给出一个 hybrid LLM+CAS 例子: pure GPT-4o 和 pure CAS 都失败, 但 GPT-4o 先提出 substitution 再让 Mathematica 求简化式时成功。
- 这支持一个重要结论: 有效工具使用不是简单调用 CAS, 而是先做数学重构再调用 CAS。

### 明确局限

- 这是 benchmark/data paper, 不是新的求解模型或训练算法。
- 所有题目仍由 100 个 seed 扩展而来, seed-level dependence 不能完全消除。
- 作者做了 cluster residual 分析, 但这仍是内部统计处理, 不是外部复现实验。
- 题目集中在大学级符号操作, 不覆盖 theorem proving、几何证明、word-problem parsing、真实工程建模。
- Equivalence perturbations 有意构造得不自然, 更像 stress test 而非真实考试题分布。
- 如果模型直接训练 ASyMOB 的 perturbation generator, benchmark 会受到 methodological contamination。
- 作者把这种 contamination 解释为“训练出更强数学泛化也是好事”, 但作为固定 benchmark 仍需重新生成题目。
- validation 依赖 SymPy 和数值抽样, 对 CAS 自身无法处理的等价形式仍可能排除一些有效答案。
- LLM 被用于 LaTeX-to-SymPy fallback, 虽不判分, 但仍引入了非完全确定的解析环节。
- 论文没有外部第三方审计 dataset labels 和 validation scripts。
- 反直觉结果很多, 例如 frontier models no-code 强于 code, 但论文主要给现象解释, 没有机制证明。
- CAS failure 例子很有启发, 但数量有限, 不足以系统刻画 CAS 与 LLM 的互补边界。
- 对 `0616-fex` 而言, 本文不直接解决 PDE symbolic solver 或 RL controller 的搜索复杂度问题。
- 它也没有测试 FEX、SSDE、SymPlex、PySR、DSR 等 symbolic regression / symbolic PDE solver。
- 因此它不能作为 FEX 性能比较 benchmark, 但可以作为“符号表达式扰动鲁棒性”的诊断范式。

### 跟我们的关系

- 对 `0616-fex` 的直接撞车风险低: ASyMOB 做 LLM 符号数学 benchmark, 不是 FEX/PDE 求解器。
- 如果我们的 claim 是“用扰动生成防记忆 symbolic benchmark”, 本文已经覆盖很大一部分, 需要明确差异化。
- 如果我们的 claim 是“FEX controller 的搜索、先验、candidate pool 或 active collocation”, 本文是相关评测思想, 不是直接竞品。
- 最可用的是 perturbation protocol: symbolic parameters、numeric extremes、equivalence-preserving identities、variance subsets。
- 这些可转化为 FEX stress tests: 对同一个 PDE 真解表达式系统化改写, 看 FEX 是否仍能搜索到等价结构或低 residual 解。
- Numeric-All-N 可用于测试 FEX 参数优化对大系数、尺度变化、conditioning 的敏感性。
- Equivalence perturbation 可用于测试 FEX expression search 是否被等价表达式膨胀拖垮。
- ASyMOB 的 CAS failure 章节提醒我们, FEX 不能只依赖 SymPy/Mathematica 化简作为唯一 correctness gate。
- 更好的 FEX validation 可以采用 symbolic check + randomized numeric check + residual/boundary check 的多重验证。
- LLM+CAS hybrid 例子对 LLM+FEX 很有启发: LLM 最有价值的角色可能是提出 substitution、normal form、operator-set prior, 而不是直接输出完整解。
- 对 FEX+LLM skeleton idea, 本文支持“soft, recoverable prior”而非 hard template, 因为错误符号结构会被小扰动放大。
- 对 FEX active collocation idea, ASyMOB 的 high-resolution perturbation family 提醒我们应报告 seed-level cluster effects, 不要把所有变体当 IID。
- 该引: 如果 0616-fex 写 benchmark、robustness、LLM+CAS/FEX、符号数学评测、anti-memorization, 应引用。
- 不必作为核心竞品引用: 如果文章只讨论 PDE residual minimization、FEX convergence theory 或 high-dimensional PDE solver performance。
- 差异化空间: 把 ASyMOB 的扰动思想移植到 PDE solution-expression search, 加入真实 FEX/SSDE/SymPlex/PySR 对照, 并报告搜索复杂度、multi-seed 和等价类剪枝收益。

## Read by: 0616-fex (2026-06-17)

本次从 `0616-fex` 视角看, ASyMOB 不直接撞 FEX, 因为它评测 LLM 的符号数学鲁棒性, 不做 PDE 闭式解搜索, 也不比较 FEX controller、SSDE、SymPlex 或 PySR。它真正有价值的是评测范式: 用 symbolic、numeric、equivalence-preserving perturbations 把“是否记住 seed”与“是否能处理等价符号结构”拆开。FEX 后续如果要证明搜索不是靠特定表达式模板, 可以仿照 ASyMOB 为 PDE 真解、边界条件、候选算子池生成变体, 并报告 seed-cluster uncertainty、multi-seed search variance 和等价化简失败率。

可用工具包括: Numeric-All-N 测 FEX 参数优化 conditioning, Equivalence-All-Hard 测表达式膨胀和 CAS gate 脆弱性, Numeric-All-N-S 测同一 seed 的 search stability, 以及 symbolic+numeric 双验证替代单一 CAS 化简。该引的场景是 robustness/benchmark/LLM+FEX/tool-use; 若我们的工作只写 FEX 搜索复杂度或高维 PDE 性能, 它应作为背景而非核心竞品。差异化应放在 PDE residual setting、真实 symbolic regression solvers、搜索成本、controller prior 和 candidate-pool robustness, 而不是重复提出“数学题扰动 benchmark”。
