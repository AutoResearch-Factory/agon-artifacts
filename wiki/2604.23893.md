# Algebraic structure behind Odrzywołek's EML operator (2604.23893)

- **Year**: 2026.04
- **Authors**: Tomasz Stachowiak (tex 中未标注机构；联系域名 monodromy.group)
- **Venue**: arXiv
- **Keywords**: EML operator, single binary operator, abelian group, anti-associativity, functional inverse, elementary functions, symbolic trees, elliptic functions
- **One-liner**: 把 EML 生成性还原为群结构与逆函数链。

## 精华提炼

### 管线/方法
- 论文从 Odrzywołek 的 EML 结果出发：一个二元算子 `EML(x,y)=exp(x)-ln(y)` 加常数 1，可以递归生成所有初等函数。
- 作者的目标不是再做穷举搜索，而是解释 EML 为什么有效，以及怎样构造同类单算子族。
- 输入对象是一个待泛化的二元算子，而不是数据集或训练任务。
- 核心重写是 `S(x,y)=M(f(x), f^{-1}(y))`。
- 其中 `f` 是可逆函数，`g=f^{-1}`；`M` 是类似减法或除法的二元抵消操作。
- 作者把 `M` 写成 `x \boxminus y`，把从 `M` 诱导出的群运算写成 `x \boxplus y`。
- `M` 需要满足三条形式公理：存在中性元 `e`，自抵消 `M(x,x)=e`，以及反结合式 `M(x,M(y,z))=M(z,M(y,x))`。
- 这三条公理覆盖了减法的关键行为：能取消、能翻转、能从嵌套表达式中恢复被遮蔽的项。
- 作者证明式地说明：满足这些公理的 `M` 背后总能构造一个阿贝尔群。
- 构造方式是 `A \boxplus B := M(A,M(e,B))`，逆元是 `\iota(A):=M(e,A)`。
- 反向也成立：给定一个阿贝尔群运算 `\boxplus` 和逆元 `\iota`，定义 `M(A,B)=A \boxplus \iota(B)` 即满足这些公理。
- 这把 EML 中的 `-` 从“偶然选择的减法”提升为“阿贝尔群差”的一个实例。
- 许多可用的 `M` 可由普通减法共轭得到：`M(u,w)=phi^{-1}(phi(u)-phi(w))`。
- 例子包括除法，对应 `phi(u)=ln(u)`；也包括 `sqrt(x^2-y^2)`，对应 `phi(u)=u^2`。
- 作者明确指出这些共轭会带来定义域、分支切割和复数域上的人工修正问题。
- 在有了 `S`、`M`、`f`、`g` 和常数 `c` 后，常数必须满足 `g(c)=e`。
- 第一阶段生成 `f(x)`：`S(x,c)=f(x)\boxminus g(c)=f(x)`。
- 第二阶段生成 `f(x)\boxminus y`：把第二个输入替换为 `f(y)`，得到 `S(x,f(y))=f(x)\boxminus y`。
- 第三阶段恢复逆函数 `g(x)`：用抵消结构把 `f(z)\boxminus(f(z)\boxminus g(x))` 化为 `g(x)`。
- 第四阶段恢复底层差操作 `x\boxminus y`：计算 `S(g(x),f(y))`。
- 第五阶段恢复群逆元 `\iota(x)`：用嵌套的 `\boxminus` 表达式从 `x\boxminus y` 中消去 `x`。
- 第六阶段恢复群加法 `x\boxplus y`：利用 `x\boxminus \iota(y)=x\boxplus y`。
- 这六步给出一个通用生成链：先从单算子拆出 `f` 和 `g`，再拆出群差、群逆和群加法。
- 之后只要 `f` 对群加法有加法公式 `f(x\boxplus y)=F(f(x),f(y))`，就能继续生成新的运算 `F`。
- 原始 EML 的关键特例是指数函数：`exp(x+y)=exp(x)exp(y)`。
- 因此，恢复 `ln` 和加法后，可以通过 `exp(ln(x)+ln(y))` 得到乘法。
- 幂函数也由 `exp(y ln(x))` 跟随得到。
- 作者强调：`ln(x)` 在 EML 树中的特定长度不是对数的本质，而是通用逆函数恢复链的结果。
- 文中给出的通用逆函数恢复在 Polish notation 下长度为 7。
- 作者还讨论了一个更短但有奇点的负号恢复路径：`M(e,x)=S(g(e),f(x))`。
- 对原始 EML，这会碰到 `g^2(c)=ln(0)`，需要扩展算术中的无穷值；作者选择绕开该奇点。
- 常数 `c` 的地位被指出是难以消除的，因为通用形式需要一个满足 `g(c)=e` 的起点。
- 若强行用 `S(x,x)` 得到中性元，会要求 `f=g`，也就是 `f^2=Id`，条件很强。
- 作者还给出三元插值算子 `B(x,y,z)`，可以在两个二元算子之间选择，从而较容易嵌入常数或组合算子。
- 但这种三元 IF-THEN-ELSE 风格的构造牺牲了 EML 单一二元算子的简洁性。

### 关键技术细节
- 论文没有训练模型、数据集或实验平台；它是一篇 225 行 tex 的数学结构说明。
- 主定义是 `S(x,y)=M(f(x),f^{-1}(y))`，其中 `M` 服从中性元、自抵消、反结合三条公理。
- 对应群构造为 `A \boxplus B := M(A,M(e,B))`，逆元为 `\iota(A)=M(e,A)`。
- 原始 EML 对应 `f=exp`，`g=ln`，`M(x,y)=x-y`，`e=0`，`c=1`。
- EDL 变体把 `M` 换成除法结构，`e=1`，仍依赖指数函数生成初等函数。
- `cos/arccos` 例子设 `S(x,y)=cos(x)-arccos(y)`，`c=1`。
- 该例子可恢复 `cos`、`arccos`、加减法、`arccos(0)=pi/2`，进而恢复 `sin(x)`。
- `cos` 例子的乘法瓶颈来自 `cos(a+b)+cos(a-b)=2cos(a)cos(b)`，只能得到 `F(x,y)=2xy`。
- 用 `f(x)=2cos(x)` 可修正因子 2，因为 `f(x+y)+f(x-y)=f(x)f(y)`。
- 但即便修正缩放，没有根式和有理函数时仍不能完整恢复除法。
- `arccot/cot` 例子设 `S(x,y)=arccot(x)-cot(y)`，`c=pi/2`。
- 该例子可恢复 `cot`、`tan`、正负号和倒数 `1/x=cot(arctan(x))`。
- `tan` 加法公式给出 `F(x,y)=(x+y)/(1-xy)`，但没有除法时仍不能抽取乘法。
- `tanh/artanh` 例子可生成 Lorentz boost 风格的速度加法语言。
- Weierstrass 例子设 `S(z,t)=wp(z)-wp^{-1}(t)`，`c=infty`，但必须限制到合适子域以回避双周期不可逆问题。
- 椭圆函数的加法公式需要 `wp` 和导数 `wp'` 两个坐标，单函数不够。
- 作者因此引入第二个二元算子 `R(z,t)=wp'(z)-(wp')^{-1}(t)`，用两个算子恢复椭圆曲线上的阿贝尔群加法。
- 最后一个无外部常数例子使用分段自反函数：非负区间为 `exp(-x)-1`，`(-1,0)` 区间为 `-ln(x+1)`。
- 该自反 `f` 满足 `f^2=Id`，所以可让 `S(x,x)=e` 直接产出中性元。
- 但这个构造只在 `-1<t,s<0` 的受限区间内给出乘法 `ts=f(f(s)+f(t))-t-s`。
- 由于只能恢复 `e=c=0`，无法自然产生 1，因此除法仍被卡住。

### 核心结果
- 最核心结果是：EML 的生成性不是孤立奇迹，而可分解为“阿贝尔群差 + 可逆函数 + 加法公式”三部分。
- 论文把 Odrzywołek 的穷举式发现转化为一个可手工构造的族。
- 对任意满足公理的 `M` 和合适的 `f/g/c`，作者给出恢复 `f`、`g`、`\boxminus`、`\iota`、`\boxplus` 的明确表达式链。
- 该链解释了为什么 EML 能先得到 `exp` 和 `ln`，再得到加减乘除和幂。
- 反直觉点：`ln` 的 EML 树长度不是因为 `ln` 复杂，而是任何 `g=f^{-1}` 都可用同样的第三步恢复。
- 另一个反直觉点：把 `f` 换成 `cos` 后，三角函数很容易得到，但普通乘法和除法反而变困难。
- `2cos` 缩放能修复一个乘法常数因子，却不能自动补足除法。
- `arccot/cot` 能直接给倒数，但加法公式把系统带入 Möbius 型组合律，仍无法分离普通乘法。
- 椭圆函数例子说明：单函数加法公式不是所有阿贝尔结构都具备；椭圆曲线需要两个坐标。
- 常数自由生成仍然没有被解决；作者最后明确把 constant-free generator 留作开放问题。

### 明确局限
- 论文的许多推导是形式性的，开头就说明函数复合的集合与定义域被“略微滥用”。
- 对复数域、周期函数和分支切割的处理不是完整理论，而是点出问题。
- 原始 EML 中 `ln(exp(z))=z` 在复数域只形式成立；周期性会破坏严格逆消去。
- 没有给出可执行的 EML 树搜索程序，也没有复现实验。
- 没有标准 benchmark、没有真机或数值实验，科学贡献主要是结构解释。
- 对“生成所有初等函数”的完整性依赖原始 Odrzywołek 论文，本文自身不重新完整证明科学计算器基底。
- 多个例子只说明能生成“有趣语言”，但没有量化表达能力、树深、搜索复杂度或优化难度。
- 常数 `c` 的选择仍是强假设；没有解决如何从纯算子无常数地产生足够常数的问题。
- `cos`、`cot`、`wp`、自反分段函数例子都暴露了同一限制：可恢复群结构不等于能恢复完整初等函数库。
- 对 0616-fex 若目标是可训练表达式发现，本文没有给训练算法，只能提供算子设计和理论解释。
