# High-recall causal discovery for autocorrelated time series with latent confounders (2007.01884)

- **Year**: 2020.07 (NeurIPS 2020)
- **Authors**: Andreas Gerhardus, Jakob Runge — German Aerospace Center (DLR), Institute of Data Science, Jena, Germany
- **Venue**: NeurIPS 2020 (spotlight/poster); arXiv v3 2021.02
- **Keywords**: constraint-based causal discovery, latent confounders, time series, FCI, PAG/MAG, autocorrelation, conditional independence effect size, PCMCI, order-independence
- **One-liner**: 提出 LPCMCI —— 把 PCMCI 的"条件于父节点以提高 CI 检验效应量"思想扩展到含潜在混杂的自相关时序 FCI 框架, 通过纠缠 edge 删除与定向阶段大幅提高 recall, 且在 oracle 下 sound+complete+order-independent.

## 精华提炼

### 管线/方法
LPCMCI (Latent PCMCI) 是一个 constraint-based 因果发现算法, 目标是在**允许潜在混杂 (latent confounders)、无 selection bias、平稳、无环**的离散时间 SVAR 过程上, 估计带滞后 (lagged) 与同期 (contemporaneous) 因果关系的 PAG。整体管线:

1. **初始化**: 把 $\mathcal{C}$ 初始化为完全图, 滞后边初始化为 $\LMM{\ohead}$, 同期边初始化为 $\QMM{\oo}$ (带 middle mark)。
2. **Preliminary phase (第 2–4 行, 循环 $k$ 次)**: 每次调用 **Algorithm 2 (Ancestral removal phase)** 删除部分错误边并反复施加定向规则, 识别一批 (非)祖先关系 (heads/tails); 把学到的 parentships 记住, 在下一轮 re-init 的图上作为默认条件 $\SSet_{def}$ 从头使用。$k$ 是超参数, 记作 LPCMCI($k$)。
3. **Final phase (第 5–6 行)**: 再跑一次 Algorithm 2, 然后调用 **Algorithm 3 (Non-ancestral removal phase)** 删除剩余错误边 (两端互不为祖先的对), 角色类似 (SVAR-)FCI 的第二次删除阶段。
4. **输出**: PAG $\mathcal{C}=\mathcal{P}(\mathcal{G})_{statAO}^{\tau_{max}}$。

核心机制是**纠缠 (entangle) edge 删除与 edge 定向**: 传统 FCI 必须先得到完整骨架才能定向, 而 LPCMCI 用新定向规则在只做了少量 CI 检验时就确定 (非)祖先关系, 从而能提前把已知 parents 放进条件集来提高效应量。

### 关键技术细节
- **诊断的失败根因**: 现有 latent 方法 (FCI / SVAR-FCI / SVAR-RFCI / tsFCI) 在自相关时序上 recall 低。根因是 **CI 检验的 effect size 太小** —— 强自相关抬高方差、压低信噪比, 使真链的偏相关不显著被误删 (false negative), 进而级联出 false positive 和错误定向。作者把 effect size 定义为 $\min_{\SSet} I(A;B|\SSet)$ (对所有被测条件集取最小的 CI 检验统计量)。
- **两条设计原则** (Sec. MCI): (a) 只在 $A,B$ 的祖先构成的条件集里搜 (Lemma: 限制到祖先不破坏 m-separation); (b) 用已知 parents 作 **default conditions** $\SSet_{def}=\pa(\{A,B\})$ 扩充条件集 —— 既提高效应量又不引入伪依赖。
- **Effect-size 定理** (信息论框架): 在两个假设 (存在落在 $\adj(A)$ 或 $\adj(B)$ 的最小化集; 存在 $\mathcal{Q}\subset\SSet_{def}$ 使 interaction information $\mathcal{I}(A;B;\SSet_{def}\setminus\mathcal{Q}|\SSet^*\cup\mathcal{Q})<0$) 下, 条件于 parents 严格增大 effect size。这把 PCMCI/PCMCI+ 的 MCI 思想推广到潜在混杂情形。
- **Middle marks (中间标记)**: 在 PAG 边上加 `?/L/R/!/空` 五种标记, 定义 **LPCMCI-PAG**, 使图在算法**每一步都有无歧义因果解释** (即使中途打断也 sound)。`!`=两向都成立, 空=确定相邻, `?`=无承诺。配套 **APR** (ancestor-parent-rule) 和 **MMR** (middle-mark-rule) 更新规则。
- **新定向规则**: Strong unshielded triple rule (Lemma) 把 FCI 的 $\R0/\R1$ 推广为 $\ER0a$–$\ER0d$、$\ER1$; $\R2,\R3,\R8,\R9,\R{10}$ 推广为 $\ER\cdot$ (加"中间变量在弱最小分离集中"的要求); $\ER4$ 基于 RFCI 判别路径规则。规则**可在任意时刻、任意顺序施加**。
- **Weakly minimal separating sets (弱最小分离集)**: 因为 LPCMCI 会用祖先扩充条件集, 找到的分离集不再最小, 但仍"弱最小" ($\SSet=\SSet_1\dot\cup\SSet_2$, $\SSet_1\subseteq$祖先)。定向规则的正确性依赖弱最小性。
- **apds / napds 集**: 比 FCI 的 PDSEP、SVAR-FCI 的 $pds_t$ 更紧的 D-Sep 近似, 用已识别的 (非)祖先关系 + 时间序 + collider 结构收缩搜索空间。
- **Order-independence**: 删除只在遍历完全部有序对后统一执行; 用 $I^{min}$ 排序 $\SSet_{search}$; 冲突用 `x` 标记消解; 用**改良多数规则** (在 apds 里搜分离集 + 把 Alg2/3 找到的分离集也纳入投票) 保证 completeness (未改良的 FCI 多数规则非完备, 见 SM 反例)。
- **平稳性**: 每次删/定向一条边, 所有等价时移边 (homologous) 同步处理。
- **CI 检验**: 线性用 ParCorr, 非线性用 GPDC (非参), 离散用 $G$-test。第二删除阶段限制 $|\SSet|\le 3$ (不含 default conditions) 控制运行时。

### 核心结果
- **理论**: LPCMCI 在 oracle (完美 CI 判定 + faithfulness + 无环 + 无 selection) 下 **sound + complete** (返回目标 PAG), 且 **order-independent** (与变量编号无关)。
- **仿真** (SCM: $V^j_t=a_jV^j_{t-1}+\sum_i c_i f_i(V^i_{t-\tau_i})+\eta^j_t$, 500 realizations, $\lambda=0.3$ 未观测): 对自相关连续变量, LPCMCI **recall 远高于 SVAR-FCI/SVAR-RFCI, 且增益随自相关增强**, 对**同期链尤其显著** (k=4 时同期定向 recall 高 >0.4, 滞后定向 recall 高 ~0.1)。$k=0\to1$ 增益最大, $k=4$ 收敛。
- **False positive 控制**: k>0 时 CI 检验校准更好, FP 控制优于基线 (基线在小 $N$、高自相关下 FP 失控)。作者明确**无法证明** FP 控制, 但也不知其他 latent 约束方法有此证明。
- **效应量-recall 关联**: recall 增益主要归因于 effect size 提升 (图中作为 min|ParCorr| 展示), 代价是条件集 cardinality (~5) 和运行时上升, 但因后期 $l$-step 收敛快 + CI 结果缓存, 运行时增幅有限。
- **真实数据** (上多瑙河三条支流日流量, N=3, $\tau_{max}=2$): LPCMCI 对 k=1..4 正确找到同期 $X_t\to Y_t$ (Iller→Danube), 对所有 k 找到 $X_t\leftrightarrow Z_t$ (天气混杂, 合理); 但 k=3,4 错误找到 $Z_t\to Y_t$。

### 明确局限
- **faithfulness 假设** (与 FCI 同): 违背会致错误结论; 未尝试更弱的 adjacency-faithfulness (潜在情形下是 open problem)。
- **无法区分 Markov 等价类全部成员** (约束型通病), 不如 SCM 型 (TS-LiNGAM/TiMINo) 精细, 但后者限制依赖类型。
- **无 FP 控制证明**。
- **非时序 (iid) 与离散变量情形**: LPCMCI(k=0) 与基线持平, 但 **k=4 出现 FP 膨胀 / recall 下降** —— 高 cardinality 条件集在低样本/离散下损失估计自由度、CI 检验失校准。作者假设是多轮 preliminary 累积了错误 ancestorship 当默认条件。
- **假设**: 平稳、无 selection bias、无环。真实数据例中平稳性明显被极端降雨事件破坏 (作者自陈), 且只有 1 个 3 变量小例、部分链错误。
- **$\tau_{max}$ 是分析选择而非可调参数** (潜在情形下增大 $\tau_{max}$ 可能删掉窗口内已有边)。
- 我读出的额外局限: 真实数据评测极弱 (无标准 benchmark、无 ground truth、单个小系统); 计算成本对高 $N$/高 $\tau_{max}$ 仍可爆炸 (SM 中部分实验 24h 不收敛)。

## Read by: 0710-causal-scs (2026-07-10)

**审读视角**: 评估 LPCMCI 对"因果效应不稳定性作为强对流预警指标"课题的撞车/可用性/差异化。

- **撞车评估**: 无直接撞车。LPCMCI 是"把自相关引起的估计不可靠**修掉**"的方法论文; 我们是"把假设违背下的估计**不稳定性当信号用**"。这与 idea review 已识别的四篇前作 (CNM 2412.16235 因果强度确定性机制、NPG 跨方法失效、Causal-Audit 2604.02488 违背风险评分、ST-CND 2606.17553 空间因果网络+恢复率) 都不同 —— 本文既不做预警也不做诊断, 是纯粹的**发现算法**。因此它是我们的**工具/引用来源**, 不是竞争对手。
- **可用工具**:
  1. **LPCMCI 本体** —— 补 PCMCI+ 不处理 latent confounder 的短板, 是 Stage 2 (真实 3km 环境场, 必有天气混杂) 的候选方法; 现成于 tigramite, 与课题已选的 PCMCI+/J-PCMCI+ 同栈, 集成成本低。
  2. **effect size = $\min_\SSet|ParCorr|$** —— 一个 within-method 标量, 可直接作为滑动窗口的 fragility/不稳定性度量, 回应 review 对"跨方法量纲不可比、fragility score 需方法内标准化"的批评; 建议 Stage 1 就把它纳入不稳定性指标集 (与滚动方差、一阶差分、DAG edge 变化并列)。
  3. **interaction-information effect-size 定理** —— 给"自相关↑ → effect size↓ → CI 失效"提供解析机制, 是我们"接近 regime-change → 估计脆弱性↑"理论桥梁的一半; 但注意本文机制是**平稳**假设下推的, 我们要的是**非平稳/临界**下的行为, 尺度/假设匹配仍需补 (与 review 指出的"快变对流 vs 慢变 CSD"错配问题一致)。
- **差异化**: 我们不碰算法内部, 而是量化 LPCMCI/PCMCI+ 在**预事件滑动窗口**上的 effect-size 退化、跨方法分歧、相邻窗口 DAG edge 变化, 检验其是否对 CAPE/shear、经典 EWS 提供**增量**、事件特异的预警信息。本文恰好证明了"这些估计量对自相关/条件集选择高度敏感" —— 这既是它要消除的缺陷, 也正是我们要利用的信号源, 可在引言里作为"敏感性真实存在且有解析根源"的支撑。
