# GenSR: Symbolic Regression Based in Equation Generative Space (2602.20557)

- **Year**: 2026.02
- **Authors**: Qian Li (SJTU), Yuxiao Hu (PolyU), Juncheng Liu (Imperial College London), Yuntian Chen (Eastern Institute of Technology)
- **Venue**: ICLR 2026
- **Keywords**: symbolic regression, generative latent space, CVAE, CMA-ES, Bayesian inference, ELBO, equation discovery
- **One-liner**: GenSR reparameterizes the discrete equation space into a continuous generative latent space via dual-branch CVAE, enabling efficient "map construction -> coarse localization -> fine search" symbolic regression.

## 精华提炼

### 管线/方法

GenSR 遵循"地图构建 -> 粗定位 -> 精搜索"三阶段范式：

**阶段一：地图构建 (预训练)**
- 预训练一个双分支 CVAE (dual-branch Conditional VAE)，将离散符号方程空间重参数化为连续生成式潜空间。
- 后验分支 (posterior branch)：输入数值样本 X + 方程符号 F，学习后验分布 q(z|X,F)=N(mu1, sigma1^2 I)
- 先验分支 (prior branch)：仅输入数值样本 X，学习先验分布 p(z|X)=N(mu2, sigma2^2 I)
- 训练目标：重建损失 (交叉熵) + KL 散度（对齐后验与先验），等价于最大化 ELBO
- KL 退火 (KL annealing) 防止后验崩塌：lambda 从 0 线性增加到 1.0

**阶段二：粗定位 (推理)**
- 推理时只用先验分支：给定输入数值 X，先验分支输出初始高斯分布 N(mu_0, sigma_0^2 I)
- 该分布粗定位到潜空间中可能包含最佳方程的区域

**阶段三：精搜索 (CMA-ES)**
- 用修改版 CMA-ES 迭代收缩初始分布：
  1. 从当前分布采样多个潜向量 z_j
  2. 解码为候选方程 f_j(x)
  3. BFGS 精化常数
  4. 评估适应度 Fitness = R^2 - omega * complexity
  5. 选 top-p 更新分布参数
- CMA-ES 修改：（1）对角协方差矩阵近似（与 VAE 独立高斯假设一致）（2）仅更新方差最大的 top-k 维度 (k=256)

### 关键技术细节

- **模型架构**：双分支 CVAE — 共享 8 层 Transformer encoder/decoder (8 heads, 512 hidden units)，后验网络和先验网络为 MLP，特征融合为线性层
- **潜空间维度**：512（消融实验确定最优），备选测试 64,128,256,512,768
- **数值输入编码**：base-10 浮点分词 -> sign + mantissa(0-9999) + exponent(E-100~E100)，嵌入维度 d_n=64
- **方程编码**：二叉树 prefix order -> token 序列，常数同样用浮点分词，padding 到固定长度 m
- **训练数据**：约 500 万合成方程-样本对，控制方程长度将复杂方程推向潜空间外围
- **训练配置**：Adam, batch size 256, 200 epochs (1000 steps/epoch), lr=1e-3, Noam schedule (w=8000 warm-up), KL annealing 前 50% steps 从 0 到 1.0
- **参数量**：162.1M trainable parameters
- **硬件**：2x NVIDIA H800 (80GB each)，每 epoch ~1 小时
- **CMA-ES 超参数**：population size s（饱和点~50），initial step size t~1.1，fitness weight omega 适中
- **推理流程**：先验分支定位 -> CMA-ES 迭代 -> BFGS 常数精化 -> 输出 final equation

### 核心结果

**Feynman 数据集 (119 个物理方程)**：

| 噪声水平 | R^2 | Time (s) | Complexity |
|---------|------|----------|-----------|
| epsilon=0 | 0.9872 | 192.3 | 22.94 |
| epsilon=0.001 | 0.9883 | 279.3 | 23.5 |
| epsilon=0.01 | 0.9872 | 486.8 | 23.36 |
| epsilon=0.1 | 0.9886 | 618.9 | 23.62 |

**Strogatz 数据集 (14 ODE 系统)**：

| 噪声水平 | R^2 | Time (s) | Complexity |
|---------|------|----------|-----------|
| epsilon=0 | 0.9918 | 33.0 | 20.53 |
| epsilon=0.001 | 0.9951 | 41.6 | 20.45 |
| epsilon=0.01 | 0.9936 | 44.6 | 20.42 |
| epsilon=0.1 | 0.9773 | 257.6 | 20.21 |

**Black-box 数据集 (57 个未知函数任务)**：R^2=0.8416, Time=2725s, Complexity=35.05

对比 18 个基线（GPlearn, Operon, GP-GOMEA, AFP, EPLEX, SBP-GP, FEAT, DSR, SPL, TPSR, SNIP, E2ESR, NeurSR, RSRM, MDLformer, RAG-SR, BSR, AIFeynman2 等）。

**关键发现**：
- 在 Strogatz 上噪声水平 epsilon=0.001 时 R^2=0.9951 最高
- 搜索时间远快于大部分 GP 方法 (Strogatz 上仅 33s vs GP-GOMEA 2760s)
- 方程复杂度稳定在 20-24 token 之间，远低于 FEAT (119-205)、SBP-GP (489-712)
- 消融实验：潜维度 512 最优，CMA-ES 的 k=256 达到饱和性能
- GenSR vs 标准 Bayesian Optimization: CMA-ES 在高维潜空间表现远超 BO

### 明确局限

**作者自述局限**：
1. CMA-ES 在潜维度增大时计算量线性增长（虽然用 top-k 缓解）
2. 未来方向包括更丰富的生成先验和物理约束
3. 更先进的生成模型可以构建更强大的方程空间

**读出来的局限**：
1. **需要大规模预训练**：500 万合成样本 + 162.1M 参数 CVAE + 200 epoch 训练 (~200 GPU-hours) —— 预训练成本极高
2. **仅面向通用 SR**：实验只做了 SRBench (Feynman/Strogatz/Black-box)，完全没做 PDE 求解验证。与 FEX 求解 PDE 的场景无关
3. **无真机验证**：纯 synthetic benchmark，没有任何真实物理实验数据验证
4. **潜空间非物理可解释**：虽然方程可解码，但潜空间坐标 z 没有物理意义，是学习到的数值特征
5. **搜索仍依赖 CMA-ES 超参数**：population size, step size, fitness weight 都需要调参
6. **对比基线不包括 FEX 方法族**——FEX, FEX-PG, H-FEX 等 PDE 求解 SR 方法完全未提及
7. **高噪声退化**：epsilon=0.1 时 Strogatz 上 R^2 降到 0.9773 (仍好但趋势下行)
8. **没有收敛率理论保证**：虽然有 Bayesian ELBO 的理论视角，但没有给出搜索算法（CMA-ES）在什么条件下、以多快速度收敛到全局最优

- **R1 Evidence Grounding (4)**: 实验数据完整，Pareto 图、噪声实验、消融实验都呈现在主文和附录中。但 claim "best balance" 依赖 Pareto front 排名，没有看到严格的统计检验（如配对 t-test）。
- **R2 Verification Independence (3)**: 使用标准 SRBench benchmark 和 18 个公开基线，但所有实验都是作者自己跑的，没有独立第三方复现或验证。
- **R3 Honesty (3)**: 消融实验完整报告了失败模式（如维度 768 性能下降），但未报告 CMA-ES 在哪些方程上搜索失败的比例，也未讨论对特定方程族的系统失效。
- **R4 Reproducibility (3)**: 提供了架构、超参数、训练配置的详细信息，声明将发布代码和 checkpoint。但 5M 合成样本的精确生成过程描述不够详细（只说了用 e2e 和 snip 的生成策略），可能难以精确复现。
- **R5 Substantiation (3)**: Bayesian 视角的 ELBO 推导干净，但推理用 CMA-ES 做 ELBO 的后验近似优化这一连接没有严格证明（只是直观类比）。插值实验展示了生成式空间的连续性，但数值实验的统计效度（样本量、seeds、variance）没有系统报告。

### 跟我们的关系

**与 FEX (0616-fex) 的对比**：

**一致/可借鉴**：
1. 都做 Symbolic Regression，都用二叉树表示方程
2. GenSR 的潜空间连续性分析为 FEX 提供了 "在连续空间中搜索比离散空间有效率" 的理论直觉支撑
3. GenSR 的 BK 退火策略对 FEX 的 controller 训练可能有借鉴意义（防止 RL 策略崩塌）
4. BFGS 精化常数这一步与 FEX 的 "Adam + BFGS 两步优化" 完全一致

**差异/不撞车**：
1. **核心方法不同**：GenSR = CVAE 生成式潜空间 + CMA-ES；FEX = RL policy gradient + Adam/BFGS。搜索范式完全不同。
2. **目标场景不同**：GenSR 面向通用 SR (SRBench)；FEX 面向 PDE 求解。GenSR 未做任何 PDE 实验。
3. **预训练 vs 零搜索**：GenSR 需要大规模预训练 (5M 样本)；FEX 从零开始搜索每个新问题。
4. **理论视角不同**：GenSR 用 Bayesian ELBO 视角；FEX 用 Holder 函数空间的 approximation theory。
5. **结果形式不同**：GenSR 输出 R^2/Fitness 得分；FEX 追求机器精度 (1e-7 MSE on PDE)。
6. **GenSR 不涉及 PDE 求解中的维度灾难问题**（FEX 的核心动机）。

**对我们课题的可用性**：
- GenSR 的 CVAE 潜空间可以作为 FEX 的 warm-start 参考——用预训练生成模型初始化 FEX 的 controller 或搜索空间。这是 landscape 中识别出的一个结构性缺口。
- 如果 FEX 考虑扩展到通用 SR benchmark，GenSR 是一个重要对比基线。
- GenSR 的 "map construction -> coarse localization -> fine search" 三阶段范式可以启发 FEX 设计类似的分层搜索策略。

**引用决策**：应引用。作为对比基线（如果 FEX 扩展到 SRBench）或作为预训练+搜索结合的参考工作。

## Read by: 0616-fex (2026-06-16)

GenSR 是与 FEX 方法族正交但互补的新兴 SR 工作。其核心贡献——用双分支 CVAE 构建兼具符号连续性和数值平滑性的生成式潜空间——为 FEX 提供了一条不同于 RL 策略梯度的技术路径。GenSR 的预训练+搜索范式（特别是 CVAE 潜空间 + CMA-ES 精化）可以直接启发 FEX 设计 hybrid 方法：用预训练生成模型为 FEX controller 提供初始先验，或用于缩小搜索空间。但需注意 GenSR 的预训练成本极高（~200 GPU-hours），且其基准测试（SRBench）与 FEX 的目标（PDE 求解）完全无关。不撞车，建议引用作为竞争基线或预训练+搜索融合的参考。
