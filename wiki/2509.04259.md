# RL's Razor: Why Online Reinforcement Learning Forgets Less (2509.04259)

- **Year**: 2025.09
- **Authors**: Idan Shenfeld, Jyothish Pari, Pulkit Agrawal (Improbable AI Lab, MIT)
- **Venue**: ICLR (投稿使用 iclr2025 模板)
- **Keywords**: catastrophic forgetting, RL fine-tuning, SFT, KL divergence, on-policy, policy gradient, continual learning, foundation models
- **One-liner**: 发现 RL 微调比 SFT 遗忘更少，因为 on-policy RL 隐式地偏向 KL 散度最小的解（RL's Razor），KL 散度是新任务上遗忘程度的可靠预测器。

## 精华提炼

### 管线/方法

**整体实验管线**：
1. 选取预训练基础模型（LLM: Qwen 2.5 3B-Instruct；Robotics: OpenVLA 7B）
2. 在新任务上分别用 SFT 和 RL（GRPO）微调，**RL 不使用显式 KL 正则化**
3. 对每种方法做超参数 sweep（学习率、batch size、epoch 等），生成数十个模型
4. 在新任务测试集上评估性能，在多样化旧任务 benchmark 上评估遗忘
5. 绘制新任务性能 vs 旧任务性能的 Pareto 前沿
6. 对比两种方法的 Pareto 前沿位置

**控制的 ParityMNIST toy setting**：
- 预训练：3 层 MLP（785→512→256→10），在 ParityMNIST + FashionMNIST 上联合预训练
- ParityMNIST 构造：MNIST 数字分类重定义为奇偶预测，任意偶数标签对偶数图片都算正确
- 这保证了"多个不同输出分布可以达到相同性能"——模拟 LLM 生成任务的特性
- 微调只在 ParityMNIST 上，遗忘在 FashionMNIST 上测量

**四象限算法对比**（Section 5）：
- GRPO：on-policy + negative examples
- 1-0 Reinforce：on-policy + 无 negative examples（正确样本 reward=1，错误=0，等价于采样后用 SFT 训练正确答案）
- SFT：offline + 无 negative examples
- SimPO：offline + negative examples（用外部模型采样错误答案做负例）

### 关键技术细节

- **基础模型**: Qwen 2.5 3B-Instruct（主要 LLM 实验）、Qwen 2.5 7B/14B（scaling 实验）、OpenVLA 7B（机器人实验）
- **RL 算法**: GRPO（Group Relative Policy Optimization），group size=64，每 prompt 生成 8 个响应，Dr. GRPO loss type，无 KL 正则化（KL coeff=0）
- **训练超参数**: LR sweep {1e-5, ..., 5e-5}，AdamW optimizer，constant w/ warmup scheduler，warmup=50 steps，max grad norm=1，bfloat16 precision，weight decay=0
- **SFT 数据构造**: Math 任务用 DeepSeek R1 生成推理链（过滤正确答案，96% 标注成功率）；Science Q&A 用 GPT-4o 生成
- **评估 benchmark**: Hellaswag, TruthfulQA, MMLU, IFEval, Winogrande, HumanEval（LLM）；开/关抽屉任务（机器人）
- **ParityMNIST**: 500 张图片预训练，15 个 learning rates（3e-6 到 1e-3 log-spaced），constant/cosine scheduler，1-2 epochs，约 500 runs/method
- **KL 测量**: Forward KL = E_{x~τ}[KL(π₀ || π)]，在**新任务**输入上评估（不是旧任务）
- **R² 拟合**: MNIST 上 Forward KL vs Forgetting 二次拟合 R²=0.96；LLM 上 R²=0.71（噪声来自近似 KL 和 accuracy 估计）
- **Oracle SFT 分布**: 闭式解 q* = argmin_q KL(π₀||q) s.t. q^T R = 1，其中 R 是 binary correctness indicator

### 核心结果

| 实验 | 关键发现 |
|------|---------|
| LLM Math/Science/Tool Use | RL Pareto 前沿始终在 SFT 之上，RL 达到相同新任务精度时遗忘显著更少 |
| 机器人 Pick-and-Place | 同样 trade-off 在 embodied policy 上复现 |
| ParityMNIST toy | Forward KL vs Forgetting 跨 SFT/RL 落在同一条曲线上（R²=0.96），证明 KL 是统一预测器 |
| Oracle SFT | 用 KL-minimal 分布训练的 SFT 遗忘比 RL 还少，证明关键是分布而非算法 |
| 四象限消融 | on-policy 方法（GRPO/1-0 Reinforce）KL 小→遗忘少；offline 方法（SFT/SimPO）KL 大→遗忘多。关键变量是**on-policy**，不是 negative gradients |
| SFT Distillation | 用 RL 训练模型的输出做 SFT，匹配 RL 的 accuracy-forgetting trade-off |
| 替代假设排除 | 权重变化 L1/L2/Fisher、激活变化、更新稀疏性/秩、Reverse KL、TV 距离——都不如 Forward KL 预测力强 |
| CKA 表示分析 | RL 模型保持高表示相似度（CKNNA=0.94），SFT 显著漂移（CKNNA=0.56） |
| Scaling 实验 | SFT 在 3B/7B/14B 上都表现出相同的 trade-off，更大模型只提高起点但不改变 trade-off 形态 |
| 理论 | 证明 policy gradient + binary reward 等价于 EM with I-projection + M-projection，收敛到 KL-minimal optimal policy |

**Surprising 发现**：
- Mukherjee et al. (2025) 声称 RL 更新稀疏导致遗忘少，但本文发现稀疏性是 bfloat16 精度造成的假象——用 float32 训练后稀疏性消失但性能相同
- SFT 不需要"差"，只要显式引导到 KL-minimal 分布，甚至可以优于 RL

### 明确局限

1. **无真机实验**：机器人实验全在 SimplerEnv 仿真中进行，未在真实机器人上验证
2. **模型规模有限**：最大只测到 Qwen 2.5 14B，frontier scale (70B+) 未知
3. **只研究了 on-policy RL vs SFT**：未涵盖 off-policy RL 算法（如 DQN、SAC），这些在传统 RL 中很流行
4. **缺乏机械论解释**：只建立了 KL→遗忘的预测关系（empirical law），但没有解释**为什么**新任务上 KL 变大会破坏旧知识——是通过表征干扰、隐式容量限制还是其他机制？
5. **Forward KL 在 LLM 上测量困难**：需要计算完整 output distribution 的 KL，实际只能近似；R²=0.71 虽然可接受但不如 toy setting
6. **任务多样性有限**：只测了 math、science QA、tool use 三种 LLM 任务和一种机器人任务
7. **未与显式 KL 正则化方法系统对比**：只在 MNIST 上做了 GRPO+KL reg 的一个数据点
8. **理论假设受限**：定理假设 e-flat (exponential family) 策略族和线性约束 P*，实际神经网络不满足；binary reward 假设简化了场景

**潜在研究方向**：
