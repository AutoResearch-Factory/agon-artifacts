# CDE: Curiosity-Driven Exploration for Efficient Reinforcement Learning in Large Language Models (2509.09675)

- **Year**: 2025.09
- **Authors**: Runpeng Dai, Linfeng Song, Haolin Liu, Zhenwen Liang, Dian Yu, Haitao Mi, Zhaopeng Tu, Rui Liu, Tong Zheng, Hongtu Zhu, Dong Yu; Tencent AI Lab, UNC Chapel Hill, University of Virginia, University of Maryland College Park, Tencent Multimodal Department
- **Venue**: arXiv
- **Keywords**: RLVR, Curiosity-Driven Exploration, PPL bonus, multi-head critic, entropy collapse, calibration collapse, count-based exploration, Qwen3-4B
- **One-liner**: 用PPL和多头critic不确定性增强RLVR探索。

## 精华提炼

### 管线/方法
- 论文目标是解决 RLVR 训练 reasoning LLM 时的探索不足、早熟收敛和 entropy collapse。
- 作者先尝试把传统 count-based exploration 搬到 CoT 轨迹上。
- Count baseline 的输入是 prompt-response 轨迹 `(q,o)`。
- 它取 LLM last-layer hidden states，形成轨迹 embedding `h_{q,o}`。
- embedding 方案包括最后 token、倒数第二个 token、或全轨迹 mean pooling。
- 然后用随机投影矩阵 `A in R^{kxd}` 做 SimHash，得到 k-bit hash cell。
- 每个 cell 的访问数 `n(b)` 被当作 pseudo-count。
- intrinsic reward 形式是 `r + omega_t * beta_t / sqrt(n(b))`。
- 这个 baseline 的问题是 CoT 轨迹太长太复杂，固定 embedding 表达力不足。
- 图 1 显示 hash-cell 访问高度集中，多数 response 塌到相同或邻近网格。
- 因此论文放弃外部 count，转向模型内部的 curiosity signal。
- CDE 包含两个分支：actor curiosity 和 critic curiosity。
- Actor curiosity 用当前 policy 对自己生成 response 的平均负 logprob，即 response-level PPL。
- 公式是 `B_actor(q,o) = -1/T * sum_t log pi(o_t | o_<t, q)`。
- 高 PPL 表示当前 actor 认为这条 response 更陌生。
- 直接加 PPL 会鼓励高 PPL 垃圾输出，所以作者加了 adaptive clipping。
- Actor reward shaping 写成 `r(q,o) + omega_t * min(|r(q,o)|/kappa, alpha * B_actor(q,o))`。
- `omega_t` 是随训练衰减的 bonus weight。
- `kappa` 控制 bonus 上限，避免 curiosity 反转原始 correctness reward。
- `alpha` 缩放 PPL bonus，决定它多容易触顶。
- 这个 actor bonus 可直接接入 GRPO，也可接入 PPO。
- 在 GRPO 中，修改后的 sentence-level reward 用来计算 group-normalized advantage。
- 在 PPO 中，作者同时引入 multi-head critic 作为 critic curiosity。
- Critic curiosity 的输入是 prompt 和 partial response prefix。
- PPO critic 不再是单个 value head，而是共享 LLM backbone 的 K 个 value heads。
- 每个 head 在 collected trajectories 的不同无放回子集上训练，子集比例是 `zeta`。
- 对某个 prefix，K 个 heads 的 value 标准差就是 `B_critic`。
- Actor update 中，GAE advantage 被加上 clipped `omega_t * min(|A|/kappa, alpha * B_critic)`。
- Critic update 的数据集是 `(q, o_{i,<=t}, r_i)` 级别的 partial-response triplets。
- 每个 critic head 最小化自己的 squared value loss。
- 方法整体只改 reward/advantage shaping 和 critic head，不需要额外 reward model。
- 训练使用 rule-based verifier 比较最终答案和 ground truth。
- 训练 prompt 是数学题逐步解答模板，最后必须输出 `Answer: $Answer`。

### 关键技术细节
- 训练数据是 DAPO-17K。
- 模型是 Qwen3-4B-Base。
- 框架是 `verl`。
- 评测集是 MATH、AMC23、AIME24、AIME25。
- MATH 报 Avg@1。
- AMC23、AIME24、AIME25 报 Avg@16 和 Pass@16。
- GRPO baseline 和 PPL-GRPO 的 actor learning rate 是 `1e-6`。
- GRPO 的 max prompt length 是 2K，max response length 是 3K。
- GRPO 的 train batch size 是 256。
- GRPO 的 rollout.n 是 8。
- GRPO 的 total training steps 是 300。
- GRPO 的 sample temperature 是 1.0。
- GRPO 的 clip ratio 是 0.20。
- GRPO 的 KL coefficient 是 0.0。
- PPO 的 actor learning rate 是 `1e-6`。
- PPO 的 critic learning rate 是 `1e-5`。
- PPO 的 critic warmup 是 10。
- PPO 的 train batch size 是 512。
- PPO 的 rollout.n 是 4。
- PPO 的 PPO mini-batch size 是 256。
- PPO 的 total training steps 同样是 300。
- PPL bonus 的实验设置是 `kappa=3, alpha=1, omega_t=Staircase`。
- Multi-head critic 的设置是 `kappa=3, alpha=0.5`。
- 2-head 和 4-head critic 使用 `zeta=1`。
- 8-head 和 16-head critic 使用 `zeta=0.5`。
- Multi-head critic 实验中 `omega_t` 不衰减。
- 作者比较了 PPL bonus 的 No decay、Linear、Cosine、Staircase 四种 schedule。
- 最优是 Staircase：早期保持强探索，后期突然去掉 bonus 以利于收敛。
- 论文的 calibration theorem 把 PPL bonus 解释为：高 PPL 正确 response 相对概率增幅更大。
- 同一个 theorem 也声称：低 PPL 错误 response 相对概率下降更大。
- Critic theorem 在 linear MDP 假设下，把 multi-head bootstrap value std 连接到 `sqrt(phi^T Lambda^{-1} phi)`。
- 这个理论只在 linear MDP 和 ridge/bootstrap 近似中成立，不直接证明 LLM setting。
- 正文公式和附录 proof 对 `kappa` 的写法存在轻微记号风险：正文用 `|r|/kappa`，proof 中转写为 `min{kappa, ...}` 且要求 `kappa<1`，但实验表写 `kappa=3`。

### 核心结果
- Qwen3-4B-Base 未 RL 时 overall Avg 是 9.2。
- GRPO baseline overall Avg 是 48.2。
- GRPO + PPL bonus overall Avg 是 50.6，提升 +2.4。
- GRPO + PPL 在 MATH Avg@1 上从 87.3 到 87.7。
- GRPO + PPL 在 AMC23 Avg@16 上从 63.6 到 67.8。
- GRPO + PPL 在 AMC23 Pass@16 上从 89.1 到 89.5。
- GRPO + PPL 在 AIME24 Avg@16 上从 20.8 到 23.3。
- GRPO + PPL 在 AIME24 Pass@16 上从 41.9 到 48.5。
- GRPO + PPL 在 AIME25 Avg@16 上从 21.0 到 23.5。
- GRPO + PPL 在 AIME25 Pass@16 上从 39.2 到 42.5。
- PPO baseline overall Avg 是 46.5。
- PPO + PPL bonus overall Avg 是 47.7，提升 +1.2。
- PPO + 2 Heads overall Avg 是 46.6，基本无提升。
- PPO + 4 Heads overall Avg 是 48.5，提升 +2.0。
- PPO + 8 Heads overall Avg 是 48.1，提升 +1.6。
- PPO + 16 Heads overall Avg 是 48.6，提升 +2.1。
- AIME25 Pass@16 上，PPO baseline 是 33.7，4 Heads 是 45.5，提升 +11.8。
- AIME24 Pass@16 上，PPO baseline 是 36.0，8 Heads 是 46.4，提升 +10.4。
- 训练曲线显示 CDE early phase 学得更慢，后期超过 baseline。
- 作者解释为探索信号抑制了过早 exploitation。
- PPL schedule 消融中 No decay overall Avg 是 48.2。
- Linear decay overall Avg 是 48.7。
- Cosine decay overall Avg 是 49.7。
- Staircase decay overall Avg 是 50.6。
- Entropy dynamics 显示 baseline GRPO entropy 下滑明显。
- PPL bonus 缓解 entropy collapse。
- No decay 会造成持续波动，收敛不稳定。
- Calibration 分析中，naive GRPO 早期正确 response 的 PPL 低于错误 response。
- 随训练推进，这个差距缩小并消失，作者称为 calibration collapse。
- 加 PPL bonus 后，正确和错误 response 的 PPL 分离保持更久。
- Multi-head critic 的 `B_critic` 均值随训练下降，支持它类似 visitation count 的解释。
- Training set 的 value-head std 低于 AMC23 和 GPQA，符合未见数据不确定性更高的直觉。
- `zeta` 消融中 4-head 和 16-head 在 `zeta=0.5/1` 下 overall Avg 都约 48.4 到 48.6，说明不敏感。

### 明确局限
- 论文是 arXiv preprint，没有 peer-reviewed venue 信息。
- 没看到公开代码或可直接复现实验 artifact。
- 虽然使用 `verl`，但实现细节不足以第三方完整复现所有曲线。
- 所有主实验都在 Qwen3-4B-Base 上，未验证更大模型或其他模型族。
- 训练预算是 reduced setting，300 steps，不能证明长训仍稳定。
- 任务主要是数学 reasoning，未覆盖 coding、agent tool-use、信息抽取等 RLVR setting。
- 只有 rule-based final-answer verifier，没有过程级 reward 或人工审查。
- Count-based baseline 主要是作者自己的 SimHash 版本，未和更强 semantic trajectory novelty 方法系统比较。
- Actor PPL bonus 和 entropy bonus 的区别讲清楚了，但与 2025-2026 后续 token-level entropy allocation 方法的关系还需要更新。
- Multi-head critic 理论依赖 linear MDP、sub-Gaussian noise 和 ridge bootstrap，和 LLM PPO value head 的实际非线性 setting 有差距。
- Calibration collapse 的证据来自训练 batch PPL stratification，不是标准 ECE/Brier/NLL calibration benchmark。
- 论文没有报告置信区间、多 seed 统计或显著性检验。
- 多数 claim 由同一团队生成和验证，缺少独立 replication。
- 公式中的 `kappa` 记号在正文、proof、实验表之间需要小心核对。
- 反引文显示 2025.10 到 2026.06 已有大量相邻工作，CDE 的 novelty window 已经很拥挤。

### 跟我们的关系
- 对 `0616-fex`，如果主题是 RLVR 中的 exploration/exploitation、feedback shaping 或 uncertainty-driven training，这篇是高度撞车论文。

- 直接撞车的 claim 3：用 multi-head value disagreement 作为 critic-side exploration bonus。

- 可复用工具：PPL stratified by correctness 的训练诊断图。

## Read by: 0616-fex (2026-06-16)

本次按 `0616-fex` 作为 RLVR exploration/calibration 相关 topic 来读。CDE 是这个方向必须纳入的高相关工作，因为它同时覆盖 actor-side PPL novelty、critic-side epistemic uncertainty、entropy collapse 和 calibration collapse 四条线。

撞车风险很高：如果我们的核心方案是“把 PPL/低概率轨迹作为探索奖励”，CDE 已经做了 response-level 版本，并给了 GRPO/PPO 实验；如果我们的方案是“多头 critic 不确定性驱动探索”，CDE 也已经做了 K-head value std 版本，并把它理论连接到 count-based bonus。

可直接复用的部分包括：DAPO-17K + Qwen3-4B-Base + AIME/MATH 的基线设置，PPL by correctness 的 calibration diagnostic，Staircase bonus decay，以及 multi-head value std 随训练下降的 coverage diagnostic。

差异化应避开“再加一个 PPL bonus”这种表层 claim。更可行的方向包括：token/position-aware entropy 或 PPL allocation，semantic-space trajectory exploration，和后续反引文中 DiPO/ICPO/SALT/PAEC/OPEFO 对比的系统实验；也可以转到多模态、tool-use、检索或 agent setting，验证 CDE 在非数学 final-answer verifier 下是否仍成立。

该文应该作为 `0616-fex` 的强 baseline 和相关工作核心引用。下一轮搜索关键词建议围绕 `perplexity policy optimization`, `intrinsic confidence RLVR`, `entropy collapse RLVR`, `semantic-space exploration RLVR`, `multi-head critic uncertainty`, `group-based policy optimization rollouts`。
