# On the Design of KL-Regularized Policy Gradient Algorithms for LLM Reasoning (2505.17508)

- **Year**: 2025.05
- **Authors**: Yifan Zhang (Princeton/UCLA), Yifeng Liu (UCLA), Huizhuo Yuan (UCLA), Yang Yuan, Quanquan Gu (UCLA), Andrew Chi-Chih Yao (Tsinghua)
- **Venue**: ICLR 2026 (submission format uses iclr2026_conference.sty)
- **Keywords**: KL regularization, policy gradient, REINFORCE, off-policy RL, LLM reasoning, importance sampling, GRPO, DAPO, unnormalized KL, RPG-Style Clip
- **One-liner**: 统一了 KL 正则化策略梯度中 forward/reverse KL、normalized/unnormalized KL、fully-differentiable/REINFORCE-style 的推导框架，提出 RPG-Style Clip 稳定 off-policy 训练，在数学推理上超越 DAPO 最多 +6 个百分点。

## 精华提炼

### 管线/方法

**RPG 框架整体流程**：
1. **输入**：当前策略 π_θ^(t)，参考策略 π_old^(t)（上一轮策略），奖励函数 R(x)
2. **Step 1 - 构建 KL 正则化目标**：J(θ) = E_{π_θ}[R] - β·Divergence，其中 Divergence 可选 FKL/RKL/UFKL/URKL
3. **Step 2 - 推导 off-policy 策略梯度**：使用 importance weight w(x)=π_θ(x)/π_old(x)，在 π_old 采样的数据上计算梯度
4. **Step 3 - 构造 surrogate loss**：两种选择——Fully Differentiable（直接对 loss 求导得梯度）或 REINFORCE-style（stop-gradient 分离权重与 logπ）
5. **Step 4 - 优化更新**：梯度下降得到 π_θ^(t+1)
6. **迭代**：周期性更新 π_old ← π_θ（每 K 步或 token-level KL 超阈值 κ 时触发）

**具体 Loss 公式（unnormalized 版本）**：

| 正则化类型 | Fully Differentiable Loss | REINFORCE-style Loss |
|-----------|--------------------------|---------------------|
| UFKL (Forward) | Z_old·E[-wR + β(w - log w - 1)] | -E[SG(Z_old(wR - β(w-1)))·log π_θ] |
| URKL (Reverse) | Z_old·E[-wR + β(w log w - w)] | -E[SG(Z_old·w(R - β log w))·log π_θ] |

Normalized 版本在附录中，结构类似但无 Z_old 和 mass correction 项。

### 关键技术细节

**模型架构与训练配置**：
- Base models: Qwen3-4B 和 Qwen2.5-7B-Instruct
- 训练数据: DAPO-Math-17k (13.9k 英文样本)
- 框架: verl (HybridFlow) + vLLM 推理引擎
- 硬件: 8×NVIDIA H100 GPU
- Optimizer: AdamW, lr=1×10⁻⁶, weight decay=0.1, gradient clipping=1.0
- 总步数: 400（含 10 warmup 步，之后 constant lr）
- Global batch size: 512，每样本 rollout 16 responses（temperature=1.0）
- Per-GPU mini-batch: 32
- 最大序列长度: 2K/4K/8K token 对应不同 context length 设置
- KL 系数 β = 1×10⁻⁴
- Clip 参数: RPG/RPG-REINFORCE/DAPO 用 (ε₁, ε₂)=(0.2, 0.28)，GRPO 用 (0.1, 0.1)

**RPG-Style Clip（Dual-Clip）机制**：
- 对 importance weight w(x) 做双重截断
- 正 advantage: w clip 到 [1-ε₁, 1+ε₂]
- 负 advantage: 加 lower bound c>1（c=2.25），防止负 advantage 时 loss 过于激进
- REINFORCE-style 版本：将权重分解为 w(x) 和 regularized advantage Â_reg(x)，分别 clip

**k₃ estimator 与 UKL 等价性**：
- k₃(y) = y - 1 - log y
- Forward: E_{π_old}[k₃(π_θ/π_old)] = UKL(π_old || π_θ)
- Reverse: E_{π_θ}[k₃(π_old/π_θ)] = UKL(π_θ || π_old)
- 因此 GRPO 使用的 k₃ penalty 本质是在做 unnormalized reverse KL 正则化

**GRPO KL term 的 off-policy bias**：
- GRPO 直接在 loss 中加 -β·k₃(π_ref/π_θ)，但采样来自 π_old 而非 π_θ
- 遗漏了 importance weight w = π_θ/π_old
- 修正后的 off-policy estimator: E_{o~π_old}[w·k₃(π_ref/π_θ)]
- 论文指出了 bias 但未做修正后的 GRPO 对比实验

**迭代式 reference 更新**：
- 定期设 π_old ← π_θ（每 K optimizer steps 或 KL 超 κ 阈值）
- 实现 practical KL trust region，避免过度正则化到初始 checkpoint
- 实现技巧：π_old 的概率预计算存储，训练时只需一个模型在 GPU 内存中
- 对比实验显示无迭代 reference 时性能明显下降

**NPG 连接**（附录）：
- RPG 目标 J(θ) - β·KL(π_θ_k || π_θ)
- 对 return 做一阶 Taylor，对 KL 做二阶 Taylor（得到 Fisher F）
- 解得 Δθ* = (1/β)·F⁻¹·∇J，即 natural policy gradient 方向

### 核心结果

**8K Context Length（主表，Qwen3-4B）**：

| Method | AIME24 Last/Best | AIME25 Last/Best | AMC23 Last/Best |
|--------|---------------------|---------------------|--------------------|
| GRPO | 0.375/0.440 | 0.335/0.406 | 0.911/0.930 |
| DAPO | 0.544/0.574 | 0.447/0.474 | 0.938/0.943 |
| RPG-UFKL | **0.594/0.618** | 0.470/0.487 | **0.949**/0.952 |
| RPG-URKL | 0.454/0.526 | 0.526/0.494 | 0.941/**0.954** |
| RPG-REINFORCE-UFKL | 0.591/0.596 | 0.483/0.503 | 0.945/0.947 |
| RPG-REINFORCE-URKL | 0.571/0.578 | **0.507/0.521** | 0.940/0.947 |

- RPG-REINFORCE-URKL 在 AIME25 达到 52%，超越官方 Qwen3-4B-Instruct (47%)
- 最大提升：RPG-UFKL vs DAPO 在 AIME24 上 +5.9pp (Best)，AIME25 上 +4.7pp (Best)

**4K Context Length**：
- RPG-REINFORCE-URKL 在 AIME24 和 AIME25 上均为第一
- RPG-REINFORCE-UFKL 在 AIME24 第二，AIME25 第二

**2K Context Length**：
- RPG-REINFORCE-UFKL 在 AIME24/AIME25 均为 Best 第一
- RPG-UFKL 在 AIME24 Last 第一

**Qwen-2.5-7B-Instruct 实验结果**：
- RPG/RPG-REINFORCE 系列在 AMC23 上显著超越 baselines
- RPG-REINFORCE-FKL AIME24 Best=0.167 (vs GRPO 0.141)
- 7B 模型上的收益不如 4B 明显

**Surprising/反直觉发现**：
1. Fully Differentiable 和 REINFORCE-style 在性能上接近，但 REINFORCE-style 在大规模训练中更稳定
2. UFKL (Forward unnormalized) 在多个 setting 下优于 URKL (Reverse)，与常见直觉（reverse KL mode-seeking 更适合 LLM）不完全一致
3. 更高 clip ratio (0.2, 0.28) 比 (0.1, 0.1) 显著提升 actor entropy，DAPO 的 "clip-higher" 策略通过提高 entropy 来提升性能
4. 迭代 reference 更新对性能至关重要——无迭代时性能明显下降
5. β=1e-4 优于 β=1e-3，更弱的 KL 约束反而更好

### 明确局限

**作者自述局限**：
- RPG-Style Clip 引入可控的 bias-variance trade-off，(ε₁,ε₂) 需要人工调参，缺乏 principled schedule
- 使用 LLM 辅助润色论文

**审读发现的局限**：
1. **无真机验证**：纯 LLM 数学推理 benchmark，没有 robot/agent 真机实验
2. **仅数学推理**：只在 AIME/AMC/Math 类任务上验证，未涉及 coding、agent 等多步推理场景
3. **模型规模有限**：最大 7B，未验证 scaling 到 70B+ 时的行为
4. **GRPO bias 发现未实证**：论文指出 GRPO KL term 的 off-policy importance weight 缺失，但未做"修正后 GRPO vs 原始 GRPO"的对比实验来证明这个修正的实际效果
5. **未与 RLOO/VinePPO 对比**：只对比了 GRPO/DAPO/REINFORCE++
6. **2K 下收益不明显**：2K context 时 RPG 与 DAPO 差距缩小
7. **未开源代码**：虽有 GitHub 链接但未验证代码可用性
8. **β 和 clip 参数的交互未探索**：ablation 只单独研究了 clip ratio 和 β，未研究二者交互
9. **缺少统计显著性检验**：所有结果只报均值，无标准差/置信区间
10. **Critic 模型价值未分析**：RPG 框架兼容 critic-based 方法，但实验只用 value-free 变体

### 跟我们的关系

**对 0616-fex 课题的评估**：

- **撞车评估**：中低风险。RPG 关注 KL 正则化策略梯度的统一推导和 off-policy 重要性采样修正，是一个 RL 算法设计框架。0616-fex 如果关注的是 feature extraction / feature engineering / 特征空间的探索策略，则不直接撞车——RPG 不涉及特征空间设计。

- **可用工具/方法**：
  1. **RPG-Style Clip 可迁移**：如果 0616-fex 涉及 off-policy RL 训练，Dual-Clip 截断是可直接借鉴的稳定化技术
  2. **迭代 reference update 策略**：周期性地同步 reference policy，可作为任何 KL 正则化 RL pipeline 的通用组件
  3. **k₃ = UKL 的等价性洞察**：在设计 KL penalty 时可以直接用 UKL framing 替代 ad-hoc estimator
  4. **REINFORCE-style vs Fully Differentiable 的选择**：如果追求大规模稳定训练，REINFORCE-style + stop-gradient 是更好的工程选择
  5. **verl 框架作为实验基础设施**：如果想做 LLM RL 实验，verl + vLLM 已被 RPG/DAPO/GRPO 多个工作验证

- **差异化方向**：
  1. RPG 没有涉及特征/状态空间的 representation learning——这是 0616-fex 的可能切入方向
  2. RPG 没有考虑多模态或非文本 reward——fex 如果涉及视觉/具身特征会有差异
  3. RPG 没有讨论 exploration strategy 的设计空间——fex 如果在探索策略上有创新则不重叠

- **该引用**：如果 0616-fex 涉及任何形式的 KL 正则化策略梯度，必须引用本文作为 KL 设计空间的基础 reference。
