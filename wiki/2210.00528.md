# Data-driven Automated Negative Control Estimation (DANCE): Search for, Validation of, and Causal Inference with Negative Controls (2210.00528)

- **Year**: 2022.10
- **Authors**: Erich Kummerfeld (Institute for Health Informatics, U Minnesota), Jaewon Lim (Biostatistics, U Washington), Xu Shi (Biostatistics, U Michigan)
- **Venue**: arXiv (no confirmed peer-reviewed venue in metadata as of read date)
- **Keywords**: negative control validation, unmeasured confounding, vanishing tetrad test, trek separation, subcovariance rank constraint, proximal/double negative control, graphical causal model, GMM estimation
- **One-liner**: 首个可证明正确的负控制变量数据驱动验证检验(vanishing tetrad)，替代纯专家知识识别，聚合多对负控制估计因果效应。

## 精华提炼

### 管线/方法

- 输入：一组候选 negative control (NC) 变量、处理变量 T、结局变量 O、观测数据（协变量可选，作为额外调整项）。
- 第一步 — DNCT (disconnected negative control triplet) validation test：对任意三个候选 NC 组成的三元组 {X, Y, Z}，构造 6 个 vanishing tetrad 检验（3 个涉及 T，3 个涉及 O），全部检验都不拒绝"行列式为零"的零假设时，判定该三元组为合法 DNCT。
- 第二步 — FindNC 算法：对候选 NC 集合做蛮力枚举（所有三元组，复杂度 O(|V|^3)），逐一跑 DNCT validation test，输出所有通过检验的 DNCT 集合（Algorithm 1）。
- 第三步 — Aggregated Negative Control Estimation：对每个通过验证的 DNCT，其内部三个变量可以组成 6 种 double-NC pair（Z、W 互换角色），分别用 GMM 结合 outcome confounding bridge function h(W,T;α) 求解处理效应 Δ；再按每个 NC pair 被选中的频率做加权平均，得到最终点估计和置信区间（Algorithm 2）。
- DANCE = FindNC + Aggregated NCE 串联（Algorithm 3），整个流程不需要用户预先手工指定哪些变量是合法 NC，只需要提供一批"候选"变量。
- 若没有任何三元组通过 DNCT validation test，DANCE 明确报告"无满足条件变量"而不是勉强给出估计——这是一个设计上的诚实止损点。

### 关键技术细节

- 理论基石：Sullivant (2010) 的 trek separation 定理 + Spirtes (2013) 的 extended trek separation 定理，把图上的 t-separation 关系映射为 subcovariance matrix 的 rank 上界，从而把"图结构假设"转成"可用行列式检验的统计假设"。
- vanishing tetrad 检验采用经典 Wishart (1928) 检验：在 Gaussian 假设下，用渐近正态分布对行列式估计量 D̂ 构造 z 统计量并计算双侧 p 值；作者提到 Bollen (1993) 的 distribution-free tetrad test 可放松高斯假设，但计算量更大、有限样本 power 更低，因此正文和仿真统一使用 Wishart 检验。
- "Simple NC model" 定义（Definition 1）：只有一个 unmeasured 变量 U（可以是多个 latent 因子的线性组合，如"就医行为倾向"由多个子因子构成，但在图上仍表示为单一节点），U 直接导致 M 中所有变量；M 中有两个特权变量 T、O，T 可以导致 O 但没有其他变量直接连接 T 或 O。
- 三条核心假设：Assumption 1（|M|≥5，至少 3 个候选 NC）；Assumption 2（tetrad faithfulness，即 tetrad 为零不是系数巧合导致的"意外零"）；Assumption 3（M 中变量间关系是线性无环 SEM）。
- FindNC 的显著性阈值 alpha 默认取 n^{-1}（n 为样本量），论文声称这是仿真中在大范围样本量下都表现均衡的启发式默认值。
- Double-NC 估计沿用 Miao et al. (2018) 的 GMM 框架，moment restriction 结合 outcome confounding bridge function h(W,T,X;α)；线性 SEM 特例下有 Kuroki (2014) 给出的协方差闭式解 Δ = [cov(T,O)cov(Z,W) − cov(W,O)cov(T,Z)] / [cov(T,T)cov(Z,W) − cov(T,Z)cov(T,W)]。
- 多 NC pair 聚合的方差估计：aggregated Δ̂ 是各唯一 NC pair 估计量的加权线性组合，用 GMM sandwich estimator（A_n^{-1} B_n A_n^{-1}）或非参数 bootstrap 求整体方差（Appendix B）。
- 复杂度对比：FindNC 是 O(|V|^3) 蛮力搜索，作者强调这在候选 NC 数量可行的实际问题规模下是可接受的，远优于因果发现算法搜索图空间的超指数复杂度；作者明确把"更高效的搜索"列为 future work，未在本文实现。

### 核心结果

- 仿真设计两种图：Simple Graph（4 个候选 NC，其中 2/4 三元组是真 DNCT）和 Complex Graph（7 个候选 NC，其中 12/35 三元组是真 DNCT），样本量覆盖 10–10,000，200 次重复。
- ROC 验证：弱边强度下样本量 ≥300、强边强度下样本量 ≥100 时，DNCT validation test 对 valid/invalid NC 的区分接近完美分离。
- 偏差与覆盖率（n=1000/3000）：DANCE 的 bias 接近 0（-1.82~1.11 ×10⁻³），95% CI coverage 0.92–0.96，接近名义水平；Random method（随机选 NC pair）bias 明显更大（19~97 ×10⁻³），且在 complex graph + 大样本下 coverage 崩溃到 0.05（几乎完全失效）。
- 附录鲁棒性检查：强边强度场景下结果类似（DANCE coverage 仍 0.94–0.96）；全变量为 Bernoulli 二值变量场景下 DANCE 的 ROC 和偏差表现依旧良好，尽管正式正确性证明只覆盖连续线性情形。
- 真实数据应用：SUPPORT 研究（5,735 名 ICU 重症患者，评估右心导管术 RHC 对 30 天生存的效应），以 72 个协变量为候选 NC 池。DANCE 找到 43 个 DNCT，产生 164 个唯一 double-NC pair（总组合 258 对）。最常被选中的 pair 是 dementhx（痴呆/中风/帕金森病史）和 gibledhx（上消化道出血史）。
- RHC 结果对比：忽略未测混杂的简单回归给出 RHC 效应 -1.29（95% CI [-1.83, -0.75]，不含 0，显著负效应）；DANCE 最常见 NC pair 给出 -2.98（[-14.96, 8.99]，覆盖 0）；聚合全部 164 对 NC pair 给出 -0.19（[-1.50, 1.12]），比单一 pair 精确得多，同时依然覆盖 0——即数据驱动的未测混杂校正消解了"简单回归"给出的虚假显著性。
- 一个有说服力的内部一致性证据：把 dementhx/gibledhx 的 W、Z 角色互换后，两次估计结果几乎相同（-2.98 vs -2.80），符合理论预期（在图 1 结构下 W、Z 角色应可互换而不改变估计），间接印证识别出的 NC 对确实满足假设的因果图结构。

### 明确局限

- Validation test 只识别符合特定 DAG（图 1 的 disconnected NC 结构）的 NC，会把满足其他合法 NC 图结构（fundamental NC assumption 的更一般情形，见 Shi et al. 2020 综述）但不满足 disconnected 结构的真实 NC 排除掉——检验是保守的，可能漏检。
- 正确性证明（Theorem 1，Appendix A）仅覆盖连续变量、线性无环 SEM 情形；binary/categorical 变量下没有形式化证明，只有仿真观察到"表现良好"（Appendix C），作者自己承认这是空白。
- Simple NC model 假设只有一个（可能是多因子线性组合的）unmeasured confounder U；若存在多个作用于不同变量子集的独立 unmeasured confounder，模型未覆盖，作者在 Discussion 中明确列为 future work。
- FindNC 是蛮力 O(|V|^3) 搜索，没有更高效的搜索算法；候选 NC 数量很大时的可扩展性未被系统评估。
- RHC 案例没有金标准 ground truth 可比对，作者的论证止步于"CI 覆盖 0 说明存在未测混杂的不确定性"，而非确证真实效应值，这是观测性因果推断方法论文常见的解释边界，值得读者注意其证据强度上限。
- 正文/附录均未见开源代码或 replication package 链接，虽然算法伪代码和仿真参数表（Table datgen）详尽到可以重新实现，但缺少作者自己发布的实现，独立复现门槛依赖读者自行编码。

### 跟我们的关系

- 课题 `0710-causal-scs`（causal-scs-indicator）关心的问题是：滚动因果效应估计（用 PCMCI+、LKIF 在耦合鞍结分岔仿真轨迹上做因果强度的滚动窗口估计）的不稳定性（dispersion + jump 双通道）是否是 severe convective storm（强对流风暴）发生前的早期诊断信号；预注册 pilot 的冻结 GPU 结果是 NEGATIVE（dispersion AUROC 0.392、jump AUROC 0.517、joint AUROC 0.428，均显著低于 raw EWS baseline 的 0.919）。
- 不撞车：DANCE 解决的是"如何从一批候选变量中用数据驱动方式挑出合法 negative control，从而在存在未测混杂的横截面/i.i.d. 处理效应估计问题中去偏"，本质是静态因果效应估计 + 变量验证问题；而 `0710-causal-scs` 关心的是"因果强度估计量在时间序列上的不稳定性能否作为分岔早期预警信号"，是动力系统状态转移检测问题。两者共享"causal inference"、"unmeasured confounder / nonstationarity 干扰"等词汇，但目标估计量（ATE vs. 时变因果强度轨迹）、数据结构（i.i.d. 单元 vs. 自相关时序轨迹）、验证目标（变量是否满足 NC 假设 vs. 信号是否具有判别力/AUROC）完全不同，不存在 claim 级别的重叠或竞争关系。
- 方法论可借鉴之处：DANCE 的核心精神——"不要默认相信一个假设（某变量是合法 NC），要在使用它之前设计一个数据可以拒绝的验证检验"——与 `0710-causal-scs` pilot 的预注册 gate 设计（dispersion/jump AUROC 阈值 + paired-bootstrap 95% 下界 + 相对 baseline 的提升阈值）是同一种"先验证工具本身、再信任其输出"的方法论姿态。DANCE 提供了一个把这种姿态形式化为可证明正确的统计检验（vanishing tetrad rank test）的具体范例，写作或方法论类比时可以引用作为"causal inference 领域如何做数据驱动假设验证"的正面例子，但不能直接复用其检验本身。
- 工具复用可能性低：vanishing tetrad / trek separation rank test 要求可观测变量满足线性（可选高斯）SEM 且只有一个 unmeasured confounder 节点，这与 `0710-causal-scs` pilot 中鞍结分岔耦合动力系统的非线性、非平稳、时序自相关结构不匹配；直接把 DANCE 的 FindNC/DNCT test 拿来当作与 PCMCI+/LKIF 并列的第三种"因果强度估计方法"并不合适，二者面向的估计目标（ATE 点估计 vs. 因果强度时间序列）本身就不是同一类输出。
- 引用价值：可以作为"负控制变量的数据驱动验证"这一子领域的代表性方法在 related work 或方法论讨论中提及，用来对比"因果推断中验证工具假设"的不同技术路线（rank constraint 检验 vs. AUROC 判别式验证），但不构成核心相关文献，引用优先级中等偏低。

## Read by: 0710-causal-scs (2026-07-11)

DANCE 是负控制变量数据驱动验证这一细分方向的代表性方法，核心贡献是把"候选变量是否满足负控制假设"从纯专家判断变成可证明正确的统计检验（vanishing tetrad rank test）。与 `0710-causal-scs-indicator` 课题（测试滚动因果效应估计的不稳定性能否作为强对流风暴早期预警信号）相比：问题设定完全不同（横截面 ATE 去偏 vs. 时间序列因果强度轨迹的判别力评估），不存在方法或 claim 层面的撞车风险。唯一可迁移的是方法论姿态——"先验证再信任"的预注册/假设检验思维——这与本课题 pilot 已经采用的 AUROC + paired-bootstrap 预注册 gate 设计本质同源，可以在写作中作为因果推断领域同类方法论的旁证引用，但不提供可直接复用的算法或工具。鉴于本课题当前 pilot 已经给出 NEGATIVE 结果（因果强度不稳定性判别力低于原始状态 baseline），DANCE 类"验证候选因果结构假设"的检验思路对诊断失败原因（例如：估计器本身是否在提议的耦合鞍结系统下满足所需的图结构/独立性假设）或许有旁敲侧击的参考价值，但需要专门设计新检验，DANCE 原始检验不能直接套用。总体优先级：了解即可，非核心相关文献。
