# Discovering Physical Directions in Weight Space: Composing Neural PDE Experts (2605.14546)

- **Year**: 2025.05
- **Authors**: Pengkai Wang (HK PolyU), Pengwei Liu (ZJU), Yuanyi Wang (HK PolyU), Guanyu Chen (ZJU), Xingyu Ren (ZJU), Xiaolong Li (ZJU), Zhongkai Hao (Tsinghua), Yuting Kong (ZJU), Qixin Zhang (NTU), Dong Ni (ZJU)
- **Venue**: arXiv (NeurIPS 2026 format)
- **Keywords**: Neural Operators, PDE Surrogate, Model Merging, Weight-Space Directions, Fine-Tuning, Calibration, Extrapolation, Physical Coordinates
- **One-liner**: 发现从同一 anchor 微调的 PDE endpoint experts 的权重残差可以分解为 family-shared adaptation 和 signed physical direction，并基于此提出 Calibration-Conditioned Merge (CCM) 实现零训练成本的跨 regime 迁移。

## 精华提炼

### 管线/方法

完整的 CCM 管线：

1. **Family anchor 训练**：在 support regimes $\Lambda_{\mathrm{sup}}$ 上训练一个基础神经算子 $\theta_0$（FNO 或 DPOT-style backbone）
2. **Endpoint expert 微调**：从 $\theta_0$ 出发，分别在低 regime $\lambda_{\mathrm{low}}$ 和高 regime $\lambda_{\mathrm{high}}$ 上微调得到 $\theta_{\mathrm{low}}$ 和 $\theta_{\mathrm{high}}$
3. **残差分解**：计算 endpoint--anchor 残差 $\Delta_{\mathrm{low}} = \theta_{\mathrm{low}} - \theta_0$, $\Delta_{\mathrm{high}} = \theta_{\mathrm{high}} - \theta_0$，然后分解为：
   - $\Delta^{+} = (\Delta_{\mathrm{low}} + \Delta_{\mathrm{high}})/2$（shared adaptation，两个 end 共同学到的东西）
   - $\Delta^{-} = (\Delta_{\mathrm{high}} - \Delta_{\mathrm{low}})/2$（signed physical direction，低→高的变化方向）
4. **坐标族定义**：$\theta(\alpha) = \theta_0 + \Delta^{+} + \alpha \Delta^{-}$，其中 $\alpha$ 是组合坐标。$\alpha = -1$ 恢复低端 expert，$\alpha = +1$ 恢复高端 expert，$|\alpha| > 1$ 外推
5. **坐标选择（CCM）**：根据可用信息选择 $\hat{\alpha}$：
   - **CCM-Coord**：直接用归一化物理坐标 $s(\lambda)$ → $\hat{\alpha} = \text{clip}(s(\lambda))$
   - **CCM-Scale**：物理坐标缩放 → $\hat{\alpha} = \text{clip}(\gamma \cdot s(\lambda))$，$\gamma$ 在 validation 上选一次
   - **CCM-Prefix**：用 rollout 前 $K$ 步做 prefix 在 $\alpha$ bank 中搜索最小 loss 的 $\alpha$
6. **部署**：单个合并 checkpoint 做完整 rollout，无需 fine-tuning、routing 或 ensembling

### 关键技术细节

- **架构**：FNO（Fourier Neural Operator）为主，验证了 3 个规模（small: w48/modes12/l4, base: w64/modes16/l4, large: w80/modes20/l4）；DPOT-style backbone（width 512）为跨架构验证
- **训练数据**：三个 PDE 家族——DiffReact（PDEBench 双通道扩散-反应）、NS2D（周期涡度形式不可压缩 Navier-Stokes，128x128 网格）、RDB（PDEBench 浅水方程径向溃坝）
- **训练超参数**：
  - RDB FNO：base lr=$5\times10^{-4}$，expert lr=$2\times10^{-6}$
  - NS2D FNO：base lr=$10^{-4}$，expert lr=$5\times10^{-5}$
  - DPOT DiffReact：base lr=$5\times10^{-4}$，expert lr=$10^{-4}$
- **物理坐标归一化**：$s(\lambda) = 2(\lambda - \lambda_{\mathrm{mid}})/(\lambda_{\mathrm{high}} - \lambda_{\mathrm{low}})$，endpoints 固定为 $s = \pm 1$
- **评估指标**：自回归 rollout L2 error（全 horizon），CCM-Prefix 使用 future-only 协议（$K=4$ prefix 用于选择，后续帧报告 L2）
- **α bank**：$\mathcal{A} = \{-1.50, -1.25, -1.00, ..., 0, ..., 1.50\}$，13 个候选值
- **硬件**：单台 8×NVIDIA A800 服务器

### 核心结果

**OOD 外推成绩（主要发现）**：
| Benchmark | Base | Endpoint Avg | CCM | OOD 误差降低 |
|-----------|------|-------------|-----|------------|
| DiffReact f2 dense | 0.188 | 0.183 | 0.086 (CCM-Coord) | 54.2% |
| NS2D viscosity | 0.591 | 0.586 | 0.338 (CCM-Scale) | 42.8% |
| RDB high-center | 0.471 | 0.450 | 0.406 (CCM-Prefix) | 13.8% |

**关键对比发现**：
- Endpoint averaging（$\alpha=0$）在 merge-friendly axes 上有效（如 DiffReact $D_v$：error 从 0.178→0.142），但在需要 signed direction 的轴上反而有害（$D_u$ medium-gap：avg 0.253 vs base 0.251）
- CCM-Coord 在 DiffReact 上从 mean 0.132→0.082（37.4% 降低），OOD 部分从 0.188→0.086（54.2%）
- **Wrong-sign control 关键证据**：翻转坐标符号（$\alpha = s \rightarrow \alpha = -s$）使 OOD error 增加 3.91x，证明方向是有物理意义的而非随意插值
- **坐标对齐证据**：DiffReact $r=1.00$，NS2D $r=0.99$，RDB $r=0.79$（RDB 标量元数据较弱，需要 prefix calibration）

**Scalability**：
- FNO scale sweep：从 small→base→large，endpoint averaging 相对 gain 从 9.6%→20.2%→24.8%，说明更强的基础模型使 endpoint 更新更具可组合性
- DPOT 验证：Base 0.045→Avg 0.039→CCM-Coord 0.018，匹配 oracle

**Seed reruns**：DiffReact CCM-Coord OOD mean = 0.0864±0.0002（3 seeds），RDB CCM-Prefix OOD = 0.0289±0.0002，结果稳定

**Generic merge baselines（Task Arithmetic / TIES / DARE）**：在 merge-friendly axis 上远不如 endpoint averaging（TA error 4579 vs avg 0.113），因为这些方法设计给无序 task 而非有序物理坐标

### 明确局限

- **作者说的**：
  - CCM 假设局部几何、兼容架构和稳定微调动态，不给 universal OOD guarantee
  - CCM-Prefix 需要短观察 horizon（$K=4$），不是真正 zero-shot
  - 坐标假设在 endpoint gap 太大、离 anchor 太远或 curvature 高时崩溃
  - 当前只处理单标量物理方向，多参数 PDE 族需要 cross-axis experts 或高阶 charts
- **读出来的**：
  - 没有真机实验，全是数值仿真 benchmark
  - 只用标准 benchmark（PDEBench 等），没有在真实科学计算场景测试
  - 只在 FNO 和 DPOT 两个架构上验证，未测试更现代的架构（如 Transformer-based operators）
  - RDB 上 CCM-Prefix gain 相对较小（13.8%），prefix 信息可能不定够
  - CCM-Prefix 的 prefix 选择需要额外推理开销（对 13 个 α 各跑 K 步 rollout），虽不需训练但要多次 forward
  - 没有讨论 α bank 的最优设计或自适应 bank
  - 没有在非 PDE 任务上验证 shared/signed 机制是否通用

### 跟我们的关系

**对 0616-fex 课题**：

- **撞不撞车**：不直接撞。CCM 研究的是 PDE 场景下的 weight-space merging，核心贡献是揭示物理坐标在权重空间中的编码。0616-fex 如果涉及 feature extraction 或 representation learning in weight space，CCM 的 shared/signed decomposition 是一种全新的分析工具——但两者的研究对象和应用场景完全不同。

- **可用工具/方法**：
  1. **Shared/Signed decomposition**：$\Delta^{+}$ 和 $\Delta^{-}$ 的分解是通用方法论的——任何从同一初始化微调的两个 expert 都可以做这种残差分析，看两个 expert 的差异中有多少是共享的、多少是方向性的。这对 fex 分析多个 fine-tuned models 之间的关系极为有用。
  2. **Physical coordinate calibration**：CCM-Coord/Scale/Prefix 的三级信息条件框架（metadata → calibrated → prefix-based）可以迁移到其他领域，用来在不同信息条件下选择模型组合系数。
  3. **Wrong-sign negative control**：翻转方向验证因果性的实验设计可以直接复用——当声称某个 weight direction 有语义含义时，wrong-sign control 是最干净的验证。
  4. **α-sweep diagnostics**：沿一维坐标线扫描 loss 的诊断方法，揭示 checkpoint line 上的光滑性和最优坐标偏移

- **差异化**：0616-fex 如果关注 weight space 中的可解释特征/方向，CCM 提供了一种"物理坐标→权重方向"的校准范式，但与 fex 的差异在于 CCM 依赖外部物理坐标（viscosity, diffusivity 等），而 fex 可能更关注从权重本身无监督发现特征方向。

## Read by: 0616-fex (2026-06-21)

### 碰撞评估
**不撞车。** CCM 的核心问题是 "PDE fine-tuned experts 的权重更新是否编码物理坐标"，0616-fex 的核心问题（若关于 feature extraction in weight space）与之共享对 "权重空间方向" 的兴趣，但 CCM 的 setting 非常专门化（PDE + physical coordinate + same-anchor fine-tuning），与通用 feature extraction 不重叠。

### 可迁移工具
1. **Shared/Signed 分解**（最核心）：对任何两个 same-anchor experts 做 $\Delta^{+}/\Delta^{-}$ 分解，检查 shared 和 signed component 各自的贡献，可作为 fex 分析工具
2. **Wrong-sign control**：验证 weight direction 语义含义的标准实验范式
3. **α-sweep diagnostics + coordinate law**：诊断一维权重方向上的 loss landscape 光滑性和语义对齐
4. **三级信息条件框架**：CCM-Coord/Scale/Prefix 的设计可作为从不同信息条件下选择模型组合系数的模板

### 注意事项
- CCM 依赖外部标量物理坐标（非无监督），fex 如果要无监督发现方向，需要不同的方法论
- CCM 的理论 bound（Theorem 1）假设 $C^2$ 光滑性，实际使用中需验证
- 同组作者（Yuanyi Wang 等）同时在 model merging 方向大量产出，关注其后续工作可能对 fex 有趣
