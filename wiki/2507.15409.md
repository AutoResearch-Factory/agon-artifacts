# PDEformer-2: A Versatile Foundation Model for Two-Dimensional Partial Differential Equations (2507.15409)

- **Year**: 2025.07
- **Authors**: Zhanhong Ye, Zining Liu, Bingyang Wu, Hongjie Jiang, Leheng Chen (Peking University); Minyan Zhang, Xiang Huang, Qinghe Meng, Jingyuan Zou, Hongsheng Liu (Huawei); Bin Dong (Peking University, corresponding)
- **Venue**: arXiv preprint (Springer Nature journal format, sn-jnl class)
- **Keywords**: PDE foundation model, computational graph representation, graph Transformer, implicit neural representation, neural operator, zero-shot prediction, few-shot adaptation, inverse problems
- **One-liner**: PDEformer-2 用计算图编码 PDE 符号形式 + Graphormer + Poly-INR，在 40TB 多类型 PDE 数据上预训练后实现跨方程零样本预测和高效少样本适配，并展示逆问题求解能力。

## 精华提炼

### 管线/方法

**输入 → 输出完整 pipeline：**

1. **PDE 符号编码（计算图构建）**：将 PDE 和边界条件的符号表达式转化为有向无环图 (DAG)。节点类型包括：UF（未知场变量）、SC（标量系数）、CF（系数场）、VC（可变系数）、IC（初始条件）、SDF（有符号距离函数）、微分运算 (dt,dx,dy)、代数运算 (+、×、−、²、=0)、特殊函数 (sin, cos, exp10, log10, AT/任意变换)。边表示运算的操作数关系。

2. **数值信息嵌入**：标量系数经 MLP 编码（2 隐藏层、256 神经元）→ 768 维。函数/场经 CNN 编码（输入 128×128 单通道 → 3 层 Conv2d(32→128→768)、stride 4、kernel 4×4、ReLU）→ 4 个 branch node 的 768 维特征。域形状通过 SDF 函数编码并经 CNN 嵌入。

3. **图 Transformer 编码（Graphormer）**：12 层、d_e=768、FFN 隐层 1536、32 注意力头。注意力偏置包含最短路径距离项 b^+_φ(i,j) + b^-_φ(j,i) + d_ij（d_ij 用于断开节点间 mask）。输出为每个待解变量 u_j 的潜在向量 μ_j ∈ R^{L×d_e}（L=11）。

4. **INR 解码器（Adapted Poly-INR）**：11 隐藏层、每层 768 神经元（base）/ 256（fast）。输入时空坐标 (t,x,y)，每层由 μ_j^ℓ 经 hypernet 生成 scale/shift 调制。激活函数为 scaled sine (σ(x)=√2·sin(x))。输出为任意坐标点的解场值 û_j(t,x,y)。

5. **域与边界处理**：域形状以 SDF 表示（优于特征函数：信息更丰富、对拓扑变化敏感）。非周期边界条件通过计算图节点显式编码；周期边界默认无额外节点。法向导数用 ∂_n u = ∇SDF_Ω · ∇u 计算。

### 关键技术细节

- **预训练数据**：8 种 PDE 通用形式 → 随机系数生成具体 PDE → Dedalus V3（谱方法）和 FEniCSx（有限元）求解。总计 3.29M 训练样本、40 TB。
  - DCR（扩散-对流-反应）：745k 样本、4.85 TB（6 个子数据集：base/npX/npY/disk/sJ3/inhom）
  - Wave：605k、4.34 TB（5 个子数据集）
  - MV-DCR（多变量 DCR）：660k、12.24 TB（4 子集，2-4 变量）
  - DC-DCR（散度约束 DCR，含 INS）：400k、7.49 TB（2 子集：icA/icV）
  - MV-Wave：320k、6.20 TB（3 子集，2-4 变量）
  - DC-Wave：200k、2.62 TB（2 子集）
  - G-SWE（广义浅水方程）：100k、1.93 TB
  - Elasticity（稳态弹性）：255k、0.38 TB

- **预训练**：MindSpore、8 NPU、192 CPU 核、768 GB RAM。动态数据缓冲区（本地 ~50 文件 × 1k 样本，远程 40TB）。持续学习策略（10+ 次增量训练，每次 1-2 天）。每阶段 200-500 "epochs"，batch size 80（每设备 10）。Adam 无权重衰减，梯度裁剪 max norm=1。最大学习率 10^{-4}（早期）→ 6×10^{-5}（后期），cosine 衰减，10 epoch 线性 warmup。

- **训练损失**：Monte-Carlo nRMSE（每样本随机采 8192 个时空点），稳定化 ε=0.05。

- **模型规模**：PDEformer-2-base 82.65M 参数，PDEformer-2-fast 71.07M（INR 宽度从 768 降为 256，其他相同）。

- **预训练精度（nRMSE）**：base 总体 8.7%、fast 总体 10.3%。各类型范围：DCR 6.9%～Wave 6.5%～Elasticity 3.8%（最佳）～DC-DCR 17.0%（最差）。

- **Baseline 对比模型**：
  - **专用神经算子**：DeepONet (MLP/CNN 两变体)、U-Net、FNO (2D/3D)、Geo-FNO
  - **PDE 基础模型**：MPP-B (115.6M)、Poseidon-B (158M)
  - **传统求解器**：Dedalus（谱方法，CPU）

- **下游适配数据集**：
  - Sine-Gordon（分布内）：u_{tt}−aΔu−sin(u)=0，周期边界
  - INS-Tracer（轻微分布偏移）：INS + 示踪粒子，周期边界
  - INS-Pipe（轻微分布偏移）：管道 INS，顶底无滑移边界
  - Wave-Gauss（远距离迁移）：PDEgym 的 Gaussian 初始条件波方程
  - Wave-C-Sines（远距离迁移）：RIGNO 的圆盘域正弦叠加波方程

- **逆问题设置**：
  - 标量系数恢复：PSO (population 100, max gen 200)，25 组初始条件，128 空间点 × 平均 20 观测/点，1% 噪声
  - 系统辨识：PSO (swarm 150, max iter 150)，恢复 10 个系数含稀疏结构
  - 源场恢复：Adam (lr=0.1, 150 iter)，λ=0.01 L2 正则 + 梯度惩罚
  - 波速场恢复：Adam (lr=0.02, 20000 iter)，λ=0.001，9 种不同常数初始化择优

- **推理速度**（Sine-Gordon 全时空预测）：base 0.631s、fast 0.202s。相比 FNO2D 0.015s（最快）、Dedalus 2.398s（128 分辨率）。Encoder-Only（仅输出 INR 隐向量）仅 0.020s。

### 核心结果

| 实验 | 关键发现 |
|------|---------|
| Zero-shot 预训练 | base 总 nRMSE 8.7%，覆盖 8 种 PDE 类型、不同域形/边界/变量数/时间依赖 |
| 变量数外推 | 预训练最多 4 变量 → 5 变量 reasonable zero-shot（~14-18% nRMSE） |
| 对非物理模式的鲁棒性 | 20 个 Dedalus 与 PDEformer-2 差异最大样本中，10 个最极端的是 Dedalus 错误（PDEformer-2 更接近 FEniCSx 参考解） |
| 超越教师求解器 | 在 Dedalus 失败的 100 个样本上，PDEformer-2 能给出合理预测的个别案例（但不普遍） |
| 少样本适配 | Sine-Gordon（分布内）：zero-shot 已超所有 baseline；INS-Pipe：所有数据量下均领先；INS-Tracer：低数据领先，FNO 在足够数据后反超 |
| 适配速度 | INS-Pipe 80 样本上 PDEformer-2 仅需 ~10 秒收敛，其他模型慢一个量级以上 |
| 不完美 PDE 输入 | 等效形式→不完整→未知→错误 PDE：zero-shot 逐步退化，但足够微调数据可弥合差距 |
| 标量系数恢复 | 40 个 PDE 大部分系数成功恢复（对角线对齐良好），对不同噪声/观测点数/初始条件数鲁棒 |
| 系统辨识 | 10 个 case 中 29 个系数仅 6 个误判为非零（阈值 0.1），无需显式稀疏约束 |
| 源场恢复 | 40 个不同源场平均 nRMSE 0.159 |
| 波速场恢复 | Adam + 9 种初始化择优策略可克服局部极小值 |

**Surprising/反直觉发现**：
- PDEformer-2 在 Dedalus 产生非物理振荡的样本上比 Dedalus 更准确——模型归纳偏置胜过教师求解器（特定意义上）。
- 预训练数据集来自单一求解器（Dedalus 主导），但模型未继承其系统性伪影。
- 即使输入错误 PDE（Lorenz ODE），模型仍可通过足够微调数据适应——显式 PDE 输入并非必要条件。
- 5 变量外推（预训练未见 5 变量 PDE）表现合理，暗示模型学到了变量间交互的某种通用规则。

### 明确局限

**作者自述**：
1. 仅支持 2D PDE，3D 扩展"challenging task requiring further research"
2. 推理速度慢于网格化 baseline（全时空预测 base 0.631s vs FNO2D 0.015s）——归因于 mesh-free INR 设计 + 多 PDE 并发学习所需模型容量
3. 精度在充足数据下未必优于专用模型（如 Wave-Gauss、Wave-C-Sines 数据集上被 FNO/Geo-FNO 反超）
4. 预训练数据集分布范围有限，远离分布时泛化差
5. 不能处理更复杂域形、时变系数/域、积分-微分方程
6. 未在真实世界数据和任务上验证
7. 预训练数据集主要来自 Dedalus 单一求解器，建议多求解器合成数据集

**读出的额外局限**：
8. 所有 time-dependent PDE 限定在 t∈[0,1] 区间，无法处理长时间演化
9. 计算图表征依赖符号形式已知——在物理机制不完全清楚的真实场景有限制（尽管展示了部分鲁棒性）
10. 函数编码器要求 128×128 均匀网格输入，mesh-free 仅是输出端而非输入端
11. 预训练"epoch"不是标准 epoch（动态缓冲区只遍历本地 50k 样本），实际训练可能不够充分
12. 没有系统性的超参数搜索消融（hyperparameters selected based on "previous experience"）
13. 压力变量 p 在 DC-DCR/DC-Wave 数据集中被丢弃，可能丢失物理信息
14. 评估未使用独立验证集做 early stopping——直接在测试集上选最优，存在过拟合到测试集的风险
15. 未与最近的 PDE 基础模型（如 PROSE-PDE、Unisolver、ICON 系列）做直接实验对比——仅引用了它们

### 跟我们的关系

**对 0616-fex 课题**：

**不撞车**：PDEformer-2 的核心贡献是"用计算图编码 PDE 符号形式 + 大规模多类型 PDE 预训练 → 跨方程泛化"，这是一个完整的 PDE 基础模型系统。0616-fex 如果聚焦于 PDE 的**特征提取**（feature extraction for PDEs），则与 PDEformer-2 的方法存在正交性——PDEformer-2 的特征提取（标量编码器 + 函数编码器 → 图节点特征）是相对轻量的（MLP + 浅层 CNN），未做专门的表示学习设计。

**可用工具/方法**：
1. **计算图表示体系**：PDEformer-2 的 DAG 节点类型定义（UF/SC/CF/IC/SDF 等 18 种）可直接复用于 feature extraction pipeline，作为 PDE 输入的标准化编码。
2. **SDF 域表示**：用 SDF 将复杂域形状编码为可学习的函数输入，比特征函数/网格掩码更优，可集成到下游模型。
3. **函数扩展公式 f^ext(r) = f(argmin_{r'} d(r,r'))**：处理非均匀域上定义的函数到均匀网格的映射。
4. **Graphormer 注意力偏置设计**：最短路径距离 + 断开连接 mask，对 PDE 图结构的特征传播有参考价值。
5. **预训练数据集设计思路**：8 种通用形式 + 随机系数生成 → 多样化的具体 PDE，可作为数据增强模板。
6. **nRMSE 损失与 Monte-Carlo 采样训练**：对高维时空输出的 PDE 模型训练实用技巧。

**该引不该引**：该引——PDEformer-2 是当前最大的 2D PDE 基础模型系统，代表了"符号形式编码 + 大规模预训练"路线的 SOTA。如果 0616-fex 涉及 PDE 的表示学习，必须引用并与之对比/差异化。

**差异化**：
- PDEformer-2 的特征编码（scalar encoder MLP + function encoder CNN）是被设计成服务于下游解场预测的组件，而非独立的通用特征提取器。0616-fex 如果聚焦于如何从 PDE 中提取更丰富、更解耦、可迁移到其他任务（不只是求解）的特征表示，则是 PDEformer-2 未涉及的领域。
- PDEformer-2 的图节点特征维度固定为 768（等于 Graphormer 嵌入维度），没有做特征压缩或多尺度特征金字塔。0616-fex 可以探索更灵活的 PDE 特征层级结构。
- PDEformer-2 的预训练目标单一（解场预测的 nRMSE），缺少对比学习、掩码预测等多样化预训练目标。0616-fex 可探索更适合特征提取的预训练策略。

## Read by: 0616-fex (2026-06-24)

本次审读从 0616-fex（PDE 特征提取）视角出发。

**核心发现**：PDEformer-2 是一个"全栈"PDE 基础模型——从符号编码到求解预测的完整管线。其最直接可借鉴的部分是**计算图表征体系**和**SDF 域编码**，两者共同实现了 PDE 符号+几何信息的统一嵌入。但 PDEformer-2 的 feature encoder（scalar MLP + function CNN）是相对浅层的设计——MLP 仅 2 层、CNN 仅 3 层（无残差连接、无归一化层），且输出直接嵌入 Graphormer 节点特征空间，未形成独立的特征表示层。

**撞车评估**：不撞车。PDEformer-2 的目标是"求解 PDE"，特征编码是手段而非产出。0616-fex 如果聚焦于 PDE 的特征提取本身（作为可迁移表示输出给下游多种任务），则与 PDEformer-2 正交——后者未尝试将计算图编码产生的特征独立出来作为通用 PDE 表示。PDEformer-2 的 latent code μ_j 是解场特定的、且通过 INR decoder 直接约束到解空间，不适合作为通用 PDE 特征。

**可复用组件**：
1. DAG 节点类型定义（18 种）→ 可直接作为 PDE 特征提取的 token vocabulary
2. Graphormer + attention bias 设计 → 可作为 PDE 图结构特征提取 backbone
3. SDF 域编码 → 可作为几何特征提取的通用方法
4. 预训练数据生成方案（8 通用形式 + 随机系数）→ 可为特征学习提供多样化训练数据

**潜在改进方向（feature extraction lens）**：
- 在 Graphormer 输出端增加独立的 feature head，与 INR decoder 解耦
- 引入对比学习或掩码节点预测等自监督目标增强特征质量
- 设计层级化的特征金字塔（节点级→子图级→全图级）
- 探索跨 PDE 的特征迁移能力（不依赖解场监督）
