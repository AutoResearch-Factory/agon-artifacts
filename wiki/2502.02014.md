# Analytical Lyapunov Function Discovery: An RL-based Generative Approach (2502.02014)

- **Year**: 2025.02
- **Authors**: Haohan Zou, Jie Feng (UC San Diego), Hao Zhao (EPFL), Yuanyuan Shi (UC San Diego)
- **Venue**: arXiv preprint; source TeX uses an ICML 2025 accepted-style template
- **Keywords**: analytical Lyapunov function discovery, symbolic transformer, risk-seeking policy gradient, SHGO falsification, genetic programming, SMT verification, nonlinear control, power systems
- **One-liner**: 用 Transformer+RL 搜索可形式验证的解析 Lyapunov 函数。

## 精华提炼

### 管线/方法

1. 输入是自治非线性动力系统的解析 ODE：$\dot{x}=f(x)$，默认平衡点在原点。
2. 目标是找一个解析表达式 $V(x)$，满足 $V(0)=0$、$V(x)>0$、$L_fV(x)<0$，从而证明局部渐近稳定。
3. 方法不先构造大规模 Lyapunov 数据集，而是针对每一个给定系统单独训练。
4. 动力系统的每个 ODE 分量先被转成表达式树，再用 preorder traversal 变成 symbolic token 序列。
5. 常数采用两种编码：整数拆成 base-10 digits，实数用 4 位有效数字的科学计数法。
6. 系统动态 token 由 `SOS`/`EOS` 分隔后送入 transformer encoder。
7. encoder 同时编码两类信息：系统动力学 $f(x)$ 和已选 token 的 hierarchical tree state。
8. 模型把这两类表示拼接成 latent representation，再给 decoder 生成候选 Lyapunov 表达式。
9. decoder 是 vanilla transformer decoder 外加线性层，输出 symbolic library 上的 token probability。
10. 候选表达式按 preorder 自回归采样，采完后减去 $V(0)$，强制满足 $V(0)=0$。
11. 每个 epoch 采样 $Q=500$ 个候选表达式。
12. symbolic library 在实验中固定为 $\{+, -, \times, \sin, \cos, x_i\}$。
13. 候选表达式先经过 Genetic Programming 继续变异、交叉、选择，补足 transformer 在组合搜索上的不足。
14. GP 使用 DEAP 实现，mutation 和 crossover 概率都设为 0.5。
15. GP 产生的高 reward 表达式组成 elite set，大小为 $0.1Q$。
16. elite set 被当作专家示例，用 weighted cross-entropy 的 expert guidance loss 更新 decoder。
17. 候选验证不是直接在训练内调用 SMT，而是先用 SHGO 做 global-optimization-based numerical verification。
18. SHGO 分别找 $V$ 的全局最小点和 $-L_fV$ 的全局最小点，也就是最容易违反正定性和下降性的区域。
19. 作者的直觉是：无效候选的 violation 往往集中在这些 minimizer 的邻域。
20. 每个候选在两个 minimizer 邻域各采样 800 个点，再在全域额外采样 800 个点。
21. 被发现违反 Lyapunov 条件的点作为 counterexamples 加回训练集。
22. 训练 reward 使用经验 Lyapunov risk：$\max(0,L_fV(x))+\max(0,-V(x))$ 的样本平均。
23. 为避免 unbounded risk 给 risk-seeking policy gradient 带来偏差，reward 设为 $R=1/(1+\mathcal{L})$，范围在 $[0,1]$。
24. 缺变量或解析不完整的表达式 reward 直接设为 0。
25. policy gradient 采用 risk-seeking 目标，只优化 top quantile 的候选，而不是平均 reward。
26. 实验统一使用 $\alpha=0.1$，也就是聚焦 batch 中 reward 最高的一小部分候选。
27. 训练循环是：采样表达式 -> GP refinement -> SHGO falsification -> 计算 reward -> risk-seeking PG 更新 -> expert guidance 更新 -> 加入 counterexamples。
28. 一旦找到通过数值验证的候选，再用 dReal SMT 做最终形式验证。
29. 最终 SMT 设置为 tolerance error $\epsilon=e^{-3}$、precision $\delta=e^{-12}$。
30. 验证域排除 $\mathcal{B}_{\epsilon}(0)$，避免原点附近的数值问题。

### 关键技术细节

31. transformer embedding dimension 为 128。
32. attention head 数为 2。
33. system dynamics encoder 是 2 层 transformer encoder。
34. hierarchical tree state encoder 是 3 层 transformer encoder。
35. decoder 是 6 层 transformer decoder。
36. 每轮 sampling batch size 为 500。
37. SHGO 初始点数设为 2048，迭代 3 次。
38. 作者比较过 `[1024, 2048, 4096, 8196]` 个 SHGO starting points，2048 在效率上最好。
39. 所有实验在 AMD Ryzen Threadripper 2920X、RTX 2080Ti 11GB、RTX 2080 8GB 上完成。
40. 测试系统覆盖 2D 到 10D，既有 polynomial，也有 pendulum、quadrotor、power system 等 non-polynomial 或工程系统。
41. 低维基线包括 neural Lyapunov 方法 ANLC、FOSSIL 2.0。
42. 解析基线包括 Alfarano et al. 2024 的 pretrained symbolic transformer 和 SOSTOOLS/SOS。
43. neural baselines 依赖小网络以降低 verification cost，但在高维或三角动力学上容易失败。
44. SOS 在低维 polynomial global cases 很快，但局部稳定和 non-polynomial recasting 会引入大量约束。
45. 本文核心取舍是训练慢一些，但生成可解释、验证快、可被控制工程师读懂的解析公式。

### 核心结果

46. 2D polynomial system：68s，SMT verification 2ms，找到 $V=9x_1^2+2x_2^2$，成功率 100%。
47. 2D Van der Pol：126s，verification 1ms，找到 $V=x_1^2+x_2^2$，成功率 100%。
48. 2D pendulum：288s，verification 1ms，找到 $V=2(1-\cos x_1)+x_2^2$，成功率 100%。
49. 3D polynomial：112s，verification 1ms，找到 $V=9x_1^2+x_2^2+x_3^2$，成功率 100%。
50. 3D trig dynamics：157s，verification 1ms，找到 $V=1-\cos(x_1)^2+x_2^2+\sin(x_3)^2$，成功率 100%。
51. 4D lossy power system：3632s，verification 621s，找到含 $\omega_2-\sin(\delta_1)+\sin(\delta_2)$ 的局部 Lyapunov 函数，成功率 100%。
52. 6D polynomial：1667s，verification 7ms，找到 $\sum_i x_i^2$，成功率 100%。
53. 6D quadrotor：3218s，verification 1ms，找到 $\sum_i x_i^2$，成功率 80%，作者进一步判定全局渐近稳定。
54. 6D lossless power system：18094s，verification 2ms，找到能量函数结构，成功率 60%。
55. 9D synthetic system：27047s，verification 6.6s，找到包含 $\sin(x_7)^2$ 和 $-\cos(x_9)+1$ 的表达式，成功率 60%。
56. 10D polynomial：64223s，verification 2ms，找到 $\sum_{i=1}^{10}x_i^2$，成功率 60%。
57. 与 neural baselines 比：2D/3D 上 ANLC 和 FOSSIL 更快，但成功率较低；6D/8D 上二者成功率为 0。
58. 与 pretrained global transformer 比：其只在低维、全局稳定 benchmark 成功，对本文局部稳定 benchmark 全失败。
59. 与 SOS 比：SOS 在低维 polynomial 上秒级完成，但在 6D/8D/10D 局部 polynomial case 失败。
60. 对 non-polynomial case，SOS 的 recasting 需要领域专家手工引入辅助变量；3D trig 上求解超过 1 小时，而本文为 157s。
61. 最有新意的结果是 2-bus lossy power system：作者声称找到了此前文献中未知的局部解析 Lyapunov 函数。
62. 消融一显示 $\alpha=0.1$ 明显优于 $\alpha=0.5$ 和 vanilla policy gradient；无 GP 的 3D trig 成功率分别为 66.67%、33.33%、0%。
63. 消融二显示 SHGO 和 random sampling 在 6D polynomial 上都能 100% 成功，但 SHGO 找到的公式更结构化、最终验证更强。
64. 消融三显示 transformer-only 也能成功，但 6D polynomial 需要 168 epochs/8399s；加 GP 和 expert guidance 后降到 18 epochs/2467s。

### 明确局限

65. 该方法本质是 per-system training，不是一个一次训练后可直接泛化到所有系统的模型。
66. 高维搜索仍然很慢，10D polynomial 平均训练时间达到 64223s。
67. 成功率随维度下降，6D/9D/10D 中多个任务只有 60% 到 80%。
68. 当前 symbolic library 很小，主要是四则运算、sin、cos 和变量；复杂函数族、参数化常数、分式/指数等没有充分覆盖。
69. 作者承认尚未高效纳入物理常数，如重力、质量、惯量等，只能通过简化设定处理。
70. 最终保证依赖 dReal SMT 在指定局部域内验证，不等价于全局稳定证明。
71. 局部域和参数设置仍由作者指定，方法没有自动证明 region of attraction 的最大范围。
72. 反例生成的 SHGO 本身是数值方法，训练内的 verifier 不是严格形式证明。
73. 成功案例虽然多，但每个任务只有 3 到 5 个 random seeds，统计置信度有限。
74. power system 的“未知 Lyapunov 函数”claim 很强，需要后续独立控制理论审查和更一般 N-bus 验证。
75. 论文给出 GitHub 链接，但 wiki 读到的 TeX 没有说明完整 artifact、环境锁定、复现实验日志是否齐全。
76. 对真实机器人/电网闭环部署没有实机实验，主要是解析系统和仿真/验证 benchmark。
77. 如果目标是控制器综合，本文目前只发现 Lyapunov 函数；作者把 control Lyapunov function 扩展列为 future work。

## Read by: 0616-fex (2026-06-16)

从 `0616-fex` 的审读角度，这篇论文的最大价值是把“生成”和“检验”拆开：transformer/GP 只负责提出解析候选，SHGO 和 dReal 负责否决。这个结构对任何需要形式证书的 FEX 类课题都很有借鉴意义。

撞车风险取决于我们的具体 claim。如果我们要做“RL/生成模型自动发现解析 Lyapunov 函数”，本文已经是直接 prior；尤其是 symbolic transformer、risk-seeking PG、counterexample feedback、SMT final verification 这四件事不能再作为新贡献讲。如果我们的方向是发现别的证书，如 barrier certificates、control Lyapunov functions、RoA certificates，或者引入更强的独立验证/更大表达式库/参数化物理常数，那么仍有清晰差异化空间。

最值得复用的是 SHGO falsifier 的训练内角色：它不是最终证明器，而是把最可能违反正定性和下降性的点喂回 reward。这个机制比随机采样更适合高维稀疏 violation，也比在训练内频繁调用 SMT 更现实。另一个可复用点是 GP elite set 作为 decoder 的监督信号，能把进化搜索找到的局部好结构反哺给生成模型。

下一轮搜索应重点看 `2606.10045`，因为它是 2026 年直接替代路线：不用 transformer/RL，而把 Lyapunov discovery 写成 constrained symbolic regression，并用 branch-and-bound-and-check 解。若 `0616-fex` 需要 novelty，应该比较本文与 `2606.10045` 在约束建模、搜索完备性、表达式复杂度、局部/全局稳定、verification independence 上的差异。
