# WeatherReal: A Benchmark Based on In-Situ Observations for Evaluating Weather Models (2409.09371)

- **Year**: 2024.09
- **Authors**: Weixin Jin, Jonathan Weyn, Pengcheng Zhao, Haiyu Dong (通讯) 等, Microsoft Corporation
- **Venue**: arXiv preprint (未见正式会议/期刊版本)
- **Keywords**: in-situ observations, benchmark, weather model evaluation, quality control, ISD, Synoptic, ERA5 bias, extreme events, leaderboard
- **One-liner**: 用全球地面站实测替代 ERA5 评测 AI 天气模型, 开源 QC+评测框架, 揭示再分析对极端/局地天气的系统性低估

## 精华提炼

### 管线/方法

整体是"数据整编 → QC → 数据分析验证 → 模型评测 → 任务/榜单定义"五段式基准构建管线：

1. **数据源三路**: (a) WeatherReal-ISD — NCEI 的 Integrated Surface Database (>14,000 活跃站, 含 SYNOP/METAR 100+ 原始来源), 自研后处理+QC; (b) WeatherReal-Synoptic — Synoptic Data PBC 商业平台 (150,000+ 站, 含 CWOP 公民气象站), 用其自带 QC, 2023 年经 Time Series API 实时采集; (c) MSN Weather 用户报告 (温度区间/天空状况/是否降水), 论文中仅做 case study, 未随 v1 发布。
2. **变量抽取**: 12 个近地面变量 (t/td/sp/msl/ws/wd/云量 okta/1-3-6-12-24h 降水), 从 ISD 原始列抽取后重采样到逐小时 (整点最近记录优先, t 与 td 必须同记录)。
3. **站点合并**: 元数据相似度 (经纬距离 25km e-folding + 高程差 100m e-folding + 站名 Jaccard 相似度加权, 阈值 0.3, 权重沿用 Rennie et al. 2014) 筛候选 → 同时观测值 ≥70% 一致才合并; 按变量分别合并 (同站温度可合并而气压不合并的实例)。
4. **QC 系统** (仅施加于 ISD 子集): 值域检查 → 时序检查 → 跨变量检查 → 邻站检查 → flag 精修与集成, 输出 normal/suspect/erroneous 三级 flag, 仅删 erroneous, 共移除约 1% 数据。
5. **评测框架**: 格点预报双线性插值到站点, 与点预报 (MS-Point) 直接可比; 指标含 RMSE/MAE/Accuracy (阈内比例)/large-ERROR (超阈比例)/ETS (降水、云的阈值化) ; 输出 scorecard 与 leaderboard。

### 关键技术细节

QC 算法是本文最有复用价值的部分, 逐条记录：

- **Known records**: WMO 全球极值记录做绝对上下限; 站压 300–1100 hPa; 3h 降水上限 = 3× 1h 极值, 6h 上限 = 12h 极值; 逻辑限制 (降水/风速非负, 风向 0–360, 云量 0–8)。
- **Distributional gap (核心)**: 站点观测与 ERA5 差值序列做高斯拟合 — 均值取中位数, σ 取 1.48×MAD (抗离群); 概率密度 0.01 以外逐值向外搜索超过 MAD 倍数的"间隙", 间隙外全部 flag。既保留站点相对 ERA5 的独特局地信息, 又剔除真离群。
- **Cluster deviation**: 离群点占多数时高斯拟合失效 (如仪器长期故障的露点), 用 DBSCAN 对 obs-ERA5 差值聚类, 只保留离 ERA5 最近的簇 (即使它点数更少), 其结果仅用于修正 distributional gap 的中位数/MAD 计算。
- **Spike**: 3h 内变化超固定阈值 → 双向搜索 spike 终点 (反向跳变且回到起点值 ±一半初始跳变), 中间值全 flag; 渐变回落 (大日较差站常见) 不 flag。
- **Persistence**: 变长滑窗内标准差低于下限即 flag (如 24h 同值, 或每 3h 同值持续 48h); 降水/风的 0 值与云的 0/8 okta 豁免。
- **跨变量**: 过饱和 (td>t 双删); 风速风向一零一非零双删; 降水时段一致性 (长时段 < 短时段则双删; 长时段为 0 反推短时段为 0; 短时段可加总补长时段)。
- **邻站检查**: 300km + 500m 高差内候选, 记录重叠 ≥1/3, 按四象限每象限取最近 2 站, 总数 ≥3 才执行; 用 distributional gap 同款算法两两比较, 每时步 ≥3 邻站记录且 2/3 一致才定 flag。关键作用: 交叉验证其它算法的 flag (多站同现的"异常"是真实天气, 如台风气压谷)。
- **Flag 精修**: 低压极小值恢复 (日内谷值 + 前后单调 → 撤销 distributional gap/spike 的 erroneous); 温度日循环检查 (24h 正弦拟合, 振幅取日内极值, 相位锚定日出时间, 最低温限定日出前 3h–日出后 1h, 拟合好则降级 flag, 限 60°S–60°N)。
- **集成规则**: suspect 段两端皆 erroneous → 整段升级; 一月滑窗内 >50% suspect/erroneous → 窗内 suspect 全升级; 站内 >50% erroneous → 整站删除。
- **规模数字**: ISD 2023 温度 12,901 站 (8,655 逐时), Synoptic 温度 53,325 站 (42,038 逐时); Synoptic 日降水站近 20,000 (CWOP/COOP)。
- **评测配置**: Aurora-9km (IFS 分析场初始化, 6h 步长, 2023-01-02~12-19 每日 00UTC), ECMWF IFS 运行版, GFS (0.25°), MS-Point (微软消费级点预报, 基于 12UTC 旧一轮 IFS, 有 8h 延迟劣势); accuracy/large-error 阈值: 温度 1.7/5.6 K, 风速 1/3 m/s, msl 2/5 hPa, 云 2/5 okta。

### 核心结果

- **ERA5 vs 站点系统性差异**: t/td RMSE 多在 2°C 内但高海拔区显著更大; sp 的 RMSE 因 ERA5 地形与真实站点高程差而比 msl 大一个量级 (高程订正可降一个量级); ERA5 日较差 (DTR) 在全球 80% 站点偏低, 西北美/南欧最明显; ERA5 高估弱风 (<4 m/s) 频率、低估强风 (>12 m/s) 频率 (美国中西部尤甚)。
- **极端事件 case**: 2023-07 南欧热浪, 希腊 Skyros 站实测最高 44.0°C, ERA5 同点仅 31.6°C, 低估 12.4°C, 且 ERA5 日较差 <3°C vs 实测 >10°C; 台风 Talim 登陆湛江, 站点 msl 975.8 hPa, ERA5 显著低估中心低压强度; Hurricane Idalia 降水用 MRMS QPE Pass2 对比 (ERA5 降水偏差太大直接弃用), 站点雨量 166.9 mm/6h 与 MRMS 一致。
- **模型排名 (medium-range 榜单, vs WeatherReal-ISD 2023)**: t2m RMSE — MS-Point 2.258 K < Aurora-9km 2.417 < ECMWF 2.766 < GFS 3.168; ws10 — MS-Point 1.753 m/s 最优; msl — Aurora 2.939 hPa 最优; 云 — MS-Point 2.723 okta; 降水 ETS 仅 ECMWF 有值 0.248。
- **反直觉发现 1**: Aurora vs ECMWF 的相对排名与 ERA5 上评测结论基本一致 — "总体预报精度对评测真值不敏感", 但**极端场景例外**: 南欧热浪 12UTC (日最高温时刻) lead ≤2 天时 Aurora 对 ECMWF 的优势几乎消失, 而以站点观测训练的 MS-Point 依然显著领先 → 再分析训练的模型学不到局地极端 (午后辐射增温) 特征。
- **反直觉发现 2**: 数据驱动模型因回归损失 + 多步微调而输出"偏 ensemble-mean 的模糊预报", 在 RMSE 类指标上随 lead time 反而相对占优 — 榜单指标选择本身会奖励模糊化。
- **Synoptic 复评**: 结论与 ISD 基本一致但误差整体更高、噪声更大 (风速日循环噪声显著) — 归因于站点选址/传感器质量更杂 + 模型在稀疏同化区泛化更差; 说明"换一套更密但更脏的真值"不翻转结论但会放大误差绝对值。
- **任务定义** (4 个, 附打分方法): medium-range (00/12UTC 起报, 6h 步长至 168h, RMSE/ETS 头指标)、short-range (4 起报/日, 1h 步长至 72h)、nowcasting (逐小时起报, 1h 步长至 24h)、S2S (周一/四 00UTC, 日/周平均, 建议概率预报用 CRPS, 头指标 week3-4/5-6)。

### 明确局限

- 作者自述: 站点分布不均 (非洲/南美稀疏); QC 纯统计方法仍有漏杀/误杀; 仅发布 2023 一年 (训练用需更长记录); 用户报告未发布; 风/云/降水仅做了基础 QC (t/td/sp/msl 才是完整 QC)。
- 读出来的局限: **无对流尺度内容** — 变量止于近地面标量, 无雷达/无环境廓线/无对流指数/无事件标签, nowcasting 任务虽列出但没有任何对流基准实现; 降水评测在自家榜单上只有 ECMWF 一行有数 (数据驱动模型普遍不输出可比降水); Synoptic 子集依赖商业 API, 复现受限; MS-Point 是自家产品且在自家榜单夺冠, 虽披露了延迟劣势但仍有利益关联; leaderboard 是"provisional" (各模型起报时刻覆盖不全); 无不确定性量化 (观测误差没有随数据发布)。
- 无真机/无标准 benchmark 之辨: 本文自己就是在立 benchmark, GitHub (microsoft/WeatherReal-Benchmark) 公开 ISD-2023 数据 + QC 代码 + 评测代码, 这一点执行到位。

### 跟我们的关系

对 0713-scs-benchmark (强对流环境–雷达多模态基准) 课题：

- **不撞车**: WeatherReal 是"全球近地面标量 × 通用天气预报评测"基准; 我们是"风暴事件为中心 × 环境场长时窗 (HRRR -15d/+5d) + 雷达 (MRMS) + 事件标签 (Storm Events)"的 SCS 专用数据产品。变量、结构、任务、空间组织 (站点 vs 事件包) 全部不同。它反而是我们 Related Work 里"现有基准没把环境场当一等公民"论点的又一例证 — 它连对流环境变量都没有。
- **必引**: 作为"观测真值取代再分析做评测"路线的代表作, 且其 ERA5 极端低估证据 (热浪 -12.4°C、DTR 全球偏低、强风低估) 直接支撑我们"SCS 研究必须用高分辨环境场+实测"的动机。
- **可复用方法** (对我们价值最大的部分): ① 三级 flag (normal/suspect/erroneous) + 算法间交叉验证/降级的 QC 集成架构, 可平移到我们的 Storm Events 标签噪声与 MRMS 异常值处理; ② distributional gap (中位数+1.48×MAD 高斯 + 间隙搜索) 和邻站 2/3 多数决, 可用于我们事件包内站点/格点一致性检查; ③ "任务定义 + headline metric + provisional leaderboard"的发布结构是 NeurIPS D&B 式打包模板; ④ 格点插值到点 + 点/格预报同台评测的框架设计。
- **警示**: 它的 nowcasting 任务停留在纸面 (无对流实现), 降水列几乎全空 — 说明"任务定义了但没有 baseline 能跑"会显著削弱榜单说服力; 我们的 canonical tasks 必须每个都配至少一个可跑 baseline。
- **反引文里的真正竞品**: 引用它的 Extreme Weather Bench (2605.01126)、RealBench (2605.24945) 才是离我们更近的"极端天气评测"工作, 需优先精读排撞。

## Read by: 0713-scs-benchmark (2026-07-14)

**撞车评估**: 无撞车。WeatherReal 做的是全球近地面标量观测的通用预报评测基准; 我们做的是风暴事件为中心的多模态 (HRRR 环境场长时窗 + MRMS 雷达 + Storm Events 标签) SCS 基准。它没有雷达、没有环境廓线/对流指数、没有事件标签、没有以事件为单位的数据组织; 其 nowcasting 任务只有定义没有实现。二者互补: 它管"预报对不对", 我们管"对流环境-事件链条可不可研究"。

**动机弹药**: 其 ERA5 偏差证据链 (热浪最高温低估 12.4°C、全球 80% 站点 DTR 偏低、强风频率低估、ERA5 降水差到直接弃用改用 MRMS) 是我们"SCS 必须用 HRRR 级环境场 + 实测雷达, 不能用再分析凑合"的现成论据, 引用即可, 无需自证。

**方法搬运清单**: ① QC 三级 flag + 跨算法交叉验证/升降级的集成架构 → 用于 Storm Events 标签质量标记与 MRMS 异常剔除; ② distributional gap (median + 1.48×MAD + gap search) 与 DBSCAN cluster deviation → 用于我们事件包的环境场-观测一致性审计; ③ 邻站四象限 2/3 多数决 → 事件聚类区域内站点互验; ④ "任务 + headline metric + leaderboard + scorecard"发布模板与"每任务必须有可跑 baseline"的反面教训 (其降水/nowcasting 列近乎全空)。

**差异化表述建议**: Related Work 中将其归入"观测真值评测基准"谱系 (WeatherBench2 → WeatherReal → EWB/RealBench), 然后指出该谱系全部缺失"环境场时空演变作为一等公民"与"事件包组织", 由此引出我们的贡献。
