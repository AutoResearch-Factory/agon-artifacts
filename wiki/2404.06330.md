# Generative Pre-Trained Transformer for Symbolic Regression Base In-Context Reinforcement Learning (2404.06330)

- **Year**: 2024.04
- **Authors**: Yanjie Li, Weijun Li (corresponding), Lina Yu, MinWu, JingyiLiu, WenqiangLi, MeilanHao, SuWei, YusongDeng — ANNLAB, Institute of Semiconductors, Chinese Academy of Sciences
- **Venue**: arXiv (NeurIPS 2024 format, likely submitted but status unclear)
- **Keywords**: symbolic regression, algorithmic distillation, in-context reinforcement learning, SetTransformer, FormulaGPT, pre-training, DSR, DSO
- **One-liner**: 将基于强化学习的符号回归算法（DSR/DSO）的搜索历史蒸馏进 Transformer，使模型在新数据上能自回归地生成RL搜索过程并自动在上下文中更新策略。

## 精华提炼

### 管线/方法
完整的 FormulaGPT 管线分为三个阶段：训练数据采集 → 模型训练 → 推理。

**训练数据采集**：
1. 随机生成表达式骨架（二元树的前序遍历形式），使用符号集合 [+, -, *, /, sin, cos, log, sqrt, C, x_1, ..., x_n]
2. 通过 Arity(s) 函数控制生成（binary op=2, unary op=1, variable/constant=0），count 计数直到为 0 获得完整表达式
3. 对生成约束：(1) 三角函数不可嵌套；(2) log/sqrt 内不能取负值
4. 在 [-10,10] 采样 X 并计算对应 y
5. 将 [X, y] 送入 DSR 和 DSO 搜索，每轮选 R² 最高的表达式，拼接其前序遍历序列和 R² 值（四舍五入到两位小数），R² < 0 设为 0.00
6. 当 R² > 0.99 或序列长度达 1024 token 时停止；若未达 0.99 则丢弃该对数据
7. 额外提取 "shortcut" 版本：从训练历史中移除 R² 振荡部分，仅保留单调上升路径
8. 总计收集 1.5M 对 [X, y] + 搜索过程序列

**模型架构**：
- Encoder: SetTransformer (Lee et al. 2019, PMLR v97)，利用 Induced Set Attention Blocks (ISABs) 处理数据的排列不变性
  - 输入 [X, y] ∈ R^{n×d} → 仿射层升维到 h_n ∈ R^{d_h} → 多层 ISAB（可学习向量作 query，数据作 key/value 交叉注意力）→ dropout → 最终交叉注意力固定输出维度
- Decoder: 标准 Transformer decoder (Vaswani et al. 2017)
  - 带 causal mask 的多头自注意力 → 交叉注意力（与 SetTransformer 提取的数据特征 z 交互）→ FFN
  - 16 层 decoder，16 注意力头，embedding dim=512
- 训练目标：交叉熵损失（预测序列 vs 真实 DSR/DSO 搜索序列）

**推理过程**：
1. 新数据 [X, y] → SetTransformer 提取特征 z
2. Decoder 从 start token 开始自回归生成符号序列
3. 每生成一个完整表达式 → BFGS 优化常数占位符 C → 计算 R² → 将 R² 拼入序列继续生成
4. 直到 R² > 0.99 或达到最大序列长度 2048
5. Beam width = 16

### 关键技术细节
- **训练数据规模**：1.5M 对（DSR + DSO 搜索历史）。消融实验显示：10k→50k 提升有限，50k→500k 大幅提升，500k→1.5M 稳定提升但增速放缓
- **R² 离散化**：100 个桶 [0.00, 0.01, ..., 0.99, 1.00]，将连续 R² 值四舍五入到两位小数转为字符串 token
- **Shortcut 机制**：从振荡上升的 R² 序列中提取"一路向上"的捷径路径。消融实验证明：Shortcut 版 FormulaGPT 平均推理过程经历 12.5 个表达式 vs 无 Shortcut 版 16.8 个 vs DSO 19.6 个 vs SPL 23.2 个
- **常数优化**：BFGS 算法在推理时对每个完整表达式优化 C 占位符处的常数值
- **训练超参数**：batchsize=128, epochs=1000, lr=0.0001, momentum=0.99, dropout=0, 16-bit 训练
- **SetTransformer 超参数**：num_inds=50, num_heads=8, n_l_enc=5, dim_hidden=512, dec_layers=5, dec_pf_dim=512, output_dim=60

- **噪声鲁棒性**：FormulaGPT 弱于 DSO 但强于 SNIP 和 NeSymReS，与 SPL 持平。测试 10 个噪声水平 [0.00-0.10]
- **Versatility（泛化区间）**：训练数据在 [-10,10] 采样。在 [-2,2] 到 [-20,20] 的 10 个区间测试，FormulaGPT 在 < 10 的区间几乎不受影响，在 > 10 的区间影响远小于 SNIP/NeSymReS

### 明确局限
- **噪声鲁棒性仍有差距**：Fig 2a 显示 FormulaGPT 在噪声下明显弱于 DSO，作者承认"有较大提升空间"
- **复杂表达式表现不佳**：在 Korns 和 Vladislavleva 数据集上弱于 DSO，作者归因于"训练数据有限"
- **无真实世界数据测试**：所有数据为合成数据，虽有噪声实验但仍是人工加噪，未在真实物理/工程数据上验证
- **训练数据依赖 DSR/DSO 性能天花板**：蒸馏的目标策略（DSR/DSO）本身有性能上限，FormulaGPT 无法超越 teacher 算法的能力边界（除非 shortcut 机制起到数据增强效果）
- **无代码/模型开源**：仅提供超参数表，无可复现的代码或预训练权重
- **无计算资源报告**：未披露训练使用的 GPU 数量、训练时间、推理延迟的绝对数值
- **Versatility 测试仅改采样区间**：只测试了 X 采样范围变化，未测试变量维度变化、数据类型变化、或分布偏移（如从均匀分布变为高斯分布）
- **仅与 4 个 baseline 比较**：缺少与更近期工作的比较，如 TPSR (2024)、SR-GPT (2024)、MMSR (2024) 等

对 **0616-fex** 课题的关联分析：

- **撞车评估**：FormulaGPT 专注于符号回归（从数据中发现显式数学公式），与 fex（特征提取/特征工程）有交叉但不直接撞车。如果 fex 涉及从观测数据中自动发现可解释的特征变换（如 sin(x1*x2), log(x1+x2) 等），FormulaGPT 的方法可视为一种"符号化特征发现"方案。但 FormulaGPT 的目标是拟合函数而非提取下游任务特征。

  4. **in-context policy update**：让 Transformer 在上下文中自动改进策略，可用于 fex 的自适应特征选择

### 审读视角：特征提取/自动化特征工程相关性

**核心判断**：FormulaGPT 与 fex 课题**不直接撞车**，但提供了有价值的方法论借鉴。FormulaGPT 的目标是发现精确拟合数据的显式数学公式（符号回归），而 fex 通常关心的是提取对下游预测任务有用的特征表示。两者的输出性质不同：FormulaGPT 输出 f(x) ≈ y 的闭式表达式，fex 输出变换后的特征向量。

**可用技术迁移**：
1. **Algorithmic Distillation 框架**：如果 fex 中存在搜索/优化驱动的特征选择过程（如自动特征工程中的 transformation graph search），可以将搜索轨迹序列化训练 Transformer，实现快速 in-context 特征发现。这是本文最可迁移的思想。
2. **Shortcut 数据增强**：从噪声优化的历史轨迹中提取单调改善路径，该技术适用于任何基于迭代优化的特征选择流程。
3. **SetTransformer + 条件生成**：用 SetTransformer 编码原始数据特征，用 decoder 条件生成搜索/优化序列的架构模式可复用于 fex 的特征变换生成。

**不应引用的理由**：如果 fex 不涉及符号搜索或迭代优化轨迹建模，本文的引用价值有限。FormulaGPT 的实验完全在合成符号回归 benchmark 上，与 fex 典型的下游任务评估（分类/回归精度提升）有本质区别。

**建议**：若 fex 方向涉及"从数据中自动发现可解释特征变换"或"用 RL/搜索做特征工程"，应将此文列为方法论参考（特别是 Algorithm Distillation 范式），但明确与本文在目标和评估上的差异。
