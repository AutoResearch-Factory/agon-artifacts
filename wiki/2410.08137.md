# Generalized Fixed-Depth Prefix and Postfix Symbolic Regression Grammars (2410.08137)

- **Year**: 2024.10
- **Authors**: Edward Finkelstein (SDSU)
- **Venue**: arXiv (Springer LNCS format, possibly submitted to a CS proceedings venue)
- **Keywords**: symbolic regression, prefix notation, postfix notation, fixed-depth grammar, Monte Carlo Tree Search, Genetic Programming, Particle Swarm Optimization, Simulated Annealing, Random Search, expression representation
- **One-liner**: 提出无错（faultless）固定深度前缀和后缀文法用于符号回归，在统一 C++/Eigen 框架下对比 5 种搜索算法在两种文法下的表现，发现 ground-truth 表达式树每层平均节点数是决定 prefix vs postfix 优劣的最强指标。

### 管线/方法

整体管线：输入 feature 向量和标签 → 固定深度 N → 根据 prefix 或 postfix 文法逐 token 生成表达式 → 使用 Levenberg-Marquardt (5 iterations) 优化常数 token → 计算 score = 1/(1+MSE/N_dat) → 返回最优表达式。

文法部分是该论文的核心贡献：
1. **Prefix Grammar**：起始步若 N=0 则仅允许 leaf nodes，否则仅允许 operators (unary + binary)。后续每步：unary/binary 仅在添加后深度 ≤ N 时允许；leaf node 仅当 `num_leaves == num_binary + 1` 且添加后最小深度 < N 且 `num_leaves == num_binary` 同时为 false 时才允许。需要额外保证表达式在达到深度 N 之前不会提前完成。
2. **Postfix Grammar**：起始步仅允许 leaf nodes。后续每步：unary 允许当 `num_leaves ≥ 1` 且添加后深度 ≤ N；binary 允许当 `num_binary ≠ num_leaves - 1`；leaf node 允许当添加后最小深度 ≤ N。postfix 文法更简单，因为不需要像 prefix 那样额外防止表达式提前完成。
3. 深度和完整性通过 stack-based 算法计算（prefix 用 trincot 的 getPNDepth 算法，postfix 用 trincot 的 getRPNDepth 算法，均来自 Stack Overflow，论文内添加了缓存优化）。

五种搜索算法在固定深度文法框架下的实现：
- **Random Search**：随机选择合法 token 直到完成 depth-N 表达式。
- **MCTS**：采用 UCT 公式选择动作，c 初始化为 √2。每 N_iter 步（文中设为 50000）若 best score 未改进则 c += √2（增加探索），否则 c = √2（专注利用）。Q(s,a) 更新为 max(Q(s,a), score)。
- **PSO**：每个 particle 位置表示合法 token 集合中的索引（position 模 N_allowed_tokens 取整）。速度更新使用标准 PSO 公式（惯性 0.721，phi1=2.8，phi2=1.3），c 项初始为 0，若 best score 在 N_iter 步内未改进则 c = U(-m, m)，m 每 N_iter 步递增 1。
- **GP**：初始种群 2000 个随机个体。每代通过 crossover (p=0.2) 和 mutation (p=0.8) 生成约 2000 新个体，选择最优 2000 进入下一代。mutation/crossover 直接在 expression list 上进行，不创建中间树结构。寻找 depth-n 子表达式通过计算每个 token 的 grasp bound（前/后缀分别用右/左 grasp bound）完成。mutation 用随机生成的 depth-n 子表达式替换原表达式中的 depth-n 子表达式；crossover 在两个体间交换 depth-n 子表达式。这些操作不改变表达式的固定深度 N。
- **SA**：单个体，每步做类似 mutation 的扰动（但 depth 范围为 [0,N] 而非 [0,N-1]）。接受概率 P = e^(Δ/T)。温度 T_min=0.012, T_max=0.1, T_init=T_max。温度更新规则：T = rT，r = (T_min/T_max)^(1/(i+1))。若 N_iter 步未改进则 T=min(10T, T_max)，否则 T=max(T/10, T_min)。

### 关键技术细节

- **框架**：全部实现在 C++/Eigen 模板库中，单一文件 `PrefixPostfixSR.cpp`
- **常数优化**：Levenberg-Marquadt 算法，5 次迭代
- **评分函数**：Score = 1/(1 + ||Y_hat - Y||^2 / N_dat)，范围 [0,1]
- **基准测试**：10 个 ground-truth 表达式
  - Hemberg (2008) 5 个：表达式深度 4-10，2 个输入变量 (x, y)，采样范围 [-3,3]，20 个数据点。操作符：+, -, *, /, ^, const
  - AI Feynman 5 个：表达式深度 4-8，5-9 个输入变量，采样范围 [1,5]，10^5 个数据点。操作符：sin, sqrt, cos, +, -, *, /, ^, const
- **实验设置**：每个配置 50 次 2 分钟运行，每 6 秒采样一次 MSE
- **硬件**：MacBook Pro M1, ~16 GB RAM
- **N_iter**：统一设为 50,000（经显著预调优）
- **GP 参数**：种群 2000，crossover prob 0.2，mutation prob 0.8，持续 N_gen 代（文中未明确指定代数，仅说明过程重复 N_gen 次）
- **MCTS 参数**：c 初始 √2，UCT 公式，Q 值更新为 max
- **PSO 参数**：惯性 0.721，phi1=2.8（全局），phi2=1.3（局部），r_g, r_p ~ U(0,1)
- **SA 参数**：T_min=0.012, T_max=0.1，使用 4 字节浮点精度故选择了非标准 T_min

### 核心结果

1. **Postfix 文法比 Prefix 文法更简单**：因为 postfix 不需要额外的"防止提前完成"逻辑。

2. **决策树揭示了最关键的影响因素**（图 PrefixPostfixDecisionTree，70% 准确率）：
   - 最强指标：ground-truth 表达式树的**每层平均节点数**
   - 次要指标：搜索算法类型
   - 再次：ground-truth 表达式深度
   - 最弱：数据维度

3. **不同算法下 prefix vs postfix 表现各异**：
   - 从 Hemberg Benchmark 结果图可见，各种算法在 prefix/postfix 下的表现没有一致的赢家，取决于具体表达式
   - GP 和 Random Search 在多个 benchmark 上表现相对稳定
   - MCTS 在某些表达式中表现突出但方差较大

4. **令人惊讶的结果**：预期中 postfix 会在高复杂度表达式中表现更好（基于 Hemberg 2008 的结论），但本文发现表达式树的**形状**（每层节点分布）比绝对深度更能预测 prefix/postfix 的相对性能。

5. **Benchmark 规模**：10 个表达式 x 2 种文法 x 5 种算法 = 100 种配置，每种 50 次运行 = 5000 次独立实验。

### 明确局限

1. **作者明确说的局限**：
   - 计算资源有限，未进行大规模超参数调优
   - 仅测试了 10 个 ground-truth 表达式，样本量小
   - 仅比较了 prefix 和 postfix，未包含 acyclic graph 等表示
   - 未来工作可在更大语料库和超参数搜索上扩展

2. **审读发现的局限**：
   - **无真机实验**：纯软件 benchmark，没有在任何真实物理系统或机器人上验证
   - **无与 SOTA 的对比**：仅内部比较 5 种算法在 prefix/postfix 下的表现，没有与 PySR、Operon、uDSR 等当代 SOTA SR 框架做直接对比
   - **单作者论文**：没有合作者交叉验证，可能是课程项目或独立研究
   - **Stack Overflow 来源**：核心深度计算算法直接引用 Stack Overflow 答案（虽经缓存优化），非原创算法
   - **仅 2 分钟运行时间**：对于复杂表达式（如 Hemberg 5, depth 10），2 分钟可能不足以找到真实解
   - **MCTS 实现较简单**：没有 neural network policy/value estimator，仅使用原始 UCT 公式
   - **GP 操作未改变深度**：mutation 和 crossover 保持固定深度 N，这可能限制了 GP 的表达能力
   - **没有 ablation 研究**：例如没有测试不同 N_iter 值的影响，没有验证常数优化（Levenberg-Marquadt 迭代次数）的敏感性
