# Vertical Symbolic Regression via Deep Policy Gradient (2402.00254)

- **Year**: 2024.02
- **Authors**: Nan Jiang, Md Nasim, Yexiang Xue (Purdue University, Computer Science)
- **Venue**: arXiv preprint (N.b.: this is a follow-up to their PKDD 2023 VSR-GP paper 2312.11955)
- **Keywords**: symbolic regression, vertical discovery path, deep policy gradient, REINFORCE, control variable experiment, context-free grammar, multi-variable equation discovery, RNN-based expression sampling
- **One-liner**: Proposes VSR-DPG: a vertical symbolic regression method where expressions are built by a deep RNN sequentially sampling context-free grammar rules, trained via policy gradient, scaling to 50+ variables via control variable experiments.

## 精华提炼

### 管线/方法

VSR-DPG follows a "vertical discovery path" -- discovering equations by progressively adding one free variable at a time through control variable experiments:

1. **Initialization**: Start with n total variables. In round 1, control n-1 variables (hold them constant) and only allow 1 variable to vary. The RNN searches for the best reduced-form equation matching the data under this controlled setting.

2. **Expression Sampling via RNN**: At each round, a deep RNN (3-layer LSTM, hidden dim 512) takes the start symbol as initial input. At each timestep t, it outputs a categorical distribution over grammar rules (vocabulary covering variables, constants, math operators). A rule is sampled from this distribution, fed back as input for the next step. After L steps (max 20), the sequence of grammar rules is converted into a symbolic expression following context-free grammar rules of the form `A→(A+A)`, `A→(A-A)`, `A→A×A`, `A→A÷A`.

3. **Fitness Evaluation**: Each sampled expression's constants are fitted using BFGS optimizer on a batch of 1024 data points. The reward is `1/(1+NMSE)`. The expression with the best fitness across all sampled expressions is the round's prediction.

4. **Policy Gradient Training**: Using the REINFORCE algorithm with a baseline (average reward of the sampled batch). Sampling N=1024 sequences per epoch, 30 epochs per round. Adam optimizer with learning rate 0.009.

5. **Vertical Expansion**: After each round, constants in the best expression are classified as "summary" (high variance across multiple control variable trials → they depend on controlled variables) or "standalone" (low variance → they stay). Summary constants are replaced with placeholder symbol "A" to form the start symbol of the next round. In the next round, one more variable is freed; the RNN's output vocabulary is restricted to grammar rules involving only the newly freed variable.

6. **Termination**: After n rounds (all variables freed), the expression with the best fitness score across all rounds is returned.

### 关键技术细节

- **Grammar Representation (key innovation)**: Unlike DSR which predicts pre-order traversal of expression trees, VSR-DPG predicts sequences of production rules in an extended CFG. This allows the start symbol to be any partial expression (e.g., `x1×A−A`), ensuring the RNN only samples expressions that are reducible to previously discovered knowledge. This avoids the gradient propagation and engineering difficulties of direct tree-based VSR+DL integration.

- **RNN Architecture**: 3-layer LSTM, 512-dim input embedding, 512-dim hidden state. Input/output vocabulary size depends on the round's free variables. Linear layer + softmax over vocabulary for categorical distribution.

- **Data Oracle Assumption**: The method assumes access to a DataOracle that generates data with specified variables controlled. This is described as the cost of "conducting control variable experiments in the real world." When such an oracle is expensive, they cite prior work (Keren et al., 2023) using DNNs to learn a data generator.

- **Constant Classification Threshold**: A constant is classified as "standalone" if the empirical variance of its fitted values across K trials is below threshold ε'. Otherwise it is "summary" and gets expanded in the next round. If the best-predicted expression doesn't consistently achieve close-to-zero NMSE, all constants are treated as summary.

- **Multi-processing**: Open constants in expressions are fitted in parallel using 8 CPU cores.

- **Implementation Stack**: TensorFlow for RNN, Sympy for expression manipulation, Scipy (BFGS) for constant optimization. Code at github.com/jiangnanhugo/VSR-DPG.

### 明确局限

1. **Data Oracle Assumption is Strong**: The method requires the ability to query data with arbitrary variables controlled -- essentially, the ability to design and run experiments interactively. This is NOT a passive regression from a fixed pre-collected dataset. The authors acknowledge this but treat it as the "vertical discovery cost."

2. **No Real-World Physical Experiment Validation**: All experiments use synthetically generated data from known ground-truth equations. No application to real experimental data where noise structure, measurement error, and unknown variable interactions exist.

3. **Limited Operator Set**: Operators are restricted to {+,-,×,sin,cos,const}. No division (except in the paper's description), no exponentiation, no log, no nested functions. Scaling to richer operator libraries is unvalidated.

4. **Sequential Dependency Chain**: If an early round produces a wrong reduced-form equation, the error propagates through all subsequent rounds. The "summary vs standalone" constant classification uses heuristic variance thresholds (ε, ε') that are not learned.

5. **No Comparison with FEX/SNIP**: The paper does not compare against FEX (Finite Expression method), which is another RL-based symbolic regression approach from the Yang group. Also omits comparison with transformer-based methods beyond E2ETransformer (e.g., no comparison with NeSymReS or other pre-trained encoder-decoder methods).

6. **Computational Cost**: Each round runs 30 epochs, each sampling 1024 expressions and fitting constants via BFGS. With n variables this requires n×30×1024 BFGS optimizations. Multi-processing (8 CPUs) mitigates but doesn't eliminate this. For the n=50 setting, this is 50 rounds of training.

7. **No Theoretical Guarantees**: The grammar rule representation and vertical expansion reduce hypothesis space exponentially, but there is no formal proof of correctness or complexity bounds. The reduction relies on heuristics for constant classification.

8. **Restricted to Continuous Variables Only**: All variables are real-valued. No categorical or discrete inputs.

- **R1 Evidence (4)**: Claims are well-supported by tables with specific NMSE numbers across 8 datasets, plus exact recovery and large-scale extension experiments. Table data is precise. However, no confidence intervals or statistical tests are reported -- only median values.
- **R2 Independence (2)**: All experiments run by the authors. No independent verification. The test set is separately generated but there is no held-out benchmark or external evaluation. The "summary vs standalone" constant classification is decided by the same algorithm with heuristic thresholds, not an independent module.
- **R3 Honesty (3)**: The authors disclose the strong data oracle assumption and limitations of direct VSR+DL integration. They include cases where VSR-DPG doesn't win (E2ETransformer edges it on some configs). However, they omit ablation studies on key hyperparameters (embedding size, number of LSTM layers, number of epochs per round, baseline choice in REINFORCE) and don't show NMSE variance/std across runs.
- **R4 Reproducibility (4)**: Code is open-sourced on GitHub. Hyperparameters are listed in detail (Appendix). Datasets are provided in the repo. Implementation stack (TensorFlow, Sympy, Scipy) is specified. However, the DataOracle for control variable experiments is synthetic -- reproducing the exact same controlled datasets requires running their generation scripts.
- **R5 Substantiation (4)**: This is real algorithmic research, not LLM-generated fluff. The grammar rule representation is a genuine theoretical contribution that elegantly solves the VSR+DL integration problem. The experimental results are internally consistent and show meaningful improvement patterns. The differential equation extension (Lorenz, MHD, Glycolysis) shows the method works beyond the algebraic equation setting.

### 跟我们的关系

**对 0616-fex (FEX: RL for Symbolic Regression) 课题**：

1. **撞车程度：高度相关但方向互补，不直接撞车**。VSR-DPG 的核心贡献是 "vertical discovery path"（控制变量实验逐步扩展），FEX 的核心贡献是 finite expression 表示和 RL 搜索。VSR-DPG 可以视为 FEX 在"主动数据采集/实验设计"方向上的扩展。两者解决的是符号回归的不同瓶颈：FEX 解决表示和搜索效率，VSR-DPG 解决多变量扩展性。

2. **可借用的工具/方法**：
   - CFG grammar rule representation for expressions: 将表达式表示为 grammar rule 序列而非 expression tree，这一 trick 可以用于改进 FEX 的表达式采样策略
   - Control variable experiment 框架：如果 FEX 要扩展到多变量场景，vertical discovery path 是直接可借用的思路
   - REINFORCE + baseline variance reduction：简单的 baseline trick 可以参考

3. **该引**：是。这篇和 FEX 都做 RL+符号回归，是直接相关的工作。必须引用以 situate FEX 在当前 landscape 中的位置。

4. **差异化空间**：VSR-DPG 依赖 data oracle（控制变量实验），这在很多实际场景下不可得。FEX 如果不依赖这个假设，就是明确的差异化优势。另外 VSR-DPG 的 constant classification 是启发式的，有改进空间（例如用贝叶斯方法或 learned classifier）。此外，VSR-DPG 在 6+ 变量时 recovery rate 降到 40-60%，FEX 如果能在这个 regime 取得更好结果就是明确贡献。

5. **要注意的是**：这篇论文的作者（Nan Jiang, Yexiang Xue, Purdue）和 FEX 的作者（Haizhao Yang 组）应该互相认识（都在符号回归和 AI for Science 领域）。投稿时有可能撞到同一个 reviewer pool。

## Read by: 0616-fex (2026-06-16)

**审读视角**：这篇论文和 FEX 同属 "RL for Symbolic Regression" 方向，核心差异在于 VSR-DPG 引入了一条"垂直发现路径"——通过控制变量实验逐步添加自变量，将大的 hypothesis space 分解为一系列小的搜索空间。这对 FEX 的启示：如果 FEX 要做多变量（>5 variables），可以考虑类似的 vertical decomposition，但 FEX 的优势在于不需要 data oracle（不假设可以主动设计控制变量实验）。

**撞车评估**：不直接撞车。VSR-DPG 的 novelty 在 "vertical path" 和 "grammar rule representation"，FEX 的 novelty 在 "finite expression" 表示和相应的 RL 搜索策略。两者可以在同一个 pipline 中组合（用 VSR 的 vertical decomposition + FEX 的 finite expression search）。

**可用的方法论**：
- Grammar rule 序列表示：比 tree traversal 更适合 vertical expansion，可以考虑在 FEX 的 expression 构建中引入类似机制
- Constant classification heuristic（summary vs standalone）：FEX 在做多变量扩展时可以借鉴
- 实验 setup（Trigonometric dataset, differential equation benchmarks）：可作为 FEX 的对比基准

**差异化机会**：
- FEX 不需要 control variable experiment 假设 → 明确优势
- VSR-DPG 在 >5 active variables 时 recovery rate 降到 40% → FEX 可以瞄准这个 regime
- VSR-DPG 的算子集仅限于 {+,-,×,sin,cos} → FEX 支持更丰富的算子库
