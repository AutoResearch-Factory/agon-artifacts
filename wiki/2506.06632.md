# Curriculum Reinforcement Learning from Easy to Hard Tasks Improves LLM Reasoning (2506.06632)

- **Year**: 2025.06
- **Authors**: Shubham Parashar, Shurui Gui, Xiner Li 等, Texas A&M University
- **Venue**: arXiv; TeX 使用 ICLR 2026 conference 模板, 但 arXiv metadata 未给正式录用信息
- **Keywords**: curriculum reinforcement learning, GRPO, RLVR, LLM reasoning, difficulty scheduling, task decomposition, DAPO, sample complexity
- **One-liner**: 用难度课程RL提升小模型推理泛化

## 精华提炼

### 管线/方法

1. 输入是一组 reasoning / planning benchmark, 目标是用 RL post-training 提升小模型在困难题和 OOD 题上的表现。
2. 作者把 reasoning 定义成: 从简单任务抽取原则, 并迁移到更难任务的能力。
3. RL 形式化为 token-level MDP: state 是 token prefix, action 是 vocabulary token, policy 是预训练 LLM。
4. 输出格式沿用 DeepSeek 风格 `<think>...</think><answer>...</answer>`, reward 在最终答案出现后给。
5. reward 主要是稀疏 outcome reward, 实现细节里又加入格式奖励和答案正确奖励。
6. 第一阶段是 task decomposition: 把训练集拆成 trivial, easy, medium, hard 四个难度桶。
7. 有人工难度标注的任务直接按任务属性分桶。
8. Blocksworld 按 plan length: trivial=1, easy=2, medium=4, hard=6, OOD=8。
9. Countdown 按 operand 数: trivial=2, easy=3, medium=4, hard=5, OOD=6。
10. MATH 按 level: level 1/2/3/4 训练, level 5 作为 OOD。
11. 无人工难度的 GSM8K 和 AQuA 用 base model error rate 估计难度。
12. 对 GSM8K/AQuA 每题采样 20 个 1-shot CoT response, error rate = incorrect / 20。
13. Error rate 0-25% 标为 trivial, 25-50% easy, 50-75% medium, 75-100% hard。
14. 第二阶段是 scheduler: 训练时按时间变化的概率从不同难度桶采样。
15. Balanced baseline 固定以 1/K 概率从每个难度桶采样。
16. Traditional curriculum baseline 在固定训练步数阈值后从 easy 切到 hard。
17. E2H-C 用 cosine interpolation, 前期偏 easy, 后期偏 hard, 不需要调参。
18. E2H-G 用 Gaussian likelihood 调度, 把训练进度映射到一维难度位置。
19. E2H-G 的公式是 exp(-((x_t - mu_k)^2)/(2 sigma^2)), x_t=(t/T)^beta(K-1), mu_k=k-1。
20. E2H-G 调两个超参 beta 和 sigma, 论文测试 (0.25,0.75), (0.5,0.5), (0.75,0.25)。
21. 核心直觉是: 简单题前期提供较密集 reward, 但必须及时淡出, 否则会 overfit easy shortcuts。
22. 论文还把 E2H 接到 DAPO 上, 让 difficulty scheduler 先塑造采样分布, DAPO 再过滤 zero-advantage batch。

### 关键技术细节

1. 所有 post-training 实验都用 GRPO, 默认 balanced scheduling 作为 GRPO baseline。
2. 训练库是 Hugging Face TRL。
3. 训练模型包括 Qwen 2.5 1.5B Instruct, Qwen 2.5 3B Instruct, LLaMA 3.2 3B Instruct。
4. GRPO learning rate 是 1e-6。
5. LR scheduler 是 cosine。
6. per-device train batch size 是 2。
7. gradient accumulation steps 是 4。
8. max steps 是 1600。
9. num_generations 是 8。
10. GRPO beta 是 0.001。
11. 使用 bf16 和 tf32, 并启用 gradient checkpointing。
12. LoRA rank r=32, alpha=64, dropout=0.1。
13. LoRA target modules 是 q_proj 和 v_proj, task type 是 CAUSAL_LM。
14. 训练使用 vLLM, vLLM GPU memory utilization 是 0.2。
15. 硬件最多 3 张 80GB A100, 其中 1 张给 vLLM inference, 其余用于训练。
16. 平均训练耗时 16-18 小时。
17. 结果平均跨 3 个 seed。
18. 普通 inference temperature=0.0。
19. pass@k 评测 temperature=0.7, top_p=0.9, top_k=50。
20. 样本效率分析固定 1600 iterations 和 effective batch size 8, 总训练样本 12,800。
21. 在 Blocksworld 上, E2H-G(0.5,0.5) 实际采样 trivial/easy/medium/hard = 1628/2997/4595/3580。
22. 相比 GRPO(HARD) 的 12,800 hard samples, E2H-G 用 3,580 hard samples 得到更强 hard/OOD 表现。
23. 理论部分用 Approximate Policy Iteration 分析 curriculum MDP 序列。
24. 设任务分布 d_1 到 d_K 逐渐从 easy 过渡到 hard target, 分析 final policy pi_K 与 target optimal policy 的 gap。
25. 定理给出包含 actor convergence bias, evaluation error, policy-update error, curriculum approximation error 的 performance bound。
26. 有限样本证明借用 linear function approximation 下 TD-learning / API 的已有框架。
27. Sample complexity 定理声称在合适几何误差分配和 L_k 难度增长下, curriculum 总样本数可小于 direct learning。
28. 这里的理论和实际 GRPO + LoRA + autoregressive LLM 之间有明显抽象落差, 更像 conceptual support 而不是可直接审计的训练保证。

### 核心结果

1. RQ1: 只训练 hard 样本通常学不起来, 尤其在 Blocksworld 上 Qwen 1.5B 的 hard-only 全部 split 近乎 0。
2. 加入 trivial/easy/medium 后, Blocksworld 从 hard-only 的 0/0/0/0/0 提升到 98/100/83.3/21.1/2.6。
3. Countdown hard-only 还能学到一部分, 但 mixed difficulty 仍提升 trivial/easy/medium。
4. MATH 因 base model 初始能力较强, hard-only 并非完全失败, 但 OOD 仍低于 E2H。
5. RQ2: balanced 是强 baseline, 但没有显式调度, 在 sparse reward hard tasks 上不够。
6. Qwen 1.5B 上, Blocksworld GRPO(All) hard/OOD = 21.1/2.6, E2H-G = 32.9/7.3。
7. Qwen 1.5B 上, Countdown GRPO(All) medium/hard/OOD = 28.8/18.1/9.2, E2H-G = 70.4/41.0/19.2。
8. Qwen 1.5B 上, MATH GRPO(All) hard/OOD = 46.3/25.7, E2H-G = 48.7/27.6, E2H-C OOD = 28.6。
9. LLaMA 3.2 3B 上, Blocksworld GRPO(All) hard/OOD = 38.9/13.3, E2H-G = 44.1/17.6。
10. LLaMA 3.2 3B 上, Countdown GRPO(All) hard/OOD = 47.9/2.7, E2H-G = 46.5/24.3。
11. LLaMA 3.2 3B 上, MATH GRPO(All) hard/OOD = 22.0/10.2, E2H-C = 34.3/15.8。
12. Qwen 2.5 3B 上, Countdown GRPO(All) hard/OOD = 41.7/9.4, E2H-G = 60.7/32.3。
13. Qwen 2.5 3B 上, Blocksworld GRPO(All) hard/OOD = 38.3/14.6, E2H-G = 53.3/23.2。
14. GSM8K/AQuA 无人工难度标签时, error-rate proxy 仍有用: Qwen 1.5B 上 E2H-G GSM8K avg = 78.7, AQuA avg = 66.1。
15. 在 DAPO ablation 中, DAPO baseline Countdown hard/OOD = 22.0/10.9, DAPO+E2H-G = 59.4/30.1。
16. DAPO baseline Blocksworld hard/OOD = 24.3/3.4, DAPO+E2H-G = 46.7/9.3。
17. DAPO baseline MATH hard/OOD = 47.9/28.3, DAPO+E2H-C = 50.2/29.2。
18. DAPO+E2H 降低 zero-advantage batch fraction, 说明难度条件采样能让 batch 更贴近模型当前能力。
19. AIME24/OlympiadBench 扩展评测显示小幅提升: Qwen 1.5B MATH-trained GRPO AIME pass@1/pass@32 = 3.3/10.0, E2H-C = 6.7/16.7。
20. OlympiadBench 上 Qwen 1.5B GRPO pass@1/pass@32 = 7.3/33.3, E2H-C = 10.0/40.0。
21. 一个反直觉结果是 E2H-C 在某些任务上明显失败, 例如 LLaMA 3B Blocksworld easy/medium/hard/OOD 直接掉到 0。
22. 另一个重要结果是 SFT 在 Blocksworld 上可能强于 RL: Qwen 1.5B SFT hard/OOD = 55.3/16.5, 高于 E2H-G 的 32.9/7.3。
23. 但 SFT 在 Countdown OOD 为 0.0, 而 E2H-G 对 Qwen 1.5B 和 LLaMA 3B 分别达到 14.2 和 24.3。
24. 因此论文最强结论不是"E2H 全面胜过所有方法", 而是"当 hard reward sparse 且 SFT 泛化差时, 难度调度显著改善 RLVR"。

### 明确局限

1. 作者自己承认 scheduler 仍是简单 Gaussian/cosine, 没有训练中自适应。
2. E2H-G 需要调 beta 和 sigma, 论文常报告三组参数中最好的数。
3. E2H-C 的失败说明"从 easy 到 hard"不是充分条件, 调度速度和任务 reward sparsity 很关键。
4. 实验主要是小模型 1.5B 到 3B, 没有证明在 7B/30B/70B 或强 reasoning model 上同样成立。
5. benchmark 以 Blocksworld, Countdown, MATH, GSM8K, AQuA 为主, 都是离散 reasoning toy-to-standard tasks, 没有真实科学发现或 PDE/SR 搜索。
6. 难度标签对 Blocksworld/Countdown/MATH 是人工规则, 对 GSM8K/AQuA 是 base model error proxy; 换模型后难度桶可能变化。
7. 训练成本不低: 每个模型任务可到 16-18 小时 A100 级训练, 对大量 ablation 有成本压力。
8. 论文有代码链接, 但 TeX 里没有 artifact hash, 训练日志, 完整 seeds 输出或独立复现实验。
9. 理论部分把 LLM post-training 抽象成 API/linear-function-approximation 框架, 与 GRPO + LoRA 实际算法有距离。
10. 证明里有一些值得谨慎的地方, 例如 concentrability / C_k 处理和若干 bound 的符号使用并不完全让人放心。
11. "reasoning" 被定义为 easy-to-hard generalization, 这个定义对本论文方便, 但不覆盖更广泛的 mechanistic reasoning 或 planning validity。
12. 对 <topic_slug> 0616-fex 来说, 最大局限是没有符号表达式树, 没有连续参数拟合, 没有 PDE residual, 没有 FEX 的 Adam+BFGS 内层优化。
13. 因此它不能作为 FEX 搜索复杂度或 FEX curriculum 的直接证据, 只能作为课程调度和 RLVR sparse reward 的方法学参考。

### 跟我们的关系

1. 对 0616-fex, 这篇不直接撞车 FEX 主线。
2. 它研究的是 LLM reasoning RL post-training, 不是 symbolic regression, 不是 PDE solver, 也不是 FEX 的有限表达式搜索。
3. 如果我们的 claim 是"课程学习本身能改善 RL", 那会被它和 Self-Evolve / Online Difficulty Filtering / DAPO 等工作预占。
4. 如果我们的 claim 是"FEX 表达式树搜索需要从浅树/简单 PDE/低维结构逐步推进", 它是相关 prior, 但不是 novelty killer。
5. 最可用的工具是 E2H-G scheduler: 用 tree depth, residual basin difficulty, operator count, variable dimension 或 zero-reward rate 定义 difficulty buckets。
6. FEX 中可类比的 trivial/easy/medium/hard 可以是: 低树深 closed form, 分离变量结构, 低维 Poisson, 高频/多尺度/PIDE/高维 hard cases。
7. 论文的核心经验很适合 FEX: easy examples 早期提供 dense reward, 但停留太久会 overfit shortcuts。
8. 对 FEX-RL controller, 这意味着浅树或过简单 PDE 只能作为 warm-up, 不能长期主导 replay buffer。
9. DAPO+E2H 的 zero-advantage batch 分析可移植到 FEX: 监控一批表达式是否 reward 全 0 或全相同, 作为 curriculum 是否有效的在线指标。
10. 论文理论部分可以作为"课程比 direct hard learning 更省样本"的参考语言, 但不能直接用于 FEX 证明。
11. FEX 需要处理连续参数优化、符号等价类、PDE residual landscape 和表达式 grammar, 这些都超出本文理论假设。
12. 若做 `fex-curriculum-depth`, 应引用本文作为 LLM/RLVR 课程调度 prior, 同时明确差异: 我们研究 finite expression / PDE residual / symbolic search 的课程。
13. 若做 `fex-llm-skeleton`, 本文价值较间接: 它提示 prior 粒度和训练阶段可以调度, 但 LLM skeleton 本身不在本文覆盖范围内。
14. 下一轮搜索应围绕 "curriculum symbolic regression", "difficulty aware expression tree RL", "zero advantage RLVR", "adaptive difficulty sampling symbolic regression", "curriculum PDE symbolic solver"。

## Read by: 0616-fex (2026-06-16)

本次从 FEX / RL symbolic regression 角度审读, 结论是: 这篇是强相关方法学 prior, 但不是 FEX 直接竞品。它证明了在 LLM reasoning 的 sparse reward RLVR 场景中, 训练样本的难度分布和时间调度会显著影响 hard/OOD 泛化; 这与 FEX controller 在深树、多尺度、高维 PDE 上遇到的 sparse reward / search instability 很像。可借鉴的具体方法包括 E2H-G Gaussian scheduler, error-rate proxy 难度估计, zero-advantage batch 诊断, 以及把 scheduler 与 DAPO 类过滤器组合的思路。

对 `fex-curriculum-depth` 来说, 本文会压缩 novelty: 不能把"从易到难的 RL 课程"作为核心创新。差异化必须落在 FEX-specific 机制上, 例如用表达式树深度、算子集合复杂度、PDE residual landscape、连续参数优化失败率、symbolic equivalence class 来定义 difficulty, 并证明这些定义能减少 FEX 的实际搜索成本。对 `fex-llm-skeleton` 来说, 本文不是 collision, 但提醒我们 LLM skeleton 不应只是一次性 prior, 也可以作为课程中的逐级先验粒度: operator-set -> subtree -> full skeleton -> oracle skeleton。

该引。引用位置建议放在 curriculum / RLVR sparse reward 背景段, 同时和 FEX 领域文献分开处理: E2H 是 LLM reasoning prior, DSO / SSDE / SymPlex / DSR 才是 symbolic regression 或 PDE expression-search prior。下一轮若继续补文献, 优先查本文反引文中的 difficulty-adaptive RLVR 工作, 以及 symbolic regression 里是否已有 "curriculum expression tree search" 或 "difficulty-aware symbolic RL"。
