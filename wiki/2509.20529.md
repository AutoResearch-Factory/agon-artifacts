# MDBench: Benchmarking Data-Driven Methods for Model Discovery (2509.20529)

- **Year**: 2025.09
- **Authors**: Amirmohammad Ziaei Bideh, Aleksandra Georgievska, Jonathan Gryak; CUNY Graduate Center / Queens College
- **Venue**: arXiv; AAAI 2026 style source
- **Keywords**: model discovery, symbolic regression, dynamical systems, ODE, PDE, SINDy, benchmark, equation fidelity
- **One-liner**: 系统评测 ODE/PDE 方程发现方法的基准。

## 精华提炼

### 管线/方法
- 这篇论文提出 MDBench, 一个用于 data-driven model discovery 的开源基准和实验框架。
- 输入对象是动力系统观测数据, 包括 ODE 轨迹和 PDE 时空场。
- ODE 部分采用 ODEBench: 63 个来自 Strogatz 教材和 Wikipedia 的一到四维系统。
- PDE 部分包含 14 个系统: 7 个常用 PDE 和 7 个作者新增的更现实物理系统。
- 常用 PDE 包括 Advection, Burgers, KdV, Kuramoto-Sivashinsky, NLS, Reaction-Diffusion, Advection-Diffusion。
- 新增 PDE 覆盖 Heat (Solar) 1D/2D/3D, Heat (Laser), NS (Channel), NS (Cylinder), RD (Cylinder)。
- 每个数据集被转成统一的 compressed NumPy `.npz` 格式。
- 对 ODE, 输入是状态变量 `u`, 目标是时间导数 `du/dt`。
- 对 PDE, 输入是状态变量和空间导数组成的符号特征集合。
- PDE 符号特征包括每个状态变量及其沿每个空间维度的一到四阶空间导数。
- 空间导数使用 `findiff` 做二阶精度有限差分。
- PDE 的时间导数也用同一有限差分方案估计。
- 噪声设置是对干净状态变量施加乘性 Gaussian noise。
- SNR 水平为 40 dB, 30 dB, 20 dB, 10 dB。
- 噪声公式为 `u -> (1 + xi)u`, 其中 `xi ~ N(0, sigma)`, `sigma = 10^(-SNR_dB/20)`。
- 训练/验证/测试按时间维度切成 60%/20%/20%。
- 超参数调优在验证集上做 grid search。
- 最终模型用训练加验证数据重新训练, 再在测试集评测。
- 统一 fitness 为 `1/(1+NMSE) + lambda exp(-l/L)`。
- 论文中设置 `lambda=1`, `L=200`。
- 每个 method-dataset experiment 限时 12 小时。
- 运行环境是 Ubuntu 22.04.3, 24-core Intel Core Ultra 9 285K CPU, 96 GB RAM。
- 需要 GPU 的 EQL, uDSR, ODEFormer, End2end 使用单张 NVIDIA RTX 4000 Ada 20 GB。
- DeepMoD, uDSR, End2end 因内存或运行时间限制, 在最多 10,000 points 的子采样数据上训练。
- 输出包括导数预测误差, 方程复杂度, 干净条件下的人工 equation fidelity 判断。

### 关键技术细节
- 评测 12 个算法, 覆盖 Linear Models, Genetic Programming, Deep Learning, Large-Scale Pretraining 四类。
- Linear Models 包括 PDEFIND, SINDy, WSINDy, ESINDy/EWSINDy, Bayesian, DeepMoD。
- Genetic Programming 包括 PySR 和 Operon。
- Deep Learning 包括 EQL 和 uDSR。
- Large-Scale Pretraining 包括 ODEFormer 和 End2end。
- NMSE 定义为导数误差平方和除以真实导数平方和加 `epsilon`。
- `epsilon` 固定为 `1e-10`, 用于避免分母为零。
- 方程复杂度由 SymPy 解析表达式树后计算节点、常数项和操作数。
- 干净数据下还人工判断是否恢复 ground-truth equation。
- full equation discovery 的系数容忍范围是 ground truth 的 `±5%`。
- partial correct 表示只缺一个项或多一个项, 其他项系数在 `±5%` 内。
- PDEFind/WSINDy 的候选项包括 polynomial, sin, cos; derivative order 到 4。
- PDEFIND threshold 为 `np.logspace(-7, 0, 16)`, alpha 为 `1e-5` 或 `1e-4`, max iterations 200。
- WSINDy 还调 integration points 为 200 或 2000, optimizer 为 SR3。
- E(W)SINDy 额外调 ensemble 数量 10/20/50, subset ratio 0.5/0.7/0.9。
- Bayesian 只用 polynomial basis, threshold 同样为 `np.logspace(-7,0,16)`。
- DeepMoD 用 4 层 `[50,50,50,50]`, learning rate 0.001, threshold 0.1/0.3/0.5。
- SINDy threshold 为 `np.linspace(0.001,1,10)`, max iterations 20/50/100。
- PySR 使用 100 iterations, 1000 cycles/iteration, 20 populations, population size 100。
- PySR max size 40, max depth 20, operators 包括 `+ - * /`, unary 包括 sin, exp, log, sqrt。
- Operon symbols 包括 add, sub, mul, aq, sin, constant, variable; pool/population size 都是 1000。
- uDSR 使用 add/sub/mul/div/sin/cos/exp/poly/const, batch size 1000, n samples 100000。
- ODEFormer beam size 50, beam temperature 在 0.05 到 0.5 之间调。
- End2end max input points 200, refine 100 trees, rescale=True。

### 核心结果
- MDBench 覆盖 77 个动力系统: 63 ODE + 14 PDE。
- 摘要级结论: PDE 上 linear methods 的导数预测误差最低, ODE 上 genetic programming methods 最低。
- 另一个核心结论: linear models 总体更抗噪。
- ODE clean/low-noise 设置中, GP 方法通常比其他方法有更低 NMSE。
- 例子: PySR 相对 ESINDy 在 low-noise/clean ODE 上显著更低, Wilcoxon `W=9774, p <= 0.0001`。
- ODEFormer 在 SNR >= 30 dB 和 clean 数据上导数预测较好。
- ODEFormer 在 SNR <= 20 dB 下降明显, 作者认为可能与大参数量过拟合有关。
- SINDy/ESINDy 在 noisy ODE 条件下产生更简单方程。
- SINDy 方程复杂度显著低于 next simplest EQL, `W=6951.5, p<1e-8`。
- GP 方法在 noisy ODE 下更容易产生复杂且不准的表达式。
- End2end 在 ODE 上误差最高之一, 作者归因于预训练数据没有覆盖动力系统。
- SINDy 和 Operon 是 ODE 训练最快的代表, EQL 和 uDSR 慢几个数量级。
- PDE 比 ODE 更难, 因为空间导数和高阶导数会放大噪声。
- PDE clean 到 40 dB 的误差比对数均值为 `3.13 ± 2.09`。
- ODE 对应值为 `0.88 ± 2.12`, 显示 PDE 对噪声更敏感。
- PDE clean/low-noise 中, 除 DeepMoD 外多数 LM 比 GP 误差更低、更稳。
- WSINDy 相对 PySR 和 Operon 在 PDE low-noise/clean 中显著更低, 分别 `W=455.0` 和 `W=466.0`, 均 `p<1e-4`。
- EQL 和 End2end 被排除出 PDE aggregate evaluation, 原因是高维输入处理失败或实现限制。
- End2end 预训练最多 10 features, 不适合高维 PDE 或多状态变量系统。
- Heat (Solar) 维度从 1D 到 3D 增加时, 所有方法都更难。
- 论文正文有一处文字称 NMSE decreases, 但上下文和表格含义应为 error increases; 这是需要注意的表述问题。
- Heat (Laser) 和 RD (Cylinder) 是最难的 PDE, 因为空间相关 forcing 或 piecewise forcing 加上高状态维。
- RD (Cylinder) 有 6 个状态变量, 对库方法和搜索方法都非常困难。
- 在 NS (Channel) 上, PySR/Operon/uDSR 可以得到低 NMSE, 但不一定恢复真实动力学。
- 这个现象说明低导数误差不能替代 equation fidelity。
- 作者明确指出当前缺少成熟的 equation similarity/fidelity metric。
- 作者建议考虑 PDE 的离散 Sobolev semi-norm 等函数空间指标。
- PDE runtime 范围从几秒到数小时。
- PDEFind 在 Heat Solar 从 1D 到 2D/3D 相比会慢约 40x 和 465x。
- PySR 在同组数据上 2D 慢约 33x, 3D 约 12x, 高维扩展反而比某些 LM 更稳。
- 这支持一个实际结论: 低维 PDE 用 LM 更合适, 高维或时间受限时 GP/uDSR 可能更稳。

### 明确局限
- 这不是新方程发现算法, 主要贡献是 benchmark、数据集、统一 pipeline 和系统性负面结果。
- 所有实验由作者在同一框架内完成, 没有第三方独立复现实验。
- 数据集主要是仿真数据, 不是真实实验室传感器数据。
- PDE 依赖有限差分导数, 导数估计本身在噪声下会引入强 bias 和方差。
- 多数方法隐含空间/时间均匀物理参数, 不适合 Heat (Laser) 和 RD (Cylinder) 这类异质 forcing。
- 高维 PDE 和多状态变量系统仍是明显短板。
- EQL, End2end 等方法大量失败, 这既反映算法短板, 也反映公开实现工程质量不稳定。
- 对 equation fidelity 的判断在 clean ODE 上是人工比较, 标准化和可扩展性不足。
- NMSE 会在真实导数接近零的测试区域被放大, 因此不能单独解释模型质量。
- Benchmark 的算法实现来自不同仓库, 依赖环境复杂, 复现成本不低。
- 有些引用解析结果来自 S2 时包含噪声条目, 说明自动引用清洗还不完美。
- 对 0616-fex 而言, 如果目标是提出新方法, MDBench 更像评测基线而不是直接撞车方法。

### 跟我们的关系
- 对 `0616-fex` 的直接撞车风险取决于题目是否也是"方程发现 benchmark"。
- 如果 `0616-fex` 是新算法, 这篇不直接撞车, 但会成为必须引用的 benchmark paper。
- 如果 `0616-fex` 要做 ODE/PDE model discovery, MDBench 的 77-system suite 可直接作为评测环境。
- 该文最有用的是 failure map: 噪声、高阶导数、异质 forcing、高维 PDE、多状态变量都会击穿现有方法。
- 可以借鉴它的统一 fitness, NMSE + complexity 指标, 以及 `±5%` 系数级 fidelity 判定。
- 不能只复用 NMSE; 论文自己说明 NMSE 会掩盖错误方程, 尤其 NS (Channel) 这类场景。
- 差异化方向包括 noise-aware derivative estimation, equation equivalence metric, heterogeneous coefficient/forcing discovery。
- 另一个差异化方向是面向 PDE 的 LLM/Transformer 预训练, 因为 End2end/ODEFormer 对 PDE 和高维输入仍弱。
- 对 `0616-fex` 下一轮检索, 应重点查 2026 的 LLM-ODE、governing differential equations discovery 和 equation fidelity metric。

## Read by: 0616-fex (2026-06-16)

这篇论文对 `0616-fex` 的价值主要是基准和反例库, 不是新算法本身。若 `0616-fex` 关注的是方程发现、feature/expression extraction 或科学模型发现, MDBench 应该被引用为当前 ODE/PDE 统一评测的强基线。它的核心结论对选题很有约束力: 只报导数 NMSE 不够, 因为低误差可能对应错误结构; 噪声和高阶空间导数会快速放大误差; 异质 forcing、空间变系数、高维 PDE、多状态变量是现有方法的集中失败点。

撞车方面, 如果 `0616-fex` 计划做"再造一个 ODE/PDE model discovery benchmark", 那撞车非常高, 需要明确新增维度, 例如真实实验数据、统一 equation equivalence metric、LLM agent 自动验证、或异质参数 PDE。若 `0616-fex` 是新方法或新 evaluator, 则不撞车, 反而可以直接拿 MDBench 的 77 个系统、12 个 baseline、SNR 设计、60/20/20 split、NMSE + complexity + fidelity 框架做对照。最推荐的差异化是: 让生成方程和验证方程分离, 加入独立 verifier; 设计对函数等价更敏感的 metric; 针对 Heat (Laser)/RD (Cylinder) 这类 forcing-heavy PDE 展示明确优势。
