# SURFACEBENCH: A Geometry-Aware Benchmark for Symbolic Surface Discovery (2511.10833)

- **Year**: 2025.11 (arXiv v2: 2025-11-13; tex 中标注 TMLR accepted camera-ready: 2026.03)
- **Authors**: Sanchit Kabra, Shobhnik Kriplani, Parshin Shojaee, Chandan K. Reddy; Virginia Tech
- **Venue**: TMLR / arXiv
- **Keywords**: symbolic regression, symbolic surface discovery, geometry-aware benchmark, implicit surfaces, parametric surfaces, Chamfer distance, Hausdorff distance, LLM scientific discovery
- **One-liner**: 用 183 个 3D 曲面任务检验符号发现的几何泛化。

## 精华提炼

### 管线/方法
- 论文提出 SurfaceBench，目标是把 symbolic regression 从标量函数 \(y=f(x)\) 推到 3D surface equation discovery。
- 输入侧不是单纯的一维或低维表格，而是从 3D 曲面方程采样得到的点数据。
- 输出侧要求恢复可解释的符号表达式，覆盖显式、隐式、参数化三种表示范式。
- 显式形式是 \(z=f(x,y)\)，强调二元输入到单输出的曲面函数。
- 隐式形式是 \(f(x,y,z)=0\)，强调 level set、拓扑结构和等值面恢复。
- 参数化形式是 \((x(u,v),y(u,v),z(u,v))\)，强调多个坐标函数的耦合发现。
- 数据集包含 183 个解析构造的曲面方程。
- 这 183 个任务分布在 15 个结构类别中，包括 piecewise、oscillatory、implicit high-degree、parametric multi-output 等。
- 每个曲面提供 5k train samples、500 test samples、500 OOD samples。
- benchmark construction pipeline 分为八步：domain selection、equation selection、operator distribution、compositional augmentation、novelty check、stability check、data generation、expert validation。
- domain selection 选择 optics、fluid dynamics、electromagnetics、materials science、robotics 等会自然产生连续 3D 曲面的科学背景。
- equation selection 先收集显式、隐式、参数化的 analytic seed equations。
- operator distribution 估计常见 symbolic operators 的分布，例如 trigonometric、exponential、rational、polynomial。
- compositional augmentation 借鉴 LLM-SRBench，做函数嵌套、加乘混合、坐标重参数化、operator substitution。
- novelty check 会把生成方程与 Feynman、SRBench 等既有 benchmark 对比，丢弃重复或过于平凡的形式。
- stability check 检查边界、零点、渐近行为，避免数值不稳定或解析退化。
- data generation 在有界 3D domain 内采样，高曲率区域使用 adaptive sampling density。
- expert validation 是人工视觉检查，确认曲面平滑、完整、解析一致。
- evaluation pipeline 先让 symbolic regression framework 从 sampled 3D data 生成候选表达式。
- 然后用 symbolic accuracy、NMSE、Chamfer distance、Hausdorff distance 四类指标评估。
- symbolic accuracy 使用 GPT-4o 作为自动等价判断器，判断 ground truth 与 hypothesis 是否存在常数重标定下的数学等价。
- NMSE 作为传统 regression-style error，用来和标量 symbolic regression benchmark 对齐。
- Chamfer distance 在 object space 比较两个采样点云的平均几何偏差。
- Hausdorff distance 捕捉最坏局部偏差，对裂缝、缺失结构、局部爆炸更敏感。
- 论文会对 predicted/reference surfaces 采样成 dense point clouds，并在 similarity transform 下对齐后计算几何距离。
- 对隐式曲面，任务被 cast 成 scalar regression：在 \((x,y,z)\) 上拟合 \(\hat f(x,y,z)\)，再抽取 \(\hat f=0\) 的 zero-level set。
- 对参数化曲面，OpenEvolve 可联合优化 \((\hat x,\hat y,\hat z)\)，PySR 则分别拟合三个坐标函数。
- 多数 SR baselines 因为没有稳定 multi-output support，被排除出 parametric task。
- OOD 评估是 range shift：训练在 \([-5,5]\)，测试在外部区间 \([-10,-5]\cup[5,10]\)。
- 噪声评估选 13 个代表方程，加 1%、5%、10% Gaussian noise，主要看 Chamfer/Hausdorff 的退化斜率。

### 关键技术细节
- LLM-based baselines 包括 LLM-SR、LaSR、SGA、OpenEvolve。
- Non-LLM baselines 包括 TPSR、NeSymReS、E2E、DSR、uDSR、PySR、gplearn。
- LLM backbones 包括 GPT-4o-mini、Llama-3.1-8B-Instruct、Qwen3-8B。
- 作者让所有 LLM-guided baselines 每个 problem 使用 1k LLM calls。
- non-LLM methods 也给等价数量的 calls / evaluation budget，以控制预算公平性。
- OpenEvolve 设置 temperature \(\tau=0.8\)。
- OpenEvolve 初始 prompt 采样 5 个 equation program hypotheses。
- OpenEvolve 每个 program hypothesis execution 的 time limit 是 30s。
- OpenEvolve 使用 Scipy BFGS 做 equation skeleton 的参数优化。
- SGA 使用 PyTorch 的 `torch.nn.Module` 实现。
- SGA 使用 MSE loss 提供 data-driven feedback。
- SGA 使用 Adam optimizer 做 differentiable parameter optimization。
- LaSR 设置 iterations=25、cycles per iteration=550、populations=10、population size=33、maximum size=30。
- LaSR operators 包括 `+`、`*`、`-`、`/`、power、exp、log、sqrt、sin、cos、tan、cosh。
- LaSR 的 LLM 权重为 `llm_mutate=0.005`、`llm_crossover=0.005`、`llm_gen_random=0.005`。
- LaSR 从 concept library 取 Top-K=20 concepts，并使用 PySR 默认配置做参数优化。
- LLM-SR 设置 temperature \(\tau=0.8\)、batch size \(b=4\)、parallel evaluators \(e=4\)。
- LLM-SR 每个 program hypothesis time limit 是 30s，memory limit 是 2GB。
- LLM-SR 使用 10 islands 维持 population diversity，每个 prompt 放 2 个 in-context examples。
- LLM-SR 限制 equation skeleton 最多 10 个 parameters，并用 BFGS 优化参数。
- memorization resistance 检查了 SurfaceBench 与 AI-Feynman、SRBench、LLM-SRBench 的 operator-set similarity 和 AST-signature similarity。
- SurfaceBench 的 operator-set similarity 均值是 0.284，没有方程超过 0.7。
- SurfaceBench 的 AST-signature similarity 均值是 0.407，没有方程超过 0.65。
- 与 AI-Feynman 的平均相似度最低：operator 0.118，AST 0.191。
- 与 SRBench 的平均相似度也低：operator 0.138，AST 0.272。
- 与 LLM-SRBench 最接近，但仍不高：operator 0.284，AST 0.407。

### 核心结果
- exact equation recovery 很低：LLM-based frameworks 聚合只有 4%，traditional SR baselines 聚合只有 6%。
- 这个数字是论文最重要的负面结论之一：surface-level symbolic discovery 远难于标量函数恢复。
- 在显式曲面上，models 往往能找到正确 structural family，但几何距离仍然大。
- 作者解释为 parameter calibration 不够，结构猜对不代表 scale、shift、coupling constants 被拟合好。
- OpenEvolve + GPT-4o-mini 在 explicit 上的 SA=0.50、NMSE=0.98、Chamfer=2.69、Hausdorff=4.88。
- explicit non-LLM 中 PySR 最强：SA=0.25、NMSE=0.18、Chamfer=0.13、Hausdorff=0.41。
- explicit non-LLM 的几何指标明显强于 LLM methods，说明传统迭代搜索和参数拟合仍有优势。
- 在 implicit 上，几何距离可以很好但 symbolic accuracy 较低。
- 这说明在 object space 接近 ground truth 不一定意味着 algebraic form 被精确恢复。
- OpenEvolve + GPT-4o-mini 在 implicit 上 SA=0.12、NMSE=0.71、Chamfer=1.85、Hausdorff=4.96。
- LLM-SR + Qwen3-8B 在 implicit 上 Chamfer=1.51，但 SA 只有 0.02，体现几何近似和符号精确性之间的张力。
- implicit non-LLM 中 PySR 的 NMSE=0.60、Chamfer=1.64、Hausdorff=5.53，几何上很有竞争力。
- parametric surfaces 只有 OpenEvolve 和 PySR 被稳定评估。
- OpenEvolve + GPT-4o-mini 在 parametric 上 SA=0.07、NMSE=0.84、Chamfer=1.22、Hausdorff=3.98。
- PySR 在 parametric 上 SA=0.10、NMSE=0.61、Chamfer=2.52、Hausdorff=5.53。
- domain-prior prompt 对部分 LLM methods 有正收益，但不是根本解决方案。
- OpenEvolve explicit Chamfer 从 2.93 降到 1.16，delta +1.77。
- LLM-SR explicit Hausdorff 从 27.43 降到 20.77，delta +6.66。
- implicit 上 prior 的收益较小，例如 OpenEvolve Chamfer 从 2.41 到 1.82，Hausdorff 从 4.99 到 4.82。
- 噪声实验显示 traditional non-LLM SR 的 geometric error 变化较小。
- LLM-based methods 随噪声增加明显退化，说明 symbolic hypothesis generation 方差较高。
- Hausdorff 比 Chamfer 更容易陡增，表明噪声常导致局部 worst-case defect，而非整体均匀漂移。
- OOD range shift 会造成稳定 generalization gap。
- 很多表达式在 \([-5,5]\) 内像局部 interpolant，但在外推区间失去正确增长、渐近或振荡结构。
- failure analysis 把 LLM failure 分为 search failure 和 equation-fitting failure。
- search failure 是生成表达式用了错误 primitive family，例如该用 trigonometric 却生成 polynomial。
- equation-fitting failure 是 primitive family 找对了，但 term order、composition、constants 没拟合对。
- 作者观察到 LLM methods 在约 200 iterations 后很少能从错误 structural family 恢复。
- 论文结论是：LLM 有早期 structural prior，但缺少紧耦合的 iterative optimization / feedback mechanism。
- 对 future work，最直接的方向是 post-structure geometric calibration、multi-output joint optimization、LLM proposal + classical optimizer 的更强闭环。

### 明确局限
- SurfaceBench 是 synthetic analytic benchmark，不是真实传感器或实验物理系统的原始数据。
- 论文称 science-inspired，但许多方程是经过 compositional augmentation 的 stress tests，不是完整 governing physical laws。
- expert validation 主要是作者侧视觉检查，没有独立第三方复现实验或 blind validation。
- symbolic accuracy 依赖 GPT-4o 自动判断数学等价，不是完全形式化证明。
- GPT-4o evaluator 与 LLM baselines 处在同一模型生态，存在 evaluator bias 风险。
- parametric task 的 baseline 覆盖不足，因为多数方法没有 multi-output support。
- 论文没有给所有曲面逐项难度分布的细粒度分析，主结果以 aggregate table 为主。
- 噪声和 OOD 图给出了趋势，但 tex 中没有把所有数值完整列成机器可读表。
- benchmark 的 novelty check 主要相对 Feynman、SRBench、LLM-SRBench，不能排除更宽泛几何处理/图形学文献中的相似 analytic families。
- 对 0616-fex，如果课题要证明某个新 agent 会做 symbolic equation discovery，这篇的 4%/6% exact recovery 是很好的痛点；但如果我们声称提出第一个 geometry-aware surface benchmark，会与本文强撞。

### 跟我们的关系
- 对 `0616-fex`，最重要的撞车面是 benchmark claim：如果我们也做 3D surface / geometry-aware symbolic regression benchmark，本文已经占住“first geometry-aware benchmark for symbolic discovery of three-dimensional surfaces”。
- 如果 `0616-fex` 是方法论文，本文更像强 benchmark 和 motivation source，不必视为直接方法撞车。
- 可直接使用的工具是 SurfaceBench 数据集、Chamfer/Hausdorff object-space evaluation、explicit/implicit/parametric 三分任务设计。
- 可直接引用的痛点是 exact recovery 极低：LLM-based 4%，traditional SR 6%。
- 可直接引用的 failure taxonomy 是 search failure vs equation-fitting failure。
- 差异化方向一：不要再做静态 benchmark，而是做能够把 LLM proposal、symbolic simplification、BFGS/gradient fitting、geometric loss 联起来的 closed-loop optimizer。
- 差异化方向二：专注参数化 multi-output discovery，因为本文显示多数 SR methods 不支持这类任务。
- 差异化方向三：把 GPT-4o symbolic judge 替换为 CAS / equality saturation / numeric falsification 的独立验证器。
- 差异化方向四：从 synthetic surfaces 推到真实实验/仿真数据，加入 measurement protocol、uncertainty、physical constraints。
- 差异化方向五：做 per-claim evidence trace，让每个发现表达式都附带 residual maps、counterexamples、OOD falsification，而不仅是 aggregate score。

从 `0616-fex` 的角度看，这篇最值得警惕的是 claim 层面的抢占：它已经把“3D surface symbolic discovery benchmark + geometry-aware Chamfer/Hausdorff evaluation + explicit/implicit/parametric 三表示”完整讲完。如果我们的课题只是做一个类似 benchmark，撞车严重；如果我们做方法，则它是合适的评测平台和问题动机。

可用部分很明确：SurfaceBench 的 183 个任务、15 类方程、5k/500/500 split、OOD range shift、1/5/10% noise stress test、search failure vs equation-fitting failure taxonomy，都可以转化成我们方法的实验协议。最强可引用结论是现有方法 exact recovery 只有 LLM 4%、traditional 6%，说明问题仍远未解决。

差异化建议是把工作重心放在“发现后校准”和“独立验证”上。本文暴露 LLM 的问题不是不会猜函数族，而是猜到后无法稳定拟合 constants、composition 和 multi-equation coupling。因此 `0616-fex` 若能做 LLM 结构生成 + CAS/equality graph + BFGS/gradient/geometric loss + OOD counterexample mining 的闭环，会避开单纯 benchmark 的撞车，同时正面利用本文最强痛点。
