# Policy Optimization in Hybrid Discrete-Continuous Action Spaces via Mixed Gradients (2605.14297)

- **Year**: 2026.05
- **Authors**: Matias Alvo, Daniel Russo, Yash Kanoria (Columbia University, Graduate School of Business)
- **Venue**: NeurIPS 2026 (preprint, under review)
- **Keywords**: hybrid action space, policy gradient, pathwise gradient, score-function estimator, differentiable simulation, exogenous MDP, joint replenishment problem, switched LQR, mixed gradient estimator, credit assignment
- **One-liner**: 提出 HPO 框架，在 hybrid discrete-continuous action space 的 exogenous MDP 中结合 pathwise gradient 和 score-function gradient，在高维连续动作空间中大幅优于 PPO，并证明 cross term 在离散策略接近 best response 时消失。

## 精华提炼

### 管线/方法

HPO 的完整 pipeline：

1. **问题建模**：将问题形式化为 hybrid MDP —— 状态 s 为连续向量，动作 a = (x, b)，x 为离散动作（从有限集合 X 中选），b 为连续动作。transition f 和 cost c 对 (s, b) 光滑，但对 x 不要求光滑。

2. **策略架构（Towered Policy）**：
   - 离散策略 π^X_φ(s) → softmax → 采样 x_t
   - 连续策略 π^B_κ(s) → 为每个可能的离散动作输出一个候选连续动作 b^j
   - 执行时选择 b_t = π^B_κ(s)[x_t]
   - 两个网络独立参数化（φ 和 κ），不共享参数

3. **混合梯度估计**（Theorem 1, Algorithm 1）：
   - ∇_φ J：标准 SF estimator（score function），通过 PPO clipped objective 优化
   - ∇_κ J = PW term + Cross term
     - PW term：固定采样的离散动作序列，通过 simulator 反传计算 cost 对 κ 的梯度
     - Cross term：∇_κ log π^X_φ(x_t|s_t(κ)) · Q(s_t, a_t)，捕获连续动作变化如何通过状态影响离散决策概率

4. **训练流程**（Algorithm 2, HPO-PPO）：
   - 每 batch：rollout → 计算 GAE advantage → 全 batch 更新 κ（mixed estimator）→ 5 个 PPO epoch 更新 φ（4 minibatches/epoch），同时更新 value function
   - HPOFull：使用完整 mixed gradient
   - HPONoCross：丢弃 cross term，仅用 PW gradient 更新 κ

5. **PS-MDP 转化**：将有有限个 action discontinuity 的 piecewise-smooth MDP 转化为 hybrid MDP，离散动作选择 smooth region，连续动作在该 region 内优化。

### 关键技术细节

- **网络架构**：两层 hidden layer，每层 512 units，tanh 激活。权重正交初始化（hidden layer gain √2，policy output 较小 gain，value head gain 1.0）
- **优化器**：Adam，lr=1e-3（HPO），lr=1e-4（PPO），ε=1e-5，gradient clip norm=5
- **GAE**：γ=0.99，λ=0.96，advantage 在 minibatch 内 normalize 到 zero mean unit variance
- **PPO clipping**：ε_clip=0.15，value loss coefficient=0.15
- **Entropy bonus**：初始 0.5，线性衰减
- **KL early stopping**：threshold=0.015
- **可行动作强制**：JRP 用 softplus 保证非负；更复杂约束（simplex/budget）可用 softmax/sigmoid 实现
- **JRP 设置**：T=100 periods，lead time L=2，K=64p（每产品固定成本恒定），p ∈ {1,3,5,10,20,30,40,50,60}，B ∈ {16,64}，产品异构（underage/holding/mean demand 随机采样），Poisson demand
- **S-LQR 设置**：T=20 periods，p ∈ {1,3,5,7,10,13,16,20,24,32,40,48}，J ∈ {2,4,6,8} modes，A=1.10I+0.05N（不稳定系统），B_j 对角线（本 mode 1.0，其他 0.15），Q_j=I，R_j=0.1I，初始状态 U[-10/√p, 10/√p]
- **计算资源**：NVIDIA A40 GPU (46GB)，单 trial 单 GPU，多数 <60min 完成

### 核心结果

**Finding 1: HPO 在高维连续动作空间收敛远快于 PPO**

| 问题 | 维度范围 | HPO vs PPO 关键数字 |
|------|---------|-------------------|
| JRP (B=16) | p=1-60 | p≤20 相当；p≥30 开始 PPO 急剧退化；p=60 时 PPO 均值 89.29 vs HPOFull 50.28（差距 ~77%）|
| JRP (B=64) | p=1-60 | p=60 时 PPO 均值 59.18 vs HPOFull 40.70（差距 ~45%）|
| S-LQR | p=1-48 | p>10 后 PPO 急剧退化；p>24 时 PPO 中位数无法收敛；HPO 在所有配置（含 p=48）500 updates 内达 5% gap |
| S-LQR (p=48, J=2) | - | HPOFull 53.89 vs PPO 79.36（差距 ~47%）|

**Finding 2: Mixed gradient 信号质量远超 SF gradient**

- SF estimator alignment：p=1,3 时 >0.05，随 p 增大迅速降至接近 0
- Mixed estimator alignment：30-35% gap 时保持 >0.8（所有 p），5-10% gap 时下降但仍远好于 SF
- SF estimator RMSE 随 p 快速增大，直接解释了 PPO 的性能退化

**Finding 3: Cross term 在离散 near-best-response 时消失**

- Theorem 2: ‖cross term‖ ≤ ε × L_π L_s (1 + Cγ/(1-γ))，其中 ε 为离散策略的 suboptimality
- 实验验证：p=20 JRP，随 discrete policy 接近 optimal，(s,S) 策略的 ordering threshold 越来越 sharp → cross term norm → 0
- JRP 中 HPONoCross vs HPOFull：差异通常 ≤2%，仅 p=60/B=64 时退化 7%
- S-LQR 中 HPONoCross 一致优于 HPOFull，gain 随 p 增长（p=48 时约 0.6-1.2 单位，~1%）

**Finding 4: HPO vs Riccati baseline 验证**

- J=1 (standard LQR)：HPO 在所有 p 达到 near-optimal，p=48 时 gap <0.2%
- J=4 (S-LQR)：HPO 比 best single-mode Riccati controller 提升最高 13%，证明学到了有效的 mode switching

**Finding 5: PPO 超参数鲁棒性研究**

- 6 种 PPO 变体（改变 lr/entropy/epochs/update structure），无一能追上 HPO
- 专门 mirror HPO update structure 的变体（离散多步+连续单步）也失败 → HPO 优势来自 gradient quality，非 update schedule

### 明确局限

1. **需要 exogenous simulator**：训练时需要能 replay 同一 scenario 的 simulator（知道 (f,c) 和外生随机性 ξ），不适用于完全 black-box 环境。这是作者明确说的 "limited scope"。
2. **仅 discrete + continuous hybrid**：纯 discrete 或纯 piecewise-smooth 但 region 极多的问题可能不适用。
3. **Cross term 需要 privileged access**：计算 cross term 需要访问离散策略的内部结构（log-probability gradients），不能仅通过行为观察。去中心化训练仅在 near-best-response 时可行。
4. **没有真机实验**：所有实验均为仿真（JRP + S-LQR），没有 real-world robotics 验证。
5. **没有标准 benchmark 比较**：未与 Parameterized DQN、HyAR 等 hybrid action RL 方法直接比较（作者说这些方法解决 complementary problems，但不能 exploit differentiability）。
6. **离散动作数量有限**：实验仅涉及 2（JRP）到 8（S-LQR）个离散动作。大量离散动作的 scaling 未研究。
7. **Cross term variance 问题**：near optimal 时 cross term 的 variance 增大（因离散策略在 decision boundary 处 sensitivity 极高），作者观察到但未提出解决方案（除丢弃 cross term 外）。
8. **PS-MDP 转化在实践中的挑战**：理论上任何 PS-MDP 可转化为 hybrid MDP，但 region 划分在实际中可能不 trivial。
9. **无收敛理论**：没有证明 HPO 收敛到全局最优或 stationary point 的理论保证。
10. **仅 continuous 维度 scaling 研究**：主要 focus 是连续动作维度 p 增长的影响，离散动作数量 J 的 scaling 不够系统。

### 跟我们的关系

对 0616-fex（foundation model + execution / PDE / operator learning）课题：

1. **不撞车**：HPO 解决的是 hybrid discrete-continuous action RL 中 gradient estimation 的问题，核心是 PW+SF 混合梯度。0616-fex 关注 foundation model 在 PDE/physics simulation 中的应用，两者问题域不同。

2. **可用的方法论工具**：
   - **混合梯度思想**：如果我们的 pipeline 中有 hybrid 决策（如离散选择物理模型 + 连续参数回归），HPO 的 PW+SF 混合梯度框架可以直接借鉴
   - **Exogenous MDP 建模**：如果我们用 simulator/solver 生成训练数据，HPO 的 "replay same scenario against different policies" 思路适用于我们的 setting
   - **PS-MDP 转化**：如果 PDE 求解器在某些区域不光滑（如 shock、phase transition），可以将 domain 划分为 smooth regions，用 HPO 框架处理
   - **Gradient quality analysis toolkit**：alignment/RMSE/signal ratio/cross-alignment 等梯度量度可以直接用于评估我们的 gradient estimator 质量

3. **差异化**：
   - HPO 依赖已知的 (f,c) 做 backprop，而 0616-fex 的核心是学习/近似 f 本身（通过学习 PDE solution operator）
   - HPO 的 PW gradient 来自精确 simulator backprop，而我们可能面临 approximate solver 的 bias-variance tradeoff
   - 我们的 hybrid 可能不是 discrete/continuous action，而是 discrete（物理模式选择）/functional（连续解函数）

4. **该引不该引**：如果 0616-fex 涉及 hybrid action space 或 mixed gradient estimation，应该引用；如果纯 supervised learning of PDE operators，相关性较弱。
