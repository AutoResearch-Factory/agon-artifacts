# Priors For Symbolic Regression (2304.06333)

- **Year**: 2023.04
- **Authors**: Deaglan J. Bartlett (CNRS & Sorbonne Universite / Institut d'Astrophysique de Paris), Harry Desmond (University of Portsmouth), Pedro G. Ferreira (University of Oxford)
- **Venue**: GECCO '23 Companion / arXiv
- **Keywords**: symbolic regression, model selection, minimum description length, Bayesian evidence, Fractional Bayes Factor, function prior, Katz back-off language model, cosmology
- **One-liner**: 用语言模型结构先验和 FBF 参数先验改造符号回归模型选择。

## 精华提炼

### 管线/方法

1. 论文目标不是提出新的方程生成器，而是重做符号回归候选方程的排序准则。
2. 输入是一个由任意 SR 算法产生的候选函数集合，以及观测数据 `D`。
3. 作者在实验中使用 Exhaustive Symbolic Regression (`esr`) 生成候选函数，以避免搜索算法漏掉候选式。
4. 对每个候选函数 `f_i`，先拟合其参数 `theta_i`，并计算数据似然。
5. 候选函数以树结构表示，节点包括变量、常数、二元算子和一元算子。
6. 基础比较对象包括纯 likelihood、PySR 风格 Pareto-front score、ESR 的 MDL 准则。
7. Bayesian 路线使用模型后验：`-log P(f_i | D) = -log P(f_i) - log Z(D | f_i)`。
8. 证据 `Z` 用 Laplace 近似计算，而不是只用 BIC 的大样本简化式。
9. Hessian / Fisher 信息项用有限差分计算，工具包是 `numdifftools`。
10. MDL 路线把数据拟合误差和函数编码长度放到同一信息尺度，即 nats。
11. ESR-MDL 的函数编码惩罚包括树节点数 `k`、唯一算子数 `n`、参数个数 `p`、整数常数 `c_alpha`、以及参数精度项。
12. 作者指出 MDL 可被解释成一种隐式函数先验：少节点、少算子、少参数、少复杂常数的表达式先验更高。
13. 参数先验部分的核心问题是：SR 中有成千上万个候选函数，手工给每个函数设 proper prior 不现实。
14. uniform prior 若无边界是 improper；若随意设边界，Bayes factor 会被边界宽度任意缩放。
15. Jeffreys prior 和 reference prior 也常常 improper，不能直接用于模型证据比较。
16. 论文引入 Fractional Bayes Factor (FBF) 来处理不适定参数先验。
17. FBF 思路是把全数据似然的一部分 `P(D|theta)^b` 用来把 improper prior 转成可归一化的训练后验。
18. 作者采用 `b = N^{-1/2}`，这是 O'Hagan 建议的稳健选择之一。
19. 对 uniform 初始参数先验，FBF 近似下只需一次最大似然优化。
20. 对 uniform 初始先验，证据近似变成 `(1-b) log P(D | f_i, theta_hat) + (p/2) log b`。
21. 作者也指出这个 FBF 形式不会像 ESR-MDL 那样惩罚 finely tuned parameters，这是潜在缺点。
22. 函数结构先验部分把表达式树视作一个语言序列问题。
23. 精确树概率按深度链式展开：每一层节点条件依赖所有更浅层节点。
24. 实际近似为 `n`-gram：某深度的 sibling set 只依赖有限长度的祖先节点。
25. 同一深度的不同 sibling set 被近似视为相互独立。
26. 条件概率使用 Katz back-off language model 估计。
27. back-off 让未见过的表达式仍有非零概率，因此不是只检索训练语料中出现过的方程。
28. 论文选择 `k=0` 的 Katz back-off 阈值。
29. 折扣估计使用 Good-Turing estimate。
30. 每个表达式先由 `sympy` 读取，再由 `esr` 转换成函数树。
31. 对每个 sibling，作者把左右节点分开建模：左节点概率乘以给定左节点后的右节点条件概率。
32. 训练语料是 161 条科学方程。
33. 其中 100 条来自 Feynman Symbolic Regression Database。
34. 另有 20 条 FSReD bonus equations，来自物理教科书。
35. 再加 41 条 Wikipedia "List of scientific equations named after people" 链接页中的科学方程。
36. 语料覆盖经典力学、电磁学、相对论等科学公式。
37. 训练语料中的算子包括 `+`, `-`, `times`, `div`, `pow`, `sqrt`, `exp`, `log`, `sin`, `cos`, `arcsin`, `arccos`, `tanh`。
38. 语料方程变量数从 1 到 11。
39. 作者比较 `n=1,2,3` 的 back-off 长度，发现 parent node 带来大量信息。
40. `n=2` 与 `n=3` 的差异较小，因此主实验采用 `n=2`。
41. 最终比较六种模型选择准则：Likelihood、Score、MDL、MDL+LM、MDL+FBF+LM、Bayes+FBF+LM。
42. benchmark 使用 Nguyen-8、Korns-1、Korns-4、Korns-6、Korns-7 五个标准函数。
43. 每个 benchmark 先采样 `10^5` 个 `x` 估计 `y` 的标准差。
44. 加入高斯噪声，噪声标准差取 `y` 标准差的一半。
45. 每个函数和样本量设置用 5 个随机种子独立生成数据。
46. 候选表达式由 `esr` exhaustive search 生成，复杂度上限为 8。
47. benchmark 算子基为 `{x, a, sqrt, square, sin, cos, +, times, -, div, pow}`。
48. Korns-7 的真函数含指数，但作者不加入 `exp`，因为可用 `pow` 写出更低复杂度变体。
49. cosmology 实验复用 ESR 对 Pantheon+ 超新星数据的候选函数。
50. 目标变量是 `y(x=1+z)=H^2(z)`，标准 LambdaCDM 形式为 `theta_0 + theta_1 x^3`。
51. Pantheon+ 样本包含 1590 个 Cepheid-calibrated supernovae。
52. 观测量不是 `H^2` 本身，而是通过积分计算得到的 distance modulus `mu`。
53. cosmology likelihood 是带公开协方差矩阵 `Sigma` 的多元高斯。
54. 作者复评 ESR 复杂度最高到 10 的候选函数，并列出各准则 top-4。
55. 纯 MDL 在超新星问题上选出 `theta_0 x^x`，复杂度低但物理上不自然。
56. 引入 LM function prior 后，`x^x` 类函数不再进入 top-4。
57. MDL+LM 的 top-4 包括 LambdaCDM 形式 `theta_0 + theta_1 x^3`，排第 4。
58. Bayes+FBF+LM 的 top-4 也出现与 `x^3` 相关的形式，但第一名仍是较不美观的幂函数。
59. 作者把这种问题归因于 Katz back-off 的局限：未见短语若由常见子短语组成，概率可能被高估。
60. 作者公开 back-off model code 和训练方程，链接为 `https://github.com/DeaglanBartlett/katz`。

### 关键技术细节

- Bayesian evidence 使用 Laplace 近似，不采用单纯 BIC。
- Hessian / Fisher 信息项用 `numdifftools` 有限差分。
- FBF 的 split fraction 设为 `b=N^{-1/2}`。
- 函数结构 prior 是 `n=2` 的 Katz back-off language model。
- 训练语料规模是 161 条科学方程：100 FSReD + 20 bonus + 41 Wikipedia-derived equations。
- benchmark 函数数为 5：Nguyen-8、Korns-1、Korns-4、Korns-6、Korns-7。
- benchmark 每个设置有 5 个随机种子，先用 `10^5` 个样本估计输出尺度。
- benchmark 噪声为输出标准差的 `0.5` 倍。
- benchmark exhaustive search 复杂度上限为 8。
- cosmology 应用使用 Pantheon+ 的 1590 个超新星样本和公开协方差矩阵。
- cosmology 中 ESR 候选函数来自复杂度最高 10 的搜索。
- cosmology 所有 top functions 的 mean Mahalanobis distance 在 0.8783 到 0.9223 之间。

### 核心结果

| Setting | 关键发现 | 解释 |
|---------|----------|------|
| benchmark likelihood-only | 没有任何 case 把真函数放进 top-2 | 噪声下高复杂度函数可任意贴合，过拟合严重 |
| benchmark score | Nguyen-8 上表现较好，但很多 case 真函数不在 Pareto front | Pareto-front score 无法处理略微离开前沿的真函数 |
| benchmark MDL | 整体最稳，Nguyen-8 总能选中真函数 | 信息长度惩罚有效抑制过拟合 |
| benchmark MDL+LM | Korns-7 上超过其他方法 | 科学方程语料偏好帮助识别更像真实科学关系的形式 |
| cosmology MDL | top-1 是 `theta_0 x^x` | 低复杂度但物理上不合理 |
| cosmology MDL+LM | top-4 全部去掉 `x^x`，LambdaCDM 排第 4 | 结构先验提升物理合理性 |
| cosmology FBF+LM | 去掉 `x^x`，但 top-1 仍不美观 | Katz back-off 对未见组合可能高估 |

Surprising/反直觉点：

- 简单 MDL 在标准 benchmark 上最强，但在真实 cosmology setting 中会偏好物理上怪异的 `x^x`。
- 语言模型先验不是普遍提升 benchmark；Nguyen-8 上反而可能让真函数跌到第 3。
- domain prior 的价值主要体现在真实物理应用中，而不是人工 benchmark 上。
- LM prior 会偏好除以参数而不是乘以参数，作者指出这可能来自训练语料偏置。
- FBF 修复 improper parameter prior 的形式问题，但 uniform 初始 prior 下会丢掉对 finely tuned parameters 的部分惩罚。

### 明确局限

- 论文重点是模型选择，不解决候选函数搜索的召回率问题。
- 实验中使用 `esr` exhaustive search 是为了控制变量；真实大规模 SR 中 exhaustive search 可能不可行。
- benchmark 只含 5 个函数，覆盖面明显小于当前常用 SR benchmark suite。
- benchmark 结果主要以图展示，表格中没有逐设置给出完整数字。
- 函数等价性由 `esr` simplification 后再人工 visual inspection 判断，仍有主观步骤。
- 训练语料只有 161 条科学方程，规模小，且偏物理。
- 结构 prior 的好坏高度依赖语料；换领域需要换 corpus。
- Katz back-off 只是浅层语言模型，作者自己指出它会高估由常见子短语拼成的未见表达式。
- `n=2` 只建模 parent-child 关系，不能完整捕捉深层表达式结构。
- FBF 部分只采用 uniform 初始参数 prior；没有系统比较其他弱信息 prior。
- 论文没有给出外部独立复现报告。
- 公开的是 back-off model code 和训练方程，不等于完整 benchmark pipeline artifact。
- 没有做 train/test split 与 Bayesian/MDL 复杂度惩罚之间的系统对照；作者明确说这值得进一步研究。
- 对 0616-fex 若目标是新式 equation generator，这篇不是直接 generator baseline；它更适合作为 ranking / prior / selection baseline。

### 跟我们的关系

- 对 `0616-fex` 的直接撞车点：如果我们的 claim 是“用先验知识改进符号回归方程排序”，这篇已经做了强先例。
- 具体撞车 claim 1：把科学公式语料转成函数结构 prior，本文用 Katz back-off LM 已覆盖。
- 具体撞车 claim 2：把 Bayesian evidence/MDL 作为 SR 统一 model-selection objective，本文做了系统比较。
- 具体撞车 claim 3：用 FBF 处理 SR 中 improper parameter prior，本文给出清晰公式化方案。
- 不撞车点：本文不做新的候选生成器，不做 neural proposal，不做 LLM-based formula extraction，不做多模态文档抽取。
- 可借用工具：`esr` exhaustive candidate set、Laplace evidence、FBF evidence approximation、Katz back-off over expression trees。
- 可借用实验：Nguyen/Korns noisy benchmark、Pantheon+ cosmology case、比较 Likelihood/Score/MDL/MDL+LM/FBF+LM 的 ablation。
- 差异化方向 1：把 prior 从 161 条手工科学方程扩展到更大、更可审计的领域语料。
- 差异化方向 2：用结构更强的 tree model / grammar model / neural prior 替代 Katz back-off，并专门测试未见组合校准。
- 差异化方向 3：把模型选择和候选生成打通，例如 proposal model 采样函数 prior，再用独立 verifier 做 veto。
- 差异化方向 4：做跨领域 benchmark 和 out-of-distribution extrapolation，不只在 physics/cosmology 上验证。
- 差异化方向 5：明确分离 generator 与 verifier，避免自产自检，提高 R2。

## Read by: 0616-fex (2026-06-16)

本次按 `0616-fex` 视角读，这篇最重要的启发是：公式发现不能只看拟合误差，必须把“表达式是否像该领域会出现的公式”变成可计算的 prior。本文已经把这个 prior 做成了 expression-tree 上的 Katz back-off LM，并和 MDL、Bayesian evidence、FBF 放到同一套 model-selection 框架中。

撞车评估：若 `0616-fex` 的核心 claim 是“给符号回归引入领域公式先验来改进排序”，本文高度撞车；若我们的核心是“从文档/实验/数据中抽取候选公式、或用更强模型生成公式”，本文只撞模型选择层，不撞生成层。最直接需要引用的是 2304.06333 和 2211.11461；若涉及 Bayesian posterior / marginal likelihood，还应关注 2512.10849（Bayesian Symbolic Regression via Posterior Sampling，2025-12-11，近 6 个月 arXiv 检索发现）以及 2606.06567 的 UQ 综述。

可用方法：把候选公式统一转换成树；用领域语料估计 parent-child / sibling 结构概率；把该概率替换 MDL 的 `k log n` 复杂度项；对参数先验不适定问题使用 FBF；最后用真实任务检查 prior 是否只是在 benchmark 上过拟合。本文的负面教训也很关键：浅层 Katz back-off 会偏好一些“局部片段常见但整体奇怪”的表达式，因此 verifier 不能只看 prior。

差异化建议：`0616-fex` 应该明确把 generator、prior scorer、independent verifier 分开。生成器可以是神经/LLM/grammar/ESR 任意来源；prior scorer 学领域表达式分布；verifier 用 held-out data、物理约束、维度一致性、外推和不确定性评估否决候选。这样能避开本文 R2 较弱的自产自检问题，也能把贡献从“又一个结构先验”推进到“可审计的公式发现闭环”。
