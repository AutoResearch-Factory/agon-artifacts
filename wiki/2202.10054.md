# Fine-Tuning can Distort Pretrained Features and Underperform Out-of-Distribution (2202.10054)

- **Year**: 2022.02
- **Authors**: Ananya Kumar, Aditi Raghunathan, Robbie Jones, Tengyu Ma, Percy Liang (Stanford University)
- **Venue**: ICLR 2022 (LaTeX source uses `iclr2022_conference.sty`/`.bst`)
- **Keywords**: fine-tuning, linear probing, transfer learning, out-of-distribution generalization, feature distortion, pretrained representations, implicit regularization, two-layer linear networks, distribution shift, LP-FT
- **One-liner**: Full fine-tuning distorts pretrained features and can be worse than linear probing OOD at every step of training; linear-probe-then-fine-tune (LP-FT) fixes this both theoretically and empirically.

## 精华提炼

### 管线/方法
- Predictor parameterization: f_{v,B}(x) = v^T g_B(x), with feature extractor g_B (a deep net in experiments, a linear map in theory) initialized from a pretrained B_0, and linear head v.
- Two baselines compared: (i) **linear probing (LP)** — freeze B=B_0, train only v to convergence (logistic loss for classification / squared loss for regression); (ii) **full fine-tuning (FT)** — jointly update v and B via gradient descent from (v_init, B_0), v_init random/zero.
- Theory pipeline (linear/regression setting): feature extractor B ∈ R^{k×d}, head v ∈ R^k, well-specified ground truth w*=B*^Tv*. Pretrained B_0 within principal-angle distance ε of B*. Training data X spans an m-dim subspace S with 1≤m<d−k (overparameterized: OOD directions exist outside the training span, and there's no unique global minimizer). Gradient flow (vanishing-step-size continuous-time GD) makes the FT/LP trajectories analytically tractable.
- Main theorem (Thm 3.1): lower-bounds the FT trajectory's OOD error **at every time t** via (a) the principal angle between rowspace(B_0) and the orthogonal complement of the training span, and (b) the "head alignment error" φ²=|(v_0^Tv*)²−(v*^Tv*)²|. Proved via a "balancedness" invariant (from Du et al. 2018 / Arora et al. 2018 two-layer-linear-network implicit-regularization literature) that couples how v and B move jointly under FT.
- Comparison theorems: LP's OOD error upper bound → 0 as B_0→B* (Thm 3.2/formal Thm C.1), while FT's OOD lower bound stays bounded away from 0 — LP asymptotically dominates FT OOD given "good" features + large enough shift. Conversely FT reaches exactly 0 ID error (fits training data exactly), while LP's ID error is strictly positive whenever B_0 is imperfect — FT dominates LP ID (Prop 3.3).
- **LP-FT recipe** (the proposed practical algorithm): (1) linear-probe on frozen B_0 to get a converged head v_LP^∞; (2) initialize full fine-tuning at (v_LP^∞, B_0) instead of a random head, then run ordinary FT. Compute cost ≈ vanilla FT since the LP step is a cheap convex fit on frozen features.
- Empirical pipeline: 10 ID→OOD benchmark pairs (Breeds-Living17, Breeds-Entity30, DomainNet sketch→{real,clipart,painting}, CIFAR-10→STL, CIFAR-10→CIFAR-10.1, FMoW NA→{Africa,Europe}, ImageNet→{ImageNetV2,ImageNet-R,ImageNet-A,ImageNet-Sketch}). Backbones: ResNet-50 from MoCo-v2 (unlabeled ImageNet), CLIP ResNet-50/ViT-B16 (WebImageText), MoCo-TP (Ayush et al., unlabeled FMoW satellite imagery). Per dataset: 6-LR sweep (3 for ImageNet), cosine schedule, ID-validation model selection, 3 seeds averaged with 90% CI (1 seed for ImageNet due to cost).
- Mechanistic diagnostic: Euclidean distance ‖g_B(x)−g_{B0}(x)‖ of penultimate-layer features before/after adaptation, averaged separately over held-out ID vs OOD examples, compared between FT and LP-FT — a direct test of the "distortion is asymmetric" mechanism, not just an accuracy correlation.

### 关键技术细节
- Feature-extractor distance: d(B,B')=min_U‖B−UB'‖_2 over k×k rotation matrices U (handles the rotation ambiguity of the B*,v* factorization).
- Principal angle cos∠_max(A,B) = σ_r(E^TF), the smallest of the r=min(dim A,dim B) singular values of E^TF for orthonormal bases E,F — this "largest principal angle > 0" is the non-degeneracy/coverage condition used throughout.
- Main bound (informal): √L_OOD(FT_t) ≥ √σ_min(Σ)·[cos∠(R_0,S^⊥)/√k · min(φ,φ²/‖w*‖)/(1+‖w*‖)² − ε], holding for **all** t — so early stopping cannot rescue FT.
- φ² is provably large under standard random/zero head init (Gaussian anti-concentration lemma: v_0~N(0,σ²I) ⟹ φ²≥Ω(‖v*‖²) w.h.p.) — this randomness is the mechanistic root cause of distortion, since FT's balancedness invariant v_tv_t^T−B_tB_t^T=const couples head movement to feature movement.
- LP-FT proposition (perfect-feature case, B_0=UB*): with v_init set to the LP solution, the training-loss gradient at (v_LP^∞,B_0) is exactly zero for both v and B — FT literally does not move, L_OOD stays 0, vs. L_OOD>0 for all t under random init.
- ImageNet recipe: CLIP ViT-B/16, 10 epochs (5+5 split for LP-FT to equalize compute), batch 128; LR grids {1e-4,3e-4,1e-3} FT / {0.01,0.03,0.1} LP stage / {1e-5,3e-5,1e-4} FT stage of LP-FT. Other 9 datasets: ResNet-50, 20 epochs (50 for smaller DomainNet/FMoW splits ~20K examples), ℓ2-regularized logistic regression for LP.
- Extra Living-17 ablations beyond LP/FT/LP-FT: 10× head LR, L2-SP-style regularization toward B_0 (λ∈{1,0.1,0.01,0.001,1e-4}, 30-point grid), side-tuning (Zhang et al. 2020: frozen backbone + trainable side network), training-from-scratch (numbers from Santurkar et al. 2020, 450 epochs).
- Effective-robustness framework (borrowed from Taori/Miller): plots OOD accuracy vs ID accuracy across 3 backbones on Living-17 to check whether a method's OOD gain is simply predictable from its ID gain (confounded) or a genuine additional effect.

### 核心结果
- 10-dataset averages: FT 85.1% ID / 59.3% OOD; LP 82.9% ID / 66.2% OOD; **LP-FT 85.7% ID / 68.9% OOD** — LP-FT pareto-dominates both baselines on average (+0.6% ID / +9.6% OOD vs FT; +2.8% ID / +2.7% OOD vs LP).
- ID: FT beats LP on 5/6 reported sets (exception: DomainNet, small ~20K-example split). LP-FT wins/ties ID on 5/6 (only FMoW falls between LP and FT).
- OOD: LP beats FT on 8/10 sets; the 2 exceptions (CIFAR-10.1, ImageNetV2) are exactly the shifts deliberately constructed to be "minute" — consistent with theory requiring a genuinely large shift. LP-FT matches/beats both on 10/10 OOD sets.
- Early-stopping-on-OOD ablation (an unfair advantage to FT): LP still wins on average (67.1% vs 61.3%) — empirically confirms the "for all t" claim, ruling out "FT would win with the right stopping point."
- Feature-drift on Living-17: FT moves ID features 0.0188±0.0001 vs OOD features 0.0167±0.0001 (ID>OOD as predicted); LP-FT moves both ~20× less. Across all reported cells, ID-drift > OOD-drift in 9/10 FT cases — direct mechanistic confirmation, not just an accuracy-level correlation.
- Boundary checks where theory predicts FT should win, and it does: worse features (MoCo-v1, 10% worse) flips the Living-17 ranking (FT 96%/71% > LP 92%/66%); ID≈OOD (CIFAR-10→10.1) has FT winning OOD (92.3% vs 82.7%). LP-FT still wins outright in the latter case (93.5%).
- Architecture-robustness check: swapping to CLIP-ResNet-50 / CLIP-ViT-B16 on Living-17 reproduces the same pattern (FT worse OOD by 11.3pp / 6.1pp); LP-FT closes 86% / 75% of that gap while still winning ID.
- Heuristic comparison on Living-17: 10×-head-LR, L2-SP regularization, and side-tuning each partially close the OOD gap (80.0–81.0%) but none matches LP-FT's 82.6% OOD while also matching its 97.8% ID.
- **Counter-intuitive headline finding**: fine-tuning — strictly more expressive and roughly equal-compute to linear probing — can be OOD-worse than freezing the backbone entirely, at *every* point along its optimization trajectory, contradicting the common "FT can only help, just early-stop" intuition.

### 明确局限
- Theory covers only linear feature extractors + squared loss + gradient flow (vanishing step size); real backbones are deep nonlinear nets with finite-step SGD/Adam and cross-entropy loss — paper explicitly flags nonlinear activations, layerwise LRs, explicit regularization as open extensions.
- The core lower bound requires cos∠(R_0,S^⊥)>0, argued to hold "almost surely" under random-subspace assumptions but not empirically verified on real pretrained CNNs.
- No true upper bound on FT's OOD error is given for the general imperfect-feature case — only the perfect-feature case (Prop 4.4) is fully closed-form; the general result is asymptotic (B_0→B*) or needs a Gaussian-covariate assumption for a quantitative threshold (Thm C.2).
- LP-FT is explicitly not claimed as SOTA — described as "a simple, principled way to get good ID and OOD accuracy" meant to validate the theory, not to be the best possible robustification method.
- All OOD evaluation is on curated benchmark shift pairs (Breeds/DomainNet/CIFAR-STL/FMoW-geo/ImageNet-variants); no real-world deployment, streaming, or temporal-shift setting; no non-vision domain (time-series, tabular, geophysical) is touched.
- Purely supervised single-task image classification; no multi-task/generative/sequence modeling; no causal-inference angle at all — "distribution shift" here means covariate/domain shift, not a causal-graph or intervention shift.
- ImageNet experiments use only 1 seed per config (cost), so those OOD cells report no CI (marked "(-)"), weaker evidence than the 3-seed/90%-CI cells used elsewhere.
- I found no explicit code-release URL in the tex sources read (abstract/intro/experiments/appendix) — cannot confirm a public code artifact from the paper text itself.

### 跟我们的关系
Pure computer-vision transfer-learning paper (ResNet/ViT/CLIP backbones on natural-image OOD benchmarks) — **zero direct overlap** with causal discovery (PCMCI+/LKIF), early-warning-signal theory, or severe-convective-storm nowcasting; not a competing paper for any core claim of a causal-indicator project. Its only relevance is as an off-the-shelf **fine-tuning recipe (LP-FT)**: if a project ever moves from training a small CNN from scratch to initializing it from a pretrained feature extractor and fine-tuning on a small labeled event set, LP-FT is a near-zero-cost, theoretically-grounded default that guards against the pretrained features being "destroyed" by naive full fine-tuning under limited labeled data. The **effective-robustness framework** (plotting a target metric against a baseline metric across many operating conditions to check whether the gain is a genuine additional effect vs. simply correlated with/confounded by baseline strength) is a reusable falsification lens for any project that fuses a new feature source with a strong existing baseline. The **MoCo-TP / geography-aware self-supervised pretraining** citation (Ayush et al., unlabeled FMoW satellite imagery) is a same-domain (geospatial-raster) precedent for self-supervised pretraining on unlabeled large-scale gridded data, relevant only if a project wants a pretrained-then-fine-tuned backbone instead of from-scratch training on scarce labeled events. No citation obligation either way — this is a technique reference, not prior art to be distinguished from.

## Read by: 0710-causal-scs (2026-07-13)

本文是纯视觉迁移学习理论+实验论文（ResNet/ViT/CLIP 骨干网络在自然图像 OOD 基准上做微调 vs 线性探测），与 causal-scs-indicator 的核心命题（滚动因果效应估计的不稳定性/跳变是否携带超出原始状态诊断量的事件信息）**完全不撞车**——没有因果推断、没有早期预警信号、没有对流风暴场景，不构成任何 claim 的竞争性前置工作。

可借鉴之处：
1. **LP-FT 微调技巧**：若未来轮次把 v10/v10b 中"从零训练的小型 3D CNN"换成基于预训练大气/卫星特征提取器的微调（例如在大量无标注 HRRR/再分析场上做自监督预训练，再在稀缺的 SCS 标注事件上微调——v9 README 已明确指出"32 cases per fold"的小样本瓶颈正是 LP-FT 类方法设计初衷所针对的场景），LP-FT（先线性探测再全量微调）是一个几乎零额外成本、有理论保证的默认选择，能防止小样本微调把预训练特征"搞坏"而在跨区域/跨季节/跨年份泛化时变差。
2. **Effective robustness 分析框架**（借自 Miller/Taori 2107.04649，本文直接使用）：可作为项目现有 gate 之外的额外证伪视角——把"(D,J) 融合特征相对 raw-only 的增量"对"raw-only 基线本身的强弱"作图，检验因果特征的增益是否只是在 raw 基线本就强的区域"搭便车"，还是在 raw 基线弱的区域也有独立增益。这与项目 v7/v9/v10 反复检验"raw EWS 是否已经吃掉了所有信息"这一问题在方法论上同构，是一个现成的可视化/统计工具。
3. **Geography-aware self-supervised pretraining**（Ayush et al., 2011.09980，本文 MoCo-TP 预训练的来源）：为"在无标注卫星/大气栅格数据上做自监督预训练"提供了同域（地理栅格数据）先例，仅在项目决定不再从零训练 CNN 时才用得上。

差异化：causal-scs-indicator 用因果发现方法 (PCMCI+/LKIF) 提取的滚动因果效应不稳定性作为事件先兆特征，在严格配对对照、sham 条件化、exchangeable-null 校准下检验增量价值；本文完全不涉及因果推断，其"分布偏移"是协变量/领域偏移而非因果图/干预偏移。无引用必要性，仅作为未来可能用到的微调技术参考保留。
