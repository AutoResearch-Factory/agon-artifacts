# Adaptive Algorithms for Nonconvex Bilevel Optimization under PŁ Conditions (2512.24291)

- **Year**: 2025.12
- **Authors**: Xu Shi, Yinglin Du, Rufeng Xiao, Rujun Jiang (School of Data Science, Fudan University)
- **Venue**: arXiv
- **Keywords**: Nonconvex bilevel optimization, Adaptive method, Polyak-Łojasiewicz condition, First-order oracle, Near-optimal complexity, AdaGrad-Norm, Auto-conditioned gradient method
- **One-liner**: 提出首个完全不依赖问题参数先验知识的全自适应一阶双层优化算法 AF²BA 和 A²F²BA，在 PŁ 条件下达到 Õ(1/ε²) 近最优一阶 oracle 复杂度。

## 精华提炼

### 管线/方法
完整的 AF²BA / A²F²BA 算法 pipeline（双循环结构）：

1. **输入**：初始点 x₀, y₀, z₀，初始步长 a₀, b₀, c₀ > 0，scale 参数 α > 1，初始 Lipschitz 因子 L₀,₁, L₀,₂ > 0，误差容忍度 ε > 0
2. **外层循环**（T = 1/ε² 次迭代）：
   - **Step 1**：求解下层子问题 min_z g(x_t, z) → 得到 ẑ ≈ y*(x_t)
     - AF²BA 用 **AdaG-N**（AdaGrad-Norm）：α_{k+1}² = α_k² + ||∇h(x_k)||²，x_{k+1} = x_k - (1/α_{k+1})∇h(x_k)
     - A²F²BA 用 **AC-GM**（Auto-Conditioned Gradient Method）：γ_{k+1} = max{L₀,...,L_k}，x_{k+1} = x_k - (1/αγ_{k+1})∇h(x_k)，L_{k+1} 用差分估计
   - **Step 2**：求解惩罚子问题 min_y σf(x_t, y) + g(x_t, y) → 得到 ŷ ≈ y_σ*(x_t)，σ = ε
   - **Step 3**：计算近似超梯度 ∇̂φ = ∇_x f(x_t, ŷ) + (∇_x g(x_t, ŷ) - ∇_x g(x_t, ẑ))/σ
   - **Step 4**：自适应更新 x：a_{t+1}² = a_t² + ||∇̂φ||²，x_{t+1} = x_t - (1/a_{t+1})∇̂φ
3. **输出**：ε-stationary point（min_t ||∇φ(x_t)|| ≤ O(ε)）

### 关键技术细节
- **PŁ 条件**：2μ_h (h(x) - h*) ≤ ||∇h(x)||²，比强凸弱很多，允许非凸函数和多个极小值点
- **Penalty 方法**：利用 φ_σ(x) = min_y {f(x,y) + (g(x,y) - g*(x))/σ} 作为 φ(x) 的平滑近似
- **超梯度存在性**：在 penalty function g_σ = σf + g 对所有 0 ≤ σ ≤ σ̄ 满足 PŁ 条件下，∇φ(x) 存在且 Lipschitz 连续
- **AdaG-N 两阶段**：Stage 1 步长小于阈值 C_b 时 O(1/ε_z²) 迭代，Stage 2 步长超过阈值后 O(log(1/ε_z²)) 迭代收敛
- **AC-GM 线性收敛**：h(x^{k+1}) - h* ≤ C_h (1 - μ_h/(8 max{L₀, L_h}))^{k+1} (h(x⁰) - h*)，最优缩放因子 α = 2
- **超梯度误差 bound**：||∇φ - ∇̂φ|| ≤ C̄_σ σ + (L_f dist(ŷ, Y_σ*) + (L_g/σ)dist(ŷ, Y_σ*) + (L_g/σ)dist(ẑ, Y*))
- **参数设置**：σ = ε，ε_z = ε_y = ε²，T = 1/ε²
- **Lipschitz 常数**：L_φ = (L_f + l_f ρ_g/μ)(1 + L_g/μ)²

**Surprising/反直觉结果**：
- AF²BA 的 oracle complexity O(1/ε⁶) 比非自适应的 F²BA 的 Õ(1/ε²) 差很多（差 ε⁻⁴），但 A²F²BA 通过换用 AC-GM 子程序弥补了这个差距，回到了 Õ(1/ε²)
- AC-GM 在 PŁ 条件下取得了线性收敛率（含显式因子），而原始 AC-PGM 论文未给出显式收敛因子
- 自适应步长策略本质上不影响外层的迭代复杂度 O(1/ε²)，只影响内层解子问题的开销

### 明确局限
1. **作者自述**：目前是双循环算法，期望 future work 做 single-loop adaptive；目前仅限 deterministic setting，未扩展到 stochastic；仅限 PŁ 条件，未扩展到更一般的 KL 条件
2. **无真机实验**：纯理论论文，没有任何数值实验或仿真验证
3. **无标准 benchmark 对比**：只给了理论复杂度对比表，没有跑任何 bilevel optimization benchmark（如数据超清洗、元学习等）
4. **Lipschitz 假设仍强**：虽然不需要知道具体值，但 Assumption 1 要求 f 和 g 满足 Lipschitz 梯度/Hessian 连续性，这在实际中不一定成立
5. **σ = ε 的选择**：penalty parameter 直接设为 ε，这是一个理论上方便的选择，但没有讨论实践中如何调优
6. **双循环开销**：每次外层迭代都要重新求解两个子问题到高精度（ε_z = ε_y = ε²），这对于大规模深度学习问题可能非常昂贵
