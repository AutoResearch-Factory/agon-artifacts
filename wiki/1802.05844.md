## 精华提炼

### 管线/方法

这是一篇理论+实证论文，管线分两层：理论推导链条，和验证该链条的实证设计。

**理论推导链条**：
1. 把经典 filter 特征选择目标 `S* = argmax_{S⊂F} P(C|S)` (Eq.1_0) 通过条件似然的 KL 散度分解 (Eq.11-16)，重新表述为互信息最大化 `S* = argmax I(C;S)` (Eq.3_2)，并用 Fano/Hellman 的贝叶斯错误率上下界 (Eq.3_4-3_5) 证明这个目标等价于最小化贝叶斯错误率。
2. 证明 **Theorem 3-1**：在忠实性假设下，`MB(C)`（父+子+配偶）是 `I(C;S)` 在所有 `S⊆F` 上的唯一最大化子集（分三种情况用互信息链式法则证明，覆盖 `S⊆MB`、`S⊆F\MB`、`S` 跨越两者三种情形）。
3. 证明 **Corollary 3-2**：Kohavi & John (1997) 经典意义下的"强相关特征集合"在忠实性假设下**恰好等于** `MB(C)`——由此因果特征选择（MB 发现）与非因果特征选择（强相关特征选择）被证明是**同一个目标函数**的两种求解路径，不是两个不同的目标。
4. 把主流非因果方法（MIM、MIFS/mRMR/FCBF、JMI/CIFE/CMIM、RelaxMRMR）统一表示为对 `I(X;C|S)=I(X;C)-I(X;S)+I(X;S|C)` (Eq.28_2) 做不同阶数的朴素贝叶斯结构假设（k-DB，k=0/1/2）线性低阶近似，见 Table 1 (tb-yu1)。
5. 把主流因果方法（IAMB/Inter-IAMB "同时发现"族；MMMB/HITON-MB/IPC-MB/STMB "分治"族）表示为求解**同一个** `I(X;C|S)` 前向-后向贪心最大化，但不做结构假设——即对完整/高阶条件集做精确条件独立检验，只是"同时发现"族条件化在整个当前 `S` 上，"分治"族只在 `S` 的子集上搜索（更省样本但仍指数级最坏复杂度）。
6. 推导条件独立检验（G²/卡方）的最小可靠样本量公式 `N ≥ ξ·r_X·r_C·r_S` (Eq.7-01, ξ=5, Agresti 经验值)，其中 `r_S` 是条件集内所有变量取值数的乘积——非因果方法因为条件集阶数 ≤2 而样本需求低，因果方法条件集阶数达到 `|S|`（最坏情况下整个 MB 大小）而样本需求指数增长。
7. 推导误差上界链条（Theorem 7-1, Corollary 7-2）：`P_err(MB) ≤ 1/2·H(C|MB) ≤ P_err(近似MB/PC超集) ≤ 1/2·H(C|PC)`——这是**总体层面**（无限样本极限）的严格排序，对有限样本估计误差不作任何声明。
8. 用合成 ALARM 贝叶斯网络（37节点，已知真实图）和 25 个真实数据集（分四组：大样本低维、小样本高维、多分类、类不平衡）做实证验证，用 NBC/KNN 分类器测预测准确率、AUC、Kappa、特征数、运行时间。

### 关键技术细节

- **k-DB 朴素贝叶斯家族**：0-DB=朴素贝叶斯（特征间给定C条件独立）、1-DB=TAN（每特征至多1个额外父节点）、2-DB=RelaxMRMR 的假设（至多2个额外父节点）。k 越大，非因果方法离"无结构假设"越近，但计算成本从 O(n) 涨到 O(ψ³n)。
- **非因果方法的因果解释**（Table tb-yu1，用 Properties 1-4 证明）：MIM/MIFS/mRMR/FCBF 只偏好 `PC(C)`（直接因/直接果），不主动发现配偶；JMI/CIFE/CMIM/RelaxMRMR 额外含类条件相关项 `I(X;F_i|C)`，因此也偏好配偶（Property 3：配偶 `I(X;C)=0` 但 `I(X;Y|C)>I(X;Y)`，条件化后才显形）。
- **因果 MB 发现两族算法**：(1) 同时发现族（IAMB/Inter-IAMB）条件化在完整 `S` 上，平均复杂度 O(n|S|)，样本需求 ∝ `r_{S_max}`；(2) 分治族（MMMB max-min heuristic / HITON-MB interleaving heuristic / IPC-MB、STMB backward heuristic）用 γ-greedy 子集枚举找 PC，再找配偶，复杂度 O(n|S|²2^|S|)，需要"对称性校正"消除 PC 发现阶段混入的假阳性后代节点（Figure 4-new1 的 D 节点例子：C→B→D 与 C→B←A→D 两条路径共同导致 D 在未条件化到 {A,B} 之前一直表现为与 C 相关）。
- **样本量公式**（Eq.7-01）：`N ≥ 5·r_X·r_C·r_S`，`r_S=Π_{F_i∈S} r_{F_i}`——这是离散列联表 G² 检验的经验法则（Agresti 2011），专为离散变量设计，不直接适用于连续变量的偏相关/CMI 检验。
- **实验设置的一个方法论不对称点**：非因果方法的选择特征数 `ψ`（例如 mRMR/CMIM/JMI）在几乎所有实验中都是**事后网格搜索最优值**（ψ∈{5,10,...,25或40}，报告最高准确率那组），而因果方法的 MB/PC 大小由算法自动确定、不需要调参——论文结论部分承认"如何为非因果方法选择合适的 ψ 是个难题"，但正文实验并未用固定/交叉验证 ψ 做对照，这给非因果方法一个真实部署中不可行的 oracle 优势。
- **数据集**：合成 ALARM（37 节点，"HR"节点真实 MB=1父+4子+3配偶=8变量）；大样本低维组（mushroom/kr-vs-kp/madelon/gisstee/spambase/bankruptcy，3K-7K样本，22-5000特征）；小样本高维组（prostate/dexter/arcene/dorothea/leukemia/breast-cancer，72-800样本，6K-100K特征）；多分类组（connect-4/splice/waveform/landsat 3K-67K样本 vs. lung/lymph/NCI9 60-96样本，3-9类）；类不平衡组（hiva/ohsumed/acpj/sido0/thrombin/infant，少数类占比1.3%-7.55%）。

### 核心结果

- **ALARM N=5000**：MMMB/HITON-MB 精确找到真实 MB（precision=recall=1），与 TrueMB 打平最高准确率（KNN 98.99%）；mRMR/CMIM/JMI 只找到部分 MB（precision/recall 0.625-0.75）但准确率仍具竞争力（98.17-98.67%），因为它们总能拿到完整 `PC(C)`（5/5）——在这个具体网络里，PC 本身已接近饱和可达准确率，配偶贡献的增量很小。
- **ALARM N=50（小样本）**：mRMR 在全部8个方法中准确率最高（KNN/NBC 95.40%/95.94%）——不是因为找到更多真 MB，而是因为假阳性最少（3个方法 FP 数对比中最低）；IAMB 的 recall 从 N=5000 时的 0.6875 崩到 N=50 时的 0.1250（样本量降100倍，recall跌82%）。
- **真实数据大样本低维组**：因果与非因果接近，NBC下因果法4/6略胜，KNN下 MMMB/HITON-MB 在2/6最优。
- **真实数据小样本高维组**：非因果方法（mRMR/CMIM/JMI）在准确率和 Kappa 稳定性上几乎全面胜出；MMMB/HITON-MB 在 dorothea/leukemia 等3/6数据集上因组合爆炸直接算不出结果（表中"-"）。论文原话："this validates our analysis of sample requirement in Section 5"。
- **多分类组**：大样本4个数据集（connect-4/splice/waveform/landsat, 3K-67K样本）因果法有竞争力甚至更优；小样本3个数据集（lung 73/lymph 96/NCI9 60样本，7-9类）**全部5个因果算法输出"-"（无结果）**，只有非因果方法能跑通。
- **类不平衡组（用 AUC 作为指标，与我们 pilot 的指标一致）**：因果与非因果**几乎打平**，且都表现平庸（例如 hiva 数据集 NBC下8个方法 AUC 全部落在 0.54-0.68 区间）。论文原话："both the causal feature selection methods and non-causal feature selection algorithms cannot deal with class-imbalanced datasets well"——这是本文唯一一个**因果/非因果分野没有出现**的 regime，与小样本高维组形成对照。
- **运行时间**：非因果方法比分治族因果方法快1-3个数量级（如 gisstee 数据集：JMI 52秒 vs HITON-MB 107,870秒）。

### 明确局限

**作者自己承认的**（结论section列出5点未来挑战）：小样本高维、类不平衡、大MB导致的组合搜索爆炸、非因果方法ψ超参数难选、因果FS用于local-to-global贝叶斯网络学习的效率问题。

**我读出的、对我们更关键的局限**：
1. **全文零时间序列、零动力学系统、零"趋近分岔"概念**——所有数据要么来自固定 i.i.d. 分布的合成 BN，要么是静态真实表格数据。论文里"因果方法需要更多样本"的全部证据来自"样本量N本身太小"，不是"因为动力学趋近临界点导致条件独立检验的有效信噪比下降"。这意味着本文的小样本证据只能作为"CI检验功效随条件集阶数增长而下降"这一统计机制的**另一个体现**，不能直接当作"causal conditioning 在 fragility regime 下变弱"的证据——两者是同一机制家族的不同触发条件，需要在动力学 regime 里独立验证。
2. **ψ 的 oracle 调参**（见上）系统性高估非因果方法在小样本/高维组的优势幅度，论文没有用固定 ψ 或交叉验证 ψ 的消融实验来隔离这个混淆。
3. **因果方法只覆盖离散数据上的 MB/PC 判别算法族**（IAMB系、MMMB/HITON-MB系），CI 检验全部是 G²/卡方检验，样本公式 `N≥ξ·r_X·r_C·r_S` 专为离散列联表设计。PCMCI+ 依赖的偏相关/条件互信息检验是连续变量、非参或高斯假设下的检验，有限样本行为不同（例如偏相关的方差随条件集维度增长的方式与离散列联表的自由度增长方式不是同一个函数形式），本文公式只能类比迁移，不能直接套用量级。
4. **没有任何 regime 报告过因果方法的判别力被打到 chance 水平**——即使在最不利的小样本高维组，因果方法的绝对 accuracy 通常仍在 70-90% 区间（不是随机水平 50%）。这与我们 pilot 里 causal AUROC=0.482（≈chance）的失效程度有明显量级差距，提示 fragility regime 的失效可能比本文记录的任何经典 regime 都更极端，或者 pilot 结果里还混入了本文完全没涉及、reviewer 已指出的额外混淆轴（非线性、隐藏驱动、窗口参数等）。

### 跟我们的关系

**撞不撞车**：不撞车，但是**必读的机制理论源头**。这篇论文完全在 i.i.d. 静态表格数据领域，零涉及时间序列因果发现、动力学系统、分岔/regime-shift、早期预警指标——我们的应用场景（PCMCI+ 在趋近分岔的动力学时间序列上做 fragility 判别）在这条文献脉络里完全空白，不存在方法或应用层面的直接竞争。

**回答本轮问题 (a)**——这些经典框架是否形式化了"causal/invariant 预测器在某些 regime 下系统性弱于非因果预测器"及其边界条件：**部分形式化，但机制和我们设想的不同**。本文没有触及"不变性/跨环境泛化"意义上的 causal（不是 ICP/anchor regression/IRM 那一支，全文没有"环境"、"分布偏移"、"intervention"这些词），而是纯粹在单一固定分布内比较"精确 MB 条件化"vs"低阶边际/pairwise 近似"两种同一目标函数的求解策略。它给出的边界条件是**样本量 N 相对条件集势 `r_S` 的比值**：`N` 大、`r_S` 小（即MB不太大、变量基数不太高）→ 因果法占优或打平；`N` 小、`r_S` 大（高维+小样本）→ 非因果法占优；**类不平衡是第三种 regime，两者同时失效、不发生分野**——这对我们很重要：不是所有"困难 regime"都会制造 causal vs non-causal 的 Delta，有些困难 regime（如不平衡）是两类方法共同的盲区。这提示我们在设计 fragility regime 对照实验时，应该同时准备一个"两方法同时失效"的阴性对照（类比类不平衡），以确认 fragility regime 制造的是**真实的分野**而不是**两方法通用的失效**。

**回答本轮问题 (b)**——是否提供可迁移到我们受控 factorial 实验设计的方法论或诊断量：**是，三个直接可迁移的工具**：
1. `N ≥ ξ·r_X·r_C·r_S` (Eq.7-01) 给出了"条件独立检验可靠性随条件集阶数增长而下降"的**形式化诊断量**，我们可以为 PCMCI+ 用的偏相关/CMI 检验构造类比量（如"有效自由度 = N − |S| − 常数"或检验统计量方差随 |S| 的膨胀率），预测：AUC_conditional 的坍塌应该恰好发生在这个类比量跌破可靠阈值的地方；如果 fragility regime 中这个类比量并未显著恶化但 AUC_conditional 仍然坍塌到 chance，就证明 fragility 失效的机制**不是**本文这种"小样本/高条件阶数"机制的动力学版本，而是别的东西（比如 reviewer 怀疑的非线性/隐藏驱动混淆）。
2. Table tb6_13 式的 **PC/SP/FP 分解**（不只报总体 precision/recall，而是拆成"找到几个真父子"、"找到几个真配偶"、"混入几个假阳性"）是一个诊断利器：可以用来区分我们 pilot 里 causal AUROC 坍塌的具体来源——是 recall 端（条件独立检验把真正的因果父母/子女都检验掉了）还是 precision 端（检验功效不足导致假阳性大量混入，稀释判别力）。这两种失效模式指向完全不同的修复方向。
3. **受控合成 benchmark 方法论**（已知真实图的 ALARM BN + 固定图结构只变一个轴 [样本量] + 对照真实 MB/PC 算 precision/recall + 配对比较同一 ψ 下的因果/非因果方法）与 round1 从 1911.07147/2002.09414 得到的建议一致：单轴受控 + 已知 ground-truth 图 + 配对 Delta 计算，是我们下一版实验设计应该照抄的模板。

**新颖性提醒（重要）**："非因果方法在小样本/高维/某些 regime 下打赢因果方法"本身**完全不是新发现**——这篇2018年 JMLR 投稿论文加上其117篇反引文（包括2025-2026年仍在扩展这个方向的新论文，如 arXiv 反引文列表中的"Causal Feature Selection With Imbalanced Data"、"Label-Aware Causal Feature Selection"）已经把这个现象研究得很透彻。我们的创新点必须落在：(1) **动力学触发机制**——是趋近分岔的临界慢化（critical slowing down）导致的有效信噪比下降，而不是经典的小N/高维/类不平衡；(2) **应用场景**——早期预警指标设计（该用条件化/因果信号还是边际/相关信号）在分岔/regime-shift 文献里是空白，这条经典 FS 理论脉络完全没有触及；(3) 用本文式的诊断分解（PC/SP/FP、样本-阶数类比量）把"是什么导致 fragility 坍塌"精确定位到一个新机制上，而不是重新发现"causal 在困难 regime 下更弱"这个已知事实。

## Read by: 0710-causal-scs (2026-07-11)

本文是 round1 从 1911.07147 wiki 中预判的"机制2"（CI检验统计功效随条件集阶数增长而系统性下降）理论源头，本轮精读后**确认预判成立**：Section 5 用 k-DB 结构假设正式统一了"非因果=低阶近似"vs"因果=精确高阶条件化"，并用 Eq.7-01 给出显式样本量公式、用 ALARM+25真实数据集的多 regime 实证验证了"小样本/高维→非因果占优，大样本→因果占优，类不平衡→两者同时失效"三分格局。

对 idea 而言，本文的价值是**方法论工具箱**而非直接证据：它的样本-阶数机制发生在静态 i.i.d. 数据里，触发条件是 N 太小，而我们的 fragility regime 触发条件是动力学趋近分岔导致的有效信噪比下降——这是同一个"CI检验功效"机制家族的两种不同触发路径，本文不能直接当作 fragility 现象的证据，但可以：(1) 提供一个类比诊断量（检验可靠性阈值）来预测/证伪 AUC_conditional 坍塌的位置；(2) 提供 PC/SP/FP 式的失效分解，把"causal conditioning 丢失信号"精确定位到 recall 端还是 precision 端；(3) 提醒我们"非因果打赢因果"不是新发现，必须在动力学触发机制和 EWS 应用场景上做出真正差异化，且类不平衡这个"两者同时失效"的反例说明不能想当然地假设任何困难 regime 都会制造 causal/non-causal 分野。

下一步优先读反引文"Causal feature selection using a conditional independence test with a balanced trade-off between Type I and Type II errors"（2026）——如果它把 Type I/II 权衡显式公式化为可调参数，可能直接给出我们需要的诊断量的现成版本。
