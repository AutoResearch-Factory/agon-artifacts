# Transformer Semantic Genetic Programming for Symbolic Regression (2501.18479)

- **Year**: 2025.01
- **Authors**: Philipp Anthes, Dominik Sobania, Franz Rothlauf (Johannes Gutenberg University, Mainz)
- **Venue**: GECCO 2025
- **Keywords**: Genetic Programming, Transformer Models, Semantic Operators, Symbolic Regression, Geometric-Semantic GP, Variation Operators, Sequence-to-Sequence
- **One-liner**: 用预训练的 encoder-decoder transformer 替代 GP 的语法变异算子，通过语义相似性搜索训练后，能在未见数据集上生成语义相似且结构紧凑的 offspring，在 5 个黑箱 SR benchmark 上显著超越 stdGP/SLIM_GSGP/DSR/DAE-GP。

## 精华提炼

### 管线/方法

TSGP 分两阶段：**Model Building（一次性）** 和 **Model Sampling（搜索时重复使用）**。

**阶段一：Model Building**
1. **Solution Generation**：在 50 个合成 SR 问题上运行 stdGP（pop=2000），生成大量数学函数集合 F。合成问题的输入输出来自随机线性回归模型，加入高斯噪声增加多样性。数据标准化（mean=0, var=1）。
2. **Semantic Similarity Search**：对 F 中每个函数 f_i，用 FAISS 库做 k-NN 搜索（k=3），以 Euclidean distance 度量语义距离 SD(s(f_i), s(f_j)) = ||f_i(X) - f_j(X)||_2。筛选 SD≠0 且 SD<100 的 pair，共生成 5M 对训练数据。
3. **Model Training**：将每个 (f_i, f_o, SD) 作为训练样本。f_i → encoder，decoder 自回归生成 f_o 的 token 序列，同时 SD 作为额外输入传给 encoder 和 decoder。Parse tree 通过 prefix-order traversal 转为 token 序列。损失函数：next-token prediction cross-entropy。

**阶段二：Model Sampling（GP 搜索）**
- 用训练好的 transformer 替代 subtree crossover 和 subtree mutation。
- Parent f_i + 目标 SD^d=0.1 → transformer → offspring f_o。
- Token-wise 采样直到 EOS 或 max_length=100。
- 使用 syntax control（Wittenberg 2022）：将导致无效树的 token 概率置零。
- 搜索框架：DEAP，pop=100，50 generations，tournament selection (size=5)，RHH 初始化 depth 2-5，max tree depth=17。

### 关键技术细节

- **Transformer 架构**：标准 encoder-decoder (Vaswani 2017)，8 attention heads，hidden dim=128，2 encoder stacks + 2 decoder stacks，max input length=100 tokens。这是小模型——相比现在的大语言模型非常轻量。
- **训练配置**：AdamW optimizer，lr=1e-3（固定），8 epochs。作者明确说"Systematic hyper-parameter tuning was not feasible due to the complexity of the task"——这其实是个明显的局限。
- **训练数据**：5M 对语义相似的函数 pair，全部来自 4-feature 合成线性回归问题的 stdGP 结果。
- **Primitive set**：{V, ERC, +, -, ×, %}，ERC ∈ [-0.5, 0.5]，stepsize 0.1。
- **对比方法**：stdGP（subtree crossover 90% + subtree mutation 10%）、SLIM_GSGP（SLIM+SIG2，inflation rate 0.2）、DSR（RNN + RL，reward=-RMSE）、DAE-GP（denoising autoencoder LSTM，5% edit corruption）。
- **Benchmark**：PMLB 中 4-feature 且样本>100 的全部黑箱数据集——ERA(1000)、ESL(488)、Galaxy(323)、LEV(1000)、Pollen(3848)。50/50 train-test split，30 次独立运行。
- **评估指标**：RMSE（test set），solution size（node count），semantic distance（Euclidean distance between parent-offspring semantics）。
- **统计检验**：Wilcoxon rank sum test，α=0.05。

### 核心结果

| 数据集 | TSGP | stdGP | SLIM_GSGP | DSR | DAE-GP | TSGP优势 |
|--------|------|-------|-----------|-----|--------|----------|
| ERA | **0.797** | 0.817 | 0.810 | 0.852 | 0.904 | 显著优于 stdGP/DSR/DAE |
| ESL | **0.379** | 0.502 | 0.418 | 0.507 | 0.595 | 显著优于全部 |
| Galaxy | 0.327 | 0.337 | **0.305** | 0.434 | 0.468 | 与最优无显著差异 |
| LEV | **0.672** | 0.703 | 0.681 | 0.773 | 0.842 | 显著优于 stdGP/DSR/DAE |
| Pollen | 0.518 | **0.514** | 0.569 | 0.750 | 0.752 | 与最优无显著差异 |

- **收敛速度**：TSGP 在 ERA/ESL/LEV 上 10 代即收敛到高质量解（ESL: TSGP 0.381 vs stdGP 0.565 vs SLIM 0.503）。
- **稳定性**：TSGP 的 IQR 在所有数据集上明显小于 stdGP，与 SLIM_GSGP 相当。
- **Solution size**：TSGP 始终显著小于 SLIM_GSGP（Pollen 上减少 42%），与 stdGP 无显著差异。
- **语义相似性**：TSGP 的 parent-offspring semantic distance 在绝大多数数据集上低于 stdGP 和 SLIM_GSGP，说明 transformer 成功学会了生成语义相似的变异。
- **DSR 和 DAE-GP 表现最差**：作者归因于它们需要在搜索过程中在线训练神经网络，数据量不足。TSGP 只需要一次离线预训练。

### 明确局限

1. **仅限 4-feature 数据集**：训练数据和测试数据都是 d=4。作者承认需要扩展到不同维度。
2. **训练数据来源单一**：合成数据仅来自线性回归模型的 stdGP 结果。可能限制了 transformer 见过的函数多样性。作者自己也提到"greater variety of training data could have a positive impact on generalizability"。
3. **无超参数调优**：transformer 架构、学习率、epoch 数都没有系统调优，可能远非最优。
4. **SD^d 固定为 0.1**：step size 控制在搜索中是固定的，没有自适应策略。作者在 Future Work 中提到要系统控制 step size 以更好地逃离局部最优。
5. **仅在 SR 上测试**：虽然方法框架声称通用，但只验证了符号回归任务。
6. **无真机/物理实验**：纯计算实验。
7. **训练成本未报告**：没说训练 5M pairs 需要多少 GPU 时间。
8. **Standardized data only**：所有数据都标准化了，未测试非标准化场景。
9. **有限的对比方法**：缺少与更近期的 SR 方法（如 TSPR、SymFormer 端到端方法）的直接预测质量对比——虽然在 Related Work 中提到了它们。
10. **只在黑箱 SR 上测试**：没有在已知 ground-truth 方程的 SR benchmark（如 Nguyen, Koza, Feynman）上测试恢复能力。

### 跟我们的关系

对 0616-fex 课题：
- **不撞车**：TSGP 是做符号回归的搜索方法，使用 transformer 作为变异算子。0616-fex 如果关注的是特征提取（feature extraction），二者问题域不同——但如果 FEX 涉及搜索数学表达式作为特征，则有交集。
- **可用的工具/方法**：
  1. "语义距离引导的变异" 思路：用神经网络学习什么样的变异产生语义相似的后代，这个范式可以迁移到任何基于搜索的优化。
  2. "离线预训练 + 在线搜索" 的分离设计：一次性训练变异模型，然后在不同问题上复用，避免在线训练的数据不足问题。
  3. k-NN + FAISS 做语义相似性搜索的数据构建方法：可复用于构建训练 pair。
  4. Syntax control（token 掩码保证语法正确）：任何 tree/graph 生成的通用技术。
- **差异化**：如果 0616-fex 做的是特征提取而非符号回归，则核心问题、评估指标、搜索空间都不同，不存在直接竞争。
- **该引不该引**：如果 0616-fex 论文涉及 (a) 语义引导的搜索 (b) 神经网络作为搜索算子 (c) learned variation operators，应该引用此工作作为 related work。

## Read by: 0616-fex (2026-06-25)

TSGP 与 0616-fex 课题的核心区别在于问题域：TSGP 解决的是符号回归（SR）——从数据中恢复数学表达式；而 0616-fex 关注的是特征提取（feature extraction）。

**撞车评估**：不撞车。但如果 0616-fex 涉及以下技术路线则需注意：
1. 如果 FEX 使用 GP/进化算法搜索特征表达式，TSGP 的 semantic-aware transformer 作为变异算子的思路可以直接迁移。
2. 如果 FEX 使用神经网络生成特征，TSGP 的"离线预训练 + 在线复用"模式值得借鉴。

**可用工具/方法**：
- Semantic similarity search (k-NN + FAISS) 构建训练 pair 的方法可用于任何需要学习"相似变换"的场景。
- Syntax control 技术可用于保证生成的 feature extractor 语法正确。
- "预训练变异模型 → 跨问题复用"的分离设计降低了在线学习的样本需求。

**不适用之处**：
- TSGP 的 primitive set (+, -, ×, %) 和 tree-based 表达可能不适用于 FEX 的特征空间。
- 如果 FEX 使用神经网络而非符号表达式做特征提取，则 TSGP 的 tokenization 和 tree 表示不适用。
