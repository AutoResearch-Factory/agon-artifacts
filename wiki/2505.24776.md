# Diffusion-Based Symbolic Regression (2505.24776)

- **Year**: 2025.05
- **Authors**: Zachary Bastiani, Robert M. Kirby, Jacob Hochhalter, Shandian Zhe (University of Utah)
- **Venue**: arXiv (NeurIPS 2025 format)
- **Keywords**: symbolic regression, discrete diffusion, masked diffusion, GRPO, reinforcement learning, risk-seeking policy, expression generation, SRBench
- **One-liner**: 提出 DDSR——用单 token 随机掩码离散扩散替代自回归采样生成表达式，结合 token-wise GRPO 和长短期风险寻求策略，在 SRBench 上显著优于 DSR。

## 精华提炼

### 管线/方法

DDSR 完整 pipeline：

1. **表达式表示**：将表达式树按 BFS 顺序展开为 token 序列，用 one-hot 编码矩阵 $\X_0 \in \mathbb{R}^{M \times d}$（M=最大 token 数=32, d=词汇表大小）。Token 库包含：变量 {1, c, x_i}，一元函数 {sin, cos, log, sqrt, exp}，二元函数 {+, -, *, /, ^}。

2. **前向扩散（训练时）**：每步随机选择一个未被掩码的 token 位置将其掩码（置零），共 M 步完成全掩码。相比 D3PM 的全局扰动，单 token 掩码保留了表达式结构信息——这是本文最核心的设计选择，实验证明比标准 D3PM 在 Strogatz 数据集上准确率提高 35.7%。

3. **反向去噪（采样时）**：从全零矩阵 $\X_M$ 开始，每步用 Transformer $\phi_\theta$ 预测 $\X_0$ 的 token 分布，采样被掩码 token，保留已重建 token，逐步恢复到 $\X_0$。若最终表达式不合法，随机替换非法 token。

4. **RL 训练**：每 epoch 采样 B=1000 个表达式，选 top α=5% 按 NRMSE 奖励。随机选扩散步 t，对采样表达式的掩码版本预测原始 token，最大化 token 级对数似然 × 相对奖励。用 token-wise GRPO（trust region clip + KL 正则化）更新模型。

5. **长短期风险寻求（LST）**：每 epoch 将当前 top-α% 表达式加入全局候选池 $\Scal_\alpha$，用全局最小奖励 $R_\alpha$ 计算 advantage。每 epoch 后移除池中最差 α% 表达式。本质是 DSR 风险寻求 + priority training queue 的混合。

6. **常数优化**：每个发现的表达式用 Levenberg-Marquardt 算法优化常数 token。表达式限制最多 10 个常数，三角函数不许嵌套。

### 关键技术细节

- **模型架构**：Transformer，1 encoder layer + 1 decoder layer，1 attention head，embedding dim=15，FF hidden size=2048。2D 位置编码：token 位置 l 和扩散时间步 t 分别用正弦编码后拼接。
- **训练超参数**：Batch size=1000，α=5%，LR=1e-4（ADAM），epochs=600，entropy coefficient λ=0.0005，GRPO clip ϵ=0.2，KL 正则化 β=0.01，C=5（每 epoch 更新步数），G=5（每 5 epoch 更新 reference model）。
- **硬件**：NCSA Delta cluster 的 A40 GPU，每个 A40 并行跑 8 个 trial，每个 problem × noise 组合跑 8 次，每次最多 4 小时。
- **评测指标**：Symbolic solution rate（SymPy 等价 或 R²=1.0）、Accuracy rate（R²>0.999）、Simplified complexity（SymPy 简化后 token 数）、R² score、Model size。
- **数据集**：SRBench 的 Feynman（119 静态系统问题）和 Strogatz（14 非线性 ODE 问题），共 133 已知解 + 120 黑盒问题。噪声水平 0%、0.1%、1%、10%。

### 核心结果

| 方法 | Sym. Solution (0% noise) | Accuracy (0% noise) | Simplified Complexity | Runtime (s) |
|------|--------------------------|---------------------|----------------------|-------------|
| DDSR | **46.54%** | 60% | 17.33 | 14441 |
| DSR-W/C | 24.81% | 38% | 16.57 | 27132 |
| DSR-W/OC | 19.71% | 24% | 13.14 | 631 |
| GP-GOMEA | 43.08% | **71%** | 25.73 | 2072 |
| TPSR | 36.09% | 68% | 57.11 | 173 |
| AIFeynman | **61.84%** | 74% | 83.29 | 29591 |

- **vs DSR**：DDSR 在 0%/0.1%/1% 噪声下 symbolic solution rate 全面碾压 DSR-W/C（46.54% vs 24.81% @0% noise），accuracy rate 60% vs 38%。运行时约为 DSR-W/C 的一半。但在 10% 噪声下 solution rate 被 DSR-W/OC 超越（10.69% vs 16.61%），作者归因于更大的 token 空间使搜索更不稳定。
- **vs GP**：symbolic solution rate 超过多数 GP 方法，GP-GOMEA 在 accuracy 上更强但表达式复杂度更高（35.4 vs 17.7）。
- **黑盒问题**：DDSR 在 Pareto 前沿（R²=0.730, size=23.8），与 Bingo、GP-GOMEA 接近。超过 4/5 的 ML 方法（linear fit, random forests, AdaBoost, MLP），但被 XGBoost 在 R² 上超过（0.776 vs 0.730，代价是模型大 713 倍且不可解释）。
- **Ablation**：Masked diffusion vs D3PM → +35.7% accuracy on Strogatz；GRPO vs RSPG → 平均快 30 epochs 收敛，Strogatz 上 +3.5%；LST vs ST → +1.3% Feynman, +3.5% Strogatz。

### 明确局限

1. **高噪声敏感性**：10% 噪声下 solution rate 降到 10.69%，不如 DSR-W/OC（16.61%）。作者承认鲁棒性不足。
2. **复杂问题运行时间长**：4 小时时间限制内，复杂表达式可能无法找到解。每个问题需要从头训练（无预训练）。
3. **无代码开源**：论文未提及代码或模型权重发布。所有结果来自作者自跑，第三方复现困难。
4. **常数优化分离**：常数 token 通过外部 Levenberg-Marquardt 算法优化，未集成到扩散/RL 框架中。这降低了端到端优雅性。
5. **最大深度限制**：表达式树深度限制 32，更复杂的表达式无法表示。
6. **未与最新 LLM-based SR 对比**：仅与 DSR、TPSR、uDSR 等对比，缺少与更近期方法（如基于 LLM 的 SR）的直接比较。
7. **单 token 掩码的代价**：虽然提高了稳定性，但扩散步数 = token 序列长度（最多 32），限制了采样效率的进一步提升空间。

### 跟我们的关系

对 **0616-fex**（特征提取/表达式 landscape）课题：

**撞车评估**：
- DDSR 的核心目标是**给定数据集的表达式发现**（symbolic regression），与 fex 方向试图从表达式中**提取特征**或**分析表达式的结构性质**有本质区别。不是直接竞品。
- 但 DDSR 的**表达式生成管线**（离散扩散 + RL）如果用于 fex 生成合成表达式数据或作为表达式先验，存在间接重叠。需关注我们是否也在做"用扩散模型生成表达式"这件事。

**可用工具/方法**：
1. **单 token 掩码扩散（核心可复用技术）**：如果 fex 需要生成表达式（如数据增强、负采样），这个比自回归/D3PM 更稳定。可直接搬用 BFS ordering + 掩码方案。
2. **Token-wise GRPO**：per-token 信任域策略可推广到任何 token 序列 RL 训练场景。如果 fex 涉及用 RL 优化表达式变换，GRPO 比普通策略梯度更高效。
3. **长短期风险寻求（LST）**：混合当前 epoch 和历史全局最优的思路简单有效，适合任何迭代优化场景。fex 的搜索/优化阶段也可用类似 buffer 机制。
4. **2D 位置编码**（位置+时间步）：扩散模型需要时间步条件，这种编码方式可直接复用于 fex 中任何需要时间条件的表达式建模。

**差异化**：fex 若聚焦"理解/分析/变换已有的表达式"而非"从头发现表达式"，则与 DDSR 完全互补。DDSR 可作为 fex pipeline 的表达式生成前端。

**该引不该引**：如果 fex 涉及扩散模型或 RL 表达式生成，必须引用。如果 fex 纯做特征提取不涉及生成，可列为 related work 中的生成式 SR 方法。

## Read by: 0616-fex (2026-06-25)

### 撞车评估
DDSR 是**表达式发现**方法（input: data → output: expression），而 0616-fex 是**特征提取/表达式分析**方向（input: expression → output: features/analysis）。两者方向相反，不构成直接竞争。但若 fex 需要表达式生成作为子模块（数据增强、负采样、先验建模），DDSR 的扩散式生成管线是直接可用方案。

### 可复用的技术组件
1. **单 token 掩码扩散**：对表达式 token 序列做渐进式掩码/去掩码，稳定且高效。fex 中任何需要表达式生成的场景都可直接复用。
2. **Token-wise GRPO + LST 风险寻求**：per-token trust region RL 训练策略。如果 fex 涉及用 RL 搜索表达式变换空间（如寻找最优特征提取规则），这套训练框架可直接套用。
3. **BFS ordering + 2D position encoding**：表达式序列化方案和位置编码设计，适用于任何需要把表达式树编码为序列的场景。

### 差异化定位
DDSR 关注"找到哪个表达式"，fex 关注"表达式长什么样/如何改进"。如果 fex 能利用 DDSR 的扩散先验来约束搜索或过滤候选表达式，可能产生有趣的新方向。例如：用扩散模型生成候选表达式，再用 fex 特征分析筛选——这在 DDSR 论文中未被探索。
