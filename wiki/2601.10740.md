# Neuro-Symbolic Activation Discovery: Transferring Mathematical Structures from Physics to Ecology for Parameter-Efficient Neural Networks (2601.10740)

- **Year**: 2026.01
- **Authors**: Anas Hajbi, School of Computing, Mohammed VI Polytechnic University (UM6P)
- **Venue**: arXiv
- **Keywords**: neuro-symbolic AI, activation function discovery, genetic programming, symbolic regression, transfer learning, scientific ML, parameter efficiency, geometric transfer
- **One-liner**: 用 GP 从科学数据中发现符号激活函数，并测试跨连续领域迁移的参数效率。

## 精华提炼

### 管线/方法

1. 论文提出的核心对象不是一般符号回归器，而是一个“先发现公式，再把公式编译成神经网络激活函数”的 neuro-symbolic pipeline。
2. 输入是二分类科学数据集 `(x_i, y_i)`，其中特征是数值向量，标签是二分类。
3. Discovery phase 先从原数据中抽取 10% stratified subset，作为 GP 的公式搜索数据。
4. GP 工具使用 `gplearn`，不是 PySR，也不是 RL controller。
5. GP population 设为 500，generations 设为 5，parsimony coefficient 为 0.01。
6. GP function set 为 `add`, `sub`, `mul`, `sin`, `cos`, `abs`。
7. 论文声称 5 代已经足够，因为函数集合很小、复杂度惩罚较强、二分类 fitness 信号清晰。
8. 每个数据集先发现一个带特征索引的表达式，例如 HIGGS 得到 `mul(cos(X25), sub(X12, X3))`。
9. 关键变换是把所有 `X_i` 统一替换成 layer input `x`，把特征交互表达式泛化为 element-wise activation。
10. 因此 HIGGS 的表达式从 `cos(X25) * (X12 - X3)` 变为 `sigma_phys(x) = x * cos(x)`。
11. Forest Cover 的表达式从 `add(sin(X13), mul(X8, X22))` 变为 `sigma_forest(x) = sin(x) + x^2`。
12. Spambase 的表达式从 `add(X6, mul(X52, cos(X51)))` 变为 `sigma_spam(x) = x + x * cos(x)`。
13. Injection phase 用动态代码生成把公式字符串编译成 PyTorch module。
14. 编译逻辑包括将 `X\d+` 替换成 `x`，将 `sin/cos` 等操作替换成 PyTorch op 字典中的可微函数。
15. 论文的网络实验控制了 architecture，只改变 activation，从而隔离激活函数本身的影响。
16. Heavy baseline 是 2 hidden layers，宽度 200 -> 100，激活函数 ReLU。
17. Light models 是 2 hidden layers，宽度 64 -> 32，比较 ReLU/GELU/SiLU/Hybrid。
18. HIGGS Light 参数量为 4,161，Heavy 参数量为 26,601。
19. Forest Cover Light 参数量为 5,825，Heavy 参数量为 31,801。
20. Spambase Light 参数量为 6,017，Heavy 参数量为 32,401。
21. 为了稳定非单调且无界的发现激活函数，作者在每个 activation 前使用 Batch Normalization。
22. 训练使用 Adam optimizer，learning rate 0.001，batch size 1024，15 epochs。
23. loss 是 Binary Cross-Entropy，特征做 standard scaling。
24. 每个配置重复 3 个 seed，seed 为 42, 43, 44。
25. transfer experiment 重点是把 HIGGS 发现的 `x*cos(x)` 迁移到 Forest Cover 和 Spambase。
26. 论文定义参数效率分数 `E_param = AUC / log10(Params)`，用 AUC 除以参数量的十进制对数。
27. 这个 metric 强调低参数模型的性价比，但它不是社区标准指标，需要谨慎引用。
28. 本文源码读取情况：1 个 `.tex` 文件，`main.tex` 共 535 行，已按 `wc -l` 核对并完整读完。

### 关键技术细节

1. 数据集包括 HIGGS、Forest Cover、Spambase，全部被设为二分类任务。
2. HIGGS 使用 100,000 samples，28 features，数据类型是 continuous kinematics。
3. Forest Cover 使用 100,000 samples，54 features，数据类型是 terrain/cartographic continuous features。
4. Spambase 使用 4,601 samples，57 features，数据类型是 word frequency / discrete-like text features。
5. Discovery subset 为原始数据 10%，作者没有报告多 seed GP discovery，只固定 discovery seed 为 42。
6. GP 搜索空间很小，且 generations 只有 5，这使 pipeline 很便宜，但也让“发现了数学结构”的 claim 偏弱。
7. 公式泛化步骤是本文最关键也最脆弱的步骤：从 feature-index expression 到 scalar activation 没有理论证明。
8. `X25`, `X12`, `X3` 等原物理特征的语义在泛化成 `x` 后被丢弃。
9. 作者认为保留下来的是 oscillatory/multiplicative functional form，而不是原 feature semantics。
10. HIGGS specialist activation 是 `x*cos(x)`，它是非单调、振荡、幅度随 `|x|` 线性增长的函数。
11. Forest specialist activation 是 `sin(x)+x^2`，带周期项和二次项，在负向输入也会快速增大。
12. Spam specialist activation 是 `x+x*cos(x)`，相当于线性项加振荡调制。
13. BatchNorm 是稳定性关键，若去掉 BatchNorm，深层网络中 `x^2` 或 `x*cos(x)` 可能带来梯度和尺度问题。
14. 论文只测试 2 层 MLP，没有测试 CNN、Transformer、ResNet、PDE solver、FEX 表达式树或更深网络。
15. 参数效率结论建立在 small MLP 和 tabular datasets 上，不能外推为一般神经网络效率结论。
16. 论文声称 GP discovery 一次性成本约 5-10 分钟每数据集，但没有给硬件、实现细节或方差。
17. Code Availability 给了 GitHub URL，但本次任务没有下载或审查非 arXiv 仓库。
18. 实验指标主要是 accuracy、AUC、parameter count、`E_param`。
19. 没有报告 calibration、训练时间、inference latency、FLOPs、显存、统计显著性检验或置信区间。
20. 没有把发现激活函数与更强的 learnable activation baseline 大规模比较，例如 PAU 的完整调参版本。

### 核心结果

| Dataset | Best / key model | Accuracy | AUC | Params | Efficiency |
|---------|------------------|----------|-----|--------|------------|
| HIGGS | Heavy ReLU | 0.718 +/- 0.000 | 0.791 +/- 0.000 | 26,601 | 0.179 |
| HIGGS | Light ReLU | 0.710 +/- 0.002 | 0.784 +/- 0.002 | 4,161 | 0.217 |
| HIGGS | Light Hybrid Specialist | 0.707 +/- 0.002 | 0.777 +/- 0.002 | 4,161 | 0.215 |
| Forest | Heavy ReLU | 0.834 +/- 0.003 | 0.915 +/- 0.001 | 31,801 | 0.203 |
| Forest | Light Hybrid Transfer | 0.824 +/- 0.003 | 0.904 +/- 0.001 | 5,825 | 0.240 |
| Forest | Light ReLU | 0.811 +/- 0.004 | 0.894 +/- 0.002 | 5,825 | 0.237 |
| Spambase | Heavy ReLU | 0.931 +/- 0.008 | 0.978 +/- 0.002 | 32,401 | 0.217 |
| Spambase | Light Hybrid Specialist | 0.920 +/- 0.003 | 0.967 +/- 0.004 | 6,017 | 0.256 |
| Spambase | Light Hybrid Transfer | 0.832 +/- 0.008 | 0.898 +/- 0.006 | 6,017 | 0.238 |

1. 论文最强结果是 Forest Cover 上的 Physics -> Ecology transfer。
2. HIGGS-derived `x*cos(x)` 在 Forest Cover 上达到 82.4% accuracy，超过 Light ReLU 的 81.1%、Light GELU 的 81.0%、Light SiLU 的 80.1%。
3. Forest Hybrid Transfer 与 Heavy ReLU 的 accuracy gap 只有 1.0 percentage point，但参数量少 5.5x。
4. Forest Hybrid Transfer 的 efficiency 为 0.240，高于 Heavy ReLU 的 0.203，作者称提升 18.2%。
5. HIGGS 本域上，Hybrid Specialist 并没有超过 Light ReLU，反而略低。
6. 这使“源域学到的结构在源域最优”这个直觉不成立。
7. 作者把这一点解释为 GP 学到的是 robust geometric primitive，而不是 HIGGS-specialized optimum。
8. Spambase 上 Physics -> Text transfer 明显失败，accuracy 从 specialist 的 92.0% 降到 transfer 的 83.2%。
9. 这个失败结果是论文中比较可信的 boundary condition：连续几何型数据和稀疏/离散词频数据不共享同一类 activation prior。
10. Spambase specialist 的 variance 比标准 light baselines 低，作者用它支持“领域特化激活函数可稳定训练”。
11. HIGGS 上真正最高 efficiency 是 Light ReLU，不是 Hybrid，这削弱了 universal neuro-symbolic activation 的叙事。
12. Forest specialist 比 transfer 更差，说明“目标域直接发现”未必好，可能因小样本或噪声过拟合。
13. 因此本文更像一个有趣 empirical observation，而不是成熟的 activation discovery 方法论。

### 明确局限

1. 作者自己承认只测试了三个数据集，不能建立 universal geometric transfer。
2. 作者自己承认只测试了 2-layer shallow networks，深层网络的梯度动力学未知。
3. 作者自己承认 GP discovery 有 stochasticity，但没有系统报告不同 GP seeds 的公式方差。
4. 作者自己承认公式可解释性有限，知道公式长什么样不等于知道为什么有效。
5. 本文没有真机、没有物理实验、没有独立实验室复现。
6. 本文没有标准 SR benchmark，也没有 PDE benchmark，因此和 FEX 主线不是同一评测空间。
7. 参数效率 metric 是作者自定义的，`AUC/log10(params)` 的选择合理但主观。
8. Heavy baseline 并非最强模型，只是更宽的同类 MLP。
9. Light ReLU/GELU/SiLU 的调参深度不清楚，可能存在 baseline under-tuning。
10. 仅 15 epochs 训练，未说明早停、学习率 schedule 或是否所有模型都充分收敛。
11. 没有测试符号 activation 与 learnable scaling/normalization 的组合，例如 `a*x*cos(b*x)`。
12. 没有做 formula ablation，例如去掉 `cos`、替换成随机非单调函数、或只用 `x*sin(x)`。
13. 没有做 dataset leakage 检查，虽然没有明显证据显示泄漏。
14. S2 references 里有若干正文句子被误识别为引用，说明文献元数据质量一般。
15. 对 FEX 来说，本文没有 RL、没有表达式树 controller、没有 PDE residual objective、没有 approximation/search complexity 理论。

### 跟我们的关系

1. 对 `0616-fex`，本文不是直接竞品，因为它不用 RL 做 symbolic regression，也不求 PDE/ODE 的解析解。
2. 它的对象是 neural activation function discovery，而 FEX 的对象是 expression-tree solution / law discovery。
3. 不撞车的核心原因：本文把 GP 发现的表达式压缩成 element-wise scalar activation，FEX 保留完整表达式树作为最终解或物理 law。
4. 如果 `0616-fex` 要写“RL/FEX 是从函数空间中搜索可解释表达式”，本文只能算远邻 neuro-symbolic architecture prior。
5. 如果 `0616-fex` 要做“用已有科学数据自动发现可迁移数学 primitive”，本文会有局部 overlap，需要引用。
6. 可用方法一：从 source domain 发现简单 operator motif，再迁移到 target domain，是 FEX 设计 source-domain prior 的一个启发。
7. 可用方法二：把 discovery seed 与 training seeds 分开，是 FEX benchmark 中评估 controller prior 是否过拟合的可借鉴 protocol。
8. 可用方法三：用 negative transfer 建立 boundary condition，适合 FEX 写“哪些 prior 不能跨任务迁移”。
9. 可用方法四：GP 搜出的 `sin/cos/mul` motif 可作为 FEX operator-set prior，但它本身不提供 RL 搜索效率改进。
10. 不建议借鉴 `AUC/log10(params)` 作为 FEX 主指标，因为 FEX 的核心是 residual/error、expression complexity、搜索预算和解析可验证性。
11. 对 FEX 的真正价值是提醒：从大而干净的 source domain 学来的公式 bias，可能比小 target domain 直接搜索更稳。
12. 差异化方向：FEX 可以强调保留完整符号表达式与 PDE residual 验证，而本文只把公式当 activation，缺少可解释解的层级。
13. 差异化方向：FEX 可以引入独立 verifier、CAS/e-graph 等价检查、PDE residual gate，弥补本文 R2 verification independence 很弱的问题。
14. 该引不该引：如果写的是 FEX/RL 搜索复杂度理论，不必核心引用；如果写 LLM/GP/RL prior 如何注入 FEX 搜索，应作为弱相关引用。
15. 撞车 claim：不要声称“首次用符号回归发现神经网络组件/激活函数”，这已被本文和更早 activation search/learnable activation 文献覆盖。
16. 不撞车 claim：仍可声称“针对 PDE/FEX 表达式树搜索的 RL controller 改进、搜索复杂度诊断、e-graph 剪枝或 LLM skeleton 引导”，本文没有覆盖。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 视角看，这篇是弱到中等相关的 neuro-symbolic prior，而不是 FEX 的直接竞品。它用 GP 在 tabular scientific datasets 上找短公式，再把公式退化成 element-wise activation，例如把 HIGGS 的 `cos(X25)*(X12-X3)` 变成 `x*cos(x)`。这和 FEX 的表达式树搜索有表面相似性，都会利用 `sin/cos/mul` 这类 primitive，但目标完全不同：本文最终产物是神经网络组件，FEX 最终产物是 PDE 解或物理 law 的解析表达式。

撞车风险主要在 claim 层：如果 `0616-fex` 后续想说“从科学数据中自动发现可迁移的数学 primitive”，本文已经给了一个 activation-level 例子，需要引用并区分。但如果我们的贡献是 RL controller、FEX 搜索效率、表达式树剪枝、LLM skeleton 引导、PDE residual verifier 或搜索复杂度理论，本文没有覆盖这些核心点。

可借鉴的是它对 transfer boundary 的处理。Physics -> Ecology 成功、Physics -> Text 失败，说明公式 prior 的迁移需要数据几何结构匹配。FEX 如果要从简单 PDE、低维解析解或频谱工具中提取 prior，再迁移到高维 PDE/PIDE，也应该显式报告 negative transfer setting，而不是只报成功案例。

本文的可靠性不够强，不适合作为主要证据。它只有 3 个数据集、2 层 MLP、3 seeds、自定义 efficiency metric，且 HIGGS 本域 Hybrid 并未击败 Light ReLU。对 FEX 最有用的引用方式是把它放在“symbolic prior injection into neural architectures”或“domain-specific mathematical primitive transfer”的 related work 中，作为动机而非 baseline。真正的 FEX baseline 仍应是 SymPlex、SSDE、EGG-SR、LLM-SR/LLM-Feynman/SR-LLM、FunctionEvolve、Deliberate Evolution 等表达式树搜索或 SR 系统。
