# Finite Element Representation Network (FERN) for Operator Learning with a Localized Trainable Basis (2510.26962)

- **Year**: 2025.10
- **Authors**: Zecheng Zhang (Notre Dame), Hao Liu (HKBU), Guosheng Fu (Notre Dame), Hayden Schaeffer (UCLA), Guang Lin (Purdue)
- **Venue**: arXiv preprint
- **Keywords**: operator learning, finite element method, ReLU activation, local basis functions, DeepONet, adaptive FEM, neural operator, hat function
- **One-liner**: 用 ReLU 构造的浅层 FEM 帽函数替代 DeepONet 深层 trunk network，每个基函数仅 2 个可训练参数（中心和宽度），在 7 类 PDE 上达到与 DeepONet 相当或更好的精度，参数减少 3-5 倍，且基函数自适应地集中到解的局部特征区域。

## 精华提炼

### 管线/方法

**输入**：函数 u 的离散化传感器值（例如 22 或 100 个均匀采样点）

**整体架构**（继承自 DeepONet 的 branch-trunk 框架）：
1. **Branch network**（系数网络 c_k）：FC 22×20→20×1，Tanh 激活。对每个基函数 k 输出一个标量系数 c_k(u)。N 个基函数则有 N 个并行的 branch network。
2. **Trunk network**（基函数 φ_k）：**本文核心创新**——不用深层网络，而是用 2 参数 ReLU 网络构造 FEM 帽函数：
   - p_{a,h}(x) = σ(x-(a-h)) - 2σ(x-a) + σ(x-(a+h))
   - 其中 σ = ReLU，a 是中心，h 是半宽。仅 2 个可训练参数。
3. **输出**：G_θ(u)(x) = Σ_{k=1}^N c_k(u) · p_{a_k,h_k}(x)

**训练**：余弦退火学习率调度，2000 epochs。基函数中心均匀初始化于定义域，宽度初始化为 h=0.05。

**对比方法**：
- DeepONet-L：7 层 FC trunk network（深层全局基函数）
- DeepONet-S：2 层 FC trunk network（浅层全局基函数）
- POD：用输出函数 SVD 分解的 POD mode 作为固定基函数（仅适用于同一 mesh）

### 关键技术细节

**基函数构造**：
- 每个帽函数只需 2 个参数：中心 a_k 和宽度 h_k（半宽）
- ReLU 的 piecewise linear 性质使帽函数能被**精确**构造，无近似误差
- 在 1D 中就是标准 FEM 线性 Lagrange 基函数（但中心和宽度可学习）
- 帽函数天然具有局部支集：在 [a-h, a+h] 外为 0

**参数效率的数理逻辑**：
- Liu et al. 2024 理论显示 trunk network 深度需 O(-log ε) 才能达到 ε 精度——深层隐含地需要大量参数
- FERN 每个基函数仅 2 参数，总基函数参数 = 2N
- 例如 AC 2-DOF：DeepONet 97,100 参数 vs FERN 38,560（主要是 branch 减少）

**自适应机制**：
- 训练过程中 a_k 和 h_k 通过梯度下降优化
- 等效于 AFEM 中的 h-adaptive 网格细化
- 观察到的自适应行为：
  - FK 方程：基函数中心向 x=0.5（解峰值位置）聚集
  - KS 方程：更多基函数集中在 bump 区域，且宽度更小
  - KdV 方程：中心集中在 bump 所在的左半区域
  - AC/CH 方程：特征均匀分布时，中心保持均匀但宽度缩小，增强局部化

**实验设定关键数字**：
- 训练样本：42-1000 个 IC-solution pairs（因 PDE 而异）
- 基函数数量：10-250（因实验而异）
- 输出 mesh：64-100 点
- 输入 sensor mesh：22-100 点
- 激活函数组合：branch 用 Tanh（AC 1-DOF）或 ReLU（AC 2-DOF），基函数用 ReLU

**关键发现**：
- 13 个对比测试中 10 个 FERN 精度更好或方差更小
- FERN 的参数效率优势在所有例子中一致：基函数参数从 ~54,700 降到 80-160（降幅 340-680x）
- DeepONet-S（2 层 trunk）在所有例子中表现极差（28%-63% error），证明"不是浅层就行，FEM 结构是关键"
- POD 在低秩数据时最好（0.15% AC），但在高秩（2 DOF）时退化（7.76% vs FERN 4.44%）
- 误差随基函数数量大致一阶收敛
- 基函数学习到自适应分布：more bases near bumps + smaller supports near sharp features

### 明确局限

1. **仅 1D**：所有实验都是一维空间域，文中完全没有 2D/3D 示例。2D 的 FEM 帽函数构造复杂得多（涉及 triangulation），本文的方法直接推广不显然。
2. **仅 P1 元素**：仅使用分段线性 Lagrange 基函数（P1）。高阶 FEM（P2+）或不同多项式族未探索。未来工作也提到了这一点。
3. **无理论保证**：自适应行为的描述完全是经验性的。没有证明基函数中心/宽度收敛到最优分布的定理。与 Liu et al. 2024 的深度理论形成对比。
4. **对比不够全面**：只与 DeepONet 和 POD 比较。未与 FNO、attention-based operators（PROSE、BCAT）、或者 LOCA/LAVA 等 mesh-based 算子学习方法比较。
5. **训练数据量不均**：FK、AD、KS 只有 42 个训练样本（很少），而 CH 2-DOF 有 1000。小样本设置下优势可能被高估。
6. **Branch 网络参数仍然占主导**：虽然 trunk 参数大幅减少，但 branch（系数）网络参数仍然占计算主导。例如 AC 2-DOF 中 branch 占 38,400/38,560 = 99.6%。
7. **没有讨论推理速度**：虽参数数量减少，但推理时的前向传播计算量（N 个 branch 前传 vs 1 个 trunk 前传）未讨论。
8. **非均匀 mesh 场景有限**：FK 和 KS 的非均匀采样设定较简单，真正的非结构化 mesh 未测试。
9. **初始化的敏感性**：h=0.05 和均匀中心初始化的影响未系统研究。
10. **基函数的碰撞/重叠**：未讨论两个帽函数中心重合或间距过小的退化情况；也未讨论正则化或约束防止基函数坍缩。

## Read by: 0616-fex (2026-06-24)

本次从功能表达性（functional expressivity）视角审读 FERN。

**核心洞察**：FERN 展示了 structured shallow architecture（2 参数 ReLU 帽函数）可以匹敌 deep trunk network 的表达能力，关键在于利用 ReLU 的 piecewise linear 性质精确嵌入 FEM 的 inductive bias。这对 0616-fex 研究有两个启示：

1. **Inductive bias 降低 expressivity 需求**：FEM 结构使 N 个 2 参数基函数达到 7 层网络（数万参数）的精度。这意味着在适当的结构先验下，"浅层+结构化"可以取代"深层+通用"。0616-fex 的理论框架应考虑这种 structured expressivity。

2. **ReLU 的构造能力是核心**：本文的理论基础（Liu et al. 2024, 2410.00357）证明了 ReLU 网络表达能力的 scaling law。FERN 的实践反过来验证了该理论在应用中的有效性——exact ReLU construction of hat functions 实现了 zero-approximation-error basis building。

**差异化路径**：0616-fex 不应沿着 FERN 的 "operator learning architecture" 方向走，而应利用 FERN 作为一个 concrete example 来说明 "expressivity is not just about depth/width, but about how well the architecture matches the target function class"。这可以成为 0616-fex 理论框架中的一个 motivating case study。
