# All elementary functions from a single binary operator (2603.21852)

- **Year**: 2026.03
- **Authors**: Andrzej Odrzywolek, Jagiellonian University
- **Venue**: arXiv
- **Keywords**: EML operator, elementary functions, Sheffer operator, symbolic regression, exp-log functions, binary expression trees, analog computing, continuous optimization
- **One-liner**: 用单个二元算子生成科学计算器函数。

## 精华提炼

### 管线/方法

1. 论文目标不是提出一个新的数值近似器，而是问一个结构性问题：标准科学计算器的常数、初等函数和二元运算能否由一个二元算子加一个常数生成。
2. 作者固定一个具体的“科学计算器”基底，表 1 中共有 36 个 primitive。
3. 这 36 个 primitive 包括常数/变量：$\pi,e,i,-1,1,2,x,y$，共 8 个。
4. 它还包括 20 个一元函数：$\exp,\ln,\inv,\half,\minus,\sqrt{\cdot},\sqr,\sigma$，三角/反三角函数，双曲/反双曲函数。
5. 它还包括 8 个二元操作：$+,-,\times,/,\log,\pow,\avg,\hypot$。
6. 完备性判定是：候选基底能否重构这 36 个 primitive；本文不处理最一般的微分代数 elementary function 定义，而采取计算器式 finite expression 定义。
7. 作者先做 ablation：从 Wolfram/Mathematica 的较小 primitive set 出发，逐步剥离冗余操作。
8. 递减链条是 Base-36 -> Wolfram primitives -> Calc 3 -> Calc 2 -> Calc 1 -> Calc 0 -> EML。
9. Wolfram 对照基底约为 $\pi,e,i,\ln,+,\times,\wedge$，作者声称进一步压缩到 $\{1,\eml\}$。
10. 核心算子是 $\eml(x,y)=\exp(x)-\ln(y)$，paired constant 是 1。
11. 关键中和机制是 $\ln(1)=0$，因此 $\eml(x,1)=e^x$。
12. 自然对数的示例构造是 $\ln z=\eml(1,\eml(\eml(1,z),1))$。
13. 在没有外部变量的口袋计算器视角下，只需要两个按钮：常数 1 和二元 EML。
14. 在有变量的表达式视角下，变量作为额外 terminal 加入 grammar。
15. 表达式 grammar 极简：$S \to 1 \mid \eml(S,S)$；有变量时可加入 $x,y,\ldots$ terminal。
16. 由此所有表达式都是同构的 full binary tree；这也是作者把它类比为连续数学中的 NAND/Sheffer primitive 的依据。
17. 搜索阶段不是直接符号穷举所有等式，而是 hybrid numeric bootstrapping。
18. 对候选公式，作者用代数独立性强的超越常数数值替换变量，例如 Euler-Mascheroni 常数和 Glaisher-Kinkelin 常数。
19. 对目标操作先数值求值，再在由当前基底生成的表达式空间中做常数识别/公式识别。
20. 作者承认这个筛选依赖 Schanuel conjecture 风格的“偶然相等概率很低”直觉，严格证明独立放在 Supplementary Information。
21. 搜索以 RPN 程序长度 $K$ 作为复杂度，实践中通常搜到 $K \le 9$。
22. bootstrapping 过程维护两个集合：已验证可用的 $S_i$，待生成的 $C_i$。
23. 每找到一个可由 $S_i$ 表达的 primitive，就把它从 $C_i$ 移到 $S_{i+1}$。
24. 第一个直接例子是 $e=\eml(1,1)$。
25. 该过程最终生成表 1 中所有 primitive 后，认为基底完备。
26. 作者还指出 EML 不是唯一，给出两个近亲：$\edl(x,y)=\exp(x)/\ln(y)$ 配常数 $e$，以及 $\ln(x)-\exp(y)$ 配常数 $-\infty$。
27. EML 的实际计算需要复数域，特别是 $i$、$\pi$ 和三角函数经由 $\ln(-1)$ / Euler 关系生成。
28. 因此实函数的 EML 表达式内部可能经过复数中间值；作者把这类内部复数类比为量子计算用复振幅得到实概率。
29. 论文另一个应用管线是把 EML tree 作为 symbolic regression 的统一搜索空间。
30. 对 univariate 情况，EML 的每个输入可被参数化为 $\alpha_i+\beta_i x+\gamma_i f$，其中 $f$ 是上一层 EML 输出。
31. 通过设置 $\alpha,\beta,\gamma$ 为 one-hot，可选择 terminal 1、变量 $x$ 或前一层子表达式。
32. level-2 master formula 有 14 个参数；一般 level-$n$ master formula 有 $5 \times 2^n-6$ 个参数。
33. 作者提出用 simplex/softmax logits 表示这些离散选择，再用优化器训练，最后将权重 snap 到 0/1。
34. 这使得 SR 问题从异质操作 grammar 搜索，变成同构 EML tree 中的连续参数优化。
35. 第一个 proof-of-concept 用 level-3 binary tree 拟合 $\ln x$ 的数值数据。
36. 该例中 simplex reparameterization 后自由参数从 34 降到 20。
37. 作者用 Mathematica 的 `NMinimize` 找到权重，MSE 达到数值精度，snap 后公式正好是 $\ln x$。
38. 后续 PyTorch 实验用 `torch.complex128`，目标函数包括由 EML 组合出的二元函数。
39. 训练采用多阶段：Adam 常规训练，hardening phase 推向 0/1，最后 clamp 为精确符号值。
40. 工程问题主要是多重 exp 导致 overflow，以及复数算术中的 NaN；作者用 clamp 和实部/虚部检查规避。

### 关键技术细节

1. EML 算子：$\eml(x,y)=\exp(x)-\ln(y)$，非交换。
2. 最小终端：常数 1；变量在函数表达时作为额外 terminal。
3. 计算域：内部使用 $\mathbb{C}$ 和主支复对数。
4. 基准 primitive 数量：36 个，含 8 常数/变量、20 一元函数、8 二元操作。
5. 复杂度计量：RPN 长度 $K$，作者称典型 Kolmogorov-style complexity $K \lesssim 7$，实践搜到 $K=9$。
6. 搜索实现：Mathematica `VerifyBaseSet`，后续 Rust re-implementation 可在秒级重查 EML。
7. 对数表达式叶数：$\ln x$ 的 EML compiler/RPN 长度为 7，RPN 示例 `11xE1EE`。
8. 指数表达式叶数：$e^x$ 长度为 3，即 $\eml(x,1)$。
9. 乘法的直接搜索长度为 17，compiler 长度为 41。
10. 除法的直接搜索长度为 17，compiler 长度为 105。
11. 幂函数 $x^y$ 的直接搜索长度为 25，compiler 长度为 49。
12. 常数 $\pi$ 的 compiler 长度为 193，直接搜索达到 $>53$ 仍未给出更短式。
13. 常数 $i$ 的 compiler 长度为 131，直接搜索达到 $>55$。
14. master formula 参数量：level-$n$ 为 $5 \times 2^n-6$。
15. SR 训练深度结果：随机初始化 blind recovery 在 depth 2 为 100%，depth 3-4 约 25%，depth 5 低于 1%，depth 6 在 448 次中未观察到。
16. 若从正确 EML tree 的权重附近加 Gaussian perturbation，则 depth 5 和 6 也能 100% 回到精确值。
17. 成功时 snap 后 MSE 在 machine epsilon squared 量级，约 $10^{-32}$。
18. 可复现材料：作者称代码、脚本和 SI 复现材料在 SymbolicRegressionPackage，提交版本归档到 Zenodo DOI 10.5281/zenodo.19183008。
19. 论文明确提到 EML 可在 Mathematica 符号环境、IEEE754 C、NumPy、PyTorch 中工作，但 pure Python/Julia 或数值 Mathematica 会遇到特殊浮点/overflow 问题。
20. Lean 4 中 `Complex.log 0 = 0` 的 total-function 设计会破坏直接形式化链条。

### 核心结果

1. 主要数学 claim：$\{1,\eml\}$ 可生成具体计算器基底中的所有初等函数、常数和算术操作。
2. 表 2 给出从 36 primitive 到 EML 的完整压缩链。
3. 作者称 EML 是连续数学中类似 NAND 的 single sufficient binary primitive。
4. EML expression tree 统一为 full binary tree，grammar 简化到一个 production rule。
5. 这会把“表达式结构搜索”转化为固定架构/固定节点类型的树搜索。
6. SR 结果显示浅层树能从数值数据中 exact recover 初等表达式。
7. 最成功、最干净的例子是 $\ln x$：level-3 tree，20 个自由参数，snap 后外推也正确。
8. PyTorch 大规模随机初始化实验显示 EML basin 存在，但深度增长后 basin 很难命中。
9. 反直觉点 1：单个看似随意的 $\exp-\log$ 非交换组合足够生成整套科学计算器。
10. 反直觉点 2：复杂初等函数的完备表达空间可用完全同质的 binary tree 表示。
11. 反直觉点 3：复数内部计算不是附带异常，而是生成 $\pi,i$ 和三角函数所需的机制。
12. 负面结果同样重要：目前 EML 需要 distinguished constant，不能像 NAND 那样从任意输入自举出全部常数。
13. 另一个负面结果：纯实域 continuous Sheffer 似乎不可行，作者搜索替代三角/双曲函数对也未找到。
14. 对 SR 的核心数字：depth 6 随机初始化 448 次无 blind recovery，这说明表示完备不等于优化容易。

### 明确局限

1. 论文核心完备性证明主要依赖 Supplementary Information，主文只给出关键例子、表格和证明草图。
2. 搜索发现阶段带启发式色彩，候选等式先由数值替换和公式识别发现。
3. Schanuel conjecture 只提供偶然相等不太可能的论证背景，不是实际证明的一部分。
4. 完备性是针对作者定义的 36 个 scientific-calculator primitive，不等于覆盖任意微分代数意义下的 elementary extension。
5. EML 表达式可能非常长，compiler 形式对 $\pi,i$、平方根、均值等的叶数很高。
6. 表中 direct search 多处是 lower bound 或 timeout，不是已证明最短式。
7. EML 需要常数 1；是否存在无需 distinguished constant 的二元 continuous Sheffer 仍是开放问题。
8. 内部复数和 branch cut 会带来实现细节，特别是负实轴对数符号、$\ln 0$、无穷、signed zero。
9. 在一些语言/证明助手中，默认 log/inf/NaN 语义会使 EML 链条不直接成立。
10. SR 实验的成功主要在浅层；深层随机搜索成功率迅速下降。
11. 论文没有标准 SR benchmark 上的大规模对比，没有和 PySR、DSR、AI Feynman 做系统公平评测。
12. 没有真机 analog circuit 或 FPGA 实现，analog computing 部分仍是应用展望。
13. 单作者、自产生候选、自验证链条，独立验证不足。
14. AI use disclosure 说明 LLM 用于语言编辑和编码辅助，但核心发现和验证由作者完成；仍应优先看代码和 SI 是否足够可审计。
