# Deep Generative Symbolic Regression with Monte-Carlo-Tree-Search (2302.11223)

- **Year**: 2023.02
- **Authors**: Pierre-Alexandre Kamienny (Meta AI & Sorbonne), Guillaume Lample (Meta AI), Sylvain Lamprier (Univ. d'Angers), Marco Virgolin (CWI)
- **Venue**: ICML 2023
- **Keywords**: symbolic regression, Monte-Carlo Tree Search, neural mutation, expert iteration, pre-training, transformer, SRBench, genetic programming
- **One-liner**: 将预训练的 Transformer 变异模型嵌入 MCTS 框架，通过专家迭代在线微调，在 SRBench 上实现 SOTA 的准确率-复杂度 Pareto 前沿。

## 精华提炼

### 管线/方法

完整 pipeline：

1. **预训练阶段**（离线）：
   - 用手工设计的生成器（Lample & Charton 2019）随机采样合成表达式 f*，维度 1-10，内部节点 5-25
   - 对每个 f*，从完整表达式开始 **逐步拆解**（procedural dismantling）：随机选一个内部节点，移除该节点（二元节点还需移除一个子树 B），连接父节点与剩余子节点 A
   - 反转拆解序列，得到训练序列：∅ → m₁ → f⁽¹⁾ → m₂ → f⁽²⁾ → ... → f*
   - 训练 Transformer encoder-decoder（M_θ）：输入为 (数据集 D, 当前表达式 f⁽ℓ⁻¹⁾)，输出为变异 mℓ = <A, op, B>
   - A 用节点索引表示（整数），op 从预定义算子表选取（见下），B 为新的子表达式
   - 预训练在 8 GPU 上跑了 12 天

2. **MCTS 搜索阶段**（在线）：
   - 搜索树节点 = 符号表达式，边 = 变异
   - 根节点 = 空表达式
   - 每次 search trial = 1000 次 MCTS 迭代，包含三步：
     - **Selection**：从根到叶的路径选择，用 PUCT（Predictor Upper Confidence for Trees），选择使 V(f') + p_uct · E(f') · M_θ(f'|f,D,θ) 最大化的子节点
     - **Expansion**：对叶节点 f，从 M_θ 采样 K 个变异，生成 K 个子节点 {f'_k}；每个表达式评估 R²（可选常数优化）
     - **Back-propagation**：将新节点的值回传给所有祖先节点
   - 节点值：解决问题（R²≥0.99）得 v=1.0，否则用 critic C_ψ 估计 v=C_ψ(D,f)
   - C_ψ 与 M_θ 共享权重，额外加一个标量输出头

3. **在线学习/专家迭代**：
   - search trial 结束后更新 M_θ 和 C_ψ
   - M_θ 从成功的变异序列中学习（模仿学习/交叉熵损失）
   - C_ψ 从 MCTS 节点的访问值中学习
   - 同时在多个数据集上训练（transfer learning）
   - 继续从合成数据采样防止灾难性遗忘（类似 AlphaStar/HTPS）

4. **分布式架构**：
   - 1 个 controller 调度数据集
   - 4 个 worker 执行 search trial
   - 4 个 trainer 更新模型参数
   - 每个 worker/trainer 1 GPU + 1 CPU

### 关键技术细节

**模型架构**：
- Transformer encoder-decoder（基于 Kamienny et al. 2022 的 E2E 架构）
- 输入上下文限制为 100 个数据点
- 最多 10 个输入变量
- 表达式用 prefix notation 表示
- 数据集用 tokenized (x_i, y_i) 对表示
- 浮点数用 3 token 表示：符号 + 尾数(0-9999) + 指数(E-100 到 E100)

**变异算子**：
- 一元：A→cos(A), sin(A), tan(A), exp(A), log(A), A^0.5, A^{-1}, A², 0→B
- 二元：A→A±B, A→A×B, A→A/B, A→B±A, A→B×A, A→B/A
- 变异只能增大表达式（不能缩小）
- 限制表达式最多 60 个节点
- 禁止嵌套复杂算子（如 cos(cos(x))）

**超参数采样**（每次 search trial 随机采样）：
- K ∈ [8, 16]（每次 expansion 的采样数）
- Temperature ∈ [0.5, 1.0]
- Length penalty ∈ [0, 1.2]
- Depth penalty ∈ {0.8, 0.9, 0.95, 1}
- p_uct = 1（exploration 常数）

**常数优化**：
- BFGS 算法，batch size 256
- 早停：10 次迭代无改善
- 超时：1 秒
- 在 MCTS 树中存储未优化表达式（避免优化后常数 OOD 影响 M_θ）

**评估标准**：
- 数据集"解决"定义为 R² ≥ 0.99
- 预算：500,000 次 evaluations，24 小时时限
- SymPy 简化后计算表达式大小

### 核心结果

**SRBench 基准**：
| 指标 | DGSR-MCTS | E2E | GP-GOMEA |
|------|-----------|-----|----------|
| Black-box median R² | **0.846** | 0.797 | ~0.84 |
| Black-box expr size | **41** | 61 | ~20 |
| Feynman solved (R²≥0.99) | 80% | 87% | ~75% |
| Feynman expr size | **33** | 121 | ~15 |

- DGSR-MCTS 在 black-box 和 Feynman 数据集上都位于 Pareto Rank 0 前沿
- 与 GP-GOMEA 在准确率-复杂度权衡上互有胜负
- 比 GP-GOMEA 在 expression evaluations 数量上更高效（得益于预训练）

**消融实验**：
| 消融 | 结论 |
|------|------|
| Mutation size | @10 最优 (OOD: 44%); @1 差 (26.8%); @∞ OOD 崩塌 (16.8%) |
| K (breadth vs depth) | K=8-16 最优; K=1 (greedy) 几乎无法解决 (0.8% OOD) |
| 常数优化 | All expressions: 79.6%/66.2% vs Never: 74.8%/44.0% |
| 同时多数据集训练 | yes+yes: 0.846/0.796 vs no+no: 0.801/0.655 |

**Surprising/反直觉结果**：
- Mutation size ∞（即 E2E 风格，一步生成整个表达式）在 in-domain 表现不差（72.4%），但 OOD 严重崩塌（16.8%），证实搜索的重要性
- 用原始 R² 做节点值会产生 deceptive rewards（类似国际象棋中弃子），critic 网络显著更好
- ~90% 的采样变异是有效表达式（Transformer 学会了表达式语法）

### 明确局限

**作者自述的局限**：
- Transformer 上下文长度限制（最多 100 数据点，10 变量），需要 LongFormer 等解决
- GPU 依赖：采样变异需要 GPU forward pass，而 GP 可以纯 CPU 运行
- 变异只能增大表达式，不能缩小或替换（留作 future work）

**读出的额外局限**：
- 仅在 SRBench 上评估，未在 PDE 发现、ODE 系统等任务上测试
- 预训练 12 天 / 8 GPU 成本高昂
- 表达式生成器（合成数据）的设计限制了表达能力：最多 25 个内部节点，固定算子集
- 特征选择限制：只处理 ≤10 变量的数据集
- 未与 LLM-based approaches（如用 GPT/Codex 生成表达式）比较
- 常数优化超时 1 秒限制了对复杂常数的恢复能力

## Read by: llm-mcts-sr (2026-06-05)

本文是 **Neural Model + MCTS for Symbolic Regression** 范式的代表作，发表于 ICML 2023。核心贡献是将预训练的 Transformer 变异模型嵌入 MCTS，通过 expert iteration 在线微调。

**撞车评估**：如果 llm-mcts-sr 的思路是"用 LLM 替代自定义 Transformer 做变异策略 + MCTS 搜索"，则与本文存在显著方法论重叠。差异化必须聚焦在 LLM 带来的 **质变**（而非量变）上：例如利用 LLM 的自然语言推理能力、物理先验知识、多步 chain-of-thought、代码生成能力等，而不仅仅是换一个更大的模型。

**可用工具**：MCTS 框架（selection/expansion/backprop）、变异表示设计、expert iteration 训练范式、SRBench 评估协议均可直接借鉴。

**差异化关键**：
1. 表示层面：LLM 可以用自然语言/Python 代码表示变异，而非 prefix notation token 序列
2. 推理层面：LLM 可以做 chain-of-thought 分析数据模式，而非纯粹的 pattern matching
3. 知识层面：LLM 的预训练知识包含物理定律、数学公式，可以做 informed search
4. 架构层面：LLM 可以同时做 proposer 和 critic（self-evaluation），无需额外 critic 网络
5. 灵活性：LLM 支持 shrink/replace 变异，不限于只增大表达式
