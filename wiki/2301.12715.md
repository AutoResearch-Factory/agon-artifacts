# Fine-Tuning Deteriorates General Textual Out-of-Distribution Detection by Distorting Task-Agnostic Features (2301.12715)

- **Year**: 2023.01
- **Authors**: Sishuo Chen, Wenkai Yang, Xiaohan Bi, Xu Sun (Center for Data Science / MOE Key Lab of Computational Linguistics, Peking University)
- **Venue**: EACL 2023
- **Keywords**: OOD detection, distribution shift, fine-tuning, pretrained language models, Mahalanobis distance, feature distortion, probing classifiers, RoBERTa
- **One-liner**: 评测文本OOD方法后发现微调利于语义偏移、损害非语义偏移检测，提出融合两类特征距离的GNOME分数。

## 精华提炼

### 管线/方法

1. **问题形式化**：OOD 检测视为二分类，检测器 D(x) 由置信度分数 S(x) 与阈值 γ 决定 ID/OOD。
2. **基准构建**：构造 8 组 ID/OOD 数据对，分两类分布偏移：
   - 非语义偏移 (NSS，同类别不同背景)：情感分析 SST-2 ↔ IMDB；毒性检测 Twitter ↔ Jigsaw（互为 ID/OOD）。
   - 语义偏移 (SS，OOD 是 ID 任务中未见过的新类别)：新闻分类 AGNews、News Category (NC)；对话意图分类 ROSTD、CLINC（各自保留部分类别作为 OOD）。
   - 另建 2 个跨任务基准（两类偏移同时发生）：SST-2 和 20 Newsgroups 分别作 ID，TREC-10 / WMT16 / Multi30k / RTE / SNLI 作 OOD。
3. **模型配置**：在 ID 数据上微调 RoBERTa-base(110M) 作为分类器；MD_pre 直接用预训练（未微调）RoBERTa-base 抽特征；PPL 基线微调 GPT-2-small(117M) 做语言模型。
4. **主评测**：在 3 大类、约 12 种 OOD 打分方法上，用 AUROC↑/FAR95↓ 两个指标、5 个随机种子取平均，跑遍全部 8+2 个基准。
5. **归因分析**：t-SNE 可视化微调前后特征分布；用 SentLen / BShift 两个探针任务（probing classifier）沿微调 epoch 追踪"任务无关语言知识"的保留程度，与 ID 准确率、NSS 场景 AUROC 对比走势。
6. **正则化消融**：测试 3 种旨在"微调时保留预训练特征"的正则化技术（RecAdam、分类头 10× 学习率、LP-FT），验证是否能弥合 MD_ft 与 MD_pre 在 NSS 上的差距。
7. **提出 GNOME**：对 MD_pre(x) 和 MD_ft(x) 各自做 z-score 标准化（均值/标准差在 ID 验证集上估计），再取均值聚合，取负作为置信度分数——无需额外训练，只需额外一次预训练模型的前向推理。
8. **验证与消融**：在全部基准上验证 GNOME；消融归一化开关、聚合算子 (mean vs max)、分数级融合 vs 特征级融合、预训练特征池化方式 (last-CLS / last-avg / first-last-avg)、骨干网络 (BERT-base / RoBERTa-base / RoBERTa-large)；在等推理开销约束下与模型集成 (model ensemble)、MC Dropout 对比。

### 关键技术细节

- **骨干网络**：RoBERTa-base 110M（主实验）；BERT-base-uncased 110M、RoBERTa-large 355M（泛化性检验）；GPT-2-small 117M（PPL 基线用语言模型）。
- **训练超参**：Adam，lr=2e-5，batch size=16，5 epoch，按 ID 验证集准确率选最优 checkpoint；所有骨干/损失函数统一此配置。
- **指标**：AUROC（越高越好）与 FAR95（真阳率 95% 时的假阳率，越低越好），均为 5 个随机种子平均值；论文表格中**未报告标准差/置信区间**。
- **Mahalanobis 距离公式**：MD(x) = min_c (z-μ_c)ᵀΣ⁻¹(z-μ_c)，S(x) = -MD(x)；μ_c、Σ 由 ID 训练集按类估计。
- **GNOME 公式**：Norm(MD) = (MD-μ)/σ（μ,σ 由 ID 验证集估计）；S_GNOME(x) = -mean(Norm(MD_pre(x)), Norm(MD_ft(x)))。
- **置信度类基线**：MSP（最大 softmax 概率）、Scaling（ODIN 式温度缩放，固定 T=1000，因文本离散不使用图像 ODIN 的输入扰动步骤）、Energy Score（logits 的 log-sum-exp）、D2U（输出分布到均匀分布的 KL 散度）。
- **密度类基线**：PPL——ID 数据微调的 GPT-2-small 语言模型困惑度的倒数。
- **距离类基线**：LOF（微调特征上的局部离群因子）；KNN_pre/KNN_ft（k=10 最近邻距离，沿用 Sun et al. 2022 的视觉域方法）；MD_ft+L_scl / MD_ft+L_margin（Zhou et al. 2021 的监督对比损失/margin 对比损失微调，τ=0.3，λ=2）。
- **探针任务**：SentLen（句长分桶）、BShift（二元语法交换检测），来自 Conneau & Kiela 2018 SentEval，每个任务 100k/10k/10k 训练/验证/测试样本，用作"embedding 中残留多少任务无关语言学信息"的代理指标。
- **正则化消融方法**：RecAdam 优化器；仅分类头使用 10× 学习率；LP-FT（先冻结骨干只训分类头，再解冻整体微调）。
- **硬件**：单张 NVIDIA TITAN RTX (24GB) 可跑全部实验，仅 RoBERTa-large 微调需要 4 张。
- **代码开源**：https://github.com/lancopku/GNOME。

### 核心结果

- **核心权衡**：所有已有方法至少在一个场景失败——MD_pre/KNN_pre 在 NSS 上最优（FAR95 低至 ~29.90%/28.67%），但在 SS 上表现差（FAR95 最高到 70.68%）；MD_ft 系与 KNN_ft 在 SS 上最优，但在 NSS 上表现差（FAR95 最高到 80-89%）。
- **8 基准套件平均**：GNOME 平均 FAR95 = 36.50%，比此前最优基线 MD_ft+L_margin (44.63%) 低 8.13 个点；GNOME 自身 NSS/SS 分别为 36.88%/36.10%，是唯一在两个场景间基本均衡的方法。
- **跨任务场景**：GNOME 平均 FAR95 = 13.02%，比次优 MD_pre (17.90%) 低 4.88 点；20 Newsgroups 为 ID 时 GNOME 达到 0.01% FAR95（NSS 主导场景下与 MD_pre/KNN_pre 持平）；SST-2 为 ID 时 GNOME (26.02%) 优于此前最优的 MD_ft+L_margin (29.43%)。
- **探针相关性**：SST-2 微调过程中，ID 测试准确率单调上升，但 SentLen/BShift 探针准确率与 NSS-AUROC（以 IMDB 为 OOD）同步下降——为"微调破坏任务无关特征"的解释提供直接经验证据。
- **正则化消融结果**：RecAdam / 头部 10× 学习率 / LP-FT 只能部分弥补 NSS 差距（最好的 LP-FT：平均 FAR95 50.92%，NSS 60.26%），仍远不及纯 MD_pre 的 29.90%——说明"正则化微调后的模型"不足以解决问题，从而引出 GNOME 直接复用未改动预训练模型的思路。
- **GNOME 自身设计消融**：去掉 z-score 归一化会让 NSS 和跨任务场景的 FAR95 各恶化约 7 个点（尽管 SS 反而改善约 3 点）——说明不归一化的简单平均会被数值尺度更大的一侧主导；mean 聚合优于 max；分数级融合优于特征级融合（拼接/平均特征），后者在 SS 场景恶化达 +20 个 FAR95 点。
- **泛化性**：NSS/SS 权衡现象在 BERT-base-uncased 和 RoBERTa-large 上同样复现；RoBERTa-large 的 MD_pre/MD_ft 原始数值反而劣于 RoBERTa-base——更大的模型未必带来更好的 OOD 检测能力。
- **等推理开销对照**：固定 2 次前向推理预算下，GNOME (36.50 平均 FAR95) 大幅优于 2-模型集成的 MD_ft (56.09) 和 2 次 MC-Dropout (72.68)——说明收益specifically来自融合预训练+微调两种表示，而非单纯"集成有效"。

### 明确局限

- **作者自陈**：GNOME 在任一单一场景下都不是最优方法——NSS 上略逊于 MD_pre/KNN_pre，SS 上略逊于 MD_ft(+margin)/KNN_ft；论文明确将其定位为在偏移类型未知时的稳健折衷，而非帕累托最优解。
- **无理论保证**：GNOME 的"两个归一化 Mahalanobis 分数取均值"纯粹由经验动机得出，没有关于何时/为何该聚合方式最优的形式化分析。
- **机制性解释的证据强度有限**：核心论断"微调破坏任务无关特征"仅由 2 个代理探针任务 (SentLen, BShift) 加单一数据集 (SST-2→IMDB) 上沿训练轨迹的相关性支撑，不是受控干预实验；作者自己也用"we speculate""empirically indicate"等措辞留有余地。
- **无方差/置信区间报告**：虽然 5 个随机种子取平均，但所有表格只报点估计，没有标准差或显著性检验。
- **额外推理开销**：需要额外一次预训练骨干的前向推理（论文承认并论证为"弱需求"，也在等开销下与集成方法公平对比，但对部署仍是真实成本，论文未尝试消除）。
- **覆盖范围窄**：全部实验限于 encoder-only Transformer 分类器 (BERT/RoBERTa，≤355M) 和英文学术 NLU 基准（情感、毒性、新闻主题、意图），2023 年 1 月投稿，早于生成式 LLM 主导期；未测试 decoder-only LLM、非英语任务，也无真实部署试验。
- **与时序/因果推断毫无方法交集**：整篇论文没有混杂 (confounder)、非平稳性或时间动力学的概念——这里的"分布偏移"始终是静态的训练/测试数据集配对差异，不是演化中的物理系统。
