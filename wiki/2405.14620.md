# Closed-form Solutions: A New Perspective on Solving Differential Equations (2405.14620)

- **Year**: 2024.05 (ICML 2025)
- **Authors**: Shu Wei, Yanjie Li, Lina Yu, Min Wu, Weijun Li, Meilan Hao, Wenqiang Li, Jingyi Liu, Yusong Deng (Institute of Semiconductors, CAS)
- **Venue**: ICML 2025
- **Keywords**: symbolic regression, differential equations, reinforcement learning, closed-form solution, PDE solving, risk-seeking policy gradient, recursive exploration, expression tree
- **One-liner**: SSDE uses RL to generate symbolic closed-form solutions for ODEs/PDEs, with a recursive strategy to handle high-dimensional PDEs by solving one dimension at a time.

## 精华提炼

### 背景与问题

微分方程是数学和物理科学的基石，但寻找其解析解需要大量数学专业知识。传统数值方法（有限体积法）存在离散化误差；PINNs 等深度学习方法牺牲了解释性且计算效率低；遗传编程（GP）方法计算成本高，且在高维问题中容易产生冗长复杂的表达式。

核心挑战在于三个目标的平衡：(1) 严格遵守物理定律，(2) 计算效率，(3) 人类可解释的符号形式。

### 管线/方法

SSDE 的整体管线如下：

1. **表达式骨架生成** (图 a): 使用 RNN 自回归地采样 token，生成表达式的 pre-order traversal（即表达式骨架）。每个常数 token 是一个列向量（长度=|c_dim|），使得单个骨架在 |c_dim| != 1 时能对相同输入产生多个输出。
   - RNN 输入包括当前 token 的父节点和兄弟节点的表示
   - 使用 ApplyConstraints 过滤不合法 token（如除法中分母为零的 token）

2. **局部常数优化 (LCO)** (图 b-c-d): 对每个采样的表达式骨架，用自动微分计算物理约束（PDE 残差 + 边界条件 + 初始条件）的 MSE 损失，然后用 BFGS 算法优化骨架中的常数。
   - 损失函数: L_s-t = lambda0 * MSE_F + lambda1 * MSE_B + lambda2 * MSE_I
   - 数据点从时空域随机采样

3. **评估器** (图 e): 使用 squash 函数将物理约束的 RMSE 平均值映射为奖励信号。奖励范围在 (0,1]。
   - R_s-t = 1 / (1 + Average(sqrt(MSE_F) + sqrt(MSE_B) + sqrt(MSE_I)))

4. **策略梯度训练**: 采用 risk-seeking policy gradient，只优化 top epsilon 分位数的 episode。训练时使用分层熵正则化促进探索。
   - J_risk(theta; epsilon) = E[R(u) | R(u) >= R_epsilon(theta)]
   - 分层熵: H(theta) = lambda_H * E[sum gamma^(i-1) * H[p(tau_i|...)]]

### Risk-Seeking Constant Optimization (RSCO)

RSCO 是 SSDE 的关键加速技术。核心思想：在每轮训练中，先仅用确定性条件（边界条件+初始条件）对所有批次骨架进行常数优化，计算粗粒度奖励。取 top epsilon 的骨架，再使用完整物理约束（包括 PDE 残差）做精确优化，然后用精确奖励计算策略梯度。

这样做避免了在明显违反物理约束的表达式中浪费 PDE 残差计算成本。RSCO 带来 4.65x 加速（平均求解时间从 934s 降到 200s）。

### 递归探索策略

这是 SSDE 解决高维 PDE 的核心创新：

1. **参数化表达式**: 将解表示为关于单个变量 x_i 的表达式，其他变量作为波动参数 alpha(x_{-i})
2. **递归调用**: 从 k=0 开始（解 = 一个常数参数 alpha_0），逐维展开。第 k 轮找出关于 x_k 的参量化骨架，将上一轮的参数替换为新的子表达式。
3. **参数退化为常数判定**: 如果某个参数向量在采样点上的方差低于阈值 delta，则视为常数并固定为均值。

最终解形式: u = u1(x1; u2(x2; ... ud(xd; c)))

该策略假设 PDE 的解是变量可分离的（additive separable 或 multiplicative separable），对于高度耦合的 PDE 系统不适用。

### 关键技术细节

- **RNN 架构**: 单层 RNN，cell size=32
- **学习率**: 0.001
- **熵权重 lambda_H**: 0.07
- **熵衰减系数 gamma**: 0.7
- **Risk factor epsilon**: 0.05
- **Batch size**: 1000
- **Max epoch**: 200
- **损失权重**: lambda_0 = lambda_1 = lambda_2 = 1
- **常数优化**: BFGS 算法
- **符号库**: L0 = {+, -, *, /, sin, cos, exp, log, x1, const}
- **计算平台**: Intel Xeon Gold 6138 CPU @ 2.00GHz（支持 GPU 加速）
- **代码**: https://github.com/Hintonein/SSDE

| Poisson3D | 5.70e-5 | **100%** | 2.60e+0 | 1.80e-1 |
| Heat2D | 1.47e-6 | **100%** | 3.21e+0 | 2.15e-1 |

**Nguyen 基准（Poisson 方程版本，12 个方程）**：
- SSDE 在所有 12 个方程上取得 100% 恢复率
- PINN+DSR 仅在部分简单 ODE（Gamma 1-3, 5-6, 8）上成功，在 2D PDE 上全面失败
- 关键发现：PINN 的数值近似误差会在后续符号回归中传播放大

**额外基线（相对 L2 误差）**：
- SSDE: 1e-5 ~ 1e-6
- Mathematica: 不能求解任何方程（marked as fail）
- FEX: 1e-1 ~ 1e-2
- DSR: 1e-1 ~ 1e+0
- PINN: 1e-2 ~ 1e-3

### 明确局限

1. **变量可分离性假设**: 递归探索策略假设 PDE 的解可以按变量分解。对于高度耦合的非线性 PDE（如 Navier-Stokes），如果解的形式不是 additively/ multiplicatively separable，SSDE 无法工作。
2. **符号库依赖性**: 符号库需要预先包含正确的基函数。如果真实解涉及符号库中不存在的函数，SSDE 无法恢复。
3. **Benchmark 简单性**: 所有测试方程的解都是比较简单的多项式/三角函数/指数函数组合，没有测试真正复杂的 PDE（如含奇异性的问题、混沌系统）。
4. **没有真实验证**: 所有实验基于已知真解的合成基准，没有在真实物理系统中验证。
5. **效率信息不完整**: 论文报告 RSCO 4.65x 加速，但未报告各基线方法的绝对运行时间对比。PR-GPSR 需要约 1 天完成 1500 代，但 SSDE 的总时间未明确给出（200 epoch x 1000 batch 的采样+BFGS 优化）。
7. **代码可复现性**: 代码已公开（GitHub），但论文没有报告使用 GPU 与否；所有实验在 CPU 上运行，RSCO 加速的数据来自哪个配置不够明确。

### 跟我们的关系

**0616-fex 课题**: SSDE 直接与 FEX 方法对比，将其定位为基线并声称大幅超越（SSDE L_PHY ~1e-5 vs FEX L2 ~1e-1）。这是直接竞争关系。

**撞车分析**:
- SSDE 和 FEX 都使用 RL 搜索符号表达式来求解 PDE
- SSDE 的核心差别在于：(1) RSCO 加速常数优化，(2) 递归逐维求解策略
- 这些创新对 0616-fex 课题是直接的威胁：如果 SSDE 的方法确实有效，FEX 方向需要找到差异化点

**可用工具/方法**:
- RSCO 加速策略可以移植到 FEX 管线中（先用边界条件快速筛选，再精细优化）
- 递归逐维求解的思想与 FEX 的高维求解策略可以交叉融合
- SSDE 的 parametric expression 形式（常数 token 作为向量）是一个值得借鉴的实现细节

**差异化方向**:
- SSDE 解决的 PDE 类型有限（需要变量可分）；FEX 可以探索不可分耦合 PDE
- SSDE 对符号库高度敏感；FEX 可以通过权重网络减少对精确符号匹配的依赖
- SSDE 的"100% 恢复率"只在简单 synthetic benchmark 上，在真实 PDE 上未必成立
- 需要验证 SSDE 的递归策略在更高维（10+ 维）或非标准域上的表现

**撞车评估**: 本文与 FEX 课题直接竞争。SSDE 声称在 2D/3D Poisson、Heat、Wave 方程上全面超越 FEX（相对 L2 误差小 3-4 个数量级）。如果 SSDE 的 claim 经得起复现验证，FEX 在简单可分离 PDE 上的优势就不存在了。

**可用工具/方法**: RSCO 加速策略（先用边界条件粗筛再精细优化的两阶段常数优化）是通用的加速技巧，可以移植到任何 RL-based 符号回归管线。递归逐维求解策略对 FEX 的高维求解也有借鉴价值，但需要解决它对变量可分假设的依赖。

**差异化**: FEX 相对于 SSDE 的潜在优势在于：(1) FEX 的权重网络结构在处理非精确符号匹配时可能更鲁棒（SSDE 强调精确符号恢复，FEX 强调近似但可解释的解）；(2) 对于非可分离的 PDE 系统，SSDE 的递归策略失效，FEX 可能还能工作；(3) SSDE 的所有实验基于已知真解的合成基准，在未知真实 PDE 上的表现未知。建议在 FEX 的后续工作中：(a) 测试 SSDE 论文中的 benchmark 以公平对比，(b) 探索 SSDE 无法处理的 PDE 类型（不可分离、奇异边界等），(c) 将 RSCO 或类似加速策略引入 FEX。
