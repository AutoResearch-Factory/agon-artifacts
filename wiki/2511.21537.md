# Context-Specific Causal Graph Discovery with Unobserved Contexts: Non-Stationarity, Regimes and Spatio-Temporal Patterns (2511.21537)

- **Year**: 2025.11
- **Authors**: Martin Rabel (University of Potsdam, Institute of Computer Science; ScaDS.AI Dresden/Leipzig), Jakob Runge (同机构; ERC Starting Grant "CausalEarth" PI)
- **Venue**: arXiv preprint (cs.LG, math.ST) — 未见会议/期刊投稿信息
- **Keywords**: hidden-context causal discovery, 非平稳因果图, regime/context 检测, 约束型因果发现 (PC/FCI/PCMCI 族), marked conditional independence test (mCIT), 不可能性定理, 时空 persistence pattern
- **One-liner**: 用三值CIT直接判定隐藏regime存在性, 证明两条根本不可能定理, 套壳标准因果发现算法.

## 精华提炼

### 管线/方法

1. **输入**: 多变量 (IID 或时间序列) 数据, 假设存在隐藏/未知的 regime 结构 —— 即非平稳 SCM `X_i^t = η_i^t + Σ_j R_ij(t) f_ij(X_j^t)`, 其中二值指标 `R_ij: T→{0,1}` 决定某条边在时刻/位置 `t` 是否激活, `T` 可以是时间、空间或任意索引集.
2. **核心改写**: 不先重建"何时处于哪个 regime"(间接路线: CPD/聚类 → 逐 regime 跑 CD), 而是直接修改 CIT 本身, 让每次 `X⊥Y|Z` 查询返回三值: (a) 全局独立, (b) 全局依赖, (c) "存在真实 regime"(在某子集独立, 在另一子集依赖). 这个三值测试称为 marked CIT (mCIT).
3. **mCIT 内部流程** (对每个测试 `X⊥Y|Z`):
   - a. 选定 persistence pattern (默认: 不重叠的时间对齐 block, 大小 `B`; 可泛化到空间分块、图网络邻域分块, 或时空乘积 pattern). 这个先验 pattern 是必需的, 因为 Impossibility Result A 证明: 若不利用索引集上的任何非 IID 结构, 混合分布的可辨识性一般不成立 (任意密度都可形式化写成"更小分布 ⊕ 残差"的混合, 系数上界 `Ξ` 可任意接近 1).
   - b. 在每个 block 上计算依赖分数 `d̂` (数值实验中用偏相关).
   - c. **同质性检验优先**: 用二项检验判断 block 集合是否 IID (若非齐性, 视为存在某种 regime 结构).
   - d. 若齐性 → 标准依赖检验 (全局独立 vs. 全局依赖).
   - e. 若非齐性 → **弱 regime 检验**: 低依赖簇的均值 (经截断正态理论修正选择偏差和高依赖簇"杂质"污染后得到的 acceptance interval) 是否与 `d0=0` 相容? 是 → 判定"真实 regime"; 否 → 归为"全局依赖"(因为 Impossibility Result B 证明这个模糊地带在有限样本下无法被任何检验彻底厘清, 只能默认更安全的标签).
4. **状态空间构建** (两阶段算法, 输入是 mCIT 标记出的 marked test 集合 `J`):
   - Phase I `discover_model_indicators`: 对每条被标记为变化的边, 用"indicator implication test"(与弱 regime 检验同构的 acceptance-interval 逻辑, 但比较对象换成另一个 indicator 而非零) 找到一个不被其他变化边"感染"的最小代表检验, 使其对应的 detected indicator 恰好等于真实的 model indicator.
   - Phase II `represent_indicator`: 利用"去掉一条边只能关闭 d-connecting 路径、不能打开新路径"的单调性引理, 把每个被标记检验的 detected indicator 表示为已发现 model indicator 的单调布尔组合 (AND-of-ORs), 避免朴素 `2^(2^k)` 组合爆炸.
5. **核心算法** (`run_hccd`, Algorithm 1): 包装任意标准约束型 CD 算法 (PC/PC-stable/FCI/PCMCI/PCMCI+/LPCMCI). 用一个"pseudo-CIT"喂给底层 CD: 对已知状态依赖的测试直接返回当前假设状态 `s` 规定的值; 对未知测试调用 mCIT, 若返回"真实 regime"则记入 `J'` 并默认按"依赖"处理 (故第一轮总是先恢复 union graph). 迭代: 由 `J` 构造状态空间 → 对每个可达状态跑一次 CD → 收集新发现的 `J'` → `J:=J∪J'` → 重复至 `J'=∅`. 因为 union graph 已经执行了任一单一 regime 骨架所需测试的超集 (稀疏性下更密的图蕴含被执行测试的超集), 实践中通常恰好 2 轮收敛.
6. **输出**: 每个可辨识状态对应一张图 (等价类); 在无环假设下边的朝向不会跨 regime 改变 (只有存在/缺失会变), 因此朝向可在状态间无损传递, 最终折叠成单张"带标记 union-graph"(变化的边着色/加标签), 与 Fig.1 toy-model 的展示一致.

### 关键技术细节

- **两条不可能性定理** (全文理论核心, 详细形式化证明见 App. §Persistence/§Weak Regimes):
  - **Impossibility Result A**: 不利用索引集上任何 pattern 时, marked independence (进而整个 HCCD 问题) 一般不可辨识. 证明构造性: 任意联合密度 `p` 都可写成 `β·p0 ⊗ p0' + (1-β)·p1`, 其中 `β` 上界 `Ξ` 对强相关的二元正态可任意接近 1 (`Ξ ≥ sqrt((1-|ρ|)/(1+|ρ|))`).
  - **Impossibility Result B**: 在"弱 regime"子问题中 (两个 regime 依赖值 `d0≠d1`, 关注 `d0` 是否为 0), 不存在任何有限样本 `N` 的检验能同时 (a) 在真实 regime 上控制 FPR 且对全局独立有非平凡功效, 或 (b) 在全局依赖上控制 FPR 且对真实 regime 有非平凡功效 —— 由 `Pr(T̂=R|d0,Δd)` 在 `(d0=0,Δd=0)` 附近的连续性论证得出, 是方法无关的根本限制, 不是某个具体估计器的工程缺陷.
- **同质性检验**: 二项检验, 比较低于估计的 `β`-分位数阈值的 block 数量; 需要 `B→∞` 且 `B/Λ(N)→0` (`Λ` 为 regime 增长尺度) 才有功效, 除非存在无偏分位数估计器 (罕见情形下固定 `B` 即可, 一个有趣的理论旁支).
- **弱 regime "acceptance interval" 检验**: 基于截断正态理论推导的区间 `[μ_{0,σB}^c - σ_α/√n_c, σ_α/√n_c]`, 在 `χ≈0`(少量无效 block) 且 `c<q_a`(截断点小于独立占比分位数) 下控制 FPR.
- **mCIT 收敛定理** (Thm. mCIT): 在弱假设 + 任意与 regime 元模型渐近 `Λ`-相容 (`Λ→∞`, 增长速度任意慢) 的 pattern 下, 组合检验"弱渐近正确"(Bayesian 平均意义, 非逐点/一致). 覆盖典型的"无信息先验"情形 (典型 regime 长度仅随 `log(N)` 增长).
- **核心算法可靠性定理** (Thm. core_algo / Thm. state_space_construction): 给定 oracle mCIT、一致的底层 CD 算法、可靠且完备的状态空间构造, 算法可证明在有限轮内终止, 且对每个 CD 可辨识状态恰好输出一张正确的图 —— 完整的可靠性+完备性证明链条横跨约 3000 行附录.
- **时间序列适配**: 吸收 PCMCI 的 MCI 思想 (先用 PC1 阶段找滞后父节点超集, 再对 momentary 独立性做条件测试), 仅在 MCI 阶段执行完整 mCIT. 论文坦承在时间序列 (非 IID) 场景下 union-graph 恢复质量随 `N` 增大而**退化**, 且未能完全解释原因 (候选解释: 同质性优先的检验顺序与 MCI 阶段的相互作用; 或"`k` 条长度 `n` 的序列 ≠ 一条长度 `kn` 的序列"导致零假设分布变宽) —— 明确列为未解决问题, 而非回避.
- **数值实验规模**: 数十条扫描轴 (样本量 `N=10²..10^4.5`、节点数、链接密度、变化指标数 [模块化 vs. 同步]、regime 尺度、6 种噪声分布族含 Cauchy), 附录表格记录具体 run 数 (如全 mCIT 扫描单项 16,496 次). 对比基线: Regime-PCMCI (3 组超参)、k-means 聚类+PCMCI、"事后最优"滑动窗口+PCMCI (明确标注为作弊式上界)、平稳 PCMCI.
- **代码**: 开源 Python, `github.com/martin-rabel/Causal_GLDF`, 依赖 tigramite (PCMCI 族) 与 causal-learn (PC/FCI 族).

### 核心结果

- **随 N 的 scaling 行为是全文最核心的实验发现**: 本文方法 (gLD-PCMCI/gLD-PCMCI+) 与"事后最优"滑动窗口是仅有的随数据增多而**变好**的方法; Regime-PCMCI 与聚类+CD 在无信息先验下随 N 增大反而**系统性变差** —— 原因是这类"先重建 regime、再测试"的两阶段管线在与独立性检验的收敛速度赛跑: 检验灵敏度以 `N^{-1/2}` 提升, 若 regime 重建误差下降更慢, 最终检验会探测到重建误差本身. 这是一个既有理论论证又有数值确认的、方法无关的"间接管线"失效机制.
- **随节点数 scaling**: 本文方法 (graph-local) 几乎不受节点数增加影响; 所有"全局"对比方法均明显退化 —— 直接验证了 locality 论点.
- **随变化指标数 (regime 结构复杂度) scaling**: Regime-PCMCI 在独立变化的指标数超过约 3 个后灾难性崩溃 (收敛失败); gLD-PCMCI 仅温和退化.
- **小规模对照表** (5–10 节点时间序列): gLD-PCMCI(+) 达到 F1 0.78–0.80、运行时间约 0.04 秒, 相比之下 Regime-PCMCI 的 F1 仅 0.33–0.58、耗时 569–1877 CPU 秒 (差 4–5 个数量级); 更复杂的 10 节点/9 链接设置下 gLD-PCMCI(+) 仍以 F1 0.52–0.64 领先 Regime-PCMCI 的 0.05–0.14.
- 方法的 regime 检测按设计是高精度/较低召回 (样本不足时默认"无 regime"而非猜测).
- **一个真实存在、未被作者解决的异常**: 仅在时间序列 (非 IID) 场景下, 本方法自身的 union-graph 恢复质量也随 N 增大而下降, 只是初步归因, 尚未完全解决 (换用 PCMCI+ 并跳过 PC1 阶段的同质性检验可部分缓解).

### 明确局限

- **只检测图结构 (父集/边存在性) 变化**, 明确不检测、不利用效应强度或噪声分布的变化 (相同图但不同系数的"regime"对本方法不可见) —— 作者声明这是刻意的范围选择, 非疏漏.
- **全文零真实数据实验**, 100% 依赖模拟/合成基准; 作者明确表示真实数据验证留给"专门的后续文章", 因篇幅限制未纳入本文.
- 数值实验与开源实现限定于**线性模型 + 偏相关依赖分数**; 非参数/非线性依赖分数只在 Future Work 讨论, 未实现.
- 正文实验默认**因果充分 (无隐混杂) + 无环**; 状态空间构造定理 (Thm. state_space_construction) 在形式上**要求**无环性和"常规骨架发现"假设, 有环模型与 inducing-path/隐混杂情形被明确列为需要额外机制的 Future Work (§Cyclic Models, §Inducing Paths).
- 需要**相对大的数据集** (论文称主要收益在 ≳1000 数据点以上才显现) —— 这恰与同一篇论文自己的实验显示 Regime-PCMCI/聚类在小样本区间仍具竞争力形成对照, 即本方法不适合小样本场景.
- 其弱 regime/同质性检验在时间序列大 N 下对 union-graph 恢复质量存在**未解决**的退化效应 (见"核心结果"), 是作者自己承认无法完全解释的开放问题.
- mCIT 把"弱 regime"坍缩为"依赖"是 Impossibility Result B 强加的、**故意舍弃信息**的简化 —— 即在 `Δd` 小的最难情形下, 方法设计上就不追求"确定性地证明这不是隐藏 regime", 只报告更安全的 union-graph 标签. **这正是 0710-causal-scs 课题想把这种模糊地带本身当作信号、而非丢弃的区域.**

### 跟我们的关系

- **定位**: 在 `0710-causal-scs` 现有 landscape 的分组框架下, 本文横跨 D 组 (课题核心因果方法/工具: 直接扩展 PC/PC-stable/FCI/PCMCI/PCMCI+/LPCMCI) 与 C 组 (稳健因果变点检测, robust CPD, 取向与 idea 相反) —— 是目前读到的、把"是否存在隐藏 regime"做成可证明可靠判定这条路线里理论最重的一篇 (可靠性+完备性证明 + 两条不可能性定理), 比此前 D 组收录的 PCMCI+/J-PCMCI+/LPCMCI (均只是底层工具, 未处理 regime 判定本身的可靠性边界) 更进一步.
- **撞车评估**: 对 idea `causal-scs-indicator` 的核心 claim ("估计过程在假设违背下的统计脆弱性本身可作为强对流预警信号, 而非需要修正掉的噪声") **不构成直接撞车**。HCCD 的目标始终是给出 SOUND 的"regime 是否真实存在"判定, 遇到无法可靠判定的情形就退化为默认标签, 而不是把这种不确定性本身当作对外部事件 (强对流爆发) 的预测特征。它与 landscape C 组 (Causal-RuLSIF CPD/kernel-copula CPD/Causal-CCP/MEND) 同属"稳健探测器, 取向与 idea 相反"阵营, 而非 idea 想要的"不稳定性即信号"阵营。
- **但 Impossibility Result B 是目前文献中对 idea Claim 3 ("统计脆弱性 ≠ 确定性因果强度变化, 需要判据区分二者") 最硬的理论支点**: 它证明在弱分离 (`Δd` 小, 即变化幅度接近噪声量级) 区域, 不存在任何有限样本检验能同时控制两类错误 —— 这是方法无关的不可能定理, 不是某个具体估计器的工程局限。可据此论证: 在 `Δd` 小的区域, 判定的不稳定性不是"用更好方法就能消除的噪声", 而是问题本身有限样本不可判定性的表现, 这与 idea 想反用的现象在机制上高度吻合 (尽管 HCCD 本身没有在这个方向上做任何下游应用)。
- **新增候选工具** (可补入 landscape D 组"不稳定量化"工具清单): mCIT 内部的 acceptance-interval 宽度/与判定边界的距离 (`§weak_regime`, Lemma Acceptance Interval) 是一个**方法内 (within-method) 连续脆弱度分数**, 且自带 FPR 控制的理论保证, 可与已收录的 LPCMCI `min|ParCorr|` (方法内)、DAGgr `s_ij` (跨方法)、Bagged-PCMCI+ per-link bootstrap 频率并列为候选量化装置, 但独有"有校准保证"的优势。三值 mCIT 决策在滑动窗口/超参数扰动下的翻转频率, 也可类比 Bagged-PCMCI+ 的思路作为量化装置。
- **Regime-PCMCI (Saggioro et al. 2020, arXiv:2007.00267) 未被现有 24 篇 landscape 收录, 但被本文系统性 benchmark**, 并证明其在大 N 下因"重建 regime vs. 独立性检验收敛速度竞速"而性能倒退——这是一条新的、量化的"先重建后测试"两阶段管线失效机制, 补强了 idea 反对传统"先修正假设违背再看效应"路线的论据 (与 idea 自身"不去寻找更准确的因果效应"的哲学高度平行, 只是 HCCD 把这个直觉用在"图结构是否变化"而非"效应估计本身"上)。**建议将 Regime-PCMCI 补入 landscape。**
- **空间维度直接相关**: 本文显式将 persistence/pattern 泛化到空间分块 (§Examples for Patterns, 给出邻近网格/河网/图网络邻域等例子), 且引言直接以"climate applications... spatially gridded time series"为动机场景——与 topic 的 3km 环境变量场 (时空) 数据形态直接匹配, 是目前 landscape 中少数显式处理"空间"而非仅"时间"persistence 的方法论文。
- **建议**: 不必作为需要"超越"的正面竞品 baseline (机制/目标都不同, 不构成竞品), 但 (a) 应引作 Claim 3 的理论支撑 (Impossibility Result B); (b) mCIT 的 acceptance-interval 宽度值得加入 Stage-1 的"跨估计器不稳定性量化"候选列表; (c) Regime-PCMCI 的失效机制描述值得引用以论证"reconstruct-then-test"两阶段管线的系统性缺陷; (d) 反引文中的 arXiv:2510.26485 ("时间序列+空间结构"因果发现) 与本课题数据形态高度相关, 建议下一轮精读。
