# On-line Learning in Tree MDPs by Treating Policies as Bandit Arms (2605.04979)

- **Year**: 2026.05
- **Authors**: Anvay Shah, Ramsundar Anandanarayanan, Sharayu Moharir, Shivaram Kalyanakrishnan (IIT Bombay)
- **Venue**: AAMAS 2026
- **Keywords**: Tree MDPs, PAC learning, regret minimization, bandit algorithms, negative cylinder dependence, imperfect information games, confidence bounds, best response
- **One-liner**: 将 bandit 算法 LUCB/UCB 推广到 Tree MDP，通过策略间数据共享的浓度不等式实现多项式复杂度，在 Kuhn/Leduc Poker 和千万状态 RBT 上验证。

## 精华提炼

### 管线/方法

**整体 pipeline**：T-MDP 定义 → 终端状态值分解（Proposition 1）→ 基于共享数据的置信界（Theorem 1, 核心创新）→ LUCB-T（PAC）/ UCB-T（Regret）算法 → 高效策略搜索（PU 过程）→ 理论分析（instance-dependent upper bounds）→ Kuhn/Leduc/RBT 实验验证。

**管线细节**：
1. **输入**：已知 reward 函数、未知 transition 的 T-MDP；T-MDP 是从 root state s₁ 出发的有限视界树形 MDP
2. **Step 1 - 值分解**：将任意策略 π 的值 V(π) 分解为终端状态 return 的凸组合：V(π) = Σ_{σ∈X(π)} q(σ)·ρ(σ)，其中 q(σ) 是到达终端状态 σ 的概率，ρ(σ) 是路径上的折扣累积奖励（已知）
3. **Step 2 - 置信界构造**：维护每个终端状态 σ 的 play count n(σ) 和 reach count n₊(σ)；策略 π 的 play count n(π) = min_{σ∈X(π)} n(σ)；UCB = V̂(π) + β(n(π), δ)，LCB = min{V̂(π), 1} - β(n(π), δ)，其中 β(m,δ) = √[8/(3m) ln(1/δ)]
4. **Step 3 - LUCB-T (PAC)**：每轮选 V̂ 最高和 UCB 次高的两个策略，若 LCB₁ ≥ UCB₂ - ε 则停止返回 π₁
5. **Step 4 - UCB-T (Regret)**：每轮选 UCB 最高的策略执行
6. **Step 5 - 高效实现 (PU 过程)**：通过递归过程 FindMaxUCB(s) 在 O(|S||A|k_max + |Σ|) 时间内找到 UCB 最大策略；若与 V̂ 最大策略相同，则再做一次递推找出不同策略

### 关键技术细节

**核心数学贡献**：
- **Negative Cylinder Dependence (NCD)**：证明 B 变量（终端状态是否被到达的 Bernoulli 变量）满足 NCD 性质（Lemma 2）
- **Twin variables 技巧**：为每个 B 变量构造独立同分布的 twin B₀ 变量，证明 V̂（基于 NCD 变量的估计）的浓度不慢于 V̂₀（基于独立变量的估计）（Lemma 3）
- **Bernstein 不等式应用**：对独立 twin V̂₀ 应用 Bernstein 不等式，得到置信界（Lemma 4）

**算法复杂度**：
- 策略总数 |Π| = |A|^|S|（指数级），但算法只需 poly(|S|, |A|) 操作/每步
- 内存：Θ(|Σ|) 个计数器（n(σ), n₊(σ)），polylog(t) 内存（LUCB-T）/ Θ(t) 内存（LUCB-T-Uniform）
- PU 过程：O(|S||A|k_max + |Σ|) 操作，k_max 为最大分支因子

**理论界**：
- **LUCB-T 样本复杂度**：O(Σ_{σ∈Σ} |Σ|/(Δᵋ_σ)² · (log 1/δ + log Σ 1/(Δᵋ_σ)² + |S|log|A|))
  - Δᵋ_σ = min_{π∈Y(σ)} max{Δ_π, ε}：与终端状态 σ 一致的所有策略中的最小 gap
- **UCB-T 遗憾界**：R_T ≤ c · Σ_{σ: Y(σ)⊄Π_opt} (|Σ|Δᵐᵃˣ_σ/(Δᵐⁱⁿ_σ)² · ln T + ln|Π|)
  - Δᵐⁱⁿ_σ = min_{π∈Y(σ)\Π_opt} Δ_π，Δᵐᵃˣ_σ = max_{π∈Y(σ)\Π_opt} Δ_π
- 关键优势：界中 sum 是对终端状态求和（多项式），而非对策略求和（指数级）
- LUCB-T-Uniform 变体可去除 |Σ| 因子，但需要 Θ(t) 内存

**实验设置**：
- 3 个游戏：Kuhn Poker (3-card: 64 policies, 5-card: 1024 policies)、Leduc Poker (144 states)、RBT (~10⁷ states for x, ~2×10⁷ for o)
- 对手固定为 CFR+ 计算的 ε-Nash 策略
- PAC baseline: BPI-UCRL, MDP-GapE；Regret baseline: MCCFR, OPF, UCT
- 所有实验用简化的置信界 β(m,δ) = √[C/m · ln(t/δ)]，手动调参 C
- Kuhn/Leduc 本地跑，RBT 需 96 核 + 120GB RAM

### 核心结果

1. **LUCB-T vs vanilla LUCB on Kuhn Poker**：
   - 3-card Kuhn: vanilla LUCB 最优（0.473M episodes for ε=0.05），LUCB-T 需 2.170M
   - 5-card Kuhn: LUCB-T 反超（3.637M vs vanilla 7.947M），验证了策略数指数增长时数据共享的优势
   - LUCB-T-Uniform 在小问题好、大问题差（丢弃太多数据）

2. **Leduc Poker PAC**：LUCB-T 的 value gap 下降速率显著优于 BPI-UCRL 和 MDP-GapE

3. **RBT PAC**：BPI-UCRL 和 MDP-GapE 完全无法运行；LUCB-T 相比 regret 算法（UCT, MCCFR, OPF）在 value gap 下降上表现更具竞争力

4. **Regret 实验**：UCB-T 在 Leduc 和 RBT 上都一致优于 MCCFR 和 OPF；UCT 在小问题上 regret 更低，但在大问题（RBT player o）上 UCB-T 的优势显现

5. **反直觉发现**：理论上更紧的 LUCB-T-Uniform 在实践中不如 LUCB-T，因为丢弃数据在大问题上危害更大

### 明确局限

1. **假设 known rewards**：reward 函数必须已知。作者承认这对一般 MDP 是限制，但对 imperfect-information games 合理（utility 在 terminal state 确定）
2. **无一般 MDP 推广**：方法严重依赖树结构。一般 MDP 可展开为树但 terminal state 数爆炸（指数级）
3. **T-MDP 无 sample-complexity lower bound**：作者承认暂无 T-MDP 特化的下界
4. **置信界超参数需调参**：理论 δ 使用 t^{|Σ|+4} 因子在实践中过于保守，实验中使用简化版 β(m,δ) = √[C/m · ln(t/δ)]，C 需手动调
5. **RBT 上 PAC baselines 不可用**：BPI-UCRL 和 MDP-GapE 在 RBT 上无法运行，PAC 比较变成与 regret 算法对比
6. **|Σ| 因子未消除**：LUCB-T 的 sample complexity 仍有 |Σ| 因子（来自 union bound），理论上可消除但实践中代价大
7. **无函数近似**：RBT 虽大但仍是 tabular 表示，未涉及神经网络等函数近似方法
8. **仅 deterministic policies**：只考虑 deterministic policy 空间，未扩展到 stochastic policies
