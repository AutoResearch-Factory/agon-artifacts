# Knowledge Integration for Physics-informed Symbolic Regression Using Pre-trained Large Language Models (2509.03036)

- **Year**: 2025.09
- **Authors**: Bilge Taskin, Wenxiong Xie, Teddy Lazebnik (Jonkoping University, University of Haifa)
- **Venue**: arXiv preprint
- **Keywords**: physics-informed symbolic regression, LLM-guided optimization, domain knowledge integration, prompt engineering, expression tree distance, genetic algorithm SR, automated scientific discovery
- **One-liner**: 将预训练 LLM 作为物理知识评估器嵌入 SR 损失函数，通过 prompt 引导生成维度一致性/简洁性/物理合理性三维评分，替代人工特征工程实现 PiSR。

## 精华提炼

### 管线/方法
完整的 LLM-integrated SR pipeline 如下：
1. **数据输入**：合成物理数据 D (n×m 特征矩阵)，500 样本，含 5 个独立初始条件参数（质量 0.1-10kg、特征长度 0.01-0.5m、初始位移 1-100m、阻尼系数 0-1、时间），运行至平衡或 1000 步。
2. **SR 搜索**：遗传算法（DEAP/gplearn/PySR）在操作符空间 {+, -, *, /, exp, log, sin, cos} 中进化表达式树种群（pop=100, gen=50），交叉概率 0.6（DEAP），变异概率 0.05，锦标赛选择 size=3，最大树深 8。
3. **LLM 评分**：每个候选方程转为字符串送入 LLM（Falcon 7B / Mistral 7B / Llama 2 7B），temperature=0，由 prompt 引导返回 Python-list [dim_corr, simp, sim, "feedback"]。三维评分：维度一致性 dim_corr (0-1)、句法简洁性 simp (0-1)、物理合理性 sim (0-1)。
4. **复合损失**：L = w1*MSE + w2*size + w3*c，其中 c = 1 - (dim_corr + simp + sim)/3，w1+w2+w3=1。LLM 评分直接进入优化目标。
5. **输出**：最佳拟合解析方程。使用表达式树距离（expression tree distance）比较与 GT 方程的结构相似性，转换为 expression tree score = 1 - d(e1, e2)。

### 关键技术细节
- **LLM 选择**：全部 7B 参数级开源模型（Mistral 7B, Llama 2 7B, Falcon 7B），本地部署，temperature=0 确保确定性输出。故意不使用 GPT-4o/Gemini 等 API 模型（成本+数据隐私考虑）。
- **Prompt 设计原则**：四段式结构——ROLE 角色定义（scientific-reasoning assistant）、METRICS 三轴评分标准（0-1 数值区间明确锚定）、FEW-SHOT 三例（kinematics/单位错误/嵌套冗余）、TASK 严格输出格式（ONLY Python-list）。关键技巧：FEW-SHOT 示例锚定评分逻辑，跨模型输出可比。
- **三个物理场景**：(1) 自由落体 v=sqrt(2gh)；(2) 简谐运动 x=Acos(sqrt(k/m)t+φ)；(3) 阻尼电磁波 E=E0*exp(-αt/2)*cos(kx-ωt)。复杂度递增：节点数 6→9→14。
- **数据生成**：500 实验 × 5 独立参数均匀采样，先算理想 GT 值，再加 1% 标准差零均值高斯噪声（SNR≈40dB），保留 SI 单位不归一化。
- **SR 超参数**：种群 100、50 代、early stopping 阈值 0.1% 三代无改进。PySR 用 Huber loss + complexity penalty；DEAP 用 crossover 0.6 / mutation 0.05 / 树深 8；gplearn 用 operator-count 复杂度惩罚。
- **Prompt 敏感性实验（8 变体）**：A=无上下文（基线）、B=变量描述、C=实验描述、D=GT 公式、E=B+C、F=B+D、G=C+D、H=B+C+D。组合 prompt E-H 全部达到 expression tree score = 1.00。
- **噪声鲁棒性实验（3×5 设计）**：噪声注入 3 种位置（feature only / target only / both）× 5 个等级（1%-5% std），按 Keren et al. 2023 方案实施。
- **表达式树距离算法**：自顶向下递归比较。相同叶子节点距离 0；不同变量距离 1；不同数值距离 α|v1-v2|（α∈[0,1]，上限 1）；操作符不匹配距离 1；交换律操作符(+,*)取最小子树匹配距离；非交换律严格左对左、右对右。

### 核心结果
| 场景 | 最佳配置 | MAE | R² | Tree Score |
|------|---------|-----|-----|-----------|
| Dropping ball | Mistral+PySR | 0.090 | 0.95 | 0.99 |
| Simple Harmonic | Mistral+PySR | 0.060 | 0.97 | 1.00 |
| EM Wave | Mistral+PySR | 0.030 | 0.99 | 1.00 |
| Baseline dropping ball (no LLM) | PySR only | 0.150 | 0.84 | 0.81 |

LLM 集成一致优于 baseline（R² 提升 0.10-0.15，expression tree score 提升 0.10-0.20）。

**Prompt 效果排序**：H≈G≈F≈E > D > C > B > A。变量描述+实验描述（E）的组合就足以达到 tree score=1.00，说明 GT 公式泄露不是必要的。更有信息量的 prompt 不仅提升数值拟合，更直接帮助发现 exact GT 方程。

**噪声鲁棒性**：PySR 表现最优，5% 噪声下 expression tree score 仍有 0.60-0.88（取决于场景复杂度）。加入 LLM 后各 SR 模型在所有噪声水平下均优于 baseline。

**Surprising 结果**：Baseline 中 gplearn 在 dropping ball 场景的 tree score (0.90) 反而高于 PySR (0.81) 和 DEAP (0.75)，但加入 LLM 后 PySR 反超。说明 LLM 引导改变了搜索动力学。

### 明确局限
1. **零样本 LLM**：LLM 未针对 SR 评分任务微调，occasional invalid suggestions 导致需要 prompt revision 和增加训练时间。
2. **无人工验证 LLM 评分质量**：无法区分性能提升来自真正物理合理性还是 LLM 捕捉到的表面特征（如方程长度）。这是一个关键的 confounding factor——LLM 可能只是做了一个更聪明的复杂度惩罚。
3. **仅合成数据**：已知 GT 的三个人工场景，远不能代表真实世界物理系统的复杂性和噪声水平。未在真实实验数据上验证。
4. **物理场景极简单**：3 个场景均为经典本科物理教材内容，LLM 训练语料几乎必然包含这些公式，因此 LLM 的"物理知识"可能是记忆而非推理。
5. **未探索权重 w1/w2/w3 的敏感性**：loss 函数权重直接决定了 LLM 评分的影响力，但没有任何 ablation 分析不同权重配置的影响。
6. **仅遗传算法 SR**：未与基于 RL（如 DSR, FEX, VSR）或稀疏回归（SINDy）的 SR 方法比较，无法判断 LLM 引导在这些不同 search strategy 下的效果。
7. **7B 小模型限制**：仅使用 7B 级模型，Mistral 优于 Llama＞Falcon 的排序可能仅反映模型在科学文本上的训练质量差异，而非方法的一般性。

| R1 Evidence | R2 Independ | R3 Honesty | R4 Reprod | R5 Subst | Tier |
|-------------|-------------|------------|-----------|----------|------|

- **R1 Evidence** (3): 主要结果有完整表格（MAE/MSE/R²/Tree Score），数据生成过程可追溯，但 LLM 评分本身没有与人工专家评分做一致性校验。
- **R2 Independence** (2): LLM 评分和 SR 搜索是同一个优化循环内的组件，没有独立的 external validator 检验 LLM 评分是否真的衡量物理合理性。表达式树距离虽是独立指标但仅用于后验评估。
- **R3 Honesty** (4): 诚实披露了 LLM 评分未人工验证、仅合成数据、零样本限制等问题。Prompt sensitivity 实验显示 GT 公式提供会提升性能但未隐藏这一发现。

- **Tier**：B (15/25)。方法有趣但核心 claim 缺乏独立验证，科学深度有限。

### 跟我们的关系
对 **FEX (RL-based SR)** 课题：

**撞车评估**：不撞车。FEX 用 RL（蒙特卡洛策略梯度 + RNN 表达式生成器）搜索表达式空间，本文用遗传算法 + LLM 辅助损失。两者在搜索策略层面完全正交。但本文的核心 insight——"LLM 可以作为物理知识的外部评估器嵌入损失函数"——可以直接移植到 FEX 中。

**可用的工具/方法**：
1. **LLM-guided loss function 架构**：FEX 的 RL reward 函数可加入类似的 LLM-based physics score term，实现 domain-knowledge-informed RL 搜索。
2. **表达式树距离 metric**：比单纯的 MSE/R² 更能衡量"是否找到了正确的物理方程"，适合作为 FEX 的辅助评估指标。
3. **Prompt 设计方法论**：四段式 prompt (ROLE/METRICS/FEW-SHOT/TASK) 可作为 FEX 整合 LLM 时的 prompt 模板起点。
4. **Prompt sensitivity 实验设计**：A-H 八变体消融方案可复用，用于分析 FEX+LLM 中不同知识注入方式的效果。

**差异化**：FEX 的 RL 搜索天然比遗传算法更适合与 LLM 协作——LLM 的评分可作为 RL reward 的一个连续项，且 RL 的 exploration-exploitation 机制可能与 LLM 引导产生更丰富的交互。这是本文未探索的方向。

**该引不该引**：应当引用。本文是 LLM+SR 领域的最新工作，且提供了可复现的 baseline。但需指出其 LLM 评分缺乏独立验证的局限性，并论证 FEX 的 RL 框架如何更好地利用 LLM 引导。

### 撞车评估
不撞车。本文方法（遗传算法 SR + LLM loss term）与 FEX（RL policy gradient + RNN 表达式生成器）在搜索机制上完全不同。本文的贡献点在"LLM 如何作为损失项"而非"SR 搜索算法"。

### 对 FEX 的可用性
1. **LLM-guided reward shaping**：FEX 可以借鉴本文的三维 LLM 评分（维度一致性/简洁性/物理合理性），作为 RL reward 的额外 shaping term，实现物理先验的自动注入。这比手动设计物理约束更灵活、更可泛化。
2. **表达式树距离**：这是一个独立于数值误差的结构评估指标，FEX 当前缺少类似方法。可直接用作 FEX 实验中的辅助 metric。
3. **Prompt 工程设计**：本文的四段式 prompt + few-shot 示例 + 严格格式约束的设计模式可以直接套用到 FEX 的 LLM 模块中。
4. **关键风险**：本文未能区分 LLM 的"物理推理"和"记忆检索"（三场景公式全在 LLM 训练语料中）。在 FEX 中使用 LLM 引导前必须先做 controlled experiment 验证 LLM 是否真正在做物理推理。

### 差异化思路
本文 LLM 评分是静态的（同一 prompt 反复调用同一 LLM）。FEX 的 RL 框架可以做到动态交互：LLM 可以观察 RL agent 的当前搜索状态（已探索区域、当前最优表达式），进行"搜索策略建议"而非仅"候选方程评分"。这是一个本文完全未触及的方向。
