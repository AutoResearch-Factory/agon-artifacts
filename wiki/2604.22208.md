# Finite Expression Method with TranNet-based Function Learning for High-Dimensional Partial Differential Equations (2604.22208)

- **Year**: 2026.04
- **Authors**: Toan Huynh, Feng Bao, Haizhao Yang, Ahmed Zytoon; affiliations are not explicitly listed in the tex source.
- **Venue**: arXiv
- **Keywords**: finite expression method, FEX, TransNet, high-dimensional PDE, symbolic expression search, neural candidate pool, policy gradient, Monte Carlo error
- **One-liner**: 用 TransNet 神经算子扩展 FEX 求解高维 PDE。

## 精华提炼

### 管线/方法

- 论文目标是扩展 Finite Expression Method (FEX)，让候选算子池不只包含手工解析算子，也包含 TransNet 训练出的浅层神经网络算子。
- 输入是一类高维、定常 PDE，加 Dirichlet 边界条件；实验覆盖 60D Poisson、60D stationary reaction-diffusion、55D semilinear elliptic。
- FEX 把 PDE 解写成一个有限数学表达式，表达式来自固定深度的二叉树。
- 二叉树节点可放 unary 或 binary operator；叶子输入高维变量，根节点输出标量解近似。
- unary 节点携带连续参数，主要是 scaling parameter alpha 和 bias beta。
- 对高维输入，直接作用在输入上的 unary operator 是 componentwise 的，scaling parameter 可以是向量。
- 方法把求解 PDE 转成组合优化：在有限表达式空间 S_k 中最小化 PDE 相关 functional L(u)。
- 本文使用的 functional 是 PDE residual 加 boundary residual：L(u)=||D u(x)-f(u(x),x)||^2 + lambda ||B u(x)-g(x)||^2。
- 三个测试中都取 boundary penalty lambda=100。
- boundary residual 用 1000 个边界点近似。
- interior residual 在前两个 60D case 中用 500 个内部点，在 55D semilinear case 中用 600 个内部点。
- 表达式结构搜索分两层：先搜索 operator sequence e，再优化连续参数 theta。
- 对每个 operator sequence，score 定义为 S(e)=(1+L(e))^{-1}，把 loss 转成 [0,1] reward。
- 精确求 L(e)=min_theta L(u(T,e,theta)) 太贵，所以用短程优化估分。
- score 估计先用 Adam 做 T1 步，再用 BFGS 做 T2 步。
- 实验中 T1=2，Adam learning rate=0.001；T2 最多 20，BFGS learning rate=1。
- controller chi_phi 对二叉树每个节点输出 operator 分布。
- operator sequence e 由每个节点从 controller 的 categorical distribution 采样得到。
- 搜索时使用 epsilon-greedy 以保留随机探索。
- controller update 使用 policy gradient，目标是最大化 sampled expression 的 expected score。
- 具体还采用 Petersen et al. 风格的 risk-seeking/quantile objective，只奖励超过 score quantile 的样本。
- 搜索阶段维护 top-K candidate pool，避免由于随机 theta 初始化导致好结构被一次低分淘汰。
- 搜索后对 candidate pool 中的每个 expression 再做 fine-tuning。
- fine-tuning 使用 Adam，初始 learning rate=0.01，T3=15000。
- 最终返回 fine-tuned loss 最小的候选表达式。
- 本文把 TransNet 用作候选算子生成器，而不是直接用 TransNet 解高维 PDE。
- 神经候选函数形式是 single-hidden-layer fully connected network: sum_m alpha_m sigma(w_m^T y + b_m)+alpha_0。
- TransNet 的关键思想是先构造 transferable neural feature space P_NN，再用 least-squares 求 output coefficient alpha。
- 隐层 neuron 先重参数化成 location parameters (a_m,r_m) 和 shape parameter gamma_m。
- location sampling 使用 X_m/sqrt-like normalization 得到 a_m=X_m/||X_m||_2，并取 r_m=U_m。
- 文中写 U_m 是 [0,1] 上的 uniform random variable。
- shape parameter 简化为所有 neuron 共享同一个 gamma。
- gamma 通过 Gaussian random field auxiliary functions 调参。
- gamma tuning 是一维 grid search：对每个 gamma，拟合 K 个 GRF realization，取 average least-squares MSE 最小者。
- 训练好的 TransNet approximation 记为 TN_f，例如 TN_x^2、TN_exp、TN_sin(x)。
- FEX candidate pool 把 0、1、Id 与多个 TN_f 放在一起，供表达式搜索选择。
- 因此本文的 novelty claim 不是新 TransNet，也不是新 FEX 主循环，而是把 TransNet-trained neural candidates 接入 FEX operator pool，并测试高维 PDE 表达式恢复。

### 关键技术细节

- 固定二叉树 depth=2 用于前两个 60D 实验。
- 60D Poisson 的 domain 是 (-1,1)^60。
- 60D Poisson 的 manufactured true solution 是 u(x)=0.5 sum_i x_i^2。
- Poisson pool P1 包含 {0,1,Id,TN_x^2,TN_x^3,TN_x^4,TN_exp,TN_sin(x),TN_cos(x)}。
- Poisson pool P2 移除 TN_x^2，并用 TN_sin(x^2) 替换，测试缺少真实二次结构时的 fallback。
- Poisson P1 搜索 700 iterations；P2 搜索 1200 iterations。
- 60D reaction-diffusion 的 domain 是 (0,1)^60。
- reaction-diffusion 方程是 -nu Delta u + mu u = f(x)。
- reaction-diffusion manufactured true solution 是 u(x)=sum_i x_i^3。
- reaction-diffusion P1 与 Poisson P1 类似，保留 TN_x^3。
- reaction-diffusion P2 移除 TN_x^3，并用 TN_{x sin(x)} 替换；作者明确指出 x sin(x) 与 x^3 的 local behavior 不像。
- reaction-diffusion 使用与 Poisson 相同的 search iteration 设置：含真算子 700，不含真算子 1200。
- 55D semilinear elliptic 方程是 -nu Delta u + mu u + u^2 = f(x)，domain 是 (-1,1)^55。
- 55D true solution 是 exp((1/d) sum_j cos(x_j))。
- semilinear pool 包含 {0,1,Id,TN_x^2,TN_x^3,TN_x^4,TN_exp,TN_sin(x),TN_cos(x)}。
- semilinear case 的难点是 true solution 不是简单 componentwise additive nonlinearities，而是 exp 与 average cos 的组合。
- 二维可视化 slice 的分辨率：Poisson 和 reaction-diffusion 为 200x200，semilinear 为 300x300。
- relative L2 error 通过 Monte Carlo 积分估计。
- Table of L2 errors 使用 2000 个 interior points，并重复 50 次，报告 mean 和 standard deviation。
- 论文没有给出 batch size N、candidate pool size K、epsilon、quantile nu、random seed、hidden neuron number M、GRF realization count K、gamma grid range等完整复现实验参数。
- tex 中没有看到公开代码、数据仓库、seed 表或硬件运行时间。

### 核心结果

- 60D Poisson，candidate pool 含 TN_x^2 时，FEX 找到涉及二次候选和 sin 候选的表达式，relative L2 mean = 4.17e-07，std = 1.51e-08。
- 60D Poisson，移除 TN_x^2 后，FEX 用 TN_x^4 与 TN_sin(x^2) 的组合近似二次结构，relative L2 mean = 3.13e-03，std = 1.24e-04。
- 这个结果说明 true operator 在 pool 中时几乎机器精度恢复；缺失时仍能拟合形状，但精度损失约四个数量级。
- 60D reaction-diffusion，candidate pool 含 TN_x^3 时，relative L2 mean = 7.76e-07，std = 2.20e-08。
- 60D reaction-diffusion，移除 TN_x^3、替换为 TN_{x sin(x)} 后，relative L2 mean = 5.17e-03，std = 1.73e-04。
- reaction-diffusion 的替代函数 local behavior 与 x^3 不像，所以误差比 Poisson 的替代场景更大，这个实验对 candidate relevance 很敏感。
- 55D semilinear elliptic 中，FEX 未恢复 exact expression exp((1/d)sum cos x_j)。
- semilinear case 产生了涉及 exp(-(.)^2) 的近似表达式，形状接近 exp(cos(.))，但 magnitude 存在差异。
- semilinear relative L2 mean = 9.33e-04，std = 3.14e-05。
- 论文核心正结果是：TransNet-trained candidates 可以作为 FEX pool 元素使用，并在高维 manufactured PDE 上得到小误差。
- 论文核心负结果同样重要：FEX 的表现强依赖 candidate pool 是否含有接近真实解结构的 operator。
- 一个有价值的 surprise 是：即便缺少 TN_x^2 或 TN_x^3，搜索仍可用其他 neural candidates 合成可用近似，但精度明显下降。
- 另一个直接信号是：composite expression 比 additive polynomial expression 更难，semilinear case 没有恢复真实解析结构。

### 明确局限

- 实验全部是 manufactured solution setting，f 和 boundary condition 由真解反推，不能直接证明对真实科学 PDE 的泛化。
- 没有和 PINN、Deep Ritz、DGM、FNO、DeepONet 或原始 FEX 的统一运行时间/精度 benchmark 做系统对比。
- 没有真机实验，因为这是数值 PDE solver 论文；更准确地说，没有外部物理数据或真实观测校验。
- 没有展示代码与完整超参数，第三方复现成本较高。
- 没有报告 wall-clock time、memory footprint、GPU/CPU 设置，尽管引言强调高维 PDE 成本。
- 没有系统 ablation TransNet candidate 的 hidden width M、activation、gamma、GRF tuning、candidate pool size。
- controller 的 epsilon、batch size、quantile nu、top-K size 对搜索质量很关键，但 tex 中未明确列出。
- 随机性只在 Monte Carlo L2 evaluation 上给 50 次 mean/std；搜索训练本身没有 multi-seed stability。
- 三个实验的 true solution 都偏低复杂度，并且候选池显式包含或接近所需基础非线性。
- 对 55D semilinear，作者承认没有恢复 exact true expression，只得到形状接近的 approximation。
- 论文没有证明 TransNet-based pool 相比 hand-crafted analytic pool 在表达能力或搜索效率上严格提升。
- novelty 更像 preliminary numerical integration，而不是一个完整的新理论框架。
- 对 `0616-fex` 课题而言，最需要警惕的撞车点是“用神经网络训练候选算子扩展 FEX operator pool”已经被这篇直接覆盖。
