## 精华提炼

### 管线/方法
- 这不是一篇新算法论文，而是一篇 Review，核心贡献是给数据驱动微分方程发现建立两个组织框架。
- 第一条主线是“equation discoverability phase diagram”：用两个轴描述一个方程发现问题的难度。
- 横轴是 structural complexity：从 closed-form library、expandable library 到 open-form symbolic/generative representation。
- 纵轴是 coefficient complexity：从 constant coefficients，到可由显式函数表示的空间/时间/参数依赖系数，再到随机场或高维 irregular field 这类不可显式表达的系数。
- 作者把现有方法放进这个二维相图，强调低左角问题通常可由 SINDy/PDE-FIND 一类稀疏库方法处理，右上角仍是未充分探索的前沿。
- 论文把一般微分方程发现写成隐式算子形式：F(z, u, grad u, grad^2 u, ...)=0。
- 实际算法常用演化形式：u_t = sum_k theta_k Phi_k(u, grad u, grad^2 u, ...)，其中 Phi 是结构项，theta 是系数。
- 作者反复提醒：这种“结构项/系数/噪声”的划分更多是算法表述产物，不一定是自然系统的本体分类。
- 第二条主线是 REO 框架：Representation、Evaluation、Optimization。
- Representation 部分比较四类表示：structured matrix/library、symbolic tree/graph/string、embedding latent space、LLM sequence representation。
- Structured matrix 对应 SINDy/PDE-FIND/PySINDy 等，把候选项评估成矩阵，再做稀疏选择。
- Symbolic representation 用 expression tree、graph 或 gene-like encoding 表达方程，代表包括 GP、SGA-PDE、DLGA、DISCOVER 等。
- Embedding representation 把方程映射到连续 latent space，适合神经生成和跨系统泛化，但可解释性与局部几何都更难校验。
- Sequence/LLM representation 把方程序列化成 token，通过 prompt、上下文、解码策略引导生成，适合人机交互但需要强验证。
- Optimization 部分覆盖 sparse regression、genetic algorithms、reinforcement learning、gradient-based optimization、prompt-based optimization。
- Sparse regression 适合有限候选库，使用 LASSO/LARS/elastic net/STRidge/group sparsity/MIO 等求稀疏系数。
- GA 通过 selection/crossover/mutation 搜索方程结构，适合离散组合和非光滑搜索空间。
- RL 把方程生成建模成 sequential decision process，用 reward 平衡拟合误差、复杂度和合法性。
- Gradient descent 适合可微 embedding 或神经 surrogate，但单纯拟合误差不保证物理合法、可解释或结构正确。
- Prompt-based optimization 用自然语言、示例和反馈迭代提示 LLM，优点是容易注入人类先验，弱点是控制间接且需要外部验证。
- Evaluation 部分把评价拆成四类：data-fitting accuracy、mathematical conciseness、physical consistency、solvability。
- Data-fitting accuracy 包括 u_t 回归误差、MAE、R^2，以及有真值 benchmark 时的 term-level recall。
- Mathematical conciseness 包括 terms 数量、symbolic length、AIC、BIC。
- Physical consistency 包括维度一致性、守恒律、对称性、已知物理约束和物理可解释假设。
- Solvability 是作者强调的新增评价维度：发现的方程是否真能被数值或解析求解，而不只是拟合数据。
- 应用综述按四个范式组织：噪声/不完全观测、微观到宏观桥接、closure model 发现、新 governing laws 探索。
- 噪声/不完全观测部分覆盖薄层弱湍流、部分观测物理系统、复杂网络、电磁场、地下水、土壤水流、随机微分方程。
- 微观到宏观桥接部分覆盖 active matter、granular flow、chemistry/plasma reduced models、single-cell biology。
- Closure model 部分覆盖 RANS turbulence closure、多相流 closure、wind turbine wake forcing/boundary terms、geophysical eddy/cloud parameterization。
- Novel governing laws 部分覆盖 chromatography relation、viscous gravity currents、terrain-precipitation relation、rogue waves、breaking waves、traffic PDE、E. coli taxis、Green's functions。
- 论文最后把未来方向从“恢复方程”推进到“形成科学概念”：修正既有概念、把科学直觉数学化、闭环发现新概念。

### 关键技术细节
- 主文 LaTeX 源码共 1009 行，只有一个 `main.tex`，没有额外 section `.tex` 文件。
- 综述引用规模为 Semantic Scholar 返回的 153 篇参考文献。
- 作者列出 ODE benchmark：Gennemark-Wedelin (>40 systems, 2009), BioPreDyn-bench (6 systems, 2015), AI Feynman (100 formulas, 2020), dysts (131 systems, 2021), Strogatz dataset (252 systems, 2021), ODEBench (63 ODEs, 2024), LLM-SRBench (239 total / 68 ODE tasks, 2025), cp3-bench (7 tasks, 2025), MDBench (63 ODE + 14 PDE, 2026).
- 作者列出 canonical PDE benchmark systems：KdV, Burgers, Schrodinger, nonlinear Schrodinger, Kuramoto-Sivashinsky, wave, sine-Gordon, Fisher, FitzHugh-Nagumo, Klein-Gordon, convection-diffusion, Keller-Segel, Chafee-Infante, PDE Compound, PDE Divide, heat, Fisher-KPP, Fujita-Storm, 2D Cahn-Hilliard, 2D reaction-diffusion, 2D Navier-Stokes, 2D Burgers, 2D advection-diffusion, 2D wave。
- 软件表覆盖 PySINDy, DataDrivenDiffEq.jl, DeePyMoD/DeePyMoD_torch, EDL, PySR, ODEformer, SINDy-PI, SymINDy, IRK-SINDy, symmetry-informed ODE discovery, symbolic-regression-ode-systems, SymDer, PDE-FIND, WSINDy_PDE, WSINDy_PDE_OL, EPDE, DISCOVER, SGA-PDE, EquationDisco, EqGPT。
- Box 1 对 PDE-FIND 做方法拆解：表示为 U_t = Theta(U,Q) theta，核心假设 theta 稀疏。
- PDE-FIND 的原始验证被描述为 6 个 benchmark PDE，数值解用 spectral differentiation 和 Runge-Kutta-45，噪声示例为 1% solution standard deviation white noise。
- PDE-FIND 用 local polynomial interpolation 估计导数，STRidge 做 ridge regression + thresholding + refit。
- Box 2 对 DSR/DISCOVER 拆解：DSR 可发现 ODE RHS f(u)，不能处理 RHS 导数；DISCOVER 扩展到 PDE expression tree。
- DISCOVER/DSR 表示为 preorder traversal sequence，内部节点是 +, -, *, / 或 differential operators，叶子是变量、坐标、常数。
- DSR reward: R = 1/(1+NRMSE)，DISCOVER reward: R = (1 - zeta_1 k - zeta_2 d)/(1+RMSE)，其中 k 是函数项数、d 是树深度。
- DISCOVER 的示例噪声设置：KdV/Burgers/Chafee-Infante 为 10%，PDE_divide/PDE_compound 为 1%。
- Box 3 对 ODEFormer 拆解：目标 u_t = f(u)，encoder-decoder 对齐 trajectory embedding 和 symbolic function embedding。
- ODEFormer 数值 tokenization 词表为 10,203 tokens，表达式用 prefix notation。
- ODEFormer 训练数据为 50M synthetic ODEs，模型为 86M parameter transformer，inference 用 beam sampling size 50，并可做 post-hoc constant optimization。
- 作者强调 LLM/生成模型可能带来 reusable priors，但必须区分“统计上像方程”和“物理上有意义的方程”。
- 变量系数部分清楚区分 grouped sparse regression、low-rank grouped sparse regression、Stepwise-DLGA、SEQL/HEQL、GraphED。
- 随机场/不可显式系数部分提到 Karhunen-Loeve expansion 与 HIN-PDE 的 spatial kernel sparse regression。
- 论文没有报告新的实验数据、训练曲线、消融表或代码；主要证据来自文献组织、分类表、方法盒子和应用案例。

### 核心结果
- 核心发现 1：微分方程发现的难度不能只按算法族分类，应同时看方程结构复杂度和系数复杂度。
- 核心发现 2：closed-library 方法在低复杂度、强先验、小候选库问题中最稳，但遇到非库内项、嵌套函数、fractional/rational/composite forms 会失效。
- 核心发现 3：open-form symbolic/generative methods 提升可表达性，但把主要瓶颈转移到组合搜索、可辨识性和评价可靠性。
- 核心发现 4：系数复杂度经常被低估；现实系统中的空间/时间变化系数、随机场、closure term 可能比结构项本身更难发现。
- 核心发现 5：evaluation 目前明显落后于 representation/optimization，尤其缺标准 PDE discovery benchmark、统一 noise/sparsity protocol、non-public test set 和 solvability 指标。
- 核心发现 6：真实科学价值不是“拟合出一个方程”，而是发现能修正概念、解释机制、被求解和复用的数学对象。
- 核心发现 7：作者把真实应用分成四类，说明 equation discovery 已从 synthetic rediscovery 走向环境、流体、生命、材料、交通等真实系统。
- 核心发现 8：作者对未来的判断是 closed-loop discovery 和 human-AI collaboration：候选方程提出、主动选择新实验、验证/修正、形成可解释概念。

### 明确局限
- 这是一篇 Review，不是 benchmark 论文；没有统一跑各方法，也没有新的定量比较。
- Phase diagram 中方法点位是概念相对位置，不是性能排名，不能直接推出哪个算法最好。
- 对 PDE discovery benchmark 的缺口说得很清楚：当前仍缺被广泛采用、可比较、包含真实复杂系统的标准 benchmark。
- 论文覆盖面大，但每个具体算法的实现细节、失败案例和超参数敏感性只做综述级摘要。
- 对 LLM/生成模型的讨论较积极，但实验可靠性主要依赖被引用论文；本文没有独立复核 hallucination、data leakage 或 benchmark memorization。
- 作者团队自身也参与了多个被讨论方法和应用方向，例如 EqGPT、GraphED、PDE discovery 系列工作；综述观点需要按引用证据而不是权威背书使用。
- 对“solvability”提出了概念，但没有给出可执行的标准测试协议。
- 对真实世界应用的案例很丰富，但不是系统 meta-analysis；不同领域案例的数据规模、公开程度和验证强度差异很大。
