## 精华提炼

### 管线/方法

1. 输入是高维参数点集合 `P={x_k}_{k=1}^m` 和函数值或动力学导数 `y`。
2. 论文假设定义域是张量积区间 `D=⊗[a_i,b_i]`，但数据点本身不要求落在完整网格上。
3. 每个坐标轴先选择一维基函数向量 `psi(x)=[psi_1(x),...,psi_p(x)]`。
4. 多维特征映射写成可分变量张量积 `varphi(x)=⊗_{i=1}^n psi(x_i)`。
5. 朴素矩阵展开会得到 `p^n` 个特征列，系数张量 `C` 也是 `p^n` 规模，直接构造不可行。
6. 作者提出 row-wise tensor product，也称 m-tensor product。
7. 对每个输入维度 `i`，先构造 core 矩阵 `Psi_i in R^{m x p_i}`，第 `k` 行是第 `k` 个样本在第 `i` 个坐标上的一维基函数值。
8. m-tensor 回归算子定义为 `Phi = Lotimes_i Psi_i`，元素满足 `Phi[k,i_1,...,i_n]=prod_j Psi_j[k,i_j]`。
9. `Phi` 的 mode-1 展开等于 face-splitting product / transposed Khatri-Rao product，但论文保留高阶张量结构，避免索引映射和显式展开。
10. 回归方程写成 `Phi ltimes C = y`，其中 `ltimes` 是 n-way contracted product。
11. 在线预测不显式存储 `C`，而是使用 Gram/kernel 型矩阵 `P=Phi ltimes Phi^T in R^{m x m}`。
12. 最小二乘解采用行满秩伪逆形式：`C_hat=Phi^T(P^{-1}y)`。
13. 实际预测为 `f_hat(x)=<varphi(x) ltimes Phi^T, z>`，其中 `z=P^{-1}y`。
14. `P` 是对称正定矩阵，作者建议用 Cholesky 分解替代显式求逆。
15. m-product 通过 Hadamard product of core Gram matrices 计算：`Phi_1 ltimes Phi_2^T = ⊙_i Psi_i^1 (Psi_i^2)^T`。
16. 这个公式是整篇的核心 computational trick，把本来 `O(m^2 p^n)` 的张量内积降为 `O(m^2 p n)`。
17. 单点推理复杂度是 `O(m p n)`，仍依赖样本数 `m`。
18. 为降低 `m` 维成本，论文提出 ALI regularization，从全部 `m` 行中选出 `tilde_m` 个 Almost Linearly Independent rows。
19. ALI 的目标是用子集张量 `tilde Phi` 近似全部行，使 `||Phi - W tilde Phi||^2 <= epsilon`。
20. 子集选择有两个版本：greedy streaming 版本和 optimal selection 版本。
21. greedy ALI 从第一行初始化，逐行计算新样本到已选行 span 的距离 `delta_k`。
22. 若 `delta_k > epsilon` 就保留该行并增量更新 Cholesky factor；否则只更新权重。
23. greedy ALI 不显式求逆 `tilde P^{-1}`，而是用三角回代计算距离和投影权重。
24. optimal ALI 每轮在剩余行中选能最小化整体距离和的行，代价更高但更接近全局选择。
25. 回归阶段只用 `tilde Phi` 和 `tilde y` 求解，推理成本降为 `O(tilde_m p n)`。
26. 论文把 ALI 与 kernel recursive least squares 的 ALD criterion 联系起来，并继承平均投影误差由 `epsilon` 控制的解释。
27. 论文还给出 Tikhonov regularization：`z_2=(Phi ltimes Phi^T + lambda^2 I)^{-1}y`。
28. spectral truncation 则通过 `P` 的 SVD 选前 `r` 个 left singular vectors，投影到低秩子空间。
29. kernel interpretation 是：`k(x,x')=<varphi(x),varphi(x')>`，`P` 就是 kernel matrix。
30. 与普通 kernel method 的区别是，本文先显式定义 tensor-product feature map，再从其内积导出 kernel。
31. 作者强调这样不受少数标准 kernel 形式限制，可根据一维基函数自由构造核。
32. 方法实质上是“可分变量 feature map + kernel trick + tensor algebra bookkeeping + 行子集 regularization”。

### 关键技术细节

1. 基础数学对象是 partial tensor product，不是 TT/CP/Tucker 分解。
2. m-tensor 的 `r-dim` 是样本行维，`c-dim` 是多维特征列维。
3. r-dim contraction 对每个 rank-1 row tensor 只需 `O(np)`，而 c-dim contraction 会退化到 `O(mp^n)`，因此算法尽量只用 r-dim contraction。
4. m-product 的复杂度是 `O(m^2pn)`，构造 Gram 矩阵时主导离线成本。
5. Tikhonov、spectral truncation 和 ALI 三种 regularization 都只操作 `m x m` 或 `tilde_m x tilde_m` 矩阵。
6. 最小二乘和 Tikhonov 需要求解 `m x m` 系统；若 `m=alpha n`，离线复杂度表中写为 `O(alpha^3 n^3)`。
7. ALI construction 给出的上界是 `O(alpha^2 n^3)`，在线推理是 `O(tilde_m n)` 或一般写法 `O(tilde_m p n)`。
8. Rosenbrock 实验使用 `n in {20,50,100,200,300}`。
9. Rosenbrock 的样本数设为 `m=alpha n`，`alpha in {5,10,20,30,40}`。
10. Rosenbrock 的一维 polynomial basis 到 4 次：`[1,x,x^2,x^3,x^4]`，所以 `p=5`。
11. 训练点用 Latin Hypercube Sampling，每个设置重复 3 次。
12. Rosenbrock 测试用 `3m` 个 unseen LHS points，报告 relative `L2` error。
13. Lorenz 实验用标准参数 `sigma=10, rho=28, beta=2.667`。
14. Lorenz 初值是 `[0,1,1.05]^T`，时间步长 `Delta t=0.001`。
15. Lorenz 训练只用 500 steps，然后 rollout 25000 steps。
16. Lorenz basis 是 `[1,x]`，spectral truncation rank 和 ALI retained rows 都设为 8。
17. Lorenz 还在 50 个随机 unseen initial conditions 上测试 ALI regression 的轨迹误差。
18. Kuramoto 实验维度是 `n in {3,10,100}`，每个维度跑 5 次。
19. Kuramoto 自然频率 `omega_i` 从 `[-5,5]` 随机采样，coupling `K=2`。
20. Kuramoto 使用两组 basis：`[1,sin(x)]` 和 `[1,cos(x)]`。
21. Kuramoto 训练 1000 timesteps，预测 50000 timesteps。
22. Kuramoto 的 ALI 平均保留约 65、360、370 行，分别对应 `n=3,10,100`。
23. 作者指出高维场景会发生数值 overflow，因为 kernel trick 等价于累加 `p^n` 项。
24. 实验中低维 `<10` 不缩放，高维输入用 `1e-7` 缩放因子，但系统化 normalization 尚未解决。
25. 代码和数值例子在文中声明有 GitHub repository，但本文源码没有给出完整复现实验脚本内容。

### 核心结果

| 实验 | 设置 | 主要发现 |
|------|------|----------|
| Rosenbrock | `n=20..300`, `m=alpha n`, degree-4 polynomial basis | 构造和推理时间符合理论 scaling；高维下三种 regularization 约在 2% relative error 量级 |
| Rosenbrock/ALI | 同上 | ALI 的单位推理时间对维度近似线性，且因 `tilde_m << m` 明显快于 full least squares |
| Lorenz | 500 training steps, 25000 rollout, basis `[1,x]` | LS、spectral truncation、ALI 都能从短轨迹识别动力学并长期 rollout；ALI 在 50 个 unseen initial conditions 上展示泛化 |
| Kuramoto | `n=3,10,100`, 1000 seen steps, 50000 rollout | 即使 `n=100` 且训练步数固定，模型仍能维持约 2% error；ALI 更快但通常误差略高 |

1. 最重要的正结果不是某个 SOTA 数字，而是复杂度机制：所有核心内积都通过 core-wise matrix products 和 Hadamard products 实现。
2. 作者声称 m-tensor formalism 在最坏情形下把维度相关复杂度压到 cubic scaling，并且 memory use 对维度线性。
3. Rosenbrock 实验显示线性采样 `m=alpha n` 下仍能得到可用误差，这是对“稀缺数据高维 regression”的主要经验支持。
4. Lorenz 和 Kuramoto 说明该方法不只是静态函数 surrogate，也能用于 data-driven operator/dynamics identification。
5. 反直觉点：Rosenbrock 中低维更难调 hyperparameter，误差反而在较高维更稳定接近 2%。
6. 另一个反直觉点：Kuramoto 维度从 10 到 100 时 ALI retained rows 平均只从 360 到 370，说明 ALI dictionary size 不一定随维度线性膨胀。
7. spectral truncation 的在线成本与 least squares 一样，主要额外代价在 SVD 和 inversion。
8. ALI 是唯一同时减少系统规模和在线推理样本维度的方法。

### 明确局限

1. 作者自己承认 numerical overflow 是实际实现中的关键问题。
2. 高维缩放因子 `1e-7` 是经验处理，尚无自动 normalization 规则。
3. 一维 basis `psi` 的选择仍是人工指定，方法成功依赖 basis 与目标函数结构匹配。
4. 本文没有给出系统的 basis-selection 或 library-learning 机制。
5. 实验主要是合成 benchmark：Rosenbrock、Lorenz、Kuramoto；没有真实工程高维 scarce-data 数据集。
6. 论文没有和 Gaussian process、random feature、SINDy、MANDy、kFSA、TT regression、FEX/PDE solver 做统一 runtime/accuracy 表格。
7. 许多结果在 figure 中呈现，正文缺少可直接读取的数值表。
8. GitHub repository 被声明存在，但源码审读没有验证代码质量、依赖、seed、完整复现实验命令。
9. Lorenz/Kuramoto 都是在已知方程生成的数据上做识别，不是未知真实系统。
10. Lorenz 例子是过定问题，不能证明极端稀缺数据下复杂动力系统一定稳定。
11. Kuramoto 训练和测试沿用同一初始条件进行长 rollout，跨初值泛化只在 Lorenz 中展示。
12. ALI 的 epsilon 选择没有系统 calibration；不同问题的 tolerance 如何选仍不清楚。
13. 论文没有做噪声鲁棒性实验。
14. 方法输出是数值 surrogate / kernel predictor，不输出简洁符号表达式。
15. 因此它不替代 FEX 的“可解释有限表达式”目标。

差异化很清楚：如果我们的目标是 cross-dimension FEX lift/audit，应强调 symbolic skeleton、有限可交换宏 grammar、searchability vs liftability、top-k PDE probe 和 parameter-only high-d optimization；本文只证明 separable basis regression 可以高效做数值预测，不会告诉我们 FEX controller 是否能搜到正确结构。建议引用，但放在“tensor/kernel high-dimensional regression and scarce-data surrogate”相关工作中，不应把它列为直接 FEX 竞品。
