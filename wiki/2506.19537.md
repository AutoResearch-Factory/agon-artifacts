# Dimension Reduction for Symbolic Regression (2506.19537)

- **Year**: 2025 (AAAI 2025 submission)
- **Authors**: Paul Kahlmeyer, Markus Fischer, Joachim Giesen (Friedrich Schiller University Jena)
- **Venue**: AAAI 2025
- **Keywords**: symbolic regression, dimension reduction, variable substitution, functional dependence, beam search, expression DAG, Chatterjee coefficient, Codec, KMAc, out-input substitution, input substitution
- **One-liner**: 用功能依赖度量来搜索有效的变量替换,迭代降低符号回归问题的维度,将恢复率提升 10-30 个绝对百分点。

## 精华提炼

### 管线/方法

整体管线是一个 beam search 过程,在每个节点执行以下步骤:

1. **输入**: 给定 n 个观测数据点 (x^(i), y^(i))，x ∈ R^d，y ∈ R。
2. **候选替换生成**: 使用 UDFS 框架枚举小规模的 expression DAG（限制为最多 1 个中间节点 + 1 个输出节点,即标量函数 k=1）。每个 DAG 代表一个候选替换函数 g 或 h。
3. **有效性检验**: 对每个候选替换,将观测数据转换为低维数据,然后用功能依赖度量(Codec/KMAc/Volume heuristic)计算得分(0 到 1)。得分接近 1 表示替换后仍保持功能依赖关系,即替换有效。
4. **Beam search 决策**: 在搜索树的每一层,保留 beam-size 个得分最高的节点,广度优先扩展。搜索结束后,将得分最高的路径上的降维问题传给任意符号回归算法。
5. **符号回归求解**: 对降维后的回归问题,用任意符号回归器求解。对于 input substitution,直接得到 f_g；对于 out-input substitution,解得 g 后通过 SymPy 解方程 h(x_I, y) = g(x_\I) 得到 f_g。
6. **重构**: 沿路径从根节点反向翻译所有变量,组装出原始空间的完整符号表达式 f(x)。

两种替换类型的形式化定义:
- **Input substitution g**: 存在 f_g 使得 f(x) = f_g(g(x_I), x_\I)，其中 |I|>1, k<|I|。
- **Out-input substitution h**: 存在 g 和 f_g 使得 h(x_I, f(x)) = g(x_\I) 且 f(x) = f_g(g(x_\I), x_I)。概念上是用简单的 h 来间接找到复杂的 input substitution g。

### 关键技术细节

- **Expression DAG 枚举**: 继承自 Kahlmeyer et al. 2024 (UDFS)。DAG 相比 expression tree 可以共享公共子表达式,用更少节点表示更多函数。候选空间限制为 ≤1 个中间节点 + 1 个输出节点。
- **功能依赖度量**:
  - Chatterjee 系数 (Chatterjee 2020): 单变量 rank-based, ξ_n ∈ [0,1]。利用排序后相邻点的秩差之和。
  - Codec (Azadkia & Chatterjee 2021): 多变量推广。利用最近邻点的秩差: Codec_n = (Σ n·min{r_i, r_ν(i)} - l_i^2) / (Σ l_i(n-l_i))。
  - KMAc (Deb et al. 2020): 最通用的核方法框架,用几何图(k-NN 图或欧氏最小生成树)编码样本点"邻近"关系,用 RKHS 核度量 Y 值距离。本文使用 1-NN 图 + 高斯 RBF 核。
- **运行时间**: Codec ~3ms/替换, KMAc ~24ms/替换, Volume heuristic ~21ms/替换。Codec 比 KMAc 快近一个数量级,性能相当。
- **Beam size**: 实验评估 1-5,发现 beam size=1 已足够,增加 beam size 仅边际改善。
- **硬件**: Intel Xeon Gold 6226R 64核, 128GB RAM, Python 3.10。
- **CAS 工具**: SymPy 用于解方程、验证替换有效性、检查恢复等价性。
- **评测指标**:
  - Recovery rate: 符号等价恢复的比例(通过 SymPy 检查 f - f̂ 化简为常数或 f̂/f 化简为常数)
  - Jaccard index: 子表达式集合的交并比,作为 recovery 的软版本
  - NRMSE: 归一化均方根误差(测试集 20%)
  - Complexity: SymPy 简化后 expression tree 的节点数

### 核心结果

- **降维率**: 在 Feynman 数据集上平均降维 ~50%,Wikipedia 数据集上 ~35%。有噪声时仅 Codec 和 KMAc 仍有效,volume baseline 崩溃。
- **恢复率提升** (Feynman, 含 beam search vs 不含):
  | 算法 | 基础恢复率 | +Beam 恢复率 | 提升 |
  |------|-----------|-------------|------|
  | UDFS | 0.58 | 0.69 | +11pp |
  | DSR (RL) | 0.37 | 0.66 | +29pp |
  | Transformer | 0.01 | 0.11 | +10pp |
  | PySR (GP) | 0.51 | 0.62 | +11pp |
  | Operon (GP) | 0.41 | 0.52 | +11pp |
  | Polynomial | 0.06 | 0.34 | +28pp |

- **替换类型消融**: out+input > input only >> AIFeynman(仅4种二元运算替换)。Wikipedia 数据集上: 0.34 vs 0.19 vs 0.08,Feynman 上: 0.49 vs 0.39 vs 0.39。
- **Model fit (NRMSE)**: 普遍大幅改善(如 DSR 从 0.14 降到 0.03)。
- **未恢复问题也有改善**: 即使问题未被完全恢复,Jaccard index 也提高,说明找到的表达式包含了更多真实子表达式。
- **Surprising 发现**: 连最基本的二阶多项式回归也从 6% 恢复率提升到 34%,说明降维后即使简单模型也能大幅受益。Transformer 基础恢复率仅 1%,显得非常弱,但 beam search 后仍达 11%。

### 明确局限

- **搜索空间非常小**: DAG 枚举仅限 1 中间节点 + 1 输出节点,意味着只能找到非常简单的替换函数。更复杂的替换需要通过 out-input substitution 间接发现,但这也有限制。
- **依赖功能依赖度量的质量**: 在小样本或高噪声情况下,Codec/KMAc 可能给出误导性高分。论文只在已知 ground truth 的数据集上验证了替换的有效性,在实际未知问题上的可靠性未经验证。
- **仅测试了两个 benchmark 数据集**: Feynman (95 公式) 和 Wikipedia eponymous equations (880 公式)。两者都是来自物理/科学的"自然"公式。没有在合成表达式、工程应用或其他领域评估。
- **无真实科学发现案例**: 所有实验都是在已知公式上做 recovery,没有展示在真实未知科学数据上发现新公式的能力。
- **计算开销**: 每个节点需要对所有候选替换计算功能依赖得分,枚举 DAG 空间虽然小但仍需 O(所有可能 DAG) 时间。
- **没有与端到端深度学习方法比较**: 只比了传统符号回归算法+Transformer,没有与最近的 LLM-based 符号回归方法比较。
- **Recovery 判据严格**: 用 SymPy 符号等价判定恢复,可能低估实际效果(表达式语义等价但符号形式不等价的情况)。
