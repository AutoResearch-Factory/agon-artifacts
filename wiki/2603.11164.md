# Learning to Unscramble: Simplifying Symbolic Expressions via Self-Supervised Oracle Trajectories (2603.11164)

- **Year**: 2026.03
- **Authors**: David Shih, NHETC, Dept. of Physics and Astronomy, Rutgers University
- **Venue**: arXiv (单作者)
- **Keywords**: symbolic simplification, self-supervised learning, oracle trajectories, transformer, permutation equivariance, multi-label loss, dilogarithm, scattering amplitudes, spinor-helicity, MDP
- **One-liner**: 通过"正向 scramble + 反向记录 oracle trajectory"自监督范式，训练置换等变 Transformer 策略网络做 step-by-step 符号化简，在多对数恒等式和散射振幅两个问题中达到 ~99.9% 求解率。

## 精华提炼

### 管线/方法

1. **数据生成（Oracle Trajectory Generation）**：
   - 构造简单 target expression（来自 goal set G）
   - 正向 scramble：随机应用数学恒等式 n_scr 次，使表达式逐渐复杂化
   - 反向得到 oracle trajectory：[complex → simple] 的 state 序列
   - 对每对相邻状态 (s_i, s_{i-1})，暴力搜索 s_i 上哪些 action 能产生 s_{i-1}，记录为 oracle action(s)
   - 核心洞察：complexification 是容易的（总有可用的正向恒等式），simplification 是难的（组合爆炸），但反转 scramble 序列可免费获得 expert demonstration

2. **MDP 形式化**：
   - State: 当前数学表达式
   - Action: 对表达式特定部分应用特定恒等式
   - Transition: 确定性（恒等式应用结果唯一）
   - Goal: 达到满足"简单性"标准的表达式（非单一 canonical form，存在多个等价目标态）

3. **训练**：
   - Transformer encoder（无 positional encoding，embedding dim=64, 4 heads, 3 layers, FF dim=128, ~165K 参数）
   - Multi-label soft loss：当 k 个等价的 oracle action 都能导到同一结果时，每个分配 1/k 目标概率
   - AdamW (lr=1e-4, linear warmup 5 epochs, weight decay 0.01, grad clip max norm 1.0)
   - 200-400 epochs

4. **推理**：
   - 贪心选择 + 反循环检测（已访问的 state-action 对屏蔽）
   - 回溯（checkpoint 局部最优复杂度状态，失败后退回到最佳 checkpoint 尝试次优 action，最多 10 次）
   - Reject Term Increase (RTI)：屏蔽会导致 term 数增加超过阈值 N 的动作
   - 对超大表达式：先用 contrastive grouping 分解子问题，再套 beam search

### 关键技术细节

- **多标签损失的影响**：对振幅问题（Schouten 恒等式平均产生 ~6.2 个等价逆 action），从单标签切换到多标签，4-point solve rate 从 73% 跳到 98%
- **架构选择**：无 positional encoding（因为加法可交换，项是无序集合）；per-term equivariant policy head（dilogarithm）或 global CLS-based policy head（amplitude，因为 bracket 跨 term）
- **Dilogarithm 设置**：特征向量 17 维（系数 log/sign、分子分母多项式系数度数等），最多 15 项，45 个离散 action (3 identities × 15 terms)
- **Amplitude 设置**：特征为每 term 的 bracket 编码（type、power、动量 one-hot），4pt 83维/term，10 项上限；action space 4pt=1,452 / 5pt=4,180 / 6pt=29,760
- **Action masking**：非目标动作（padding term、无效操作）设 logit 为 -∞
- **Beam search for YM 5pt**：beam size K=20，用 contrastive encoder embedding 按 cosine similarity 分组（最多 25 terms/组），多相似度阈值（0.6→0.5→0.4），MD5 哈希去重，patience=20 步无改善则终止

### 核心结果

| 问题 | 本文模型 | 前人最佳 | 失败率降低倍数 |
|------|----------|----------|---------------|
| Dilog (target-rel) | 99.9% (4,731/4,737) | 92% (DSZ) | ~80× |
| 4pt Amplitude (target-rel) | 99.9% (9,858/9,869) | 98.2% (CDS) | ~16× |
| 5pt Amplitude (target-rel) | 99.6% (10,220/10,256) | 96.0% (CDS) | ~11× |
| 6pt Amplitude (target-rel) | 99.4% (9,871/9,934) | 96.9% (CDS) | ~5× |
| YM 5pt Gluon (full simplify) | **100%** (103/103) | <50% for >100 terms (CDS) | — |

- **泛化能力**：dilog 模型训练最多 7 scramble，在 10 scramble 测试集上无性能下降
- **步数效率**：模型找到的化简路径平均比 scramble 步数短 10-20%（ratio 0.80-0.90），说明学会了 bypass scrambling 中的冗余
- **Beam search** 使得在 228-term 的 Feynman diagram 级表达式上从训练分布（≤25 terms）泛化到完全化简
- 两个模型的 failure sets 几乎不重叠（>96% 互相覆盖），暗示未来可做 hybrid

### 明确局限

1. **无真机/无实验验证**：纯计算问题，不存在物理实验对照。但数学等价性可以通过数值代入验证（文中做了 numerical validation）。
2. **只涉及两类问题**：dilogarithm 和 spinor-helicity amplitude，尚未推广到更一般的符号化简场景
3. **依赖已知目标形式**：oracle trajectory 生成需要已知简单表达式的构造方法（即 goal set G 的结构已知），不适用于"不知道简单形式长什么样"的问题
4. **大表达式依赖 contrastive grouping**：模型自身输入上限 25-30 terms，处理实际物理表达式需要外部聚类+beam search pipeline
5. **单步 action 的计算成本**：amplitude 推理时 identity application 可能很贵，需 per-step 30s timeout 和 per-sample 120-180s timeout
6. **Schouten 等价性导致多标签**：amplitude 问题中平均 6.2 个等价逆 action，增加了训练复杂度，但被 multi-label loss 解决了
7. **所有工作由 Claude Code 完成**：作者明确声明 "Claude did all of the hands on work"，包括写代码、训练、评估、分析、绘图、写论文。这是一篇 human-in-the-loop + AI-agent 论文。
8. **未探索 RL 混合方法**：作者在 discussion 中提到，可以探索当 self-supervision 不够时，用 self-supervised policy 作为 RL warm start

本文与 FEX（Feature Expression foundation model）课题的关系：

**撞车评估**：不撞车。本文是符号化简（已知复杂表达式→更简单等价形式），FEX 是表达式发现（从数据中搜索可解释的符号表达式）。两者的输入输出和问题定义正交，不存在直接竞争。

**可迁移技术**：
1. **置换等变 Transformer 编码**：将表达式表示为无序 term set（无 positional encoding + [CLS] token），这个方案可以直接用于 FEX 的表达式 encoder
2. **Oracle trajectory 自监督范式**：如果 FEX 中能定义"scramble"操作（等价变换但不改变数值），可生成合成训练数据做 pretraining。例如：从简单表达式出发，通过等价变换生成复杂变体，让模型学习"哪些变换不改变数值"
3. **Multi-label soft loss**：FEX 中可能遇到多个等价表达式表示同一函数的情况，可作为训练技巧
4. **Contrastive grouping + beam search**：处理大型表达式（多 term）的分解搜索策略，可能用于 FEX 的表达式生成阶段
5. **Inference 技巧套装**：anti-cycle detection、backtracking、action masking 对任何 step-by-step 搜索都有参考价值

- 这可能是第一篇完全由 Claude Code（AI agent）执行的科研论文，在方法论上对 agent-factory 项目有参考价值
