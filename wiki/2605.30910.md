# PINNs Failure Modes are Overfitting (2605.30910)

- **Year**: 2026.05
- **Authors**: Nigel T. Andersen, Takashi Matsubara (Hokkaido University, RIKEN AIP)
- **Venue**: arXiv preprint (NeurIPS 2026 formatting)
- **Keywords**: PINNs, failure modes, overfitting, double backpropagation, gradient regularization, PDE solvers, collocation points, L-BFGS, spectral bias
- **One-liner**: 揭示 PINN 失败模式的根本原因不是优化困难而是过拟合于配点（collocation points），通过将 double backpropagation 扩展到全部残差项实现 SOTA 性能，仅需 1/23 的配点数量。

## 精华提炼

### 管线/方法

**输入**: 偏微分方程 (PDE) 定义（域微分算子 F、边界条件 B、初始条件 I），空间域 Omega，时间域 [0,T]，配点网格

**步骤**:
1. 用 4 层 MLP（每层 128 神经元，tanh 激活）参数化解 u_theta(x,t)
2. 输入归一化到 [-1,1]
3. 计算三个损失分量：域残差 L_F（在随机/网格配点上求 MSE）、边界条件损失 L_B、初始条件损失 L_I
4. 原始 PINN loss = sum of all terms with lambda=1 权重
5. **Double PINN（本文核心方法）**: 对每个损失项额外施加输入梯度正则化：
   - L_total = L_F + (lambda_r/2) * ||grad_{x,t} F||^2 + sum over boundaries of (L_Bj + (lambda_r/2) * ||grad_t B_j||^2) + sum over IC of (L_Ij + (lambda_r/2) * ||grad_x I_j||^2)
6. 用 L-BFGS 优化器（memory=100，匹配 PyTorch 实现）
7. 收敛判据：每 100 次迭代检查 loss 相对变化 vs sqrt(machine_epsilon)，FP32: 3.453e-4，FP64: 1.490e-8
8. **诊断方法**: 同时监控 train loss 和 test loss（大随机点集上的 loss），二者发散 = 失败模式标志

**替代正则化方案**（同样消除过拟合但性能差一些）: L^2 weight decay, L^1 weight decay (sparsity 过度限制表达力)

**输出**: u_theta(x,t) 在整个时空域的近似解

### 关键技术细节

**模型架构**:
- 4 层隐藏层，每层 128 神经元
- tanh 激活函数
- Glorot normal 初始化
- 输入归一化到 [-1,1]（原始 vanilla PINN 设置）
- 纯 MLP，无 attention、无状态空间模型、无 Fourier features

**训练配置**:
- 优化器: L-BFGS（Optax 实现，memory 从默认 10 提升到 100 以匹配 PyTorch）
- 精度: FP32 用于初步实验（Titan RTX），FP64 用于最终 SOTA 结果（A100 GPU）
- 最大迭代: 10^6 次 L-BFGS 迭代
- 早停条件: 100 次迭代内 loss 相对变化 < sqrt(machine_epsilon)
- 实现: JAX + Flax NNX + Optax + Orbax

**配点策略（极简）**:
- Convection: 400 域点 + 200 边界点 + 200 初始条件点 = 800 总点（对比主流方法 10201-10403）
- Reaction: 256 域点 + 100 + 100 = 456 总点
- Wave: 144 域点 + 100 + 100 + 100 = 444 总点
- Allen-Cahn: 4096 域点 + 100 + 100 + 100 = 4396 总点（需要更多因为陡峭界面）
- Test 集: 10000 随机域点 + 10000 每边界/初始条件点

**Double backpropagation 扩展**:
- 原始 double backpropagation (Drucker & Le Cun, 1992) 仅正则化最终 loss 对输入的梯度
- Double PINN 对每个 PINN loss 项（域、边界、初始条件）分别施加输入梯度惩罚
- 域项: 正则化 ||grad_{x,t} F(u_theta)||^2
- 边界项: 正则化 ||grad_t B_j(u_theta)||^2
- 初始条件: 正则化 ||grad_x I_j(u_theta)||^2
- 理论计算量: 2x（需要额外梯度），但实际因收敛更快 + 更少点而更省

**四个 benchmark PDEs**:
1. Convection (linear advection): beta=50, 周期边界, u_exact = sin(x - beta*t)
2. Reaction: rho=5, 周期边界, 有闭式解
3. Wave: beta=3, Dirichlet 边界, 有闭式解
4. Allen-Cahn: 无闭式解（用 spectral solver 高分辨率解作为参考），周期性边界 + Neumann

### 核心结果

**诊断发现**:
- 失败模式中 train loss 被优化到 ~10^{-1} 的低值，但 test loss 很大——经典的 train/test divergence，即过拟合标志
- 域残差在配点处被最小化，但配点之间的区域残差巨大——直接可视化证实了 memorization
- 同样低精度下，减少配点会使 FP64 也失败（Wave 方程 144 点 vs 10201 点对比），证明 FP64 成功部分依赖于"过定问题"（大量点使网络难以记忆）

**Table 1 完整对比** (double PINN FP64 vs 其他方法):

| PDE | 指标 | PINNsFormer | PINNMamba | PINN_FP64 | **double PINN (ours)** |
|-----|------|------------|-----------|----------|----------------------|
| Convection | rMAE | 3.3e-2 | 1.8e-2 | 5.9e-3 | **4.9e-6** |
| | N (总点数) | 10403 | 10403 | 10403 | **800** |
| Reaction | rMAE | 1.5e-2 | 9.2e-3 | 2.7e-2 | **1.1e-5** |
| | N | 10403 | 10403 | 10403 | **456** |
| Wave | rMAE | 3.5e-1 | 1.9e-2 | 8.0e-3 | **3.8e-4** |
| | N | 10504 | 10504 | 10504 | **444** |
| Allen-Cahn | rMAE | 9.9e-1 | 1.4e-1 | 1.6e-2 | **9.6e-5** |
| | N | 10504 | 10504 | 10504 | **4396** |

- 误差提升 **超过一个数量级**（Convection: 5.9e-3 → 4.9e-6，约 1200x 改善）
- 配点数量减少: 最多减少 23x（800 vs 10403）
- PINNsFormer 和 PINNMamba 在 Allen-Cahn 上几乎完全失败（rMAE ~1）

**正则化对比实验** (Convection, beta=50, FP32):
- 无正则化: 失败模式，train/test divergence
- L^2 weight decay: 成功克服失败模式，但收敛较慢，最终 test loss 较高
- L^1 weight decay: 避免过拟合但稀疏性过度限制网络表达力，最终误差大
- Double PINN: 最快收敛（~5000 iteration），最低 test loss
- 三种正则化方法都消除了 train/test divergence

**Allen-Cahn 特殊分析**:
- FP32: train 和 test loss 同步运动但优化 stall——这是 stiff optimization dynamics，不是失败模式
- FP64 无正则化: 成功学到解，但后期 test loss 上升——学到解后开始记忆数据点
- FP64 + double PINN: 最佳性能

**Surprising 发现**:
1. FP64 训练不是万能药——减少配点后 FP64 也会触发失败模式
2. 简单 L^2 weight decay（深度学习最基础的正则化）就能让 FP32 PINN 克服失败模式——之前文献几乎没人在 PINN 上用正则化
3. 失败模式在过去被归因于"loss landscape 复杂/精度不足"，而实际原因简单得多：过拟合

### 明确局限

**作者承认的局限**:
1. Double backpropagation 理论计算量翻倍（需计算额外梯度），虽然实际被更少迭代和更少点抵消
2. L^2 和 L^1 简单且便宜但绝对性能和收敛速度不如 double PINN
3. 方法直接针对失败模式机制，但不改进优化过程本身——对不失败但 stiff 的方程无直接加速作用（但可改善后期精度）
4. 没有测试高维 PDE（curse of dimensionality 下 dense collocation 不可行，预期本文方法更重要，但未验证）

**审读发现的未讨论局限**:
1. 所有实验限于 1D 空间 + 时间（最大 2D 输入），没在 2D/3D 空间 PDE 上验证
2. 没有讨论 lambda_r（正则化强度）的调优策略或敏感性——这个超参数对性能影响多大？
3. 停止条件用 sqrt(machine_epsilon) 在某些问题上可能过早停止
4. 没有对比 Sobolev training (Son et al. 2023) 或 gradient-enhanced PINN (Yu et al. 2022) 的直接实验——只在 related work 中讨论
5. 仅用了 L-BFGS 优化器，没有测试 Adam 或其他一阶方法 + double PINN 的组合
6. 没有讨论在 inverse problem（PINN 的另一大应用场景）中失败模式是否存在同样的过拟合解释
7. 初始化方式固定为 Glorot normal，没有测试不同初始化对失败模式的影响

### 跟我们的关系

**与 0616-fex 课题的关系**:

**撞车评估**: 不撞车。该论文是 PINN 领域的诊断+修复工作，聚焦于 PDE 求解场景下的 failure mode 机理分析。0616-fex 课题的 failure exploration 方向如果是针对通用深度学习训练失败，这篇论文提供的是一个领域特化的 case study 而非直接竞争。

**差异化**:
- 本文的失败模式定义非常领域特化（PDE 残差配点过拟合），与我们关注的通用训练失败（梯度问题、loss 不降、mode collapse 等）有本质区别
- 本文的贡献在于"重新诊断已知问题"而非"发现新失败类型"

**可用的工具/方法**:
1. **train/test loss divergence 作为 failure mode 的标志**: 这是一个非常简单但被 PINN 文献长期忽略的诊断工具，可迁移到任何有 training set / evaluation set 分离的场景
2. **过拟合配点的可视化诊断方法**: 画 residual 在 collocation points 和非 collocation 区域的值，直观展示 memorization——这个可视化技术可用于 debug 任何基于采样点训练的模型
3. **Double backpropagation 作为梯度平滑正则化**: 在输入空间上惩罚 loss 的梯度，使模型输出在训练点附近平滑——可迁移到任何需要防止 memorization 的场景
4. **失败模式分类方法**: 区分"优化困难"(train/test loss 同步不降) vs "过拟合失败"(train loss 降 test loss 不降)——这是一个简洁的 failure taxonomy 维度
5. **"减少关键资源 -> 测试鲁棒性"的实验设计**: 减少 collocation points 来揭示 FP64 成功的真因——这种"subtractive experiment"设计值得借鉴

**该引**: 如果 0616-fex 工作涉及 failure mode 分类学，该论文提供了"overfitting vs optimization difficulty"这一分析维度，以及 train-test divergence 作为判别标准。
