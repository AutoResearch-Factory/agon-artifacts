# Causal-Audit: A Framework for Risk Assessment of Assumption Violations in Time-Series Causal Discovery (2604.02488)

- **Year**: 2026.04
- **Authors**: Marco Ruiz (Institute for Systems and Robotics, Instituto Superior Técnico, Lisbon), Miguel Arana-Catania (Univ. Oxford), David R. Ardila (JPL/Caltech), Rodrigo Ventura (ISR/IST)
- **Venue**: Journal of Causal Inference (2026), open access
- **Keywords**: causal discovery, time series, assumption validation, calibrated risk assessment, selective prediction, abstention, isotonic calibration, effect-size diagnostics, PCMCI+, VAR-Granger
- **One-liner**: 把时序因果发现的"假设违背"形式化为**可校准的方法失败风险评分**(四维 R + 不确定区间)加**弃权决策**——在自建 500-DGP 合成 atlas 上 AUROC>0.95、推荐集假阳性降 62%、严重违背样本 78% 弃权。

## 精华提炼

### 管线/方法

Causal-Audit 是一个**"因果发现之前"(pre-discovery / pre-hoc)** 的三阶段流水线,回答的问题不是"这个因果图有多可信",而是"我到底该不该在这份数据上跑因果发现"。核心动机是 **silent failure(静默失败)**:当平稳性、规则采样、有界时间依赖等假设被违背时,结构学习会输出**自信但错误**的因果图,且不给任何警告(引 Cliff 2020:自相关违背可把假阳性率推到 100%;Granger-Newbold 1974:非平稳过程产生伪回归)。

- **输入**:多变量时间序列 `X ∈ R^{T×N}`(pandas DataFrame + DatetimeIndex)。
- **Stage I — Diagnostic Auditing(诊断审计,方法无关)**:对五个假设族计算**效应量**(effect size,而非二元 p 值,以便后续聚合)诊断向量 `d = [d1..d5]`:
  1. **Stationarity(平稳性)**:ADF 单位根检验 + KPSS(以平稳为原假设,与 ADF 互补)+ Bai-Perron 多重结构断点(动态规划,报断点位置与幅度,单位为标准差)。
  2. **Irregularity(不规则采样/缺失)**:Gap 变异系数 `CV_gap = std(Δ)/mean(Δ)` + Little MCAR 检验 + 季节性缺失检验。
  3. **Persistence(持续性/自相关)**:积分自相关时间 `τ_int = 1/2 + Σρ(ℓ)` + 有效样本量 `T_eff = T/(2τ_int)` + Ljung-Box。
  4. **Nonlinearity(非线性)**:ΔRMSE 检验——线性自回归 vs 随机森林 3 折时序 CV,`ΔRMSE_rel = (RMSE_lin - RMSE_RF)/RMSE_lin`,>0.30 标记非线性。**仅报告,不聚合进风险分**(因校准语料是线性 VAR(1),且后果依方法而定)。
  5. **Confounding proxies(混淆代理)**:VIF `= 1/(1-R_j²)` + Chow 检验(系数跨子样本是否恒定)。注意:这是"与混淆相关的可观测症状(共线性/参数不稳定)",**不是**对潜混淆的直接检验。
  - 多变量诊断用 Benjamini-Yekutieli 控制 FDR,聚合取变量间最大校正 p 值(保守)。
- **Stage II — Risk Estimation(风险估计+不确定量化)**:把诊断映射成四个校准风险分 `R = (R_nonstat, R_irreg, R_persist, R_confound) ∈ [0,1]^4`。
  - **Logistic 聚合**:`R_k = σ(α_k + Σ w_kj·x_j)`,权重从领域知识初始化(见下"关键技术细节"),再经**分层贝叶斯校准**(PyMC v5.10,4 链×2000 draws,`R̂<1.01`)在合成 atlas 上精炼,冻结后验均值部署。
  - **Isotonic 二次校准**:对每维拟合单调映射 `f_k` 最小化 `Σ(f(r_i)-y_i)²`,`y_i` 为二元失败标签。
  - **Bootstrap 不确定区间**:B=100 次,对诊断加高斯噪声 `ε_j ~ N(0, max(0.1|x_j|, 0.05))`;并按 `√(T/T_eff)` 放大标准差(自相关越强区间越宽)。95% UI = `[R̄ ± 1.96·σ_adj]`。
  - SHAP 验证每维确由其"应该主导"的诊断主导(如持续性由 τ_int 主导,`|φ|=0.81`)。
- **Stage III — Selective Recommendation & Abstention(选择性推荐/弃权)**:
  - 复合风险取**最坏情况聚合** `R_composite = max(四维 R)`(任一维严重违背即足以致失败;均值会掩盖单维严重违背)。
  - 决策理论门槛来自效用函数(Berger 1985):`U`=成功 `u₊`、失败 `-u₋`、弃权 `-u∅`,排序 `u₋ ≫ u∅`。弃权阈值 `θ_abstain = (u₊+u∅)/(u₊+u₋)`;当所有方法 `R_m ≥ θ_abstain` 则弃权。
  - **方法-假设约束表**(硬/软约束):Granger(VAR) `R_comp≤0.30`、PCMCI+ `≤0.70`、LPCMCI† `≤0.80`、Transfer Entropy† `≤0.90`(†尚未接入决策管线)。
  - 四类强制弃权:`T_eff/T<0.30`、任一风险区间宽>0.5、灾难性违背(`R_nonstat>0.85` 或非平稳>0.70 且混淆>0.85 或 `max(R)>0.90`)、无方法置信>0.60。
- **两级使用**:Tier 1 = 仅 Stage I,做透明的假设审计报告;Tier 2 = 全流水线,自动方法选择+弃权。

### 关键技术细节

- **失败的定义**(生成 ground-truth 标签用):`R_k = P(Y=0|d_k)`,方法失败 = **FPR>0.50 或 FNR>0.80**(FPR 门槛更严,因假阳性会污染下游因果claim;FNR 更宽容,因假阴性只是知识不全)。标签源自**生成过程参数**(断点幅度、缺失率、谱半径)而非诊断输出,防止 feature-label 泄漏。
- **Logistic 初始权重**(表 3):`R_nonstat = σ(-2.0 + 1.0·break_mag + 2.0·drift + 0.5·adf + 0.5·kpss)`;`R_irreg = σ(-1.5 + 2.5·gap_cv + 2.0·missing + 1.5·seasonal_miss)`;`R_persist = σ(-1.5 - 3.0·(T_eff/T) + 2.0·τ_int)`(负权:T_eff/T 越低风险越高);`R_confound = σ(-6.5 + 2.0·chow + 1.8·resid_var + 0.5·vif)`(保守截距,VIF 权重小)。
- **合成 DGP Atlas(核心 artifact)**:500 数据集 / 10 违背族,共享一阶 VAR `X_t = A·X_{t-1} + ε_t`。F1 干净基线;F2 结构断点→R_nonstat;F3 不规则采样/缺失(15-35%)→R_irreg;F4 近单位根谱半径 ρ(A)∈[0.92,0.98]→R_persist;F5 潜混淆(L∈{1,2})→R_confound;F6 季节性;F7 非线性(tanh/sin/ReLU);F8 重尾/Laplace 噪声;F9 混合违背;F10 边界条件(N∈3-12, T∈200-2000)。F1-F9 用 N∈{5,6,7,8}, T∈{500,750,1000}。496 通过质量过滤 → **396 校准 / 100 holdout**,按违背族分层。
- **外部 benchmark**:TimeGraph(Ferdous 2025,18 类,ABCD 四组:线性/非线性/趋势季节/缺失,含 confounded "C" 变体)+ CausalTime(Cheng 2024,神经网络拟合真实数据生成,PM2.5 空气质量 36 变量 / Traffic 20 / Medical 20,T=40 短序列高维)。
- **两个 baseline**:no-audit(无脑跑 PCMCI+,代表现状)+ simple heuristic gate(ADF p<0.05 二元 + T_eff/T<0.3)。
- **校准工具**:isotonic regression(无参数、保 ROC 凸包)、分层贝叶斯(跨族 pooling + 族特定截距)、参数 bootstrap、SHAP。
- **算法/软件**:开源 MIT 包 `causal-audit`(github.com/marcoruizrueda/causal-audit),四模块:AssumptionAuditor / RiskQuantifier / MethodRecommender / RiskAwareGatekeeper;输出结构化 JSON,阈值走 YAML 配置。

### 核心结果

- **风险校准**(holdout 100):四维全部校准斜率∈[0.9,1.1]、ECE<0.05、**AUROC>0.95**(R_nonstat 0.981 / R_irreg 0.985 / R_persist 0.979 / R_confound 0.952)。
- **消融**:去掉 isotonic 校准,斜率 1.01→0.82、ECE 0.039→0.094(翻倍);去掉持续性诊断只轻微退化(ECE 0.039→0.051)。
- **决策/弃权**:默认框架**选择性 FPR 从 0.38 降到 0.14(降 62%)**,coverage 68%,selective F1 0.61→0.79。严格阈值(R<0.50)FPR 降到 0.09 但 coverage 51%。
- **按违背强度分层**:干净数据三方法都 FPR 0.08(等价);中度违背 Causal-Audit 把 FPR 从 0.31 压到 0.19;**严重违背** no-audit FPR 0.67,而框架**弃权**——150 个严重样本弃权 117 个(**78%**);160 次总弃权中 133(83%)确实对应 PCMCI+ FPR>0.50 的样本。
- **内部验证**:10 折 CV AUROC `0.978±0.011`(稳定,说明退化来自分布漂移非过拟合);**OOD 族 holdout**(F9-F10 只测不训)AUROC 0.974→0.918(**降 6%**),诚实报告泛化到未见违背组合会退化。
- **外部**:TimeGraph 18 类 + CausalTime 3 类 = **21/21 推荐-或-弃权决策与 benchmark 规范一致**(Group C 趋势 / D3 复合违背 → 弃权;Group A 干净线性 → 推荐)。CausalTime 三个数据集 R_cf=1.0 却仍推荐——因为高维共线性(20-36 变量)而非潜混淆(软约束正确识别)。
- **计算开销**:代表数据集(N=10,T=1000)总审计 30s ≈ PCMCI+ 发现时间的 10%;持续性诊断因 O(T²) 自相关计算主导耗时(12s)。

### 明确局限

- **无真机/无真实科学数据结论**:全部定量校准都在**作者自建的合成 atlas** 上;外部 benchmark(TimeGraph/CausalTime)也是合成/半合成,且**只测决策一致性,不测校准**(没有 ground-truth 失败率)。
- **架构设计用了全 atlas 的探索性分析**(诊断选择、聚合规则、约束阈值),396/100 split **只验证参数估计,不验证这些高层设计决策**——作者自己点破,这是最大的方法学软肋。
- **只支持 2 个方法**(PCMCI+ / VAR-Granger),32% 弃权率部分是方法目录太小所致;TE/LPCMCI 尚未接入。
- **混淆代理 ≠ 潜混淆**:VIF/Chow 只是共线性/不稳定的代理,高维会误报(CausalTime R_cf=1.0)。
- **非线性未校准**(校准语料是线性 VAR(1));非线性只作诊断报告不进风险分。
- **失败定义有离散边界**(FPR>0.5/FNR>0.8),敏感性分析只做了初步。
- **21/21 全一致**略显"太干净",且"expected decision"由作者自行依违背 profile 判定,无独立裁决者。
- **跟我们的关系**:见"跟我们的关系"与"Read by"章节——这篇是 topic 的**最贴近的对立面/工具库**,既是必引 baseline 又提供现成诊断武器。

### 跟我们的关系

对课题 **0710-causal-scs(因果效应不稳定性作为强对流预警指标)**,这篇是**最贴近的"哲学对立面 + 工具库"**,同时是**审稿人必然拿来打的 baseline**。

- **同底反向**:两者操作的是**同一现象**——因果发现在假设违背(非平稳、混淆、持续性)下的**不稳定/不可靠**。Causal-Audit 把这不稳定当**噪声**去量化并**弃权规避**;我们的 topic 把它当**信号**去提取并用作 SCS 预警。这个"信号/噪声反转"正是 topic 的差异化核心,且据现有文献**无人做过**用违背诱导的不稳定作为预测特征。
- **是否撞车?** 核心 claim **不直接撞**:Causal-Audit 声称"能校准方法失败概率并弃权";topic 声称"会触发弃权的那种不稳定本身携带 SCS 环境演化的物理信息"。二者正交。**但有致命张力**:Causal-Audit 用 AUROC>0.95 强证据表明"违背诱导的不稳定是**可从数据诊断预测的、可校准的统计噪声**"。这威胁 topic 的核心假设——若我们的不稳定只是 Causal-Audit 式的可校准违背噪声,则"缺失过程/环境演化"的物理解释站不住。→ **topic 必须证明:因果效应不稳定指标对 SCS 事件的预测力,超出 Causal-Audit 四维风险分(及其滑窗动态)作为通用违背噪声所能解释的部分。** 这是一条现在必须加进去的 baseline/消融。
- **直接可用的工具/方法**:
  1. **Stage I 效应量诊断电池**(ADF/KPSS/Bai-Perron/gap-CV/Little-MCAR/τ_int/T_eff/VIF/Chow/ΔRMSE)= 恰好是 topic 想追踪的"假设违背"的现成量化;topic 可**在滑动时间窗上算这些诊断,看其时序演化(而非绝对值)是否预示 SCS**。
  2. **四维校准风险分 R + 不确定区间**可直接作为"不稳定量化";topic 的"不稳定性"本质≈ Causal-Audit 风险分,而 topic 用其**时间导数/方差/滑窗轨迹**作指标。
  3. **开源 `causal-audit` 包**(MIT,AssumptionAuditor / RiskQuantifier 模块)可当诊断引擎直接复用。
  4. **T_eff-aware 不确定放大 √(T/T_eff)**——对 topic 高自相关滑窗分析有用。
  5. **合成 atlas 方法学**(VAR(1)+可控违背注入,10 族)可作 topic Stage-1 模拟验证的参照;topic 用**耦合 Lorenz(分岔)** 而非 VAR,恰是一个差异化点。
- **该不该引? 必引**——(a) 作为"时序因果发现中假设违背不稳定"的最近对立框架与天然 baseline/foil;(b) 作为诊断工具箱来源。定位话术:"Causal-Audit 把违背诱导的不稳定当作弃权理由;我们反转它,检验同一不稳定是否是强对流的物理前兆信号。"
- **差异化(topic 安全区)**:①领域(强对流/大气 vs 领域无关合成);②目标(物理事件早期预警 vs 方法可靠性 QC);③信号/噪声反转;④不稳定的**时间动态/跨方法分歧**(vs 静态 per-dataset 风险分);⑤方法(J-PCMCI+ 多 regime、LKIF 信息流——Causal-Audit 未覆盖);⑥Stage-1 用耦合 Lorenz 分岔(动力系统)vs VAR(1) atlas。

## Read by: 0710-causal-scs (2026-07-10)

**审读视角结论(撞车评估 / 可用工具 / 差异化):**

1. **撞车评估**:核心 claim **不直接撞车**,但存在**必须正面回应的致命张力**。Causal-Audit 用强证据(AUROC>0.95)证明:时序因果发现在假设违背下的不稳定是**可从数据诊断预测、可校准的统计噪声**。这直接威胁本 topic 的立论——若我们观察到的"因果效应不稳定"只是这种可校准的违背噪声,则把它解释为"缺失过程/强对流环境演化的物理信息"就缺乏根据。**必须新增的 baseline**:用 Causal-Audit 四维风险分(及其滑窗动态)作为 null model,本 topic 的因果效应不稳定指标必须在 SCS 预测上**显著超越**它,才能主张物理信号性。这是审稿人第一刀会砍的地方,建议在 idea/proposal 阶段就把它写成显式对照实验。

2. **可用工具(可直接复用)**:
   - 开源 MIT 包 `causal-audit` 的 **AssumptionAuditor**(Stage I 诊断)可当**滑窗诊断引擎**直接用——把 ADF/KPSS/Bai-Perron/τ_int/T_eff/VIF/Chow/ΔRMSE 沿时间窗滑动计算,得到"违背强度的时序轨迹",其**波动/突变/分歧**正是本 topic 想要的 instability 原料。
   - **RiskQuantifier**(四维校准风险分 + √(T/T_eff) 不确定放大)可作为不稳定的量化基线;本 topic 的贡献在于用其**时间导数/跨方法分歧**而非静态值。
   - 其 **VAR(1)+可控违背注入** 的 atlas 生成方法学,可移植为本 topic Stage-1 的一个对照 DGP(与耦合 Lorenz 分岔互补)。

3. **差异化(本 topic 的安全区,建议写进 novelty 论证)**:①领域=强对流/3km 大气环境场(Causal-Audit 领域无关,仅测合成+空气质量/交通/医疗);②目标=物理事件**早期预警**(Causal-Audit=方法可靠性 QC / 弃权);③**信号-噪声反转**——用不稳定,而非规避不稳定;④刻画对象=不稳定的**时间动态与跨方法分歧**(Causal-Audit=静态 per-dataset 风险分);⑤方法覆盖=J-PCMCI+(多 regime)、LKIF(信息流,低成本)——Causal-Audit 只做 PCMCI+/VAR-Granger,明确未接入 LKIF;⑥Stage-1 模拟=耦合 Lorenz 已知分岔(动力系统),vs Causal-Audit 的线性 VAR(1)。

4. **一句话定位**:Causal-Audit 是本 topic 的"必引对立面"——它把违背诱导的不稳定当作**弃权理由**,本 topic 反转它,检验同一不稳定是否是强对流的**物理前兆信号**;二者共享诊断工具箱,但目标、价值取向、时间维度处理完全相反。
