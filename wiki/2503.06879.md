# Reinforcement Learning-Based Symbolic Regression for Load Modeling (2503.06879)

- **Year**: 2025.03
- **Authors**: Ding Lin, Han Guo, Jianhui Wang, Meng Yue, Tianqiao Zhao
- **Venue**: arXiv
- **Keywords**: symbolic regression, actor-critic, risk-seeking policy gradient, expression tree, load modeling, power systems, candidate pool, dynamic load
- **One-liner**: 用 Actor-Critic 生成可解释负荷模型表达式，并在仿真电网负荷上优于多数基线。

## 精华提炼

### 管线/方法
1. 任务输入是一组负荷测量数据 `{X, y}`，其中 `X` 是扰动下的输入信号，`y` 是目标负荷响应。
2. 论文把动态负荷建模写成寻找未知函数 `y=f(x_1,...,x_n)` 的符号回归问题。
3. 方法先固定一个可训练二叉表达式树，树深 `L` 控制表达式复杂度。
4. 树的内部节点表示一元或二元算子，叶子节点表示变量或操作数。
5. 每一层在一元算子层和二元算子层之间交替。
6. 一元算子示例包括 ReLU、sigmoid、exp、tan 等可微变换，论文没有完整公开算子库清单。
7. 二元算子示例包括加法和乘法，用来组合上一层节点输出。
8. 每个算子节点带有可训练系数和偏置，例如 `a tan(bx)+c` 或 `a(bx_1 * cx_2)+d`。
9. Actor 网络负责按概率为表达式树节点选择算子序列。
10. Critic 不是普通 value network，而是当前算子序列对应的可训练表达式树。
11. Actor 采样一批 operator sequences `{a^(1),...,a^(N)}`。
12. 对每个候选序列，Critic tree 前向计算预测值 `T_w(x,a)`。
13. Critic 的训练目标是 MSE：`M(X,a)=1/n sum_i (y_i-T_w(x_i,a))^2`。
14. Critic 参数 `w` 通过反向传播和梯度下降更新。
15. 训练后的表达式用 RMSE 转成有界 reward：`R(a)=1/(1+RMSE)`。
16. Actor 使用 policy gradient 更新算子采样概率。
17. 论文重点采用 risk-seeking policy gradient，而不是只优化平均 reward。
18. risk-seeking 目标只强化 batch 中 reward 高于经验分位阈值 `R_epsilon` 的样本。
19. 论文比较了 `epsilon=0.3, 0.5, 0.7` 和 standard policy。
20. 训练过程中维护固定大小候选池 `P`，保存 reward 排名前 `C` 的表达式。
21. Actor-Critic 训练结束后，对候选池中的表达式逐个做一阶优化微调。
22. 最终返回候选池中 MSE 最小的表达式，并用中序遍历生成公式。
23. 整体 pipeline 是：仿真/测量负荷数据 -> 固定深度表达式树 -> Actor 采样算子 -> Critic 拟合系数 -> reward 计算 -> risk-seeking policy update -> candidate pool -> gradient fine-tuning -> 显式负荷模型表达式。

### 关键技术细节
24. 表达式树深度是核心超参数，论文测试 `L=3,5,7`。
25. 最终选择 `L=5`，理由是整体精度、复杂度和过拟合风险之间折中最好。
26. risk-seeking 超参数测试 `epsilon=0.3,0.5,0.7`，最终选择 `epsilon=0.5`。
27. reward 用 RMSE squash 到 `(0,1]`，低误差表达式得到更高 reward。
28. Actor 的 action 是整棵树的 operator sequence，而不是单步连续控制。
29. Critic 的参数包括所有节点的系数和偏置。
30. 算法伪代码中给出 `I_1` Actor 迭代次数、`I_2` Critic 迭代次数、`I_3` fine-tuning 迭代次数、候选池大小 `C`、action batch size `N`。
31. 论文没有给出这些训练迭代数和 batch size 的具体数值，这是复现上的主要缺口。
32. 数据集来自 modified IEEE 39-bus system 和 33-node feeder model 的仿真。
33. 扰动在 `3.5s` 注入。
34. 评测包含 bus fault 和 line fault 两类故障场景。
35. 负荷类型包括 constant impedance `CI`、induction motor `IM`、photovoltaic distributed energy resources `PV-DER`。
36. 系统中包含 `10` 个 CI、`3` 个 IM、`4` 个 PV，以及其他负荷。
37. 每个故障场景选取一个 CI、一个 IM 和一个 PV-DER 展示结果。
38. 观测/预测指标是 voltage `V`、active power `P`、reactive power `Q`。
39. 对比基线是 ANN、ZIP 和 Polynomial Regression。
40. 论文还比较了 standard policy gradient 与 risk-seeking policy gradient。
41. 论文强调表达式简单性和可解释性，但没有展示最终学到的完整公式列表，这是证据链上的弱点。

### 核心结果
42. risk policy 对比中，`epsilon=0.5` 被选为默认，因为多数负荷/故障组合下总体最稳。
43. bus fault + CI 下，`epsilon=0.5` 的 `V/P/Q` RMSE 为 `0.055/0.070/0.083`。
44. bus fault + IM 下，`epsilon=0.5` 的 `V/P/Q` RMSE 为 `0.044/0.059/0.014`。
45. bus fault + PV 下，`epsilon=0.5` 的 `V/P/Q` RMSE 为 `0.042/0.034/0.025`。
46. line fault + CI 下，`epsilon=0.5` 的 `V/P/Q` RMSE 为 `0.043/0.069/0.085`。
47. line fault + IM 下，`epsilon=0.5` 的 `V/P/Q` RMSE 为 `0.039/0.019/0.057`。
48. line fault + PV 下，`epsilon=0.5` 的 `V/P/Q` RMSE 为 `0.040/0.039/0.023`。
49. tree depth 对比中，`L=5` 通常优于 `L=3`，但部分单项指标 `L=7` 更低。
50. 例如 bus fault + CI 的 `V` 在 `L=7` 为 `0.053`，低于 `L=5` 的 `0.055`。
51. 作者仍选择 `L=5`，说明其判断基于总体稳定性而不是单项最小值。
52. 不同模型对比中，proposed model 在大多数 `V/P/Q` 指标上为最优或接近最优。
53. bus fault + CI：proposed `0.055/0.070/0.083`，ANN `0.051/0.078/0.092`，ZIP `0.652/0.783/0.943`，Polynomial `2.188/3.068/5.394`。
54. bus fault + IM：proposed `0.044/0.059/0.014`，ANN `0.040/0.066/0.012`，ZIP `0.536/0.694/0.253`，Polynomial `1.759/2.579/2.629`。
55. bus fault + PV：proposed `0.042/0.034/0.025`，ANN `0.047/0.036/0.029`，ZIP `0.579/0.461/0.382`，Polynomial `1.427/1.385/3.173`。
56. line fault + CI：proposed `0.043/0.062/0.085`，ANN `0.047/0.069/0.088`，ZIP `0.428/0.727/0.983`，Polynomial `1.526/3.577/5.851`。
57. line fault + IM：proposed `0.039/0.019/0.057`，ANN `0.042/0.021/0.066`，ZIP `0.403/0.279/0.596`，Polynomial `1.339/1.231/3.358`。
58. line fault + PV：proposed `0.040/0.039/0.023`，ANN `0.050/0.033/0.028`，ZIP `0.416/0.529/0.296`，Polynomial `1.735/1.636/2.729`。
59. surprising 点：ANN 在少数 `V` 或 `Q` 指标上仍优于 proposed model，因此论文的优势主要是准确率接近/超过 ANN 同时保留显式表达式，而不是逐项全面压制 ANN。
60. 另一个反直觉点：`epsilon=0.7` 并不总是更好，过度 risk-seeking 会牺牲整体稳定性。

### 明确局限
61. 没有公开代码、数据生成脚本或完整训练配置，第三方复现实验会比较困难。
62. 没有列出完整 operator library，无法判断搜索空间和 FEX/DSR/PySR 等方法是否公平可比。
63. 没有报告 wall-clock time、sample count、GPU/CPU 配置或搜索预算。
64. 没有展示最终发现的表达式全集，只展示预测曲线和 RMSE 表，削弱了可解释性 claim。
65. 评测数据来自仿真系统，不是真实 PMU 或现场扰动数据。
66. 只有 modified IEEE 39-bus 和 33-node feeder，系统规模和分布外泛化没有系统测试。
67. 没有和 PySR、DSR、AI Feynman、NeSymReS、FEX、SSDE 等更强 SR/PDE symbolic solver 对比。
68. 对 ANN 的比较较粗，未说明 ANN 架构、参数量和训练预算。
69. 对 ZIP 和 Polynomial 的比较说明 proposed model 能处理非线性，但这些基线偏弱。
70. `L=5` 的选择有经验性，缺乏复杂度-泛化的理论或统计置信区间。
71. 候选池机制合理，但没有消融 candidate pool 是否真正贡献收益。
72. risk-seeking policy 有 epsilon 消融，但没有分析 reward landscape 或失败样本。
73. 论文没有说明噪声、缺测、传感器偏差下的鲁棒性。
74. 没有真机或电网在线部署实验，因此工程可用性仍停留在仿真验证。
75. 没有讨论表达式是否满足物理约束、稳定性约束或电力系统可解释规则。

### 跟我们的关系
76. 对 `0616-fex` 来说，这篇是方法学近邻，不是 PDE/FEX 主线的直接竞品。
77. 直接重叠点是：固定深度表达式树、RL/policy-gradient 搜索、risk-seeking reward、候选池/后处理微调。
78. 如果我们的 claim 是泛化的“用 RL 搜索表达式树做 symbolic regression”，它已经覆盖一部分应用层证据，必须引用。
79. 如果我们的 claim 是 FEX 的 `S_k` approximation theory、PDE residual solver、高维 PIDE、Hamiltonian/stochastic FEX，则不撞车。
80. 它没有 FEX 的 Holder 函数近似理论，也没有证明搜索 sample complexity。
81. 它没有把表达式树用于 PDE 边界条件、PDE residual、collocation 积分或高维维度灾难问题。
82. 它可作为一个跨领域信号：FEX 式表达式树搜索不只适用于 PDE，也能迁移到电力系统负荷响应建模。
83. 它最可借鉴的是 candidate pool + fine-tuning 的两阶段做法，和 FEX 的 Adam/BFGS 后处理有相似精神。
84. risk-seeking epsilon 消融也值得借鉴：FEX controller 的 reward quantile 不应默认越激进越好。
85. 它暴露的弱点对我们有帮助：若只报 RMSE 而不报公式、预算、失败模式，解释性 claim 会很虚。
86. 对 `0616-fex` 后续写作，建议把它放在“RL-based symbolic regression applied outside PDE”或“power-system load modeling SR”相关工作中。
87. 它不应作为 FEX PDE solver 的核心 baseline，因为任务、损失、约束和维度都不同。
88. 如果做 FEX in real systems/data-driven physical law discovery，这篇应该引用为动态负荷建模应用先例。
89. 差异化重点应放在：PDE residual + boundary condition、`S_k` 理论、可审计表达式、搜索复杂度/失败模式诊断、真实或标准化 benchmark。
90. 对下一轮搜索，应该查 actor-critic SR、risk-seeking SR、power-system load symbolic regression、FEX candidate pool、PDE symbolic solver 与 expression-tree policy gradient。
