# PI-MFM: Physics-informed multimodal foundation model for solving partial differential equations (2512.23056)

- **Year**: 2025.12
- **Authors**: Min Zhu (Yale), Jingmin Sun (JHU), Zecheng Zhang (Notre Dame), Hayden Schaeffer (UCLA), Lu Lu (Yale, corresponding)
- **Venue**: arXiv
- **Keywords**: PDE, multimodal foundation model, physics-informed neural networks, multi-operator learning, derivative computation, fine-tuning, PROSE, Polish notation
- **One-liner**: 提出 PI-MFM 框架，将物理信息（PDE 残差）自动注入 PDE-encoding 多模态基础模型的预训练和微调中，在稀疏数据、部分域、噪声等场景下显著优于纯数据驱动方法，并展示 zero-shot physics-only 微调到未见 PDE 族。

## 精华提炼

### 管线/方法

PI-MFM 框架的完整管线：

1. **输入**：初始条件 $u_0(x)$（由正弦波叠加生成，Min-Max 归一化到 $[0,1]$）+ PDE 符号表达式 $s$（以 Polish/前缀记法编码为 token 序列）
2. **数据编码器（Data Encoder）**：将初始条件嵌入为向量，输出形状 $(B, N_{emb})$
3. **符号编码器（Symbol Encoder）**：将 Polish 记法 token 序列编码，输出形状 $(B, N_{emb})$
4. **融合模块（Fusion Module）**：将数据和符号两条流的嵌入融合，输出 context $(K,V)$ 给 decoder
5. **数据解码器（Data Decoder）**：以 query 坐标 $(t,x)$ 的嵌入为 $Q$，通过 cross-attention 预测解 $u(t,x)$，输出 $(B, M)$
6. **向量化 PDE 损失计算（Algorithm 1）**：
   - 单次 batched forward pass 在 collocation points $TX$ 上获得所有 $U$
   - 分析 batch 内所有 PDE 的 token 序列，确定所需导数类型集合 $\mathcal{D} \subseteq \{u_t, u_{tt}, u_x, u_{xx}, \ldots\}$
   - 若用 AD：通过向量化 forward-mode JVP 在单次扫描中计算所有导数（每个坐标方向一个 JVP）
   - 若用 FDM：构造移位 collocation blocks，一次 forward pass，组装有限差分
   - 将 $U$ 和 $\{U_d\}_{d\in\mathcal{D}}$ 打包为 $\mathbf{Z} \in \mathbb{R}^{B \times M \times C}$
   - 为每个 batch item $b$ 解析 token 序列 $s_b$ 为表达式树，在 $\mathbf{Z}_b$ 上递归求值，得到残差向量 $r_b$
   - 计算 MSE: $\mathcal{L}_{PDE} = \frac{1}{BM}\sum_{b,m} r_{b,m}^2$

7. **总损失**：$\mathcal{L} = \omega_{PDE}\mathcal{L}_{PDE} + \omega_{IC}\mathcal{L}_{IC} + \omega_{IC'}\mathcal{L}_{IC'} + \omega_{data}\mathcal{L}_{data}$

**关键设计决策**：
- 周期性 BC 不是通过显式损失项施加，而是通过将输入坐标用周期特征增强（periodic feature augmentation）来内置
- Collocation points 每 iteration 随机重采样（Resample 策略），这是提升精度的关键
- Layer normalization 替代 Batch normalization，因为 BN 会在 query index 维度上混合统计量，破坏 Jacobian 的块对角结构

### 关键技术细节

**网络架构**（基于 PROSE 实现）：
- 数据 encoder + 符号 encoder → self-attention stacks → fusion module → cross-attention decoder
- 所有 self-attention 和 cross-attention 层都使用 Layer Normalization
- 训练精度：模型存储和反向传播用 float32；FDM 实验对比了 float64/float32/float16/bfloat16

**训练配置**：
- 优化器：AdamW, lr=$10^{-4}$, weight decay=$10^{-4}$, $\beta_1=0.9$, $\beta_2=0.999$, $\epsilon=10^{-6}$
- LR schedule：linear warmup（前 3200 iter）到 base lr，然后 cosine decay 到 0（共 $T_{max}=32000$ iter）
- Batch size：128（sparse/few-func 场景），100（small_func 场景中 $\mathcal{Z}^{data}$ 的 batch size）
- Collocation points：默认 500 个，每 iter 随机重采样；resample 实验 range $[20, 2000]$
- 四类 loss 中 $\mathcal{Z}^{PDE} = \mathcal{Z}^{IC} = \mathcal{Z}^{IC'} = \mathcal{Z}^{data}$（大部分场景）

**数据集**：
- 13 族 1D time-dependent PDE（Advection、Diffusion 及变体、Conservation Law 族、Burgers、Wave、Klein-Gordon、Sine-Gordon）
- 10 族用于训练/测试，3 族（Diff-Bi, Burgers, Wave）留给下游 zero-shot fine-tuning
- 训练/验证/测试：500K/100K/100K labeled pairs（standard），或 $10N_{func}$ / 100K / 100K（few-func 场景）
- 每个 PDE 族的参数从中心值 ±10% 区间随机采样
- 基准网格：$64 \times 128$（time × space）；训练在偶数时间片，测试在奇数时间片（这比在相同时间片上测试更难）
- 数值求解器生成 ground truth：spectral solver（linear PDEs）、explicit Runge-Kutta 4th order（nonlinear PDEs）

**评测指标**：$L^2$ relative error 和 $H^1$ relative error

### 核心结果

**1. 稀疏标注数据（Sparse Labeled Data）**：
- 在 $2\times8$ 到 $32\times128$ 五种分辨率下，w/ Phys. 在所有分辨率都大幅优于 w/o Phys.
- 最极端：$4\times16$ 分辨率，w/ Phys. 平均 L² 误差 ~1%，w/o Phys. 平均 L² 误差 ~11.5%；w/o Phys. 的 H¹ 误差达 35%
- 10 族 PDE 全部受益，家族分布图显示 w/ Phys. 在中位数和 spread 上全面胜出

**2. 部分时间域标注（Partial Temporal Domain）**：
- 只在 $t \in [0, 1/2]$ 训练时，w/o Phys. 在 $t > 0.5$ 外推区误差急剧发散
- w/ Phys. 在整个时间域（含外推区）保持低且稳定的误差
- 训练时间域越短，物理损失的增益越大

**3. 函数空间稀疏数据（Few Function Pairs）**：
- $N_{func}$ 从 10 到 50K：w/ Phys. 在 $N_{func}=10$ 时 L² 误差约 3%，w/o Phys. 约 16%
- w/ Phys. 的验证误差持续下降，w/o Phys. 在 $N_{func}$ 小时很快 plateau
- 当 $N_{func} > 2000$，两者的差距缩小，曲线趋于饱和

**4. 噪声鲁棒性**：
- SNR=1（$\gamma=1$，即 100% label noise）下，$N_{func}=50000$ 仍能达到 ~3% L² 误差
- 关键发现：$N_{func}$ 小时（100-1000），w/o Phys. 严重过拟合（error 先降后升），w/ Phys. 保持稳定并持续改善
- 物理损失充当了有效的 physics-guided prior 和 regularizer

**5. Collocation Point 重采样的影响（Surprising Result）**：
- Resample 策略在所有 collocation point 数量下都显著优于 Non-resample
- 例如：2000 个点 Resample 的误差 ≈ 200 个点 Non-resample 的误差，说明 Resample 策略比单纯增加点数更有效
- 对 KG 和 SG 等重尾 PDE 族，Resample 特别抑制了 outlier

**6. 微分方法对比（AD vs FDM）**：
- FDM float32 在调好 step size 后可匹敌 AD 精度（L² ~1.04% vs AD ~1.04%），但 GPU 内存和时间都更少（17.96 GiB / 0.203 s/iter vs 53.60 GiB / 0.561 s/iter）
- FDM float64 内存 34.78 GiB，时间 0.258 s/iter，精度几乎等于 AD
- Forward AD bfloat16/float16 精度几乎没有退化（~1.07% / ~1.02%），因为 AD 的精度不依赖于数值精度
- Reverse AD float32 最贵（87.34 GiB / 0.642 s/iter）且精度与 forward AD 相同
- **建议**：FDM float32（需 step size tuning）或 Forward AD float16（无需 tuning）
- 所有 FDM 精度曲线呈 U 形，验证了截断误差 vs 舍入误差的理论分析

**7. Zero-shot Physics-only Fine-tuning（最亮眼的结果）**：
- 从 w/ Phys. pretrained model（$N_{func}=100$）出发，仅用 PDE 残差 + IC/BC（无任何 labeled solution data），微调到未见 PDE 族
- 3 个下游 PDE 族（Burgers, Wave, Diff-Bi）的 L² 误差快速降到 ~1%
- 显著优于 physics-only training from scratch（scratc 模型收敛慢且 plateau 在数%）
- 注意：Wave 方程 zero-shot prediction（iteration=0）的误差接近 100%，但 fine-tuning 后迅速降到 1% 以下

### 明确局限

1. **仅限 1D time-dependent PDE**：所有实验都在 1D 空间 + 1D 时间上进行。作者在 future work 中提到扩展到 2D/3D，但当前阶段完全没有验证
2. **周期性 BC 的取巧处理**：用 periodic feature augmentation 避开 BC loss，但这意味着模型假设了周期性 BC，对 Dirichlet/Neumann BC 的泛化未验证
3. **仅 13 族 PDE 的 toy-ish benchmark**：虽然是同类工作中最大的 benchmark，但 PDE 多样性有限（都是 scalar、1D、time-dependent、periodic BC），与真实工程场景（Navier-Stokes、多物理场耦合）差距大
4. **无标准 benchmark 对比**：没有在 PDEBench、PDEArena 等社区标准 benchmark 上评测，全都是自建数据集
5. **未与 PINO 基线对比**：虽然声称与 physics-informed 方法不同（因为 PINO 是 SOL 不是 MOL），但应该提供一个 PINO-per-PDE 的对比来展示 MOL 单模型 vs SOL 多模型的效率/精度 trade-off
6. **硬件要求高**：forward AD float32 需 53.6 GiB GPU 内存，reverse AD 需 87.34 GiB，远超普通学术 GPU（如 RTX 3090/4090 的 24 GiB），虽然 FDM 可降到 10-18 GiB
7. **PROSE 作为唯一 backbone**：声称框架 architecture-agnostic，但只验证了 PROSE 一种实现
8. **Polish notation 的局限**：符号 PDE 编码依赖人工预先写出正确的前缀表达式，不适合从数据中自动发现的 PDE
9. **Zero-shot fine-tuning 仅在 3 族上测试**：未见 PDE 族太少，且都是与训练集有结构相似性的一阶/二阶 PDE

4. **FDM 作为 AD 的低成本替代**：float32 FDM + 合适 step size 可匹敌 AD 精度但节省 ~60% 内存和 ~65% 时间

**差异化**：
- PI-MFM 的核心创新是 "物理注入 PDE-encoding 基础模型"，0616-fex 可根据自身目标做一个 "物理注入特征提取" 的类比设计
- PI-MFM 的 Polish notation 方法需要预先知道 PDE 的符号形式，如果 0616-fex 面对的是未知系统，可以探索从数据中学习 PDE 约束的方法（与 PI-MFM 正交）
