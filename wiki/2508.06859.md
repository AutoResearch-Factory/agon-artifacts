# MeteorPred: A Meteorological Multimodal Large Model and Dataset for Severe Weather Event Prediction (2508.06859)

- **Year**: 2025.08
- **Authors**: Shuo Tang, Jian Xu† 等 (中科院自动化所 MAIS / 国科大 / 中关村学院 / 中国气象局 CMA, 通讯 Jian Xu)
- **Venue**: arXiv (CVPR 2026 投稿模板, paperID 29850)
- **Keywords**: severe weather warning, multimodal LLM, ERA5, benchmark dataset, 4D meteorological data, plug-and-play fusion, CMA warnings, LLM-as-a-judge
- **One-liner**: 用 42 万对 ERA5 4D 场×CMA 预警文本构建 MP-Bench, 加三个即插即用模块让 VLM 直接吃 185 通道 4D 气象数据生成灾害天气预警文本。

## 精华提炼

### 管线/方法
完整 pipeline：
1. **数据源**：ERA5 再分析（0.25°, 逐小时, 37 层气压 1–1000 hPa, 5 变量 z/u/v/t/q → 185 通道）中国区域 + CMA 全国 2,412 站 2023–2024 年灾害天气预警文本。
2. **清洗**：预警只保留发布时间/地点/类型/等级；同站同类型 2 小时内合并取最高等级 → 371,703 条有效预警；按地区和季节均匀采样补 49,660 条"正常天气"负样本 → 共 **421,363 对**。
3. **对齐**：以预警发布时刻 t 为锚，取 **[t, t+11] 的 12 小时窗口**（1h 间隔），不做时间平均——这是与 CLLMate（日平均）等前作的核心差异。
4. **QA 构造**：四种任务——MC（选择题，主类型 7+1 类 / 子级别蓝黄橙红）、T/F（判断题）、RSW（区域灾害天气识别）、NSW（全国尺度结构化预警生成 [地名][类型][等级]）。
5. **模型 MMLM**：4D 张量 (T×C×H×W) 并行过三个即插即用模块 → concat → 可学习 3D 卷积 Fusion Layer → MLP 投影到 backbone 输入维度 → LLM 生成预警文本。
6. **评测**：MC/T/F/RSW 用 Acc + Macro-F1；NSW 用 GPT-4o LLM-as-judge（专家定 0–5 rubric）；2023 训练 / 2024 测试（年际切分）；另用 1,000 条 NOAA Storm Events 做跨区域泛化。

### 关键技术细节
三个模块（公式完整可复现）：
- **DTGF**（Dynamic Time-Gated Fusion，时间维）：Δx_t = ‖x_t − x_{t−1}‖₂（通道维 L2），g_t = Sigmoid(MLP(Δx_t))，x̃_t = g_t·x_t——用相邻时步差分驱动时间门控，突出气象场突变时段。
- **TGS**（Text-Driven Gaussian Spatial Masking，空间维）：从问题文本提取 N 个经纬度 → 最近格点 (h_i,w_i) → 2D 高斯 G_i(h,w)=exp(−((h−h_i)²+(w−w_i)²)/2σ²) → 求和成 mask M。
- **TGCA**（Text-Driven Channel Attention，垂直/通道维）：P=Linear(文本嵌入 y)，V=Mean(X)（时空平均的通道描述子），Y = X·Sigmoid(Softmax(VPᵀ)P)——文本引导的 185 通道重加权。

训练配置：8×A800 40GB；backbone 四选：Qwen2.5-VL-7B-Instruct / LLaVA-NeXT-Video-7B / Video-LLaVA-7B / InternVL3-8B；LoRA rank 8, alpha 16, dropout 0, target=all linear + gating_mlp/linear/text_proj/mlp_channel；lr 5e-5 cosine, batch 2×grad-accum 8, 3 epochs, bf16, cutoff_len 9500, max_samples 220k；LLaMA-Factory 风格配置全表公开；结果为 3 次独立运行平均。

类别分布（2023 训练集）：Gale 43.54%、Rain Storm 22.24%、Heat Wave 9.74%、Cold Wave 5.30%、Frost 3.36%、**Hail 仅 2.71%**、Snow Storm 1.50%、Normal 11.62%——重度长尾。

### 核心结果
主表（全量训练, 2024 测试, Acc/Macro-F1 0–100, NSW 0–5）：

| 模型 | MC-main | MC-sub | T/F | RSW | NSW |
|------|---------|--------|-----|-----|-----|
| GPT-4o 零样本（3 通道图像） | 11.92/2.92 | 6.51/0.88 | **0.19** | 14.03 | 0.1 |
| Baseline 最好（3 层通道 LoRA 微调） | 59.77/26.88* | 49.01/9.78* | 69.23 | 65.31 | 1.7 |
| MMLM-Qwen2.5-VL（185 通道+三模块） | **72.37/50.88** | **58.19/29.31** | **87.13** | **71.23** | **2.1** |

（* baseline 各列最优分属不同 backbone）

- GPT-4o 零样本几乎完全失效（T/F 0.19% 远低于随机）→ 未微调闭源模型读不懂气象场。
- 消融（5,000 样本子集, Qwen2.5-VL）：无模块 42.73 MC-main → 三模块全开 58.27；T/F 58.47→79.21；TGCA+TGS 组合对 T/F 增益最大（+11.97）；三模块并行融合最优。
- 时间窗消融：1h/6h/12h → MC-main 54.29/58.01/58.27，T/F 67.17/72.23/79.21，RSW 52.71/54.63/58.63——**12h 窗口一致最优**，长时间上下文对所有任务有益。
- 跨区域（NOAA 美国 1,000 样本, MC-main）：64.20 Acc / 35.18 Macro-F1，有降但可用。
- **物理一致性分析（surprising/有价值）**：TGCA 通道权重自动集中在 500 hPa 经向风（中层槽脊）、950 hPa 纬向风（近地面大风）、300 hPa 温度（高空冷心/急流）、825–875 hPa 位势/湿度（低层斜压性与水汽）——与天气学诊断吻合；DTGF 时间权重差（红 vs 蓝预警）匹配 CMA 预警时效定义（红色冰雹 2h 内、橙色 6h；红色暴雪 6h/≥15mm vs 蓝色 12h/≥4mm；红色大风 6h vs 蓝色 24h）。
- 误判案例：COE（attention magnitude/angle）分布上错样本更弥散、偏高值区；复合天气（雨+风）易把暴雨误报成大风。

### 明确局限
作者自述：NSW 绝对分仍很低（2.1/5），全国尺度结构化生成远未解决；误判集中在复杂气象场，需引入物理约束和多源数据。
我读出的局限：
1. **标签是 CMA 预警而非观测事实**——学的是"预报员会发什么预警"（含漏报/空报偏差），不是"实际发生了什么"；没有任何事件验证环节。
2. **无真正的预报提前量**：输入是预警发布后 [t, t+11] 的 ERA5 再分析（事件同期的准真值场），本质是"事后诊断/看图说预警"而非提前预报；业务化需换 NWP 预报场，性能必然再降，论文未测。
3. **非 SCS 专用**：无雷暴/闪电/龙卷/短时强降水类别；"Gale"混合对流性大风与系统性大风；冰雹仅 2.71% 且作者承认 0.25° ERA5 分辨率难以捕捉。
4. MC-sub Macro-F1 仅 29.31——严重度等级判别基本未解决。
5. 仅 2023/2024 两年一次切分；NSW 依赖 GPT-4o 判分（作者自引了 LLM-judge 自偏好文献但未做人工校验）。
6. 代码数据"将公开"，投稿时未释出。
7. 主表 Baseline（3 通道）vs MMLM（185 通道+模块）混合了通道数与模块两个因素；模块的净贡献只在 5,000 子集消融里分离，两表数字不可直接比。

### 跟我们的关系
对 **0713-scs-benchmark**（SCS 环境场 benchmark 课题）：
- **撞车评估——中度撞车，必须绕开**："大规模 ERA5 4D 场 × 中国灾害天气预警文本 benchmark + MLLM QA"这个 claim 已被 MP-Bench 占住（421k 对、7 类、4 种 QA、12h 无平均对齐）。若我们的 benchmark 也是"ERA5 配预警文本给 LLM 做 QA"，novelty 直接死。
- **幸存的差异化轴**（我们该占的位）：(a) **SCS 专用类别**——雷暴大风/冰雹/龙卷/短时强降水，MP-Bench 完全没有对流/非对流区分；(b) **观测事实标签**（强对流天气实况报告）而非预警文本——测"发生了什么"不是"预报员发了什么"；(c) **真实提前量**——用事件前环境场（如 [t−12, t)）构造 pre-storm environment 任务，MP-Bench 用事件同期场、零提前量；(d) **物理/环境指标维度**——CAPE/shear/因果指标作为 benchmark 的诊断轴，MP-Bench 是端到端黑盒+事后权重可视化；(e) **预报检验指标**——POD/FAR/CSI 等气象界标准检验，MP-Bench 只有 Acc/F1/LLM-judge。
- **可直接借用的工具/方法**：12h 逐小时无平均对齐策略；2h 同站合并清洗规则；年际切分协议（2023 train/2024 test）；DTGF/TGS/TGCA 公式级可复现，可作我们 benchmark 的 baseline 模块；NOAA 跨区域泛化协议；其 Table 1 的数据集对比图谱（CrisisLex/ClimaBench/Climate-FEVER/GridRad-Severe/SEVIR/HR-Extreme/WeatherQA/ClimateIQA/CLLMate/OmniEarth-Bench）可直接用作我们的 related-datasets 起点。
- **对 causal-scs-indicator 的旁支价值**：DTGF/TGCA 的权重分析本质是 attention 式时间/通道归因（500 hPa v、950 hPa u、300 hPa T 等），是"4D 时间特征携带预警信息"的正面证据；我们的因果指标（LKIF 信息流）可把这种 attention 级证据升级为因果级，并检验其物理一致性结论是否稳健。
- **该引**：任何 SCS/severe-weather benchmark 提案的 must-cite 最近邻 prior work。
