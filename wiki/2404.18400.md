# LLM-SR: Scientific Equation Discovery via Programming with Large Language Models (2404.18400)

- **Year**: 2024.04 (ICLR 2025)
- **Authors**: Parshin Shojaee (Virginia Tech), Kazem Meidani (CMU), Shashank Gupta (AI2), Amir Barati Farimani (CMU), Chandan K. Reddy (Virginia Tech)
- **Venue**: ICLR 2025
- **Keywords**: symbolic regression, LLM, equation discovery, evolutionary search, program synthesis, scientific prior knowledge, equation-as-program, experience buffer
- **One-liner**: 用 LLM 生成方程骨架程序 + 外部优化器拟合参数 + 岛屿模型进化搜索，在自定义科学 benchmark 上大幅超越传统 SR 方法，尤其 OOD 泛化。

## 精华提炼

### 管线/方法

LLM-SR 的核心管线分三步循环迭代：

**Step 1: Hypothesis Generation（假设生成）**
- 输入：结构化 prompt = Instruction + Problem Specification（自然语言描述问题和变量物理含义）+ Evaluation/Optimization Function + Experience Demonstration（从 buffer 采样的 k=2 个 in-context examples）
- LLM（GPT-3.5-turbo 或 Mixtral-8x7B）在 temperature τ=0.8 下一次采样 b=4 个方程骨架程序
- 方程表示为 Python 程序：`def f(x, params): ... return y`，其中 params 是占位符参数向量（最多 10 维）
- 程序执行检查：超时（30s）或崩溃的程序直接丢弃

**Step 2: Hypothesis Optimization & Assessment（假设优化与评估）**
- 将方程发现解耦为两阶段：(i) LLM 搜索离散的方程结构（骨架），(ii) 外部优化器优化连续参数
- 两种优化器：numpy+BFGS（scipy，适合少参数）和 torch+Adam（PyTorch，适合大规模/可微编程）
- 评分函数：Score = -MSE(y_pred, y_true)，即负均方误差

**Step 3: Experience Management（经验管理）**
- 岛屿模型（Islands Model）：m=10 个独立岛屿，每个岛屿维护一个方程种群
- 新假设加入其源岛屿（采样 prompt 的那个岛屿），仅当 score 优于当前最佳时
- 岛屿内按 score 聚类，保持多样性
- 定期重置（每 4 小时）：最差的 m/2 个岛屿被丢弃，用存活岛屿的最佳程序重新初始化
- 采样策略：(a) 均匀选岛 → (b) Boltzmann 采样选簇（偏好高分）→ (c) 指数采样选程序（偏好短程序）

**关键设计决策：**
- 方程作为程序（而非表达式树/前缀序列）：允许多行逻辑、if-else 条件（如 sigmoid 平滑近似）、step-by-step 推理注释
- 骨架+优化器分离：LLM 只搜离散结构，连续参数交给可靠优化器，解决了 LLM 不擅长精确数值的问题
- 自然语言问题描述注入先验：让 LLM 利用其训练数据中的科学知识

### 关键技术细节

- **LLM backbone**: GPT-3.5-turbo (OpenAI API) 和 Mixtral-8x7B-Instruct (4× NVIDIA RTX 8000 48GB)
- **搜索预算**: ~2.5K 迭代（对比 SR baseline 的 2M+ 迭代），每次 4 个采样
- **温度**: τ=0.8，平衡探索与利用
- **岛屿数**: m=10，Boltzmann 温度 T₀=0.1，N=10000
- **in-context examples**: k=2 个 best-shot
- **参数向量上限**: 10 维
- **执行超时**: 30s per program，内存上限 2GB
- **并行评估器**: e=4 个并行
- **重置周期**: 每 4 小时重置最差 m/2 岛屿
- **Loss 函数**: MSE (Normalized MSE 用于报告)
- **GPT-4o 额外实验**: Oscillation 1 OOD: 6.75e-5, Oscillation 2 ID: 4.27e-10（最好）

**主结果表 (NMSE, 越低越好):**

**关键发现：**
1. **OOD 优势显著**：E.coli 问题 LLM-SR OOD NMSE ~0.004 vs baseline >1.0，差距 250 倍+
2. **搜索效率**: 2.5K 迭代 vs 2M+，少 3 个数量级的搜索量
3. **LMX/FunSearch 对比**: 这两个 LLM 优化框架在 SR 上远不如 LLM-SR（Osc1 OOD: LMX 48.93 vs LLM-SR 0.0005）
4. **消融实验关键排序**: w/o skeleton+optimizer > w/o iterative refinement > w/o prior > w/o program（从影响最大到最小）
5. **噪声鲁棒性**: 在 σ=0.05 高斯噪声下 LLM-SR 仍远优于 PySR，先验知识对噪声数据尤其重要
6. **Feynman benchmark 问题**: LLM 对 Feynman 方程有严重记忆化（perplexity 3.1 vs 新 benchmark 5-7），几乎单 pass 就回忆出精确公式
7. **发现方程可解释性强**: LLM-SR 输出的方程带有 driving/damping/restoring force 等物理注释，baseline 的方程没有物理意义

**Surprising finding:**
- Oscillation 2: LLM-SR 发现的最终方程 `5(1-exp(x))` 与真实方程 `-5x·exp(0.5x)` 在数据范围 x∈(-2,2) 内近似等价——LLM 找到了一个等效但形式不同的表达式
- Stress-strain 问题：LLM-SR 利用 sigmoid 函数平滑近似 if-else 条件来建模分段非线性行为，这是传统表达式树 SR 做不到的

### 明确局限

**作者明确提到的：**
1. 性能与 LLM 训练数据的质量和覆盖范围绑定，某些科学领域可能存在偏差或空白
2. 迭代 LLM 查询和参数优化的计算成本对大规模问题可能过高
3. numpy+BFGS 优于 torch+Adam 主要是因为 LLM 更擅长生成 numpy 代码，而非优化方法本身的优劣
4. Benchmark 只有 4 个问题（2 个振荡器 + 1 个细菌生长 + 1 个应力-应变），规模偏小

**我读出来的局限：**
5. 没有与 MCTS 类 SR 方法（如 SPL-MCTS, Sun et al. 2023）做直接实验对比，只在 Related Work 中提及
6. 没有测试 PDE 发现（只测了 ODE 和代数方程），无法评估对偏微分方程的适用性
7. 岛屿模型的超参数（m=10, 重置周期 4h, Boltzmann 温度等）没有系统调优分析
8. 没有与人类科学家的发现过程对比，也没有 human-in-the-loop 实验
9. 所有 benchmark 的 ground truth 方程都是已知的（人工设计的），没有在真正未知方程的开放问题上测试
10. 对 LLM 的科学先验质量没有量化分析——哪些先验有用、哪些有害不清楚

### 跟我们的关系

对 **llm-mcts-sr** 课题的关联分析：

**撞车评估：**
- LLM-SR 的核心思路（LLM 生成方程骨架 + 搜索优化）与 llm-mcts-sr 高度相关，但搜索策略不同：LLM-SR 用的是进化搜索（岛屿模型），而非 MCTS
- 如果 llm-mcts-sr 的核心 claim 是"用 MCTS 替代进化搜索来引导 LLM 进行方程发现"，则不撞车。LLM-SR 未涉及 MCTS
- 但如果 claim 只是"用 LLM 先验 + 搜索做 SR"，那 LLM-SR 已经很好地覆盖了这个思路

**可借鉴的关键设计：**
1. **骨架+优化器分离**：这是 LLM-SR 最重要的设计，证明 LLM 搜离散结构 + 外部优化器拟合连续参数远优于 LLM 端到端生成完整方程（消融差 3 个数量级）
2. **方程作为程序**：允许多行逻辑、条件分支、sigmoid 平滑近似，比表达式树表达力强
3. **自然语言问题描述注入先验**：简单但有效，消融显示 w/o prior 性能显著下降
4. **自定义 benchmark 设计方法**：修改已知物理方程避免 LLM 记忆化，perplexity 验证有效

**差异化方向：**
- MCTS 可以提供更结构化的探索/利用权衡，可能优于随机岛屿进化
- MCTS 的 tree policy (UCT) 可以比 Boltzmann 采样提供更系统的搜索引导
- 可以将 LLM 的 rollout 与 MCTS 的 value estimation 结合
- LLM-SR 的 benchmark 可以直接复用（他们的代码和数据都已开源）

**必须引用此论文。** 它是 LLM 用于 SR 的旗舰工作（ICLR 2025, 93 citations），任何做 LLM+SR 的论文都需要对比。

## Read by: llm-mcts-sr (2026-06-05)

本次审读从 llm-mcts-sr（LLM + MCTS 用于符号回归）视角分析：

**核心发现：** LLM-SR 是 LLM 用于 SR 的代表性工作（ICLR 2025），验证了三个关键设计选择对 llm-mcts-sr 课题至关重要：
1. 骨架+优化器分离（消融显示比端到端好 1000 倍）
2. 方程作为程序（允许条件逻辑和多步推理）
3. 自然语言先验注入（尤其提升 OOD 泛化）

**与 llm-mcts-sr 的关键差异：** LLM-SR 使用岛屿模型进化搜索，搜索策略是 Boltzmann 采样 + 随机岛屿选择，这是一种相对松散的探索策略。MCTS 可以通过 UCT 提供更原则化的探索/利用权衡。LLM-SR 引用了 SPL-MCTS (Sun et al. 2023 ICLR) 但没有直接对比。

**差异化空间：** (1) MCTS 的 tree-structured search 可以系统地记录和复用搜索历史，而 LLM-SR 的岛屿模型只保留最佳方程; (2) MCTS 可以对方程空间做分层搜索（先选大类结构再细化），而 LLM-SR 是 flat 的; (3) MCTS 的 rollout 可以用 LLM 来做，value estimation 可以结合 data fitness。
