## 精华提炼

### 管线/方法

本文是综述，不是提出新方法。它构建了一个全面的分类体系：

1. **Forward Modeling 三大范式**：
   - **Data-driven Surrogates**（数据驱动代理模型）：纯粹从数据学习 PDE 解映射，分为依赖离散化（structured grid / irregular mesh / Lagrangian particles）和独立于离散化（DeepONet 系列 / Physical Space 算子 / Fourier Space 算子如 FNO）
   - **Physics-driven Surrogates**（物理驱动代理模型）：在训练中嵌入物理知识，分为 PINN（物理约束在 loss 中）和 Constraint-Informed（离散化约束嵌入网络结构）
   - **ML-assisted Numerical Solutions**（ML 辅助数值解）：用 ML 替换传统 solver 的部分组件，包括粗尺度模拟、预处理器学习、超分辨率等杂项

2. **Inverse Design & Control**：
   - 逆设计：PDE-constrained（hPINN, gPINN, Bi-PINN）vs Data-driven（先训 surrogate 再 backprop + 直接逆向映射如 INN/cINN）
   - 控制：Supervised Learning / RL / PDE-constrained (Control PINNs)

3. **Applications 七大领域**：Aerodynamics, Combustion, Atmosphere & Ocean, Biology Fluid, Plasma, Symbolic Regression, Reduced Order Modeling

### 关键技术细节

- **覆盖时段**：2020-2024，聚焦最近五年
- **文献量**：系统梳理了约 100+ 篇论文，分在各级分类下
- **Benchmark 数据集**：总结 21 种 PDE 和 13 种流体问题（含 DeepXDE、PDEBench、PINNacle、BLASTNet、AirFRANS）
- **核心模型家族**：
  - CNN 类：DPUF, TF-Net, EquNet, Phygeonet, PeRCNN
  - GNN 类：GNS (ICML 2020), MGN (ICLR 2021), MP-PDE (ICLR 2022), BSMS-GNN, CARE (NeurIPS 2023), BENO (ICLR 2024)
  - Fourier 类：FNO (ICLR 2021) → PINO, GEO-FNO, F-FNO (ICLR 2023), CFNO (ICLR 2023), GINO (NeurIPS 2023), FCG-NO (ICML 2024)
  - DeepONet 类：DeepONet → PI-DeepONet, MIONet, NOMAD (NeurIPS 2022), Shift-DeepONet (ICLR 2023), HyperDeepONet (ICLR 2023)
  - PINN 类：PINN-SR (Nat. Commun.), NSFnet, Meta-Auto-Decoder (NeurIPS 2022), PINNsFormer (ICLR 2024), DATS (ICLR 2024)
  - Transformer 类：GNOT (ICML 2023), ICON (PNAS), FactFormer (NeurIPS 2023), Transolver (ICML 2024)
  - Constraint-Informed 类：DGM, EDNN, INSR (ICML 2023), PPNN, **FEX** (Tree-based, Arxiv 2024)

- **与 FEX 直接相关**：
  - FEX（Finite Expression）在分类体系中出现在两个位置：
    1. **Physics-driven Surrogates → Constraint-Informed**（Table `tab:method_physics`），标注 backbone 为 "Tree"，scenarios 为 Hindmarsh-Rose, FHN
    2. **Applications → Symbolic Regression** 中作为代表性工作被讨论
  - FEX 在 Constraint-Informed 类别中被描述为 "represents dynamics on complex networks using binary trees composed of finite mathematical operators with minor prior knowledge"
  - FEX 在 Symbolic Regression 上下文中被描述为 "uses deep reinforcement learning to approximate the governing PDEs with symbolic trees"

### 核心发现

- **ML 正在显著改变 CFD 研究**：从纯数据驱动到物理嵌入到混合 solver 的多条路线并行推进
- **Fourier 空间算子是主流**：FNO 及其变体是最活跃的子方向，ICLR/ICML/NeurIPS 顶会密集出现
- **从"替代 solver"到"增强 solver"的转变**：纯粹端到端 surrogate 在长时间 rollout 中误差累积严重，ML-assisted numerical solutions（粗尺度 + 学习修正）是务实路线
- **科学基础模型（Scientific Foundation Model）是终极目标**：PROSE、ICON、Unisolver、OmniArch、DPOT、MPP 等都在探索跨多物理系统的通用 PDE 求解能力
- **自动科学发现（Auto-lab）闭环**：用 surrogate 做 verifier，用 ML 做 proposer，过滤后做真实实验/高精度仿真，反馈丰富数据集——这个 pipeline 在材料科学已经有效，CFD 是自然延伸
- **扩散模型进入逆设计和控制**：CinDM、SMDP 等方法展示扩散模型在高维设计空间采样的优势

### 与 FEX 关系的关键点

本文并非 FEX 的 primary source。FEX 的方法论文是：
- `song2024finite`（FEX 在 Constraint-Informed 场景，Arxiv 2024）：用 binary tree + finite mathematical operators 表示复杂网络动力学
- `jiang2023finite`（FEX 在 Symbolic Regression 场景）：用 deep RL 逼近 PDE 的符号树表示

本文作为综述为 FEX 提供了 landscape 定位：
- FEX 在分类中属于 **Physics-driven Surrogates → Constraint-Informed** —— 即通过离散化约束将物理知识嵌入网络结构
- 同时 FEX 也是 **Symbolic Regression** 方法的一种——用于从未知数据中恢复控制方程
- FEX 的独特卖点是 **可解释性**：输出符号树而非黑箱数值，这对科学发现场景极为重要

### 明确局限

- **综述本身局限**：
  - 覆盖截至 2024 年中，缺乏 2024 年下半年至 2025 年的最新进展
  - 缺少定量 Benchmark 对比——只分类列举，不给性能比较表
  - 对 FEX 的讨论较浅，仅一两句话，没有深入分析其与 PINN、SINDy 的区别
  - 对 RL-based symbolic regression（如 FEX 的核心技术）的方法学讨论不够
  - 中文作者团队，语言质量参差不齐（有少量语法错误和拼写错误如 "Advectionantageous"、"Reynold" 应为 "Reynolds"）

- **FEX 相关局限**（从综述视角看）：
  - FEX 的实验仅在 Hindmarsh-Rose 和 FHN（低维神经动力学方程）上，尚未在真实 CFD 场景（如 Navier-Stokes）上验证
  - FEX 与 PINN-SR、SINDy 等 symbolic regression 方法的定量对比缺失
  - FEX 的 tree 表示是否可扩展到高维 PDE（3D NS 方程）未知

**撞车评估**：无直接撞车风险。综述本身不提出新方法。FEX 的方法论文 (song2024finite, jiang2023finite) 需要单独精读。

**FEX 的 landscape 定位**：
- FEX 属于小众但有特色的方向——Constraint-Informed 类别中大部分方法（DGM, EDNN, PPNN）输出的是数值解，而 FEX 输出的是符号表达式
- FEX 与 PINN-SR、SINDy 构成直接竞争：都是 "从数据中恢复方程"，但 FEX 使用 tree + RL，PINN-SR 使用 PINN + sparse regression，SINDy 使用 LASSO
