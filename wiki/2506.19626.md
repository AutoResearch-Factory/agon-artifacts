# Scaling Up Unbiased Search-based Symbolic Regression (2506.19626)

- **Year**: 2024 (IJCAI)
- **Authors**: Paul Kahlmeyer, Joachim Giesen, Michael Habeck, Henrik Voigt (Friedrich Schiller University Jena / University Hospital Jena)
- **Venue**: IJCAI 2024
- **Keywords**: symbolic regression, expression DAG, unbiased search, variable augmentation, Pareto front, subexpression recovery, SRBench
- **One-liner**: 用 expression DAG 的无偏随机搜索 + 变量增强技术（variable augmentation），在 SRBench 上超越所有 SOTA 符号回归方法，恢复率和鲁棒性均为最高。

## 精华提炼

### 管线/方法

完整 pipeline（对应 Figure 3 workflow）：

1. **输入**：一个回归问题，即带标签的训练数据 `(x_i, y_i)`，n 个输入变量，m 个输出变量。
2. **选取变量增强（Variable Augmentation）**：
   - 用无偏 DAG 搜索（Section 4）在小空间（无参数节点 p=0）中采样潜在变量增强表达式 `z_Δ`。
   - 用标准回归器族 F（如多项式回归）在增强数据 `(x_i, z_i, y_i)` 上拟合，计算 `1 - R^2` 作为 scoring function。
   - 选 top-k 个最好的增强变量。
3. **求解增强后的回归问题**：对每个选中的变量增强，用 UDFS (Section 4) 搜索 DAG frame。
   - DAG 参数 p=1（单个参数节点），intermediary nodes i=5，最多 200,000 个 DAG skeletons。
4. **Pareto 前沿筛选**：对每个增强问题的 regressor，计算 R²（模型拟合）和 complexity（DAG 节点数），保留 Pareto 前沿上的模型。
5. **最终输出**：Pareto 前沿上 complexity 低于阈值的 best-fitting 模型；若无，则返回最小模型。

### 关键技术细节

**Expression DAG 表示**：
- 4 种节点：variable nodes（输入变量）、parameter nodes（可优化常数）、intermediary nodes（中间运算）、output nodes（输出）。
- 输入节点是 DAG 的 root，输出节点是 leaf。只有 intermediary nodes 有 incoming + outgoing edges。
- 支持 binary operators: `+, -, ×, ÷`；unary operators: `-, ⁻¹, sin, cos, log, exp, ², √`。
- DAG 比 expression tree 更紧凑：common subexpression 被共享，例如 `x²(x²+x)` 的 tree 有 4 个 operator nodes，DAG 只有 3 个。

**DAG 搜索空间参数化**：
- 由 `(p, i)` 定义：p 是 parameter nodes 数量（固定 p=1），i 是 intermediary nodes 数量（实验中 i=5）。
- 搜索空间大小递归公式：`S_n(0) = n(n+1)`，`S_n(i) = S_n(i-1)(n+i)(n+i+1)`，随 i 指数增长。

**两阶段搜索**：
1. **采样 DAG skeleton**：按拓扑序编号 intermediary nodes 1..i；
   - Unary output/intermediary node：从合法前驱中 uniform random 选 1 个
   - Binary node：从合法前驱中 uniform random 选 1 对
   - 删除无后继的 intermediary nodes（dead nodes）。
2. **穷举 operator labeling**：对每个 skeleton，穷举所有可能的二元/一元 operator 组合（共 `|binary_ops|^b × |unary_ops|^u` 种）。

**参数优化**：
- 不调 BFGS 等外部优化器（太慢），而是用两级 hierarchical grid search。
- Grid size=50，两级从 `[-1, 1]` 起始。可并行评估整个 grid 的单次前向传播。
- 大的常数可通过运算生成（如 `e^c` 或 `c²`），所以 grid 限制不构成实际障碍。

**超参数设置**（默认）：
- 5 个 intermediary nodes
- 最多 200,000 DAG skeletons
- 1 个 parameter node
- 变量增强用 polynomial regression，k=1 个增强，增强问题 DAG 搜索最多 30 nodes
- 平台：Intel Xeon Gold 6226R 64-core, 128 GB RAM, Python 3.10

### 核心结果

**SRBench 结果（130 ground-truth 问题）**：
- **Recovery rate**：UDFS + Aug 最高，显著超过第二名 AIFeynman。DAG（无增强）已经排第三。
- **Robustness（噪声鲁棒性）**：在所有噪声级别（除最高档），UDFS + Aug recovery rate 最高。噪声增大时 UDFS + Aug 与 DAG bare 的差距缩小（噪声使增强更难找到）。
- **Model complexity**：UDFS + Aug 和 DAG 都产出最简洁的模型（simplified complexity 最低），而 AIFeynman 在不能 recovery 时会输出又大又差的模型。
- **R² test**：UDFS + Aug 中位数接近 1.0，与其他 top regressors 持平。
- **Pareto front**：UDFS 和 UDFS + Aug 都在第一 Pareto 前沿上（complexity vs R² rank）。

**Jaccard 指数（子表达式部分恢复）**：
- 论文提出 Jaccard index 作为软性恢复度量——不要求完整符号等价，只看子表达式集合的交并比。
- 在 Jaccard 度量下 DSR 超过 AIFeynman（DSR 更鲁棒），但 DAG 系列仍然最高。

**可扩展性实验**：
- 随 intermediary nodes i 和 skeletons 数 s 增加，recovery 单调提升。
- i=1→4，s=10k→80k：Feynman 上 recovery 从 0.11→0.43。
- Nguyen-2 (`x⁴+x³+x²+x`) 能被 DAG 解决因为其紧凑 DAG 表示（3 intermediary nodes 重用 x² 和 x²+x）。
- Nguyen-6 (`sin(x)+sin(x²+x)`) 需要 4 intermediary nodes，在 200k skeletons 下概率不足，未恢复。
- Variable augmentation 对 Nguyen-4 和 Nguyen-12（需要更多 intermediary nodes 的）有决定性帮助。

**与 ESR 对比**：
- DAG + Aug: 100% Univ recovery, DAG bare: 80%, ESR: 50%
- 时间：DAG + Aug 26.12s, DAG 388.82s, ESR 1293.63s

**Runtime**：
- 默认设置（50k skeletons, i=3, p=1, 10 parallel）下，runtime 随 i 指数增长，随 p 多项式增长，随骨架数线性增长（直到覆盖整个搜索空间后饱和）。

### 明确局限

1. **不适用于非常深的表达式**：搜索空间随 i 指数增长，即使有 variable augmentation，深层嵌套的表达式也超出可达范围。i=5 是实验上限。
2. **依赖随机采样**：DAG skeleton 采样是随机的，不能保证覆盖到目标表达式的 skeleton（如 Nguyen-6 未能恢复）。
3. **参数优化简化**：Hierarchical grid search 代替非线性优化，对需要精确常数的表达式可能不够精确（虽然论文论证说可以通过运算生成大常数）。
4. **仅单参数节点**：p=1 限制了直接表示多个独立常数的能力（但论文说可通过函数组合补偿）。
5. **仅限给定 operator set**：不支持的运算（如 tanh, erf, Bessel functions 等）无法表示。
6. **无真机实验**：纯 benchmark 评估，没有在真实物理/工程问题上验证发现的表达式是否有科学意义。
7. **Variable augmentation 依赖标准回归器族**：如果 F（如多项式回归）不适合目标问题，augmentation scoring 会失效。
8. **没有与 LLM-based SR 方法对比**：论文投稿时间是 2024 IJCAI，未涉及 2024 年后出现的基于 LLM 的符号回归方法。

## Read by: 0616-fex (2026-06-25)

这篇论文提出 UDFS（Unbiased DAG Frame Search），结合 expression DAG 的无偏随机搜索和 variable augmentation 技术，在 SRBench 130 个 ground-truth 问题上达到了最高的符号恢复率和最强的噪声鲁棒性。

对 0616-fex 课题的核心启示：
1. **Expression DAG 作为紧凑表示**：如果 [0616-fex] 需要表示/搜索数学公式或特征表达式，DAG 比 tree 更优——子表达式复用天然内置。
2. **Variable augmentation 即特征提取**：这个思路直接对应特征工程——先搜索有信息量的子表达式（特征），增强输入空间，再在主任务上搜索。这本质上是一种 "symbolic feature extraction"。
3. **Pareto 前沿选择**：不预设复杂度-精度 trade-off，保留所有 Pareto 最优解——这种策略适合探索性科学研究场景。
4. **需要注意的 gap**：本文的实验局限于 SRBench 的已知 ground-truth 问题上，没有展示在真实科学发现场景下的表现。如果 [0616-fex] 面向真实应用，这一点需要额外验证。
