# Guiding Multi-Objective Genetic Programming with Description Length Improves Symbolic Regression Solutions (2605.22374)

- **Year**: 2026.05
- **Authors**: Gabriel Kronberger (University of Applied Sciences Upper Austria), Fabricio Olivetti de Franca (Federal University of ABC), Deaglan J. Bartlett (University of Oxford), Harry Desmond (University of Portsmouth), Pedro G. Ferreira (University of Oxford)
- **Venue**: arXiv
- **Keywords**: symbolic regression, genetic programming, MDL, description length, fractional Bayes factor, BIC_SR, NSGA-II, model selection
- **One-liner**: 用DL/FBF从GP帕累托前沿选短而稳的符号回归式。

## 精华提炼

### 管线/方法
- 论文研究的问题是：GPSR 在有噪声数据上容易过拟合和 bloat，最终从一堆候选表达式里选哪个模型很难。
- 输入是一组回归数据 `{x_i, y_i}`，假设误差是 i.i.d. Gaussian，噪声方差未知。
- 每个候选表达式 `f(x; theta)` 都先用 Levenberg-Marquardt 做非线性最小二乘参数优化。
- 噪声方差 `sigma_err^2` 不外部给定，而是由候选模型残差的 MSE 估计。
- 所有模型选择准则都基于同一个 Gaussian negative log likelihood (NLL)。
- 比较的准则包括 AIC、BIC、BIC_SR、DL 和 FBF。
- AIC 只罚参数数：`2*NLL + 2p`。
- BIC 只罚参数数但更强：`2*NLL + p log m`。
- BIC_SR 是作者加的结构复杂度版本：`2*NLL + 2*func_comp + p log m`。
- DL 是 `NLL + function complexity + parameter complexity`。
- FBF 是 fractional Bayes factor 近似，使用 `b=m^{-1/2}`，共享同一个 `func_comp` 结构先验。
- `func_comp = k log n + sum log c_i`，其中 `k` 是表达式节点数，`n` 是表达式内 distinct symbols 数。
- 重要实现细节：作者不把 fitted parameter symbols 和 constants 计入 `k log n`，常数用 `sum log c_i`，参数由 parameter complexity 处理。
- DL 的 parameter complexity 来自 observed Fisher information matrix (FIM)。
- 与 Bartlett ESR 早期做法不同，这篇不只用 FIM 对角项，而是算 full FIM 并做 SVD。
- SVD 后把 MLE 参数旋转到近似独立方向，用 FIM eigenvalues 表示 stiff/sloppy directions。
- 参数复杂度项是 `1/2 sum max(0, log S_ii - log 3 + log |theta_rot_i|)`。
- full FIM + SVD 的代价是 `O(p^3)`，但实验里参数数通过表达式长度间接限制，通常最高约 50。
- 作者测试三种搜索/选择策略。
- 第一种是 MO-Length：NSGA-II 风格多目标 GP，同时优化 NLL 和 program length，再用 AIC/BIC/BIC_SR/DL/FBF 从 Pareto front 选一个模型。
- 第二种是 MO-DL：仍多目标，但第二目标从 expression length 换成 DL penalty，即 function complexity + parameter complexity。
- 第三种是 SO：直接把 AIC/BIC/BIC_SR/DL/FBF 当 single-objective fitness 最小化。
- 三种策略的参数拟合步骤一致，差别只在 GP 选择压力和最终 model selection。
- 论文的核心建议不是“把 DL 当 fitness”，而是“先保留 accuracy/length 多样性，再用 DL/FBF/BIC_SR 选模型”。

### 关键技术细节
- GP population size 是 1000。
- tournament selection group size 是 2。
- 大多数 synthetic、SRBench、Nikuradse 数据集跑 200 generations。
- Friction 和 Tower 数据集跑 400 generations。
- crossover probability 是 90%。
- mutation 是 10% individuals，且每个 node 5%。
- 初始化是 grow method，depth limit 5。
- 最大 code length：Synthetic 和 SRBench 为 100。
- 最大 code length：Nikuradse 和 Friction 为 200。
- 最大 code length：Tower 和 Chemical Competition 为 300。
- function set 是 `{+, -, *, div, x^2, sin, cos, exp, log|x|, sqrt|x|, |x|^y}`。
- terminal set 是 `{var, param, var * param}`。
- 多目标算法是 NSGA-II：non-dominated sorting + crowding distance + elitist generational replacement。
- evaluation 时先抽取表达式里的参数初值，再用 LM/NLL 优化，最后把优化后的参数写回 expression。
- 表达式长度定义为 tree nodes，包括 operators、variables、fitting parameters。
- 统计比较不用 NHST，而用 Bayesian Bradley-Terry (BBT) 模型。
- BBT 比较的是成对 selection criterion 在同一个 Pareto front 中选出的模型 test MSE 谁更好。
- ROPE 设为 `45%-55%`，`dagger` 表示 99% 以上 posterior mass 在 ROPE 外且大于 50%。
- 训练/测试规模：synthetic problem 每个 setting 采样 100 个 dataset 或 noise realization。
- SRBench 多数数据集做 100 个 random splits，训练 2/3、测试 1/3。
- 大型 SRBench 数据集 `1193 BNG lowbwt` 和 `1199 BNG echo months` 每次采 1000 条训练，剩余测试。
- 工程数据包括 Nikuradse、Friction、Chemical Tower、Chemical Competition。
- Salustowicz 示例：训练 100 个 grid points，范围 `[0.05, 10]`，测试 220 点，范围 `[-0.5, 10.5]`。
- Salustowicz 加 10% noise，`sigma_err = 0.033`，测试集无噪声。
- Friedman-2 noise study：训练集 100 observations，10 维输入，生成表达式长度 22。
- Friedman-1 sample-size study：`n=50..1000`，生成表达式长度 24。

### 核心结果
- MO-Length 上，BIC_SR、DL、FBF 的行为非常接近，明显优于 AIC/BIC 的过拟合倾向。
- MO-Length test MSE 的 BBT：`P(DL > AIC)=88.8%`，显著。
- MO-Length test MSE 的 BBT：`P(DL > BIC)=73.0%`，显著。
- MO-Length test MSE 的 BBT：`P(DL > BIC_SR)=51.8%`，在 ROPE 内，近似等价。
- MO-Length test MSE 的 BBT：`P(DL > FBF)=51.7%`，在 ROPE 内，近似等价。
- BIC_SR、DL、FBF 都包含 function complexity penalty，因此比 AIC/BIC 选出更短的表达式。
- AIC 经常选最长表达式，BIC 比 AIC 稍好但仍常过拟合。
- Salustowicz 上，AIC 在 10-20 generations 后开始明显过拟合。
- Salustowicz 上，BIC 大约 70 generations 后也开始过拟合。
- Salustowicz 上，BIC_SR/DL/FBF 的 training RMSE 收敛到已知 noise threshold 附近。
- Salustowicz final Pareto front 中，function-complexity criteria 在 expression length 约 23 附近形成清晰 minimum。
- 该 length 约等于 Salustowicz 生成表达式长度，但 best test-error model 平均略长，约 30。
- Nikuradse 2D 上，AIC/BIC 选出的模型有明显不稳定 extrapolation；DL/FBF 模型 in-sample 无 poles，外推更合理。
- Friedman-2 noise sweep 上，MO-Length+DL 会随 noise 增大自动选更短表达式。
- Friedman-2 低噪声时，DL 选出的表达式长度约 20-30，接近生成表达式长度 22。
- Friedman-1 sample-size sweep 上，小 `n` 时 DL 更偏短表达式，`n>=100` 才较稳定接近 irreducible error。
- MO-DL 能找到更低 DL 的最终表达式，`P(MO-DL > MO-Length+DL)=89.1%`。
- 但 MO-DL 的 test RMSE 与 MO-Length+DL practically equivalent：`P=50.2%`，HPD `48.9%-51.4%`。
- 这说明“把 DL penalty 放进第二目标”主要改善 DL 值，不明显改善泛化。
- SO 直接优化 BIC_SR/DL/FBF 往往失败：复杂度压力太早太强，diversity 下降，表达式过短，underfit。
- MO-Length+DL 相比 SO 的 win probabilities：对 SO-AIC 为 `79.5%`，SO-BIC 为 `61.8%`，SO-BIC_SR 为 `78.3%`，SO-FBF 为 `77.4%`，SO-DL 为 `76%`。
- SO-BIC 是 SO 里最接近 MO-Length+DL 的，因为它允许更长表达式，但在多个数据集上又会过拟合。
- Tower 是例外：AIC 表现最好，作者怀疑原因是该 multivariate time series 被 shuffle 后 train/test split，不满足 i.i.d. Gaussian likelihood 假设。
- 总体结论：DL、FBF、BIC_SR 是模型选择准则，而不是直接替代 GP fitness 的万能目标。

### 明确局限
- 只测试 Gaussian likelihood；分类、count data、heteroscedastic likelihood 都没测。
- DL 和 FBF 理论上可配任何 likelihood，但本文没有验证这些场景。
- function complexity 的具体形式会影响结果，`k log n` 的权重在个别数据集上可能过强或过弱。
- 作者没有给出对 function prior 的完整校准方法，只是比较一个经验上表现不错的设定。
- full FIM + SVD 的 DL 计算比 BIC_SR/FBF 更贵；对大量 Pareto-front 外候选计算不划算。
- MO-DL 虽然降低 DL，但不提升 test RMSE，因此额外计算成本未必值。
- 实验是作者自评估，没有独立复现实验或外部 benchmark report。
- 没看到随论文源码一起发布完整实验代码、随机种子和中间结果 archive；可复现性依赖论文中超参数与公共数据集说明。
- synthetic truth 已知，但 real-world datasets 仍只看 held-out prediction，不能证明学到真实机理。
- SRBench 子集被作者自己认为不适合评价复杂表达式恢复能力，因为多数数据集短表达式就会过拟合。
- 对于我们的方向，若需要 LLM/agent 生成候选公式，本文只处理 GP 生成的候选表达式，不覆盖自然语言到表达式、符号约束解析或人机审查环节。

## Read by: 0616-fex (2026-06-16)

这篇对 `0616-fex` 的主要价值是把“候选表达式生成”和“最终表达式选择”拆开：GP 负责产生 NLL/length Pareto front，DL/FBF/BIC_SR 负责后处理选模型。若我们的方向涉及 formula/expression discovery 或 noisy symbolic regression，这个拆分应当作为默认 baseline。

撞车点很明确：如果我们声称“用 description length 或 Bayesian-ish score 选择短而泛化的符号表达式”，这篇已经系统做了，而且覆盖 AIC/BIC/BIC_SR/DL/FBF、MO-Length/MO-DL/SO 三类策略和 34 个 problem variants。尤其不能再把“DL 直接作为 fitness 会更好”当新点；本文反而证明这常导致 premature convergence 和 underfitting。

可用方法也很清楚：BIC_SR 是便宜近似，几乎追平 DL/FBF；DL 的 FIM-SVD parameter complexity 可作为高成本 verifier；BBT 适合比较不同 selection rules 在同一 Pareto front 的表现。若我们要差异化，建议放在非 Gaussian likelihood、LLM/agent 候选生成、独立 verifier、有约束/物理先验、或自动校准 function prior 上。
