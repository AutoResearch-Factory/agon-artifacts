# Selftok: Discrete Visual Tokens of Autoregression, by Diffusion, and for Reasoning (2505.07538)

- **Year**: 2025.05
- **Authors**: Bohan Wang, Zhongqi Yue, Fengda Zhang, Shuo Chen, Li'an Bi, Junzhe Zhang, Xue Song, Kennard Yanting Chan, Jiachun Pan, Weijia Wu, Mingze Zhou, Wang Lin, Kaihang Pan, Saining Zhang, Liyu Jia, Wentao Hu (Selftok Team, Media Technology Institute, Huawei); Tech Lead: Hanwang Zhang; Project Lead: Wei Zhao
- **Venue**: arXiv
- **Keywords**: discrete visual tokenizer, autoregressive prior, reverse diffusion, visual RL, Bellman equation, policy improvement, VLM, text-to-image generation, image editing
- **One-liner**: 提出一种抛弃空间先验的离散视觉 tokenizer（Selftok），利用反向扩散过程的 AR 递归性将图像编码为因果自回归 token，从而在纯 AR 框架中统一扩散与自回归，并在视觉生成上首次实现与 LLM 同等有效的 RL post-training。

## 精华提炼

### 管线/方法

**整体管线**：输入图像 → Selftok Encoder（双流 MMDiT，图像流+token流）→ 量化器（lookup codebook 32768 words）→ K 个离散 token → 两种解码路径：(a) 多步扩散 Decoder 或 (b) 一步 Renderer → 重建图像。训练后的 Selftok 作为 VLM 的视觉 tokenizer，VLM（Llama3-8B 扩展 vocab）做纯离散 AR 训练，然后 GRPO-based RL 做 visual post-training。

**Selftok 训练**：
- Encoder：双流 MMDiT Transformer，图像流用预训练 SD3 VAE latent（8× 下采样，256×256→32×32, D=512），token 流用可学习的连续 token embedding（K×512）。Co-attention 中 token stream 可以 attend image stream，但反过来不行（节省计算）。Token-aware AdaLN 为每个 token index 学习 scale/shift 参数。
- 量化器：Bottleneck 512→16 维，cosine similarity 做最近邻查找，straight-through estimator 做梯度传播。Codebook 32768 words。量化 loss = commitment loss + entropy loss（促进均匀 codebook 使用）。EMA 更新 codebook（γ=0.8），dead-code reactivation（EMA 概率低于 0.0125/C 则随机重新赋值）。
- Decoder：SD3 MMDiT 架构初始化，token stream 从零训练，image stream AdaLN 去除 CLIP text embedding 依赖。目标函数：MSE ‖x₁ - Dec((1-t)·x₀+t·x₁, V_{≥k(t)})‖² + L_Q。
- Token schedule k(t)：自定义非均匀分配——小 t 分配更少 token（因为扩散早期路径对重建影响小）。
- 一步 Renderer：用 learnable "canvas" token 替换扩散时间步输入，从 decoder 权重初始化，联合 MSE+LPIPS+GAN loss 训练（MSE 30k iters 后用 GAN）。

**VLM 训练（两阶段）**：
- Stage1 跨模态对齐：4 种数据格式（Text-to-Image, Image-to-Text, Image-Only, Text-Only），530M 高质量 image-text 对，48 节点 384 张 Ascend 910B2，lr=1e-4→1e-5 cosine decay，200k iters。
- Stage2 跨任务对齐：SFT 在 text-to-image（252k 样本）、image editing（1.49M 样本）、visual comprehension 三个任务上。Llama3-8B 扩展 vocab +32768 视觉词。

**Visual RL 管线**：
- Reward Model 两类：(1) Program-based：MM-Detection 检测器做 object counting/position 检查，每个 prompt 有多项测试取平均；(2) QA-based：InternVL/GPT-4o 做 prompt 分解为语义 tuple→VQA 打分（0-1），多问题平均。
- Policy Gradient：简化版 GRPO（无 importance sampling），batch 内标准化 advantage，KL 正则化（π_old 作为 anchor）。
- 与 Diffusion DPO 的本质差异：Diffusion DPO 无法获取真正的生成轨迹（inversion 难），退化为 contrastive learning；而 Selftok 将完整扩散路径离散化为 state-action 轨迹。

### 关键技术细节

- **模型架构**：双流 MMDiT backbone，token 流和图像流各有独立参数。Encoder N blocks（未明确 block 数，推测与 SD3 一致）。
- **训练数据规模**：Tokenizer 在 ImageNet-1k train split (256×256 center-crop) 上训练。VLM pretrain 530M pairs，含 COYO、DataComp、LAION-5B 子集 + 自研数据。
- **超参数**：Codebook size C=32768 (2^15), token dim D'=16, temperature τ=0.1, EMA γ=0.8, dead-code η=0.99。Renderer λ₁=0.1→0.5, λ₂=0→0.5（30k iters 切换）。VLM lr=1e-4→1e-5, batch 7680 (pretrain), 32 (SFT T2I), 128 (SFT edit/VQA)。Logit adjustment: γ=10, τ=2.0 (T2I); γ=7.5, τ=2.0 (edit)。
- **Loss 函数**：Tokenizer = MSE reconstruction + commitment loss + entropy loss + EMA codebook。Renderer = MSE + 0.5*LPIPS + 0.5*GAN。VLM = CE loss。RL = -(advantage - λ*KL)。
- **RL 算法**：简化的 GRPO，无重要性采样。KL 用 π_old/π - log(π_old/π) - 1 估计。Program-based reward 使用 detection threshold 0.6。
- **真机/平台**：Huawei Ascend 910B2。Tokenizer 训练规模不详。VLM 在 48 nodes × 8 GPUs = 384 GPUs 上做 pretrain。
- **评测指标**：Tokenizer: rFID, PSNR, SSIM, LPIPS。T2I: GenEval（7 子任务）和 DPG-Bench（5 子任务）。Editing: Structure Distance, PSNR, LPIPS, MSE, SSIM, CLIP score。VQA: MME benchmark。

|---------|------------|------------|-------------|-------------|

| Counting | 58 | 66 | **90** (+24) | 59 |

|-----------|------------|-------------|

**Surprising 发现**：
1. Selftok-Zero **不使用任何 pairwise supervision**，仅靠 RL reward 在 GenEval 上从 74→95，超越所有现有模型（包括 GPT-4o 的 85 和 HiDream-I1 的 83）。
2. Position 子任务暴涨 45→98（+53 点），说明 RL 对空间关系的提升极为显著。
3. Program-based reward（GenEval +21）远优于 QA-based reward（DPG +3.77），说明结构化 reward 信号质量更高。
4. Janus-Pro（空间 token）做同样 RL 仅从 79→85（+6），而 Selftok 从 74→95（+21），验证了 AR 属性对 RL 的关键性。
5. Selftok token 在 comprehension + generation 双任务训练中始终表现出正 synergy（Δ>0），而空间 token 在某些数据点出现负 synergy（冲突）。

### 明确局限

**作者自己说的**：
1. LLM token 生成速度远慢于扩散模型——512 token/frame × 24fps × 60s = 737,280 tokens 对视频是巨大吞吐挑战。
2. 模型规模受限（Llama3-8B），尚未展示跨模态 emergent capabilities。
3. 分辨率仅 256×256（但展示了多分辨率 Selftok 的初步 512×512 结果）。
4. RL reward 模型设计还很初步（仅 program-based + QA-based），更有效的 reward 待探索。
5. Token schedule k(t) 的最优设计仍是 open problem。
6. 缺乏 standardized image editing 评测指标，只能依赖 human eval。
7. Logit adjustment 仍需手动调参（γ, τ），不够优雅。

**读出来的局限**：
1. **无标准 benchmark 以外的评测**：GenEval 和 DPG-Bench 是最常用的 T2I benchmark，但没有在 COCO FID、ImageReward、PickScore 等更广泛指标上验证。
2. **RL 实验缺乏 ablation**：未比较不同 RL 算法（PPO vs GRPO vs DPO），未做 reward model 选择的系统消融（仅区分了 program-based vs QA-based）。
3. **VLM 与纯 LLM 的视觉 RL 对比不够**：只在 GenEval/DPG 上比较，没有在更广泛的 VLM 能力（如 captioning、VQA）上验证 RL 是否有负面影响。
4. **没有开源代码/checkpoint**：paper 中的 project page 只有 report。
5. **计算成本未披露**：384 GPUs pretrain 的具体 GPU-hours 未报告。
6. **与 concurrent work 的比较不足**：很多 concurrent work（CaTok, UniFlow 等）在反引文中出现，论文缺少与它们的详细对比。

### 跟我们的关系

对 **0616-fex** 课题的关联分析：

**撞车风险**：
- Selftok 的核心贡献是 "AR visual tokenizer + RL for visual generation"，而 0616-fex 关注的是 feature extraction / search complexity。**不直接撞车**——Selftok 没有提出新的 feature extraction 方法，它的创新在 tokenizer 结构化方向。
- 但是，如果 0616-fex 涉及任何 "visual tokenization for downstream tasks" 或 "discrete representation learning"，Selftok 是一个必须引用的强 baseline。

**可用工具/方法**：
1. **AR 属性验证方法**：Section 2.1 的 token entropy curve 分析（用 shuffled orders 验证 AR 结构）可直接复用来验证其他 tokenizer 的因果性。
2. **Tokenizer 评估协议**：rFID + PSNR + SSIM + LPIPS 四维重建评估，可标准化用于任何 visual tokenizer 评测。
3. **Synergy metric Δ**：双任务训练的正负协同性量化方法（Δ = Performance_both - Performance_single），可泛化用于任何多任务 representation learning。
4. **Bellman equation 论证框架**：如果 0616-fex 做 RL-based 视觉搜索，Selftok 的 MDP 形式化（discrete state-action space → AR property → Bellman optimality）可复用为理论支撑。

**差异化方向**：
- 0616-fex 可以关注 **token 的信息论属性**（entropy, mutual information）而非 AR 属性，这与 Selftok 互补。
- 0616-fex 可以研究 **tokenizer 选择对 downstream search/retrieval 性能的影响**，这是 Selftok 未覆盖的方向。

本次审读从 **0616-fex**（feature extraction / search complexity）视角分析：

**撞车评估**：不撞车。Selftok 是一个 visual tokenizer + VLM + visual RL 的完整系统，其核心创新在于"用扩散过程赋予视觉 token 以自回归先验"。0616-fex 关注的是搜索和特征提取，与 Selftok 的具体技术路线无直接重叠。但如果 0616-fex 涉及 visual representation 或 discrete tokenization 作为中间表示，Selftok 是必引的方法。

**最可复用的工具**：
1. AR 属性验证协议（entropy curve + shuffled orders）——如果 0616-fex 需要验证任何序列化表示的因果结构。
2. Synergy metric Δ——量化多任务训练正负协同性，在评估 representation learning 是否泛化时有通用价值。
3. Tokenizer 四维评估（rFID/PSNR/SSIM/LPIPS）——标准化评测任何 visual compression 方法。

**需警惕**：反引文中 CaTok (2603.06449) 和 (1D) Ordered Tokens (2604.15453) 是 2026 年新工作，如果 0616-fex 做 1D tokenization + search，这两个是直接竞争对手/基线。

**差异化**：0616-fex 可以关注 tokenizer 的信息论性质（如每 token 的 entropy、mutual information with query），而 Selftok 只关注了 AR 结构下的预测 entropy 下降趋势。
