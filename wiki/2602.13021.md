- **Year**: 2026.02
- **Authors**: Jing Xiao, Xinhai Chen, Jiaming Peng, Qinglin Wang, Menghan Jia, Zhiquan Lai, Guangping Yu, Dongsheng Li, Tiejun Li, Jie Liu (National University of Defense Technology)
- **Venue**: ICML 2026 (submitted, uses icml2026 style)
- **Keywords**: Symbolic Regression, Prior Constraints, Pseudo-Equation Trap, Scientific Discovery, LLM-Guided Search, Rademacher Complexity, PACE Mechanism
- **One-liner**: 提出 PG-SR，通过显式编码领域先验为可执行约束程序并渐进退火，解决符号回归中的伪方程陷阱问题，在生物学/材料/化学/物理五个 benchmark 上实现 SOTA 并给出理论泛化界保证。

## 精华提炼

### 管线/方法
1. **问题定义**：将 SR 重新定义为 argmax S(f, D, C)，其中 S 同时考虑 MSE 分数和先验约束检查器 V(f, C) 的布尔输出。权重函数 w(V) 给满足约束的候选者更高权重但保留不满足者的贡献以维持多样性。
2. **先验约束检查器**：LLM-Assisted 工作流——LLM 从问题描述提取领域知识 → 人类专家结合统计模式筛选 → LLM 生成可执行约束程序（Python 代码形式）。例如 E. coli 生长模型中约束包括：db=0 when b=0、极端温度/pH 下 db≤0、对温度和 pH 的单峰响应、高温不对称衰减。
3. **三阶段管线**：
   - **Stage I (Warm-up)**：基于数据分析（采样点、变量范围、相关性、非线性贡献）引导 LLM 生成方程骨架 → L-BFGS-B 优化参数 → 不满足先验则触发修复流程 → 循环分配到岛屿
   - **Stage II (Evolution)**：并行岛屿 + 两级采样（cluster-level softmax 基于 PACE 分数，cluster 内按长度偏好短程序）→ LLM 结合历史高分数候选和 insight pool 生成新候选 → L-BFGS-B 优化 + 随机重试（最多10次）
   - **Stage III (Refinement)**：残差增强精炼 + 反思机制。对每岛最优候选：残差统计量（NMSE/bias/skewness/kurtosis）+ SHAP + 决策树高误差区域分析 → 识别缺陷变量 → 从 E 中检索残差向量相似的参考候选 → LLM 生成改进方程 → 反思成功/失败经验写入 Insight Pool
4. **PACE 机制**：核心创新。随时间 t = N_curr/N_max 渐变：
   - 有效候选 (V=1): S = (1-β) + 2β·S_MSE_norm
   - 无效候选 (V=0): S = σ(t)((1-β) + 2β·S_MSE_norm) - δ(t)
   - σ(t) = max(0, 1 - η·φ(t)), δ(t) = α·φ(t), φ(t) = (B^t - 1)/(B - 1), B=60
   - 早期 t≈0 → σ≈1, δ≈0，允许探索；晚期 t→1 → σ 收缩, δ 增大，强制收敛到先验一致子空间

### 关键技术细节
- **LLM Backbone**: GPT-4o-mini（主实验），Llama-3.3-70B（消融鲁棒性）
- **参数优化器**: L-BFGS-B（SciPy）
- **岛屿数**: 10，带 reset_period=4h 防止早熟收敛
- **采样预算**: max_sample_num=10000，samples_per_prompt=4
- **PACE 参数**: β=0.6, α=1.2, η=1.0, exp_base=60.0
- **Cluster 采样温度**: 初始 0.1，衰减周期 30000
- **L-BFGS-B 重试**: 最多 10 次（先验检查失败时重新初始化参数）
- **Refinement 触发**: refine_start_ratio=0.01（预算1%后开始），每100样本触发
- **推理成本**: ~17.85M tokens/run（13.36M input + 4.49M output），约 \$4.70/run，单 GPU RTX 4090 + i9-14900HX，约10小时/run
- **实验数据集**: LLM-SRBench 的5个 benchmark，数据切分为 train/ID validation/OOD validation（时间维度切分，T=[0,20) 作为 OOD）

### 核心结果
- **定量结果**：PG-SR 在所有5个数据集上 ID 和 OOD NMSE 均最优。以 GPT-4o-mini 为 backbone 时：
  - E. coli: ID 1.32e-3, OOD 1.59e-3
  - Stress-Strain: ID 1.49e-2, OOD 4.63e-2
  - CRK: ID 1.74e-11, OOD 1.22e-8
  - Oscillator 1: ID 6.62e-14, OOD 2.95e-8
  - Oscillator 2: ID 1.79e-11, OOD 8.32e-11
- **Oscillator 2 几乎精确恢复真方程**：简化后与 ground truth 逐项匹配，系数几乎完全一致
- **Backbone 鲁棒性**：Llama-3.3-70B 和 GPT-4o-mini 结果接近，表明框架不敏感于 backbone 选择
- **先验增强实验**：给 PySR 和 LLM-SR 加相同的 prior checker + PACE → OOD 改善，但仍不如 PG-SR，说明 PG-SR 框架整体设计优于简单外挂先验约束
- **先验质量敏感性**：Wrong Priors（设错误最优温度15°C/pH4）→ 性能退化但仍接近 No Prior 水平，PACE 机制使错误先验不会导致崩溃
- **Ablation**：去 Warmup → OOD NMSE 从 1.59e-3 崩到 5.09e-2；去 PACE → OOD 崩到 2.98e-1（最严重）；去 Refine → ID 崩到 8.75e-2
- **噪声鲁棒性**：σ=0.1 时 PG-SR 仍保持远优于 baselines 的 OOD 性能
- **数据稀缺鲁棒性**：5%数据时 PG-SR OOD 性能仍稳定（先验约束主导搜索，数据量不敏感）
- **Surprising**：PACE 去除是最致命的 ablation（OOD 恶化最严重），说明硬过滤比渐进约束危险得多。Wrong Priors 不崩溃也是反直觉的——先验质量差不会让框架爆炸。

### 明确局限
1. **人工干预依赖**：构造可执行先验约束程序仍需人类专家参与（从 LLM 输出筛选、结合统计模式），fully autonomous 尚未实现。作者承认这是主要 limitation。
2. **未做标准 SR benchmark 对比**：只在 LLM-SRBench 上实验，没有和经典 SR benchmark（如 Feynman、ODE-Strogatz、Nguyen）比较，不清楚在简单/经典问题上是否也有优势。
3. **真机/真实实验验证不足**：Stress-Strain 用了真实实验数据（Al 6061-T651），但其他4个数据集仍是 synthetic/simulated。没有 robot/物理实验的在线验证。
4. **计算成本高**：每 run ~10 小时 + $4.70，比传统 search-based 方法（PySR 通常几分钟）慢得多。
5. **先验约束构造依赖 LLM 提取 + 人类验证**：如果人类专家不熟悉领域，可能出错。没有自动化先验质量评估。
6. **没有 open-source code 声明**：正文中未提代码开源，reproducibility 存疑。
7. **对比方法版本问题**：LLM-based baselines 用的是 GPT-4o-mini（低成本模型），没有和更强模型（GPT-4o、Claude 等）比较，不清楚 PG-SR 优势是否部分来自 baseline LLM 能力不足。
8. **理论 bound 相对 trivial**：Rademacher complexity 减小的证明基本上说 H_C ⊆ H 所以 R(H_C) ≤ R(H)，这是平凡的；真正的贡献在实际约束设计而非理论深度。
