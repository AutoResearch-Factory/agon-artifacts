# Towards Scaling Laws for Symbolic Regression (2510.26064)

- **Year**: 2025.10
- **Authors**: Dave Otte, Jörg K.H. Franke, Arbër Zela, Fábio Ferreira, Frank Hutter (University of Freiburg / ELLIS Institute Tübingen / Open-Sci Collective / LAION / Prior Labs)
- **Venue**: NeurIPS 2025 MATH-AI Workshop
- **Keywords**: symbolic regression, scaling laws, transformers, encoder-decoder, compute-optimal, power laws, end-to-end SR, FLOPs, token-to-parameter ratio, synthetic data
- **One-liner**: 首次系统研究 symbolic regression 中 transformer 的 scaling law，发现 solved rate 和 loss 在 3 个数量级 compute 上遵循 power-law，最优 token-to-parameter ratio ≈ 15。

## 精华提炼

### 管线/方法

整体 pipeline 由两部分组成：**数据生成** 和 **模型训练与评估**。

**数据生成（两阶段）**：
1. **表达式生成（Expression Generation）**：递归地从变量 $E_0 = \{x_1, ..., x_n\}$ 出发，逐深度应用一元运算符（exp, sin, neg, sqrt）和二元运算符（+, -, ×, ÷）生成完整表达式树集合 $E$。每步用 Sympy 做 canonicalization 和去重（expand + 自定义排序），保证训练数据干净无重复。最终取前 $|E| = 100,000$ 个表达式。
2. **数据采样（Data Sampling）**：对每个 base expression 采样最多 $k = 3600$ 个 expression-dataset pair。先以概率 $p=0.2$ 在每个变量和一元运算符位置插入随机整数常数（范围 [-9, 9]），然后通过 Gaussian mixture 采样 64 个数据点的 dataset（最多 5 个 cluster，带随机 Haar 旋转），失败重试最多 5 次。

**模型架构（Encoder-Decoder Transformer）**：
- **输入 Embedding**：不同于前人将每个数据点 merge 成单个 embedding，本工作对表格数据集中每个 cell 独立做 embedding——将 mantissa（含符号）和 exponent 分别 up-project 到 embedding 维度后相加。这样每行 × 每列产生一个 token embedding。
- **Encoder**：借鉴 tabular foundation model 设计（如 TabPFN），每层同时做 **row-wise attention** 和 **column-wise attention**，跨不同变量和不同数据点做信息聚合。
- **Decoder**：标准 seq2seq transformer decoder，cross-attend 到 encoder 输出的 target cell embedding，自回归生成 LaTeX 表达式（常数按 digit-wise tokenization）。
- **Loss**：cross-entropy，预测 token vs 真实表达式的 token。

**评估**：从模型采样 128 个表达式，选 $R^2$ 最高的那个，计算 $\text{Acc}_{\text{solved}}$（完美匹配率）和 $\text{Acc}_{R^2>0.99}$（$R^2>0.99$ 比例），3 seed 平均。

### 关键技术细节

**模型规模（5 档）**：

| Model | Dim | Enc Layers | Dec Layers | Heads | Head Dim | MLP Dim | Params |
|-------|-----|-----------|------------|-------|----------|---------|--------|
| 6.5M  | 256 | 3 | 3 | 4 | 64 | 1024 | 6.48M |
| 13.5M | 320 | 4 | 4 | 5 | 64 | 1280 | 13.40M |
| 24M   | 384 | 5 | 5 | 6 | 64 | 1536 | 24.01M |
| 45.5M | 448 | 7 | 7 | 7 | 64 | 1792 | 45.53M |
| 93M   | 512 | 11| 11| 8 | 64 | 2048 | 93.08M |

**训练超参数**：
- Optimizer: AdamCPR (Constrained Parameter Regularization，一种 Adam 变体，带 kappa 参数约束)
- LR schedule: linear warmup (前 5% steps) + cosine annealing
- Weight decay: 0.1, Clip value: 1.0, Beta1=0.9, Beta2=0.98, Eps=1e-9
- Dropout: residual 0.1, attention 0.1
- Precision: fp32
- Max output length: 256 tokens

**Compute 估计**：采用 Kaplan et al. 的 FLOP 估计适配 encoder-decoder：
$\text{FLOPs} \approx 6 \cdot (N_{enc} \cdot D_{in} + N_{dec} \cdot D_{out})$，其中 $N_{enc}, N_{dec}$ 为 feed-forward 参数量，$D_{in}, D_{out}$ 为 input/output tokens 数。

**Token-to-parameter ratio 搜索范围**：5 到 80。

**数据规模**：训练集 100K 表达式 × 3600 pairs = 360M expression-dataset pairs；验证集和测试集各 1000 表达式（新采样常数和数据集）。

### 核心结果

1. **Power-law scaling**：在 3 个数量级 compute（~4.5×10¹⁵ 到 ~1.5×10¹⁹ FLOPs）上，$\text{Acc}_{\text{solved}}$ 和 validation loss 都遵循 power-law 趋势。Acc_solved 从 0.03 提升到 0.6，按 scaling law 外推在 3.8×10²¹ FLOPs 可达 0.8。

2. **最大模型（93M）的最佳结果**：Acc_solved = 0.5967，Acc_{R²>0.99} = 0.8830，Final Val Loss = 0.1047（最高 compute 下）。

3. **最优超参数随 compute 增长**：optimal batch size 和 learning rate 都随模型规模增大而增大。注意 learning rate 的上升趋势与 LLM 的 scaling law 发现相反——这体现了不同 task 之间的差异。

4. **最优 Token-to-Parameter Ratio ≈ 15**：在实验的 compute regime 内约为 15，且随 compute 增长有轻微上升趋势，意味着训练数据量应比模型规模增长略快。

5. **Acc_{R²>0.99} 提升速度远快于 Acc_solved**：说明精确匹配表达式非常困难，即使 $R^2$ 已接近完美匹配。

### 明确局限

1. **表达式域限制**：仅限最多 2 变量 + 小整数常数（[-9, 9]），real-world SR 涉及更多变量和浮点常数。Scaling law 可能不能直接迁移到更复杂的表达式设置。
2. **Single-seed 训练**：因计算资源限制，每个配置只跑一个 seed，结果有 variance。作者自己也指出需要更多 seed 来减小方差。
3. **Compute 范围有限**：仅跨越 3 个数量级，外推未验证。
4. **未与 baseline 比较**：专注于 scaling insight，不做与 GP-based SR 或现有 deep SR 方法的对比。难以判断这些 scaling law 在实际竞争中意味着什么。
5. **合成数据局限**：仅用 Gaussian mixture 生成数据，无真实世界表格数据验证。真实数据可能有不同的分布特性。
6. **最大模型仅 93M 参数**：相比 LLM 的 scaling law 研究（十亿到千亿参数），规模仍非常小。更大模型下的 scaling behavior 未知。
7. **表达式深度限制**：max tree depth = 3，复杂的嵌套表达式不在范围内。

### 跟我们的关系

对 **0616-fex**（feature extraction / function extraction）课题：

- **撞车评估**：不直接撞车。本工作聚焦 SR 的 scaling law 本身，而非 FEX 的具体应用。但如果我们用 transformer-based SR 做 feature extraction 的 core component，这篇的 scaling insight 直接相关——告诉我们怎么 scale 模型和数据。
- **可用工具/方法**：
  1. **数据生成 pipeline**：递归表达式生成 + canonicalization 的方法可以直接用于生成 FEX 训练数据，比 Lample 2019 的采样方法更可控。
  2. **Cell-level embedding + row/col attention**：对表格数据的编码策略可直接用于我们的 feature extractor encoder，比将整行 merge 成单个 embedding 更 expressive。
  3. **AdamCPR optimizer**：constrained parameter regularization 可能在我们的训练中也有用。
  4. **Scaling law 方法论**：如果我们要 scale FEX 模型，这篇的 FLOP 估计、Pareto front 选取、power-law fitting 流程可直接借鉴。
  5. **Evaluation metrics**：Acc_solved + Acc_{R²>0.99} 的双指标设计值得采纳。
- **差异化**：我们做的是将 SR/表达式发现用于具体 feature extraction 场景，而非通用 SR scaling。我们需要关注的是在真实表格数据（非合成 Gaussian mixture）上的效果，以及如何把发现的表达式 embedding 到下游任务中——这些这篇完全没做。

## Read by: 0616-fex (2026-06-25)

### 撞车评估
不撞车。本文研究 SR transformer 的 scaling law 本身（power-law 趋势、最优超参数随 compute 变化），不是 FEX 的应用。但如果我们计划用 transformer-based SR 做 feature extraction，本文的 scaling insight 是必须读的 reference。

### 可用技术点
1. **递归表达式生成 + Sympy 去重**：比 Lample 2019 的随机采样更可控，可直接用于 FEX 训练数据 pipeline
2. **Cell-level input embedding + row/col dual attention encoder**：比 E2E 的单 embedding 更 expressive，适合我们的表格数据 encoder 设计
3. **AdamCPR**：可尝试替代 AdamW
4. **Scaling law 分析框架**：FLOP 估计公式、Pareto 选取、power-law fitting——如果我们有 scale FEX 的计划，这套方法直接可用
5. **Token-to-parameter ratio ≈ 15**：给我们一个训练的初始 guess

### 关键限制（对我们）
- 仅在合成 Gaussian mixture 数据上验证，无真实数据
- 仅 2 变量、整数常数、depth 3——FEX 实际面对的问题 space 可能复杂得多
- 最大模型仅 93M——我们的 target model size 可能也需要在这个范围或更大
- Single-seed 实验，variance confidence 不足
