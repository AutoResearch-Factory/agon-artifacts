# A constrained symbolic regression approach for Lyapunov function discovery (2606.10045)

- **Year**: 2026.06
- **Authors**: Ilias Mitrai (The University of Texas at Austin), Wentao Tang (North Carolina State University)
- **Venue**: arXiv
- **Keywords**: Lyapunov function discovery, symbolic regression, expression tree, MINLP, branch-and-bound-and-check, stability certification, dynamical systems, process control
- **One-liner**: 把 Lyapunov 搜索写成约束符号回归，并在 B&B 中内嵌验证。

## 精华提炼

### 管线/方法
- 输入是自治连续动力系统 $\dot{x}=f(x)$，目标是在平衡点 $x=0$ 附近寻找 Lyapunov 函数 $V(x)$。
- 训练数据是若干条从不同初值出发的状态轨迹：第 $i$ 条轨迹含 $N_i$ 个采样点。
- 论文假设状态方程模型 $f$ 已知；若模型未知，仍可搜索候选 $V$，但不能做全域有效性验证。
- 候选 Lyapunov 函数被表示为固定深度 $d$ 的 perfect binary expression tree。
- 节点集合为 $\mathcal{N}=\{1,\dots,2^{d+1}-1\}$，叶节点为 $\mathcal{T}=\{2^d,\dots,2^{d+1}-1\}$。
- 每个节点可以分配二元算子 $\mathcal{B}=\{+,-,*,/\}$、一元算子 $\mathcal{U}=\{\exp,\log\}$，或操作数 $\mathcal{L}=\{x_1,\dots,x_{N_x},cst\}$。
- 实验部分为了降低复杂度，实际使用的二元算子是 $\{+,-,*\}$。
- 二进制变量 $y_{on}$ 表示算子/操作数 $o$ 是否分配给节点 $n$。
- 连续变量 $c_n\in[c_{lb},c_{ub}]$ 表示常数节点取值。
- 连续变量 $v_{ijn}\in[v_{lb},v_{ub}]$ 表示第 $i$ 条轨迹第 $j$ 个数据点在树节点 $n$ 的计算值。
- 根节点值 $v_{ij1}$ 就是该数据点上的 $V(x_{ij})$。
- 第一组约束是 tree-defining constraints，保证每个节点最多分配一个算子/操作数。
- tree-defining constraints 还保证二元/一元算子的 arity 合法，例如二元算子必须有左右孩子，一元算子只需要右孩子。
- 论文额外要求所有状态变量都必须至少出现在最终表达式中。
- 第二组约束是 value-defining constraints，保证树的数值计算与节点算子一致。
- 对加法、减法、乘法、除法和常数，作者使用 big-M 约束来表达节点值关系。
- 对变量节点，作者用上下界约束把节点值绑定到对应状态变量 $x_{kij}$。
- 第三组约束是 Lyapunov-defining constraints。
- 正定性约束要求非零状态点满足 $v_{ij1}\geq\|x_{ij}\|^2$。
- 平衡点约束要求在 $x=0$ 的数据点满足 $v_{ij1}=0$。
- 若轨迹没有到达原点，作者建议人为加入一个 $x=0$ 辅助数据点。
- 导数负定性不通过符号求导建第二棵树，而是用有限差分近似。
- 具体约束为 $(v_{i,j+1,1}-v_{ij1})/\Delta t\leq-\alpha\|x_{ij}\|^2$。
- 这里 $\alpha\geq0$ 是收敛速率，也是可优化变量。
- 论文指出也可替换成指数稳定形式：$(v_{i,j+1,1}-v_{ij1})/\Delta t\leq-\alpha v_{ij1}$。
- 总体学习问题是一个非凸混合整数非线性规划；在实验设定下是 MIQCQP。
- 默认目标可写成 $-w_1\alpha+w_2\sum y_{no}+w_3\sum c_n^2$。
- 这让同一框架既能追求最大收敛速率，也能追求表达式复杂度更低或常数更小。
- 关键求解策略是 branch-and-bound-and-check。
- 标准 branch-and-bound 在搜索过程中会不断发现 integer feasible solutions。
- 每个 integer feasible solution 都对应一个满足树结构、数据点正定性、有限差分负定性的候选符号 Lyapunov 函数。
- 但作者强调：数据点满足约束不等于全域有效，有限差分满足也不等于真实 $\dot{V}$ 满足。
- 因此他们在 Gurobi callback 中一旦发现候选函数，就立刻构造符号表达式并做验证。
- 验证步骤检查 $\min_{x\in\mathcal{D}}V(x)\geq0$、$\max_{x\in\mathcal{D}}\dot{V}(x)\leq0$、以及 $V(0)=0$。
- 若三项成立，候选函数被接受为整个域上的 Lyapunov 函数。
- 若验证失败，branch-and-bound 继续搜索，而不是等全局最优解出来后才做后验检查。
- 这点是论文最有价值的工程设计：把 verification gate 放进 combinatorial search loop。
- 输出是一个符号表达式形式的 Lyapunov 函数、收敛速率 $\alpha$、以及是否通过全域检查。

### 关键技术细节
- 论文的实现使用 Pyomo 建模。
- 求解器是 Gurobi v11.0.0。
- 动力系统轨迹用 explicit Runge-Kutta 方法仿真。
- Case 1 系统为 $\dot{x}_1=-x_1+4x_2$，$\dot{x}_2=-x_1-x_2^3$。
- Case 1 初始训练轨迹为 $t\in[0,10]$、100 个离散点，初值 $[-0.61,0.24]^T$。
- Case 1 算子集合为 $\{+,-,*\}$，常数边界 $c_{lb}=1,c_{ub}=5$，节点值边界 $v_{lb}=-50,v_{up}=50$。
- Case 1 默认目标权重为 $w_1=1,w_2=0,w_3=0.1$。
- 深度 1 的表达式树对应 3 个节点，问题有 419 个变量，其中 12 个 binary，1828 条约束；Gurobi presolve 判 infeasible。
- 深度 2 的表达式树对应 7 个节点，1.5 秒后也判 infeasible。
- 深度 3 的表达式树对应 15 个节点，问题有 1697 个变量，其中 66 个 binary，10562 条约束。
- 深度 3 时，45 秒找到 $V(x_1,x_2)=2x_1^2+8x_2^2$，$\alpha=0.194$，通过全域检查。
- 若继续搜索，85 秒找到 $V(x_1,x_2)=x_1^2+4x_2^2$，$\alpha=0.10$。
- 当固定 $\alpha=0.1$ 且只最小化常数平方和时，40 秒全局最优得到 $x_1^2+4x_2^2$。
- 采样率实验固定 $T=5$，$N_{data}=5$ 时 0.1 秒得到的函数不是全域有效。
- 同一实验中，$N_{data}=10,20,50,100$ 均返回 $x_1^2+4x_2^2$，求解时间分别为 0.23、2、42、206 秒。
- 采样率提高会提高导数有限差分近似质量，但变量和约束数量线性增加。
- 时间跨度实验固定 $\Delta t=0.1$，较短 horizon 会失败：$T=1$ 返回 $x_1^2+2x_2$，$T=2$ 返回 $x_1^2+3x_2^2$，均不是全域有效。
- $T=5$ 时返回 $x_1^2+4x_2^2$，对应 897 个变量、4648 条约束、33085 个 B&B 节点、42 秒。
- 多初值实验表明短 horizon 下增加轨迹数可以弥补覆盖不足：$T=0.2,N_t=5$ 无效，但 $N_t=10,15$ 有效。
- 对 $T=1$，$N_t=3$ 无效，$N_t=5,10,15$ 返回有效的 $x_1^2+4x_2^2$。
- 最简单函数实验先用 $\sum y$ 做目标，130 秒找到 13 节点的 $x_1^2+4x_2^2$。
- 同一实验 227 秒后找到 9 节点的 $x_1^2+3.97x_2^2$，形式接近但系数偏离。
- 作者把这种系数偏差解释为有限数据、导数近似，以及 $\sum y$ 目标下常数退化共同导致。
- 改用复杂度上界 $\sum y\leq N_C$ 后，$N_C=15,13,11,9$ 全部找到有效函数。
- 其中 $N_C=9$ 只用 20 秒找到 $x_1^2+4x_2^2$，作者认为这是给定二元算子集合下的最简单树。
- Case 2 系统为 $\dot{x}_1=x_2-x_1^3$，$\dot{x}_2=-x_1-x_2^3$。
- Case 2 使用一条 $t\in[0,50]$、200 个离散点轨迹。
- Case 2 学习问题有 3297 个变量，其中 66 个 binary，18148 条约束。
- 以 $-\alpha+0.1\sum c_n^2$ 为目标时，8 秒、93 个 B&B 节点找到 $V=x_1^2+x_2^2$，$\alpha=0.0097$。
- 改成纯 feasibility objective $0$ 且不用 warm start，30 秒、2272 个 B&B 节点找到同一个函数。

### 核心结果
- 本文证明性结果不多，核心证据主要来自可解释优化建模和数值 case studies。
- 最重要的正结果是：对两个 2D 非线性系统，框架能找到解析 Lyapunov 函数。
- Case 1 最终稳定函数是 $x_1^2+4x_2^2$ 或同尺度变体 $2x_1^2+8x_2^2$。
- Case 2 最终稳定函数是 $x_1^2+x_2^2$。
- 深度不足直接导致 infeasible，这说明表达式树容量是硬瓶颈，不是仅靠优化时间能解决。
- 数据采样太稀会产生假 Lyapunov 函数；$N_{data}=5$ 时返回的表达式未通过全域验证。
- 时间 horizon 太短也会产生假函数；论文明确展示短轨迹无法捕获足够动态行为。
- 多初值可以减少短轨迹覆盖不足，但会显著增加连续变量和约束。
- 反直觉点：求全局最优的学习问题并不保证得到全域有效 Lyapunov 函数，因为目标和数据约束本身可能偏离真实 Lyapunov 条件。
- 另一个反直觉点：把表达式复杂度作为约束而不是目标，反而更稳地找到简单有效函数。
- branch-and-bound-and-check 的价值不是提升最优性证明，而是提前把无效候选挡掉，避免等到全局最优才发现不可用。

### 明确局限
- 作者自己承认学习 formulation 假设数据无噪声；真实实验数据需要 slack variables，并在目标中惩罚 slack。
- 作者自己承认有限差分近似 $\dot{V}$ 会破坏保证，即使训练点约束全满足也可能不是有效 Lyapunov 函数。
- 作者自己承认有限轨迹覆盖不足会造成全域无效，需要 verification step 或更多数据。
- 作者自己承认如果动力学模型未知，就不能对全域做有效性认证。
- 实验全部是低维解析/仿真系统，没有高维、混杂、接触、神经动力学或真实过程控制平台。
- 没有标准 benchmark 横向比较，也没有与 transformer/RL/neural Lyapunov 方法在同一测试集上跑耗时和成功率。
- 没有公开代码、数据或 Gurobi callback 实现细节，第三方复现实操成本较高。
- 全域验证本身如何被数值求解、域 $\mathcal{D}$ 如何选、验证优化是否全局可靠，正文没有展开到可审计程度。
- 表达式树是固定深度，算子集合在实验里很小；复杂函数可能迅速变成不可解的 MINLP。
- 常数范围、节点值 big-M 边界、采样策略都会影响结果，论文没有系统给出鲁棒性区间。
- 对 0616-fex 来说，若目标是可扩展的自动形式表达发现，这篇更像一个严谨但小规模的 optimization baseline。

### 跟我们的关系
- 对 `0616-fex` 的关系取决于该 topic 的精确定义；按“formal/executable expression discovery / certificate extraction”视角，它是高度相关的 baseline。
- 如果我们要 claim “从轨迹数据发现 Lyapunov certificate 的符号表达式”，这篇直接撞车。
- 撞车点具体是：expression tree superstructure、把正定/负定条件写成约束、用混合整数全局优化搜索、在搜索中嵌入 verification gate。
- 如果我们只是做 LLM agent 辅助建模、自动生成候选表达式、自动调参或自动构造验证脚本，它不完全撞车，但必须引用。
- 可复用的方法包括：固定深度表达式树、arity constraints、big-M value constraints、有限差分 Lyapunov constraints、Gurobi callback 中的 branch-and-check。
- 可复用的实验设计包括：采样率、时间 horizon、初值数量、复杂度约束、feasibility objective 的 ablation。
- 差异化方向一：把验证器独立化，例如 SMT/SOS/interval arithmetic/auto-diff 精确导数，而不是同一优化回路自检。
- 差异化方向二：处理噪声和未知动力学，加入 slack、置信界、系统辨识误差传播。
- 差异化方向三：扩展到高维或复杂系统，并建立统一 benchmark，而不是只做两个 2D case。
- 差异化方向四：用 agent/LLM/搜索策略提出表达式或切分问题，再由严格 verifier veto，提高可扩展性。
