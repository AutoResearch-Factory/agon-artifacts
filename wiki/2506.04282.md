# DrSR: LLM based Scientific Equation Discovery with Dual Reasoning from Data and Experience (2506.04282)

- **Year**: 2025.06
- **Authors**: Runxiang Wang, Boxiao Wang, Kai Li, Yifan Zhang, Jian Cheng (UCAS, CASIA -- C2DL Lab)
- **Venue**: arXiv preprint (NeurIPS 2025 style)
- **Keywords**: symbolic regression, LLM, scientific discovery, data-aware insight, inductive reasoning, equation discovery, dual reasoning, idea library
- **One-liner**: DrSR augments LLM-based symbolic regression with data-structure analysis and reflective idea extraction, significantly outperforming prior LLM-SR on 6 scientific benchmarks.

## 精华提炼

### 管线/方法

DrSR builds upon the LLM-SR pipeline (generate equation skeleton, optimize parameters via BFGS, evaluate fitness, cache best equations) and adds two synergistic LLM-powered modules:

1. **Data-aware Insight** (`pi_data`): An LLM that analyzes raw data to extract structural patterns. At each iteration, 100 input-output pairs are uniformly sampled, fed to `pi_data` to produce structured descriptions (monotonicity, nonlinearity, correlations). When a better equation is found, residuals are computed and used to construct an augmented dataset `D_t = {(x_i, y_i, res_i)}`, which is resampled to produce refined insights. Insights evolve from coarse global patterns to finer local/higher-order structures over iterations.

2. **Inductive Idea Extraction** (`pi_idea`): An LLM that categorizes each generated equation into Positive (outperforms current best), Negative (worse), or Invalid (syntax/compile errors). For each category, it extracts structured "ideas" (generation strategies, common pitfalls, error-avoidance heuristics) stored in a JSON Idea Library. In each iteration, ideas from the latest 50% entries per category are sampled and injected into `pi_main`'s prompt.

3. **Main Generation** (`pi_main`): Generates equation skeletons (with learnable parameters) conditioned on: (a) task description + variable constraints, (b) data-aware insights from `pi_data`, (c) sampled ideas from the Idea Library. Four candidate skeletons per iteration, each optimized via BFGS. Best-performing equations cached as in-context examples.

All three modules share the same LLM backbone (Mixtral-8x7B-Instruct-v0.1 or LLaMA3.1-8B-Instruct).

### 关键技术细节

- **LLM backbone**: Mixtral-8x7B-Instruct-v0.1 (Mixtral of Experts, 46.7B total params) and LLaMA3.1-8B-Instruct, both quantized on NVIDIA H100 GPUs. LLaMA3.1-8B uses ~8GB VRAM in 4-bit quantized form.
- **Sampling for data insight**: 100 input-output pairs uniformly sampled from original dataset (or residual-augmented dataset). Sampling temperature 0.6, top-k=30, top-p=0.3.
- **Equation generation**: 4 skeletons sampled per iteration, all with learnable parameters optimized via BFGS (L-BFGS-B).
- **Iterations**: DrSR runs 1000 iterations per dataset; baselines run 2000+ to ensure convergence.
- **Idea sampling**: Per iteration, up to 3 ideas drawn from latest 50% entries per idea category (positive/negative/error).
- **Validation**: Tested on 6 datasets spanning physics (nonlinear oscillators), biology (E. coli growth), materials science (stress-strain), chemistry (LSR-Synth CRK0), and transformed Feynman equations (LSR-Transform).
- **Runtime**: ~6 hours per dataset on LLaMA3.1-8B, ~12 hours on Mixtral-8x7B on H100 GPUs.

### 核心结果

| Metric | DrSR (best) | Best baseline | Improvement |
|--------|-------------|---------------|-------------|
| Oscillator 1 NMSE | 3.14e-7 (Mixtral) | 2.55e-5 (LLM-SR Llama3.1) | ~81x better |
| Oscillator 2 NMSE | 1.80e-12 (Mixtral) | 4.59e-5 (LLM-SR Mixtral) | ~25,500x better |
| Oscillator 2 ACC_0.001 | 99.94% (Mixtral) | 8.2% (LLM-SR Llama3.1) | ~12x better |
| E. coli NMSE | 0.0195 (Mixtral) | 0.1349 (LaSR Llama3.1) | ~7x better |
| Stress-Strain ACC_0.1 | 88.28% (Mixtral) | 76.21% (LLM-SR Llama3.1) | +12% |
| LSR-Transform ACC_0.1 | 96.32% (Llama3.1) | 93.73% (LLM-SR Llama3.1) | +2.6% |
| LSR-Synth CRK0 NMSE | 1.75e-8 (Llama3.1) | 6.52e-8 (LaSR Llama3.1) | ~3.7x better |

- Valid solution rate: DrSR 0.4-0.6 vs LLM-SR sub-0.3 on complex tasks (E. coli, Stress-Strain).
- Ablation: Removing data insight or idea extraction both degrade performance, with compounded losses when both removed.
- Noise robustness: At sigma=0.002 Gaussian noise, DrSR maintains 95.39% ACC_0.01 (ID) vs LLM-SR's 82.08%.
- OOD generalization: DrSR OOD NMSE on Oscillator 2 = 2.80e-11 vs uDSR's 9.59e+3 (severe overfitting).

### 明确局限

- **作者自称局限**:
  1. LLM stochasticity can produce verbose, repetitive, or overly complex outputs requiring expert intervention.
  2. BFGS optimizer may not find globally optimal parameters; more robust optimization strategies needed.
  
- **读出来的局限**:
  1. **仅 4 个候选方程/iteration**: 搜索空间覆盖非常有限。相比传统 GP 方法（人口规模 50-1000），4 次采样严重依赖 LLM 先验质量。
  2. **小规模问题**: 所有 benchmark 变量数在 4-8 之间，未测试高维（>10 变量）或高复杂度场景。
  3. **无真实验证**: 所有实验都是数值拟合 benchmark，没有真机实验或真实世界验证。
  4. **无开源代码**: 论文没有提供代码仓库链接（作者声称 "we will release"）。
  5. **计算成本高**: 6-12 小时 per dataset on H100，远超传统 SR 方法（PySR 等数分钟可跑完）。
  6. **Prompt 依赖**: 所有 insight/idea 提取依赖精心设计的 prompt templates（Appendix C 展示），鲁棒性未知。
  7. **LLM 先验可能泄漏**: 部分 benchmark（如 LSR-Transform 来自 Feynman equations）的 ground-truth 可能已被 LLM 在训练中见过，导致 data insight 模块实际是在触发 LLM 记忆而非从数据中学到结构。

### 跟我们的关系

- **撞车评估**: 对 `0616-fex` 课题（fex = flexible expression / formula exploration?），DrSR 提出了一个通用框架而非特定 fex 方法。它和 fex 的核心区别在于：DrSR 是 LLM-based SR（依赖 LLM 先验 + BFGS 参数优化），而 fex 系列 idea 更多关注 representation learning、pretrained 模型、或特定 inductive bias。低度直接撞车。
- **可用工具/方法**:
  - Data-aware Insight 的 "residual-based refinement" 思路可以借鉴到 fex 的 active learning / selective sampling 中。
  - Idea Library 的分类存储策略（positive/negative/error）可用于 fex 的 failure case analysis。
  - 但 DrSR 整体 pipeline 太重（6-12h on H100），和我们轻量级目标可能不匹配。
- **该引不该引**: 作为 LLM-based SR 的最新 work 可以引用，但要注意区分我们的方法是否也走 LLM-generation 路线。如果我们的 fex 方法不用 LLM，则 DrSR 属于 orthogonal 方向，只需 briefly mention as related work。
- **差异化**: 我们的核心优势（如果 fex 走 pretrained encoder/decoder 路线）可以是（1）更快的推理速度（非 iterative LLM sampling），（2）可证明的理论 guarantee（非 prompt-based），（3）可扩展性（支持 high-dimensional input）。
