# SyMANTIC: An Efficient Symbolic Regression Method for Interpretable and Parsimonious Model Discovery in Science and Beyond (2502.03367)

- **Year**: 2025.02
- **Authors**: Madhav R. Muthyala, Farshud Sorourifar (Ohio State U), You Peng (Dow Chemical), Joel A. Paulson (Ohio State U, corresponding)
- **Venue**: Submitted to ACS journal (JACS Au format)
- **Keywords**: Symbolic Regression, Sparse Regression, Compressed Sensing, Mutual Information, Feature Expansion, Pareto Optimization, GPU Acceleration, Information-Theoretic Complexity
- **One-liner**: SyMANTIC 提出基于互信息筛选 + 自适应特征扩展 + ℓ0 稀疏回归的高效符号回归算法，在 20 个基准方程上恢复率超 95%（PySR 仅 50%），中位训练时间不到 10 秒。

## 精华提炼

### 管线/方法

SyMANTIC 的完整 pipeline：

1. **输入**：训练数据 `{(x_i, y_i)}`，操作符集合 O（默认包含 `+, -, *, /, exp, log, sqrt, inv, sq` 共 10 个操作符）

2. **Step 1: 互信息预筛选** — 计算每个输入特征 x_i 与目标 y 之间的互信息 MI(y; x_i)（通过核密度估计 KDE 近似），按 MI 值降序排序，丢弃 MI 低于阈值 γ=0.1 或排名超过 n_screen=20 的特征。对高维输入（如 D=1444），只保留 top 1% MI 特征；对中等维度（D=57），保留 top 50%。最终得到 d ≤ 20 个筛选特征 z。

3. **Step 2: 递归特征扩展** — 对筛选后的特征 z 递归应用操作符 O，生成扩展特征 φ_l(x)。第 l 级特征数 d_l ≈ m_b^{2^l-1} * d^{2^l}，呈双重指数增长。默认最大扩展级数 n_exp=3。若无 MI 预筛选，高维输入无法扩展到第二级。

4. **Step 3: C²-SISSO** — Complexity-Constrained SISSO 模块：
   - (a) 复杂度过滤：移除结构复杂度 C(f) > λ_c 的所有扩展特征
   - (b) SIS（Sure Independence Screening）：计算特征与目标的相关性权重 ω = Φ^T y，保留 top k=20 个特征
   - (c) ℓ0 回归：穷举所有 t-term 模型（t=1 到 T=3），每个 t 组合数为 C(tk, t)。对每个组合做最小二乘拟合，记录残差 r_t
   - (d) 残差引导迭代：用最佳 t-term 模型的残差 r*_t 作为新目标，再次运行 SIS 扩展特征子空间，逐步构建 t+1 term 模型
   - (e) 每轮 C²-SISSO 测试最多 Σ_{t=1}^T C(tk, t) 个模型

5. **Step 4: 自动化超参数搜索** — 遍历复杂度阈值 λ_c（n_comp=4 个值，按复杂度分位数均匀选取）和扩展级数 l（n_exp=3），每次调用 C²-SISSO 后更新近似 Pareto 前沿 P_i。若某个模型的 RMSE 达到阈值，提前终止。默认设置下最多测试 420240 个模型。

6. **输出**：Pareto 前沿上的表达式集合，以及"utopia point"（在 loss-complexity 平面上最接近原点的模型）。

### 关键技术细节

- **结构复杂度度量**：C(f) = K(f) * log2(B(f))，其中 B(f) 是表达式中出现的唯一基函数（包括变量和操作符）的数量，K(f) 是基函数被使用的总次数。这个度量来自最小描述长度（MDL）原理，取的是 Wu & Tegmark (2019) 的信息论框架的第一项（省略参数编码长度），使其对参数数值不敏感。例如 f(x) = 0.5*x_1*x_2^2 有 B=4（x_1, x_2, *, ^2），K=5，C=10。

- **操作符集合**：默认 O = {I, +, -, *, /, exp, log, sqrt, inv, sq}。包含 2 个一元操作符（exp, log, sqrt, inv, sq 实际是 5+ 个一元）、4 个非对称二元操作符。用户可以自定义操作符，包括 Bessel 函数等特殊函数。

- **SIS vs 前向/后向选择**：SIS 只需要一次矩阵-向量乘法 O(nd) 来计算特征重要性，而前向/后向选择需要为每个特征训练回归模型。这使 SIS 能扩展到 10^10 甚至更多的特征空间。论文在 Supporting Information 中提供了数值对比。

- **ℓ0 回归精确求解**：对于小 T（默认 T≤5），在所有 C(tk, t) 组合中穷举搜索最优模型是可行的，因为 t×k 通常不超过 60（t=3, k=20 时 C(60,3)=34220 组合）。

- **PyTorch 实现优势**：
  - `torch.linalg.lstsq` 批量并行求解最小二乘
  - 结构复杂度在特征矩阵构建时通过 tensor 操作并行计算
  - 对比以前用 SymPy 串行计算复杂度，有数量级加速
  - 支持 GPU 加速（NVIDIA A100 上加速 2-3 倍）
  - 包名 `symantic`，pip install 即可使用

- **维度分析**：使用 SymPy 符号跟踪单位，阻止不一致单位的操作（如加法要求相同单位），进一步减少无效特征生成。

- **默认超参数**：
  | 参数 | 默认值 | 作用 |
  |------|--------|------|
  | n_screen | 20 | MI 预筛选保留的最大特征数 |
  | γ | 0.1 | MI 阈值 |
  | k | 20 | SIS 保留特征数 |
  | T | 3 | 模型最大项数 |
  | n_comp | 4 | 复杂度阈值数量 |
  | n_exp | 3 | 最大扩展级数 |

### 核心结果

1. **20 方程基准测试**（5 合成+5 深树+5 EmpiricalBench+5 SRSD-Feynman）：
   - SyMANTIC: ~95% 恢复率（5 次重复平均）
   - PySR: ~50%
   - PyOperon, GP-GOMEA, gplearn, DSO: 远低于 50%
   - SyMANTIC 中位训练时间 < 10 秒，4 种对比方法在 5 分钟时限内未能完成
   - 所有实验在 Intel Xeon Gold 6444Y (16 cores, 512GB RAM) 上运行

2. **Pareto 前沿对比**（相对论动量方程 p = mv/√(1-v²/c²)）：
   - SyMANTIC 同时发现了经典解 p=mv 和相对论真解
   - PySR 只找到经典解和几个过拟合的复杂表达式
   - SyMANTIC 的 Pareto 前沿提供了"复杂度 vs 精度"的系统性权衡视图

3. **异方差噪声鲁棒性**（Rydberg 公式，噪声幅值 v=0-1.0）：
   - SyMANTIC 在所有噪声水平下 NRMSE 显著低于 PySR
   - SyMANTIC 误差随 v 增加趋势更平滑可预测
   - PySR 变异性更大

4. **Dow Chemical 工业竞赛数据**（57 输入特征，747 训练/319 测试）：
   - SyMANTIC test R²=0.781, PySR test R²=0.453
   - SyMANTIC test RMSE=0.0255 < 1 分钟
   - HeuristicLab test RMSE~0.154 需要 ~50 分钟
   - SyMANTIC 在 MI 筛选时保留 top 50%（而非默认 20 个特征）

5. **混沌 Lorenz 系统**（仅 5 个时间点，噪声=0）：
   - SyMANTIC 精确还原了 Lorenz 方程的混沌动态
   - PySINDy 快速偏离真值并收敛到不存在的稳态
   - PySINDy 需要约 10 个时间点才能表现良好

6. **分子性质预测**（氧化还原电位，1444 个 PaDEL 描述符，115 训练/1000 测试）：
   - SyMANTIC test R²=0.88（MI 筛选保留 top 1%）
   - PySR test R²=0.70
   - LASSO test R²=-6.05, NN test R²=-881.40（严重过拟合）
   - SyMANTIC 训练时间 9.21 秒

7. **GPU 加速**（4-term 模型，500-3000 数据点）：
   - GPU 相比单核 CPU 加速 2-3 倍
   - 支持多 GPU 通过 PyTorch DDP

### 明确局限

**作者自己说的局限**：
1. 无法发现非线性出现的未知常数（如 sin(2πx) 中的 2π 必须在操作符集合中显式包含）
2. 只探索到最大深度 n_exp 的表达式，无法像遗传编程那样探索更深的结构
3. 当前仅支持 MSE loss（基于 i.i.d. 高斯假设），不支持自定义损失函数
4. 多输出只做独立单输出回归，不建模输出间相关性
5. 动态系统学习需要导数信息（可用数值微分，但在稀疏测量时不可靠）
6. k > 50 时 ℓ0 回归精确求解不再可行，需要替代方法（如 LASSO）
7. 特征预筛选的 MI 方法不考虑特征间冗余（建议用户手动去相关）

**读出来的局限**：
1. 20 个基准方程的测试集偏小，且 15 个方程来自已知的 EmpiricalBench/SRSD-Feynman 数据集，可能存在"数据集友好型"偏置。需要更大规模（如 SRBench 的 252 题）评估。
2. 5 分钟时限对 GP 方法不公平——遗传编程通常需要更长时间才能收敛，但作者论点（"用户不知道需要多长预算"）有一定说服力。
3. 没有在所有问题上报告测试集性能（大部分只用训练集恢复率），仅 Dow Chemical 和分子数据有 train/test split。
4. 复杂度度量 C(f) 只关注结构，不关注参数数值大小（对整数 vs 实数参数的编码成本被忽略），可能导致"看起来简单"的表达式实际包含不精确的参数。
5. ℓ0 回归穷举搜索的本质意味着 T 增加时计算量组合爆炸（T=4 时若 k=20，需检查约 150 万模型），这在更复杂问题上可能成为瓶颈。
6. 没有真机/机器人实验（这与我们 0616-fex 课题的方向可能部分相关——如果 fex 涉及物理系统的 feature expansion）。

### 审读视角：特征扩展（Feature Expansion）维度

SyMANTIC 对 0616-fex 课题的核心价值不在于其符号回归的整体目标，而在于其**特征扩展管道**的设计：

**可借鉴的组件**：
1. **递归特征扩展框架**（Step 2）：操作符集合 O 递归应用于特征向量，生成非线性组合特征。这是泛化的特征扩展机制。如果 0616-fex 需要从原始特征中系统性地生成高阶/非线性特征，这个递归架构是很好的参考。关键参数：扩展级数 n_exp 控制深度，操作符集合 O 控制生成类型的广度。

2. **MI 预筛选**（Step 1）：在输入维度高时（>10-20），用互信息做快速非参数筛选。不假设线性关系——这对捕获非线性特征-目标关系至关重要。0616-fex 如果有高维特征输入场景，可以直接采用这个筛选策略。

3. **复杂度约束的搜索**（Step 3a）：用结构复杂度 C(f) 过滤候选特征，避免在扩展过程中过度生成无意义的高复杂度特征。这个"生成前先过滤"的思路对任何特征扩展管道都有价值。

4. **ℓ0 回归的穷举搜索策略**：虽然 SyMANTIC 的 T≤3 限制看起来小，但结合 SIS 筛选的 k=20 个候选特征，实际探索了 C(20,1)+C(40,2)+C(60,3)=34240 种组合。这种"筛选后穷举"的模式在特征空间可控时效果很好。

**与 LLM 驱动特征扩展的差异点**：
- SyMANTIC 的特征扩展是**确定性的穷举**：给定 O 和 z，生成的特征空间是唯一确定的。LLM 驱动的扩展可能是**生成式的**：LLM 提议有意义的特征组合。
- 确定性穷举的优势：可重复、可分析、不依赖 prompt 设计。劣势：无法利用先验知识"跳跃"到有意义的组合，必须逐级扩展（可能生成大量无用特征）。
- 如果将两者结合：用 SyMANTIC 的递归扩展生成候选池，用 LLM 对候选池做语义层面的筛选/排序/组合提议，是一种可能的 hybrid 方向。

**撞车判断**：不撞车。SyMANTIC 的特征扩展是纯数值操作符驱动的，0616-fex 的特征扩展（根据名称推测）涉及 LLM/skeleton/lifting 等概念，属于不同范式。但 SyMANTIC 是值得引用的相关工作，尤其当 0616-fex 涉及从数据中自动发现特征结构时。
