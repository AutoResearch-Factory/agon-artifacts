# The Entropy Mechanism of Reinforcement Learning for Reasoning Language Models (2505.22617)

- **Year**: 2025.05
- **Authors**: Ganqu Cui, Yuchen Zhang, Jiacheng Chen, Lifan Yuan, Zhi Wang, Yuxin Zuo, Haozhan Li, Yuchen Fan, Huayu Chen, Weize Chen, Zhiyuan Liu, Hao Peng, Lei Bai, Wanli Ouyang, Yu Cheng, Bowen Zhou, Ning Ding (Shanghai AI Lab, Tsinghua, UIUC, PKU, NJU, CUHK)
- **Venue**: arXiv
- **Keywords**: policy entropy, RL for LLM reasoning, entropy collapse, exploitation-exploration tradeoff, covariance regularization, Policy Gradient, GRPO, scaling predictability
- **One-liner**: 发现 RL 训练 LLM 推理时策略熵系统性坍缩，建立 R=-a exp(H)+b 可预测关系揭示性能由熵透支驱动，理论证明熵变化≈协方差(动作概率, 优势)，提出 Clip-Cov/KL-Cov 突破熵瓶颈。

## 精华提炼

### 管线/方法

整个研究 pipeline 分四个阶段：

1. **大规模观察 → 熵坍缩**：4 个模型家族 (Qwen2.5 0.5B–32B, Mistral 7B/Nemo/24B, LLaMA 3B/8B, DeepSeek-Math 7B) × 4 种 RL 算法 (GRPO, REINFORCE++, PRIME, RLOO) × 2 任务域 (Math, Coding) × 2400 gradient steps。所有实验一致观察到：策略熵在训练前 400 步消耗 87%，前 800 步消耗 94%，之后趋于零附近（<0.05），性能同时饱和。

2. **定量拟合 → 可预测定律**：对 200+ 数据点拟合 R = -a exp(H) + b，仅需 2 个系数精确描述熵-性能关系。用前 36 步（15% 训练量）拟合曲线可预测后续 200 步性能，RMSE 仅 0.9%（math）/ 1.2%（code）。系数 a,b 与模型参数规模呈 log-linear 关系，可用小模型系数外推大模型性能。

3. **理论分析 → 协方差驱动**：对 softmax 策略（LLM 的本质属性），一阶近似下熵变化 = -Cov(log π(a|s), z^{k+1} - z^k)。在 Policy Gradient 下退化为 -η·Cov(log π, π·A)；在 Natural Policy Gradient 下退化为 -η·Cov(log π, A)。协方差在训练全程保持正值，解释了熵单调递减。

4. **介入方法 → Clip-Cov/KL-Cov**：基于理论，按 token 级协方差 Cov(y_i) = (log π(y_i) - μ_log) · (A(y_i) - μ_A) 排序。Clip-Cov 随机采样高协方差区间 [ω_low, ω_high] 的 token 并 detach 梯度（r=2e-4）；KL-Cov 对 top-k（k=2e-3/2e-4 for 7B/32B）高协方差 token 施加 KL 惩罚（β=1）。

### 关键技术细节

- **熵定义**：token-level average entropy，H = -E_D[log π(y_t|y<t)]，按 batch 采样计算
- **RL 算法实现**：基于 veRL 框架 (HybridFlow)，learning rate = 5e-7（policy）/ 1e-6（PRM in PRIME），batch size 256，micro-batch 128，rollout 每 step 512 prompts × 8 responses
- **PPO 默认**：ε=0.2，ref-KL coefficient=0（故意不加 KL 正则以观察纯净熵动态）
- **温度**：训练时 temperature=1，eval 时 AIME/AMC 用 0.6，其余 greedy
- **训练数据分层**：Qwen/Mistral-24B 用 Eurus-2-RL-Math（更难），其他用 GSM8K（更简单）；coding：Qwen/Mistral-24B 用 AceCode + Eurus-2-RL-Code + KodCode
- **评测集**：MATH500, AIME 2024/2025, AMC, OlympiadBench, OMNI-MATH, Minerva, Eurus-2-RL-Code test, KodCode test
- **熵正则实验**：测试了 entropy loss (α=0.0001/0.001/0.005/0.01) 和 ref-KL (β=0.001/0.002/0.005/0.01)，结论是 naively 应用传统熵正则要么无效（小系数）、要么熵爆炸（大系数），要么性能下降
- **协方差分布**（Qwen2.5-7B step 1）：top 0.02% token 的协方差均值=5.654，top 0.2% = 3.112，top 2% = 1.385，全局均值仅 0.003。极少数 token 主导熵坍缩

### 核心结果

| 发现 | 具体数字 |
|------|---------|
| 熵坍缩速度 | 73% 熵消耗 + 76% 性能增益发生在前 200 步（1/12 训练） |
| 前 800 步占比 | 94% 熵消耗 + 93% 性能增益 |
| 拟合精确度 | 2 系数拟合 200+ 数据点，RMSE ~0.9–1.2% |
| 小模型预测大模型 | 系数 a,b 与 log(参数规模) 线性，可外推 |
| 传统熵正则失效 | entropy loss 要么无效要么熵爆炸；ref-KL 稳定熵但压低性能 |
| Clip-Cov (7B) | Avg +1.8% vs GRPO，AIME25 +6.2% |
| KL-Cov (7B) | Avg +2.0% vs GRPO，AIME24 +1.4%，AMC +2.7% |
| KL-Cov (32B) | Avg +6.4% vs GRPO，AIME24 +15.0%，AIME25 +14.6%，OlympiadBench +5.4% |
| 熵维持 | KL-Cov 在后期保持比 GRPO 高 10× 以上的熵值 |
| Clip-higher 对比 | 早期涨分但后期不稳定（32B 上 MATH500 反而跌到 77.2），Cov 方法全程稳定 |

**Surprising 发现**：
- 熵-性能关系对所有 RL 算法（GRPO/RLOO/PRIME）一致，说明系数 a,b 反映 model+data 的内在属性，与 advantage estimation 方式无关
- 仅干涉 10^-4 ~ 10^-3 比例的 token 就能完全改变熵曲线
- 更大模型（32B）从熵干预中获益更多（+15% AIME24 vs +1.4% 7B），暗示预训练潜力被熵坍缩锁住

### 明确局限

- **作者自述局限**：(1) 可预测性不普适——DeepScaler 等用不同策略模型或 off-policy 数据时熵行为不同；(2) 最优熵值仍未知——有控制的熵与性能之间未发现明确关系；(3) 理论推导基于 tabular softmax 假设，实际 LLM 的连续状态动作空间更复杂
- **审读发现的局限**：(1) 仅在数学推理和编程任务上验证，无 agents/对话/creative writing 等开放域；(2) 所有实验基于 GRPO 等 group-based PG 方法，未覆盖 RLHF/Reward Model 范式下的 PPO；(3) 没有真机/机器人实验；(4) Cov 定义在 response-level（bandit setting），token-level 细粒度分析缺失；(5) 未讨论 semantic entropy（不确定性分解为 aleatoric/epistemic）；(6) 训练数据量固定，未分析数据量对系数 a,b 的系统影响；(7) 所有实验在 veRL 框架上，跨框架泛化性未验证
