# Humanoid Locomotion and Manipulation: Current Progress and Challenges in Control, Planning, and Learning (2501.02116)

- **Year**: 2025.01
- **Authors**: Zhaoyuan Gu et al.; Georgia Institute of Technology, USC, TUM, Google DeepMind, The AI Institute, IHMC, Duke, Stanford, CNRS/LIRMM, NVIDIA, CMU, Harbin Institute of Technology
- **Venue**: arXiv survey
- **Keywords**: humanoid locomotion, loco-manipulation, multi-contact planning, MPC, whole-body control, reinforcement learning, imitation learning, tactile sensing, robot foundation models
- **One-liner**: 系统综述 humanoid loco-manipulation 的规划、控制、学习与 foundation model 挑战。

## 精华提炼

### 管线/方法

1. 这是一篇 survey，不提出单一新算法，而是把 humanoid locomotion and manipulation (HLM) 拆成一条方法谱系。
2. 论文对象限定为 bipedal humanoid robots，强调 whole-body motion，不重点讨论纯多指 dexterous hand manipulation。
3. 输入侧的核心问题是：机器人在真实人类环境中同时移动自身、移动物体、维持平衡、处理多接触和不确定感知。
4. 综述主线从背景定义开始：bipedal locomotion、bipedal navigation、whole-body manipulation、loco-manipulation。
5. 作者把 loco-manipulation 定义为同时包含 object movement 和 robot self mobility；whole-body loco-manipulation 还要求可使用全身表面接触。
6. 方法总图把领域分成四层：tactile sensing、traditional planning/control、learning-based approaches、foundation models。
7. 传感层先讨论视觉和 proprioception 已经成熟，再把重点放到更少被系统综述的 tactile sensing。
8. tactile sensing 被分为三类：hands tactile、feet tactile、whole-body tactile。
9. hand tactile 主要支撑 grasp force regulation、slip detection、object state/property estimation、in-hand manipulation。
10. foot tactile 主要支撑 ground reaction force、support region、terrain property、uneven foothold recognition。
11. whole-body tactile 主要支撑人机物理交互、全身 compliance、navigation among movable objects、whole-body grasp/lift。
12. 传统 planning/control 第一层是 multi-contact planning，要同时决定 contact mode、contact location、contact force 和 robot trajectory。
13. multi-contact planning 被分为 search-based、optimization-based、learning-based 三类。
14. search-based 方法用图搜索或采样展开 contact modes，再用 pose optimization 或 motion optimization 验证可行性。
15. optimization-based 方法核心是 Contact-Implicit Trajectory Optimization (CITO)，将 contact mode、force、position、motion 一起放进轨迹优化。
16. learning-based contact planning 的作用是加速 contact sequence 或 contact affordance prediction，例如学习 centroidal dynamics/contact sequence。
17. 传统 planning/control 第二层是 MPC，论文给出统一 OCP 形式：优化 state、control、constraint force trajectories，满足 dynamics、equality task、inequality task。
18. MPC 的动态模型分三类：simplified models、whole-body models、mixed-fidelity models。
19. simplified models 包括 SRBM、LIPM 等，用于高频在线规划，但对接触和物体动力学表达不足。
20. whole-body models 使用 centroidal dynamics、whole-body kinematics 或 joint-space whole-body dynamics，表达更准确但计算更重。
21. mixed-fidelity models 在同一 horizon 中组合不同精度模型，近端保真、远端简化，以获得可计算性和长期视野。
22. MPC 与物体交互建模被分成两路：把物体当 external wrench，或把物体动力学并入统一 robot-object system。
23. 对 static environment，MPC 主要规划接触力和质心/动量轨迹，不需要预测环境状态。
24. 对 manipulated object，MPC 需要 object state、mass、inertia、contact wrench evolution，难度显著上升。
25. 对 dynamic environment 或 deformable object，论文明确说 humanoid loco-manipulation 中仍 largely unexplored。
26. 传统 planning/control 第三层是 WBC，即在当前时刻解 inverse dynamics，输出 joint torque、constraint force、generalized acceleration。
27. WBC 与 MPC 的区别：MPC 解 receding horizon，WBC 解 instantaneous control；MPC 可用 reduced/full model，WBC 通常用 full-order Euler-Lagrangian dynamics。
28. WBC 任务统一写成线性 dynamic task，可作为 equality、inequality 或 quadratic cost。
29. WBC 方法分 closed-form 和 optimization-based 两类。
30. closed-form 包括 inverse dynamics control、operational-space control、stack of tasks，优点是快，缺点是 inequality task 难处理。
31. optimization-based WBC 通常是 QP，可用 strict task hierarchy 或 weighted soft tasks 管理冲突。
32. loco-manipulation WBC 分两种 interaction model：external wrench model 和 unified robot-object model。
33. learning 部分把 skill learning 目标分成 robustness/agility/safety 与 versatility/generalization。
34. RL-from-scratch 适合发现 novel behavior，但依赖 reward engineering、curriculum、sim-to-real、domain randomization。
35. IL 被分为 robot experience data 和 human data 两类。
36. robot experience 包括 policy execution 和 teleoperation，数据和机器人同分布但昂贵、稀缺。
37. human data 包括 motion capture 和 internet video，数据规模大但缺动作、触觉、力、环境交互信息，并有 embodiment gap。
38. hybrid methods 把 RL、IL、MPC、TO/WBC 组合起来，例如 teacher-student、trajectory reference tracking、residual trajectory augmentation。
39. versatile skill representation 分为 MOE、motion latent representation、goal representation、state-transition/world-model representation。
40. foundation model 部分先区分两种用法：用现成 LLM/VLM 做 high-level planning，或训练 robot foundation model / VLA 直接输出动作。
41. LLM/VLM 用于 humanoid 的当前主流是 skill selection、task graph、code/reward/pose/contact generation，而不是直接闭环控制全身。
42. humanoid RFM/VLA 的难点是高维 action space、unstable bipedal dynamics、全身多模态数据稀缺、onboard inference latency。
43. 论文把未来 humanoid RFM 输出建议为 end-effector/body poses，再由低层 controller/policy 保证 balance/safety。
44. 最终讨论比较 model-based 与 learning-based：model-based motion accuracy 高，RL robustness 高，IL versatility 高，VLA generalizability 有潜力但硬件证据不足。
45. 论文同时讨论 hardware bottlenecks：QDD actuators、电池、低 limb inertia、whole-body sensing、torsional/thermal limits。

### 关键技术细节

46. MPC 表中 SRBM-based humanoid loco-manipulation 可达 20 Hz 到 300 Hz；例如 Li et al. 2023 的 SRBM 外力建模 reported 300 Hz QP。
47. CD-based humanoid loco-manipulation 通常更慢：Dai centroidal kinematics 约 5 Hz/offline，Polverini 2020 约 10 Hz interior-point，Elobaid 2023 约 5 Hz。
48. WBD-based Memmo 2021 表中 reported 100 Hz DDP，但需要 motion memory/warm start，依赖数据覆盖。
49. NMPC speed-up 表把方法分为 dynamics modeling、NLP structure exploitation、linearization、warm start。
50. simplified model accuracy 低但效率高；whole-body model accuracy very high 但效率 very low；mixed-fidelity 两者居中。
51. direct methods 的 NLP 复杂度可达 O(N^3)，稀疏 direct multiple shooting/collocation 可把结构利用到接近 O(N)。
52. ReLU-QP 被列为 GPU-accelerated QP，论文引用其在 high-dimensional balancing 中把 control frequency 从 206 Hz 提升到最高 2600 Hz。
53. sampling-based MPC 依赖 MPPI、MJPC、MuJoCo/IsaacLab GPU/CPU parallel rollout，适合不连续接触但高维 humanoid 仍难。
54. WBC 的 full-order dynamics 写成 M qddot - J^T lambda - S^T tau = -C qdot - G。
55. contact constraint 写成 J_c qdot = 0，因此 J_c qddot + Jdot_c qdot = 0。
56. WBC decision variables 是 qddot、lambda、tau，因此动力学方程和 task template 在单步上可成为线性约束。
57. external wrench WBC 策略包括 simultaneous optimization、pre-optimization、post-optimization。
58. unified robot-object WBC 把被操作物体作为 passive object 或另一个 robot，并用 action-reaction force pair 耦合。
59. RL 算法对比中，PPO/A3C 被归为 on-policy、sample inefficient 但 policy update 快且可扩展，适合 parallel simulation。
60. SAC/TD3/DDPG 被归为 off-policy、sample-efficient 但训练更新更重，适合真实机器人数据昂贵场景。
61. sim-to-real 方案包括 domain randomization、system identification、domain adaptation、real-to-sim alignment。
62. 论文指出 MuJoCo soft contact 可能产生 penetration artifacts，PhysX friction cone pyramid approximation 也会扩大 sim-to-real gap。
63. teleoperation 数据对 humanoid 的限制在于多数系统只覆盖 upper-body manipulation，完整 gait 需要 IMU suit、mocap 或 exoskeleton。
64. human motion data 的核心缺口不是 pose 数量，而是缺 tactile/force、object motion、ego observation、intent/cognitive action。
65. foundation model 成本例子：LLaMA 训练用了 34 days on 992 NVIDIA A100-80B GPUs。
66. VLA 输出动作的 tokenization 难点在 humanoid 上更严重，因为高 DoF action 用 stringified numbers token-inefficient。
67. pi0 被引用为 diffusion head 可实现 up to 50 Hz bimanual manipulator with wheelbase，但 humanoid 全身还没有同等级证明。
68. hardware 细节：QDD actuator 通常 gear ratio 小于 10:1，backdrivable、force-control bandwidth 高，但大电流带来 power electronics strain 和 overheating。
69. MIT Humanoid 被指出约 75% mass 在 torso，用 remote actuation 降低 limb inertia。
70. 电池例子：Apptronik Apollo reported 4 hours，Unitree G1 reported 2 hours，但很多实验仍依赖 tethered power。

### 核心结果

71. 论文不是 benchmark paper，没有单一 SOTA 数字；核心结果是高质量分类框架和跨范式比较。
72. 最重要的结论之一：humanoid loco-manipulation 不能只靠 locomotion 或 manipulation 子模块拼接，需要显式处理 shared contact、balance、object dynamics。
73. 另一个结论：model-based 和 learning-based 不是对立关系，sim-to-real RL 本身也依赖 simulator dynamics，可视作隐式 model-based。
74. model-based 方法在 motion accuracy 和可解释约束上占优，适合高精度、强安全约束、任务可建模场景。
75. learning-based 方法在 robustness、versatility、human-data reuse 上更有潜力，但受 sim-to-real、数据质量和 benchmark 缺失制约。
76. whole-body tactile sensing 被提升为关键 modality，因为 vision/proprioception 对 occluded contact、patch contact、unexpected collision 不够。
77. dynamic environment 与 deformable object 是论文明确标出的欠探索方向，尤其缺少和 humanoid balance/whole-body planning 紧耦合的方法。
78. Contact-implicit planning 对 unified contact/motion 很有吸引力，但 humanoid real-time loco-manipulation 还没有成熟硬件级解法。
79. MPC 的工程核心不是求最精确模型，而是在 model fidelity、constraint relaxation、solver speed、online robustness 中做任务相关折中。
80. WBC 短期仍会是 humanoid low-level control 主力，尤其是 optimization-based QP WBC。
81. IL from robot experience 是目前获取 expert-level skill 的可靠路线，但 teleoperation/full-body data scaling 是瓶颈。
82. IL from human data 是通往 humanoid generalist 的长期路线，但必须越过 embodiment gap 和 missing force/action modalities。
83. Foundation models 在 humanoid 上近期更实际的角色是高层语义规划、skill selection、reward/code/pose/contact authoring。
84. End-to-end humanoid VLA/RFM 需要更多真实机器人数据、合成数据和安全低层控制栈，当前证据主要来自 manipulation 或 simulation。
85. benchmarking 缺失是重大限制：HumanoidBench、Mimicking-Bench、SMPLOlympics 还只覆盖部分技能，且 sim-to-real 价值未定。

### 明确局限

86. 作为 survey，它不提供统一实验、独立 benchmark 或跨方法复现实验；很多比较依赖各论文自报结果。
87. 表格里也承认 algorithm performance 很多是 anecdotal，受实现、机器人平台、调参和场景强影响。
88. 反引文显示 2026 年已经有大量 work 沿着 HLM、whole-body manipulation、humanoid data generation、MPC-guided RL 继续推进，本 survey 的前沿性会快速被刷新。
89. deformable object 只在 MPC interaction section 简短讨论，没有系统梳理 soft-object dynamics、cloth/rope/fluid/food 等细分任务。
90. tactile sensing 强调很到位，但缺少对 tactile sensor benchmarking、sensor cost、long-term durability 的定量比较。
91. foundation model 讨论包含 GR00T N1、Gemini Robotics 等 2025 work，但这些系统的公开技术细节和独立复现很有限。
92. paper 对 humanoid manipulation hand design 的讨论主要放在草稿/删除文本里，主文只保留较少手部设计和 dexterous manipulation 分析。
93. 缺少 soft-body/soft-object 与 rigid-object loco-manipulation 的明确 taxonomy。
94. 缺少对 BFM/behavior foundation model 如何具体处理 force/tactile/contact tokens 的细化方案。
95. 对 safety/legal/economic 的讨论偏展望，缺少严格的 risk model 或 failure taxonomy。
96. 对 real-world deployment 的描述覆盖工厂、家庭、医院、救援等，但没有可审计 deployment dataset。
97. 对 learning data 的观点很有价值，但还没有给出可操作的数据 schema 或 benchmark protocol。

## Read by: 0612-soft-bfm (2026-06-12)

这篇 survey 不和 `0612-soft-bfm` 正面撞车，因为它不提出 soft-object manipulation、deformable object dynamics 或 BFM/RFM 的新方法；它是 humanoid loco-manipulation 的总图。它的价值在于把我们课题所在的空位说清楚：rigid-object humanoid HLM 已有 MPC/WBC/RL/IL/VLA 的方法谱系，但 deformable objects 只被简短列为 underexplored，且还缺 object deformation state、contact patch、material property、tactile/force feedback 的统一接口。

对我们的可用工具包括四类。第一，使用 tactile sensing 作为 soft-object contact 的核心观测，不只依赖 RGB/RGB-D。第二，用 unified robot-object model 或 learned object dynamics 替代 external wrench/static approximation。第三，让 BFM/RFM 只输出 object-centric subgoals、contact proposals、body/end-effector pose references，再交给 MPC/WBC/RL policy 保证 balance 和 safety。第四，构造数据集时不要只收人或机器人的 pose，而要收 object deformation、contact/force/tactile、egocentric vision、task language 和 intent label。

差异化建议：把贡献明确落在 soft/deformable object 的 physics-grounded representation 与 evaluation 上，而不是泛泛说 humanoid foundation model。可引用本文证明 humanoid HLM 的传统栈、tactile 缺口、数据稀缺和 foundation model 接口难点；再强调我们的 novelty 是软物体状态/接触/材料属性如何被 BFM 表征、预测和用于控制。下一轮应优先查 `tactile whole-body manipulation`, `deformable object humanoid manipulation`, `contact-aware humanoid object interaction`, `robot foundation model force tactile`, `soft object dynamics foundation model`, `humanoid loco-manipulation benchmark`。
