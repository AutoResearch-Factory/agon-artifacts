# LLM-ODE: Data-driven Discovery of Dynamical Systems with Large Language Models (2603.20910)

- **Year**: 2026.03
- **Authors**: Amirmohammad Ziaei Bideh, Jonathan Gryak (The Graduate Center, CUNY; Queens College and The Graduate Center, CUNY)
- **Venue**: GECCO '26 / arXiv
- **Keywords**: LLM-guided symbolic regression, genetic programming, ODE discovery, dynamical systems, Pareto front, PySR, SINDy, ODEFormer
- **One-liner**: 用 LLM 替代 GP 随机变异算子, 在 91 个 ODE 系统上提升符号方程发现效率。

## 精华提炼

### 管线/方法

1. 输入是一个未知自治 ODE 系统的观测轨迹 $\mathbf{X}\in\mathbb{R}^{N\times D}$。
2. 目标是恢复每个状态变量的导数方程 $\dot{x_i}=f_i(\mathbf{x})$。
3. 论文不直接搜索完整系统, 而是把系统拆成 $D$ 个单方程 SR 问题。
4. 对每个状态变量 $x_i$, 先用有限差分估计训练和验证导数 $dx_i/dt$。
5. 每个方程由一个 LLM-aided symbolic regression 子过程独立搜索。
6. 搜索空间是表达式树, 复杂度 $\mathcal{C}(f)$ 定义为树节点数。
7. 系统复杂度是各个方程复杂度之和 $\mathcal{C}(S)=\sum_i\mathcal{C}(f_i)$。
8. 单方程搜索采用 island-based evolutionary strategy, 多个 island 并行维护候选表达式种群。
9. 每个 island 初始化为 2 个随机生成的复杂度为 3 的简单方程。
10. 每轮进化先计算每个候选的 fitness, 即负 MSE: $-\mathrm{MSE}(\dot{\mathbf{x}}, f_i(\mathbf{x}))$。
11. 在 island 内按 $\exp(s)$ 的 softmax 权重抽取 $k$ 个候选作为 LLM in-context examples。
12. LLM 根据这些 elite 或较高分候选生成 $b$ 个新的表达式。
13. Prompt 要求 LLM 把所有数值常数写成占位符 `C`, 后续由数值优化器拟合。
14. Prompt 中显式列出可用算子: `+`, `-`, `*`, `**`, `/`, `sin`, `log`, `exp`, `abs`。
15. 代表性 prompt 设定是给 8 个从差到好的方程, 要 LLM 输出 3 个从简单到复杂的新方程。
16. LLM 生成的新表达式先固定符号结构, 再用 BFGS 优化常数和系数。
17. 常数优化目标是在训练导数上最小化 MSE。
18. 优化后的表达式被插回对应 island, 继续参与后续选择。
19. 每隔 $n_{\text{refine}}=5$ 轮执行一次 refinement。
20. refinement 会剪枝所有 island, 并从每个 island 抽取若干方程加入另一个随机 island。
21. 这个交换机制用于避免单个 island 过早集中在局部表达式区域。
22. 每个方程搜索结束后, 在验证数据上计算复杂度和误差的 Pareto front。
23. 对 $D$ 个状态变量, 方法取各方程 Pareto front 的 Cartesian product 得到候选系统池。
24. 再对完整系统按总复杂度和轨迹级 MSE 计算 system-level Pareto front。
25. 最终系统从 system Pareto front 上选择最大边际收益点。
26. 选择规则是最大化 $\Delta\mathcal{T}/\Delta\mathcal{C}$, 并要求轨迹 fitness 至少达到 $h\mathcal{T}_{\max}$。
27. 论文设 $h=0.1$, 用于避免过早选择过简单但明显不准的系统。
28. 系统级 fitness 通过数值积分候选 ODE 后与观测轨迹比较得到。
29. 这使得单方程导数拟合和完整轨迹重建被分成两级验证。
30. 关键设计是把 LLM 当作 evolution operator, 而不是把 LLM 当作一次性方程生成器。

### 关键技术细节

1. LLM 后端包括 `Qwen3-30B-A3B-Instruct-2507`, `Ministral-3-14B-Instruct-2512`, `Olmo-3.1-32B-Instruct`。
2. 三个模型都通过 vLLM serving, 依赖 PagedAttention 提高吞吐。
3. LLM-ODE 使用 4 个 independent evolutionary islands。
4. 每个 island 初始化 2 个 seed expressions。
5. 搜索总轮数是 200 iterations。
6. in-context examples 数量是 $k=8$。
7. 每次 prompt 采样新表达式数量是 $b=3$。
8. refinement frequency 是 $n_{\text{refine}}=5$。
9. 论文没有在正文中给出 $n_{\text{mix}}$ 的具体数值。
10. 导数估计使用 fourth-order finite-difference scheme。
11. 常数优化使用 BFGS。
12. 候选系统积分使用 SciPy ODE solver, 明确说是 explicit Runge-Kutta integration。
13. 为降低 LLM 记忆常见方程的风险, prompt 删除系统描述、状态变量名和物理单位。
14. 输入变量统一写作 `x_0`, `x_1`, `x_2` 等, 避免让模型直接识别 Lorenz、SIR 等系统。
15. 评测数据包含 91 个真实模型仿真的 dynamical systems。
16. 数据集从 ODEBench 的 63 个系统开始, 再补充高维和 chaotic dynamics。
17. 维度分布为 $D=1$ 有 23 个系统, $D=2$ 有 28 个, $D=3$ 有 22 个, $D=4$ 有 18 个。
18. 每个系统生成两条不同初值轨迹, 一条训练, 一条测试。
19. 轨迹积分区间是 $t=0$ 到 $t=10$。
20. 采样间隔是 0.1 秒。
21. 参数优化使用 $t=0$ 到 $t=5$ 的轨迹片段。
22. 模型选择使用完整训练轨迹。
23. 成功发现的指标是测试轨迹 NMSE 低于阈值 $\lambda$。
24. 阈值从 $10^{-1}$ 到 $10^{-6}$ 分层报告。
25. Baseline PySR 配置为 200 iterations, 4 islands, 每轮 3 cycles, 20 individuals, 且使用同一算子集。
26. Baseline SINDy 使用 STLSQ, 网格包括 threshold 0.05/0.1/0.15, alpha 0.025/0.05/0.075, max iterations 20/100。
27. SINDy polynomial degree 搜索 $1$ 到 $10$。
28. SINDy basis function 有三组: polynomial; polynomial+sin+cos+exp; polynomial+sin+cos+log+exp+sqrt+inverse。
29. Baseline ODEFormer 设置 beam size 50, temperature 0.1。
30. ODEFormer 的 50 个 beam outputs 被当作 Pareto front 候选系统。
31. 代码和数据生成流程声明发布在 `https://github.com/gryaklab/llm-ode`。

### 核心结果

1. 论文主表报告不同维度、不同 NMSE 阈值下成功发现的系统数量。
2. 在 $D=1$, 严格阈值 $10^{-6}$ 下, Mistral/Olmo/Qwen 都发现 8/23 个系统, PySR 发现 6/23, SINDy 3/23, ODEFormer 0/23。
3. 在 $D=2$, 阈值 $10^{-6}$ 下, Qwen 发现 3/28, SINDy 3/28, Mistral/PySR/Olmo 各为 1/28。
4. 在 $D=3$, 阈值 $10^{-4}$ 下, Mistral 和 Qwen 各发现 2/22, PySR 和 SINDy 各 1/22, ODEFormer 0/22。
5. 在 $D=4$, 阈值 $10^{-6}$ 下, 只有 Mistral 还发现 3/18 个系统, 其他方法均为 0/18。
6. 按总数看, 阈值 $10^{-3}$ 下 Qwen 发现 32/91, Mistral 29/91, Olmo 29/91, SINDy 29/91, PySR 25/91, ODEFormer 11/91。
7. 阈值 $10^{-6}$ 下 Mistral 发现 12/91, Qwen 11/91, Olmo 9/91, PySR 7/91, SINDy 6/91, ODEFormer 0/91。
8. 低维系统明显更容易恢复, 维度升到 3 和 4 后所有方法都有大幅退化。
9. SINDy 在 $D=2$ 和部分 $D=3$ 阈值下仍很强, 说明 sparse library 方法在中低维合适库设定中仍有竞争力。
10. SINDy 到 $D=4$ 基本崩溃, 作者归因于候选 library terms 指数增长。
11. ODEFormer 在全部维度和阈值上都显著弱于 LLM-ODE variants。
12. PySR 是最强非 LLM GP baseline, 但在高维和严格阈值下落后于 LLM-ODE。
13. 收敛曲线显示 LLM-ODE variants 在前 50 iterations 就能达到 PySR 200 iterations 后仍达不到的 discovery rate。
14. 在 $\lambda=10^{-3}$ 和 $\lambda=10^{-5}$ 这类阈值上, 早期迭代优势尤其明显。
15. Pareto front 分析显示 LLM-ODE 在训练初期获得的 system Pareto front 点数更多。
16. 这个优势随迭代继续会缩小, 说明随机 GP 最终也能补上部分探索覆盖, 但样本效率较低。
17. 作者把 Pareto front richness 解释为更好的多目标探索和结构多样性。
18. 一个反直觉点是 SINDy 在 $D=2$ 的多个阈值上不输 LLM-ODE, 所以论文不是简单的 LLM 全面碾压故事。
19. 另一个反直觉点是三个 LLM backend 没有单调强弱关系, Mistral 在最严格阈值和 $D=4$ 上表现更稳。
20. 结果更像是 "LLM prior 帮助 GP 搜索早期覆盖高质量结构", 而不是 "LLM 单独会发现正确方程"。

### 明确局限

1. 作者明确承认 LLM inference latency 是计算瓶颈。
2. LLM serving 需要专用硬件, 普通用户不一定能复现同样吞吐。
3. 量化或低秩近似模型可能降低成本, 但会牺牲性能。
4. 方法虽然比 classical GP 更能扩到 $D=4$, 但高维问题仍然困难。
5. $D=4$ 的 system pool 在最后迭代达到 $O(10^4)$, 仍可管理, 但已经显示 Cartesian product 组合会快速膨胀。
6. system Pareto front 计算昂贵, 因此只在 GP loop 结束后执行, 不是每轮闭环反馈。
7. 数据全部是仿真系统, 不是真实实验观测的 noisy trajectory。
8. 论文没有系统报告噪声鲁棒性。
9. 论文没有独立第三方复现实验。
10. 防记忆主要靠删除系统 metadata, 但 benchmark 本身包含 Lorenz、SIR、Van der Pol 等常见方程族, 仍不能完全排除结构记忆。
11. 论文声称是首个 LLM+evolution framework 用于 coupled ODE systems, 但最近半年已有多篇 LLM equation discovery 和 LLM-aided SINDy 邻近工作, 需要在 2026 版 related work 中更细地区分。
12. 算子集固定, 不讨论算子集错配时的失败模式。
13. 常数优化依赖 BFGS, 对非凸参数拟合的失败没有专门诊断。
14. LLM 生成的表达式如果结构正确但常数初值或拟合不稳, 可能被早期低分淘汰。
15. 作者报告了 Pareto front 点数, 但没有直接展示表达式等价类去重, 所以 "多样性" 可能部分来自语法差异而非语义差异。
16. 论文没有把 wall-clock 时间作为主要指标, 因此 iteration-level 加速不一定等于实际运行更快。
17. 论文没有和 2026 年新出现的 granular feedback、agent repair、LLM policy optimization 系列方法正面对比。
18. 对 FEX 来说, 最大局限是问题 setting 不同: 本文做数据驱动 ODE system identification, 不是 PDE boundary-value/physics residual 求解。

### 跟我们的关系

1. `0616-fex` 的核心是 RL for symbolic regression, 尤其是 FEX 及其 LLM/FEX warm-start 方向。
2. LLM-ODE 与 FEX 不在同一任务设置: 它发现 ODE governing equations, FEX 主要搜索 PDE 解或物理律表达式。
3. 方法层面有明显重叠: 都是表达式搜索、常数优化、复杂度-误差 tradeoff、从数据/残差信号中选择符号结构。
4. 若我们的 claim 是 "用 LLM 从 elite expressions 中生成新候选来加速符号搜索", 本文已经覆盖了很大一块, 撞车风险中高。
5. 若我们的 claim 是 "LLM 生成 PDE 解表达式骨架作为 FEX policy-gradient controller 的 warm-start", 本文是强 related work 但不直接撞车。
6. 关键差异化应放在 FEX 的 PDE/physics residual setting、boundary/initial condition 约束、FEX 的 $S_k$ 理论保证、以及 high-dimensional PIDE/Poisson/Schrodinger 这类任务。
7. 本文没有学习一个持久 RL policy, 而是用 LLM 作为在线 evolutionary operator; FEX 可以强调 RL controller 的策略学习和可迁移搜索分布。
8. 本文的 "elite equations -> LLM proposals -> BFGS constants -> Pareto front" 可以直接作为 FEX LLM-skeleton idea 的 baseline。
9. 本文的 anti-memorization 处理很值得复用: prompt 不给系统名、变量名、物理单位, 只给匿名变量和候选表达式。
10. 对 FEX, 需要进一步做 oracle skeleton、operator-set-only、LLM skeleton、random skeleton、corrupted skeleton 的先验粒度相图, 否则容易被本文和 LLM-SR 系列压缩 novelty。
11. 本文没有处理 equality graph/equivalent expression pruning; 这给 FEX + EGG 或 Rao-Blackwellized policy gradient 留出空间。
12. 本文没有 wall-clock 成本归一化, FEX 若声称加速, 必须报告 token cost、LLM latency、BFGS 次数和实际时间。
13. 最近 6 个月 arxiv-tool 检查还显示 IGSR (2605.29184)、KeplerAgent (2602.12259)、PiT-PO (2602.10576)、STRIDE (2605.17790)、LLM-DMD (2601.05632) 等邻近工作, 说明 LLM+SR/equation discovery 已非常拥挤。
14. 本文应引用, 特别是在 "LLM as evolutionary operator for symbolic regression / dynamical systems" 段落中。
15. 下一轮搜索建议围绕 `LLM evolutionary operator symbolic regression`, `LLM-aided SINDy ODEBench`, `FEX LLM skeleton warm-start`, `granular feedback symbolic regression`, `Pareto front symbolic regression dynamical systems`。

## Read by: 0616-fex (2026-06-16)

从 FEX/RL symbolic regression 视角看, LLM-ODE 是必须收录的强相关工作。它不是 FEX, 也不是 PDE solver, 但它已经把 "LLM 根据高分表达式生成新候选 + BFGS 拟合常数 + GP/Pareto 搜索" 这条路线在 coupled ODE discovery 上跑通, 并用 91 个 dynamical systems 给出强于 PySR 和 ODEFormer 的证据。

撞车风险取决于我们怎么表述 FEX idea。若我们只说 "用 LLM 引导表达式搜索", 本文会构成明显 prior work。若我们表述为 "LLM 生成不同粒度的 PDE 解骨架, 作为 FEX policy-gradient controller 的 warm-start, 并研究 operator set / subtree / full skeleton 对 FEX 搜索效率的相图", 则本文是相关基线而非直接撞车。

可复用模块包括: elite equation prompt, 常数占位符 `C`, BFGS 后拟合, island exchange, equation-level Pareto 到 system-level Pareto 的两级筛选, 以及匿名变量防记忆设置。需要警惕的是本文没有 wall-clock/token 成本对照、没有 noisy real data、没有 equality graph 去重, 这些正好是 FEX 后续可以补强的差异点。

引用建议: 该引。它应放在 LLM-guided evolutionary symbolic regression / dynamical system discovery 相关工作中。差异化时要明确 FEX 的任务是 PDE/physics residual 或高维解析解搜索, 不是从轨迹恢复 ODE 右端项; FEX 的核心资产是表达空间理论、RL controller 和 Adam+BFGS 精调, 而不是单纯让 LLM 生成候选表达式。
