# SymPlex: A Structure-Aware Transformer for Symbolic PDE Solving (2602.03816)

- **Year**: 2026.02
- **Authors**: Yesom Park (UCLA Math), Annie C. Lu (UCLA Math), Shao-Ching Huang (UCLA OARC), Qiyang Hu (UCLA OARC), Y. Sungtaek Ju (UCLA Mech&Aero), Stanley Osher (UCLA Math)
- **Venue**: ICML 2026
- **Keywords**: symbolic PDE solving, structure-aware Transformer, reinforcement learning, tree-relative attention, analytical solutions, grammar-constrained decoding, curriculum learning
- **One-liner**: 用结构感知 Transformer + RL 在符号表达式空间中搜索 PDE 解析解，无需真解，对非光滑和参数化 PDE 达 100% 符号恢复率。

## 精华提炼

### 管线/方法

**问题定义**：给定一个 PDE（含边界条件和初始条件），要找到其解析解（closed-form 符号表达式），且无法获得 ground-truth 表达式。

**整体流程**：
1. **SymFormer 自回归生成**：以 PDE 的坐标/参数为条件，通过结构感知 Transformer 解码器自动生成候选符号表达式（前缀序抽象语法树 AST）。每一步都受语法约束和深度预算约束。
2. **常数优化**：对每个生成的表达式树 T，使用梯度下降（Adam, lr=0.02, 50步）优化其中的可训练常数 c。这是离散结构搜索与连续参数拟合的首次解耦。
3. **PDE 奖励计算**：用 PINN 风格的 L2 残差计算 PDE 损失 + BC 损失 + IC 损失，再取倒数作为奖励 R = 1/(1+sqrt(E))。对 Hamilton-Jacobi 型方程使用基于特征的损失替代标准 PINN 残差以避免多解性。
4. **多样感知 Top-K 记忆**：维护 K=10 个候选池，同时检查结构多样性（Levenshtein 距离）和语义/行为多样性（在同测试点计算数值差异），确保池中表达式在语法和功能上都真正不同。
5. **策略优化**：rank-based 奖励 + 深度加权 + 熵正则化 (\lambda_ent = 0.3) + 模仿损失（从 top-k 池借力），Adam lr=5e-4, ReduceLROnPlateau, grad clip 5.0。
6. **课程学习**：3 阶段递进——Stage 1 只用空间变量学初始条件，Stage 2 引入时间学完整 PDE（参数固定），Stage 3 引入参数 kappa 学完整参数化解。每阶段最多 500 epoch，奖励 >0.99 提前推进。

### 关键技术细节

- **SymFormer 架构**：4 层 Transformer 解码器、embedding dim 64、FFN hidden dim 128、8 头注意力。
- **树关系相对自注意力**：6 种 token 间关系（self/parent/child/sibling/ancestor/other），每种关系有可学习的 embedding R_rij，与 key 相加后再算 attention score。AST 根据 token arity 动态重建，不需要外部 parse tree。
- **遍历感知位置编码**：保留传统 sinusoidal positional encoding 沿前缀位置编码，与树关系注意力互补。
- **语法约束自回归生成**：每一步根部分树状态确定合法 token 集合（叶节点只能选变量/常数，内部节点必须满足 arity），过滤退化子表达式（x-x, x/x），深度预算感知。
- **词汇表**：二元运算符 {+, -, *, /}、一元运算符 {neg, relu, sqrt, square, sin, cos, exp, ...}、变量 {x0,...,xn, t, kappa}、const。
- **超参**：最大深度 7-10，batch size 64，lambda_BC = lambda_IC = 10，N_PDE=200, N_BC=N_IC=80，entropy coeff 0.3，top-k size 10，模仿损失 temp 0.1，常数优化 200 步重优化每 10 epoch。
- **训练硬件**：单张 NVIDIA GV100 (TITAN V)。

**与数值/神经方法对比**（Burgers 方程）：
- SymPlex：误差 0.0，推理时间 6.27e-5 s，存储 0.07 MB
- WENO：误差 1e-4，推理 0.26 s（需重算），存储 131.17 MB
- PINN：误差 7e-4，推理 0.073 s，存储 69.94 MB

**Surprising/反直觉发现**：
- SymPlex 的推理速度比 PINN 快约 1000 倍（6.27e-5 vs 0.073 s），存储压缩比约 1000 倍（0.07 vs 69.94 MB）——符号解在推理和存储上的效率远超神经方法。
- 所有 7 个测试 PDE 都达到了精确符号恢复（MSE=0, SRR=100%），包括含 kinks 的非光滑 PDE（Eikonal、Burgers），这是此前所有方法都没做到的。
- FEX 在 Burgers 上也能达 7.59e-15 的极低 MSE，但 SRR 仅 10%——低数值误差不等于正确的符号结构。

### 明确局限

- **高维 PDE 仍困难**：搜索空间随维度和深度组合爆炸增长。
- **符号词表选择**：对于复杂 PDE，选择合适的算子和函数需要物理/数学领域知识，缺乏自动方法。
- **训练昂贵**：Transformer + RL 训练需 642s（Burgers），峰值内存 251 MB，远高于 WENO 的 0.35s。
- **收敛性缺乏严格保证**：理论保证仅假设全局最优可实现，对随机梯度训练的实际收敛性不提供保证。
- **误差稳定性分析未做**：当 PDE 解析解不在假设类中时，小残差是否意味着小 L2 误差没有定量保证。
- **仅测试了 2D 问题**：空间维度限于 2D，未见 3D 或更高维实验。
- **没有真机/真实物理实验**：全是合成 PDE 基准。

### 跟我们的关系

**与 0616-fex 课题的关系**：
- SymPlex 和 FEX（Finite Expression Method）直接相关：SymPlex 在实验中以 FEX 作为主要基线之一，并且均在"符号表达式搜索 PDE 解"这一范畴内。
- SymPlex 在实验结果中全面碾压 FEX：所有 7 个测试示例 SymPlex 均达 SRR=100%，而 FEX 仅在 Poisson（100%）和 Burgers（10%）上有非零 SRR。
- **关键的差异化方向**：FEX 用确定性搜索组合算子，没有学习组件；SymPlex 用 Transformer + RL 学习策略。如果 0616-fex 方向的重点在确定性/可解释的符号方法，那 SymPlex 代表"以学习代搜索"的竞争范式。
- **可用工具**：SymFormer 的树关系注意力和语法约束解码可以直接用在符号回归任务中。课程学习策略（空间->时空->参数）是一个可直接复用的训练调度。
- **撞车评估**：如果 0616-fex 的 idea 是"用 FEX 风格的确定性搜索 + 更高效的组合策略"，与 SymPlex 不直接冲突。但如果 0616-fex 包含神经网络/Transformer/RL 组件，则需仔细对照 SymPlex 的 claim。
