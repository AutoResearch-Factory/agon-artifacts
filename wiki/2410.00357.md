# Neural Scaling Laws of Deep ReLU and Deep Operator Network: A Theoretical Study (2410.00357)

- **Year**: 2024.10
- **Authors**: Hao Liu (HKBU), Zecheng Zhang (Notre Dame), Wenjing Liao (GaTech), Hayden Schaeffer (UCLA)
- **Venue**: arXiv preprint
- **Keywords**: deep operator learning, neural scaling law, approximation theory, generalization theory, DeepONet, curse of dimensionality, low-dimensional structure
- **One-liner**: 为 DeepONet 建立了完整的逼近误差与泛化误差理论框架，量化了模型/数据 scaling law，并证明利用输入函数低维结构可将对数收敛率提升为幂律收敛率。

## 精华提炼

### 管线/方法

1. **问题设定**：学习 Lipschitz 算子 G: U→V，其中 U⊂L²(Ω_U)、V⊂L^∞(Ω_V)。输入为 n 对函数样本 {(u_i, v_i)}，v_i = G(u_i) + noise。

2. **DeepONet 架构**（Chen & Chen 风格）：
   - Branch Net（分支网络）：以输入函数 u 的离散化向量 ub ∈ R^{n_x} 为输入，输出 N 个系数 {ã_k(ub)}
   - Trunk Net（主干网络）：以输出域点 y ∈ Ω_V 为输入，输出 N 个基函数值 {q̃_k(y)}
   - 最终输出：G_NN(ub)(y) = CL_{β_V}(Σ_{k=1}^N ã_k(ub)·q̃_k(y))，经 clipping 确保有界

3. **证明管道**：
   - Step 1：用 Trunk Net（Theorem 3 / function approximation）将输出函数在 Ω_V 上做 piecewise constant 逼近（partition of unity），将算子学习转化为 N 个 functional 的学习问题
   - Step 2：用 Branch Net（Theorem 6 / functional approximation）逼近每个 functional f_k(G(u)) = G(u)(c_k)，其中 c_k 是 Ω_V 上的离散点
   - Step 3：泛化误差分析 → 覆盖数（covering number）估计 → bias-variance trade-off → 最终界

### 关键技术细节

- **网络架构超参数**（一般情况）：
  - Trunk Net：L₁ = O(log ε⁻¹), p₁ = O(1), K₁ = O(log ε⁻¹), κ₁ = O(ε^{-d₂-1})
  - Branch Net：L₂ = O(ε^{-2d₁} log ε⁻¹), p₂ = O((C_F√c_U)^{c_U} ε^{-c_U})，极其庞大
  - N（基函数数量）= O(ε^{-d₂})，c_U（输入域覆盖球数）= O(ε^{-d₁})

- **低维情况改进**：
  - 假设输入函数 U 可由 b_U 个正交基 {ω_k} 展开（线性子空间结构）
  - Branch Net 简化为：L₂ = O(log ε⁻¹), p₂ = O(ε^{-b_U}), K₂ = O(ε^{-b_U}(log ε⁻¹ + n_x))
  - 网络规模从超指数级降为多项式级

- **证明核心技巧**：
  - 对 Ω_U 做 δ-覆盖（Euclidean balls），利用 partition of unity 构造连续函数逼近
  - Lipschitz functional 的逼近：先通过 partition of unity 将 functional 转化为有限维 Lipschitz 函数，再用 ReLU 网络逼近
  - 泛化误差分解：T₁（训练误差上界，bias-variance）+ T₂（generalization gap，用 covering number + moment generating function 控制）
  - Covering number 估计：基于参数离散化（uniform grid on parameters）

- **关键假设**：
  - U 中函数：Lipschitz（常数 ≤ L_U）、有界（‖u‖_∞ ≤ β_U）
  - V 中函数：Lipschitz（常数 ≤ L_V）、有界（‖v‖_∞ ≤ β_V）
  - G 是 Lipschitz 算子：‖G(u₁)-G(u₂)‖_{L^∞} ≤ L_G‖u₁-u₂‖_{L²}
  - 低维情况额外假设：U 中函数可被 b_U 个正交基展开，且离散网格满足 quadrature 条件

### 核心结果

| 场景 | 逼近误差（模型 scaling） | 平方泛化误差（数据 scaling） |
|------|------------------------|---------------------------|
| 一般 Lipschitz 算子 | (log N_# / log log N_#)^{-1/d₁} | (log n / log log n)^{-2/d₁} |
| U 由 b_U 个正交基展开 | N_#^{-1/(b_U+d₂)} | n^{-2/(2+b_U+d₂)} log³ n |

- **一般情况收敛极慢**：近似误差以 log N_# 的幂律衰减，而非 N_# 的幂律。这意味着理论上学习一般 Lipschitz 算子存在维度灾难，网络规模需随精度呈超指数增长。
- **低维结构是关键**：当输入函数具有低维线性结构时，收敛率从 log 的幂律跃升为 n 和 N_# 的幂律。这解释了为什么实际中 DeepONet 能在 PDE 问题上表现良好——真实的 PDE 解算子通常具有低维结构。
- **与经验观察一致**：低维情况下的 n^{-2/(2+b_U+d₂)} 速率与 lu2021learning 和 de2022cost 的经验幂律观察一致。
- **最优性讨论**：逼近速率与 mhaskar1997neural 的下界一致（仅差一个 log log N 因子），说明结果是近似最优的。
- **d₁ 是瓶颈**：一般情况下的慢收敛率主要由输入域维度 d₁ 决定（而非输出域维度 d₂），这与函数逼近中的维度灾难一致。

### 明确局限

1. **纯理论论文，无实验验证**：所有主张均为数学推导，没有数值实验支持。无法判断理论界与实际训练行为的匹配程度。
2. **仅考虑线性低维结构**：低维结构假设（Assumption 5）只覆盖线性子空间（正交基展开），未处理非线性流形结构。作者明确将此列为 future work。
3. **网络架构是构造性的存在性证明**：证明构造的网络架构极其庞大（branch net 宽度和参数量呈超指数增长），在实践上不可训练。这只给出了理论上界，不代表实际中需要或能达到。
4. **L^∞ 误差度量**：输出误差用 L^∞ 范数度量，与 lanthaler2022error 使用的 L² 误差不同。L^∞ 更强但也更难达到。
5. **假设 Lipschitz 算子**：算子需满足 ‖G(u₁)-G(u₂)‖_{L^∞} ≤ L_G‖u₁-u₂‖_{L²}，这个混合范数的 Lipschitz 条件在特定 PDE 例子中成立（gravity pendulum、transport equation），但不是所有 PDE 都满足。
6. **不考虑训练动态**：泛化界基于 Empirical Risk Minimization 的全局最优解，不涉及 SGD 训练动态、优化 landscape、或隐式正则化。
7. **噪声假设为 sub-Gaussian**：仅考虑了轻尾噪声，未涉及 heavy-tailed noise 或模型误设（misspecification）。
8. **没有与 FNO 等架构对比**：仅为 DeepONet（Chen-Chen 风格）做分析，不涉及 Fourier Neural Operator、PCA-Net 等流行架构。
