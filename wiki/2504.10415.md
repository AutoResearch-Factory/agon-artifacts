# LLM-SRBench: A New Benchmark for Scientific Equation Discovery with Large Language Models (2504.10415)

- **Year**: 2025.04
- **Authors**: Parshin Shojaee, Ngoc-Hieu Nguyen, Kazem Meidani, Amir Barati Farimani, Khoa D. Doan, Chandan K. Reddy (Virginia Tech, VinUniversity, CMU, Capital One)
- **Venue**: ICML 2025 (uses icml2025.sty)
- **Keywords**: scientific equation discovery, symbolic regression, LLM benchmark, memorization, data-driven reasoning, Feynman equations, scientific AI
- **One-liner**: 提出了 LLM-SRBench，一个包含 239 道跨越四个科学领域的方程发现问题 benchmark，通过数学变换和合成新项防止 LLM 背诵，最佳系统仅达 31.5% 符号准确率。

## 精华提炼

### 动机与问题
当前科学方程发现（scientific equation discovery / symbolic regression）的评估主要依赖 SRBench（Feynman 物理方程 + Strogatz 动力系统）和 SRSD。但这两个 benchmark 的问题都是教科书级别的已知方程，LLM 可以通过背诵（memorization）而非真正的数据驱动推理来解决。作者通过实验证明：在 Feynman 问题上，LLM 采样在前几次迭代内 NMSE 急剧下降且符号错误远低于 LLM-SRBench 问题，说明是背诵而非逐步发现。此前 LLM-SR 等人手动构造了 5 个防背诵问题，但规模小、多样性不足。

### 管线/方法

**LLM-SRBench 由两大类别组成：**

**1. LSR-Transform（111 道题）**
- 输入：Feynman benchmark 的 100 个物理方程（含自然语言问题描述、变量名）
- 步骤：
  a) 收集 Feynman 原始方程及其输入/输出变量、问题描述
  b) 选择一个输入变量作为新的目标变量（pivot variable）
  c) 特征-目标角色互换（交换输入变量和目标变量）
  d) 用 SymPy 对新目标变量进行符号求解（解析变换）
  e) 可解性检查：仅保留可解析求解的变换（排除超越方程、高次多项式等）。471 个候选方程 → 418 个可解
  f) 数据集精炼：过滤掉超出定义域的数据点（如平方根下负数、除零）
  g) 复杂度过滤：约束变换后方程的表达式树节点数匹配原始 Feynman 分布，排除 307 个高复杂度候选，最终保留 111 个
  h) 用 GPT-4o 生成每个变换问题的新自然语言描述
- 设计目的：测试 LLM 在"同一物理问题但不同数学表示"下是否能进行推理而非背诵

**2. LSR-Synth（128 道题）**
- 跨四个科学领域：化学 36（反应动力学）、生物 24（种群增长）、物理 43（阻尼谐振子）、材料科学 25（应力-应变）
- 生成步骤：
  a) 选定科学问题（如化学反应动力学 dA/dt）
  b) 用 GPT-4o 生成 known terms（如 -kA 一阶动力学，-kA^2 二阶动力学，-k exp(-k_s t) 指数衰减）
  c) 用 GPT-4o 生成 synthetic terms（如 kA^2/(1+βA^4) 非线性高阶饱和，kA exp(-γ/t)/t^2 量子隧穿效应）
  d) 可解性检查：用 SciPy solve_ivp（RK45）数值验证可解性
  e) 新颖性检查：用 GPT-4o 评估合成项在给定科学上下文中的新颖性
  f) 数据点生成：对动态系统 t∈[0,60] 采样 5000 点，对材料应力-应变 ε∈[0,0.6]、T∈[273,573]K
  g) 专家验证：两名领域专家交叉验证合理性
  h) OOD 测试集：最后 500 个时间点/温度点作为 OOD 测试集

**评估协议：**
- 数据保真度：Acc_τ（最大相对误差≤τ的二元指标）+ NMSE（归一化均方误差），在 ID 和 OOD 数据上分别评估
- 符号准确度：用 GPT-4o 作为评估器，移除常数和参数后比较预测表达式与真值表达式的数学等价性。两名作者在 130 个随机采样问题上人工验证，GPT-4o 与人类一致率达 94.6%

**被评估的方法（3 种 + 1 个基线）：**
1. **LLM-SR**（Shojaee et al. 2024）：将方程假设生成为 Python 函数骨架（程序搜索），结合 LLM 科学知识与多岛进化搜索（m=10 个岛屿），用 BFGS 优化参数，每问题 1k LLM 调用
2. **LaSR**（Grayeli et al. 2024）：概念学习框架，从成功假设中提取抽象文本概念（ConceptAbstraction），通过概念进化（ConceptEvolution）建立概念库，结合 PySR 进行进化搜索和 LLM 引导搜索，每问题 25 次迭代、每次 550 cycles
3. **SGA**（Ma et al. 2024）：双层优化，LLM 产生离散方程假设（用 PyTorch nn.Module），物理模拟提供连续参数优化（Adam 优化器）
4. **Direct Prompting (DataBlind)**：纯依靠 LLM 领域知识直接生成假设，无法访问数据——用于评估背诵程度

### 关键技术细节
- 方程复杂度度量：表达式树节点数（遵循 SRBench/La Cava 标准）
- 数据生成质量保证：解需满足 (1) 数值容差内可解、(2) 有意义的物理行为（不发散/非常数）、(3) 唯一性（RMSE 阈值去重）
- 公平比较：所有方法统一 1k LLM 调用/问题预算，保持核心算法设计和超参数
- LLM 骨架：Llama-3.1-8B-Instruct（开源）、GPT-4o-mini、GPT-3.5-turbo
- 代码开源：github.com/deep-symbolic-mathematics/llm-srbench

### 核心结果

**主表（Table 2）：**

| Method (best LLM) | LSR-Transform SA/Acc0.1 | LSR-Synth Chem SA/Acc0.1 | LSR-Synth Bio SA/Acc0.1 | LSR-Synth Phys SA/Acc0.1 | LSR-Synth MatSci SA/Acc0.1 |
|---|---|---|---|---|---|
| DataBlind (GPT-4o-mini) | 7.21 / 6.31 | 0 / 13.88 | 0 / 4.16 | 4.54 / 9.09 | 0 / 0 |
| SGA (GPT-4o-mini) | 9.91 / 8.11 | 0 / 16.66 | 4.16 / 12.51 | 4.54 / 9.09 | 0 / 36.11 |
| LaSR (GPT-4o-mini) | 6.31 / **50.45** | 2.77 / 38.92 | 8.33 / 20.83 | **9.91** / 31.81 | **28.12** / 72.04 |
| LLM-SR (GPT-4o-mini) | **31.53** / 39.64 | **11.11** / 52.77 | **25.30** / **58.33** | **9.91** / 36.36 | 20.24 / **88.28** |

**关键发现：**
1. **整体性能低**：最佳系统 LLM-SR 仅达 31.53% 符号准确率，说明科学方程发现对当前方法仍是巨大挑战
2. **DataBlind 极差**：纯靠 LLM 知识而无数据几乎无法正确发现方程，证明了需要数据驱动推理
3. **方法间分工**：LaSR 数值精度最好（Acc0.1 领先），LLM-SR 符号准确度最优；在材料科学领域 LaSR 符号准确度反超 LLM-SR
4. **GPT-4o-mini > Llama-3.1-8B > GPT-3.5-turbo**：较小/推理能力更强的模型在方程搜索中表现更好
5. **LSR-Synth 难度 > LSR-Transform**：发现变换后的已知方程能力不延伸到含合成新项的场景
6. **同复杂度下 LSR-Transform 远难 Feynman**：即使表达式树节点数相同，变换后的问题显著更难
7. **符号准确度与 OOD 泛化强相关**：验证了 LLM 评估方法的有效性，同时确立了 OOD 评估在方程发现中的重要性
8. **与 PySR（非 LLM 基线）对比**：PySR 数值精度有竞争力甚至更好（LSR-Transform Acc0.1 = 56.76%），但符号准确率极低（化学/生物/材料科学为 0%），说明纯数据驱动方法缺乏科学直觉

### 明确局限

**作者承认的：**
1. LSR-Transform 的数学变换在代数上有效，但科学意义因上下文而异。部分变换缺乏清晰的物理可解释性（如解出角频率时涉及无直观物理意义的差平方根），某些变换模糊了因果关系（力→加速度 变 加速度→力）
2. 仅靠数学有效性不足以构成有意义的科学贡献，需要结合人类领域科学家评估
3. 化学/生物/材料科学仅 128 道题，物理最多 111 道，覆盖面有限

**我们读出的：**
4. 没有真机/物理实验验证——所有数据均为合成/数值生成，没有真实实验数据
5. 评估器（GPT-4o）本身就是被评估方法使用的同类模型，存在循环偏差风险（尽管人工验证 94.6% 一致）
6. 防背诵设计依赖"LLM 没见过这些变换/合成项"的假设，但未来 LLM 训练数据可能包含本 benchmark
7. 整个 benchmark 的合成项由 GPT-4o 生成——可能存在生成偏差（synthetic terms 偏向 LLM 自己容易/难生成的类型）
8. 没有分析不成功的原因分布（是科学推理失败、搜索效率不足、还是数学表达困难？）
9. 1k LLM 调用/问题的预算设置缺乏消融——不同方法对调用次数的敏感度不详

**可用资源**：
- 239 道防背诵方程发现问题（111 LSR-Transform + 128 LSR-Synth），跨物理/化学/生物/材料科学四个领域
- 完整的评估协议：符号等价（GPT-4o 评估器，94.6% 人类一致率）+ 数值精度（Acc_τ, NMSE）+ OOD 泛化

- PySR 纯进化算法基线：在各领域数值精度 25%-68%，符号准确率 0%-8.11%，提供了非 LLM 方法的上限参考
- 三个 LLM-based 方法（LLM-SR, LaSR, SGA）的完整实现细节和 prompt 公开在附录

**值得注意的局限**：数据全部合成，无真实实验数据；评估器与被评估方法使用同类 LLM；合成项由 GPT-4o 生成可能有偏差；不含 0616-fex 可能关注的特定数据模态（如表格数据、时间序列预测、工业数据）。
