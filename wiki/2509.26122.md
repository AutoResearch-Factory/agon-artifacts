# Trustworthy AI in numerics: On verification algorithms for neural network-based PDE solvers (2509.26122)

- **Year**: 2025.09
- **Authors**: Emil Haugen (Univ. Oslo), Alexei Stepanenko (Cambridge), Anders C. Hansen (Cambridge & Univ. Oslo)
- **Venue**: arXiv
- **Keywords**: Trustworthy AI, PDEs, Verification algorithms, Physics-informed NNs, a posteriori error bounds, neural network derivatives, activation bounds, quadrature estimates, undecidability
- **One-liner**: 为 NN-based PDE solver 构建后验验证算法，通过 rigorous 的 NN 导数 bound 计算 + PDE 残差估计，certify NN 解是否满足 ε-accuracy，绕过优化不可判定问题。

## 精华提炼

### 管线/方法

**问题设定**：给定 PDE 真解 u 的一个 NN 近似 v，判断 ‖u - v‖_X < ε 是否成立。

**整体 pipeline**：
1. **残差框架**：利用 PDE 的 well-posedness 建立 ‖u - v‖_X ≤ C · ‖φ[v]‖_Y 型不等式（如 heat eq 的 energy estimate、wave eq 的 Strichartz estimate），将误差估计转化为计算已知函数 φ[v]（NN 及其导数的线性组合）的 L^p 模。
2. **数值积分**：将 Ω 分解为不相交超立方体 ∪_k B_ε(y_k)，用中点求积（midpoint quadrature）近似 ∫_Ω |φ[f](x)|^p dx ≈ Σ_k ε^d |φ[f](y_k)|^p。
3. **求积误差 bound**（本 paper 核心贡献）：利用 Proposition 3.1 的局部求积误差界 ≤ p·ε^{d+1}·√d/2 · sup_x |φ[f](x)^{p-1} · ∇φ[f](x)|，将问题转化为计算 NN 及其导数的局部上界。
4. **导数 bound 算法**（§3）：对 ReLU/RePU NN 构造 activation bounds → 逐层传播 → 得到 ∂^α f 在 B_ε(y) 上的局部一致上界。
5. **验证算法**（§4）：对 heat equation，用上述 bound 计算 PDE 残差 L^2 模和初始条件残差 L^2 模的求积估计，代入 energy estimate 验证是否满足 ε-accurate。

### 关键技术细节

**NN 架构**：
- 全连接网络 f ∈ F_L，L 个隐层，输入 x ∈ R^d，输出 ∈ R
- 激活函数：ReLU(t) = max{0, t} 和 RePU(t) = max{0, t}^n（重点分析 n≥3 用于三阶导数）
- F_L^{(n)}：使用 n 阶 RePU 的 F_L 元素

**激活 bound（Activation bounds）**：
- 核心量：D^k(x,y) = μ'(z^k(x)) - μ'(z^k(y))，表示神经元激活状态变化
- 计算 Q^k_j 使得 |μ'(z^k_j(x)) - μ'(z^k_j(y))| ≤ Q^k_j，∀x ∈ B_ε(y)
- ReLU：Q̂_relu(E) = 0（如果 ε·E < |z(y)|/√d，即输入未跨越零点）或 1（否则）
- RePU：Q̂_repu(E) = Σ_{m=1}^{n-i} (εE)^m/m! · |μ^{(m+i)}(z(y))| + n!/(n-i)! · (εE)^{n-i} · Q̂_relu(E)
- 交替计算方案：先算 E^1（只需 W^1），算 Q^1，然后逐层向上：已知 Q^1...Q^{k-1}，用 Bound_{k-1} 算 E^k，再算 Q^k

**导数公式**：
- 一阶：∂_l f(x) = W^{L+1}[μ'(z^L) ⊙ W^L[μ'(z^{L-1}) ⊙ ... W^2[μ'(z^1) ⊙ W^1 e_l]...]]
- 高阶：用 Faà di Bruno 公式迭代计算（Bell number B_m 控制项数：B_2=2, B_3=5, B_4=15, B_5=52...）

**算法总结**：
- Algorithm 1（一阶导 bound）：输入 f, α=e_l, y, ε → 输出 A_1(α,f,y,ε) 满足 |∂_l f(x) - ∂_l f(y)| ≤ A_1，适用于任意层数 L
- Algorithm 2（二阶导 bound）：仅适用于 L=2，RePU n≥2
- Algorithm 3（三阶导 bound）：仅适用于 L=2，RePU n≥3
- 导数值 bound：E^{(α)}(y,ε) = |∂^α f(y)| + A_{|α|}(α,f,y,ε)
- f 本身 bound（α=0）：|f(x)| ≤ E^{(0)}(y,ε) = |f(y)| + ε√d/2 · (Σ_j E^{(j)}(y,ε)^2)^{1/2}

**Heat equation 验证的具体设定**：
- U = [-1,1]^{d-1}，U_T = U × [0,T]，d-1 空间维 + 1 时间维
- χ(x) = (1-x_1^2)...(1-x_{d-1}^2) 强制边界条件，v = χf
- φ_pde[f] = (∂_t - κΔ)(χf)，φ_init[f] = (χf)(·, 0) - g
- Energy estimate 常数：C = √d/(π√κ)
- 验证准则：√d/(π√κ)·(I₂(φ_pde)+B_pde(ε)) + (I₂(φ_init)+B_0(ε)) < ε₀

**数值实验**：
- 1D heat equation：d-1=1 空间维，g(x) = sin(πx)，κ = 1/π²，T=1
- NN：F_2^{(3)}，2 隐层，每层宽度 100
- 求积网格 ε 从 0.04 到 2.5e-5
- 最高分辨率 (ε=2.5e-5)：NVIDIA A10 GPU ~10 小时
- 其他计算：laptop CPU 可完成

**关键发现**：
- 求积误差 bound R 随 ε→0 以 O(ε) 速度收敛到 0（与 proof 一致）
- Global verification algorithm 理论上保证 halt于有限步（因为存在收敛子序列 + bound 随网格细化而收紧）
- 对 ReLU 网络，activation bound 本质上是检测神经元是否跨越零点（激活状态变化）
- 对 RePU 网络，高阶导数的 activation bound 包含多项式项 + 离散跳跃项

### 明确局限

1. **高阶导数限于 2 隐层**：Algorithm 2 和 3 只能处理 L=2。因为 Faà di Bruno 的 Bell number 爆炸 + 每层递归一步，更深网络的计算复杂度不可行。
2. **仅支持 ReLU 和 RePU**：sigmoid/tanh 虽有提及但未实现。ReLU 仅一阶可微，高阶导数需 RePU。
3. **仅 1D 空间演示**：实验仅在 d-1=1 的 heat equation 上做。高维情况计算复杂度 O(ε^{-d}) 会爆炸。
4. **无有限元/经典方法对比**：没有与经典 a posteriori error estimation（如 Verfürth 2013 的 finite element 方法）的精度或效率比较。
5. **无真机实验**：全部是数值模拟。没有在真实物理系统上的验证。
6. **无标准 benchmark 对比**：没有在 PDE benchmark 上与其他 verification 方法（如 Hillebrecht & Unger 2022）比较。
7. **计算成本高**：高精度（小 ε）需要 O(ε^{-d}) 个积分点，每个点要做完整的导数 bound 计算。10 小时 GPU 仅处理 1D 空间。
8. **需要 PDE-specific residual bound**：每种新 PDE 需要单独推导 energy estimate 或 Strichartz estimate，不是通用黑箱。
9. **需要 g 的导数 bound**：Assumption 4.1 要求初始条件 g 的导数有已知一致上界 G^i。
10. **Local algorithm 不保证 halt**：仅当 v 确实足够接近 u 时才 halt。需要 Global algorithm 扫描 NN 序列。
