# Walrus: A Cross-domain Foundation Model for Continuum Dynamics (2511.15684)

- **Year**: 2025.11
- **Authors**: Michael McCabe (Flatiron Institute/NYU), Payel Mukhopadhyay (Cambridge), Tanya Marwah, Bruno Régaldo-Saint Blancard, François Rozet, et al. — The Polymathic AI Collaboration
- **Venue**: arXiv (ICLR 2026 format)
- **Keywords**: foundation model, continuum dynamics, PDE emulation, transformer, autoregressive, multi-physics, 2D-3D joint training, patch jittering, adaptive compute, physical simulation, FSDP load balancing
- **One-liner**: 1.3B transformer foundation model pretrained on 19 diverse 2D+3D physical scenarios with harmonic-analysis-based stabilization and compute-adaptive tokenization, achieving SOTA downstream transfer across 26 continuum emulation tasks.

## 精华提炼

### 管线/方法

**整体 Pipeline**: 给定物理系统 S 的历史快照序列 U_t (2D: 6步历史, 3D: 3步历史) → 可逆归一化 (RMS over time×space) → Patch Jittering (随机平移+padding) → Compute-Adaptive Patching (CSM 卷积下采样到固定 token 数: 2D=32², 3D=16³) → Space-Time Factorized Transformer (40 blocks, hidden=1408, MLP=5632, 16 heads) → 反向 CSM 上采样 → Inverse Jittering → 输出 Δu_{t+1} → 反归一化 → u_{t+1}=u_t + Δu_{t+1}

**Encoder**: hMLP (Hierarchical MLP) + StrideMod (Convolutional Stride Modulation, 来自 Overtone 论文)。Encoder 层使用 GELU 激活（历史遗留，与 processor 的 SwiGLU 不一致）。

**Decoder**: 转置 hMLP + 转置 StrideMod。Encoder/Decoder 参数在各 device 上复制而非 shard（因为 hMLP 轻量）。

**Processor**: Space-time split transformer blocks。每个 block 内：(1) Space mixing: Parallel Attention (MLP + Attention 并行, PaLM 风格) + Axial RoPE 位置编码 + QK Norm。(2) Time mixing: Causal Attention (为 KV caching 设计) + T5 风格 Learned Relative Position Encoding。

**归一化策略**:
- 内部: RMSGroupNorm (GroupNorm 实现为 RMSNorm，group 数=attention head 数=16)
- 输入: 按 field 做 `RMS_{Time×Space}(U_t)` 归一化
- 输出: 按 field 做 `RMS_{Time×Space}(ΔU_t)` 反归一化（非对称设计，因为 Δ 的分布不同于原始值）

**Pretraining**: 96×H100 GPU, FP32 单精度 (BF16 精度下降显著，FP16 训练不稳定), ~400K steps, 2D micro-batch=192, 3D micro-batch=96, AdamW (lr=2E-4, weight decay=1E-4), inverse sqrt schedule + linear warmup + inverse quadratic cooldown。每个 2D 数据集约 4M samples，每个 3D 数据集约 2M samples。

**Finetuning**: 4×H100, 50 pseudo-epochs (500K samples), lr 减半=1E-4。Finetuning 特有改动：Learnable RoPE (每个维度复制频率参数并设为可训练)、可选 APE (仅在 anisotropic 域使用)。

### 关键技术细节

**1. Patch Jittering (核心创新)**:
- 理论来源：调和分析 (harmonic analysis)。对 ViT 风格的 strided convolution + transposed convolution 的编解码过程做频域分析，发现输出信号可写为 `v̂[k] = h̄̂[k] ĝ[k] û[k] + Σ_{j=1}^{P-1} h̄̂[k] ĝ[k+jM] û[k+jM]`——第二项是 aliasing 项。
- 解法：在 patchify 前随机平移输入数据，decode 后反向平移。利用傅里叶平移性质 `F u(x-s)[k] = e^{-i2πsk} û[k]`，期望消除 aliasing 项 (Cauchy 积分定理：绕复单位圆的围道积分为零)。
- 实现：Periodic 边界→`torch.roll`；Open/Closed 边界→padding + 有限 roll。实际不使用平均而仅单次采样就有效果。
- 效果：在 17/19 pretraining 数据集上改善长期预测，中位改善 54%，高误差 (VRMSE ≥1) 从 10 降到 3。

**2. 2D→3D Joint Training**:
- 所有 2D 数据加 singleton 维度 zero-pad 到 3D（如 velocity {v_x, v_y} → {v_x, v_y, 0})
- 应用 tensor-law-aware 变换：对 order-1 张量场 → Rv，对 order-2 张量场 → RvR^T
- 旋转采样自 SO(3) 的轴对齐子群（octahedral group, 90° 旋转和反射）
- Variable time striding (1-5 步随机采样)，迫使模型从上下文推断时间尺度

**3. Compute-Adaptive Tokenization (CSM)**:
- Convolutional Stride Modulation: 动态调整编码器/解码器中卷积的 stride
- Pretraining 时固定每条轴的 token 数（2D=32², 3D=16³），根据输入分辨率调整下采样因子
- 使得不同分辨率的数据产生大致相同的 token 数，平衡计算负载

**4. Topology-Aware Distributed Sampling (HSDP)**:
- 问题：FSDP 的 AllGather 是阻塞同步操作，不同 resolution/dimensionality 的数据导致负载不均在 sharding group 内产生死重
- 解法：同一 sharding group 内所有 rank 每步采样同一数据集，不同 group 独立采样 → 保证 AllGather 瓶颈处数据同质
- 结合 gradient accumulation 做 randomized load balancing
- 成果：相比朴素 FSDP，吞吐量提升 262%

**5. 其他架构细节**:
- Drop Path Rate: 0.05
- Gradient norm clipping: 10.0
- Encoder dimension=352, Hidden dimension=1408, MLP dimension=5632
- 2D: time history=6, base token shape=32²; 3D: time history=3, base token shape=16³
- Projection dimension=48

### 核心结果

**Downstream Transfer (Table in finetune_table.tex, Figure downstream_2d.pdf)**:
- vs MPP-AViT-L (407M), Poseidon-L (628M), DPOT-H (1.15B)
- 2D下游任务: CE-RM, FB Skelenton, BML PoolBoilSubcool, PDEA ConditionedINS
- 3D下游任务: PDEB CNS 3D Turb/Rand, RSG Convective Envelope, Post Neutron Star Merger
- One-step: 平均 63.6% 降低 vs baselines
- Short rollout (1-10步): 平均 56.2% 降低
- Medium rollout (11-30步): 平均 48.3% 降低
- 56/65 个 tracked metrics 上取得最佳

**Cross-Domain (Table tab:training_set_comps, 19 pretraining 数据集)**:
- Finetuned Walrus vs heavily finetuned baselines (4.5M samples for 2D, 2.5M for 3D)
- One-step: 18/19 数据集最佳，平均 52.2% 降低
- 20步: 17/19 最佳，30.5% 降低
- 20-60步: 12/19 最佳，6.3% 降低
- DPOT (AFNO-based) 在 acoustics/wave propagation 中最接近 Walrus，甚至更长期限上超越
- Poseidon 在 inviscid fluids (Euler quadrants) 上竞争力强（其 pretraining 数据 4/6 为 Euler 方程）
- 结论：Walrus 在各领域均接近或超越各模型的优势领域，展现了真正的跨域泛化

**Diversity Ablation (HalfWalrus, Figure CleanedUpTransferPlot.pdf)**:
- 用 half-size Walrus (640M, 30 blocks, hidden=1088) 在 8 个 2D 数据集上做对照
- 关键发现：在 pretraining metrics 上，naive (无 diversity augmentation) 模型表现更好，但在 downstream transfer 上 HalfWalrus 显著更好
- 警示：不能仅靠 pretraining loss 判断模型质量，diversity 在 pretraining 期间的"负面"影响是 deceptive
- 2D-only pretrained 模型在 3D CNS 任务上只有微弱提升，甚至可能成为阻碍；full 3D-pretrained Walrus 有压倒性优势

**Jitter 消融 (Table app_tab:jitter_metrics)**:
- 最显著改善：TRL 2D (76.9→0.75), MultiQuadrantsO (1.82→0.50), Inclusions (1.30→0.18), Maze (1.07→0.34)
- 无明显变化/轻微变差的：Active Matter (0.98 vs 0.98), MultiQuadrantsP (0.62→0.60)
- 没有严重变差的数据集

### 明确局限

**作者自己说的**:
1. RSG 外边界产生 artifacts，内部忠实；PNS 中 bulk dynamics 好但不正确的重元素产量估计 → 科学验证仍是必需的
2. 确定性训练（full reconstruction loss）可能是 poorly observed/stochastic 系统的表征瓶颈
3. Non-uniform geometries 尚未处理
4. History-based vs parameterized mode 的 tradeoff 未解决
5. 部分架构决策来自惯性而非仔细测试（encoder/processor 激活不一致、additive corner handling）
6. Latent diffusion models 可能在更高压缩比下提供更稳定的 rollout 和概率估计，但尚未推广到 multi-physics

**我读出来的额外局限**:
7. FP32 训练要求高：96×H100 单精度训练，BF16/FP16 均不稳定，计算成本极高
8. 时间历史长度有限（2D=6步, 3D=3步），对需要长记忆的系统可能不足
9. 边界条件处理粗糙：仅做简单拓扑分类 (Open/Closed/Periodic)，未利用 ghost-node 等数值方法，且 corner handling 有已知问题
10. Finetuning 后仍需每步自回归预测，无多步预测加速能力
11. 仅处理 Cartesian/Angular/Spherical 域的 Euclidean 数据，不支持非结构网格或 Lagrangian 描述
12. 代码开源但 KV caching 未实现（论文中承认），推理优化尚未完成
13. 1.3B 参数但 pretraining 数据仅 19 个场景且 3D 数据很少——可能存在 overfitting 风险于特定 simulation 的数值特征而非物理规律
14. 评测依赖 VRMSE，但 VRMSE > 1 仅表示比均值预测差——对混沌系统这个标准可能太弱

### 跟我们的关系

对 **0616-fex** 课题的评估：

**撞车评估**:
- Walrus 的核心关注点是 continuum dynamics 的通用 foundation model，包含流体、等离子体、声学、天体物理等。0616-fex 的关注点需要具体对照其物理 domain。如果 0616-fex 涉及的是 fluid-like continuum systems 的 data-driven emulation，则 Walrus 是直接竞品。
- Walrus 的 **patch jittering** 技术是一般性方法，可用于任何 ViT-style 物理 emulator。如果 0616-fex 使用类似架构但未采用 jittering，可能处于劣势。
- Walrus 的 2D-3D joint training 和 compute-adaptive tokenization 构成了较高的工程壁垒——如果没有类似的多维度和多分辨率训练策略，难以与 Walrus 的泛化能力竞争。

**可用工具/方法**:
- Patch jittering 可直接复用——实现简单（torch.roll + padding），对稳定性提升显著
- CSM (Convolutional Stride Modulation) 可参考 Overtone 论文 (2507.09264) 实现
- 非对称输入/输出归一化策略（RMS over time×space for input, RMS of Δ for output）
- HSDP 分布策略可参考，如果也面临异构数据训练的负载均衡问题
- Learnable RoPE + APE 的 finetuning 策略值得借鉴

**差异化方向**:
- Walrus 训练完全是确定性的 (L1 loss)，如果是 stochastic systems 需要概率输出，可以做 latent diffusion 或 ensemble 方法
- Walrus 不支持 non-uniform/non-Euclidean 几何，如果 0616-fex 涉及此方向可差异化
- Walrus 的时间上下文有限（6/3步），如果需要长程记忆可改进
- Walrus 没有利用已知 PDE 信息 (physics-agnostic)，如果 0616-fex 能用 physics-informed 方法可能在某些场景更高效
- 1.3B 参数的训练成本高，如果 0616-fex 能实现更小模型达到类似效果会是关键优势
