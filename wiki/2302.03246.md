# CDANs: Temporal Causal Discovery from Autocorrelated and Non-Stationary Time Series Data (2302.03246)

- **Year**: 2023.02
- **Authors**: Muhammad Hasan Ferdous, Uzma Hasan, Md Osman Gani (Causal AI Lab, Dept. of Information Systems, University of Maryland, Baltimore County)
- **Venue**: Machine Learning for Healthcare (MLHC) 2023, PMLR vol. 219
- **Keywords**: temporal causal discovery, autocorrelation, non-stationarity, changing modules, PCMCI+, CD-NOD, kernel conditional independence (KCI), surrogate variable, ICU oxygen therapy, MIMIC-III
- **One-liner**: Constraint-based algorithm (CDANs) that jointly recovers lagged, contemporaneous, and "changing module" (non-stationary) causal edges by first shrinking the conditioning set via lagged-parent detection, evaluated on synthetic AR data and an ICU oxygen-therapy cohort.

## 精华提炼

### 管线/方法

Input: multivariate time series X = {X¹,...,X^N} observed at each timestamp t, with a max lag τ_max. Output: a temporal causal graph over (Xₜ ∪ lagged parents ∪ surrogate variable C).

Four-step pipeline:

1. **Step 1 — Lagged parent detection.** For each target Xₜʲ, run PCMCI-style iterative CI testing against all lagged candidates X_{t-τ}ⁱ: first unconditional tests recording p-value + effect size, then build a candidate lagged-parent set sorted by effect size, then iteratively test conditional independence adding variables in descending effect-size order and pruning non-significant links. Produces LPA(Xₜʲ) for every variable. This is the core trick that avoids conditioning on the *entire* past (the standard high-dimensionality failure mode of plain constraint-based methods like PC/FCI applied naively to time series).
2. **Step 2 — Undirected graph construction.** Build a partially-complete undirected graph G over (Xₜ ∪ LPA(Xₜ) ∪ C), where C is a single surrogate variable standing in for "time" / the unobserved non-stationary confounder — directly borrowed from CD-NOD's (Zhang et al. 2017) "changing module" formalism and its "pseudo causal sufficiency" assumption (a single smooth latent confounder is assumed to explain all non-stationarity).
3. **Step 3 — Changing-module + contemporaneous skeleton detection.** Kernel-based CI tests (KCI, Zhang et al. 2012) between each contemporaneous Xₜʲ and C, conditioning on subsets of (LPA(Xₜʲ) ∪ Xₜ) \ {Xₜʲ}; separately, marginal/CI tests between contemporaneous pairs Xₜⁱ, Xₜʲ conditioning on subsets of (Xₜ ∪ LPA(Xₜⁱ) ∪ LPA(Xₜʲ) ∪ C). This is the key differentiator versus "extended CD-NOD" (Huang et al. 2020), which detects changing modules using *only* contemporaneous variables — CDANs argues that omitting lagged parents from this conditioning set causes false positives.
4. **Step 4 — Orientation.** Lagged edges are oriented trivially by time order (past → future). C→Xₜʲ edges are oriented forward whenever adjacent. For unshielded triples (C − Xₜⁱ − Xₜʲ), the algorithm recalls the conditioning set used in Step 3's CI test between C and Xₜʲ: if Xₜⁱ was *not* in that set, orient as a collider C→Xₜⁱ←Xₜʲ; otherwise orient as a chain C→Xₜⁱ→Xₜʲ. When both Xₜⁱ and Xₜʲ are adjacent to C (the rule above doesn't apply), direction is decided via an **extended Hilbert–Schmidt Independence Criterion (HSIC)** asymmetry statistic — reusing the exact causal-direction machinery from extended CD-NOD (Huang et al. 2020): the direction whose Gram-matrix-based dependence score ∂_{X→Y} is *smaller* is preferred (interpreted as: cause's marginal + effect's conditional look "more independent" than the reverse pairing).

Complexity claim: worst case is polynomial in (N+1) — N variables plus the one surrogate — rather than exponential in N·τ_max as in plain PC, because lagged-edge orientation is polynomial (argument borrowed from Runge 2019) and the changing-module/contemporaneous step only iterates over the already-shrunk conditioning sets. **This claim is asserted, not empirically benchmarked** — no runtime numbers appear anywhere in the paper.

Code + data: https://github.com/hferdous/CDANs. Baselines (PCMCI+, FGES, GFCI, GRaSP, SVAR-GFCI) are run via the public `tetrad` Java package (https://github.com/cmu-phil/tetrad), not reimplemented.

### 关键技术细节

- **CI tests**: PCMCI-style tests (partial-correlation-flavored, consistent with the Tigramite/PCMCI lineage) for Step 1 lagged-parent detection; **KCI (Zhang et al. 2012)** specifically for Step 3 changing-module/contemporaneous edges. No kernel bandwidth, permutation count, or significance-level values are reported for the KCI tests.
- **Orientation statistic**: ∂_{X→Y} = tr(G_X H G_{YX} H) / (tr(G_X H)·tr(G_{YX} H)), where G_X, G_{YX} are Gram matrices of conditional mean embeddings μ̂_{X|C} at surrogate values C = c₁,...,c_N and H is the centering matrix — identical machinery to extended CD-NOD.
- **Synthetic data**: 4/6/8-variable models × lags {2,4,6,8}. Explicit nonlinear AR generating equations, e.g. Xₜ¹ = 0.6·Xₜ₋₁¹ + ε; Xₜ² = 0.8·Xₜ₋₁¹ + 1.5·sin(t/50) + ε (the "changing module" is literally a deterministic sine/cosine function of the global time index — not stochastic regime-switching, not a changepoint process, not covariate-driven drift). 4-var/lag-2 ground truth: 1 changing module, 1 contemporaneous edge, 2 autocorrelated edges, 2 lagged edges (6 edges total, deliberately sparse).
- **Real-world clinical dataset**: MIMIC-III-derived oxygen-therapy (OT) cohort, ICU patients on invasive mechanical ventilation, following the Panwar et al. 2016 pilot-RCT protocol (conservative vs. liberal oxygenation) and the same lab's earlier structural-causal-model paper (Gani et al. 2023, arXiv 2010.14774). 12 variables sampled every 4 hours over up to 2 weeks (~88 time points): FiO₂, hemoglobin, lactate, PaCO₂, PaO₂, SaO₂, SpO₂, minute ventilation, peak air pressure, PEEP, pH, tidal volume.
- **Ground truth for real data**: none exists for the temporal graph. Evaluation is against a *non-temporal* reference graph from Gani et al. 2023, itself built by majority voting across 7 causal-discovery algorithms plus manual domain-knowledge correction over 26 variables (reduced to the 12 overlapping ones) — i.e., a soft, algorithmically-derived pseudo-ground-truth, not a true gold standard.
- **Baselines**: PCMCI+ (Runge et al. 2020), FGES, GFCI, GRaSP, SVAR-GFCI (all via `tetrad`), and CD-NOD (Zhang et al. 2017) for a contemporaneous-only comparison (CD-NOD cannot produce lagged edges at all).
- **Metrics**: TPR (recall), FDR (false discovery rate), SHD (structural Hamming distance) — standard skeleton/orientation metrics, formulas given in Appendix A.

### 核心结果

Synthetic data (Table 1, 4/6/8 vars × lag 2/4/6/8, 6-method comparison):
- CDANs achieves the highest or tied-highest **TPR** in 10 of 12 (variable-count, lag) cells — e.g., 6-variable models: TPR 0.89/0.89/0.73/0.89 across the four lags, vs. PCMCI+'s 0.78/0.67/0.56/0.67.
- **FDR/SHD are mixed, not dominated by CDANs.** E.g. 4-var/lag-4: CDANs gets TPR=1.00 but FDR=0.54, SHD=7 — worse than PCMCI+'s FDR=0.43/SHD=5 at TPR=0.67. This means CDANs is trading precision for recall in several settings; the paper's narrative ("CDANs consistently outperforms") is more favorable than the raw FDR/SHD columns fully support.
- Head-to-head vs. CD-NOD on changing-module + contemporaneous-only subgraphs (Table 2): CDANs **ties** CD-NOD exactly on TPR (1.0/1.0, 0.75/0.75, 0.43/0.43 for 4/6/8 vars) but has equal-or-lower FDR and strictly lower SHD (1 vs 2, 4 vs 5, 8 vs 10). I.e., CDANs' advantage over CD-NOD here is entirely about fewer spurious edges among the *same* true edges found — not about finding more true edges.

Real clinical data (oxygen therapy, MIMIC-III, 12 vars, lag 2):
- Non-temporal skeleton comparison (Table 4): CDANs TPR=0.27, FDR=0.63, SHD=13, vs. PCMCI+ and CD-NOD both at TPR=0.09, FDR=0.80/0.83, SHD=14/15. CDANs "wins" relatively, but absolute recall of 0.27 is low in absolute terms.
- CDANs identifies FiO₂ and VT as changing modules; CD-NOD identifies FiO₂ and PaCO₂ — no ground truth exists to adjudicate correctness; this is face-validity only.
- One directed edge (PEEP→FiO₂) and an undirected pH–PaCO₂ edge are cited as consistent with physiological domain knowledge — a single supporting anecdote, not systematic validation.

### 明确局限

Authors' own stated limitations (Discussion section):
- Uses conventional (non-classifier-based) CI tests; suggest CCMI (Mukherjee et al. 2020) or a GAN-based CI test (CIGAN) as future alternatives.
- Does not jointly estimate contemporaneous parents alongside lagged parents in one pass — explicitly deferred to their own immediate follow-up, **eCDANs** (arXiv 2303.02833, published ~1 month later as an MLHC student abstract).
- Not evaluated at larger scale (variable count capped at 8 synthetic / 12 real; real time series length ~88 points).

Additional limitations observed while reading:
- **No true ground-truth temporal graph anywhere** — the clinical validation leans on a non-temporal reference graph from a sister paper by the same lab, which is itself a causal-discovery output (majority vote + manual correction), not an independent gold standard.
- **Non-stationarity model is narrow**: the "changing module" in synthetic data is a deterministic sine/cosine of the global time index — no regime-switching, no changepoints, no heteroskedastic noise, no covariate-driven distribution shift.
- **Single surrogate variable C** encodes all non-stationarity ("pseudo causal sufficiency," same assumption as extended CD-NOD) — breaks down if there are multiple independent sources of non-stationarity.
- **No runtime/complexity benchmark** despite an explicit polynomial-vs-exponential complexity claim.
- **No variance/seed reporting** — Table 1 synthetic results are single point estimates; number of random trials averaged is not stated.
- KCI hyperparameters (kernel bandwidth, permutation count) are unspecified, despite materially affecting test power/size.
- Venue is MLHC (a clinical-ML-focused PMLR workshop track), not a core causal-discovery venue (UAI/NeurIPS/JMLR main track) — reflects a clinical-application framing over deep methodological scrutiny.

### 跟我们的关系

课题 0710-causal-scs 的核心主张是反常规的：**因果效应估计在假设违背（confounder、non-stationarity）下产生的不稳定性本身**可以作为强对流事件的预警指标——不追求更准确/更鲁棒的因果图，而是主动保留假设违背、量化由此产生的效应波动/分歧/突变。候选方法明确包含 PCMCI+ / J-PCMCI+。

- **哲学上直接对立，构成有用的"反向读法"**：CDANs（连同其直接前身 PCMCI+ 和其效仿对象 extended CD-NOD）站在完全相反的立场——目标是**吸收/建模非平稳性**（引入代理变量 C 显式表示 "changing module"，并用定向规则把它变成图中可解释的一部分），从而让因果图估计在非平稳条件下依然准确。这是"通过建模消解不稳定性"的传统范式，与课题"把不稳定性本身当作信号"的范式正好相反。Proposal 的 Related Work 里可以把 CDANs/PCMCI+/extended CD-NOD 这条线统一归为前者，用于凸显差异化定位。
- **不撞车**：CDANs 完全没有涉及"效应不稳定性/波动/分歧作为预警指标或特征"的思路；应用域是 ICU 供氧治疗（MIMIC-III），与强对流/大气科学无关；没有滑动时间窗口分析，没有跨方法分歧对比，也没有 bifurcation/regime-switching 动力系统实验。
- **可复用之处**：
  1. CDANs 是 PCMCI+ 的直接扩展，如果课题用 PCMCI+ 作为候选因果发现方法之一，CDANs 给出了一个具体范式——如何在 PCMCI+ pipeline 里用代理时间变量 C 显式表示 "changing module"。课题不需要照搬其"纠正非平稳性"的目的，但可以借用其"用 C 定位模块变化"的表示方式，转而观察 C 相关边（强度、方向、显著性）在滑动窗口/不同 CI 检验假设下的**波动**，把 CDANs 原本用来"消解不确定性"的机制，反过来当作"产生可测量不确定性"的观测面板。
  2. Step 1 "先做 lagged-parent 检测再收窄 conditioning set" 是应对高维诅咒的通用工程技巧；若课题在真实 3km 网格环境变量场（变量数可能远超论文测试的 8 个）上跑 PCMCI+/J-PCMCI+ 遇到维度灾难，这是可直接复用的加速手段，属于实现层面而非哲学层面的借用。
  3. Table 1 的数值提示一个课题实验设计需要警惕的混淆变量：约束型方法（PCMCI+ 及变体）本身在变量数、lag 增大时 TPR/FDR/SHD 都明显退化（8 变量/lag 8 下 TPR 仅 0.73–0.82，FDR 普遍 > 0.5）。这意味着课题在高维真实环境场上观测到的"效应不稳定"，可能混杂了"方法固有的高维检测力下降"和"环境真实非平稳性导致的效应波动"两种来源——课题已规划的阶段一（模拟耦合 Lorenz 系统等已知分岔行为）应当专门设计消融，把这两种噪声源分离开，而不仅仅验证核心假设本身。
  4. 后续工作 eCDANs（2303.02833，同实验室 1 个月后发表）声称联合估计 contemporaneous 与 lagged parents，如果课题未来需要刻画"conditioning set 选择本身导致的效应分歧"作为不稳定性来源之一，eCDANs 值得下一轮追踪（本文未深入，留给下一轮搜索）。
- **与姊妹课题 "Causal Discovery Uncertainty" 的关系**：CDANs 的贡献本质上是"降低非平稳性导致的 DAG 不确定性"，这与姊妹课题（量化/比较不同因果发现方法得到的 DAG 不确定性）的主题更加贴近，比与本课题（把不确定性当作预警信号）更贴近。若姊妹课题启动，CDANs 应作为核心相关工作复用；本课题目前只需将其作为"传统范式代表"引用，无需深入采用其方法本身。
