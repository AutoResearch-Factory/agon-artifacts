# Active Symbolic Discovery of Ordinary Differential Equations via Phase Portrait Sketching (2409.01416)

- **Year**: 2024.09
- **Authors**: Nan Jiang (Purdue CS), Md Nasim (Cornell CS), Yexiang Xue (Purdue CS)
- **Venue**: AAAI 2025 (uses aaai25.sty, includes AAAI reproducibility checklist)
- **Keywords**: symbolic regression, ODE discovery, active learning, phase portrait, REINFORCE, policy gradient, grammar-guided search, dynamical systems, sequential decoder, butterfly effect
- **One-liner**: 用 REINFORCE policy gradient 训练 Transformer decoder 生成候选 ODE 的语法规则序列, 并通过 phase portrait sketching 主动选择高信息量区域查询新轨迹数据, 比被动学习方法在 Strogatz 和 ODEBase 数据集上 NMSE 更低.

## 精华提炼

### 管线/方法

完整 pipeline 分 3 步 (Figure 1):

1. **Grammar-rule 序列采样** (Figure 1a): 输入 start symbol `phi -> A, B` 到 sequential decoder (embedding layer + multi-head self-attention + softmax, dim=256). decoder 输出 categorical distribution over grammar rules, 每步采样一个 rule token 作为下一步输入. 最多采样 20 步. 概率由 decoder parameters theta 参数化: p_theta(s) = prod_i p_theta(s_i | s_1...s_{i-1}).

2. **序列→ODE 表达式** (Figure 1b): Context-free grammar 定义: V = {A, B} (每个变量一个非终结符), Sigma = {x_1,...,x_n, const}, R = 数学操作 (+, -, *, /, sin, cos 等). 从 start symbol 出发逐条应用采样到的 grammar rules, 替换第一个非终结符, 直到所有符号均为终结符. 如 `phi -> A, B` → `B -> B * B` → ... → `phi -> x_2, c_1 * sin(x_1)`. 开常数 (coefficients) 用 BFGS optimizer 在 queried data 上拟合, batch size=1024, 并行 20 CPU cores (pathos library).

3. **Phase Portrait Sketching 选信息区域** (Figure 1c): 
   - 对 M 个候选 ODE {phi_1,...,phi_M}, 随机选 K=10 个 region (n 维立方体, 边长=变量区间长的 1/4)
   - 每个 region u_k 内均匀采样 L 个初始条件, 用 RK45 数值积分 (自实现, 非 scipy.integrate.solve_ivp 因其自适应步长太慢) 生成短轨迹
   - 计算 pairwise informative score: IF(phi_m, phi_m', u_k) = (1/L) sum_l ||tau_{m,k,l} - tau_{m',k,l}||^2
   - 总信息量: IF(u_k) = (1/M) sum_{m} sum_{m'>m} IF(phi_m, phi_m', u_k)
   - 选 IF 最大的 region u*, 从中采样 m 个初始条件, 查询 oracle O(x_i, T) 获得 ground-truth 轨迹

4. **Policy Gradient 更新**: reward(s) = 1/(1 + NMSE(phi)). REINFORCE 梯度估计: grad_theta J ≈ (1/N) sum_i (reward(s^i) - b) * grad_theta log p_theta(s^i), 其中 baseline b = mean of batch rewards (variance reduction). N=100 (batch size of expressions). Adam optimizer, lr=0.009, 50 epochs.

### 关键技术细节

- **Decoder 架构**: embedding (dim=256) → multi-head self-attention → softmax. Pytorch 实现. 输入/输出 vocabulary 为 grammar rules 集合.
- **Grammar 设计**: 以 2-var ODE 为例, A 表示 dx_1/dt 的子表达式, B 表示 dx_2/dt 的子表达式. 规则包括 A->(A+A), A->(A-A), A->A*A, A->A/A, A->x1, A->x2, A->const, 以及 B 的对应规则.
- **系数拟合**: 表达式中的占位符 `const` → 用 BFGS (scipy.optimize.minimize, method=BFGS) 在 queried data 上拟合具体数值. batch size=1024.
- **数值积分**: 自实现 4 阶 Runge-Kutta (RK45), 避免 scipy.integrate.solve_ivp 自适应步长导致的极慢速度 (某些候选 ODE 超过 12 小时无法返回轨迹).
- **奖励函数**: reward = 1/(1+NMSE). 含占位符或超过 20 个开常数的表达式直接赋 fitness = -inf.
- **Region 参数**: 相对边长 = 1/4 (每个 region 边长为变量区间边长的 1/4). K=10 regions.
- **评测指标**: NMSE (主), R^2 score, Kendall tau ranking distance. NMSE = (1/sigma^2)*(1/n)*sum_i (x(t_i) - hat{x}(t_i))^2, sigma 为 empirical std.
- **数据集**: 
  - Strogatz dataset: 80 个 ODE 实例, 来自 Strogatz 教科书, 由 ODEFormer 形式化为 benchmark. 按变量数分 n=1,2,3,4.
  - ODEBase dataset: 114 个实例, 来自化学和生物学. 按变量数分 n=2,3,4,5.
  - 三种条件: noiseless (sigma^2=0), noisy (multiplicative Gaussian, sigma^2=0.01), irregular time (随机 drop 10% 时间点, alpha=0.1).
- **训练/测试设置**: 训练用 100 个随机初始条件, 演化 1 秒, time step=0.001s. 测试用 100 个随机初始条件, 演化 10 秒, time step=0.001s.
- **硬件**: Milan CPUs @ 2.45GHz, 8GB RAM, max 24h per run. 并行 20 CPU cores. Python 3.10.

### 明确局限

1. **Resolution 问题**: Phase portrait sketching 的 region width 太大或采样点太少时可能遗漏 fine-grained dynamics. 这是作者在 Appendix B 明确承认的.
2. **不适用于 PDE**: 作者明确说 "It is unclear if the proposed phase portrait sketching idea is applicable to partial differential equations". 这意味着整个 active query 策略限于 ODE.
3. **依赖 data oracle 假设**: 方法假设存在一个可以任意查询的 data oracle O(x_0, T). 在真实实验中 (如生物实验), 这种灵活查询可能是昂贵的或不现实的.
4. **仅验证到 n<=5**: 声称可扩展到高维, 但实验仅覆盖到 n=5. 维度诅咒的实际表现未经验证.
5. **没有与最新 SR 方法对比**: 缺少与 DGSR-MCTS、TPSR、SR-GPT 等 2023-2024 SR 方法的对比. 只比较了 ODEFormer、SINDy、SPL、ProGED、E2ETransformer.
6. **Grammar 需手动设计**: 算子集 {+,-,*,/,sin,cos,const} 对 Strogatz/ODEBase 有效, 但新 domain 需要重新设计.
7. **REINFORCE 高方差**: 虽然用了 baseline subtraction, 但 policy gradient 的方差仍然可能在高维系统中成为问题.
8. **没有与 Purely Neural 的隐式 ODE 方法对比**: 没与 Neural ODE、PINN 等做定量比较, 虽然这些方法不生成符号表达式但可能拟合更好.
9. **系数拟合阶段分离**: 符号搜索 (decoder) 和系数优化 (BFGS) 两阶段分离, 没有端到端的梯度流动.
10. **单 region 选择**: 每轮只选一个 region, 可能错过多个信息区域的组合.

### 跟我们的关系

**对 0616-fex (FEX: RL for Symbolic Regression) 课题:**

1. **撞车评估: 中低**. APPS 使用的是 REINFORCE policy gradient + grammar-guided search 这一与 FEX 相同的核心范式. 但应用场景不同: APPS 做的是 ODE discovery from data (给定轨迹数据, 反推 ODE 表达式), 而 FEX 做的是 PDE solving (给定 PDE, 搜索近似解). APPS 的 "data oracle" 假设也与 FEX 的 "已知 PDE 形式" 假设有本质区别. 但在方法论层面, APPS 的 sequential decoder + grammar rules 框架与 FEX 的 controller + 表达式树搜索 高度重叠. 如果 FEX 要扩展到 "从数据中发现 PDE" 的场景, APPS 是最直接的竞争.

2. **可用工具/方法:**
   - **Phase portrait sketching 可适配到 FEX**: APPS 的 region-based active query 策略可以迁移到 FEX 的搜索中: 在评估候选解时, 不在整个 domain 上均匀采样, 而是在解的 phase portrait 中找出最分歧的区域重点评估. 这对 FEX 的 PDE 求解场景可能也有用 (如在边界层或激波附近重点采样), 但需要将 ODE phase portrait 泛化到 PDE 的流场或解空间.
   - **自实现 RK45 的优化经验**: APPS 发现 scipy.integrate.solve_ivp 的自适应步长在某些候选 ODE 上极慢, 因此自实现固定步长 RK45. FEX 在评估候选 PDE 解时也可能遇到类似的数值积分瓶颈, 这个工程经验有参考价值.
   - **Grammar-based expression representation**: APPS 的 grammar rules 表达方式比 FEX 的二叉树更为结构化. FEX 可以借鉴 grammar 机制来约束搜索空间 (例如定义 PDE 解的特殊结构).

3. **该引不该引**: **该引**. 作为 grammar-based RL symbolic regression 的主动学习变体, 应在 related work 中引用. 特别是 FEX 的 RL search 部分与 APPS 的 decoder policy gradient 有方法学相似性. 可以在 discuss future work 时提及 APPS 的 active query 思想作为 FEX 搜索效率改进的潜力方向.

4. **差异点:**
   - FEX 有 S_k 理论保证 (no CoD), APPS 无理论
   - FEX 做 PDE solving, APPS 做 ODE discovery from data
   - FEX 用二叉树 (preorder traversal), APPS 用 grammar rules
   - FEX 用 Adam + BFGS 两步优化参数, APPS 用 BFGS-only 拟合系数
   - FEX 的 RL controller 是搜索 PDE 解的表达式, APPS 的 decoder 是发现 data-generating ODE 的表达式

5. **填补 landscape gap 的方向**: APPS 的 active learning 思路可能帮助解决 FEX landscape gap #3 (搜索效率瓶颈) — 用 active region selection 替代 uniform evaluation, 减少候选解的评估成本. 这也与 gap #9 (synchrosqueezed/butterfly 先验) 有关联: phase portrait sketching 本质上就是在寻找 butterfly effect 最显著的 region 来最大化信息增益.

## Read by: 0616-fex (2026-06-16)

APPS (2409.01416) 使用 REINFORCE policy gradient + Transformer decoder + grammar-guided search 做 ODE 的主动符号发现, 与 FEX 的方法学核心 (RL + 表达式树搜索) 高度重叠但应用场景不同 (ODE discovery from data vs PDE solving). 撞车风险中低: 方法相似但任务不同, FEX 若扩展到 data-driven PDE discovery 则需要仔细对照.

**FEX 可借用的关键 insight**: phase portrait sketching 的 active region selection 策略—在候选解差异最大的区域重点查询数据—可能帮助 FEX 减少候选解评估的计算成本 (landscape gap #3). 但需要解决 PDE 场景下的 phase portrait 泛化问题 (APPS 作者自己也不确定可否扩展到 PDE).

**FEX 应引用的理由**: APPS 代表了 grammar-based RL symbolic regression 的主动学习前沿, 与 FEX 共享方法学血缘. 作者 Nan Jiang 同时也是 EGG-SR (2511.05849) 的第一作者, 后者已在 FEX landscape 中标注为可集成模块.

**差异化核心**: FEX 有 S_k 无维度灾难的理论保证, APPS 无理论; FEX 做求解, APPS 做发现; FEX 的 Adam+BFGS 两步优化 vs APPS 的 BFGS-only 系数拟合; FEX 的二叉树 vs APPS 的 grammar rules. 两者可视为 RL+SR 范式下互补的两个分支.
