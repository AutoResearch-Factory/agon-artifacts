# Robust Optimization for Mitigating Reward Hacking with Correlated Proxies (2604.12086)

- **Year**: 2026.04
- **Authors**: Zixuan Liu, Xiaolin Sun, Zizhan Zheng, Tulane University
- **Venue**: arXiv preprint
- **Keywords**: reward hacking, robust RL, correlated proxies, max-min optimization, occupancy measure, chi-squared divergence, linear rewards, RLHF
- **One-liner**: 用相关约束下的最坏奖励训练抗 reward hacking 策略。

## 精华提炼

### 管线/方法
1. 输入是一个参考策略 `pi_ref`、一个可观测 proxy reward、一个相关系数 `r`，以及隐藏的 true reward 假设。
2. 论文沿用 Laidlaw et al. 的 r-correlated proxy 定义：proxy 与 true reward 只要求在 `pi_ref` 访问到的 state-action 分布下相关。
3. reward hacking 被定义为：优化 proxy 后得到的策略在 true reward 下低于参考策略。
4. 与 ORPO 不同，本文不是针对一个固定 proxy 做 occupancy regularization。
5. 它把 reward hacking 写成 robust RL：策略最大化在所有满足相关约束的奖励函数中的最坏返回。
6. 不确定集 `R_corr` 包含所有在 `mu_pi_ref` 下均值为 `M`、标准差为 `V`、与 normalized proxy 相关为 `r` 的奖励函数。
7. 用 Radon-Nikodym derivative `L(s,a)=mu_pi(s,a)/mu_pi_ref(s,a)` 把 `E_mu_pi[R]` 改写成 `E_mu_pi_ref[L R]`。
8. 这样 inner minimization 的目标和约束都在参考 occupancy 上表达。
9. 对一般 reward 函数，作者对 inner minimization 写 Lagrangian。
10. 固定策略时，最坏 reward 的闭式形式为 `R*(s,a)=(L - lambda_1 R_proxy/V - lambda_2)/(2 lambda_3)`。
11. 对 `mu_pi_ref(s,a)=0` 的状态动作，约束不起作用，最坏 reward 理论上可以任意差。
12. 代回 dual 变量后，outer objective 化简成 proxy return 减去一个 robust penalty。
13. 最终目标近似为 `r E_mu_pi[R_proxy] - sqrt(1-r^2) sqrt(chi^2(mu_pi||mu_ref)-E_mu_pi[R_proxy]^2)`。
14. 这个式子是核心贡献：它比 ORPO 的 `proxy - lambda sqrt(chi^2)` 多了 `E_proxy^2` 校正项。
15. 作者证明 `chi^2(mu_pi||mu_ref) >= E_mu_pi[R_proxy]^2`，因此平方根项非负。
16. 他们还给出一个 lower-bound theorem：如果 true reward 在不确定集内，该目标下界 true improvement over reference。
17. 算法上，Max-Min Policy Optimization 用 PPO 更新策略。
18. 每轮采样当前策略轨迹，并用 discriminator 估计 occupancy ratio。
19. discriminator 目标是 logistic density-ratio loss，估计 `d_phi(s,a)=log mu_pi/mu_ref`。
20. `chi^2` 用 `E_D_pi[exp(d_phi)-1]` 近似。
21. 对离散环境，则可以直接用采样频率估计 discounted occupancy。
22. proxy reward 先在参考策略下估计均值和方差，再 normalization。
23. `E_mu_pi[R_proxy]` 和平方项用 double-sampling 估计，避免平方均值估计偏差。
24. 论文指出原 ORPO 实现里的 discriminator 每轮训练不足，会低估 occupancy shift。
25. 因此他们定义 ORPO*：只把 discriminator 训练日程改成和本文方法一致，用于隔离该因素。
26. 第二个方法是 Linear Max-Min。
27. 它假设 reward 是已知 feature 的线性组合 `theta^T phi(s,a)`，未知的是非负 feature weights。
28. 线性不确定集要求 `theta^T phi` 在参考 occupancy 下均值 0、方差 1、与 proxy 相关为 `r`。
29. 这样最坏 reward 不再是任意函数，而是 feature 权重的 adversarial reweighting。
30. 作者先构造 `Q=sum mu_ref phi phi^T`。
31. 再通过 Cholesky / matrix inverse square root whitening，让 `Q` 变成 identity。
32. whitening 后，inner problem 对每个 feature coordinate 可分解。
33. 最优 `theta_i*` 是一个 clipped quadratic solution：`max(0, q_i/(2 lambda_3))`。
34. dual 变量用 SciPy root solver / Levenberg-Marquardt 求 stationary point。
35. 线性版本的实际意义是可解释：最坏 reward 权重显示哪些 proxy feature 容易被 exploitation。
36. 论文把 Linear Max-Min 与一般 Max-Min 区分：前者更可解释，但当 feature prior 错时会变弱。
37. 实验覆盖五个环境：Traffic、Pandemic、Glucose Monitoring、Tomato Watering GridWorld、RLHF。
38. Traffic 使用 highway merging，proxy 用 velocity/acceleration/headway，true 用 commute/acceleration/headway。
39. Pandemic 使用 SEIR 类 simulator，proxy 漏掉 political cost，true 包含 infection、political、stage、smoothness。
40. Glucose 使用 SimGlucose，proxy 是 expected patient cost，true 是 `magni_bg` 健康风险。
41. Tomato 是 gridworld，proxy 在 sprinkler state 加 bonus，true 是 watered tomatoes。
42. RLHF 是 contextual bandit，proxy reward model 是 Pythia-70M，true reward model 是 Llama 3 Tulu V2 8B。
43. baseline 包括 ORPO、ORPO*；RLHF 还包括 reward model Ensemble。
44. 训练使用 PPO，非 LLM 环境沿用 ORPO 的 policy architecture 和 checkpoints。
45. 论文对 `r` 做 grid search，最终选择 Traffic 0.3、Pandemic 0.7、Glucose 0.9、Tomato 0.4、RLHF 0.4。
46. 评价包括 true reward、proxy reward、general worst-case reward、linear worst-case reward、unseen occupancy。
47. 对连续环境，作者用 discriminator 输出 infinity 作为 reference policy 未覆盖区域的近似信号。
48. 对 Tomato 这种离散环境，直接估计 unseen state-action occupancy。
49. 理论部分还给出 discriminator occupancy-ratio estimation 的 `O(n^{-1/4})` 型误差界。
50. Max-Min 收敛分析借用 General Utility RL，把最坏 reward 看成 occupancy utility 的 pseudo-reward。
51. 收敛界形式包含三项：`1/T`、`1/N`、以及 occupancy-ratio estimation error。
52. Linear Max-Min 的收敛没有完整新界，作者把 inner oracle 最优解与 alternating min-max 结果联系起来。

### 关键技术细节
- 一般 Max-Min 目标：`max_pi min_R E_mu_pi[R]`，其中 `R` 满足相关、均值、方差三个约束。
- 关键约束分布：所有相关定义都只在 `mu_pi_ref` 下成立，不在新策略访问的新区域成立。
- 最坏 reward 闭式解：来自 Lagrangian stationarity，核心变量是 occupancy ratio `L`。
- 实现里设置 `M=0, V=1`，因为它们只平移或缩放 objective，不改变策略排序。
- penalty 项不是单纯 `chi^2`，而是 `chi^2 - E_proxy^2`。
- discriminator network：两层 hidden layer，每层 256 units，ReLU。
- discriminator batch/epoch：Traffic 40000 batch、200 epochs、lr `5e-3`；Glucose 100000 batch、20 epochs、lr `1e-2`；Pandemic 3860 batch、15 epochs、lr `5e-4`。
- PPO policy network：Traffic 4 层 512，Pandemic 2 层 128，Tomato 4 层 512，Glucose 3 层 LSTM 每层 64。
- PPO 训练轮数：Traffic 250，Glucose 500，Pandemic 260，Tomato 500。
- PPO batch size：Traffic 40000，Glucose 100000，Pandemic 3860，Tomato 3000。
- discount factor 均为 0.99；RLHF 用 contextual bandit，`gamma=0`。
- Linear Max-Min 的 feature whitening 需要 `Q` full-rank；作者在 Traffic/Pandemic 实验中声称经验上 full-rank 且 condition 较好。
- 线性最坏权重 `theta` 被限制非负，最后手动 clip。
- 训练硬件：单张 RTX 4090 24GB 和 i9-13900KF。
- 训练时间：ORPO* / Max-Min / Linear Max-Min 约 Traffic 10h、Glucose 37h、Pandemic 19h、Tomato 1h。

### 核心结果
| 环境 | 关键发现 |
|------|----------|
| Traffic | ORPO true reward 最高，但 Max-Min worst-case 从 ORPO 的约 `-1.96e4` 提升到 `-268.31`，且 unseen occupancy 为 0。 |
| Traffic | Linear Max-Min true reward 接近 ORPO，linear worst-case 最好，为 `0.20±0.01`。 |
| Pandemic | Linear Max-Min true/proxy reward 最高，true `3.65±0.11`，proxy `7.60±0.13`。 |
| Pandemic | Max-Min general worst-case 最好，从 ORPO 的 `-5.31e6` 到 `-63.29±3.35`。 |
| Glucose | Max-Min worst-case `-1.71±0.25`，显著优于 ORPO `-27.54±0.32` 和 ORPO* `-8.79±0.27`。 |
| RLHF | Max-Min worst-case `-0.10±0.01`，优于 ORPO `-1.84±0.03` 和 Ensemble `-1.70±0.04`。 |
| Tomato | ORPO* 的 observed worst-case 更好，但 Max-Min unseen occupancy 最低，面对任意差的 unseen reward 更稳。 |
| robust r evaluation | Traffic/Pandemic 上 Max-Min 在多个 evaluation `r` 下平均 reward 高且方差低。 |
| discriminator ablation | ORPO* 常显著优于 ORPO，说明 occupancy-ratio 估计质量本身就是关键变量。 |

比较反直觉的点：
- ORPO 在 Traffic 的 true reward 最高，但 worst-case 极差，说明平均 true reward 与 robust safety 不是一回事。
- 训练更好的 discriminator 不一定提高每个具体 true reward 指标，例如 ORPO* 在 Traffic/Tomato 的 unnormalized true reward 可能低于 ORPO。
- Linear Max-Min 在结构 feature 正确时可解释且强，但在 unseen true feature 下可能退化。
- 对 `r` 的选择不是越高越好；高 `r` 过信 proxy，低 `r` 过保守，中间值更稳。

### 明确局限
- 一般 reward 空间只在线性 / 任意函数两端展开；若 reward 用 neural network 表示，inner minimization 不再有闭式解。
- Linear Max-Min 的 whitening 需要 `O(d^2)` 空间和 `O(d^3)` 时间，feature 维度高时很重。
- 低秩近似不一定安全，因为小 eigenvalue 在 inverse whitening 中可能很重要。
- 方法强依赖有意义的 feature prior；Glucose 和 Tomato 都暴露了 feature 设计困难。
- 框架假设固定 proxy、固定 reference policy、预先给定或调参得到的 `r`，没有在线 reward refinement。
- `r` 的选择没有原则性方案；作者只给 grid search、Fisher z confidence interval、minimax regret 方向。
- 对 continuous environment，unseen state-action 的检测依赖 discriminator divergence，可能漏掉 rare event。
- 结果没有第三方复现；代码可用但没有独立 artifact report。
- RLHF 实验使用较小 proxy model 与较大 true reward model，但仍是 reward-model-to-reward-model 的 proxy，并非真实人类评价。
- 论文自述使用 ChatGPT 改写语言和辅助找 prior work，但称算法、证明、实验由作者完成。

## Read by: 0616-fex (2026-06-17)

从 `0616-fex` 视角看，这篇论文的真正 claim 不只是“reward hacking 要 regularize”，而是把 r-correlated proxy 下的安全性写成一个可解的 max-min 问题。它的可撞车点很清楚：如果我们的方案也依赖 reference policy 上的 proxy-true correlation，并用 occupancy ratio 控制新策略离开 reference support 的风险，那么必须把本文当作最近 prior。

最值得复用的是三件事。第一，`chi^2 - E_proxy^2` 这个 penalty 结构给了比 ORPO 更细的 conservative bound。第二，ORPO* 说明很多“robustness gain”可能来自 discriminator 训练更充分，而不只是新 objective，因此我们的实验必须把 occupancy estimator 质量作为 ablation。第三，Linear Max-Min 的 adversarial `theta` 是很好的 feature vulnerability diagnostic，可以帮助解释 proxy 哪些分量最容易被 hack。

差异化方向也很明确。本文没有解决 `r` 如何可靠估计，尤其没有在线 human feedback 或 active query；没有解决 neural reward uncertainty 的 inner optimization；对 rare/unseen state-action 的处理依赖 discriminator，连续环境仍不可靠。如果 `0616-fex` 做 feature-level reward diagnosis，可以定位在“主动发现和修复 proxy feature”而不是只训练 robust policy。如果做 RLHF，可避开单一 reward-model proxy，研究多 evaluator、verifier 或 human audit 对 `r` 区间的约束。

近 6 个月 arXiv 关键词检索没有发现直接复现本文 max-min 公式并引用它的后续工作；但 reward hacking 方向很热，邻近工作已经在 RLHF reward modeling、auditing、benchmark 和 early-warning signals 上推进。写作时建议把本文列为“policy-side robust optimization under correlated proxies”，把 BNRM、ARA、PRIME、Hack-Verifiable Environments 列为并行的 reward-model/audit/evaluation 路线。
