# Complexity-Aware Deep Symbolic Regression with Robust Risk-Seeking Policy Gradients (2406.06751)

- **Year**: 2024.06
- **Authors**: Zachary Bastiani, Robert M. Kirby, Jacob Hochhalter, Shandian Zhe (Kahlert School of Computing & SCI Institute & Dept. of Mechanical Engineering, University of Utah)
- **Venue**: AISTATS 2026 (accepted, uses aistats2026.sty)
- **Keywords**: symbolic regression, deep reinforcement learning, risk-seeking policy gradient, DCT attention, BIC reward, GRPO, expression tree, decoder-only transformer, dual-indexed position encoding, frequency-domain attention
- **One-liner**: CADSR proposes a decoder-only transformer that attends in frequency space, uses BIC for automatic complexity-accuracy trade-off, and eliminates tail-barrier policy gradient zeroing via ranking-based weighted updates with GRPO.

## 精华提炼

### 管线/方法

CADSR 是 DSR (Deep Symbolic Regression, Petersen 2019) 框架的全面改进。总体 pipeline：给定输入输出数据对 {(x_i, y_i)} → 1) 用 decoder-only transformer actor 自回归生成表达式树 → 2) BFS 层序遍历逐层生成节点 → 3) 每个节点 token 由 DCT 频域注意力 + DPE 双索引位置编码 → 4) 采样一批表达式后，用 Levenberg-Marquardt 优化常数 token → 5) BIC 奖励函数评分表达式 → 6) Ranking-based risk-seeking policy gradient + GRPO 更新模型 → 7) Experience replay buffer 保留历史 top 表达式 → 循环迭代直到收敛。

### 关键技术细节

**模型架构**：
- Decoder-only transformer (无 encoder)，SRBench 用 1 decoder layer / 1 head / FFN 2048 / embedding dim 10；Fracture Mechanics 用 2 decoder layers / 3 heads / embedding dim 15
- DCT (Discrete Cosine Transform) 频域注意力：对嵌入矩阵 H 做 DCT 得到频域表示，截断保留前 M 个低频分量（SRBench: M=8，原始 dim=10；Fracture: M=12，原始 dim=15），在频域做 multi-head self-attention，再通过逆 DCT 零填充恢复维度
- 引入 DCT 的理论依据：高频分量通常对应噪声/离群值/参数扰动，滤除高频=去噪+稳定训练。实测减少 21% 采样时间（RTX 3080：DCT 3.576ms/expression vs 标准 attention 4.514ms）

**双索引位置编码 (DPE)**：
- 两层编码：深度 d（正弦编码，周期 10000）+ 水平位置 h（正弦编码，周期 10），各占 D 维
- 水平位置递归计算：左子 h = h_parent - h_parent/2^d，右子 h = h_parent + h_parent/2^d，根节点 h = 1/2
- 配合 BFS 层序遍历生成，使得已生成节点顺序不变，仅需一次向量化

**BIC 奖励函数**：
- BIC(τ) = k·log(S) - 2·log(p(τ))，其中 k = 表达式树节点数 + 常数 token 数，S = 训练样本数，p(τ) = Gaussian likelihood
- 无额外超参数，自动平衡复杂度与数据拟合；可替换为 Student-t 或 mixture of Gaussians likelihood
- 常数 token 惩罚的三重目的：(1) 常数需从数据估计，增加模型自由度 (2) 阻止滥用常数 token (3) 降低优化常数开销

**Robust Risk-Seeking Policy**：
- 定义 Tail Barrier：当 top-α% 采样表达式的映射奖励全部等于 R_α 时，policy gradient 权重为零，模型无法有效更新
- 引理 1 (Proof in Appendix)：任何连续 reward mapping 都存在产生 tail barrier 的风险
- 解决方案：rank-based linear weighting f(z) = λ·ReLU(1 - |{τ: R(τ) > R(z)}| / (αB/100))，只给 top-α% 非零权重，且高排名赋予大权重
- 引理 2 (Proof in Appendix)：该 step function 产生无偏梯度，且永不遭遇 tail barrier

**GRPO 集成**：
- 在 PPO/TRPO 框架内使用 group-relative advantage
- 对 top-α% 表达式中的每个 token，计算概率比 g = p(token|context,θ)/p(token|context,θ_old)，clip 到 [1-ε, 1+ε]（ε=0.2）
- KL 散度正则化：β·KL[p(·|θ)||p(·|θ_ref)]，θ_ref 每 G=5 epochs 更新一次
- Experience replay buffer：保留历史 top-α% 表达式，与当前 epoch 的 top 表达式合并用于梯度计算

**训练超参数**：
- SRBench: 600 epochs, batch 1000, α=5%, lr=1e-4 (Adam), λ=0.2, entropy coeff=0.005, β=0.01, C=5 steps/epoch, max depth=32, oversample γ=2
- 硬件：NVIDIA A40 GPU, 4 小时/问题上限，无 early stop
- Fracture Mechanics: batch 2000, 500 epochs, β=0.1, entropy coeff=0.0005, oversample γ=3

### 核心结果

**SRBench (133 problems with known solutions, 4 noise levels × 8 trials)**：
| Metric | Noise 0% | 0.1% | 1% | 10% |
|--------|----------|------|-----|------|
| CADSR Symbolic Solution Rate | 41.83% | **34.27%** | **33.57%** | **29.25%** |
| DSR-OCT Sol. Rate | 24.81% | 24.42% | 17.53% | 10.48% |
| AIFeynman Sol. Rate | **61.84%** (noise-free) → 0.86% (10%) | | | |
| CADSR Accuracy Rate | 55% | 54% | 55% | 54% |
| CADSR Simplified Complexity | 16.30 | 15.69 | 15.89 | 15.03 |

- 在 noise 0.1%/1%/10% 三个噪声级别上 Symbolic Solution Rate 全部第一
- 10% noise 时 CADSR 比 DSR-OCT 提升 179%（29.25% vs 10.48%），这是最显著的差异
- Simplified Complexity 始终最低档，说明 BIC 有效控制了表达式长度
- Black-box 120 问题上 CADSR 在 Pareto front（Model Size vs R^2），比 DSR-NCT 更准确，比 DSR-OCT 更简洁

**Ablation 关键发现**：
- BIC vs NRMSE：10% noise 时 BIC 的 Symbolic Solution Rate 提升 111%（30.93% vs 14.65%）
- BIC vs SPL/TPSR 复杂度奖励：BIC 无需调参即超越 SPL(η=0.99) 和 TPSR(λ=0.1) 在所有设置下的最佳表现，10% noise 时分别领先 3% 和 20%
- GRPO 移除导致 solution rate 下降 4.5%，且平均收敛 epoch 从 168 延至 257
- DPE 替代为 1D PE：noise-free 时微升 0.5%，但 10% noise 时下降 3%；DPE 平均每 epoch 发现 473 个 novel expressions，1D PE 仅 284 个

**Fracture Mechanics 应用**：
- 预测裂纹启动的 Fatigue Indicator Parameters (FIPs)，6 个晶粒级特征
- CADSR: R^2=0.626, Complexity=22.25 — 最佳 interpretability-accuracy trade-off
- GP-GOMEA: R^2=0.636 但 Complexity=73, 表达式几乎不可解释
- TPSR 完全失效 (R^2=0.046)，因为预训练模型分布外泛化失败

**Sensitivity Analysis (Strogatz, 0% noise)**：
- DCT cutoff M：不裁剪 (M=10) 导致 solution rate 下降 21.4%，M=4 仅降 0% 但复杂度微增
- Trust region ε：0.2 最优，增大到 1.0 或减小到 0.01 都导致性能下降
- KL penalty β：0.1 最优，过大或过小均降性能

### 明确局限

**作者承认的局限**：
1. 常数优化效率低——每个表达式顺序用 LM 优化而非并行，导致训练慢
2. 缺乏 early stopping 机制——固定 600 epochs 运行，浪费计算
3. 仅在 SRBench 和单一工程问题上验证，缺少更多真实世界应用

**审读发现的局限**：
1. 无真机实验——纯计算研究，不涉及物理机器人系统
2. DSR 在 SRBench 报告的版本是 DSR-NCT（无常数 token），作者补充了 DSR-OCT 作为公平比较，但仍缺少与 uDSR、DySymNet 的全面直接对比
3. BIC 在 noise-free 时比 NRMSE 略差 2%，虽然作者解释为 BIC 与表现的弱相关性，但说明 BIC 并非在所有场景都最优
4. 模型规模极小（1 decoder layer, 1 head, emb=10），是否能在更复杂问题上扩展未知
5. 表达式生成仅支持预定义 token 库（sin/cos/exp/log/sqrt/+/-/*///^），无法发现 token 库外的函数形式
6. 没有与 LLM-based 方法（如 GPT-based SR）的对比
7. 4 小时/问题时间上限在真实工程中可能不够
8. Experience replay buffer 可能引入分布偏移风险——论文未分析这种风险

### 跟我们的关系

对 0616-fex (soft object BFM / contact predictive pretrain) 课题：
- **不撞车**：CADSR 做的是 discovery-style symbolic regression——从数据中自动发现表达式；0616-fex 做的是 contact dynamics 物理建模中的表征学习/预训练。两个问题域完全不同
- **可用的工具/方法**：

  2. Ranking-based risk-seeking policy + GRPO 组合可以用于任何 RL-based 模型发现任务——特别是当奖励函数值可能高度倾斜时，可避免 tail barrier
  3. DCT frequency-domain attention 的思路可参考用于物理信号去噪——contact force 信号通常含高频噪声，频域注意力可能在特征提取中有用
  4. DPE 双索引位置编码适用于树/图结构化生成——如果 futures 工作涉及 BFM 模型的图结构搜索/生成
- **不该引为直接竞争**：这是一篇 method paper，与 0616-fex 的核心贡献（contact predictive pretrain / BFM / soft object manipulation）不重叠，可作为工具性引用
- **差异化**：CADSR 是通用 SR 工具，0616-fex 是 domain-specific 任务；如果 0616-fex 需要在 contact model discovery 场景中用 SR，CADSR 是可用的 baseline 之一（特别是在噪声数据下表现优秀）
