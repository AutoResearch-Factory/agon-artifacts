# Zero Collapse: A Failure Mode of Policy Gradient Methods in Discontinuous Reward Environments (2605.30896)

- **Year**: 2026.05
- **Authors**: Nishant Kumar, Enrique Areyan Viqueira, A. Greenwald
- **Venue**: arXiv
- **Keywords**: zero collapse, policy gradient, discontinuous reward, sparse reward, actor-critic, auction RL, first-price auction, step-size control
- **One-liner**: 不连续奖励会让 policy gradient 过冲后陷入零信号区。

## 精华提炼

### 管线/方法
1. 论文从一维阈值奖励函数入手建模: action 为 bid, 阈值为 h, 低于 h 时 reward 为 0, 高于 h 后 reward 为递减函数 f(a)。
2. 这个 reward landscape 有三个核心结构: 大片零奖励平坦区, 阈值处的跳变, 阈值右侧的单调递减收益区。
3. 最优动作位于不连续边界附近, 因此 policy gradient 的更新方向很容易把策略均值推向边界。
4. 策略带有随机探索, 参数更新又有有限步长, 所以策略分布可能跨过高收益边界并落入零奖励平坦区。
5. 一旦进入平坦区, 轨迹回报大量相同或为 0, score-function estimator 的权重失去区分度。
6. 作者把这种机制称为 zero collapse: 策略进入低信号区域后, 恢复极端低效。
7. 论文强调这不是普通的 sparse reward 难题, 因为高奖励区域一开始可能能被找到, 失败发生在找到后过冲。
8. 实验环境是 auction-like simulation, 对应 ad exchange / first-price auction 的阈值获胜结构。
9. agents 在有限 horizon 中竞拍 impression opportunities, campaign 有 market segment, reach R, budget B。
10. reward 被定义为 value minus cost, 通常需要终局满足 reach threshold 才能得到正向价值。
11. 为了分析, 多智能体环境被化约为单学习 agent, 其他 agent policy 固定并进入环境随机性。
12. joint policy 假设按 agent 条件独立分解, 因此每个 agent 的 policy gradient 只依赖自身 log-prob。
13. 评测算法包括 REINFORCE, REINFORCE with baseline, actor-critic (SARSA) with baseline, actor-critic (TD), 以及一个 Q-based variant。
14. 所有方法共享一个 Gaussian policy network, hidden activation 使用 ReLU 或类似非线性, final output 可换成 smooth activation。
15. 训练设置明确给出: 5000 epochs, 每 epoch 200 trajectories, learning rate 5e-7, average cumulative reward 最大值为 25。
16. optimizer 主体是 SGD, 作者认为 Adam / momentum 会更容易因累积梯度导致过冲。
17. 论文没有把核心贡献放在新算法, 而是把 reward geometry 与 policy optimization dynamics 的交互作为 failure mode。

### 关键技术细节
18. 核心 policy gradient 形式是 REINFORCE estimator: sum_t grad log pi(a_t|s_t) * Qhat_t。
19. baseline 版本使用 advantage: Qhat_t - V(s_t), 不改变期望但改变单条 trajectory 的更新符号。
20. flat zero-reward region 中, raw return 近似相同, 未加 baseline 时梯度信号容易消失或被噪声主导。
21. 加 baseline 后, 低于 baseline 的 flat-region trajectory 会得到负 advantage, 可形成把概率质量推离坏区域的恢复力。
22. 但 baseline 也可能在阈值附近制造来回振荡, 作者称为 thrashing。
23. step-size mitigation 的核心公式是 alpha = sqrt(epsilon / (||grad J||^2 + eta))。
24. 当梯度范数很大时, 该公式使参数空间更新范数约为 sqrt(epsilon), 相当于 normalized gradient step。
25. 作者指出这个方法只约束 parameter-space shift, 不直接约束 policy-space shift。
26. 因此 TRPO / PPO 这类 trust-region 或 clipped policy update 被提出为自然后续方向, 但论文没有实验证明。
27. 初始化策略被认为很关键: 随机初始化可能把大量概率质量放进 flat region。
28. 推荐初始化在 informative non-flat region, 即 reward 有变化、能产生方向信号的区域。
29. 作者说不需要精确知道非零阈值, 粗略知道一个 reward 变差的 upper bound 就足够。
30. 对 digital advertising, 可以用历史竞价、domain knowledge 或 simulator 估计边界。
31. policy parameterization 方面, 作者主张 final layer 使用 softplus 等 smooth activation。
32. 理由是 ReLU 正区间斜率恒定, 固定参数步长对应近似固定 action-space 步长, 临近阈值时仍可能过冲。
33. softplus 的斜率连续变化, 在部分区域自然衰减输出变化, 等价于更温和的 effective step size。
34. 作者明确不偏好 tanh / sigmoid 这类强饱和激活, 因为它们会过度压制梯度并拖慢学习。
35. actor-critic 的失败解释聚焦 critic mismatch。
36. value network 虽然原则上能拟合不连续函数, 但训练中学到的是过于平滑的 value surface。
37. 这种平滑 critic 会低估阈值边界的尖锐程度, 给 actor 一个错误的可优化斜坡。
38. actor 比 critic 更快改变 action distribution, critic lag 导致 actor 基于不成熟价值估计过冲。
39. bootstrapping 会传播 value error, 所以 TD/SARSA actor-critic 比 Monte Carlo REINFORCE 更脆弱。
40. Appendix 给出 Markov game, single-agent reduction, baseline advantage, TD advantage, SARSA-style advantage 的推导。
41. 代码框架提到 Derby, 类似 OpenAI Gym 的 env.init / reset / step loop, 但链接被 redaction 包住。

### 核心结果
42. pre-mitigation 结果: 所有 policy gradient 变体先从强负 reward 改善到接近 0, 然后大多 collapse 到 near-zero reward。
43. actor-critic variants collapse 最明显, 短暂改善后陷入低 reward 区域。
44. REINFORCE with baseline 曾短暂达到 near-optimal reward, 但之后出现高方差 reward rain pattern 并最终回到 near-zero。
45. 作者把这个现象解释为策略成功移动到边界附近, 但继续更新时越过最优窄区间进入 flat region。
46. post-mitigation 结果: step size control, informed initialization, smooth parameterization, baseline 组合后, REINFORCE with baseline 稳定接近最优。
47. REINFORCE without baseline 也明显改善, 但后期仍有 reward rain, 表明不加 baseline 稳定性不足。
48. actor-critic 在 mitigations 后仍失败, TD 和 SARSA variants 仍早期改善后快速 collapse。
49. Q-based variant 改善更慢, 未达到高 reward 并出现退化。
50. 最反直觉的结果是: baseline 既能防止 flat-region 完全无信号, 又可能在边界附近引发 oscillation。
51. 另一个关键发现是: critic 本来是降低方差的工具, 但在不连续 reward 中可能因平滑 bias 加速错误更新。
52. 论文的 empirical evidence 主要来自曲线图, 没有提供多 seed 统计表、置信区间或完整 ablation 数字。

### 明确局限
53. 论文是 arXiv preprint, 未见 peer review 信息。
54. 实验是 stylized auction-like simulator, 不是真实 ad exchange 或公开 auction benchmark。
55. 没有报告随机种子数、error bars、confidence intervals, 图中曲线很难判断统计显著性。
56. 关键 mitigation 是组合使用, 但每个组件的单独贡献没有系统量化。
57. 作者提到 Adam 更容易 collapse, 但说是 preliminary experiments, 没有放完整结果。
58. TRPO/PPO 被提出为自然方向, 但没有被实测, 因此不能把 trust-region 有效性当作本文结论。
59. policy network 架构只粗略描述为 dense / Gaussian policy, 具体层数、width、std parameterization、normalization 细节不足。
60. Derby framework 和源码链接被 redaction 包裹, 可复现性不足。
61. zero collapse 的理论说明是机制性和经验性, 没有给出正式概率界、escape time bound 或 impossibility result。
62. reward landscape 的一维阈值图很清楚, 但高维、多阈值、组合动作空间中的 collapse 条件还没有形式化。
63. 对 FEX 这种离散表达式树 RL, reward discontinuity 的来源不是 bid threshold, 而是表达式合法性、PDE residual、参数优化成功与否、复杂度惩罚等非平滑评分。
64. 因此本文不能直接证明 FEX controller 会 zero collapse, 只能提供可迁移的失效模式假设。

### 跟我们的关系
65. `0616-fex` 的主题是 FEX: RL for Symbolic Regression, 即用 policy gradient / RL controller 搜索符号表达式树。
66. 本文不直接撞车 FEX, 因为它处理的是连续 bid action 和 auction threshold reward, 不是 PDE symbolic solver 或 expression tree search。
67. 但它与 FEX 的最大理论缺口高度相关: FEX 有 approximation theory, 但 RL controller 搜索复杂度和失效机制缺少分析。
68. 如果 FEX reward 来自 PDE residual, 参数优化结果, 表达式复杂度, 语法合法性和 benchmark success, reward surface 很可能也是非光滑、阶跃式、稀疏的。
69. zero collapse 可转译为 FEX 版假说: controller 学到某些局部高分表达式模式后, 因 policy update 过大进入大量无效或低分表达式区域, 之后梯度由低信号 samples 主导。
70. actor-critic mismatch 对 FEX 尤其值得警惕, 因为 critic 若把表达式树 reward 平滑化, 可能高估跨语法边界或跨结构编辑的收益。
71. 对 `fex-failure-taxonomy`, 本文可作为一个 failure class: discontinuous reward / flat-region collapse / critic smoothing bias。
72. 对 `fex-search-complexity`, 本文不是理论证明, 但提示可以研究 escape time, flat-region measure, policy entropy 和 update norm 的关系。
73. 对 `fex-curriculum-depth`, 本文支持小步长、warm-start 和 staged search 的必要性, 因为深度增加会让 low-signal region 体积变大。
74. 对 `fex-llm-skeleton`, LLM skeleton 可看作 informed initialization, 把 controller 初始分布放在 informative expression region。
75. 对 `fex-active-collocation`, baseline / advantage shaping 可以和 residual-based evaluator 结合, 避免所有差表达式都获得近似同一低 reward。
76. 该引不该引: 如果论文讨论 FEX 的 RL 不稳定、reward shaping、controller collapse、频率分离导致 RL dynamics instability, 应引用。
77. 如果只写 FEX approximation theory 或 PDE solver accuracy, 本文不是必要引用。
78. 差异化策略: 不要泛泛声称发现了 policy gradient 在非光滑 reward 中会失败, 本文已经覆盖了这个机制。
79. 更好的新 claim 是: 在 symbolic PDE expression search 中定义并量化 zero-collapse-like failure, 给出 FEX-specific diagnostics 和修复。
