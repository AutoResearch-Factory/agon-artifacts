# PACMANN: Point Adaptive Collocation Method for Artificial Neural Networks (2411.19632)

- **Year**: 2024.11
- **Authors**: Coen Visser, Alexander Heinlein, Bianca Giovanardi (TU Delft)
- **Venue**: arXiv / submitted to Journal of Computational Physics
- **Keywords**: PINNs, adaptive sampling, collocation points, PDE residual gradient, min-max optimization, high-dimensional PDEs, gradient ascent
- **One-liner**: PACMANN moves PINN collocation points toward high-residual regions using gradient-based optimization of the squared PDE residual, matching SOTA in low dimensions and outperforming it in high dimensions with low computational overhead.

## 精华提炼

### 管线/方法

1. **问题设定**：PINN 求解 PDE，损失函数含 PDE 残差项在 collocation points 上计算。
2. **核心思想（min-max formulation）**：将训练视为 min-max 问题——神经网络参数 theta 最小化损失，同时 collocation points X_r 最大化残差损失。
3. **PACMANN 算法流程**：
   - 初始用均匀采样（Hammersley 序列或等距网格）放置 N_r 个 collocation points
   - 训练 P 次迭代（resampling period = 50）
   - 暂停训练，计算每个 collocation point 处的平方残差 r^2 对输入坐标的梯度
   - 用梯度信息沿 residual 增长方向移动 collocation points（梯度上升），重复 T 步
   - 移出域外的点用均匀分布重新采样替代
   - 恢复训练，循环往复

### 关键技术细节

- **激活函数**：全部实验使用 tanh
- **训练协议**：总计 50000 次迭代，分 5 个 phase，每 phase 包含 7000 次 Adam（lr=10^{-3}）+ 3000 次 L-BFGS；仅在 Adam 阶段进行 collocation point 重采样
- **默认 resampling period**：P=50
- **推荐 steps 数**：T=5（在精度与效率间平衡）
- **测试的平台**：NVIDIA Tesla V100S GPU on DelftBlue supercomputer
- **代码库**：基于 DeepXDE + PyTorch 1.12.1；开源在 https://github.com/CoenVisser/PACMANN
- **重采样方式**：只在 Adam 训练阶段进行（L-BFGS 阶段不重采样以避免破坏收敛）
- **神经网络架构**：沿用 DeepXDE 各 test case 的默认架构（未做架构搜索）
- **每次实验跑 10 个 random seed**，报告均值和标准差

**六种 collocation point 移动优化器**：
1. Gradient Ascent：直接沿梯度方向移动 x_{i+1} = x_i + s * f'(x_i)
2. Nonlinear Gradient Ascent：用 tanh 压缩梯度 x_{i+1} = x_i + s * tanh(f'(x_i))
3. RMSprop：自适应 stepsize，S_{i+1} = beta*S_i + (1-beta)*(f'(x_i))^2
4. Momentum：加权历史梯度平均
5. Adam：组合 RMSprop + Momentum，带 bias correction
6. Golden Section Search：沿梯度方向一维线搜索

Adam 优化器超参数：beta1=0.9, beta2=0.999, V0=0, S0=0, epsilon=1e-8

### 核心结果

**Burgers 方程 (1D, 2500 collocation points)**：
| 方法 | L2 相对误差 (Mean) | 1 SD | 平均运行时间 |
|------|---------------------|------|-------------|
| Uniform grid | 25.9% | 14.2% | 425s |
| Hammersley | 0.61% | 0.53% | 443s |
| Random resampling | 0.40% | 0.35% | 423s |
| RAR | 0.11% | 0.05% | 450s |
| RAD | 0.16% | 0.10% | 463s |
| RAR-D | 0.24% | 0.21% | 503s |
| **PACMANN-Adam** | **0.07%** | **0.05%** | 461s |

PACMANN-Adam 精度远超 RAR（0.07% vs 0.11%），但运行时略高于 RAR。

**Allen-Cahn 方程 (1D, 2500 points)**：
| 方法 | L2 相对误差 (Mean) | 1 SD | 平均运行时间 |
|------|---------------------|------|-------------|
| Uniform grid | 44.34% | 18.58% | 634s |
| RAR-D | 0.28% | 0.13% | 632s |
| **PACMANN-Adam** | **0.16%** | **0.07%** | 632s |

同运行时下 PACMANN 精度明显优于所有 baseline。

**高维 Poisson 方程 (5D, 750 points)** —— 关键结果：
| 方法 | L2 相对误差 | H1 半范数 | 平均运行时间 |
|------|-----------|-----------|-------------|
| Uniform grid | 40.32% | 62.58% | 744s |
| Random resampling | 11.47% | 13.36% | 751s |
| RAR | 62.00% | 69.97% | 773s |
| RAD | 11.69% | 13.19% | 841s |
| RAR-D | 89.31% | 93.83% | 784s |
| **PACMANN-Adam** | **8.35%** | **10.32%** | 786s |

**Surprising findings**：
- RAR 和 RAR-D 在 5D 完全失效（精度比 random resampling 还差很多），验证了 Wu et al. 报告的"高维扩展性差"
- PACMANN 在 5D 不仅精度最高，运行时间比 RAD 还低（786s vs 841s）——实现了"高维效率优势"
- 在超大 step size 下，Adam/RMSprop 将所有点移出域外，此时 PACMANN 退化为 random resampling（精度收敛到 random resampling 水平），这是一个非常 clean 的 behavior 验证

**3D Navier-Stokes (300 points)**：
所有方法精度相近，PACMANN 对 u/v/w 三个分量分别达 0.46%/0.49%/0.33% L2 误差，均为最优或并列最优。

**Re-entrant corner (不规则域, 75 points)**：
PACMANN-Adam 在 L2 误差上最优（0.43%），点集体向原点（代数奇点位置）移动——直观展示了方法的有效性。

**超参数敏感度发现**：
- **Stepsize s 是主导超参数**，影响显著；Number of steps T 影响较小
- 推荐 T=5 固定（移除一个超参数）
- Resampling period 增大时 PACMANN 精度下降慢于其他方法——方法更鲁棒
- 不同优化器的最优 stepsize 相差可达数个数量级（梯度上升：1e-6 到 Adam：1e-5 到非线性梯度上升：1e-4）

### 明确局限

1. **无理论收敛保证**：min-max 公式有 remark 连接 L-infinity 范数，但没有收敛性证明或误差界；这是纯实验论文
2. **未做广泛架构搜索**：直接用了 DeepXDE 默认架构——不同架构下 PACMANN 是否仍最优未知
3. **无真机实验**：全是数值仿真 benchmark
4. **未探索与其他 PINNs 改进组合**：loss weighting（如 Wang et al. 的 LAA）、causal training、curriculum learning 等——PACMANN 只关注 collocation point 分布
5. **梯度计算成本**：需要额外 forward+backward 来计算 residual gradient on collocation points，高维时可能成为瓶颈（虽论文声称成本低）
6. **对激活函数未做消融**：全用 tanh，sin/ReLU/GELU 未测试
7. **仅测试了 forward 和简单 inverse 问题**——未涉及复杂 inverse（如参数场反演）、stiff PDEs、多尺度问题
8. **对比方法有限**：只对比了 RAR/RAD/RAR-D、未包含 DAS-PINNs、因果采样等更 recent 方法在全部实验中的对比
9. **代码基于老版本**：PyTorch 1.12.1，可能需更新
