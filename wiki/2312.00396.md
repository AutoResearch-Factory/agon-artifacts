# GFN-SR: Symbolic Regression with Generative Flow Networks (2312.00396)

- **Year**: 2023.12
- **Authors**: Sida Li (UChicago), Ioana Marinescu (Princeton), Sebastian Musslick (U Osnabrück, Brown)
- **Venue**: arXiv (submitted to NeurIPS 2021 AI for Science Workshop)
- **Keywords**: symbolic regression, GFlowNet, trajectory balance, expression tree, deep learning, diverse candidate generation, reward baseline, noisy regression
- **One-liner**: 用 GFlowNet 替代 RL 做符号回归,通过生成与 reward 成比例的多样化候选表达式,在噪声数据下显著优于 DSR 和 BSR。

## 精华提炼

### 管线/方法

整个 GFN-SR 流程：

1. **状态空间建模**：将表达式树的构造过程建模为 DAG 上的遍历。source state s₀ 是空树,terminal states 是所有完整表达式树。采用 pre-order traversal 顺序插入节点。

2. **前向策略 P_F**：LSTM 网络接收 parent 和 sibling token 的 one-hot 编码,输出一个 categorical distribution over token library L。Library 包含: sqrt, log, exp, sin, cos, (·)², ×, ÷, +, -, 变量 xᵢ, 常数 token。

3. **In-situ 约束剪枝**（采样前执行）：
   - Composition constraints: 禁止 trig(trig(...))、禁止互逆函数直接组合（如 (√x)²）
   - Depth constraints: 达到最大深度时只允许变量和常数
   - Constant constraints: unary 不接受常数参数;binary 常数只能在右子节点

4. **Trajectory Balance Loss**：由于 pre-order 下每个 state 有唯一前驱, P_B ≡ 1, TB loss 简化为:
   ```
   L_TB(τ) = (log(Z_θ · Π P_F(s_t|s_{t-1}) / R(s)))²
   ```

5. **Reward Baseline 增强**：
   - vanilla reward: R(s) = 1/(1+NRMSE(s))
   - 调整后: R_B(s) = R(s) · (1 + γ·(R(s)/B - 1))
   - B 初始化为第一批平均 reward,训练中逐步提高至 top-performing 表达式的平均 reward
   - γ=0.9（控制调整程度）

6. **常数优化**：表达式中常数 token 不参与构造（不赋值）,而是用 BFGS 在完整表达式上优化常数值以最大化 R(s)。首次出现做 BFGS 并缓存;后续命中从缓存读取,定期重新优化。

### 关键技术细节

- **LSTM 架构**: 2 recurrent layers, 250 hidden states per layer. Adam optimizer, lr=5e-4
- **Batch size**: 250
- **训练预算**: 每 run 约 2500 万次表达式评估
- **GFN 训练机制**: mini-batch gradient descent on TB loss,同时学习 Z_θ 和 P_F 参数
- **表达式生成**: 训练后从 P_F 采样,取最频繁出现的表达式作为解（比 DSR/BSR 取 Pareto front 更严格）
- **计算资源**: NVIDIA Quadro RTX 6000 GPU, Intel Xeon Gold 6242 CPU
- **比较方法**: DSR (DSO package), BSR (autoRA theorist 实现), GP (GPLearn)

### 核心结果

**Nguyen Benchmark (无噪声, 20 runs)**:

| Benchmark | GFN-SR | DSR | GP | BSR |
|-----------|--------|-----|-----|-----|
| Nguyen-4 | .02(.002) | **0(0)** | .05 | .01 |
| Nguyen-5 | **.15(.021)** | .35 | .51 | .19 |
| Nguyen-6 | .25(.082) | **.012(.005)** | .90 | .47 |
| Nguyen-10 | .053(.014) | **.013(.019)** | .044 | .025 |
| Nguyen-12 | **.037(.014)** | .040(.004) | .080 | .082 |

GFN-SR 整体 competitive, 但不总是最优。关键发现：GFN-SR 不是 return-maximization 目标,但表现接近 DSR。

**Noisy Synthetic Benchmark (10% noise, 20 runs, recovery rate %)**:

| Equation | BSR | DSR | GFN-SR |
|----------|-----|-----|--------|
| f₁ = x³+x²+x | 30% | 75% | **100%** |
| g₁ = sin(x)(√x+eˣ) | 5% | 5% | **15%** |
| f₂ = log(x+1) | 0% | 5% | **15%** |
| g₂ = 0.5x | 90% | 40% | **95%** |
| f₃ = x²-4x+3 | 5% | 5% | **95%** |
| g₃ = 4(1-√x) | 75% | **90%** | 85% |

6 个方程中 GFN-SR 赢了 5 个。核心洞察：噪声模糊了 reward landscape,DSR 容易被错误的高 reward mode 困住（如 f₁→g₁ 的 cross-mode confusion）,GFN-SR 通过生成多样化候选来规避此问题。

**Surprising 发现**: g₁ 的 recovery rate 整体惨淡（最高 15%）,即使在低噪声下这些符号不同但数值接近的方程几乎不可区分。

### 明确局限

1. **基准太小**: 仅在 Nguyen (12 个玩具方程) 和自建 6 方程 synthetic 上测试。作者承认需要 SRBench 和 Feynman database 等更大基准
2. **No real-world noisy data**: 所有噪声实验都是 synthetic,没有真实世界噪声数据的验证
3. **Pre-order 限制**: 强制 pre-order traversal 严重限制 state space 的可能转移数;如果允许从任意未完成节点扩展,DAG 更大、探索更充分但更慢
4. **LSTM 可能被替代**: 作者自己承认 Transformer 可能更好地捕捉表达式的 global structure
5. **未与其他技术结合**: landajuela2022 的 unified framework（SR+GP+多项式回归）值得探索
6. **常数优化开销**: BFGS 优化常数需要额外计算;cache 机制缓解但不消除
7. **没有真机/真实物理实验**: 纯数值实验
8. **Recovery 定义严格**: GFN-SR 取最频繁出现表达式 vs DSR/BSR 取 Pareto front 上的任意匹配 — 前者更严格但作者未说明这种差异对公平性的影响

## Read by: 0616-fex (2026-06-24)

本文对 FEX（函数表达式提取）课题的核心价值在于 **多样性生成机制** 和 **噪声鲁棒性**。

**撞车评估**: 不直接撞车。GFN-SR 解决 "给定 (X,y) 找表达式" 的传统 SR 问题;FEX 更可能是从文本/代码/对话等非结构化数据中提取函数表达式。但若 FEX 包含 candidate expression search/verification 步骤,GFN-SR 的多样候选生成是直接可复用的技术。

**可用工具**:
1. GFlowNet + TB loss 的表达式树生成框架 — 如果 FEX 需要从多个可能的表达式中做 selection/ranking
2. Reward baseline R_B(s) 公式 — 用于 FEX 中任何需要区分 candidate quality 的 scoring step
3. 三类 in-situ constraints — 直接用于限制候选表达式空间,降低 search complexity
4. BFGS 常数优化 + cache 机制 — 如果 FEX 需要精确拟合表达式中的常数参数

**差异化重点**: FEX 是 "从什么媒介提取" (extraction from media),GFN-SR 是 "从数据拟合" (fitting from data)。两者可互补：FEX 提取候选表达式结构 → GFN-SR 做参数优化和验证。
