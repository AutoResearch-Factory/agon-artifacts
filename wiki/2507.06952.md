# What Has a Foundation Model Found? Using Inductive Bias to Probe for World Models (2507.06952)

- **Year**: 2025.07
- **Authors**: Keyon Vafa (Harvard University), Peter G. Chang, Ashesh Rambachan, Sendhil Mullainathan (MIT)
- **Venue**: ICML 2025 / arXiv
- **Keywords**: world models, inductive bias probe, extrapolative predictability, orbital mechanics, Othello, lattice tasks, symbolic regression, next-token partition
- **One-liner**: 用归纳偏置探针检验模型是否真学到世界模型。

## 精华提炼

### 管线/方法
- 论文目标不是问模型在原始 next-token 任务上准不准，而是问模型迁移到新任务时的归纳偏置是否符合给定世界模型。
- 输入是一组序列样本 `x`、任务输出 `y`、以及研究者显式指定的世界模型状态映射 `phi(x)`。
- 世界模型由状态空间 `Phi` 和状态函数 `phi: X -> Phi` 表示；若输出可写成 `y = g(phi(x))`，数据集就与该世界模型一致。
- Foundation model 被形式化为一个学习算法：给定小数据集 `D` 后，返回预测函数 `m_D`。
- 探针构造很多与世界模型一致的小型 synthetic datasets，而不是只做一个 probe classifier。
- 每个 synthetic dataset 都用于 fine-tune 或 in-context adapt foundation model。
- 然后在 held-out inputs 上收集模型对不同 `D` 的外推函数。
- 核心量是 extrapolative predictability：一个输入的模型输出能否预测另一个输入的模型输出。
- 如果模型真的按世界模型外推，那么世界模型状态相近的输入应该表现出更高的可预测性。
- 论文引入 oracle foundation model：oracle 直接访问真实 `phi(x)` 和 admissible function family `G`。
- 连续状态版本把模型的 extrapolative predictability 与 oracle calibration curve 对比；理想情况接近 45 度线。
- 有限状态加二元输出时，探针退化为两个指标：R-IB 和 D-IB。
- R-IB 衡量同一状态内预测是否一致，类似 precision。
- D-IB 衡量不同状态间预测是否可区分，类似 recall。
- 实验不依赖内部表示探针，主要看模型被少量新标签适配后的行为。
- 物理实验中，输入是行星轨道序列，指定的世界模型是牛顿力学状态向量。
- Othello 实验中，输入是棋局 move sequence，指定状态是 8x8 board。
- lattice 实验中，输入是 `L, perp, R` 的移动序列，指定状态是线性格点位置。
- 论文还提出 next-token coarsening 分析，用 legal next tokens 的等价类解释模型可能真正用的偏置。
- 对 VLN 课题可复用的主线是：把导航轨迹预测能力和可迁移空间状态理解拆开测。

### 关键技术细节
- 轨道数据每条序列有 1,000 个观测，表示太阳和 1 到 10 个行星的位置。
- 轨道采样：行星数来自 `Unif([1,...,10])`。
- 偏心率来自 `Beta(alpha=0.867, beta=3.03)`。
- 半长轴来自 `Unif(0.3, 42)` AU。
- 行星质量来自 `LogUniform(1e-7, 1e-3)`。
- 恒星质量来自 `Unif(0.5, 5)`。
- 轨道序列一半使用 6 个月间隔，一半使用 1 周间隔，并在序列开头放特殊 token 标注间隔。
- 训练集规模是 10M sequences 和 20B tokens。
- 物理模型是 109M 参数 transformer，12 层、12 heads、768 embedding dimensions。
- 训练目标是 discretized coordinate next-token prediction；每个 `(x,y)` 坐标轴被离散到 7K bins，范围是 -50 到 50 AU。
- 轨道 transformer 训练 25 epochs，使用 8 张 H100。
- 训练优化器是 Adam，学习率 `6e-4`，2,000 warmup iterations，weight decay `0.1`，gradient clipping `1`。
- 通用模型比较包括 RNN、LSTM、Transformer、Mamba、Mamba-2。
- RNN 在 Othello 中用 6 个单向 RNN layers 和 768 embedding dimensions；lattice 中用 2 层。
- Mamba 用 24 Mamba layers、768 embedding dimensions、SSM state expansion factor 16、block expansion factor 2、convolution width 4。
- Mamba-2 使用同样规格，但 mixer 换成 Mamba-2 module。
- physics IB probe 创建 100 个数据集，每个数据集 100 个样本。
- 每个数据集先采样 100 条序列，再用 50 个 `6 x 1` 高斯随机矩阵选择一个一维投影。
- 指标计算时从所有输入随机采 2,000 个样本，按 oracle pairwise distance 分 20 个等宽 bins。
- force-vector 实验用太阳系 8 条轨道序列，随机标注 1% 真实 force vectors。
- force-vector fine-tune 使用 MSE，10,000 steps，学习率 grid 是 `1e-6` 到 `5e-4`，最佳 `2e-4`。
- force-magnitude symbolic regression 用 9K two-body train systems 和 1K test systems。
- 对每条 test sequence 加入两个随机 timestep observation，避免完全陌生序列。
- 再选 5,000 个与训练状态最近的 test timesteps，确保测试点接近训练分布。
- force-magnitude fine-tune 10,000 steps，batch size 64。
- 对 1,000 个随机 test points 做 PySR symbolic regression。
- PySR 搜索 max size 20，binary operators 是 addition 和 multiplication，unary operators 是 sine、cosine、exponentiation、inverse。
- PySR 每次三次 random restarts，每次 100 iterations。
- oracle force baseline 是基于真实 state 的 kNN，`k=2`。
- LLM physics appendix 测了 o3、Claude 4 Sonnet、Gemini 2.5 Pro。
- LLM 实验每个模型看 5 个 solar systems，每个 450 observations，只给 10 个 force magnitude labels。
- lattice 使用长度 100 的 valid move sequences，训练集 10M tokens，hold-out 100K tokens。
- Othello 使用 20M games 训练，3.8M hold-out games，序列最多 60 moves。
- Othello 为避免 permutation shortcut，构造 depth 10 openings，选一个重复出现很多次的 board，得到 210 个 openings 和 2,100 个 subsequences。

### 核心结果
- 轨道 next-token prediction 极准，held-out trajectory 的 `R^2 > 0.9999`。
- 物理 1-step MSE：transformer 是 `1.90e-8`，previous-position baseline 是 `1.16e-4`，per-orbit mean 是 `7.53e-2`。
- 物理 5-step MSE：transformer 是 `1.56e-8`，previous-position baseline 是 `1.37e-4`，per-orbit mean 是 `5.53e-2`。
- 物理 100-step MSE：transformer 是 `3.74e-5`，previous-position baseline 是 `4.04e-2`，per-orbit mean 是 `1.39e-1`。
- 但同一个 transformer 的 inductive bias calibration 远离 oracle 45 度线。
- 关键反直觉结论：会高精度预测轨道，不等于有牛顿力学世界模型偏置。
- force-vector 图中，transformer 预测的力方向和大小明显偏离真实牛顿力。
- 对 transformer force magnitudes 做 symbolic regression，5 个 galaxy sample 都没恢复 `F proportional to m1 m2 / r^2`。
- 示例错误 law 包括只依赖 `m1`、`m2` 或 `sin/cos/exp(r)` 的非物理表达式。
- oracle kNN 在同样 symbolic regression 流程下能恢复正确万有引力形式。
- two-body center-of-mass 版本也失败：10M two-body systems、10 epochs、质量范围 `1e-4` 到 `1e-2` solar masses，仍偏离 oracle。
- LLM physics 也失败：o3 和 Gemini 2.5 Pro 的 symbolic regression 近似只得到 `F proportional to m1`。
- Claude 4 Sonnet 的 LLM symbolic regression 近似得到 `F proportional to 1/(m2 - 0.50)`。
- lattice 和 Othello 中，所有模型在 next-token validity 上表现很好。
- lattice next-token test 对所有模型是 `1.00`。
- Othello next-token test：RNN `0.992`，LSTM `0.996`，Transformer/Mamba/Mamba-2 均 `0.999`。
- 但 Othello IB 明显低于 next-token 表现，说明合法动作预测不是完整 board state 理解。
- NTP-trained Othello Transformer 的 R-IB/D-IB 是 `0.703/0.624`。
- NTP-trained Othello Mamba 的 R-IB/D-IB 是 `0.682/0.728`。
- NTP-trained Othello Mamba-2 的 R-IB/D-IB 是 `0.653/0.694`。
- lattice 5-state 上，NTP-trained LSTM 表现较强，R-IB/D-IB 是 `0.782/0.921`。
- lattice 5-state 上，NTP-trained Transformer 较弱，R-IB/D-IB 是 `0.483/0.677`。
- next-token partition 分析显示，同为不同真实状态时，legal-next-token 相同的 pair 更容易被模型混在一起。
- Othello Transformer 的 `D-IB_q=` 是 `0.458`，`D-IB_q!=` 是 `0.625`。
- 这个差异支持“模型学的是合法下一步集合的粗化状态，而不是完整 board state”。
- transfer learning 结果与 IB 指标相关，尤其 Edge Balance 的 NLL/ACC 相关达到 `0.970/0.960`。
- 论文的 strongest claim 不是“模型没有任何世界表示”，而是“模型的迁移归纳偏置不等于给定真实世界模型”。

### 明确局限
- 方法必须先指定候选世界模型 `phi`，不能自动发现模型内部真正使用的状态。
- 如果研究者指定的 `phi` 不完整或粒度错误，探针只能否定这个候选，而不能穷尽所有可能世界模型。
- 物理实验主要是 synthetic orbital simulations，不是真实天文观测数据。
- 行星间交互被忽略，因为行星质量远小于太阳，这让物理设置更干净但也更简化。
- transformer 的失败不直接推出所有架构都无法学习物理；作者也指出带物理归纳偏置的 GNN 是有希望的。
- Othello 和 lattice 都是状态可精确定义的 toy/closed-world domain，迁移到开放环境需要重新定义状态。
- R-IB/D-IB 依赖大量 fine-tuning probes，计算成本比静态 linear probe 高。
- LLM 实验是小规模 appendix，因成本只用了 5 个 solar systems 和 2,250 observations per LLM。
- 文中没有真机 embodied navigation 实验，也没有 VLN benchmark。
- 对 0616-vln 来说，它不能直接回答 VLM/LLM 是否能在真实室内导航中构建 metric/topological map。
- 论文提供的是诊断框架，不是一个新的 VLN policy、planner 或 mapping module。
- 如果我们的课题是“用轨迹预测证明空间世界模型”，这篇会构成直接反例和审稿风险。
- 如果我们的课题是“评估或增强 VLN agent 的空间归纳偏置”，这篇是强相关方法来源而非撞车。

### 跟我们的关系
- 对 `0616-vln`，最重要的启发是把“能预测下一步/下一视角/下一动作”和“有可迁移空间世界模型”拆开。
- VLN agent 在 R2R、RxR、REVERIE、HM3D 等任务上高成功率，仍可能只学到 instruction-action shortcut 或 legal-action affordance。
- 可直接借鉴 inductive bias probe：构造小数据集，让模型从少量导航轨迹标签迁移到新的空间函数。
- VLN 版 `phi(x)` 可定义为房间拓扑节点、agent pose、可见地标集合、metric distance-to-goal、shortest-path tree、或 object-room relation graph。
- R-IB 可测同一拓扑状态或同一语义位置下，模型对新导航属性的预测是否一致。
- D-IB 可测不同空间状态是否被模型区分，防止把“下一步可行动作相同”的位置混为一类。
- next-token partition 对 VLN 很关键：两个不同位置可能拥有相同可行动作集合，模型若只看 action affordance 会通过常规导航指标但缺乏地图理解。
- 与我们的课题不直接撞车，除非我们声称“VLN 轨迹预测自然涌现完整世界模型”。
- 该引：应该引用为世界模型评估和负面证据，尤其用于论证为什么要设计空间状态 probe。
- 可用方法：oracle calibration、R-IB/D-IB、`D-IB_q=` vs `D-IB_q!=`、symbolic/structured readout、synthetic task adaptation。
- 差异化方向：把 probe 从封闭棋盘/轨道扩展到 embodied navigation 的拓扑地图、语言指令 grounding、动态可见性和真实 simulator。
- 实验设计上，我们应同时报告 task success 和 state-transfer probe，避免只用 SPL/SR 证明世界模型。

## Read by: 0616-vln (2026-06-18)

从 VLN 角度看，这篇不是导航论文，但它非常适合作为“为什么仅靠轨迹/动作预测不能证明空间世界模型”的方法论引用。它最有用的地方是把模型能力分成两层：原任务预测能力和新任务适配时的归纳偏置。VLN 中也会出现类似现象：agent 能选择合法动作、跟随常见指令模板、在 benchmark 上拿到高 SR/SPL，但未必能在少量新标签下外推到拓扑距离、不可见目标方位、房间连接关系或指令中隐含的空间约束。

对 `0616-vln` 不构成直接撞车，因为本文没有做视觉语言导航、没有用真实/仿真室内环境，也没有提出 VLN policy。真正的撞车点只在 claim 层面：如果我们的论文声称“foundation model 经过 VLN 轨迹预测后自然形成世界模型”，这篇会要求我们给出比 success rate 更强的证据。建议把它作为评估框架引用，并把我们的差异化写成 embodied setting、language-conditioned spatial state、topological/metric map probe，以及 next-action affordance coarsening 的诊断。
