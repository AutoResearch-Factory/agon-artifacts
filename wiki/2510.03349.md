# AgentCaster: Reasoning-Guided Tornado Forecasting (2510.03349)

- **Year**: 2025.10
- **Authors**: Michael Chen (单作者, Caltech, Computing + Mathematical Sciences)
- **Venue**: arXiv preprint (NeurIPS 2025 preprint 模板; 附完整 NeurIPS checklist, 应为投稿版)
- **Keywords**: LLM agent benchmark, tornado forecasting, HRRR, multimodal spatiotemporal reasoning, contamination-free evaluation, GeoJSON risk polygon, IoU verification, hallucination metrics, practically perfect forecast (PPF), SPC outlook
- **One-liner**: 免污染 LLM-agent 龙卷预报基准：交互读 HRRR 图/探空产出风险多边形，SPC 专家 18.3% 远超最佳 LLM 8.5%，且 reasoning 越强分数越低。

## 精华提炼

### 管线/方法

完整 pipeline（输入 → 处理 → 输出 → 验证）：

1. **数据生成（每日）**：NOAA HRRRv4（3-km 对流可分辨模式，WRF-ARW 内核）00:00 UTC run 的原始 GRIB2 → 自建解析管线 → 渲染 145 种产品图（CAPE/CIN、风切变、螺旋度、湿度、温度、模拟雷达反射率等），预报时效 f12–f36 每小时一张 → 每天 3,625 张 CONUS 底图 PNG（统一投影 + 地理参考叠加）。
2. **探空按需生成**：HRRR BUFKIT 数据 → agent 给定 (lat, lon, forecast_hour) → Haversine 距离找最近站（每小时 1,605 站可选，全天理论池 40,125 张）→ 用修改版 SHARPpy 渲染标准 skew-T log-P 图（温度/露点廓线、风羽、热力学与动力学参数）。每日配额默认 50 次，强制 agent 做战略性资源分配。
3. **Agent 交互循环**：系统提示词设定 "AI meteorologist" 角色 + 评分规则全透明（把 ground truth 生成方法直接写进 prompt）→ agent 依次调用 4 个工具：`list_available_map_types` → `request_hrrr_map`（PNG base64 嵌入对话）→ `request_sounding`（含剩余配额提示）→ `submit_tornado_prediction`（一次性提交 GeoJSON FeatureCollection）。每轮附 token 用量提示。无人在环，禁止提问。
4. **预测格式**：SPC 风险等级 2%/5%/10%/15%/30%/45%/60% 的嵌套多边形（高风险必须空间包含于低风险内），MultiPolygon 表示不连通区域；预测目标为"24 小时内（12UTC–次日 12UTC）任一点 25 英里内出现龙卷的概率"。
5. **Ground truth 生成（改进版 PPF, Hitchens et al. 2013）**：SPC 龙卷报告（12UTC–12UTC 聚合）→ NCEP Grid 211（80-km Lambert Conformal）上做归一化 Gaussian KDE（σ≈120 km）→ 双线性插值到 ~5-km 网格 → 与 40-km 半径均匀圆盘核卷积 → 乘以格元面积得期望龙卷数 λ → Poisson 关系 P=1−e^(−λ) → 按 SPC 阈值离散化为风险带 → 栅格转矢量多边形 → 重投影 WGS84 存为逐日 ground truth。
6. **评测**：预测与 ground truth 多边形在投影坐标系下做逐风险带（disjoint bands）几何 IoU + 质心距离分析。

### 关键技术细节

- **TornadoBench（主指标）**：日分数 TB_d —— GT 全 0% 且预测全 0% → 1；GT 全 0% 但预测有风险 → 0；GT 有风险 → 对出现过的风险带集合 S_d 求平均 IoU（0% 带用补集 IoU）。总分 = Σ(TB_d·W_d)/ΣW_d，其中 W_d 为当日 GT 最高风险的数值（0%→1, 5%→5, 30%→30），即高风险日权重大。
- **TornadoHallucinationSimple**：安静日（GT=0%）误报任何 ≥2% 风险的天数占比。
- **TornadoHallucinationHard**：两类幻觉（安静日报风险 + 风险日预测与 GT 零重叠）按预测最高风险数值加权的平均惩罚。
- **质心指标**：Overall risk centroid（≥2% 风险区 unary_union 的质心）与 Max risk centroid（最高风险带质心），Lambert Conformal 投影下欧氏距离（km）。
- **数据集**：40 天连续窗口（2025-03-01 至 2025-04-09），522 条龙卷报告；日最高风险分布 = 0%×22 天 / 2%×6 / 5%×4 / 10%×3 / 15%×2 / 30%×3（无 45%/60% 日）。共 145,000 张预处理图 + 1,605,000 可生成探空 + SPC outlook 基线 + ground truth，约 244 GB，托管 HuggingFace `agentcaster/agentcaster`，代码 `github.com/agentcaster/agentcaster`，NOAA NODD 开放 license。
- **被测模型（12 个，知识截止均早于 2025-03-01 以保证免污染）**：GPT-5 家族四档 reasoning（minimal/low/medium/high）、gpt-4.1、o3、o4-mini-high、claude-3.7-sonnet（±thinking 32k）、gemini-2.5-pro/flash-thinking、grok-4。
- **人类基线**：SPC 首发 Day 1 Convective Outlook（12UTC 周期），与 agent 预测走完全相同的评测管线。
- **统计**：非参数 bootstrap 1000 次、percentile 法 ±2σ 置信区间（允许不对称）。API 总成本约 $500。

### 核心结果

| 模型 | TornadoBench | HalluSimple | HalluHard | 质心误差(avg, km) |
|------|-------------|-------------|-----------|------------------|
| SPC 人类专家 | **18.31%** | 0.275 | 0.70 | 182 |
| gpt-5-minimal | 8.51% | 0.385 | 2.56 | 358 |
| gpt-5-low | 7.23% | 0.444 | 1.92 | 417 |
| claude-3.7-sonnet | 6.79% | 0.400 | 3.30 | 405 |
| claude-3.7-sonnet:thinking | 6.64% | 0.359 | 3.10 | 474 |
| gpt-5-medium | 6.28% | 0.484 | 2.65 | 398 |
| gpt-4.1 | 5.63% | 0.444 | 3.64 | 361 |
| gemini-2.5-pro | 4.26% | 0.406 | 4.50 | 494 |
| grok-4 | 3.85% | 0.538 | **8.85** | 450 |
| gpt-5-high | 3.54% | 0.500 | 2.30 | 449 |
| o3 | 3.27% | 0.550 | 5.50 | 478 |
| gemini-2.5-flash:thinking | 1.57% | 0.625 | 4.50 | 601 |

- **Surprising #1（反直觉主发现）**：GPT-5 家族内 reasoning 越强 TornadoBench 单调下降（8.51 → 7.23 → 6.28 → 3.54）；claude-3.7 非 thinking 版也略胜 thinking 版（6.79 vs 6.64）。更多推理 ≠ 更好的空间-地理预报。
- **Surprising #2**：所有 LLM 系统性**过度预测风险强度**（Max Risk Over 占 59–90% 天数，grok-4 达 89.7%；SPC 为 40%），且幻觉严重（HalluHard 是 SPC 的 3–13 倍）。
- **Surprising #3**：探空工具严重利用不足——多数模型平均每天 <5 次请求（配额 50），gpt-5 系列几乎不用（0.05–0.4 次/天）；说明 agent 不会主动做垂直结构分析。
- 质心误差 LLM 普遍 400–600 km vs SPC 182 km——不是"差一点"而是"整体错位"。
- 有效输出也是瓶颈：gemini-2.5-flash:thinking 仅 16/40 天产出合法 GeoJSON，gpt-5-high 30/40，gpt-5-medium 31/40。
- 大 outbreak 日（2025-03-14, 30% 风险日, 104 报告）最佳 LLM 日分 9.45% ≈ SPC 9.51%——高信号日 LLM 能接近专家，拉开差距的是低信号日与安静日。

### 明确局限

作者自述：40 天窗口无法覆盖多年气候学多样性；探空配额 50/天是因现有模型长上下文一致性差而设的工程妥协；交互协议在真实感与可复现性间做了取舍；HRRRv4 是当前最先进对流可分辨模式（同时给分析场与预报场）。未来方向：nowcasting、气候尺度、更多 NWP 模式。

我读出的局限：
- **输入不对称**：SPC 预报员实际可用多模式集合、观测、卫星等全部信息，agent 只有 00Z HRRR 单一确定性 run——"人类 vs LLM"的差距部分来自信息差而非纯推理差；论文未讨论。
- **单季节偏倚**：3–4 月只代表春季 Dixie/Plains 龙卷季，冷季/热带型事件全缺。
- **指标设计可商榷**：0% 日的 0/1 二值打分极苛刻（一个错误 2% 多边形整天归零）；W_d 按风险数值加权是 ad hoc 选择；未并报气象界标准的 CSI/POD/FAR/reliability diagram，与气象验证文献接轨不足。
- **无改进侧实验**：纯评测，没有微调、RAG、集合策略、数值场直读（非 PNG）等干预实验来定位失败根因（视觉读图误差 vs 空间推理误差 vs 输出格式误差未解耦）。
- **单作者小成本项目**（$500 API），12 模型 × ≤40 天样本，风险日仅 18 天，bootstrap CI 很宽（SPC [10.2, 28.3] 与最佳 LLM [4.8, 12.6] 上界接近重叠）。
- 环境场信息以 PNG 图形式喂给模型——**数值时空场从未作为一等公民进入 agent 上下文**，恰是这类工作的通病。

### 跟我们的关系

对 0713-scs-benchmark（强对流环境–雷达多模态开放基准数据集）：

- **撞不撞车：不撞车，且是有力的"gap 印证"**。AgentCaster 是 *LLM-agent 评测协议*（评模型能力），不是 *面向训练/分析的数据产品*。它不含 MRMS 雷达、不含长时窗环境场（我们是事件前 15 天/后 5 天，它只有 f12–36 单 run）、事件标签只有龙卷报告（我们覆盖 Storm Events 全事件类型）、环境场只以 PNG 渲染图交付而非数值场。它反而证明了我们 topic 的核心判断——现有资源里"环境场及其时空演变"不是一等公民：连 2025 年最新的 agent 基准也只是把环境场画成图片喂模型。
- **可复用工具/配方**（代码开源）：(1) HRRR GRIB2 → 145 产品可视化管线；(2) BUFKIT + 修改版 SHARPpy 的按需探空渲染；(3) **PPF ground truth 配方**（KDE σ120km → 5km 插值 → 40km 盘卷积 → Poisson）——我们的事件标签概率场谐一化可直接借鉴，这是把离散 Storm Events 报告变成连续验证目标的成熟方法；(4) SPC 风险带嵌套多边形 + 逐带 IoU 的评测代码；(5) 244GB 级 NOAA 衍生数据在 HF 托管 + NOAA NODD license 的先例（license 风险的直接参照）。
- **该不该引：必引**。引用位置：相关工作中"现有基准按用途割裂/不以环境场为一等公民"的最新例证；以及"下游应用"论证——我们的数据集可为 AgentCaster 式 agent 评测提供更丰富的输入模态（数值环境场 + 雷达 + 多事件标签），它的失败模式（空间错位、过度预报）正是需要更好环境场表征的证据。

## Read by: 0713-scs-benchmark (2026-07-14)

**撞车评估**：无撞车。AgentCaster 与本 topic 的交付物形态、数据模态、用途三个维度全部错开：它评 LLM（模型为被试），我们建数据底座（数据为产品）；它单模态渲染图 + 探空图，我们数值环境场 + MRMS 雷达 + 全类型事件标签；它 40 天单 run 短窗，我们事件为中心的长时窗（−15d/+5d）。最重要的是它把环境场降维成 PNG 的做法反向验证了本 topic 的立项 gap。

**可用工具**：PPF 概率场配方（KDE→盘卷积→Poisson，可直接用于 Storm Events 标签谐一化为连续验证场）；HRRR GRIB2 解析渲染管线；SHARPpy 探空生成；逐风险带 IoU + 质心距离评测代码；HF 大体量托管 + NOAA NODD license 先例。

**差异化/机会**：(1) 我们的 canonical tasks 可包含一个 AgentCaster 式 outlook 验证任务，但输入换成数值场 + 雷达序列，形成"同任务、更富模态"的升级；(2) 它揭示的 LLM 失败模式（地理错位 400–600 km、过度预报、探空利用不足）可作为我们示范分析章节的动机证据——环境场时空演变的显式表征正是当前 agent 缺的输入；(3) 它没做的 CSI/POD/FAR 标准验证与多年窗口，是我们基准设计时应补上的点。
