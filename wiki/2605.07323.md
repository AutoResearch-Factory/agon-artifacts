# Discovering Ordinary Differential Equations with LLM-Based Qualitative and Quantitative Evaluation (2605.07323)

- **Year**: 2026.05
- **Authors**: Sumie Song, Bong Gyun Shin, Jae Yong Lee (Chung-Ang University, Daejin University)
- **Venue**: ICML 2026
- **Keywords**: Symbolic Regression, Ordinary Differential Equations, Large Language Models, Scientific Discovery, Multi-Agent System, Qualitative Evaluation, Quantitative Evaluation
- **One-liner**: DoLQ 引入多智能体架构，用 LLM 进行物理语义定性评估并融合定量消融分析，在 ODE 发现上达到 7/8 成功率（NMSE test），显著优于纯定量方法

## 精华提炼

### 管线/方法

DoLQ (Discover ODEs with LLM-based Qualitative and Quantitative evaluation) 是一个三智能体迭代管线：

1. **Sampler Agent** (LLM, Gemini 2.5 Flash Lite, temperature=0.9): 每轮生成 3 个候选方程假设。每个假设包含每个维度的若干符号项（如 "np.sin(x1)"、"x0*x0"），每个项附带自然语言物理合理性解释。输入包括：(a) 系统自然语言描述 T（不含公式），(b) Scientist Agent 的累积知识和逐项评估反馈，(c) 已移除项黑名单。输出格式为结构化 JSON。

2. **Parameter Optimizer** (数值优化): 将符号项转为可执行 Python 函数（每个项分配一个可学习参数 params[i]），采用三种策略并选择最优：(a) BFGS 单独优化，(b) 差分进化 (DE) 单独优化，(c) 混合策略（先 DE 全局搜索后 BFGS 局部精化）。优化目标为残差 MSE（而非积分 MSE），避免多维系统积分发散问题。超参数：DE 种群大小 20，DE tolerance 1e-5，BFGS tolerance 1e-9，最多 8 参数每项。

3. **Scientist Agent** (LLM, Gemini 2.5 Flash Lite, temperature=0.6): 双轨评估——(a) **定性评估**：LLM 判断每项与系统描述的语义一致程度（good/neutral/bad）；(b) **定量评估**：简单消融测试——暂将该项系数置零，计算残余 MSE 变化率 delta = (MSE_ablated - MSE_base)/MSE_base，>5% 为 good（正面贡献），<-5% 为 bad（负面），其余为 neutral。最终决策矩阵：如果语义 bad 则直接 remove；如果语义 good 且定量 good 则 keep；其他情况 hold，连续两次 hold 转为 remove。移除项进入黑名单（但以 0.01 概率随机复活，防止永久丢失）。

### 关键技术细节

- **LLM 模型**: Gemini 2.5 Flash Lite（Sampler 和 Scientist 均使用），对比了 GPT-4o mini、DeepSeek-V3.2、Grok 4.1 Fast
- **搜索预算**: 100 轮迭代（=100 次 LLM 调用）
- **候选生成**: 每轮 3 个候选假设，每项最多 10 个符号项
- **无显式稀疏正则化**: 稀疏性通过项上限 + Scientist Agent 反馈 + 消融过滤控制
- **系统描述 T 设计**: 只含物理概念的自然语言描述，移除方程名、显式符号形式和直接函数线索，防止 prompt 泄漏 ground truth
- **数值微分**: 用 NumPy gradient 二阶边缘处理计算时间导数
- **数据生成**: Runge-Kutta 4 阶方法，1000 均匀采样点，无噪声（sigma=0）
- **评估指标**: 残差 NMSE（逐点导数匹配）和积分 NMSE（轨迹积分预测），ID 和 ID-Ext 两个时间域
- **基准测试**: 从 ODEbench 选取 8 个 ODE 系统，涵盖多项式/有理/三角/指数函数形式，2D 和 4D

### 核心结果

| 评估指标 | DoLQ (ours) | ICSR | LASR | LLM-SR | EDL |
|----------|------------|------|------|--------|-----|
| NMSE test (8 题通过数) | **7** | 2 | 2 | 4 | 0 |
| Term test (8 题通过数) | **5** | 0 | 2 | 2 | 0 |
| CDIMA 积分 NMSE (ID) | **2.35e-8** | 3.78e-4 | 2.90e-1 | 3.14e-3 | 1.33e5 |
| Glider(4D) 积分 NMSE (ID) | **8.29e-7** | 2.55e-1 | 4.94e-1 | 8.54e-7 | 1.37e4 |
| SIR 积分 NMSE (ID) | 1.55e-8 | 1.42e-8 | 7.60e-9 | 1.61e-8 | 1.40e1 |

**关键发现**:
- CDIMA(2D) 场景突出体现了纯定量方法的陷阱：ICSR、LASR、LLM-SR 残差 NMSE 看似很低但积分 NMSE 很高，表明未捕捉到真正的物理动力学；只有 DoLQ 在两个指标上都保持低误差
- 定性评估加速收敛：在 Glider(2D) 上，含 Scientist Agent 的完整 DoLQ 在第 27 轮发现正确方程，不含 Scientist 的消融版本在第 62 轮才发现
- Token 效率显著优于 LLM-SR：在 Glider(4D) 上，DoLQ 总 token 约 784K+264K，LLM-SR 高达 1852K+1997K
- 混合优化器（DE+BFGS）在复杂系统（如 Glider）上被更频繁选用，因为 BFGS 单独在崎岖 loss landscape 上常失败

**不同 LLM 骨干对比**（Glider 2D）：
- Gemini 2.5 Flash-Lite: 积分 NMSE ~10^-8（成功恢复）
- DeepSeek-V3.2: 积分 NMSE ~10^-8（成功恢复，轻微额外项）
- Grok 4.1 Fast: 积分 NMSE ~10^-8（成功恢复，轻微额外项）
- GPT-4o mini: 积分 NMSE 高达 ~10^-3 到 ~10^-1（定性结构错误）

### 明确局限

1. **噪声敏感**: DoLQ 不能内在过滤观测噪声。sigma=0.001 时 ID NMSE 从 ~10^-8 退化到 0.1086；sigma=0.1 时达 32.58。作者明确承认 "its absolute accuracy deteriorates as sigma increases"。
2. **参数优化依赖数值微分**: 用有限差分估计导数，对测量噪声敏感。
3. **推理固定 (reasoning fixation)**: LLM 偶尔陷入过早的机制性 storytelling，对探索性项赋予自信的物理叙述但缺乏充分数值证据，导致过早收敛到不合理假设。
4. **依赖系统描述质量**: 当描述被修改/误导时，DoLQ 缺乏纠正错误先验知识的内在机制。完全错误描述下残差 NMSE 从 ~10^-8 升至 ~10^-2。
5. **未扩展到 PDE**: 作者指出 "extension to PDEs presents considerable difficulty"。
6. **无平均统计**: 只报告 best of 3 runs，消融曲线只展示单次代表性轨迹而非多次平均。
7. **消融简化**: 在消融测试中不重新优化剩余系数（设零后直接评估），虽经 pilot 验证对决策影响极小但理论上不精确。
8. **API 依赖**: 需要 Gemini（或等价商业 LLM）API 访问，非完全离线/开源。

### 跟我们的关系

对 0616-fex（RL for Symbolic Regression）课题：

1. **问题空间重叠**：DoLQ 做的是 SR 中 ODE 发现，与 FEX 课题共享 "从数据中发现符号方程" 这一核心任务。DoLQ 的 benchmark（ODEbench 8 系统）和定量/定性双评估范式值得借鉴。
2. **方法差异**：DoLQ 用 LLM 多智能体（非 RL），我们更关注 RL-based SR。但 DoLQ 的定性评估机制（Scientist Agent 用 LLM 判断物理合理性）可被 FEX 借鉴为 reward shaping 或 action masking 的语义信号。
3. **不直接撞车**：DoLQ 不涉及 RL，FEX 的核心竞争壁垒（RL policy 学符号搜索策略）不受威胁。但 DoLQ 的定性评估 baseline 是一个有力的比较对象——如果 FEX 只在定量指标上做文章，无法说服审稿人为何不需要 LLM。
4. **可用的工具/方法**：
   - Scientist Agent 的 ablation-based 定量评估机制可直接复用
   - Dual evaluation 决策矩阵（keep/hold/remove）是优雅的候选筛选策略
   - CDIMA 场景中 "低残差 NMSE 但高积分 NMSE" 的陷阱是 FEX 需要特别防范的
   - Hybrid optimizer (DE+BFGS) 可考虑作为 FEX 的参数优化子模块
5. **差异化方向**：FEX 可以主打 "无需 LLM 知识注入的纯数据驱动 RL 方法"，与 DoLQ 的 LLM-centric 路线形成对比，论证 RL 能学到更通用的搜索策略而不依赖先验语义。
6. **需要关注的局限**：DoLQ 的噪声敏感问题，FEX 如果用 RL 需要证明噪声鲁棒性更好。
