# An Alternate Policy Gradient Estimator for Softmax Policies (2112.11622)

- **Year**: 2022.03 (AISTATS 2022)
- **Authors**: Shivam Garg, Samuele Tosatto, Yangchen Pan, Martha White, A. Rupam Mahmood (University of Alberta / Huawei Noah's Ark Lab)
- **Venue**: AISTATS 2022
- **Keywords**: policy gradient, softmax saturation, exploration, biased critic, non-stationary RL, gradient bandits, actor-critic
- **One-liner**: 提出 alternate PG estimator，利用 reward noise 和 critic bias 帮助 softmax 策略逃离子最优饱和区域，在非平稳环境中显著优于 regular estimator。

## 精华提炼

### 管线/方法

整体方法 pipeline：输入状态 S → 软最大化动作偏好 θ(s) 得到策略 π(a|s) → 采样动作 A → 执行获得奖励 R → 用 alternate estimator 计算梯度 → 更新动作偏好 → 更新 critic baseline。

**Regular Estimator（标准 likelihood ratio）**:
- 梯度形式: `ĝ^REG = (R - b) · (e_A - π)`，其中 e_A 是采样动作的 one-hot 向量，π 是策略概率向量
- 同时更新所有动作偏好，复杂度 O(|A|)
- 饱和时 (π ≈ e_A)：梯度期望 → 0，方差 → 0，策略卡死

**Alternate Estimator（本文提出）**:
- 梯度形式: `ĝ^ALT = (R - b) · e_A`，仅更新被采样动作的偏好
- 只更新单个动作偏好，配合 tree sampling 复杂度 O(log|A|)
- 饱和时梯度期望 → 0，但方差 → σ²(c) ≠ 0，允许随机游走逃离
- 等价于去掉 softmax 归一化常数的梯度: ∇_w log(e^[θ]_A / 1) = ∇_w [θ]_A

**扩展到 MDP**：将 ∇_w log π(A|S) 替换为 ∇_w [θ(S)]_A，适配 REINFORCE、Online AC、TRPO、PPO。

### 关键技术细节

- **模型架构**: 
  - Bandit: 纯 softmax 策略，动作偏好 θ_a
  - Tabular MDP: 每状态独立的动作偏好表，5 状态 chain 环境
  - Linear FA: tile-coding（4 tiles/dim × 8 tilings + bias），状态特征维度 d，线性权重 W ∈ R^{d×|A|}，critic 同理
  - Neural: 2×10×10×9 策略网络 + 2×10×10×1 critic 网络，ReLU 激活，Adam 优化器

- **训练配置**:
  - Bandit: 1000 timesteps，150 runs，α ∈ {2^{-6},...,2^1}，β ∈ {2^{-4},...,2^0}
  - Tabular REINFORCE: 100 episodes，γ=0.9，episode cutoff=100
  - Linear AC (MountainCar): 200k steps，γ=1，cutoff=1000，α ∈ {2^{-13},...,2^5}，β ∈ {0.01,...,2}
  - Linear AC (Acrobot): 400k steps，其余同上
  - Neural AC (DotReacher): 100k steps，Adam default config

- **核心理论结果**:
  - Lemma 2.1: biased update 的固定点分类 —— (1) b 介于 reward 之间时无固定点 (2) b = r(a) 时 simplex 面上任意点 (3) b < r_1 或 b > r_k 时有唯一内点 π*
  - Theorem 1: 悲观 baseline (b < r₁) → π* 为 repellor；乐观 baseline (b > r_k) → π* 为 attractor，KL 散度单调递减
  - Proposition: 策略饱和时 regular estimator 方差为零，alternate estimator 方差非零

- **Tree Sampling**: 平衡二叉树数据结构，每个节点存 e^{θ_a} + 左子树累积和，O(log|A|) 采样 + O(log|A|) 更新

### 核心结果

| 实验 | Setting | 关键发现 |
|------|---------|---------|
| Exp 1 | 3-arm bandit, ε~N(0,1) | Alternate 在饱和初始化下学会最优策略，Regular 完全失败 |
| Exp 2 | 同上，ε=0 | 无 reward noise 时所有方法都无法逃离饱和 |
| Exp 3 | 乐观/悲观 baseline | 乐观 b=+4 大幅帮助 alternate，悲观 b=-4 严重损害 |
| Exp 4 | Tabular Chain MDP, REINFORCE | Alternate+baseline 在饱和策略下远优于所有方法 |
| Exp 5 | 乐观/悲观 value init | 乐观 b=+4 显著帮助 alternate；悲观 b=-4 损害性能 |
| Exp 6 | MountainCar/Acrobot 非平稳 | Alternate 能恢复而 Regular 无法恢复（MountainCar best config） |
| Exp 7 | RealisticMountainCar | Alternate 解决任务，Regular 几乎完全失败（~ -700 vs ~ -200） |
| Exp 8 | DotReacher 神经网络 | 趋势一致但差异减小 |
| Exp 9 | 对比 entropy reg 和 escort transform | Alternate 整体优于 entropy+regular；Escort 在部分设置更优但对 p 敏感 |

**反直觉结果**:
- Alternate estimator 在极大 stepsize (α=2^{16}=65536) 下仍保持良好性能，而 regular estimator 崩溃 —— 因为 alternate 只更新单动作偏好，泛化性低，不易全局破坏策略
- 梯度噪声注入在 bandit 中有效但在 MDP 中对 regular estimator 几乎无效 —— 无偏随机噪声缺乏方向性信号，不足以逃离 MDP 级别的饱和
- 乐观 baseline 并非越乐观越好：RealisticMountainCar 上 b=+500 反而比 b=0 差，因为 bias 越大、消除 bias 需要的时间越长

### 明确局限

1. **仅限离散动作空间**（作者明确声明）：方法天然不适用于连续控制
2. **仅限 log-likelihood 族 PG 估计器**：不适用于确定性策略梯度等其他 PG 家族
3. **理论分析仅限 bandit 设置**：MDP 下的理论分析缺失，且 bandit 分析是渐近的
4. **未测试 batch PG 算法**（PPO/A2C）：作者承认这很重要但留作 future work；batch 算法有太多交互部件使分析困难
5. **未在真正大规模任务上测试**：最大实验是 Acrobot/MountainCar/DotReacher，与 Atari/MuJoCo 等标准 benchmark 有差距
6. **Uniform policy 并非总是好的**：Modified chain 实验显示当均匀策略不利于探索时 alternate 性能差于 regular
7. **依赖乐观 critic 初始化**：方法需要调优 baseline 初始化值，增加了超参数调优负担
8. **未讨论连续控制中的类比方案**：虽然作者声明仅限离散，但没讨论是否可以通过 Gumbel-softmax 等技巧扩展到连续动作

### 跟我们的关系

对于 **0616-fex** 课题：

**撞车评估**: 不直接撞车。Alternate estimator 是一种 PG 估计器层面的改进，而 fex 关注的是功能/特征探索（function/feature exploration）。但在 "如何帮助 RL agent 更好地探索" 这个大目标上存在交叉。

**可用工具/方法**:
1. **Alternate estimator** 本身可以嵌入到任何离散动作 PG 算法中（REINFORCE, AC, TRPO, PPO），作为标准 PG 估计器的替代品
2. **Optimistic critic initialization** 作为探索机制：暂时保持 critic 的乐观偏差可以鼓励策略探索，渐进收敛时偏差消失 —— 这个思想可能适用于 fex 的探索策略
3. **Tree sampling 数据结构**: O(log|A|) 的采样和更新对大规模离散动作空间的探索有价值
4. **策略熵作为诊断 metric**: 作者用熵跳变来解释 alternate 在非平稳时的恢复能力，这个方法可用于 fex 的诊断
5. **非平稳实验范式**: 切换动作映射来引入非平稳性，这种实验设计可复用于测试 fex 的适应性

**差异化**: Alternate estimator 解决的是 PG 估计器的数值问题（饱和导致零梯度），不涉及特征表示学习、隐空间探索或好奇心驱动探索 —— 这些都是 fex 可能的核心方向。Alternate estimator 可以作为底层优化器与 fex 的高层探索机制正交组合。

## Read by: 0616-fex (2026-06-22)

### 撞车评估

不直接撞车。Alternate estimator 是策略梯度估计器层面的改进，解决的是 softmax 策略在饱和时梯度消失的数值问题。而 0616-fex 关注功能/特征层面的探索机制。两者在 "增强 RL 探索" 这一高层目标上有交集，但技术路径正交 —— alternate estimator 可以嵌入任何离散动作 PG 算法作为底层优化器，与 fex 的探索机制组合使用。

### 可迁移工具

1. **Optimistic critic 初始化作为探索策略**: 这是本文最有启发的思想。在 fex 中，如果涉及 value-based 或 actor-critic 架构，可以有意识地让 critic 短暂保持乐观偏差来自动增强探索，然后让偏差随学习渐进消失。这比显式加熵奖励更优雅。

2. **策略熵作为非平稳检测信号**: 本文用策略熵的跳变来验证 alternate estimator 在非平稳时的恢复机制。fex 可以用类似方法检测 agent 是否遇到了分布偏移、需要重新探索。

3. **动作偏好空间的随机游走**: Alternate estimator 在饱和时本质是做随机游走。如果 fex 涉及在隐空间或特征空间中探索，可以借鉴这种 "在无梯度信号时依靠随机性逃离" 的机制。

4. **Tree sampling O(log|A|)**: 如果 fex 涉及大规模离散动作或离散特征选择，tree sampling 数据结构可以直接用于高效采样和更新。

### 该引不该引

- 如果 fex 涉及 PG-based 探索增强 → **该引**，作为相关工作的 baseline 比较
- 如果 fex 涉及非平稳环境适应 → **该引**，作为对比方法
- 如果 fex 纯粹是特征表示学习且不用 PG → 相关性弱，不必引用

### 差异化方向

Alternate estimator 是被动的"逃离饱和"机制，不主动寻求信息增益。fex 可以在这方面做差异化：主动探索未知特征区域、使用信息增益/好奇心作为探索驱动、在特征空间而非动作偏好空间做探索。Alternate estimator + fex 的组合：底层用 alternate 增强鲁棒性，上层用 fex 指导探索方向。
