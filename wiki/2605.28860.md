# Mechanistic origins of catastrophic forgetting: why RL preserves circuits better than SFT? (2605.28860)

- **Year**: 2026.05
- **Authors**: Jeanmely Rojas Nunez (Algoverse AI Research), Viraj Sawant, Nathan Allen, Nomgondalai Amgalanbaatar, Yannis Zongo, Vasu Sharma, Maheep Chaudhary (Independent)
- **Venue**: ICML 2026 (submission, [accepted] option in style file)
- **Keywords**: mechanistic interpretability, catastrophic forgetting, reinforcement learning, supervised fine-tuning, circuit analysis, differential binary masking, Dr.GRPO, circuit preservation
- **One-liner**: 通过 Differential Binary Masking 在注意力头层面分析 RL vs SFT 对基础模型内部电路的影响，发现 RL (Dr.GRPO) 比 SFT 保留更多基础电路（72.5% vs 59.0%），且 RL 的电路保留在训练中可恢复，而 SFT 单调衰减。

## 精华提炼

### 管线/方法

整体三阶段管线：

**Phase I：复现分布偏移效应**
- 基础模型：Qwen2.5-3B-Instruct
- Task A（微调任务）：SciKnowEval 科学问答，用 New Task Score (NTS) 衡量——模型给正确答案分配最高 geometric-mean token probability 的题目比例
- Task B（保留评估）：HellaSwag, TruthfulQA, MMLU, IFEval, WinoGrande, HumanEval（常识推理、事实性、指令遵循、代码生成）
- SFT：completion-only cross-entropy loss，直接微调 base model
- RL：从 SFT checkpoint 出发，用 Dr.GRPO 继续训练。Dr.GRPO 基于 DeepSeekMath 的 GRPO：group size=64，μ=2 refinement steps，无显式 KL 惩罚，用组内相对归一化估计 advantage
- 关键设计选择：RL 从 SFT 模型出发而非从 base 直接 RL，因此比较的是 "SFT 后加 RL" vs "纯 SFT"
- 量化漂移：对 retention tasks 计算期望 KL divergence π_base || π_θ

**Phase II：通过 Differential Binary Masking (DBM) 识别电路**
- 分析粒度为 attention head 级别
- DBM 学习 mask m_h ∈ [0,1]，在 base activation a_h^base 和 source activation a_h^source 间插值：ã_h = (1-m_h) a_h^base + m_h a_h^source
- 通过退火将 mask 推向二值化，得到稀疏因果电路
- 三元组构造（chemistry QA）：answer-key swap、molecule swap、task-type swap 三种反事实假设
- 目标函数：L_DBM = -log P(y_target|x,ã) + λ Σ m_h（最大化目标答案概率 + 稀疏正则）
- 评分：geometric mean of token probabilities p(y|x) = exp((1/T) Σ log P(y_i|x,y_<i))
- 对 π_base, π_SFT, π_RL 独立运行 circuit discovery

**Phase III：跨模型电路比较**
- Circuit Faithfulness: Faithfulness(C,M) = F(C|M) / F(M)，接近 1 表示子电路恢复了模型全部行为
- Head-level mask shift: Δm_h(M) = m_h^M - m_h^base，识别被保留/放大/削弱的 head
- Vulnerable heads 定义：{h: m_h^SFT < m_h^RL - δ}（SFT 下比 RL 下退化更多的 head）
- Necessity/Sufficiency 双维度分析：necessity 衡量移除 head 后 log-prob 下降量，sufficiency 衡量仅保留该 head 时恢复行为的程度

### 关键技术细节

- **模型**：Qwen2.5-3B-Instruct，36 层 transformer，共 576 个 attention heads（36层 × 16 heads/layer）
- **RL 算法**：Dr.GRPO（Domain-specific Group Relative Policy Optimization），group size=64，μ=2 refinement steps，无 KL penalty，binary reward signals
- **电路发现**：DBM 在 attention head 级别学习 mask，退火至二值化
- **Base circuit**：297 heads（占全部 attention heads 的 51.6%）
- **SFT circuit**：~265 heads（46.0%），base overlap 52%
- **RL circuit**：~296 heads（51.4%），base overlap 68%
- **Circuit faithfulness**：Base 1.02, SFT 1.04, RL 1.12（均 >1.0，提取的子电路可靠）
- **Necessity vs Δm_h 相关性**：SFT r=-0.125 (p=0.12, 弱负相关), RL r=0.022（几乎为零）
- **训练数据**：SciKnowEval 科学问答数据集
- **评估指标**：NTS (New Task Score)，KL divergence，circuit faithfulness，Δm_h，necessity/sufficiency

### 核心结果

| 指标 | SFT (Epoch 1) | SFT (Epoch 2) | RL (Epoch 1) | RL (Epoch 2) |
|------|--------------|--------------|-------------|-------------|
| Base circuit retention | 63.5% | 59.0% | 69.8% | 72.5% |
| DCM score | 10.4 | 6.3 | 15.8 | 10.6 |

关键发现：
1. **RL 的电路保留在训练中恢复**：RL Epoch 1→2 从 69.8% 升至 72.5%，而 SFT 从 63.5% 降至 59.0%。这是 surprising 的发现——RL 训练可以恢复之前被破坏的电路组件，而非单调退化。
2. **SFT 是"压缩器"，RL 是"分布式适配器"**：SFT 将新任务行为集中到少数高 necessity+高 sufficiency 的 specialist heads；RL 在更多 head 间分散适配，不产生任何单一不可替代的 head。
3. **输出空间 KL 不能预测内部遗忘**：RL 模型的输出空间 KL divergence 比 SFT 更大，但电路保留和先前任务保留更好。这说明表面行为和内部表征转移可以分离——RL 改变了输出分布但保留了底层计算，SFT 表面漂移小但内部重组更激进。
4. **两种目标都不追踪 base model 的 head 重要性**：SFT 和 RL 的 Δm_h 与 base necessity 之间均无正相关，说明两者都不"聪明地"优先修改最重要的 head；SFT 广泛重组并提拔新的 specialist，RL 均匀调整。
5. **高低 NTS 区域的行为分离**：低 NTS 时两者电路保留相当；高 NTS 时 SFT 出现"断裂点"——从保守适配转向激进电路重构，RL 保持稳定。
6. **Layer-wise 分析**：RL 在大多数层 retained heads > forgotten heads；SFT 的 forgotten heads 遍布所有层，中后层尤其严重。

### 明确局限

1. **单一模型**：仅在 Qwen2.5-3B-Instruct 上测试，未验证 Gemma/Mistral/Llama/Pythia 等架构，也未测试不同参数规模
2. **仅分析 attention heads**：未扩展到 MLP 层、residual stream features，电路分析不完整
3. **任务狭窄**：仅科学问答（SciKnowEval），未覆盖多语言推理、事实回忆、安全、工具使用等
4. **无真机/无真实部署**：纯离线 benchmark 评估，无用户研究或生产环境验证
5. **RL 算法单一**：仅 Dr.GRPO，未比较 PPO/DPO/其他 RL 变体
6. **RL 从 SFT 出发**：实验设计中 RL 总是从 SFT checkpoint 开始，不能直接比较 "纯 RL from base" vs "纯 SFT from base"
7. **DBM 方法的局限性**：mask 退火到二值化的过程中可能丢失信息；仅分析 attention head 级别的 mask，不涉及 head 内部子结构
8. **因果方向未确定**：观察到电路保留与遗忘减少之间的关联，但不能严格证明因果关系——可能两者都是某个第三因素的共同结果
9. **NTS 的定义依赖特定评分方式**：geometric mean of token probabilities 可能不是所有任务的合适指标
