# Dynamic Bayesian Optimization Framework for Instruction Tuning in Partial Differential Equation Discovery (2601.00088)

- **Year**: 2026.01
- **Authors**: Junqi Qu, Yan Zhang, Shangqian Gao, Shibo Li (Florida State University)
- **Venue**: arXiv
- **Keywords**: dynamic instruction tuning, Bayesian optimization, PDE discovery, symbolic regression, LLM scientific discovery, prompt optimization, STRidge, equation discovery
- **One-liner**: 用 BO 动态选择提示策略来提升 LLM PDE 发现的恢复率。

## 精华提炼

### 管线/方法

1. 输入是一组 PDE 观测数据 $\mathcal{D}=\{(x_i,t_i,u_i)\}_{i=1}^N$，目标是恢复 $u_t=\mathcal{N}(u,u_x,u_{xx},\ldots)$ 的符号形式。
2. 作者把 LLM equation discovery 中的 prompt 选择定义为 sequential decision problem，而不是一次性 prompt engineering。
3. 框架名为 `NeuroSym-BO`，由三个 agent 组成：Symbolic Proposer、Numerical Evaluator、Prompt Optimizer。
4. Symbolic Proposer 是 LLM，负责根据当前 prompt 生成候选 PDE 结构。
5. Numerical Evaluator 解析候选 symbolic skeleton，并对自由系数做数值拟合。
6. Prompt Optimizer 是 Bayesian Optimization agent，基于 evaluator 的反馈选择下一轮 instruction strategy。
7. 每轮 prompt 形式为 $P_t=I_{task}\oplus \mathcal{H}_{top}\oplus I_{strategy}^{(k_t)}$。
8. $I_{task}$ 固定描述变量、算子库、输出格式等约束。
9. $\mathcal{H}_{top}$ 是历史 top-N 方程和 fitness score，相当于给 LLM 提供 in-context search memory。
10. $I_{strategy}^{(k_t)}$ 是可优化指令，来自 strategy bank。
11. strategy bank 由 GPT-4o 生成，规模 $K=100$。
12. 指令覆盖 exploration、parsimony、mutation、refinement 等搜索模式。
13. 典型 exploration 指令让模型忽略当前最优、提出新结构。
14. 典型 parsimony 指令让模型删掉贡献小的冗余项。
15. 典型 mutation 指令保留主结构但替换非线性 interaction term。
16. 典型 refinement 指令在结构看起来正确时微调函数形式。
17. BO 的离散变量是 strategy index $k\in\{1,\ldots,K\}$。
18. 黑盒目标 $f(k)$ 是采用该策略后得到的最佳方程 fitness。
19. surrogate 是 Gaussian Process，拟合历史 `(strategy, fitness)` 观测。
20. acquisition function 是 Expected Improvement (EI)。
21. 初始探索阶段 $t\le K_{init}$ 随机选择策略，论文实验中 $K_{init}=10$。
22. 后续每步选择 $k_{t+1}=\arg\max_k EI(k)$。
23. 每轮 LLM 生成 $M=5$ 个候选方程。
24. 每个候选方程用 STRidge 做 sparse coefficient fitting。
25. fitness 同时考虑误差和复杂度：$S(\hat{u})=(1-\lambda\cdot complexity(\hat{u}))/(1+NRMSE)$。
26. 实验中 parsimony penalty $\lambda=0.01$。
27. NRMSE 定义为 $\sqrt{MSE/Var(u)}$。
28. complexity 计数方程项数，用于惩罚过复杂表达式。
29. 每轮最佳候选会更新 history buffer $\mathcal{H}$ 和 BO observation set $\mathcal{O}$。
30. 这个闭环让系统能在“探索新结构”和“压缩/修正已有结构”之间切换。

### 关键技术细节

31. backbone LLM 使用 Llama-3.2-3B-Instruct；tex 脚注链接却写成 Meta-Llama-3-8B-Instruct，存在模型名不一致风险。
32. baseline 是 Fixed Prompt，使用同样 top-5 history，但 instruction 不随搜索进度变化。
33. NeuroSym-BO 和 baseline 都运行 300 iterations。
34. 每个 benchmark 跑 5 trials，报告 train/test $R^2$ 均值。
35. prompt optimizer 用 BoTorch 实现，surrogate 为 standard GP，acquisition 为 EI。
36. Appendix 给出的复杂度分析认为 exact GP 每步 fitting 为 $\mathcal{O}(t^3)$。
37. 总体复杂度写作 $\mathcal{O}(T^4+T\cdot M\cdot C_{LLM})$，实际主成本是 LLM inference。
38. 实验配置声称 $T=300,K=100,M=5$，单 A100 GPU 每轮 15-30 秒。
39. 论文说 LLM inference 占约 80% runtime，GP overhead 小于 1%。
40. PDE benchmark 共 5 个：Burgers、Fisher、Chafee-Infante、Divide、Allen-Cahn。
41. Burgers/Fisher/Chafee/Divide 来自 LLM4ED benchmark suite。
42. Allen-Cahn 是作者自生成 synthetic dataset。
43. Burgers 设置：$x\in[-8,8]$，$t\in[0,10]$，grid $256\times201$，目标形式 $u_t+uu_x=0.1u_{xx}$。
44. Fisher 设置：$x\in[-1,1]$，$t\in[0,1]$，grid $200\times100$，目标形式 $u_t=u_{xx}+u(1-u)$。
45. Chafee-Infante 设置：$x\in[0,3]$，$t\in[0,0.5]$，grid $301\times200$，目标形式 $u_t-u_{xx}=u-u^3$。
46. Divide 设置：$x\in[1,2]$，$t\in[0,1]$，grid $100\times251$，目标形式 $u_t=0.25u_{xx}-u_x/x$。
47. Allen-Cahn 设置：$u_t=0.1u_{xx}+5.0(u-u^3)$。
48. Allen-Cahn 生成域为 $x\in[-10,10]$，$t\in[0,10]$，grid $256\times201$。
49. Allen-Cahn 初值为 $\sin(2\pi x/L)+0.5\cos(4\pi x/L)+\epsilon$，$L=20$，$\epsilon\sim U(0,0.2)$。
50. Allen-Cahn 使用 periodic boundary conditions。
51. Allen-Cahn 空间导数用 FFT pseudo-spectral method。
52. Allen-Cahn 时间积分用 `scipy.integrate.solve_ivp` 的 RK45。
53. Allen-Cahn solver relative/absolute tolerances 都设为 $10^{-8}$。
54. Allen-Cahn 数据按时间轴 80%/20% split 为 train/test。
55. 图 2 报告五个 PDE 的 test $R^2$ trajectories，红线为 NeuroSym-BO，蓝虚线为 fixed prompt，阴影为 $\pm$SEM。

### 核心结果

56. Allen-Cahn：Fixed train/test $R^2$ 为 0.7968/0.7824，Ours 为 0.9107/0.8914。
57. Burgers：Fixed train/test $R^2$ 为 0.8102/0.8242，Ours 为 0.8699/0.8791。
58. Chafee：Fixed train/test $R^2$ 为 0.9894/0.9886，Ours 为 0.9951/0.9947。
59. Divide：Fixed train/test $R^2$ 为 0.9927/0.9922，Ours 为 0.9942/0.9941。
60. Fisher：Fixed train/test $R^2$ 为 0.9952/0.9953，Ours 为 0.9999/0.9999。
61. 最大绝对 test $R^2$ 提升在 Allen-Cahn：0.7824 到 0.8914，约 +0.109。
62. Burgers 的 test $R^2$ 提升约 +0.0549，也属于作者强调的 challenging case。
63. Fisher/Chafee/Divide 本身 baseline 已接近满分，提升较小但方向一致。
64. 作者的解释是 fixed prompt 在 50-100 iterations 内 plateau，而 NeuroSym-BO 通过策略切换出现 step-wise jumps。
65. 论文声称动态 instruction selection 带来 compounding benefits，并降低 trial variance。
66. 反直觉点：方法并没有训练 LLM 或学习新的 equation representation，只是在离散 prompt strategy 上用 BO 做闭环控制。
67. 另一个关键点：本文把“prompt optimization”的目标从找全局单一最佳 prompt 改成按搜索状态选择不同 prompt。
68. 这个 framing 比单纯“更好的 prompt”更像 adaptive search controller。

### 明确局限

69. 作者承认方法依赖 backbone LLM 的数学知识；如果 LLM 不知道某类算子，prompt optimization 不能补足。
70. 作者承认当前只评估 1D PDE，高维系统和 chaotic behavior 仍是 future work。
71. 未见真实物理实验数据或真机/实验室数据，所有 evaluation 都是标准/合成数值数据。
72. benchmark 数量只有 5 个，且 LLM4ED 相关任务可能与 LLM prompt 先验高度贴合。
73. 没有看到与 LLM-SR、LLM4ED、PySR、SINDy、NeSymReS 等强方法的直接 head-to-head 数值表，只和 fixed prompt baseline 比。
74. fixed prompt baseline 是否足够强不清楚；如果 baseline prompt 不经调优，提升可能部分来自 prompt bank 的人工/LLM 设计。
75. strategy bank 由 GPT-4o 生成，但复现需要完整 bank；tex 没有列出 100 条 strategy。
76. 代码、数据、随机种子、prompt 模板、解析失败处理、STRidge 细节未在 tex 中充分展开。
77. Llama-3.2-3B 与脚注 Meta-Llama-3-8B-Instruct 的不一致会影响 reproducibility。
78. 表格只报均值，没有置信区间、标准差或显著性检验；图中有 SEM 但 tex 没给具体数值。
79. fitness 里的 complexity penalty $\lambda=0.01$ 未见敏感性分析。
80. BO 在离散 strategy index 上建 GP，strategy 之间的距离/核如何编码未详细说明；若只按整数 index 建核，语义邻近性可能不合理。
81. 没有 ablation：没有去掉 history、没有随机 strategy、没有 bandit/EA prompt optimizer、没有不同 strategy bank size。
82. 没有分析失败案例，例如动态策略何时选错、LLM 何时生成不可解析方程。
83. 没有说明 parser 如何把 LLM 文本稳健转换成 symbolic skeleton。
84. Scientific claim 是 promising，但证据主要支持“比固定 prompt 好”，尚不足以证明比现有 equation discovery 方法整体更强。

97. 本文该引用，尤其在“adaptive prompt/instruction control for equation discovery”和“LLM PDE discovery”相关段落。

## Read by: 0616-fex (2026-06-16)

这篇论文对 `0616-fex` 的主要价值是把“LLM 生成过程中的指令选择”从经验 prompt engineering 转成有反馈的离散优化问题。它的可用部件很清楚：策略库、top-N 历史、数值 evaluator、BO/EI controller、parsimony-aware fitness。若我们的方向涉及 agentic science、自动实验搜索、公式发现或 prompt-controlled search，它是必须引用的近邻。

撞车程度取决于 `0616-fex` 的核心 claim。如果我们也声称“根据数值反馈动态优化 LLM 指令以发现 PDE/方程”，那会直接撞车；如果我们强调更强的独立验证、多任务泛化、高维/混沌系统、真实数据或非 PDE 科学任务，则可以保持差异化。本文证据强度中等：相对 fixed prompt 有稳定提升，但没有强 baseline、完整 ablation、开源策略库或独立复现。后续构思时应避免只复刻 BO 选 prompt，而要把 verification、benchmark breadth、ablation 和 reproducibility 做成主贡献。
