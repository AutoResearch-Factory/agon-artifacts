# From Equations to Insights: Unraveling Symbolic Structures in PDEs with LLMs (2503.09986)

- **Year**: 2025.03
- **Authors**: Rohan Bhatnagar, Ling Liang, Krish Patel, Haizhao Yang; University of Maryland, College Park; University of Tennessee, Knoxville
- **Venue**: arXiv
- **Keywords**: LLM-informed FEX, finite expression method, symbolic PDE solving, operator-set prediction, high-dimensional PDEs, postfix notation, LoRA fine-tuning, policy-gradient symbolic learning
- **One-liner**: 用 LLM 预测 PDE 解的算子集来加速 FEX。

## 精华提炼

### 管线/方法
- 论文的核心问题不是直接用 LLM 解 PDE，而是学习 PDE 数据到解表达式算子集合的映射。
- 输入对象是强形式 PDE `Du=f`、边界条件 `Bu=g`、PDE 类型、边界条件类型，以及 `f/g` 中出现的符号算子。
- 作者把目标设成：给定 `f` 和 `g` 的符号结构，预测解析解 `u` 里需要出现哪些 unary/binary operators。
- 数据生成从随机解析解 `u` 开始，而不是从随机 PDE 开始。
- 每个随机 `u` 由 binary computational tree 生成，树深度固定为 3。
- 树节点使用 unary operators 与 binary operators 组成表达式。
- unary operators 包括 `sin`, `cos`, `tan`, `exp`, `lg`, `ln`, `sqrt`, `abs`, powers 等。
- binary operators 包括 `+`, `-`, `*`, `/`。
- 每个 unary operator 还带两个标量参数 `alpha` 和 `beta`，用于形成 affine-adjusted nonlinear transform。
- 数据生成流程是：先指定 PDE 类型和边界条件类型，再随机生成 `u`，再由 `u` 反推 RHS `f` 和边界/初值数据 `g`。
- 实验覆盖两类 PDE：Poisson equation 和 Linear Conservation Law。
- 每类 PDE 生成 99,000 条样本，总数据规模 198,000。
- 每类 PDE 配三类边界条件：Cauchy、Dirichlet、Neumann。
- `u`, `f`, `g` 都被转换成 postfix notation，也就是 Reverse Polish Notation。
- 选择 postfix 的理由是消除括号并保留相对算子位置，更适合序列模型。
- 训练输入 prompt 极简：`Type:<PDE>|RHS:<ops>|BC_Type:<bc>|Solution:`。
- 训练目标是 `Solution:` 后面的算子 token 序列，而不是完整数值解。
- tokenizer 使用 SentencePiece，PAD token 被设置成 EOS token。
- prompt 最大长度为 256 tokens，target 最大长度为 64 tokens，拼接后最大长度为 300。
- 训练模型包括 BART、T5、LLaMA-3B、LLaMA-8B。
- 训练目标是 cross-entropy loss。
- 优化器是 AdamW，learning rate 为 `2e-4`，weight decay 为 `0.01`。
- 训练 12 epochs，batch size 为 32。
- 训练硬件是 8 张 NVIDIA A6000 GPU，每张 48 GB。
- 训练使用 BF16 mixed precision、gradient checkpointing 和 LoRA。
- LoRA rank 为 16，scaling factor 为 32，dropout 为 0.1。
- 推理阶段模型输出 postfix token sequence。
- 后处理先抽取 unique operators，再过滤拼写错误、malformed token 和无法识别的符号。
- 评价指标是 operator-set mismatch：把预测和真值算子集编码为固定字典上的 binary vector，计算 squared `l2` distance。
- 这个指标本质上数的是多预测或漏预测的算子数量。
- LLM 预测出的算子集随后喂给 FEX，作为动态 operator pool。
- 原始 uninformed FEX 使用固定 binary set `{+, -, *}`。
- 原始 uninformed FEX 使用固定 unary set `{0, 1, Id, ^2, ^3, ^4, exp, sin, cos}`。
- LLM-informed FEX 把固定全集改成逐样本预测的小算子集。
- FEX 仍然是强化学习式符号搜索：policy 采样表达式程序，执行后得到显式 surrogate `u_theta`。
- FEX 的物理损失沿用 least-square PDE residual 和 boundary mismatch：`||Du-f||^2 + lambda ||Bu-g||^2`。
- FEX reward 被写成 `R(theta)=(1+L(u_theta))^{-1}`。
- LLM 的实际作用是剪枝搜索空间，不承担 residual 检验和最终表达式搜索。
- 因此这篇工作最准确的定位是 LLM-guided operator selection for FEX。

### 关键技术细节
- 理论部分只对 Poisson equation with Dirichlet boundary condition 给出 proof-of-concept。
- Poisson 设定是 `-Delta u=f` in compact domain `Omega`，`u=g` on boundary。
- 假设 domain distance function `D(x)=dist(x,boundary Omega)` 有显式解析表达。
- 假设 `f` 是 Holder continuous，`g` 是 `C^2(overline Omega)` 函数在边界上的限制。
- 定理声称对任意 `delta in (0,1)`，存在近似函数 `u_tilde`，满足 `||u-u_tilde||_{L2(Omega)} <= delta`。
- `u_tilde` 只需要使用 `+`, `*`, square，以及 `D`, `f`, `g` 本身 evaluation 中已有的算子。
- 近似复杂度是 `poly(delta^{-1}) * (ops(f)+ops(g))`。
- 证明使用 Brownian motion / walk-on-spheres 表示 Poisson 解。
- 边界项用 Monte Carlo 采样 `g(X_tau)` 近似。
- 源项用离散 walk-on-spheres 路径和随机单位向量近似。
- 这个定理支持“解的算子可由数据算子诱导”的直觉，但只覆盖较窄的 Poisson setting。
- 论文还给 FEX policy-gradient 搜索写了随机投影策略梯度理论。
- policy 参数是 `phi in Phi`，`Phi` 是 nonempty closed convex set。
- 假设 `p_phi` support 不随 `phi` 变，且 score-function gradient 合法。
- 目标写作 `J(phi)=E_{theta~p_phi}[R(theta)]`，带约束 indicator。
- 算法是 stochastic projected policy gradient method。
- 每步采样 batch size `B` 个 `theta_i`，计算 reward 和 score gradient，做 stochastic gradient step，再投影回 `Phi`。
- 收敛定理假设 `J` 是 `L`-smooth，固定步长 `eta in (0,1/(2L))`，梯度估计方差有上界 `sigma^2`。
- 定理给出 stationarity distance 的上界，由 `sigma^2/B` 和 `Delta/T` 两项控制。
- 选择 `B=O(epsilon^{-2})`、`T=O(epsilon^{-2})` 时，总 sample complexity 是 `O(epsilon^{-4})`。
- 这个理论主要说明通用 projected policy gradient 的一阶 stationarity，不直接证明 LLM operator prediction 的泛化。

### 核心结果
- operator prediction 实验中，decoder-only LLaMA 系列明显优于 encoder-decoder 模型。
- 第 1 epoch 的 average mismatch：LLaMA-8B 为 0.82697，LLaMA-3B 为 0.80510，T5 为 1.71635，BART 为 8.52542。
- 第 12 epoch 的 average mismatch：LLaMA-8B 为 0.38394，LLaMA-3B 为 0.40697，T5 为 1.29627，BART 为 1.64541。
- 最终 LLaMA-8B 最好，LLaMA-3B 接近，T5/BART 明显落后。
- LLM-informed FEX 实验使用 fine-tuned LLaMA-8B。
- FEX 下游实验共 100 个随机 PDE：50 个 Poisson，50 个 Linear Conservation Law。
- 每个实例运行 5 次，报告平均迭代数、平均时间、误差。
- Linear Conservation Law 上，uninformed FEX 平均迭代 28.52，LLM-informed FEX 平均迭代 5.23。
- Linear Conservation Law 的迭代 speedup 是 `5.45x`。
- Linear Conservation Law 上，uninformed FEX 平均时间 23.67 min，LLM-informed FEX 平均时间 3.95 min。
- Linear Conservation Law 的时间 speedup 是 `6.00x`。
- Poisson 上，uninformed FEX 平均迭代 28.73，LLM-informed FEX 平均迭代 6.47。
- Poisson 的迭代 speedup 是 `4.44x`。
- Poisson 上，uninformed FEX 平均时间 62.07 min，LLM-informed FEX 平均时间 14.09 min。
- Poisson 的时间 speedup 是 `4.41x`。
- 作者声称精度保持 competitive，并且 boxplot 显示稳定性更好。
- appendix 的逐例表说明不是所有例子 LLM-informed 都更准，有些例子 LLM version error 更大。
- 例如某些 `exp/sin/cos` 组合上，LLM-informed 的误差高于 uninformed，但速度仍通常更快。
- 论文最强的实证 claim 是“算子集预测可以显著降低 FEX 搜索迭代和时间”。
- 论文较弱的实证 claim 是“精度也提升”，因为整体表没有给出平均误差汇总和统计检验。

### 明确局限
- 数据是合成的，而且是先生成真解 `u` 再反推 `f/g`，这和真实建模中从物理方程出发寻找解的分布不同。
- 训练和测试都来自同一套表达式 grammar，泛化到新算子、新树深、更复杂 PDE 还没有证明。
- 只实验了 Poisson 和 Linear Conservation Law，没有展示 nonlinear Schrodinger、Navier-Stokes、reaction-diffusion 等真实复杂 PDE。
- 边界条件只覆盖 Cauchy、Dirichlet、Neumann，缺少 Robin、mixed、复杂几何上的系统评测。
- 理论只覆盖 Dirichlet Poisson，并依赖显式 distance function 和解析解存在假设。
- 理论证明的是存在 operator-preserving approximation，不是 LLM 学到该关系的泛化保证。
- policy-gradient 收敛定理是通用一阶 stationarity 结果，没有解决 FEX 离散表达式搜索的全局最优问题。
- 论文没有看到 LLM 训练代码、数据生成器、数据集或 checkpoint 的公开链接。
- 只给了原始 FEX 仓库链接，不等于本文 LLM-informed FEX 可复现。
- 缺少 train/validation/test split 的具体规模说明。
- 缺少 operator-set size reduction 的定量表，例如平均保留多少 unary/binary operator。
- 缺少 LLM 错误类型分析：漏算子、多余算子、非法 token 分别多少。
- 缺少 uncertainty 或 fallback 机制：一旦 LLM 漏掉必要 operator，FEX 搜索空间会被错误剪枝。
- 下游只比较 uninformed FEX，没有比较其他 symbolic regression 系统或 neural PDE solver 作为最终解质量 baseline。
- 时间单位表中写 `m`，但 FEX 下游硬件配置未明确，影响速度结果复现。
- 精度图是 boxplot，正文没有给出均值、方差、显著性检验和失败率。
- 有些公式和证明符号存在 typos，例如 `D`/`mathcal D` 混用、`M_i` 未清晰定义、maximize/minimum 文字不一致。
- 因此这篇论文适合作为方向性和 pipeline prior，不宜当作已经封闭验证的 benchmark paper。

### 跟我们的关系
- 对 `0616-fex` 来说，这是直接相关甚至部分撞车的工作。
- 如果我们的 claim 是“用 LLM 预测 PDE 解的候选算子来加速 FEX”，这篇已经做了。
- 如果我们的 claim 是“LLM 不是 solver，而是 operator selector / search-space pruner”，这篇也是非常接近的 prior。
- 可直接借鉴的工具链包括 postfix/RPN 表达、operator-set binary vector mismatch、LoRA fine-tuning、LLaMA-8B decoder-only 选择、FEX 动态 operator pool。
- 可以引用它作为 LLM-informed FEX 的最近先例，尤其是 4.4x-6.0x 时间加速结果。
- 差异化空间在于：公开可复现 benchmark、真实 PDE/复杂几何、uncertainty-aware soft operator gating、漏算子 fallback、与多种 symbolic regression backend 对比。
- 另一个差异化方向是把 hard pruning 改成 probabilistic prior，让 FEX 在高置信算子上优先搜索，但不彻底删除低置信算子。
- 还可以做 evaluator separation：LLM 只产生候选，独立 residual verifier 和 holdout PDE family 决定是否接受。
- 若我们的研究更偏 FEX 理论，需要注意本文的 `O(epsilon^{-4})` 是通用 SPPGM stationarity，不是 LLM+FEX 端到端理论。
- 若我们的研究更偏系统，应把这篇视为 baseline：复现它，再加入校准、ablation 和失败分析。
