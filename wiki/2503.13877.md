# Shock with Confidence: Formal Proofs of Correctness for Hyperbolic Partial Differential Equation Solvers (2503.13877)

- **Year**: 2025.03
- **Authors**: Jonathan Gorard (Princeton University), Ammar Hakim (Princeton Plasma Physics Laboratory)
- **Venue**: arXiv (ACM SIGPLAN format)
- **Keywords**: formal verification, hyperbolic PDEs, finite volume methods, shock-capturing schemes, Racket DSL, floating-point theorem proving, automatic code generation, CFL stability, flux conservation, computational physics
- **One-liner**: 在 Racket 中构建了一个形式化验证管线，用于从双曲 PDE 系统规范自动生成可验证正确性的 C 代码有限体积求解器，证明了 L² 稳定性、通量守恒和物理有效性等性质。

## 精华提炼

### 管线/方法
完整的端到端验证管线：
1. **输入**：用户在 Racket DSL 中用四个列表描述 PDE 系统：`cons-exprs`（守恒变量）、`flux-exprs`（通量向量）、`max-speed-exprs`（最大波速估计）、`parameters`（模拟参数，如状态方程变量）
2. **代码生成**：`convert-expr` 函数递归地将 Racket 符号表达式转换为功能等价的 C 代码字符串（含显式括号以保证运算顺序一致），条件表达式转为三元运算符或 if 语句，支持 `abs`/`max`/`min` 等数学函数映射到 C 的 `fabs`/`fmax`/`fmin`
3. **符号化简**：`symbolic-simp-rule` 递归地将表达式化简为规范代数形式，基于全局合流且强规范化的重写系统；核心约束是只允许 IEEE 754 浮点算术下正确性保持的代数变换（允许交换律但不允许结合律）
4. **自动微分**：`symbolic-diff` 实现标量函数的符号微分，支持加法（线性）、乘法（积法则）、`sqrt` 等标准函数；`symbolic-jacobian`/`symbolic-gradient`/`symbolic-hessian` 基于此构建
5. **定理证明**：
   - CFL 稳定性：计算通量雅可比特征值 → 取绝对值 → 与 `max-speed-exprs` 比较 → 验证 CFL 常数 ∈ (0,1]
   - 双曲性保持：`is-real` 函数判断特征值实性，`are-distinct` 判断严格双曲性（特征值互异）
   - 局部 Lipschitz 连续性（物理正确性）：计算通量 Hessian → 验证半正定性（特征值非负）→ 用 `is-non-negative` 判定
   - Roe 通量守恒：代数验证跳变条件两端化简到同一规范形式
6. **二阶精度扩展（通量限制器）**：
   - `variable-transform`：执行 $r \to 1/r$ 变换验证对称性
   - `evaluate-limit`：在扩展实数上求极限，用强化的 Sweby 条件（$r\to 0, r\to 1, r\to 2, r\to \infty$ 极限 + 凹性）验证 TVD 性质
7. **输出**：独立可运行的验证 C 代码，或可集成到 Gkeyll 多物理框架的验证模块；每个生成的证明本身是可独立运行的 Racket 代码，作为正确性证书

### 关键技术细节
- **语言和实现规模**：Racket 实现，总计约 15,000 行；核心定理证明器、自动微分和符号极限求值各仅几百行
- **重写系统设计原则**：全局合流（confluent）+ 强规范化（strongly normalizing），排除交换律重写规则以保证终止性
- **浮点算术约束**：严格遵循 IEEE 754 标准，不假设加法/乘法结合律；例如 $(10^{30} + (-10^{30})) + 1 = 1$ 但 $10^{30} + ((-10^{30}) + 1) = 0$ 被正确区分
- **代数规则示例**：0 的左加性恒等元、1 的左乘性恒等元、0 的左乘性零化子、数值常数左置、同类项合并（分配律）、纯数值直接求值、$\sqrt{x^2}=x$、$|\text{-}x|=|x|$、$\max(x,y)=\frac{1}{2}(x+y)+\frac{1}{2}|x-y|$
- **矩阵处理**：所有矩阵操作限制在 2×2；高维 PDE 系统通过"分解为耦合 PDE 对"处理（如 Maxwell → 4 对，等温 Euler → 2 对）
- **Roe 矩阵构造**：$\mathbf{A}(\mathbf{U}_i, \mathbf{U}_{i+1}) = \frac{1}{2}[\nabla_{\mathbf{U}}\mathbf{F}(\mathbf{U}_i) + \nabla_{\mathbf{U}}\mathbf{F}(\mathbf{U}_{i+1})]$
- **四个测试方程系统**：线性平流（linear, scalar）、无粘 Burgers（nonlinear, scalar）、完美双曲 Maxwell（linear, vector, 8 分量）、等温 Euler（nonlinear, vector, 4 分量）
- **四种通量限制器**：minmod、monotonized-centered、superbee、van Leer
- **代码公开**：Racket 管线 https://github.com/ammarhakim/gkylcas/ ；验证 C 代码 https://github.com/ammarhakim/gkylzero/

### 核心结果
全部以"证明步数"量化，而非传统数值精度指标：

| PDE 系统 | 双曲性(Lax) | 严格双曲性(Lax) | CFL稳定(Lax) | Lipschitz(Lax) | 双曲性(Roe) | 严格双曲性(Roe) | 守恒(Roe) |
|---------|------------|--------------|------------|---------------|----------|--------------|--------|
| 线性平流(scalar) | 33 | - | 45 | 39 | 55 | - | 92 |
| 无粘Burgers(scalar) | 90 | - | 116 | 96 | 120 | - | 188 |
| Maxwell (Ey,Bz) | 499 | 502 | 575 | 272 | 616 | 619 | 412 |
| Maxwell (Ez,By) | 627 | 630 | 711 | 452 | 826 | 829 | 643 |
| Maxwell (Ex,φ) | 729 | 735 | 803 | 450 | 871 | 877 | 642 |
| Maxwell (Bx,ψ) | 731 | 737 | 805 | 450 | 873 | 879 | 642 |
| Euler (ρ,ρu) | 1462 | 1465 | 1558 | **失败** | **失败** | **失败** | **失败** |
| Euler (ρv,ρw) | 1189 | **失败** | 1329 | 251 | 1315 | **失败** | 498 |

通量限制器结果：

| 限制器 | 对称性 | 二阶TVD |
|--------|--------|--------|
| minmod | 863 | 513 |
| monotonized-centered | 4171 | 2251 |
| superbee | **失败** | 2125 |
| van Leer | 244 | **失败** |

**关键发现（反直觉）**：
- 定理证明器的"失败"往往是正确的：等温 Euler 中 $\rho$ 和 $\rho u$ 分量的 Lipschitz 连续性证明被正确阻塞在需要 $\rho > 0$ 的前提条件上；Roe 双曲性证明受阻于密度正性条件
- 只有 Roe 通量守恒对 $\rho,\rho u$ 是真正的"不应失败但失败了"的案例，原因是符号化简算法的当前代数能力不足
- superbee 对称性和 van Leer TVD 的失败也是代数能力不足，而非性质本身不成立

### 明确局限
- **仅处理 1D 问题**：所有模型方程都是一维的；未讨论多维推广
- **仅一阶时间精度**：二阶精度只针对空间，使用半步时间推进但未达到二阶时间精度
- **仅 Lax-Friedrichs 和 Roe 两种通量函数**：未涉及 HLL、HLLC、Kurganov-Tadmor 等其他广泛使用的格式
- **仅 2×2 特征值计算**：更高维矩阵需要显式编码特征值求解公式
- **仅四种通量限制器**：未涉及更现代的 WENO 或 DG 限制器
- **无真机/实际实验验证**：只证明了算法的数学/物理性质，没有实际运行模拟并对比解析解
- **等温 Euler 的不完整证明**：Roe 求解器在非线性向量情况下证明能力严重不足
- **重写系统局限**：当前代数规则集无法处理 superbee 对称性和 van Leer TVD 的情况
- **无性能评估**：没有报告生成的 C 代码在运行时的性能
- **未明确与 SMT solver 或 Coq/Lean 等通用定理证明器对比**：自建定理证明器的完备性和可靠性与成熟工具链的关系未经讨论
