# Reinforcement Learning for Laser Additive Manufacturing Scan-Order Optimisation: A Bilevel Proxy--FEA Diagnostic Framework for Reward and World-Model Diagnosis (2605.25063)

- **Year**: 2026.05
- **Authors**: Xian Wu, Haoran Li, Dongbin Zhao, Ruiyao Zhang, Yuanqi Chu, Bin Wang; Brunel University London, Chinese Academy of Sciences, ISIS Neutron and Muon Source
- **Venue**: arXiv
- **Keywords**: reinforcement learning, laser additive manufacturing, scan-order optimisation, Proxy-FEA alignment, finite-element analysis, reward modelling, world-model diagnosis, residual stress, distortion
- **One-liner**: 用稀疏 Abaqus FEA 诊断 RL 扫描顺序 proxy reward 是否错配。

## 精华提炼

### 管线/方法

1. 论文研究的问题不是“训练出了一个最优 RL agent”，而是“未来如果用 RL 优化激光增材制造扫描顺序，reward/world-model 信号是否可信”。
2. 输入是一个简化 LDED32 stripe benchmark：构件被离散成 32 条 scan tracks，每个完整扫描策略是这 32 条 track 的一个排列。
3. 作者没有让 RL 在线调用昂贵 FEA，而是提出双层诊断框架：lower-level cheap proxy 负责快速生成/筛选候选，upper-level sparse FEA 负责给少量完整策略打高成本 reference label。
4. lower-level proxy 将完整扫描序列 \(\tau\) 映射为一组便宜 descriptor：path spacing、jump distance、neighbour gap、clustering、boundary exposure、symmetry、thermal-memory-like indicators。
5. proxy 可组合成 \(J_{\mathrm{proxy}}(\tau;\alpha)=\alpha^T \tilde p(\tau)\)，但作者明确说这不是 validated surrogate，只是候选 reward/world-model 信号。
6. upper-level FEA 用 Abaqus 对选出的完整扫描策略做稀疏 reference evaluation，不在每一步 action 上调用。
7. FEA 输出三个最终冷却残余指标：residual Mises stress、U3 vertical distortion range、PEEQ-based plastic fraction。
8. 残余 Mises 指标采用 scan region 内排除边界条件支配区域后的 top-5 mean，避免单个极端节点主导。
9. U3 指标是 scan region 内 \(max(U3)-min(U3)\)，作为主要 vertical distortion proxy。
10. PEEQ 指标是 scan region 内超过阈值的 plastic fraction；在本文数据中区分度弱。
11. FEA composite score 由归一化 Mises、U3、PEEQ 加权得到，低分代表更好。
12. 核心诊断不是回归误差，而是 preference structure：cheap proxy 排出的策略顺序是否与 FEA reference 排序一致。
13. 作者用 pairwise ranking disagreement 衡量 proxy 与 FEA 对任意两个策略的偏好是否相反。
14. 这很适合 RL 语境：policy 不一定要预测精确 FEA 值，但 reward 至少不能系统性偏好 FEA 上更差的策略。
15. 实验策略集固定为 10 个完整 scan orders，不是 exhaustive optimisation，也不是 trained policy rollout。
16. 10 个策略包括 raster_left_to_right、odd_even_interlaced、center_out、edge_in、greedy_maximin、smartscan_proxy、multilag_jump、block_quarters、windowed_dispersion、center_edge。
17. 整个 workflow 是：32-track permutation space -> proxy descriptors -> select/evaluate 10 strategies -> Abaqus final-cooling labels -> ranking robustness -> Proxy-FEA alignment。

### 关键技术细节

18. benchmark 是 LDED-style stripe configuration，但使用 whole-track heating abstraction，每次激活一整条 track。
19. 该抽象故意不解析完整 moving heat-source physics，所以不能当作 calibrated high-fidelity LDED simulation。
20. FEA reference label 来自 final-cooling last frame，论文反复强调是 simulation-based reference，不是实验 ground truth。
21. 边界条件支配区域被排除后再算 Mises 和 U3，这一点对可解释性重要，因为固定边界会制造非扫描顺序主导的应力/位移模式。
22. baseline FEA score 用三项归一化 residual metrics 的加权和，随后用 alternative weightings 做 ranking robustness。
23. ranking robustness 的作用不是找物理最优权重，而是测试“center_out 是 compromise”的解释是否对权重敏感。
24. v1 proxy alignment 中，最强 proxy 是 `proxy_jump_mean`。
25. `proxy_jump_mean` 对 composite FEA score 很弱：Pearson -0.045，Spearman 0.249。
26. `proxy_jump_mean` 对 Mises 有中等相关：Pearson 0.526，Spearman 0.505，但作者不把它验证成 stress proxy。
27. `proxy_jump_mean` 对 U3 更强：Pearson -0.842，Spearman -0.730，说明 cheap path-spacing 类指标主要抓到 distortion/U3 相关趋势。
28. v1 pairwise agreement 是 0.614，只是略高于随机，无法作为可靠 reward。
29. v2 proxy 增加 neighbour-gap、hot-cluster、boundary/symmetry、stress-risk、distortion-risk、compromise-oriented descriptors。
30. v2 最好的 composite raw signal 是 `all_window_dispersion_mean`，Pearson 0.120，Spearman 0.439，仍然不能行动化。
31. v2 对 Mises 的 `early_window_pairwise_distance_mean` Pearson 0.858、Spearman 0.854，但作者标为 wrong-sign diagnostic。
32. v2 对 U3 的 `proxy_stress_risk_candidate` Pearson -0.622、Spearman -0.782，也被解释为 wrong-sign diagnostic。
33. v2 最好 pairwise agreement 是 `edge_first_ratio` 的 0.667，仍只是 exploratory signal。
34. 论文的数据与代码在 GitHub 仓库 `BrunelXian/RL-LAM-ScanOpt`，但 large Abaqus ODB 不公开，只公开 derived metrics、tables、figures 和 scripts。

### 核心结果

35. FEA reference ranking 第一名是 `center_out`：Mises top-5 = 203.481 MPa，U3 range = 0.452 mm，PEEQ frac = 99.997%，score = 0.266。
36. 第二名是 `odd_even`：Mises 239.073 MPa，U3 0.897 mm，PEEQ 99.405%，score 0.380。
37. 第三名是 `block_quarters`：Mises 301.565 MPa，U3 0.490 mm，PEEQ 99.406%，score 0.405。
38. `raster` 的 Mises 最低，194.164 MPa，但 U3 最高，1.607 mm，所以 composite 只排第 8。
39. `edge_in` 的 U3 最低，0.276 mm，但 Mises 很高，407.721 MPa，所以 composite 排第 7。
40. 这构成本文最重要的 empirical finding：扫描顺序不是单一 monotonic quality objective，而是 stress-distortion trade-off。
41. `center_out` 不是每项最优，而是在 stress 和 distortion 两个 endpoint 之间保持较稳健折中。
42. `smartscan_proxy` 反而排最后：Mises 353.251 MPa，U3 0.527 mm，PEEQ 99.268%，score 0.505。
43. 这个结果直接支持作者的 reward-mismatch 警告：proxy-informed strategy 未必是 FEA-informed strategy。
44. PEEQ 基本都在约 98.5%-100% 区间，区分度弱，主要结论由 Mises 和 U3 驱动。
45. residual field maps 支持 ranking table 的解释：raster 的低 Mises 伴随大 U3 distortion，edge_in 的低 U3 伴随高 stress，center_out 避开两个极端。
46. ranking robustness 显示，当权重偏 stress 时 raster-like behaviour 更有利；当权重偏 distortion 时 edge-in behaviour 更有利。
47. 因此未来 RL reward 不能只奖励 long jumps、spatial separation、local cooling 或 reduced reheating，否则可能只优化某个残余响应。
48. cheap proxy 的合理定位是 candidate generation / preliminary screening，而不是 stand-alone FEA replacement。
49. sparse FEA 的合理定位是 diagnostic correction layer：在大规模 policy training 前发现 reward/world-model 偏差。
50. 论文最终贡献更像一个 reward/evaluator QA protocol，而不是制造工艺或 RL 算法本身的最终突破。

### 明确局限

51. 没有训练完成的 RL policy，也没有证明 PPO 或其他 RL 算法能收敛到好 scan order。
52. 只有 10 个手工/规则生成的完整策略，策略集小且结构化，不能代表高熵搜索空间。
53. LDED32 是简化 whole-track heating，不是 fully resolved moving heat-source LDED。
54. Abaqus FEA 是 simulation reference，不是实验验证；没有真机制造、残余应力测量或形变实测闭环。
55. FEA 模型没有宣称 calibrated high-fidelity physics，所以物理外推要保守。
56. ODB 等大二进制 simulation 文件未公开，第三方只能复现 derived analysis，不容易从原始 FEA 场重新审计。
57. PEEQ threshold 在当前策略集上几乎饱和，作为第三项 composite metric 的贡献弱。
58. proxy descriptor v2 虽然更丰富，但仍未识别出 FEA 认为最好的 `center_out` compromise。
59. 相关性分析样本数只有 n=10，Pearson/Spearman/pairwise agreement 都只能作 qualitative diagnostics。
60. 作者没有证明 sparse FEA labels 足够指导后续高熵 RL training，只说明当前 benchmark 暴露了 proxy mismatch。
61. 下一步应转向 LPBF-like 2D patch/grid benchmark，带 variable masks、random subset targets、local action consequences 和 geometry generalisation。
62. 对 0616-fex 而言，本文不提供 expression grammar、symbolic PDE solver、FEX controller 或 SR benchmark 结果。

### 跟我们的关系

63. 对 `0616-fex`，这篇论文不直接撞车：它研究的是激光增材制造扫描顺序优化，不是 FEX、不是 symbolic regression、不是 PDE 闭式解搜索。
64. 其 action space 是 32-track permutation；FEX 的 action space 是 expression tree / operator sequence，两者技术对象不同。
65. 其 high-fidelity evaluator 是 Abaqus FEA；FEX 的 high-fidelity verifier 更可能是 dense PDE residual、held-out collocation grid、exact/semantic expression equivalence、ground-truth solution error 或结构恢复指标。
66. 真正可借鉴的是 reward/world-model 诊断范式：cheap training reward 不能自动等同于最终科学目标。
67. FEX 中的 cheap proxy 可能是训练点 PDE residual、短 rollout reward、低精度 quadrature、浅树先验或 operator-frequency heuristic。
68. FEX 中的 sparse teacher 可以设计成：高密度 residual audit、跨 domain extrapolation、symbolic equivalence/e-graph canonicalization、profile likelihood 参数可辨识性、或多组边界/初值上的 generalisation test。
69. 本文的 pairwise ranking agreement 很适合移植到 FEX：比较 cheap reward 排序与 verifier 排序是否一致，防止 controller 学到“低残差但结构错”的表达式。
70. 若 `0616-fex` 要写 RL reward misspecification、multi-fidelity verifier、candidate screening + sparse high-fidelity audit，这篇可以作为跨领域方法论引用。
71. 若 `0616-fex` 只写 FEX theory、SR complexity、LLM skeleton 或 e-graph pruning，这篇相关性低，不应放核心 related work。
72. 差异化很清楚：FEX 的 novelty 边界在 symbolic expression search / PDE solving / search complexity；本文的 novelty 在 AM scan-order reward diagnosis。
73. 可用工具/方法：bilevel evaluator 架构、pairwise ranking disagreement、target-specific proxy diagnosis、stress-distortion trade-off 类比成 multi-objective expression quality。
74. 不建议把本文当成 FEX 直接竞品；更适合当“RL reward 要有独立 verifier gate”的支持性证据。
