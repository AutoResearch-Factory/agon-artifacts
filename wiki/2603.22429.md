# Neural Structure Embedding for Symbolic Regression via Continuous Structure Search and Coefficient Optimization (2603.22429)

- **Year**: 2026.03
- **Authors**: Fateme Memar, Tao Zhe, Dongjie Wang; Department of Electrical Engineering and Computer Science, University of Kansas
- **Venue**: arXiv
- **Keywords**: symbolic regression, structure embedding, Transformer prior, postfix notation, coefficient abstraction, continuous structure search, gradient coefficient optimization, Feynman benchmark
- **One-liner**: 用结构嵌入和系数优化做符号回归。

## 精华提炼

### 管线/方法

1. 输入是符号回归数据集：连续输入 $\mathbf{x}\in\mathbb{R}^d$ 和标量输出 $y$。
2. 目标是恢复一个可解释表达式 $f_S(\mathbf{x};\mathbf{w})$，其中 $S$ 是符号结构，$\mathbf{w}$ 是数值系数。
3. 论文把 SR 写成嵌套优化：先在可接受结构空间 $\StructSpace$ 中找结构，再在固定结构下最小化 MSE。
4. 第一阶段是结构语料构建：对每个训练数据集运行 GP 型 SR 工具 `gplearn`，生成候选公式。
5. `gplearn` 的设置包括 population size 2000、20 generations、maximum depth 4、parsimony coefficient 0.001、固定随机种子。
6. 每个方程生成 $M=1000$ 个候选公式，跨 Feynman synthetic 和 Feynman real-world 两个 benchmark 聚合为结构语料。
7. 使用的基础函数集合是 $\{+,-,\times,\div,\sin,\cos\}$。
8. 语料预处理把所有数值常数替换成统一的 `COF` 占位符，从而把“结构”和“系数值”解耦。
9. 预处理后每个表达式被确定性转换成 postfix / reverse Polish notation token 序列。
10. malformed 序列会被丢弃，并用长度上限控制复杂度。
11. 第二步训练 Transformer 序列模型，学习 coefficient-abstracted postfix 结构的自回归分布。
12. token embedding 加 positional encoding 后进入 Transformer，输出上下文 hidden states。
13. softmax head 预测下一个 token，训练目标是最大化所有结构序列的自回归 log-likelihood。
14. 训练后的模型被解释为结构先验 $p_\theta(S)$，同时诱导一个连续结构表示空间。
15. 第二阶段是结构搜索：从 start token 开始按 Transformer prior 自回归采样 postfix 序列。
16. 采样使用 temperature scaling 和 top-k sampling，直到 end token 或最大长度 $L_{\max}$。
17. 每轮重复采样 $M$ 次，产生一批候选结构。
18. 候选结构先经过 stack-based parser 检查 postfix 语法，要求最终化约成一个合法表达式树。
19. 之后做 semantic filtering，丢弃显然 undefined 的表达式，例如 trivial division-by-zero pattern。
20. 再用复杂度约束 $\mathcal{C}(S)\leq C_{\max}$ 保持可解释性。
21. 论文还加了一个 lightweight proxy scorer，用采样过程中的 temperature-scaled top-k log-prob 累加分数做结构 plausibility 排序和日志记录。
22. 这个 proxy score 只用于候选排序/记录；最终结果仍由系数拟合后的 test metrics 决定。
23. 第三阶段是系数优化：每个 `COF` 被替换成不同的可学习参数 $w_i$。
24. 固定结构后，优化问题变成对 $\mathbf{w}$ 的连续 MSE 最小化。
25. 论文使用 gradient-based coefficient fitting，并带随机初始化、最大迭代预算、loss plateau early stopping 等 safeguard。
26. 每个结构拟合后用 MSE、$R^2$、Pearson $\rho$ 评估。
27. 最终表达式在性能指标和复杂度预算下选择。
28. 这套管线的核心主张是：用 `gplearn` bootstrapping 产生结构语料，用 Transformer 形成结构 prior，再用解耦的 gradient coefficient fitting 提高数值精度。

### 关键技术细节

1. 结构表示：变量 token、operator/function token、`COF`、special sequence markers 组成 postfix token 序列。
2. 训练模型：Transformer sequence model，论文给出 token embedding、positional encoding、softmax next-token objective，但没有给出层数、hidden size、head 数、batch size、learning rate 等训练超参数。
3. 结构先验训练数据：来自两个 Feynman-based symbolic regression benchmarks，每个方程用 `gplearn` 产生 1000 个候选公式。
4. Benchmark 1 是 Feynman--real-world：easy/medium/hard 分别 30/40/50 个方程，输入维度分别 2/3/4。
5. Benchmark 2 是 Feynman--synthetic：同样 30/40/50 个方程和 2/3/4 输入维度，从 ground-truth expression 采样 train/test。
6. 所有报告结果都在 held-out test split 上计算，SRCO 只用 training split 做结构学习和系数拟合。
7. Baselines：DSO、FFX、EFS、`gplearn`。
8. 评测指标：MSE、$R^2$、Pearson correlation $\rho$；部分表格报告 $\log(\mathrm{MSE})$。
9. 结构搜索约束：语法合法性、语义定义性、表达式长度或 operator count 等复杂度预算。
10. 系数优化 ablation：在 Feynman-bonus.1 上比较 gradient fitting (`model+`) 与 stochastic hill-climbing/random search (`model-`)。
11. 噪声鲁棒性实验：只扰动 test input features，目标 $y$ 和 test split 固定；$\epsilon_{ij}\sim\mathcal{N}(0,(\eta\sigma_j)^2)$，$\sigma_j$ 来自 training split。
12. 噪声水平从 0% 到 100%，固定已发现方程结构和系数，只改变测试输入。
13. 推理效率实验：测最终方程在 test split 上的一次 forward evaluation wall-clock time，平均六种设置，即 2 个 benchmark 乘 3 个 difficulty tiers。
14. 参数敏感性实验：单独改变 `max_term` 和 `max_trig_vars`，看 test $R^2$ 和 Pearson $\rho$。
15. `max_term` 从 4 增大到 26 时 Pearson 从 0.987 到 0.999，$R^2$ 从 0.972 到 0.997，18--22 后收益递减。
16. `max_trig_vars` 的主要收益在 3--4 附近出现，6--10 只带来边际变化。

### 核心结果

1. Feynman--synthetic easy：SRCO $R^2=0.99821$，Pearson 0.96556，$\log(\mathrm{MSE})=30.4281$。
2. Feynman--synthetic medium：SRCO $R^2=0.99555$，Pearson 0.99786，$\log(\mathrm{MSE})=67.14446$。
3. Feynman--synthetic hard：SRCO $R^2=0.93708$，Pearson 0.94424，$\log(\mathrm{MSE})=65.8075$。
4. Feynman--real-world easy：SRCO $R^2=0.95045$，Pearson 0.96984，$\log(\mathrm{MSE})=39.7806$。
5. Feynman--real-world medium：SRCO $R^2=0.99936$，Pearson 0.99940，$\log(\mathrm{MSE})=67.1605$。
6. Feynman--real-world hard：SRCO $R^2=0.94961$，Pearson 0.96152，$\log(\mathrm{MSE})=65.8739$。
7. 论文声称 SRCO 在两个 benchmark 的 easy/medium/hard tier 上都超过 DSO、FFX、EFS、`gplearn`。
8. 一个显著现象是 baseline 在 hard tier 上会出现极端负 $R^2$，例如 synthetic hard 的 `gplearn` 为 $-1.84\times10^{13}$，DSO 为 $-2.37841\times10^5$。
9. 系数优化 ablation 中，gradient fitting 在 Feynman-bonus.1 上达到 $R^2=0.999$、$\rho=0.999$、MSE 约 0.001。
10. 相同 template 和预算下，random-search hill-climbing 只有 $R^2=0.928$、$\rho=0.931$、MSE 约 0.016。
11. 噪声鲁棒性：0% noise 时 $R^2=0.9990$；10% noise 时 0.9973；40% noise 时 0.9828；70% noise 时 0.9521；100% noise 时 0.9009。
12. Pearson 在高噪声下也平滑下降，100% noise 时为 0.9009。
13. 推理时间：SRCO 每方程 0.00649 s，EFS 0.00651 s，DSO 0.0171 s，FFX 0.0396 s，`gplearn` 0.250 s。
14. 按论文报告，SRCO 与 EFS 基本同速，比 DSO 快 2.6 倍，比 FFX 快 6.1 倍，比 `gplearn` 快 38.5 倍。
15. 反直觉点：论文把方法称为 continuous structure search，但 Stage 2 的实际生成仍是 postfix token 的 constrained sampling；连续性主要来自 Transformer embedding/prior，而不是对离散结构变量直接做可微优化。
16. 另一个需要留意的点：表格中的 $\log(\mathrm{MSE})$ 数字为正且很大，和噪声实验里的负 log MSE 不一致；论文说会标注 transform，但这里可读性/定义仍不够清楚。

### 明确局限

1. 作者明确承认评估只限于两个 Feynman benchmark。
2. 作者明确承认当前结构语料依赖 `gplearn` bootstrapping，后续应减少这种依赖。
3. 作者明确提出 future work：更广和更高维 benchmark、更丰富 operator library、扩展 constrained sampling 之外的 proposal/selection 策略。
4. 论文没有报告代码、数据处理脚本或 artifact 链接，第三方复现需要自行补齐很多实现细节。
5. Transformer 的关键训练配置缺失，包括模型规模、优化器、学习率、batch size、训练步数、token vocabulary 细节、候选过滤比例。
6. Stage 2 的采样规模、temperature、top-k、$L_{\max}$、$C_{\max}$ 等核心搜索超参数没有完整表格化。
7. coefficient optimizer 的具体实现不完整：只说 gradient-based optimization、随机初始化、early stopping，没有明确 optimizer 类型和迭代/重启策略。
8. 实验主要证明在 Feynman 风格方程上的拟合能力，没有覆盖高维真实科学系统、噪声训练、out-of-distribution extrapolation 或单位/量纲约束。
9. 没有报告 exact symbolic recovery rate，也没有系统分析表达式是否与 ground truth 代数等价。
10. 因为结构语料由 `gplearn` 生成，SRCO 的结构 prior 可能继承 `gplearn` 的搜索偏好和 blind spots。
11. 没有做训练语料去重/泄漏分析；Feynman synthetic/real-world 共享 equation family 时，结构模板可能存在近邻泄漏风险。
12. baseline 对齐细节不足，特别是 DSO、FFX、EFS 和 `gplearn` 的预算、重启次数、constant fitting 设置是否公平不完全可审计。
13. 没有置信区间、随机种子方差或多次运行统计，稳定性 claim 主要来自平均表格和少量 ablation。
14. 没有说明 Transformer prior 是否跨 equation 留一训练，还是在包含同一 benchmark 全部方程的结构语料上训练后再评估。
15. 对 0616-fex 若关注“可信的科学发现/自动研究”，这篇更像 SR 组件论文，不是完整科学假设生成和验证系统。

## Read by: 0616-fex (2026-06-17)

这篇论文对 0616-fex 的价值主要在“符号发现工具链”而不是“完整自动研究系统”。SRCO 的可复用设计是三段式：先用 `gplearn` 生成结构语料并把常数抽象成 `COF`，再训练 Transformer prior 采样 postfix 结构，最后固定结构做 gradient coefficient fitting。若我们的课题需要从实验数据中提取可解释方程，这个 pipeline 可以作为候选方程 proposer。

撞车风险取决于 0616-fex 的 claim 粒度。若 claim 是“用连续结构嵌入做符号回归并解耦系数优化”，SRCO 已经覆盖核心思路；近邻检索还显示 EQUATE、UniSymNet、SAGE-Fit、FunctionEvolve 等 2025-2026 工作分别覆盖 embedding search、Transformer-guided SR、参数优化瓶颈和 structure-aware fitting。若 claim 是“把符号回归嵌入到自动文献-假设-实验-验证闭环，并让检验者独立于生成者”，SRCO 没有覆盖。

可借鉴工具包括：postfix 表达式规范化、`COF` 系数占位、stack-based syntax gate、semantic/complexity filters、held-out test metrics、noise robustness check、coefficient optimizer ablation。需要避开的弱点包括：只在 Feynman benchmark 上证明、缺少 exact recovery audit、缺少实现超参数、没有开源 artifact、没有独立复现。对 0616-fex 的差异化建议是把 SRCO 当作 equation proposal backend，同时增加单位/量纲约束、benchmark leakage audit、独立 verifier veto 和真实实验/仿真闭环。
