# A Unified Data-Driven Framework for Efficient Scientific Discovery (2509.07303)

- **Year**: 2025.09
- **Authors**: Tingxiong Xiao, Xinxin Song, Ziqian Wang, Boyang Zhang, Jinli Suo (Tsinghua University, Dept. Automation)
- **Venue**: arXiv preprint (submitted to Nature Communications format — uses wlscirep.cls)
- **Keywords**: symbolic regression, dimensional analysis, Buckingham Pi theorem, scientific discovery, formula discovery, physics-informed, symbolic simplification
- **One-liner**: Proposes FIND — a physics-informed two-stage framework that uses Buckingham Pi theorem for latent variable discovery and Taylor's theorem for polynomial expression, reducing formula search from hundreds of millions of iterations to tens via dimensional constraints and coarse-to-fine grid search.

## 精华提炼

### 管线/方法
FIND 将公式发现分解为四个阶段：
1. **两阶段复合函数建模** — 输入 x 先通过 latent layer f1 映射到低维潜变量 z（乘幂形式），再通过 expression layer f2 映射到输出 y（多项式形式）。f1 受 Buckingham Pi 定理启发，f2 受 Taylor 定理启发。
2. **结构辨识** — 用 SHAP 值排序选出有贡献的输入变量；对稠密数据计算 PPMCC 矩阵推断潜变量个数、输入-潜变量连接关系和权重比。
3. **参数优化** — 施加量纲不变性约束(DI-1 搜索中约束 / DI-2 搜索后约束)，C2F (粗到细) 网格搜索以步长从 1.0→0.5→0.2→0.1 逐级细化，多级优化逐个学习潜变量。
4. **表达式简化** — 用 PySR (genetic programming symbolic regression) 将多项式简化为紧凑符号形式。

### 关键技术细节
- **模型架构**：$z_i = \prod_{j=1}^{p} x_j^{w_{ij}}$ (latent layer)，$y = a_0 + \sum a_i z_i + \sum a_{i_1 i_2} z_{i_1} z_{i_2} + \ldots$ (expression layer，默认 5 阶多项式)
- **SHAP 输入筛选**：使用 SHAP (Lundberg & Lee 2017) 对每个输入变量计算对输出的平均贡献，保留 top-k
- **PPMCC 结构辨识**：定义 $\rho_j = x_j \frac{\partial y}{\partial x_j}$，计算 $\rho_j$ 与 $\rho_k$ 的 Pearson 相关系数。若相关系数 ≥ 0.95，则认为 $x_j$ 和 $x_k$ 连接到同一潜变量，且比值 $\rho_j/\rho_k \approx w_{ij}/w_{ik}$
- **量纲约束**：构建维度矩阵 D (d×p) 和 d (d×1)，要求 $D W_i^T = d$，通过求齐次解基向量 E 和特解 e*，将 W 的搜索转化为 λ 的搜索，维度从 s×p 降到 s×(p-r(D))
- **C2F 网格搜索**：先以步长 1.0 在 [-2,2]^c 采样 5^c 个候选，计算 R^2 选 top 候选；再用步长 0.5 细化，重复至 0.2 或 0.1
- **多级优化**：串行学习潜变量 z1→z2→...，将搜索空间从 5^{s×p} 降至 s×5^p
- **导数估计**：支持后向差分法和多项式拟合法两种偏导数计算方式
- **符号回归工具**：PySR (Cranmer et al. 2023)
- **数据集规模**：11 个数据集，涵盖天体物理(NASA 行星数据 9 个天体)、系外行星、氢 Knudsen 压缩机(120 测量)、激光-金属作用、3D 打印孔隙率、Rayleigh-Benard 对流、电催化反应、弹簧-质量-阻尼器(8 个时间序列×800 点)、Karman 涡街、RLC 电路(交流 31250 样本)、RLC 无源电路

### 核心结果
- **搜索效率**：从数亿次搜索降到几十次（DI-1 + 输入辨识组合下，搜索 d 的输出公式从 464 次降到 6 次迭代）
- **物理定律发现**：从 NASA 太阳系行星数据发现 58 个公式，57 个可由理论验证；1 个新公式 $m \approx 6.8\times 10^{26} v_e^2/t_d^2$ 主要适用于大质量远距离行星。系外行星数据在 26 次搜索/0.14 秒内验证 Kepler 第三定律
- **无量纲数发现**：在氢 Knudsen 压缩机数据上，681 次搜索/1.13 秒发现 10 个 R^2>0.99 的无量纲数，其中 $\sqrt{\Delta T \lambda_2^3 / (H^3 T_c)}$ 为最高 R^2 的新发现。3 种气体共享 163 个无量纲数，趋势高度一致
- **PDE 发现**：SINDy + FIND 组合从弹簧-质量-阻尼器时间序列恢复 $\frac{dx}{dt} = \xi_2 x + \xi_3 \frac{d^2x}{dt^2}$，FIND 进一步发现 $\xi_2 = -k/c, \xi_3 = -m/c$，恢复完整原始 PDE。同样验证了 Navier-Stokes 方程和 RLC 电路方程
- **电路临界参数**：从 31250 样本中识别出 $|Z| = f_{|Z|}(1/R, \omega^2 L^2, 1/(\omega C))$ 和 $\Delta\phi = f_{\Delta\phi}(1/R^2, \omega L, 1/(\omega^2 C^2))$，PySR 简化为标准理论形式 $|Z| = \sqrt{R^2 + (\omega L - 1/(\omega C))^2}$，$\Delta\phi = \arctan((\omega L - 1/(\omega C))/R)$
- **综合比较**：FIND 在运行时间上仅次于纯 PR(多项式回归)但公式复杂度远低于 PR；DA(量纲分析)的复杂度低但 OOM 且 R^2 差；SR(符号回归)慢且公式复杂
- **潜结构辨识精度**：在 Δx=0.01 的稠密采样下，输入辨识 Precision 接近 1.0，潜变量个数估计比值接近 1.0，连接辨识 Precision 约 0.9-1.0，权重比估计约 0.8-1.0

### 明确局限
- **多项式阶数未知**：作者默认使用 5 阶多项式，但缺少对最优阶数的自适应估计
- **数据稠密度要求**：潜结构辨识依赖偏导数估计，只适用于稠密数据（适合可控变量实验，不适合稀疏观测）
- **潜变量形式受限**：潜变量仅表达为乘幂形式 $\prod x_j^{w_{ij}}$，无法表达 $z=\cos(x_1)+x_2$ 等形式
- **量纲依赖**：FIND 高度依赖物理量的单位/量纲信息进行搜索空间约简；在无物理量纲的纯黑箱问题中，DI 约束失效，效率优势大打折扣
- **真机实验**：没有真实机器人/物理实验数据，全部是仿真或已发表数据集
- **没有标准 benchmark 对比**：未与标准 symbolic regression benchmark (如 SRBench/Feynman datasets) 做系统对比，只在自己选的 11 个数据集上评估
- **多潜变量搜索仍可优化**：多级优化将搜索从 5^{s×p} 降到 s×5^p，但 s 大时仍是瓶颈
- **符号回归仅作后处理**：PySR 只用于表达式简化阶段，未与前置搜索过程联合优化

### 跟我们的关系
- **撞车评估**：不撞。FIND 用物理约束+网格搜索做公式发现，FEX 用 RL 做符号回归。两者方法路线完全不同。FIND 甚至可以视为 FEX 的对比方法/related work。
- **DI 约束与 RL 的结合**：FIND 的量纲不变性(DI)约束可以把搜索空间从 5^p 降到 (p-r(D)) 维——这种搜索空间约简技巧可以直接嵌入 FEX 的 RL action space 设计，使 agent 只在物理合法的公式空间内采样。对于有物理量纲信息的问题，这是强力加速器。
- **两阶段分解启发**：FIND 的 latent layer + expression layer 分解可以启发 FEX 引入 hierarchical RL 结构——先用 RL 发现潜变量 z=f1(x) 的数学形式，再用 RL 或回归发现 y=f2(z) 的形式。两阶段比单阶段更容易处理高维输入。
- **该引不该引**：应该引。FIND 是 2025 年与 FEX 问题域高度重叠的工作（都是公式发现/data-driven scientific discovery），而且它引用了 FEX 方向的核心工作（Udrescu & Tegmark 2020 AI Feynman 系列、Brunton 2016 SINDy、Cranmer 2020 PySR 等）。作为对比方法它很合适，可以突显 FEX 不需要量纲先验的优势。
- **差异化**：FEX 的优势在于（a）不需要物理量纲信息——纯黑箱场景也可用；（b）RL 学到的策略可以迁移到新的公式发现任务；（c）FEX 可以表达任意符号表达式，不限于乘幂+多项式。FIND 的优势在于物理可解释性、搜索效率（在量纲已知时极快）、以及不需要训练。两者在不同场景互补。

## Read by: 0616-fex (2026-06-17)

本次审读以 FEX (RL for Symbolic Regression) 课题视角分析 FIND。

**撞车评估**：不撞车。FIND 的核心方法是物理先验(量纲分析+Buckingham Pi)+ 网格搜索 + 多项式回归，完全不使用 RL。FEX 用 RL 学习表达式树的搜索策略。两者方法路线正交。

**可用工具/方法**：
1. **量纲约束作为 search space pruning**：FIND 的 DI 约束（$DW_i^T = d$）可将候选参数空间从指数级降至多项式级。在 FEX 处理物理公式发现任务时，可将 DI 约束编码为 RL agent 的 action mask——禁止选择量纲不合法的 token。
2. **两阶段分解 (latent + expression)**：FEX 可借鉴此思路引入 hierarchical decomposition——先发现中间潜变量的解析形式，再发现输出表达式。这对高维输入的公式发现问题尤其有价值。
3. **PPMCC 结构辨识**：对于稠密数据，可通过 $\rho_j = x_j \partial y/\partial x_j$ 的相关系数矩阵推断变量间的连接关系，这种结构先验可指导 FEX 的 RL 策略初始化。

**该引不该引**：应该作为 related work 引用。FIND 的目标问题（从数据自动发现物理公式）与 FEX 高度重叠，且 FIND 引用了 FEX 方向的关键 baseline (AI Feynman, PySR, SINDy)。引用 FIND 可以（a）定位 FEX 在文献景观中的位置；（b）论证 FEX 不需要量纲先验的优势场景。

**差异化**：
- FEX 不需要物理量纲信息 → 可用于纯黑箱/无物理先验的公式发现问题
- FEX 的 RL 策略可跨任务迁移 → 一次训练可用于多种公式发现
- FEX 可表达任意符号表达式 → 不限于乘幂+多项式
- FIND 在量纲已知时搜索极快 → 但依赖物理专家提供量纲信息
