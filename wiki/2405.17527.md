# Unisolver: PDE-Conditional Transformers Towards Universal Neural PDE Solvers (2405.17527)

- **Year**: 2024.05
- **Authors**: Hang Zhou, Yuezhou Ma, Haixu Wu, Haowen Wang, Mingsheng Long (Tsinghua University, School of Software, BNRist)
- **Venue**: ICML 2025
- **Keywords**: neural PDE solver, universal PDE solving, conditional Transformer, PDE components embedding, LLM symbolic encoding, domain-wise/point-wise conditioning, zero-shot generalization
- **One-liner**: Unisolver 用完整 PDE 分量（方程符号、系数、边界条件、外力、几何域）作为 domain-wise 和 point-wise 深度条件注入 Transformer，在三个大规模 benchmark 上实现 SOTA zero-shot PDE 泛化。

## 精华提炼

### 管线/方法

**完整 pipeline：输入 → 嵌入 → Transformer → 输出**

1. **输入**: 初始条件观测 X（在 mesh M 上离散化的物理场），完整的 PDE 分量 C_PDE（方程符号、系数、外力、几何域 mask、边界类型、边界值函数）
2. **PDE 分量嵌入** (Section 3.2):
   - **方程符号**: 用 LLaMA-3 8B 将 LaTeX 代码（如 `u_{tt} - a^2 u_{xx} = f(x,t)`）编码，取最后一个 Transformer block 输出的序列平均 → 4096 维向量，经 MLP 对齐到模型 hidden dim
   - **Domain-wise 分量**（系数、边界类型）：两个线性层 + SiLU → 条件向量
   - **Point-wise 分量**（外力场、几何 mask、边界值函数）：与输入相同的 patchify embedding
   - **条件整合**: 同类别内部 add 聚合
3. **PDE-Conditional Transformer** (Section 3.3):
   - **子空间解耦**: hidden dim 沿 channel 维度均分，一半由 domain-wise condition 影响，一半由 point-wise condition 影响
   - **条件注入**: 每层 attention 和 FFN 前后各施加 scale/shift/select 三路调制，形式为 `I_select ⊙ Module(I_scale ⊙ LN(X) + I_shift) + X`
   - 共 N=8 层（HeterNS）/ 8 层（1D）/ 6 层（2D）
4. **输出**: 最后一层输出经 scale/shift 调制后线性投影 → 预测量 Y

### 关键技术细节

- **LLM 编码细节**: LLaMA-3 8B，prompt 使用方程符号而非具体数值（如用符号 "a" 而非其数值），取 last Transformer block 输出的 sequence-mean pooling → 4096 维 → MLP 投影。预处理阶段离线完成，不参与训练反向传播
- **PDE 分量分类理论依据**: 从弦振动方程解析解出发，将分量按"全域一致影响"vs"逐点影响"分为两大类——domain-wise（方程符号、系数、边界类型）和 point-wise（外力、几何域、边界值函数）
- **模型参数量**: HeterNS: 4.1M (对比 FNO 4.7M, ViT 4.8M)；1D PDE: 19M (对比 PDEformer 22M)；2D Mixed PDE: 33M (对比 DPOT 30M)
- **训练配置**:
  - HeterNS: Adam, lr=0.0005, cosine annealing, batch=60, 300 epochs, Relative L2 loss
  - 1D PDE: Adam, lr=6e-4, batch=1024, 500 epochs, Relative L2 loss, PolyINR decoder
  - 2D Mixed PDE: AdamW, lr=1e-3, OneCycle scheduler, batch=320, 1000 epochs, warmup=200
  - 计算资源: 32 NVIDIA A100 (40GB) GPUs；总训练: ~3000h (1D PDE) + ~800h (2D) + ~24h (HeterNS)
- **HeterNS benchmark**: 自建，NS equation（涡量形式），5 种粘度 ν×3 种外力频率 ω=15 种组合，15,000 训练样本，分辨率 (64,64,10)；In-dist test 仅变初始条件，Zero-shot 变粘度或外力
- **1D Time-Dependent PDE benchmark**: PDEformer 提供，3M 样本，576 种方程类型（6 个系数可为 0 或非零 ×3 种 κ/s 状态 ×3 种边界类型组合），分辨率 (256,100)
- **2D Mixed PDE benchmark**: DPOT 收集，12 个数据集，74.1k 样本，来自 FNO、PDEBench、PDEArena、CFDBench，分辨率 (128,128,10)
- **评测指标**: 全部使用 Relative L2 = ||u_pred - u_gt|| / ||u_gt||
- **数据不平衡处理**: 2D 采用 DPOT 的 balanced data sampling 方法

### 核心结果

| Benchmark | Setting | Unisolver | Best Baseline | Promotion |
|-----------|---------|-----------|---------------|-----------|
| HeterNS (Viscosity) | In-dist | **0.0098** | ViT 0.0156 | 37.2% |
| HeterNS (Viscosity) | Zero-shot | **0.0374** | ViT 0.0514 | 27.2% |
| HeterNS (Force) | In-dist | **0.0428** | ViT 0.0593 | 27.7% |
| HeterNS (Force) | Zero-shot | **0.1053** | ViT 0.3158 | **66.7%** |
| 1D PDE | In-dist | **0.0108** | PDEformer 0.0225 | 52.0% |
| 1D PDE | 4 zero-shot avg | **—** | — | **27.1% avg** |
| 2D Mixed PDE | 12 datasets avg | **4.54** | DPOT 5.50 | 17.5% |

**关键反直觉发现**:
- 在 Force zero-shot 上提升最大 (66.7%)，远超 In-dist (27.7%)，说明条件建模在零样本泛化中尤其关键
- LLM 嵌入方程符号带来约 5.76% 提升——仅编码 6 个 PDE 分量中的 1 个，这个幅度显著
- 用 LLM 编码"错误"方程信息（*→/，指数取倒数）性能下降，但比完全去掉 LLM 好，说明 LLM 确实注入了有用先验
- 仅编码非零项数量（非完整方程）性能不如完整 LLM 嵌入，证明 LLM 捕捉了更多数学结构信息
- 30% 概率缺失分量仍能超越 FNO 全信息情况（平均 21.6% 的完整信息 boost）

**Fine-tuning 性能**:
- 1D PDE: 100 epochs 微调后误差比零样本降低 **61%**，比 PDEformer 同条件微调好 **59.3%**
- 2D Mixed PDE: 200 epochs 微调后比零样本降低 >**12%**，比 DPOT 同条件微调好 **14%**

**Scalability**: 数据 60x (50k→3M) 和参数 21x (3M→63M) 均呈现清晰的 scaling law (log-log 线性)

### 明确局限

**作者自己说的** (Appendix Limitations):
- 仅限结构化网格数据（grid data），因为 point-wise 分量的 patchify 嵌入需要规则网格
- 不规则网格 PDE 数据集缺乏且生成成本极高
- 建议用 Transolver 替换标准 Transformer 来处理不规则网格

**我读出来的**:
- 未做真机实验——全部是数值仿真数据的 surrogate model 验证
- 对 1D PDE benchmark 和 2D PDE benchmark 的计算开销极大（3000h + 800h GPU hours），硬件门槛高，中小团队难以复现
- 方程类型的广度有限：1D 仅 3 阶多项式以内的演化方程族，2D 主要是流体力学相关 PDE；没有覆盖椭圆方程、特征值问题等
- LLM 嵌入是离线预处理，端到端 fine-tune LLM 是否更好未探索
- 未与 Physics-Informed 范式做更深入的混合（仅对比了 PINO）
- 没有严格的理论泛化保证，完全依赖经验验证
- 与 0616-fex 课题的关系：这是 PDE solver 方向的通用模型工作，使用了 Transformer + conditional 策略；与 FEX（符号回归/PROSE 系）方向不同——PROSE 是用 Transformer 做 PDE 的符号表达式预测，Unisolver 是在已知 PDE 情况下做快速求解器

### 跟我们的关系

对 **0616-fex** 课题（符号回归/PROSE 方向的 PDE 符号表达式预测）:

**不撞车**：Unisolver 和 FEX 是互补关系而非竞争。Unisolver 假设 PDE 已知，目标是替代数值求解器的快速 inference。FEX 方向是从观测数据推断 PDE 符号形式（逆向问题）。

**可用工具/方法**:
1. **LLM 方程嵌入方法**：用 LLaMA 将 LaTeX PDE 编码为语义向量——这个可以直接借鉴到 FEX 模型中作为 PDE 的 representation
2. **Domain-wise vs Point-wise 条件分类**：这个分类思路可迁移——FEX 在做符号回归时，对不同类型的 PDE 项（扩散项、对流项、源项）可以设计不同的编码策略
3. **Conditional Transformer 的 scale/shift/select 调制机制**：可以用于 FEX 中根据 PDE 结构条件化 decoder 输出
4. **HeterNS / PDEformer 数据集**：可用作 FEX 的训练数据——从这些仿真数据中提取 PDE 符号表达式作为 label
5. **子空间解耦思想**：在 FEX 中分离方程结构的全局影响和物理场的局部影响

**差异化**：
- Unisolver 是前向问题（已知 PDE 求解），FEX 是逆向问题（从解推断 PDE）
- Unisolver 没有做符号回归或方程发现的能力
- 两者可以串联：FEX 推断 PDE → Unisolver 快速验证，形成闭环

## Read by: 0616-fex (2026-06-24)

**撞车评估**: 不撞车。Unisolver 是前向 PDE 求解器（已知 PDE → 解），FEX 是逆向符号回归（解 → 推断 PDE 形式）。两者是上下游互补关系而非竞争。

**可用工具**:
1. **LLM LaTeX 编码** (Section 3.2): 用 LLaMA 将 PDE LaTeX 表达式转为语义向量——FEX 可用此技术提高方程结构的表示质量
2. **PDE 分量分类体系**: domain-wise（方程符号、系数、边界类型）vs point-wise（外力场、几何域）——可指导 FEX 对不同类型项采用不同解码策略
3. **Conditional Transformer 机制**: scale/shift/select 三层调制——FEX 的 decoder 可用此机制根据 PDE 结构条件化符号序列生成
4. **数据集复用**: HeterNS、PDEformer 3M 样本——可从这些数据中提取方程符号和数值解形成 (solution, equation) 监督对供 FEX 训练

**差异化**: Unisolver 不做符号发现；FEX 不做数值求解。两者结合可形成"符号发现→快速验证→迭代优化"闭环。该论文对 0616-fex 的主要价值在于方法借鉴（嵌入、条件化）而非直接竞争。
