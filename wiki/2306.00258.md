# Towards Foundation Models for Scientific Machine Learning: Characterizing Scaling and Transfer Behavior (2306.00258)

- **Year**: 2023.06
- **Authors**: Shashank Subramanian (LBNL), Peter Harrington (LBNL), Kurt Keutzer (UC Berkeley), Wahid Bhimji (LBNL), Dmitriy Morozov (LBNL), Michael W. Mahoney (LBNL/ICSI/UC Berkeley), Amir Gholami (ICSI/UC Berkeley)
- **Venue**: NeurIPS 2023 (推断自 acmart sigconf 格式和主题)
- **Keywords**: foundation models, scientific machine learning, transfer learning, Fourier Neural Operator, PDE, scaling laws, neural operators, pre-training
- **One-liner**: 首次系统性验证了"预训练+微调"范式在 SciML 领域的可行性：FNO 预训练模型在下游 PDE 任务上能以数量级更少的数据达到同等精度，且收益随模型规模增大而增强。

## 精华提炼

### 管线/方法

完整的 pipeline：
1. **数据生成**：通过数值方法（谱方法）求解三类 PDE（Poisson、Advection-Diffusion、Helmholtz），在 128×128 网格上生成输入-输出对。输入包括源函数 f（高斯径向基线性组合，稀疏度 20%-80% 均匀采样）和 PDE 系数（扩散张量 K 的 eigenvalue e、角度 θ；速度向量 v；波数 ω）。所有输入变量同时采样以构造多样化数据集。
2. **输入归一化**：per-instance normalization——每样本源函数按训练集中位数归一化，系数再按全局中位数参考值归一化。这是关键组件，缺少时性能显著下降（系数跨度量纲可达 100× 差异）。
3. **模型架构**：Fourier Neural Operator (FNO)，核心超参数为 embedding 维度 d 和 Fourier mode 截断数 m。输入 tensor ∈ R^{h×w×c}，先 project 到 embedding 维度 d，通过 L 个 FNO block（点乘 + FFT → 频域线性变换 → IFFT → 非线性激活），最后 project 回输出。
4. **预训练**：Adam optimizer + cosine LR decay，500 epochs，4×A100 GPUs，ℓ₂ loss（MSE），best model 按最低 validation loss 保存。预训练数据量：2^15 (32K) 样本。
5. **下游适应**：zero-shot（直接评估）和 few-shot fine-tuning（O(10) 下游样本微调），与 from-scratch 训练对比。每个实验组合做 grid-search 超参调优（5 个 learning rates）。
6. **模型缩放**：固定 m 缩放 d ∈ {32, 128}，固定 d 缩放 m ∈ {4, 16}，参数从 64K 到 256M（每次 16×，共 4000× 范围）。

### 关键技术细节

- **PDE 系统**：
  - SYS-1 (Poisson): −div(K∇u) = f，扩散张量 K = R⁻¹·diag(1,e)·R，e 控制各向异性程度
  - SYS-2 (Advection-Diffusion): −div(K∇u) + v·∇u = f，引入 Ψ = ‖v·∇u‖/‖div(K∇u)‖ 量化 advection/diffusion 竞争比
  - SYS-3 (Helmholtz): −∇²u + ωu = f，ω 为波数，控制高频振荡程度
- **数据规模**：每系统 2^15 训练 + 2^12 验证 + 2^12 测试样本，128×128 分辨率
- **源函数分布**：高斯径向基函数线性组合 f(x) = Σᵢ φᵢ(x)pᵢ，σ=1/32（4× 网格分辨率），pᵢ ~ U(0,1)，sparsity s ∈ [0.2, 0.8]
- **FNO block 公式**：X^{l+1} = σ(W_l X^l + F⁻¹[K_l(F(X^l))])，K_l 在 Fourier 空间参数化为复数权重 Φ ∈ C^{d×d×m×m}
- **模型缩放方案**：m=4→16，d=32→128（分两阶段，每阶段 ~16× 参数增长），总数 4000× from 64K to 256M
- **归一化方案**：f_ref = median_i(‖f_i‖₂)，sample 级别归一化：f_i ← f_i / (‖f_i‖₂/f_ref)，系数级归一化：各系数除以训练集中位数参考值
- **评测指标**：μ_ℓ₂ = 1/N · ‖u_pred − u_true‖₂ / ‖u_true‖₂（mean relative ℓ₂ error）
- **硬件与训练时间**：4× NVIDIA A100 (Perlmutter)，完整预训练 ~2.5 小时
- **OOD 分级**：None / Mild / Med / Large 四档，通过 PDE 系数分布重叠程度量化（见表1）

### 核心结果

| 实验维度 | 关键发现 | 数字 |
|---------|---------|------|
| Q1: 下游数据缩放 | TL 数据效率远超 from-scratch | 128 examples TL ≈ 8K examples from-scratch（~100× 数据节省），SYS-1 Med OOD |
| Q2: 模型规模缩放 | 大模型从 fine-tuning 获益更多 | 64K 参数时误差饱和 > 1e-2；256M 参数时 TL 误差最低，且与 from-scratch 差距最大 |
| Q3: 物理外推 | 逐步 OOD 时 zero-shot 性能递减，fine-tuning 可恢复 | No OOD: zero-shot 最优，fine-tuning 无额外收益；Mild OOD: zero-shot 仍显著优于 from-scratch；Large OOD: few-shot TL 仍优于 from-scratch 但需要更多数据 |
| Q4: 多算子 | 单模型预训练于混合 PDE 可迁移到所有下游 PDE | Mixed pretraining 在 Poisson / AD / Helmholtz 三个下游任务上均保留 TL 增益，与单系统预训练性能相当 |
| 数据节省量化 | TL consistently 节省 O(1K-10K) 训练样本 | 除 SYS-1 Large OOD 外，所有 shift 级别均观察到显著数据节省 |
| Helmholtz 挑战 | Helmholtz 对波数极敏感，OOD 外推最难 | Mixed pretraining 仍与单 Helmholtz 预训练性能持平 |

- **Surprising/反直觉结果**：
  - 仅通过简单的零通道 padding 区分不同 PDE operator（无显式 token/embedding），模型就能学会"选择"正确的算子——说明 FNO 的 Fourier 空间表征已经隐式编码了算子结构
  - 在 "large target data" 区域（下游数据接近预训练数据量），TL 与 from-scratch 性能趋同——这与 NLP 的 scaling law 行为吻合
  - 输入归一化是"make-or-break"组件：不做归一化时性能极差（cross-channel scale 差异可达 100×）
  - 小模型（64K）完全饱和，增加数据和 fine-tuning 均无效——capacity bottleneck 占主导

### 明确局限

1. **单一架构**：仅研究 FNO，未涉及 ViT、DeepONet、GNN 等其他 SciML 架构。作者自己明确承认这是主要局限。
2. **限于 2D 空间系统**：未扩展到 3D 问题或时空耦合系统（Navier-Stokes 等）。
3. **仅限于稳态 PDE**：无 time-dependent PDE，无 transient dynamics。
4. **仅限于周期性边界条件**：FNO 的 FFT 机制天然适应周期边界，对 Dirichlet/Neumann 边界需要额外处理（文中提到但未实验）。
5. **预训练规模有限**：32K 样本的"预训练"远不及 NLP/CV 的 foundation model 规模（通常是 billions）。仅三个 PDE 系统的组合。
6. **无 self-supervision**：未使用 PDE 残差作为额外的无监督损失（物理信息 loss），而这在 PINN 文献中是常见手段。
7. **无 in-context learning 对比**：与同期工作 Yang et al. (2023) 的 in-context operator learning 是互补的，未做对比实验。
8. **未考察 memory/compute envelope**：缩放实验仅到 256M 参数，未做实际部署的延迟/吞吐量分析。
9. **随机种子实验仅在 Med OOD 场景**：5 trials 仅用于两个系统的中等 OOD，其他设置未报告方差。
10. **分布偏移量化粗糙**：仅用系数范围重叠程度定义 OOD 级别，缺少 Wasserstein distance 或更精确的分布距离度量。
