# Accelerating evolutionary exploration through language model-based transfer learning (2406.05166)

- **Year**: 2024.06
- **Authors**: Maximilian Reissmann, Yuan Fang, Andrew S. H. Ooi, Richard D. Sandberg (University of Melbourne, Dept. Mechanical Engineering)
- **Venue**: arXiv (IEEE journal format)
- **Keywords**: Gene Expression Programming, Transfer Learning, Symbolic Regression, Language Model, Transformer, Evolutionary Algorithm, Karva String
- **One-liner**: 用小参数量的 transformer 语言模型从历史 GEP 优化中学习子表达式模式，在新任务初始化时生成有偏好的候选解种群，在 UCI 和 CFD 问题上实现 15-36% 收敛加速。

## 精华提炼

### 管线/方法

完整 pipeline（Fig.1 五阶段）：

1. **源任务优化**：对源任务 T_s 运行标准 GEP，收集多轮优化中的近似解（不仅最后一代，而是整个探索过程的精英个体 + 接近最优解 15% 以内的个体），生成约 5×10^4 条训练序列
2. **语言模型训练**：将收集到的表达式按 Karva 字符串线性表示进行 tokenization（字符级），配合控制 token（`<link>` 等）和数值占位符（skeleton 方式，常数离散化为 {0.1, 0.5, -0.1, -0.5}），训练 encoder-decoder transformer
3. **目标任务表示**：给定新任务 T_t 的 (X, Y) 数据，通过 encoder 生成 latent vector z̃
4. **序列生成**：decoder 以 z̃ 为条件自回归生成 Karva 字符串，过程中用 mask vector（利用 GEP 的 head/tail 约束：t ≥ |H| 时非终结符概率 → -∞）保证句法合法性
5. **种群注入与进化**：将生成的个体按比例（0.1-0.75）混入初始种群，其余随机初始化，然后运行标准 GEP 优化

### 关键技术细节

**Encoder（PointNet/T-Net）**：
- 输入：(X, Y) 矩阵，维度 n × m（n=数据点数，m=特征数+1）
- 三层 1D Conv（d_emb → 2d_emb → 4d_emb），每层后接 BatchNorm + ReLU
- GlobalMaxPooling 压缩到 4d_emb，再经两层 FFN 压缩到 d_emb
- 输出：固定维度 d_emb 的 latent vector z̃
- 特点：对输入数据点顺序不变（permutation invariant）

**Decoder（Modified LLaMA）**：
- 基础架构：LLaMA 的改进版 transformer block（RMSNorm + SwiGLU + Rotary Embedding）
- 关键修改：在 self-attention 后增加 cross-attention 层（以 encoder latent z̃ 为条件）
- UCI 配置：d_emb=32, 8 attention heads, 8 transformer blocks, context window=64
- CFD 配置：d_emb=128, 12 transformer blocks, context window=256
- 总参数量极小（UCI 约几十万级别，CFD 约百万级别），远小于 GPT-3 的 4-30%

**Tokenization**：
- 字符级 tokenization，每个符号（+,-,*,/,sqr,sqrt, x1...xn, 常数占位符）是一个 token
- 数值离散化：只保留 {0.1, 0.5, -0.1, -0.5} 四个常数占位符（skeleton 方式），避免浮点数 token 爆炸
- 控制 token：`<link>` 分隔不同 gene，`<head>`/`<tail>` 标记结构
- 序列固定长度 64（UCI）或 256（CFD），不足补零

**GEP 参数**：
- 种群大小 100，head 长度 5-6（总长度 11-13）
- UCI: 3 genes, 最大 100 代, function set {+,-,*,/, sqr, sqrt}, 4 个常数 token
- CFD: 4×3 genes（tensor regression）, 最大 30 代, function set {+,-,*}
- Mutation 0.05, 1-point crossover 0.3-0.5, 2-point crossover 0.2-0.4

**训练配置**：
- UCI: 50 epochs, batch size 128, Adam + weight decay scheduler
- CFD: 500 epochs, batch size 256
- 硬件：NVIDIA A100（训练），32-core 2.0GHz CPU（进化搜索）
- 源任务：Feynman benchmark equation set（UCI），Re_τ=180 Ratio=1 duct flow（CFD）

### 核心结果

**定量结果（UCI 8 个数据集，25 runs average MAE）**：

| 数据集 | GEP | IGEP_0.25 (best) | 改善 | 备注 |
|--------|-----|------------------|------|------|
| Gas Turbine | 6.33 | 5.12 | ~19% | 最一致改善 |
| Power Plant | 5.61 | 4.56 | ~19% | 但有 0.75 比例反而变差 |
| Concrete | 9.18 | 7.10 | ~23% | 0.75 比例最佳 |
| Airfoil Noise | 10.99 | 7.35 | ~33% | 最大改善幅度 |
| Real Estate | 0.22 | 0.09 | ~59% | 方差大 |
| Energy Eff. | 1.17 | 1.16 | ~1% | 几乎无改善 |
| Yacht | 2.27 | 2.14 | ~6% | 0.5/0.75 反而变差 |
| Hardware | 22.05 | 21.91 | ~1% | 几乎无改善 |

**与 baseline 对比**：IGEP > GEP > gplearn > GP-GOMEA（部分 case）> DSR。GP-GOMEA 在 case 2,4 上明显优于所有 GEP 变体。

**收敛加速**：
- Case 1-2：标准 GEP 100 代达到的 fitness，IGEP 在 22-28 代就达到 → 节省约 6700-6300 次 fitness evaluation
- Case 4：标准 GEP 的 fitness 在 IGEP 约 72 代达到 → 节省 28 代

**CFD 结果**：
- 速度变化场景（Re_τ 180→360）：IGEP 初始种群优于 GEP 和 CGEP，最优情况达到标准 GEP 30 代也无法企及的解；最差情况与随机初始化持平
- 空间变化场景（Ratio 1→7）：IGEP 从一开始就显著优于 GEP（35%）和 CGEP（25%），但优化曲线趋于平缓（早期收敛到局部最优）
- 时间节省：30 代提前 × 140s/代 × 10 runs ≈ 4.2×10^5 秒（约 11 小时）

**Surprising Findings**：
- 25% 的注入比例普遍最优，而非更高比例——50% 和 75% 在某些 case 反而 worse than standard GEP（negative transfer）
- 语言模型生成的个体在初始种群中质量更高（Fig.3，中位数误差低 30-32%），但遗传算子（mutation/crossover）会破坏这些优质结构，导致优势随迭代衰减
- 8 维以上特征的外推效果急剧下降（case 3 中 75% 注入反而使中位数 fitness 从 126 升到 200）

### 明确局限

1. **维度扩展性差**：超过 8 个输入特征时性能显著下降，encoder 需要所有特征维度预先定义，新特征需 zero-padding（作者自己指出）
2. **遗传算子破坏性**：mutation 和 crossover 会随机破坏语言模型精心生成的结构，类似 Haslam 2016 的观察（作者自己指出）
3. **Negative transfer 风险**：当目标任务与源任务不相似时，注入个体反而有害；缺乏不确定性量化机制来决定何时不进行推理（作者自己指出）
4. **无真机实验**：所有实验均为数值仿真，CFD case 也是数值求解 RANS 方程，无物理实验验证
5. **模型规模极小**：d_emb=32/128 的 transformer 表达能力有限，可能无法捕捉复杂数学结构
6. **Skeleton 方式的局限**：常数离散化为固定 token 集合，无法表达任意常数值
7. **只做了 GEP**：没有在 GP、GA 等其他进化算法上验证通用性
8. **源任务依赖**：需要已有足够的源任务优化数据来训练语言模型，冷启动问题未解决
9. **未开源**：论文未提及代码/模型 release
10. **对比不够全面**：未与最新的 pre-trained SR 方法（如 end-to-end transformers）做直接对比，只在 table 中引用了别人的 DSR/gplearn/GP-GOMEA 结果

2. **可复用的技术组件**：
   - Karva string + mask 保证句法合法性：如果在 fex 中需要生成数学表达式形式的特征变换（如 `log(x1*x2 + x3)`），可以直接用这个技巧

4. **警示**：genetic operators 破坏学习到的结构这个发现很重要——如果在 fex 中也用进化搜索 + 预训练的组合，需要设计保护机制防止优质特征变换被 mutation/crossover 破坏。
