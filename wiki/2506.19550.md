# Discovering Symmetries of ODEs by Symbolic Regression (2506.19550)

- **Year**: 2025.06
- **Authors**: Paul Kahlmeyer, Niklas Merk, Joachim Giesen (Friedrich Schiller University Jena)
- **Venue**: Submitted to AAAI 2025
- **Keywords**: Lie point symmetries, symbolic regression, ODE solving, nonlinear dynamical systems, DAG-based regression, computer algebra systems, symmetry generators, Hamiltonian symmetry, canonical coordinates, multi-output regression
- **One-liner**: 用基于 DAG 的搜索式符号回归替代 CAS 代数方法，在 10 个非线性 ODE 系统上自动发现 Lie 点对称性生成元，Mathematica 的 SYM 包全败。

## 精华提炼

### 管线/方法

完整 pipeline 为四步（见图 flowchart.tex）：

1. **输入**：给定 ODE 系统 y' = f(t, y) 的符号形式 f。
2. **数值采样**：用 Runge-Kutta（RK45, Dormand & Prince 1980）从随机初始条件数值求解 ODE，产生时间序列数据集 D = {(t_i, y(t_i))}_{i=1,...,n}。每个系统用 3 条时间序列以保证可辨识性（n 维 ODE 需要 n+1 条时间序列以确认参数空间是 n 维的）。
3. **Loss 构建**：基于简化后的对称性条件（已排除 Hamiltonian 对称）构建 MSE loss。对候选生成元 η*(t, y): R^{d+1} -> R^d，在采样点计算每维的对称性条件残差 S^k_η*(t,y)，loss 为所有点所有维度的残差平方均值。
4. **符号回归搜索**：使用 Kahlmeyer 2024 的 UDFS（Universal DAG-based Function Search）框架搜索表达式 DAG，该框架先生成 DAG skeletons（无标签操作节点的图结构），再通过合法标注操作节点（+,-,×,÷,sin,cos,exp,log,sqrt 等）生成表达式 DAG 族。选 loss 最小的 DAG 对应的表达式作为输出生成元。自动微分通过 PyTorch 实现。丢弃 η* 中位绝对 < 0.01 的平凡解。

### 关键技术细节

- **核心数学简化**：每个 ODE 都有 Hamiltonian 对称 X_H = ∂_t + <f, ∇_y>（时间演化对称），但对求解无帮助。作者显式排除此对称：设 ξ=0，只搜索 η*(t,y)，其对应的生成元为 X = <η*, ∇_y>。简化后的对称性条件为：
  ∇η* · [1; f] - ∇f · [0; η*] = 0

- **多输出处理**：大多数符号回归器将多输出问题拆为单输出问题逐个解，而 UDFS 框架能直接处理多输出（d 维 ODE 需要输出 d 维向量 η*），这是选择 UDFS 的关键原因。

- **搜索空间控制**：搜索空间由 DAG 大小（节点数/边数）决定，按可用计算预算设定上限。搜索空间随 DAG 大小指数增长。

- **实验设置**：
  - 10 个手工构造的非线性 2D ODE 系统（ODE1-ODE10，包含指数、对数、三角函数、多项式等非线性项）
  - 数值求解用 RK45，初值范围和积分区间大多为 [1,2]（少数为 [-1,0]）
  - 硬件：Python 3.10, Intel Xeon Gold 6226R 64-core, 128 GB RAM
  - 每个问题跑 50 次独立实验取平均

- **对比基准**：Mathematica 的 SYM 包（Stylianos 2005）
  - SYM 自动模式：推导 PDE 约束，用 Mathematica 求解 → 成功 = 约束 PDE 可解
  - SYM-Z：交互模式，将所有但一个 component 设为零，求解剩下的
  - SYM-S：交互模式，将所有但一个 component 设为已知生成元的值（不现实的 oracle 模式）

- **评测指标**：(1) 是否找到生成元 (2) 数值 loss（MSE）(3) 符号验证（sympy 验证对称性条件符号恒为零）

- **额外特性**：独立生成元搜索。排除 Hamiltonian 后，多生成元独立意味着它们提供不同信息。用非线性系统 y1'=√(y1)·t, y2'=y1·y2·t 演示了两个独立生成元，分别导出不同维度约化 ODE，最终得到相同解 y1=t⁴/16, y2=exp(t⁶/96)。

### 核心结果

| 方法 | 成功数/10 |
|------|----------|
| SYM 自动模式 | 0/10 |
| SYM-Z（设一个未知，其余为零）| 1/10（仅 ODE4）|
| SYM-S（设一个未知，其余已知）| 7/10（ODE8,9,10 仍失败）|
| **本文方法（符号回归）** | **10/10** |

- 所有找到的生成元数值 loss 在 1e-13 到 1e-24 量级内，且 sympy 验证符号损失全为零。
- 运行时间：1-6 分钟/问题，生成元表达式越简单越快。生成元表达式树大小在 3-11 节点之间。
- Surprising：即使是 oracle 模式的 SYM-S（已知除一个 component 外所有的生成元），Mathematica 仍无法求解 3/10 个系统的剩余 PDE 约束。说明 CAS 的代数 solvers 在面对某些非线性偏微分方程时本质上有盲区。

### 明确局限

- **作者承认的局限**：
  1. 搜索空间随 DAG 节点数指数增长，高维 ODE 系统很难（目前仅演示 2D 系统）
  2. 即使是困难生成元也只能近似为简单表达式（"even difficult generators can still be approximated by simpler expressions"）
  3. 可辨识性问题：从有限时间序列识别 ODE 需要足够多的初始条件，但这依赖于用户提供

- **读出来的局限**：
  1. **没有标准 benchmark**：10 个 ODE 是手工构造的，无公开标准测试集，难以与其他方法公平比较
  2. **只和 SYM 比**：仅对比 Mathematica SYM，未对比 Maple symgen（不能处理 ODE 系统）或 MathLie（需人工交互）。缺少与其他数值/ML 方法的直接对比（如 AI Poincare 2.0 虽主要找守恒量，但有重叠）
  3. **仅 2D ODE**：所有实验在 2 维 ODE 上，没测试更高维系统
  4. **没有与 NN-based 方法比较**：AI Poincare / Bondesan 2019 等用 NN 学习坐标变换再符号化，本文只用纯符号回归，缺少两路线的对比分析
  5. **只找点对称**：仅处理 Lie 点对称（point symmetries），不处理更一般的 contact、generalized 或 non-local 对称
  6. **计算效率瓶颈**：DAG skeleton 生成 + 标注 + 评估的成本随 DAG 大小指数增长，难以扩展到大规模系统
  7. **没有真机/实际物理系统**：所有例子是人工构造的数学表达式，无真实物理/生物/工程数据测试
  8. **不包含预测/prediction evaluation**：只验证了生成元满足对称性条件，未演示如何用找到的生成元实际降阶求解（仅 supplement 中有两个演示）
