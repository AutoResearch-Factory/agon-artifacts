# Transformer-based Planning for Symbolic Regression (2303.06833)

- **Year**: 2023.03 (NeurIPS 2023)
- **Authors**: Parshin Shojaee, Kazem Meidani (equal contribution), Amir Barati Farimani (CMU), Chandan K. Reddy (Virginia Tech)
- **Venue**: NeurIPS 2023
- **Keywords**: symbolic regression, Monte Carlo tree search, transformer decoding, planning, fitting-complexity tradeoff, equation discovery, pre-trained SR models, non-differentiable feedback
- **One-liner**: 用 MCTS 做预训练 Transformer SR 模型的解码策略，将 fitting accuracy + complexity 的非可微反馈注入方程生成过程，显著超越 beam search/sampling baseline。

## 精华提炼

### 管线/方法

TPSR 的整体 pipeline 如下：

1. **输入**: 观测数据集 D = {(x_i, y_i)}，预训练好的 Transformer SR 模型（如 E2E）
2. **方程生成建模为 MDP**: state = 当前生成的 token 序列（可能是不完整方程），action = 从词表中选择下一个 token，terminal = 生成 EOS token
3. **MCTS 四步循环**（每一步解码一个 token 位置时调用 MCTS 做 lookahead planning）:
   - **Selection**: 使用 P-UCB（融合了预训练 Transformer logits 的 UCB 变体）选择要探索的节点: `P-UCB(s,a) = Q(s,a) + β · P_θ(a|s) · √(ln(N(s))/(1+N(s')))`
   - **Expansion**: 不是随机展开词表，而是仅用 Transformer 的 **top-k logits** 作为候选 action（k_max=3），避免无效方程，大幅缩小搜索空间
   - **Evaluation**: 不用随机 rollout，而是调用 Transformer 的 **beam search** (beam size b=1) 从当前 partial sequence 生成完整方程，对完整方程计算 reward
   - **Backpropagation**: 将 reward 沿路径回传，更新 Q 值：Q(s',a') ← max(Q(s',a'), r)
4. **MCTS 完成后**: 返回搜索过程中 reward 最高的方程
5. **常数优化**: 生成的方程经过 BFGS 优化其中的数值常数

关键设计：Transformer 和 MCTS 的信息共享发生在两个地方——expansion 阶段用 Transformer top-k 限制搜索空间，evaluation 阶段用 Transformer beam search 代替随机 rollout。这使得 MCTS 不需要从零搜索，而是在 Transformer 学到的 prior 基础上做精细化探索。

### 关键技术细节

**Reward 函数设计**:
```
r(f̃|x,y) = 1/(1+NMSE(y, f̃(x))) + λ·exp(-l(f̃)/L)
```
- 第一项: fitting accuracy，NMSE = (1/n · ||y - f̃(x)||²) / (1/n · ||y||² + ε)
- 第二项: complexity penalty，l = prefix notation 序列长度，L = 最大序列长度 (200)
- λ: 控制 fitting vs complexity 的权衡参数，推荐默认 λ=0.1
- ε = 1e-9 防止数值不稳定

**MCTS 超参数**:
- k_max = 3（每个节点最多展开 3 个子节点）
- r = 3（rollout 次数）
- b = 1（simulation 时的 beam size）
- β = 1（P-UCB 中的探索-利用平衡参数）
- λ ∈ {0, 0.1, 0.5, 1.0}（推荐 0.1）

**Backbone 模型**:
- 主要基于 E2E (Kamienny et al., NeurIPS 2022)：端到端预测完整方程（含常数），d_max=10，N_max=200
- 也验证了 NeSymReS (Biggio et al., ICML 2021) 作为 backbone 的效果

**Caching 机制**:
- **Top-k caching**: 缓存已计算的 top-k logits，避免重复前向传播
- **Sequence caching**: 当 b=1 时，缓存完整的贪心生成序列；如果后续 MCTS 迭代中某个 state 是已缓存序列的前缀，直接复用
- 两种 caching 合计可减少约 28% 推理时间

**Bags 策略**: 对超过 200 点的数据集（如 SRBench），分成 B 个 bags（每个 200 点），迭代解码：从第一个 bag 开始 MCTS，若 R²>0.99 即停止，否则继续下一个 bag，最多 B=10 个

**硬件**: 4x Quadro RTX 8000 GPUs (48GB)

### 核心结果

**SRBench 主表（TPSR λ=0.1 vs E2E+Sampling）**:

| 数据集 | 指标 | TPSR(λ=0.1) | E2E+Sampling | 提升 |
|--------|------|-------------|-------------|------|
| Feynman (119) | R²>0.99 | 0.949 | 0.848 | +10.1% |
| Strogatz (14) | R²>0.99 | 0.785 | 0.357 | +42.8% |
| Black-box (57) | median R² | 0.945 | 0.864 | +8.1% |
| In-domain (400) | R²>0.99 | 0.708 | 0.640 | +6.8% |

- λ=0 时 fitting 最好但 complexity 爆炸（Feynman: complexity 84.42 vs 50.73）
- λ=1 时 complexity 最低（47.24）但 fitting 仍显著超越 baseline（0.916 vs 0.848）
- **TPSR 达到了 SRBench 的第一 Pareto 前沿**（fitting-complexity 权衡）

**NeSymReS backbone 验证**: 在 52 个 Feynman 方程（D≤3）上，NeSymReS+TPSR(λ=0.1) 的 R²>0.99 从 0.635 提升到 0.808，complexity 不变（9.98）

**Extrapolation**: λ>0 时外推性能显著优于 λ=0（后者因过拟合在 σ>8 时性能下降）

**Noise robustness**: MCTS 引导的解码在噪声下性能下降幅度小于 baseline

**Inference time**: 比 E2E 慢（~100-300s vs ~10s per equation），但比 GP-based 方法快 1-2 个数量级（Operon ~3hrs, SBP-GP ~30hrs for d=9）

**Surprising/反直觉发现**:
- 在 Strogatz（ODE 时序数据）上提升最大（+42.8%），尽管 E2E 没有在时序数据上预训练
- 低维问题（d=3）适合大 λ，高维问题（d=8,9）适合小 λ——与直觉一致但量化了这种关系
- In-domain 数据集上提升相对小，因为 E2E 对这些数据已有很好的 prior（训练分布内）

### 明确局限

1. **推理时间增加**: 比简单 beam search/sampling 慢 10-30 倍，虽然比 GP 方法快
2. **强依赖预训练模型**: TPSR 的性能上限受限于 backbone 模型的 prior 质量——如果 backbone 不好，MCTS 也救不了
3. **结构约束继承**: 受限于 backbone 的输入维度（d≤10）、序列长度（L=200）、词表定义
4. **不更新模型权重**: MCTS 只在推理时工作，不 fine-tune backbone——对 out-of-distribution 数据的适应能力有限
5. **没有用 LLM**: 使用的是专用预训练 SR transformer（E2E/NeSymReS），不是通用 LLM

### 跟我们的关系

**对 llm-mcts-sr 课题的关键意义**:

- **直接相关**: 这是 MCTS + Transformer SR 的开创性工作之一，是 llm-mcts-sr 课题的核心 baseline
- **方法可借鉴**: P-UCB 结合 Transformer logits 的设计、fitting+complexity 双目标 reward、top-k/sequence caching 都是成熟可复用的组件
- **差异化空间**: TPSR 用的是专用 SR transformer（E2E），不是 LLM；如果用 LLM（如 GPT/Claude）做 SR 并结合 MCTS，需要解决 LLM 输出空间更大、方程表示不同、prompt 设计等新挑战

- **改进方向**: (1) 用 LLM 替代专用 SR transformer 做 backbone；(2) MCTS 中加入物理约束作为 reward；(3) 在 MCTS 过程中 fine-tune 模型权重；(4) 并行 MCTS 提高效率

## Read by: llm-mcts-sr (2026-06-05)

本文是 MCTS + Transformer SR 的代表性工作（NeurIPS 2023），对 llm-mcts-sr 课题高度相关。

**撞车评估**: 如果 llm-mcts-sr 的方向是"用 LLM 替代专用 SR transformer 做 backbone，结合 MCTS 做方程搜索"，则与 TPSR **不直接撞车**但属于同一技术路线的延伸。TPSR 用的是 E2E/NeSymReS 这种专用预训练模型，而非 GPT/Claude 这类通用 LLM。差异化的关键在于：(1) LLM 的搜索空间组织方式不同（natural language vs prefix notation）；(2) LLM 具有更强的 few-shot/in-context 能力但缺乏专用 SR 训练；(3) LLM 可能需要完全不同的 reward 设计和 MCTS 结构。

**可用工具/方法**: P-UCB 公式（将 Transformer logits 融入 UCB）、fitting+complexity 双目标 reward 设计（1/(1+NMSE) + λ·exp(-l/L)）、top-k caching 和 sequence caching 的工程实现、SRBench 评测框架和标准 benchmark。代码开源于 GitHub: deep-symbolic-mathematics/TPSR。

**差异化方向**: (1) 用 LLM 的 in-context learning 能力取代预训练，无需专门训练 SR 模型；(2) 引入物理先验/维度分析作为 MCTS reward 的额外信号；(3) 让 LLM 以更灵活的方程表示（如 LaTeX 或 Python 代码）生成方程，不局限于 prefix notation。
