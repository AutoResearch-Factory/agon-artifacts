# The Need for Verification in AI-Driven Scientific Discovery (2509.01398)

- **Year**: 2025.09
- **Authors**: Cristina Cornelio (Samsung AI, Cambridge), Takuya Ito (IBM Research), Ryan Cory-Wright (Imperial Business School), Sanjeeb Dash (IBM), Lior Horesh (IBM)
- **Venue**: arXiv preprint (not yet peer-reviewed; 139 references)
- **Keywords**: AI-driven scientific discovery, verification bottleneck, symbolic regression, neuro-symbolic AI, scientific method, derivable models, theorem proving, LLM
- **One-liner**: A position paper arguing that without rigorous verification mechanisms, AI-generated hypotheses will overwhelm rather than accelerate scientific discovery; surveys the full pipeline from data-driven to derivable models.

## 精华提炼

### 管线/方法
本文是一篇综述/观点文章（非原创方法论文），核心 argument 是：
1. **输入**：当前 AI（ML + LLM）可以前所未有地快速产生科学假设（hypothesis generation）
2. **瓶颈**：验证（verification）管道跟不上生成速度，导致 unverified hypotheses 堆积
3. **输出**：呼吁将验证作为 AI 辅助科学发现的基石，需要 automated verification 机制

论文沿三个维度组织方法图谱（Fig.2）：数据驱动的程度、知识驱动的程度、计算复杂度。核心分类：
- **Knowledge-Aware** (中等验证性)：PINNs, Equivariant Networks, physics-inspired networks
- **Derivable Models** (高验证性)：AI-Descartes (generator-verifier), AI-Hilbert (theory-guided synthesis)
- **LLM/Foundation Models** (验证性可强可弱)：LLM-SR, ChemCrow, SciAgents, Galactica, NatureLM

### 关键技术细节
- **AI-Descartes**：Generator-Verifier 框架。SR（mixed-integer nonlinear programming）产生候选公式，Theorem Prover 根据 background theory axioms 验证。提出 "reasoning distance" 概念衡量候选公式与理论可推导公式之间的差异。输入为四元组 (B, C, D, M) = (背景公理, 假设类, 数据集, 建模偏好)
- **AI-Hilbert**：将数据+理论统一为单一合成问题。假设空间限制为多项式/有理式。4步流程：(1) 构建多项式优化问题 Pr，(2) 用 SOS optimization 转化为半定规划 Pr^{sd}，(3) 混合整数锥优化求解，(4) 输出候选公式 + 可推导性证书
- **PINNs**：损失函数 = lambda_d * L_data + lambda_f * L_physics + lambda_b * L_boundary
- **HNNs**：学习 H(q,p) 标量函数，通过 dq/dt = dH/dp, dp/dt = -dH/dq 导出动力学
- **Symbolic Regression**：PySR (evolutionary), AI Feynman (physics-inspired heuristics + neural fitting), RSRM (MCTS + RL)
- **RLVR (RL with Verified Rewards)**：只验证最终答案不验证中间推理步骤；spurious rewards 问题（负奖励也能提升 benchmark）；性能与 base model 强相关
- **Formal Proof Assistants**：Lean (dependently typed), Coq, Isabelle；但无自然科学通用公理集，翻译 informal→formal 易出错

### 核心结果
本文为纯综述无实验数据。核心 argument 证据链：
1. 历史上验证失败导致严重科学错误：Adams (1925) 引力红移误测、砷基生命假说 (2011) 被撤稿、催化剂合成计量错误导致撤稿
2. 当前 LLM 的可靠性问题：RLVR 不能提供逻辑验证保证（参见 Kambhampati 2025 / Stechly 2025 / Shao 2025），LLM 产生错误推理步骤到达正确答案
3. 附录中的 GPT-5 实验：GPT-5 在 binary star 数据上做 symbolic regression 失败（产生多种候选但未找到 Kepler 第三定律），但在简单公理推导任务上 GPT-5 胜 GPT-4
4. 核心观察：RL 应用到 LLM 只能改进 "surface reliability"，不能解决 "principled, automated logic verification against background theory" 的需求
5. 各科学领域的验证策略不同：物理学靠演绎+受控实验，生物学靠本体论框架+手动实验，临床科学靠统计推断+随机对照试验

### 明确局限
- **读出来的局限**：
  - 完全是立场文章，无方法贡献、无代码、无数据集、无实验
  - AI-Descartes/AI-Hilbert 仅限于物理科学中的特定问题表述，不能泛化到其他领域
  - "可推导性"(derivability) 概念强烈依赖背景理论 B 的完整性和一致性，但作者承认大多数科学领域的理论是 incomplete/inconsistent/ontological 而非 formal
  - 讨论范围主要在 symbolic/physical science 领域，对机器学习特征工程的直接指导有限
  - 没有讨论如何将验证集成到 LLM/深度学习 pipeline 中的具体工程方案
