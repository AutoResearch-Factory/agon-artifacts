# Parsing the Language of Expression: Enhancing Symbolic Regression with Domain-Aware Symbolic Priors (2503.09592)

- **Year**: 2025.03
- **Authors**: Sikai Huang, Yixin Wen, Tara Adusumilli, Kusum Choudhary, Haizhao Yang; Purdue University, University of Florida, University of Maryland College Park
- **Venue**: arXiv
- **Keywords**: symbolic regression, finite expression method, tree-structured RNN, domain-aware priors, KL regularization, reinforcement learning, formula constraints, scientific discovery
- **One-liner**: 用领域符号先验提升树 RNN 符号回归效率与鲁棒性。

## 精华提炼

### 管线/方法

1. 论文目标是把来自不同科学领域的数学表达式统计规律显式注入 symbolic regression，而不是只依赖统一的 operator set 或人工规则。
2. 输入包括待拟合数据 `{X, y}`、一个固定表达式树结构 `T`、候选 operator 字典、以及从领域表达式语料中提取的符号先验。
3. 语料侧输入来自 arXiv 中 physics、biology、chemistry、engineering 等领域论文的数学表达式；每个领域声称选取 10,000 篇相关论文抽取公式。
4. 每个抽出的表达式被转换为作者提出的 general multi-branch tree，而不是传统 strictly binary expression tree。
5. 该表示允许一个 binary operator 连接多个 unary-operator sequences，尤其适合连续加法、乘法和多项并列项。
6. 树的 root 是 unary operator，例如 `Id`, `log`, `exp`, `sin`, `cos`, `tan`, `sqrt`, `square`。
7. 树中 binary operator set 为 `{+, *, /}`，主要作为多个 unary sequences 的组合节点。
8. leaf node 输入变量集合包括 `f`, `x_i`, 一阶偏导 `f_{x_i}` 和二阶偏导 `f_{x_i x_j}`。
9. leaf 输出不是单变量，而是对变量集合逐元素应用同一 unary operator 后的线性组合：`sum_i gamma_i mu(v_i)`。
10. 非叶 unary operator 带 affine transform：`alpha * mu(I) + beta`，增加表达能力。
11. 作者定义 subsequence 为 root-to-leaf 路径上的 unary/binary operator 序列。
12. 作者定义 width 为第一层 unary sequences 的数量。
13. 作者定义 depth 为最长 root-to-leaf subsequence 中 unary operator 的数量。
14. 先验提取分为 vertical priors、horizontal priors、domain-specific components 和其他结构先验。
15. vertical prior 估计某节点符号在 parent、level、ancestor/path 上下文下的条件分布。
16. horizontal prior 估计同一 binary parent 下 sibling unary operators 的条件共现分布。
17. 对 sibling context，作者最多记录三个 sibling；超过三个时只保留最常出现的三个，以控制组合爆炸。
18. 对表达式内变量数造成的重复计数偏差，作者用 normalized count：单表达式内 raw count 除以变量总数，再跨表达式平均。
19. domain-specific components 指工程中的 `cos(.) + sin(.)`、化学中的 `exp(. / .)` 等高频结构块。
20. 这些结构块被加入 operator dictionary，作为更粗粒度的 building blocks 来加速搜索。
21. 生成侧核心 agent 是 tree-structured RNN policy。
22. 每个树节点输出一个 categorical distribution，用于采样该节点 operator。
23. parent hidden activation 会传给所有 child nodes。
24. sibling 之间也有横向信息传递，用于建模 sibling dependency。
25. sibling sequential sampling 让后一个 sibling 的分布条件依赖前一个 sibling 和 parent。
26. 训练目标是 reinforcement learning 形式，先采样 operator sequence `e`，再优化连续参数 `theta={alpha,beta,gamma}`。
27. 连续参数优化沿用 FEX 风格的 alternating optimization，coarse tune 用 Adam 与 BFGS，fine tune 再对候选池做更细优化。
28. reward 定义为 `R(e)=1/(1+L(e))`，其中 `L(e)=min_theta NRMSE`。
29. policy update 用 risk-seeking policy gradient，只强化高于 reward 分位阈值的样本。
30. domain prior 作为 soft constraint 进入 KL regularization：最小化 `KL(P*(s_i | S_i, s_p, h) || y_i)`。
31. 总 objective 是 reward 减去平均 KL 项，KL 权重记为 `ell`。
32. 作者还设计 dynamic scheduling，使 `ell` 随训练指数衰减：`ell_0 * exp(-lambda_d t)`。
33. 早期较大 KL 权重让搜索贴近领域先验，后期衰减以允许数据驱动地偏离先验。
34. hard constraints 来自 General Formulation Rules 和领域中几乎不出现的组合。
35. `HC_1` 是数值不稳定或违反符号规则的组合，采样时直接 mask 到 0。
36. `HC_2` 是很罕见但未必非法的组合，保留极小概率 `epsilon`，避免完全封死探索。
37. 整体输出是最佳表达式结构和连续参数 `(T*, theta*)`。

### 关键技术细节

1. 树表示重点解决 binary tree 对连续加法造成的深度增长和不平衡问题。
2. `Id` 可以出现在 root、leaf 和 sequence 中，但在 sequence 中只允许单独出现，避免无意义拉长表达式。
3. 公式语料的领域包括 physics、biology、chemistry、engineering。
4. 每个领域声称抽取 10,000 篇 arXiv 相关论文中的公式，但论文没有给出可复现的检索 query、解析器、清洗规则或抽取脚本。
5. 结构先验图显示：physics depth 约 4-6 常见，width 约 2-5 主导。
6. biology depth 约 3-5 更常见，width 主要 2-4，超过 5 的概率明显下降。
7. chemistry depth 分布约 4-7，width 在 3、4 附近有峰值。
8. engineering depth 约 5-7 概率较高，width 约 3-6 有峰值。
9. root node distribution 中 `Id` 在四个领域都是最高概率 root，大约在 0.28-0.34 范围。
10. engineering 的 `exp` root 概率相对较高，图中约 0.15。
11. chemistry 中 `log`, `exp`, `sqrt`, `square` 分布更均衡，符合反应速率、平衡和分子性质建模。
12. vertical prior 图显示，在多种 root 下 `+` 的条件概率通常最高，其次是 `*`，`/` 最低。
13. engineering 的 root=`exp` 情况下 `+` 和 `*` 都明显，而 `/` 很低。
14. horizontal prior 图在 parent=`+`, first sibling=`exp` 时显示 second sibling 最常为 `exp` 或 `Id`。
15. physics 中 `exp + exp` 与 `exp + Id` 都高，`sin`、`cos` 小但非零。
16. biology 中 `exp + exp` 最高，`exp + Id` 次之，其他几乎为零。
17. chemistry 中 `exp + exp` 与 `exp + Id` 主导，`square` 有小概率。
18. engineering 中 `exp + exp`、`exp + Id` 主导，`sqrt` 和 `square` 有小概率。
19. benchmark 使用 Feynman Benchmark for Physics 和 ODEbase for Biology。
20. Feynman benchmark 每个 equation 使用 10,000 data points。
21. ODEbase 选取 200 representative biological ODE expressions。
22. recovery criterion 是生成表达式与 target expression 的差可化简为常数。
23. 论文正文前后出现一次 10 seeds、一次 100 seeds 的说法；最终实验段落写的是每个 benchmark expression 重复 100 random seeds。
24. 噪声水平为 `0`, `0.01`, `0.05`, `0.07`, `0.1`。
25. 对比方法包括 RNN, RNN+prior, Tree-RNN, Tree-RNN+prior, FEX, FEX+prior。
26. 四个 domain case 包括 Fluid Dynamics、Evolution of cell populations、Reaction rate equation、Hamiltonian expression。
27. 前两个 case 的 learning rate 为 0.003，batch size 1000，risk factor 0.05，KL parameter 0.5。
28. 后两个 case 的 learning rate 为 0.001，batch size 1000，risk factor 0.05，KL parameter 0.35。
29. appendix 给出 Hamiltonian, biology tumor-cell ODE, chemistry reaction-rate equation, engineering fluid-dynamics inspired deep function 四个表达式。
30. Hamiltonian case 使用 `A_hat=2.0`, `m_N=1.5`, `g=0.8`。
31. biology case 参数包括 `b=0.5`, `gamma_B=0.1`, `mu_B=0.3`, `K2=1.0`, `d_B=0.05`, `s=2.0`, `k=0.8`, `omega_B=0.2`。
32. chemistry case 使用 `Vmax=1.0`, `Km=0.5`, `Ki=0.3`。
33. engineering case 使用 `alpha=1.2`, `beta=0.8`, `gamma=2.0`, `delta=0.5`, `epsilon=0.1`, `eta=1.5`, `theta=0.3`, `zeta=0.05`, `lambda=0.01`。

### 核心结果

1. benchmark 图中 Tree-RNN+prior 在 biology 和 physics 两个 benchmark 上几乎全噪声水平最高。
2. biology benchmark 在 noise=0 时 Tree-RNN+prior 约 82%，RNN/FEX/prior variants 大约 75%-81%。
3. biology benchmark 在 noise=0.1 时 Tree-RNN+prior 约 59%，RNN 约 45%，FEX 约 46%，Tree-RNN 约 49%，RNN+prior/FEX+prior 约 56%-57%。
4. physics benchmark 在 noise=0 时各方法都在约 82%-84%，Tree-RNN+prior 略高。
5. physics benchmark 在 noise=0.1 时 Tree-RNN+prior 约 49%，RNN 约 35%，FEX 约 37%，Tree-RNN 约 39%，RNN+prior 约 46%，FEX+prior 约 44%。
6. 趋势上，噪声越高，prior 和 tree structure 的优势越明显。
7. RNN 和 FEX 无 prior 版本对噪声更敏感，recovery rate 下降更快。
8. 四个 case study 的 average reward 曲线显示 Tree-RNN with prior 通常最快到达接近 1.0 的 reward。
9. Reaction rate equation 中 Tree-RNN with prior 大约 60-65 iterations 到达接近 1.0，是最清楚的加速案例。
10. Hamiltonian expression 中 Tree-RNN with prior 大约 35 iterations 到达接近 1.0，RNN+prior 约 60 iterations，FEX+prior 约 110 iterations，Tree-RNN 无 prior 接近 180 iterations。
11. Evolution of cell populations 中 Tree-RNN with prior 约 80 iterations 到达 1.0，RNN+prior 约 90 iterations，FEX+prior 后期也接近 1.0。
12. Fluid Dynamics 是作者承认的弱点：带 prior 的方法未显著压过其他方法，说明先验可能和复杂表达式不匹配。
13. 作者最重要的实证 claim 是：domain priors 与 tree-structured RNN 互补，能同时改善收敛速度与高噪声 recovery robustness。
14. 最值得保留的反直觉发现是：prior 并不总是有利，在复杂 fluid dynamics case 中 bias 会限制表现。
15. 论文没有给出逐方程的数值表、置信区间、统计显著性检验或失败案例列表；核心结果主要靠图展示。

### 明确局限

1. 最大可复现性问题是公式语料构建过程不透明：没有公开 query、parser、cleaning pipeline、expression normalization code。
2. "每领域 10,000 篇 arXiv 论文" 是很强的数据声明，但文中没有列出领域分类标准和去重规则。
3. 先验分布图有概率柱状图，但没有公开原始计数、样本量、方差或 bootstrap uncertainty。
4. benchmark 结果主要以曲线图呈现，缺少可审计的 per-equation recovery table。
5. 论文正文对随机种子次数有不一致表述：一处说 10 次，一处说 100 random seeds。
6. 对比方法实现细节不足，尤其是 RNN/FEX baseline 是否同等调参、是否共享 continuous optimizer。
7. FEX+prior 的具体 prior 注入方式不如 Tree-RNN+prior 讲得清楚。
8. tree structure 被假设固定，但如何选 tree、如何扩展到可变 tree search 没有完全展开。
9. hard constraint 与 rare-but-allowed soft constraint 的阈值选择没有系统消融。
10. KL schedule 的 grid search 范围没有给出。
11. 没有代码链接、artifact、环境说明或训练时间统计。
12. 没有真机或真实实验平台问题；主要是标准公式 benchmark 与 synthetic/domain-inspired equations。
13. 没有跟 PySR、AI Feynman、DSO 等主流库做充分现代横向比较，只覆盖了论文选择的六个 variants。
14. 没有分析公式抽取误差如何传播到 prior，再影响 downstream symbolic regression。
15. 对 <topic_slug>=0616-fex 来说，这篇更像 FEX/RL symbolic regression 的 prior-augmentation，而不是 FEX PDE solver 本身的数值分析论文。

## Read by: 0616-fex (2026-06-16)

这篇论文对 0616-fex 的关系很近，但撞车范围要精确切分。它不是新的 FEX PDE solver，也没有解决 FEX 在复杂 PDE、oscillatory solution、complex domain 或 eigenvalue problem 上的数值表达能力；它做的是 symbolic regression search policy，把从 arXiv 公式语料中挖出的 horizontal/vertical operator priors 注入 tree-RNN 和 FEX-style alternating optimization。若我们的题目主张 "FEX 加领域符号先验提升恢复率"，这篇已经覆盖了大半 claim，必须调整为更可审计的 prior extraction、uncertainty-aware prior、task-adaptive prior 或 PDE-specific prior。
