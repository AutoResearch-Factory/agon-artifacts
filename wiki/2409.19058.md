# CLLMate: A Multimodal Benchmark for Weather and Climate Events Forecasting (2409.19058)

- **Year**: 2024.09 (v1; 后续版本补入 Gemini-2.0-flash / Qwen2.5-VL / Janus-Pro 等 2025 模型)
- **Authors**: Haobo Li, Zhaowei Wang, Jiachen Wang, YueYa Wang, Alexis Kai Hon Lau, Huamin Qu (HKUST)
- **Venue**: arXiv preprint (ACL 模板, preprint 模式)
- **Keywords**: multimodal benchmark, weather and climate event forecasting (WCEF), ERA5, MLLM evaluation, news-derived events, expert annotation, visual instruction tuning, LLaVA fine-tuning
- **One-liner**: 首个新闻事件×ERA5 对齐的天气气候事件预报 (WCEF) 多模态基准; 23 个 MLLM 多选评测均 <50% 准确率, 微调 7B 反超闭源大模型。

## 精华提炼

### 管线/方法

提出 WCEF 任务: 从数值气象栅格数据预测**文本化**的天气气候事件及其级联后果, 形式化为 `E_{C:C+F-1} = f_φ(R_{C-H+1:C}, Instruction)`。注意 C 同时出现在输入与输出侧 —— 实际实现是**当日气象场 → 当日事件**的映射 (检测/诊断), 并非带 lead time 的预报。

数据集构建管线 (CLLMate, 7,747 事件):

1. **新闻源**: 从 Wisers 商业采购 26,156 篇环境新闻 (2015.07–2023.06, 含标题/正文/日期/出版方/媒体类型)。
2. **事件抽取**: GPT-4o-mini + zero-shot CoT + few-shot, 输出五元组 [subject, 'cause', object, location, time]; 自动标出 6,352 篇含潜在事件。
3. **专家核验**: 3 名气象领域专家 (博后/博士生) 对每篇抽取结果二元判定, 多数票定案。仅 2,575 篇 (40.54%) 通过 → 7,747 条时空落地事件。IAA 成对一致率 81.83%, Fleiss κ=0.63。
4. **地理编码**: GPT-4o-mini 为 1,049 个地区生成经纬度包围盒, 人工在地图上目检校验; 范围至少外扩 5°以纳入周边气象条件。
5. **层级分类体系**: 自底向上语义聚类 + 自顶向下领域分类法结合, 两大类: 气象现象 (3,979 事件; 3 个一级/8 个二级: 大气类高温/低温/低能见度, 风类一般风/龙卷/台风, 降水类固态/液态) 与后果 (3,768 事件; 5 个一级/18 个二级: 灾害/生物影响/损毁/社会影响/交通影响)。
6. **事件–气象对齐**: ERA5 single-level 4 变量 (2m 温度, 10m u/v 风, 总降水), 逐时数据 2015.07–2023.07, 空间分辨率 27.75 km; 取事件日期+区域裁剪, 编码为归一化 RGB 伪图像 (R=温度, G=风速模, B=降水) + 区域统计量 (各变量 max/min/mean)。
7. **评测格式**: 多选题。Prompt = System + 位置信息 + 统计量 + RGB 图 + 选项 (跨样本随机化次序防位置偏差) + 三步 Zero-Shot CoT (统计分析→空间格局分析→综合), 输出限定为选项编号。
8. **切分**: 按时间顺序 90/10 训练/测试 (测试集仅 775 事件: 现象 492 + 后果 283)。

### 关键技术细节

- **被测模型 23 个**: 闭源 6 (GPT-4o-mini/GPT-4o/GPT-o1, Gemini-1.5-pro/2.0-flash, Claude-3.5-Sonnet); 开源 12 (Janus-Pro-7B, InternVL2-8B, Cambrian-1-8B, LLaVA-1.5-7B/13B, LLaVA-1.6-vicuna-7B/13B/34B, LLaVA-1.6-mistral-7B, Qwen2-VL-7B, Qwen2.5-VL-7B/72B); 微调 5 (LLaVA-1.5-7B/13B, LLaVA-1.6-vicuna-7B/13B, LLaVA-1.6-mistral-7B)。
- **微调配置**: CLIP ViT-L/14 视觉编码器 + LLaVA 两层 MLP 适配器均冻结, 只调 LLM; DeepSpeed ZeRO-3, 8×A800, lr 2e-5, batch 16, 1 epoch (7B 1 小时 / 13B 2 小时)。
- **指标**: 四个子任务的 accuracy (现象一级/二级, 后果一级/二级)。
- **启发式基线**: 随机猜 33.33/12.50/20.00/5.56; 多数类猜 (训练集最频类: 液态降水、洪水) 40.12/26.07/40.11/4.90 —— 类不平衡使多数类基线非常强。
- **数据分布**: 空间集中于东亚/北美/欧洲 (新闻报道基础设施差异); 时间上夏冬双峰, 2021.07 郑州特大暴雨造成显著离群峰。现象类内部不平衡: 大气 43.7%, 降水 40.5%, 风 15.8%。
- **数据格式**: 每事件 JSON 含因果链指针 (caused by / cause)、news_id、事件文本、层级类目、日期、地点、包围盒、RGB 图路径、三变量统计。CC BY 4.0 (承诺审后释出; ERA5 为 Copernicus 许可, 新闻原文来自商业源)。

### 核心结果

| 模型 (代表) | 现象一级 | 现象二级 | 后果一级 | 后果二级 |
|---|---|---|---|---|
| 随机猜 | 33.33% | 12.50% | 20.00% | 5.56% |
| 多数类猜 | 40.12% | 26.07% | 40.11% | 4.90% |
| Claude-3.5-Sonnet (闭源最佳) | 37.60% | 19.72% | 40.28% | 15.90% |
| Qwen2.5-VL-72B (开源最佳) | 40.85% | 20.73% | 39.58% | **16.25%** |
| LLaVA-1.6-vicuna-34B | **42.28%** | 15.45% | 17.67% (< 随机!) | 3.89% |
| GPT-o1 (反直觉地弱) | 32.93% | 10.97% | 17.67% | 10.60% |
| FT-LLaVA-1.5-7B | **45.93%** | 23.37% | 43.46% | 9.89% |
| FT-LLaVA-1.6-vicuna-7B | 44.91% | **29.67%** | **44.17%** | 8.48% |

- 零样本 MLLM 大多打不过多数类基线; 二级现象无一超过 26.07%。
- 微调 7B 全面反超 34B 未调模型 (后果一级 44.17% vs 17.67%, +26.5 pts) 和闭源旗舰 → 任务对齐 > 规模。
- **消融** (Gemini-2.0-flash / LLaVA-1.6-mistral-7B): 去图像重创现象预测 (37.80→25.81; 后果 39.22→13.78); **去统计量反而常常涨分** (Gemini 后果一级 30.39→38.87; LLaVA 39.22→44.52) → 统计摘要与栅格图冗余甚至添噪; 去 CoT 重创后果推理 (30.39→17.67)。
- **规模非单调**: LLaVA-34B 现象最强但后果崩到低于随机; Qwen 7B→72B 后果大涨但现象微降。
- **案例研究**: 高温事件全员答对; 加州山火案例中 GPT-4o 与 Claude-3.5 都推理出"火险条件"却最终选了 Drought (因果综合失败), 只有 Qwen2.5-VL-72B 把强风解读为助燃因子而答对 Fire; 微调模型直接给答案无推理 (且案例 2 答错)。
- 全部模型四任务准确率 <50% —— 任务远未解决, surprising 的是推理链正确、最终选项错误的"临门一脚"失败模式。

### 明确局限

作者自述: (1) 缺卫星影像、台站时序等更多模态 (声明可按时空信息轻松接入); (2) 抽出的因果三元组/知识图谱未被利用。

我读出的局限 (对我们更关键):
- **环境场是单日快照**: 事件当日的日均 ERA5, 无任何时间演变窗口 —— "预报"实为同日诊断, 无 lead time 概念。
- **ERA5 27.75 km + 4 个地表变量**对强对流完全不够 (无 CAPE/shear/探空类环境量, 无逐时演变)。
- **RGB 三通道压缩**是信息瓶颈: 变量数被锁死在 3, 消融显示模型对这种编码的利用也不稳定。
- **新闻标签偏差**: 40.54% 通过率意味着 LLM 自动抽取近六成不合格; 地理覆盖偏向媒体发达地区; 事件频次跟着媒体热度走 (郑州峰值)。
- 无真机/业务对接; 无 CSI/POD/FAR 等气象业务指标; accuracy 单指标 + 单次运行, 无置信区间; 测试集仅 775 例, 二级类目样本极稀。
- 数据"审后释出"承诺, 截至本文版本 GitHub 链接在源码中被注释掉, 复现性暂受限。

### 跟我们的关系

对 0713-scs-benchmark (强对流环境–雷达多模态开放基准):

- **不撞车**。CLLMate 是"新闻文本事件 × ERA5 当日快照 → MLLM 多选题"基准; 我们是"NOAA Storm Events 官方记录 × HRRR 3km 逐时长窗环境场 (-15d/+5d) × MRMS 雷达 → 以风暴为中心的事件包 + 可追溯研究数据产品"。事件源、环境数据、模态组合、时间结构、交付形态五个维度全部不同。
- **反而强化我们的 gap 论述**: 连最接近"事件+气象对齐"的基准也只把环境场做成单日 RGB 缩略图 —— "环境场及其时空演变作为一等公民"在现有基准中确实缺位, 可在 related work 中直接引用 CLLMate 作为证据。
- **可借用的设计**: (1) LLM 抽取 + 多专家多数票 + κ 报告的标注 QA 协议 (其 40.54% 通过率是"新闻挖掘标签噪声"的量化警示, 反衬我们用官方 Storm Events 的优势); (2) 多数类基线暴露类不平衡的做法; (3) 层级分类 + 多选题作为 NeurIPS D&B track 的 canonical task 模板之一; (4) 按时间顺序切分防泄漏; (5) 模态消融实验设计; (6) RGB 压缩的失败教训 → 我们应保留原生栅格/NetCDF 访问路径而非只给图像化产物。
- **引用义务**: related work 必引 (WCEF 任务定义者); 其引文 WeatherQA/ClimateIQA 与反引文 MeteorPred/WXImpactBench 构成"气象 MLLM 基准"谱系, 我们的差异化叙事要覆盖这一整条线。

## Read by: 0713-scs-benchmark (2026-07-14)

以 SCS 环境–雷达开放基准视角审读:

- **撞车评估: 无撞车**。CLLMate 的 claim 是"首个把文本事件与气象数值数据对齐的监督多模态数据集 + MLLM 基准", 其环境场是事件当日 ERA5 日均 4 变量的 RGB 缩略图, 无时间窗、无雷达、无官方事件库、无 CONUS 强对流聚焦。我们的核心 claim (长时窗 HRRR 环境场 + MRMS 雷达 + NOAA Storm Events 的以风暴为中心事件包, 环境场时空演变为一等公民) 未被触及。
- **对我们最有用的三件东西**: (1) 标注 QA 协议范式 (多专家多数票 + Fleiss κ + 通过率透明化) —— 我们的 Storm Events 标签虽为官方记录, 但事件–雷达聚类的对应关系仍需类似的质检报告; (2) 多数类基线 + 按时间切分, 直接纳入我们 canonical tasks 的评测协议; (3) 其消融证明"统计摘要与图像冗余、RGB 压缩不可靠" —— 支持我们"交付原生栅格 + 训练 ready 双层数据"的三层交付设计。
- **差异化叙事**: 在 related work 把 CLLMate/WeatherQA/ClimateIQA/MeteorPred 归为"MLLM 评测导向"的事件基准 (环境信息被压缩成图像或文本摘要, 服务于模型评测), 而我们是"数据产品导向" (完整保真的多模态源数据 + provenance, 服务于气候学/过程研究/nowcasting/预警全链路)。CLLMate 全员 <50% 的结果还说明: 缺乏环境演变信息时事件推断困难 —— 恰是我们长时窗设计的动机证据。
