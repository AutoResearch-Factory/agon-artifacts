# ParFam - (Neural Guided) Symbolic Regression via Continuous Global Optimization (2310.05537)

- **Year**: 2023.10
- **Authors**: Philipp Scholl, Katharina Bieker, Hillary Hauger, Gitta Kutyniok; Ludwig-Maximilians-Universität München / Munich Center for Machine Learning
- **Venue**: arXiv
- **Keywords**: symbolic regression, continuous global optimization, rational functions, basin-hopping, Set Transformer, SRBench, Feynman benchmark, DL-ParFam
- **One-liner**: 把符号回归转成有理函数连续优化，并用 Set Transformer 加速。

## 精华提炼

### 管线/方法
- 输入是回归数据集 `(x_i, y_i)_{i=1..N}`，其中 `x_i in R^n`，目标是找到简洁可解释的符号函数 `f: R^n -> R`。
- ParFam 的核心改写是把离散表达式树搜索改成连续参数优化。
- 用户先给定一组非有理的一元基函数，例如 `sqrt`, `cos`, `exp`；论文实验中主 SRBench 设置使用这三个。
- 对每个基函数 `g_j`，ParFam 学一个有理函数输入 `Q_j(x)`。
- 输出层再学习一个有理函数 `Q_{k+1}`，它接收原始 `x` 和所有 `g_j(Q_j(x))`。
- 主模型写作 `f_theta(x)=Q_{k+1}(x, g_1(Q_1(x)), ..., g_k(Q_k(x)))`。
- 这可以看作一层 residual-like neural network，但连接不是线性层，而是有理函数层。
- `Q_j` 的可学习参数是分子多项式和分母多项式的系数。
- 离散结构搜索被压缩为少量 model parameters：输入层有理函数分子/分母次数、输出层有理函数分子/分母次数、基函数集合。
- 对给定 model parameters，训练只需优化连续参数 `theta`。
- 损失函数是 MSE 加稀疏正则：`L(theta)=1/N sum_i (y_i-f_theta(x_i))^2 + lambda R(theta)`。
- 正则项取 `R(theta)=||theta||_1`，目的是推动公式稀疏、便于解释。
- 优化器选择 SciPy 的 basin-hopping，全局随机跳跃加局部 BFGS。
- basin-hopping 后还有一轮 finetuning：把低于阈值的系数置零，再用 L-BFGS 优化保留下来的系数。
- 置零阈值从 `1e-5` 逐步增至 `1e-2`，用于进一步压缩公式。
- 对高层结构选择，ParFam 不一次性跑最大模型，而是按从简单到复杂的顺序遍历候选 model parameters。
- 遍历顺序先试多项式，再试纯有理函数，再逐步加入基函数和更高次数。
- 这种搜索类似 exhaustive symbolic regression，但一个 ParFam 参数族一次覆盖大量表达式，因此候选数更少。
- 论文说相同一维设置下，exhaustive 方法可需要超过 100,000 个函数，而 ParFam 约搜索 500 个参数族。
- DL-ParFam 是 neural-guided 版本：先用预训练网络预测 ParFam 的 model parameters，再运行 ParFam 优化连续系数。
- DL-ParFam 的输入仍是无序数据点集合 `(x_i,y_i)`。
- 预训练网络 encoder 使用 Set Transformer，原因是数据点顺序不应影响预测。
- classifier 是全连接 ReLU 网络，输出五类 one-hot model parameters。
- 输出参数包括第一层分子次数、第一层分母次数、第二层分子次数、第二层分母次数、基函数类型。
- 推理时取概率最高的 `m=3` 组 model parameters 给 ParFam 试跑。
- 推理时还额外加入多项式和纯有理函数参数族，因为这些更便宜。
- 训练 DL-ParFam 的合成数据直接从 ParFam 参数族采样，而不是从 Lample 等人的表达式生成器采样。
- 这样做是因为他们需要直接得到 ParFam model parameters 作为监督标签。
- 合成数据采样 `x_i ~ U(1,5)^n`，维度 `1 <= n <= 9`。
- 每个函数采样 200 个数据点。
- 系数不直接任意采样，而是限制每个多项式只有 1 到 3 个非零系数，以避免不现实的复杂公式。
- 还要求非零系数覆盖“最小必要参数族”，避免同一函数有多个正确标签。
- 对生成函数做过滤：若 `y_i` 为 NaN、无穷或超出 `[-1000,1000]`，最多重采样对应 `x_i` 6 次，仍失败则丢弃函数。

### 关键技术细节
- 主 SRBench 参数：第一层分子最大次数 2，第一层分母最大次数 2。
- 主 SRBench 参数：输出层分子最大次数 4，输出层分母最大次数 3。
- 主 SRBench 基函数：`sqrt`, `cos`, `exp`。
- 主 SRBench 最大变量幂次：3，即排除 `x_1^4`，但允许 `x_1^3 x_2`。
- 主 SRBench 优化器：global optimizer 为 basin-hopping，local optimizer 为 BFGS。
- ParFam 的 basin-hopping 最大迭代数设为 10。
- DL-ParFam 的 basin-hopping 最大迭代数设为 1，用网络预测来省搜索。
- 最大数据集长度设为 1000。
- 正则参数 `lambda=0.001`，论文还做了 `0.0001/0.001/0.01` 灵敏度分析。
- Feynman 数据集原有 119 个物理公式；作者按 SRBench 做法删除含 arcsin/arccos 的 3 个和 1 个缺失 ground truth 的数据集。
- 无噪声实验每个 Feynman 问题取 500 个训练点，有噪声实验取 1000 个训练点。
- Strogatz 数据集包含 14 个非线性 ODE 问题，来自 7 个二维 chaotic dynamic systems。
- 黑盒实验只看最多 10 个独立变量的数据集，因为高维全局优化太贵；这留下 77 个数据集。
- Ground-truth benchmark 时间预算沿用 SRBench：8 CPU 小时和 1,000,000 次函数评估。
- 黑盒 benchmark 时间预算为 48 CPU 小时。
- 评测指标包括 symbolic solution rate、`R^2`、accuracy solution rate。
- accuracy solution rate 定义为测试集 `R^2 > 0.999` 的比例。
- 加噪声实验使用 `epsilon_i ~ N(0, sigma^2 * 1/N sum_i y_i^2)`。
- 有理函数分母正则有一个细节：直接正则分子/分母会被共同缩放绕过。
- 论文用分母系数的 `L2` 归一化来避免 `gamma a, gamma b` 等价缩放破坏 `L1` 稀疏正则。
- 不能用 `L1` 归一化分母，因为分母归一后 `L1` 恒为 1，无法促进稀疏。
- 数值稳定处理：`sqrt(Q(x))` 改为 `sqrt(abs(Q(x)))`。
- 数值稳定处理：`exp(Q(x))` 用 `min(exp(Q), exp(10)+abs(Q))` 近似避免 overflow。
- 数值稳定处理：分母绝对值小于 `1e-5` 时替换为 `1e-5`，避免除零。
- DL-ParFam 训练集约 4,000,000 个合成函数。
- DL-ParFam 主训练跑 88 epochs，在 4 个 epoch 不再提升后停止。
- 数据生成耗时 1.3 小时；训练耗时 93 小时，硬件是一张 RTX3090。
- 之后又在 4,000,000 个 noisy functions 上 finetune 30 小时。
- 对比中 EndToEnd 使用 800 GPU 小时，NeSymRes 使用 10,000,000 个函数并在 RTX 2080 上训练 72 小时。
- Set Transformer 参数：hidden dimension 256，inducing points 128，multi-head 每层 4 heads，encoder 8 层，classifier 4 层。
- DL-ParFam optimizer 是 Adam，learning rate `0.0001`，batch size 1024，gradient clipping 1。

### 核心结果
- 主文结论：PySR、ParFam、AI Feynman、DL-ParFam、uDSR 在 symbolic solution rate 上明显领先其他算法超过 20 个百分点。
- 在这 5 个强方法中，PySR 的 symbolic solution rate 最好，ParFam 第二。
- ParFam 在 accuracy solution rate 上无噪声和有噪声都优于所有竞争方法。
- DL-ParFam 的 symbolic recovery 低于 ParFam，但速度目标达成，最高可比竞争方法快约 100 倍。
- 论文指出 DL-ParFam 在低噪声条件下恢复精确符号解的能力受限，这一点不是被隐藏的失败。
- 灵敏度分析中，`lambda=0.0001` 的 accuracy solution rate 为 94.7%，symbolic solution rate 为 50.0%，复杂度 227。
- 灵敏度分析中，`lambda=0.001` 的 accuracy solution rate 为 91.7%，symbolic solution rate 为 55.6%，复杂度 131。
- 灵敏度分析中，`lambda=0.01` 的 accuracy solution rate 为 94.7%，symbolic solution rate 为 52.2%，复杂度 243。
- 反直觉结果：`lambda` 增大时公式复杂度不一定下降，作者认为原因可能是 `L1` 只是稀疏 proxy，且模型参数遍历破坏单调关系。
- 与 EQL-with-division 在不含 `sqrt/log/exp` 的 96 个 SRBench 公式上比较：ParFam accuracy 93.8%，symbolic 69.8%。
- 同一 reduced benchmark 上 EQL accuracy 75.0%，symbolic 16.7%。
- 与 NeSymRes 在最多 3 变量的 SRBench 子集比较：DL-ParFam symbolic 58.1%，accuracy 87.1%，训练/推理时间 53 秒。
- 同一子集上 NeSymRes symbolic 48.7%，accuracy 59.0%，时间 1389 秒。
- 与 Bayesian model-parameter selection 比较：Bayesian 50 calls symbolic 34.9%，accuracy 85.3%，time 7678s。
- Bayesian 500 calls symbolic 38.0%，accuracy 89.1%，time 10937s。
- DL-ParFam 在该表 symbolic 45.9%，accuracy 83.5%，time 234s。
- Grid search ParFam symbolic 55.6%，accuracy 93.2%，time 12860s。
- 这说明 DL-ParFam 是速度优先策略；完整 grid-search ParFam 质量更高但慢很多。
- Nguyen benchmark 平均：ParFam 91.2%，SPL 94.5%，NGGP 92.4%，GP 38.2%。
- Nguyen-11 `x^y` ParFam 为 0%，因为该函数不在 ParFam 当前结构表达范围内。
- SRSD-Feynman easy：ParFam symbolic 76.7%，accuracy 93.3%，time 7620s；DL-ParFam symbolic 60.0%，accuracy 66.7%，time 72s。
- SRSD-Feynman easy：PySR symbolic 90.0%，accuracy 93.3%，time 5796s；uDSR symbolic 66.7%，accuracy 86.7%，time 7470s。
- SRSD-Feynman medium：ParFam symbolic 47.5%，accuracy 80.0%，time 8478s；DL-ParFam symbolic 40.0%，accuracy 65.0%，time 138s。
- SRSD-Feynman medium：PySR symbolic 62.5%，accuracy 85.0%，time 8022s；uDSR symbolic 40.0%，accuracy 82.5%，time 7740s。
- 作者把 SRSD 结果解释为：DL-ParFam 在与训练分布不同的物理量范围上仍有可用表现，可能受益于输入归一化。

### 明确局限
- ParFam 的主要优势也是局限：预定义一层有理函数加一元基函数结构会限制函数空间。
- 作者用表达式树计数分析说明限制不严重，但这仍是理论上的平均/近似论证，不等于覆盖所有科学公式。
- 不能表达多层一元函数复合，例如 `cos(sqrt(x))` 这种路径上有超过一个 unary node 的表达式。
- 高维问题困难；作者明确说超过 10 个变量时全局优化代价很高。
- 参数数量随变量数和多项式次数增长，作者写为 `O(n^d)`。
- DL-ParFam 的最大维度当前是 9，黑盒数据集高于 9 维的部分无法直接使用。
- DL-ParFam 依赖昂贵预训练，灵活性低于裸 ParFam。
- DL-ParFam 用 synthetic functions 训练，分布和真实 Feynman/黑盒数据仍有 gap。
- SET Transformer 在 synthetic 上 top-10 correct model parameters 为 71.2%，在 Feynman 上 top-10 只有 45.6%。
- 这说明网络更多是有用先验，而不是可靠的结构识别器。
- 作者承认 DL-ParFam 可在没预测到“正确 model parameters”时仍恢复正确函数，因为参数族之间有包含关系。
- 这也意味着用 model-parameter accuracy 衡量网络并不完全等价于最终公式成功率。
- 没有真机实验；这是算法/benchmark 论文，主要证据来自标准符号回归数据集。
- 论文没有独立第三方复现实验报告，虽然提供代码和实验设置。
- 主要结果依赖作者自己的实现和同机 rerun；PySR、uDSR、EndToEnd 等有部分超参选择，存在比较设置敏感性。
- PySR 有噪声过滤而 ParFam 没有，这是竞争比较中的重要差异。
- 图中主 SRBench 结果很多是 PDF 图，不是完整机器可读表；源码中只保留部分 appendix 表格数值。
- 对黑盒数据集的结论较弱，因为 ground truth 不存在，只能比较 median `R^2`、复杂度和时间。
- 若目标是从真实实验数据发现物理规律，仍需要噪声模型、单位约束、可辨识性检查和领域先验。
- 若目标是 LLM-agent 自动科研，ParFam 只是候选 equation discovery 工具，不提供实验设计或证据审计。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 视角看，ParFam 是必须放进 related work 的近邻方法。它不是又一个黑箱神经网络，而是把符号回归的表达式搜索约束成一层有理函数参数族，然后用 basin-hopping/BFGS 搜连续系数。若我们的课题主张从数据中抽取可解释公式，撞车点在“连续优化替代离散表达式搜索”和“神经网络预测搜索结构”两个 claim 上，尤其 DL-ParFam 已经用 Set Transformer 预测 model parameters 并显著提速。

可直接借用的部分是评测协议和工程细节：SRBench/Feynman/Strogatz/Nguyen/SRSD-Feynman，`R^2>0.999` accuracy solution rate，symbolic solution rate，公式复杂度和时间预算；实现上可参考有理分母 `L2` 归一化、`sqrt(abs(.))`、`exp` 截断、分母 `1e-5` 稳定化、系数阈值剪枝加 L-BFGS finetuning。需要避开的 claim 是“用 transformer 指导符号回归”本身，因为 DL-ParFam 和多篇后续工作已经覆盖。更稳的差异化是加入独立验证、单位/物理约束、主动实验设计、PDE/真实测量数据、或把 ParFam 作为 agent pipeline 中的可否决候选生成器。
