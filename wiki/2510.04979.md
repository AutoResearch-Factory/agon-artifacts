# Federated Computation of ROC and PR Curves (2510.04979)

- **Year**: 2025.10
- **Authors**: Xuefeng Xu, Graham Cormode (University of Warwick)
- **Venue**: arXiv preprint (no venue listed in metadata; not yet confirmed peer-reviewed)
- **Keywords**: federated learning, ROC curve, PR curve, differential privacy, quantile estimation, hierarchical histogram, secure aggregation, area error bound
- **One-liner**: 用分层直方图估计正负样本分位数、跨隐私持有方可加聚合重建全局ROC/PR曲线，给出Area Error的O(1/Q)与DP下Õ(1/Q+1/(nε))确定性worst-case界，全文不涉及跨异质子集的统计置信区间构造。

## 精华提炼

### 管线/方法

端到端流程（输入 → 每一步 → 输出）：

1. 每个 FL client 本地持有一批样本的分类器 prediction score 与真实标签（正/负）。
2. Client 侧：对 positive、negative 两类样本的 score 分别建 hierarchical histogram（分支因子 `b`，高度 `h`，即 `b^h` 个最细叶子 bin，均匀覆盖 score 值域）。
3. 若启用差分隐私：client 给每一层每个 bin 加 i.i.d. 噪声（连续噪声采样自 Gamma 分布，离散噪声采样自 Pólya 分布，实现 Distributed DP／DDP；隐私预算 `ε` 按 `h` 层平分）。
4. Client 把 `(h^+, h^-)` 两个直方图发给 server，经由 Secure Aggregation（server 只看到跨 client 求和结果，看不到任一 client 的原始直方图）。
5. Server 侧：对所有 client 的直方图逐 bin 求和，得到全局 `H^+, H^-`；若启用 DP，对求和后的直方图做后处理（post-processing）以强制层级一致性与非负性。
6. Server 从 `H^+, H^-` 抽取 `Q` 个分位点 `q^+, q^-`，用 PCHIP（分段三次 Hermite 插值，保单调、不引入振荡）或线性插值重建两条 ECDF `Φ^+(s), Φ^-(s)`。
7. 对任意阈值 `s`：`F(s)=1-Φ^-(s)`（FPR），`T(s)=1-Φ^+(s)`（TPR），`P(s)=T(s)n^+/(T(s)n^++F(s)n^-)`（Precision）。
8. 输出：由离散阈值点连成的 ROC 曲线 `(F(s),T(s))` 与 PR 曲线 `(T(s),P(s))`；可进一步用梯形法则算 AUC-ROC/AUC-PR。

**理解本文能否为我们所用的关键区分**：这是"许多 client 各自持有同一底层分布的一部分数据 → 服务器把它们加总还原成好像从未拆分过的单一全局曲线"的问题（估计目标是唯一的 pooled 曲线），不是"许多独立的、内部存在真实异质性的评测单元（storm/season）各自贡献一条曲线，再对这些曲线做统计意义上的跨单元汇总与抽样不确定性量化"的问题（估计目标是跨单元效应量分布本身）。这个区分决定了本文的核心算法在方法论层面是否可以迁移到我们的 storm-cluster bootstrap 需求。

### 关键技术细节

- Hierarchical histogram 参数：高度 `h`，分支因子 `b`（实验取 `b=2`），`h = ceil(log_b Q) + c`（`c` 取 2 或 3）；每 client 通信量 `O(b^h)=O(Q)`。`Q=1024` 时每个直方图约 `2^13` 个 bin，约 8K 个 32-bit 整数（32KB）。
- **Area Error (AE)** 定义：真实曲线与估计曲线之间"绝对误差"在 `[0,1]` 上的积分（ROC: `∫|T(f)-T̂(f)|df`；PR: `∫|P(t)-P̂(t)|dt`），比 Point Error（只看阈值点误差，参见 Barczewski2025）和 `|AUC 差|` 更严格：AE 小 ⟹ AUC 误差小，反之不然。
- 四条主定理：(1) ROC-AE：`Q` 个精确分位数下 ROC 的 AE 上界 `O(1/Q)`；(2) PR-AE：类别比 `r=n+/n- >= 0.1` 时 PR 的 AE 为 `Õ(1/Q)`，极端不平衡 `r<<1` 时为 `Õ(1/(Qr))`；(3) AE-SA：安全聚合（无隐私噪声）下 ROC 为 `O(1/Q)`，PR 为 `Õ(1/Q)`；(4) AE-DDP：分布式差分隐私下 ROC/PR 均为 `Õ(1/Q + 1/(nε))`。
- 证明技巧：把 ROC/PR 空间按分位点切成 `Q-1` 个竖条，用梯形面积公式给出每条的最坏情形误差，再用 `sum_i Delta_i <= 1` 求和得到整体 `O(1/Q)`；PR 曲线额外需要"well-behaved score distribution"（`Φ+, Φ-` 均为 `Θ(1)`-Lipschitz）这一光滑性假设，否则分位数位置误差可能在 score 分布有尖峰（大量样本同一取值）时被放大成任意大的 T/F 误差（Lemma abs-error-score, abs-error-curve）。
- 隐私机制：Distributed DP（DDP, Goryczka2017）——每个 client 独立加噪，加总后满足 `ε`-DP；连续噪声用 Gamma，离散用 Pólya，也可扩展到 `(ε,δ)`-DP 的 Skellam / Distributed Discrete Gaussian 机制。
- 实验设置：3 个真实数据集（Bank 45K 行 pos:neg=0.132，Adult 33K 行 0.317，Cover 581K 行 0.574，代表三种不平衡程度），两个分类器（XGBoost、Logistic Regression）；对比线性 vs PCHIP 插值、SA vs DDP 不同 `ε`、"separate"（正负样本分位数分别估计）vs "combine"（合并估计）两种 PR 策略，以及与 Barczewski2025 range-query 方法的直接对比。代码开源：github.com/xuefeng-xu/fedcurve。

### 核心结果

- `Q≈100` 时：ROC 的 AE 常在 `~1e-3` 量级，PR 的 AE 在 `~1e-2` 量级以下，且在强隐私（`ε<=1`）下依然保持较低误差。
- "separate" 策略显著优于"combine"策略——因为 separate 保证 precision 分子天然 `<=` 分母，不需要截断；combine 策略在数值上可能需要 clipping 到 1，引入额外误差。
- 与 Barczewski2025 的 range-query DP 方法相比：渐近阶相当（本文 `O(1/Q + log^{3/2}Q/(nε))` vs 对方 `O(log^{3/2}N/(nε))`），但本文方法随 `Q` 增大更稳定；range-query 方法在 `Q<100` 低精度区间偶尔更优（尤其在高度不平衡的 Cover 数据集上），因其用固定阈值网格同步取点，而本文对正负样本各自独立取分位数在小 `Q` 时可能出现阈值错位。
- 类别不平衡实验证实 PR 曲线 AE 随 `r` 减小而增大（与 `Õ(1/(Qr))` 理论一致），但当分类器在极端不平衡下性能退化到接近"随机分类器"（PR 曲线趋于 `y=n+/n` 的水平线）时，观测误差反而可能低于理论最坏情形。
- 扩展讨论（非实证）：多分类可用 one-vs-rest/one-vs-one 分解为多个二分类任务，再用 macro/加权平均合并结果，误差界据称可传递（未给出具体权重选择算法或加权平均后的严格证明，只是一句话论证）；该框架可直接扩展到 AUC（AE 直接给出 `O(1/Q)` 的 AUC 误差界）、Mann-Whitney U 统计量、DET 曲线，Precision-Recall-Gain 曲线留作未来工作。

### 明确局限

- 作者自陈局限：`Q` 较小时本文方法可能不如 range-query 方法，尤其当阈值对齐很关键时；提及混合方法作为可能补救，但未给出对应 PR 曲线的误差界。
- Well-behaved（Lipschitz）假设是所有误差界成立的前提；若 score 分布有尖峰，理论保证会失效，论文只用"经验上不是病态分布，结果依然稳健"回应，未给出病态情形下的替代保证。
- 三个数据集均为标准 UCI 类表格数据（Bank/Adult/Cover），分类器为 XGBoost/LR，与我们强对流 4D 时空张量、CNN、SPC/HRRR storm-level 场景在数据模态、规模、评测协议上完全不同，全文没有任何时序或空间结构的讨论。
- 隐私/通信约束是本文的核心驱动力，我们的审计任务没有联邦学习场景，不需要这层约束——论文为解决"隐私+通信"设计的全部机制（直方图分箱、DDP 加噪、post-processing 一致性）对我们没有直接适用性。
- **针对本轮 gap 问题的最关键局限**：全文没有任何 bootstrap、重抽样、跨 client/跨组的置信区间或方差估计方法。"多 client 异质性"在本文里被处理为"服务器看不到内部结构、只需重建加总后的全局曲线"的鲁棒性属性（"agnostic to data heterogeneity and skew"），而不是"内部结构（client 间差异）本身值得量化的统计对象"。这与我们需要的"以 storm 为重抽样单元估计 Delta AUPRC 抽样分布"的目标不是同一个统计问题。

### 跟我们的关系

**直接回答本轮 gap(2) 的问题**：本文核心算法**不能**为 storm-cluster 级别的 AUPRC 计算与跨 season 结果聚合提供可复用的具体算法。理由分四层：

1. **"异质子集"类比在表面成立、在统计目标上不成立**：论文里的 FL clients 确实是异质数据持有者，字面上可以类比"不同 storm/season 各自持有一部分观测"。但论文要重建的估计对象是"若所有 client 数据汇聚一处会得到的那一条 pooled ROC/PR 曲线"——一个近似确定性的量（随机性只来自有限 `Q` 和 DP 噪声）；而我们需要的是"以 storm 为独立单元估计 Delta AUPRC 的抽样分布"——一个随机性来自 storm 间真实差异（between-cluster variance）的统计推断问题。两者的 estimand 根本不同。
2. **曲线聚合技巧不可迁移**：论文的聚合是"每个 client 本地对 score 分布分箱，server 逐 bin 线性求和（Secure Aggregation）"，前提是所有 client 共享同一分箱方案，聚合后精确（或 DP 噪声下近似）等价于中心化计算。这是 within-pool 的可加聚合，不是 between-cluster 的统计合成；不能直接用来"先算每个 storm 的 AUPRC，再合成跨 storm 的点估计与方差"。
3. **没有可复用的置信区间构造技巧**：全文的"不确定性"全部是 Theorem/Lemma 给出的 worst-case 确定性界（依赖 `Q` 与 DP 参数 `ε, n`），没有出现 bootstrap、permutation、jackknife 或任何基于重抽样的 CI。这与 storm-cluster bootstrap（对 storm 做重抽样、构造 Delta AUPRC 95% CI）是完全不同的不确定性量化范式——一个是"近似/隐私噪声误差界"，一个是"抽样变异性置信区间"，没有算法可以直接借用。
4. **最接近但仍不可用的一处**：Discussion 里"Extension to Multi-class Setting"提到 macro/加权平均多条曲线、误差界可传递——这是唯一在字面上涉及"合并多条曲线"的段落，理论上可类比"跨 season 加权平均"，但论文只给出一句话论证，未给出具体权重选择算法，且仍是确定性平均（不产生置信区间），不构成可操作的算法可以摘录进我们的 protocol。

**间接、次要的借鉴价值**（非本轮 gap 直接要求，但值得记录）：
- Area Error（曲线级绝对误差积分）vs Point Error vs `|AUC 差|` 的三层区分，提示我们在报告 Delta AUPRC 时，若想更细致地定位"哪个 recall 区间差异最大"，可以参考这种"整条曲线积分误差"的度量思路，而不仅报标量 AUPRC——但这只是度量设计的启发，不是可执行算法。
- "separate" 优于"combine"的分位数策略再次印证 PR 曲线对正负样本比例的敏感性，与我们 proposal 里反复强调"PR/AUPRC 对不平衡敏感、需要谨慎处理跨 storm 流行率差异"的关切方向一致，属于背景知识加固而非方法迁移。
- Related Work 提到 Fawcett2006 的"heuristic ROC 合并方法"（在固定 FPR 处平均 TPR，或在固定阈值处平均 FPR-TPR 对）"缺乏理论保证，不适合 FL 中的异质数据"——这句话本身确认了：即使在这篇论文引用的前人工作里，"跨异质子集合并曲线"也只有启发式方法，没有成熟的、带置信区间的统计合成算法，间接印证我们需要的 storm-cluster bootstrap CI 技巧应该从 ML 评测统计文献（如 clustered/paired bootstrap for AUC/AUPRC 比较）里找，而不是从联邦学习曲线计算文献里找。

**结论**：本文不撞车（我们的 proposal 不涉及联邦学习、隐私预算或通信约束），也没有直接可用的 storm-cluster CI 或跨 season 曲线聚合算法；下一轮搜索应转向"clustered/paired bootstrap confidence interval for AUC or AUPRC comparison"、"block bootstrap for spatially or temporally correlated evaluation units"等 ML 评测统计方向，而非继续深挖联邦学习曲线计算文献。
