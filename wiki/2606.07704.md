# FunctionEvolve: Structure-Guided Symbolic Regression with LLMs (2606.07704)

- **Year**: 2026.06
- **Authors**: Zeyu Xia (Bosch AI / Tsinghua), Jun Zhu (Tsinghua), Dong Yan (Bosch AI)
- **Venue**: NeurIPS 2026 (arXiv preprint)
- **Keywords**: symbolic regression, LLM-guided evolution, abstract syntax tree, expression tree, structure-aware optimization, scientific equation discovery, genetic programming, LLM-SRBench
- **One-liner**: LLM + explicit AST 结构引导进化搜索，在 LLM-SRBench 上以 82.9% SA@50 大幅超越此前方法 4.5x，同时揭露 benchmark 中的可辨识性问题。

## 精华提炼

### 管线/方法

FunctionEvolve 是一个神经符号进化框架，把 LLM 的语义推理能力和显式 AST 树操作结合起来做符号回归。整体管线分为四个阶段：

1. **Generator（初始化）**：读取任务上下文（变量名、物理含义、科学背景），让 LLM 提取领域先验知识，然后生成 20 个多样化的种子表达式。种子覆盖两类：基础函数结构（单项/多项式/幂律/指数/对数等）和知识驱动结构（从领域先验衍生的公式形式）。若 LLM 产出不够，用固定回退种子池补足。

2. **Selector（父代选择）**：每轮进化迭代，系统提取进化树的摘要（每个节点的 AST 深度、参数数量、算子类型计数、训练 NMSE、拟合参数值），把摘要 + 任务上下文 + 历史选择记录给 LLM，LLM 选择 5 个父代用于下一轮。选择策略是非贪婪的：优先选结构简单 + 低 NMSE + 代数结构多样的节点，避开已经充分探索的方向。

3. **Mutator（变异）**：对每个选中的父代施加三类操作：
   - **规则添加**（Rule Addition）：将基础函数库（线性/三角/幂律/指数/对数/有理分数）通过加法、乘法、除法、一元函数包装附加到父表达式上。
   - **规则删除**（Rule Deletion）：剪枝或解包 AST 子树，测试当前公式部分是否冗余。
   - **LLM 引导变异**（LLM-guided Proposal）：给 LLM 提供父表达式、带标注的 AST、拟合参数、规则已生成的候选摘要、搜索历史，请求生成 20 个非模板的局部精修候选（ADD 保留父代附加新材料，SUBST 替换不匹配的子树）。
   - 候选经过结构指纹去重和参数规范化后进入表达式数据库。

4. **Optimizer（系数优化）**：对每个固定骨架拟合系数并评分，利用 AST 结构做多阶段优化：
   - **变量投影**（Variable Projection）：识别线性参数，用 OLS 解析求解，只对非线性参数做数值搜索。
   - **结构化预搜索**：对幂指数参数枚举固定候选值（{1,2,1/2,3,1/3,...})，对偏移参数扫描 100 个等距候选点。
   - **全局搜索**：trust-region least squares + CMA-ES + differential evolution 在约束参数空间搜索。
   - **局部精修**：L-BFGS-B 在参数边界内做梯度优化。
   - **参数 snapping + 重拟合**：将指数类参数 snaping 到最接近的有理数，然后重拟合验证。
   - **符号简化**：合并冗余项、简化零项、消去恒等变换。

### 关键技术细节

- **模型 backbone**：Claude Opus 4.6（主要）、GPT-5.2-medium、DeepSeek-V4-Pro、Qwen3.6-27B、Llama-3.1-8B
- **基准测试**：LLM-SRBench 合成子集（129 个任务，4 个领域：Chemistry 36、Biology 24、Physics 44、MatSci 25）+ AI-Feynman（120 个任务）
- **运行配置**：最多 30 轮进化迭代/任务，每轮选 5 个父代，每个父代 LLM 提 20 个候选，LLM 调用预算理论值 182 次/任务（实际平均 66.86 次因 early stopping 减少）
- **选择器 LLM 参数**：temperature 0.3
- **变异器 LLM 参数**：temperature 0.7，max_tokens 128000
- **CPU 评估协议**：每轮平均评估 126.09 个后代，每个后代 120s 超时
- **GPU**：本地 Llama-3.1-8B / Qwen3.6-27B / DeepSeek-V4-Pro 用 16 张 NVIDIA H20-3e，vLLM 0.20.0
- **LLM 代币用量**：FunctionEvolve 总 340.36M tokens（Selector 占 85.3% = 290.44M，主要是 prompt tokens），价格加权成本 85.56M 低于 LLM-SR 的 143.45M
- **评估指标**：SA@k（前 k 个候选中有精确符号匹配）、Acc_tau（所有点的最大相对误差 <= tau）、NMSE
- **符号等价性判断**：双模型（GPT-5.2 + Claude Opus 4.6）独立判断，不一致分歧手动 adjudication（500 样本审计：96.4% / 98.6% 准确率）
- **代码可用性**：https://github.com/Phoinikas03/FunctionEvolve（匿名化）

**关键观察**：
- FunctionEvolve 在 SA@50 上比最强的同 backbone baseline（LLM-SR 和 OpenEvolve）高出 **4.5x**（107 vs 24）
- SA@1 上超过 PiT-PO 的 top-1 结果 **3.6x**（72 vs 20）
- No LLM 变体（纯程序化 AST 变异 + 结构感知优化器）已达 29/129，接近 PiT-PO 的 20
- AI-Feynman 120 个任务全部 SA@1 恢复
- LLM 调用次数平均 66.86/任务，远低于 LLM-SR（1000）和 OpenEvolve（201.5）

**消融实验关键发现**（Claude Opus 4.6）：
- Full：107 (72)
- w/o All：34 (10) -- 去掉所有 LLM 组件 + 结构优化器
- w/o LLM Mutator：46 (31) -- LLM 引导变异最重要
- w/o Structure-aware Optimizer：53 (22) -- 结构感知优化同样关键
- w/o AST Structure：60 (40) -- 去掉 AST 可见性
- w/o Selector：74 (46)
- w/o AST Mutator：84 (50)
- w/o Generator：91 (58)

**噪声鲁棒性**：
- Clean：107 (72), Acc_0.1=113
- 1% noise：54 (24), Acc_0.1=39
- 5% noise：40 (13), Acc_0.1=23
- 噪声下符号恢复显著下降，因为噪声让不同结构都能解释数据

### 明确局限

作者说：
1. **基准测试可辨识性缺陷**：MatSci 子集中 T（温度）和 epsilon（应变）呈近似完美线性相关（R^2 ~ 1），Arrehnius 因子 exp(-c/T) 变化极弱（0.11%-2.06%），导致不同公式结构可获得相同 NMSE。
2. **无紧凑闭式解的复杂动力系统**（如 PDE 控制的过程）超出当前范围。
3. LLM-SRBench 任务最多 3 个输入变量，真实问题可能有更多候选特征。
4. **噪声敏感**：1% 噪声使 SA@50 从 107 降至 54。
5. AI-Feynman 结果可能受 LLM 预训练数据污染影响（round 0 匹配率高）。
6. 符号精确恢复仍是主要瓶颈，数值低 NMSE 不完全等价于正确结构。

我自己读出的局限：
1. **无真机/物理实验验证**：全部在合成 benchmark 上评估，没有真实物理实验数据。
2. **LLM 成本**：Selector 占 token 消耗的 85%（长上下文演化树摘要），虽然调用次数少但价格仍可观。
3. **多变量扩展能力未验证**：最多 3 变量，真实科学问题可能有 10+ 变量。
4. SA@50 和 SA@1 间存在较大差距（107 vs 72），说明正确公式被生成但未被正确排序——终选策略仍有改进空间。
5. 与我们的 soft-body physics 方向关系较远：本文聚焦纯数学符号回归，不涉及物理仿真、接触动力学等。

### 跟我们的关系

这篇论文对 topic `0616-fex`（假设为 "function expression / symbolic regression in expressive systems"）的参考价值如下：

**不撞车**：本文是标准的 LLM-driven symbolic regression，聚焦于从 (x, y) 数据恢复数学闭式。不涉及 soft-body physics、接触动力学、物理仿真等具体领域。方法论上跟我们的方向正交。

**可用的工具/方法**：
1. **AST-visible evolution framework**：如果我们做 symbolic policy / dynamics learning，可以借鉴其结构感知的进化搜索框架
2. **Structure-aware optimizer**：Variable projection + parameter snapping 方法通用，适用于任何基于符号表达的系数优化任务
3. **LLM 引导的局部变异策略**：保留父代有用子结构、用 AST 标注指导局部替换的方法可移植性高
4. **进化树摘要选择器**：用 LLM 做父代选择的 prompt 设计值得借鉴

**该引不该引**：如果我们的工作涉及 symbolic regression 或从数据恢复可解释表达式，该引（尤其是其结构引导 LLM 进化的方法贡献）。如果我们的工作只是 soft-body 物理仿真，不涉及符号形式发现，不该引。

**差异化方向**：我们如果做 FEX（函数表达式相关），可以考虑将本文的 AST-guided 方法扩展到：(1) 更复杂的物理系统（PDE/ODE）；(2) 带噪声的真实传感器数据；(3) 多变量 + 特征选择场景。
