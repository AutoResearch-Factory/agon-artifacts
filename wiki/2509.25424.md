# Polychromic Objectives for Reinforcement Learning (2509.25424)

- **Year**: 2025.09
- **Authors**: Jubayer Ibn Hamid, Ifdita Hasan Orney, Ellen J. K. Xu, Chelsea Finn, Dorsa Sadigh (Stanford University)
- **Venue**: arXiv
- **Keywords**: RLFT, set reinforcement learning, polychromic objective, PPO, vine sampling, entropy collapse, trajectory diversity, pass@k
- **One-liner**: 用集合目标和 vine PPO 防止 RLFT 多样性坍缩。

## 精华提炼

### 管线/方法

1. 输入是一个已经预训练好的策略 `pi_beta`，目标是在下游 RLFT 中提升成功率，同时保留多样化行为。
2. 论文把 RLFT 的失败模式定义为 entropy/diversity collapse：策略把概率质量集中到少数高奖赏行为上。
3. 他们不是只加 token/action 级 entropy bonus，而是把优化对象从单条 trajectory 扩展到 trajectory set。
4. 核心框架叫 **set reinforcement learning**：优化 `E_{tau_1:n ~ pi_theta}[f(s0, tau_1, ..., tau_n)]`。
5. set RL 的关键约束是：同一个 trajectory set 内的所有轨迹共享同一个学习信号。
6. 如果 set objective 只是单条 reward 的平均值，则会退化成标准 RL；作者明确要求目标不能被改写成单 trajectory objective。
7. 论文定义了 set value `V^sharp_pi(s; f)` 和 set Q-function `Q^sharp_pi(s, a_1:n; f)`。
8. 理论上，他们把 state visitation 看成一棵每个状态分出 `n` 个动作的树。
9. 在这个树设定下，作者证明了 set-RL 版本的 performance difference lemma。
10. 这个 lemma 给 Poly-PPO 提供了合理性：如果新策略在访问状态上的 set advantage 为正，set value 会提升。
11. 实际目标叫 **polychromic objective**，用于同时奖励成功和多样性。
12. 具体形式是 `f_poly(s, tau_1:n) = (1/n) * sum_i R(tau_i) * d(s, tau_1:n)`。
13. `R(tau_i)` 是 trajectory return，`d(s, tau_1:n)` 是集合内轨迹多样性，二者都归一化到 0 到 1。
14. 这个乘积形式很重要：只有成功但同质的 set、或多样但全失败的 set，都不会被高分鼓励。
15. 因为 set 内共享 advantage，某些还没有直接产生 reward 但贡献多样性的轨迹也能得到正向梯度。
16. 实用算法是 **Polychromic PPO / Poly-PPO**。
17. Poly-PPO 相对 PPO 有两处核心改动：数据采样改为 vine sampling，advantage 改为 set-level polychromic advantage。
18. vine sampling 流程：先从 `pi_beta` roll out 若干主轨迹。
19. 然后从每条主轨迹中选 `p` 个 rollout states。
20. 在每个 rollout state 上重置环境并采样 `N` 条 vine trajectories。
21. 论文主实验使用等间隔选点：如果主轨迹长度为 `T`，选 `T/(p+1), 2T/(p+1), ...`。
22. 对每个 rollout state，作者从 `N` 条 vine trajectories 里构造多个 size `n` 的 trajectory sets。
23. 每个 set 的 score 是 `f_poly`。
24. baseline 是同一个 rollout state 上 `M` 个 set score 的平均值。
25. set 内 trajectory 上从 rollout state 开始的动作都拿到同一个 advantage：`score(g_i) - mean_j score(g_j)`。
26. 非 rollout state 仍然使用标准 PPO 的 GAE advantage。
27. 作者还加入每状态 KL penalty `KL(pi_beta(.|s) || pi_theta(.|s))` 来稳定训练。
28. BabyAI/MiniGrid 中使用一个 polychromic window：不只更新 rollout state `s_t`，还把 `s_t...s_{t+W}` 的 advantage 设为 polychromic advantage。
29. 这个 window 的目的，是防止策略在 rollout state 探索，但之后状态又被标准 PPO 拉回 exploitative 行为。
30. Algorithmic Creativity 中 `W=0`，因为图上路径一旦分叉，不容易重新合并。

### 关键技术细节

31. set size `n=4`，每个 rollout state 的 vine 数 `N=8`。
32. 每个 rollout state 构造 `M=4` 个 trajectory sets。
33. 每条主轨迹选 `p=2` 个 rollout states。
34. 总 trajectory budget 设为 `B=136`，满足 `N + N^2(p - 1) <= B`。
35. BabyAI/MiniGrid 的 polychromic window `W=5`。
36. Algorithmic Creativity 的 polychromic window `W=0`。
37. PPO epochs 为 2，minibatch size 为 64。
38. discount `gamma=1.0`，GAE `lambda=0.95`。
39. PPO clip `epsilon=0.2`。
40. actor learning rate 为 `1e-5`，critic learning rate 为 `1e-4`。
41. value loss coefficient 为 `0.5`。
42. KL coefficient sweep 为 `{0.005, 0.01, 0.05, 0.1}`。
43. max grad norm 为 `0.5`，temperature 为 `1.0`。
44. BabyAI policy 输入 grid image、agent direction embedding、mission text；mission 用 GRU 编码。
45. BabyAI policy 是 CNN-GRU，并输出 BabyAI/MiniGrid 标准离散动作 logits。
46. MiniGrid FourRooms 没有 mission text，使用 compact MLP 处理 flattened image observation。
47. BabyAI/MiniGrid 动作空间包括 left、right、forward、pickup、drop、toggle、done。
48. 预训练使用专家 demonstrations；Goto/Pickup 等任务用 80/20 train-test split。
49. Synthseq 和 BossLevel 使用完整数据集，因为作者认为需要全量数据才能得到足够强的 base policy。
50. RLFT 在 50 个固定 configurations 上进行。
51. 除 BossLevel 和 Synthseq 外，RLFT configurations 来自 pretraining test set。
52. BabyAI/MiniGrid reward：如果 horizon `H=100` 内到达目标，reward 为 `1 - 0.5 * t/H`，否则为 0。
53. 作者特意把默认时间惩罚从 `0.9` 降到 `0.5`，因为较轻的 time penalty 对 diversity 更友好。
54. Algorithmic Creativity 使用 Triangle Discovery：agent 输出未观察图中的 edge token 序列，目标是构成有效三角形。
55. Triangle Discovery 最大序列长度为 11，动作 vocabulary size 为 1017。
56. Triangle Discovery 预训练模型是 decoder-only Transformer，loss 是 masked cross-entropy。
57. Triangle Discovery 预训练数据来自 10 个图，每图 15,000 samples，每个图有 999 nodes。
58. Algorithmic Creativity RLFT 在 3 个固定图上做，reward 是 sparse `+1` valid triangle，否则 0。
59. BabyAI/MiniGrid 的 diversity function：两条轨迹访问的 room set 不同，则视作 semantically distinct。
60. Algorithmic Creativity 的 diversity function：两条轨迹访问的 node set 不同，则视作 distinct。
61. 如果 set 内所有 trajectories 访问同一组 rooms 或 nodes，则 `d=0`。
62. 理论分析限制在 horizon `H=1`、binary rewards、discrete actions、softmax policy。
63. 理论中作者引入 scaffold value `Lambda_f(a_1:n; pi)`，用来刻画某个 action set 吸引概率质量的倾向。
64. 他们证明 homogeneous successful sets 在概率足够高时 scaffold value 变负，因而 polychromic objective 会抑制继续坍缩。
65. 他们也证明包含成功动作的 heterogeneous sets 有正 scaffold value，因此会吸引概率质量。

### 核心结果

66. BabyAI/MiniGrid 平均 reward 和 success rate 上，Poly-PPO 基本匹配或超过最佳 baseline。
67. Goto：Poly-PPO 达到 `(0.575, 80.2%)`，高于 REINFORCE `(0.533, 73.0%)` 和 PPO `(0.406, 46.2%)`。
68. Pickup：Poly-PPO+UCB 达到 `(0.486, 65.6%)`，Poly-PPO 为 `(0.452, 63.2%)`，明显高于 PPO `(0.283, 33.4%)`。
69. Synthseq：REINFORCE+UCB `(0.361, 47.8%)` 最强，Poly-PPO `(0.341, 47.0%)` 很接近；这不是全任务无条件胜出。
70. BossLevel：Poly-PPO+UCB `(0.379, 46.8%)`，Poly-PPO `(0.378, 45.2%)`，高于 PPO `(0.336, 38.8%)`。
71. MiniGrid FourRooms：REINFORCE+UCB `(0.672, 92.6%)` 略高，Poly-PPO `(0.666, 92.4%)` 基本持平。
72. pass@k 是论文最有说服力的结果，因为它测的是多次尝试下覆盖多少 configurations。
73. baseline RL 方法会在小 k 上提升成功率，但大 k 时可能低于 pretrained policy，说明 RLFT 丢掉了 pretrained diversity。
74. Poly-PPO 在 BabyAI pass@k 上显著高于 baselines，并且几乎所有 k 上不低于 pretrained policy。
75. 作者报告 Poly-PPO 的 pass rate 约到 `k=80` 仍继续上升，而 baselines 大多在 `k=20` 左右饱和。
76. BossLevel 中 Poly-PPO 在大 k 处约高出 baselines 15 个百分点，这是 diversity preservation 的强证据。
77. 加 UCB 后，REINFORCE 和 PPO 在部分环境有改善，但收益常较早饱和。
78. Poly-PPO+UCB 在多数 k 上进一步保持或提高 coverage，说明 UCB 与 polychromic objective 可互补。
79. Algorithmic Creativity 中，PPO 提高 validity，但降低 creativity 和 diversity。
80. Poly-PPO 的 validity 略低于 PPO，但 diversity 与 creativity 高于所有 baseline，包括 pretrained policy。
81. 这是一个反直觉点：更强的单次 valid rate 不等于更强的 creative/pass@k 覆盖。
82. 初始状态扰动泛化实验中，Poly-PPO 表现很强。
83. Goto perturbed pass@1：Poly-PPO `60.6%`，pretrained `30.2%`，REINFORCE `41.3%`，PPO `21.1%`。
84. Pickup perturbed pass@1：Poly-PPO `33.4%`，pretrained `15.2%`，REINFORCE `22.0%`，PPO `12.5%`。
85. Synthseq perturbed pass@1：Poly-PPO+UCB `32.1%`，Poly-PPO `30.6%`，pretrained `20.0%`。
86. BossLevel perturbed pass@1：Poly-PPO `34.3%`，PPO+UCB `28.2%`，pretrained `23.8%`。
87. FourRooms perturbed pass@1：Poly-PPO `88.7%`，REINFORCE `82.7%`，PPO `15.3%`。
88. 泛化实验支持作者主张：学习多样行为不只是提高 pass@k，也能抵抗 state-visitation distribution shift。

### 明确局限

89. 最大工程限制是 vine sampling 需要能把环境 reset 到访问过的中间状态。
90. 如果不能精确 reset，作者要求 MDP 足够 deterministic，使得 replay actions 能近似回到同一状态。
91. 对很长 horizon，保证 sufficient vine coverage 会困难。
92. diversity function 是手工设计的：rooms/nodes 这种离散环境容易，连续控制或开放式语言语义难得多。
93. polychromic advantage 用 Monte Carlo baseline，作者承认在别的 setting 可能高方差。
94. 实验没有展示真实机器人或真实用户环境，主要是可重置模拟器和算法任务。
95. 论文动机强烈面向 LLM RLFT，但主实验不是大规模 LLM reasoning benchmark。
96. Algorithmic Creativity 是有 verifier 的离散图任务，不能直接代表开放式创意生成。
97. 理论分析做了 `H=1`、binary reward、discrete action 的强简化。
98. 论文没有从 tex 中看到代码 release 或第三方复现实验。
99. 与 UCB 的比较相对充分，但没有系统比较 learned diversity metric 或 embedding-level diversity metric。
100. 对实际 RLHF/RLAIF 的 reward model 噪声、preference ambiguity、verifier bias 讨论不足。

## Read by: 0616-fex (2026-06-17)

这篇论文对 `0616-fex` 的价值主要在于：它把“RLFT 后多样性坍缩”从经验观察提升成了一个可优化目标，并给出可实现的 PPO 改造。如果我们的方向涉及探索、覆盖率、多样化策略、agent fine-tuning 或 pass@k/test-time compute，这篇必须作为核心 prior 处理。

撞车风险集中在四个 claim：第一，使用 trajectory set 而不是 single trajectory 作为 RL objective；第二，用 reward 与 semantic diversity 的乘积定义 set score；第三，把同一个 set-level advantage 分配给集合内多条轨迹；第四，用 vine sampling 在中间状态构造多条 on-policy rollouts。只要 `0616-fex` 也在讲这些点，就不能再声称方法主线新颖。

可复用方法包括：pass@k 作为 coverage 指标、pretrained-vs-RLFT 在大 k 的反转诊断、room/node set diversity 的语义多样性定义、initial-state perturbation 泛化测试、以及 UCB bonus 与 diversity objective 互补的实验设计。差异化最好放在本文弱点上：不要求环境 reset、使用 learned/verifier diversity、处理 reward uncertainty、落地到真实 LLM reasoning 或 tool-use agent，并引入独立 verifier。
