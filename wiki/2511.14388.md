# Integral Bayesian symbolic regression for optimal discovery of governing equations from scarce and noisy data (2511.14388)

- **Year**: 2025.11
- **Authors**: Oriol Cabanas-Tirapu, Sergio Cobo-Lopez, Savannah E. Sanchez, Forest L. Rohwer, Marta Sales-Pardo, Roger Guimera; Universitat Rovira i Virgili, Virginia Commonwealth University, San Diego State University, ICREA
- **Venue**: arXiv
- **Keywords**: integral Bayesian symbolic regression, governing equation discovery, Bayesian Machine Scientist, noisy time series, numerical integration, MCMC, SINDy, bacterial growth
- **One-liner**: 用积分贝叶斯符号回归从稀疏噪声时序发现微分方程。

## 精华提炼

### 管线/方法

1. 输入是原始时间序列观测 `D = {(t_k, x_k)}`，核心约束是只观测坐标、不观测导数。
2. 传统符号回归式的 ODE 发现通常先估计 `dx/dt`，再拟合 `dot{x} = f(x, theta)`。
3. 本文认为这个中间导数估计会放大噪声，并引入 spurious correlations。
4. I-BMS 直接在候选微分方程 `f_i(x, theta)` 上工作，但评估时比较其积分轨迹。
5. 对任意候选 `f_i`，先数值积分得到 `F_i(t, theta, x_0)`。
6. `x_0` 被当成积分形式的额外参数一起优化。
7. 候选模型的后验写成 `p(f_i | D) = p(F_i | D) = exp[-L(F_i,D)] / Z`。
8. 描述长度近似为 `L(F_i,D) = B(F_i,D)/2 - log p(f_i)`。
9. `B(F_i,D)` 是把积分轨迹和原始坐标观测比较得到的 BIC。
10. 先验 `p(f_i)` 仍然作用在微分方程表达式上，而不是作用在积分后的黑箱曲线上。
11. 这点保留了符号表达式的可解释性，也避免把导数估计误差混入 likelihood。
12. 模型选择使用 MAP，也就是最小描述长度模型。
13. 对 SINDy 类基线，作者在相同多项式库内做公平比较。
14. 在该受限空间中，I-BMS 穷举评估所有允许的线性组合，并选择积分描述长度最小者。
15. 对更开放的表达式空间，作者使用 MCMC 采样任意符号表达式，而不限定为预定义 library 的线性组合。
16. MCMC 每步修改表达式树，再用积分形式重算参数和描述长度。
17. 采样中使用 parallel tempering，减少符号表达式空间局部最优问题。
18. 对二维 Lotka-Volterra，两个微分方程耦合，参数估计和 SSE 计算要同时考虑两个变量。
19. 更新二维系统时，先更新一个变量的表达式，再更新另一个变量，但接受后会刷新两个方程的最优参数。
20. 对细菌生长应用，作者额外加入 physics-informed 搜索偏置。
21. 第一种细菌设置不约束模型形式。
22. 第二种设置强制包含线性增长项 `dot{B} = rB + g(B)`。
23. 第三种设置把增长率本身写成种群大小函数 `dot{B} = alpha B h(B)`。
24. 最终仍按 description length 在候选 run 中选最合理的 governing equation。

### 关键技术细节

25. 合成 benchmark 包含一维 logistic equation 和二维 Lotka-Volterra system。
26. Logistic 的积分真值为 `x(t)=A/(B+exp(C-Dt))`。
27. Logistic 参数为 `A=1, B=1, C=0, D=1`。
28. Logistic 默认在 `t in [-6,6]` 上以 `delta t=0.1` 采样，得到 `N=120`。
29. 每个 Logistic 噪声水平生成 40 个数据集。
30. 固定噪声、改变样本数的 Logistic 实验中，图注给出的固定噪声为 `sigma=0.05`。
31. Lotka-Volterra 真值为 `dot{x}=ax-bxy`, `dot{y}=dxy-cy`。
32. Lotka-Volterra 参数为 `a=0.1, b=0.02, c=0.02, d=0.4`。
33. Lotka-Volterra 从 `t_0=0` 积分到 `t_1=80`，时间步长 `delta t=0.1`。
34. Lotka-Volterra 初值为 `x_0=10, y_0=5`。
35. Lotka-Volterra 每个噪声水平也生成 40 个数据集。
36. 图注中 Lotka-Volterra 固定点数噪声实验使用 `N=180`，固定噪声变点数实验使用 `sigma=3.5`。
37. FD-BMS 的导数估计使用 central difference。
38. 首尾点分别使用 forward difference 和 backward difference。
39. SD-BMS 使用 PyNumDiff，通过二阶多项式和 21 点窗口平滑后求导。
40. MCMC 的表达式树操作包括 `exp()`, `pow2()`, `pow3()`, `-`, `+`, `*`, `/`, `**`。
41. Parallel tempering 使用 `T_0=1`，并给副本设置 `T_k=1.02^k, k in [1,20]`。
42. Learnability 图注说明：Logistic 和常规 Lotka-Volterra MCMC 每个数据集跑两个独立链、每链 3,000 steps。
43. I-BMS 在 Lotka-Volterra 上使用 4,000 steps。
44. 细菌数据来自 6 个物种：`C. sedlakii`, `E. coli`, `E. aerogenes`, `K. pneumaniae`, `P. vulgaris`, `S. aureus`。
45. 细菌实验持续 30 到 31 小时。
46. 观测是 `OD_600`，每 10 到 15 分钟测量一次。
47. 每个物种原始有 96 种 substrate，但只保留有明显增长的 substrate。
48. 六个物种分别保留 29、40、24、35、28、13 个 substrate。
49. 训练集是五个物种实验的一半，随机选择。
50. 测试集包含这五个物种的另一半实验，以及剩余一个物种的全部实验。
51. 细菌应用中每种搜索偏置跑 3 个独立 sampling processes，每个 3,000 MCMC steps。
52. 细菌增长基线是 logistic growth model 和 Gompertz model。
53. Logistic 基线为 `dB/dt = rB(1-B/K)`。
54. Gompertz 基线为 `dB/dt = rB log(K/B)`。
55. 本文发现的 I-BMS 细菌模型为 `dB/dt = B r (1 + c0 (c1 B exp(c2 B))^(B^3))`。
56. 该模型的 `r, c0, c1, c2` 对每个 bacteria-substrate 数据集不同。
57. 作者报告 `c0` 和 `r` 符号相反，因此模型形式上包含增长项和竞争/抑制项。
58. 该模型有 carrying capacity，但不能写成参数的简单组合。

### 核心结果

59. 在受限多项式 library 中，W-ESINDy 和 ESINDy 可近似轨迹，但当 library 超过二次项后很少精确找回真方程。
60. FD-BMS 表现最差，对噪声和数据稀缺最敏感。
61. SD-BMS 比 FD-BMS 更抗噪，但在低噪声或数据充足时会因过度相信平滑后的数据而过拟合。
62. I-BMS 在这些比较中表现最好，作者概括为最大抗噪性和最低数据点需求。
63. 在开放表达式空间的 learnability 实验中，非积分 BMS 在较低噪声下就会让 ground truth 变成 unlearnable。
64. 平滑导数虽然提升高噪声韧性，但降低低噪声 learnability。
65. I-BMS 使 ground truth 在更高噪声水平下仍保持 learnable。
66. 对预测任务，作者用 `RMSE / sigma = 1` 作为噪声下近似最优预测的标准。
67. I-BMS 的预测在所有噪声水平上接近该最优行为。
68. 非积分方法在 learnable/unlearnable 转换区附近预测尤其差。
69. 细菌应用中，I-BMS 模型能覆盖 lag phase、growth phase 和 stationary phase。
70. 作者强调其能表示 double ramp-ups、更快/更慢增长、非标准 lag/growth transition。
71. I-BMS 没有用导数训练，却能给出合理的经验导数曲线。
72. Logistic 和 Gompertz 常在初始阶段和指数增长阶段拟合不足。
73. I-BMS 在训练和测试的 description length 上优于 logistic 和 Gompertz。
74. RMSE 也总体更好，但测试集上相对 logistic 的 RMSE 差异不显著。
75. 论文主文没有给出图中检测率、learnability 或 RMSE 的精确数值表，只给出曲线和定性比较。

### 明确局限

76. 这是 arXiv 预印本，主文没有会议或期刊接收信息。
77. 本次 arxiv source 中 `find` 只发现 `main.tex`，虽然主文引用了 `SI`，但源码包中没有可读的 `SI.tex`。
78. 关键 performance 数字多数在图中，没有主文表格给出精确检测率或置信区间。
79. 代码和数据声明在 GitHub 可用，但本次任务按规则没有下载或验证仓库。
80. 合成系统只覆盖 Logistic 和 Lotka-Volterra，不能证明对高维、stiff、partial observation 或 chaotic 系统同样稳定。
81. I-BMS 每个候选表达式都要数值积分和参数优化，计算成本可能明显高于导数回归式方法。
82. MCMC 的 3,000 到 4,000 step 规模能展示效果，但不是全局最优或充分混合的证明。
83. 细菌应用的真实测量是 `OD_600`，它是 population proxy，不是直接的生物机制观测。
84. 细菌 substrate 经过“有明显增长”筛选，结论不覆盖弱增长、无增长或复杂群落场景。
85. 发现的细菌方程虽然闭式，但竞争项比较复杂，生物机理解释仍偏 phenomenological。
86. 参数按 bacteria-substrate 数据集变化，说明共享的是形式，不是统一机制参数。
87. 测试 RMSE 相对 logistic 不显著，说明新增复杂度在泛化误差上不是所有指标都压倒性胜出。
88. 验证主要由同一作者团队完成，没有独立复现或独立实验 veto。
89. 对神经 ODE、深度符号回归、最新 foundation model 式 equation discovery 的横向比较在主文中不充分。
90. 作者承认实践中真实模型先验 `p(f)` 不知道，所以 Bayes optimal 是在先验假设意义下的。

## Read by: 0616-fex (2026-06-17)

从 `0616-fex` 视角看，这篇论文的核心不是提出一个新的符号回归语法，而是把 ODE 发现的评分面从导数空间移回原始轨迹空间。它最有价值的 claim 是：候选方程可以很复杂，但验证必须通过数值积分后和观测轨迹比较，因此避免把有限差分或平滑导数的偏差当作数据事实。

如果我们的课题也要从稀疏、带噪时间序列里发现 governing equations，那么本文高度撞车。尤其是“积分式验证 + Bayesian description length + MCMC expression search”这组三件套已经覆盖了一个完整方法闭环。我们不能再把“避免导数估计”作为单独新意，只能把它作为 baseline 或组件。

可用方法上，本文给了一个很清晰的验证器设计：对每个候选表达式优化参数和初值，积分得到轨迹，用 BIC 和表达式先验算 description length，再用 MAP 选模型。这个验证器可以接到 LLM/程序搜索/遗传编程的候选生成器后面，形成 generation 与 verification 分离的 pipeline。

差异化上，`0616-fex` 更应该强调本文没覆盖的部分：候选生成器是否更高效，是否支持 partial observation 或高维系统，是否能做主动采样，是否有独立 verifier veto，是否能输出 calibrated posterior/credible intervals，是否在公开 benchmark 上系统比较最新 neural/symbolic 方法。若应用到细菌增长或类似生物时，还需要补真实机制验证，否则容易停留在 phenomenological curve fitting。

结论：这篇必须引用。它是“积分式符号 ODE 发现”方向的近邻 prior art，对任何声称在 noisy/scarce trajectories 上做 equation discovery 的工作都有直接约束。
