# Convergence and Sample Complexity of First-Order Methods for Agnostic Reinforcement Learning (2507.04406)

- **Year**: 2025.07
- **Authors**: Uri Sherman (Tel Aviv University), Tomer Koren (Tel Aviv University / Google Research), Yishay Mansour (Tel Aviv University / Google Research)
- **Venue**: arXiv
- **Keywords**: agnostic reinforcement learning, policy optimization, variational gradient dominance, policy mirror descent, conservative policy iteration, Frank-Wolfe, sample complexity, non-Euclidean optimization
- **One-liner**: 用 VGD 假设给 agnostic RL 的一阶策略优化建立收敛和采样复杂度界。

## 精华提炼

### 管线/方法

1. 论文研究的是 agnostic policy learning: 给定策略类 `Pi`, 目标不是找到全局最优策略, 而是找到接近 `Pi` 内最优策略的 policy。
2. 关键区别是: 不假设 `Pi` 包含最优策略, 因此不是 realizable RL, 而是 best-in-class learning。
3. 作者把策略学习问题改写为 policy functional space 上的一阶优化问题, 优化变量是每个状态上的 action distribution。
4. 基础等式来自 policy gradient theorem: `E_{s ~ mu^k}[H <Q^k_s, pi_s>] = <grad V(pi^k), pi>`。
5. 每轮用 on-policy rollout 得到 `s ~ mu^k` 和无偏 `Q^k_s` 估计, 形成经验 surrogate objective。
6. 通用 update 模板是最小化 `hat Phi_k(pi) = (H/N) sum_s <hat Q^k_s, pi_s> + proximity/divergence term`。
7. proximity/divergence term 可以是平方范数、Bregman divergence, 也可以为 0 后接 Frank-Wolfe/CPI step。
8. 为了使用 local smoothness, 算法通常在 `Pi^eps = (1-eps)Pi + eps uniform` 这个探索化策略类中运行。
9. 理论的核心假设是策略类满足 `(c_vgd, eps_vgd)` variational gradient dominance。
10. VGD 形式为 `V(pi)-V*(Pi) <= c_vgd max_{tilde pi in Pi}<grad V(pi), pi-tilde pi> + eps_vgd`。
11. VGD 比 completeness + coverage 更弱, 因而能覆盖一部分非 realizable / agnostic setting。
12. 论文把 completeness/coverage 解释为 VGD 的充分条件: coverage 下 `c_vgd = H D_infty`, 近似 completeness 进入 `eps_vgd`。
13. 作者构造了三个主要算法族: SDPO, CPI/DA-CPI, PMD。
14. SDPO 来自 constrained steepest descent, 用非欧几里得 action norm 的平方距离做 proximal term。
15. CPI 被重新解释为 policy space 上的 Frank-Wolfe: 先找 greedy/improvement direction, 再做 convex combination。
16. DA-CPI 额外用第二个 oracle 近似 convex combination step, 避免 CPI 需要保存线性数量 actor。
17. PMD 被作为 Bregman proximal point / mirror descent 的策略优化实例重新分析。
18. 论文的 pure optimization 部分先抽象出 compact convex set `X`, local norm, local smoothness, local Lipschitz, VGD。
19. 对 constrained steepest descent, 作者引入 potential gradient mappings `G_x = {1/eta (x-y): y in X}`。
20. 再定义 constrained steepest descent magnitude, 用它连接下降量和 Frank-Wolfe 风格的 stationarity gap。
21. 对 Frank-Wolfe, 作者给出 approximate FW 在 VGD 目标上的收敛界, 不依赖目标凸性。
22. 对 DA-FW, 作者加入第二个近似投影/组合步骤, 并在 local norm 下控制误差传播。
23. 对 Bregman proximal point, 作者把函数值近似最优转成近似一阶 optimality condition, 再套已有 PMD/VGD 分析。
24. 采样部分用一个 action-value estimation sampler: 先按 `pi` rollout, 每步以 `1-gamma` 接受当前状态, 从而得到 `s ~ mu^pi`。
25. 接受状态后对动作做 uniform exploration, 再 rollout 到随机终止, 构造 `hat Q` 的重要性采样估计。
26. 经验目标到 population 目标的统一集中界依赖 `Pi` 在 `||.||_{infty,1}` 下的 covering number。
27. 整体样本复杂度不直接依赖状态空间基数 `S`, 而是依赖 policy class covering number。

### 关键技术细节

1. 折扣 MDP 记号: `H = 1/(1-gamma)`, `A = |A|`, `S = |S|`; 主文为简化先写有限状态空间。
2. SDPO update: `argmin_{pi in Pi^eps} E_{s~mu^k}[H<Q^k_s,pi_s> + (1/(2eta))||pi_s-pi^k_s||_circ^2]`。
3. SDPO theorem 使用 `L1` action norm, 要求 `Pi` convex 且满足 VGD。
4. SDPO ideal iteration bound: `O(c_vgd^2 A H^4 / K^{2/3} + c_vgd H^3 sqrt(A) K^{1/6} sqrt(epsilon) + eps_vgd)`。
5. CPI update 先解 `argmin_{pi in Pi} E_{s~mu^k}<H Q^k_s, pi_s-pi^k_s>`, 再设 `pi^{k+1}=(1-eta_k)pi^k+eta_k tilde pi^{k+1}`。
6. CPI 被证明等价于 approximate Frank-Wolfe, 使用 `||.||_{infty,1}` 下 `2H^3` smoothness。
7. CPI theorem 不要求 `Pi` convex, 采用 `eta_k = 2 c_vgd/(k+2)`。
8. CPI bound: `V(pi^K)-V*(Pi) <= 8(2c_vgd^2+1)H^3/K + 2c_vgd epsilon + eps_vgd`。
9. DA-CPI 第一 oracle 找 greedy policy, 第二 oracle 近似 `(1-eta)pi^k + eta tilde pi`。
10. DA-CPI 使用 `L1` action norm, bound 为 `O(c_vgd^2 A H^3(1/K^{2/3}+epsilon^{1/3}+epsilon^{2/3}K^{2/3}))`。
11. PMD 使用 action regularizer 的 Bregman divergence; 欧氏 `L2` regularizer 下给出 agnostic sample-complexity bound。
12. PMD theorem bound: `O(c_vgd^2 A^{3/2}H^3/K^{2/3} + (c_vgd + H^2 A K^{1/6}) epsilon^{1/4})`。
13. SDPO 相比 PMD 的主要理论改进是 error dependence 从 `epsilon^{1/4}` 改为 `sqrt(epsilon)`。
14. SDPO/DA-CPI 在 `L1` action norm 下也改善了对 action cardinality `A` 的依赖。
15. Sample complexity 里假设 `gamma <= 1/2`, 这看起来是为了简化 sampler 的 sub-exponential tail bound。
16. SDPO learning bound: `O(c_vgd^2 A^2 H^7 sqrt(log(n C(Pi)/delta))/n^{2/15} + c_vgd H^3 sqrt(A)n^{1/30}sqrt(eps_erm) + eps_vgd)`。
17. CPI learning bound: `O(c_vgd^2 A H^4 log(n C(Pi)/delta)/n^{1/3} + eps_vgd)`。
18. DA-CPI learning bound: `O(c_vgd^2 A^2 H^5 sqrt(log(n C(Pi)/delta))/n^{2/15} + eps_vgd)`。
19. PMD learning bound: `O(c_vgd^2 A^3 H^7 sqrt(log(n C(Pi)/delta))/n^{2/15} + eps_vgd)`。
20. 实验评估的不是最终算法 SOTA, 而是 VGD condition 的经验合理性。
21. 实验环境: Cartpole-v1, Acrobot-v1, SpaceInvaders-MinAtar, Breakout-MinAtar。
22. 实验代码基于 Gymnax, 部分基于 purejaxrl。
23. 每个环境跑 10 个 seed, error bar 是最大/最小值。
24. 实验维护两个 actor: 一个 SDPO actor, 一个 VGD actor。
25. VGD actor 用来估计 `max_{tilde pi in Pi}<grad V(pi^k), pi^k-tilde pi>`。
26. VGD ratio 估计为 `(V(pi^k)-V_hat*) / <hat grad V(pi^k), pi^k-tilde pi^{k+1}>`。
27. VGD actor 使用 AdamW, step size `5e-4`, `n_epochs_vgd=100`, `n_mbs_vgd=4`。
28. Cartpole: `K=40`, `N=4x500`, `n_epochs=100`, `n_mbs=4`, `n_rep=5`, `eta_opt=2e-4`, `eta=0.01`。
29. Acrobot: `K=100`, `N=8x500`, `n_epochs=100`, `n_mbs=4`, `n_rep=20`, `eta_opt=4e-4` with linear annealing, `eta=0.1`。
30. SpaceInvaders-MinAtar: `K=600`, `N=16x1000`, `n_epochs=4`, `n_mbs=8`, `n_rep=5`, `eta_opt=5e-3` with linear annealing, `eta=0.1`。
31. Breakout-MinAtar: `K=200`, `N=16x1000`, `n_epochs=100`, `n_mbs=8`, `n_rep=5`, `eta_opt=5e-3` with linear annealing, `eta=0.1`。
32. MinAtar 每 seed 运行约 90-120 分钟, GPU 为 NVIDIA RTX A5000; Cartpole/Acrobot 10 seeds 总计低于 20 分钟。

### 核心结果

1. 论文给 agnostic RL 提供了一个统一的一阶优化框架, 把策略学习和非欧几里得 constrained smooth optimization 对齐。
2. 理论上, VGD 可以替代强 completeness/coverage 假设, 至少在 best-in-class 目标上得到有意义的收敛界。
3. SDPO 是新算法, 其核心收益是把 PMD agnostic bound 的 oracle/generalization error 项从 `epsilon^{1/4}` 改到 `sqrt(epsilon)`。
4. CPI 的重新解释很干净: 原始 conservative policy iteration 本质上是 policy space 的 Frank-Wolfe。
5. CPI 用 `eta_k ~ 1/k` 可从经典 `O(1/sqrt(K))` 型复杂度改善到 `O(1/K)` iteration complexity。
6. DA-CPI 解决了原始 CPI 的工程硬伤: 原始 CPI 的 mixture policy 需要保留历史所有 actors, actor memory 随 K 增长。
7. DA-CPI 通过第二个 oracle 把 mixture actor 压回一个 actor, 因此 actor-oracle efficient。
8. PMD 的新分析给出与 policy class 参数化无关的 sample complexity, 这比只分析特定 log-linear/NN 参数化更通用。
9. 实验上, 四个环境中的估计 VGD ratio 保持 moderate, 并在收敛附近下降到约 1 或以下。
10. Cartpole/Acrobot 中作者认为算法基本收敛到全局最小值; MinAtar 中存在 local optima, 由 `eps_vgd` 解释。
11. 表格对比显示本文算法全部支持 general parameterization 和 agnostic guarantee; SDPO/DA-CPI/PMD actor-oracle efficient, 原始 CPI 不是。
12. 反直觉点: 论文不是追求新的深度 RL empirical SOTA, 而是说明一个较弱、可测的 landscape 假设可以解释 policy optimization 收敛。
13. 另一个反直觉点: CPI 这个老算法的较差 rate 很大程度来自 step-size/分析视角, 而不是算法本身必然慢。

### 明确局限

1. VGD 仍是强 landscape 假设; 论文展示它比 completeness/coverage 弱, 但没有证明常见神经策略类普遍满足。
2. SDPO/DA-CPI/PMD 的主要定理要求 `Pi` convex; 神经网络参数化策略类通常不是 convex。
3. CPI 虽然不要求 convex `Pi`, 但原始 CPI actor-oracle inefficient, 工程上不适合维护大量 actor mixture。
4. 优化 oracle access 是抽象假设; 实际神经网络训练只会给近似解, 且误差项可能很大。
5. Sample complexity bound 指数如 `n^{-2/15}` 很弱, 更像理论可行性证明, 不是实用样本效率承诺。
6. 主文为简化先写 finite state space; 虽说可扩展到 infinite state space, 但需要额外技术。
7. Sample complexity 部分假设 `gamma <= 1/2`, 折扣接近 1 的长期任务需要额外处理。
8. 实验只是 VGD condition 的 proof-of-concept, 环境数量少, 没有标准深度 RL 大规模 benchmark。
9. 实验把同一 rollout 中的状态近似当成独立样本, 作者自己在伪代码注释中说明这不是严格独立。
10. 实验没有开源 artifact 信息; 可复现性主要来自超参数描述和论文公式。
11. 对符号回归/FEX 没有直接实验, 没有表达式树、PDE residual、BFGS 内层优化或 candidate operator library。
12. 对离散结构搜索中的 sparse/tail reward, policy collapse, 等价表达式重复等问题没有覆盖。

### 跟我们的关系

1. 对 `0616-fex`, 本文不直接撞车: 它不做 FEX、符号回归、PDE 求解、表达式树搜索或 LLM+SR。
2. 但它高度相关于 `0616-fex` landscape 中的最大理论缺口: FEX 的 RL controller 搜索是否有收敛率/采样复杂度。
3. 可借用的核心工具是 VGD framing: 不要求策略类 realizable, 只需证明 best-in-class suboptimality 被某种 policy-gradient gap 控制。
4. FEX controller 可以被看成生成表达式树的 policy; best-in-class 可以对应固定深度/固定 operator set 下最优表达式分布。
5. 如果能定义 FEX reward/objective 的 policy-gradient theorem analog, 本文的 SDPO/PMD/FW 框架可作为理论蓝图。
6. FEX 的困难比本文更尖锐: reward 来自 PDE residual + 内层连续参数优化, 不像 MDP value function 那样天然有 occupancy measure 和 Q function。
7. FEX 的策略类通常是神经网络 over tree tokens, 非 convex; 本文的 convex `Pi` 假设无法直接套。
8. FEX 搜索还存在等价表达式重复、稀疏高分尾部、operator library mismatch, 这些都不在本文理论中。
9. 若要做 FEX 搜索复杂度理论, 可以先尝试一个 relaxed/mixture expression distribution, 把表达式树分布空间 convex 化。
10. 另一条路线是把 FEX controller 视作 bandit / contextual bandit 而非 MDP, 只借用本文的 VGD/error-floor 思想。
11. 对 “RL+表达式树搜索 PDE 解” 的工程 claim, 本文不构成竞品; 对 “FEX RL 搜索有理论收敛保证” 的理论 claim, 本文是必须引用的邻近理论。
12. 补充 novelty 检查: 用 `arxiv_tool.py` 对 2026 S2 做 policy-gradient/sample-complexity 查询, 返回的是一般 RL/control/MDP 理论, 没看到直接覆盖 FEX/RL-symbolic PDE 搜索复杂度的工作; arXiv 源检索当时被 429/503 限流。
