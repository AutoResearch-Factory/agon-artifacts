# Influence-Guided Symbolic Regression: Scientific Discovery via LLM-Driven Equation Search with Granular Feedback (2605.29184)

- **Year**: 2025.05
- **Authors**: Evgeny S. Saveliev*, Samuel Holt* (equal contrib.), Nabeel Seedat, David L. Bentley, Jim Weatherall, Mihaela van der Schaar; University of Cambridge, Thomson Reuters, U. Colorado Anschutz, AstraZeneca
- **Venue**: ICML 2026 (accepted)
- **Keywords**: symbolic regression, LLM, MCTS, influence scores, per-term feedback, interpretable models, scientific discovery, equation discovery
- **One-liner**: 用 LLM 提 basis function + per-term influence score 做 granular 反馈 + MCTS 搜索，发现可解释闭式方程，并通过湿实验验证了一个新的 DNA 甲基化假说。

## 精华提炼

### 管线/方法

IGSR 的核心 pipeline 如下：

**输入**：数据集 (train/val/test) + 科学问题文本描述 + 变量语义
**输出**：稀疏线性方程 f(x) = Σ w_j ψ_j(x)

**Propose-and-Prune Cycle（核心迭代步骤）**：
1. **Propose（LLM 生成阶段）**：一个 LLM agent（"Propose" agent）基于以下 context 生成 ~5 个新候选 basis function ψ_j(x)（首轮 10 个）：
   - 数据集和问题描述（包括变量语义）
   - 当前活跃的 terms 集合
   - 迄今最优方程
   - 历史记录（之前哪些 terms 被保留/丢弃及对应 MSE 变化）
   - 数据预览片段

2. **Fit（线性回归拟合）**：新提出的 terms 与已有 terms 合并，用 OLS 在训练集上拟合权重 w。

3. **Score（Per-term Influence 计算）**：对每个 term ψ_j 在验证集上计算 influence score Δ_j = MSE_{-j} - MSE_full，即将 w_j 置零（其他权重不变）后验证集 MSE 的增量。Δ_j 大 → term 重要；Δ_j ≈ 0 → term 可丢。这是 **no-refit** 策略（不需要对每个 term 重新训练模型），计算效率很高。

4. **Prune（剪枝）**：
   - **确定性剪枝（默认 IGSR）**：按 Δ_j 排序，保留 top-K（K=6）。对多输出场景，用 max 聚合。
   - **Agentic 剪枝（IGSR-Agent）**：将 (ψ_j, w_j, Δ_j) 三元组传给第二个 LLM agent（"Prune" agent），它结合 influence 启发式 + 语义推理做选择。

5. **记录历史**：保留/丢弃的 terms 及 MSE 变化写入 history buffer，供下一轮 Propose agent 做 in-context learning。

**MCTS 搜索框架**：
- 每个节点 = 一个方程状态（basis function 集合 + 权重）
- 每条边 = 一次 propose-and-prune cycle 的转移
- 从一个父节点可生成 n_successors=5 个子节点（利用 LLM 随机性）
- UCT 公式平衡 exploration/exploitation：UCT(i) = Q(i) + c√(ln N(p)/N(i))，c=√2
- 默认 Heuristic MCTS：直接用节点的 -val_MSE 作为 reward，不做 rollout
- 搜索预算：total_budget=30 次 node expansion，depth_limit=10
- 输出：整个搜索过程中验证 MSE 最低的方程

**IGSR-TLO 变体（Term-Local Optimization）**：允许 basis function 内部包含可调常数 c(init)（如 np.sin(c(1.0)*x1)），用 L-BFGS-B 优化这些内部常数。外层权重 w_j 仍由 OLS 确定。

### 关键技术细节

- **模型形式**：f(x) = Σ_{j=1}^{M} w_j ψ_j(x)，对 ψ_j 是线性的（保证全局可解释性），但 ψ_j 本身可以是任意非线性表达式（由 LLM 提出）
- **Influence Score 数学定义**：Δ_j = MSE_{-j} - MSE_{full} = (2w_j / n_val) · φ_j^T r_val + (w_j² / n_val) · ||φ_j||²。在训练集上交叉项为零（OLS 残差正交性），但在验证集上可能不为零，因此 Δ_j 可以略微为负。
- **Refit-aware 变体**：实现了两种 refit-aware influence（full refit 和 efficient refit），但实验表明与 no-refit 在准确性上无显著差异（ANOVA p≥0.09），且更贵。
- **LLM Backbone**：主实验用 GPT-4o (2024-11-20)；LLM-SRBench 用 GPT-4o-mini（控制成本）；案例研究用 o3 (2025-04-16)
- **默认超参数**：terms_per_round=5, first_round_n_candidates=10, keep_n_terms=6, total_budget=30, n_successors=5, exploration_constant=√2, depth_limit=10
- **温度**：默认 1.0（允许生成随机性以产生多样子节点）
- **嵌套验证分析**：将验证集拆为 Val-inner（pruning 用）和 Val-outer（MCTS reward 用），与共享验证的默认配置在测试 MSE 上无显著差异（paired t-test p>0.05），排除了搜索时 overfitting。

### 核心结果

**6 个 Benchmark 数据集**（25 seeds, GPT-4o）：
| 数据集 | IGSR MSE | 最佳白盒基线 | 黑盒最佳 |
|--------|---------|------------|---------|
| Lung Cancer | 5.64e-5 | ICL: 0.0557 | RNN: 1.33e-3 |
| Lung Cancer+Chemo | 0.0013 | ICL: 21.2 | RNN: 6.85e-3 |
| Lung Cancer+Chemo+Radio | 0.0141 | ICL: 63.3 | RNN: 0.013 |
| COVID-19 | 5.01e-8 | ICL: 9.35e-8 | RNN: 5.88e-8 |
| RNA Polymerase (263 features) | 0.0111 | ICL: 0.0119 | Transformer: 7.24e-3 |
| Warfarin | 0.565 | ICSR: 0.497 | DyNODE: 0.593 |

IGSR 在 5/6 数据集中白盒方法最优，平均排名 1.17。

**LLM-SRBench**（128 个 discovery 问题，4 个科学领域，GPT-4o-mini，5 seeds/problem，300k token 预算）：
- IGSR 在 ID 和 OOD 预测性能上均取得最佳平均排名（NMSE ID: 1.35, OOD: 1.77）
- Symbolic Accuracy 和 Term Recall 同样最优或次优
- 显著优于 LLM-SR、ICSR、LaSR、D3、ICL

**消融实验**（Lung Cancer+Chemo+Radio）：
- Full IGSR: NMSE 0.000787
- 去掉 MCTS（仅迭代+influence）: 0.626（↑ 795x）
- 去掉 influence（MCTS+标量反馈）: 0.293（↑ 372x）
- 两者都去掉: 4.85（↑ 6163x）
→ MCTS 和 influence feedback 都至关重要，且有协同效应

**跨 seed 稳定性**：25 seeds 中发现的 terms 高度一致。4 个频率 ≥80% 的 canonical terms 全部是 ground truth。86/300 seed pairs 共享完全相同的 term 集合。

**跨 LLM 鲁棒性**：在 9 个不同 LLM（含开源 Llama-3.3-70B, DeepSeek R3）上，IGSR 一致优于 ZeroShot/ZeroOptim/ICL。

**计算效率**：
- 263 特征时运行时间仅比 10 特征增加 ~25%（亚线性扩展）
- 峰值内存 2.782±0.003 GB（263 特征）
- 在固定 ~265k token 预算下，IGSR MSE 比第二好的 ICSR 低约 50 倍

**RNA Pol II 湿实验验证**：
- IGSR 发现 DNA 甲基化 (CpG methylation) 在基因体内抑制转录暂停的新假说
- 实验：用 5-azacytidine 处理 HCT116 细胞去除甲基化 → 暂停频率和强度显著增加
- Wilcoxon p=4.3×10⁻⁵² (频率), p=3.5×10⁻³³ (强度)
- 假说在 7/10 seeds（原始 replicate）、6/10（独立 replicate）、8/10（100 特征子集）中被恢复

### 明确局限

1. **LLM 先验限制**：如果所需函数形式完全不在 LLM 预训练分布中，系统不太可能提出它
2. **线性组合约束**：基本假设 f(x) = Σ w_j ψ_j(x)，不擅长发现深度嵌套非线性动力系统（相比微分方法）
3. **IGSR-TLO 在复杂问题上不稳定**：L-BFGS-B 可能陷入局部最优，在 Chemo+Radio 和 RNA Pol 上表现反而更差
4. **没有真正的非线性组合**：LaSR-like 的概念进化或完全自由形式的表达式搜索不在能力范围内
5. **缺少显式 parsimony 目标**：搜索仅优化验证 MSE，没有在 MCTS 中直接权衡复杂性-准确性 Pareto 前沿
6. **Material Science 领域 symbolic recovery 偏弱**：倾向于用线性展开近似 Arrhenius 等函数形式，导致结构恢复得分低
7. **真机验证仅限 RNA Pol II 案例**：其他基准都是仿真或已知 ground truth

### 跟我们的关系

**对 llm-mcts-sr 课题**：
- **高度相关**：这是 LLM + MCTS + symbolic regression 的直接组合，可能是该方向最具代表性的工作之一
- **撞车评估**：如果我们的 idea 也是用 LLM 在 MCTS 框架中做符号回归/方程发现，那与 IGSR 有明显重叠。需要在反馈机制、搜索策略、模型形式或应用领域上差异化
- **可能的差异化方向**：
  1. 突破 f(x)=Σw_j ψ_j(x) 的线性组合约束 → 嵌套/递归结构
  2. 在 MCTS 中加入显式 parsimony/complexity 目标
  3. 多目标搜索（accuracy vs interpretability Pareto）
  4. PDE/ODE 结构发现而非静态回归
  5. 用 influence-like 反馈但应用到不同的模型类（如 neural ODE）
- **能用的工具/方法**：
  - Per-term influence score 机制可直接借鉴
  - MCTS + LLM 的搜索框架成熟且有开源代码
  - LLM-SRBench 是标准评测基准
  - Propose-and-prune 的 history buffer + in-context learning 设计值得参考
- **引用建议**：必引。如果我们做 LLM+MCTS+SR，这是最直接的 baseline 和对比对象

## Read by: 0616-fex (2026-06-16)

### FEX 课题视角下的分析

FEX 是用 RL 做符号回归的方法。IGSR 与 FEX 有本质区别：IGSR 用 LLM 生成 basis function + influence score 剪枝 + MCTS 搜索，而 FEX 用 RL agent 直接操作表达式空间。两者共享"符号回归"这个最终目标，但方法路径完全不同。

**撞车评估**：
- **方法层面不撞车**：IGSR 的核心创新在于 per-term influence score 作为 granular feedback 来驱动 LLM 的 propose-and-prune cycle，这与 RL 驱动的符号回归（如 FEX 的 policy gradient 或 value-based 方法）是完全不同的技术路线。FEX 不依赖 LLM 的 generative priors，而是用 RL agent 在表达式空间做探索。
- **问题层面部分重叠**：两者都解决符号回归（发现闭式方程），但 IGSR 强在"受 LLM 先验引导的高维特征选择"，FEX 强在"自由形式的表达式搜索"。IGSR 的线性组合约束 f(x)=Σ w_j ψ_j(x) 与 FEX 的完全自由形式搜索有根本差异。
- **评测基准可能重叠**：两者可能在 LLM-SRBench、PKPD 等标准基准上直接对比。如果 FEX 也在这些基准上评测，IGSR 是必须引用的最强 LLM-based 基线。

**对 FEX 课题的启发**：
1. **IGSR 的 influence score 可以作为 FEX 的 reward shaping 参考**：per-term Δ_j 衡量 term 的边际贡献，这个信号可以用来设计更细粒度的 RL reward（而不是简单的全局 MSE）。
2. **MCTS 框架 vs RL 框架**：IGSR 证明 MCTS+UCT 对符号回归搜索空间有效。FEX 的 RL 框架可以考虑将 MCTS 作为 baseline 对比，或结合 MCTS 做 tree-based policy evaluation。
3. **IGSR 的局限正是 FEX 的机会**：
   - 线性组合约束 (f=Σ w_j ψ_j) → FEX 可以直接探索嵌套/递归结构
   - LLM 先验限制 → FEX 不依赖预训练，对 LLM 分布外函数形式可能更强
   - 缺少 parsimony 目标 → FEX 可以天然在 reward 中包含复杂度正则化
   - Material Science 领域 symbolic recovery 弱 → FEX 可能更好地处理 Arrhenius 等函数形式
4. **IGSR 的实证方法论值得借鉴**：
   - 跨 LLM 鲁棒性验证（9 个 LLM）
   - 跨 seed 稳定性分析（Jaccard 相似度）
   - 嵌套验证消除 search-time overfitting 疑虑
   - 消融实验的 MCTS x Influence 双因子设计
5. **湿实验验证的标杆**：IGSR 的 RNA Pol II 湿实验验证是 SR 领域罕见的闭环验证，FEX 应以此作为验证标准的目标。

**引用建议**：必引。IGSR 是 LLM+MCTS+SR 路线的 SOTA，FEX 作为 RL+SR 路线必须与之对比和差异化。建议在 Related Work 中明确区分两种范式。
