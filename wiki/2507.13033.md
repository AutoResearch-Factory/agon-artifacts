# (Exhaustive) Symbolic Regression and model selection by minimum description length (2507.13033)

- **Year**: 2025.07
- **Authors**: Harry Desmond, Institute of Cosmology & Gravitation, University of Portsmouth
- **Venue**: arXiv; review article in the Royal Society discussion-meeting issue on symbolic regression in the physical sciences
- **Keywords**: symbolic regression, exhaustive search, minimum description length, Bayesian evidence, function prior, astrophysics, model selection, genetic programming
- **One-liner**: 用ESR+MDL把符号回归变成可穷举排序的问题。

## 精华提炼

### 管线/方法

1. 论文先把传统符号回归的问题拆成两层：生成候选函数和选择最终函数。
2. 传统 GP/GA/PySR/Operon 类方法用随机搜索生成表达式树，因此无法知道某个好函数是不是被漏掉。
3. 传统评估把误差和复杂度放在 Pareto 平面上，但复杂度定义和最终选点规则都依赖用户启发式。
4. Exhaustive Symbolic Regression (ESR) 的第一步是固定算子集合和最大复杂度。
5. 这里复杂度定义为表达式树节点数，包括变量、参数、一元算子、二元算子等。
6. ESR 枚举所有可能的树模板，每个节点只标注 arity。
7. 对每个树模板，ESR 用用户给定的算子集合装饰节点，生成所有结构合法的表达式。
8. 之后用符号规则去重，去掉行为等价但结构不同的函数。
9. 去重规则包括 tree reordering、parameter permutations、simplifications、reparametrisation invariance、parameter combinations。
10. 对每个 unique function，ESR 用非线性优化求最大似然参数。
11. 对等价类中其他表达式，通过变换 Jacobian 广播最大似然结果。
12. 关键点是：等价函数最大似然相同，但描述长度可能不同，因此 MDL 排序仍要考虑等价类内不同写法。
13. ESR 的输出不是一个函数，而是给定复杂度上界内的全函数空间及其得分。
14. 论文给出的典型复杂度上限是 10；复杂度 10 的完整 ESR 约需 200 CPU-hours。
15. 计算瓶颈来自函数数目指数增长，以及去重阶段需要跨函数比较，不能完全 embarrassingly parallel。
16. 作者强调只有参数拟合依赖数据；函数生成和去重只依赖算子集，因此可以复用预计算函数集合。
17. ESR 代码公开，文中脚注给出 GitHub: DeaglanBartlett/esr。
18. model selection 部分使用 Minimum Description Length (MDL) 原理，把精度和复杂度统一成 nats。
19. MDL 总目标写成 L(D)=L(H)+L(D|H)，即传达函数本身和传达数据残差所需的信息量。
20. 残差项 L(D|H) 在 Shannon-Fano 最优编码下等于最小负 log-likelihood。
21. 函数结构项用 k log n 编码，其中 k 是树节点数，n 是表达式中不同算子数。
22. 自然数常量或化简产生的整数系数用 sum log(c_j) 编码。
23. 自由参数的数值编码依赖精度 Delta_i，在舍入损失和编码长度之间优化。
24. 最优参数精度为 Delta_i=(12/I_ii)^(1/2)，I 是最大似然点处的 observed Fisher information。
25. 最终描述长度公式包含负 log-likelihood、k log n、参数数目项、整数常量项、Fisher 信息项和参数绝对值项。
26. 论文说明 MDL 与 Bayesian evidence 在 Laplace approximation 下近似等价。
27. 结构复杂度惩罚可解释为函数先验 P(f_i)，即 -log P(f_i)=k log n + sum log c_alpha。
28. 因此 MDL 不是任意惩罚项，而是把 Bayesian model selection 推广到函数结构空间。
29. 作者还讨论了替代函数先验：Katz back-off model。
30. Katz 先验从领域内成功方程训练集中学习算子组合概率，惩罚训练集中罕见的组合。
31. 这个先验可以让物理上常见的结构更便宜，例如 sum of sines 相比 nested sines。
32. Katz 模型被作者作为 k log n 先验的 drop-in replacement，并给出单独代码库 DeaglanBartlett/katz。

### 关键技术细节

33. Benchmark 数据是 Penn Machine Learning Benchmarks 中的 feynman_I_6_2a。
34. 该 benchmark 有 10^5 个无噪声一元函数数据点。
35. 对比算法包括 ESR、PySR、DataModeler、FFX、QLattice、Operon。
36. ESR 在复杂度 7 处找到明显 MSE cliff，其他算法在复杂度 5 以后只给出边际改进。
37. ESR 找到的复杂度 7 函数是 y=theta_1 theta_0^(x^2)。
38. 拟合参数 theta_0=0.6065、theta_1=0.3989，非常接近 1/sqrt(e) 和 1/sqrt(2pi)。
39. 将精确常数代入后 MSE=3e-33，达到机器精度意义上的零。
40. Operon 也找到标准正态形式，但过参数化到复杂度 11；其余方法没有暴露真实结构。
41. 天体物理应用一是 H(z) 宇宙膨胀率，使用 cosmic chronometers 和 Type Ia supernovae。
42. cosmic chronometers 数据集包含 32 个点，来自 Moresco et al. 2022。
43. Pantheon+ SNe 数据集包含 1590 个超新星，并使用协方差矩阵进入似然。
44. H(z) 应用的算子集是 {x=1+z, theta, inv, +, -, times, div, pow}。
45. 对 CC 数据，MDL 最佳函数为 H^2(z)=theta_0(1+z)^2。
46. 对 SNe 数据，MDL 最佳函数为 H^2(z)=theta_0(1+z)^(1+z)。
47. 两个函数复杂度都是 5，低于简单 Friedmann 方程的复杂度 7。
48. 相比简单 Friedmann 方程，CC 和 SNe 上分别优 7.12 和 4.91 nats。
49. 这对应概率比 1240 和 136。
50. ESR 找到 38 个优于 Friedmann 方程的 CC 函数和 36 个优于 Friedmann 方程的 SNe 函数。
51. 作者明确指出这不等于推翻 Friedmann 方程，因为有限红移范围内 Taylor 展开到二阶相同，数据不足时 MDL 会偏向更简单函数。
52. 应用二是 galaxy radial acceleration relation (RAR) 和 MOND interpolating functions。
53. MOND 设定低加速度尺度 a_0 约为 1.2e-10 m/s^2。
54. RAR 应用的算子集是 {g_bar, theta, exp, sqrt, square, inv, +, -, times, div, pow}。
55. ESR 找到许多比经典 MOND IF 更好的函数。
56. 多数好函数满足 Newtonian limit，但很少满足 deep-MOND limit。
57. 最佳函数更常见的低 g_bar 极限是 g_obs 趋于常数。
58. 但作者用 RAR IF 生成 mock data 后也看到同样现象，因此不能直接断言 MOND 被数据排除。
59. 应用三是单场慢滚 inflation 的 inflaton potential V(phi)。
60. 使用的观测约束是 A_s=(0.027+-0.0027) M_pl、n_s=0.9649+-0.0042、r<0.028 at 95% C.L.
61. inflation 算子集 A 是 {phi, theta, inv, exp, log, sin, sqrt, square, cube, +, -, times, div}。
62. inflation 算子集 B 是 {phi, theta, inv, exp, log, +, -, times, div, pow}。
63. 用 k log n 先验时，两组算子的 MDL 函数都是 exp(-exp(exp(exp(phi))))，复杂度 6。
64. 该函数可产生良好的 slow-roll region，但作者认为很难来自合理粒子物理理论。
65. 用 Katz 先验并以 Encyclopaedia Inflationaris 为训练集后，set A 的 MDL 函数为 theta_0(theta_1+log(phi)^2)。
66. 用 Katz 先验后，set B 的 MDL 函数为 theta_0 phi^(theta_1/phi)。
67. 在 operator set B 与 k log n 先验下，Starobinsky、quadratic、quartic 模型排名分别为 1272、8697、10839。

### 核心结果

| 任务 | 数据/设置 | 主要发现 | 解释 |
|------|-----------|----------|------|
| SRBench feynman_I_6_2a | 10^5 无噪声点 | ESR 在复杂度 7 找到标准正态结构 | 随机 SR 方法可能漏掉简单真式 |
| H(z) cosmic chronometers | 32 点 | H^2=theta_0(1+z)^2，比 Friedmann 优 7.12 nats | 有限数据下更简单函数被 MDL 偏好 |
| H(z) Pantheon+ SNe | 1590 SNe | H^2=theta_0(1+z)^(1+z)，比 Friedmann 优 4.91 nats | 与 Friedmann 在低阶 Taylor 上近似 |
| RAR/MOND | galaxy dynamics | 许多函数优于 MOND IF，但低加速度极限多为常数 | 数据动态范围不足，不能强判 MOND |
| Inflation | Planck A_s,n_s,r | k log n 先验偏好 exp(-exp(exp(exp(phi)))) | 统计上好但物理性弱 |
| Inflation + Katz | Inflationaris 方程训练集 | theta_0(theta_1+log(phi)^2) 或 theta_0 phi^(theta_1/phi) | 领域先验能改变可解释性 |

### 明确局限

68. 这篇是 review，很多实验结果来自作者及合作者已有论文，不是完整的新实验论文。
69. ESR 的穷举保证只在用户给定算子集和复杂度上限内成立。
70. 如果真函数需要未包含算子、更高复杂度、多变量结构或特殊常数，ESR 不会自动发现。
71. 复杂度 10 已约 200 CPU-hours，继续提高上限受到指数增长限制。
72. 论文未来工作提到 Julia rewrite、automatic differentiation、更快去重，目标把复杂度上限提高到约 13。
73. 当前 ESR 主要是低复杂度函数空间的确定性工具，不是高复杂度无限表达式空间的全局求解器。
74. 多变量建模、参数 snapping 到整数/有理数/基本常数在文中仍是未来 ESR 2.0 功能。
75. Benchmark 只展示一个非常简单的无噪声一元函数案例，不能覆盖高噪声、多变量、PDE residual 等设置。
76. 天体物理应用的核心结论多是有效拟合优于标准公式，不等于发现新的物理定律。
77. H(z) 和 RAR 部分都诚实说明数据质量/动态范围不足，不能用 MDL 最优函数直接否定标准理论。
78. Inflation 部分显示默认结构先验可偏好物理上不自然的表达式，说明先验选择会显著影响科学解释。
79. 本文没有讨论 FEX 这类 RL controller 的搜索收敛率或采样复杂度。
80. 本文的 ESR 是穷举式，不解决 FEX 中 policy gradient 是否会稳定找到好表达式的问题。
81. 论文有公开 ESR/Katz 代码线索，但 review 本身没有给完整复现实验脚本和逐实验配置。
82. 与 `0616-fex` 最相关的局限是：它证明随机 SR 会漏好式，但没有给 RL 搜索的理论或工程修复方案。

## Read by: 0616-fex (2026-06-16)

这篇对 `0616-fex` 的价值不在于直接竞争，而在于提供一个很干净的“穷举 oracle + MDL scoring”参照系。FEX 用 RL controller 在有限表达式空间里采样 PDE 解，ESR 则在低复杂度上完整枚举同类表达式空间；两者都依赖算子集、表达式树、参数优化和复杂度控制。区别是 ESR 放弃学习式搜索，换来不漏函数的确定性保证；FEX 追求高维 PDE 场景的可扩展性，但搜索过程本身缺少收敛率和采样复杂度保证。

对 FEX 后续 idea，最可用的是三点。第一，用 ESR 小空间穷举当 FEX policy 的 diagnostic oracle：固定同一算子集和深度，在低维 PDE residual 上比较 FEX 是否找到 ESR/MDL 前沿。第二，把 MDL 引入 FEX reward，替代单纯 residual loss 加手工复杂度惩罚；但 PDE residual 需要先定义成 likelihood 或 pseudo-likelihood。第三，把 Katz function prior 移植到 FEX/LLM+FEX：从 PDE 解、物理 law、Inflationaris 一类方程库学习算子组合先验，用来初始化 controller、约束 action prior 或重排 beam candidates。

撞车评估：不撞车。本文没有 PDE symbolic solver、没有 FEX 的 S_k 近似理论、没有高维 PIDE、没有 RL/LLM 搜索。但它会削弱任何“随机/学习式 SR 已足够可靠”的叙事，因为文中 benchmark 明确显示 PySR、FFX、QLattice 等可能漏掉非常简单的真函数。若 `0616-fex` 继续走 RL 搜索效率、频率先验、e-graph 剪枝或 LLM skeleton 方向，应引用本文作为搜索可靠性和模型选择标准的背景。
