# PROSE: Predicting Operators and Symbolic Expressions using Multimodal Transformers (2309.16816)

- **Year**: 2023.09
- **Authors**: Yuxuan Liu (UCLA Math), Zecheng Zhang (FSU Math), Hayden Schaeffer (UCLA Math)
- **Venue**: arXiv preprint (not yet published in a top venue as of writing)
- **Keywords**: multi-operator learning, symbolic regression, multimodal transformers, Polish notation, neural operators, equation discovery, ODE systems, feature fusion
- **One-liner**: PROSE 用一个多模态 Transformer 同时学习多个不同 ODE 系统的解算子和符号控制方程，用 Polish notation 编码方程，通过特征融合实现跨模态信息交互，数据预测误差约3-6%，方程生成有效率 >99%。

## 精华提炼

### 管线/方法

PROSE 的完整 pipeline（端到端训练）：

1. **数据输入**：ODE 解轨迹在 $[0,2]$ 上的 64 个均匀采样点（含 2% 高斯噪声），每个点 $(\bm{u}(t_i), t_i)$ 经线性层嵌入为 Data Feature
2. **符号输入**：ODE 方程用 Polish notation（前缀表达式）编码为 token 序列。浮点数用 sign + mantissa + exponent 三元组表示（如 `[+ 260 E-2]` 表示 2.6）。词汇量约 $10^4$。符号输入可有噪声（系数未知/项删除/项添加）
3. **数据编码器**：2 层 self-attention，时间 $t_i$ 作为 positional encoding，隐藏维度 512
4. **符号编码器**：4 层 self-attention，sinusoidal positional encoding，隐藏维度 512
5. **特征融合**：将数据特征和符号特征拼接，通过 8 层 self-attention（8 头）。添加可学习的 modality-type embedding 标记各 token 来源。这是核心的 cross-modality interaction 位置
6. **数据解码器**：8 层 cross-attention，以融合特征为 source、query locations 为 query，构建算子。Query locations 互相独立（类似 branch-trunk 结构），可在任意时间点评估
7. **符号解码器**：8 层 encoder-decoder attention，自回归生成 Polish notation 序列。从 start token 开始，每次预测下一个 token，直到 end token。推理时用 greedy search
8. **损失函数**：$\mathcal{L} = \alpha \mathcal{L}_d + \beta \mathcal{L}_s$，$\alpha=6.0$（数据 loss 权重），$\beta=1.0$（符号 loss 权重）。$\mathcal{L}_d$ 是 relative squared error，$\mathcal{L}_s$ 是 cross-entropy

### 关键技术细节

- **模型规模**：隐藏维度 512，FFN 隐藏维度 2048，8 个 attention heads。数据编码器 2 层，符号编码器 4 层，融合层 8 层，数据解码器 8 层，符号解码器 8 层。总参数量未明确给出，但按此配置估算在 100M+ 级别
- **训练数据**：512K 训练样本（每个 ODE 系统采样 20 个初始条件），25.6K 验证样本，102,400 测试样本。15 个不同 ODE 系统：12 个 3D（Thomas、Lorenz、Aizawa、Chen-Lee、Dadras、Rössler、Halvorsen、Rabinovich-Fabrikant、Sprott B、Sprott-Linz F、Four-wing、Duffing），2 个 4D（Lorenz 96 N=4、Double Pendulum），1 个 5D（Lorenz 96 N=5）
- **ODE 系数采样**：每个 term 的系数从 $[F-0.1F, F+0.1F]$ 均匀采样，F 为参考值。OOD 测试扩展到 $[F-0.2F, F+0.2F]$
- **训练配置**：AdamW 优化器，lr=$10^{-4}$，weight decay=$10^{-4}$，inverse square root scheduler（10% warmup），batch size 256/GPU，gradient norm clip=1.0，80 epochs（每 epoch 2000 steps），2× RTX 4090 (24GB)，约 19 小时
- **数据细节**：初始条件从 $[-2,2]^d$ 均匀采样，ODE 用 BDF 方法求解（absolute tolerance $10^{-6}$，relative tolerance $10^{-5}$），总区间 $[0,6]$。输入使用 $[0,2]$ 上 64 点，输出预测 $[2,6]$ 上 128 点
- **方程编码**：Polish notation（前缀表达式），二元树保证一一对应。浮点数用 base-10 三元组表示（sign, mantissa length=3, exponent），如 `[+ 260 E-2]` = 2.6。维度分隔 token `|`。操作符（cos, sin, pow 等）作为单个 trainable token
- **评估指标**：(1) 相对 $L^2$ 预测误差；(2) 相对 $L^2$ 表达式误差（在 $[-5,5]^d$ 内 50 点 Monte Carlo 估计）；(3) 有效表达式百分比（输出能否解析为合法数学表达式）

### 核心结果

| 实验设置 | 相对预测误差 (%) [2,4]/[2,6] | 相对表达式误差 (%) | 有效表达式 (%) |
|----------|------|------|------|
| Known（方程已知） | 2.74 / 2.94 | 0.00 | 100.00 |
| Skeleton（系数未知） | 3.39 / 4.59 | 2.10 | 99.98 |
| Unknown 3D | 3.43 / 4.63 | 2.11 | 99.95 |
| Unknown Multi-D | 3.95 / 5.66 | 1.88 | 99.94 |

| OOD λ | 预测误差 [2,4]/[2,6] (%) | 表达式误差 (%) | 有效表达式 (%) |
|--------|------|------|------|
| 0.10 (ID) | 3.43 / 4.63 | 2.11 | 99.95 |
| 0.15 | 3.89 / 5.71 | 3.21 | 99.44 |
| 0.20 | 4.94 / 7.66 | 4.83 | 97.89 |

**关键发现**：
- 数据解码器直接预测 vs 符号解码器生成方程后用 BDF 求解：前者 4.59%，后者 14.69%——说明学习到的算子比用学习到的方程直接求解更准确
- 消融实验：完整 PROSE 显著优于纯数据模态版本。随着输入传感器数量减少，纯数据模型性能下降，而 PROSE 几乎不变——符号模态提供了互补信息
- 跨模态注意力图（Figure 6）显示 bottom-left/upper-right block 存在非零值，证实了 cross-modality interaction 确实发生

### 明确局限

**作者说的**：
- 仅处理 ODE 系统，未扩展到 PDE（虽然提到是未来方向）
- 训练数据需要已知方程标签（虽然可以带噪声），不是纯无监督方程发现
- 方程系数恢复到 2 位精度（mantissa=3），精度有限，需要后处理优化（如 BFGS）

**我读出来的**：
- **无真机实验**：全部是数值仿真数据，ODE 用 BDF 求解器生成。没有真实物理系统数据
- **无标准 benchmark 对比**：没有和 SINDy、PINN-SR 等方程发现方法做定量比较。只在消融实验中和自己比。对 operator learning 方法（FNO、DeepONet）也只是引用，没做直接对比实验
- **ODE 系统规模有限**：15 个系统，最大 5 维。全是已知混沌吸引子，结构类型有限（多项式+三角函数）。没有 rational、exponential、logarithmic 项
- **新 ODE 泛化未测试**：声称可以泛化到新 ODE 系统，但所有实验都是在 seen ODE systems 上做的（只是 unseen 初始条件和系数范围）。没有测试完全 unseen 的 ODE 形式
- **符号表达式解析依赖 Polish notation 的封闭性**：如果生成了非法序列，直接丢弃——没有纠错机制
- **计算开销大**：2×4090 训练 19 小时，对于 15 个 3-5D ODE 系统来说开销不小
- **没有开源代码链接**：论文中未提供代码仓库 URL（按 arXiv 提交时间 2023.09 正常，但截至审读时未见公开代码）

### 跟我们的关系

对 **0616-fex**（finite expression methods / 方程发现）课题：

**撞车评估**：
- **不直接撞车**：PROSE 目标是同时学算子和方程，0616-fex 如果是纯方程发现（从数据中提取符号表达式），则定位不同
- **部分重叠**：PROSE 的符号解码器本质上是一个方程发现模块。如果 0616-fex 也用了 Polish notation + transformer 做符号回归，那在方法论上有交集。但 PROSE 的核心贡献在"多模态 + 多算子"，不是纯符号回归
- **关键差异**：PROSE 需要已知方程（带噪声）作为输入，不是从零发现；0616-fex 如果是从数据直接发现方程，则任务更硬

**能用什么**：
- **Polish notation 编码方案**：浮点数三元组表示（sign, mantissa, exponent），维度分隔 token，操作符作为 single token。这是一套成熟的方程编码体系，可以直接复用
- **特征融合架构**：hierarchical attention + modality-type embedding 的融合方式，对多模态科学 ML 任务有参考价值
- **损失函数设计**：relative squared error + cross-entropy 的加权组合，$\alpha=6, \beta=1$ 的比例
- **评估协议**：有效表达式百分比 + Monte Carlo 相对 $L^2$ 误差，比单纯的 token-level accuracy 更合理
- **数据集参考**：15 个混沌 ODE 系统列表可作为 benchmark 的一部分

**差异化空间**：
- PROSE 不做纯方程发现（需要符号输入），可以做真正的 unsupervised / weakly-supervised 方程发现
- PROSE 不处理 PDE，扩展到 PDE 方程发现是明确的 gap
- PROSE 没有和 SINDy/PINN-SR 对比，可以做系统性 benchmark
- PROSE 的方程恢复精度有限（mantissa=3），提高精度或加入 refinement 步骤是改进方向

## Read by: 0616-fex (2026-06-24)

本次审读视角：PROSE 是用多模态 transformer 同时做算子学习和方程发现的方法，对 0616-fex 课题的参考价值主要在于（1）Polish notation 方程编码体系可直接复用；（2）特征融合机制可作为符号-数值多模态的参考架构；（3）实验设置（15 个 ODE 系统、评估指标）可作为 benchmark 设计参考。

**撞车风险**：低。PROSE 需要符号表达式作为输入（即使带噪声），本质上是"已知方程结构 + 未知系数"的设定，而非从纯数据中 ab initio 发现方程。0616-fex 如果定位于无监督/弱监督方程发现，则不撞车。

**可复用组件**：Polish notation 的浮点数编码方案（sign-mantissa-exponent 三元组）、有效表达式验证的解析方法、relative $L^2$ 误差 + 有效表达式率的两维评估协议。

**差异化方向**：纯方程发现（无符号输入）、扩展到 PDE、加入物理约束（如守恒律验证）、与 SINDy 等传统方法的系统对比、提高符号精度（mantissa 长度 > 3）。
