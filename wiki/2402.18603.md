# MMSR: Symbolic Regression is a Multi-Modal Information Fusion Task (2402.18603)

- **Year**: 2024.02
- **Authors**: Yanjie Li, Jingyi Liu, Min Wu, Lina Yu, Weijun Li, Xin Ning, Wenqiang Li, Meilan Hao, Yusong Deng, Shu Wei (AnnLab, Institute of Semiconductor, Chinese Academy of Sciences)
- **Venue**: arXiv (submitted to Nuclear Physics B)
- **Keywords**: Symbolic Regression, Multi-modal, Information Fusion, Contrastive Learning, Modal Alignment, SetTransformer, Pre-training, Expression Skeleton
- **One-liner**: 将符号回归重构为多模态信息融合任务，用 SetTransformer 提取数据特征、Transformer decoder 生成表达式，通过对比学习对齐数据与表达骨架两个模态，在 SRBench 等 13 个数据集上达到 SOTA。

## 精华提炼

### 管线/方法

整体 pipeline 为"数据编码 → 骨架编码 → 模态对齐 → 特征融合 → 自回归解码 → 常数优化"：

1. **训练数据生成**：随机采样表达式骨架 → 在占位符位置填入常量（[-1,-0.1]∪[0.1,1]）→ 计算 y=f(X) → 生成 10M 对 (X,y, 骨架, 常量序列) 训练数据
2. **数据编码**：SetTransformer (5 层 ISAB blocks, 50 inducing points, dim_hidden=512, 8 heads) 将 point cloud (X,y) 映射到固定长度特征 F_data ∈ R^{b×h}
3. **骨架编码**：前 8 层 transformer decoder（无 cross-attention）作为骨架编码器，将前序遍历符号序列映射为 F_skeleton ∈ R^{b×h}
4. **模态对齐**：在 F_data 与 F_skeleton 间计算对比损失 L_CON（对称 InfoNCE），使匹配对靠近、不匹配对远离
5. **特征融合**：后 8 层 decoder 通过 cross-attention（Q 来自 skeleton, K/V 来自 data）融合两个模态特征
6. **自回归解码**：用 beam search 从融合特征生成符号序列（骨架）和常量序列
7. **常数精修**：用 BFGS 对生成的常数做局部优化得到最终表达式

### 关键技术细节

- **模型架构**：双 encoder + decoder。数据 encoder = SetTransformer (5 ISAB layers, 50 inducing points, 512 hidden dim)；骨架 encoder = 前 8 层 transformer decoder（无 cross-attention）；decoder = 后 8 层 transformer decoder（含 cross-attention）。共 16 层 decoder layer。
- **训练数据规模**：10M 对 (X, y, skeleton, constants)。消融实验验证 10K→100K→1M→5M→10M 数据规模 scaling，5M 达到 avg R²>0.9，10M 达到 avg R²>0.98。
- **超参数**：batch size 未明确但在算法伪代码中提及；学习率 lr=0.0001（SetTransformer）；θ（对比温度）=0.07；λ₁（CE loss）=1.0，λ₂（MSE loss）=1.0，λ₃（对比 loss）=1.0
- **常数编码策略**：科学记数法 C = c_m × 10^{c_e}，c_e ∈ [-5, 5]（11 类占位符 C-5 到 C5），c_m ∈ [-1,-0.1] ∪ [0.1, 1]。每个表达式骨架和常量序列都 padding 到最大长度 N=60。
- **Loss 函数**：L = λ₁·L_CE + λ₂·L_MSE + λ₃·L_CON
  - L_CE：预测符号序列与真实序列的交叉熵
  - L_MSE：预测常量与真实常量的均方误差（分母归一化用真实均值方差）
  - L_CON：数据-骨架对的双向 InfoNCE 对比损失
- **模型参数**：约 86M；所有模型的 transformer 使用 16 注意力头、embedding dim=512
- **推理**：beam search 生成 n 个候选表达式，BFGS 优化常数，选 R² 最优者
- **评测指标**：拟合能力 R²（20 次重复取平均 + 0.95 置信区间）、表达式复杂度（前序节点数）、模型复杂度（参数量 + 推理时间）
- **测试数据集**：13 个主流数据集，含 SRBench (Feynman, Strogatz, Black-box) 和经典 benchmark (Nguyen, Keijzer, Korns, Constant, Livermore, Vladislavleva, R, Jin, Neat, Others)，共约 400 个表达式。另外还单独测试了 AI Feynman 120 个方程。

### 核心结果

| 方法 | 平均 R² | 平均节点数 | 参数量 | 推理时间 |
|------|---------|-----------|--------|---------|
| **MMSR** | **0.9934** | **21.3** | 86M | 26.6s |
| TPSR | 0.9264 | 24.4 | 86M | 86.3s |
| End2End | 0.7892 | 23.9 | 86M | 24.4s |
| NeSymReS | 0.7528 | 26.2 | 84M | 27.3s |
| SymbolicGPT | 0.6311 | 32.4 | 88M | 28.4s |

**Surprising 发现**：
- MMSR 在表达式简洁性（平均节点数 21.3）和拟合精度上同时最优，打破了"更复杂表达式才能更好拟合"的假设
- 联合训练（one-step）优于分离训练（two-step），且数据量越大差距越明显——作者解释为特征提取与融合模块的"磨合"效应
- 模态对齐权重 λ₃ 从 0→1.0 时 avg R² 从 0.9328→0.9934，提升幅度约 6 个百分点，远超预期
- 常数编码（MSE loss）对性能贡献显著：λ₂ 从 0→1.0 时 avg R² 从 0.9069→0.9934，提升近 9 个百分点

**消融实验结果**：
- 模态对齐：λ₃=1.0 → R²=0.9934; λ₃=0.1 → R²=0.9633; λ₃=0.0 → R²=0.9328
- 常数编码：λ₂=1.0 → R²=0.9934; λ₂=0.1 → R²=0.9613; λ₂=0.0 → R²=0.9069
- 模态对齐可视化：对比学习后，diagonal cosine distance 显著低于非对角，证明两个模态在特征空间确实被拉近

### 明确局限

1. **抗噪性能差**（作者自述）：对含噪声数据拟合效果不佳
2. **符号集不可扩展**：训练完成后无法新增运算符类型，需重新训练
3. **最大变量数固定**：训练时指定的变量维度不能事后扩展
4. **无真机 / 物理约束验证**：仅做数值拟合评估，未验证发现的表达式是否满足物理一致性（量纲、边界条件等）
5. **无标准 benchmark 以外的泛化测试**：未在真实科学发现场景（如材料科学、物理实验数据）上验证
6. **对比方法较陈旧**：baselines 中 TPSR (2024), End2End (2022), NeSymReS (2021), SymbolicGPT (2021)，未对比最新方法如 DGSR (2023), UDSR (2022) 等
7. **训练数据为合成数据**：所有训练数据为随机生成表达式，与真实世界数据分布可能存在 domain gap
8. **对比学习引入的额外计算开销**：需计算 batch 内所有样本对的 similarity matrix，大 batch 下计算开销显著

### 跟我们的关系

**对 0616-fex（特征提取/工程）课题**：

**撞车评估**：不直接撞车。MMSR 的核心贡献是"将 SR 视为多模态任务+对比学习对齐"，而非特征提取方法本身。其 SetTransformer 数据编码器和对比学习对齐机制属于 feature extraction 的子话题，但聚焦于符号回归这一特定下游任务。

**可用工具/方法**：
1. **SetTransformer 作为 permutation-invariant 特征提取器**：如果你需要处理无序点云 / 集合输入，直接可用。5 层 ISAB + 50 inducing points 的结构设计可作为参考起点。
2. **对比学习模态对齐范式**：InfoNCE 用于对齐不同模态特征的方法可迁移到其他需要融合异构数据的任务（如传感器融合、多模态机器人感知）
3. **联合训练 vs 分离训练的实证结论**：one-step joint training 优于 two-step separate training 的发现可指导你的训练策略设计
4. **科学记数法常量编码**：将连续值离散化的技巧（c_m × 10^{c_e}）可借鉴用于需要将连续参数纳入离散序列生成的场景

**差异化**：0616-fex 如果关注的是通用特征提取（如图像/视频/传感器信号 → 特征向量），与 MMSR 的 (X,y) point cloud → 表达式 的任务完全不同。文中的 contrastive alignment 策略可以作为一种跨模态特征对齐手段被引用和借鉴。

**该引不该引**：如果你的工作涉及 (a) 多模态特征对齐、(b) permutation-invariant 特征提取、(c) 符号回归任务，均需引用。纯粹做单模态特征提取可以不引。
