# FePySR: A Neural Feature Extraction Framework for Efficient and Scalable Symbolic Regression (2605.12704)

- **Year**: 2026.05
- **Authors**: Zhiming Yu, Wangtao Lu (Zhejiang University), Xin Lai (Tampere University)
- **Venue**: arXiv
- **Keywords**: symbolic regression, feature extraction, neural network, evolutionary search, expression trees, combinatorial optimization, NP-hard, heterogeneous activation units
- **One-liner**: 提出 FePySR 两阶段框架——先用异构神经网络 FMN 从观测数据中提取非线性特征以缩小搜索空间,再用 PySR 在优化特征空间中重构表达式,在 5 个标准基准和 75 个复杂合成方程上超越 SOTA。

## 精华提炼

### 管线/方法

FePySR 是一条两阶段 SR 管线：

**Stage 1 — FMN (Feature Mapping Network)**:
- 输入：观测数据 $\mathcal{D} = \{x, y\}$（自变量和方程解值）
- 前向传播通过多层异构层（heterogeneous layers），每层由并行 HAU 阵列构成
- 每个 HAU 从预定义原语库 $\{(\cdot)^2, \sin(\cdot), \cos(\cdot), \exp(\cdot), +, \times\}$ 中选取一个数学操作作为激活函数
- 层间采用 dense connectivity：$y_i = [y_{i-1}, y_{i,1}, \cdots, y_{i,j}]$
- 输出层为线性全连接：$\hat{y} = W_{L+1} y_L$
- 训练完成后，每层每个 HAU 提取其最大绝对权重对应的输入特征，应用其原语得到特征 $\phi_{i,j}$
- 多次独立训练（16 runs, 随机初始化），用 SymPy 化简特征并统计频率
- 选出 top-K 特征（默认 K=4），与原始数据拼接送入 Stage 2
- Univariate 用 3 层，multivariate 用 4 层

**Stage 2 — PySR**:
- 在 FMN 增强的表示空间上运行 PySR（基于 GP 的 SR 求解器）
- PySR 的 unary operators: $\{\sin(\cdot), \cos(\cdot), \exp(\cdot), \sqrt{(\cdot)}\}$
- PySR 的 binary operators: $\{+, -, \times, /\}$

### 关键技术细节

**FMN 超参数**:
- batch size: 50, learning rate: 0.5, epochs: 100
- 原语库: $\{(\cdot)^2, \sin(\cdot), \cos(\cdot), \exp(\cdot), +, \times\}$
- 网络深度: univariate 3 层, multivariate 4 层
- 并行训练: num_experiments=16, num_workers=8
- 选中特征数: pysr_num=4

**Loss 函数** = $\mathcal{L}_2 + \mathcal{L}_{\text{sparse}} + \mathcal{L}_{\text{contrast}}$:
- $\mathcal{L}_2 = (\hat{y} - y)^2$ — 标准 MSE
- $\mathcal{L}_{\text{sparse}} = \lambda_1 \sum_{i,j} |W_{ij}|$, $\lambda_1=0.08$ — $L_1$ 稀疏正则化，抑制无关特征
- $\mathcal{L}_{\text{contrast}} = \lambda_2 \cos(W_1, W_2)$, $\lambda_2=0.001$ — 防止二元对称算子（$+, \times$）的两路权重趋同导致功能坍缩

**训练稳定性技巧**:
1. 梯度归一化 (unit-wise gradient normalization)：用 $\frac{\nabla_W \mathcal{L}}{\|\nabla_W \mathcal{L}\|_2 + \epsilon}$ 替代原始梯度，防止不同原语（sin, exp, log）梯度尺度差异导致训练失衡
2. Mask-based 结构约束：当 exponential HAU 的输入直接来自前一层的 exponential 输出时，mask 掉该连接权重，防止级联指数运算导致的数值上溢（3 层以上网络特有）

**噪声鲁棒性评估**:
- 对目标变量 $y$ 添加 $\mathcal{N}(0, \sigma)$ 高斯噪声，$\sigma = \alpha \cdot \text{RMS}(y)$
- 测试 $\alpha \in \{0, 0.02, 0.04, 0.06, 0.08, 0.1, 0.2\}$ 共 7 个噪声水平
- 每噪声水平做 256 次独立 FMN 运行（16 CPU 核并行）
- 三个评估指标：EFR（有效特征频率占比）、DCG-1（排序质量）、DCG-2（排序+置信度）

### 核心结果

**标准基准 (Table 1-3)**:
| 数据集 | FePySR | 最佳竞争方法 |
|--------|--------|-------------|
| Nguyen (12 方程) | 97.75% | NGGP 91.4%, PySR 77.0% |
| Livermore (22 方程) | 77.45% | DSR 83.58% (唯一下风) |
| R (3 方程) | 41.33% | PySR 34.67% |
| Nguyen' (6 方程) | 89.67% | PySR 76.33% |
| Jin (6 方程) | 87.00% | PySR 76.17% |
| Constant (8 方程) | 91.38% | PySR 88.50% |
| **全部平均** | **81.11%** | NGGP 74.92% |

**LLM 合成复杂方程 (75 个)**:
- 可恢复 36 个：FePySR 平均恢复率 85.78%，vs PySR 20.24%，vs DSO 36.11%
- 不可恢复 39 个：FePySR 在 87.2% 的方程上 MSE 更低（vs PySR），其中 13 个误差比 < 0.5，3 个改进超 10 倍（error ratio < 0.1），Unrecover-9 达到 $2.79 \times 10^{-5}$ 极端比值

**Surprising 发现**:
- **无效但结构相似的特征仍有引导作用**：对 Recover-10 ($x^3+x^2+x+\sin(x)+\sin(y^2)$)，$\sin(y)$ 虽然不是有效特征（目标是 $\sin(y^2)$），但作为 partial approximation 降低了候选方程的 MSE，在 Pareto 前沿中被 GP 保留，进而引导搜索到 $\sin(\cdot)$ 区域，最终通过随机替换/交换操作找到 $\sin(y^2)$。这是一个重要 insight——特征提取不需要精确命中，近似即可提供 heuristic guidance。
- **FMN 的噪声鲁棒性与 PySR 的噪声敏感性脱钩**：Stage 1 的 FMN 在噪声下保持特征选择质量（多数方程 EFR/DCG 指标偏差 ≤5%），但 Stage 2 的 PySR 在最低噪声水平 ($\alpha=0.02$) 下恢复率即降至 0%。这说明两阶段的噪声鲁棒性是功能解耦的。
- **Nguyen-12 的关键突破**：所有 SOTA 在 Nguyen-12 上 0% 恢复，FePySR 达到 95%。原因是 FMN 提取出 $x^2$ 和 $y^2$，将方程从 17 个 token 的因子化形式压缩到 11 token ($x^2(x^2-x)+0.5y^2-y$)，从根本上简化了搜索树深度。

**生物 ODE 案例**:
- Tyson 双蛋白调控网络：100 次运行中 PySR 0 次恢复真结构（最佳结果：线性多项式，MSE=$1.23 \times 10^{-10}$）
- FePySR 24/100 恢复精确非线性动力学（MSE=$7.04\times 10^{-16}$），恢复的方程与 ground truth 结构高度一致

### 明确局限

1. **常数处理能力弱**：FMN 能识别 $\sin(x)$ 和 $\cos(y)$ 但不能识别 $\sin(1.5x)$ 和 $\cos(0.5y)$ 中的常数因子。作者提出可能的解决方案是用 LLM 维护动态特征库（类似 LASR），但尚未实现。
2. **Stage 2 噪声敏感性**：FMN 在噪声下稳定，但 PySR 在含噪数据上恢复率归零。需要 noise-robust SR 机制（如 denoising preprocessing 或 noise-aware loss）。
3. **没有真机实验**：所有评估均在合成数据和仿真 ODE 上进行，没有真实物理/生物实验数据验证。
4. **与 DSO 比较不完整**：由于核心算法差异（GP vs RL），作者承认无法在相同资源约束下与 DSO 公平比较，选择主要与 PySR 对比。
5. **Liermore-7/Livermore-8（双曲函数）0% 恢复**：$\sinh(x)$ 和 $\cosh(x)$ 是两个 FMN 原语库缺失的函数，导致完全无法恢复。
6. **对高层操作嵌套的局限**：$\sin(\cos(x) + \tan(yz))$ 这类深度嵌套需要原语库中包含 $\tan$ 等额外函数。
7. **特征提取质量依赖随机初始化和多次运行**：需 16 次独立训练取频率统计，计算成本不低。
8. **无理论保证**：没有关于 FMN 能找到真有效特征的概率下界的理论分析。

## Read by: 0616-fex (2026-06-24)

### 撞车评估
**不撞车。** FePySR 解决的是符号回归（SR）问题，目标是给定观测数据恢复闭合形式的数学表达式。我们的 0616-fex 课题聚焦 PDE/物理系统的 feature extraction 和 foundation model，问题域不同。

但 FMN 作为一个 "用可解释数学原语做 feature extraction" 的神经网络，其设计思想与我们的 feature discovery 有交集。特别是它证明了：
1. 异构激活函数（每种原语一个 HAU）比 EQL 的单体网络更能学到 diverse features
2. 通过多次随机初始化+频率统计可以区分 signal 和 noise
3. 近似特征（不完全等于真特征）仍能提供 heuristic guidance

### 可用工具
- **HAU 设计模式**：用数学原语替换标准激活函数做 feature engineering
- **Contrastive loss for operator anti-collapse**：$\lambda_2 \cos(W_1, W_2)$ 简单有效
- **梯度归一化**：处理 multi-operator 训练时的梯度尺度失衡
- **Mask-based 数值稳定**：截断级联指数传播防止数值溢出，适用于 deep operator networks

### 差异化与下一步
FePySR 是"先提取特征再搜索表达式"的 pipeline，而我们关注的是"特征本身的表示学习"。可以把 FMN 的思路拆出来——用可解释算子构建的异构 NN 做 feature extractor——这个组件本身与我们的方向高度兼容。建议关注 FMN 在各噪声水平下的 EFR/DCG 指标作为 feature extraction 质量的评估参考。
