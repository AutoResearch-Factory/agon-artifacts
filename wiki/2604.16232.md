# Neuro-Symbolic ODE Discovery with Latent Grammar Flow (2604.16232)

- **Year**: 2026.04
- **Authors**: Karin L. Yu (ETH AI Center), Eleni Chatzi (ETH Zurich), Georgios Kissas (Swiss Data Science Center)
- **Venue**: arXiv preprint
- **Keywords**: neuro-symbolic, equation discovery, ODE discovery, grammar-based representation, discrete latent space, flow matching, symbolic regression, grammar quantisation autoencoder
- **One-liner**: 用文法编码 ODE 结构进离散潜空间，再通过离散 flow 模型和稳定性/阶数引导采样来发现显式和隐式 ODE。

## 精华提炼

### 管线/方法

LGF 是一个两阶段嵌套的 neuro-symbolic 方程发现框架：

**阶段 1 — 训练（离线）**
1. **Grammar parsing**: 将 ODE 表达式解析为上下文无关文法 (CFG) 的产生式规则序列。数值常量替换为占位符 C，后续用 Nelder-Mead 优化。
2. **Grammar Quantisation Autoencoder (GQAE)**: 编码器用 3 层 CNN (kernel 7/8/9) + 2 层残差网络将 one-hot 规则序列映射到 latent 分布；用 FSQ (Finite Scalar Quantization, Mentzer et al. 2023) 将连续 latent 向量离散化；解码器用 2 层双向 GRU (hidden dim=80, ELU 激活) 重建规则序列。
3. **Behavioural loss (Wasserstein distance)**: 对每个 ODE 结构，用 25 次随机标量采样 + 2,048 个向量场采样点 (t: [0,10]×8, u: [0,100]×16, du: [0,500]×16) 计算均值/标准差，再用 log-Wasserstein-2 距离作为语义距离。该 loss 将行为相似的方程拉近 (\( \mathcal{L}_W = ||z_{e,i}-z_{e,j}||_2 - \lambda_W W_2 \))。
4. **Discrete flow model**: 用 Campbell et al. 2024 的离散状态空间 flow matching 在 GQAE 的离散 latent 空间中建模分布。训练 denoising model (2 层 MLP, hidden=5000, ReLU, 20% dropout, 300 epochs)。
5. **Predictors (guidance models)**: 针对 order (分类)、stability (分类)、optimization objective (连续) 分别训练 predictor MLP (1 层 hidden=500, ELU, 20% dropout, 200 epochs)。扩展 Nisonoff et al. 2025 的离散 guidance 到多 predictor：\( p^{\phi}(y|x,t) = p^{\phi_1}(y_1|x,t) \prod_i p^{\phi_i}(y_i|y_{1:i-1},x,t) \)。

**阶段 2 — 发现（在线）**
1. **Outer loop (skeleton discovery)**: 离散 flow 模型从噪声开始采样 latent code → 解码器生成 ODE skeleton（规则序列）。初始种群 \(n_{pop,0}\)，之后每轮 \(n_{pop}=100/250\)。最多 \(i_{max}^{out}=10\) 轮。收敛条件：新 unique 表达式数 < \(\varepsilon_{unique} n_{pop}\) 且 top-k 目标停滞。
2. **Inner loop (constant optimization)**: Nelder-Mead 算法两阶段优化标量常量。阶段 1 (tol=0.1/0.5, max 50 iter) → 若 loss < 25 则阶段 2 (tol=0.001/0.1, max 150-250 iter)。梯度不可用故选导数自由法。
3. **Evaluation strategy**: 先用 residual loss \( \mathcal{L}_{DE} \) (ODE evaluation MSE)，若 < 阈值 200 再算 solution loss \( \mathcal{L}_{SOL} \) (trajectory SMSE)，避免对差表达式求解 ODE。
4. **Equation simplification**: 消除求和项（平方和 < 0.01）或乘法项（与 1 的绝对误差和 < 0.01）。
5. **Stability guidance**: 对自治系统用 Lyapunov 间接法（线性化 → Jacobian 特征值）；对非自治系统用数值耗散性近似。

### 关键技术细节

- **Grammar**: CFG 包含 terminal/nonterminal 符号集、~20-30 条产生式规则。标量不直接编码，用 C 占位符。
- **GQAE latent space**: 维度 d 按 Benchmark 不同 (23/24/16)，FSQ levels \(n_{FSQ}=7/9/10\)，channels \(n_{FSQ,cha}=4\)。等效 codebook size = \((n_{FSQ})^{n_{FSQ,cha}}\)。
- **FSQ (Finite Scalar Quantization)**: 本质是对每个 channel 做 round 到离散 level，无需训练 codebook。比 VQ-VAE 训练更简单。
- **Training data generation**: 每个 benchmark 的 grammar + dataset 复用 GODE (Yu et al. 2025) 的 GitHub repo。合成 ODE 数据 + 5% Gaussian noise。
- **Benchmark 1**: 30 个一阶显式 ODE（23 个 ODEBench + 7 个 GODE），population 100，10 outer iterations，最多 200 次 scalar optimization iter。GQAE: d=23, n_FSQ=10, beta_W=1e-4, n_res=60, n_shape=92, S=5。
- **Benchmark 2**: 10 个混合 ODE（5 线性 + 5 非线性），使用 order + stability guidance。GQAE: d=24, n_FSQ=9, beta_W=1e-4, n_res=60, n_shape=96, S=5。Inner max iter=250。
- **Benchmark 3**: 7 个二阶动力系统（pendulum, Duffing, Van der Pol, damped, exp stiffness, nonlinear damping ×2），加速测量 (ddot) 作为观测，局部可观测。GQAE: d=16, n_FSQ=7, beta_W=1e-5, n_res=40, n_shape=64, S=4。Population 250，inner max=300 iter。
- **Flow model 超参**: noise eta=10, temperature x1=1, time step=0.01, guide temperature=1。Convergence: top-k=10% population, epsilon_IC,k=0.01, epsilon_unique=0.1。
- **Nelder-Mead**: fatol=0.1→0.001, xatol=0.5→0.1, 两阶段共 200-350 iter。
- **Objective**: \( \mathcal{L}_{IC} = 0.1\mathcal{C} + 0.9 n_s \mathcal{L}_{accuracy} \)，复杂度 C = 操作数+变量数+常量数。
- **Computational cost**: LGF 探索 ~1,000-2,500 skeleton expressions。对比 ODEFormer 需要 ~50M 训练样本和 60.7M 参数，PySR 技术可探索 4.56M-18M 表达式。LGF 算竞品中较节约的。

Benchmark 3 (LGF vs PySR, 7 systems): LGF 在 6/7 个系统上全面优于 PySR（ut/udot/uddot 三个指标）。PySR 只在 "Nonlinear damping 1" 上胜出。显著案例：Van der Pol 振荡器 LGF rel L2=0.115 vs PySR=20.517 (ut)；Pendulum LGF=0.015 vs PySR=3.707 (ut)。

**主要发现**: PySR 作为函数逼近器不求解 ODE，其 residual low loss 不代表解轨迹匹配。LGF 通过求解 ODE 并与轨迹对比，发现更准确的方程。Benchmark 3 中 PySR 的某些案例看似低 loss 但解轨迹完全偏离真实。

### 明确局限

1. **仅支持一维 ODE**（一阶和二阶 single-degree-of-freedom），未扩展到高维或 ODE 系统。作者声称可扩展但未验证。
2. **Grammar 需人工设计**: 每个问题域需要设计 CFG，可迁移性受限。对比 PySR 是无 grammar 的。
3. **Nelder-Mead 可扩展性**: 导数自由的标量优化对常量数增加时效率下降。无梯度不可自动微分。
4. **稳定性判断近似**: 非自治系统用数值近似而非严格 Lyapunov 方法，不保证正确性。
5. **仅 5% Gaussian noise**: 噪声水平的鲁棒性未被系统研究。真实物理系统噪声可能更高且非高斯。
6. **无开源代码**: 论文未声明代码开源（对比 GODE 有 GitHub repo）。复现困难。
7. **GQAE + flow model 需要预训练**: 对每个新 grammar 需要重新训练 GQAE 和 flow model，训练成本不低。
8. **Benchmark 规模有限**: 最多 30 个 ODE，缺乏更大规模系统比较。与 ODEFormer 相比在 B1 上仍有差距 (0.117 vs 0.084 PySR)。

**撞车评估**:

- LGF 的核心新意在于 (a) 用 FSQ 做离散 latent embedding 替代连续 VAE (b) 用 discrete flow matching 替代 evolution strategy 做 latent 采样 (c) 扩展 of discrete guidance 到多 predictor。若 0616-fex 的 proposal 包含这些中的任何一项，需要差异化。
- 如果 0616-fex 是做 soft object BFM / robotics 的应用，则 LGF 是方法论上游，不直接撞车。

**可用工具/方法**:
- GQAE + behavioural Wasserstein loss 可以用于任何需要结构+语义联合嵌入的场景。
- Discrete flow matching with multi-predictor guidance 是一个通用的 controlled generation 思路，可以移植到其他离散结构生成问题。
- 两阶段标量优化（粗搜→精搜）可复用。

**差异化建议**:
- 将 LGF 扩展到高维/系统 of ODEs（作者留的未来工作）。
- 在真实物理系统（而非合成数据）上验证。
- 将 grammar 换成 learning-based tokenization（如 transformer），去掉手工设计 grammar 的约束。
- 结合 diffusion 替代 flow matching，或探索更高效的标量优化（如梯度-based 方法用于连续部分）。
