# Softmax Policy Gradient Methods Can Take Exponential Time to Converge (2102.11270)

- **Year**: 2021.02, revised 2022.12
- **Authors**: Gen Li, Yuting Wei, Yuejie Chi, Yuantao Gu, Yuxin Chen (UPenn/CMU 等；tex 题注列出 UPenn 与 CMU 作者信息)
- **Venue**: arXiv; presented in part at COLT 2021
- **Keywords**: softmax policy gradient, tabular MDP, discounted reinforcement learning, lower bound, crossing time, state visitation, natural policy gradient, regularization
- **One-liner**: 构造硬 MDP 证明软最大 PG 可超指数慢。

## 精华提炼

### 管线/方法

- 论文研究对象是 vanilla softmax policy gradient，而不是 NPG、mirror descent、actor-critic 或带正则的策略优化。
- 输入问题类是 $\gamma$-discounted infinite-horizon tabular MDP。
- 策略参数化为 softmax：每个状态的动作 logits $\theta(s,a)$ 经过 softmax 得到 $\pi(a|s)$。
- 算法假设能访问 exact policy gradient，因此 lower bound 不来自采样噪声。
- 初始化使用 uniform policy，作者刻意避免把困难藏在坏初始化里。
- 初始状态分布也是 uniform over states，作者刻意避免把困难藏在单点初始分布或极端 exploration failure 里。
- 主要输出是一个显式构造的 hard MDP family。
- hard MDP 的每个状态至多有 3 个动作，动作集合是 $\{a_0,a_1,a_2\}$ 的子集。
- 证明主线不是经验实验，而是完整追踪 softmax PG 的参数动力学。
- 作者先构造原始 MDP，证明许多 primary states 在指数长时间内过不了 value threshold。
- 然后修改 booster states，把单点慢收敛提升为 average value error 慢收敛。
- 证明围绕 crossing time 展开：$t_s(\tau)$ 是状态 $s$ 的 value 第一次达到阈值 $\tau$ 的迭代数。
- 类似地定义 adjoint state 和 buffer state 的 crossing time。
- 从 buffer states 的启动时间开始，作者建立 primary state 的递推式。
- 递推式的核心是：状态 $s$ 的启动依赖更早的状态 $s-2$，且等待时间按 $1.5$ 次幂放大。
- 这个幂次放大沿着 $H=\Theta(1/(1-\gamma))$ 层传播，最终得到 $|S|^{2^{\Omega(1/(1-\gamma))}}$。
- 因此困难来自 horizon 和 state size 的耦合，而不是单独的常数因子。
- 论文把 NPG、policy mirror descent、log-barrier 或 entropy regularization 放在对照位置，说明 naive softmax PG 的更新几何可能是问题本身。

### 关键技术细节

- 主定理假设 $0.96 < \gamma < 1$。
- 步长假设为 $0 < \eta < (1-\gamma)^2/5$。
- 状态规模需要 $|S| \ge c_3(1-\gamma)^{-6}$。
- 主定理给出的迭代下界为 $(c_1/\eta)|S|^{2^{c_2/(1-\gamma)}}$。
- 在这么多迭代之前，average value error 仍不能降到 $0.07$ 以下。
- hard MDP 中 action 数不超过 3，这排除了“大动作空间导致慢”的简单解释。
- 状态集合包含 absorbing state $0$。
- 还包含两个 buffer sets：$S_1$ 和 $S_2$。
- 还包含 primary states $\{3,\ldots,H\}$。
- 还包含 adjoint states $\{\bar{1},\ldots,\bar{H}\}$。
- 还包含 booster sets $\hat{S}_s$ 与 $\hat{S}_{\bar{s}}$，用于放大平均误差。
- 有效层数取 $H=c_h/(1-\gamma)$。
- $|S_1|=c_{b1}(1-\gamma)|S|$。
- $|S_2|=c_{b2}(1-\gamma)|S|$。
- 每个 booster set 的规模为 $c_m(1-\gamma)|S|$。
- 关键阈值 $\tau_s=0.5\gamma^{2s/3}$。
- 转移概率参数 $p=c_p(1-\gamma)$。
- primary state 的奖励尺度为 $r_s=0.5\gamma^{2s/3+5/6}$。
- 最优策略在所有非吸收状态选 $a_1$。
- 最优 value 形如 $V^\star(s)=\gamma^{2s}$，$V^\star(\bar{s})=\gamma^{2s+1}$。
- buffer state 1 中 $a_0$ 和 $a_1$ 的 Q 值分别为 $-\gamma^2$ 与 $\gamma^2$。
- buffer state 2 中 $a_0$ 和 $a_1$ 的 Q 值分别为 $-\gamma^4$ 与 $\gamma^4$。
- primary state $s \ge 3$ 的 $a_0$ 直接去 absorbing state，并得到带 $p\tau_{s-2}$ 项的即时奖励。
- primary state 的 $a_1$ 转到 $\bar{s-1}$，这是最优但 softmax PG 会很晚才把概率推上去的动作。
- primary state 的 $a_2$ 以概率 $p$ 转到 $\bar{s-2}$，否则去 absorbing state。
- adjoint state $\bar{s}$ 的 $a_1$ 回到 $s$，$a_0$ 则终止到 absorbing state。
- booster states 最初用于制造 visitation lower bound，最终版本还承担 average error 的质量。
- 证明需要反复控制 discounted state visitation probabilities。
- primary 和 adjoint states 在关键时间前的 visitation 量级约为 $(1-\gamma)^2$。
- 单个 buffer state 的 visitation 量级约为 $(1-\gamma)/|S|$。
- 这个 visitation 差异使得 buffer 启动慢，而后续状态又继承并放大这种慢。
- 初始阶段中，在 $t_{s-2}(\tau_{s-2})$ 前，$\theta(s,a_1)$ 会被压低。
- 同期 $\theta(s,a_0)$ 和 $\theta(s,a_2)$ 保持非负，并且常有 $a_0 \ge a_2 \ge a_1$ 的排序结构。
- 中间阶段中，哪怕 $s-2$ 已经启动，$a_1$ 也无法立刻追上。
- blowing-up lemma 给出关键放大：$t_s$ 相对 $t_{s-2}$ 至少有约 $[t_{s-2}]^{1.5}$ 级别的延迟。
- 归纳式最终形如 $t_s(0.5)\ge c_1|S|^{2/3}\eta^{-1}(c_2|S|)^{(1/3)1.5^{\lfloor s/2\rfloor}}$。
- 因为 $s$ 可以走到 $\Theta(1/(1-\gamma))$，$1.5^{s/2}$ 变成 horizon 指数。
- 修改后的 booster state 有两个动作，让与慢 primary 或 adjoint state 关联的大量 booster states 也保持 suboptimal。
- 最终 average error 不是靠少数罕见状态，而是靠一大批 booster states 的总质量支撑。

### 核心结果

- 最重要结论：exact softmax PG 仍可能需要 $(1/\eta)|S|^{2^{\Omega(1/(1-\gamma))}}$ 次迭代。
- 这个下界同时依赖状态数 $|S|$ 和 effective horizon $1/(1-\gamma)$。
- 结论比“常数慢”或“多项式慢”强得多，是对 naive softmax PG 的结构性负面结果。
- 下界在 uniform initialization 下成立，因此不能简单归咎于初始策略太差。
- 下界在 uniform initial distribution 下成立，因此不能简单归咎于初始状态分布太偏。
- 下界在 exact gradient 下成立，因此不能归咎于 Monte Carlo 方差或估计误差。
- 下界在三动作 MDP 下成立，因此动作空间复杂度不是主因。
- 对照结果是 NPG 已有 dimension-free 或 horizon 依赖更清晰的 upper bound。
- 论文强调 softmax PG 的更新方向和参数化会使最优动作概率长期受抑制。
- 令人意外的点是：最优动作 $a_1$ 的优势并非不存在，但 logits 动力学仍可让它迟迟无法变主导。
- 另一个反直觉点是：探索分布看起来友好，仍不能阻止 crossing time 被层层放大。
- 本文的贡献不是提出更快算法，而是说明为什么需要调整 update rule 或加入合适正则。

### 明确局限

- 这是 worst-case 理论构造，不是常见 benchmark 的经验失败案例。
- 论文没有真机实验，也没有 deep RL benchmark。
- 分析限定在 tabular discounted MDP。
- 分析不覆盖函数逼近、神经网络策略、continuous control 或 partial observability。
- 分析使用 exact gradient，不处理采样噪声、off-policy data 或 finite-sample estimator。
- 结论针对 vanilla softmax PG，不能直接外推到 NPG、PPO、TRPO 或 mirror descent。
- 步长条件是 constant stepsize 且 $\eta<(1-\gamma)^2/5$，不覆盖所有 adaptive stepsize 策略。
- hard MDP 是精心工程化的，论文没有证明这种慢收敛在自然任务中普遍发生。
- 作者没有给出可运行 artifact，但证明本身在 tex 中足够完整。
- entropy regularization 和 log-barrier 等 remedy 只作为动机或对照，不是本文主结果。
- average error 的最终构造依赖 booster states，读者需要区分“单个链条慢”和“总体平均慢”的两层证明。
- 如果我们的课题使用经过归一化、trust-region、entropy 或 verifier-gated 的更新，本论文更多是负面对照，而不是直接撞车。

- 对 `0616-fex` 来说，这篇论文主要是一个负面理论基线。
- 如果课题 claim 是“plain softmax PG 在长 horizon tabular RL 中高效”，这篇会直接撞车。
- 如果课题使用 NPG、mirror descent、trust region、entropy/log-barrier regularization 或 verifier-gated reward，本篇不直接撞车。
- 它能提醒我们在写方法时不要把 softmax 参数化和 policy-gradient 更新混为“天然稳定”。
- 它能作为引用支撑：naive PG 在精确梯度和友好初始化下也可能慢。
- 对实验设计的启发是：需要单独报告 horizon、state coverage、action suppression、distribution mismatch 和 slow crossing 现象。
- 对理论叙事的启发是：如果我们的方法声称解决 long-horizon credit assignment，需要说明是否改变了 update geometry。
- 对差异化最有用的角度是：把我们的机制定位为避免最优动作概率被长期压低，而不是只说提高样本效率。
- 适合被引在 related work 的“policy optimization lower bounds”和“why regularization or natural gradients matter”两段。
- 不建议把它当作 deep RL 或 LLM RL 的直接反例，除非我们明确建立 tabular abstraction 或 pattern-selection MDP 映射。

## Read by: 0616-fex (2026-06-17)

本次审读结论：这篇不太像会和 `0616-fex` 的正向方法 claim 撞车，除非我们的 claim 明确落在 vanilla softmax PG、tabular discounted MDP、exact gradient、无正则更新这组设定上。它更适合作为负面对照和理论风险提示：即使初始化和初始状态分布都很温和，softmax PG 的 logits 动力学也可能让最优动作概率被长期压低，并通过 long-horizon 结构放大到超指数迭代复杂度。

可用的方法线索有三类。第一，任何 FEX 方法如果用 policy gradient，需要明确区分 softmax PG 与 natural gradient、mirror descent、trust-region 或 entropy/log-barrier regularized updates。第二，评测时应检查动作概率是否出现 valid action suppression、state visitation 是否被少数路径主导、crossing time 是否随 horizon 爆炸。第三，写 related work 时可以引用本文支撑“plain PG 的失败不是采样噪声，而可能是 update geometry 本身”的论点。差异化可以放在：我们的机制改变 reward/verification/update 的闭环，使低概率但关键的有效动作不会长期失去梯度通道。
