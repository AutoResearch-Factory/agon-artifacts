- **Year**: 2023.06 (NeurIPS 2023 Datasets and Benchmarks track)
- **Authors**: Cheng Tan, Siyuan Li (equal contribution), Zhangyang Gao, Wenfei Guan, Zedong Wang, Zicheng Liu, Lirong Wu, Stan Z. Li (通讯); Westlake University AI Lab + 浙江大学 + 西安电子科技大学
- **Venue**: NeurIPS 2023 Datasets and Benchmarks
- **Keywords**: spatio-temporal predictive learning, video prediction benchmark, recurrent-free, SimVP, MetaFormer, WeatherBench, nowcasting baseline, unified codebase
- **One-liner**: 统一代码库复现 14 算法/24 模型跑 7 个时空预测数据集，发现 recurrent-free 模型以约 1/5 FLOPs 在低频宏观任务（天气/交通）上全面胜过 recurrent 模型。

## 精华提炼

### 管线/方法

OpenSTL 不提出新预测算法，核心贡献是一个**统一实现 + 统一评测协议**的 benchmark 框架：

1. **问题形式化**: 输入过去 T 帧张量 X ∈ R^{T×C×H×W}，预测未来 T' 帧 Y ∈ R^{T'×C×H×W}，模型 F_Θ 最小化预测帧与真值帧的 discrepancy（确定性回归，MSE 系 loss）。
2. **方法二分类**（本文的组织性贡献）:
   - **Recurrent-based**（11 个）: F_Θ 由共享参数的 f_θ(x^i, h^i) 逐帧自回归复合，隐状态 h 传递历史。含 ConvLSTM (NeurIPS 2015)、PredNet (ICLR 2017)、PredRNN (NeurIPS 2017, ST-LSTM)、PredRNN++ (ICML 2018, Causal-LSTM + gradient highway)、MIM (CVPR 2019, 非平稳性建模)、E3D-LSTM (ICLR 2019, Conv3D)、CrevNet (ICLR 2020, 信息保持)、PhyDNet (CVPR 2020, PDE 解耦 PhyCell)、MAU (NeurIPS 2021, 运动感知)、PredRNNv2 (TPAMI 2022, 课程学习+记忆解耦)、DMVFN (CVPR 2023, 多尺度体素流, 逐帧预测)。
   - **Recurrent-free**（3 个 + 10 个 MetaVP 变体）: encoder(2D conv 降维) → 时间模块（把 T 帧沿通道堆叠成 T×C 一次性处理）→ decoder(2D 反卷积)，并行推理。含 SimVP (CVPR 2022, IncepU)、TAU (CVPR 2023, 时间注意力)、SimVPv2 (gSTA)。
3. **MetaVP 抽象**（本文的方法性贡献）: 论证 recurrent-free 的能力主要来自"encoder–时间模块–decoder"通用架构而非特定时间模块，于是把时间模块替换为任意 MetaFormer——实现了 ViT、Swin、Uniformer、MLP-Mixer、ConvMixer、PoolFormer、ConvNeXt、VAN、HorNet、MogaNet 共 10 种，把时空预测重构为"视觉 backbone 设计的下游任务"。
4. **代码库三层结构**: core layer（dataloader/模块/metric 统一接口）→ algorithm layer（PyTorch 复现，对照官方代码）→ user interface layer（config/训练/评估/可视化脚本），遵循 OpenMMLab 设计规范，吸收 OpenMixup 与 USB 的元素。

### 关键技术细节

- **7 个数据集**（5 类任务）: Moving MNIST 三变体（10k 训练/10k 测试, 64×64, 10→10 帧）；KTH 人体动作（4,940/3,030, 128×128, 10→20/40）；Human3.6M（73,404/8,582, 3ch 128×128, 4→4）；Kitti&Caltech 驾驶场景（3,160/3,095, 128×160, 10→1）；TaxiBJ 交通流（20,461/500, 2ch 32×32, 4→4, 30min 间隔）；WeatherBench-S 单变量（2,167/706, 32×64 即 5.625° 或 128×256 即 1.40625°, 12→12, 1h 间隔, 2010-2015 训练/2016 验证/2017-2018 测试）；WeatherBench-M 多变量（54,019/2,883, 4ch(t/r/u/v), 4→4, 6h 间隔, 1979-2015 训练）。
- **鲁棒性三设置**（Moving MNIST 上）: Missing（20% 随机丢帧）、Dynamic（数字速度加高斯噪声）、Perceptual（24×24 黑块随机遮挡）。
- **指标四组**: 误差（MSE/MAE/RMSE）、相似性（SSIM/PSNR）、感知（LPIPS, 只用于真实视频）、计算（Params/FLOPs/FPS, 单卡 V100）。
- **超参**: Adam；lr ∈ {1e-2 … 1e-4} 网格取最优；drop path ∈ {0, 0.1, 0.2}；epoch 200(MMNIST)/100(KITTI,KTH)/50(其余)；recurrent-free 统一 hid_S 32-64、hid_T 256-512、N_S 2-4、N_T 6-8。
- 无误差棒/多随机种子（checklist 中作者自己承认 answerNo），每格取网格搜索最优单次结果。

### 核心结果

- **Moving MNIST**（高频合成）: recurrent-based 最优——PredRNN++ MSE 22.06 vs 最好的 recurrent-free TAU 24.60；但 PredRNN++ 171.7 GFLOPs / 38 FPS vs TAU 16.0 GFLOPs / 283 FPS，约 10 倍效率差。
- **Kitti&Caltech**（真实驾驶）: PredRNN++ MSE 125.5 (1803 GFLOPs) vs SimVPv2 129.7 (96.3 GFLOPs)——精度差 3% 而计算差 ~19 倍。
- **WeatherBench 单变量**（低频宏观）: **recurrent-free 全面碾压**。温度: SimVPv2 MSE 1.105 vs 最好 recurrent MAU 1.251；湿度: Swin 31.33 vs MAU 34.53；风: Uniformer 1.485 vs PredRNN++ 1.873；云量: 所有 recurrent-free 均优于所有 recurrent-based。高分辨率温度 (128×256): SimVPv2 0.650 vs PredRNN++ 0.854 且 PredRNN++ 只有 2 FPS。
- **WeatherBench-M 多变量**: MogaNet 版 MetaVP 四变量全部最优（温度 MSE 4.83 vs PredRNN 5.60）。
- **鲁棒性**: Missing/Perceptual 下 recurrent-free 几乎无退化（TAU 26.77/26.87 vs 干净 24.60），recurrent-based 大幅崩坏（PredRNN 23.97→46.05/46.04）；Dynamic 噪声下所有方法都显著退化（PredRNN++ 40.85 最好）。
- **反直觉点**: (1) PhyDNet 在 WeatherBench 上灾难性失败（温度 MSE 285.9, 其他方法 ~1.2-1.8），物理先验模块在真实气象场上完全不 work；(2) DMVFN 在 Human3.6M 上最优（MSE 109.3）但在 Moving MNIST 上近乎失败（123.67），逐帧预测模型极度依赖数据频率特性；(3) 10 种 MetaFormer 时间模块之间差异远小于"有无 recurrent"这一架构选择，支持"架构 > 模块"论断。
- **总解释框架**: recurrent 提供局部（逐帧）时间归纳偏置 → 适合高频信号；recurrent-free 堆叠全部帧 → 全局时间归纳偏置 → 适合低频信号。天气（宏观、缓变）属于低频，故 recurrent-free 胜。

### 明确局限

- **作者自述**: 无误差棒、无算力总量报告；开放问题是如何融合两类模型优势、能否继续改进 MetaFormer 时间模块。
- **审读读出的局限（对我们最重要）**:
  1. **指标缺口**: 只有 MSE/MAE/RMSE/SSIM/PSNR/LPIPS——**没有任何气象分类技巧评分（CSI/HSS/FSS/POD/FAR）、没有极值敏感指标、没有概率/ensemble 指标（CRPS）**。确定性 MSE 训练+评测对强对流这类小概率高影响事件会系统性奖励模糊预测。
  2. **无对流尺度任务**: WeatherBench 是 5.625°/1.40625° 的全球再分析低分辨率场（t2m/r/u/v/tcc），**无雷达、无 SEVIR/HKO-7、无对流风暴事件**，"weather forecasting"结论只覆盖宏观缓变场。
  3. **无环境场-事件耦合**: 各数据集都是单一模态自回归预测，没有"环境条件 + 观测 + 事件标签"的多模态结构。
  4. 数据集时间划分是简单年份切分（气象）或随机切分（视频），无 leakage 讨论。
  5. Benchmark 后续已被社区扩展（OpenSTL-v2/后续 citing 工作加了 Mamba 类、diffusion 类），本文快照止于 2023 中。

### 跟我们的关系

对 **0713-scs-benchmark**（强对流环境–雷达多模态开放基准数据集, ESSD / NeurIPS D&B 方向）：

- **撞车评估：不撞车，完全互补**。OpenSTL 是**方法 benchmark**（统一算法库+评测协议，用现成数据集），我们是**数据 benchmark**（新数据产品：Storm Events 事件标签 + HRRR 长时窗环境场 + MRMS 雷达的事件包）。OpenSTL 没有雷达、没有对流事件、没有环境场耦合、没有气象技巧评分——这四个缺口恰是我们的核心贡献面。二者在 NeurIPS D&B track 是"数据集论文 vs 系统/库论文"两个不同赛道。
- **可用工具（直接复用价值高）**:
  1. **Baseline zoo**: 我们 benchmark 的 MRMS nowcasting canonical task 需要 baselines，OpenSTL 的 14 算法统一实现（尤其 ConvLSTM/PredRNN 系 + SimVP/TAU）可直接跑我们的事件包数据，省掉自己复现的工程量；其 config 系统支持换 dataloader 即接入新数据集。
  2. **论文模板**: 它是 NeurIPS D&B 收录的时空预测 benchmark 先例——其"方法分类表 + 数据集统计表 + 指标四组（含 Params/FLOPs/FPS 效率维度）+ 鲁棒性 setting"结构可作为我们 benchmark 论文实验章节的模板。
  3. **科学问题移植**: 其核心发现"recurrent-free 在低频任务占优、高频任务 recurrent 占优"给我们送来一个现成的 benchmark 示范分析问题——**对流尺度雷达回波（高频）+ 缓变环境场（低频）的多模态预测中哪类架构占优**？这正好用我们的双时间尺度事件包回答，且 OpenSTL 的频率解释框架预言两种模态答案相反，天然有戏剧性。
- **该引**: 必须引。引用位置：(a) related work 的 benchmark 段（方法 benchmark vs 数据 benchmark 的区分）；(b) baseline 选择依据；(c) 指标章节——指出其指标体系对极端事件的不足以论证我们引入 CSI/FSS/CRPS 的必要性。
- **差异化**: 我们不做算法复现贡献，做数据/整编/任务定义贡献；我们的评测协议要补齐它缺的事件感知指标与防泄漏时间划分——这两点可以在论文里明确写成"beyond OpenSTL"的表格对比。

## Read by: 0713-scs-benchmark (2026-07-14)

**审读视角**: 我们要建 SCS 环境-雷达-事件多模态基准数据集，OpenSTL 是 NeurIPS D&B 同 track 的方法 benchmark 先例。

- **撞车**: 无。它无雷达/对流/环境场/事件标签，是算法库论文；我们是数据产品论文。它反而是我们 canonical task 的 baseline 供应商。
- **直接可用**: (1) 其开源代码库（github.com/chengtan9907/OpenSTL）作为 MRMS nowcasting baseline zoo——换 dataloader 即接入；(2) 其"数据集统计表 + 指标四组 + 效率维度"的论文结构作 D&B 提交模板；(3) 其鲁棒性 setting 设计（丢帧/噪声/遮挡）可移植为我们的"传感器缺失/QC 噪声"stress test。
- **差异化弹药**: OpenSTL 的指标体系（MSE/SSIM 系）无 CSI/HSS/FSS/CRPS、无极端事件敏感度、时间划分无泄漏讨论——在我们论文里可作为"现有时空 benchmark 不适配强对流评估"论据的具体靶子。其"recurrent-free 胜于低频、recurrent 胜于高频"发现可转化为我们的示范分析：双时间尺度事件包（缓变环境场 vs 高频雷达回波）上架构优势是否翻转。
- **行动**: baseline 复用优先级高；论文引用必须；无需跟进其后续版本细节（社区扩展由 citing 工作覆盖）。
