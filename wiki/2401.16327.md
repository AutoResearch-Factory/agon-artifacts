# PICL: Physics Informed Contrastive Learning for Partial Differential Equations (2401.16327)

- **Year**: 2024.01
- **Authors**: Cooper Lorsung, Amir Barati Farimani (Carnegie Mellon University, MechE/BME/MLD)
- **Venue**: arXiv (AIP/REVTeX format)
- **Keywords**: contrastive learning, neural operator, Fourier Neural Operator, PDE surrogate, physics-informed pretraining, multi-system generalization, operator learning
- **One-liner**: 提出 Physics Informed Contrastive Learning (PICL) 框架，利用幅度感知余弦相似度和物理信息距离函数进行对比预训练，显著提升 FNO 在多个 PDE 系统间的泛化能力。

## 精华提炼

### 管线/方法

完整的 PICL 两阶段训练流程：

**阶段一：对比预训练 (Contrastive Pretraining)**
- 输入：来自 Heat、Burgers'、Linear Advection 三个方程的多条 trajectory（每个方程 3072-5000 条样本，1D 或 2D）
- 每条样本包含：初始条件 $u^t$、下一帧 $u^{t+1}$、方程系数向量 $\theta = [\alpha, \beta, \gamma]$
- FNO 编码器将输入映射到 latent embedding $z = G_\theta(u)$
- 计算三个关键组件：
  1. **System distance** $d_{system} = u_i^{t+1} - u_j^t$ — 输入帧间差异
  2. **Update distance** $d_{update} = F(G_\theta(u_i^t)) - G_\theta(u_j^t)$ — 其中 $F(\cdot)$ 是物理信息数值更新算子（有限差分近似）
  3. **Physics distance** $d_{physics} = \|d_{system} - d_{update}\|^2$ — 锚定系统距离与更新距离
- **Similarity metric** $\psi(\theta_i, \theta_j) = \sqrt{|\theta_i \cdot \theta_j|} / \max(\|\theta_i\|_2, \|\theta_j\|_2)$ — 幅度感知余弦相似度
- 损失函数：Generalized Contrastive Loss (GCL)：$\mathcal{L} = \psi \cdot d^2/2 + (1-\psi) \cdot \max(\tau - d, 0)^2/2$
- 预训练时使用 projection layer 将 latent 投影到与输入相同维度以使用有限差分更新
- 1D 预训练：20 epochs（fixed-future）或 5 epochs（autoregressive）

**阶段二：微调 (Fine-tuning)**
- 冻结丢弃 projection layer，使用标准监督训练
- 在每个单独方程上 fine-tune（不冻结权重，经验发现冻结会降低精度）
- 优化目标：相对 $L^2$ 范数
- 1D fine-tune：500 epochs（fixed-future）或 100 epochs（autoregressive）
- OneCycle learning rate scheduler 比 step scheduler 效果更好

### 关键技术细节

**模型架构：**
- Backbone：Fourier Neural Operator (FNO)
- 1D FNO：hidden width=32, 8 modes, 3 layers
- 2D FNO：hidden width=48, 4 modes, 4 layers
- Projection layer：将 latent embedding 投影到与输入相同维度

**训练超参数（1D Fixed-Future）：**
- Pretrain：batch size=512, lr=1E-2, weight decay=1E-8, dropout=0, τ=5
- Fine-tune/Baseline：batch size=32, lr=1E-3, weight decay=1E-4, dropout=0

**训练超参数（1D Autoregressive）：**
- Pretrain：batch size=10, lr=1E-2, weight decay=1E-8, τ=1
- Fine-tune：batch size=16, lr=1E-2 或 1E-3（FNO baseline）

**训练超参数（2D Fixed-Future）：**
- Pretrain：batch size=256, lr=1E-2, weight decay=1E-7, τ=Mean（每批次的平均距离值）
- Fine-tune/Baseline：batch size=32, lr=1E-2, weight decay=1E-7

**数据生成：**
- 1D：WENO5 方法生成 Heat 和 Burgers' 数据；解析解生成 Advection 数据
  - 空间离散：200 点 → 降采样至 50；时间离散：200 步 → 降采样至 50
  - 系数：α∈{0.5,1.0,2.0,5.0}, β∈{0.2,0.5,1.0,2.0,5.0}, γ∈{0.5,1.0,2.0,5.0}
  - 初始条件：5 个正弦波叠加，$A_j\sim\mathcal{U}(-0.5,0.5)$, $l_j\in\{1,2,3\}$, $\phi_j\sim\mathcal{U}(0,2\pi)$
  - Δt=0.0603, L=128
- 2D：有限差分法，domain $[-1,1]^2$, 32 timesteps, Δt=0.02
  - 64×64 网格 → 降采样至 32×32
  - Heat: β∈[0.02,0.03]; Advection: c∈[2.5,3.0]²; Burgers: β∈[0.005,0.0075], a∈[1.0,1.25]²
  - 系数范围更窄更连续 → 系统间更相似，预训练任务更难

**数值更新方案（用于 $F(\cdot)$）：**
- Heat: 空间二阶中心差分 + 时间一阶后向差分
- Burgers': 扩散项用二阶中心差分 + 非线性对流项用二阶迎风格式 + 一阶后向时间差分
- Advection: 二阶迎风空间差分 + 一阶后向时间差分
- 注意：数值更新精度低于数据生成精度（数据生成用 WENO5/解析解，更新用低阶有限差分）

**评估指标：** 相对 $L^2$ 范数；5 个随机种子，报告均值±标准差

### 核心结果

**1D Fixed-Future（初始条件→t=49帧）：**
- FNO Baseline: Heat 4.82±0.14, Burgers 6.75±0.08, Advection 23.20±0.60, Combined 15.09±0.48 (×10⁻²)
- PICL Pretrained: **Heat 1.45±0.13, Burgers 4.27±0.31, Advection 4.05±1.16, Combined 3.09±0.23** (×10⁻²)
- 提升幅度：Heat 70%, Burgers 37%, Advection 82%, Combined 79%

**1D Autoregressive Rollout：**
- 累积误差减少 2 个数量级
- 即使增加 fine-tune 数据量，PICL 优势仍保持
- Combined dataset 上随样本数增加差距扩大
- 注意：baseline FNO rollout 不稳定，尽管 next-step 预测精度与 PICL 相当

**2D Fixed-Future（初始条件→最终状态）：**
- Heat: FNO 2.21→**1.85** (16% 提升), Burgers: 4.33→**4.13** (5%), Advection: 72.32→72.38（无提升）

**2D Autoregressive Rollout：**
- Heat: 0.388→**0.378**, Burgers: 1.078→**1.032**, Advection: 6.736→**6.688**
- 提升约 2.6%-4.3%，幅度有限但一致

**T-SNE 分析（关键发现）：**
- PICL 预训练后的 latent embeddings 展示出物理直觉一致的聚类：
  - Region A：弱扩散 Heat 系统聚集
  - Region B：弱扩散和弱对流的 Burgers/Heat 混合
  - Region C：中等对流 Burgers 系统
  - Region D：中等扩散系统
- 从左到右系数由小到大
- Advection 系统与 Heat/Burgers 系统明显分离
- 同一系数值的系统高度聚集（γ=5 聚集率 100%, γ=2 聚集率 99.5%, α=5 聚集率 99.3%, β=5 聚集率 87.1%）

**Passthrough 消融（无物理信息的对比预训练）：**
- 预训练后 fine-tune 完全无法学习（模型崩塌）
- t-SNE 显示按系数组合的整齐聚类，但不符合物理直觉——扩散主导 Burgers 与 Heat 没有表现出更相似的行为

**超分辨率（零样本）：**
- PICL 在不同分辨率下误差增加幅度与 baseline 相当，但绝对值更低（意味着始终优于 baseline）

### 明确局限

1. **必须已知控制方程**：PICL 仅适用于方程信息完全已知的场景，无法处理未知物理或数据驱动发现的场景
2. **额外预训练计算开销**：两阶段训练增加了计算成本
3. **不支持 forcing terms**：当前版本未纳入静态或时变 forcing terms
4. **仅测试同质 PDE**：所有方程都是齐次的，周期性边界条件
5. **仅测试 FNO backbone**：未在其他神经算子架构（DeepONet, Transformer, UNet 等）上评估
6. **2D 提升有限**：2D 实验中系数范围窄、系统间相似度高，预训练任务难度大，提升幅度小（尤其 Advection 几乎无提升）
7. **无真机/真实数据验证**：全是合成数据，没有真实物理系统实验
8. **无标准 benchmark 对比**：未在 PDEBench 等标准 benchmark 上评测
9. **CFL 稳定性问题**：1D 实验中上风方案的 CFL 数=1.17，超过稳定性约束 1，但声称 PICL 对此具有鲁棒性——这更像是巧合而非论证过的性质
10. **τ 超参数敏感**：τ 选择需手动调节，建议从"未训练模型的 $d_{physics}$ 同量级"开始尝试，缺乏自动化方法
11. **作者自己承认**：更高维系统（3D）中距离函数幅度变化更大，学习差异更具挑战性

### 跟我们的关系

对 0616-fex 课题的关键分析：

**是否撞车：**
- 与 0616-fex（PROSE 分支的 feature extraction / multi-operator learning）**不直接撞车**，但高度相关
- PICL 和 PROSE 都试图解决多算子学习问题，但方法完全不同：
  - PROSE：用多模态 transformer 同时预测多个 operator 和符号表达式
  - PICL：用对比学习预训练让单个 FNO 更好地区分和泛化多个系统
- PICL 的 contrastive pretraining 思想可以作为 PROSE 的**补充增强策略**——在 PROSE 训练前加入对比预训练阶段

**可用的工具/方法：**
1. **幅度感知余弦相似度**：可直接用于任何需要度量 PDE 系统间相似度的场景
2. **物理锚定距离函数** ($d_{physics}$)：将数值更新与模型输出锚定的技巧，可用于训练正则化
3. **GCL loss 框架**：对比学习在 PDE 预训练中的应用模式可供参考
4. **Two-stage pretrain+finetune 范式**：与 PROSE 的多任务学习可以结合
5. **Passthrough 消融教训**：没有物理信息的对比预训练会导致 fine-tune 失败——物理约束是关键
6. **τ 选择经验**：以未训练模型 batch 平均距离为起点

**差异化方向：**
- 0616-fex 可以探索：将 PICL 的对比预训练 + PROSE 的多模态符号推理结合，形成 "contrastive PROSE"
- PICL 只做数值预测，不涉及符号推理——这是 0616-fex 的核心优势
- 0616-fex 的 feature extraction 可以从 PICL 的 t-SNE embedding 分析中学到评估中间表示质量的方法

## Read by: 0616-fex (2026-06-24)

PICL 为 0616-fex（PROSE 分支的 multi-operator / feature extraction 方向）提供了重要的对比预训练视角。

**撞车评估：不撞车。** PICL 做的是纯数值预测（surrogate modeling），不涉及符号表达式推理或 operator discovery。PROSE/PICL 两者的目标函数和输出空间完全不同——PICL 输出的是数值解，PROSE 输出的是 operator + 符号表达式。两者是互补关系而非竞争关系。

**可用组件：**
1. 幅度感知余弦相似度可嵌入到 PROSE 的 multi-task 训练中作为辅助损失
2. $d_{physics}$ 锚定机制可以用于确保 PROSE 的数值 head 输出与物理规律一致
3. Two-stage pretrain+finetune 是对 PROSE 从零训练的有益补充——先用对比预训练让 backbone 学会区分不同系统，再进行多模态微调

**关键教训：**
- passthrough 消融证明：纯数据驱动的对比预训练（无物理信息）会导致下游 fine-tune 崩塌。这意味着任何应用于 PROSE 的对比预训练方案必须嵌入物理约束
- 2D 实验中系统太相似（窄系数范围）导致预训练困难——在设计 0616-fex 实验时应确保跨方程数据有足够的"区分度"
- τ 的选择高度依赖数据规模——在 batch 较大时用 mean 效果更好

**建议的下一步方向：**
- 探索 "Contrastive PROSE"：用 PICL 式对比预训练初始化 PROSE encoder，然后多模态微调同时学习数值预测和符号表达式
- 在 PROSE 的多模态 embedding 空间中也做 t-SNE 分析，验证是否获得类似 PICL 的物理直觉聚类
