# Correlated Proxies: A New Definition and Improved Mitigation for Reward Hacking (2403.03185)

- **Year**: 2024.03
- **Authors**: Cassidy Laidlaw, Shivam Singhal, Anca Dragan, UC Berkeley
- **Venue**: ICLR 2025 / arXiv
- **Keywords**: reward hacking, correlated proxy reward, occupancy measure regularization, chi-squared divergence, RLHF, PPO, ORPO, reward misspecification
- **One-liner**: 用 reference-policy 下的代理-真奖励相关性定义 reward hacking, 并用 chi2 occupancy regularization 缓解。

## 精华提炼

### 管线/方法

1. 论文先指出 reward hacking 缺少一个能覆盖现实案例的形式化定义。
2. 核心观察是代理奖励之所以可用, 通常只是在某个参考行为分布上看起来合理。
3. 他们把这个参考行为分布形式化为 reference policy `pi_ref` 的 state-action occupancy measure。
4. 给定 MDP、未知 true reward `R`、可优化 proxy reward `R_tilde` 和 reference policy `pi_ref`。
5. 代理奖励定义为 `r`-correlated proxy: 在 `mu_pi_ref(s,a)` 分布下, 标准化后的 proxy reward 与 true reward 的相关系数为 `r > 0`。
6. reward hacking 定义为: 优化 proxy 后得到的 policy `pi` 在 true reward 上低于 reference policy, 即 `J(pi,R) < J(pi_ref,R)`。
7. 如果 proxy 的最优 policy 中存在这种真奖励下降, proxy 就是 hackable correlated proxy。
8. 这个定义同时解决两个问题: 什么代理奖励算合理, 以及 reward hacking 的失败阈值在哪里。
9. 失败阈值不是“true reward 任意下降”, 而是低于 reference policy 的 true return。
10. 论文随后证明一个 true-reward improvement lower bound。
11. 对任何满足 `mu_pi << mu_pi_ref` 的 policy, true reward improvement 可由 proxy improvement 减去 occupancy divergence 项下界。
12. 下界中的 regularizer 是 `sqrt((1-r^2) chi2(mu_pi || mu_pi_ref))`。
13. 由此得到优化目标: maximize `J(pi, R_tilde) - lambda sqrt(chi2(mu_pi || mu_pi_ref))`。
14. 理论建议 `lambda = sigma_proxy sqrt(1-r^2)`。
15. 直觉是 proxy 与 true reward 的相关性越弱, policy 越需要贴近 reference occupancy。
16. 论文强调这个 lower bound 不保证任意 MDP 都能提升 true reward。
17. 但如果正则化目标能被优化到正的 lower bound, 就能证明 true reward 至少超过 reference。
18. 即使不能提升, 优化这个 lower bound 也能提供避免 reward hacking 的保守路径。
19. 实际算法命名为 Occupancy-Regularized Policy Optimization, 简称 ORPO。
20. ORPO 用 discriminator 估计当前 policy 与 reference policy 的 occupancy log-ratio。
21. discriminator loss 是二分类式 logistic loss, 区分 `mu_pi` 样本和 `mu_pi_ref` 样本。
22. 最优 discriminator 输出近似 `d_hat(s,a)=log(mu_pi(s,a)/mu_ref(s,a))`。
23. chi2 divergence 估计为 `E_{mu_pi}[exp(d_hat(s,a))-1]`。
24. policy reward 被替换为 `R'(s,a)=R_tilde(s,a)-lambda/sqrt(chi2_hat) * exp(d_hat(s,a))`。
25. 训练流程是交替采样当前 policy 轨迹、采样 reference policy 轨迹、训练 discriminator、再用 PPO 更新 policy。
26. 论文也给了 OM-KL 版本: augmented reward 中用 `-lambda d_hat(s,a)`。
27. 对 RLHF, 当前单轮 prompt-response 形式可视为 contextual bandit。
28. 在 contextual bandit 中 action-distribution divergence 与 occupancy-measure divergence 等价。
29. 但论文指出多轮对话、tool use、agentic LLM 会破坏 contextual bandit 假设。
30. 因此 OM regularization 是面向未来 multi-turn / tool-use RLHF 的更一般形式。

### 关键技术细节

31. 理论设置是 infinite-horizon discounted MDP, 有有限 `S` 和 `A`, discount `gamma in [0,1)`。
32. return 定义为 normalized discounted return: `(1-gamma) E[sum gamma^t R(s_t,a_t)]`。
33. occupancy measure 同样带 `(1-gamma)` 归一化, 总质量为 1。
34. correlated proxy 的方差 `sigma_proxy^2` 和 `sigma_true^2` 都在 `mu_pi_ref` 下计算。
35. Theorem 1 使用 Cauchy-Schwarz 将 true/proxy reward 的相关性与 occupancy chi2 divergence 联系起来。
36. 论文证明了 near-optimal reference policy 时的 suboptimality corollary。
37. 论文也证明 lower bound 不总是 optimizable: 存在 MDP 里 proxy 和 true reward 都可提升, 但 lower bound 仍不能大于 0。
38. 反过来也证明存在 MDP 对任意 `r` 都能让 lower bound 为正, 且优化 lower bound 得到 true-reward optimal policy。
39. 对 action-distribution regularization, 论文证明几乎任意 f-divergence AD regularizer 都不能给 true reward improvement guarantee。
40. 关键反例是小 action 分布变化会改变长时序 state occupancy, 导致 AD 距离小但 true reward 下降。
41. 非 LLM 实验用 RLLib 和 PyTorch 实现。
42. traffic / pandemic / tomato policy 网络分别是全连接 2x128、4x512、4x512。
43. glucose policy 用 3 层 64 hidden units 的 LSTM, 因为观测包含血糖历史和 insulin 历史。
44. discriminator 对所有非 LLM 环境都用 2 层 256 hidden units 的全连接网络。
45. PPO/ORPO 训练迭代数: tomato 500, traffic 250, glucose 500, pandemic 260。
46. batch size: tomato 3000, traffic 40000, glucose 100000, pandemic 3860。
47. discount 都是 0.99, optimizer 都是 Adam。
48. ORPO 中 discriminator reward clipping 分别为 tomato 1000, traffic 10, glucose 1e10, pandemic 0.1。
49. `sigma_proxy` 分别为 tomato 0.05, traffic 0.0002, glucose 0.05, pandemic 0.08。
50. chi2 估计用 trimmed mean, 每侧 trim 1%, 用来降低 exponentiated discriminator outlier 的影响。
51. RLHF 实验沿用 Coste et al. 的实现, 基于 OpenAssistant 和 trlX。
52. RLHF proxy reward model 是 Pythia-70M 在 AlpacaFarm preference data 上 fine-tuned。
53. RLHF true reward model 是 AI2 的 Llama 3 Tulu V2 8B RM。
54. RLHF reference policy 是 Coste et al. 从 Pythia-1.4B 在 AlpacaFarm SFT data 上 fine-tuned 的 SFT policy。
55. RLHF 中直接在 PPO loss 加 AD chi2 或 AD KL loss term, 因为 contextual bandit 下 AD 和 OM 等价。

### 核心结果

56. 论文在 5 个环境上验证定义: traffic control, pandemic mitigation, glucose monitoring, RLHF, tomato-watering gridworld。
57. Figure 2 的 top row 显示 reference policy 下 proxy 与 true reward 都正相关。
58. Figure 2 的 middle row 显示直接优化 proxy 后相关性崩坏, true reward 低于 reference。
59. Figure 2 的 bottom row 显示 OM regularization 能在多数案例中提升 true reward。
60. Table 1 是主要结果, 报告 5 seeds 的 median true reward 和 std。
61. Traffic: reference 为 -2.28e3, 无正则为 -57.38e3, true reward 训练上限 -0.93e3。
62. Traffic 最佳是 state-action OM chi2, -1.15e3, 超过 reference 且接近 true-reward upper bound。
63. Pandemic: reference -12.26, 无正则 -29.57, true reward upper bound -2.65。
64. Pandemic 最佳是 state OM KL -10.24, state OM chi2 -10.68, 都超过 reference。
65. Glucose: reference -72.6e3, 无正则 -599.0e3, true reward upper bound -43.4e3。
66. Glucose 最佳是 state-action OM chi2, -47.6e3, 明显优于 AD regularization。
67. RLHF: reference true RM score 16.37, 无正则 9.16。
68. RLHF AD chi2 达到 16.94, AD KL 为 16.81, chi2 更高且 std 更小。
69. Tomato gridworld: reference 5.86, 无正则 2.35。
70. Tomato 最佳是 state-action OM chi2 9.17, 高于 true reward 训练 baseline 8.54。
71. AD chi2 在 pandemic 和 glucose 不能可靠超过 reference, 说明 AD 约束对长时序 occupancy drift 不够。
72. chi2 通常对 regularization coefficient 更稳健, 尤其 RLHF win-rate ablation 中 chi2 median win rate 52.83, KL 为 51.50。
73. KL win-rate range 为 11.93-53.75, chi2 range 为 51.98-53.98, 说明 KL 在某个 seed 中严重 reward hacking。
74. ORPO ablation 显示先训 discriminator 再训 policy 略好, pandemic 中反过来训练会显著增加方差。
75. removing discriminator clipping 会 NaN, 论文没有报告无 clipping 结果。

### 明确局限

76. 最大理论局限是 true reward improvement lower bound 不总能被优化到正数。
77. 论文自己给了 counterexample, 说明 correlated proxy + OM regularization 不是万能充分条件。
78. 理论要求 `mu_pi << mu_pi_ref`, 即 learned policy 不能访问 reference occupancy 零概率的 state-action。
79. 实际深度 RL 中这个绝对连续性条件只能近似满足, 依赖采样覆盖和 regularization。
80. reference policy 的质量至关重要: 如果 reference policy 很差, lower bound 只保证相对它的改善或不劣。
81. true reward 在 RLHF 中由另一个 reward model 代理, 不是人类真实偏好, 因此 RLHF 结果仍是 surrogate evaluation。
82. 非 LLM 环境多为 simulator 或 benchmark, 没有真机自动驾驶、真实医疗闭环、真实公共卫生部署。
83. ORPO 需要 reference policy 采样, 对离线日志或不可交互系统可能成本较高。
84. discriminator 估计 occupancy ratio 在高维长时序环境中可能不稳, 论文靠 clipping 和 trimmed mean 稳定。
85. chi2 对 outlier 惩罚强, 理论上是优点, 工程上也带来 exponentiation blow-up 风险。
86. RLHF 当前只实验了 contextual bandit setting, 没有真正 multi-turn/tool-use agent 的 OM regularization 实验。
87. 论文没有证明 discriminator 近似误差下的 finite-sample safety guarantee。
88. 结果表中不同环境最佳 regularizer 不完全一致, pandemic 的最佳主表是 state OM KL 而不是 state-action OM chi2。
89. 近期 arXiv 已有直接 follow-up 用 robust optimization 扩展 correlated proxies, 说明“correlated proxy + ORPO”已不是空白点。

## Read by: 0616-fex (2026-06-17)

这篇对 0616-fex 的价值在于它把“代理奖励为什么一开始合理、为什么优化后崩坏”拆成了一个可引用的形式化框架: reference policy occupancy 下 proxy 与 true reward 正相关, 但优化把 policy 推到 reference distribution 外, 相关性失效。若 0616-fex 的核心是 feedback exploitation/reward hacking/f-divergence RLHF, 则本文是高撞车 prior: correlated proxy 定义、chi2 occupancy regularization、ORPO discriminator 估计、AD regularization failure theorem 都已覆盖。可差异化的位置包括真实人类反馈而非 surrogate RM、multi-turn/tool-use agent 的非 bandit occupancy drift、inference-time hacking、以及检测/预警而非 policy regularization。近期 arXiv 检索还显示 2604.12086 已把 correlated proxies 扩展成 worst-case robust optimization, 所以若要主张“对所有 r-correlated proxy 做鲁棒优化”也需要避开该后续工作。本文应作为必引 baseline, 但不能把“用 chi2/OM 替代 KL 防 reward hacking”包装成新贡献。
