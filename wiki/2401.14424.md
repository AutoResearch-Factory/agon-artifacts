# Discovering Mathematical Formulas from Data via GPT-guided Monte Carlo Tree Search (2401.14424)

- **Year**: 2024.01
- **Authors**: Yanjie Li, Weijun Li, Lina Yu, Min Wu, Jingyi Liu, Wenqiang Li, Meilan Hao, Shu Wei, YuSong Deng — ANNLAB, Institute of Semiconductors, Chinese Academy of Sciences
- **Venue**: ICML 2024
- **Keywords**: symbolic regression, Monte Carlo Tree Search, GPT, self-play, expression recovery, policy-value network, S_NRMSE, information entropy loss
- **One-liner**: 将 GPT 作为 policy+value 网络引导 MCTS 搜索数学表达式，通过类 AlphaZero 自对弈循环互相优化，在 222 个 benchmark 表达式上达到 74.9% 恢复率 SOTA。

## 精华提炼

### 管线/方法

SR-GPT 的完整 pipeline 如下：

1. **输入**：数据点 $[x_1,...,x_m, y]$，符号库 $S$（含 5 个二元算子 $+,-,\times,\div,\bullet^\bullet$，5 个一元算子 $\sin,\cos,\exp,\sqrt{},\ln$，变量 $x_1,...,x_n$，常数占位符 $c$）。

2. **表达式表示**：表达式以二叉树表示，用 preorder traversal 展开成离散符号序列 $[a_1, a_2, a_3, ...]$。用 counter（初始=1）和 arity 追踪是否完整：$counter = counter + arity(a) - 1$，counter=0 时表示完整表达式。

3. **GPT 网络**：单一 GPT 模型 $\mathcal{N}_\theta$ 同时担任 policy 和 value 网络。输入为当前节点的 parent 和 sibling 节点，输出 $(p, v) = \mathcal{N}_\theta(s)$，其中 $p$ 是各符号的选择概率，$v$ 是当前状态的价值估计。

4. **MCTS 模拟阶段**：在选择每个新符号前，执行多次 MCTS 模拟：
   - **Select**：从根节点递归选择 UCT 值最大的子节点，$UCT(s,a) = Q(s,a) + c_{puct} \cdot P(s,a) \cdot \frac{\sqrt{N(s)}}{1+N(s,a)}$
   - **Expand**：遇到非终止叶节点时，创建所有子节点，用 GPT 生成的先验概率 $P(s',\bullet) = \mathcal{N}_\theta(s')$ 选择最高概率的节点展开
   - **Evaluate**：用 GPT 计算展开节点的 value $V(s')$
   - **Backpropagate**：更新路径上所有节点的 $N_{visit}$ 和 $Q(s,a) = \frac{1}{N(s,a)} \sum V(s')$

5. **Self-Search 阶段**：模拟完成后，通过归一化访问次数得到搜索概率 $\pi_a \propto N(s,a)^{1/\tau}$，贪心选择 $\arg\max(\pi)$ 作为下一个符号。生成完整表达式后计算 reward $z = 1/(1+S_{NRMSE})$ 并进行整条路径的反向传播。

6. **GPT 训练**：从 self-search 中收集训练数据 $(s_t, \pi_t, z)$，存入大小为 1000 的 FIFO 队列，随机采样 batch 训练 GPT，使 $p$ 逼近 $\pi$，$v$ 逼近 $z$。

7. **常数优化**：表达式中出现常数占位符 "C" 时，用 BFGS/L-BFGS-B 进行非线性常数优化。

8. **输出**：满足终止条件（reward 达到阈值或找到目标表达式）时返回最优表达式。

### 关键技术细节

- **Loss 函数**：$\mathcal{L} = (z-v)^2 - \pi^T\log p - p^T\log p + \xi||\theta||^2$
  - 第一项：value 预测误差
  - 第二项：policy 与搜索概率的交叉熵
  - 第三项：**信息熵 loss**（本文创新），鼓励 GPT 产生低熵概率分布，使预测更"自信"。消融实验显示加入此项后平均信息熵从 2.64 降到 1.78，搜索时间从 226s 降到 133s。
  - 第四项：L2 正则化

- **S_NRMSE Loss**（本文创新）：$S_{NRMSE} = NRMSE(y,\hat{y}) + \lambda \sum_j NRMSE(x_j, \hat{x}_j)$
  - 除了传统的 y 匹配误差，还加入了变量存在性惩罚
  - 如果预测表达式包含变量 $x_j$，则 $\hat{x}_j = x_j$，loss 为 0
  - 如果缺失变量 $x_j$，则 $\hat{x}_j = 0$，产生大 loss
  - 有效解决多变量回归中的变量遗漏问题

- **搜索空间约束**：
  1. 限制表达式最大长度
  2. 禁止互逆函数连续出现（如 $\exp(\log(x))$）
  3. 禁止三角函数重复嵌套（如 $\sin(\cos(x))$）
  4. 禁止非法表达式（如 $\log$ 后接可能产生负值的函数）

- **GPT 架构**：标准 GPT transformer，输入为 parent+sibling 节点拼接
- **训练数据管理**：大小 1000 的 FIFO 队列，每次训练随机采样 batch
- **评测指标**：精确符号等价恢复率（用 SymPy 验证），每个表达式测试 100 次
- **数据采样**：仅用 20 个点，$x \in [-1,1]$ 均匀采样
- **硬件**：Intel Xeon Gold 5218R CPU (2.10GHz, 20 cores)，无 GPU

### 核心结果

| 数据集 | SR-GPT | DGSR-MCTS | SPL | TPSR | NeSymReS |
|--------|--------|-----------|-----|------|----------|
| Nguyen (21) | **96%** | 93% | 91% | 91% | 56% |
| Keijzer (15) | **80%** | 76% | 67% | 77% | 52% |
| Korns (15) | **77%** | 71% | 69% | 66% | 27% |
| AI Feynman (103) | **65%** | 57% | 44% | 49% | 18% |
| 平均 (222 expr) | **74.9%** | 68.5% | 63.1% | 65.1% | 25.1% |

- **R² 平均 0.9917**：即使无法精确恢复公式，也能找到拟合良好的等价表达式
- **抗噪声**：在所有噪声水平 (0-10%) 上优于除 DSO 在 1-2% 处外的所有基线
- **泛化性**：在 [-1,1] 到 [-10,10] 各区间上表现稳定，优于依赖预训练的 DGSR-MCTS、TPSR、NeSymReS（它们在训练范围 [-5,5] 外急剧下降）
- **效率**：Nguyen 数据集平均 147.5s，略慢于 DGSR-MCTS (122.5s) 和 TPSR (125.9s)，但远快于 SPL (237.5s)
- **消融实验**：去掉信息熵 loss、约束、S_NRMSE 各自都导致显著性能下降，但无灾难性退化

### 明确局限

**作者声明的局限**：
1. 变量数过多时搜索空间大，效率低，有时无法找到目标表达式
2. 未来计划用大量数据预训练 policy network 提升性能
3. 未利用物理单位约束来缩小搜索空间

**我读出的局限**：
1. **仅 CPU 运行**：GPT 模型在 CPU 上跑，规模受限，无法利用大规模 transformer
2. **GPT 输入设计简陋**：仅用 parent+sibling 两个节点作为输入，没有利用完整的部分表达式序列
3. **与 AlphaZero 的关系未充分讨论**：方法本质上是 AlphaZero 范式在符号回归的直接应用，但论文未深入讨论与 AlphaGo Zero 的异同
4. **仅 20 个数据点**：测试条件非常有限，实际应用中数据量和噪声分布更复杂
5. **无与最新 LLM 的比较**：GPT 此处是小型 transformer，非大语言模型，名称有误导性
6. **ICML 2024 论文但对比基线偏旧**：缺少与 2023-2024 年更新的 SR 方法的对比

### 跟我们的关系

对 **llm-mcts-sr** 课题而言：

1. **直接相关**：本文是 "LLM/transformer + MCTS + symbolic regression" 这条路线的代表性工作，发表于 ICML 2024。任何做 LLM-guided MCTS for SR 的工作都必须引用和对比。

2. **核心方法模式**：类 AlphaZero 的 policy-value 网络 + MCTS 自对弈循环。这个范式直接可用——问题是 GPT 模型很小（仅用 parent+sibling 作输入），有大量改进空间（用真正的 LLM、用完整序列输入、用预训练模型初始化等）。

3. **可复用的工具**：
   - S_NRMSE loss 对多变量问题的变量遗漏问题是个有效解决方案
   - 搜索空间约束（反函数、三角嵌套、非法表达式）是标准工程
   - 信息熵 loss 提升搜索效率的思路值得借鉴

4. **差异化方向**：
   - 本文的 "GPT" 实际上是小型 transformer，不是大语言模型——用真正的 LLM 做 SR 是可行差异化
   - 可以引入更丰富的上下文（完整部分表达式、数据特征、物理约束）而非仅 parent+sibling
   - 可以结合预训练 + 在线微调，兼顾泛化性和搜索效率

## Read by: llm-mcts-sr (2026-06-05)

本文是 LLM-MCTS-SR 课题的核心相关论文之一。SR-GPT 提出了类 AlphaZero 范式的 GPT+MCTS 自对弈框架用于符号回归，在 ICML 2024 发表。

**撞车评估**：如果我们的课题也是做 "LLM 引导 MCTS 做符号回归"，则与本文高度撞车。差异化需要在以下方面发力：(1) 使用真正的大语言模型而非小型 transformer；(2) 更丰富的状态表示（完整部分表达式序列、数据统计特征、物理约束）；(3) 预训练+在线微调的混合策略。

**可用工具/方法**：S_NRMSE loss、信息熵正则化、搜索空间约束、counter-arity 表达式完整性检查，这些都是直接可复用的工程组件。AlphaZero 式 self-play 循环是核心算法范式。

**差异化方向**：本文的 "GPT" 名不副实——实际是小 transformer，输入仅 parent+sibling。真正用预训练 LLM 的语义理解能力（如理解物理含义、数据模式识别、多步推理）来引导 MCTS 搜索，是明确的差异化方向。同组后续的 ChatSR 和 MMSR 已经朝这个方向迈进。
