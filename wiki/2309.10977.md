# PAGER: A Framework for Failure Analysis of Deep Regression Models (2309.10977)

- **Year**: 2023.09
- **Authors**: J. Thiagarajan (Lawrence Livermore National Labs), V. Narayanaswamy (Lawrence Livermore National Labs), Puja Trivedi (University of Michigan), Rushil Anirudh (Amazon)
- **Venue**: ICML 2024 / arXiv
- **Keywords**: failure detection, deep regression, anchored training, epistemic uncertainty, manifold non-conformity, OOS/OOD risk, calibration-free risk regimes
- **One-liner**: 用 anchoring 同时估计不确定性和流形不一致性，给深度回归样本分风险档。

## 精华提炼

### 管线/方法

1. 论文目标不是给每个回归样本精确预测误差，而是把测试样本组织成 ID、Low Risk、Moderate Risk、High Risk 四类风险 regime。
2. 输入包括训练好的/待训练的深度回归模型、带标签训练集 `D={(x_i,y_i)}`、无标签测试样本，以及可选的校准集。
3. 核心前提是：回归 failure 的定义依应用而变，所以直接估计 sample-level true risk 很难；按风险档排序更可操作。
4. 作者区分三种测试区间：ID 表示训练支持内样本；OOS 表示仍在同一输入空间但训练支持外；OOD 表示输入空间本身和训练分布不相交。
5. PAGER 的第一条信号是 forward anchoring 得到的 epistemic uncertainty。
6. Anchored training 把普通输入 `x` 改写成 `[r, x-r]`，其中 anchor `r` 从训练集随机抽取。
7. 模型训练目标仍是预测原来的 `y`，损失函数使用 MAE 或 MSE；论文实验主文说明全部模型使用 `L1` objective。
8. 推理时对一个测试样本 `x_t` 采样 `K` 个训练 anchor，计算 `F([r_k,x_t-r_k])` 的均值作为预测。
9. 同一批 anchored predictions 的标准差作为 uncertainty：`sigma = sqrt(1/(K-1) sum_k (F([r_k,x_t-r_k])-mu)^2)`。
10. PAGER 的第二条信号是 manifold non-conformity, MNC，用来补足 uncertainty 对 OOS/OOD 风险排序不稳定的问题。
11. `Score_1` 使用 reverse anchoring：把测试样本 `x_t` 放到 anchor 位置，把训练样本 `r` 放到 query residual 里，调用 `F([x_t,r-x_t])`。
12. 因为训练样本 `r` 的真实标签 `y_r` 已知，可以无测试标签地计算 `||y_r - F([x_t,r-x_t])||_1`。
13. `Score_1(x)=max_{r in D} ||y_r - F([x,r-x])||_1`，实践中作者说可用约 100 个随机训练样本近似。
14. `Score_1` 的直觉是：如果一个测试点不能作为“参照 anchor”让模型恢复训练样本标签，它很可能不符合训练任务流形。
15. `Score_2` 是更重的 MNC：优化一个虚拟样本 `x_bar`，让 `F([x_bar,r-x_bar])` 能匹配训练样本标签，同时保持在输入流形上。
16. `Score_2` 的优化目标包含 `||y_r - F([x_bar,r-x_bar])||_1 + lambda R(x_bar)`。
17. 正则项 `R(x_bar)` 由 anchored autoencoder `A` 给出，包含双向 cyclical reconstruction：`A([x,x_bar-x])` 和 `A([x_bar,x-x_bar])` 都要一致。
18. `Score_2` 最终用 `||x_t - x_bar||_2` 衡量需要把 query 移动多远才能落回可解释的 manifold。
19. 作者给出三个算法：predictive uncertainty estimation、`Score_1` computation、`Score_2` computation。
20. `Score_2` 算法输入包括 anchored model、anchored autoencoder、训练子集、学习率 `eta`、权重 `lambda`、迭代数 `T`。
21. 风险分箱时，PAGER 分别把 uncertainty 和 MNC score 按 conditional quantile 切为 low `[0,25]`、moderate `[25,75]`、high `[75,100]`。
22. ID 定义为 low uncertainty 且 low/moderate MNC。
23. Low Risk 定义为 low uncertainty + high MNC，或 moderate uncertainty + low/moderate MNC。
24. Moderate Risk 定义为 moderate uncertainty + high MNC，或 high uncertainty + low/moderate MNC。
25. High Risk 定义为 high uncertainty + high MNC。
26. 这个规则显式表达论文的中心 claim：uncertainty 必要但不充分，必须和 task-aware non-conformity 联合使用。
27. 作者还提出四个 failure detector 指标：FN、FP、`C_low`、`C_high`，都越低越好。
28. FN 衡量 ID/Low Risk 档里混入 true risk top 20% 样本的比例，是高代价场景最关键指标。
29. FP 衡量 Moderate/High Risk 档里混入 true risk bottom 20% 样本的比例。
30. `C_low` 比较 ID 的 90th percentile error 与 Low Risk 的 10th percentile error，用来衡量低风险相邻档混淆。
31. `C_high` 类似，但衡量 Moderate Risk 与 High Risk 的混淆。

### 关键技术细节

32. 1D benchmark 包含四个函数：分段二次函数 `f1`、`sin(2*pi*x)`、带指数和余弦项的 `f3`、以及 `sin(x)cos(5x)cos(22x)`。
33. 每个 1D 函数使用 200 个 uniform-grid 测试样本来计算指标。
34. 高维 regression benchmark 覆盖 2D 到 32,000D：Camel、Levy、Airfoil、NO2、Kinematics、Puma、Boston Housing、Ailerons、DTI。
35. 高维实验构造两类目标空间 shift：Gaps 只让训练暴露 target 的 0-30th 与 60-100th percentile；Tails 只暴露 0-70th percentile。
36. 额外真实 shift 使用 Skillcraft 数据集，shift 来自 league index 的变化。
37. Image regression 包含 chair yaw angle、cell count、CIFAR-10 rotation prediction。
38. CIFAR-10 任务随机旋转 32x32x3 图像，角度范围 0-90 度；评估使用 10K randomly rotated images。
39. 主文写 tabular benchmark 使用 4 层 MLP，每层 hidden dimension 128。
40. Appendix 对其他 regression benchmark 写的是标准 MLP，5 个 hidden layers，ReLU activation 和 batchnorm，训练 5000 epochs。
41. Appendix 给出其他 regression benchmark 的 optimizer 为 ADAM，learning rate `5e-5`。
42. Cell Count 和 Chair Angle 使用 anchored WideResNet 40-2，batch size 128，训练 100 epochs。
43. WideResNet 40-2 的 ADAM momentum parameters 为 `(0.9,0.999)`，固定 learning rate `1e-4`。
44. Anchored autoencoder 使用 convolutional encoder-decoder：encoder 两个 3x3 conv layer 加 max pooling，decoder 两个 stride-2 transposed conv layer。
45. Anchored autoencoder batch size 128，训练 100 epochs，ADAM `(0.9,0.999)`，learning rate `1e-3`。
46. CIFAR-10 regression backbone 是 ResNet-34，论文说使用 standard training configurations。
47. 对图像输入，anchored tuple 在 channel 维拼接；3-channel 图像变成 6-channel tensor。
48. 对 vector-valued data，anchored tuple 通过 concat anchor 与 residual 扩展输入维度。
49. Baseline 包括 DEUP 和 DataSUITE；DEUP 训练 post-hoc risk/error predictor，DataSUITE 用 task-agnostic data inconsistency。
50. 作者强调两个 baseline 都依赖额外 curated calibration data，而 PAGER 主设定不需要 calibration data。

### 核心结果

51. 1D benchmark 上，PAGER 在多数函数和指标上优于 DEUP/DataSUITE；`Score_2` 常在 FN、FP 或 high-risk confusion 上最好。
52. 1D 推理 runtime 用 1000 个测试样本、单 GPU 统计：DataSUITE 29.8s，DEUP 18.2s，PAGER `Score_1` 1.55s，PAGER `Score_2` 40.9s。
53. 这个 runtime 结果说明 `Score_1` 是在线可用版本，`Score_2` 是高成本但更细的版本。
54. 高维 Gaps/Tails 实验中，作者报告 PAGER 对 baseline 的 FN 和 FP 有超过 50% reduction，并显著降低 `C_low` 与 `C_high`。
55. 在 Gaps 表里，DTI 32,000D 的 FN 从 DEUP 16.51 / DataSUITE 29.18 降到 PAGER `Score_1` 9.26。
56. 在 Gaps 表里，Ailerons 的 FP 从 DEUP 15.96 / DataSUITE 8.03 降到 PAGER `Score_2` 1.33。
57. 在 Tails 表里，Camel 的 FN 从 DEUP 10.53 / DataSUITE 3.84 降到 PAGER `Score_1` 0.0。
58. 在 Tails 表里，Ailerons 的 FP 从 DEUP 12.01 / DataSUITE 12.0 降到 PAGER `Score_2` 0.38。
59. Image regression 的 `R^2` 说明 anchoring 不损害预测性能：CIFAR observed 0.93 vs standard 0.92，unobserved 0.81 vs 0.77。
60. Chairs yaw angle 的 `R^2`：observed standard 0.97、anchoring 0.97；unobserved standard 0.73、anchoring 0.75。
61. Cells count 的 `R^2`：observed standard 0.88、anchoring 0.89；unobserved standard 0.69、anchoring 0.72。
62. CIFAR ablation 显示单独 UQ 或单独 MNC 都不足：UQ-only FP/FN/`C_low`/`C_high` 为 12.54/13.45/14.75/9.15。
63. CIFAR ablation 中 MNC-only 为 13.08/13.24/12.91/11.38，而 PAGER 联合后为 5.05/10.38/6.58/2.09。
64. Test-time corruption 实验使用 defocus blur 与 frost；作者称 severity 增大时 `Score_2` 同时降低 FP 和 FN。
65. PAGER 可用于非 anchored backbone：冻结标准训练 feature extractor，只训练 anchored regression head，在 CIFAR rotation gaps setting 中仍接近 fully anchored PAGER。
66. Post-hoc calibration 可进一步改善结果：DTI 上 PAGER 从 9.09 FP / 6.67 FN / 19.94 `C_low` / 13.35 `C_high` 变为 4.27 / 3.88 / 17.55 / 10.76。
67. CIFAR-10 上 PAGER calibration 后从 7.86 FP / 3.34 FN / 3.28 `C_low` / 5.34 `C_high` 变为 4.81 / 2.59 / 1.70 / 4.48。
68. 最反直觉的结果是：低 uncertainty 的 OOS 样本仍可能有 moderate risk；而高 uncertainty 的 OOD/OOS 样本也不必然是最高 risk。
69. 论文最强证据不是某个单一 SOTA 数字，而是多个 synthetic/real shift 设置下同一模式反复出现：UQ 和 MNC 联合优于任一单项。

### 明确局限

71. 作者没有给出形式化 guarantee；PAGER 是经验风险分档框架，不是 conformal prediction 那类覆盖率保证方法。
72. Quantile bins `[0,25]`、`[25,75]`、`[75,100]` 是 demonstration choice，真实应用需要按风险容忍度重新设定。
73. `Score_1` 使用 max over training samples，实践中用随机子集近似；不同子集大小和抽样策略可能影响稳定性。
74. `Score_2` 计算开销显著高于 `Score_1`，1D 1000 样本上已达 40.9s，真实高维在线场景要谨慎。
75. `Score_2` 还需要训练 anchored autoencoder，并依赖优化 `x_bar` 的超参数 `eta`、`lambda`、`T`。
76. 高维和图像 benchmark 的 Gaps/Tails 多为人为构造 target shift，不能完全代表真实部署中的复杂 feedback shift。
77. Skillcraft 是真实 shift，但论文没有在医疗、科学仿真、RL 控制闭环中做端到端部署实验。
78. Baseline 主要是 DEUP 和 DataSUITE，未系统比较所有 contemporary OOD/regression uncertainty 方法。
79. 论文强调不需要 calibration data，但实验指标仍需要 held-out labels 计算 true risk top/bottom 20%，评估本身不是无标签。
80. 主文和 appendix 对 tabular MLP 层数写法有轻微不一致：主文为 4 layers hidden 128，appendix 写其他 regression benchmarks 为 5 hidden layers。
81. Reproducibility 受代码、数据处理脚本和具体 random anchors 影响；tex 源码没有说明 artifact 或独立复现。
82. PAGER 假设训练样本标签可信，reverse anchoring 对 label noise 或 target heterogeneity 的鲁棒性没有充分展开。
83. 该方法适合 regression；classification failure detection 只作为相关工作背景，不是本文实验范围。
84. 对非常大的训练集，`Score_1`/`Score_2` 都有 anchor subset selection 问题，论文只给出小批随机样本建议。
85. 论文没有讨论多输出 regression、heteroscedastic aleatoric uncertainty 与 PAGER score 的交互。

### 跟我们的关系

1. 对 `0616-fex`，如果课题关注 failure/extrapolation/OOS risk，这篇是直接相关 prior，需要引用。
2. 最容易撞车的 claim 是“单靠 epistemic uncertainty 不足以刻画 regression failure，必须加入 task-aware non-conformity”。
3. 如果我们的方案也使用 anchored training、reverse anchoring、或用训练标签恢复误差定义无标签 test non-conformity，那会和 PAGER 正面重叠。
4. 如果我们的方案只是研究 LLM/agent 的 execution failure、文本任务 failure、或分类/OOD 检测，则不算直接撞车，但 PAGER 的风险分档思想可迁移。
5. 可用的方法包括：用 paired score 而不是单一 uncertainty；用 quantile bins 生成风险档；用 FN/FP/相邻档 confusion 评估 detector。
6. 可用的差异化方向包括：给风险档提供 conformal/coverage guarantee；研究 anchor subset selection；在真实闭环系统做 deployment evidence。
7. 另一个差异化点是把生成者和检验者分离，例如一个模型生成预测，另一个独立模型或规则系统 veto high-risk cases。
8. 如果 `0616-fex` 的目标是做最新工作，必须补充 2025-2026 年的 regression failure / uncertainty / extrapolation 文献搜索，因为本文 S2 反引文为 0，不代表方向停滞。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 角度看，PAGER 是 regression failure / extrapolation risk 的强相关 prior。它的核心贡献不是新的 backbone，而是一个 failure characterization 框架：用 anchored model 的 forward pass 估计 epistemic uncertainty，再用 reverse anchoring 从训练标签构造无测试标签 MNC score，最后把两种 score 交叉分箱成风险档。若 `0616-fex` 的 claim 是“在 OOS/OOD 或 extrapolation 场景中，比单独 uncertainty 更可靠地识别回归失效样本”，这篇会构成直接撞车，尤其是 anchored/reverse-anchored non-conformity 这一路线。若我们的方向是 agent/tool execution failure、LLM 失败归因、或带独立 verifier 的 failure evidence，则主要是可引用的评估框架和 cautionary prior，不是方法撞车。

可借鉴的具体工具是四个指标 FN、FP、`C_low`、`C_high`，以及把 risk detector 评估从“单点误差估计”改成“风险档混淆评估”。可差异化的地方也清楚：PAGER 没有形式保证，缺少独立复现和真实部署验证，`Score_2` 成本高，且 anchor subset/quantile threshold 都是经验选择。`0616-fex` 若要避开重叠，应强调生成-检验分离、可审计 evidence、近期 2025-2026 setting、或在非回归/真实闭环任务里的 failure characterization。
