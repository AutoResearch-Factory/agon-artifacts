# WxC-Bench: A Novel Dataset for Weather and Climate Downstream Tasks (2412.02780)

- **Year**: 2024.12
- **Authors**: Rajat Shinde, Christopher E. Phillips, Kumar Ankur, Aman Gupta, Simon Pfreundschuh, Sujit Roy 等（六位共同一作）; UAH Earth System Science Center + Stanford + NASA Marshall + Colorado State（NASA IMPACT 团队）
- **Venue**: arXiv preprint（wlscirep 模板，Scientific Data 投稿体例）
- **Keywords**: ML-ready dataset, weather-climate foundation model, downstream tasks, multi-modal, MERRA-2, ERA5, benchmark, Prithvi WxC
- **One-liner**: 面向天气气候基础模型评测的 6 任务多模态 ML-ready "dataset of datasets"，从航空湍流分类到自然语言预报生成，附 per-task 基线验证。

## 精华提炼

### 管线/方法

定位：为天气气候基础模型（FM，如 Prithvi WxC 2.3B）的下游任务微调与泛化评测提供统一的 ML-ready 数据。总管线 = 原始非 ML-ready 源（再分析/卫星/飓风库/飞行员报告/文本讨论）→ per-task 预处理（网格化、coarse-graining、归一化、文本清洗）→ HuggingFace 托管的 per-task 数据集（NetCDF/HDF5/grib2/CSV）+ `dataset.py` 加载脚本 → 训练/微调任意 AI 模型 → per-task 技术验证基线。

6 个任务按"ML 现象"组织（分类/参数化/搜索/预测/多模态生成），空间尺度覆盖 meso-β（20-200 km）到 synoptic（2500 km）：

1. **航空湍流检测**（分类，microscale）：输入 MERRA-2（0.625°×0.5°，最低 40 个气压层至 100 hPa，仅 1800 UTC 单时刻快照）的 T、RH、u、v、ω、地形、位势高度；标签 PIREPs 飞行员报告（Iowa State 档案，2003–2023.11，CONUS）。报告按飞行高度分 LOW（0–14,900 ft）/MIDDLE（15,000–29,900 ft）/HIGH（≥30,000 ft，去掉 >50,000 ft），重分类为 moderate-or-greater（MODG）二值；按日 bin 到最近 MERRA-2 格点，MODG 频率 >25% 记 turbulent。原为 3 小时 binning，因每格报告太稀改为按日。
2. **重力波（GW）参数化**（回归，mesoscale）：ERA5（0.3°，137 层取 122 层，1.45 hPa–地面，去顶部 15 层 sponge），对全风场做 Helmholtz 分解取 divergent 分量 + 去前 21 次谐波高通滤波，通量 (F_u,F_v)=(u_div·ω′, v_div·ω′) 的 coarse-grain；输入为 coarse-grain 到 2.8°（64×128 Gaussian grid）的全风场 u/v/θ。4 年（2010/2012/2014/2015，按南半球冬季地形 GW 活动挑选），~2.87 亿列训练数据。
3. **天气 analog 搜索**：仿 ECMWF 子网格法，把 MERRA-2 全球切成 623 个约 1800 km 边长的 subgrid（西/南边各重叠 10 格点），1800 km 依据 1989 年经典研究（对兴趣点 12 小时预报足够且历史 analog 数量稳健）。
4. **S2S 长期降水预报**（预测 4 周）：输入三套卫星记录 —— GridSat-B1（静止轨道可见光/红外）、PATMOS-x（极轨 AVHRR+HIRS）、SSM/I CDR（微波）；标签 PERSIANN-CDR（1983–2000.6）+ IMERG Final（2000.6–今）拼接，统一插值到 0.625°×0.5°。Split：train 1983–2017（~1.2 万天文件/源）、val 2018、test 2019–2022，附评测脚本与 NWP 官方基线对比协议。
5. **飓风轨迹与强度**：HURDAT（NHC）1980–2022，大西洋 649 + 东太平洋 810 = 1459 个命名风暴；用 `tropycal` 包提取轨迹、强度（10m 风速、最低 SLP）、尺寸（最大风半径、34kt 风半径），6 小时 best-track 三次插值到 3 小时对齐 MERRA-2。
6. **自然语言预报报告生成**（多模态生成）：HRRR 分析场为输入、NOAA SPC 全国严重天气讨论存档为标签，1249 样本（2014.8–2017.12，CONUS）；文本清洗 = 去标点/特殊字符、展开缩写、统一小写、去数字与停用词；训练标签只取报告的 "...SUMMARY..." 节。

### 关键技术细节

- GW 归一化：输入 (x−μ)/3σ 缩到 [−2,2]，θ 除以 1000；输出通量因强间歇性（Laplace 分布）做全局标准化后再取**立方根**把极端值拉近 1。
- 湍流类分布：LOW 15,910 / MIDDLE 8,486 / HIGH 13,551 样本，turbulent 仅 6–7%（强不平衡），用 class-weighted loss 缓解。
- 基线模型逐任务：① 湍流 = 5 隐层 ANN（100/60/60/40/20，LeakyReLU + sigmoid）；② GW = Attention-UNet CNN；③ analog = conv encoder-decoder（输入 32×25 图像，100 epochs，lr 1e-3，batch 16），余弦相似度检索 + SSIM 评估；④ 降水 = ResNeXt-block 自回归 CNN（per-source stem → 共享空间编码器降 4× 两级 → 时间编码器 → propagator（encoder-decoder，取前两步 latent 状态推进）→ 共享 decoder 上采样 → 逐点卷积 head），quantile loss 预测 32 分位，AdamW lr 1e-4 余弦退火 50 epochs，预测至 16 天，输入通道 drop 2%/整传感器 drop 5% 增稳，经度周期 padding + 纬度反射 padding；⑤ 飓风 = FourCastNet（FNO+ViT，MERRA-2 上训练）；⑥ NLP 报告 = ViT 编码器 + GPT-2 解码器 captioning 微调。
- 数据访问：HuggingFace `nasa-impact/WxC-Bench`，每任务一个 `dataset.py`（`trust_remote_code=True` 加载）；代码 github.com/NASA-IMPACT/WxC-Bench。格式混杂：NetCDF/HDF5（xarray 读）、grib2（Pygrib 读）、文本（pandas 读）。

### 核心结果

| 任务 | 基线结果 |
|------|---------|
| 湍流分类 | OA ~80%，但 PoD 仅 38.9%（MIDDLE）–56.5%（LOW），FAR ~17.5%，F1 17.5–31.5% —— 不平衡 + MERRA-2 分辨率粗是主因 |
| GW 通量回归 | 中纬对流层 R² 最高 ~0.6，热带 ~0.3；学到 Rockies/Andes/东亚山脉热点、ITCZ 对流通量、南大洋 storm-track 通量 |
| Analog 搜索 | top-4 检索结果 SSIM >0.5（第 5 名 <0.2），仅单参数（温度或气压）搜索 |
| S2S 降水 | ML 基线小干偏差 vs NWP（ECMWF/UKMO）湿偏差；相关系数 NWP 前 10 天占优，10 天后 ML 反超——但此时两者都已降到月气候学水平以下 |
| 飓风（Michael 2018） | SLP 相关 2 天 0.97 → 5 天 0.3；MSLP RMSE 1 天 118 Pa → 3 天 282 Pa；track 误差 <40–50 km 尚可，但强度误差 50–60 hPa（很大） |
| NLP 报告生成 | ROUGE-L 0.28–0.42；能捕捉极端事件词（severe thunderstorms）、地理（southern Plains）、时间（this evening）等要素 |

Surprising/值得记的点：ML 降水基线"10 天后反超 NWP"其实是评测偏置（ML 的训练标签与评测参照同源 IMERG）+ 双方都已无技巧，作者对此明说，诚实度高。

### 明确局限

作者自述：湍流任务受 MERRA-2 粗网格限制（湍流本质 microscale），预期就难；NLP 生成的系统性评估留作 future work；analog 搜索目前只支持单参数检索；ERA5 只解析 150–200 km 以上波长的 GW（不声称完整解析）。

我读出的局限（对我们最重要）：
- **"dataset of datasets" 实为松散并置**：6 任务各自独立目录、独立数据格式、独立时空范围，没有统一的事件/样本封装结构，没有跨任务的统一评测协议或 leaderboard —— 严格说是数据集论文而非 benchmark 论文（仅降水任务有完整 split + 评测脚本 + 外部基线协议）。
- **无雷达模态**（无 MRMS/NEXRAD），无强对流事件标签（tornado/hail/wind report），无临近预报任务。
- **环境场无时间演变设计**：湍流任务只取 1800 UTC 单时刻快照配当日标签 —— 恰好反证"环境场时空演变作为一等公民"是空白。
- 时间对齐/泄漏处理未显式记录（PIREP 日聚合 vs MERRA-2 单时刻的错位；降水任务 ML 标签与评测参照同源）。
- 无 license/provenance/不确定性标记讨论，无 DOI 版本化（托管 HF + GitHub）。
- 真机/业务验证无（纯离线基线）；FourCastNet 飓风基线是自训 MERRA-2 版，训练细节不全。

- **R3=4**：明说 PoD 低的原因、ML 降水反超时已无技巧、飓风强度误差大、ROUGE-L 评估不充分 —— 不粉饰。

- **R5=4**：NASA 团队真实工程整编，6 任务各有实验图表；非 LLM 空壳。

对 0713-scs-benchmark（强对流环境–雷达多模态事件包基准）：

## Read by: 0713-scs-benchmark (2026-07-14)

**审读视角**：本 topic 要建以强对流事件为中心的环境场（HRRR，-15d/+5d 长时窗）+ 雷达（MRMS）+ 事件标签（NOAA Storm Events）多模态基准，目标 ESSD / NeurIPS D&B。

**撞车评估**：不撞车。WxC-Bench 的 6 任务均非强对流事件预警/nowcasting，无雷达模态、无 Storm Events 标签、无事件包封装。它对环境场的用法（湍流任务的 1800 UTC 单时刻快照）恰好是我们 gap 声明（"没有数据集把环境场时空演变当一等公民"）的实证案例——写 related work 时点名对比。它属于我们 landscape 表中"多任务 ML-ready 集合"一档（与 ClimateLearn 同类），而非"风暴尺度事件数据集"一档（SEVIR/Grad-Severe/MYRORSS）。

**可用工具/方法**：① 点报告→网格标签 binning 策略（时间窗聚合 + 频率阈值二值化 + class-weighted loss）可移植到 Storm Events → 事件标签整编，其"3h 窗太稀改日窗"的教训提示我们的事件聚类时间窗要做敏感性检查；② `tropycal` 包（HURDAT 提取）如果我们扩展到热带气旋事件可直接用；③ HuggingFace per-task `dataset.py` 托管模式与 Scientific-Data 式论文结构（Background → Data Description → Technical Validation）可作 ESSD 投稿组织参照；④ 其降水任务的 split + 评测脚本 + 官方 NWP 基线协议是我们 canonical task 配置的最低标准线。

**差异化**：我们的事件包结构（storm-centric、长时窗环境演变、多模态对齐、显式 provenance/license/不确定性标记）在 WxC-Bench 完全缺位；WxC-Bench 也自证了"松散并置"路线的短板（格式混杂、无统一评测协议）。我们应把"统一事件封装 + 可追溯整编"作为与其最鲜明的对比轴。
