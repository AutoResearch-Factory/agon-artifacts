# Meta-Learned Basis Adaptation for Parametric Linear PDEs (2604.09289)

- **Year**: 2026.04
- **Authors**: Vikas Dwivedi, Monica Sigovan, Bruno Sixou; CREATIS Biomedical Imaging Laboratory, INSA/CNRS/Inserm/Universite Lyon 1
- **Venue**: arXiv
- **Keywords**: KAPI, meta-learned basis adaptation, parametric linear PDE, PIELM, Gaussian RBF, predictor-corrector, physics-informed learning, shallow interpretable basis
- **One-liner**: 元学习PDE参数到RBF几何，再最小二乘校正。

## 精华提炼

### 管线/方法
- 论文目标不是直接学习一个黑盒 solution operator，而是学习“每个 PDE 参数实例应该用什么近似空间”。
- 输入是低维 PDE 参数 `lambda` 和 query coordinate `z`，输出先是 KAPI predictor 的粗解 `u_pred(z; lambda)`。
- KAPI 的核心内部模块是参数调制网络 `h_psi(lambda)`，它输出一组 task-adaptive Gaussian basis geometry。
- 这组几何包括 basis center `mu_j(lambda)`、width/scale `sigma_j(lambda)`、activity gate `g_j(lambda)`、amplitude/coefficient `a_j(lambda)`。
- 对 steady PDE，basis geometry 是空间上的；对 unsteady PDE，basis geometry 可以是 space-time 或动态移动中心。
- Predictor ansatz 是浅层 RBF 展开：`u_pred = sum_j g_j(lambda) a_j(lambda) phi(z; mu_j(lambda), sigma_j(lambda))`。
- 训练方式是 physics-informed meta-training，对 `lambda ~ rho(lambda)` 抽样，在 interior、boundary、initial collocation 上最小化 residual。
- 通用 loss 包括 PDE residual、BC residual、IC residual 和 geometry regularizer。
- Poisson 案例中用 `x(1-x)y(1-y)` hard enforce homogeneous Dirichlet boundary，loss 主要是 PDE residual 加 gate sparsity。
- Linear advection 案例中 regularizer 控制 moving center 的时间平滑性和 width collapse。
- Predictor 不是最终求解器，而是 basis generator；它要负责指出“哪里需要表达能力”。
- Corrector 接收 predictor-generated geometry，选出 activity/amplitude 较高的 inherited kernels。
- Corrector 还加入由 predictor gradient/residual cue 触发的 refinement basis。
- Corrector 最后加入 fixed background basis，保证全局覆盖，避免只在 predictor active region 附近有支撑。
- Corrector dictionary 是 `Theta_corr = Theta_inh union Theta_ref union Theta_bg`。
- 因为本文只处理 linear PDE families，固定 hidden basis 后的系数求解是线性 least squares。
- Corrected solution 是 `u_corr(z; lambda)=sum_m c_m(lambda) phi(z; theta_m(lambda))`。
- 系数通过对 PDE residual、BC、IC collocation 组装 `A_lambda c approx b_lambda`，再做 ridge/least-squares 一步求解。
- 整体 inference pipeline 是：`lambda -> Gamma_psi(lambda) -> u_pred -> Theta_corr(lambda) -> u_corr`。
- 这是一种 amortized targeted refinement：offline meta-training 后，每个新 task 只需 predictor pass 加一次 LS solve。
- 与 PINN 的差异：不是每个新 PDE instance 重新优化网络参数。
- 与 PIELM 的差异：PIELM hidden basis 通常固定或手工设定；这里 basis geometry 由 predictor 按 task 生成。
- 与 residual-adaptive PINN/RAR 的差异：RAR enrich collocation，但通常不改变网络架构；这里改变/生成 hidden basis。
- 与 neural operator 的差异：DeepONet/FNO 学 function-to-function map；本文学 parameter-to-approximation-space map。

### 关键技术细节
- Test Case 1 是 2D Poisson with Gaussian source，domain `[0,1]^2`，参数 `lambda=(x0,y0,nu)`。
- Poisson training range: `x0 in [0.4,0.6]`, `y0 in [0.4,0.6]`, `nu in [0.05,0.10]`，`nu` log-uniform。
- Poisson predictor 使用 `M=128` Gaussian kernels，2-layer internal parameter-conditioning network，width 64。
- Poisson predictor 训练 2000 epochs，Adam，learning rate `1e-3`，4 tasks/batch。
- Poisson collocation 混合 localized near-source sampling 和 uniform interior sampling。
- Test Case 2 是 periodic constant-coefficient 1D linear advection，domain `[0,1) x [0,1]`，参数 `lambda=(x0,nu)`。
- Advection PDE 是 `u_t + u_x = 0`，periodic BC，periodic Gaussian IC。
- Advection predictor 使用 `M=32` dynamic kernels，task encoder width 64，dynamic net width 96，time Fourier features。
- Advection predictor 训练 5000 epochs，Adam，learning rate `1e-3`，4 tasks/batch，包含 interior/IC/periodic BC/near-IC samples。
- 论文中 advection training range 有文字不一致：test-case 段写 `x0 in [0.2,0.8]`，而一处 baseline 图注写 `[0.35,0.65]`；后续 predictor-corrector 表又把 `x0=0.30` 标为 in-range。
- Test Case 3 是 advection-diffusion，domain `[0,1] x [0,0.5]`，参数 `lambda=(a,nu)`。
- Advection-diffusion PDE 是 `u_t + a u_x = nu u_xx`，`a in [0.5,1.0]`，`nu in [0.01,0.05]`。
- Advection-diffusion 使用 exact reference family `1/sqrt(4t+1) exp(-(x-xc-at)^2/(nu(4t+1)))`，`xc=0.2`。
- Advection-diffusion predictor 使用 `M=48` dynamic kernels，3-layer dynamic predictor，width 128，time Fourier features。
- Advection-diffusion 采用 viscosity curriculum：前半训练 `nu in [0.03,0.05]`，后半扩展到 `[0.01,0.05]`。
- Test Case 4 是 variable-speed advection，`u_t + a(x;beta)u_x=0`，`a(x;beta)=1+beta sin(2pi x)`。
- Variable-speed 参数为 `(x0,nu,beta)`，`x0 in [0.2,0.8]`，`nu in [0.03,0.12]`，`beta in [0.20,0.60]`。
- Variable-speed predictor 使用 `M=64` dynamic kernels，hidden width 128，periodic base dictionary。
- Variable-speed predictor 训练 5000 epochs，Adam，learning rate `5e-4`，用 width curriculum。
- Corrector 在 Poisson 中选 informative spatial kernels，加 source-local enrichment 和 structured background grid，ridge `1e-8`，另有 weak anchor term。
- Corrector 在 advection 中用 selected predictor-guided spacetime basis、本地 refinement、periodic background，并 sweep ridge candidates。
- Corrector 在 advection-diffusion 中用 inherited predictor basis、background basis、predictor-informed refinement centers，ridge `1e-8`。
- Corrector 在 variable-speed advection 中用 background spacetime basis、selected dynamic centers、gradient-informed refinement，ridge `1e-8`。
- 所有 predictor 都是 physics-informed 训练，不使用 labeled solution snapshots。
- Ground truth: Poisson 用 second-order finite-difference five-point stencil 直接解；linear advection 用解析 wrapped solution；advection-diffusion 用解析 family；variable-speed 用 characteristic travel-time map 数值反演。

### 核心结果
- Predictor-only baseline 比较覆盖 Poisson 和 linear advection，对手是 FiLM-HyperPINN 和 physics-informed DeepONet。
- Poisson predictor in-range: KAPI `2.008e-2` / `1.195e-2`，FiLM-HyperPINN `1.657e-2` / `6.055e-3`，PI-DeepONet `2.946e-2` / `1.345e-2`。
- Poisson shifted out-of-range `(0.30,0.30,0.06)`: KAPI `3.569e-1`，FiLM-HyperPINN `2.717e-1`，PI-DeepONet `4.129e-1`。
- Poisson narrow out-of-range `(0.50,0.50,0.03)`: KAPI `3.869e-2`，FiLM-HyperPINN `9.953e-2`，PI-DeepONet `1.543e-1`，KAPI 对 width extrapolation 更稳。
- Linear advection predictor: KAPI 在四个代表 case 上均优于 FiLM-HyperPINN 和 PI-DeepONet。
- Linear advection narrow out-of-range `(0.50,0.02)`: KAPI `3.656e-1`，FiLM-HyperPINN `3.752e-1`，PI-DeepONet `7.383e-1`。
- Full predictor-corrector 的强项是 Poisson、advection-diffusion 和 variable-speed advection。
- Poisson corrected errors: `7.032e-4`, `5.596e-4`, `1.122e-3`, `1.656e-3`，相比 predictor 大多提升 1-2 orders。
- Poisson shifted out-of-range 从 `3.569e-1` 降到 `1.122e-3`，这是论文最强证据之一。
- Linear advection corrected errors: `3.428e-2`, `2.040e-2`, `1.153e-1`, `3.885e-1`。
- Linear advection 的 correction gain 很小，最难 narrow out-of-range 还从 `3.656e-1` 变差到 `3.885e-1`。
- 这个负面结果说明：如果 predictor geometry 没有足够 sharp support，LS corrector 无法凭空恢复 fine scale。
- Advection-diffusion corrected errors: `1.959e-4`, `2.225e-4`, `1.846e-4`, `9.882e-4`。
- Advection-diffusion transport-dominated in-range `(0.95,0.015)` 从 `1.924e-1` 降到 `1.846e-4`，提升超过三 orders。
- Advection-diffusion low-viscosity out-of-range `(0.75,0.008)` 从 `3.904e-1` 降到 `9.882e-4`。
- Variable-speed advection predictor 很弱：四个 case predictor errors 为 `4.111e-1`, `5.614e-1`, `6.249e-1`, `1.075e0`。
- Variable-speed corrector 仍有明显帮助：降到 `2.673e-2`, `4.484e-2`, `5.715e-2`, `3.896e-1`。
- Variable-speed strongest extrapolation `beta=0.70` 仍然偏高，说明 basis geometry extrapolation 不够。
- Interpretability plots 显示 Poisson basis centers 聚集到 source response region。
- Linear advection 的 active centers 在 `(x,t)` 中沿 transported packet/characteristic-like path 移动。
- Corrector geometry 图显示 inherited predictor basis 负责定位重要区域，refinement/background basis 负责局部解析度和全局覆盖。
- Uniform-only PIELM ablation 证明 gain 不是“加 LS corrector”本身，而是 predictor-guided basis placement。
- Poisson guided corrector vs best uniform-only: `7.032e-4` vs `1.138e-2`, `5.596e-4` vs `2.659e-2`, `1.122e-3` vs `4.425e-2`, `1.656e-3` vs `3.445e-1`。
- Advection-diffusion guided corrector vs best uniform-only: `1.959e-4` vs `1.126e-2`, `2.225e-4` vs `3.640e-3`, `1.846e-4` vs `2.380e-2`, `9.882e-4` vs `3.704e-2`。
- Single-instance PINN ablation 给出公平 trade-off：单任务重训可以更便宜或更准，但 KAPI 是 family-level amortized solver。
- Poisson fixed task: KAPI predictor `1.294e-2` 优于 single-instance PINN `5.099e-2`；训练时间 `231.19s` vs `58.23s`。
- Advection fixed task: single-instance PINN `2.897e-2` 优于 KAPI `4.022e-2`；训练时间 `150.96s` vs `369.38s`。
- Multi-source Poisson appendix 显示两源和四源 Gaussian forcing 下同一 predictor-corrector 思路仍有效，但正文没有给表格数字。
- Mexican-hat advection appendix 显示 non-Gaussian sign-changing profile 上 predictor 已很准，corrector 小幅提升；errors 约 `5.677e-3` 到 `1.100e-2`。

### 明确局限
- 最大边界：只处理 parametric linear PDE families，因为 corrector 依赖线性 PDE 下的 linear least-squares solve。
- 对 nonlinear PDE，需要 nonlinear correction 或 sequential linearization；本文没有实验。
- 参数空间是低维 vector conditioning，不是 distributed coefficient、forcing field、boundary function 这种 function-valued conditioning。
- 这意味着它不是 full neural operator replacement，而是 low-dimensional parametric solver。
- 所有 predictor 都是 shallow Gaussian RBF basis，解释性强，但遇到 anisotropic、multiscale、strongly nonlinear structures 可能不够。
- Corrector 的上限取决于 predictor-generated basis quality；linear advection narrow extrapolation 的变差就是证据。
- 实验只有四类 canonical PDE，加 appendix extension；没有真实工程 benchmark、inverse problem、高维 nonlinear benchmark。
- 论文说 full code will be released，但当前文本本身不是完整 artifact；复现性依赖后续开源。
- 没有独立复现实验，也没有 confidence interval / seed variance。
- Baseline 设置有一定信息，但 FiLM-HyperPINN 和 PI-DeepONet 的 tuning 公平性仍需代码审计。
- Advection training range 在不同段落/图注/表格中存在不一致，是需要作者修正的 reproducibility 问题。
- Corrector 的 refinement cue 有时写成 gradient/residual/local structure，但具体 selection algorithm 细节不足。
- 论文没有比较 FNO 正式数值，只在 related framing 中讨论 neural operator 区别。
- 对 FEX 课题而言，它不产出 closed-form finite expression，也不解决 RL expression-tree search complexity。
- KAPI 的 output 是 basis geometry 和 LS coefficients，解释性是 geometric，不是 symbolic formula。
- 因此它不能直接替代 FEX 的“解析表达式求解器”主张。

## Read by: 0616-fex (2026-06-19)

从 FEX 视角，这篇是“非符号的结构迁移”相关工作，而不是直接竞品。它证明了一个有用经验：对 parametric PDE family，先学一个跨任务共享的结构先验，再在每个 instance 上做轻量 physics-constrained correction，可以显著减少每例重训成本。FEX 可以借鉴这个思想，但应把结构先验换成 symbolic skeleton / operator prior / collocation prior，而不是 RBF basis geometry。

撞车点主要在 `fex-active-collocation` 和 `fex-curriculum-depth`：KAPI 的 predictor-guided inherited/refinement/background basis 三分法，可以直接转译成 FEX 的 active residual sampling、局部表达式 refinement、global sanity coverage；width/viscosity curriculum 也支持我们做从浅树/宽峰/高黏性到深树/尖峰/低黏性的 staged search。它对 `fex-search-complexity`、`fex-llm-skeleton`、`fex-dim-lift-skeleton` 的核心 claim 不构成 novelty blocker，因为它不分析 RL 搜索复杂度、不用 LLM、不输出 finite expression，也没有跨维 symbolic skeleton lifting。

建议引用方式：把它放在“parametric PDE family amortization / learned solver priors / adaptive basis placement”相关工作中，用来说明非符号 PDE solver 已经在学 task-to-geometry map；然后强调 FEX 的差异化是最终对象可解析、搜索空间是 expression tree、可结合 e-graph/LLM/curriculum 约束来解决 symbolic search bottleneck。可用作 baseline 的不是完整 FEX 竞品，而是 continuous probe：先跑 KAPI/类似 RBF predictor 找 active regions，再让 FEX 在这些区域强化 residual 和 operator selection。
