# Fine-Tune Language Models as Multi-Modal Differential Equation Solvers (2308.05061)

- **Year**: 2023.08
- **Authors**: Liu Yang, Siting Liu, Stanley J. Osher (Department of Mathematics, UCLA)
- **Venue**: ICML 2024
- **Keywords**: in-context operator learning, multi-modal learning, differential equations, language model fine-tuning, scientific machine learning, ICON-LM, next-function prediction, caption-based operator description
- **One-liner**: 将 in-context operator learning 改造为多模态框架，通过自然语言"caption"注入人类对算子的先验知识，并用 GPT-2 级语言模型架构（ICON-LM）替代 encoder-decoder，实现参数减半、性能反超。

## 精华提炼

### 管线/方法

**整体流程**：
1. **输入构造**：caption tokens（自然语言描述 + LaTeX 方程）+ function tokens（condition/QoI key-value pairs）+ query tokens
2. **Token 嵌入**：
   - Caption tokens → caption embedding layer → caption embeddings
   - Function tokens（key + value，去除 one-hot index）→ shared function embedding layer（linear or shallow MLP）→ function embeddings
3. **位置编码**：caption embeddings 使用标准 LM 位置编码；function embeddings 使用可学习位置编码，同类型 tokens 共享同一编码以确保置换不变性
4. **单 Transformer 处理**：合并原 ICON 的 encoder+decoder 为单一 transformer，施加定制 attention mask
5. **输出映射**：仅保留 query token 对应位置的输出 → head layer（linear/MLP）→ 预测 QoI 函数值
6. **训练目标**：MSE loss，autoregressive "next-function prediction"——模型基于前 k 个 example 预测第 k+1 个 QoI，单次前向同时训练 0-shot 到 5-shot

**多模态训练**：
- Total loss = loss_with_captions + loss_without_captions
- 每个 prompt instance 随机从训练 caption 池中选取 vague 或 precise caption
- Precise caption 中的参数占位符在训练时替换为真实参数值

### 关键技术细节

**模型架构**：
- 单模态 ICON-LM：6 层 transformer，8 头 attention，d_model=256，d_ff=1024，总参数 15.8M（encoder-decoder ICON 的 31.6M 的一半）
- 多模态 ICON-LM：fine-tune GPT-2（124M 参数），full-parameter fine-tuning
- Embedding layer: 单模态用 linear；多模态用 shallow MLP（单隐层 1024 维）
- Head layer: 同上

**Attention Mask 设计**（核心难点，满足 5 个约束）：
1. Caption tokens: 下三角 mask（类 GPT 自回归）
2. 每个 example 内的 condition tokens 可互相关注（置换不变性），但不可关注当前 example 的 QoI tokens（防泄漏）
3. 当前 example 的 QoI tokens 不被任何 token 关注
4. Query tokens 互相不可关注（对角 mask），保证预测独立
5. Mask 满足 M²=M（幂等性），防止间接信息泄漏

**训练配置**：
- 优化器：AdamW，β1=0.9，β2=0.999，weight decay=1e-4
- 学习率：warmup-cosine-decay，peak lr=1e-4，总步数 1M，warmup 占前 10%
- 单模态 ICON-LM：batch size 24，单卡 RTX 4090 24GB，约 37.5h
- 多模态 GPT-2：batch size 10，双卡 RTX 4090，约 5.5 天
- 全局梯度裁剪：norm clip=1.0

**数据集**：
- 继承自 ICON (yang2023context)：19 类问题（forward/inverse ODE×6 + forward/inverse PDE×6 + MFC×5 + damped oscillator×2）
- 每类 1000 个训练 operator + 100 个测试 operator，每个 operator 有 100 个 condition-QoI pairs
- 测试集：19×500 个 learning cases，支持 J=0 到 5 shot
- Caption 数据：GPT-4 辅助生成，每 operator 160 个训练 caption + 40 个测试 caption（vague/precise 各半）
- 每个 condition/QoI 函数由 41-50 个 key-value pairs 表示

**Baseline 对比配置**：
- FNO：n_modes=16, hidden_channels=512，参数 20.2M
- DeepONet：trunk/branch 各 6 层 MLP，hidden width 1024，参数 14.8M
- FNO/DeepONet 预训练 0.1M steps，微调用 AdamW constant lr=1e-5

### 核心结果

**单模态对比（ICON-LM vs encoder-decoder ICON）**：
- ICON-LM（15.8M 参数）在所有 19 类问题的平均相对误差上一致优于 encoder-decoder ICON（31.6M 参数），从 1-shot 到 5-shot
- 训练时间更短（37.5h vs 41.5h），参数量减半

**In-Context vs Classic Operator Learning**：
- 在 Problem #14（inverse nonlinear reaction-diffusion PDE）的 5-shot 场景：
  - ICON-LM 单次前向的误差远低于 FNO/DeepONet 逐 case 微调
  - FNO/DeepONet 对偏离均值分布的 testing operator 泛化差，ICON-LM 对所有 operator 一致表现良好
  - ICON-LM 在训练时只用了 41-50 tokens/函数，测试时直接泛化到 101 tokens，无需微调

**多模态效果（fine-tuned GPT-2）**：
- 误差递降：no caption → vague caption → precise caption
- Zero-shot（precise caption）表现惊人：仅靠文字描述即可预测，无需任何 function example
- Vague caption 的 zero-shot 在某些问题上也好（如 damped oscillator、inverse Poisson、MFC），因为 condition 本身隐含了 operator 参数信息

**Ablation——预训练 vs 随机初始化 GPT-2**：
- Zero-shot + precise caption 平均相对误差：
  - Pretrained GPT-2: Train 2.30%, Test 2.44%, Gap 0.14%
  - Unpretrained GPT-2: Train 2.07%, Test 3.08%, Gap 1.01%
- 预训练语言模型的先验知识显著提升了多模态科学 ML 任务的泛化能力，训练-测试 gap 缩小约 7 倍

### 明确局限

**作者自述**：
1. 函数表示仅限于散点（scattered points），未探索其他表示形式（如 Fourier 基、mesh、图像等）
2. 多模态 autoregressive training scheme 的可扩展性留待 future work

**审读发现的局限**：
3. 数据集规模较小（19 类问题，每类 1000 operators），远未达到 "foundation model" 级别
4. Caption 由 GPT-4 生成——依赖 LLM 先验，若 LLM 不理解的物理系统则无法生成有效 caption
5. 无真机实验，所有验证均为数值仿真
6. 只在 1D/2D 域上验证，未涉及 3D 或高维问题
7. GPT-2 124M 训练需要 5.5 天双卡——大规模模型的训练成本未讨论
8. 未与同期多模态科学 ML 工作（如 PROSE 2309.16816）对比
9. 论文声称 "opened up a new avenue for LLM in scientific ML"，但仅在 toy-scale 问题上验证，离实际科学计算差距大
10. 未开源代码（声称 "open-sourced alongside code" 但 arxiv 页面无链接）

### 跟我们的关系

对 **0616-fex** 课题：

**不撞车**：本文是 in-context operator learning 的多模态扩展，核心关注如何用 caption 注入算子知识。0616-fex 课题如果做的是 feature extraction 相关（从你的课题名推断），双方关注点不同——本文关注"用什么架构和训练方式让模型学会从多模态输入中推断算子"，而非"如何从数据中提取更好的 feature representation"。

**可用工具/方法**：
1. **ICON-LM 的 attention mask 设计**（Figure 3）：精心设计的 mask 保证了置换不变性 + 防泄漏 + 幂等性，如果你需要在 transformer 中同时处理多种异构输入（数值+文本），这个 mask 设计思路直接可用
2. **Next-function prediction 训练范式**：autoregressive 地在单次前向中同时训练 0-shot 到 K-shot，比 ICON 的固定 example 数训练更高效
3. **Learnable positional encoding 按 token type 共享**：保证同类型 token 置换不变性的简洁方案
4. **GPT-2 fine-tuning 作为多模态 backbone**：证明了预训练 LM 对科学计算任务有正向迁移

**该引不该引**：如果你的工作涉及 (a) 用 transformer/LM 做科学计算，(b) 多模态科学数据融合，(c) in-context learning for scientific tasks，应该引用。否则仅作参考。

**差异化**：你可以从本文的局限出发——(a) 本文只用散点表示函数，你可以用更丰富的表示；(b) 本文 caption 依赖 GPT-4 生成，你可以探索不需要外部 LLM 的知识注入方式；(c) 本文只做 1D/2D，你可以扩展到更高维度或更实际问题。

## Read by: 0616-fex (2026-06-24)

本次审读聚焦于这篇 ICML 2024 论文对 **0616-fex（feature extraction for scientific ML）** 课题的参考价值。

**核心发现**：本文证明了 (1) 预训练语言模型的先验知识可以通过 fine-tuning 迁移到数值计算任务，(2) 自然语言描述（caption）是注入人类算子知识的有效媒介，(3) 精心设计的 attention mask 可以同时处理异构输入并保证数学约束。

**对 0616-fex 的启示**：
- 如果你在做 scientific data 的 feature extraction，ICON-LM 的 function token 嵌入方式（key-value 直接拼成向量 + 共享可学习位置编码）是一个轻量但有效的方案
- Caption 机制本质上是一种 **semantic feature conditioning**——将人类对系统的语义理解编码为 latent condition，引导数值预测。这个思路可以泛化到任意 feature extraction pipeline
- GPT-2 预训练权重的正向迁移暗示：语言模型的内部表示已经编码了某些对科学推理有用的结构——是否可以利用 LM 的 hidden states 作为一种 implicit feature extractor？

**撞车评估**：不撞车。本文关注的是"如何让模型学会算子"，而非"如何提取更好的特征表示"。但如果你计划将 LLM 用作 scientific feature extractor，本文是一个重要的 reference point。
