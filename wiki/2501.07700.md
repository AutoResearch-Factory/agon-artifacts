# Adaptive Collocation Point Strategies For Physics Informed Neural Networks via the QR Discrete Empirical Interpolation Method (2501.07700)

- **Year**: 2025.01
- **Authors**: A. Celaya, David Fuentes, Beatrice Riviere (Rice University; The University of Texas MD Anderson Cancer Center)
- **Venue**: arXiv preprint; manuscript formatted for Computer Methods in Applied Mechanics and Engineering
- **Keywords**: Physics-Informed Neural Networks, adaptive collocation, QR-DEIM, randomized SVD, residual snapshots, PDE solvers, reduced-order modeling, active sampling
- **One-liner**: 用 QR-DEIM 从 PINN residual 动态中选自适应 collocation 点，在 4 个 PDE benchmark 上显著降误差。

## 精华提炼

### 管线/方法

本文要解决的是 PINN 训练中 collocation point 的分布问题：固定随机点、等距网格、Hammersley/Latin/Sobol 等点集通常不看训练中 residual 的变化，因此会漏掉高梯度、shock、transport front 等关键区域。

作者提出两条 adaptive collocation 管线：标准 QR-DEIM 和 randomized QR-DEIM-R。两者都只处理 interior residual points；初值和边界条件在实验里通过 output transform 强约束，从而把 loss 简化为 PDE residual MSE。

标准 QR-DEIM 管线如下：

1. 输入 PDE、PINN 网络 `u_theta(x,t)`、训练 collocation set `T`、独立 snapshot candidate set `S`。
2. 初始训练集 `T` 由随机采样得到，默认 `N_train=2000`。
3. snapshot set `S` 与 `T` 不相交，默认 `N_snapshot=1000`。
4. 每个训练 iteration 都在 snapshot set 上评估 PDE residual `r(x,t)`。
5. 每隔 `P=1000` 次 iteration，把最近 `P` 次 snapshot residual 组成矩阵 `R in R^{N_snapshot x P}`。
6. 对 residual snapshot matrix 做 SVD：`R = V Sigma W^T`。
7. 根据能量阈值 `epsilon=0.005` 选择 rank `k`，使前 `k` 个 singular values 覆盖至少 `1-epsilon` 的 residual snapshot energy。
8. 取 `V_k` 后对 `V_k^T` 做 column-pivoted QR。
9. QR pivot indices 对应 snapshot set 中最能代表 residual 动态的点，将这些点作为新 collocation points 候选。
10. 同时在当前训练集 `T` 上比较上一次 update 和本次 update 的 residual magnitude。
11. 定义 convergence degree `d = log2(r_old(T) / r_new(T))`。
12. 若 top-k convergence degree 中有负值，说明候选删除点没有真正收敛或 residual 变坏，本轮不更新训练集。
13. 否则删除 `T` 中 convergence degree 最大的 `k` 个点，也就是最已解决的点。
14. 加入 QR-DEIM 选出的 `k` 个 snapshot points，保持训练集规模固定。
15. 重新采样 snapshot set `S`，继续下一轮训练。

QR-DEIM-R 管线保持同样的 add/remove 逻辑，但替换最贵的两个步骤：

1. 用 randomized SVD 近似 `R ≈ V_k Sigma_k W_k^T`，rank `k` 直接作为输入，默认 `k=100`。
2. 不对完整 `V_k^T` 做 QR pivoting，而是按 `V_k^T` column norm 构造离散概率分布。
3. 从 snapshot columns 中按概率采样 `ell = k + z` 个候选，默认 oversampling `z=50`。
4. 只在小矩阵 `C in R^{k x ell}` 上做 QR with column pivoting。
5. pivot indices 映射回原 snapshot set，得到新 collocation points。

这套方法的核心不是简单“高 residual 点优先”，而是把一段训练期间的 residual time series 看成 snapshot matrix，用低秩结构抽取残差演化模式，再通过 QR-DEIM 选出 interpolation points。

因此它能捕捉 update interval 之间的 residual dynamics；这是作者认为 RAR-G、RAD、RAR-D、R3、PINNACLE 等固定间隔 adaptive 方法容易遗漏的信息。

### 关键技术细节

- **PINN formulation**：通用 PDE 写成 `u_t = L(u) + F(u)`，residual 为 `r(x,t)=u_t-L(u)-F(u)`，导数由 automatic differentiation 计算。
- **Loss**：实验中强约束 IC/BC 后只优化 residual MSE；未把 IC/BC loss 作为额外项比较。
- **Network**：fully connected feedforward network，5 个 hidden layers，每层 64 hidden units。
- **Activation**：`tanh`。
- **Optimizer**：Adam。
- **Learning rate**：初始 `0.001`，cosine annealing schedule。
- **Training length**：wave、Allen-Cahn、Burgers 均 `100,000` iterations；convection 因 loss landscape 更病态训练 `1,000,000` iterations。
- **Training points**：所有 benchmark 默认 `2,000` collocation points。
- **Validation**：同一组 `10,000` random validation points，用最低 validation loss 的模型权重。
- **Test set**：空间-时间域 `[-1,1] x [0,1]` 上 `256 x 100` grid。
- **Metric**：relative `l2` error，报告 10 次 stochastic runs 的 mean/std。
- **Implementation**：Python + PyTorch 2.7.1。
- **Hardware**：Nvidia RTX A4000 GPU，Intel Xeon w7-3445 CPU。
- **QR-DEIM defaults**：`N_snapshot=1000`，snapshot period `P=1000`，energy threshold `epsilon=0.005`。
- **QR-DEIM-R defaults**：`N_snapshot=1000`，`P=1000`，target rank `k=100`，oversampling `z=50`。
- **Baselines**：Random、Hammersley、Random-R、RAR-G、RAR-D、RAD、R3、PINNACLE。
- **RAR-G/RAR-D**：initial points `1000`，sampling interval `1000`，每次加 `10` 点，candidate pool `10000`。
- **RAD**：`2000` initial points，每 `1000` iteration release/resample all points，candidate pool `10000`。
- **R3**：`2000` initial points，interval `1`，以 residual magnitude mean 为 threshold，release/resample `k` 个点。
- **PINNACLE**：`1000` initial points，每 `1000` iteration 加 `100` 点，`Z_pool=4000`，`Z_ref=500`，`m=100` NTK modes。

### 核心结果

主表报告 4 个 PDE 上 10 次运行的 relative `l2` error mean/std：

| Method | Wave | Convection | Allen-Cahn | Burgers |
|--------|------|------------|------------|---------|
| Random | 1.17e+00 (5.49e-01) | 1.06e+00 (4.60e-02) | 2.21e-02 (2.97e-02) | 2.84e-01 (1.65e-01) |
| Hammersley | 3.96e-01 (4.24e-01) | 1.03e+00 (3.35e-02) | 6.18e-03 (1.57e-03) | 2.66e-01 (1.86e-01) |
| Random-R | 1.80e-01 (9.65e-02) | 1.02e+00 (1.44e-02) | 9.00e-03 (1.00e-03) | 2.34e-02 (3.57e-02) |
| RAR-G | 1.50e-01 (1.24e-01) | 1.35e+00 (1.48e-01) | 5.14e-03 (9.35e-04) | 3.12e-03 (2.84e-03) |
| RAR-D | 9.68e-02 (8.09e-02) | 1.35e+00 (1.35e-01) | 5.53e-03 (9.63e-04) | 7.31e-04 (1.78e-04) |
| RAD | 5.16e-02 (5.31e-02) | 8.07e-01 (2.09e-02) | 8.61e-03 (2.70e-03) | 1.23e-03 (4.57e-04) |
| R3 | 2.25e-01 (2.39e-02) | 8.44e-01 (1.91e-02) | 1.83e-02 (2.26e-03) | 1.77e-03 (4.43e-04) |
| PINNACLE | 4.96e-01 (3.31e-01) | 1.29e+00 (2.50e-01) | 3.21e-02 (3.39e-02) | 9.60e-03 (4.13e-03) |
| QR-DEIM | **9.44e-03 (2.49e-03)** | **6.94e-02 (4.29e-02)** | 4.05e-03 (1.67e-03) | 5.96e-04 (7.07e-05) |
| QR-DEIM-R | 9.96e-03 (3.47e-03) | 7.02e-02 (1.88e-02) | **3.85e-03 (1.87e-03)** | **5.06e-04 (5.69e-05)** |

最强结果在 convection：QR-DEIM/QR-DEIM-R 的 error 约 `7e-02`，而 RAD/R3 仍在 `8e-01`，RAR/PINNACLE 超过 `1.2`，显示它对 advective temporal dynamics 很敏感。

Wave 上 QR-DEIM 从最强非本文 baseline RAD 的 `5.16e-02` 降到 `9.44e-03`，约 5 倍提升；也比 RAR-D 的 `9.68e-02` 低一个数量级。

Allen-Cahn 上差距较小但仍领先：QR-DEIM-R `3.85e-03`，QR-DEIM `4.05e-03`，RAR-G `5.14e-03`。

Burgers 上 QR-DEIM-R 最低 `5.06e-04`，QR-DEIM `5.96e-04`，RAR-D `7.31e-04` 接近但仍略差。

可视化结果显示两类 adaptation：

- Wave/convection：points 先集中在早期时间段，随后向后期时间推进，像自动学到 temporal curriculum。
- Allen-Cahn/Burgers：points 集中到 phase boundary、shock/front 等空间高梯度区域。
- QR-DEIM-R 的点分布与完整 QR-DEIM 相似，说明随机化近似保留了主要选点行为。

计算成本方面：

- 固定采样在 wave 上约 `9s / 1000 iterations`。
- QR-DEIM/QR-DEIM-R 约 `11s / 1000 iterations`。
- 额外成本主要来自 snapshot residual matrix 的重复 residual evaluation。
- 标准 QR-DEIM update 在 `1000 x 1000` residual matrix 上约 `0.2s`。
- QR-DEIM-R update 约 `0.02s`。
- 作者认为整体开销介于 R3 和 RAD/RAR/PINNACLE 之间：比 R3 贵，比 PINNACLE 便宜，和 RAD/RAR 接近。

Ablation 结论：

- QR-DEIM 的 `epsilon` 从 `0.1` 到 `0.001` 变化时，wave error 都在可接受范围；默认 `0.005` 最好，`9.87e-03`。
- QR-DEIM-R 的 rank `k` 从 `50` 到 `250` 变化时也较稳定；默认 `k=100` 最好，`9.96e-03`。
- snapshot points 与 period 的组合也相对鲁棒；QR-DEIM 在 `N_snapshot=1000, P=500` 时 wave error 最低 `7.59e-03`，默认 `P=1000` 为 `9.44e-03`。
- QR-DEIM-R 在 `N_snapshot=1000, P=1000` 默认组合下表现最好。

### 明确局限

1. **只验证 1D space + time benchmark**：wave、convection、Allen-Cahn、Burgers 都在 `[-1,1] x [0,1]`，没有 2D/3D、复杂几何、真实 multi-physics。
2. **没有真实工程数据或真机系统**：全部是标准 PDE synthetic benchmark，可控但不能说明真实科学计算工作流中的鲁棒性。
3. **没有公开代码证据**：tex 中给了 PyTorch 2.7.1、硬件和超参，但没有读到代码仓库、随机种子、完整运行脚本或 artifact。
4. **方法依赖强约束 IC/BC 的简化设置**：作者明确为了聚焦 interior points，把 IC/BC 用 output transform 强约束；对弱约束、多源观测数据、复杂边界的效果未验证。
5. **对高维扩展只是未来工作**：标准 QR-DEIM 的 SVD/QR 对 snapshot size 有压力，QR-DEIM-R 只能说明方向，不能证明高维 PDE 可行。
6. **baseline 公平性有待复查**：所有方法用了统一简单网络和 Adam，这有利于 isolating sampling effect，但不代表每个 baseline 的最佳配置。
7. **Convection 训练 1,000,000 iterations**：比其他 PDE 多 10 倍，虽然作者给了 loss landscape 原因，但实用成本需要单独评估。
8. **没有统计显著性检验**：报告 mean/std over 10 runs，但没有 confidence interval 或 paired significance。
9. **未讨论 failure cases**：ablation 说鲁棒，但没有展示 QR-DEIM 失败于何种 PDE、何种 snapshot period、何种 noisy residual。
10. **只处理 forward PDE solving**：作者明确区分 Forootani et al. 的 inverse parameter estimation；本文不是 inverse PINN 方法。

### 跟我们的关系

对 `0616-fex`，如果课题关注的是 feature extraction / function extraction / field experiment 中的“从动态信号里抽关键采样点或关键特征”，这篇不直接撞整个课题，但会撞到一个明确 claim：**用 residual snapshot 的低秩结构做 adaptive point/feature selection**。

最可复用的工具是 `residual snapshot matrix -> SVD/randomized SVD -> QR pivot / randomized QR pivot -> representative points` 这条管线。

如果 `0616-fex` 的方法也想声称“adaptive sampling based on residual dynamics”，这篇必须引用，并且需要与 QR-DEIM/R3/RAD/PINNACLE 明确区分。

差异化空间在于：

- 把 QR-DEIM 的选点从 PINN collocation 扩展到更一般的 feature extraction 或 experiment design；
- 用在线/streaming SVD 处理更长时间窗口，而不是每 `P` 步重建完整 residual matrix；
- 将 snapshot residual 换成多源特征，例如 gradient norm、uncertainty、NTK sensitivity、物理约束 violation；
- 在高维 PDE、复杂边界、多物理耦合或真实观测数据上验证，而不是只做 1D benchmark；
- 将 QR-DEIM 作为一个可解释 selector，与 learned sampler 或 active learning policy 比较。

撞车判断：**局部中高风险**。若 `0616-fex` 的核心 novelty 是“从 residual/feature snapshots 中用 SVD/QR 选关键点”，本文已经覆盖了 PINN forward PDE 场景；若课题做的是其他模态或把 feature extraction 作为辅助模块，则更像可引用的 baseline 和工具来源。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 视角读，重点看它是否已经覆盖“从训练动态中抽取关键特征/关键采样点”的 claim。结论是：它不只是普通 PINN adaptive sampling，而是把 residual 在 snapshot set 上的时间序列组织成矩阵，再用 SVD/QR-DEIM 选代表性点。因此，如果我们的课题也想把动态 residual 或 feature trace 做低秩特征提取，这篇是必须对齐的近邻。

可直接复用的方法模块有三块：第一，residual snapshot matrix 的构造；第二，能量阈值或 fixed-rank randomized SVD；第三，用 QR pivot 或 randomized QR pivot 把低秩 modes 转成实际采样点。它的 pruning 逻辑也值得借鉴：删除 residual 已显著下降的点，而不是只往高 residual 区域加点。

差异化不应声称“首次把 SVD/QR 用于 PINN adaptive sampling”；这个 claim 已经被本文覆盖。更稳的方向是把 FEX 问题定义到不同对象上，例如多模态 feature traces、实验设计、真实物理传感器选点、在线 streaming selection，或把 QR-DEIM 与 uncertainty/NTK/gradient features 结合。

风险点是本文实验范围较窄：1D PDE、标准 benchmark、无代码 artifact、无复杂边界。若 `0616-fex` 能在高维、真实数据或更复杂 feature extraction 场景中证明同类 selector 的价值，就不是直接撞车，而是可把本文作为 strong baseline 和理论动机。
