# Conflict-Aware Harmonized Rotational Gradient for Multiscale Kinetic Regimes (2604.24745)

- **Year**: 2026.04
- **Authors**: Zhangyong Liang (National Center for Applied Mathematics, Tianjin University)
- **Venue**: Submitted to SIAM (siamart251216 class)
- **Keywords**: multi-task learning, asymptotic-preserving neural networks, gradient conflict resolution, Boltzmann-BGK equation, linear transport equation, Knudsen number, isometric rotation, harmonized cone
- **One-liner**: 提出 HRGrad 方法，用流形等距旋转替代欧几里得投影来解决跨 Knudsen 数连续谱训练 APNN 时的梯度冲突，保证非冲突更新的同时 100% 保留梯度模长。

## 精华提炼

### 管线/方法

HRGrad 是一个梯度预处理算子，嵌入 Adam 优化器中使用。整体训练流程：

1. **采样**：每轮从 Knudsen 数连续谱 $\varepsilon \in [\varepsilon_{\min}, 1]$ 中对数均匀采样 $m$ 个 scale parameters，$\varepsilon_k = \exp(\xi_k)$，$\xi_k \sim \mathcal{U}(\log \varepsilon_{\min}, 0)$。密度正比于 $1/\varepsilon$，保证小 $\varepsilon$ 区域有足够采样密度。

2. **APNN 前向**：对每个 $\varepsilon_i$ 使用统一 APNN 架构（micro-macro 或 even-odd 分解）计算损失 $\mathcal{L}_i = \mathcal{L}_{\mathrm{APNNs}}^{\varepsilon_i}$，包含 kinetic residual、conservation law residual、moment consistency、BC、IC。

3. **HRGrad 四阶段梯度预处理**（应用于 Adam 的 bias-corrected first moments $\hat{m}_{g_i}$）：
   - **Stage 1: Harmonized Cone & Physical Anchoring** — 用 Double Description Method (DDM) 求解 $\hat{G}^\top\hat{G}\lambda \ge 0$ 提取极端射线 $\Pi$，定义 Harmonized Cone $\mathbb{H} = \mathbb{K} \cap \mathbb{K}^*$，物理锚点 $d^* = \mathcal{U}(\sum \mathcal{U}(r_j))$ 为极端射线归一化质心。
   - **Stage 2: Manifold Isometric Rotation** — 对冲突梯度（$\bar{g}_i^\top d^* < 0$），在 2D Riemannian 切平面 $\mathcal{T}_i = \text{span}(\bar{g}_i, d^*)$ 上用 Gram-Schmidt 构建正交参考 $w_i$，然后用参数化 $SO(2)$ 旋转 $r_i(\alpha_i) = \cos(\alpha_i)\bar{g}_i + \sin(\alpha_i)w_i$，保证 $\|r_i(\alpha_i)\| \equiv 1$。
   - **Stage 3: MER Optimization via MCAE** — 联合优化所有旋转角 $\boldsymbol{\alpha} \in [0,\pi/2]^m$，目标函数 $\mathcal{L}(\boldsymbol{\alpha})$ = Global Alignment（抑制 Under-Correction Angular Error）+ $\lambda$ × Global Proximity（抑制 Over-Correction Angular Error）。自适应内层 SGD 步数：$\alpha_{\text{steps}}^{(t)} = \lfloor \alpha_{\min} + (\alpha_{\max} - \alpha_{\min}) s_t/(s_t + k_{\text{std}}) \rfloor$，其中 $s_t$ 是相对损失变化的标准差。
   - **Stage 4: Magnitude Restoration & Fair Aggregation** — 恢复原始模长 $g_i^{\text{rot}} = \|g_i\| \cdot r_i(\alpha_i^*)$（冲突）或保持原梯度（非冲突）。用 Moore-Penrose 伪逆构造公平共识方向 $g_u = \mathcal{U}[(M^\dagger)^\top \mathbf{1}_m]$，最终更新 $g_{\text{HRGrad}} = (\sum (g_i^{\text{rot}})^\top g_u) g_u$。

4. **Adam 更新**：将 $g_{\text{HRGrad}}$ 作为统一梯度估计，按标准 Adam 流程更新网络参数。

### 关键技术细节

- **APNN 架构**：统一 base model，macro-micro 分解（BGK）或 even-odd 分解（linear transport）。BGK 问题用两个独立通道编码 $f_\theta$ 和 $(\rho_\theta, u_\theta, T_\theta)$ 并解码。
- **激活函数**：$\sigma(x) = x/(1+\exp(-x))$ (BGK)，$\sigma(x) = \tanh(x)$ (linear transport)。
- **优化器**：Adam，初始学习率 $\eta_0 = 10^{-3}$，指数衰减 $\gamma = 0.96$，衰减步长 $p = 200$。
- **初始化**：Xavier 初始化。
- **硬约束**：$f_\theta$ 通过 softplus 保证正性；$\rho_\theta, u_\theta, T_\theta$ 通过构造自动满足边界条件。
- **积分**：线性输运用 30 个 Gauss 求积点，BGK 用 64 个（1D smooth 用 257 个梯形法则点）。速度截断 $V = 10$（BGK）/ $V = 8$（ES-BGK）。
- **Fourier 特征**：周期边界条件时用 $k=8$ 的 Fourier 基变换预处理输入。
- **MAD 框架**：采用 meta-auto-decoder 方法构造 MTL 框架以加速收敛。
- **收敛性证明**：对 convex 和 non-convex 设置都给出了收敛保证。关键引理：(i) Isometric Fidelity $\|g_i^{\text{rot}}\| = \|g_i\|$, (ii) Non-Conflict $(g_i^{\text{rot}})^\top g_{\text{HRGrad}} \ge 0$, (iii) Equal Cosine Similarity $S_c$ uniform across tasks, (iv) Aggregate Product Identity。非凸情况下收敛速率为 $O(1/K)$（ergodic bound），速率最多随任务数 $m$ 按 $O(m/K)$ 退化。
- **收敛率常数**：引入 manifold mapping coefficient $\kappa = \rho_{\min}/S_c \in (0,1]$，冲突自由时 $\kappa = 1$。
- **ConFIG 兼容性**：无冲突时 HRGrad 精确退化为 ConFIG。

### 核心结果

- **10 个 MTL baseline 对比**：LS, UW, DWA, GradDrop, PCGrad, CAGrad, IMTL-G, RLW, Nash-MTL, FAMO。用 Mean Rank (MR, 越低越好) 和 Average F1 score ($\overline{F_1}$, 越高越好) 评估。
- **验证问题**：BGK 方程（1D smooth + 1D Riemann/Sod）、线性输运方程（periodic + non-periodic + 2D smooth）、ES-BGK 方程、线性半导体 Boltzmann-Poisson 方程。
- **Knudsen 数范围**：从 $\varepsilon = O(1)$（kinetic/rarefied）到 $\varepsilon \to 0$（fluid/diffusive），包括 $\varepsilon = 10^{-6}$ 甚至更小的极限，log-uniform 采样保证覆盖所有 decade。
- **关键发现**：
  - PCGrad 和 ConFIG 在严重多尺度冲突下遭遇"能量裁剪"：当 $\phi \to \pi$ 时 $\|\mathcal{P}(g_{\text{micro}})\|_2 \to 0$，不可逆地抹除高频动力学特征。
  - ConFIG 在 NS2d-C, Volterra1d, Volterra system 上 5 次独立运行全部产生 NaN（原版 least-squares），作者改用伪逆实现。
  - HRGrad 的 isometric rotation 保证 $\|g_i^{\text{rot}}\| \equiv \|g_i\|$，即使 $\phi \to \pi$ 也不损失微观梯度能量。
  - 对数均匀采样策略显著提升小 $\varepsilon$ 区间的训练稳定性和外推能力。
- **ES-BGK 扩展**：额外包含各向异性 Gauss 平衡态和应力张量，测试 HRGrad 在非各向同性碰撞下的鲁棒性。
- **半导体 Boltzmann-Poisson**：加入自洽电场、Poisson 耦合和 drift-diffusion 渐近极限，比纯输运更具挑战性。

### 明确局限

1. **作者自述**：高维动力学问题（velocity space 维度高时）的计算复杂度仍有待改进；复杂碰撞项和高维积分的计算成本未解决。
2. **未做真机实验**：所有实验均为数值仿真，在 Nvidia RTX-3090-24GB GPU 上完成。未在真实物理系统上验证。
3. **无标准 benchmark 对比**：未在标准 MTL benchmark（如 NYUv2, Taskonomy, CityScapes）上测试，仅在 kinetic equations 上验证。通用 MTL 场景的有效性未知。
4. **超参数敏感**：HRGrad 引入了 $\alpha_{\min}, \alpha_{\max}, k_{\text{std}}, \lambda, \delta$ 等额外超参数，文中未给出系统的消融研究。自适应步数策略的 $k_{\text{std}}$ 选择缺少指导。
5. **DDM 计算开销**：Stage 1 需要在每次迭代求解 DDM 约束 $\hat{G}^\top\hat{G}\lambda \ge 0$，对于高维参数空间（$D$ 很大）的神经网络，这个 $m \times m$ 矩阵操作的计算成本未量化分析。
6. **仅与 Adam 集成**：算法展示仅嵌入 Adam，未讨论与其他优化器（SGD, AdamW, LAMB 等）的兼容性。
7. **对数均匀采样的假设**：采样策略假设 multiscale transitions 是乘性的，如果实际问题有非对数的尺度变化可能存在偏差。
8. **无大规模分布式训练**：所有实验单 GPU，未测试多 GPU/分布式训练场景。
9. **代码已开源**：`https://github.com/liangzhangyong/HRGrad`，但未见 artifact evaluation 或独立复现报告。

- **该引**：如果 0616-fex 涉及 multiscale PDE 的多任务预训练或参数化求解，应引用此工作作为梯度冲突解决方案。此论文也引用了 ConFIG、PCGrad、CAGrad 等，可作为 MTL 优化器对比的参考链。
- **差异化**：0616-fex 如果聚焦于 (a) 更大规模的 PDE 类型覆盖（不只 kinetic equations），(b) 网络架构创新（如 attention-based operator），(c) zero-shot/few-shot 泛化到新 PDE，则与 HRGrad 的工作重点明显不同。HRGrad 是"如何训"的方法，0616-fex 可以侧重"训什么架构"和"覆盖多少 PDE"。

## Read by: 0616-fex (2026-06-24)

本次审读视角：从 PDE foundation model 训练方法的角度审视 HRGrad。

**撞车评估**：不撞车。HRGrad 是优化方法论（how to train），0616-fex 的 PDE foundation model 侧重架构和数据覆盖（what to train）。两者互补——HRGrad 可以成为 foundation model 多尺度预训练的梯度聚合模块。

**可提取的方法组件**：
1. **对数均匀采样** — 适用于 foundation model 在连续物理参数谱上的预训练数据采样。
2. **HRGrad 梯度预处理** — 如果 foundation model 需要同时拟合多个参数 regime（如不同 Reynolds 数的 Navier-Stokes），这个算子可以替代 naive gradient averaging。
3. **等距旋转的保范性质** — 保证弱信号（小尺度物理）不被强信号（大尺度物理）淹没，对 multiscale foundation model 训练至关重要。
4. **MAD 框架** — 将物理参数编码为 latent code 注入网络，是 foundation model 条件化的有效方案。

**不应引用的理由**：如果 0616-fex 的 focus 是纯数据驱动的 neural operator（如 FNO、DeepONet 变体），不使用物理约束 loss，则 HRGrad 的 APNN 特定设计不直接适用。但 log-uniform sampling 和梯度冲突分析仍然有参考价值。

**建议**：关注 HRGrad 的 Stage 1（Harmonized Cone）中 DDM 的计算开销是否能在 foundation model 的高维参数空间中保持可行——这是将该方法集成到大规模训练的关键瓶颈。
