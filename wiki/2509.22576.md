# EPO: Entropy-regularized Policy Optimization for LLM Agents Reinforcement Learning (2509.22576)

- **Year**: 2025.09
- **Authors**: Wujiang Xu et al., Rutgers University / Adobe Inc.
- **Venue**: arXiv preprint; source uses ICML 2026 style
- **Keywords**: LLM agent RL, entropy regularization, sparse reward, multi-turn agents, PPO, GRPO, ScienceWorld, ALFWorld
- **One-liner**: 用历史熵走廊稳定多轮智能体RL。

## 精华提炼

### 管线/方法
- 论文研究对象是多轮 LLM agent 强化学习：同一个策略模型跨 30+ turn 与环境交互，只有终局任务成功/失败奖励。
- 作者把核心失败模式命名为 exploration-exploitation cascade failure：早期 turn 的熵失控或错误收敛会通过上下文传到后续 turn。
- 关键诊断是共享参数耦合：同一套 policy parameters 同时控制早期探索和后期执行，因此简单给早期/后期设置不同 entropy weight 不能真正解耦。
- 基础任务形式是轨迹 `tau=(s_0,a_0,r_0,...,s_T,a_T,r_T)`，中间奖励为 0，总回报 `R(tau)=r_T`。
- 基础 policy loss 可以接 PPO clipped surrogate 或 GRPO group-relative advantage，EPO 被定义成 on-policy RL 的附加正则框架。
- 第一步是计算多轮 policy loss `L^MT(theta)`，在所有 turn 上累积 PPO/GRPO 的策略更新。
- 第二步是计算 trajectory-level entropy：对 batch 中每条轨迹、每个 turn、每个 token 的 token entropy 求平均。
- 熵项公式是 `L^H = 1/B sum_j 1/T sum_t 1/|tau_j,t| sum_i H_j,t,i`。
- token entropy 来自词表分布：`H=-sum_v p(v|context) log p(v|context)`。
- 第三步是维护历史熵窗口 `W_k={H_0,...,H_{k-1}}`，用过去 RL step 的平均 token entropy 作为动态参考。
- 历史参考是累计均值 `bar H^{W_k}=1/k sum_{m=0}^{k-1} bar H_m`；作者明确选择 cumulative window 而非 sliding window。
- 第四步是构造 entropy corridor，对当前 token entropy 低于 `kappa_l * historical_avg` 或高于 `kappa_r * historical_avg` 的部分施加 ReLU penalty。
- penalty 形式是 `[kappa_l H_ref - H]_{+} + [H - kappa_r H_ref]_{+}`。
- smoothing loss 再按 batch、turn、token 求平均，并乘 penalty weight `alpha`。
- 完整损失是 `L^EPO = L^MT - lambda [ L^H - beta_k L^smooth ]`。
- 直观上，`L^H` 提供探索梯度，`L^smooth` 防止熵大幅偏离历史走廊，二者被显式 decouple。
- 动态系数 `beta_k = 1 + exp(-gamma * k / k_mid)`，其中 `k_mid=floor(K/2)`。
- 这个 schedule 在训练早期接近 2，随后向 1 衰减，用来早期强约束、后期保留较弱稳定项。
- 伪代码流程是：初始化策略和空 entropy history，逐步收集轨迹、算 advantage、算 `L^MT`、算 `L^H`、用历史均值算 `L^smooth`、更新策略、追加当前 batch entropy。
- 方法声称可兼容 PPO 和 GRPO，实验中实际报告 PPO+EPO 和 GRPO+EPO。

### 关键技术细节
- 基础模型按环境区分：ScienceWorld 使用 Qwen2.5-7B-Instruct，ALFWorld 使用 Qwen2.5-3B-Instruct。
- ScienceWorld 是文本科学实验环境，覆盖 physics、chemistry、life science 等 30+ task types。
- ALFWorld 是文本化 embodied household 环境，论文写有 4,639 task instances，覆盖 Pick & Place、Pick Two & Place、Pick & Cool/Heat、Pick & Clean、Look At Obj & Pick、Examine In Light 等类别。
- 评测分 IID 和 OOD：IID 是 seen task variants/categories，OOD 是 seen categories 下 unseen variants。
- 论文用两个 success 指标：`Succ.*` 是三 seed 最大 success rate 平均，`overline{Succ.}` 是收敛后平均 success rate。
- ALFWorld 的 `overline{Succ.}` 从 step 125 起平均。
- ScienceWorld 的收敛区间按 run 手动选取：GRPO baseline 为 70-120、90-120、90-120；GRPO+EPO 为 60-80、80-120、25-120。
- ScienceWorld 的 PPO baseline 区间为 100-120、105-120、105-120；PPO+EPO 为 70-120、60-120、60-120。
- 训练曲线使用 wandb 默认 running average，窗口大小 10；可视化方差缩放 0.8。
- 主文写 ScienceWorld 训练 120 RL steps、ALFWorld 150 steps；附录超参表写 ScienceWorld `K=125`，这是一个需要注意的不一致。
- ScienceWorld 超参表：max prompt length 2048，max response length 512，test frequency 5 steps。
- ScienceWorld optimizer 是 AdamW，cosine LR scheduler，`num_cycles=0.5`，warmup/min LR ratio 为 0.1/0.2。
- ScienceWorld 学习率：PPO/PPO+EPO 为 `3e-6`，GRPO/GRPO+EPO 为 `5e-6`。
- ScienceWorld batch：PPO mini-batch 64、micro-batch 8、critic micro-batch 4；GRPO mini-batch 128、micro-batch 16、无 critic。
- ALFWorld 超参表：`K=150`，max prompt 2048，max response 512，test frequency 5。
- ALFWorld 学习率全部 `5e-6`，AdamW + cosine schedule。
- ALFWorld PPO batch：mini-batch 256，micro-batch 32，critic micro-batch 16；GRPO 列在表中未给 batch 值。
- EPO 共同超参：`lambda=0.001`，`gamma=3.0`。
- 实验表给的 corridor 是 `kappa_l=0.0, kappa_r=2.0`；理论 appendix 的 practical guidance 又说 `0.5,1.5` 是 robust default，这也是一个不一致点。
- `lambda_k` 在 ScienceWorld 是 0.05，在 ALFWorld 是 0.1；论文没有把这个符号和主文损失里的 `lambda`/`beta` 解释得非常清楚。
- 训练硬件是单节点 8 张 NVIDIA H100 或 A100。
- 训练时间：ScienceWorld PPO 约 H100 23h / A100 31h，GRPO 约 H100 16h / A100 20h。
- ALFWorld PPO 约 H100 23h / A100 31h，GRPO 约 H100 18h / A100 25h。
- 作者称 EPO 相对对应 PPO/GRPO baseline 几乎没有额外训练时间开销。
- 附录提供 ALFWorld 和 ScienceWorld 的 system prompts，包括是否带历史 observation/action 的两版 prompt。
- 作者声明 LLM 只用于语言润色和少量 boilerplate coding，不用于实验结果、标注或方法决策。

### 核心结果
- ScienceWorld PPO baseline：IID `Succ.*=64.6`、`overline{Succ.}=38.4`；OOD `Succ.*=58.3`、`overline{Succ.}=39.1`。
- ScienceWorld PPO+EPO：IID `100.0/96.8`；OOD `100.0/96.2`。
- ScienceWorld PPO+EPO 相对 PPO：IID `Succ.* +54.8%`、IID avg `+152.1%`；OOD `Succ.* +71.5%`、OOD avg `+146.0%`。
- ScienceWorld GRPO baseline：IID `93.8/81.6`；OOD `91.7/80.9`。
- ScienceWorld GRPO+EPO：IID `95.8/83.8`；OOD `95.8/81.3`。
- ScienceWorld GRPO+EPO 增益较小：IID avg `+2.7%`，OOD avg `+0.5%`，说明 GRPO 自带 group-relative advantage 已较稳定。
- ALFWorld PPO baseline：IID `95.8/72.3`；OOD `87.5/70.9`。
- ALFWorld PPO+EPO：IID `85.4/73.4`；OOD `91.7/74.3`。
- ALFWorld PPO+EPO 的 `Succ.*` IID 反而下降 `-10.9%`，但收敛均值和 OOD 均提升，这是一个非单调结果。
- ALFWorld GRPO baseline：IID `87.5/63.3`；OOD `83.3/63.5`。
- ALFWorld GRPO+EPO：IID `91.7/75.8`；OOD `89.6/75.4`。
- ALFWorld GRPO+EPO：IID avg `+19.8%`，OOD avg `+18.7%`。
- 与非自实现 baseline 对比，ScienceWorld 上 PPO+EPO 的 IID/OOD 均明显高于 ReAct、AgentGym、SFT、GiGPO、RLVMR。
- ALFWorld 上 RLVMR 的 `Succ.*` 仍很强：IID 91.4、OOD 91.8；GRPO+EPO 的 IID `Succ.*=91.7` 接近，OOD `89.6` 略低。
- 训练曲线结论：ScienceWorld PPO+EPO 训练 reward 约 15 vs PPO 约 8，且更平滑。
- 论文称 ScienceWorld EPO variants 在 40 steps 内达到 IID/OOD `>0.8`，baseline 100 steps 后仍难超过 0.4。
- ALFWorld OOD 上 baseline 经常跌破 0.2，而 EPO variants 维持在 0.4 以上。
- 消融：去掉 smoothing regularizer 的 EPO-Base 在 ScienceWorld 前 40 step 几乎不学，成功率平台约 0.5-0.6。
- 完整 EPO 在 ScienceWorld step 20 左右已有明显学习，成功率可达 0.8-1.0，作者称 relative improvement 50-60%。
- 附录消融：GRPO+EPO-Base 在 ScienceWorld reward 到 step 40 仍约 2，GRPO+EPO 可到 7-8 reward 和 0.7-0.85 success。
- ALFWorld 消融显示 smoothing 更多提升收敛速度：PPO variants 最终约 12.5 reward、0.8 success，但 PPO+EPO 快约 20 episodes。
- 动态 `beta_k` 消融：常数 beta 在 ALFWorld 最终可相当，但前 40-60 episodes 更慢且方差更大。
- 模型研究：PPO+EPO-Decay 试图早期高熵、后期低熵，但在 ScienceWorld 三个指标都低于 uniform EPO-Base。
- 反直觉点：按 turn 衰减 entropy weight 并不能解决多轮探索-利用，反而会过早压低 late-step entropy，降低恢复能力。
- 与 Cheng et al. 的 entropy-based advantage shaping 对比：EA plateau 在 0.5-0.6，而 PPO+EPO 接近 1.0 success。
- 作者解释为 EA 用 detached entropy 做 advantage bonus，缺少直接 entropy gradient，且可能扰乱 credit assignment。
- 理论结果：EPO 被解释为 entropy corridor violation 的 exact penalty，`rho_k=lambda beta_k alpha`。
- 理论结果：在 corridor 约束下 cumulative entropy deviation 是 `O(T)`，标准 entropy regularization 最坏是 `O(T^2)`。
- 作者给出直观量化：T=10 时 worst-case ratio 5.5x，T=20 时 10.5x。
- 性能界包含 optimization error、entropy bias、stability cost；若最优策略具有 moderate entropy，则 stability cost 为 0。
- 这些理论都依赖 corridor、regularity、`alpha beta_max < 1` 等假设，不能等同于无条件保证。

### 明确局限
- 最大局限是只在两个文本环境 ScienceWorld/ALFWorld 上验证，没有真机、浏览器、GUI 或真实工具执行环境。
- 虽然文中引用 coding/computer-use/web-search agent 场景，但实验没有覆盖这些更开放的 agent setting。
- 没有 vision-language agent 实验；作者自己在 future work 中说 VLM agent 的视觉/文本 entropy 交互可能不同。
- 作者自己承认未利用 memory systems；当前历史信息只用来做 entropy regularization，不存储/检索成功轨迹。
- 没有独立第三方复现；结果由作者自实现 PPO、GRPO、EPO 得到。
- ReAct、AgentGym、SFT、GiGPO、RLVMR 等 baseline 的结果来自 RLVMR 论文，不是同一代码栈重跑。
- ScienceWorld 的 convergence window 是按 run 手动选定，虽然合理但会引入分析自由度。
- `Succ.*` 最大成功率指标容易受峰值和随机性影响；`overline{Succ.}` 更可靠，但同样依赖收敛区间定义。
- 只使用 Qwen2.5-7B 和 Qwen2.5-3B，没有系统探索模型规模迁移。
- ScienceWorld step 数在主文和附录表之间有 120/125 不一致。
- corridor 超参在理论建议和实验表之间有 0.5/1.5 vs 0.0/2.0 不一致。
- `lambda_k` 的角色在主文损失定义中不够清楚，容易影响复现者实现。
- 理论部分的 `O(T)` vs `O(T^2)` 是在 corridor 和 worst-case drift 假设下成立，实验证据主要是曲线和消融，不是直接测量所有中间机制。
- 论文声称 EPO 几乎无额外开销，但没有细分 entropy history、smoothing penalty 的计算/通信开销。
- 对 0616-fex 如果关注多轮 agent exploration failure，这篇是强 prior；如果关注 memory、dense process reward、tool correctness 或 real-world agent 评测，则它更像 entropy-control baseline。

### 跟我们的关系
- 对 `0616-fex`，这篇论文最重要的 prior claim 是：多轮 sparse-reward agent 的失败不只是 reward sparsity/credit assignment，也可能是 entropy dynamics 被共享参数耦合放大。
- 如果我们的方案也做“多轮 LLM agent + PPO/GRPO + entropy/exploration control”，那与 EPO 直接撞车。
- 具体撞车点包括：历史平均 entropy、entropy corridor、直接 entropy loss、dynamic smoothing coefficient、ScienceWorld/ALFWorld 上的 PPO/GRPO agent training。
- 如果我们的 claim 是“按 turn 调不同 entropy weight 可以解决早期探索/后期利用”，这篇提供了反证实验：PPO+EPO-Decay 反而更差。
- 如果我们的 claim 是“把 entropy 加进 advantage 就够了”，这篇用 Cheng et al. EA 对比指出 advantage shaping 可能 plateau 在 0.5-0.6。
- 可用工具/方法：把 `L^EPO` 当作强 baseline；记录 early/late token entropy；画 entropy-reward joint curve；做 EPO-Base、constant beta、turn-wise decay、EA shaping 四个消融。
- 可复用 benchmark：ScienceWorld 的 IID/OOD、ALFWorld 的 IID/OOD，以及 convergence-window average success。
- 差异化方向一：引入 memory/replay/success trajectory retrieval，因为作者明确没有做 memory-augmented learning。
- 差异化方向二：把 entropy corridor 扩展到 VLM/GUI/web/tool-use agent，验证视觉 observation 和 tool-call action entropy 是否有不同 failure mode。
- 差异化方向三：把生成者和验证者分离，用独立 evaluator 或 process verifier 决定轨迹质量，而不是只靠终局 reward 和 self-run curves。
- 差异化方向四：做 claim-level diagnostics，例如直接测 importance weight variance、trajectory diversity、early-turn action diversity，而不是只展示 success/reward 曲线。
- 引用建议：应引用。它是 2025 年 agent RL entropy-control 的直接相关工作，并且已有 2026 反引文沿着 adaptive entropy/exploration optimization 继续扩展。

## Read by: 0616-fex (2026-06-16)

本次从 `0616-fex` 视角看，EPO 是必须纳入的强相关 prior。它已经把“多轮 LLM agent 在 terminal sparse reward 下为何 PPO/GRPO 不稳”具体落到 entropy dynamics，并给出了可实现的 loss：trajectory-level entropy、历史平均 entropy corridor、dynamic smoothing weight。若我们的方案也想用熵、探索、PPO/GRPO 后训练来解决多轮 agent failure，就不能只声称“我们稳定探索”，需要明确和 EPO 的差别。

最容易撞车的 claim 是：用历史统计量约束当前策略熵、用动态 schedule 平衡探索和稳定、在 ScienceWorld/ALFWorld 证明 sparse-reward agent RL 能更快收敛。这些都已被 EPO 覆盖。可用的差异化路线是把 EPO 当 baseline，然后加入它缺失的部分：memory/retrieval of successful trajectories、independent verifier/process reward、VLM/GUI/tool-use 环境、或更直接的 failure diagnostics。下一轮检索应围绕 adaptive entropy modulation、agentic exploration-aware policy optimization、checklist/process rewards、trajectory diversity diagnostics 和 memory-augmented sparse-reward agent RL。
