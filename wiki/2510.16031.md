# Storm250-L2: A Storm-Centric 250 m NEXRAD Level-II Dataset for High-Resolution ML Nowcasting (2510.16031)

- **Year**: 2025.10
- **Authors**: Andy Shi (单作者, 无机构 affiliation, gmail 联系邮箱)
- **Venue**: arXiv preprint (未投会议/期刊迹象; 全文仅 2 节, 无实验)
- **Keywords**: storm-centric dataset, NEXRAD Level-II, super-resolution reflectivity (250 m), GridRad-Severe storm tracks, radar nowcasting, HDF5 event packaging, polar geometry, provenance manifest
- **One-liner**: 用 GridRad-Severe 风暴轨迹在 NEXRAD Level-II 单站原生极坐标数据上裁剪 250 m 分辨率风暴事件包, 发布 reflectivity-only 的 ML-ready 数据集 (alpha 版, 数据尚未上线)。

## 精华提炼

### 管线/方法

端到端管线 (论文 Fig. 4, 5 步):

1. **事件源**: GridRad-Severe (GR-S, Murphy et al. 2023, MWR) 提供风暴轨迹作为时空 scaffold; NEXRAD Level-II 原始数据从 AWS Open Data registry 拉取 (逐 volume 记录 S3 key)。
2. **事件筛选**: 一条 GR-S track 成为合格事件需同时满足: (a) 全程被**单个** NEXRAD 站观测; (b) 所有 track 点距该站 ≤ 250 km; (c) track 连续, ≥ ~60 个采样 (约 1 小时), 内部 gap ≤ 6 小时。
3. **时间对齐**: 对每个 GR-S 时间戳, 在 ±29 s 容差内匹配最近的 NEXRAD volume; 保留雷达原生 4–6 min 扫描节奏, **不做时间插值**。事件的 T = 成功匹配的 volume 数 (典型 10–20)。
4. **风暴中心裁剪** (核心贡献):
   - 伪合成反射率: 每个 volume 在每个 range gate 上取所有仰角的最大反射率 (column-max composite);
   - 风暴识别: 将 composite 投到 250 m 网格, 对 ≥ 20 dBZ 区域做 connected-component labeling, 用 GR-S 中心点选出主 blob (Fig. 2);
   - 窗口优化: 对整个生命周期求单一固定地理窗口, 最小包络所有时间步的 blob 边界 (以首末时刻覆盖为优化目标, Fig. 3), 外加 5 km buffer;
   - 原生坐标裁剪: 在**极坐标系内**直接裁剪, 不做笛卡尔重采样, 保留原生 250 m 径向分辨率。
5. **序列化**: HDF5 (GZIP), 4D 张量 (T, H, W, C), H = 方位射线数, W = range gates, C = 仰角数 (伪合成产品 C=1); 伴随 context 表 (独立 HDF5 + JSON schema) 逐时间步携带 GR-S 原始字段 (风暴位置、运动矢量、echo-top 高度、severe weather 标签); 全数据集生成 SHA-256 manifest + summary catalog。

### 关键技术细节

- **分辨率论证**: 现有公共数据集 1–2 km (SEVIR 1/2 km, HKO-7 ~1 km, GridRad-Severe ~2 km 水平) 会抹平亚公里尺度结构。引 Mahale et al. 2012 (WAF): 96–250 m CASA X-band 侦测到 1 km WSR-88D 漏掉的 QLCS 内多个致灾 mesovortex; 引日本 XRAIN (~250 m / 1 min) 城市 X-band 网络对短时洪水更早告警。
- **文件命名**: 站点_风暴ID_变量_T步数_HxWxC_起止时间, 如 `KTLX_12345_reflectivity_T12_360x400x1ch_20200504T220000Z_20200504T234500Z.h5` — 自描述文件名。
- **Provenance**: 每个 volume 记录精确 AWS S3 源文件 key; SHA-256 完整性 manifest; 声称 pipeline 可复现。
- **规模**: 仅说 "thousands of storm events across CONUS" — **无精确事件数、无年份范围、无存储体量、无 train/test split**。
- **发布状态**: "will be hosted on Zenodo, packaged by year and radar site" — 撰稿时数据**尚未上线**, 论文中无 DOI、无代码仓库链接。

### 核心结果

- **没有任何实验结果**。无 baseline 模型训练、无 nowcasting 指标、无与 SEVIR/HKO-7 训练模型的对比、无数据集统计表。全部"结果"即数据集设计本身与 4 张示意图 (粗细分辨率对比图、blob 识别图、窗口优化图、pipeline 图)。
- 论文自称贡献 3 条: (1) storm-centric 250 m 单站雷达数据集; (2) 可复现的 track→匹配→裁剪→序列化 pipeline (含 metadata + checksum); (3) ML-ready 封装 (HDF5 + JSON schema)。

### 明确局限

作者自陈 (alpha release, 两处重复披露):
1. 仅反射率, 无径向速度、无双偏振变量;
2. 无高级雷达 QC (杂波、衰减、亮带均未处理);
3. 事件覆盖受限于 CONUS NEXRAD + GR-S catalog 的定义。

我们读出的额外局限:
- **无环境场**: 纯雷达单模态, 无任何 NWP/探空/近风暴环境信息;
- **无 benchmark 任务**: 无 canonical tasks/splits/metrics/baselines, 不构成可评测基准, 只是数据包;
- **数据不可得**: Zenodo 未上线、无代码链接, "可复现"目前只是声明;
- **无数据集统计**: 事件数/年份/体量全部缺失, 审稿角度看远未达 ESSD/NeurIPS D&B 门槛;
- **单站假设的代价**: 要求全程单站 ≤250 km 会系统性排除长寿命/长路径风暴 (恰是最高影响事件), 论文未讨论此选择偏差;
- **伪合成 ≠ 标准 composite**: column-max over tilts 在原生极坐标下的几何畸变 (波束高度随距离抬升) 未讨论。

对 0713-scs-benchmark (强对流环境–雷达多模态开放基准):

- **不撞车, 反而是 gap 证据**: Storm250-L2 是 2025 年最新的 storm-centric 雷达数据集, 但**依旧是雷达 reflectivity 单模态、零环境场** — 直接支持我们 "现有数据集没有把环境场及其时空演变当一等公民" 的核心 gap 论述。related work 必引 (定位为 preprint/alpha, 与 SEVIR/GR-S 并列时注明其未发布状态)。
- **claim 级差异**: 它攻 "分辨率 gap" (1–2 km → 250 m, 单站极坐标), 我们攻 "模态 gap" (环境场长时窗 + 雷达 + 事件标签)。两个正交的批评维度, 互不侵蚀新颖性。
- **可借鉴的设计模式** (工程价值高于科学价值):
  1. 用已有风暴产品 (GR-S track) 做事件 scaffold + 显式合格性标准 (单站/距离/时长/gap 上限) — 我们的 Storm Events 聚类事件定义可参照这种 "criteria 表" 写法;
  2. 整生命周期单一固定窗口 (min enclosing box + buffer), 避免 moving window 的插值伪影 — 与我们 "以事件聚类区域为单位" 的设计同构;
  3. provenance 粒度: 逐 volume 记录源 S3 key + SHA-256 manifest + summary catalog — 正是 ESSD 式 provenance 的最小可行实现, 值得直接采用;
  4. context 表沿用上游 (GR-S) 原始 schema 保证互操作 — 我们封装 Storm Events/HRRR 字段时同理。
- **反面教训**: 它演示了这个 topic 的失败形态 — 无统计表、无 baseline 任务、数据未上线就发 preprint, 结果是零引用、C 级可信度。我们的三层交付物 (源数据/训练 ready/工作流代码) + canonical tasks 恰好是补齐这些缺口的路线。
- **潜在复用**: 若我们的基准未来需要亚公里雷达支路 (MRMS 之外), 其 "GR-S track → 单站 Level-II 匹配" 方法可作为高分辨率扩展模块的参考; 但当前 topic 用 MRMS 即可, 不依赖它。
