# PDEformer: Towards a Foundation Model for One-Dimensional Partial Differential Equations (2402.12652)

- **Year**: 2024.02
- **Authors**: Zhanhong Ye (PKU), Xiang Huang (Huawei), Leheng Chen (PKU), Hongsheng Liu (Huawei), Zidong Wang (Huawei), Bin Dong (PKU, New Cornerstone)
- **Venue**: ICLR 2024 Workshop on AI4DifferentialEquations in Science
- **Keywords**: foundation model, PDE solver, computational graph, graph Transformer, implicit neural representation, neural operator, zero-shot, fine-tuning, inverse problem
- **One-liner**: 用计算图表示 PDE 符号形式 + Graph Transformer 编码 + Poly-INR 解码，预训练后 zero-shot 超越专家模型，fine-tune 后 SOTA。

## 精华提炼

### 管线/方法

完整 pipeline：**PDE 符号形式 + 数值信息 → 计算图(DAG) → Graph Transformer(Graphormer) → 隐变量 μ → Poly-INR with Hypernets → mesh-free 预测解 û(t,x)**

1. **Graph Construction（计算图构建）**：
   - 将 PDE 的符号形式表达为有向无环图（DAG），节点类型包括：UF（未知场变量）、SC（标量系数）、IC（初始条件）、微分算子（dt, dx）、代数算子（+, ×, -, ², =0）
   - 节点特征向量：SC 节点的特征为系数值重复填充；IC 节点连接 N=16 个 patch 节点（受 ViT 启发），每个 patch 对应初始条件 g(x) 的 16 个网格点值；其余节点特征为零向量
   - 额外引入 L=8 个 latent modulation 节点（m₁...m₈），连接到 UF 节点，用于后续解码
   - 关键设计：DAG 表示与符号选择和加法/乘法顺序无关，u_t + cu_x = 0 和 βv_x + v_t = 0 映射到同一个 DAG

2. **Graph Encoding（图编码）**：
   - 使用适配版 Graphormer（Graph Transformer），9 层，embed_dim=512，32 heads
   - 初始嵌入 = 节点类型嵌入 + 特征编码(3层MLP) + 入度嵌入 + 出度嵌入
   - 注意力偏置包含最短路径编码和连通性 mask（断开节点间 attention mask 为 -∞，这是对原始 Graphormer 的改进）
   - 无 edge encoding（因为边是同质的），无 [VNode]
   - 输出：每个 m_ℓ 节点对应的嵌入向量 μ^ℓ ∈ R^512

3. **Solution Decoding（解解码）**：
   - Poly-INR：8 个隐藏层，每层 256 神经元，输入 (t,x)，输出 û(t,x)
   - 每层有独立的 scale/shift hypernet（3层MLP，256 hidden），将 μ^ℓ 映射为调制参数 s_ℓ^scale 和 s_ℓ^shift
   - h₀ = 1，逐层计算：g_ℓ = W_ℓ^in [t;x] + b_ℓ^in，q_ℓ = s_ℓ^scale ⊙ (W_ℓ^h (h_{ℓ-1} ⊙ g_ℓ) + b_ℓ^h) + s_ℓ^shift，h_ℓ = σ(q_ℓ)
   - 激活函数 σ：Leaky-ReLU (slope=0.2) + clip to [-256,256]
   - 最终输出：û(t,x) = W^Last h_L + b^Last
   - 与原 Poly-INR 的区别：原版 hypernet 生成 W_ℓ^in 和 b_ℓ^in（精度更高但训练慢），PDEformer 改为生成 feature modulation（效率优先）

4. **预训练**：
   - 数据集：500k 条 PDE 解，形式为 u_t + f₀(u) + f₁(u)_x - νu_xx = 0，f_i(u) = Σ_{k=0}³ c_ik u^k
   - c_ik ~ U([-3,3])，50%概率置零；log ν ~ U([log 10⁻³, log 1])；线性通量时 50%概率 ν=0
   - 初始条件 g(x)：叠加正弦波 + 绝对值函数 + 窗口截断
   - 数值求解器：Dedalus v3（谱方法），256 空间网格点，δt=4×10⁻⁴
   - 训练：每样本随机采 8192 个时空坐标点，nRMSE loss，batch_size=80，Adam lr=3×10⁻⁴，1000 epochs
   - lr schedule：在 40%/60%/80% epoch 时减半，前 10 epoch warmup
   - 数据增强：x 轴随机平移（利用周期性）
   - 计算：8 NPU (Ascend 910A)，MindSpore 2.0，约 79 小时

5. **Fine-tuning**：
   - 在目标 PDE 数据集上（如 PDEBench Burgers/Advection/Reaction-Diffusion 各 9k 样本）对预训练模型做少量迭代 fine-tune
   - 实验中使用 100-9k 样本 fine-tune

6. **逆问题**：
   - 给定带噪观测解，用粒子群优化（PSO）搜索最优 PDE 系数
   - 每次迭代用 PDEformer 前向预测解，计算与观测解的 relative L² error 作为 fitness

### 关键技术细节

- **总参数量**：19.21M（Graphormer ~14M + Poly-INR + Hypernets ~5M）
- **Graphormer**：9 layers, d_e=512, 32 heads, FFN width=512, pre-LN
- **Poly-INR**：L=8 hidden layers, dim_hidden=256, dim_in=2, dim_out=1
- **Hypernets**：per-layer, 3-layer MLP, hidden=256, not shared across layers
- **初始条件编码**：n_x=256 网格点 → N=16 patches × d_f=16（N=4 或 N=1 精度更差）
- **不同隐变量 vs 共享隐变量**：每层不同 μ^ℓ 比共享单个 μ 性能更好
- **最短路径截断**：>14 的路径截断为 14
- **损失函数**：预训练用 nRMSE，评估用 relative L² error
- **网格无关性**：预训练用 101 时间步，PDEBench 用 256 时间步，mesh-free 预测天然支持
- **对比 baseline 配置**：
  - DeepONet: branch net + trunk net，各 6 层 MLP × 256 hidden，1.65M 参数
  - FNO: modes=12, channels=20, depths=4, 0.92M 参数
  - U-Net: 4 层上下采样 CNN, 13.39M 参数
  - Autoregressive U-Net: ℓ=4 历史帧预测下一帧，rollout K=16

### 核心结果

| 数据集 | DeepONet | FNO | U-Net | PDEformer-FS | PDEformer (zero-shot) | PDEformer-FT |
|--------|----------|-----|-------|-------------|----------------------|-------------|
| Burgers ν=0.1 (ID) | 0.0699 | 0.0155 | 0.1627 | 0.0135 | **0.0103** | **0.0046** |
| Burgers ν=0.01 (ID) | 0.1791 | 0.0445 | 0.2253 | 0.0399 | **0.0309** | **0.0146** |
| Advection β=0.1 (ID) | 0.0186 | 0.0089 | 0.0873 | 0.0124 | 0.0119 | **0.0043** |
| Burgers ν=0.001 (OOD) | 0.2010 | 0.0700 | 0.2431 | 0.0645 | 0.0921 | **0.0295** |
| Advection β=1 (OOD) | 0.0187 | 0.0097 | 0.2655 | 0.0239 | 0.4000 | **0.0075** |
| Reaction-Diffusion (OOD) | 0.0015 | 0.0018 | 0.0126 | 0.0013 | 0.7399 | **0.0009** |

**Surprising 发现**：
1. **Zero-shot 超越专家模型**：在 Burgers ν=0.1 和 0.01 上，预训练 PDEformer 的 zero-shot 推理（0.0103, 0.0309）超过了所有专门训练的 baseline（FNO 0.0155, 0.0445）。作者认为这源于预训练中接触多样 PDE 学到了"可泛化的规律"，而非仅仅是架构优势——因为 PDEformer-FS（from scratch）的结果（0.0135, 0.0399）虽然也 competitive 但不如 zero-shot。
2. **OOD zero-shot 两极分化**：在 Advection β=1 (OOD) 上 zero-shot 极差（0.4000），但在 Reaction-Diffusion (OOD) 上也是 0.7399——说明当前预训练分布覆盖不足以泛化到 equation form 的 OOD。但 fine-tune 后立刻 SOTA。
3. **FNO transfer learning 不如 PDEformer**：预训练 FNO（在 Advection β=0.1 上预训练 9k 样本 1k 迭代）再 fine-tune 到 β=1 (OOD)，仅比 from-scratch FNO 有微弱提升——说明传统 neural operator 的预训练迁移效率远不如 PDEformer 的多 PDE 联合预训练。
4. **Fine-tune 速度极快**：Advection β=1 (OOD) 仅 ~100 次迭代即收敛，而 FNO from-scratch 数千次迭代仍不如。
5. **逆问题对噪声鲁棒**：即使 noise level=0.1（噪声幅值为解的 L^∞ 范数的 10%），大部分 PDE 系数仍能被有效恢复。

### 明确局限

1. **仅限 1D**：所有实验都是一维 PDE，作者承认这是"milestone towards foundation model"，2D/3D 是显然的扩展方向
2. **仅限周期性边界条件**：所有 PDE 都有周期性 BC，虽然计算图可以表示其他 BC（通过引入新节点类型），但未实验验证
3. **预训练 PDE 族有限**：仅覆盖 u_t + f₀(u) + f₁(u)_x - νu_xx = 0 形式（多项式通量 + 扩散），不包含高阶导数、非线性扩散、耦合方程组等
4. **未开源代码**：论文发表于 ICLR 2024 Workshop，使用华为 MindSpore 框架，未见代码 release
5. **逆问题方法简单**：使用 PSO 这种黑箱优化，没有利用 PDEformer 的可微性做 gradient-based inversion
6. **计算成本高**：预训练需要 8 NPU × 79 小时，且 MindSpore 生态限制可复现性
7. **未与 LLM-based 方法深入对比**：仅提及 PITT、PROSE、ICON-LM 等 LLM 方法但未做实验对比
8. **没有理论分析**：纯实验论文，没有关于为什么多 PDE 预训练能超越单 PDE 专家的理论解释
9. **Resolution 限制**：虽然声称 mesh-free，但初始条件仍需 256 个固定网格点输入
10. **Nonlinear conservation laws 未覆盖**：如 Euler 方程、浅水波方程等实际重要的 PDE

### 跟我们的关系

对 **0616-fex** 课题：

- **撞车评估**：不完全撞车。PDEformer 做的是"一个模型解多种 PDE"的 foundation model 路线，而 0616-fex 如果关注的是 expert model 架构创新或 foundation vs expert 的 trade-off 分析，PDEformer 是重要的 baseline 和对比对象。PDEformer 的核心贡献在于计算图表示 + graph Transformer 的组合使 symbolic information 可被模型处理，这个 representation 方法是可复用的。

- **可用工具/方法**：
  1. **计算图表示法**：将 PDE 符号形式转为 DAG 的思路可以直接复用，节点类型可扩展（引入新算子节点如 sin/cos/exp/log/∇² 等）
  2. **Graphormer + INR 架构组合**：作为 strong baseline
  3. **预训练数据生成 pipeline**：Dedalus + 随机系数 + 数据增强（平移）
  4. **Zero-shot vs Fine-tune 评估协议**：ID/OOD 区分、few-shot fine-tune curve

- **差异化方向**：
  1. PDEformer 仅限 1D + 周期性 BC → 可以做 2D/3D 或更复杂 BC
  2. PDEformer 使用闭源框架 → 开源复现本身就是贡献
  3. PDEformer 缺乏理论分析 → 可以做为什么 foundation model works 的理论
  4. PDEformer 的 graph 表示不支持方程组 → 可以扩展

## Read by: 0616-fex (2026-06-24)

PDEformer 是 foundation model for PDE 路线上的一个关键工作（ICLR 2024 Workshop）。与 0616-fex 课题的核心关联在于：

1. **Foundation vs Expert 的 trade-off**：PDEformer 提供了"一个模型解多种 PDE"的实证证据——预训练后的 zero-shot 推理在 in-distribution PDE 上确实能超越专门训练的 expert models（FNO/DeepONet）。这对 0616-fex 如果研究 expert model 的定位和边界有直接参考价值。

2. **计算图表示是核心创新**：将 PDE 符号形式转为 DAG、用 graph Transformer 处理——这种 symbolic + numeric 的统一表示方法可能比 LLM-based 方法（PROSE, PITT, ICON-LM）更 principled。如果 0616-fex 涉及 PDE 表示的改进，这是一个重要的 baseline 思路。

3. **预训练数据多样性是关键**：PDEformer 的预训练集仅覆盖一个 PDE 族（多项式通量 + 扩散），但已经展现出泛化能力。这暗示如果扩大预训练数据多样性，foundation model 的效果可能更好——但也说明当前离"通用 PDE foundation model"还很远。

4. **已知的 gap**：PDEformer 不开源（MindSpore + NPU）、仅 1D、仅周期性 BC、缺乏理论分析。这些都是 0616-fex 可以切入的方向。特别是如果要做 foundation vs expert 的理论分析或更公平的对比实验，PDEformer 的方法和结果可以作为 important reference point。
