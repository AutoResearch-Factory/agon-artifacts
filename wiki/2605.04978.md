# Exhaustive Symbolic Integration: Integration by Differentiation and the Landscape of Symbolic Integrability (2605.04978)

- **Year**: 2026.05
- **Authors**: Harry Desmond, University of Portsmouth
- **Venue**: arXiv preprint; manuscript formatted for Journal of Symbolic Computation
- **Keywords**: exhaustive symbolic integration, symbolic regression, integrability fraction, expression grammar, numerical fingerprinting, Risch algorithm, CAS benchmarking, derivative database
- **One-liner**: 用穷举函数库反向求导量化符号可积性。

## 精华提炼

### 管线/方法
1. 论文提出 Exhaustive Symbolic Integration (ESI)：不是直接对 integrand 做积分，而是先穷举候选原函数，再求导反查。
2. 输入是一组 operator basis、一个 complexity ceiling `k`、以及由表达式树定义的有限函数空间 `F_{<=k}`。
3. 表达式复杂度定义为 expression tree 的节点数；叶子是变量 `x` 或自由参数 `a_i`，内部节点是一元/二元算子。
4. 函数空间由 Exhaustive Symbolic Regression (ESR) 的枚举模块生成；本文只使用 ESR 的 exhaustive function generation，不做回归拟合。
5. ESR 首先生成所有可由 basis 构造、节点数不超过 `k` 的表达式树。
6. 每一层复杂度生成后，用 SymPy simplification 去除显然重复表达式；每个表达式 simplification timeout 是 60 秒。
7. 作者承认 SymPy 去重不完备，因此后面又加 numerical fingerprinting 做第二层去重。
8. 五个 operator bases 被研究：`core_maths`、`core_log_maths`、`ext_maths`、`ext_log_maths`、`trig_maths`。
9. `core_maths` 包含 `x,a`、`inv`、以及 `+,-,*,/,pow`，代表有理函数和幂函数基线。
10. `core_log_maths` 在 `core_maths` 上加 `log`，用于隔离 logarithm 的单独效果。
11. `ext_maths` 在 core 上加 `sqrt,sq,exp`，代表更丰富的 elementary basis。
12. `ext_log_maths` 在 `ext_maths` 上进一步加 `log`，用于研究 exp-log 子系统。
13. `trig_maths` 包含 `sin,cos`，且不是其他 basis 的 superset，用作结构独立对照。
14. 对每个候选原函数 `F(x)`，作者用 SymPy 机械求导得到 `F'(x)`。
15. 若 `hash(F')` 出现在 `hash(F_{<=k})` 中，则认为 `F'` 在同一 grammar 和 complexity ceiling 下有原函数。
16. 核心统计量是 integrability fraction `rho(k)=|{F in F_{<=k}: F' in F_{<=k}}|/|F_{<=k}|`。
17. 换句话说，`rho(k)` 测的是 bounded expression grammar 在“积分”方向上的闭合密度。
18. 这和 Liouville/Risch 理论互补：经典理论回答 elementary antiderivative 是否存在，本文回答在有限 complexity grammar 中这种现象有多常见。
19. Numerical fingerprinting 的流程是：统一参数名后，在随机点上高精度数值求值，再对值向量哈希。
20. 每个表达式在 `N=60` 个随机 `x` 点上求值，`x` 从 `(0.2,5)` 均匀采样。
21. 参数 `a_i` 也固定为从 `(0.5,3.0)` 均匀抽样的确定性随机值，以保证可复现。
22. 数值计算使用 `mpmath` 50 位精度，结果四舍五入到 10 个有效数字。
23. 60 个值组成 tuple 后用 MD5 生成 fingerprint；相同 fingerprint 被视为同一函数。
24. 作者给出典型情况下的 false-positive 概率估算：单点评估约 `1e-10`，60 点约 `1e-600`。
25. 这个估计不是 worst-case proof；若某个病态函数刚好在 60 个点上吻合，仍可能逃过。
26. 该 fingerprint 只能捕捉固定参数赋值下相等的表达式，不能捕捉所有重参数等价。
27. 例如 `a0*x` 与 `a0*a1*x` 是同一参数族，但固定参数值不同，可能不会合并。
28. 作者估计这类重参数漏检影响低于 1%，因此 `rho(k)` 更像保守下界。
29. 分组时，所有具有相同 derivative fingerprint 的原函数形成一个 equivalence class。
30. ESI 作为积分算法时，本质就是在这些 derivative classes 中查找 integrand 的 primitive。

### 关键技术细节
1. 最高复杂度依 basis 而定：`core_maths` 到 `k=10`，多数更丰富 basis 到 `k=8` 或 `k=9`。
2. 文中报告的最高规模从 `core_maths` 的 77,053 个 unique functions 到 `ext_log_maths` `k=9` 的 1,454,666 个 unique functions。
3. CAS 对比表中使用的 `ext_log_maths (k<=8)` 有 222,838 个 deduplicated cumulative functions，49,538 个 tested multi-member classes。
4. `core_maths (k<=10)` 有 77,053 个函数，13,993 个 multi-member classes。
5. `core_log_maths (k<=8)` 有 21,214 个函数，5,669 个 multi-member classes。
6. `ext_maths (k<=9)` 有 628,400 个函数，117,674 个 multi-member classes。
7. `trig_maths (k<=9)` 有 897,293 个函数，51,027 个 multi-member classes。
8. `ext_log_maths k=9` 的 tree generation 约需 20 CPU-hours。
9. 同一设置下，56 million candidate trees 的 symbolic deduplication 需数天、数十核、每核约 20-30 GB 内存。
10. `ext_log_maths k<=8` 在去掉无 `x` 依赖表达式后，从 319,128 个表达式经 fingerprinting 降到 222,838 个 unique hashes，减少约 30%。
11. `trig_maths` 类似去重率只有约 4%，说明 log/Abs 组合引入了更多语义重复。
12. `trig_maths k<=8` 的 fingerprinting 成功 150,388 个函数，另有 78 个 explicit timeouts 和 354 个 fingerprinting failures。
13. `rho(k)` 的误差条使用 binomial counting uncertainty `sqrt(rho(1-rho)/n)`。
14. 作者明确指出该误差不包含 deduplication 不完备带来的系统误差。
15. 对 `~1e5` 函数，单核 differentiation + hashing 大约 10 分钟；hash lookup 本身几乎瞬时。
16. CAS 对比使用六类系统/策略：SymPy、Mathematica、RUBI、FriCAS、Maxima、Giac。
17. SymPy 代表 partial Risch + heuristic fallback。
18. Mathematica 代表商业 heuristic/mixed 系统。
19. RUBI 代表 rule-based integration，大约 6,700 条人工规则。
20. FriCAS 代表最完整的开源 Risch-based 实现之一。
21. 初筛每个策略 180 秒 timeout；stress test 延长到每策略 600 秒。
22. SymPy 尝试 simplification、`conds=none`、positive parameter assumptions、manual mode。
23. Mathematica 初筛使用 `x>0, params>0` assumption，后续对主要失败样本做不同 domain rerun。
24. ESI 与 CAS 的公平对比只在 multi-member equivalence classes 上做，因为这些是 ESI 可提供非平凡 lookup 的地方。

### 核心结果
1. `core_maths` 的 `rho(k)` 从 `k=7` 约 5.5% 下降到 `k=10` 的 3.5%。
2. `ext_maths` 在 `k=5-7` 约 4%，到 `k=9` 降至 2.9%。
3. `trig_maths` 从约 5% 降至 3.7%，没有因为 sin/cos 导数闭环而显著提高。
4. `ext_log_maths` 与非 log basis 明显不同：`k=4` 为 8.1%，`k=6` 峰值 11.6%，`k=9` 降至 8.3%。
5. `core_log_maths` 也显示同样 log-driven peak：`k=6` 峰值 19.2%，`k=8` 仍有 17.4%。
6. `core_log_maths` 在 `k=8` 比 `core_maths` 高约 3.6 倍。
7. `ext_log_maths` 相对 `ext_maths` 在 `k>=5` 的 integrability fraction 约高 2.5 倍。
8. 作者据此判断 logarithm 本身就是增强可积性的主要来源，不需要和 exp 交互才出现。
9. operator decomposition 显示 `log only` subset 最可积：`k=6` peak 达 20.1%。
10. `k=8` 时，`log only` 的 `rho=14.4%`，比 `exp only` 高 3.8 倍。
11. 含任意 log 的函数比不含 log 的函数可积概率高 2.7 倍。
12. 机制解释是 `d log(f)/dx = f'/f` 产生的 rational form 容易落回已有函数空间。
13. 相比之下 `d exp(f)/dx = f' exp(f)` 会引入乘法因子和复杂度膨胀，更容易越过 `k` 上限。
14. sin/cos 虽然互相闭合，但 chain rule 仍引入 `f'`，所以无法像 log 那样提升整体 closure。
15. derivative map 的 equivalence classes 高度 clustered，最大类来自 `1`、`-1`、`1/x` 等 attractor integrands。
16. `ext_log_maths k<=8` 有 203,812 个 derivative clusters，其中 49,538 个是 multi-member。
17. `trig_maths` 有 131,734 个 clusters，其中只有 6,987 个 multi-member。
18. CAS 初筛中，SymPy 解出 `ext_log_maths` 49,538 个 integrands 中的 47,107 个，即 95.1%。
19. SymPy 剩余失败 2,431 个，多为快速返回 unevaluated integral，而非 timeout。
20. Mathematica 解出 47,489 个，即 95.9%，剩余失败 2,049 个。
21. SymPy 与 Mathematica 的失败集交叉后得到 503 个二者都失败的 integrals。
22. 对这 503 个做 stress test 后，270 个被至少一种策略解出，232 个仍抗 SymPy 与 Mathematica。
23. FriCAS 又解出这 232 个中的 218 个，说明大部分失败是实现覆盖缺口，不是 elementary antiderivative 不存在。
24. RUBI 对这 232 个没有解出，符合 rule-table 系统对深层 exp-log tower 覆盖较弱的预期。
25. 最终跨四个 CAS-tested bases，有 3 个 integrals 抗全部六个 CAS engine 的所有测试策略。
26. 第一个 headline integral 是 `((2x+1)e^x+1)/(2 sqrt(x+e^{-x}))`，ESI 给出原函数 `e^x sqrt(x+e^{-x})`。
27. 第二个是 `((2x+1)e^x-1)/(2 sqrt(|x-e^{-x}|))`，原函数 `e^x sqrt(|x-e^{-x}|)`。
28. 第三个是 `x^{x-3/2}(1-x-x log x)/(2 sqrt(|x-x^x|))`，原函数 `sqrt(|x-x^x|)/sqrt(x)`。
29. 这三者共同结构是 square-root algebraic extension 与 `e^x`、`e^{-x}` 或 `x^x` tower 耦合。
30. 作者认为这类 coupling 无法通过简单 substitution 解耦，暴露了当前 Risch 实现的具体缺口。
31. 在 47,107 个 ESI 与 SymPy 都能解的 integrals 上，SymPy 返回更低 node count 的比例是 38.7%。
32. ESI 返回更低 node count 的比例是 34.7%，两者打平 26.6%。
33. 但当 ESI 赢时累计节省 135,036 个 nodes；SymPy 赢时累计节省 32,431 个 nodes，比例为 4.2:1。
34. 这说明 ESI 赢的次数略少，但赢得更大，常给出更紧凑的 factored/nested primitive。
35. 对 RUBI 72,401 个 benchmark integrals，ESI 在当前 basis 与 complexity ceiling 下能解 184 个。
36. RUBI 自己失败的 134 个 integrals 中，122 个可转成 fingerprint；其中 3 个在 ESI derivative database 中出现。
37. 去掉一个 hyperbolic branch artifact 后，仍有两个 robust ESI solves，且 FriCAS 也能解。

### 明确局限
1. 结果强依赖 ESR 的 expression grammar 和 node-count complexity metric。
2. 改变 grammar、改变算子成本、或使用 tree depth/operator count，都会改变 `rho(k)` 的绝对数值和 peak 位置。
3. 作者给出例子：若把每个 `log` 或 `exp` 的复杂度从 1 改为 2，`rho(k)` 会变为约 8-9% 的平坦曲线，`k=6` peak 消失。
4. 因此最稳健的结论不是“峰值在 k=6”，而是“log 提升”和“高复杂度 decline”。
5. Deduplication 不完备是第二个核心局限：SymPy simplification 与 numerical fingerprinting 都只是近似。
6. Numerical fingerprinting 有 false negative：domain failure、division by zero、log negative 等会导致等价表达式未合并。
7. Numerical fingerprinting 也不能识别所有 reparametrization equivalence。
8. 因此 `rho(k)` 可能被低估，特别是参数乘积出现在 derivative 中时。
9. 计算规模仍然有限，最高只到 `k=8-10`；很多实际 CAS 难题需要更高 complexity。
10. CAS benchmark 不是全 basis 全积分的穷尽 stress test；表中若干 entries 明确是不完整或 lower bound。
11. `core_log_maths` 主要用于 log effect 诊断，没有纳入完整六引擎 CAS cascade。
12. Mathematica 是商业系统，完整复现实验需要相应环境。
13. ESI 当前主要是一元表达式上的 closed-form integration lookup，不是通用多变量 PDE/SR solver。
14. 论文不解决“如何在巨大表达式空间中高效搜索目标函数”的 RL/optimization 问题；它是穷举 cataloguing。
15. 对 FEX 而言，本文的 limitation 正是关键差异：FEX 的困难在 controller discoverability 和 coefficient optimization，ESI 只证明某个 grammar 的 landscape 可被离线穷举统计。
