# LLM-Feynman: Leveraging Large Language Models for Universal Scientific Formula and Knowledge Discovery (2503.06512)

- **Year**: 2025.03
- **Authors**: Zhilong Song, Qionghua Zhou, Chunjin Ren, Chongyi Ling, Minggang Ju, Jinlan Wang (Southeast University / Suzhou Laboratory)
- **Venue**: arXiv preprint
- **Keywords**: symbolic regression, LLM, Monte Carlo tree search, domain knowledge, feature engineering, Pareto frontier, self-evaluation, materials science
- **One-liner**: 将 LLM 的领域知识嵌入符号回归流程，结合 MCTS 解释公式含义，在 Feynman 物理公式重发现和材料科学应用上超越 SISSO/PySR。

## 精华提炼

### 管线/方法

LLM-Feynman 框架分三个顺序模块：

**模块 I — 自动数据预处理与特征工程**
- 输入：材料特征 X、目标值 y、特征物理含义和量纲
- 数据预处理：缺失值处理 + 可选归一化
- 特征工程三条路径：
  1. **互信息筛选**：用 Automatminer 生成组分/结构/电子性质特征（如 1200 个），通过 MODNet 的 MI 模块筛选（本文中筛到 45-52 个），去冗余
  2. **LLM 引导特征匹配**：LLM 从 Matminer 库建议物理有意义的描述符
  3. **迭代特征扩展**：公式发现停滞时 LLM 建议新特征
- 所有特征附带 LLM 生成的物理含义和量纲

**模块 II — LLM 符号回归 + 自评价 + 多目标优化**
- LLM 以 Python 函数形式生成 N 个初始公式
- 域知识嵌入：结构化 prompt 包含特征物理含义和量纲
- 自评价：LLM 为每个公式打可解释性分数 S ∈ [0, 1]
- 多目标 loss 函数：L = α·N(E) + β·N(C) + γ·S
  - E = 误差指标（回归：MAE, R²；分类：accuracy, precision, recall, F1, cross-entropy）
  - C = 公式复杂度
  - S = 可解释性评分
  - α, β, γ 默认均为 1
- 迭代循环：每轮选 loss 最低的 30 个公式，将其数学表达 + loss + 数据放入 prompt，LLM 生成 J=10 个新公式
- 默认 500 次迭代
- 最终所有公式做 Pareto frontier 分析（准确度 vs 复杂度）

**模块 III — MCTS 公式解释**
- 每个树节点 = LLM 生成的一个解释性假说
- UCB 公式：UCB = Q + c·√(ln N_parent / N_node)
- 四阶段迭代：Selection（选最高 UCB 节点）→ Expansion（LLM 生成细化解释）→ Self-evaluation（LLM 打分：清晰度、科学相关性、连贯性）→ Backpropagation（更新祖先节点分数）
- 输出：每个特征对目标的影响 + 公式预测目标的理论解释

### 关键技术细节

- **LLM 选择**：测试了 Falcon-Mamba-7B [43]、ChemLLM-20B [44]、LLaMA3-8B [45]
  - Falcon-Mamba-7B 表现差，后续排除
  - ChemLLM-20B 在所有配置中一致优于 LLaMA3-8B（归因于其化学/材料领域微调）
- **推理设置**：Transformers + FlashAttention + Accelerate，temperature=0.7, top-k=50, top-p=0.9
- **硬件**：NVIDIA A800 GPU（80GB VRAM），1TB 内存
- **特征工程工具**：Automatminer + Matminer + MODNet
- **基线**：SISSO [38,39]、PySR [40]、AI-Feynman [46]
- **数据集**：
  - SAC 吸附能（DFT 数据，6 个特征，4 种吸附物）
  - 2D 材料可合成性（151 正例 + 150 负例，52 个特征，8:2 分）
  - 钙钛矿可合成性（完美分类）
  - 固态电解质离子电导率（659 个数据点，50 个组分特征，9:1 分）
  - 2D 材料 GW 带隙（551 个数据点，45 个结构特征，9:1 分）
  - Feynman 物理公式（100 个简单 + 20 个复杂）

### 核心结果

| 任务 | 指标 | LLM-Feynman | 最佳基线 |
|------|------|-------------|---------|
| SAC 吸附能 ablation | 1-R², MAE | 最低误差 + 最低复杂度 | SISSO/PySR 误差更高或复杂度更高 |
| Feynman 简单公式 (noise=0) | 成功率 | 100% | AI-Feynman 100% |
| Feynman 简单公式 (noise=10⁻³) | 成功率 | 100% | AI-Feynman 85% |
| Feynman 简单公式 (noise=10⁻²) | 成功率 | 86% | AI-Feynman 67% |
| Feynman 复杂公式 (noise=10⁻²) | 成功率 | **90%** | AI-Feynman 55% |
| 2D 材料可合成性 | Acc/Prec/Rec/F1 | 0.93/0.88/1.00/0.94 | E_hull 基线 0.60/0.61/0.53/0.57 |
| 钙钛矿可合成性 | Acc/Prec/Rec/F1 | **1.00/1.00/1.00/1.00** | — |
| 固态电解质电导率 | R²/MAE | 0.855/0.673 | — |
| GW 带隙 | R²/MAE(eV) | 0.80/0.429 | — |

**Surprising/反直觉结果**：
- 纯数据驱动 LLM-SR（无域知识）已略优于 SISSO/PySR 的准确度，但复杂度更高——说明 LLM 倾向于生成过于复杂的公式
- 加入域知识后复杂度显著降低而准确度不降——域知识起到了正则化的作用
- 自评价进一步提升准确度但不增加复杂度——LLM 的科学知识可以作为有效的反馈信号
- 特征工程对比：Automatminer+MI (R²=0.855) >> LLM-suggest-iterative (0.709) >> LLM-suggest (0.687)——LLM 从零生成特征的能力仍远不如传统自动化工具

### 明确局限

**作者自述**：
- Token 长度限制阻碍大数据集（>10⁴样本）的应用
- 未来需分层数据处理、架构增强、RLHF 来优化大输入

**读出的局限**：
- 仅测试 7B-20B 的小型开源 LLM，未对比 GPT-4/Claude 等强 LLM，无法判断方法上限
- MCTS 解释模块的评价主观性强（LLM 自打分），缺乏人类专家评价对照
- 所有应用集中在材料科学，未在更广泛的物理/化学/生物场景验证
- 500 次迭代 × 每轮 10 个公式 = 5000 次 LLM 调用，计算成本未报告
- 2D 材料可合成性数据集仅 301 个样本，统计效力有限
- 未与近期 LLM-SR 工作（如 LLM-SR [Shojaee 2024]、SPL [Tenachi 2023]）系统对比
- Pareto frontier 的"最优"公式选择缺乏严格标准，依赖人工判断

## Read by: llm-mcts-sr (2026-06-05)

本文是 LLM+MCTS+SR 课题的核心参考文献之一。其框架将 LLM 的域知识嵌入到符号回归中，并用 MCTS 做公式解释，展示了三者结合的可行性。

**撞车评估**：部分撞车。本文的核心创新是 (1) LLM 域知识嵌入 SR prompt + 自评价 loss、(2) MCTS 用于公式解释。如果我们的方向是将 MCTS 用于 SR *搜索*过程本身（如将公式空间建模为树，用 LLM 做 expansion/rollout，用 UCB 选下一步搜索方向），则与本文方法有本质区别——本文的 MCTS 仅用于后处理解释，不参与公式搜索。

**可用工具/方法**：
- 多目标 loss 设计（准确度+复杂度+可解释性）可直接复用
- 域知识 prompt 模板设计思路
- Feynman 物理公式数据集作为标准 benchmark
- PU learning + MAML 处理不平衡数据的方法

**差异化路线**：
1. MCTS 深度耦合搜索：将 MCTS 的 selection/expansion/simulation/backpropagation 全部用于公式*生成*过程，而非仅用于解释
2. 更强 LLM：本文仅用 7B-20B，用 GPT-4/Claude 级别 LLM 可能获得质的提升
3. 搜索空间设计：不是让 LLM 自由生成完整公式，而是用 MCTS 控制公式树的逐步构建
4. 泛化到 PDE/ODE 发现：本文仅做代数公式，可扩展到微分方程
