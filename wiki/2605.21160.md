# Learning First Integrals via Backward-Generated Data and Guided Reinforcement Learning (2605.21160)

- **Year**: 2026.05
- **Authors**: Jingfeng Zhong, Zhengxiang Liu, Zhijie Wang, Shuai Li (Shanghai Jiao Tong University)
- **Venue**: arXiv preprint
- **Keywords**: First Integrals, Scientific Knowledge Discovery, LLMs, Symbolic Reasoning, Data Generation, Dynamical Systems, Backward Generation, Reinforcement Learning
- **One-liner**: 用 Backward Generation 从采样积分反推微分方程构建训练数据，配合 Levenshtein Distance 奖励的 GRPO 强化学习，让 1.5B 小模型在首次积分发现上超越 Mathematica 和 DeepSeek-V3。

## 精华提炼

### 管线/方法
完整 pipeline 三步走：
1. **数据生成（Data Generation）**：
   - **Forward Generation**：用现有工具（InSyDE）对符合条件的微分方程求解，产生少量 (DE, first integral) 对
   - **Backward Generation（核心创新）**：采样 m 个首次积分 V₁...Vₘ 和 n−m 个已知微分方程，通过求解线性符号系统 Jₐ(dxₐ/dt) + J_b(dx_b/dt) + ∂V/∂t = 0 来反推出未知微分方程，确保生成数据的正确性（Proposition 1 证明）。过滤掉退耦合的平凡系统。
   - **Data Synthesis**：用少量已知首次积分 fine-tune 一个生成模型（FIGenerator），让它产生大量结构相似的合成积分，再经 Backward Generation 验证生成 BwdSyn 数据集
   - **Dataset Blending**：将 Forward 生成的真实样本按比例（0.5%~2%）混合到大规模反向生成数据中，作为分布锚点实现 domain adaptation

2. **模型训练（Model Training）**：
   - **SFT**：对 Qwen2.5-Math-1.5B-Instruct 做 LoRA fine-tuning（rank=16, α=32），输入为波兰表示法的微分方程组，输出为首次积分
   - **RL（GRPO）**：从 SFT checkpoint 出发，用 GRPO 算法做强化学习。奖励函数分三档：验证通过 → R_max=100；未通过但合法 → ω·R_L（Levenshtein Distance 指数衰减奖励）；非法表达式 → −k·P 惩罚

3. **推理与验证（Inference & Verification）**：
   - Beam search（bs=5 或 70）生成多个候选首次积分
   - 用 SymPy 符号验证 dV̂/dt ≡ 0，有一个候选通过即算正确

### 关键技术细节
- **模型基座**：Qwen2.5-Math-1.5B-Instruct（仅 1.5B 参数）
- **数据表示**：Polish (prefix) notation，通过 SymPy 处理。例如 `−1/sin(x+y−2)` → `/ -1 sin + + - 0 2 x y`
- **BwdBase 数据集**：410k 样本（400k train / 10k val），积分含 0-6 个算子，70% 变量 {x,y,t} + 30% 单数字整数
- **测试集**：Normal（同分布）和 Hard（含 t 变量 + 非线性算子）
- **Fwd 数据集**：用 InSyDE S3 策略求解 d²x/dt² = (Ax+Bt+C)/(Dx+Et+F) 形式的 rational ODE，产生 7k 样本
- **BwdSyn 数据集**：用 ~2k Fwd 样本训练生成器，产生 350k 合成对
- **SFT 训练配置**：2×RTX 2080，batch size=2 per GPU，gradient accumulation=16，LoRA r=16/α=32/dropout=0.05，1 epoch，lr=2e−4，总训练 ~23h
- **RL 训练配置**：1×RTX 2080，lr=3e−6，β=0.01，ε=0.02，batch size=1，gradient accumulation=8，每 1k 样本 ~4h
- **奖励函数参数**：R_max=100，k_L=0.1，k_S=0.01，LD reward ω=50，Binary reward ω=0
- **RL 数据**：NS 用 5k BwdBase 随机样本，HS 用 1k Hard 分布样本
- **Beam search**：bs=5 或 70，bs=70 时 300s 超时限制

### 核心结果
| 测试集 | SFT-only (bs=70) | +RL(HS)(LD) (bs=70) | Mathematica | DeepSeek-V3.2-Exp (685B) |
|--------|------------------|---------------------|-------------|--------------------------|
| Normal | 72.7 | 74.7 | 74.0 | - |
| Hard   | 62.2 | 63.7 | 23.3 | 42.8 |

关键发现：
- **Backward Generation 有效性**：SFT 后 1.5B 模型在 Normal/Hard 上从 12.9%/2.4% 跃升至 43.2%/26.1%（bs=5），远超未 fine-tune 的 7B 模型
- **LD reward >> Binary reward**：LD 奖励在 Hard 上带来 +7.4%（bs=5），而 Binary reward 几乎无增益；在 bs=70 下 Binary reward 甚至因输出分布坍缩导致性能下降（49.5% vs SFT 72.7%）
- **超越 Mathematica**：Hard 集上 63.7% vs 23.3%，因为 Mathematica 依赖刚性算法规则，对非线性系统泛化差
- **超越 DeepSeek-V3.2-Exp**：1.5B vs 685B，63.7% vs 42.8%，且 token 消耗更少（1611.8 vs 2869.6 per sample）
- **Domain adaptation 有效**：在 Fwd-Eval 上，BwdSyn 训练达到 28.4%，超过 InSyDE-Base (9.8%) 和 InSyDE-S2 (22.6%)；加入 2% Fwd 数据后达到 35.4%
- **RL 修复 SFT 错误**：Case study 中 SFT 模型识别出 log(x−y) 但无法正确处理与 t 的关系（输出 t·log(x−y) 等错误形式），RL(LD) 后成功输出 log(x−y)/t

### 明确局限
- **仅限 2D 一阶无参数 ODE**：作者承认高维和参数化系统需要更大模型或更强的约束解码
- **Polish notation 非规范**：代数等价的首次积分可能对应不同 token 序列，降低了 Levenshtein Distance 塑形的保真度
- **未评估大模型 bs=70**：因计算和财务成本限制，未与其他 LLM 做公平的 beam search 对比
- **无真机/物理实验验证**：所有验证仅限符号计算（d V̂/dt ≡ 0），未在实际物理系统中测试
- **未开源代码和数据**：论文未提及代码/数据发布（2026.05 的 arXiv preprint）
- **仅覆盖特定 ODE 族**：Fwd 数据仅针对 rational second-order ODE，泛化到其他系统族需要新的 seed 数据
- **Backward Generation 的 OOD 问题**：合成数据的分布受限于采样策略，某些结构系统可能欠代表
