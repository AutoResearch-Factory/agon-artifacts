# Temporal Causal Reasoning with (Non-Recursive) Structural Equation Models (2501.10190)

- **Year**: 2025.01 (extended version of an AAAI-25 conference paper)
- **Authors**: Maksim Gladyshev (Utrecht University), Natasha Alechina (Open Universiteit / Utrecht), Mehdi Dastani (Utrecht), Dragan Doder (Utrecht), Brian Logan (University of Aberdeen / Utrecht)
- **Venue**: AAAI-25 (this arXiv version adds a new section beyond the published proceedings paper)
- **Keywords**: structural equation models, actual causality, temporal logic, LTL with past, model equivalence, model checking, non-recursive causal models, time-indexed interventions
- **One-liner**: Reinterprets SEMs as mechanisms turning exogenous time series into endogenous time series, giving a P-time model-checkable temporal logic (CPLTL) for actual causality that handles cyclic ("non-recursive") models for free.

## 精华提炼

### 管线/方法

这是一篇纯理论/形式化论文，没有数据管线，"管线"是概念构造的推进顺序：

1. 从 Halpern 式静态因果模型（signature `S=(U,V,R)`, 模型 `M=(S,F)`）出发，把静态 context `u`（一次性的外生变量赋值）替换为**时序 context** `u→: ℕ → ∏R(U)`（外生变量赋值的无穷序列）。
2. 把静态 intervention `Y←y` 改造成**时间敏感 intervention** `Y(n)←y`（在第 n 步对 Y 做干预），支持同一变量在不同时间点被多次干预的向量记号 `Ȳ(n̄)←ȳ`。
3. 定义 **computation** `C: ℕ → ∏R(V)`：从默认赋值 `v` 出发，每一步用上一步的 `(u→(i-1), C(i-1))` 调用结构方程 `F` 递推产生下一步赋值——本质上是把结构方程当成"时滞"的（1 步延迟）来展开成时间序列。
4. 证明这种时序展开天然消解了非递归（有环依赖图）模型在静态设定下的唯一解问题：因为每一步只依赖"严格更早"的一步，环不再造成不动点不唯一。
5. 定义 **CPLTL**（Causal LTL with Past）语法：外层是 intervention 算子 `[Ȳ(n̄)←ȳ]ψ` 包裹内层 PLTL 公式 `ψ`；PLTL 含 4 个基本模态：`○`(next)、`U`(until)、`⊖`(previous)、`S`(since)，并派生 `F`(eventually)/`G`(always-future)/`P`(sometime-past)/`H`(always-past)。
6. 语义：在一个 **causal scenario** `(M, u→, v)` 的时刻 `t` 评估 CPLTL 公式时，先按 intervention 生成（被干预后的）computation `C^int`，再把它当作标准 PLTL 的线性模型来跑标准语义。
7. 为了让 model-checking 可判定，限制 `u→` 是**终极周期**（ultimately periodic）的（前缀+循环的有限表示），呼应经典 LTL model-checking 对输入做有限表示的一贯做法。
8. Algorithm 1（`periodic-comp`）：给定周期 `u→`（类型 `(n,m)`）和 intervention，先算出前缀（覆盖 context 前缀长度与最晚 intervention 时刻的较大值），再继续推进直到发现"处于 context 循环同一位置 + 内生赋值相同"的重复状态对，从而确定 computation 本身也是终极周期的（返回前缀长 x、循环长 y）。
9. 把 CPLTL model-checking 归约为：对公式里每个 intervention 子公式跑一次 Algorithm 1 生成 computation，再用 Markey (2002) 的 PLTL path model-checking 技术（利用"过去时态嵌套高度" `h_P(φ)` 给出周期展开界）在多项式时间内验证，最后做布尔代入合并。
10. 证明 **Theorem 1**：CPLTL model-checking 属于 P。
11. 独立扩展一节：把 1 步延迟推广为**任意固定延迟**（`domain function D(Y)` 把变量映射到一组带延迟的父节点 `X_i^{-t}`），并证明 **Theorem 2**：任意有界延迟 TSEM 都可以被一个多项式大小（`O(|M|²)`）的等价 1 步延迟 TSEM 模拟（做法类似给每个滞后依赖显式展开一条"移位寄存器"式的链变量）。

### 关键技术细节

- 模型规模的复杂度记号刻意采用**一进制编码**数字（`|t|=t`），并在脚注中说明理由：这样"沿路径走到位置 n"是线性而非指数的——这是复杂度证明能成立的关键设计选择，不是随意选的编码。
- **Ultimately periodic**（终极周期）线性模型定义：`σ` 类型为 `(i,l)` 当且仅当 `∀k≥i: σ(k)=σ(k+l)`；这是把无穷时序对象变得"有限可表示"从而可判定 model-check 的核心手段。
- 提出两种时序模型等价概念，扩展 Beckers (2021) 的静态等价定义：
  - **temporally equivalent**（wrt 可观测变量集 `O`）：任意限制在 `O` 上的 intervention 和任意 context，两个模型总能找到匹配的默认赋值使得投影到 `O` 上的 computation 逐步相同。
  - **rescalably equivalent**（带系数 `k`）：允许两个模型的"一步"对应不同粒度的真实时间刻度（例如 1 分钟步长模型 vs 1 秒步长模型描述同一个 deadline 追踪过程），`k` 表示模型 2 需要多少步才能模拟模型 1 的一步。这是全新的、Beckers (2021) 没有的概念，论文明确反驳了 Beckers (2021) "无法给出有用的、对‘数值'属性敏感的等价概念"的论断。
- Algorithm 1 的循环检测要追踪 `(context 循环内位置, 内生赋值)` 二元组而不是只看内生赋值——因为外生 context 在循环内部的不同位置也可能取相同值，只看内生赋值会误判周期。
- 复杂度：整体 model-checking 时间是 `O(|Sub(φ)|)` 次 Algorithm 1 调用（每次 `O((max(|u→|,|φ|)+|u→|)·|M|)`），加上每个 intervention 子公式的 PLTL path-check `O((n+(h_P(φ)+1)·m)·|φ|)`（Markey 2002 技术），所有量在一进制编码下都是多项式。
- 任意延迟扩展中，"调用" `F_X` 在历史长度不足以覆盖 `X` 全部滞后父节点时，退化为"保持上一步的值"——这给了模型冷启动/初始化阶段一个干净、良定义的处理方式。
- 三个贯穿全文的例子承担了不同论证角色：
  - **Rocks**（Suzy/Billy 扔石头砸瓶子，经典 Halpern actual-causality 例子）：引入基本记号，并用"扔出到命中的延迟"版本引出为什么需要显式时滞链变量、以及"chain-free"压缩表示。
  - **Treatment**（连续两天给药才能康复，`R:=...∨R=1`）：典型的自指/非递归方程，展示时序展开如何自动给出唯一、直观的语义。
  - **Deadline**（Day/Hour/Minute 三个不同 tick 粒度的时钟变量）：直接驱动 rescalable equivalence 概念的提出——两个描述同一现实过程但计算步长差 60 倍的模型，既不是普通等价也不是完全无关。

### 核心结果

- **Theorem 1**：CPLTL model-checking（给定周期 `(M, u→, v, t)` 和公式 `φ`）属于 P——第一个把 actual-causality 式带时间戳的 intervention 与完整的过去+未来 LTL 结合起来的复杂度结果。
- **Theorem 2**（本 arXiv 版相对 AAAI-25 会议版新增的一节）：任意固定延迟的 TSEM 都可归约为一个多项式大小等价的 1 步延迟 TSEM——说明会议版里"只讨论 1 步延迟"的限制其实"unnecessary"（无损一般性），不是真正的表达力限制。
- 两个新的时序因果模型等价概念（temporally equivalent、rescalably equivalent），并给出与 Beckers (2021) 的正面理论分歧（不同意其"数值属性无法产生有用等价概念"的论断）。
- 证明非递归（有环依赖图）因果模型在时序解释下**不需要任何额外机制**就能被处理——把"actual causality 文献几乎全部限定于递归模型"重新定性为静态设定的技术产物，而非本质限制。

### 明确局限

- 作者自己在 Discussion 中列出的局限：(1) 只处理有限模型，声称扩展到无穷多变量"看起来直接"（seems straightforward）但**本文并未真正做**这个扩展，只是断言；(2) 没有给出时序因果模型之间的（双）模拟（bisimulation）概念，尽管这是证明等价性结果最自然的工具——完全留空；(3) 框架仅限**确定性**结构方程，概率化的时序 context 或非确定性结构方程被明确推给未来工作（引用 Beckers 2024 "Nondeterministic Causal Models" 作为起点，本文并未在此基础上做任何推进）。
- 全文没有任何实验、实现、代码或经验评估——100% 是定义+两个定理+证明的理论贡献。
- 全程假设因果模型 `M` 是**给定/已知**的，完全不处理"从数据中发现/学习因果结构"这件事——这与 causal-scs-indicator 课题需要从带噪真实时序中估计因果结构/效应的设定形成鲜明反差。
- 论文明确假设"不存在同时刻（contemporaneous）依赖"——理由是因果模型被假设为对现实的"完备表示"，时间粒度足够细以保证因果严格先于效果。这恰好回避了真实时序因果发现中最核心的难题（采样频率不够细导致无法区分同时发生的因果关系/混杂），也正是 causal-scs-indicator 课题动机部分所指出的"假设违背"的来源之一。
- Model-checking 复杂度结果的前提是 `u→` 精确已知且终极周期（有限可表示）；真实观测时序是有限、带噪声、非周期的，且因果模型本身不是先验给定的——这些都不在本文讨论范围内。
- 论文 Related Work 一节自己明确把本文与时序因果发现（Full Time/Window/Summary Causal Graph，PCMCI 一类方法）划清界限：本文不做任何从数据学习/估计的工作。

### 跟我们的关系

- **核心 claim 比对**：causal-scs-indicator 课题的核心 claim 是"用 PCMCI+/J-PCMCI+、LKIF 等统计因果发现方法在真实/模拟非平稳时序上计算因果效应估计的不稳定性，并把这种不稳定性本身当作强对流预警信号"。本文核心 claim 是"给定确定性因果模型的前提下，时间戳化 intervention + 完整过去/未来 LTL 的 model-checking 复杂度"。两者在方法、数据设定、claim 层面**完全不重叠**，不构成撞车风险。
- **方法可复用性低**：本文不做因果发现、不做效应估计、不处理混杂/非平稳作为信号来源，也不涉及任何统计/数据驱动组件——课题目前的实验管线（PCMCI+ gate、pilot 脚本等）用不上本文的任何技术组件。
- **可借用的是术语/背景定位**：本文 Related Work 明确梳理了 Full Time / Window / Summary Causal Graph（引用 `causal_timeseries` 教材 Ch.10 及 Assaad 2022）这条时序因果发现的标准术语脉络，与课题使用的 PCMCI+/J-PCMCI+ 系出同源。如果课题写作中需要精确定义"Window/Summary Causal Graph"之类术语，这条 Related Work 线索值得回去查一下 Assaad 2022 原文。
