# MORPH: PDE Foundation Models with Arbitrary Data Modality (2509.21670)

- **Year**: 2025.09
- **Authors**: Mahindra Rautela, Alexander Most, Siddharth Mansingh, Bradley Love, Alexander Scheinker, Diane Oyen, Nathan Debardeleben, Earl Lawrence, Ayan Biswas (Los Alamos National Laboratory)
- **Venue**: arXiv (submitted to ICML 2026)
- **Keywords**: PDE foundation model, modality-agnostic, axial attention, cross-attention field fusion, autoregressive surrogate, spatiotemporal forecasting, multi-physics pretraining
- **One-liner**: MORPH 是一个模态无关的 PDE 基础模型，用卷积视觉 Transformer 骨干网络统一处理 1D-3D 异质时空数据，通过 component-wise 卷积、inter-field cross-attention 和 axial attention 实现跨物理场迁移。

## 精华提炼

### 管线/方法
1. **输入标准化**：任意 PDE 数据集的 batch 被在线转换为 Unified Physics Tensor Format (UPTF-7) — 形状 `(B, T, F, C, D, H, W)`，其中 B=batch, T=时间步, F=物理场数, C=每个场的标量/向量分量数, D/H/W=空间维度。1D/2D/3D 数据通过填充缺失维度（D=1/H=1）统一表示。
2. **Component-wise 3D 卷积**：在 C 维度上做卷积，默认 8 个 filter，使用 1×1×1 projection + 3×3×3 conv + LeakyReLU(α=0.2)，产出 feature maps。为 ViT 骨干引入卷积归纳偏置，提升样本效率。
3. **Patching**：空间维度上按 patch size=8 做不重叠分块，最大 4096 patches。2D: 1×8×8 patch；1D: 1×1×8；3D: 8×8×8。每个 patch 被线性投影到 embedding（256-1024 维，取决于模型变体）。
4. **Inter-field Cross-Attention (场融合)**：将 F 个物理场的 patch embeddings 组成序列，用一个 learnable query 做 cross-attention（Q 为 learned vector, K/V 为各场 embedding），输出单一融合场。这是一个 permutation-invariant 操作（无场序位置编码），计算复杂度 O(F)。将多场信息压缩为单一场表示，显著降低后续 attention 的计算量。
5. **4D Axial Attention**：将完整的 (T, D, H, W) 时空自注意力分解为沿四个轴分别做 1D multi-head attention — 时间轴 (T)、深度轴 (D)、高度轴 (H)、宽度轴 (W)。每个轴独立做 MHA + residual sum。复杂度从 O((TDHW)²) 降为 O(TDHW·(T+D+H+W))。时间轴仅在 t>1 时启用，支持 AR(1) 到 AR(k) 的课程学习。
6. **Decoder**：简单的线性投影将 transformer 输出映射回物理空间。
7. **训练目标**：自监督 AR(1)，最小化 MSE(X_{t+1}, F_θ(X_t))。
8. **归一化**：使用 ReVIN (Reversible Instance Normalization) 预归一化全数据集（非 on-the-fly），缓存统计量用于推理时反归一化。

### 关键技术细节
- **模型变体**：Ti (7M, dim=256, heads=4, depth=4), S (30M, dim=512, heads=8, depth=4), M (126M, dim=768, heads=12, depth=8), L (480M, dim=1024, heads=16, depth=16)。均使用 8 个卷积 filter。
- **预训练数据**（6 个数据集，3 个 benchmark）：1D-CFD (PDEBench), 2D-DR/FHN (PDEBench), 2D-CFD-IC/Incompressible NS (PDEBench), 2D-SW/Shallow Water (PDEBench), 3D-MHD (The Well), 3D-CFD/Compressible NS (PDEBench)。覆盖 1D-3D，标量+矢量场。
- **微调数据**（7 个数据集）：1D-DR, 1D-BE/Burgers, 2D-CFD/Compressible NS, 2D-GSDR/Gray-Scott (The Well), 3D-CFD-Turb, 3D-TGC/Turbulent Cooling Gravity (The Well), 2D-FNS-KF/Kolmogorov Flow (PDEGym)。
- **预训练协议**：AdamW, LR=1e-3, weight decay=1e-2 (CFD 用 1e-4)。ReduceLROnPlateau (factor=0.5, patience=5 epochs)，20 epoch warmup。Ti/S: 200K steps; M: 100K steps; L: 125K steps。Balanced task sampling：每数据集采样权重 ∝ 1/N_trajectories（MHD3D 最多被采样，权重 0.31）。
- **LoRA 微调**：L 模型 (480M) 上应用 LoRA(rank_attn=16, rank_mlp=12)，仅训练 77M 参数（少于 M 模型的 126M 全参数）。LoRA 应用于所有 attention 投影 (Q,K,V,O) 和 MLP 线性层。
- **微调层级**：Level-1 (LoRA only), Level-2 (+encoder unfreeze), Level-3 (+decoder unfreeze), Level-4 (全参数)。Ti/S/M 用 Level-4，L 用 Level-1。
- **多 DataLoader 架构**：6 个独立 DataLoader（每个预训练数据集一个），允许 per-dataset batch size 差异化（1D-CFD=128, 2D-DR=64, 2D-CFD-IC=16, 2D-SW=64, 3D-MHD=16, 3D-CFD=4）。自定义 streaming sharding algorithm (Algorithm 1) 实现跨 DDP ranks 和 workers 的无重复样本分配。
- **数据流**：chunked streaming from storage → RAM → local shuffle → AR pairs → GPU。支持 TB 级数据集而不需要全量加载到内存。
- **计算资源**：Ti/S FM: 4×A100(40GB)；M/L FM: 2×8×H100(80GB)；独立模型: 2×A100 或 2×A6000；微调: 1×H100 或 2×A100/A6000。
- **Positional Encoding**：两种 learnable 方案 — S-Linear & T-Slice (Ti/S/M, max_ar=1) 和 ST-Bilinear (L, max_ar=16)，均为可处理可变 (t,n) 的 resampling 方案。
- **预训练规模**：数据分割 0.8/0.1/0.1 (train/val/test)。Patches 默认 ≤ 4096，patch size=8。

### 核心结果
- **Zero-shot 优势**：MORPH-FM-L 在 11/12 个数据集上取得最佳 zero-shot 性能。L 模型预训练验证 loss 收敛到 ~0.02，对比 Ti/S 的 ~0.09（~4.5× 改善）。
- **Full-shot 微调**：MORPH-FM 在 5/6 个微调数据集上优于 MORPH-SS（独立训练版）。对比 DPOT-FM 和 Poseidon-FM — MORPH 在 2D-CFD 上以 0.0423 NRMSE 大幅领先（DPOT-FM-M: 0.63255, Poseidon-FM-B: 0.59939）。在 2D-GSDR 上以 0.00497 VRMSE 刷新最优。1D-DR 上 MORPH-FM-Ti 取得 0.0008 NRMSE（全局最优）。
- **LoRA 高效性**：MORPH-FM-L (480M, 77M 可训练) 在 full-shot 微调中匹配或超越 M 模型 (126M 全参数)。
- **跨模态迁移 (GCR)**：仅用 2D 不可压 NS (CFD2D-IC) 预训练，zero-shot 迁移到 6 个不同模态目标 — 全部 GCR>0（正迁移）。最近域（1D/2D/3D CFD）迁移最强，MHD3D 和 TGC3D 等多物理场系统也有非平凡迁移。这是 MORPH 模态无关设计的最直接实证支撑。
- **数据效率**：1D-DR 上仅用 25% 微调轨迹，MORPH-FM-Ti 已超越独立模型的 100% 全量训练。2D-FNS-KF 上用 ~256 条轨迹（<1% 全量）即超越独立模型。
- **跨 FM 规模效率**：MORPH-FM-S (30M) 在 FNS-KF 任务上（128 轨迹 ≤1% 数据，200 epochs）超越 DPOT-FM-M (122M, 4×参数) 和 Poseidon-FM-B (158M, 5×参数)。MSE: MORPH 0.00162 vs DPOT 0.0301 vs Poseidon 0.0017。
- **预训练语料消融**：MORPH-FM-S* (27M) 使用与 Poseidon 相同的预训练语料，在 CE-RM (0.09044 vs 0.4181) 和 NS-SL (0.0159 vs 0.0163) 上超越 Poseidon-FM-B (158M, 6×参数)。仅在 NS-PwC 上落后 (0.00408 vs 0.0004)。证明架构优势而非数据优势。
- **架构消融**：Conv 在 3D/2D 上降误差 5-8%，1D 上效果轻微；Cross-attention 场融合优于 concat+MLP (降误差 1.5-9.7%)；Axial attention 在所有 token count 设置下优于 full/sparse attention — 在 4096 tokens 时 error 降 34% 且 FLOPs 降 3.1×。
- **Scaling**：预训练数据量增加 → train/val loss 持续下降（10 epoch 内）。但固定 10 epoch 下 Ti→L 无清晰收益，暗示需要更长的训练时间或调参来释放大模型能力。

### 明确局限
- **物理覆盖范围有限**：预训练主要覆盖流体相关 PDE，缺乏弹性力学、断裂力学、波传播、流固耦合等物理领域。
- **模型规模上限**：当前最大 480M 参数，作者认为需要数十亿参数才能充分利用更大的预训练语料。
- **规则网格限制**：卷积预处理和 patching 方案依赖规则 Cartesian 网格，无法直接处理非结构网格或不规则几何（这是一个重大局限）。
- **UPTF-7 开销**：标量到向量的 lifting 操作引入了额外的 batch 级内存和计算开销。
- **仅 AR(1) 训练**：虽然架构支持多步 AR，但当前只用 AR(1) 训练，高维 AR(k) 可能改善 rollout 长期稳定性但未充分探索。
- **无不确定性量化**：模型仅做点预测，缺乏预测置信度估计，安全关键应用存在风险。
- **LoRA 探索不充分**：仅测试了固定 rank，未系统性研究不同 rank 在不同数据集上的效果。
- **非自回归对比**：与 Poseidon 对比时，Poseidon 用的是 direct（非自回归）预测模式跨全轨迹，不是严格可比的自回归设定。

### 跟我们的关系
- **不撞车**：MORPH 是数据驱动的 PDE neural surrogate，FEX 是 RL-based symbolic regression。MORPH 学习 PDE 的数值解（黑盒预测），FEX 要发现 PDE 的符号形式（白盒可解释表达式）。本质上是互补的两种科学发现范式。
- **潜在协同**：MORPH 可以作为 FEX 搜索过程中的快速评估器——当 FEX 的 RL agent 生成候选符号表达式时，可以用类似 MORPH 的预训练 neural surrogate 来快速近似该表达式的数值解，加速 fitness evaluation（替代昂贵的数值求解器）。反过来，FEX 发现的符号表达式可以作为物理先验知识注入到 MORPH 的训练中（如物理损失项），减少 MORPH 对大数据的需求。
- **可用工具/方法**：(1) UPTF-7 统一数据格式的思想可借鉴到 FEX 的多模态数据输入标准化；(2) axial attention 的时序建模能力可用于 FEX 中处理多时间尺度的观测数据；(3) LoRA 参数高效微调方法可用于 FEX 中的 policy/value network 快速适配新方程族；(4) streaming data pipeline + 多 DataLoader 架构的工程实践经验；(5) ReVIN 归一化方案可借鉴到 FEX 的数据预处理管线。
- **差异化**：MORPH 解决 "已知方程，快速求解"；FEX 解决 "已知数据，发现方程"。两者目标相反，不存在竞争。如果未来要发展 "neuro-symbolic PDE discovery"，MORPH 的预训练模型可作为神经网络先验。

## Read by: 0616-fex (2026-06-24)

MORPH 与 FEX 分属科学机器学习的两个互补方向：MORPH 是 data-driven neural surrogate（已知方程→快速数值解），FEX 是 RL-based symbolic regression（已知数据→发现符号方程）。两者不存在竞争关系，但存在协同潜力。

**撞车评估**：完全不撞。MORPH 不涉及符号回归或 RL，FEX 不涉及 PDE neural surrogate 或 foundation model pretraining。

**可借鉴的技术**：
1. UPTF-7 统一数据格式 — 如果我们后续处理多模态物理观测数据作为 FEX 输入，可以参考这种标准化思路
2. Axial attention 的序列建模能力 — 可能用于 FEX 中处理时序观测数据的 token sequence encoder
3. LoRA 参数高效微调 — 如果 FEX 的 policy/value network 需要快速适配新方程族，LoRA 是轻量级方案
4. Streaming data pipeline — 处理大规模仿真数据时的工程参考
5. ReVIN 归一化 — 跨异构数据集归一化的实用方案

**差异化方向**：MORPH 的逆问题实验（反引文 2603.04606）暗示了一个有趣方向：用 neural surrogate 加速 PDE 参数估计。FEX 可以通过 RL 发现 symbolic 形式的逆问题求解器，这比纯 neural 方案更具可解释性。这是潜在的未来交叉点。
