# "What is a realistic forecast?" Assessing data-driven weather forecasts, a journey from verification to falsification (2602.00622)

- **Year**: 2026.02
- **Authors**: Zied Ben Bouallègue (ECMWF, Reading, UK; 单作者)
- **Venue**: arXiv preprint（copernicus 模板 `\documentclass[gmd]`，目标期刊为 GMD 系 Copernicus 期刊）
- **Keywords**: forecast realism, forecast verification, model diagnostics, falsification, physical consistency, data-driven weather models, hallucination, Murphy forecast goodness
- **One-liner**: 提出天气预报 realism 三分类（functional/structural/physical），把 Popper 式 falsification 引入数据驱动天气模型评估，与 verification、diagnostics 并列成三步评估流程。

## 精华提炼

### 管线/方法（概念框架，无实验管线）

这是一篇单作者 position/essay paper，仿照 Murphy (1993) "What is a good forecast?" 的经典三分法，回答 "What is a realistic forecast?"。核心贡献是一个概念框架 + 一个评估流程：

**三类 realism 定义**（每类配一个形式化算子）：

| 类型 | 名称 | 定义 | 形式化 | 对应评估活动 |
|------|------|------|--------|--------------|
| Type 1 | Functional realism | 单实例上预报与观测的接近程度 | V = v(x_i, y_i)，v 为 scoring function | Verification（验证） |
| Type 2 | Structural realism | 预报与观测的统计特性一致性（reliability） | D = d(X, Y)，X/Y 为预报/观测的统计特征 | Diagnostics（诊断） |
| Type 3 | Physical realism | 预报与科学知识库 K 的相容性 | F = f(x_i, K)，f 为假设检验函数 | Falsification（证伪） |

**评估旅程（journey）**：Verification（逐实例对比观测，排名模型）→ Diagnostics（平均特性，定位模型缺陷）→ Falsification（对照物理知识，识别 artifact、建立信任）。三步走完才能回答"这个预报 realistic 吗"。

**认识论框架**：理论驱动模型主要用演绎逻辑（物理定律 → 预报），数据驱动模型核心是归纳逻辑（数据 → 学出规则，规则机器可读但人不可读）。Hume 的归纳问题在此变成可解释性问题；对预报本身，则需要 falsification 来补上"规则是否符合物理"的检验。

### 关键技术细节

- **Type 1 关键概念**：elicitability 与 consistent scoring function——常规做法是"对可引出泛函用一致评分函数"（如对 ensemble mean 算平方误差）；proper score 对特定泛函提供 calibration 与 optimal score 对齐的理论保证。结果同时依赖 metric 选择和 substrate 选择（变量、区域、lead time、空间尺度）。
- **Type 2 工具**：bias、activity bias（异常方差偏差）、power spectra（Rodwell 2025，比较各尺度能量）、effective resolution（Selz 2025，大气结构以真实振幅再现的最小空间尺度）、granularity/sharpness 度量（Ebert-Uphoff 2025，针对第一代 ML 预报的模糊化）。
- **Type 1 vs Type 2 关系（论文最技术性的部分）**：
  - 概率框架：proper scoring rule 可分解为 resolution + reliability 项（Bröcker 2009 分解），改善 reliability 同时改善 score——两目标对齐，这也是 propriety 被视为基本性质的原因。
  - 确定性框架：一阶矩校准（bias correction）改善 score，但二阶矩校准（方差校正）可能恶化 score（Murphy 1988 的可预报性结果）→ **accuracy–activity trade-off**。
  - MSE 训练的数据驱动模型：优化过程平均掉低可预报性的小尺度特征 → 预报随 roll-out 变长越来越平滑，一定 lead time 后等效于条件均值。因此物理模型输出叫 "deterministic forecast"（一条物理可能的轨迹），ML 输出应叫 **"single forecast"**（统计优化问题的解），是一种新的信息载体。
- **Type 3 / Falsification 的三个层级**（超越目视检查/subjective verification 之外）：
  1. **硬约束 no-go areas**：单变量（负降水）或变量组合（总降水为零时出现正雪）的物理禁区检查；操作化建议是把预报极值与常规验证指标一起归档监控。
  2. **平衡/守恒定律**：从物理平衡出发可定量化，能定义性能指标（引 Kasteleyn 2026 PhysMetrics.Weather）；实际限制是 ML 模型只输出有限变量集，部分守恒律无法直接评估。
  3. **需要模型本体的定向测试**：扰动/受控条件下运行模型，评估响应是否合理——超长 roll-out 揭示漂移（Lehmann 2026）、局部扰动看动力学响应（Hakim 2024 dynamical tests）；由开发者执行的 stress tests，是建立信任的关键。
- **Hallucination 定义**（引 Rathkopf 2025）：生成式 ML 的新错误类别——统计上似真但领域专家一看就物理不一致的预报特征。**常规 scoring rules 对物理真实性不敏感，所以 hallucination 在常规验证中不会被发现**，需要新工具。物理真实性改善与常规评分改善之间的形式化关系尚未建立（开放问题）。
- **完美预报思想实验（层级关系）**：Type 1 完美 ⇒ Type 2、3 完美；Type 2 完美 ⇒ 预报是观测的 counterfactual（取值原则上可被观测到）但不保证 Type 1；Type 3 完美不保证 Type 1 或 Type 2（物理模型也有偏差）。层级 ≠ 应用重要性排序，后者由 fit-for-purpose 决定。
- **Realism 与 information content**：先校准再打分 = 度量 resolution/潜在性能（引 Gneiting 2025），是比较不同 reliability 预报的公平方式——后处理成为验证流程的组成部分。价值层面：多源预报并存时，信息互补性比单指标性能更重要，realism 较差的预报可能价值更高。

### 核心结果

概念论文，无数值实验。可操作的"结果"是：

- 三类 realism 的正式定义 + V/D/F 三算子 + 三步评估流程图（Fig. journey）。
- Storm Ciarán 案例（引 Charlton-Perez 2024）作为三类 realism 同时评估的示范：风暴路径抓得好（Type 1 好）、ML 预报最大风速普遍低估（Type 2 差）、风暴增强期物理过程有时表现差（Type 3 差）——一个案例三个维度得出三种结论。
- 明确论断：概率框架下 score 与 calibration 目标对齐；确定性/single forecast 框架下存在 accuracy–activity trade-off，"single forecast" 应作为新信息类型理解。
- fit-for-purpose 指引:Type 1 对短时效逐日预报最重要;Type 2 对长时间尺度（统计特性再现）与概率决策必要;Type 3 对科学探索、气候变化外推（训练样本外泛化）、human-in-the-loop 信任场景关键;若预报只作为下游后处理输入，物理不一致可在后端修复。

### 明确局限

- 作者自认：物理真实性"有时更多是期望属性而非实际需要"——两类模型（物理/ML）输出都有 artifact；falsification 对 ML 模型的必要性源于归纳问题（信任），不是所有应用都需要。
- 守恒律检查受限于 ML 模型输出变量集不全，部分诊断无法应用。
- 物理真实性改善与评分改善之间的形式化关系是开放问题（作者明说 "remains to be established"）。
- 读出的局限：无任何定量实验或新指标实现——三个 falsification 层级全部指向他人工作（Kasteleyn、Lehmann、Hakim）；框架的可操作性未经论文本身验证。essay 文体，正文仅 273 行 tex。
- 对流尺度/区域模型完全未讨论（通篇 global medium-range 视角，ERA5/AIFS 语境）；也不涉及数据集/benchmark 构建。
