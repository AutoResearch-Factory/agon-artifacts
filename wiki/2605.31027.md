# Multi-Scale Separable Fourier Neural Networks for Solving High-Frequency PDEs (2605.31027)

- **Year**: 2026.05
- **Authors**: Qihong Yang, Qiaolin He (School of Mathematics, Sichuan University)
- **Venue**: arXiv
- **Keywords**: high-frequency PDEs, separable random features, Fourier neural networks, least squares, batched QR, Helmholtz equation, neural PDE solver, analytic derivatives
- **One-liner**: 固定随机傅里叶分离基+最小二乘解高频 PDE。

## 精华提炼

### 管线/方法
- 输入是一类 PDE 边值/初边值问题：域内方程 `L u = f`，边界或初值条件 `B u = g`。
- 作者主要面向高频线性 PDE，但也把方法扩展到非线性 elliptic、Taylor-Green vortex 和 steady Navier-Stokes。
- 方法名是 MS-SFNN: Multi-Scale Separable Fourier Neural Networks。
- 核心表示不是普通 PINN 的端到端可训练 MLP，而是固定随机基函数加线性最小二乘。
- 对 `d` 维输入 `x=(x_1,...,x_d)`，构造 `d` 个单隐藏层子网络。
- 第 `j` 个子网络只接收单个坐标 `x_j`，输出一个 `M` 维向量。
- 每个子网络的权重和偏置从均值 0、方差 1 的均匀分布中独立采样。
- 这些随机权重和偏置只初始化一次，之后完全冻结。
- 每个坐标维度有一个缩放因子 `rho_j`，用于控制该维度基函数的有效频率。
- 激活函数选择 `cos`，作者认为这显式嵌入 Fourier features。
- 全局基函数通过各坐标子网络输出的逐元素乘积得到。
- 最终解的形式为 `u_M(x)=sum_i w_i prod_j cos(rho_j (w_ji x_j + b_ji))`。
- 唯一需要求解的未知量是线性组合系数 `w_i`。
- 这些系数通过把 PDE residual、边界条件、初值条件写成线性最小二乘系统求出。
- 因为基函数是显式 cos 乘积，作者不用 automatic differentiation。
- 一阶、二阶导数都有闭式表达式，直接代入 PDE operator。
- 这点对高频和 3D 问题重要，因为 AD 会保留计算图，显存开销高。
- 大规模 collocation 时，完整矩阵 `A` 的形状约为点数乘以基函数数，仍会爆显存。
- 为解决这个问题，作者提出 batched QR decomposition。
- 训练点分成域内点批次和边界点批次，分别处理。
- 算法维护一个 `M x M` 上三角矩阵 `R_t` 和一个 `M` 维向量 `b_t`。
- 每来一个 batch，就把当前 `R_t` 与该 batch 的算子矩阵纵向堆叠，再做 QR。
- 更新后只保留新的 `R_t` 和 `(Q^T b)[:M]`。
- 处理完所有域内和边界 batch 后，对 `R_t w = b_t` 求最小二乘系数。
- 这种流程避免一次性 materialize 全量 collocation matrix。
- 非线性 PDE 不是一次线性最小二乘解决，而是用 Picard iteration。
- 非线性项在每轮用上一轮近似解冻结，得到线性化系统。
- nonlinear elliptic 例子中最大迭代 100 次，系数差阈值为 `1e-16`。
- Taylor-Green vortex 和 double-cylinder steady Navier-Stokes 也采用类似 Picard 线性化。
- 对复值 acoustic car cabin 问题，作者把实部和虚部分成两个 MS-SFNN 展开分别拟合。

### 关键技术细节
- 主要实验统一使用 `M=10,000` 个基函数。
- 计算环境是 Debian 12、Intel Xeon Platinum 8358 CPU 2.60 GHz、NVIDIA A100 80GB。
- 误差指标是最大绝对误差 `L_infty` 和相对 `L_2` 误差。
- heat conduction 设置为 1D 空间加时间，视为二维输入。
- heat conduction 的精确解是 `exp(-t) sin(kx)`，`k=20pi,100pi,500pi`。
- heat conduction 的网格为 `101 x 101` 或 `401 x 201`。
- heat conduction 的缩放因子按真实频率设置：`rho_1=k, rho_2=1`。
- 2D Helmholtz 设置在 `(0,1)^2`，精确解 `sin(kx) sin(ky)`。
- 2D Helmholtz 的 `k=24pi,48pi`，网格 `201 x 201`，缩放因子 `rho_1=rho_2=k`。
- complex geometry Helmholtz 在方形域挖一个圆孔，圆心 `(0.5,0.5)`，半径 `0.15`。
- 圆孔边界用 200 个 polar points 采样。
- complex geometry Poisson 有三个圆孔和一个椭圆孔，网格仍为 `201 x 201`，每个内孔 200 个边界点。
- complex source Poisson 用 `u=sin(mu x^2)+sin(mu y^2)`，`mu=15`，`rho_1=rho_2=50`。
- 3D Helmholtz 在单位球内，候选网格 `101^3`，边界用 `500 x 500` 球面采样。
- 3D Helmholtz 的缩放因子是 `rho_1=6, rho_2=10.5pi, rho_3=2`。
- car cabin acoustic 例子使用长方体 cabin 近似域 `[-1.5,1.5] x [-2,0] x [-1,1]`。
- car cabin 有 6 个声源，频率 `v_freq=1000`，波数 `k=2pi v_freq / 340`。
- flower-shaped Helmholtz 用边界 `r=a-b cos(m theta)`，`(a,b,m)=(0.5,0.1,6)`。
- flower-shaped 例子测试 `1000 Hz` 和 `10000 Hz`，边界 Dirichlet 或 mixed boundary。
- nonlinear elliptic 例子为 `Delta u + u^2 = f`，精确解 `(x+y) cos(kx) sin(ky)`，`k=10`。
- Taylor-Green vortex 使用 `51 x 51 x 51` 空间时间网格。
- Taylor-Green 的 `u,v,p` 各有一组缩放因子，共 9 个 `rho`。
- double-cylinder steady Navier-Stokes 在半径 3 圆域中挖两个半径 0.3 障碍物。
- scale factor analysis 对 2D Helmholtz 在 `rho_1,rho_2 in [1,200]` 做网格搜索。
- 作者结论之一是 `rho` 不是装饰性超参，选错会导致误差大到数值不可用。
- 理论部分把 MS-SFNN 解释为随机 Fourier feature 诱导的 kernel method。
- 理论依据主要借鉴 Rahimi random features 与 RKHS 高概率逼近结果。
- 论文还引用一维二阶 elliptic 方程中 random Fourier features 的 spectral convergence，作为 Fourier features 有效性的间接支持。
- 但文中没有给出面向全部实验设置的完整误差界或自动选 `rho` 的理论。

### 核心结果
- heat conduction: PINN 在 `k=20pi,100pi,500pi` 都接近失败，`L_2` 约 `1`。
- heat conduction: SV-SNN 在 `k=500pi` 的 `L_infty=4.45e-2, L_2=3.75e-2`。
- heat conduction: MS-SFNN 在 `k=20pi` 和 `100pi` 达到 `1e-12` 级误差。
- heat conduction: MS-SFNN 在 `k=500pi` 明显降级到 `L_infty=5.77e-4, L_2=1.59e-4`，这是重要负面信号。
- 2D Helmholtz: PINN 在 `k=24pi,48pi` 的误差约 `1`，基本失败。
- 2D Helmholtz: SV-SNN 误差为 `1e-2` 量级。
- 2D Helmholtz: MS-SFNN 在 `k=24pi` 达到 `L_infty=5.66e-11, L_2=2.71e-12`。
- 2D Helmholtz: MS-SFNN 在 `k=48pi` 达到 `L_infty=9.65e-9, L_2=1.34e-9`。
- 圆孔 Helmholtz: MS-SFNN 在 `k=24pi` 为 `L_infty=1.08e-10, L_2=1.49e-11`。
- 圆孔 Helmholtz: MS-SFNN 在 `k=48pi` 为 `L_infty=9.81e-10, L_2=1.08e-10`。
- 多孔 Poisson: SV-SNN `L_infty=3.45e-2`，MS-SFNN `L_infty=1.18e-12, L_2=5.75e-13`。
- complex source Poisson: PINN `L_infty=1.04, L_2=1.78`，SV-SNN `L_2=4.73e-3`，MS-SFNN `L_2=5.75e-13`。
- 3D Helmholtz: PINN `L_2=2.87`，SR-HFNN `9.04e-4`，NR-HFNN `7.30e-3`。
- 3D Helmholtz: MS-SFNN `L_infty=3.30e-6, L_2=2.94e-6`，比 2D 场景弱但仍优于比较方法。
- car cabin acoustic: PINN 的实部/虚部 `L_2` 分别约 `0.994` 和 `1.02`。
- car cabin acoustic: MS-SFNN 的实部/虚部 `L_2` 分别为 `1.07e-3` 和 `8.55e-4`。
- flower Dirichlet 1000 Hz: FEM `2.60e-3`，LbNM `4.85e-6`，MS-SFNN `L_infty=2.41e-14`。
- flower Dirichlet 10000 Hz: FEM `3.81` 失败，LbNM `7.05e-5`，MS-SFNN `L_infty=7.08e-6`。
- flower mixed 1000 Hz: MS-SFNN `L_infty=1.95e-14, L_2=5.33e-15`。
- flower mixed 10000 Hz: LbNM `2.66e-6`，MS-SFNN `L_infty=1.66e-6, L_2=8.37e-7`。
- nonlinear elliptic: PINN `L_2=3.40e-1`，SV-SNN `4.05e-3`，MS-SFNN `4.51e-13`。
- Taylor-Green vortex: MS-SFNN 对 `u,v,p` 的 `L_2` 分别为 `1.11e-12,5.74e-13,4.29e-12`。
- double-cylinder NS case 1: MS-SFNN 对 `u,v,p` 达到 `1e-14` 量级。
- double-cylinder NS case 2: MS-SFNN 对非零压力也达到 `1e-14` 量级。
- 方法对比表中，2D Helmholtz `k=24pi` 上 MS-SFNN 平均 `L_2=2.61e-12, std=8.44e-13`。
- 同一表中 XPINN、FBPINN、FourierPINN、BsPINN 为 `1e-1` 到 `1` 量级，SV-SNN 为 `1.27e-2`。
- 最反直觉的结果是：固定随机基函数加线性最小二乘，在这些 manufactured 高频 PDE 上比大量可训练 PINN 变体精度高许多。
- 最需要警惕的结果是：作者几乎总是把 `rho` 设成已知真实频率或手工调好的近似值。

### 明确局限
- 作者自己承认，MS-SFNN 的精度高度依赖缩放因子 `rho`。
- 作者自己承认，暴力 grid search 选 `rho` 在高维和多变量系统中不可接受。
- 作者自己承认，在极高频率和 3D 高频设置下精度会下降。
- 作者提出未来需要自动发现近优 `rho` 的算法。
- 论文没有发布代码，Data Availability 写的是数据可向通讯作者索取。
- 这会显著限制第三方复现。
- 所有核心实验都是 manufactured solutions，右端项和边界条件从精确解反推。
- 这适合验证表达能力，但不能证明真实未知 PDE、真实观测噪声或真实几何网格下的稳健性。
- 很多 baseline 表格缺少 wall-clock、内存峰值、同等调参预算和完整误差列。
- batched QR 的内存优势有算法描述，但缺少显式内存曲线或和 dense solve 的系统对比。
- `rho` 的选择使用了真实解频率信息，例如 `k`, `k/sqrt(2)`, `2pi`。
- 这使部分实验更像“给定频率先验的近似器”，而不是完全自动 PDE solver。
- 论文没有和 FEX 或 Multi-Scale FEX 做直接比较。
- 对 `0616-fex` 来说，这意味着它不是直接证明优于 FEX 的论文。
- 输出是 10,000 项随机 Fourier 分离基的线性组合，不是简洁可解释的 symbolic expression。
- 因此它不能替代 FEX 在 closed-form / finite expression / symbolic recovery 上的定位。
- 非线性 PDE 依赖 Picard iteration，收敛性和失败模式没有充分展开。
- Taylor-Green 和 Navier-Stokes 的结果非常高，但仍是精确制造解，离真实 CFD benchmark 有距离。
- 比较方法的实现细节、训练预算、调参过程描述不足，存在 baseline strength 风险。
- 论文语言中有不少拼写和公式排版问题，反映出版本仍偏 preprint。

### 跟我们的关系
- `0616-fex` 的主线是 FEX / RL for symbolic regression / PDE expression search。
- 这篇论文不直接撞车 FEX 的核心：它不是 RL 搜索表达式树，也不输出简洁 finite expression。
- 它的解是大规模随机 Fourier feature 线性组合，可解释性弱于 FEX。
- 它也没有 FEX 的 `S_k` finite expression approximation theory。
- 但它与 FEX 的高频 PDE 求解方向强相关。
- 如果 `0616-fex` 的 idea 是 multi-scale / high-frequency FEX，那么这篇需要作为近邻引用。
- 具体撞车点是 per-dimension frequency scaling：MS-SFNN 用 `rho_j` 控制每个坐标方向的频率内容。
- Multi-Scale FEX 也关心频率分离和多尺度振荡；两者都把频率先验作为成功关键。
- 另一个可借鉴点是解析导数：FEX 表达式树本身也可自动生成解析导数，避免 AD 显存和数值误差。
- batched QR 也值得借鉴：如果 FEX 某些候选表达式的系数拟合可写成线性最小二乘，可用类似 streaming QR 降低 collocation 评估内存。
- 对 active collocation idea，这篇提供反例式提醒：高频 PDE 中大量均匀点仍可能必要，采样策略必须和频率先验联动。
- 对 synchrosqueezed / butterfly prior idea，这篇强化了“先估频率再设基函数尺度”的合理性。
- 该引不该引：如果论文讨论高频 PDE、Fourier basis、random feature solver、MS-FEX 频率先验或 collocation least-squares，就该引。
- 如果论文只讨论一般 SR 搜索效率、LLM skeleton 或 equality graph 剪枝，这篇不是核心引用。
- 差异化方向很清楚：FEX 应强调自动发现低复杂度 symbolic structure，而 MS-SFNN 依赖手工/网格选 `rho` 与 10,000 项随机基。
- 更强的 FEX claim 可以是：不用预知真实频率，用 RL/LLM/synchrosqueezed prior 自动发现频率结构，并输出简洁表达式。
- 本文最有用的 benchmark 是 2D/3D Helmholtz、flower domain、car cabin acoustic，这些可以作为 FEX 高频能力测试集候选。
- 本文最大风险提示是：若我们的 FEX 高频方法也要手工设置频率参数，就会被质疑和 MS-SFNN 一样依赖 oracle frequency。

这篇不是 FEX 的直接竞品，因为 MS-SFNN 不搜索表达式树，也不恢复低复杂度 closed-form solution。它更像一个固定随机 Fourier 分离基的高精度 least-squares PDE solver。对 FEX 来说，真正需要吸收的是三个工程和科学信号：第一，per-dimension frequency scaling 是高频 PDE 的关键控制旋钮；第二，解析导数和 batched QR 能把大规模 collocation residual evaluation 做得更稳；第三，手工或 oracle `rho` 会成为方法说服力的主要弱点。
