# When Can We Trust Cluster-Robust Inference? (2604.02000)

- **Year**: 2026-04
- **Authors**: James G. MacKinnon (Queen's University; Aarhus Center for Econometrics)
- **Venue**: arXiv preprint (econometrics); preliminary version presented at the 2025 Canadian Stata Conference, Ottawa; acknowledges Morten Nielsen, Matt Webb and two referees for comments (not a formally refereed venue as of this version)
- **Keywords**: cluster-robust inference, CRVE, wild cluster bootstrap, cluster jackknife, score-variance test, cluster heterogeneity diagnostics, effective number of clusters, partial leverage, targeted Monte Carlo, placebo regression
- **One-liner**: 综述"何时能信任聚类稳健推断"，给出一套可操作的诊断程序清单（异质性/杠杆诊断 + targeted Monte Carlo + placebo regression 交叉验证），而非提出新理论。

## 精华提炼

### 管线/方法

这是一篇实践导向的综述/方法论提炼论文，结构本身就是一套诊断流程，而非单一算法：

1. **建立基准**（§2）：一维聚类线性回归 `y_g = X_g β + u_g`，`g=1..G`；三种 CRVE——CV1（最常用，简单 sandwich）、CV2（HC2 类推广，无偏但计算贵）、CV3（cluster-jackknife，`(G-1)/G · Σ_g (β̂^(g) − β̂)(β̂^(g) − β̂)'`，通过留一聚类重估计得到，计算上等价于先算好 `X_g'X_g` 与 `X_g'y_g` 再逐个求逆，成本很低）。
2. **推断分布选择**（§2.2）：`t(G-1)` 是最常用近似；Hansen (2025/2026) 的按系数定制自由度 `d_j` + 缩放因子 `γ_j` 方法目前被作者认为是解析法里最好的（已进 Stata 19.5）。
3. **Bootstrap 替代**（§3）：Pairs Cluster Bootstrap（PCB，重抽样整个聚类）表现平庸；Wild Cluster Bootstrap（WCB）更优，四个具体变体——WCU-C/WCR-C（经典，基于未修正/受限得分）与 WCU-S/WCR-S（用 cluster-jackknife 修正过的得分，`ś_g = X'X(β̂ − β̂^(g))`，几乎零额外成本，因为 jackknife 量已经算过），一般 WCR-S 综合表现最好。
4. **诊断为何会不可信**（§4）：`G` 比 `N` 重要得多；老经验法则"`G=42` 就够"被明确否定为过度简化；真正决定可靠性的是聚类间的异质性（大小、杠杆、处理臂分布）。
5. **决定正确聚类层级**（§5）：score-variance 检验（MNW-testing）、Ibragimov-Müller 检验、Cai 随机化检验——都要求候选层级严格嵌套，且都与具体系数绑定，作者明确建议当诊断用而非当正式预检验用；双向聚类时取"两个单向 + 一个双向"里最大的标准误。
6. **评估可信度的具体工具**（§6，核心可操作部分）：异质性诊断（杠杆/有效聚类数）、targeted Monte Carlo、placebo regression——三者互相印证，一致则可信。
7. **两个真实数据全流程示范**（§7）：完整走一遍上述全部诊断，在两个已发表的真实教育经济学数据集上展示分歧与收敛。
8. **结论**（§8）：把全文压缩成一张可操作检查表（见"关键技术细节"末尾）。

### 关键技术细节

- **CV3/jackknife 计算捷径**：`β̂^(g)` 的计算不需要真正重新做 `G` 次完整回归——只需先算好每个聚类的 `X_g'X_g` 和 `X_g'y_g`（这些本来就是构造全样本 `X'X` 的中间量），逐一相减再求逆即可，除非 `G` 或 `k` 很大，否则成本接近免费。
- **杠杆/异质性诊断族**：partial leverage `L_gj = (accented x_gj)'(accented x_gj) / (accented x_j)'(accented x_j)`（对系数 `j`，`accented x_j` 是 `x_j` 对其余回归量回归后的残差）；scaled variance `V_s^j = G²/(G-1) · Σ_g (L_gj − 1/G)²`（全部聚类杠杆相同时为 0，越大越不均衡）；**有效聚类数** `G*_j(ρ)` 是 `V_s^j` 的单调递减函数，比原始 `G` 更直观——论文给出具体对照：`G=50` 但 `G*(0)=11.2` 时推断困难，`G*(0)=48.7` 时大多方法可靠。这三者可用 Stata 的 `summclust` 包一次性算出，同时给出留一聚类估计 `β̂^(g)` 的变异情况（个别聚类差异过大 = 警示信号）。
- **处理臂计数的具体规则**：若关键回归量是聚类层面的处理哑变量，设 `G1` 个处理聚类、`G0=G-G1` 个对照聚类，两者都不能太小；DiD 模型里 `G1` 尤其关键，`G0` 可为 0（只要有足够多"有时处理"的聚类）。
- **异方差检查**：按聚类算残差方差 `σ̂_g²`（式 clustvar，若有聚类固定效应则退化为按聚类哑变量回归平方残差）；更进一步可直接跑 `û_gi² = η1 + η2·T_gi + e_gi`（式 twovars）检验处理组与对照组的残差方差是否系统性不同——`η̂2` 显著为正是警示信号。
- **Targeted Monte Carlo 具体做法**：不用通用仿真设计，而是固定真实的 `X` 矩阵和真实的聚类结构，`β` 设为 0 或 `β̂`（对置信区间长度无影响，只影响检验的是哪个假设），用随机效应模型 `u_gi = v_g + ε_gi`（式 RE，`ρ=σ_v²/(σ_v²+σ_ε²)` 从 0 到 0.5 扫描）生成扰动项。**关键警示**：若模型含聚类固定效应，随机效应设计会让固定效应把 `v_g` 完全吸收掉，届时聚类内实际上不再有相关性，根本不需要聚类稳健推断，用 HC2/HC3 即可。
- **Placebo regression 具体做法**：构造一个"安慰剂回归量" `z`，构造方式要模仿真实处理回归量的结构（如真实处理是 20/22 个聚类的二元分组，则 `z` 也应是随机重新分配的 20/22 二元分组）；两种嵌入方式——替换真实回归量（Placebo-R，风险：若真实回归量本该在 DGP 里，删除它会导致设定误设从而虚假拒绝）或作为额外回归量加入（Placebo-A，代价是回归量数目和真实模型不一致）。**可行性下限**：可能的安慰剂组合数需要在约 100 到几十万之间才有意义，太少（如仅 9 种）则无法学到任何东西；组合数可枚举时应枚举而非随机抽样，避免模拟误差。
- **两个真实案例的关键数字**：案例一（Porter & Serra 2020 女性榜样实验，12 个班级聚类、4 个处理聚类）：`G*(0)=8.49`，`G*(1)=5.98`，CV1 标准误明显小于不聚类的稳健标准误（向下偏），11 种方法给出的 `t` 统计量和 `P` 值分歧很大；targeted MC（`ρ` 从 0 到 0.5）+ 两种 placebo regression（Add/Replace 几乎给出相同结果）一致显示 CV1+t(11) 过度拒绝、pairs bootstrap 拒绝不足、HC1/default logit（即不聚类）反而表现最好——因为该数据集实际组内相关性很弱。案例二（Rao 2019 Delhi 精英学校多样性，2364 观测，17 所学校/68 个校-年级聚类）：按校聚类 `G*(0)=11.121`；按校-年级聚类 `G*(0)=28.349, G*(1)=18.525`；用 score-variance 检验判定应按学校（更粗、`G` 更小）聚类，而非文献里常用的校-年级聚类；WCR-S 与 Hansen 方法的结论最终一致（结果显著）。

### 核心结果

- **"`G=42` 就够"的经验法则被明确否定**——它是严重的过度简化；决定可靠性的是异质性（杠杆分布、处理臂平衡），不是单纯的 `G`。
- **CV1+t(G-1)**（应用经济学里事实上最常用的默认组合）**几乎永远不该被无条件信任**，除非 `G` 很大且异质性很低；两个真实案例里它都明显过度拒绝。
- **CV3/jackknife、Hansen 方法、WCR-S bootstrap 是当前最好的通用默认**，但也不是处处可靠——处理聚类很少时可能严重拒绝不足。
- **有效聚类数可以远小于原始聚类数**——案例一 `G=12` 但 `G*(1)=5.98`；这是"原始 `G` 单独作为可靠性判据会误导"的具体量化证据。
- **正确的聚类层级取决于具体检验目标而非数据集固有属性**——案例二里用更粗（学校）层级比常用的更细（校-年级）层级更可靠，与文献惯例相反，且是用 score-variance 检验推出来的，不是主观选择。
- **诚实的负面结果**：多处报告"某方法在 targeted MC 里拒绝不足，但在 placebo regression 里却过度拒绝"这类互相矛盾的诊断结果，以及"WCR-S 和 Hansen 方法通常一致，但不总是"——不粉饰任何单一方法的普适性。

#### Storm-Cluster Bootstrap 可信度核查协议（草案，供 proposal 下一版直接采用/裁剪）

**阶段 A——训练前，仅用 storm 花名册（零训练成本）**：
1. 报告 `G`（总 storm 数）、`G1`（阳性事件 storm 数）、`G0=G-G1`。按本文结论第 2 条的显式规则："处理型模型不仅要看 `G`，还要看 `G1`，且 `G1/G` 越接近 1/2 越好"——这把 2406.00650 已发现的"`G1` 是第一主因"具体化为一条可预注册阈值：若 `G1<~8-10` 或 `G1/G` 远离 0.5，在 Validation Focus 里显式标记为功效风险，而不是等真实数据到手才发现。
2. 复用 2205.03288 已建议的两个朴素杠杆代理（每个 storm 贡献的阳性窗口占比、总窗口占比），用本文 `V_s^j` 的相同函数形式（`G²/(G-1)·Σ(proxy_g − 1/G)²`）算出一个"有效 storm 数"类比指标，并用本文给出的具体参照——`G*(0)` 远小于 `G`（如 50→11）判为高风险，接近 `G`（如 50→48.7）判为健康——作为预注册的单一阈值统计量，而不是留白到"看情况判断"。
3. 若计划在 P2 真实数据上做任何置换/安慰剂式检验（见阶段 C2），先算清楚该置换空间的组合数，确认落在本文给出的"至少 100、至多几十万"可行区间内（`>=50` storm 几乎必然满足，但必须显式算出并记录，不能默认）。

**阶段 B——留出预测生成后，全量 bootstrap 前（低成本，无需重训练）**：
4. 复用 2205.03288 已建议的 leave-one-storm-out jackknife（对 `ΔAUPRC` 逐一去掉一个 storm 重算），报告 `max|Δ(ΔAUPRC)|`。新增本文带来的具体判据：**若去掉单个 storm 会翻转 POSITIVE/NULL/NEGATIVE 判据，必须单独标注**——本文明确指出杠杆过高的聚类不仅威胁推断可靠性，也说明"模型对每个聚类是否都成立"本身存疑，这比单纯报告 CI 宽度更诚实。
5. 新增一项此前三轮均未提出的检查：比较 raw-only 与 raw+`(D,J)` 两条臂各自跨 storm 的结果异质性（如各 storm 的 AUPRC 或预测分数分布方差）——参照本文式 twovars（检验处理组与对照组残差方差是否系统不同）的精神，若某一条臂在 storm 间的变异明显大于另一条，应作为对称 CI 假设的警示写入报告，而不是默认两臂同方差。

**阶段 C——Bootstrap CI 构造与交叉校验**：
6. 主结果沿用已规划的 storm-cluster 重抽样 percentile/BCa CI。
7. **新增、直接回应 review 的 Validation Focus 缺口 #1**："Gate 0 新旧协议判据是否翻转只能证明 comparator 强弱会改变结论，不证明新协议本身已校准"——本文第 6/7 节的核心方法论是：当 targeted Monte Carlo 与 placebo regression（两种独立、原理不同的校准手段）结论一致时，才有理由信任由此产生的推断。对应到本课题：Gate 0 的 graph-null 校准是"targeted Monte Carlo"一侧（用真实 storm/聚类结构但已知虚无 DGP）；应补一个"placebo"一侧——在 P2 真实 storm 花名册上，把 `(D,J)` 通道与 sham/特权匹配对照通道的臂身份做重新随机分配（类比本文 placebo regressor 的"replace"构造：把真实信号替换成结构相同但内容打乱的安慰剂），直接从真实 storm 数据经验重算 `ΔAUPRC` 的零分布，与 Gate 0 的合成 graph-null 校准做交叉比对。两者一致则增强可信度；不一致本身就是应诚实报告的发现（本文原话：这是一致时"可以放心依赖"的具体判据，而不是"必须一致否则重做"）。
8. 复用 2301.04522 已建议的"多层级都报 CI"（storm-level 为主 + day-block/season-block 为敷衍成本很低的稳健性检查），并新增本文结论段给出的具体决策规则：**若更粗层级的 CI 比 storm-level 更窄，应视为抽样噪声而非更高精度的证据，报告口径以两者中更宽的为准**——这是一条此前三轮都没有的具体、可操作的"不能挑对自己有利的 CI"防呆规则。

**阶段 D——诚实局限（必须写入 Failure Modes，不能被上述协议掩盖）**：

10. IID vs regime-迁移 holdout 之间的患病率不一致问题、BH 校正与 CI 阈值判据的结合方式，本文完全没有涉及（纯线性回归推断论文，不含 ranking 指标或分布漂移内容）——继续按 proposal review 现有建议单独处理，不要指望这条文献线能解决。
11. 该协议只能提高"我们是否该信任这个 CI"的确信度，不能让一个仍然过宽、无法区分 0.01 与 0.02 的 CI 变成可判定结果——诊断全绿但 CI 依旧跨越判据阈值时，仍应诚实报告 NULL，不能因为"检查清单都通过了"就降低实用阈值。

**为什么这次可以给出具体协议而此前三轮只能给"不适用"结论**：前三篇论文各自只提供了协议的一个片段（G1 主因、留一聚类杠杆代理、聚类层级不确定性的诚实处理），且各自明确说"渐近理论/检验公式不适用"；本文作为同一作者群 2026 年的综合性方法论文，第一次把"诊断异质性→targeted MC→placebo regression 交叉校验→给出综合判断"整理成一条完整的端到端流程，而这条流程的每一步骤都只依赖"能否重算这个统计量"这一最弱前提（不需要闭式得分），因此才第一次可以被完整地映射（而非只取用碎片）到 storm-cluster `ΔAUPRC` 场景。
