# Exploring Multi-view Symbolic Regression methods in physical sciences (2509.10500)

- **Year**: 2025.09
- **Authors**: Etienne Russeil (Stockholm University), Fabrício Olivetti de França (UFABC), Guillaume Moinard (Sorbonne Université / CNRS / LIP6), Konstantin Malanchev (Carnegie Mellon University), Maxime Cherrey
- **Venue**: arXiv preprint
- **Keywords**: symbolic regression, multi-view symbolic regression, physical sciences, PySR, Operon, phy-SO, eggp, interpretable models
- **One-liner**: 比较四种 MvSR 工具在真实科学数据上的表现。

## 精华提炼

### 管线/方法

1. 论文研究对象是 Multi-view Symbolic Regression (MvSR)：从多个由同一机制产生的数据集出发，搜索同一个参数化函数形式 `f(x; theta)`。
2. 每个 view 有自己的参数 `theta^i`，函数结构在所有 view 之间共享。
3. 内层优化：对每个 view 独立拟合该 view 的参数，使 `L(f(x^i; theta^i), y^i)` 最小。
4. 外层搜索：SR 算法搜索表达式结构 `f`，再把各 view 的 loss 聚合成一个 fitness。
5. 论文给出统一形式：`min_f agg_i(min_theta_i L(f(x^i; theta^i), y^i))`。
6. 聚合函数可以是 max、mean、median 等；本文主要用 worst-fitness/max 作为比较口径。
7. 比较对象是当前公开支持 MvSR 的四类实现：Operon、PySR、phy-SO、eggp。
8. Operon 使用高性能 C++ genetic programming，MvSR 通过多个 evaluator 独立拟合 views，再聚合得分。
9. Operon 支持多种聚合函数；本文使用原 MvSR paper 中的 worst-fitness。
10. Operon 的关键限制是不能显式限制参数数量，且同一参数不能在表达式中多次出现。
11. phy-SO 使用 RNN 逐 token 生成表达式，原本强调物理单位约束；其 MvSR 版本称为 Class SR。
12. phy-SO 区分 realization-specific parameters 和 class parameters，即 view-specific 参数与跨 view 共享常数。
13. 本文数据没有 ground-truth units，因此 phy-SO 以 dimensionless 设置运行，不能测试其物理单位优势。
14. PySR 使用 Julia/Python genetic programming，多种群并行演化；v1.0 支持 Parametrized Template Expressions。
15. PySR 的 MvSR loss 是各 view loss 求和，支持最大参数数目和全局常数。
16. PySR 允许同一参数在一个表达式中重复出现。
17. eggp 使用 e-graph 记录已访问表达式及其等价形式，避免重复探索等价表达式。
18. eggp 的 MvSR 版本允许多 view 评价、最大参数数目控制、参数重复使用，并采用 worst-fitness 聚合。
19. 实验目标不是给四个工具排绝对名次，而是找出未来 MvSR 实现需要具备的特性。
20. 论文选用真实科学数据而不是合成 ground-truth benchmark，强调 practical scientific modelling。
21. 所有数据都是一维输入，这让专家更容易检查表达式形状，但也限制了结论外推。
22. 所有 view 内部随机划分 80% training 和 20% testing。
23. 表达式结构只在 training set 上搜索。
24. testing set 评估时不重新优化参数，以测量插值和局部外推能力。
25. 主要指标是按自由度归一化的 reduced MSE：误差除以 `(n_obs - n_params)`。
26. 这个指标同时惩罚拟合误差和参数数量，适合比较可解释小公式。
27. 每个 view 独立算 reduced MSE，再用最大值作为该候选表达式的最终 loss。
28. 作者的理由是：MvSR 假设所有 view 共享同一机制，不能只拟合大部分 view 而放弃最差 view。
29. 操作符集合统一为 `+`, `-`, `*`, `/`, `exp`, `log`, `sqrt`, `abs`, `pow`, `square`。
30. 搜索超参数用 2 x 2 组合：最大表达式 size 为 15 或 30；最大 view-specific 参数数为 2 或 4。
31. 每个数据集、每个方法、每个超参配置重复 10 个随机种子。
32. 对 PySR 和 phy-SO，额外测试全局常数；phy-SO 常数数目限制为 2，PySR 不限制常数数目。
33. 各方法大部分超参数保持默认；共同设置为 100 generations、population size 100、参数优化 3 iterations。
34. 每个实验设置最大运行时间 30 分钟。
35. 由于 Operon 单次实验非常快，作者额外设置 `operon_increased`：1000 generations、population size 1000、参数优化 100 iterations。
36. `operon_increased` 的结果用于说明 Operon 在更现实预算下的能力，但不能和参数受限方法完全公平比较。

### 关键技术细节

1. 五个真实多 view 数据集覆盖天体物理、生物化学、流体、网络科学等领域。
2. Supernovae light curves：6 条 light curves，每条 147 个点，是样本点最多的 view。
3. Galaxy rotation curves：26 个高质量 rotation curves，由 3200 多个 local-universe galaxies co-added 构成。
4. Michaelis-Menten：5 个 view，每个 view 平均 6 个点，是最小样本、最容易过拟合的数据集。
5. Nikuradse：6 个 pipe roughness views，每个 view 平均 60 个点，用于测试难建模湍流摩擦曲线。
6. Network degree distribution：从 KONECT 中选 8 个 views，每个 view 平均 79 个点。
7. KONECT 数据库原始规模很大，论文提到可包含最高 68,349,466 个 nodes 与 2,586,147,869 条 links 的网络。
8. 数据未使用 uncertainty / error bars，因为一些数据集没有误差条，且部分实现不能处理 uncertainty。
9. 作者明确指出只用 PySR 和 phy-SO 能处理 weighted least-squares uncertainty。
10. 训练和测试的参数处理很关键：测试时不重新拟合参数，因此 testing loss 更像“冻结参数后的泛化检查”。
11. 论文将 training loss 低于 `5e-3` 视为 good model 的经验阈值。
12. 全局统计图把所有数据集、超参和随机种子的结果堆在一起，只显示 reduced MSE 到 `5e-2` 的范围。
13. 所有超过 `5e-2` 的结果被聚合到 clipping bin。
14. 表达式复杂度用平均节点数和 testing MSE 的 Pareto front 观察。
15. 表达式表展示每个数据集、每个方法最低 `MSE_train` 的公式。
16. 作者承认实验不是实际用户使用方式：真实用户会反复调参、挑选候选表达式、手动化简。
17. 因此本文结果应看作保守下界，而不是每个工具的最好能力。
18. 计算时间差异很大：PySR 和 phy-SO 平均约 1000 秒量级，eggp 约 50 秒，Operon 原设置约 0.1 秒。
19. Galaxy 数据上 PySR 最慢，平均 1785.6 秒，并触及 30 分钟上限。
20. eggp 在五个数据集上平均约 19.4 到 83.0 秒。
21. 原始 Operon 五个数据集都约 0.1 到 0.3 秒；`operon_increased` 也只有约 1.3 到 7.9 秒。
22. 这个效率差异说明方法比较不能只看 fixed evaluations，也要看 wall-clock budget。

### 核心结果

1. 总体上，四类 MvSR 方法都能在强约束和未精调设置下发现不少准确且较简单的公式。
2. 原始 Operon 在相同 evaluations 下表现偏弱，但这主要来自计算预算过低。
3. `operon_increased` 显著提高了 good-model 频率，并且仍远快于 PySR / phy-SO。
4. 但 `operon_increased` 不限制参数数量，经常产生过参数化表达式，不能只按 MSE 宣称最好。
5. eggp 和 phy-SO 的整体结果相近，eggp 倾向于产生更多低误差模型。
6. PySR 产生 good models 的频率较低，但在多个数据集上能给出最佳或非常强的单个表达式。
7. 大多数方法从 training 到 testing 的 loss 都明显变差，作者认为这更多是 view-specific 参数拟合过度，而非表达式结构本身过拟合。
8. Supernovae 上，PySR 和 phy-SO 最物理合理，能描述 light curve 的指数上升和衰减。
9. Supernovae 上，Operon 给出类似对称 Gaussian 的形状，eggp 会让归一化 flux 超过 1，物理可信度较差。
10. Supernovae 上，PySR 公式拟合好但用了 7 个全局常数，解释性弱；phy-SO 只用 1 个常数，更适合专家接受。
11. Supernovae 中所有方法都未能描述 maximum 后约 30 天的 re-brightening，这是明确失败点。
12. Galaxy rotation curves 上，所有方法都能达到较高精度，并能描述大半径 slowdown。
13. Galaxy 上 PySR 的模型在 test set 上最准确，同时只需 2 个参数、1 个常数和较小表达式。
14. Galaxy 数据 26 个 views 使 PySR 的计算效率受到明显冲击。
15. Michaelis-Menten 上，多数方法都能在 training 上高精度拟合，Operon 因短 runtime 失败。
16. Michaelis-Menten 的 test MSE 不可靠，因为每个 view 点数太少。
17. Michaelis-Menten 上，phy-SO 的解最物理合理，符合转换率从 0 到 1 渐近的先验。
18. phy-SO 在 Michaelis-Menten 上用 2 个参数和 2 个常数给出符合边界行为的模型。
19. Nikuradse 上，phy-SO 的解存在 complex-valued outputs，不能完整显示。
20. Nikuradse 上，Operon 和 eggp 拟合总体准确，但难以精确刻画曲率。
21. Nikuradse 上，PySR 用 1 个参数和 3 个常数给出很简单且准确的表达式。
22. Network degrees 上，Operon 明显失败；phy-SO 能拟合但曲率不足。
23. Network degrees 上，PySR 和 eggp 都产生准确模型，约 3 个参数，PySR 另含 2 个常数。
24. 论文的最终工程建议有三条：参数数目控制、常数数目控制、聚合策略可配置。
25. 参数控制是 MvSR 的核心功能，因为它直接推动公式更简单、更相关。
26. 常数有用，但不限制会导致 PySR 式常数滥用，使表达式难解释。
27. worst-fitness 和 mean 聚合没有一方明显胜出；应由专家根据数据性质选择。
28. 若所有 view 都同等重要，worst-fitness 更合理；若大样本包含 outliers，mean 可能更稳。
29. 作者建议每个 MvSR 算法都应导出 top-N visited solutions，便于专家二次选择和工具分析。
30. 作者特别点名 rEGGression 这类交互式工具可用于探索候选表达式。
31. 论文强调 uncertainty handling 是 MvSR 未来必须补上的功能。
32. 结论是：MvSR 在多实验条件、多对象、多实例科学数据中有很强潜力，尤其适合寻找共享机制下的少参数经验公式。

### 明确局限

1. 所有 benchmark 都是一维输入，不能证明方法能处理高维 PDE、PIDE 或高维物理系统。
2. 数据是实证数据但没有 ground truth 公式，因此无法衡量 exact recovery。
3. 没有使用 error bars 或 uncertainty，削弱了科学建模的可信度。
4. 五个数据集的领域差异大，但任务形态仍是曲线拟合，不是 PDE residual minimization。
5. 未测试 FEX 场景最关心的高维函数空间、边界条件、微分算子 residual、积分项或 PDE 解结构。
6. 所有方法都采用统一简化超参，实际工具能力可能被低估。
7. 运行预算固定但方法计算效率差异极大，公平性只能近似。
8. Operon 的参数不受限让 `operon_increased` 结果和其他方法不完全可比。
9. PySR 常数不受限会产生解释性差但误差低的表达式。
10. phy-SO 的物理单位能力没有被测试，因为数据没有指定参数单位。
11. testing set 不重拟合参数是严格评估，但也可能低估实际使用中“新 view 参数重新拟合”的能力。
12. 论文主要比较已有实现的 feature，而非提出新的 MvSR 算法。
13. 结论更多是工程 guideline，不是理论贡献。
14. 没有开箱即用地给出 FEX/RL controller 的搜索复杂度或收敛分析。
15. 对 `0616-fex` 来说，它是高价值相关工作，但不是直接替代 FEX 的 PDE solver。

### 跟我们的关系

1. 对 `0616-fex`，这篇不是直接撞车 FEX 的核心 claim。
2. FEX 是 RL controller 搜索有限表达式空间，用于 PDE/物理方程求解和物理律发现；本文是多 view 曲线数据上的 MvSR 工具比较。
3. 直接重叠点是“符号回归如何在多个相关数据集之间共享表达式结构”。
4. 如果 FEX 后续要处理同一 PDE family、不同参数实例、不同初边值条件或多实验数据，MvSR 的 `shared structure + per-view parameters` 很值得吸收。
5. FEX 当前的表达式搜索通常把一个任务作为单独 residual minimization；MvSR 提醒我们可以把多个任务实例合并成一个结构搜索问题。
6. 这对 `fex-dim-lift-skeleton`、`fex-curriculum-depth`、`fex-synchro-prior` 都有启发：先锁定共享表达式骨架，再为每个 view/维度/频率实例拟合少量参数。
7. 本文最可复用的设计是 worst-fitness 聚合：FEX 若要求所有 PDE instances 都满足误差阈值，max residual across tasks 比 mean residual 更符合安全目标。
8. 第二个可复用点是参数数目控制：FEX 的候选表达式也应对 free coefficients 做显式惩罚，否则高 MSE 改善可能来自过参数化。
9. 第三个可复用点是 top-N solution export：FEX controller 不应只输出 best expression，而应输出 Pareto 前沿与多样候选，便于专家审查。
10. 第四个可复用点是 uncertainty/error bar 处理：若 FEX 走实验数据物理律发现路线，weighted residual 或不确定性传播必须补上。
11. 需要引用吗：如果 `0616-fex` paper 涉及多任务、多实例、多实验条件下的符号发现，应引用；若只做单 PDE solver，相关性中等，可放在 related work 的 SR benchmark/MvSR 段落。
12. 差异化：FEX 的优势仍是 PDE residual、有限表达式空间理论、Adam+BFGS 参数优化和高维 PDE/PIDE 场景；本文没有覆盖这些。
13. 撞车风险：低到中。低是因为任务形态不同；中是因为 `shared expression + per-instance parameters` 可能与 FEX family-level solver idea 重叠。
14. 最新性检查：arxiv 工具检索到 2025.12 的 SR physical sciences 综述和 2025.11 的 SR extrapolation/materials 工作，但没有发现更新的直接 MvSR benchmark 取代本文。

## Read by: 0616-fex (2026-06-17)

本次从 FEX / RL for Symbolic Regression 视角审读。结论：这篇不直接撞车 FEX 的 PDE solver 主线，但对“多实例共享表达式骨架”的 FEX 扩展很有参考价值。论文的核心贡献是把 MvSR 的工程现实讲清楚：共享结构、每 view 独立参数、loss 聚合、参数/常数数目控制、top-N 候选导出、uncertainty 处理，都会影响公式是否真正可用。

对 `0616-fex` 最有用的部分是 worst-fitness 聚合和参数控制。FEX 若要从单 PDE 扩展到 PDE family、不同初边值条件、多尺度频率实例或实验数据物理律发现，可以把多个实例作为 views，让 controller 搜索共享表达式树，再为每个实例拟合少量参数。这个设定与本文 MvSR 完全同构，但 FEX 的差异化在于 loss 是 PDE residual / boundary residual，而不是普通曲线 MSE。

撞车风险为低到中：低，因为本文不做 PDE residual、RL controller 收敛、FEX 的 `S_k` 理论或高维 PIDE；中，因为如果我们提出“多任务 FEX 共享骨架 + per-instance coefficients”，本文必须作为 MvSR prior 引用。可用方法包括：max-over-views residual 聚合、free-coefficient penalty、限制 global constants、导出 Pareto front、用 rEGGression/e-graph 工具审查等价表达式。差异化应强调 FEX 的 PDE 约束、理论表达空间、跨维/跨任务 transfer，以及可审计的 residual verification。
