# Grammar-based Ordinary Differential Equation Discovery (2504.02630)

- **Year**: 2025.04
- **Authors**: Karin L. Yu, Eleni Chatzi, Georgios Kissas (ETH Zurich, ETH AI Center, Institute of Structural Engineering)
- **Venue**: arXiv
- **Keywords**: Dynamical Systems, Ordinary Differential Equations, Symbolic Regression, Formal Grammars, Equation Discovery, Deep Generative Models, GVAE, CMA-ES, Implicit ODEs, Context-Free Grammar
- **One-liner**: GODE combines context-free grammars with a grammar variational autoencoder (GVAE) and CMA-ES stochastic search to discover ODEs from noisy/sparse trajectory data; grammar-based embedding is orders of magnitude more sample-efficient than tokenizer-based transformer approaches.

## 精华提炼

### 管线/方法

GODE pipeline 分四阶段:

1. **Grammar-generated candidate ODEs**: 使用 Context-Free Grammar (CFG) 定义候选表达式的生成规则。CFG 是 tuple {T, N, P, S}（terminal, non-terminal, production rules, start symbol）。每一条 ODE 对应 grammar 中的一串 rule sequence r = [r1, ..., r_nr]。用 padding rule r_p 补齐到最大长度 N_max。关键优势：grammar 保证所有生成表达式句法有效（syntactically valid），从根本上消除了遗传编程方法的组合爆炸问题。Grammar 通过 nltk chart parser 定义。

2. **Manifold learning via GVAE**: 使用 Grammar Variational Autoencoder (GVAE, Kusner et al. 2017) 将离散 rule sequence 嵌入连续低维 latent space。Encoder: one-hot encoding of rules → 3 层 CNN (ReLU) → latent distribution q(z|X)。Decoder: reparametrized z → 3 层 bidirectional GRU (ELU) → 预测 rule sequence p(X|z)。Loss: L_BCE (binary cross-entropy with Sigmoid) + beta_KL * L_KL (KL divergence)，beta_KL = 1e-3 到 1e-4。Adam optimizer，初始 lr=0.001，adaptive scheduler (factor=0.9, patience=400-500, min_lr=5e-5 到 1e-4)。

3. **Latent space optimization (two-stage)**: 
   - 外循环 (Stage 1): CMA-ES (Covariance Matrix Adaptive Evolution Strategy)，gradient-free non-convex optimization。Population size 100-500，5-10 generations，initial sigma=0.5。从 multivariate normal distribution 采样，自适应更新均值和协方差矩阵。
   - 内循环 (Stage 2): Nelder-Mead 方法优化常数 (scalar optimization)。GVAE 编码阶段只 embed 表达式骨架（skeleton），常数用占位符 C 表示，在第二阶段单独优化。这避免了 CFG 用 structural non-terminals 表示实数导致的极长 rule sequence（例如精度为3的数字 1.234 需要 9 次 rule application）。
   - Preprocessing: MLP (3 layers, 32 neurons, SiLU activation) + automatic differentiation 对噪声数据进行平滑并估计导数。Loss 包含 prediction loss + anti-drift loss + initial conditions loss（用于含二阶导的复杂场景）。

4. **Trivial equation filtering**: 对候选表达式在一组测试解 S* = {t, -2.5t, sin(3t), 2cos(t/4)+t/3} 上评估，若所有解的平均残差都低于阈值 epsilon，则判为 trivial 并返回高惩罚。这解决了隐式 ODE 发现的 trivial solution 问题（如 du/dt + 2 - du/dt - 2 = 0）。

### 关键技术细节

- **模型架构**: GVAE encoder 用 3 层 CNN，kernel sizes 7/8/9（benchmark-dependent），decoder 用 3 层 bidirectional GRU，hidden dim=80。Latent dimension 21-26（取决于 benchmark）。GODE 参数量远小于 ODEFormer（ODEOformer ORG: 60.7M params，GODE 的 GVAE 约数十万到百万级别）。
- **训练数据规模 (Benchmark 1)**: 仅 10,000 skeletons 用于 GVAE 训练（9:1 train/test split），15,000 skeletons 用于 ODEFormer 训练数据集（integration 耗时约 5 小时 on MacBook Pro M3）。对比 ODEFormer 原始论文使用 50 million 训练样本。GVAE 单 GPU (RTX 4090) 训练 10-20 分钟，ODEFormer 原始模型训练 3 天 on A100。
- **优化目标**: 两个 loss——L_DE (ODE residual loss) 和 L_SOL (standardized MSE of solution trajectories)。先计算 L_DE，如低于阈值 theta_DE（初始 200，后续动态调整为当前迭代 best 20 的均值 + 5% buffer）才计算 L_SOL。Model selection criteria L_IC = alpha*C + (1-alpha)*n_s*L_accuracy，alpha=0.1 (偏重 accuracy)，C 为 complexity metric（常量数+变量数+操作数，SymPy 自动提取）。
- **Grammar 设计**: Benchmark 1 (29 rules, N_max=40)，Benchmark 2 (generation: 29 rules/N_max=50, GVAE: 24 rules/N_max=70)，Benchmark 3 (generation: 26 rules/N_max=40, GVAE: 22 rules/N_max=65)。Generation grammar 使用 probabilistic CFG（更复杂的 rule 分配更低概率），GVAE grammar 更简化以增加灵活性。Dataset generation 仅需 1-2 分钟（纯 symbolic），无需解 ODE。
- **Baseline 配置**: ODEFormer (beam search: beam=50, temp=0.1, metric=R2)，PySR v0.19.4 (populations=20, pop_size=30, iterations=20, model_selection='best' or 'score')，ProGED v0.8.5 (grammar='universal', sample_size=100, max_repetitions=100)。
- **评测指标**: Relative L2 error = ||u_pred - u_true||_2 / ||u_true||_2。对所有方法做 5 次独立 run，加 5% Gaussian noise。采样频率 4-100 Hz，50-300 evaluation points。误差 >1 时 cap 到 1。

### 核心结果

| Benchmark | 任务 | 最好方法 | GODE 表现 | 关键发现 |
|-----------|------|---------|----------|---------|
| B1: 30 explicit 1D ODEs | 一阶显式 ODE | PySR (mean L2 u_t=0.084) | mean L2 u_t=0.150, 优于 ODEFormer ORG (0.148) | GODE 参数少 ~190x，训练样本少 ~1000x，仍胜 ODEFormer；但 PySR 在该简单任务上最强 |
| B2: 10 implicit LODE/NLODE | 1阶/2阶 线性非线性隐式 ODE | GODE | mean L2 u_t=0.120, L2>1 例数=0, complexity=0.90 (最低) | GODE 同时获得最高精度和最简表达式；PySR score 模式 simplest (1.04) 但精度差 |
| B3: 3 nonlinear dynamics | 工程实例 (pendulum, Duffing, Van der Pol) | GODE | Pendulum: 完全正确 (coeffs ~1% error), Duffing: 结构正确 (coeffs 偏), Van der Pol: 结构接近正确 (cos(0.00032t)≈1 for small t) | ProGED 完全失败；PySR 给出不可解释的嵌套表达式；GODE 是唯一成功的 |

**Surprising results**:
1. ODEFormer 64/128（减少参数/数据的版本）表现极差，mean L2 u_t=0.454-0.524，13/30 例子误差 >1。说明 tokenization 的 sample inefficiency 是根本性的，不是调参问题。
2. GODE 在 B1 略逊于 PySR 但 B2/B3 全面超越，说明 grammar-guided search 在复杂任务上的优势随复杂度增长而放大。
3. ProGED（也是 grammar-based）在 B2/B3 表现远不如 GODE，差异在于 GVAE 的连续 latent space search 远优于 ProGED 的 Monte-Carlo 离散搜索 + differential evolution 常数优化。

### 明确局限

**作者自己说的局限**:
1. 需要用户手动设计 grammar——提出了 grammar induction 作为 future work
2. Semantic ambiguity 问题未解决：交换律/结合律/分配律导致同一表达式有多个等价 representation（CFG 只保证 syntactic validity）
3. 目前限于 ODE，未扩展到 PDE 和守恒律
4. 未探索多模态训练（同时使用 numerical trajectory + symbolic expression）
5. ODEFormer 淘汰是因为训练成本太高无法适配新任务，不是方法本身不可行

**我读出的额外局限**:
1. 没有任何真实物理实验数据验证——所有 benchmark 都是 synthetic + 5% Gaussian noise。Real-world noise structure（sensor drift, bias, colored noise）未测试
2. 代码未公开（只说 "at the time of publication"），复现性打折扣
3. 未与 SINDy（dictionary-based）直接比较，理由是需要 pre-selected library 不公平，但这也是 domain knowledge integration 的最直接方案
4. Grammar 设计依赖专家知识——不同 benchmark 用了不同 grammar，论文未讨论 grammar 选择的敏感性分析
5. 搜索算法 (CMA-ES) 有随机性，5 次 random initialization 取 best，这在实际应用中需要多次 run 才能获得可靠结果
6. Benchmark 用的一维 ODE 规模偏小，未测试多变量耦合 ODE 系统的 scaling
7. Van der Pol 案例中 GODE 未能精确恢复非线性阻尼项（cos(0.00032t) 近似代替），说明常数优化在高维常数空间仍有困难
8. 效率对比中未包括 total inference time——GODE 需要 MLP 训练 + CMA-ES 搜索 + Nelder-Mead，而 ODEFormer 是 single forward pass

### 跟我们的关系

**对 0616-fex 课题的评估**:

1. **撞车评估**: 不直接撞车。GODE 是 ODE discovery pipeline，0616-fex 如果是关于从 LLM 提取功能表达/特征，两者层次不同。但如果 0616-fex 涉及从数据中自动发现函数形式（symbolic regression），则 GODE 的 grammar-guided search 范式提供了参考框架。

2. **可用工具/方法**:
   - **GVAE + grammar 的组合方式**: 将离散结构化空间 (grammar rules) 嵌入连续 latent space 然后搜索，这个思路可以移植到任何需要结构化搜索的场景。如果 0616-fex 涉及从 LLM 骨架中搜索最优功能表达，GVAE 可能作为候选编码器。
   - **Two-stage optimization (skeleton + constants)**: 分离结构搜索和参数优化是一个通用策略，可借鉴。
   - **CMA-ES over latent space**: 在连续 latent space 中做 stochastic search 比在离散 space 中做 MCTS/beam search 更高效，值得考虑。
   - **Trivial solution filtering**: 使用一组测试输入来检测退化解的方法可迁移到任何 generative + search pipeline。

3. **该引/不该引**: 
   - 如果 0616-fex 涉及 grammar-guided generation、latent space search、或 symbolic discovery——该引。
   - 如果 0616-fex 纯粹是 LLM pipeline 不涉及函数发现——不直接引用，但 GVAE 方法论可放入 related work。

4. **差异化**: GODE 的 grammar 是手工设计的专家规则，0616-fex 如果用 LLM 来生成/学习 grammar 或直接替换 grammar 的作用，这是直接的差异化方向。GODE 的 outlook 中也提到了 grammar induction 作为 future work。

## Read by: 0616-fex (2026-06-16)

本论文的审读角度是评估 GODE 对 functional expression extraction (fex) 相关研究的参考价值。

**直接相关度**: 中等偏高。GODE 处理的 ODE discovery 是 functional expression extraction 的一个特例——从 trajectory data 中提取 governing equation。0616-fex 如果关注的是从数据/模型中提取可解释的函数表达式，GODE 提供了以下可借鉴的范式：

1. **Grammar as structured prior**: GODE 用 CFG 定义搜索空间的边界和结构，这是一种介于完全自由搜索 (genetic programming) 和过度限制 (dictionary-based) 之间的优雅折中。0616-fex 如果涉及从 LLM 输出中约束功能表达的形式，grammar 是个可参考的工具。

2. **Skeleton-then-constants decomposition**: 分离结构发现和参数优化是提高搜索效率的关键设计。GVAE 只 embed 表达式骨架（不含常数），这样 latent space 更紧凑、训练更高效。对于 fex 场景——如果是从大量候选功能表达中筛选——这个分解策略值得采纳。

3. **Continuous latent space search vs discrete search**: GODE 的核心 insight 是 "embed discrete into continuous then search"，这比直接在离散空间搜索（遗传编程、MCTS）高效得多。0616-fex 如果面临大规模离散候选空间搜索问题，可以考虑类似策略。

4. **Trivial solution detection**: 使用 hold-out test inputs 来检测退化解的方法，对任何 generative + search pipeline 都有价值。

**不撞车**: 0616-fex 如果是 LLM-centric 的功能表达提取，与 GODE 的数据驱动 ODE 发现不直接竞争。两者可以在以下方向互补：GODE 的 grammar 是 hand-crafted，可以用 LLM 来自动化 grammar design/induction（这正是 GODE 论文 outlook 中提到的 future work）；LLM 生成 candidate expressions 的质量也可以用 grammar 来约束和验证。

**可借用的具体组件**: GVAE 架构（CNN encoder + GRU decoder over rule sequences）、CMA-ES over latent space 的搜索实现、L_IC 的 complexity-accuracy trade-off 公式。
