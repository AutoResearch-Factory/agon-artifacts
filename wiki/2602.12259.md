# Think like a Scientist: Physics-guided LLM Agent for Equation Discovery (2602.12259)

- **Year**: 2026.02
- **Authors**: Jianwei Yang, Ohm Venkatachalam, Mohammad Kianezhad, Sharvaree P. Vadgama, Rose Yu (UCSD)
- **Venue**: arXiv preprint, ICML 2026-style preprint
- **Keywords**: equation discovery, symbolic regression, LLM agent, PySINDy, PySR, symmetry discovery, physics-guided tools, DiffEq benchmark
- **One-liner**: LLM agent 先发现物理结构再配置 SR 工具, 显著提高方程发现准确率。

## 精华提炼

### 管线/方法

1. 论文目标是把 LLM 从“一次性猜方程”的角色改成“科学家式工具编排器”：先分析数据, 再调用物理结构工具, 最后配置符号回归后端。
2. 输入是观测数据集, 形式可以是一般输入-输出变量, 也可以是 ODE/PDE 系统里的状态变量和导数目标。
3. KeplerAgent 使用单个 GPT-4o-mini 主 agent, 运行在 ReAct-style 循环中。
4. 每一步输入包含四块：tool specification、user query、workspace files 摘要、experience log。
5. workspace 保存原始数据、图像、统计文件、工具输出和中间结果；experience log 保存每次工具调用参数、损失、方程、结构发现等文本摘要。
6. agent 被要求先查看 workspace 和 past experience, 避免重复调用完全相同参数的工具。
7. 如果已有 SR 结果达到 MAPE < 0.1%, system prompt 要求 agent 立即停止并返回最低误差方程。
8. 第一类工具是 Python code interpreter, 只用于探索性数据分析, 如基本统计、可视化、数据转换；prompt 明确不让它直接做最终 symbolic regression。
9. 第二类工具是 visual subagent, 用 GPT-4o-mini 读取数据图像, 输出趋势、周期性、噪声、可能函数形式、建议的 SR operators。
10. 第三类工具是 symmetry discovery, 这是论文最核心的 physics-aware 中间结构工具。
11. symmetry discovery 先训练一个 tiny 2-layer MLP surrogate, 用状态变量预测导数或 RHS。
12. 然后固定 surrogate, 优化一个 q x q 矩阵 A, 使线性向量场 v_A=(Ax)·∇ 对 surrogate 满足 equivariance loss。
13. 对 ODE/PDE 系统, A 表示 dependent variables 上的 infinitesimal symmetry generator；例如 [[0,-1],[1,0]] 表示二维旋转对称。
14. 这个 symmetry tool 只处理 dependent variables 上的线性 symmetry, 不处理作用于时间/空间变量或非线性 group action 的一般 Lie symmetry。
15. 方程求解工具有 PySINDy 和 PySR。
16. PySINDy 用 sparse regression, 假设 RHS 是候选函数库的稀疏线性组合。
17. KeplerAgent 在 PySINDy 中额外暴露 Equivariant SINDy 选项, 可以把 symmetry generator 转成 SINDy 参数空间约束。
18. 这使 agent 能完成“发现数值 symmetry -> 解释成标准物理 symmetry -> 作为 downstream SR 约束”的闭环。
19. PySR 用 genetic programming 搜表达式树, 支持更一般的 rational、nested、trigonometric 表达式, 但搜索空间更大。
20. KeplerAgent 给 PySR 暴露的关键接口是 expression_spec, 即模板表达式。
21. expression_spec 包含 placeholder functions、variable_names 和 combine 字符串, 例如 f(x1,x2)+g(x3) 或 sin(f(x))*g(y)。
22. 论文认为让 agent 选择模板比调 population size 等底层超参数更有用, 因为模板直接缩小表达式树搜索空间。
23. Agent 的 stopping rule 以 MAPE < 0.1% 为主目标, 但作者承认 noisy real-world data 可能让 ground truth 也超过这个阈值, 因此实现里加了最大 tool-call 数防止无限循环。
24. 一个反应扩散系统 case study 展示了真实 reasoning trace：初始 SINDy 得到 MAPE 70.033%, symmetry discovery 得到近似旋转 generator, agent 把它 round 成 [[0,-1],[1,0]], 再用 symmetry-constrained SINDy 降到 MAPE 15.584%。
25. 这个 case study 的亮点不是最终误差很低, 而是 agent 能把 noisy numerical symmetry 输出解释为 exact physical generator。
26. 同一 case study 也暴露了弱点：agent 后续重复尝试增加 polynomial degree 和调 threshold, 没有及时判断噪声/参数优化才是瓶颈。

### 关键技术细节

1. 主 LLM 和 visual subagent 都使用 GPT-4o-mini。
2. LLM-SR baseline 也使用 GPT-4o-mini, 因此 agent 框架与 LLM-SR 的模型能力基本对齐。
3. LLM-SR 在 ODE/PDE 数据集上最多采样 500 个 equation proposals, 每个方程最多 10 个 free parameters。
4. LSR-Transform benchmark 上 LLM-SR 最多采样 200 个 proposals, 并记录 wall-clock runtime 与 token usage。
5. PySR baseline 使用 Xia et al. 2025 setup：125 iterations, cycles per iteration 550, populations 15, population size 33, max size 30, randomization weight 0.1。
6. PySR baseline operators：binary {+, -, *, /, pow}, unary {exp, log, sqrt, sin, cos}。
7. PySINDy tool 暴露 polynomial_degree 默认 3、derivative_order 默认 2、normalize_columns、threshold、use_symmetry、lie_generator。
8. Symmetry discovery 的 equivariance loss 是 E_x || J_f(x)·((Ax)·∇)(x) - ((Ax)·∇)(f(x)) ||^2。
9. 该 loss 参考 Yang et al. 2024 Proposition 3.3 / Eq. (5), 原本用于 symmetry regularization, 这里改成优化未知 generator A。
10. Benchmark 之一是 LSR-Transform, 来自 LLM-SRBench, 包含 AI Feynman 方程的变换版本, 通过更换目标变量降低 LLM 记忆经典公式的风险。
11. LSR-Transform 实验覆盖 111 个 equations。
12. 第二个 benchmark 是作者自建 DiffEq, 包含 10 个 ODE/PDE 系统, 每个系统两个 dependent variables 和两个 target equations, 总计 20 个 equations。
13. DiffEq 的前 7 个系统来自 Strogatz nonlinear dynamics textbook, 并使用 La Cava et al. / PMLB 相关数据。
14. 前 7 个系统每个有 4 条 trajectory, 每条 100 timesteps。
15. 另外 3 个系统来自 Yang et al. 2024 symmetry experiments：damped oscillator、growth、reaction-diffusion。
16. damped oscillator 数据有 50 条 trajectory, 每条 20 timesteps, Δt=0.2, 按 trajectory 80/20 划分 train/test。
17. growth 数据有 10 条 trajectory, 每条 100 timesteps, Δt=0.02, 同样 80/20 划分。
18. reaction-diffusion 使用 Champion et al. 数据, 单条 trajectory 为 128 x 128 x 200 array, 前 160 time steps 训练, 后 40 steps 测试。
19. reaction-diffusion 训练时因 full grid 过大, 对 baselines 随机采样 2000 points。
20. noisy DiffEq 构造方式是对每个 state variable 加 white noise, σ_i = σ_R * std(x_i)。
21. 除 reaction-diffusion 外, noise level σ_R=0.01；reaction-diffusion 用 0.0001。
22. noisy data 的 target derivatives 用 central difference 从 noisy trajectories 估计。
23. symbolic accuracy 用 GPT-4o-mini evaluator 判断 hypothesis 与 ground truth 是否等价, 允许小系数误差。
24. 数值指标包括 pointwise NMSE 和 DiffEq 上的 long-term prediction NMSE at t_max。
25. long-term prediction 是从 clean test trajectory 的初始点出发, 用 discovered equation 做 ODE/PDE solver 积分到最大时间, 再比较最终状态。

### 核心结果

1. LSR-Transform 上, KeplerAgent @1 的 symbolic accuracy 是 35.14%, PySR 是 37.84%, LLM-SR 是 31.53%。
2. LSR-Transform 上, KeplerAgent @3 的 symbolic accuracy 是 42.34%, 超过 PySR 和 LLM-SR。
3. LSR-Transform 上, KeplerAgent @1 runtime 238s, tokens 42k；@3 runtime 698s, tokens 125k。
4. 对比之下, PySR runtime 2440s, LLM-SR runtime 2118s, LLM-SR tokens 209k。
5. LSR-Transform 上 LLM-SR 的平均 NMSE 最低, 论文解释为 LLM-SR 直接以类似 MAPE 的数值指标驱动, 更偏向低误差而非简洁/正确 symbolic form。
6. KeplerAgent @3 在 LSR-Transform 上平均 NMSE 为 1.21, PySR 为 2.82, LLM-SR 为 0.09；表格说明 NMSE 已按 x10^-1 scale 报告。
7. DiffEq clean data 上, KeplerAgent symbolic accuracy 75%, PySR 40%, LLM-SR 30%。
8. DiffEq noisy data 上, KeplerAgent symbolic accuracy 45%, PySR 15%, LLM-SR 10%。
9. DiffEq clean pointwise NMSE：KeplerAgent 0.04, PySR 0.16, LLM-SR 0.26。
10. DiffEq noisy pointwise NMSE：KeplerAgent 0.15, PySR 5.89, LLM-SR 4.80。
11. DiffEq noisy long-term NMSE at t_max：KeplerAgent 0.33, PySR 2.80, LLM-SR 2.62。
12. DiffEq clean long-term NMSE at t_max 上 PySR 1.56 略好于 KeplerAgent 1.65, LLM-SR 2.18。
13. DiffEq runtime 上, KeplerAgent clean 120s、noisy 147s, 与 PySR 119/120s 接近, 远低于 LLM-SR 3648/4048s。
14. DiffEq token usage 上, KeplerAgent clean 23k、noisy 30k, LLM-SR clean 182k、noisy 184k。
15. long-term prediction curves 中, clean regime 下 KeplerAgent 在几乎所有系统维持较低误差, 但 van der Pol oscillator simulation failed。
16. noisy regime 下, KeplerAgent 在 10 个系统中的 6 个最好或接近最好：predator-prey、shear flow、van der Pol、damped oscillator、growth、reaction-diffusion。
17. surprising result 之一：default PySR 在 LSR-Transform 单次 symbolic accuracy 高于 KeplerAgent @1, 但 runtime 大约是 10 倍。
18. surprising result 之二：LLM-SR 的 NMSE 很低但 symbolic accuracy 不高, 支持“数值拟合好不等于发现正确方程”的判断。
19. surprising result 之三：KeplerAgent noisy DiffEq 上 long-term NMSE 大幅优于 baselines, 说明物理结构约束对 noisy derivative estimation 很关键。
20. 最强实验 claim 是：在 coupled ODE/PDE 和 noisy data 上, 先发现中间物理结构再配置 SR, 比直接 LLM-SR 或固定 PySR 更可靠。

### 明确局限

1. 工具箱很小：只有 Python interpreter、visual subagent、symmetry discovery、PySINDy、PySR。
2. 作者明确说未来应加入 extended symmetry analysis、dimensional analysis、bifurcation structure 等工具。
3. symmetry discovery 只覆盖 dependent-variable linear symmetries, 对一般 Lie symmetry、作用于 time/space variables 的 symmetry、非线性 symmetry 都不覆盖。
4. Prompt/tool spec 增多会导致 context inflation, agent 更难记住 earlier information。
5. 作者提出 workflow graph、多节点 subagents、RAG tool selection 作为未来方向, 但本论文没有实现。
6. stopping criterion MAPE < 0.1% 对 noisy datasets 不合理；实现里只能靠 max tool calls 截断。
7. case study 证明 agent 会在后期重复尝试类似 SINDy 配置, 对失败模式和噪声诊断不足。
8. evaluation 的 symbolic accuracy 依赖 GPT-4o-mini evaluator, 不是形式化 CAS equivalence checking, 有 evaluator bias 风险。
9. LSR-Transform 的 LLM-SR NMSE 引自 LLM-SRBench, 只有 average, 不具备完整 distribution 对比。
10. DiffEq benchmark 是作者自建且规模较小：10 systems / 20 equations, 其中许多是 textbook systems。
11. 没有真实实验台架或湿实验验证；数据多为 synthetic/simulated physical systems。
12. 没有看到代码、数据 artifact 或完整 tool implementation release 信息, 可复现性受限。
13. 所有 LLM methods 使用 GPT-4o-mini, 未测试更强或开源模型下结论是否保持。
14. KeplerAgent 对 LSR-Transform 的 NMSE 不如 LLM-SR, 说明它更偏 symbolic correctness, 数值拟合不是全面胜出。
15. 对 FEX 课题而言, 它发现的是 governing equations, 不是 PDE solution expressions；不能直接当 FEX PDE solver baseline。
16. 它不使用 RL controller 搜 expression trees；核心搜索后端是 PySR/GP 和 PySINDy/sparse regression。
17. 它没有处理 PDE boundary/initial condition residual, collocation sampling, 或 FEX 的 Adam+BFGS PDE loss。
18. 它没有分析表达式树搜索复杂度、采样复杂度或 controller 收敛。
19. 它证明了“物理结构工具 + LLM agent”有用, 但没有证明 LLM 产生的完整 skeleton 对 FEX-style RL search 必然有用。
20. 与 0616-fex 的关键交集在工具编排和结构先验粒度, 不是直接方法撞车。

### 跟我们的关系

1. 对 `0616-fex` 的直接撞车风险是中低：KeplerAgent 做 data-driven equation discovery, FEX 主线是 finite expression PDE solution / RL expression-tree search。
2. 它不解决 FEX 的核心理论 gap：RL controller 搜索复杂度、sample complexity、是否逃脱维度灾难。
3. 它也不覆盖 FEX 的核心工程 surface：PDE residual loss、collocation 点、boundary/initial condition、S_k 结构、Adam+BFGS 两阶段优化。
4. 它直接挤压的是 “LLM + SR agent” 这个大叙事空间, 尤其是如果我们的 FEX idea 只写成“LLM 帮忙选工具/选算子”。
5. 对 `fex-llm-skeleton` 最重要的启发：不要只说 LLM 生成 skeleton, 要把贡献改成“结构先验粒度相图：operator set vs template/skeleton vs exact expression vs physics tool constraints”。
6. KeplerAgent 的 expression_spec 正好是一个 template-prior 接口, 可以作为我们设计 FEX skeleton prior 的对照层级。
7. 它的 symmetry workflow 提示 FEX 可以先调用物理结构工具, 再把结果转成 controller prior 或 grammar constraints, 而不是只让 LLM 从文本直接吐表达式树。
8. 对 FEX active collocation idea, KeplerAgent 的 weakness 是没有诊断 reward/evaluation budget 怎么被浪费；我们可以强调“选择最能区分候选表达式的点”是它没碰的搜索评估层问题。
9. 对 FEX failure taxonomy idea, 这篇提供一个可复用 taxonomy seed：good structure but bad score、noisy derivative floor、hard stopping threshold mismatch、tool-call repetition。
10. 对 FEX search complexity idea, 这篇没有理论, 但说明 agentic tool orchestration 正在成为竞品叙事；理论工作应明确避开 agent engineering novelty。
11. 该引。凡是写 LLM+FEX、LLM skeleton、physics-aware symbolic regression, 这篇都应作为 2026 concurrent/near-concurrent work 引用。
12. 差异化写法：KeplerAgent 是“LLM selects and configures existing SR tools after physics diagnostics”；FEX 方向应写成“LLM/physics priors change the search distribution or grammar of an RL expression-tree PDE solver, then use FEX-specific residual/BFGS objective to solve PDEs”。
13. 近 6 个月 novelty check 还显示 `2603.19241` 在 constitutive-law 场景使用 LLM physics agents, `2605.17790` 反引文提出 self-reflective automatic equation discovery agent, 所以 agentic SR 叙事已经拥挤。
14. 因此 `0616-fex` 的安全主张应避免“第一个 LLM scientist for equation discovery”之类措辞；应聚焦 FEX-specific PDE solving、搜索分布、结构先验粒度和高维/PIDE setting。

## Read by: 0616-fex (2026-06-16)

这篇对 `0616-fex` 是必须引用但不直接撞车。它的主张是 LLM agent 先调用物理结构工具, 再配置 PySINDy/PySR；FEX 的主张是用有限表达式空间和 RL controller 搜 PDE 解或物理律。因此若我们的 idea 是 FEX 搜索复杂度、e-graph 剪枝、active collocation、synchrosqueezed prior, 它只提供背景和工具灵感, 不构成核心 prior art。

对 `fex-llm-skeleton` 则要严肃对照：KeplerAgent 已经展示了 template/expression_spec、symmetry tool、workspace/experience log 这类 agentic prior 注入方式。我们不能把 novelty 写成“LLM 帮 SR 选结构”本身；更好的差异化是 FEX-specific：把 LLM/physics prior 注入 RL expression-tree policy、算子 grammar 或 warm-start 分布, 并在 PDE residual/FEX benchmark 上做 operator-set、skeleton、exact-template、physics-tool constraint 的多粒度对照。

可用工具包括：experience log 设计、workspace file registry、symmetry generator round-to-physical-structure 的处理、PySR expression_spec 作为 skeleton interface 参考、noisy DiffEq long-term prediction 评测。特别值得吸收的是它的 honesty：case study 里 agent 后期重复调参暴露了“诊断工具不足”。这正好支持 FEX failure taxonomy 和 active collocation：我们的贡献可以是识别 FEX 搜索中结构正确但参数评分失败、reward 方差、collocation 低信息点等细分瓶颈。

最新相邻工作检索中, `2603.19241` 已经使用 LLM physics agents 做 constitutive-law symbolic regression, `2605.17790` 用 self-reflective agent 做 automatic equation discovery。结论是 agentic SR 叙事拥挤；`0616-fex` 后续写作必须避免泛化口号, 只 claim FEX/PDE/高维 PIDE/搜索分布/先验粒度这些具体差异。
