# Leverage, Influence, and the Jackknife in Clustered Regression Models: Reliable Inference Using summclust (2205.03288)

- **Year**: 2022.05（Stata Journal 出版版本晚于 arXiv 首版，正文 rev1-v20 已是接近定稿的修订）
- **Authors**: James G. MacKinnon (Queen's University), Morten Ørregaard Nielsen (Aarhus University), Matthew D. Webb (Carleton University) — 与 2406.00650（logit 推广）同一作者三人组
- **Venue**: Stata Journal（伴随公开 Stata 包 `summclust`，SSC + GitHub）
- **Keywords**: cluster-robust inference, leverage, partial leverage, influence, cluster jackknife, CRVE, effective number of clusters, high-leverage clusters, regression diagnostics
- **One-liner**: 把观测级的 leverage/partial-leverage/influence 诊断推广到 cluster 级，证明 partial leverage 的 scaled variance 能预测 cluster-robust 推断（含 bootstrap）何时不可信。

## 精华提炼

### 管线/方法

- 研究对象：线性回归模型 $y_g = X_g\beta + u_g$，$g=1,\ldots,G$ 个不相交 cluster，cluster $g$ 内 $N_g$ 个观测，总样本 $N=\sum_g N_g$，回归量维度 $k$。仅假设 cluster 间独立，cluster 内允许任意异方差/相关。
- 起点：标准 cluster-robust variance estimator CV1（sandwich 形式，式 eq:CV1）是业界默认（Stata 默认），但当 (a) 个别 cluster 远大于其余，或 (b) "处理"观测只集中在少数 cluster 时，CV1 + $t(G-1)$ 推断可能严重失真——这两类情形本质上都是"某个 cluster 有高杠杆/高影响力"。
- 核心新工具三件套：
  1. **Leverage** $L_g$：cluster 级 hat 矩阵 $H_g=X_g(X^\top X)^{-1}X_g^\top$ 的迹（核范数），衡量删除该 cluster 对**拟合值/残差**的潜在影响，与具体系数无关。
  2. **Partial leverage** $L_{gj}$：针对某个特定系数 $\beta_j$，先把回归量 $j$ 对其余回归量做残差化（partial out），再取 cluster $g$ 在残差化向量中的平方和占比，衡量删除该 cluster 对**该系数估计值**的潜在影响。
  3. **Influence**：真正计算 delete-one-cluster 系数估计 $\hat\beta^{(g)}$（式 eq:delone），比较其与全样本 $\hat\beta$ 的差异，衡量删除该 cluster 对结果的**实际**（非潜在）影响。
- 关键计算技巧（这是全文能"快速计算"的根本原因）：因为是 OLS，$\hat\beta^{(g)}$ 可以只用 cluster 级充分统计量 $X_g^\top X_g$、$X_g^\top y_g$ 通过一次 $k\times k$ 矩阵求逆算出（式 eq:delone），**不需要对每个 cluster 重新跑一次完整回归**——这是线性模型独有的代数捷径，非线性模型（含 CNN）不存在等价的闭式充分统计量分解。
- 副产品：由于两个 jackknife 方差估计量 CV$_3$、CV$_{3\rm J}$（式 eq:jack、eq:jack3）本质就是这些 $\hat\beta^{(g)}$ 的方差，计算 leverage/influence 时"顺便"就得到了比默认 CV1 更保守、通常更可靠的标准误——全部打包进 Stata 命令 `summclust`，同时提供 $G^*(\rho)$（有效 cluster 数）等汇总诊断量和默认输出的四联散点图。
- 验证方式两条腿：(1) 一个真实实证例子（72-market RCT，见下）证明"cluster 数不少、大小不算太不均匀"这类传统安全感知标准会被 partial leverage 的极端异质性证伪；(2) 一个大规模仿真（3000 组设计 × 每组 10000 次 replication），用广义可加模型把六种推断方法（CV1-t、CV3-t、WCR-C/S bootstrap 等）的经验拒绝率回归到 partial leverage 的 scaled variance 上，证明这一单一诊断量对拒绝率有近乎线性的预测力，且对 bootstrap 方法同样成立（程度较轻但确实存在）。

### 关键技术细节

- Leverage 公式：$L_g=\mathrm{Tr}(H_g)=\mathrm{Tr}(X_g^\top X_g(X^\top X)^{-1})$（式 eq:traceXX），$\sum_g L_g=k$，均值 $k/G$。经验法则：$L_h\approx 2k/G$ 大概率不算高，$L_h\approx 5k/G$ 大概率算高（作者明确承认这是"a matter of judgement"，非形式化阈值）。
- 退化情形 1（估计均值，单一常数回归量）：$L_g = N_g/N$ **精确成立**（式 eq:conlev）——leverage 退化为该 cluster 的样本量占比。
- 退化情形 2（单回归量+固定效应，组内去均值后）：$L_g \propto N_g\hat\sigma^2_{x,g}$（式 eq:felev）——leverage 正比于"cluster 大小 × 组内方差"。
- 退化情形 3（cluster 级处理哑变量）：$L_g \propto N_g/\bar d$（处理组）或 $N_g/(1-\bar d)$（对照组）（式 eq:treatlev2）——**哪个臂样本量占比小，哪个臂的 cluster 杠杆就系统性更高**。
- Partial leverage 公式：残差化 $\acute x_j = (I-X_{[j]}(X_{[j]}^\top X_{[j]})^{-1}X_{[j]}^\top)x_j$，$L_{gj}=\acute x_{gj}^\top\acute x_{gj}/\acute x_j^\top\acute x_j$（式 eq:partial），均值 $1/G$。仿真证明这是全文最具诊断力的单一指标。
- Influence：$\hat\beta^{(g)}=(X^\top X - X_g^\top X_g)^{-1}(X^\top y - X_g^\top y_g)$（式 eq:delone）；奇异子样本（如 cluster 固定效应或处理哑变量只在该 cluster 非零）用 generalized inverse 处理并显式标记警告，不静默吞掉。
- 两个 jackknife 方差估计量：CV$_{3\rm J}=\frac{G-1}{G}\sum_g(\hat\beta^{(g)}-\bar\beta)(\hat\beta^{(g)}-\bar\beta)^\top$（围绕 jackknife 均值，式 eq:jack）、CV$_3=\frac{G-1}{G}\sum_g(\hat\beta^{(g)}-\hat\beta)(\hat\beta^{(g)}-\hat\beta)^\top$（围绕全样本估计，式 eq:jack3）；两者渐近等价，CV$_3$ − CV$_{3\rm J}$ 半正定，实践中数值接近。
- 异质性汇总统计量：scaled variance / 变异系数平方 $V_s(a_\bullet)=\frac{1}{(G-1)\bar a^2}\sum_g(a_g-\bar a)^2$（式 eq:coefvar），可套用到 $N_g$、$L_g$、$L_{gj}$、$\hat\beta_j^{(g)}$ 任意一组量上；此外还提供调和/几何/二次均值与算术均值的比值作为互补的尺度无关异质性指标。
- 有效 cluster 数 $G_j^*(\rho)=G/(1+\Gamma_j(\rho))$（式 eq:gstar），$\rho$ 为组内相关参数；证明 $G_j^*(0)$ 与 $V_s(L_{\bullet j})$ 存在精确的单调递减关系（即两者数学等价，只是尺度不同）。
- 仿真设计：$G\in\{20,30\}$，cluster 大小由不均匀参数 $\gamma\sim U[2,4]$ 控制（$G=20$ 时最大/最小 cluster 比可达约 12–47 倍），5 个二值回归量，组内同质概率 $p_c\in\{0.25,...,0.60\}$；用带 5 自由度光滑样条的 GAM 把拒绝率回归到 $V_s(L_{gj})$ 及 $G_j^*(0)$ 上。
- 两阶段/两维 clustering 处理：无内生支持，退化为"分别调用 `summclust` 三次（各维度 + 交集）+ 用 eq:twoway 组合方差"的工程 workaround，作者明确承认"两维 jackknife 估计量的性质目前知之甚少"。

### 核心结果

- **实证例子（Busso & Galiani 2019 的 72-market 竞争实验数据）**：72 个 cluster，大小 20–55 相对均衡，传统直觉认为 CV1+$t(71)$ 应该表现良好。但 partial leverage 从 0.0001 到 0.0642，比值超过 500 倍（leverage 本身比值也有 198 倍）。结果：CV1 给出 $P=0.046$（勉强显著），CV2/CV3/CV3J/两种 WCR bootstrap 全部给出 $P>0.08$（不显著）——**同一数据集，仅因推断方法不同，显著性结论完全翻转**，且这一风险在运行任何替代推断方法之前，仅凭 partial leverage 分布就能被识别出来。该例子的有效 cluster 数 $G^*\approx34$，不到名义 72 个 cluster 的一半。
- **仿真主结果（Panel a, Fig 3）**：CV1-t 拒绝率随 $V_s(L_{gj})$ 近乎线性上升，从 $V_s\approx0$ 时的 0.055–0.058（轻微过拒绝）一路升到 0.10+（严重过拒绝）；CV3-t 在低 $V_s$ 时反而欠拒绝（0.033–0.039），曲线远比 CV1 平缓，但 $G=20$ 时 $V_s>9.6$ 后开始下降。
- **Bootstrap 并非免疫（Panel b）**：WCR-C/WCR-S 在低 $V_s$ 时表现很好，但除 WCR-S（$G=30$）外，其余 bootstrap-$t$ 组合在 $V_s$ 增大时同样出现明显的过拒绝或欠拒绝——**这是全文对本任务最关键的一条证据：partial leverage 异质性预测的不是"CV1 特有的失效"，而是一种更普遍的、bootstrap 类推断也会（程度较轻地）继承的失效模式**。

### 明确局限

- 全部公式严格依赖 OLS 的闭式 hat 矩阵 $H=X(X^\top X)^{-1}X^\top$；本文完全没有涉及非线性/非参数估计量的推广（该推广是同一作者组后续论文 2406.00650 才做的，且仅推广到 logit/GLM 这类仍有光滑闭式 score 的参数模型）。
- Estimand 必须是回归系数 $\beta_j$ 本身（或后续论文允许的 $\beta$ 的光滑函数），完全不覆盖 AUC/AUPRC 这类基于预测排序的阶梯统计量。
- 高/低杠杆的判定阈值（"2 倍" vs "5 倍"）是经验法则，非形式化功效/决策论推导。
- 两维聚类支持是工程 workaround，作者自己承认其 jackknife 方差估计量性质"知之甚少"。
- 仿真 DGP 是特定的二值回归量合成设计，作者未声称结果外推到任意 DGP，但"partial leverage 异质性预测推断不可靠"这一定性结论从代数推导本身就有相当普适性（不依赖该 DGP 的具体参数选择）。

### 跟我们的关系

**任务问题的直接回答：不能直接套用字面公式，但存在一条有代数依据、可预注册、且几乎零额外计算成本的复用路径。**

1. **为什么字面公式套不上**：$L_g$、$L_{gj}$ 是 OLS hat 矩阵 $H=X(X^\top X)^{-1}X^\top$ 的解析函数，本质要求"参数 $\to$ 拟合值"是一个线性投影。已训练冻结的 CNN 把 4D 张量映射到概率没有这种闭式投影结构，不存在 hat 矩阵；而本 proposal 的 estimand 是 storm-cluster bootstrap 下的 paired $\Delta$AUPRC——一个基于预测排序的阶梯统计量，既不是回归系数，也不是回归系数的光滑函数（连 2406.00650 把方法推广到 logit 时都明确止步于"$\beta$ 的光滑函数"这一边界，AUPRC 不满足）。这一点在读 2406.00650（同组作者把方法推广到 logit MLE）时已被独立验证过一次，本文进一步确认了该边界的根源：不是"AUPRC 太复杂"，而是"AUPRC 缺少 MNW 全部代数依赖的那个线性/光滑参数结构"。

2. **关键澄清：MNW 这条方法线真正解决的计算问题，在本任务里根本不存在**。MNW 之所以要发明闭式 delete-one-cluster 公式（本文）以及后续的一阶线性化（2406.00650 的 CV3L），是因为对 OLS/logit 而言，"删掉一个 cluster 重新估计"意味着重新做一次完整估计（logit case 实测占该论文仿真约 70% 总计算时间）。而本 proposal 的 raw-only / raw+(D,J) 两个 CNN 在评估阶段已训练完毕并冻结，"删掉一个 storm 重算 $\Delta$AUPRC"只需过滤已算好的 held-out 预测数组后重排序（$O(n\log n)$），**从起点就已经比 MNW 想要避免的暴力重估算便宜，且比 2406.00650 的线性化版本还便宜**——因此 MNW 全部"如何算得快"的核心卖点对我们不构成任何门槛，也不需要被移植；真正能移植的是"如何用这些廉价的 delete-one-cluster 估计做诊断"这套**方法论**，而非算法本身。

3. **可直接落地的复用方案（回应任务的具体问题：识别个别 storm 是否有异常大杠杆/影响力）**：
   - 对 held-out 的每个 storm $s$，计算 leave-one-storm-out 点估计 $\Delta\mathrm{AUPRC}^{(-s)}$（从池化的 held-out 预测中剔除 storm $s$ 后重算）——这是 $\hat\beta^{(g)}$ 在评估侧（而非估计侧）的直接结构类比，计算成本可忽略。
   - 用这 $G$ 个值构造 MNW 风格的**影响力**诊断：(i) 排序 $|\Delta\mathrm{AUPRC}^{(-s)}-\Delta\mathrm{AUPRC}|$，标记显著偏离其余 storm 的极端点（直接对应本文"若 $\hat\beta^{(h)}$ 与 $\hat\beta$ 差异悬殊则 cluster $h$ 具有高影响力"的判据）；(ii) 直接套用 eq:coefvar 的 scaled variance/变异系数公式，把这组 $\Delta\mathrm{AUPRC}^{(-s)}$ 当作本文的通用量 $a_g$ 代入，得到一个可预注册阈值的单一汇总数；(iii) 顺带套用 CV$_{3\rm J}$ 公式（$\frac{G-1}{G}\sum_s(\Delta\mathrm{AUPRC}^{(-s)}-\overline{\Delta\mathrm{AUPRC}})^2$）得到一个几乎零成本的 jackknife 方差，作为独立于 storm-cluster bootstrap 之外的交叉校验——若 bootstrap SE 明显小于该 jackknife SE，是 resampling 方案本身可能未充分捕捉 cluster 结构的预警信号，直接回应 review 对"bootstrap 结果可信度前置检查"的诉求。
   - 真正意义上的**杠杆**（与结果无关，仅依赖设计/协变量）比影响力更难类比，因为 CNN 没有 hat 矩阵；但本文自身给出两个退化解析情形可作为诚实、有代数依据（而非纯类比）的**杠杆代理**，且都只需 storm 花名册数据、**可在训练任何模型之前**计算，字面满足任务"真正跑 P2 真实数据实验前"的要求：(a) 均值估计退化情形（eq:conlev, $L_g=N_g/N$）——池化 AUPRC 本质更接近"按 storm 加权聚合的排序统计量"而非通用回归，故 storm $s$ 在 held-out 正例总数（或样本总数）中的占比是一个字面有依据的朴素杠杆代理；(b) cluster 级处理哑变量退化情形（eq:treatlev2, $L_g\propto N_g/\bar d$ 或 $N_g/(1-\bar d)$）——SPC 正例本就稀疏，直接类比"处理组是稀有臂"：正例占比异常偏离整体 base rate 的 storm，是该逻辑下的高杠杆候选点，与本 proposal 已知的类别不平衡风险直接对应。
   - 需在 proposal 中诚实标注：以上两条是**有代数依据的朴素代理**，不是 CNN 的真实杠杆（后者因无 hat 矩阵而不存在闭式定义），避免读者误以为是同一概念的直接搬运。

4. **一个更窄但 100% 字面适用的场景**：如果 PCMCI+/LKIF 因果特征提取或其某个验证步骤内部涉及任何线性/GLM 回归（例如用一个简单线性模型检验 $(D,J)$ 判别力是否被个别 storm 主导），`summclust` 的字面公式可以不加修改地直接部署在那一步——这是比 CNN 级 $\Delta$AUPRC 诊断更小、但完全无需类比推理的一个廉价附加检查点。

5. **为什么这条诊断确实值得做（不是画蛇添足）**：Section 6 仿真明确证明 partial leverage 的 scaled variance 不仅预测 CV1-t 的失效，也（程度较轻但确实）预测 wild cluster bootstrap 的失效——这是一个可引用、已被量化的先例，直接证实 proposal reviewer 缺口 (2) 的担忧对 storm-cluster **bootstrap** 而非仅对 t 检验同样成立，不是无的放矢的额外複杂度。

6. **撞车检查**：不撞车。Proposal 的 non-goals 已明确不主张因果估计器/融合机制/新骨干的新颖性，引入 MNW 风格的杠杆/影响力预检查属于"协议卫生"范畴，与 proposal Contribution Focus 里已有的"审计协议模板"表述完全同类，不产生需要单独辩护的新贡献叙事。

7. **下一轮搜索方向**：(a) arXiv:2604.02000（MacKinnon 2026, *When Can We Trust Cluster-Robust Inference?*）标题直接对准"何时可信"，很可能把 leverage/influence/$G^*$ 整合成一个可操作的信任判据，是最值得精读的直接后续；(b) 仍未找到 ML/统计文献里专门针对**分组测试集上 AUC/AUPRC 类排序指标**的 jackknife/influence 诊断（econometrics 一侧的 cluster-robust 推断文献已经充分检索，需要切换到 ML 评测可靠性/DeLong 检验聚类推广方向）。

本轮任务：为 `causal-scs-indicator` proposal v1（REVISE 判定）的 Validation Focus 缺口 (2) 做针对性精读——判断 MacKinnon/Nielsen/Webb 的 cluster-level leverage/partial-leverage/influence 诊断工具（summclust）能否直接套用为 storm-cluster bootstrap 的预注册诊断工具，用于在真正投入 P2 真实数据预算前，识别是否有个别 storm 对 raw-only vs raw+(D,J) 的 $\Delta$AUPRC 估计有异常大的杠杆/影响力。
