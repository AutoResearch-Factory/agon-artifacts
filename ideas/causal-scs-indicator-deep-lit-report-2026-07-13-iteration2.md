## 三个子问题（用户原始委托）

1. 在类似小样本（几十到上百个 storm systems）遥感/天气时空数据场景下，自监督预训练后应该端到端微调整个 encoder，还是保持冻结只训练线性/浅层 probe？现有文献的经验证据和理论指导是什么？
2. codex 建议的 masked temporal prediction（预测未来帧而非纯粹的空间/时空块重建）作为预训练 pretext task，是否有实证支持优于当前设计的 masked reconstruction，特别是在下游任务样本量很小的情况下？
3. 是否有关于因果/物理衍生特征与微调后（而非冻结）预训练模型融合的先例？

## 三个子问题的结论（供 proposal-refiner 直接采纳）

### 子问题 1：frozen probe vs. 端到端微调

**没有文献支持"默认冻结"是合理选择。** 证据链自洽收敛于同一方向：

- **理论基石**（2202.10054, LP-FT）：全量微调会扭曲预训练特征，纯冻结探测则留下可改进空间；两者之间存在一个几乎零额外成本、有严格理论保证的更优策略——**先线性探测再全量微调（LP-FT）**，同时改善 ID 和 OOD 精度。
- **两个独立领域交叉验证**"全量微调破坏任务无关特征"这一机制（2301.12715 NLP OOD 检测；隐含于 2202.10054 的 10 个 CV 基准），说明该现象不是 CV 专属，但也说明**解法（LP-FT）同样跨领域适用**，不是"因此必须冻结"的理由。
- **v2 proposal 自己援引的灵感来源 Prithvi WxC 原文（2409.13598）从未使用"冻结+纯线性 probe"设计**：其 3 个下游任务的适配配方都是"冻结核心 + 多层新卷积块（含残差融合）"，比 v2 当前 `Linear(1088,1)` 复杂得多；论文自陈"仅微调新头效果不佳，需要针对性架构改造"。这是本轮最锋利的一条证据——**proposal 当前设计比它自己引用为灵感来源的论文更保守，而那篇论文自己都承认这种保守做法效果不佳**。
- **WeatherPEFT/SFAS（2509.22020，已有 wiki）提供第三条路**：Fisher 信息引导的选择性微调，backbone-agnostic，在 Aurora/Prithvi-WxC 上以 0.1%-4% 参数量追平/反超全量微调。**但其 Fisher 估计的样本稳定性仅在数千级样本验证过，`<=50-100 storm` 场景是全新的、未经验证的regime**，需要在 Gate 0 阶段先做小规模稳定性检查，不能直接当作免费午餐采纳。
- **反向警示**（2409.00489, 2409.18536）：Prithvi 只预训练了 backbone、head 从零开始，最终被"全流程预训练+同等力度调参"的对照模型反超；EO 领域系统性发现"预训练默认更强"这一假设在非同构下游任务上经常不成立，除非对"从头训练"一侧也投入同等力度的超参搜索——这对 v2 proposal 自己的 raw-only 对照臂设计也是一个独立于 codex 发现的额外提醒。

**推荐路径（按工程成本排序）**：
1. LP-FT：保留当前"预训练后训练 probe"作为初始化阶段，随后解冻整个 encoder，在 outer-training storms 上联合微调 encoder+probe（BCE）。零新增结构，直接对应 codex 建议的"SSL 初始化后接监督微调"。
2. 若担心 `<=50-100 storm` 全量微调的过拟合风险，可评估 SFAS 式 Fisher 引导选择性微调作为 LP-FT 的替代或补充，但必须先做小样本 Fisher 稳定性验证（本轮未发现任何论文在此样本量级验证过该机制）。

### 子问题 2：masked temporal prediction vs. masked reconstruction pretext

**没有找到任何论文（含 Prithvi WxC 自身）对"重建 vs 预测未来"做受控消融** —— 这是一个真实的文献空白，而非检索遗漏：Prithvi WxC 论文明确自陈"no ablations isolating individual design choices"（掩码策略、气候态相对目标、Hiera/MaxViT 架构改动三者同时变化，无法归因收益来源）。

支持 codex 建议方向的是**间接但一致的证据**，而非直接受控实验：
- SatVision-TOA（2411.17000）的成功高度依赖"下游任务本身不需要时序推理"这一前提——它是单帧 2D 云检索任务。而本课题下游信号的本质就是 96h 时序演化（这正是 (D,J) 存在的理由），纯空间/局部时空块重建给不了 encoder 学习真实前向时间动力学的强制信号（重建一个被遮盖的块，往往靠邻近时空上下文插值即可解出，不需要学会预测）。
- MAM4WF（2409.20117）证明"预测未来帧"范式在 SEVIR（强对流邻近数据集）上确实有效，但它是全监督判别式预测器，不是"预训练+小样本下游微调"框架，因此不能直接搬来回答"小样本微调时哪种 pretext 更好"这个具体问题。
- MAESTRO（2508.10894）给出时间轴处理的可执行设计依据（早期 token 级时间融合优于晚期融合），可用于重新设计 masking 方案本身。

**建议表述（避免过度声称）**：proposal 下一版若采纳 masked-temporal-prediction 或 reconstruction+forecasting 混合目标，应将其定位为"任务对齐的合理设计选择，有间接类比证据支持"，而不是"已被文献证明优于 reconstruction-only"——后者目前找不到直接支撑，即使是 codex 引用的 Prithvi WxC 范式本身也没有做过这个消融。

### 子问题 3：因果/物理衍生特征 + 微调后（非冻结）预训练模型融合先例

**两轮定向搜索 + 一轮跨领域广谱搜索（Granger 特征融合、因果图嵌入辅助输入、transfer entropy 特征融合、handcrafted+deep 混合融合四种独立措辞）均未发现直接先例。** 这是三个子问题中证据最薄弱的一个，但空白本身是本轮认真检索后的真实结论，而非放弃检索的借口——它反而支持而非削弱 proposal 现有 Novelty and Elegance Argument 段落"该两阶段设计目前无人做过"的论断，把证据基础从"仅搜到 TabPFN-CFM 一篇最接近但不构成先例"扩展到"跨四种措辞、两个搜索轮次均未命中"。

唯一可迁移的是**架构模式而非概念先例**：Prithvi-CAFE（2601.02315）的"轻量 adapter 微调的基础模型分支 + 处理额外/互补信息的并行 CNN 分支 + 逐尺度注意力门控融合"模式，可直接用于升级 v10 pilot 现有 raw 分支与 `(D,J)` 分支之间的 naive concat/mean 融合，与选择哪种 frozen/fine-tune 策略正交，可独立采纳。

Prithvi-EO-2.0 的 GPP 发现（全流程微调表征模型反超工程特征 baseline，R² 达 +20%）是一条诚实的反向提醒：不能预设因果衍生特征一定比训练充分的表征模型多带来信息，这恰好支持 proposal 现有"诚实证伪、不预设正结果"的判据设计（POSITIVE/NULL/NEGATIVE/HARM 穷尽分类），不需要改动。
